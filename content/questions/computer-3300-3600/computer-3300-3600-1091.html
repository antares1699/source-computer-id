OP meminta penciptaan waktu, tapi Ian Yang dan boehj jawaban didasarkan pada perintah touch pengaturan modifikasi , mengakses dan perubahan waktu ke waktu saat ini.

\n\n

Seperti yang dikatakan boehj (dalam komentarnya) tidak ada penciptaan waktu file ... Ini berlaku untuk sebagian besar sistem file, tetapi pada beberapa sistem file modern (seperti ext4 ) itu penciptaan (atau kelahiran ) waktu file disimpan dalam inode .

\n\n

Oleh karena itu kami dapat memberikan jawaban yang lebih benar dengan cermat sesuai dengan judul pertanyaan: 

cp -a  your-folder tmp-folder\nrm -rf your-folder\nmv tmp-folder your-folder\n

Ini akan mengubah file penciptaan waktu ( kelahiran waktu) ke waktu saat ini. Ini juga memperbarui perubahan waktu tetapi tetap tidak berubah modifikasi dan mengakses waktu.

\n\n

Sebagai informasi, perubahan waktu kadang-kadang disingkat ctime dan dapat disalahartikan sebagai penciptaan waktu (crtime atau btime). Faktanya, file perubahan waktu sebuah file adalah modifikasi waktu yang sesuai inode . Oleh karena itu, perintah yang mengubah konten inode sebagai chmod atau chattr juga menyetel perubahan waktu ke waktu saat ini.

Arahkan ke folder yang dimaksud, katakanlah, ~ / Documents / myfiles. 

$ cd ~/Documents/myfiles\n

Kemudian lakukan: 

$ touch *\n

Ini akan mengubah waktu modifikasi setiap kali Anda menjalankan perintah itu.

\n\n

Tentunya Anda dapat membuatnya lebih spesifik tergantung pada kasus penggunaan Anda, mis. 

$ touch *.doc\n

hanya akan mengubah waktu modifikasi untuk file dengan string '.doc' dalam namanya.
