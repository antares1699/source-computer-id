Biar kutebak; Anda menggunakan Chrome, bukan?

\n\n

Ini bukan virus (perhatikan bagaimana semuanya adalah "domain" 10 huruf), ini adalah server DNS pengujian Chrome untuk mendeteksi pengalihan kegagalan.

\n\n

Apa yang terjadi adalah beberapa ISP telah mengubah server DNS mereka sehingga ketika Anda mencoba menavigasi ke URL yang tidak ada, alih-alih memberi Anda halaman kesalahan sederhana, sebaliknya ISP memberi Anda halaman web sebenarnya yang memiliki iklan, hasil penelusuran, dan sampah lainnya — mungkin terkait dengan URL / istilah penelusuran yang Anda coba buka. (Chrome sendiri menawarkan ini sebagai opsi.)

\n\n

Chrome menggabungkan bilah penelusuran dan bilah alamat menjadi satu ke dalam Omnibar dan juga menyembunyikan protokol secara default (Anda tidak perlu memasukkan http:// secara khusus; diasumsikan). Selain itu, URL tidak memiliki memiliki TLD, bisa seperti http://svn/. Oleh karena itu, saat Anda mengetik kata ke Omnibar, tidak jelas apakah Anda mencoba memasukkan URL atau istilah penelusuran. Karena itu, Chrome memerlukan cara untuk mendeteksi apakah Anda sedang melakukan penelusuran atau mencoba menavigasi ke suatu situs.

\n\n

Jadi untuk menghindari pengguna yang selalu melihat kesalahan ISP / laman iklan ketika mereka mengetik istilah seperti itu di Omnibar, yang dilakukan Chrome adalah mencoba menyambungkan ke beberapa URL acak yang tidak masuk akal. Jika semuanya (secara ajaib) menyelesaikan, dan ke IP yang sama pada saat itu, maka Chrome tahu bahwa server DNS mengarahkan ke halaman kesalahan, sehingga Chrome dapat bereaksi sesuai (misalnya, memperlakukan istilah sebagai permintaan pencarian, meminta Anda untuk alternatif, istilah terkait; dll.)

\n\n

Plus, ini juga membantu untuk menghindari pembajakan DNS karena dalam skenario itu, sebagian besar jika tidak semua (tergantung pada implementasi pembajakan) permintaan DNS biasanya akan diselesaikan ke IP yang sama untuk memata-matai sebelum diteruskan (jika ada) ke IP asli .

\n\n
