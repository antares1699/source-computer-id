Saya sedang menyinkronkan banyak file antara sistem Windows dan Linux. Karena tidak semua Windows peduli dengan casing, beberapa file di sistem Windows tidak lagi memiliki casing yang sama seperti yang ada di sistem Linux. Tetapi rsync sekarang memperlakukan file-file ini sebagai berbeda dan mengupload salinan baru.

\n\n

Apakah mungkin rsync mengabaikan kasus ini?

Biasanya, Windows mengabaikan kasus, tapi diawetkan saya t. Jika nama file Anda mengubah kapitalisasi, itu pasti karena beberapa program selain rsync yang menyalin file dan mengacaukan kasus selama penyalinan. Program itu mungkin salah konfigurasi. Atau jika Anda menggunakan server Samba, mungkin itu salah konfigurasi. Secara default, Samba juga mempertahankan kasus nama file.

\n\n

Jadi, mungkin Anda dapat menyelesaikan masalah nama file yang tiba-tiba mengubah kasusnya.

\n\n

Kemudian Anda ditinggalkan dengan " hanya "masalah sensitivitas huruf pada filter. Jika itu menjadi kasus Anda, dan Anda enggan menerapkan tambalan yang disarankan oleh Wim, Anda dapat menggunakan kelas karakter dalam filter.

\n\n

Itu jelek dan menjengkelkan, tapi berhasil: 

--exclude=\"[Tt][Ee][Mm][Pp]/\"\n

atau dalam file filter: 

- Temp/\n- temp/\n- TEMP/\n

atau yang lebih umum tetapi tidak dapat dibaca: 

- [Tt][Ee][Mm][Pp]/ \n

Jika Anda ingin mengotomatiskan konversi file filter ke sintaks ini, Anda dapat menggunakan perintah perl ini: 

perl -i.bak -pe 's/([a-z])/[\\U$1\\E$1]/g' your_rsync-filter.txt\n

Dan ubah kembali ke bentuk yang dapat dibaca manusia dengan 

perl -i.bak  -pe 's/ \\[ [A-Z] ([a-z]) \\] /$1/xg' your_rsync-filter.txt\n

(Opsi -i.bak membuat cadangan dan mengonversi file di tempat alih-alih ke stdout)

\n\n

Setidaknya untuk filter dan opsi sertakan / kecualikan, akan sangat menyenangkan jika memiliki tanda peka huruf besar / kecil di rsync. Sampai saat itu, satu-satunya pilihan tampaknya adalah tambalan atau sintaks mirip regex yang berbelit-belit.

Anda dapat mencoba menggunakan sistem pencocokan fuzzy rsync:

\n
\n

-y, --fuzzy

\n

Opsi ini memberi tahu rsync bahwa ia harus mencari file basis untuk file tujuan apa pun yang hilang. Algoritme saat ini mencari di direktori yang sama dengan file tujuan untuk file yang memiliki ukuran dan waktu modifikasi yang identik, atau file dengan nama yang mirip. Jika ditemukan, rsync menggunakan file basis fuzzy untuk mencoba mempercepat transfer.

\n

Perhatikan bahwa penggunaan file --menghapus opsi mungkin menyingkirkan file kecocokan fuzzy potensial, jadi gunakanlah --delete-after atau tentukan beberapa pengecualian nama file jika Anda perlu mencegahnya.

Rsync memiliki apa yang mereka sebut "patch yang dipelihara" yang menambahkan sakelar --ignore-case. Anda perlu mengkompilasi rsync dari sumber untuk ini, dan terapkan ignore-case.patch. Anda memerlukan versi rsync yang telah ditambal pada kedua lokal sebagai sisi jarak jauh.
