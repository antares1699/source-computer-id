Saat ini saya memiliki sistem Windows XP yang melaporkan penggunaan Komit sistem kurang dari penggunaan Memori Fisik. Pemahaman saya adalah bahwa System Commit adalah jumlah total pagefile ditambah penggunaan RAM. Seperti yang diutarakan oleh Wikipedia:

\n

Jumlah biaya komit tidak termasuk semua memori fisik; itu hanya menghitung semua memori fisik yang dapat dipindahkan ke disk.

\n\n

Dari Menekan Batas Windows - Memori Virtual (Saya akan merekomendasikan membaca seluruh artikel)

\n\n
\n

Seperti yang Anda harapkan dari deskripsi jaminan commit, file\n batas komit adalah jumlah memori fisik dan ukuran paging\n file. Pada kenyataannya, tidak semua memori fisik diperhitungkan dalam\n batas komit karena sistem operasi mencadangkan bagian fisik\n memori untuk digunakan sendiri.

\n
\n\n

dan

\n\n
\n

Tidak semua memori virtual yang dialokasikan oleh suatu proses dihitung ke dalam file\n batas komit. Seperti yang Anda lihat, memori virtual yang dipesan tidak. Virtual\n memori yang mewakili file pada disk, disebut tampilan pemetaan file,\n juga tidak dihitung dalam batas tersebut kecuali jika diminta oleh aplikasi\n copy-on-write semantik, karena Windows dapat membuang data apa pun\n terkait dengan tampilan dari memori fisik dan kemudian mengambilnya\n dari file. Memori virtual di ruang alamat Testlimit di mana\n gambar DLL yang dapat dijalankan dan sistemnya dipetakan karena itu tidak dihitung\n menuju batas komit. Ada dua jenis memori virtual proses\n yang dihitung terhadap batas komit: pribadi dan didukung oleh berkas halaman.

\n
\n\n

Jadi beberapa penggunaan memori OS tidak dihitung untuk batas itu serta file yang dipetakan memori. Di Windows semua file EXE dan DLL dimuat sebagai file yang dipetakan memori. Mereka dimuat ke dalam memori fisik, tetapi karena mereka tidak dimodifikasi setelah memuat (biasanya) Windows tidak mendukung mereka dengan file halaman, karena ia tahu bahwa itu hanya dapat membacanya kembali dari file disk mereka - itulah sebabnya mereka tidak tidak dihitung terhadap batas komit.
