Anda harus memeriksa hard drive Anda dengan chkdsk 

Win + R\nType: cmd (and hit enter)\nType: chkdsk /F /R /X c: (and hit enter, replace the c in \"c:\" by the letter of your suspected drive)\n

Jika perlu dijalankan saat reboot, ia akan bertanya ... Dan jika Anda ingin memeriksa hasil setelah reboot 

Win + R\nType: eventvwr (and hit enter)\n

Di windows journal "Applications", cari entri terbaru untuk Winlogon dan buka. Ini akan memiliki informasi tentang apa yang ditemukan di ChkDsk. (Jika Anda mencari hasil pemindaian yang dijalankan pada waktu selain boot, Anda harus menemukannya di tempat yang sama, tetapi terdaftar di bawah ChkDsk)

\n\n

Anda juga bisa menggunakan alat seperti HD Tune : PINTAR. info & scan disk

Alat-alat ini hanya menggunakan api defrag windows yang tidak berubah di Windows 7.

\n\n

Alasan mereka memproses mungkin bertahan selama beberapa waktu setelah mereka keluar adalah karena operasi disk yang besar membutuhkan waktu.

\n\n

Aplikasi defrag menganalisis disk dan kemudian pada dasarnya meminta windows untuk "memindahkan file ini dari tempat ini ke tempat itu di disk". Operasi tersebut dapat memakan waktu lama, terutama jika itu adalah file besar, atau disk Anda hampir penuh, atau lambat.

\n\n

Aplikasi kemungkinan besar belum "macet", mereka hanya menunggu windows menyelesaikan operasi terakhir yang mereka buat. Dugaan saya adalah bahwa Anda memiliki beberapa file yang sangat besar (> 2GB) dan operasi defrag hanya membutuhkan waktu untuk menyelesaikan relokasi mereka. Mereka pada akhirnya akan selesai, dan prosesnya akan ditutup dengan sendirinya jika Anda mengizinkannya.

\n\n

Jika Anda memiliki file 40 GB dan defragger meminta jendela untuk memindahkannya, diperlukan waktu> 10 menit untuk menyelesaikan tugas itu dengan mudah. Anda harus menunggu selama itu, sebelum mempertimbangkan prosesnya digantung. Jika disk Anda memiliki kinerja tulis yang lambat (seperti SSD atau drive laptop 4500 rpm yang lebih lama), operasi ini akan memakan waktu lebih lama.

\n\n

Jika disk Anda baru dan cepat dan dapat menulis pada 100 MB / dtk, Anda mungkin melakukan perhitungan dan berpikir bahwa hanya perlu 200 detik untuk memindahkan file 20 GB, tetapi ketika Anda mempertimbangkan pembacaan disk yang sama dengan, waktu pencarian , dan pembaruan terus-menerus dari file log NTFS, sebenarnya akan memakan waktu lebih lama.

\n\n

Saya menggunakan "My Defrag" sepanjang waktu di Windows 7 tanpa masalah.
