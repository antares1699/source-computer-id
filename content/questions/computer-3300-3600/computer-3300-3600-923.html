Saya akan memilih MS Access karena jika pengguna harus mengisi lembar, itu menjadi rentan terhadap kesalahan. MS Access adalah database dan dapat menghitung bidang dan melindungi bidang berdasarkan entri sebelumnya. Membuat sheet lebih canggih, tetapi jika dilakukan dengan benar, akan jauh lebih mudah untuk diisi. Selain itu, lebih mudah untuk menghasilkan laporan yang tepat yang hanya menampilkan informasi yang ingin Anda tampilkan, bukan semuanya. Ini juga dimungkinkan dengan Excel, tetapi jauh lebih sulit untuk dilakukan.

\n\n

Jika tidak banyak penghitungan yang harus dilakukan, dan aplikasi tidak harus seluas itu, Excel mungkin akan melakukannya. Tapi Excel dimaksudkan sebagai lembar tempat Anda bekerja, bukan sebagai database, di mana Access dimaksudkan sebagai database dengan metode berbeda untuk mengisi informasi dan mengekstrak informasi lainnya.

\n\n

Jika Anda adalah satu-satunya orang yang bekerja di sheet sepanjang waktu, Excel mungkin merupakan pilihan yang lebih baik jika data, penghitungan, dan laporan terbatas.

Suatu tahun, sebagai guru sains di sekolah swasta kecil, saya harus mengerjakan rapor untuk 300+ siswa karena sistem komputer penilaian yang ada tidak lagi dapat diakses (semacam krisis kecil). Dengan tidak adanya anggaran untuk perangkat lunak penilaian yang lebih aktual, saya dibiarkan menggunakan Excel dan / atau Access untuk membuat kartu laporan. Saya memutuskan untuk menggunakan Excel dan Access. Inilah alasannya:

\n\n

1) Excel sudah digunakan untuk menyimpan nilai, dan secara keseluruhan jauh lebih sederhana untuk membuat tabel. Akses sangat sulit untuk dikerjakan dalam hal pengembangan tabel.

\n\n

2) Kami harus dapat menggabungkan beberapa catatan dan membuat laporan. Access dirancang untuk membuat laporan dari beberapa tabel dengan variabel tertaut.

\n\n

3) Access dapat melakukan tautan dinamis ke beberapa spreadsheet Excel, yang memperbarui database setiap kali file Excel berubah. Ini berarti kami tidak perlu memuat lebih dari satu kali.

\n\n

Semua dalam semua itu berjalan cukup baik, meskipun saya akan mengatakan bahwa sistem pasti di fritz dan crash beberapa kali dengan begitu banyak tabel yang ditautkan (ini dengan Office 2003). Bagian yang menyenangkan dari bekerja dengan perangkat lunak basis data adalah bahwa datanya terkunci dan bahkan kerusakan perangkat lunak tidak mematikan file basis data Anda. Menurut saya Excel sekarang memiliki lebih banyak kemampuan ... jadi Anda mungkin tidak memerlukan Access sama sekali, meskipun saya belum pernah membuat laporan dengan Excel.
