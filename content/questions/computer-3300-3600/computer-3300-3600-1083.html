Saya punya masalah:

\n\n

Saya tidak memiliki kata sandi root di mac saya. Saya hanya memiliki akun Admin, yang dapat melakukan banyak hal menggunakan sudo. Sekarang, saya ingin menambahkan pengguna biasa saya ke file /etc/sudoers juga. Karena tidak mengizinkan saya menulis ke file itu (bahkan menulis menggunakan sudo), saya melakukan ini: 

sudo chmod u+w /etc/sudoers\n

Itu berhasil. Tetapi sejak itu saya tidak dapat melakukan perintah sudo lagi di sistem saya. Ia mengeluh bahwa /etc/sudoers memiliki mode yang salah: 

$ sudo touch /etc/sudoers \nsudo: /etc/sudoers is mode 0740, should be 0440\nSegmentation fault\n

Kedengarannya seperti lelucon yang buruk, karena sekarang saya bahkan tidak dapat mengubah mode kembali ke 0440: 

$ sudo chmod 0440 /etc/sudoers \nsudo: /etc/sudoers is mode 0740, should be 0440\nSegmentation fault\n

Adakah cara untuk memperbaiki situasi ini? Saya perlu mendapatkan kemampuan sudo saya kembali.

Ini mungkin salah satu kasus yang jarang terjadi di mana menjalankan "Perbaiki Izin Disk" dari Utilitas Disk mungkin benar-benar dijamin.

\n\n

Jika tidak berhasil, ikuti saran @ slhck untuk mem-boot ke mode pengguna tunggal untuk memperbaikinya:

\n\n
    \n

  1. Nyalakan ulang, tahan Command-S untuk masuk ke mode pengguna tunggal. Perhatikan bahwa jika Anda menggunakan keyboard Bluetooth, Anda harus menekan ini seperti bunyi lonceng boot tertinggal . Jika Anda menekannya terlalu cepat, itu sebelum radio Bluetooth diatur ulang dan keyboard kehilangan koneksi dan tidak mengkomunikasikan penekanan tombol. Jika Anda terlambat melakukannya, proses boot akan berpindah ke boot multi-pengguna.

    2. \n

  2. Ikuti petunjuk di layar untuk memasang kembali drive root Anda, baca-tulis. Anda mungkin dapat melewati langkah fsck jika Anda melakukan boot ulang dengan anggun. Jadi Anda mungkin bisa mengetik:

    \n\n

    mount -uw /

    3. \n

  3. Perbaiki izin pada /etc/sudoers. Perhatikan bahwa Anda sudah root saat Anda dalam mode pengguna tunggal, jadi Anda tidak perlu sudo ini.

    \n\n

    chmod 0440 /etc/sudoers

    4. \n

  4. Keluar dari mode pengguna tunggal dengan mem-boot ulang lagi, cukup dengan mengetik:

    \n\n

    reboot

    5. \n

Pendekatan lain menggunakan Automator.app

\n\n

Buat alur kerja baru yang berisi sesuatu seperti berikut ini: 

do shell script \"chmod 0440 /etc/sudoers\" with administrator privileges\n

Ketika Anda menjalankannya, itu akan meminta kata sandi Anda dan (dengan asumsi Anda adalah administrator) itu akan memperbaikinya.

\n\n

Ini juga berfungsi jika Anda mengacaukan perizinan pada direktori induk (misalnya membuat / sesuatu selain 755).
