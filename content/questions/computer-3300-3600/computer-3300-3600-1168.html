Ini adalah contoh skrip FTP untuk mentransfer satu file: (Perhatikan, Anda dapat menggunakan FQDN alih-alih IP) 

#!/bin/bash\n\n# $1 is the file name for the you want to tranfer\n# usage: this_script  <filename>\nIP_address=\"xx.xxx.xx.xx\"\nusername=\"remote_ftp_username\"\ndomain = sample.domain.ftp\npassword= password\n\nftp -n > ftp_$$.log <<EOF\n verbose\n open $IP_address\n USER $username $password\n put $1\n bye\nEOF\n

Tambahkan > ftp_$$.log hanya jika Anda perlu logging.\nKemudian Anda dapat menggunakan 

crontab -e\n

perintah untuk mengedit tabel cronjob dan menambahkan skrip Anda.

\n\n

Ini adalah Contoh:

\n\n

Jika Anda suka memiliki skrip di atas, (asumsikan Anda memilikinya di rumah dan namanya myscript.sh) /home/myscript.sh, jalankan setiap hari pada jam 2 pagi, Anda harus melakukan: 

# crontab -e\n

dan kemudian Anda harus menambahkan entri berikut: 

0 2 * * * /home/myscript.sh\n

Sebagai referensi, di sini Anda memiliki parameter entri crontab yang artinya: 

* * * * * command to be executed\n- - - - -\n| | | | |\n| | | | ----- Day of week (0 - 7) (Sunday=0 or 7)\n| | | ------- Month (1 - 12)\n| | --------- Day of month (1 - 31)\n| ----------- Hour (0 - 23)\n------------- Minute (0 - 59)\n

Tutorial ini juga bisa membantu Anda.

manusia crontab akan menunjukkan kepada Anda apa yang Anda butuhkan. Anda akan menginginkan sesuatu seperti: 

0 0 * * *  yourScript.sh\n

di file crontab Anda. Perhatikan bahwa skrip di bawah cron berjalan dengan lingkungan cut-down, jadi Anda harus menentukan pengaturan env yang diperlukan skrip dalam skrip itu.
