Saya ingin mengizinkan rekan kerja Windows saya di domain kami mengakses beberapa file publik di Mac saya tanpa memasukkan kredensial apa pun. Mac ada di jaringan, tetapi tidak di domain (setidaknya saya yakin begitu, saya tidak pernah mengerti apa itu domain).

\n\n

Saya berhasil berbagi file tetapi pengguna Windows dimintai login / kata sandi untuk mengaksesnya.

\n\n

Inilah yang saya lakukan:

\n\n
    \n
  • Saya mengaktifkan berbagi file, termasuk opsi SMB untuk akun saya.
    • \n
  • Saya menambahkan direktori ("common_files") untuk dibagikan di panel "Folder bersama"
    • \n
  • Saya memastikan "semua orang" memiliki akses membaca ke direktori itu
    • \n
  • Saya memastikan berbagi file Tamu diaktifkan di panel preferensi sistem "Berbagi".
    • \n
  • Saya memastikan login tamu diaktifkan di panel preferensi sistem "akun".
    • \n
\n\n

Saya memeriksa bahwa ketika memasukkan login / kata sandi akun Mac saya, saya dapat mengakses direktori tersebut. Tidak ada masalah. Untuk melakukannya, dari Windows, saya mengetik "URI": 

\\\\my.mac.IP.address\\common_files\n

Jadi semuanya berfungsi, kecuali bagian akses tamu.

\n\n

Ada ide?

\n\n

Terima kasih.

\n\n

Jean-Denis

Sepertinya ini adalah bug dalam implementasi SMB Apple sejak Lion (jadi untuk Lion dan Mountain Lion, tapi tidak untuk Snow Leopard). Jadi tidak ada cara mudah untuk mengatasinya. Saya sendiri pernah mengalami masalah yang sama dan tampaknya orang lain juga: http://forums.macrumors.com/showthread.php?t=1186688

\n\n

Solusi yang saya lihat adalah (saya belum mencobanya sendiri):

\n\n
    \n
  1. Siapkan akun pengguna baru di mac (hanya "Tamu" diperbolehkan? Alih-alih "Pengguna Tamu" yang merupakan akun Tamu biasa). Jangan berikan kata sandi. Kemudian orang lain dapat memasukkan "kredensial" ini saat diminta. Salah satu kelemahannya adalah Anda harus memberi tahu orang-orang nama pengguna apa yang akan digunakan.
    2. \n
  2. Siapkan server SMB Anda sendiri dan gunakan server tersebut alih-alih yang ada di dalam OSX. Seseorang di tautan di atas mengatakan bahwa mereka membuat ini bekerja menggunakan MacPorts (yang memungkinkan Anda menjalankan perangkat lunak Linux di Mac).
    3. \n
\n\n

Untuk saya sendiri, saya hanya berbagi antara komputer saya (Windows 8), dan GF saya (OSX Lion). Jadi saya membuat akun untuk diri saya sendiri di komputernya dan mengautentikasi menggunakan itu.

\n\n

Saya harap ini membantu. Ini jawaban StackOverflow / StackExchange / SuperUser pertama saya.
