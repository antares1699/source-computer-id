Saat saya menggunakan SecureCRT sebagai klien ssh saya, dan saya keluar dari sesi nano, prompt muncul di bawah status terakhir layar nano. Saya sudah terbiasa bisa merujuk kembali ke konten yang saya edit, jadi saya suka perilaku ini.

\n\n

Saat saya menggunakan dempul, itu menghapus konten layar nano sebelum mengembalikan saya ke prompt, jadi satu-satunya hal yang muncul di atas prompt baru saya adalah konten terminal yang ada sebelum memasuki nano.

\n\n

Saya telah mencari di pengaturan dempul, sejauh ini tidak berhasil. Adakah ide untuk bagaimana mendapatkan perilaku yang diinginkan ketika saya sedang dalam kesulitan? (atau mungkin itu adalah pengaturan nano atau terminal, dan dempul hanya mengikuti perintah yang tidak dilakukan SecureCRT?)

Jika Anda masuk ke layar Konfigurasi Putty, di bawah kategori "Terminal", di tab "Fitur", ada kotak centang untuk "Nonaktifkan peralihan ke layar terminal alternatif". Pastikan untuk memeriksanya, dan Anda akan mendapatkan perilaku yang Anda cari.

\n

Dari Dokumentasi Putty:

\n
\n

4.6.4 Menonaktifkan peralihan ke layar alternatif

\n

Banyak terminal, termasuk PuTTY, mendukung 'layar alternatif'. Ini adalah ukuran yang sama dengan layar terminal biasa, tetapi terpisah. Biasanya program berbasis layar seperti editor teks mungkin mengalihkan terminal ke layar alternatif sebelum memulai. Kemudian di akhir proses, itu beralih kembali ke layar utama, dan Anda melihat konten layar seperti sebelum memulai editor.

\n

Beberapa orang lebih suka ini tidak terjadi. Jika Anda ingin editor Anda berjalan di layar yang sama dengan aktivitas terminal Anda lainnya, Anda dapat menonaktifkan fitur layar alternatif sepenuhnya.

\n
\n

Gunakan TERM=vt100 untuk membiarkan konten di layar setelah keluar.

\n\n

Anda cukup menguji dengan melakukan 

export TERM=vt100\n

jika menggunakan bash. Tidak perlu memulai ulang shell. Anda juga dapat menyetel nilai ini di Putty, sebagai bagian dari konfigurasi untuk koneksi itu.
