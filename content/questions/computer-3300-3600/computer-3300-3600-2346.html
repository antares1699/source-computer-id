Ini sangat meningkatkan kebutuhan saya untuk menggunakan vim dengan lebih efektif. Apa yang dapat saya lakukan untuk membantu mempelajari pintasan yang sejauh ini belum pernah saya pelajari?

Untuk menggunakan vim secara lebih efektif, saya akan berkonsentrasi mempelajari lebih banyak perintah gerakan khusus (tanda kurung untuk berpindah demi kalimat, tanda kurung untuk berpindah demi paragraf). Mereka dapat digabungkan dengan perintah pengeditan (hapus, ubah, dll.) Setelah Anda terbiasa dengannya. Selain itu, pelajari kombinasi 'a' dan 'i' untuk memilih (dan menerapkan perintah ke) objek di sekitar kursor (mis. das untuk menghapus kalimat tempat kursor berada: bandingkan dengan (d), yang memiliki tiga tombol Shift). Belajar menerapkan hitungan untuk gerakan yang lebih besar (mis. c2w untuk mengubah dua kata berikutnya, d2as untuk menghapus kalimat tempat kursor berada dan yang berikutnya). Gunakan tanda untuk melompat ke tempat yang sering digunakan.

\n\n

Saya lupa salah satu perintah gerakan favorit saya: f/t/F/T bergerak maju atau mundur (huruf kecil atau huruf besar, masing-masing) ke karakter berikutnya atau sebelum berikutnya (f atau t, masing-masing) yang Anda ketik: f* pindah ke * berikutnya ke kanan, T( bergerak tepat ke kanan tanda kurung buka berikutnya ke kiri (tukar kiri dan kanan untuk konfigurasi kanan-ke-kiri).

\n\n

\n\n

Untuk pengeditan berulang, gunakan fungsi perekaman makro q untuk merekam dan @ untuk memutar ulang urutan dari perintah yang sama.

\n\n

Gunakan register alih-alih salin dan tempel, ini akan menyimpan perjalanan bolak-balik (karena makro disimpan dalam register, Anda dapat menggunakan perintah register store / ambil untuk memperbaiki kesalahan di makro tanpa merekam ulang).

\n\n

Gunakan perintah :%s/â€¦/â€¦/g untuk membuat penggantian / penghapusan global.

\n\n
\n\n

Dan jika Anda masih belum bisa membuatnya merasa nyaman, Anda mungkin bisa melakukan pengeditan melalui plugin netrw : vim scp://target/somefile.

\n\n

Sebagai akira catatan di bawah, ini memungkinkan Anda menjalankan Vim di komputer lokal Anda di mana latensi tidak menjadi masalah. Saat Anda mengedit file jarak jauh, Vim menyalinnya ke mesin lokal Anda. Saat Anda menyimpan, Vim menyalinnya kembali ke mesin jarak jauh. Mendukung banyak protokol (WebDAV, HTTP (read-only), FTP, SSH (scp, sftp), rsync).

Nah, Vi Sudah ditulis untuk sambungan modem yang lambat, jadi Vi / Vim mungkin merupakan alat terbaik untuk digunakan dalam situasi ini.

\n\n

Apa yang saya sarankan adalah untuk mempertimbangkan apa pun yang Anda temukan berulang, dan tulis makro untuk menghindari pengulangan.

\n\n

Gunakan: petakan secara bebas, dan (karena ini Vim), fitur perekaman makro 'q' (:help recording).

\n\n

Manfaatkan hitungan berulang (seperti yang telah Anda identifikasi) dan gunakan gerakan untuk menavigasi (seperti mencari dan langsung menuju ke nomor baris) daripada menggedor h / j / k / l.

\n\n

Jadi hal-hal seperti 5d untuk turun 5 baris, } untuk mencapai baris kosong berikutnya, Ctrl-I dan Ctrl-O untuk kembali / maju melalui posisi yang baru saja digunakan dalam teks (:help jumps).

\n\n

Pertimbangkan juga untuk mengunduh file dan mengedit secara lokal, kemudian mengunggah kembali ke server.
