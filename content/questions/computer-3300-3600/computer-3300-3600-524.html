Wikipedia memiliki artikel bagus tentang Bundel Aplikasi s.

\n\n

Singkatnya, .app bukanlah sebuah file: ini adalah pohon direktori dengan struktur tertentu. Biner sebenarnya yang berjalan (yaitu, setara dengan biner yang Anda temukan di /usr/bin) adalah PackageName.app/Contents/MacOS/PackageName. Namun, Anda (umumnya) tidak bisa begitu saja menyalin file itu ke '/ usr / bin' dan menjalankannya, karena itu akan berharap memiliki banyak file lain yang dibutuhkannya di dalam folder .app juga - misalnya, banyak aplikasi memiliki file terjemahan yang memungkinkan mereka menampilkan UI dalam berbagai bahasa di bawah PackageName.app/Contents/Resources/de.lproj (di mana de berarti german).

\n\n

Dalam beberapa kasus, bundel .app mungkin berisi executable yang dapat Anda jalankan secara langsung - misalnya, saya sering menggunakan biner ebook-convert dari bundel calibre.app, dengan menjalankan /Applications/calibre.app/Contents/Resources/loaders/ebook-convert. Dalam hal ini, ebook-convert kebetulan menjadi skrip python mandiri dan sangat senang dipanggil dengan cara ini - tidak semua file yang dapat dieksekusi akan bahagia.

\n\n

Anda biasanya dapat memulai .app dengan memanggil biner utama secara langsung - misalnya, /Applications/Safari.app/Contents/MacOS/Safari akan meluncurkan Safari. Namun, ini banyak mengetik - open /Applications/Safari.app adalah pintasan yang melakukan hal yang sama.

\n\n

Mengapa Anda mencoba memindahkan file .app ke / usr / bin? Saya akan menebak bahwa yang benar-benar ingin Anda ketahui adalah "bagaimana cara membuka file menggunakan Foo.app saat saya berada di baris perintah?"

\n\n

Jika saya benar, jawabannya bukanlah memindahkan file .app ke / usr / bin. Jawaban yang benar adalah dengan menggunakan perintah open: 

$ open -a Foo some_file\n

Menggunakan open tanpa -a akan membuka file dengan aplikasi defaultnya - sama seperti jika Anda mengklik dua kali pada some_file di Finder.
