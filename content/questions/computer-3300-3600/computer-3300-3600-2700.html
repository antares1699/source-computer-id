createdb adalah perintah yang harus terhubung ke server seperti perintah klien lainnya (bahkan ketika dijalankan di server itu sendiri), dan karena itu memerlukan pengguna database untuk terhubung.

\n\n

Jika Anda tidak menentukan pengguna database dengan opsi -U username, ini mengambil nama pengguna OS Anda, maka referensi ke \"R-61\" dalam pesan kesalahan. Kata sandi yang diminta adalah kata sandi dari pengguna database R-61 yang tidak ada.

\n\n

Ada juga opsi lain seperti host default ke localhost dan port default ke 5432, lihat halaman manual untuk semuanya.

\n\n

Karena Anda menyebutkan bahwa Anda mengetahui kata sandi superuser postgres, solusi paling sederhana adalah menggunakan pengguna postgres untuk memulai perintah. Ini masuk akal juga, karena pada penginstalan baru, pengguna ini akan menjadi satu-satunya pengguna yang memiliki izin untuk membuat database.

\n\n

Coba createdb -U postgres newdatabase

\n\n

Itu harus meminta kata sandi, dan itu akan menjadi kata sandi dari pengguna database postgres.

\n\n

Jika tidak meminta kata sandi, itu berarti kebijakan keamanan di pg_hba.conf dikonfigurasi seperti itu. Jika kata sandi hilang (atau tidak ada kata sandi yang pernah disetel, yang merupakan kasus untuk beberapa penginstalan otomatis, terutama di Unix), pg_hba.conf harus diubah dan layanan dimuat ulang untuk mengotorisasi koneksi tanpa kata sandi, setidaknya untuk sementara waktu untuk menyetel kata sandi baru .

Saya tidak memiliki pengalaman langsung dengan ini tetapi saya telah memeriksanya dan saya pikir inilah yang Anda cari.\nSaya menemukan masalah serupa yang menunjukkan bahwa PostgresSQL di Windows meminta kata sandi saat menginstal.\nKata sandi itu mungkin kata sandi untuk akun administrator untuk PostgresSQL.\nAkun admin harus bekerja untuk menyambungkan ke database itu dan menyesuaikan izin untuk itu.

\n\n

Atau, mengedit file pg_hba.conf untuk mempercayai koneksi pengguna dari pengguna dan lokasi tertentu akan memungkinkan Anda terhubung juga.\nAnda akan ingin memperbaiki masalah izin dan mengembalikan baris pg_hba.conf yang telah diedit.

\n\n
