Anda bisa menjalankan tcpdump untuk membuang paket DNS dan melihat bagaimana lalu lintas DNS berperilaku. Sesuatu seperti berikut, masuk ke Terminal, harus melakukan trik: 

sudo tcpdump -i en0 -n udp port 53\n

-i en0 harus mengacu pada antarmuka aktif Anda. Di Mac, biasanya en0, tetapi jika Anda memiliki ethernet jack dan adaptor nirkabel, Anda mungkin perlu menggunakan en1. Ini akan menghasilkan keluaran seperti (Saya telah membungkus garis panjang untuk kejelasan): 

22:19:46.160992 IP 192.168.1.143.61150 > 192.168.1.1.53:\n  60237+ A? www-google-analytics.l.google.com. (51)\n22:19:46.184272 IP 192.168.1.1.53 > 192.168.1.143.61150:\n  60237 11/0/0 A 74.125.225.233, A 74.125.225.238, A 74.125.225.224,\n  A 74.125.225.225, A 74.125.225.226, A 74.125.225.227, A 74.125.225.228,\n  A 74.125.225.229, A 74.125.225.230, A 74.125.225.231, A 74.125.225.232 (227)\n

Dump ini menunjukkan permintaan dari mesin saya ke router saya (192.168.1.143 > 192.168.1.1) pada 22:19:46.160992. Router saya membalas kembali pada 22:19:46.184272 dengan tanggapan. Jadi permintaan DNS ini membutuhkan waktu sekitar 23ms.

\n\n

Jika Anda mencurigai adanya masalah kinerja DNS, jalankan perintah tcpdump dan cari permintaan DNS untuk server yang Anda minati. Jika Anda melihat penundaan yang lama atau beberapa kali mencoba, Anda tahu bahwa Anda mengalami masalah.
