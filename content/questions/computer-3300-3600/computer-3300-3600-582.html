Saya mengalami masalah saat menarik seluruh direktori file dari perangkat android yang di-rooting dengan ADB. Ada kesalahan izin khusus. Jadi saya pikir solusinya adalah dengan mengubah semuanya sehingga daemon ADB secara efektif berjalan dengan izin root (tidak dapat menjalankannya dengan izin root secara langsung). Kemudian mungkin chmod semuanya kembali seperti semula setelah saya menarik semua file dengan adb pull.

\n\n

Apakah ada risiko yang terlibat dengan pendekatan ini? Bagaimana dengan tidak mengubah izin kembali ke normal? Meskipun kasus saya khusus untuk android, saya tertarik untuk mendengar perbedaan apa pun untuk sistem linux normal.

Dengan mengubah semuanya menjadi 777, Anda melewati sebagian besar pengaturan keamanan hak istimewa Linux, dan ini akan dianggap tidak aman, dan jelas bukan praktik terbaik.

\n\n

Masalahnya adalah jika seseorang dapat menembus sistem Anda sebagai pengguna selain diri Anda sendiri, jika file Anda adalah CHMOD 777, mereka dapat membaca, menulis, menjalankan, dan menghapusnya. Ini dianggap sebagai ancaman keamanan.

\n\n

Demikian pula, memiliki file dengan izin "777" berarti bahwa jika seseorang dapat menemukan file yang tidak dapat dieksekusi, dan menimpanya dengan skrip, dan dapat mengelabui Anda untuk menjalankan file tersebut, maka file tersebut akan dieksekusi, yang merupakan risiko keamanan.

\n\n

Jika itu saya, saya pasti tidak akan melakukannya, tetapi jika Anda adalah satu-satunya pengguna sistem Anda, dan Anda tidak khawatir tentang serangan yang ditargetkan, Anda mungkin dapat melakukannya di perangkat Android (Anda rentan , tetapi agak terisolasi oleh sistem hak istimewa dan sifat perangkat pengguna tunggal) - tetapi melakukannya di semua jenis server Linux memohon agar perangkat itu diretas.
