Periksa rdesktop :

\n\n
\n

Untungnya, Linux memiliki utilitas rdesktop. Ini adalah klien untuk protokol desktop jarak jauh (RDP), yang digunakan di sejumlah produk Microsoft termasuk Windows NT Terminal Server, Windows 2000 Server, Windows XP dan Windows 2003/2008 Server. Anda tidak perlu menginstal server VNC. Yang Anda butuhkan hanyalah klien rdesktop di workstation Linux atau BSD.

Jika Windows Anda adalah versi "Pro" atau "lebih baik" dari RDP akan bekerja. Versi "Rumah" tidak memiliki sisi server. Jadi, Anda membutuhkan sesuatu seperti VNC.\nJuga RDP tidak memungkinkan untuk berbagi desktop, jadi jika Anda ingin berbagi desktop dengan pengguna lokal maka VNC kembali direkomendasikan.

\n\n

Sarankan tightvnc untuk sisi server Windows. Ada beberapa implementasi VNC sisi klien untuk kotak Linux serta setidaknya beberapa klien RDP (rdesktop, freerdp) jika Anda menggunakan rute itu.

\n\n

Anda tentu saja perlu meneruskan port ke server jika berada di belakang firewall NAT.
