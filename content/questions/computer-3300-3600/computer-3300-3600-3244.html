Ada beberapa faktor yang bekerja di sini:

\n\n

Pertama , dengan menggunakan -exec, find menghasilkan proses baru untuk grep untuk setiap file yang ditemukannya.

\n\n

Ini dapat dikelola baik dengan menggunakan xargs seperti dalam jawaban Felipe Alvarez (saya menulis file posting blog tentang ini 5+ tahun yang lalu) atau dengan menggunakan -exec grep param1 {} + (perhatikan + bukannya \\;). Saat menggunakan +, "baris perintah dibuat dengan menambahkan setiap nama file yang dipilih di akhir; jumlah pemanggilan perintah akan jauh lebih sedikit daripada jumlah file yang cocok" (dari temukan halaman manual )

\n\n

Kedua , jumlah dan ukuran file di bawah /var berpotensi sangat besar.

\n\n

Dapatkah Anda menambahkan parameter selain -type f untuk membatasi file yang Anda cari. find memiliki opsi untuk dibatasi oleh waktu, pemilik, pola nama, dll. Misalnya, Anda dapat melakukan: 

find /var -type f -name \"access*log\" -mtime -7 -exec grep param1 {} +\n

Itu akan menemukan file yang dimodifikasi dalam tujuh hari terakhir dengan nama yang dimulai dengan "akses" dan diakhiri dengan "log". Hanya file yang cocok yang akan menjadi bagian dari exec / grep.
