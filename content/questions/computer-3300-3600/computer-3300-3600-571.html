Jika Anda ingin membuatnya agar siswa tidak dapat mengakses file satu sama lain, Anda memiliki dua opsi:

\n\n
    \n
  1. Gunakan GID per pengguna (UID == GID) dan setel umask default ke 0750.
    2. \n
  2. Setel umask default ke 0700.
    3. \n
\n\n

Opsi pertama lebih cepat diimplementasikan, dan jauh lebih kecil kemungkinannya untuk merusak banyak hal. Perlu diingat bahwa tidak ada cara praktis tanpa menggunakan sesuatu seperti SELinux untuk mencegah orang secara sengaja mengizinkan orang lain mengakses file mereka.

Banyak distro secara otomatis akan membuat grup utama bersama dengan akun pengguna baru. Secara default, ini menggunakan ID berikutnya yang tersedia, dan ini biasanya berarti bahwa UID pengguna baru akan sama dengan GID-nya, kecuali Anda sebelumnya telah membuat grup dengan GID otomatis yang ditetapkan padanya, atau UID tanpa primernya sendiri. kelompok.

\n\n

Mengatakan UID == GID dengan demikian adalah keliru, karena pada banyak mesin multi-pengguna, UID mungkin tidak cocok dengan GID pengguna.

\n\n

Biasanya, semua akun pengguna juga merupakan anggota dari grup umum, biasanya users(100), jadi pada sistem di mana setiap akun memiliki grup utamanya sendiri, Anda dapat beralih dari satu paradigma ke paradigma lain dengan cepat, mis. dengan chown -R :users . file dalam sebuah folder.

Ini terutama masalah preferensi pribadi.

\n\n

Saya memiliki pertanyaan yang sama beberapa tahun yang lalu dan melakukan beberapa penelitian tentang masalah tersebut.
Sejauh yang saya bisa tentukan, tidak ada manfaat besar yang nyata untuk kedua metode tersebut. Sebagian besar distro Linux utama saat ini tampaknya menggunakan metode UID = GID secara default. Beberapa sistem Unix lain masih lebih menyukai metodologi lain.\nSepertinya tidak masalah selama Anda menggunakan UMASK dengan benar. (750 adalah standar untuk GID = UID, 700 lebih disukai jika beberapa pengguna berbagi GID.)

\n\n

Mungkin ada beberapa perangkat lunak yang mengasumsikan satu atau yang lain, tetapi untuk setiap kasus perangkat lunak yang lebih memilih GID = UID Anda dapat menemukan perangkat lunak lain yang menginginkannya sebaliknya. Dan Anda selalu bisa mendapatkan sesuatu untuk bekerja dengan beberapa penggunaan kreatif chown dan chgrp.

\n\n

Di beberapa organisasi, mungkin terdapat preferensi untuk GID bersama, karena cara penyiapan sumber daya jaringan bersama. (Izin NFS atau cara mereka memetakan izin Unix pada SMB ACL.)

\n\n

Jadi saya akan mengikuti arus: Gunakan apa pun yang ditentukan di organisasi Anda (jika ada aturan seperti itu) atau apa pun yang default di distro pilihan Anda (untuk meminimalkan masalah dengan aplikasi yang mengasumsikan sistem tertentu berdasarkan distro yang digunakan).
