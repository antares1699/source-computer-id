Bagaimana cara menambahkan margin tepian (berbeda untuk halaman ganjil dan genap) ke dokumen PDF sehingga ada ruang untuk membuat lubang setelah saya mencetaknya? Saya akrab dengan Ghostscript dan Terminal Linux tetapi tidak dapat menemukan perintah untuk memenuhi kebutuhan khusus ini.

\n\n

Saat ini beberapa konten terlalu dekat dengan tepi. Saya dapat menggunakan penskalaan di kotak dialog cetak untuk menciptakan ruang untuk penjilidan, tetapi ini akan mengurangi ukuran lebih dari yang benar-benar diperlukan dan meninggalkan celah di kedua sisi halaman.

\n\n

Solusi yang hanya menambahkan satu sentimeter ekstra ke kedua sisi halaman (kiri untuk halaman ganjil dan kanan untuk genap) akan memadai di sini karena saya kemudian dapat memilih fitur "skala agar pas" ketika saya mencetaknya.

\n\n

Pertanyaan saya awalnya ditandai sebagai duplikat dari ini , tetapi tidak ada jawaban yang memecahkan masalah saya. Saya mencoba menggunakan Briss dan PDF Scissors, tetapi mereka adalah alat pemangkasan PDF dan hanya dapat membuat halaman lebih kecil, tidak lebih besar. Pertanyaan saya juga berbeda karena saya perlu mengubah ukuran secara berbeda pada halaman ganjil dan genap. Penanya pertanyaan awal tidak memiliki persyaratan ini.

Saya bermaksud untuk mengembangkan ini menjadi jawaban yang tepat ketika saya punya waktu, tetapi inilah solusi yang saya temukan, jika ada yang membutuhkannya untuk sementara waktu. Idenya adalah menggunakan Ghostscript dengan opsi -c untuk memodifikasi file PDF menggunakan PostScript kustom.

\n\n

Pertama, Anda perlu mengetahui tinggi dan lebar PDF asli di Poin (ada 72 Poin dalam satu inci). Anda dapat menggunakan alat "pdfinfo" untuk menemukan informasi ini: 

pdfinfo \"inputFile.pdf\"\n

Misalkan Anda memiliki dokumen A4 - A4 adalah 595pts × 842pts (210mm × 297mm).

\n\n

Jika Anda ingin menambahkan selokan 10mm (= 1cm = 28pts) ke dokumen A4,\nmaka Anda perlu memberikan informasi berikut ke Ghostscript:

\n\n

-dDEVICEHEIGHTPOINTS=842 (tinggi A4)
-dDEVICEWIDTHPOINTS=623 (= 595 + 28, yaitu lebar A4 + talang yang akan ditambahkan)
{28 0 translate} {} ({geser halaman ganjil 28pts ke kanan & 0pts ke atas} {jangan lakukan apa pun ke halaman genap})

\n\n

Catatan: Menentukan penambahan lebar akan menambah spasi di sisi kanan halaman, jadi hanya halaman bernomor ganjil yang harus diterjemahkan - halaman genap sudah ditata dengan benar.

\n\n

Berikut perintah lengkap untuk Linux: 

gs -q -sDEVICE=pdfwrite -dBATCH -dNOPAUSE -sOutputFile=outputFile.pdf \\\n  -dDEVICEWIDTHPOINTS=623 -dDEVICEHEIGHTPOINTS=842 -dFIXEDMEDIA \\\n  -c \"<< /CurrPageNum 1 def /Install { /CurrPageNum CurrPageNum 1 add def\n   CurrPageNum 2 mod 1 eq {28 0 translate} {} ifelse } bind  >> setpagedevice\" \\\n  -f \"inputFile.pdf\"\n

Catatan: Perintah ini membuat PDF menjadi 1cm lebih lebar dari A4. Jika Anda kemudian mencetak pada A4 semuanya akan diperkecil agar pas dan talang sebenarnya akan sedikit lebih tipis dari 1cm (akan menjadi 9,5mm). Jika itu tidak cukup baik maka Anda harus mengerjakan sedikit aljabar untuk mendapatkan offset yang benar. Dalam hal ini, akan menjadi 29.8pts, bukan 28pts, untuk mendapatkan talang cetak 10.0mm (Anda juga perlu menyetel -dDEVICEWIDTHPOINTS=624.8). Namun, jika talang kurang dari ~ 2cm maka mungkin tidak ada gunanya melakukan penghitungan, terutama mengingat toleransi dalam pencetakan dan ukuran kertas.
