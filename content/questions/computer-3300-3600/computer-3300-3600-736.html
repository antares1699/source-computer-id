? [= Sender ObnoxiousLogger] ignore\n

Anda dapat menggunakan perintah berikut untuk menyetel tingkat pemfilteran log individu untuk proses tertentu: 

syslog -c processname -d\n

Ini akan menyetel tingkat log sedang berjalan contoh processname hingga semua pesan dengan level debug atau lebih tinggi . level yang mungkin adalah masing-masing karakter pacewnid: (Panik), Waspada, Kritis, Kesalahan, Peringatan, Pemberitahuan, Info, dan Debug .

\n\n

Jadi, untuk hanya mengaktifkan logging pesan peringatan dan lebih tinggi untuk Finder, gunakan: 

syslog -c Finder -w\n

Anda perlu mengulang ini setiap kali Anda memulai kembali proses yang dimaksud.

Solusi yang diusulkan oleh Torsten tampaknya tidak berhasil. Tampaknya menambahkan konfigurasi tersebut ke asl.conf global tidak memiliki efek yang diharapkan seperti cara menangani aturan.

\n\n

Solusi untuk masalah ini, yang saya temukan, setelah saya hampir mengamuk berkelahi dengan asl.conf utama, adalah dengan menggunakan file konfigurasi asl/* terpisah untuk setiap filter. File-file itu harus disimpan pada /etc/asl/

\n\n

misalnya untuk mengabaikan "com.example.Sender" dan memfilter spam log-nya dari system.log kita dapat membuat file /etc/asl/com.example.Sender dengan konten berikut: 

? [= Sender com.example.Sender] claim only\n* ignore\n

claim only untuk pertandingan ("Pengirim" dalam contoh) menyebabkan aturan dalam master asl.conf diabaikan yang berarti hal-hal tidak akan berakhir pada system.log.

\n\n

Jangan lupa restart syslogd, tentunya.

Saya memposting lebih detail menjawab di AskDifferent.

\n\n

Masukkan /etc/asl.conf aturan seperti berikut 

? [= Sender foobar] [<= Level error] notify com.apple.foobar\n  op   key   value   op  key   value\n  -----query 1----- -----query 2---- ---------action--------\n

Untuk mengabaikan pesan di kedua system.log dan di Console.app, Anda harus menambahkan aturan untuk mengabaikan pesan sebelum komentar berikut pada asl.conf.

\n\n

# Flat file configurations formerly in syslog.conf

\n\n

Contoh dengan pengubah S, yang mengabaikan pesan yang cocok dengan substring tersebut dari SIMBL Agent. 

? [= Sender SIMBL Agent] [S= Message warning: failed to get scripting definition from] ignore\n  op -key-- ---value---  mod --key-- --------------------value-----------------------\n  --------query 1------- -----------------------query 2------------------------------- action\n

Setelah menambahkan aturan, restart daemon dengan sudo killall HUP syslogd.

\n\n

Berhasil diuji di OS X 10.8.5.
