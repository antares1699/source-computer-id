Di Firefox, "Server tidak ditemukan" memberi tahu Anda bahwa ia tidak mengetahui alamat IP-nya † . Jadi, memang masalah DNS.

\n\n

Jika Anda mengalami masalah ini lagi, Anda bisa segera coba lihat apakah server DNS lain akan memberikan hasil yang berbeda. Jadi, jangan segarkan browser Anda, jangan bersihkan cache, tetapi jalankan perintah berikut (seperti yang sudah Anda lakukan) untuk memberi tahu Anda apa server DNS Anda saat ini tahu tentang domain yang merepotkan. Ketika Anda mendapatkan jawaban untuk berikut ini maka jenis tes gagal, tetapi mungkin yang berikut ini berhasil tidak memberi Anda balasan yang berguna sama sekali: 

dig www.python.org\n\n  [..]\n  ;; ANSWER SECTION:\n  www.python.org.  33592 IN A 82.94.164.162\n  [..]\n  ;; Query time: 168 msec\n  [..]\n  ;; WHEN: Sat Nov 14 12:56:54 2009\n

Hasil di atas mungkin telah di-cache oleh router / modem Anda, atau didelegasikan ke server DNS ISP Anda untuk apa pun yang belum diketahui oleh router. Nilai untuk Query time mungkin beberapa indikasi apakah hasilnya di-cache di modem / router atau tidak. Jadi, jalankan kembali perintah yang sama (tekan panah ke atas dan Kembali di Terminal): 

  ;; Query time: 33 msec\n  [..]\n  ;; WHEN: Sat Nov 14 12:57:14 2009\n

Di atas, melihat 168 versus 33 milidetik, kita dapat berasumsi bahwa hasil untuk perintah dig pertama tidak di-cache di modem / router, dan meminta server DNS penyedia membutuhkan waktu ekstra. Permintaan kedua mungkin telah di-cache.

\n\n

Jika waktu kueri sangat berbeda untuk Anda juga, maka karena alasan tertentu browser atau OS Anda belum meminta domain tersebut dari DNS, meskipun browser Anda mencoba menampilkan halaman itu. Itu mungkin akan memberi tahu Anda bahwa DNS komputer kacau (yang masuk akal, karena komputer lain yang menggunakan modem / router yang sama tidak mengalami masalah yang sama).

\n\n

Selanjutnya, periksa apakah, misalnya, OpenDNS menghasilkan hasil lain: 

dig @208.67.222.222 www.python.org\n\n  [..]\n  ;; ANSWER SECTION:\n  www.python.org.  83166 IN A 82.94.164.162

Di atas, hasilnya sama, tetapi jarak tempuh Anda mungkin berbeda. (Anda hanya perlu membandingkan alamat IP; jika secara eksplisit menggunakan beberapa server DNS, seperti dengan menggunakan @208.67.222.222, waktu kueri tidak akan jauh berbeda untuk permintaan berulang.)

\n\n

Jadi, setiap kali ini terjadi lagi: beri tahu kami hasil di atas? Atau, adil mengatur Mac Anda menggunakan server OpenDNS untuk sementara waktu, dan melihat apakah masalahnya masih terjadi?

\n\n

† Server yang tidak merespons akan memberi Anda "Waktu koneksi telah habis", dan mungkin perlu waktu lebih lama untuk menunjukkan kesalahan itu.
