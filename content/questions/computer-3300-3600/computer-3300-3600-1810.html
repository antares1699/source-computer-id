Dari: Perbaiki Windows 10 VPN error 789 koneksi gagal karena masalah keamanan :

\n\n

Kesalahan Akses Jarak Jauh 789 muncul ketika sistem Anda tidak diatur dengan benar untuk terhubung ke server L2TP, sehingga upaya koneksi gagal bahkan sebelum Anda membuat koneksi dengan server.

\n\n

Itu juga terkait dengan konfigurasi yang salah dari sistem operasi Anda seperti Windows 10 dalam kasus ini. Kesalahan umum ini terjadi saat negosiasi IPSec gagal untuk koneksi L2TP / IPSec.

\n\n

Penyebab lain yang mungkin termasuk:

\n\n
    \n
  • Klien VPN berbasis L2TP (atau server VPN) berada di belakang NAT.
    • \n
  • Sertifikat yang salah atau kunci yang dibagikan sebelumnya diatur di server atau klien VPN.
    • \n
  • Sertifikat mesin atau sertifikat mesin root tepercaya tidak ada di server VPN.
    • \n
  • Sertifikat Mesin di Server VPN tidak memiliki 'Otentikasi Server' sebagai EKU.
    • \n
\n\n

Berikut adalah solusi yang dapat Anda gunakan untuk memperbaiki Windows 10 VPN error 789 di komputer Anda.

\n\n
    \n
  1. Setel ulang adaptor jaringan
    2. \n
  2. Periksa sertifikatnya
    3. \n
  3. Aktifkan kembali IPSec di komputer Anda
    4. \n
\n\n

Sebelum mencoba salah satu solusi ini, pastikan bahwa opsi pass-through L2TP dan IPSec diaktifkan dari router Anda. Jika Anda mengkonfigurasi layanan VPN Anda secara manual, pastikan Anda menggunakan kunci preshared.

\n\n
    \n

  1. Setel ulang adaptor jaringan \n

    \n\n
      \n

    • Klik kanan Start dan pilih Pengaturan perangkat .

      \n\n

      • \n

    • Temukan Adaptor jaringan dan klik untuk memperluas daftar

      \n\n

      • \n

    • Identifikasi adaptor jaringan Anda dan klik kanan padanya, lalu pilih Copot pemasangan .

      • \n
    • Klik baik .
      • \n
    • Hidupkan Kembali komputer Anda. Perangkat akan menginstal ulang dan harus mengatur ulang ke pengaturan default.
      • \n
    \n\n


    Jika ini tidak memperbaiki kesalahan 789, coba solusi berikutnya.

    2. \n

  2. Periksa sertifikatnya

    \n\n

    Pastikan sertifikat yang benar digunakan di sisi klien dan server. Jika Pre Shared Key (PSK) digunakan, pastikan bahwa PSK yang sama dikonfigurasi di sisi klien, dan mesin server VPN.

    3. \n

  3. Aktifkan kembali IPSec di komputer Anda

    \n\n
      \n

    • Klik kanan Start dan pilih Lari .

      \n\n

      • \n

    • Ketik services.msc

      • \n

    • Temukan 'IKE and AuthIP IPSec Keying Modules'.

      \n\n

      • \n

    • Temukan 'Agen Kebijakan IPSec'.

      \n\n

      • \n

    • Periksa statusnya. Jika tertulis 'mulai' klik untuk memulai ulang. Jika opsi 'mulai' dinonaktifkan, aktifkan.

      • \n
    • Klik dua kali pada masing-masing dari keduanya.
      • \n

    • Pilih Jenis startup .

      \n\n

      • \n

    • Ubah ke Otomatis .

      \n\n

      • \n

    • Simpan perubahannya.

      • \n
    • Mulai ulang layanan VPN Anda.
      • \n
    4. \n
\n\n

Setelah Anda melakukan semua langkah di atas dengan hati-hati, VPN akan bekerja dengan lancar karena pengaturan protokol telah disetel ulang ke default. Namun, jika tidak berhasil, Anda harus menyetel secara manual metode enkripsi untuk server dan sisi klien, agar kompatibel.

\n\n

Jika Anda memiliki masalah khusus pengguna di komputer Anda namun Anda masih mendapatkan kesalahan 789 setelah mencoba salah satu solusi di atas, Anda juga dapat menghubungi layanan pelanggan atau tim dukungan teknis untuk penyedia VPN khusus Anda dan membagikan detailnya untuk bantuan lebih lanjut.
