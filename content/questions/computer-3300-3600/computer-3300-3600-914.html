Apakah mungkin untuk mengakses server Apache saya dari luar VirtualBox di browser Google Chrome di komputer host saya? Ini berjalan di CentOS 7 di VirtualBox. Saya telah mencoba menghubungkan ke alamat ip mesin virtual CentOS tetapi tidak berhasil. Ini menggunakan jaringan 'Bridged Adapter' dalam pengaturan VM dan saya memeriksa alamat ip menggunakan perintah 'ip addr'. Terima kasih.

Saya bertemu masalah yang sama. Saya memperbaikinya sebagai berikut:

\n\n
    \n
  1. Dalam pengaturan jaringan VM, menggunakan 'Bridged Adapter'.
    2. \n
  2. Nyalakan port CenOS7: menggunakan perintah berikut di terminal (misalnya port 8080):\n\n$ firewall-cmd --permanent --add-port=8080/tcp\n$ firewall-cmd --reload\n
    3. \n
\n\n

Lingkungan saya:
Host: Win7 64bit
Tamu: CenOS 7

Anda dapat menggunakan satu NAT (tanpa adaptor kedua):

\n\n

Setel adaptor No1 ke NAT, lalu klik "Penerusan port", ketik (misalnya) 8080 di porta Host, dan 80 di port Tamu (tidak perlu alamat ip).\nAnda mungkin juga (opsional) ingin memetakan SSH di sini ke port yang berbeda.

\n\n

Kemudian di VM CentOS 7, setelah server Apache atau NGINX Anda diinstal dan berjalan (misalnya: yum install nginx, systemctl start nginx, systemctl aktifkan nginx), konfigurasikan firewall seperti yang disarankan niaomingjian, tetapi mungkin lebih tepat: 

sudo firewall-cmd --zone=public --add-service=http --permanent\nsudo firewall-cmd --reload\n

Kemudian navigasikan di mesin host Anda ke http://127.0.0.1:8080 dan periksa apakah Anda mendapatkan halaman selamat datang.\nJika ini tidak langsung berhasil, coba hentikan dan mulai layanan firewalld, atau boot ulang VM.
