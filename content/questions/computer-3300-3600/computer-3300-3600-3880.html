Saya membuat arsip dari kumpulan file besar menggunakan alat baris perintah 7-Zip, dan saya akan menulis arsip ke DVD (2 set, agar aman).

\n\n

Dengan asumsi tepat satu file (bagian dari arsip 7-Zip) per DVD-R yang dibakar, berapa ukuran volume terbesar yang dapat saya bagi menjadi arsip saya?

","source":"https://superuser.com/questions/597929/what-is-the-largest-file-i-can-write-to-a-dvd","author":"Brendan Kidwell","replies":[],"answers":[{"answer":"

Pertama-tama, pastikan Anda membakar cakramnya UDF format alih-alih format ISO-9660. ISO-9660 hanya mendukung file hingga 2 gibibyte atau 4 gibibyte tergantung pada implementasinya.

\n\n

Format UDF tampaknya tidak memiliki banyak overhead dasar untuk volume dengan satu file di dalamnya, tetapi mari kita gunakan angka 10 mebibyte overhead (10 Ã— 1024 2 byte atau sektor berukuran 5120 DVD), untuk memungkinkan kesalahan dalam proses penulisan dan banyak ruang untuk sistem file untuk berjaga-jaga.

\n\n

(Nilai kapasitas dikutip dari http://en.wikipedia.org/wiki/Dvd#Capacity .)

\n\n
    \n

  • Lapisan tunggal DVD-R
    2.298.496 sektor, 4.707.319.808 byte
    kurangi 10MiB overhead = 4.696.834.048 byte

    • \n

  • DVD + R satu lapisan
    2.295.104 sektor, 4.700.372.992 byte
    kurangi 10MiB overhead = 4.689.887.232 byte

    • \n

  • Lapisan ganda DVD-R
    4.171.712 sektor, 8.543.666.176 byte
    kurangi 10MiB overhead = 8.533.180.416 byte

    • \n

  • DVD + R lapisan ganda
    4.173.824 sektor, 8.547.991.552 byte
    kurangi 10MiB overhead = 8.537.505.792 byte

    • \n
\n\n

Jadi, jika Anda menggunakan cakram lapisan tunggal (atau mungkin beralih antara + R dan -R), baris perintah 7-Zip adalah: 

7za a -v4689887232b FOLDER.7z FOLDER\n
