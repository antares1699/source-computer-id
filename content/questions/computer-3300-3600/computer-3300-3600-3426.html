Saya membuat program backup ini lebih dari setahun yang lalu. Itu tidak pernah terdeteksi sebagai ancaman sebelumnya. Pagi ini ketika mencoba melakukan apa yang selalu dilakukannya, Windows Defender mengkarantina, menyebutnya sebagai ancaman parah "Trojan: Win32 / Bearfoos.A! Ml"

\n\n

Ini menggunakan dua DLL yang saya tulis sendiri, salah satunya melakukan pencarian rekursif dan yang lainnya membaca / menulis file. Pada dasarnya .exe saya membaca file konfigurasinya, menimpanya jika terjadi masalah, kemudian menggunakan info yang dimuat untuk memulai pencarian rekursif dan kemudian menyalin / mengompres (menggunakan 7za.exe) file tersebut ke dalam beberapa drive.

\n\n

Ini juga Pinvoking kernel32 (GetConsoleWindow) dan user32.dll (ShowWindow) karena ini adalah aplikasi konsol.

\n\n

Saya yakin saya bisa menambahkannya ke pengecualian, ada utas lain tentang Avast tentang hal yang sangat mirip yang menyarankannya. Saya hanya bertanya-tanya mengapa?\nKenapa sekarang? Mengapa Bearfoos? Mengapa Windows Defender tidak dapat mendeteksi bahwa saya menulis program itu sendiri? Mengapa Windows Defender tidak menyadari itu hanya menyalin file di sekitar drive lokal saya sendiri? Saya bahkan menambahkannya sendiri ke Penjadwal Tugas Windows, berapa banyak lagi bendera hijau yang dibutuhkan Windows Defender !?

Mengapa ini mulai terjadi sekarang? Kemungkinan ada pembaruan terkini untuk Windows Defender yang menyebabkannya mengubah perilakunya ke program Anda. Deteksi perangkat lunak berbahaya sulit dan prosesnya tidak 100% dapat diandalkan. Terjadi kesalahan. Terkadang perangkat lunak berbahaya tidak terdeteksi dan terkadang perangkat lunak yang sah salah diidentifikasi sebagai perangkat lunak berbahaya.

\n\n

Windows Defender tidak mencoba menentukan apakah Anda menulis program dan dengan demikian memberinya hak khusus. Itu akan menjadi ide yang sangat buruk. Para programmer tidak mempercayai kemampuan mereka untuk melakukan ini dengan keandalan yang memadai. Jika ada fasilitas seperti itu, itu akan memberi malware metode potensial lain untuk menghindari deteksi. Windows Defender tidak memiliki gambaran besar tentang program yang Anda lakukan. Itu hanya dapat memeriksa file-nya untuk mengetahui pola malware yang dikenal dan memantau aktivitasnya. Dan ia melakukan semua ini dengan pengetahuan bahwa segala sesuatunya mungkin tidak seperti yang terlihat. Malware modern sangat canggih dan memiliki banyak trik sehingga akan muncul sebagai sesuatu yang lain selain apa adanya. Malware terus menyempurnakan metodenya untuk menghindari deteksi dan perangkat lunak keamanan harus menyempurnakan metode pendeteksiannya.

\n\n

Sebagian besar perangkat lunak keamanan memiliki semacam daftar pengecualian. Tetapi bahkan ini tidak sesederhana kelihatannya. Ini harus dikelola dengan sangat hati-hati atau perangkat lunak berbahaya akan menambahkan dirinya sendiri ke dalam daftar. Penulis malware mempelajari Windows Defender dan produk keamanan lainnya, selalu mencari kelemahan yang mungkin mereka manfaatkan.
