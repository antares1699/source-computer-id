saya menggunakan TortoiseSVN sebagai klien subversi saya di Windows 7. Repositori subversi saya memiliki URL https.

\n\n

Saya berada di jaringan publik dan saya ingin memastikan bahwa koneksi saya ke server aman. Saya bertanya-tanya apakah TortoiseSVN mengenkripsi kata sandi saya ketika berbicara ke repositori saya.

","source":"https://superuser.com/questions/148510/how-secure-is-the-subversion-connection-to-an-https-url","author":"reprogrammer","replies":[],"answers":[{"answer":"

Jika repositori SVN Anda memiliki URL HTTPS, maka semua lalu lintas SVN (termasuk nama pengguna dan sandi Anda) akan mengalir melalui HTTPS tersebut ( HTTP melalui SSL / TLS ) koneksi. Sejauh menyangkut lalu lintas jaringan, ini mirip dengan koneksi HTTPS lainnya, meskipun SVN menggunakan beberapa perintah HTTP WebDav / DeltaV tambahan.

\n\n

Lalu lintas SVN Anda harus sangat aman, selama administrator memelihara server web SVN yang aman dan selama Anda tetap memperbarui klien TortoiseSVN Anda.

HTTPS adalah SSL . Cukup aman.

\n

Segala sesuatu di luar pencarian DNS awal akan dienkripsi.

\n

Perhatikan bahwa saya tidak yakin bagaimana TortoiseSVN bekerja. Ini dapat membuka koneksi UDP (atau koneksi TCP tidak aman) saat mengirim data mentah ke repositori. Disarankan untuk memeriksa dengan tim pengembangan atau membuka Wireshark saat menjalankan komit dan melihat apakah ada lalu lintas tidak aman yang keluar.
