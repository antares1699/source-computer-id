Jika Anda pergi ke https://github.com/stascorp/rdpwrap/ Ada tiga cara untuk mengunduh program. Ada mengunduh kode, dari tab kode. Atau, di bawah tab rilis, ada mengunduh file MSI. Atau, juga di bawah tab rilis, ada mengunduh zip.

\n\n

Dari metode itu, metode yang saya gunakan yang berhasil, adalah pergi ke tab rilis, dan unduh Zip di sana. Jadi, bukan tab kode, bukan zip itu. Dan bukan MSI yang dirilis. Tapi zip di bawah rilis. RDPWrap-v1.6.2.zip

\n\n

Kemudian, ekstrak zip itu ke beberapa direktori. Mulai perintah cmd, jalankan sebagai administrator. Jadi prompt cmd administratif. (dari sana Anda dapat menjalankan install.bat yang lebih baik daripada berada di prompt cmd biasa dan mengklik kanan install.bat dan melakukan runas pada bat, karena tampaknya itu bisa menjadi buggy).

\n\n

Mengekstrak hasil zip

\n\n

rdpconf.exe akan mengatakan "tidak didukung"

\n\n

Sayangnya file ini yang dibundel dengan proyek sudah lama.

\n\n

Ada update.bat tetapi mendapat file ini lama yang tidak mendukung banyak versi termrv.dll.

\n\n

Jadi Anda harus pergi ke halaman github dan melihat masalah atau pulls dan melihat apakah Anda bisa mendapatkan file ini dengan versi build Anda di dalamnya.

\n\n

Untuk mengganti c:\\program files\\rdp wrap\\rdpwrap.ini Anda harus menghentikan layanan 'layanan desktop jarak jauh', untuk membuka penguncian ini, lalu menggantinya, lalu memulainya. Dan Anda melihat dari services.msc bahwa itu disebut TermService. Anda dapat melakukan sc queryex termservice untuk melihat apakah itu berjalan atau tidak. s (sc queryex sedikit lebih baik daripada sc query 'cos sc queryex menunjukkan pid). Dan Anda dapat melakukan net start termservice, net stop termService, dan taskkill jika macet 'berhenti'.

\n\n

Kemudian Anda dapat menjalankan rdpconf.exe untuk melihat apakah dikatakan berhasil .. (sebagai lawan dari 'tidak didukung'). Anda bahkan dapat menggunakan rdpconf sebelum (kembali) memulai layanan terminal

\n\n

Dan kemudian Anda dapat melakukan rdpcheck, dan melihat apakah itu mengatakan "koneksi terbatas", atau apakah itu berfungsi. (atau Anda dapat menguji dengan mstsc, tetapi rdpcheck sangat cepat untuk diuji)

\n\n

Dan kemudian Anda tahu dan semoga itu akan berhasil untuk Anda.
