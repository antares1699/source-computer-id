Saya bingung karena saya tidak dapat menemukan jawabannya di mana pun.

\n\n

Saya telah menginstal Apache 2.2 untuk Windows di laptop saya. Ini memungkinkan penyertaan sisi server berfungsi saat saya bekerja di situs saya - bahkan saat saya tidak tersambung ke internet.

\n\n

Ini dulu bekerja: 

http://example.brak\n

Tapi saya benar-benar merusak sesuatu ketika saya menambahkan domain baru.

\n\n

Saya menggunakan file host untuk menyelesaikan alamat IP host di domain ".brak" 

127.0.0.1  example.brak\n

Setiap host * .brak menjawab ping.

\n\n

Setiap kotak mendapatkan TLDnya sendiri yang memungkinkan saya mengakses setiap situs lokal saya dari kotak mana pun di apartemen saya.

\n\n

ini seharusnya berfungsi di Cygwin: 

$ /cygdrive/c/Apache22/bin/httpd -k start -n Apache2.2\n

Atau di Command Prompt: 

c:\\> \\Apache22\\bin\\httpd -k start -n Apache2.2\n

(Seseorang harus menggunakan httpd -k install untuk mengatur httpd sebagai Layanan Windows.)

\n\n

Manajer tugas menunjukkan bahwa layanan dimulai tetapi dengan sangat cepat berhenti.

\n\n

Untuk mengetahui status dari perintah sebelumnya: 

c:\\> dir\nc:\\> echo %errorlevel%\n0\n

Apa yang setara dengan apachectl configtest di Apache untuk Windows?

\n
\n\n

Menurut arus Dokumentasi Apache 2.4 untuk apachectl , configtest setara dengan httpd -t . Sebagai referensi, httpd -t memungkinkan Anda untuk:

\n\n
\n

Jalankan pengujian sintaks untuk file konfigurasi saja. Program segera keluar setelah pengujian penguraian sintaks ini dengan kode pengembalian 0 (Sintaks OK) atau kode pengembalian tidak sama dengan 0 (Kesalahan Sintaks). Jika -D DUMP_VHOSTS juga disetel, detail konfigurasi virtual host akan dicetak. Jika -D DUMP_MODULES diatur, semua modul yang dimuat akan dicetak.

\n
\n\n

Kesalahan konfigurasi untuk keduanya httpd.conf dan file virtual host juga biasanya akan ditangkap dengan manual, non-service manager start-up (httpd -k start) ex.

\n\n

\n\n

Anda harus menghentikan layanan Apache yang sedang berjalan sebelum mencoba httpd -k start (httpd -t seharusnya tidak memiliki persyaratan seperti itu).

\n\n
\n\n

Perhatikan bahwa meskipun pertanyaannya berkaitan dengan Apache 2.2, dokumentasi umum untuk apachectl configtest dan httpd -t tidak berubah antara versi 2.2. dan 2.4.

\n\n
\n\n
\n

Saya telah menginstal Apache 2.2 untuk Windows di laptop saya.

\n
\n\n

Pada tulisan ini, Apache 2.2 secara resmi sudah End-of-Life. Per 1 Januari 2018:

\n\n
\n

Proyek Server HTTP Apache telah menghentikan semua pengembangan dan tinjauan patch dari rangkaian rilis 2.2.x [...] dan tidak ada evaluasi lebih lanjut tentang risiko keamanan yang akan dipublikasikan untuk rilis 2.2.x.

\n
\n\n

Jika Anda tidak memiliki alasan kuat untuk tetap menggunakan versi 2.2 (mis. Ini untuk penggunaan pribadi dan Anda tidak memerlukan beberapa modul khusus, dll.), Anda mungkin ingin mempertimbangkan peningkatan secara serius.

\n\n

Saya akan merekomendasikan yang asli (non-Cygwin) Windows dibangun dari Apache Lounge . Mereka umumnya mengikuti rilis resmi Apache Foundation terbaru (atau tidak jauh di belakang) dan memiliki pilihan modul opsional umum yang layak.

\n\n

(Sebagai FYI kecil, peningkatan dari versi 2.2 ke 2.4 akan memerlukan beberapa perubahan sintaks kecil di file konfigurasi Anda tetapi tidak ada yang besar).
