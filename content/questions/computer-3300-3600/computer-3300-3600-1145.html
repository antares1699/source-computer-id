Jika Anda tidak ingin melakukan ini dari GUI, Anda dapat menggunakan program baris perintah cacls menggunakan opsi / T.

\n\n

(atau gunakan icacls ​​yang merupakan versi baru cacls).

\n\n

Jika Anda ingin menggunakan GUI, klik kanan folder, pilih properties, lalu klik tab security. Klik Lanjutan, lalu klik Ubah Izin. Tambahkan pengguna / grup yang ingin Anda beri izin (dan pilih izin yang ingin Anda berikan), lalu centang kotak yang bertuliskan "Ganti semua izin objek anak dengan izin yang diwariskan dari objek ini".

\n\n

MEMPERBARUI: Makan makanan anjing saya sendiri, skenario dunia nyata lainnya kemarin ketika kami perlu mengatur ulang semua izin pada folder dengan beberapa juta file di dalamnya (dan menghapus warisan):

\n\n

Ini ada di Windows Server 2008 R2, jadi Anda perlu menjalankan CMD Prompt sebagai Administrator (disarankan untuk Server 2008 R2 dan yang lebih baru). Pada prompt CMD: 

icacls E:\\Store\\*.* /grant:r \"OURDOMAIN\\Domain Admins\":F /grant OURDOMAIN\\ITAdmins:F /T /C /inheritance:r /Q\n

.

\n\n

Untuk file apa pun yang gagal, Anda mungkin perlu menjalankan TAKEOWN lalu menjalankan perintah icacls lagi (pada file yang gagal):

Anda mungkin ingin menyalurkan output ke file karena tidak ada opsi "quiet" di takeown.

Gunakan takeown /? untuk melihat opsi lain yang tersedia.

Langkah untuk mengubah izin untuk folder dan file subfoldernya.

\n

Di Linux, Anda dapat melihat berbagai folder dan file yang ada. Anda perlu mengatur izin dan kepemilikan yang sesuai untuk memastikan akses kerja yang tepat.

\n

Izin 755 diperlukan untuk mengatur direktori. (drwx.r-xr-x);

\n

Tiga jenis izin.\n• baca ulang\n• w- tulis\n• x- izin eksekusi 

            find /foldername -type d –exec chmod 755 {} \\;\n

Contoh: temukan / var / www / user –type - exec chmod 755 {};

\n

Izin 644 sangat penting untuk file. (-rw-r - r--); 

         [find (filename) -type f –exec chmod 644 {} \\;][1]\n

Contoh: find / var / www / user / -type f –exec chmod 644 {};
