Setelah meningkatkan versi Chromium dari 34 menjadi 35, kegunaan Chromium menurun drastis. Jadi saya memutuskan untuk menurunkan versi Chromium. Untungnya, versi Chromium sebelumnya masih ada di disk saya, jadi menurunkan versi (di ArchLinux) sesederhana: 

sudo pacman -U /var/cache/pacman/pkg/chromium-34.0.1847.137-1-x86_64.pkg.tar.xz\n

Setelah menjalankan perintah sebelumnya, Chromium benar-benar mulai, tetapi sayangnya, saya terus disadap oleh pesan berikut saat memulai:

\n\n
\n

Profil Anda tidak dapat digunakan karena berasal dari versi Google Chrome yang lebih baru. Beberapa fitur mungkin tidak tersedia. Harap tentukan direktori profil yang berbeda atau gunakan versi Chrome yang lebih baru.

\n
\n\n

Salah satu cara untuk menghilangkan pesan ini adalah dengan menghapus ~/.config/chromium/Default/Web Data, tetapi ini juga menyebabkan my kata kunci mesin pencari menghilang .

\n\n

Bagaimana cara menurunkan versi Chromium tanpa kehilangan data apa pun?

\n\n

Catatan: Saya menggunakan Linux, tetapi masalah ini tidak bergantung pada platform. Silakan menjawab dari perspektif sistem operasi non-Linux seperti Windows.

Catatan: Tutup Chrome dan cadangkan profil Anda sebelum mengotak-atiknya!

\n\n

Web Data adalah database SQLite, jadi saya menggunakan file program sqlite3 untuk membuka database.

\n\n
    \n

  1. Apa versi database "Data Web" saat ini? (jalankan SELECT * from meta;) 

    sqlite> SELECT * FROM meta;\nDefault Search Provider ID|34\nlast_compatible_version|55\nversion|55\nBuiltin Keyword Version|70
    2. \n

  2. Baris yang ditekankan di keluaran sebelumnya mengacu pada nomor versi database internal di Chromium. Untuk setiap versi, Anda dapat menemukan kode migrasi yang relevan di Kode sumber Chromium (autofill_table.cc) . Secara khusus, lihat file AutofillTable::MigrateToVersion metode. Karena versi database saya adalah 55, saya melihat "kasus 55" 

    case 55:\n      *update_compatible_version = true;\n      return MigrateToVersion55MergeAutofillDatesTable();

    Setelah mengetahui metode ini, saya hampir siap untuk menulis kueri SQLite untuk membalikkan migrasi.

    3. \n

  3. Menggunakan pengetahuan dari langkah sebelumnya (dan skema asli dari tabel berbeda di komputer lain), saya membuat kueri berikut (salin-tempel kueri ke sqlite3):
    \n Catatan: Kueri ini khusus untuk menurunkan Chrome 35 menjadi 34! 

    CREATE TABLE autofill_v54 (\n    name VARCHAR,\n    value VARCHAR,\n    value_lower VARCHAR,\n    pair_id INTEGER PRIMARY KEY,\n    count INTEGER DEFAULT 1);\nCREATE TABLE autofill_dates (\n    pair_id INTEGER DEFAULT 0,\n    date_created INTEGER DEFAULT 0);        \n\nINSERT INTO autofill_v54 (\n    name,\n    value,\n    value_lower,\n    count\n) SELECT name, value, value_lower, count FROM autofill;\n\nINSERT INTO autofill_dates (\n    pair_id,\n    date_created\n) SELECT pair_id, date_created\n  FROM autofill_v54 a54 JOIN autofill a USING (name, value);\n\nDROP TABLE autofill;\nALTER TABLE autofill_v54 RENAME TO autofill;\n\nCREATE INDEX autofill_name ON autofill (name);\nCREATE INDEX autofill_name_value_lower ON autofill (name, value_lower);\n
    4. \n

  4. Setelah mengembalikan migrasi, Anda dapat menurunkan versi database dengan aman: 

    UPDATE meta SET value='54' WHERE key='version' OR\n                                 key = 'last_compatible_version';\n
    5. \n

  5. Tutup database menggunakan .quit

    6. \n
\n\n

Setelah menyelesaikan semua langkah ini, saya memulai Chromium, tidak ada munculan yang mengganggu dan semua data profil masih utuh!

Saya mengonfirmasi: solusi dengan sqlite3 berfungsi dengan baik!

\n\n

Saya pengguna Ubuntu (14.04), tetapi saya mencoba metode ini bukan dengan chromium tetapi dengan google-chrome-stable.

\n\n

Saya telah menurunkan versi chrome dari 35 menjadi 34 karena tidak ada cara untuk menjalankan applet java di rilis 35. :-( 

sudo apt-get purge google-chrome-stable\nsudo rm -f /etc/apt/sources.list.d/google-chrome.list*\nsudo apt-get update\ncd ~/Downloads\nwget http://95.31.35.30/chrome/pool/main/g/google-chrome-stable/google-chrome-stable_34.0.1847.137-1_amd64.deb\n

Dengan menghapus sumber ini, google-chrome tidak akan lagi diperbarui secara otomatis.\nSaya telah menginstal rilis 34 dari nautilus dengan gdebi. Anda dapat menginstal gdebi dengan: 

sudo apt-get install gdebi\n

Masalahnya dimulai di sini ... Message = 'Profil Anda tidak dapat digunakan karena berasal dari ...'
Anda dapat menginstal sqlite3 seperti ini: 

sudo apt-get install sqlite3\n

untuk menjalankan solusi di atas (disediakan oleh Rob W), lakukan: 

cd ~/.config/google-chrome/Default\nsqlite3 Web\\ Data\n

Setelah itu cukup copy / paste langkah 1, 3, 4, 5 yang diberikan oleh Rob W.
Semoga bantuan ini. ;-)
