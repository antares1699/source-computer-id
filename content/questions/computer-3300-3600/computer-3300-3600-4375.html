Kontrol Tombol X-Mouse adalah perangkat lunak Windows yang akan menunjukkan apa yang dilakukan mouse Anda.

\n\n

Buka perangkat lunak, dan klik tombol, atau gulir roda. Daftar tarik-turun di samping setiap tindakan berkedip kuning saat Anda melakukannya - misalnya daftar tarik-turun di samping "Roda Naik" akan berkedip kuning saat Anda menggerakkan roda mouse ke atas.

\n\n

Saya baru saja mengalami masalah dengan mouse saya - dan ternyata itu adalah masalah perangkat keras ( ini adalah salah satu mouse USB Nirkabel Logitech, yang memiliki opsi 'gulir halus' di roda tombol tengah - Anda menekannya sekali, Anda memiliki 'gulir halus', tekan sekali lagi, lalu Anda memiliki gulir 'melangkah' .. . ) Ternyata, roda tengah saya macet, jadi saya harus menekannya beberapa kali sampai 'pulih' - dan saya menggunakan xev (di Ubuntu 10.04) untuk mengontrol ini.

\n\n

Untuk mengucapkan beberapa kata lagi pada mouse di bawah Ubuntu - Anda dapat memeriksa log sistem, untuk melihat apa yang dicatat saat boot, sehubungan dengan, katakanlah, perangkat USB - untuk melihat apakah perangkat tersebut dikenali sama sekali. Namun, ini dengan "log sistem" belum tentu diperbaiki - di Ubuntu 10.04, saya hanya melihat di /var/log/messages - tetapi di 11.04, string 'Logitech' muncul di file ini: 

$ grep --files-with-matches --recursive Logitech /var/log 2>/dev/null \n/var/log/spnavd.log\n/var/log/Xorg.0.log\n/var/log/Xorg.1.log\n/var/log/dmesg.0\n/var/log/Xorg.0.log.old\n/var/log/dmesg\n/var/log/kern.log.1\n/var/log/udev\n

Di bawah 11.04, saya kira salah satu caranya adalah dengan grep / mencari string 'usb' di /var/log/dmesg - atau bahkan mungkin lebih mudah; lagi untuk string 'Logitech': 

$ grep Logitech /var/log/dmesg\n[   22.323032] input: Logitech USB Receiver as /devices/pci0000:00/0000:00:1d.1/usb3/3-1/3-1:1.0/input/input7\n[   22.323529] generic-usb 0003:046D:C526.0001: input,hidraw0: USB HID v1.11 Mouse [Logitech USB Receiver] on usb-0000:00:1d.1-1/input0\n[   22.502664] input: Logitech USB Receiver as /devices/pci0000:00/0000:00:1d.1/usb3/3-1/3-1:1.1/input/input8\n[   22.509847] generic-usb 0003:046D:C526.0002: input,hiddev0,hidraw1: USB HID v1.11 Device [Logitech USB Receiver] on usb-0000:00:1d.1-1/input1\n

Apa yang Anda inginkan, adalah melihat "alamat USB" mana perangkat terhubung - di atas kita melihat "

$ grep --files-with-matches --recursive Logitech /var/log 2>/dev/null \n/var/log/spnavd.log\n/var/log/Xorg.0.log\n/var/log/Xorg.1.log\n/var/log/dmesg.0\n/var/log/Xorg.0.log.old\n/var/log/dmesg\n/var/log/kern.log.1\n/var/log/udev\n
", jadi pada dasarnya kita dapat "cat" untuk menjalankan file usbmon yang sesuai di bawah /dev (yaitu, /dev/usbmon3); ini akan memberi kami data USB "mentah" yang masuk dari perangkat; karena biner ini tidak dapat langsung dirender di terminal, kami akan menyalurkannya ke hexdump untuk diformat: 

$ sudo cat /dev/usbmon3 | hexdump -C\n00000000  80 6b 14 f1 00 00 00 00  43 01 81 02 03 00 2d 00  |.k......C.....-.|\n00000010  4c e0 75 4e 00 00 00 00  41 5a 07 00 00 00 00 00  |L.uN....AZ......|\n00000020  08 00 00 00 08 00 00 00  00 00 00 00 00 00 00 00  |................|\n00000030  00 00 ff ff 00 00 00 00  80 6b 14 f1 00 00 00 00  |.........k......|\n00000040  53 01 81 02 03 00 2d 3c  4c e0 75 4e 00 00 00 00  |S.....-<L.uN....|\n....\n

Sekarang, ketika Anda melakukan tindakan pada mouse (bergerak, tekan tombol), lebih banyak data masing-masing harus ditampilkan di terminal (akhiri proses ini pada akhirnya dengan Ctrl-C). Perhatikan Anda harus menggunakan sudo pada perintah di atas.

\n\n

Intinya adalah - jika dalam pengujian di atas, Anda mendapatkan data yang dihasilkan saat Anda menggerakkan roda gulir mouse ke atas dan ke bawah; maka setidaknya perangkat keras dan bagian driver USB dari rangkaian tersebut harus berfungsi - jika semuanya berfungsi, maka peristiwa tersebut harus disebarkan ke sistem X, dan juga dilaporkan oleh xev. ( karena dalam kasus saya, saya tidak melihat kejadian ini, saya kemudian dapat menyimpulkan masalah perangkat keras; yang, kali ini, untungnya dapat diselesaikan dengan mudah hanya dengan beberapa tekanan. Perhatikan juga - ketika semuanya bekerja, menggerakkan roda gulir juga menghasilkan data di terminal, jadi Anda tidak akan dapat menggulir terminal dengan roda gulir :) dalam hal ini, kembalikan sementara ke panah keyboard yang menggulir terminal ).
