Saya ingin membuat cadangan folder untuk dibagikan di linux nas dengan robocopy.

\n\n

Saya menggunakan commandline robocopy /mir <source> <dest>

\n\n

Setelah salinan berhasil di explorer, cap waktu sama tetapi entah bagaimana robocopy menyalin file yang sama saat berikutnya berjalan dan mengklaim file sumber lebih baru. Tampaknya menjadi masalah dengan versi seseorang di nas.

\n\n

Sekarang saya dapat menggunakan sakelar /xn untuk mencegah penyalinan file tetapi itu juga akan mencegah file disalin saat saya menjalankan pekerjaan yang sama di lain waktu.

\n\n

\n

Setelah salinan berhasil di explorer, stempel waktunya sama tetapi\n entah bagaimana Robocopy menyalin file yang sama saat berjalan dan mengklaimnya lagi\n file sumber lebih baru. Tampaknya ada masalah dengan versi seseorang\n nas.

\n
\n\n

Tambahkan sakelar /FFT dengan Robocopy: 

ROBOCOPY /MIR /FFT <source> <dest>\n

Sakelar Robocopy 

/FFT : Assume FAT File Times (2-second date/time granularity).\n

Deskripsi Lebih Baik

\n\n
\n

/ FFT menggunakan pengaturan waktu file lemak, bukan NTFS. Artinya, perincian> sedikit kurang tepat. Untuk operasi berbagi lintas jaringan\n ini tampaknya jauh lebih dapat diandalkan - jangan mengandalkan file tersebut\n pengaturan waktu agar benar-benar tepat ke detik.
