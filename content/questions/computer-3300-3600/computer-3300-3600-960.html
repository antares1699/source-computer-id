Saya mengikuti tutorial untuk mengkonfigurasi OS X Mountain Lion dan entah bagaimana setelah mengedit bash_profile saya tidak dapat menjalankan terminal dengan benar.

\n\n

Terminal mulai baik-baik saja tetapi kosong, jika menunggu cukup lama ada "[Proses selesai]" yang muncul tetapi itu saja, saya tidak dapat menjalankan perintah apa pun meskipun saya masih bisa menulis.

\n\n

Ketika saya melihat Inspektur saya melihat bahwa proses 'login' berfungsi dengan baik tetapi untuk 'bash' itu masalah lain - ID-nya terus naik dengan 'ruby'.

\n\n

Ini adalah tutorial yang saya ikuti .

\n\n

Apa yang saya tambahkan ke .bash_profile sebelum masalah muncul adalah:

Di preferensi Terminal.app Anda, cukup ubah profil Anda untuk menjalankan /bin/zsh secara default (atau shell lain yang Anda suka kecuali Bash). Menonaktifkan Jalankan di dalam cangkang .

\n\n

\n\n

Sekarang, mulai jendela terminal baru dan hapus ~/.bash_profile Anda: 

rm ~/.bash_profile\n

Hapus apa pun yang tidak Anda inginkan, atau ubah dengan vim jika Anda mau. Anda tentu saja dapat menjalankan perintah rm langsung dari startup Terminal.

\n\n

Anda bisa mulai dari sini. Anda tidak perlu mencari ~/.bashrc - Anda mungkin tidak membutuhkan file ini di OS X, karena jendela Terminal baru menjalankan shell login dan oleh karena itu sumber .bash_profile.

\n\n

Anda juga tidak perlu memiliki biner Homebrew diutamakan daripada yang sistem dengan mengubah jalur Anda. Homebrew memiliki kebijakan ketat untuk tidak menyediakan duplikat perangkat lunak OS X, jadi jika Anda menginstal - misalnya - GNU coreutils, Anda akan mendapatkan gfind, bukan find, dll.
