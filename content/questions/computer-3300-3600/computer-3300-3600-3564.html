Saya memiliki satu partisi swap, dan 3 OS Linux yang diinstal. Jadi saya hanya ingin tahu, apakah ada risiko keamanan atau potensi masalah yang dapat muncul akibat ketiga OS menggunakan partisi ini?

\n\n

Hanya satu dari mereka yang dapat di-boot pada waktu tertentu, dan masing-masing melepaskan partisi swap saat mematikan / me-reboot. Jadi jika saya, katakanlah, menutup Fedora dan melakukan booting ke Arch, mungkinkah ada masalah yang berkaitan dengan swap? Seperti data dari OS sebelumnya yang digunakan oleh OS yang saat ini di-boot? Sejauh ini saya belum mengalami masalah apa pun.

\n\n

Haruskah saya membuat partisi swap terpisah untuk setiap OS? Saya tahu bahwa mencoba berbagi file hibernasi / partisi swap dapat menimbulkan masalah, karena OS lain dapat mencoba hibernasi dari data OS lain. Tetapi saya tidak menggunakan hibernasi, karena penginstalan saya menggunakan SSD. Saya pernah mengalami masalah dengan hibernasi di masa lalu (seperti Windows yang bertindak miring).

Ya selama Anda tidak mencoba sesuatu yang aneh seperti menghibernasi satu OS, lalu mem-boot langsung ke OS lain dan tidak mengharapkan masalah. (Satu versi hibernasi akan menyimpan ram langsung ke partisi swap.).

\n\n

Yang perlu Anda lakukan adalah memastikan bahwa setiap O / S yang Anda instal memasang partisi swap yang benar.
