Dalam skrip Bash saya, saya harus menangani nama file dengan spasi. Ini adalah baris penting di dalam skrip saya: 

mp3file=\"/media/d/Music/zz_Hardcore/Sampler/Punk-O-Rama\\ Vol.5\\ \\[MP3PRO\\]/01\\ -\\ Nofx\\ -\\ Pump\\ up\\ the\\ Valium.mp3\"\necho \"Command: mp3info -x `echo $mp3file`\"\nmp3info -x `echo $mp3file`\n

Sayangnya, perintah tersebut tidak berfungsi, karena nama file dipisahkan: 

mp3info: invalid option -- '\\'\nmp3info: invalid option -- '\\'\nError opening MP3: /media/d/Music/zz_Hardcore/Sampler/Punk-O-Rama\\: No such file or directory\nError opening MP3: Vol.5\\: No such file or directory\nError opening MP3: \\[MP3PRO\\]/01\\: No such file or directory\nError opening MP3: Nofx\\: No such file or directory\nError opening MP3: Pump\\: No such file or directory\nError opening MP3: up\\: No such file or directory\nError opening MP3: the\\: No such file or directory\nError opening MP3: Valium.mp3: No such file or directory\n

Saya juga mencoba menambahkan IFS khusus saat saya membaca di beberapa forum: 

SAVEIFS=$IFS\nIFS=$(echo -en \"\\n\\b\")\n# Script like above\nIFS=$SAVEIFS\n

Tapi dengan cara ini, saya mendapatkan kesalahannya 

Error opening MP3: /media/d/Music/zz_Hardcore/Sampler/Punk-O-Rama\\ Vol.5\\ \\[MP3PRO\\]/01\\ -\\ Nofx\\ -\\ Pump\\ up\\ the\\ Valium.mp3: No such file or directory\n

Saya sudah mencoba cukup lama sekarang tetapi saya tidak bisa membuat skrip saya berfungsi. Yang aneh adalah jika saya menjalankan perintah yang sama yang harus dibuat skrip saya secara manual (menggemakannya di dalam skrip saya) di Shell, itu benar-benar berfungsi. Tapi tidak di dalam skripku. Ada petunjuk?

Bungkus nama variabel dalam tanda kutip: 

mp3file=\"/media/music/Punk-O-Rama Vol.5 \\[MP3PRO\\]/01 - Nofx - Pump up the Valium.mp3\"\necho \"Command: mp3info -x \\\"$mp3file\\\"\"\nmp3info -x \"$mp3file\"\n

Saya juga menghapus perintah echo yang berlebihan dan melarikan diri yang tidak perlu.
