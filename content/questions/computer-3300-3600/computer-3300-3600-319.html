Anda dapat menjalankan Linux di Mac seperti yang dijelaskan di lebar variasi dari situs dalam lingkungan boot ganda / tunggal. Preferensi saya menjalankan tamu Linux di VMware Fusion atau VirtualBox, untuk tujuan pengujian perangkat lunak.

Namun, dan saya tidak ingin mencoba dan membujuk Anda keluar dari ini terlalu banyak - Saya juga tidak ingin memicu pertarungan OS, saya tidak akan repot-repot.

\n\n

Saya pengguna Linux lama (~> 15 tahun), dan mempertimbangkan untuk melakukan ini ketika saya mendapatkan Macbook Pro dari kantor.

\n\n

Namun, Linux sebagai platform OS workstation biasanya berarti gangguan perangkat keras. Wi-Fi dan suara sering kali berarti saat-saat produktivitas yang hilang mencoba membuat segala sesuatunya berfungsi dengan baik. Belum lagi saya memiliki iPhone, dan menyelaraskannya dengan OS yang tidak menjalankan iTunes secara asli berarti sakit kepala menunggu untuk terjadi. Ini tidak mempertimbangkan semua gangguan perangkat lunak lain yang datang dengan menjalankan Linux.

\n\n

Mac OS X, tentu saja, bekerja sempurna dengan perangkat kerasnya, karena memang dirancang untuk itu. Itu tentu saja keuntungan utama untuk membeli Mac, adalah OS terintegrasi dengan erat. Juga, karena Mac OS X adalah file Unix bersertifikat , itu berarti dapat (dan memang) menjalankan semua cara perangkat lunak Unix (dan Linux). Dengan MacPorts proyek, kekayaan perangkat lunak berjarak port install.

\n\n

OS X menyediakan semua hal hebat yang saya suka tentang menjalankan workstation Linux (utilitas mirip Unix / Unix, baris perintah dan kemampuan skrip yang kuat, dll), dan tidak ada hal yang membuat saya frustrasi selama bertahun-tahun.
