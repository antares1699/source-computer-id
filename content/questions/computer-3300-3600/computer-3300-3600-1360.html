Saya menyadari ini adalah pertanyaan yang sangat lama, tetapi solusinya sangat mudah. Anda tidak perlu menginstal win98 penuh untuk menghapus data. Anda hanya perlu mendapatkan image disk win98 EBD. (Disk Boot Darurat)

\n\n

Anda juga memerlukan beberapa jenis penyiapan mesin virtual, seperti VMware, Qemu, atau Bochs.

\n\n

Pertama-- Di perangkat lunak mesin virtual pilihan Anda, buat dua gambar disk baru. Yang pertama harus dapat menampung konten yang tidak dikompresi dari drivepace3 volume terkompresi Anda. Yang kedua harus berukuran 2mb lebih besar dari file .000. Yang pertama mungkin harus diformat sebagai FAT32, dan yang kedua HARUS diformat sebagai FAT16.

\n\n

Baik. Sekarang salin file .000 ke dalam gambar disk FAT16 dengan sesuatu seperti winimage.

\n\n

Atur mesin virtual untuk mem-boot dari EBD, dan untuk memasang dua file image disk. Gunakan volume FAT16 sebagai volume primer dan FAT32 sebagai volume sekunder.

\n\n

Boot mesin virtual di EBD. Ini akan menanyakan apakah Anda menginginkan dukungan CDROM. Terserah kamu. Ketika selesai, itu akan meninggalkan Anda di prompt DOS.

\n\n

Pasang file .000 menggunakan fitur "tidak berdokumen" dari scandisk. 

A:\\>Scandisk /mount C:\\drvspace.000\n

Image disk yang tidak terkompresi yang menahan drvspace.000 sekarang menjadi drive host H yang tersembunyi, dan drive C sekarang menjadi wadah untuk bekerja dengan file drvspace.000. 

A:\\>C:\n\nC:\\>DIR\n

Lihat semua file Anda?\nJika demikian, Bagus. Jika tidak, coba temukan EBD yang belum dilucuti dari DRVSPACE.BIN. (Beberapa orang menanggalkannya, karena memakan sebagian besar memori konvensional, dan bersifat opsional. Dalam hal ini, KAMI MEMBUTUHKANNYA.)

\n\n

Gunakan XCOPY untuk menyalin file dari drive C ke drive D yang tidak terkompresi. 

C:\\>A:\\xcopy /e C:\\*.* D:\\\n

Tunggu hingga proses penyalinan file selesai.

\n\n

Matikan mesin virtual. Buka gambar disk FAT32 dengan Winimage, atau apa pun. Keluarkan file Anda yang sekarang tidak terkompresi dan letakkan di tempat yang aman.

Untuk setiap disk yang dikompresi

\n\n

Penyimpanan VM akan disiapkan seperti ini:

\n\n
    \n
  • IDE Pri Master: win98.vdi (boot disk)
    • \n
  • IDE Pri Slave: enc_001.vdi (gambar ruang drive disalin dari CD iso)
    • \n
  • IDE Sec Master: 001.iso (CD iso containg gambar zip drive terkompresi)
    • \n
  • IDE Sec Slave: unenc_001.vhd (tujuan untuk gambar zip yang tidak dikompresi)
    • \n
\n\n

ISO digunakan untuk mendapatkan konten disk drivespace ke enc_001.vdi. Disk vdi dan vhd dibuat per disk untuk didekripsi.
