Saya memiliki folder yang dibuat oleh MSBuild, app.publish. Seringkali, skrip build saya gagal menghapusnya, mengeluh bahwa itu tidak memiliki izin yang diperlukan. Hari ini saya menguji ini di luar skrip build, dan ternyata saya bahkan tidak dapat membukanya.

\n\n

Jika saya mencoba membukanya, saya mendapatkan pesan ini:

\n\n

\n\n

"Lokasi tidak tersedia. Akses ditolak."

\n\n

Jadi saya mencoba mengubah izin, tetapi bahkan tidak dapat melihat siapa pemiliknya, bahkan sebagai administrator:

\n\n

\n\n

Jadi saya pikir mungkin ada masalah dengan Explorer, jadi saya mencoba dari PowerShell. Sayangnya, akses juga ditolak di sana. Saya mencoba menggunakan Get-ACE -Path app.publish, tetapi itu juga menolak untuk bekerja: "Akses ditolak." Saya juga mencoba icacls app.publish /grant ijhbvhbv:f, tetapi itu juga gagal dan berkata "Akses ditolak." Saya mencoba kedua perintah ini sebagai pengguna biasa dan sebagai administrator.

\n\n

Akhirnya, saya mencoba untuk menghapus direktori induk, tetapi diberitahu bahwa saya harus "memberikan izin administrator untuk menghapus folder ini". Ketika saya melakukannya, saya mendapat pesan kesalahan yang aneh: "Anda memerlukan izin dari FWEE-ERIC-WIN8 \\ ijhbvhbv untuk membuat perubahan pada folder ini". "FWEE-ERIC-WIN8 \\ ijhbvhbv" adalah akun saya sendiri. Jadi tampaknya saya memang pemilik folder tersebut, tetapi hanya ketika saya mencoba berinteraksi dengannya sebagai administrator.

\n\n

Pertanyaan lain tentang ini di SU berbicara tentang UAC sebagai potensi sumber masalah, tetapi masalahnya tetap sama, apa pun UAC yang disetel.

Setelah menghabiskan waktu terlalu lama untuk masalah serupa, saya menemukan jawabannya.

\n\n

Ini terjadi pada folder yang:

\n\n
    \n
  • Tidak punya pemilik yang sah
    • \n
  • Terkunci karena beberapa aplikasi masih menggunakannya.
    • \n
  • Jika tidak, telah dihapus.
    • \n
\n\n

Folder berada dalam status "zombi" saat dihapus, tetapi masih digunakan oleh beberapa proses. Anda perlu menemukan proses yang menggunakannya dan menutupnya, atau, jika Anda tidak tahu yang mana, reboot mesin.

\n\n

Anda akan ditinggalkan dengan foldernya hilang. Jika seluruh subpohon telah dihapus, tetapi hanya satu folder di dalamnya yang digunakan, Anda mungkin perlu kembali dan membersihkan subpohon secara manual. Ini harus bekerja dengan hak admin.

Beberapa kemungkinan untuk menyelesaikan file terkunci:

\n\n
    \n

  1. Sistem atau aplikasi menguncinya; gunakan alat seperti CrystalRich LockHunter untuk mencari tahu apa, dan jika Anda ingin, buka kuncinya. Terkadang komputer perlu dihidupkan ulang untuk melepaskan semua kunci, terutama jika objek tersebut digunakan oleh situs web.

    2. \n

  2. Mengatur izin pada folder agak rumit; ada beberapa langkah. Lihat Cara Memberikan Izin Penuh ... .

    3. \n
\n\n
\n\n

Beberapa kemungkinan lain:

\n\n
    \n

  1. Beberapa "folder" sebenarnya adalah tautan simbolis, seperti C: \\ ProgramData \\ Data Aplikasi ; ini tidak dapat diakses. Namun, karena Anda dapat mengakses folder tersebut sebelumnya, sepertinya tidak mungkin.

    2. \n

  2. MS belakangan ini mulai menggunakan file TrustedInstaller akun, jadi bahkan Admin tidak dapat menyentuh beberapa file dan folder (meskipun dari deskripsi Anda itu mungkin bukan penyebab masalah).

    3. \n
