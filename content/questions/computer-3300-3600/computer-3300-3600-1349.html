Secara keseluruhan, pengguna "normal" seharusnya tidak diizinkan untuk membuat file atau folder di root sistem file. Membiarkannya mengarah ke segala macam masalah keamanan.

\n\n

Jika Anda memiliki sudo yang dikonfigurasi dengan benar, inilah cara yang tepat untuk membuat direktori itu: 

sudo mkdir u01\n

Jika sudo tidak berfungsi karena beberapa alasan (atau bahkan tidak diinstal - tidak mungkin, tetapi mungkin), Anda dapat menggunakan su dengan parameter perintah -c, seperti ini: 

su -c \"mkdir u01\"\n

(perhatikan bahwa perintah harus dikutip agar dapat diteruskan dengan benar ke su).

\n\n

Perbedaan penting antara menggunakan sudo vs su: ketika diminta kata sandi, sudo memerlukan kata sandi akun pengguna Anda, tetapi su membutuhkan kata sandi akun root. Pada banyak distro Linux (terutama keluarga Ubuntu / Debian) kata sandi akun root bahkan mungkin tidak pernah ditetapkan, jadi su mungkin gagal (kecuali Anda menetapkan kata sandi ke akun root sebelumnya).
