Saya telah menetapkan hak atas semua data di mac HD untuk 'sistem' pengguna menjadi hanya-baca. Saya juga melakukannya untuk 'roda' dan 'barang' grup. Hanya satu akun pengguna dengan nama 'mac01' yang memiliki semua hak atas data di mac HD saya.

\n
\n\n

Jika Anda mengacaukan izin OS X, biasanya Anda harus menjalankan Disk Utility di mode pemulihan dan lari Perbaiki Izin Disk dan Perbaiki Disk atau Pertolongan pertama (di Kapten ) untuk memperbaiki kesalahan pada HDD dan akhirnya memperbaiki boot loader. Dalam kebanyakan kasus, itu harus berhasil.

\n\n

Untuk memperbaikinya secara manual, Anda harus menjalankan mode tunggal dan coba perbaiki izinnya folder sistem secara manual, misalnya: 

chown -R root /System /Library\nchmod -R u+rwX,go+rX,go-w /System /Library\n

lalu ketik exit untuk melanjutkan proses startup dan mencatat kesalahan yang berarti.

\n\n

Jika ini tidak membantu, berikut beberapa hal yang perlu dipertimbangkan:

\n\n
    \n
  • lepaskan semua perangkat eksternal yang terhubung (jika ada),
    • \n
  • coba jalankan Mac Anda Mode aman ,
    • \n
  • periksa pembuangan inti baru /cores (ini bisa menunjukkan beberapa masalah dengan komponen tertentu),
    • \n
  • periksa apa pun yang mencurigakan di log (tail /var/log/*.log, terutama system.log),
    • \n
  • setel ulang Anda SMC , NVRAM & KERETA BAYI ,
    • \n
  • membongkar ekstensi kernel pihak ketiga (dalam mode tunggal, mungkin Anda perlu memindahkannya dari /System/Library/Extensions, tetapi jika Anda tidak menginstal apa pun (seperti vboxdrv), lebih baik biarkan saja, karena mungkin akan bertambah buruk),
    • \n

  • dalam mode tunggal, verifikasi variabel NVRAM Anda sudo nvram -p dan pertimbangkan untuk menjalankan kernel Anda dalam mode yang berbeda, misalnya:

    \n\n
      \n
    • mode debugging: sudo nvram boot-args=\"debug=0x144\"
      • \n
    \n\n

    Untuk opsi lainnya, periksa: Opsi argumen boot di OS X .\nSaat berhasil, jangan lupa untuk menonaktifkannya.

    • \n

  • pertimbangkan untuk menggunakan alat pemulihan pihak ketiga (mis. DVD pemulihan dari DiskWarrior ),

    • \n
  • pertimbangkan untuk menginstal ulang OS X Anda.
    • \n
