Itu halaman manual diucapkan secara ambigu: 

-f file    Log the specified file.\n

Ada satu contoh di halaman manual tanpa penjelasan lebih lanjut: 

logger -p local0.notice -t HOSTIDM -f /dev/idmc\n

POSIX tidak membantu:

\n\n
\n

Utilitas pencatat menyimpan pesan, dengan cara dan format yang tidak ditentukan, berisi operan string yang disediakan oleh pengguna. Pesan tersebut diharapkan untuk dievaluasi kemudian oleh personel yang melakukan tugas administrasi sistem.

\n
\n\n

Saya berharap untuk mencatat:

\n\n
    \n
  • untuk berkas
    • \n
  • sesuatu tentang berkas
    • \n
  • atau sesuatu dari berkas
    • \n
\n\n

Namun, jika saya melakukan hal berikut: 

$ echo \"contents\" > testfile\n$ logger -f ./testfile \"test message\"\n$ cat testfile\ncontents\n$ tail /var/log/messages\nAug  4 10:00:00 hostname logger: test message\n

Saya tidak ada hubungannya dengan "testfile" atau isinya atau isinya tidak berubah. Jika testfile tidak ada sebelum saya mengeluarkan perintah logger saya mendapatkan pesan kesalahan ini:

\n\n
\n

logger: ./testfile: Tidak ada file atau direktori seperti itu.

\n
\n\n

Apa yang harus dilakukan logger -f?

Itu Dokumentasi GNU sedikit lebih jelas dan memberikan petunjuk lebih lanjut\npengujian:

\n\n
\n

-f file
--file = file
Catat konten dari file yang ditentukan. Jika file adalah '-' input standar\n diasumsikan.

\n
\n\n

Ternyata menentukan pesan menimpa spesifikasi file.

\n\n

Jadi ini berhasil: 

logger -f testfile\n

dan mencatat isi file ke logfile. Jika file tersebut terdiri dari\nbeberapa baris, setiap baris menjadi entri terpisah di log.
