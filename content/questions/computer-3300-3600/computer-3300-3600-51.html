Internet tidak hanya menggunakan satu protokol. Ia bahkan tidak hanya menggunakan satu protokol pada satu waktu: ia benar-benar menggunakan beberapa sekaligus, yang menumpuk satu sama lain untuk melakukan segala macam hal yang berbeda. Jika kita menyederhanakan sedikit, Anda bisa mengatakan bahwa itu menggunakan empat.

\n\n
    \n
  • Link Layer : Protokol yang memungkinkan Anda menekan sinyal ke kabel (atau gelombang radio, atau kilatan cahaya, atau apa pun) ke komputer lain di ujung lain. Contohnya termasuk PPP, WiFi, dan Ethernet.
    • \n
  • Lapisan Jaringan : Protokol yang memungkinkan Anda mendorong sinyal melalui rangkaian mesin, sehingga Anda bisa mendapatkan data antar mesin yang tidak terhubung secara langsung. Di sinilah IP dan IPv6 hidup.
    • \n
  • Lapisan Transportasi : Protokol yang memungkinkan Anda memahami sinyal tersebut. Beberapa, seperti TCP, membuat "koneksi virtual" antara dua mesin, seolah-olah ada kabel langsung di antara keduanya. Lainnya, seperti UDP, hanya mengeluarkan sedikit data dari satu mesin ke mesin lainnya. Protokol yang berbeda memiliki kekuatan dan kelemahan yang berbeda, itulah sebabnya mengapa ada begitu banyak.
    • \n
  • Lapisan Aplikasi : Inilah yang biasanya kami anggap sebagai "protokol". Mereka dikhususkan untuk jenis data tertentu, dimaksudkan untuk tujuan tertentu. Beberapa contoh termasuk FTP, HTTP, dan BitTorrent, yang semuanya mentransfer file.
    • \n
\n\n

Protokol transfer file yang saya sebutkan biasanya ditumpuk di atas TCP (yang dengan sendirinya ditumpuk di atas IP), di situlah kami sampai pada pertanyaan khusus Anda. TCP mencoba, sebaik mungkin, untuk bekerja seperti kabel langsung di antara mesin: ketika server mengirim paket, dapat dipastikan klien mendapatkannya, dan dapat dipastikan klien mendapatkan paketnya dalam urutan yang sama server mengirim mereka. Bagian dari cara melakukannya adalah bahwa setiap paket yang dikirim server haruslah diakui oleh klien: ia mengirimkan sinyal kecil kembali yang mengatakan "Oke, saya menerima paket yang Anda kirim; saya siap untuk yang berikutnya." Jika server tidak mendapatkan pengakuan itu, itu terus mengirim paket sampai itu (atau memutuskan bahwa ini tidak akan pernah berhasil dan menyerah).

\n\n

Ini adalah kunci untuk menjawab pertanyaan Anda. Server tidak dapat mengirim Paket 2 sampai mengetahui bahwa Paket 1 berhasil, yang tidak dapat terjadi sampai Paket 1 diakui, dan bahwa tidak bisa terjadi sampai Paket 1 benar-benar selesai. Server di tengah rantai tidak perlu menyimpan data apa pun (tidak lebih dari satu paket pada satu waktu) karena pada saat mesin bahkan melihat Paket 2, ia tahu bahwa ia tidak membutuhkan Paket 1 lagi .

\n\n

Satu poin terakhir: secara teknis, ini hanya berarti Internet tidak harus cache data seperti yang Anda bicarakan. Jika seseorang benar-benar ingin menyimpan semua data ini, mereka bisa; tidak ada dalam protokol yang benar-benar menghentikannya terjadi. Tetapi Internet tidak perlu cache ini untuk bekerja.
