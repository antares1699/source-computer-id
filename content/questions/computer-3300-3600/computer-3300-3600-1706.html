Ini sulit dikodekan untuk alasan yang baik - jika Anda memiliki pegangan terbuka pada file yang sebenarnya dan kemudian mencoba untuk menyimpan, menimpa file itu, maka penyimpanan gagal karena beberapa alasan (terutama pada jaringan fileshare, tetapi bahkan hanya masalah lokal seperti kegagalan daya) , Anda sekarang akan memiliki setengah file yang tidak berguna.\nMetode yang digunakan Word di sini saat menyimpan adalah dengan menyimpan file sementara, pastikan disimpan dengan benar, lalu hapus yang asli dan 'tukar' file sementara, tentu saja semuanya secara transparan kepada pengguna.

\n\n

File "bernama aneh" yang Anda lihat adalah file meta yang memungkinkan pengguna mendapatkan pesan penguncian yang berguna - "dokumen yang Anda inginkan sedang diedit oleh Barry" - nama pengguna diambil dari file temp tersebut (itulah sebabnya ia menunjukkan nama pengguna seperti yang disimpan di Office, bukan nama akun masuk mereka).

\n\n

File AutoRecover tidak disimpan di folder yang sama dengan dokumen sebenarnya, tetapi di folder lokal profil pengguna.
