Jawaban singkat:

\n\n

Sangat tidak mungkin hal ini menimbulkan ancaman bagi mesin Anda.

\n\n

Jawaban panjang:

\n\n

Kecuali jika Anda memiliki program yang berjalan di mesin Anda yang menerima paket [biasanya disebut a daemon ], sistem tidak akan terpengaruh. Sebaiknya lakukan pemindaian port UDP dengan program seperti Nmap untuk mengetahui apakah port tersebut terbuka.

\n\n

Karena ini adalah UDP [dikenal dengan bahasa sehari-hari sebagai "Protokol Data Tidak Penting"], yang mengharuskan program di sisi penerima memulai koneksi keluar agar pengirim memiliki ide APA PUN jika berhasil lewat, ini mungkin bukan probe mencoba. Kebanyakan pemindai jaringan menggunakan paket TCP SYN untuk memindai apakah port terbuka atau tidak. Kemungkinannya adalah ini hanya masalah 91.143.232.133 hanya mengirim paketnya ke host yang salah. Mungkin host jarak jauh hanya memuntahkan paket UDP ke port 2703 dari setiap host di Internet.

\n\n

Jika Anda benar-benar khawatir, Anda dapat mengatur firewall untuk memblokir semua paket dari host itu, atau menutup port sepenuhnya jika Anda tidak menggunakannya.

\n\n

Perangkat lunak firewall Anda membuat Anda takut jika tidak perlu. Pertama: "mode siluman" sama sekali tidak tersembunyi. Ini hanya berarti bahwa mesin Anda belum mengirim respons ICMP / IP "port unreachable" ke datagram yang masuk. Kedua: UDP / IP tidak memiliki koneksi, tidak seperti TCP / IP, jadi tidak ada "percobaan koneksi" sama sekali.

\n\n

Port UDP / IP 2703 digunakan oleh Server Manajemen Sistem Microsoft untuk fungsi "obrolan jarak jauh" nya . Seseorang di alamat IP itu menggunakan SMS untuk mencoba melakukan Obrolan Jarak Jauh dengan Anda, atau siapa pun yang menggunakan alamat IP Anda sebelum Anda. Ini mungkin tidak berbahaya, seseorang yang tidak tahu bahwa siapa pun yang memiliki alamat IP Anda sebelumnya tidak lagi menggunakan alamat itu, atau bisa juga berbahaya, seseorang yang ingin melihat apakah Anda telah menginstal Klien SMS dengan tidak aman sehingga xe dapat mencoba menggunakan fungsi kendali jarak jauhnya untuk mengambil alih mesin Anda.

\n\n

Tetapi karena firewall Anda hanya menjatuhkan lalu lintas UDP / IP masuk ke lantai, bahkan tanpa mengirimkan respons ICMP / IP, dalam kedua kasus tidak ada alasan untuk bertindak di pihak Anda.
