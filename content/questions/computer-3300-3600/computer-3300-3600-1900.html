msdt.exe /cab \"%~dp0WindowsUpdateDiagnostic.diagcab\" -af \"%~dp0stillneeded.xml\"\n

Anda dapat menggunakan Get-TroubleshootingPack ( https://msdn.microsoft.com/en-us/library/dd323716(v=vs.85).aspx ) cmdlet di PowerShell untuk membuat file jawaban: 

Get-TroubleshootingPack C:\\Windows\\Diagnostics\\system\\WindowsUpdate -AnswerFile c:\\WUDAnswers.xml\n

(Perhatikan bahwa ini menggunakan jalur ke pemecah masalah yang ada di% systemroot% (biasanya C: \\ Windows), bukan file .diagcab. Saya belum dapat membuat file jawaban menggunakan file .diagcab).

\n\n

Ini seharusnya hanya menanyakan satu pertanyaan, dan Anda mengetik "1 [Enter]" untuk menjawab "Terapkan Perbaikan." Kemudian Anda dapat membuat skrip PowerShell (.ps1) hanya dengan baris ini di dalamnya: 

Get-TroubleshootingPack -Path C:\\Windows\\diagnostics\\system\\WindowsUpdate | Invoke-TroubleshootingPack -AnswerFile c:\\WUDAnswers.xml -Unattended -Result c:\\WUDResult\n

Dan Anda dapat menjalankan skrip PowerShell itu dengan perintah seperti ini: 

powershell -ExecutionPolicy Bypass -file c:\\RunWindowsUpdateDiagnostics.ps1\n

... yang seharusnya membuat 3 atau lebih file di C: \\ WUDResult (ubah jalur yang Anda inginkan).

Solusi yang disarankan secara teoritis benar tetapi tidak akan berfungsi dengan WindowsUpdateDiagnostic.diagcab karena yang satu ini menyertakan dua Paket WTP dalam satu file kabinet diagcab.

\n\n
    \n
  1. Oleh karena itu silahkan perluas file diagcab menggunakan expand.exe, 7Zip atau sejenisnya ke c: \\ wud \\
    2. \n
  2. Kemudian jalankan Powershell dengan hak istimewa yang ditinggikan (klik kanan / jalankan sebagai administrator)
    3. \n
  3. ketik "Import-Module TroubleshootingPack"
    4. \n
  4. kemudian lanjutkan menggunakan solusi yang disebutkan sebelumnya yaitu jalankan "Get-TroubleshootingPack C: \\ wud \\ BitsDiagnostic -AnswerFile c: \\ wud \\ answer_bits.xml"
    5. \n
  5. dan "Dapatkan-TroubleshootingPack C: \\ wud \\ windowsupdatediagnostic -AnswerFile c: \\ wud \\ answer_wud.xml"
    6. \n
\n\n

Sekarang Anda memiliki dua file jawaban dan direktori terkait c: \\ wud yang dapat digunakan dan dipanggil menggunakan 

Get-TroubleshootingPack -Path C:\\wud\\windowsupdatediagnostic | Invoke-TroubleshootingPack -AnswerFile c:\\wud\\answer_wud.xml -Unattended -Result c:\\wud\\log_wud\n

dan 

Get-TroubleshootingPack -Path C:\\wud\\BitsDiagnostic | Invoke-TroubleshootingPack -AnswerFile c:\\wud\\answer_bits.xml -Unattended -Result c:\\wud\\log_bits\n

Saya tidak tahu apakah seseorang dapat membuatnya menggunakan "multi-diagcab" dengan dua file jawaban dan msdt.exe ... Agaknya tidak.
