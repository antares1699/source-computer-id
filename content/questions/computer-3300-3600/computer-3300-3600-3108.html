Oke, ini penyiapannya. Saya berada di luar jaringan lokal dan terhubung ke jaringan rumah saya menggunakan iSSH (di iPad), sejauh ini baik-baik saja, saya biasa mengarahkan pusat media rumah saya ke localhost dengan membuat terowongan. Jadi saya bisa memasukkan localhost:8080 di bilah alamat saya dan menelusuri pusat media saya di rumah, itu adalah: 

local port : 8080\ndestination host : localhost\ndestination port 8080\n

Itu bekerja dengan baik.

\n\n

Sekarang saya ingin melangkah lebih jauh dan memberi tahu mediacenter yang saya sambungkan, untuk mengalihkan 192.168.1.112:8800, itulah mesin yang ingin saya akses dari mediacenter dan pada dasarnya menggunakan mediacenter sebagai gateway. Saya berasumsi bahwa saya harus menggunakan ssh -L tetapi saya tidak terlalu yakin apakah itu cara yang tepat untuk melakukannya.

\n\n

Bagaimana cara membuat terowongan ini?

Saya menggunakan huruf tebal untuk membedakan antara porta lokal yang Anda sambungkan di iPad (dicetak tebal) dan port jarak jauh yang Anda akses di pusat media (bukan cetak tebal).

\n\n

Sepertinya inilah yang selama ini Anda lakukan:

\n\n

ssh -L 8080 : localhost: 8080 namapengguna@YourMediaCenterPublicAddress.com

\n\n

The "-L 8080 : "berarti Anda akan mengambil sumber daya jarak jauh dan mengikatnya ke port lokal (dalam hal ini 8080 ) yang berarti Anda dapat terhubung ke sumber daya jarak jauh ini dengan mengetik di localhost: 8080 di iPad Anda.\nBagian selanjutnya "localhost: 8080" (dari perspektif pusat media) adalah nama host dan port sumber daya yang Anda ikat ke porta 8080 di iPad Anda. Anda dapat mengubahnya ke nama host atau alamat ip dan port mana pun yang dapat dilihat oleh pusat media Anda di jaringannya.

\n\n

Sekarang Anda ingin terhubung ke 192.168.1.112: 8800 , akan terlihat seperti ini:

\n\n

ssh -L [aLocalPort]: 192.168.1.112: 8800 username@YourMediaCenterPublicAddress.com

\n\n

maka Anda dapat terhubung ke 192.168.1.112: 8800 dengan mengetik di localhost: [aLocalPort] ke dalam program apa pun yang ingin Anda gunakan untuk menghubungkannya di iPad Anda.\n(di mana [aLocalPort] adalah beberapa port lokal yang tidak digunakan seperti 9000. Anda dapat menggunakan port yang sama yang digunakan di tujuan ( 8800 ) untuk kenyamanan jika Anda suka)

\n

Mesin, tempat perintah terowongan ssh diketik dipanggil »Host Anda« .

\n

\n
\n

\n

pengantar

\n
    \n

  1. lokal: -L Specifies that the given port on the local (client) host is to be forwarded to the given host and port on the remote side.

    \n

    ssh -L sourcePort:forwardToHost:onPort connectToHost artinya: terhubung dengan ssh ke -L Specifies that the given port on the local (client) host is to be forwarded to the given host and port on the remote side., dan meneruskan semua upaya koneksi ke lokal sourcePort ke port onPort pada mesin bernama forwardToHost, yang dapat dihubungi dari mesin connectToHost.

    \n
    2. \n

  2. jarak jauh: -R Specifies that the given port on the remote (server) host is to be forwarded to the given host and port on the local side.

    \n

    ssh -R sourcePort:forwardToHost:onPort connectToHost artinya: hubungkan dengan ssh ke connectToHost, dan teruskan semua upaya koneksi ke terpencil sourcePort ke port onPort pada mesin bernama forwardToHost, yang dapat dihubungi dari mesin lokal Anda.

    \n
    3. \n
\n

Teladan Anda

\n

Gambar kedua mewakili terowongan ini. remotehost adalah pusat perantara Anda. farawayhost adalah mesin 192.168.1.112 Anda dengan porta merah muda yang diinginkan (dalam porta gambar 456, tetapi dalam porta kasus Anda 8800). Jadi perintahnya harus terlihat seperti ini: 

ssh -L 123:192.168.1.112:8800 mediacenter\n

Tentu saja Anda dapat mengubah nomor port hijau 123 menjadi apa pun yang Anda inginkan dan kemudian mengaksesnya melalui 

localhost:123\n

di mana lagi 123 adalah nomor port sewenang-wenang (dipilih oleh perintah tunneling Anda) dari port hijau.
