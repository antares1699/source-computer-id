Manajer Layanan Terminal dapat memberi Anda informasi ini.

\n\n

Di bawah alat administratif -> Layanan Terminal -> Manajer Layanan Terminal

\n\n

Kemudian pilih host di panel kiri dan di panel tengah pilih tab "Sesi". Akan ada daftar dari semua sesi aktif dan ada kolom "IdleTime".

\n\n

Outputnya diinterpretasikan sebagai "Hari + Jam: menit". Jadi misalnya, jika outputnya adalah "4 + 11: 23", pengguna telah menganggur selama 4 hari, 11 jam, dan 23 menit.

\n\n

Perlu juga dicatat bahwa Anda dapat log off sesi ini dengan mengklik kanan dan memutuskan mereka. Meskipun mungkin menyenangkan untuk menelepon mereka terlebih dahulu: P

\n\n

Sejak Windows 2008 perintah quser bekerja.

\n\n
\n

Menampilkan informasi tentang sesi pengguna di server Host Sesi Desktop Jarak Jauh (RD Session Host).

\n
\n\n

Faktanya, skrip PowerShell yang ditautkan ke jawaban hanya menjalankan perintah ini. OP meminta cara cepat dan mudah, ini pasti lebih cepat daripada mengunduh dan menjalankan skrip PS.

\n\n

Contoh dari Windows Server 2012 R2: 

\nC:\\Users\\admin.tedd>quser\n USERNAME              SESSIONNAME        ID  STATE   IDLE TIME  LOGON TIME\n admin.xxxxxxxx                            2  Disc     15+15:12  7/20/2017 1:19 PM\n admin.xxxxx           rdp-tcp#54          3  Active       1:39  7/21/2017 5:35 AM\n xxxxxxxx                                  4  Disc      6+04:10  7/21/2017 9:25 AM\n>admin.tedd            rdp-tcp#86          5  Active          .  8/9/2017 4:40 PM\n

Angka di depan + adalah hari, jadi 15 + 15:12 adalah 15 hari, 15 jam dan 12 menit.

\n\n

Catatan! Dokumentasi mengatakan ini sama dengan menjalankan query user. Perintah query tidak ada di server saya. Tidak yakin apakah ini adalah pemasangan terpisah atau bagian dari pemasangan fitur.
