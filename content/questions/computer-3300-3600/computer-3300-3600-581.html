Ini tentang registri pada Windows 7 Home Premium 64-bit (komputer teman saya). Saya akan mengatur sistem untuk menghapus pagefile saat mematikan dengan mengatur nilai REG_DWORD ClearPageFileAtShutdown pada HKLM\\SYSTEM\\CurrentControlSet\\Control\\Session Manager\\Memory Management menjadi "1".

\n\n

Namun pertanyaan saya adalah tentang tanda tanya pada nilai ExistingPageFiles dan PagingFiles di lokasi registri ini.

Jalur tanda tanya ganda adalah Nama objek NT Object Manager . Object Manager adalah sistem yang mengatur banyak jenis sumber daya sistem (misalnya perangkat, bagian memori, stasiun jendela, peristiwa) ke dalam satu struktur seperti pohon. Anda dapat menjelajahi pohon itu dengan WinObj alat.

\n\n

Isi direktori semu \\??\\ dapat dilihat di bagian GLOBAL?? dari WinObj. Ada subpohon lain, seperti Device. Faktanya, setiap item di \\??\\ adalah tautan simbolis - pintasan atau alias - ke objek di tempat lain. \\??\\C: adalah (di sistem saya) tautan ke \\Device\\HarddiskVolume4, dan jalur lainnya yang ditampilkan di tangkapan layar Anda adalah jalur di bawah akar volume itu. Setiap huruf drive muncul di sana sebagai tautan simbolis ke volume tempatnya berada. Anda mungkin menemukan jenis perangkat lain yang sudah dikenal, seperti CON dan NUL.

\n\n

Catatan: Anda tidak dapat menggunakan jenis jalur ini untuk mengakses file atau direktori di sebagian besar aplikasi. Hanya komponen sistem tingkat rendah tertentu yang dirancang untuk bekerja dengan jalur Manajer Objek.

\n\n

Itu menjelaskan tangkapan layar Anda ExistingPageFiles, tetapi bukan PagingFiles. PagingFiles berisi pengaturan memori virtual Anda dengan jalur ke file paging yang dinyatakan sebagai jalur normal. Jika sistem Anda diatur untuk mengelola semua pengaturan memori virtual dengan sendirinya, entri itu tidak menentukan huruf drive (hanya nama file dan jalur), jadi ? hanyalah placeholder untuk drive apa pun tempat file halaman berakhir.

\n\n

Singkatnya, ya, tanda tanya itu seharusnya ada di sana.

\n\n

Bacaan lebih lanjut: Di dalam Object Manager NT , Manajer Objek (Windows)
