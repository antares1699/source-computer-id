Setelah menyederhanakan riwayat cabang Git menggunakan git rebase -i, saya mengalami masalah dengan remote: 

$ git push remote-name branch-name\nCONFLICT (add/add): Merge conflict in ...\nerror: failed to push some refs to 'ssh://...'\n$ git pull remote-name branch-name\n...\nAutomatic merge failed; fix conflicts and then commit the result.\n$ git mergetool\n$ git commit -m \"Merge\"\n

Sekarang sejarah cabang itu kacau, dengan komit saya muncul dua kali (garis paralel di gitk) setelah beberapa titik yang tampaknya sewenang-wenang sebelum komit pertama saya. Saya sudah mencoba git rebase -i, tetapi sekarang mengeluh bahwa "Tidak dapat menerapkan 123abc ...". Bagaimana cara menghilangkan komit paralel?
