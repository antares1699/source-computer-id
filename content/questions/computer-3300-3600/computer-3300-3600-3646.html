Saya bisa memutakhirkan Surface 3 baru saya ke Windows 10. Sudah bekerja dengan baik selama seminggu atau lebih. Kemudian beberapa hari yang lalu tidak seorang pun kecuali saya atau istri saya (satu-satunya dua di keluarga kami yang telah saya jadikan administrator di Surface) dapat masuk ke dalamnya. Anak-anak kita tidak bisa login sama sekali menggunakan akun mereka. Setiap upaya mereka untuk masuk menghasilkan pesan kesalahan berikut:

\n\n

"Windows tidak dapat terhubung ke layanan Klien Kebijakan Grup.\nSilakan berkonsultasi dengan administrator sistem Anda. "

\n\n

Nah, saya adalah "administrator sistem"; dan saya tidak yakin apa yang terjadi. Saya telah mencari kesalahan ini dan menemukan artikel online yang mengatakan bahwa Anda harus masuk ke aplikasi Layanan dan mengatur startup untuk layanan Klien Kebijakan Grup ke Otomatis. (Saya sudah memeriksa Surface dan disetel ke Manual.) Tapi artikel itu untuk Vista. Apakah nasehat itu masih berlaku sampai sekarang?

Sunting :

\n\n

Saya akhirnya menemukan apa yang tampaknya menjadi solusi permanen untuk masalah ini di sini https://answers.microsoft.com/en-us/windows/forum/windows_7-performance/why-wont-windows-connect-to-the-group-policy/b73107f8-8447-4599-87a5-65ecc6a63aa0?auth= 1

\n\n
    \n
  1. Buka regedit
    2. \n
  2. Arahkan ke kunci HKEY_LOCAL_MACHINE\\SOFTWARE\\Microsoft\\Windows NT\\CurrentVersion\\SVCHOST
    3. \n
  3. Buat nilai multi-string bernama GPSvcGroup dan tetapkan nilai GPSvc
    4. \n
  4. Buat kunci (folder) dan beri nama GPSvcGroup
    5. \n
  5. Buka folder GPSvcGroup yang baru dibuat dan buat 2 nilai DWORD:
      \n
    • Pertama kali dipanggil AuthenticationCapabilities dengan nilai 0x00003020 (atau 12320 dalam desimal)
      • \n
    • Kedua disebut CoInitializeSecurityParam dengan nilai 1
      • \n
    6. \n
  6. Mulai ulang
    7. \n
\n\n

Setelah melakukan ini, layanan Klien Kebijakan Grup saya berjalan saat saya boot

\n\n

Solusi orisinal :

\n\n

Saya sering mengalami masalah ini di kedua komputer Windows 10 saya. Tampaknya terjadi setiap minggu atau lebih. Ini selalu memperbaiki masalah, meskipun hanya sementara dan hanya setelah itu terjadi:

\n\n
    \n
  1. Buka prompt perintah sebagai administrator
    2. \n
  2. Jalankan sfc /scannow
    3. \n
  3. Nyalakan ulang komputernya
    4. \n
\n\n

Meskipun selalu mengatakan bahwa tidak ada masalah yang ditemukan ketika saya menjalankan pemindaian sfc, itu memperbaiki masalah tersebut.

\n\n

Apakah Anda mencoba mengubah mode penyalaan layanan? Jika itu berhasil, saya ingin tahu!

Saya mengalami masalah "Tidak dapat tersambung ke layanan klien kebijakan grup" untuk pertama kalinya pagi ini. Saya membuka sesi obrolan dukungan Microsoft dan mendapat respons yang cukup cepat yang terlihat sangat mirip dengan posting di atas (menarik).

\n\n

Sayangnya, saya bahkan tidak dapat menjalankan regedit. Itu, dan sebagian besar operasi sistem lainnya, hanya berhenti tanpa merespons (saya bahkan tidak bisa kembali ke titik pemulihan sebelumnya). Jadi saya tidak mencoba perbaikan yang disarankan ini.

\n\n

Melihat log peristiwa (klik kanan pada "start", lalu penampil acara, menurut saya ..), saya melihat beberapa kesalahan terkait antivirus McAfee (menurut saya mcupdate). Saya mencopot pemasangan McAfee. Setelah reboot, semuanya bekerja.

\n\n

Jadi, dalam kasus saya, anti-virus McAfee mungkin telah menjadi masalahnya. Saya telah mencari di web dan belum menemukan orang lain mengalami masalah McAfee Windows 10 ini, jadi mungkin masalah saya hanya satu.
