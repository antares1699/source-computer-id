Java EE secara efektif adalah sekumpulan pustaka perusahaan (dalam file .jar) yang berjalan di atas Java SE, biasanya di server. Jadi java -version dengan sendirinya hanya mengatakan "SE". Ia juga mengatakan "Server VM", yang disetel untuk menjalankan server; tetapi ini adalah VM default pada Linux 64-bit untuk semuanya. (Anda mengonfigurasi atau mengganti pilihan VM saat memulai. VM tidak "tahu jenis aplikasi apa" yang dijalankannya, jadi VM tidak memilihkan untuk Anda.)

\n\n

Java EE yang Anda tautkan menginstal server Glassfish; bundel tertentu itu memiliki JDK sendiri - sebenarnya yang lebih tua. Jadi dimanapun Anda meletakkannya, jika Anda pergi ke direktori glassfish3/jdk7 dan menjalankan bin/java -version, itu akan melaporkan java version \"1.7.0_10\", bukan _45. Perhatikan bahwa runtime "inti" adalah pada jdk7/jre/lib/rt.jar

\n\n

EE JARs ada di glassfish3/glassfish/modules. Yang utama memiliki nama yang dimulai dengan "javax", jadi Anda bisa melihatnya dengan find . -name 'javax*.jar'

\n\n

Saat Anda menjalankan aplikasi di Glassfish, aplikasi tersebut harus melakukan sihir jalur kelas untuk membuat JAR tersebut tersedia. Jika Anda menggunakan IDE, itu harus melakukan sesuatu yang serupa sehingga Anda dapat mengkompilasi kode Anda. Jika Anda memprogram secara manual, Anda harus melakukannya sendiri.

\n\n

Jadi untuk menjawab pertanyaan Anda, tidak ada "menggunakan" EE tanpa juga menggunakan SE. Dan satu-satunya persyaratan nyata untuk menggunakan EE adalah memiliki JAR-nya di classpath. Eclipse harus memiliki cara untuk "menunjuk ke Glassfish" dan membuatnya menemukan segalanya.

\n\n

Dalam kasus khusus ini, Anda juga harus menghapus JDK _10 yang lama. Kemudian letakkan _45 Anda yang sudah ada di tempatnya di glassfish3/jdk7; atau lakukan perubahan yang diperlukan pada PATH, JAVA_HOME, glassfish3/glassfish/config/asenv.conf, dll untuk menjalankannya. Anda tidak ingin secara tidak sengaja menjalankan JDK lama.

Di linux sedikit berbeda dari OS windows. Saya akan menulis bagaimana saya mengatur lingkungan di Ubuntu 18.0.4.\nAnda akan membutuhkan 4 alat untuk mulai mengembangkan JSP atau halaman Web Dinamis dengan java.

\n\n

1. JavaSDK
2. Ikan Gelas
3. Server Apache Tomcat
4. Gerhana IDE \n

\n\n

1. Instal Oracle Java
Ada banyak diskusi antara java mana yang akan digunakan pada platform Linux openjdk atau oracle-jdk. saya tidak ingin membedakan antara mereka tetapi saya akan merekomendasikan Anda untuk menggunakan oracle-jdk untuk alasan yang tidak diketahui :)

\n\n

Prosedur instalasi\ntambahkan repositori 

sudo add-apt-repository ppa:webupd8team/java\nsudo apt-get update\n

Instal java menggunakan 

sudo apt-get install oracle-java8-installer\n




\n\n

2. Pasang Glassfish
Perbedaan utama antara Java-SE dan Java-EE adalah pada library yang memiliki beberapa library tambahan dan salah satunya adalah glassfish. Saat Anda menginstal pustaka ini, Anda dapat dengan mudah mengimpor kelas di bawah paket javax.

\n\n

Instalasi Glassfish: 

sudo apt-get install glassfish-javaee\n
