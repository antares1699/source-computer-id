Saya memiliki arsip e-book yang sangat besar (sekitar 1TB) termasuk berbagai format file misalnya. PDF, DJVU, MOBI dan EPUB.

\n\n

Saya meletakkannya di folder yang berbeda berdasarkan subjek misalnya. Teknik, Pemrograman, dll. Tapi setelah bertahun-tahun, semuanya menjadi gila. Folder pemrograman itu sendiri berukuran 220GB dan nama file tidak jelas. Beberapa nama file didefinisikan dengan baik seperti: 236659889_Final_Report_of_2012_Climate_Change_Conference.pdf tetapi beberapa nama file hanya berupa nomor ISBN atau hanya download.pdf.

\n\n

Saya membutuhkan aplikasi untuk mengatur dan mencari e-book saya. Saya sudah mencoba Calibre, Mendeley dan Debenu. Tetapi semua aplikasi ini mencoba mengimpor file terlebih dahulu dan saya tidak memiliki 1+ TB cadangan untuk folder impor aplikasi.

\n\n

Luar biasa, tetapi bahkan tidak ada solusi yang baik untuk Unix / Linux - dan saya telah berusaha keras untuk menemukannya. Sebenarnya, saya sedang mencari solusi offline, tetapi saya malah mencari hal-hal online. Ini neraka.

\n\n

Ide "terbaik" saya sejauh ini adalah menggunakan mesin pencari desktop dan membatasinya ke folder "perpustakaan" (arsip-buku-e). Sekarang, saya menggunakan Tracker. Sebelumnya, saya menggunakan Beagle, yang tidak lagi dirawat. Mesin yang baik dapat membaca semua format file Anda dan membuat setidaknya database indeks.

\n\n

Ide "terbaik kedua" saya membutuhkan sedikit waktu. Periksa file Anda ... dan ganti namanya. Atau letakkan kelompok mereka ke dalam sub-folder dan berikan nama yang bagus untuk folder Anda. Nah, untuk 1TB, mungkin perlu waktu agak lama. Sebagian besar sistem file modern mendukung nama file yang sangat panjang, sehingga Anda dapat memikirkan pola yang baik untuk tag dan informasi, yang dapat menjadi bagian dari nama folder- / file-. Saya menggunakan pola seperti 

[CLASS]_Year_Author1-Author2_Name-Name-Name_[TAG1-TAG2].filetype\n

di mana kelas dapat berupa apa saja seperti "book", "prosiding", "paper", "report", "thesis" ... Memiliki pola yang baik, Anda dapat menggunakan perintah shell (seperti find, awk, grep dll) secara berurutan untuk menemukan, apa yang Anda cari. Metode ini membutuhkan disiplin, karena Anda perlu memberi nama yang baik untuk setiap entri baru di perpustakaan Anda.

\n\n

Last but not least, beberapa waktu yang lalu, saya menaruh sedikit harapan WinFS , tetapi menurut saya proyek ini telah dihentikan. Bagaimanapun, alangkah baiknya, jika seseorang dapat mengaitkan tag ke file dan menyimpannya sebagai data meta dalam sistem file.
