Nah, untuk memahami mengapa tidur lebih disukai daripada hibernasi, penting untuk mengetahui apa itu masing-masing ( sumber ):

\n\n
\n

Tidur umumnya dikenal sebagai Standby in\n Sistem Windows atau S3 di ACPI. Dalam tidur\n mode, catu daya ke\n tidak penting dan tidak kritis\n komponen ditahan, dan sebagian besar sistem\n operasi dimatikan dan dihentikan. Semua\n data dalam memori fisik (modul RAM)\n masih disimpan di memori internal , dan\n seluruh sistem ditempatkan dalam keadaan siaga\n mode, yang dapat dibangunkan dan digunakan\n hampir seketika. Dalam mode Tidur, file\n beban daya sangat berkurang, hemat\n banyak energi. Namun, kekuatannya\n tidak boleh dipotong, dan harus dipotong\n terus memasok ke komputer. Setelah listrik mati, sistem akan melakukannya\n harus memulai lagi seperti baru\n komputer boot baru saja dimulai dari daya\n off state.

\n
\n\n

...

\n\n
\n

Hibernate, atau S4 di ACPI, sementara itu\n akan menyimpan data dalam memori fisik\n ke hard disk drive (HDD), lalu\n matikan komputer. Dalam mode Hibernate\n mode, file bernama hiberfil.sys yang\n memiliki ukuran file yang sama dengan jumlahnya\n memori sistem akan dibuat\n disk lokal. Saat pengguna ingin menggunakan\n komputer lagi, komputer akan\n boot dan memuat kembali status di\n hibernasi terakhir. Keuntungan dari\n Modus hibernasi adalah tidak ada daya\n terbuang percuma untuk penghematan daya maksimum. Di\n Dormansi hibernasi, tidak ada listrik\n dikonsumsi oleh sistem. Selain itu, pulihkan\n dari Hibernate biasanya lebih cepat\n dari reboot komputer, dan benar-benar\n berbeda dari awal yang baru, sebagai pengguna\n dapat kembali ke keadaan tepat terakhir\n hibernasi dengan semua program berjalan\n dan dokumen dibuka utuh, sebagai gantinya\n dari desktop kosong. Kerugian dari\n Hibernasi adalah setelah jangka waktu\n waktu, mungkin ada fragmentasi\n mengajukan. Pengguna perlu melakukan defragmentasi\n volume yang menyimpan hibernasi\n mengajukan sering.

\n
\n\n

Hibernasi lebih lambat daripada tidur karena seluruh konten RAM harus disimpan di disk saat memasuki kondisi ini. Memiliki file 4 GB di C: partisi bukanlah hal terbaik, kadang-kadang. File itu kemudian harus dimuat kembali ke RAM saat keluar dari hibernasi. Tidur lebih cepat, tidak ada yang perlu disimpan, tidak ada yang dimuat (kebanyakan).

\n\n

Keandalan seharusnya tidak menjadi perhatian untuk tidur, atau hibernasi.

Hibernasi menggunakan ruang disk (jelas) dan tidak berfungsi pada beberapa perangkat keras (kebanyakan yang lebih tua). Tapi saat berhasil, itu solid. Anda akan tahu setelah beberapa hibernasi pertama apakah itu berfungsi dengan baik pada perangkat keras Anda. Benar-benar tidak ada risiko untuk mencobanya. Banyak orang menggunakannya setiap hari tanpa masalah, dan hanya melakukan boot sepenuhnya saat Windows membutuhkannya, atau jika aplikasi yang ceroboh menyebabkan crash.
