Saya memiliki pengaturan dengan router Cisco RV042 yang terhubung sebagai berikut: 

Broadband Motorola Surfboard -> Cisco RV042 -> Wirless Router\n |                                              |\n |-> Wirless N network                          |-> Wireless G network\n      |                                              |\n      |-> Home server w/ shared printer              |-> [Other PCs]\n

Papan Selancar memiliki jaringan N nirkabel, meskipun perute nirkabel lainnya berjalan di G untuk alasan kompatibilitas.

\n\n

Masalah yang saya alami adalah saya memiliki printer yang terhubung ke komputer Windows pada router N nirkabel (dengan alamat IP statis), dan saya memerlukan komputer di jaringan G agar dapat menggunakan printer bersama. Printer dibagikan, meskipun tidak melalui Homegroup.

\n\n

Di komputer lain di rumah (di jaringan G), saya tidak dapat melihat server dalam daftar komputer di jaringan, dan sepertinya saya tidak dapat menambahkan printer secara manual dengan mengetikkan alamat IP.

\n\n

Pertanyaannya adalah-- bagaimana saya bisa mendapatkan komputer lain untuk mendeteksi server yang terhubung ke printer, atau alamat / port / URL IP apa yang saya tentukan untuk menghubungkan ke printer bersama pada PC itu?

Apakah Anda memiliki kedua Router yang membagikan DHCP? Apakah kedua router berada di subnet IP yang sama? Jika papan selancar menangani DHCP, router G telah menonaktifkan DHCP (tetapi disetel dengan alamat statis pada subnet IP yang sama ... pada dasarnya hanya jembatan), saya tidak dapat melihat mengapa semua komputer tidak dapat melihat printer bersama.

\n\n

Sebagai contoh...

\n\n

Bagaimana ini menarik Anda ... masuk akal? Minta papan selancar menangani DHCP, dan atur kisarannya untuk mulai dari 50 atau lebih. Setel router G menjadi jembatan, atau setidaknya nonaktifkan DHCP, dan sambungkan ke papan selancar melalui salah satu port ethernet normal (bukan port Internet). Tetapkan IP yang berada di luar jangkauan DHCP, tetapi pada subnet yang sama (sehingga Anda dapat dengan mudah membukanya untuk menyesuaikan pengaturan, dll). Kemudian, semua komputer yang terhubung ke router G, akan mendapatkan alamat IP yang berada dalam kisaran yang sama dengan server dan printer. Itu seharusnya berfungsi.

Jika Anda memiliki kedua router yang dikonfigurasi untuk menggunakan subnet yang sama, (mis., 192.168.1.x / 255.255.255.0), Anda perlu mengkonfigurasi router kedua sebagai jembatan atau subnet router G akan menutupi subnet router N. Sebagian besar perute tingkat konsumen tidak memiliki opsi penghubung dalam firmware default mereka, meskipun firmware sumber terbuka seperti Tomat, OpenWRT, atau DD-WRT kemungkinan besar akan memungkinkan Anda untuk mengonfigurasi penghubungan.

\n\n

Salah satu solusi mudahnya adalah menonaktifkan DHCP di router G dan mengonfigurasinya dengan alamat IP di subnet router N. Kemudian, alih-alih menghubungkan port WAN router G ke router N, sambungkan salah satu port LAN-nya ke router N. Dengan cara ini, Anda hanya menggunakan router G sebagai titik akses. Saat klien terhubung ke router G, klien akan diberi alamat DHCP oleh router N dan akan dapat melihat server dan printernya, dengan asumsi pembagian Anda dikonfigurasi dengan benar.

\n\n

Solusi lain adalah dengan hanya mengatur router Anda untuk menggunakan subnet yang berbeda (misalnya, mengatur subnet router N ke 192.168.0.x dan subnet router G ke 192.168.1.x) dan komputer di router G seharusnya dapat melihat apa pun di router N (tetapi tidak sebaliknya).
