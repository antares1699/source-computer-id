Pertarungan Word vs PDF atau Eps yang tiada henti

\n\n

Dengan gambar raster Anda harus menemukan kompromi antara definisi dan ukuran. Jika Anda mengetahui bahwa dokumen akan dicetak pada 600 DPI, Anda dapat memutuskan untuk mengimpor gambar dengan definisi tersebut. Tetapi jika besok Anda akan memiliki printer definisi tinggi, Anda harus mulai lagi. Jika Anda meningkatkan DPI, dokumen Anda akan bertambah ukurannya dan Anda akan membutuhkan lebih banyak sumber daya untuk sistem Anda.

\n\n

Jika Anda dapat bekerja dengan file gambar vektor Anda tidak akan mengalami masalah terkait dengan definisi gambar, tetapi Anda dapat menemukan masalah terkait font yang diinstal, atau Anda dapat memiliki file yang lebih besar jika, misalnya, Anda memplot 1 miliar data ...

\n\n

Saat Anda harus mengimpor file dalam kata, Anda dapat:

\n\n
    \n

  • Pergi ke sumbernya : Anda dapat mengganti atau menambahkan format (dan akhirnya ukurannya) yang diperlukan langsung di skrip yang menghasilkan plot. Baca dari situs matplotlib [ 1 ] . Lebih baik jika Anda bisa menyimpan di file Format Grafik Vektor [ 1b ] . 

    plt.savefig(pp, format='pdf') \nplt.savefig(pp, format='png')\nplt.savefig(pp, format='svg')\n

    atau bahkan 

    fig.savefig('test.pdf')\nfig.savefig('test.png')\n
    • \n

  • Gunakan imagemagick [ 2 ] atau Inkscape [ 2b ] atau pincang [ 2b ] untuk mengonversi pdf dalam png (atau dalam format raster lainnya, tiff, jpg ...) atau dalam svg (atau dalam format grafik vektor lainnya).

    \n\n

    Ini tergantung apakah itu PDF dengan vektor grafis [ 3 ] di dalam atau tidak.
    Dalam kasus pertama, Anda harus menemukan beberapa masalah rendering atau font yang langka tetapi tidak ada masalah definisi.
    Dalam kasus terakhir, Anda harus memilih kepadatan dan dimensi untuk gambar akhir.
    Baca lebih lanjut misalnya tentang jawaban ini [ 4 ] .
    Anda akan selesai menulis sesuatu yang mirip dengan: 

    convert file.pdf file.svg                                  # If pdf with vector\nconvert -density 600 file.pdf -resize 4961x7016 mypic.png  # With fixed grid\n
    • \n
\n\n

Catatan
Jika file PDF dibuat dengan gambar Raster dengan definisi tertentu, mis. 300 DPI, Anda tidak akan terlalu berhasil dengan program apa pun yang meningkatkan DPI menjadi 400 atau 600 ... :-)
Sebagai aturan jempol (biasanya berfungsi) Anda dapat berasumsi bahwa dalam pdf raster ada string /image.
Jadi di Linux misalnya Anda dapat menjalankan grep dan menghitung kemunculan string itu: 

grep  -c -i \"/image\" *pdf\nMyRasterPdf.pdf:3    # > 0  if raster pdf\nMyVectorPdf.pdf:0    # = 0  if vector pdf\n

Terakhir, pertimbangkan Getah , mungkin dengan GUI sebagai pembuat teks .

Mulai Mei 2018, Microsoft Word dalam edisi Office 365 telah membuat dua perubahan yang mengubah jawaban atas pertanyaan ini:

\n\n
    \n
  • EPS tidak lagi didukung
    • \n
  • SVG sekarang didukung
    • \n
\n\n

Ini berarti bahwa cara terbaik untuk menangani grafik berkualitas tinggi adalah dengan membuat gambar di Matplotlib yang memiliki ukuran yang benar (menggunakan plt.figure(figsize=(width_in_inches, height_in_inches)), kemudian mengekspor gambar dengan plt.savefig('filename.svg'). Anda dapat memasukkan file ini langsung ke Word dan akan ditampilkan dengan benar , bahkan saat mengonversi ke PDF.

Di Word 2016, alur kerja saya untuk mengimpor grafik matplotlib adalah: 

import matplotlib.pyplot as plt\n

Ekspor teks yang dapat diedit sehingga saya dapat menyesuaikan gambar dan teks di Illustrator, secara default matplotlib mengekspor “font Tipe 3” yang tidak dimengerti Adobe Illustrator, jadi Anda perlu mengekspor font Tipe 2 / TrueType. 

plt.rcParams['pdf.fonttype'] = 42\nplt.rcParams['ps.fonttype'] = 42 \n

menyimpan gambar 

plt.savefig('my_figure.pdf',bbox_inches='tight',transparent = True)\n

Buka di Illustrator dan sesuaikan sesuai kebutuhan

\n\n

\n\n
