Saat Anda perlu menyambungkan dari jarak jauh ke desktop seseorang, dan komputernya berada di belakang NAT, Anda dapat menggunakan koneksi VNC terbalik. Anda melakukannya dengan menggunakan opsi "Lampirkan Penampil Mendengarkan" TightVNC Server. Ini membutuhkan pilihan manual dari opsi itu, dan pengetikan manual alamat IP.

\n\n

Adakah cara untuk menghindari aktivitas manual ini dengan membuat pintasan (baris perintah) dengan alamat IP dan port tetap?

Saya menemukan jawabannya. Untuk melakukannya gunakan perintah sederhana seperti ini:

\n\n

\"c:\\Program Files\\TightVNC\\tvnserver.exe\" -controlservice -connect ip-address:port-number

\n\n

Untuk ip-addres Gunakan alamat IP komputer yang menjalankan klien VNC aktual (yang akan mengontrol desktop)

Saya ingin VNC ke komputer lokal di IP 10.0.1.11 menggunakan pintasan desktop jadi saya cukup mengkliknya dua kali. Di atas tidak berhasil. Menggunakan bantuan (dalam CMD) saya menemukan petunjuknya sebagai:

\n\n

tvnviewer hostname :: port [OPTIONS]

\n\n

Contoh yang berfungsi menggunakan TightVNC:

\n\n
    \n

  1. Buat pintasan di desktop Anda dari folder 'Program Files' (atau x86)

    2. \n

  2. Klik kanan pintasan, pilih Properti.

    3. \n

  3. Pada baris pertama yang dapat diedit berlabel 'Target:' tambahkan ini setelah string yang sudah ada: 10.0.1.11::5900 sehingga string Anda sekarang terlihat seperti ini:

    4. \n
\n\n

"C: \\ Program Files \\ TightVNC \\ tvnviewer.exe" 10.0.1.11::5900

\n\n

Jelas ubah 10.0.1.11 ke alamat IP apa pun yang ingin Anda kontrol.\nBuka juga tab Umum dan ubah nama pintasan menjadi sesuatu yang sesuai seperti "Hubungkan ke Laptop" atau "Mac Saya", dll.

\n\n

Diuji pada PC Windows 8.1 Pro saya dengan TightVNC 2.7.10.0.
