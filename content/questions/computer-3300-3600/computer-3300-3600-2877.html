Ternyata ada perbaikan besar-besaran ke RDP dengan Windows 8. Anda dapat membaca tentang peningkatan grafis ke RDP di blog Remote Desktop dari Microsoft.

Untuk menjawab pertanyaan Anda secara khusus, berikut adalah kutipan dari blog ini:

\n\n
\n

Di Windows Server 2012 dan Windows 8, desktop lokal lengkap adalah\n disusun dan ditampilkan pada host dalam sesi jarak jauh sebelum\n gambar yang dihasilkan dikodekan dan dikirim oleh RemoteFX. Dengan kata lain,\n RemoteFX didasarkan pada rendering sisi host. Hasilnya, penuh\n pengalaman desktop desktop Windows 8 serta semua aplikasi\n antarmuka pengguna (UI) secara konsisten dikirim dari jarak jauh paling tinggi\n kualitas dengan 32 bit per piksel (BPP). Untuk menyeimbangkan pengalaman pengguna dengan\n encoding dan biaya bandwidth, frame rate target dalam sesi jarak jauh\n adalah 30 frame per detik (FPS).
