Mengapa system32 memiliki begitu banyak nama folder berbeda yang disebut Drivers?

\n\n

\n \n

pengemudi

Tidak ada bedanya apakah file sistem win32 mencoba membuka file (direktori) bernama "drivers", "Drivers" atau "DRIVERS".

\n\n
    \n

  • Pemrogram driver (vendor) mungkin memilih untuk menggunakan varian nama direktori yang berbeda saat menulis driver mereka.

    • \n

  • Nama-nama ini berasal dari file .INF yang mengasosiasikan driver dengan perangkat keras mereka. Penulis file .INF yang berbeda menggunakan variasi nama yang berbeda.

    • \n

  • Subsistem Win32 menggunakan mode tidak peka huruf besar / kecil (melindungi huruf besar / kecil tetapi tidak peka huruf besar / kecil).

    • \n
\n\n

Ini berarti bahwa aplikasi dapat memilih file pada disk meskipun nama yang diberikan berbeda dari nama yang disimpan pada disk.

\n\n
\n

Sebagai bagian dari persyaratan kepatuhan POSIX, Sistem Berkas Windows NT (NTFS) menyediakan berkas peka huruf besar kecil dan konvensi penamaan direktori. Meskipun NTFS dan subsistem POSIX masing-masing menangani sensitivitas huruf dengan baik, berbasis Windows 16-bit, berbasis MS-DOS, berbasis OS / 2, dan Aplikasi berbasis Win32 tidak .

\n \n

Di NTFS, Anda dapat membuat nama file unik, disimpan di file\n direktori, yang hanya berbeda dalam kasus. Misalnya berikut ini\n nama file dapat hidup berdampingan dalam satu direktori pada volume NTFS: 

CASE.TXT\ncase.txt\ncase.TXT\n

Namun, jika Anda mencoba untuk membuka salah satu file ini di Win32\n aplikasi , seperti Notepad, Anda hanya akan memiliki akses ke salah satu\n file , terlepas dari kasus nama file yang Anda ketik di Buka\n Kotak dialog file.

\n
\n\n

...

\n\n
\n

NTFS mendukung dua mode operasi yang sedikit berbeda yang dapat dipilih oleh subsistem aplikasi yang berinteraksi dengan NTFS. Yang pertama peka huruf besar-kecil dan menuntut agar nama file yang disediakan oleh aplikasi cocok dengan nama yang disimpan pada disk termasuk huruf besar / kecil jika file pada disk akan dipilih. Mode operasi kedua adalah melindungi huruf besar / kecil tetapi tidak membedakan huruf besar / kecil. Ini berarti bahwa aplikasi dapat memilih file pada disk meskipun nama yang diberikan berbeda dari nama yang disimpan pada disk. Perhatikan bahwa kedua mode mempertahankan case yang digunakan untuk membuat file. Perbedaan perilaku yang dicatat di sini hanya berlaku jika aplikasi perlu menemukan file yang sudah ada. POSIX memanfaatkan mode peka huruf besar / kecil, sedangkan MS-DOS, WOW, dan Subsistem Win32 menggunakan mode case insensitive .
