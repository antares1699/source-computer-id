$ (route ;echo; route -n ) | grep -v '^[KD]'\n10.3.0.0        *               255.255.255.224 U     0      0        0 eth0\n192.168.0.0     *               255.255.255.0   U     2004   0        0 wlan0\nlink-local      *               255.255.0.0     U     2      0        0 eth0\nloopback        localhost.local 255.0.0.0       UG    0      0        0 lo\ndefault         192.168.0.1     0.0.0.0         UG    2004   0        0 wlan0\n\n10.3.0.0        0.0.0.0         255.255.255.224 U     0      0        0 eth0\n192.168.0.0     0.0.0.0         255.255.255.0   U     2004   0        0 wlan0\n169.254.0.0     0.0.0.0         255.255.0.0     U     2      0        0 eth0\n127.0.0.0       127.0.0.1       255.0.0.0       UG    0      0        0 lo\n0.0.0.0         192.168.0.1     0.0.0.0         UG    2004   0        0 wlan0\n

Baru saja menjawab pertanyaan serupa:

\n\n

Mengapa hanya ada satu Default Gateway?

\n\n

Gateway "*" atau 0.0.0.0 menunjukkan bahwa jaringan dapat dijangkau secara langsung - tidak diperlukan gateway. Gateway default adalah alamat tempat paket dikirim jika tidak cocok dengan rute yang lebih spesifik - yaitu jika ditujukan untuk router inti & internet yang lebih luas.
