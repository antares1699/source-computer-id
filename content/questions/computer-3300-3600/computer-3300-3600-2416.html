Saya memiliki kartu Micro SD yang tidak berfungsi lagi.

\n\n

Latar Belakang:

\n\n

Yang terjadi adalah saya mengeluarkannya dari ponsel lama saya (berfungsi dengan baik), dan memasukkannya ke ponsel baru saya (Nokia E71). Nokia mengeluh bahwa itu dienkripsi dan diminta untuk memasukkan kata sandi. Saya tidak tahu mengapa ia berpikir bahwa itu dienkripsi.

\n\n

Itu tidak akan menerima kata sandi kosong, jadi saya mencoba 1234. Itu tidak berhasil, jadi saya menukarnya kembali ke telepon lama, yang tidak dikenali lagi. Kemudian saya mencobanya di pembaca kartu laptop saya, dan itu juga tidak berfungsi di sana. Saya memiliki kartu bagus yang diketahui berfungsi di laptop.

\n\n

Saya tidak tahu apakah saya merusak kartu secara fisik saat menarik / mendorongnya di antara ponsel, atau apakah Nokia melakukannya sesuatu untuk itu ..

\n\n

Saya ingin setidaknya memulihkan file darinya jika memungkinkan.

\n\n

Detail pemecahan masalah:

\n\n

Di laptop (Ubuntu linux) saya, ketika saya memasukkannya ke pembaca kartu saya, /dev/sdc muncul, tetapi tidak ada /dev/sdc1 - tidak ada partisi yang ditemukan.

\n\n

Saya mencoba membuat gambarnya sebagai berikut:

\n\n
\n

sudo dd if = / dev / sdc dari = brokencard.img

\n
\n\n

Tapi saya mendapatkan kesalahan ini:

\n\n
\n

dd: membuka `dev / sdc ': Tidak ada media yang ditemukan

\n
\n\n

Kemudian saya mencoba ini:

\n\n
\n

sudo fdisk / dev / sdc

\n
\n\n

Dan dapatkan ini:

\n\n
\n

Tidak dapat membuka / dev / sdc

\n
\n\n

Saya mencoba utilites photorec dan testdisk (dari keamanan ), tetapi tidak satu pun dari mereka akan menemukan /dev/sdc.

\n\n

Saya sekarang kehabisan ide. Apa lagi yang bisa saya coba?
