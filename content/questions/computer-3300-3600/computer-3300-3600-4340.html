Saya memiliki layanan prioritas tinggi yang saya mulai dengan sudo nice -n -10 process. Proses ini tidak memerlukan hak pengguna super, kecuali untuk peningkatan prioritas. Tapi bagus membutuhkan hak superuser untuk meningkatkan prioritas.

\n\n

Bagaimana cara menjalankan proses dengan prioritas tinggi, tanpa juga menjalankannya sebagai root?

","source":"https://superuser.com/questions/236118/how-do-i-elevate-the-priority-of-a-process-without-giving-that-process-superuser","author":"Myrddin Emrys","replies":[],"answers":[{"answer":"

Lihat /etc/security/limits.conf. Di sana Anda dapat menentukan prioritas mana yang diizinkan untuk digunakan oleh pengguna atau grup. Halaman manual: man 5 limits.conf

Salah satu kemungkinannya adalah ini (yang baru saja saya pikirkan setelah bertanya):

\n\n

sudo nice -10 sudo -u user command

\n\n

Tapi sepertinya harus ada metode yang lebih elegan.

","votes":"1","source":"https://superuser.com/a/236120","author":"Myrddin Emrys","replies":[]},{"answer":"

Kemungkinan lainnya adalah menggunakan sudo renice untuk mengubah prioritas proses setelah memulainya. Untuk mendapatkan PID proses, Anda dapat menggunakan ps dan sed, atau Anda dapat memulainya di latar belakang dan mendapatkan prosesnya langsung dari shell: 

process &\npid=$!\nsudo renice -10 $pid\nfg $pid                 # if you don't want it in background\n

Perhatikan bahwa jika proses memunculkan proses lain dan keluar, itu mungkin akan melakukannya pada saat renice dijalankan, jadi Anda akan ingin menggunakan ps untuk mendapatkan PID dari proses / proses anak, dan mengganti mereka.
