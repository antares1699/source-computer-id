Kami ingin menggunakan hotspot seluler ini untuk menghubungkan sistem keamanan video ( DVR Apasonik ) ke internet.

\n\n

DVR Apasonic menghosting halaman web tempat rekaman keamanan dapat diakses. Dengan koneksi internet broadband kabel, mudah untuk mengatur penerusan port dan login ke DVR dari jarak jauh.

\n\n

Login jarak jauh tidak dimungkinkan dengan hotspot seluler ini, bahkan hotspot tidak dapat di-ping. Hotspot seluler memiliki konfigurasi penerusan port tetapi ini tidak ada gunanya.

\n\n

Saya menduga bahwa hotspot nirkabel tidak cocok untuk hosting server. Dugaan saya adalah bahwa jaringan seluler dioptimalkan untuk koneksi berumur pendek yang menggunakan banyak bandwidth, dan koneksi berumur panjang yang menggunakan sedikit bandwidth adalah "musuh terburuk" dari perusahaan seluler karena penggunaan data dibayar, bukan waktu. terhubung.

\n\n

Internet seluler tampaknya memiliki kemampuan yang berbeda dari broadband, dan saya akan sangat tertarik untuk melihat lebih banyak dokumentasi tentang ini. Mungkin itu didasarkan pada Model jaringan 7 lapis . Pertanyaan ini tidak khusus untuk perangkat hotspot tertentu atau sistem keamanan yang kami gunakan. Saya pernah mendengar bahwa internet seluler mirip dengan dial-up, karena perangkat tidak memelihara koneksi konstan dan harus memulai koneksi berumur pendek. Jelas dial-up tidak cocok untuk menghosting server yang dapat menerima koneksi setiap saat, karena koneksi terputus hampir sepanjang waktu. Apakah internet seluler, broadband, dial-up, dan DSL, menempati lapisan yang sama dengan model 7-lapisan itu?

\n\n

Kembali ke percobaan ping - pengujian tersebut tidak banyak membuktikan karena modem umumnya tidak dapat di-ping, karena dikonfigurasi secara default. Server di belakang modem lebih mungkin merespons ping. Saya rasa saya telah melihat opsi konfigurasi modem yang memungkinkan modem merespons ping.

\n\n

Alamat IP hotspot seluler mungkin sering atau jarang berubah - Saya tidak yakin. Anggaplah saya tidak membuat kesalahan yang jelas dengan mencoba mengakses server di alamat IP yang salah. Bisakah hotspot seluler menghosting server?

\n\n

Singkatnya, artikel yang saya temukan tentang pertanyaan ini tampaknya lebih mementingkan penerusan porta dan firewall daripada menjawab pertanyaan mendasar tentang kemampuan hotspot seluler.

\n\n

Terima kasih!

Broadband seluler telah menggunakan NAT tingkat operator bahkan sebelum istilah itu diciptakan, jadi tidak cocok untuk menghosting apa pun. Mungkin ada layanan bisnis dengan akses non-NAT, tentu saja dengan biaya tambahan.

\n\n

Sebagian besar metode akses Internet didasarkan pada "dial-up": Anda memulai sesi, mengautentikasi diri Anda sendiri. Tentu saja, jika Anda memiliki saluran tetap, sesi ini kemungkinan akan berlangsung lebih lama, karena tidak akan terganggu oleh kehilangan sinyal atau apa pun. Namun, broadband seluler pada dasarnya sama.

\n\n
\n\n

Jadi, saya akan memutuskan pertanyaan Anda adalah " Bagaimana cara membuat ini berfungsi? ". Yang Anda inginkan adalah jangkar statis yang menghubungkan klien (tempat Anda mengakses video) dan DVR. Dengan cara ini, tidak masalah di mana (dalam hal akses Internet) klien dan DVR berada.

\n\n

Salah satu kemungkinan untuk melakukannya adalah dengan menyewa VPS dan memasang server VPN di atasnya. Anda kemudian akan menggunakan klien VPN di jaringan DVR Anda dan di mesin klien. Terowongan ini melalui semua NAT dan firewall di jalan.

\n\n

Konfigurasi ini memiliki keuntungan tambahan: DVR tidak terhubung ke Internet. Keamanan layanan web DVR dipertanyakan. Jika memungkinkan, sebaiknya tidak diekspos ke Internet.

\n\n

Alternatif untuk menyewa VPS adalah menggunakan sesuatu seperti Hamachi.

\n\n

Kedua solusi tersebut mungkin memerlukan mesin tambahan yang bertindak sebagai klien VPN di jaringan DVR. Sebuah Raspberry Pi sudah cukup, tetapi Anda perlu memeriksa kompatibilitas perangkat lunak VPN terlebih dahulu!
