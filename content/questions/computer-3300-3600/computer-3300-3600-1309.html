Selama instalasi Ubuntu 15.04 Vivid Vervet, saya membuat partisi untuk sistem file Ubuntu sebagai Ext2 menggunakan alat partisi Windows. Saya membuat partisi untuk informasi bootloader dan sistem file Ubuntu. Setelah instalasi Ubuntu, partisi Ubuntu muncul di Windows sebagai disk lokal, tetapi tidak dapat dibaca. Ketika saya mencoba membukanya, sebuah pesan ditampilkan yang menyatakan bahwa partisi rusak Saya diminta untuk memformat disk.

\n\n

Saya tidak ingin mengakses sistem file Ubuntu dari Windows.

\n\n

  • Apa penyebabnya ini?
    • \n
  • Bagaimana cara menyembunyikan partisi Ubuntu ini di Windows?
    • \n

Windows entah bagaimana memutuskan untuk mendaftarkan partisi ini, tetapi karena fakta bahwa Windows tidak memiliki driver filesystem ext2/3/4, ia tidak dapat memahami isinya. Anda dapat menyembunyikan ini di Disk Manager dengan menghapus huruf drive yang ditetapkan.

\n\n

Atau Anda dapat membiarkannya apa adanya dan mencari driver ext2/3/4 untuk Windows sehingga Anda dapat membaca (mungkin dan menulis) konten ke partisi Linux.

Solusi yang ditawarkan oleh Maxthon Chan akan berhasil; Namun, mungkin ada solusi yang lebih baik. Untuk memahami mengapa ini merupakan solusi yang lebih baik, Anda harus terlebih dahulu memahami penyebab masalahnya ....

\n\n

Karena pertanyaan Anda diberi tag untuk Windows 10, tebakan saya adalah disk yang dimaksud menggunakan Tabel Partisi GUID (GPT). Jika Anda memeriksa Artikel Wikipedia tentang GPT, Anda akan melihat bahwa partisi data Windows memiliki kode tipe EBD0A0A2-B9E5-4433-87C0-68B6B72699C7, sedangkan partisi data Linux memiliki kode jenis 0FC63DAF-8483-4772-8E79-3D69D8477DE4. Prosedur partisi Anda menggunakan alat Windows, jadi partisi Linux mungkin memiliki kode jenis Windows. Ini berarti Windows menganggap partisi Linux adalah partisi Windows. Jadi, solusi yang lebih baik untuk masalah ini adalah dengan mengubah kode tipe dari partisi tersebut. Windows kemudian akan mengabaikan partisi Linux. Solusi ini lebih baik daripada menggunakan Disk Manager di Windows karena solusi Disk Manager berlaku untuk satu penginstalan Windows - jika Anda perlu menggunakan disk darurat Windows atau jika Anda menginstal ulang Windows, partisi Linux akan muncul lagi sebagai Windows yang rusak partisi dan rentan terhadap kerusakan yang tidak disengaja.

\n\n

Untuk mengubah kode tipe, Anda dapat menggunakan salah satu dari setidaknya dua kelas program:

\n\n
    \n
  • Di GParted dan parted, Anda dapat menghapus "bendera msftdata" dari partisi. "Bendera" ini adalah cara libparted untuk mengidentifikasi partisi data Windows.
    • \n
  • Di GPT fdisk (gdisk, cgdisk, atau sgdisk), Anda dapat mengubah kode jenis dari 0700 menjadi 8300. Ini adalah singkatan kode jenis yang digunakan GPT fdisk untuk partisi data Windows dan Linux.
    • \n
\n\n

Perhatikan bahwa jawaban saya didasarkan pada asumsi bahwa disk menggunakan GPT. Jika itu adalah disk MBR, kemungkinan besar sesuatu yang serupa dapat dilakukan dengan menggunakan alat Linux fdisk, tetapi mengubah kode jenis dari 07 menjadi 83. AFAIK, memperbaiki masalah ini pada disk MBR tidak mungkin dilakukan dengan GParted atau parted, meskipun saya ' Saya tidak 100% positif akan hal itu. Tidak gunakan GPT fdisk pada disk jika itu adalah disk MBR, karena GPT fdisk akan mengubahnya dari MBR ke bentuk GPT.
