Sepertinya masalah dengan SSD ... dan kemungkinan besar masalah dengan ISP Anda. Saya mengalami masalah serupa dan menemukan bahwa itu adalah masalah dengan MTU yang dimiliki ISP saya. saya menulis kata-kata kasar tentang masalah ini beberapa waktu yang lalu ... tetapi berikut adalah langkah-langkah umum untuk mendiagnosis / menyelesaikan:

\n\n

Pertama ... lihat pengaturan MTU Anda saat ini:\nBuka prompt perintah dengan hak administratif dan kemudian jalankan perintah sebagai berikut: 

netsh interface ipv4 show subinterfaces\n

Anda akan melihat sesuatu seperti: 

   MTU  MediaSenseState   Bytes In  Bytes Out  Interface\n------  ---------------  ---------  ---------  -------------\n  1500                1  8864896253  1116506364  Local Area Connection\n

Dengan pengetahuan itu, Anda akan mengetahui bahwa MTU Anda saat ini disetel ke 1500. Jika ISP Anda bekerja dengan itu ... Anda harus dapat melakukan ping ke situs seperti msn.com atau google.com dengan ukuran paket 1500 tanpa memecah paket. 

ping www.google.com -f -l 1500\n

jika Anda melihat pesan kesalahan seperti:\nPaket perlu difragmentasi tetapi set DF.

\n\n

Anda tahu Anda harus lebih kecil. Jadi, kurangi 8 & coba lagi ... sampai Anda menemukan MTU yang valid.\nSetelah Anda menemukan salah satu yang memberi Anda balasan tanpa mengeluh tentang perlunya dipecah-pecah ... inilah waktunya untuk mengubah MTU Anda. Kembali ke command prompt admin ... dan kami melakukan ini: (ganti 1464 dengan apa pun yang Anda buat) 

netsh interface ipv4 set subinterface \"Local Area Connection\" mtu=1464 store=persistent\n

Jika Anda hanya ingin mengujinya ... tanpa melakukan perubahan ... lewati bit store=persistent ... dan reboot akan mengembalikannya seperti semula. Anda juga dapat mengaturnya kembali secara manual ke apa pun yang Anda mulai dengan ...

\n\n

Saya tidak tahu apakah itu akan berhasil untuk semua orang ... tetapi "berhasil untuk saya" (c)

Saya mengalami perilaku aneh ini kembali dengan Windows Vista, dan baru-baru ini dengan Windows Server 2012 R2 dan Windows 8.

\n\n

Ternyata fitur penyetelan otomatis global TCP yang diperkenalkan kembali di Windows Vista tidak selalu berfungsi baik karena kompatibel dengan lingkungan jaringan yang ada. Anda mungkin ingin mencoba menonaktifkan global auto-tuning di command prompt administrator. 

netsh interface tcp set global autotuning=disabled\n
