Periksa kunci registri berikut ini: 

HKEY_LOCAL_MACHINE\\Software\\Microsoft\\Windows NT\\CurrentVersion\\Image File Execution Options\n

Pada sistem 64-bit, ada salinan kedua dari kunci ini untuk aplikasi 32-bit: 

HKEY_LOCAL_MACHINE\\Software\\Wow6432Node\\Microsoft\\Windows NT\\CurrentVersion\\Image File Execution Options\n

Kunci ini bisa digunakan untuk meluncurkan debugger secara otomatis saat aplikasi yang ditentukan dimulai (Lihat juga ini dan ini untuk informasi lebih lanjut). Namun, jika nilai Debugger di subkunci dari kunci ini menunjuk ke file yang tidak ada, setiap upaya untuk memulai file EXE dengan nama yang sama dengan subkunci akan gagal dengan kesalahan "file tidak ditemukan".

\n\n

Jika Anda melihat subkunci yang seharusnya tidak ada di sana, hapus subkunci tersebut (setelah membuat cadangan konten kunci registri). Dalam penginstalan Windows 7 yang bersih, ada dua subkunci di bawah kunci ini:

\n\n
    \n
  • DllNXOptions - berisi daftar DLL yang diketahui tidak kompatibel dengan DEP;
    • \n
  • IEInstal.exe - berisi ExecuteOptions=0x00000000 (nilai ini tampaknya tidak didokumentasikan).
    • \n
\n\n

Beberapa malware mungkin menggunakan kunci ini untuk menonaktifkan komponen sistem penting (mis., explorer.exe) dan program antimalware, dan terkadang program antimalware menghapus file yang dapat dieksekusi malware, tetapi membiarkan kunci registri di bawah Image File Execution Options menunjuk ke lokasi file tersebut, yang akan mengakibatkan kesalahan Anda sedang melihat.
