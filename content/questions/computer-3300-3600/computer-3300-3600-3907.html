"Pembaruan Server Windows" yang Anda dengar mungkin adalah Layanan Pembaruan Server Windows (WSUS). Jika Anda memiliki sejumlah klien yang tidak terhubung ke internet, tetapi dapat berbicara dengan server daripada di internet, ini akan menjadi pilihan yang bagus.

\n\n

Anda akan membutuhkan mesin yang menjalankan OS Windows Server, menurut Persyaratan sistem Anda dapat kembali ke Windows Server 2003 (Windows 8 bukan OS server sehingga tidak akan berfungsi di sana). Saya tidak tahu apakah ada persyaratan lain seperti DNS dan Active Directory, dan jika semua itu berlaku untuk situasi Anda.

\n\n

Gunakan tautan di atas dan Google untuk mencari tahu cara menyiapkan server dan klien. Hasil akhirnya adalah server mengunduh pembaruan dari Internet dan klien mendapatkan pembaruan dari server.

\n\n

\n\n

Sedangkan untuk mengunduh dan menginstalnya secara manual, jawaban David adalah opsi terbaik.

Anda mungkin ingin mencobanya WSUS Offline .

\n\n

Ide dasarnya adalah bahwa itu akan membuat salinan offline dari serangkaian pembaruan yang dipilih sebelumnya untuk berbagai versi Windows target. Anda kemudian dapat mengambil salinan offline itu ke mesin target dan menerapkannya.

\n\n

\n\n
    \n

  1. Setelah mengunduh, jalankan UpdateGenerator.exe dan pilih paket yang ingin Anda unduh.

    \n\n

    \n\n

    Proses ini akan memakan waktu cukup lama ...

    2. \n

  2. Sekarang Anda dapat menyalin folder client ke media penyimpanan eksternal (sebagai alternatif, Anda dapat menggunakan file Media USB opsi di dalam generator pembaruan).

    3. \n

  3. Pada mesin target, jalankan UpdateInstaller.exe dan pilih apa yang akan diperbarui / dipasang. Pembaruan Windows "Normal" akan selalu diinstal.

    \n\n

    4. \n
