Saya mencoba memahami perbedaan antara aplikasi dan program di Windows 10.

\n
\n\n

Perbedaan antara keduanya terutama pada platform pemrograman:

\n\n
    \n

  • "Aplikasi" menggunakan Platform Windows Universal (UWP), diperkenalkan dengan Windows 10.

    • \n

  • Aplikasi "Desktop" klasik biasanya menggunakan yang tradisional Win32 / COM API atau mungkin yang lebih baru .NET Framework (meskipun Microsoft secara teknis memasukkan UWP dalam daftar ini ).

    • \n
\n\n

Pertimbangan Praktis

\n\n

Dibandingkan dengan Apps, program desktop tradisional umumnya lebih kuat dan tidak dibatasi secara keseluruhan. Karena Aplikasi dimaksudkan untuk menjadi lintas platform di antara produk Microsoft (PC / Xbox / Tablet / Telepon) dan pertimbangan perangkat keras / keamanan sangat berbeda antara perangkat ini, ada batasan bawaan dalam platform UWP yang berpotensi membatasi Aplikasi apa yang dapat lakukan dan seberapa baik mereka melakukannya.

\n\n

Ini kurang penting dengan jenis program ringan tertentu dan menjadi lintas platform bahkan dapat bermanfaat dalam beberapa kasus. Misalnya, ketika Microsoft mematikan Gadget desktop di Windows, Gadget Radio Pandora pada dasarnya dihidupkan kembali sebagai Aplikasi yang ditujukan untuk ponsel yang juga dapat digunakan pada Windows 10.

\n\n

Namun, dengan aplikasi desktop yang lebih "serius" (pikirkan game PC), UWP mulai kehilangan keuntungannya dan bahkan mungkin mulai tumbuh beberapa duri. Tingkat kontrol dan daya pemrograman yang tersedia untuk mesin game khusus berbasis Win32 tidak mudah direplikasi oleh Aplikasi karena platform UWP dan Win32 diimplementasikan dengan sangat berbeda.

\n\n

Lalu ada juga potensi masalah "kompromi" App.

\n\n

Aplikasi yang menggunakan UWP dapat "diperpanjang" untuk memanfaatkan kekuatan perangkat keras yang berbeda ... tetapi ini tidak selalu diinginkan. Mungkin saja berakhir dengan program yang bekerja jauh berbeda saat dijalankan misalnya. desktop vs. tablet, meskipun (bisa dibilang) aplikasi yang "sama".

\n\n

Sebagai solusi untuk hal semacam ini, mendapatkan program untuk bekerja dengan baik pada perangkat keras yang paling tidak kuat / paling terbatas seringkali menjadi prioritas. Jadi, program yang diimplementasikan sebagai Aplikasi mungkin membatasi ketepatan grafisnya secara keseluruhan untuk meminimalkan perbedaan visual. Demikian pula, mungkin sekumpulan kontrol di layar yang dirancang untuk antarmuka sentuh mungkin tidak berfungsi dengan baik dengan mouse ( batuk ... Tombol Windows 8 ... batuk ).

\n\n

Aplikasi desktop sering menghadapi lebih sedikit masalah ini karena spesifikasi PC (laptop atau lainnya) cenderung kurang bervariasi dalam perangkat keras generasi tertentu.

\n\n

Barang Lainnya

\n\n

Ada beberapa hal lagi yang perlu diperhatikan.

\n\n

Pertama, meskipun tersedia di PC tradisional atau bahkan konsol, Aplikasi ditargetkan tepat pada ruang seluler. Microsoft telah melakukan upaya bersama untuk membuatnya hanya tersedia dari tokonya, seperti yang dilakukan Apple untuk produknya atau monopoli de facto yang dimiliki Google di Android. Sebaliknya, program desktop masih tersedia dari berbagai sumber.

\n\n

Kedua, meskipun tampak kecil, ikon adalah perbedaan. Tidak hanya dengan ubin langsung dan tempat ikon hidup secara alami, tetapi juga kemampuan untuk menghapusnya dalam keadaan tertentu. Ikon aplikasi desktop dapat dihapus dari menu Semua Aplikasi, sedangkan Aplikasi tidak bisa. Jika Anda ingin menghapus ikon Aplikasi dari menu Semua Aplikasi, Anda harus menghapusnya.

\n\n
\n\n

Lebih luas lagi, Aplikasi secara umum diinstal dan tinggal di folder tetap yang benar-benar terpisah yang tidak dimaksudkan untuk diakses oleh pengguna. Program Win32, di sisi lain, mempertahankan kemampuan untuk diinstal hampir di mana saja dan lebih mudah diakses oleh siapa saja yang menggunakan perangkat tertentu.
