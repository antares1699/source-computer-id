RDP sebelum Windows 10 memiliki driver grafisnya sendiri untuk mengonversi hasil render\nlayar ke dalam paket jaringan untuk dikirim ke klien, yang digunakan secara eksklusif CPU. Window 8 adalah yang pertama mulai menggunakan GPU.

\n\n

Sejak Windows 10 build 1511 dan Windows Server 2016, RDP menggunakan AVC / H.264\ncodec untuk mendukung layar yang lebih besar daripada full HD.\nCodec ini menggunakan GPU,\ntetapi hanya dalam kondisi tertentu dan untuk sesi desktop penuh,\ntetapi sebaliknya kembali menggunakan CPU seperti sebelumnya.

\n\n

Menggunakan AVC / H.264 sekarang adalah default, tetapi Anda dapat menonaktifkannya menggunakan\nEditor Kebijakan Grup (gpedit.msc) dan menelusuri ke:
\n Konfigurasi Komputer -> Template Administratif -> Komponen Windows -> Layanan Desktop Jarak Jauh -> Host Sesi Desktop Jarak Jauh -> Lingkungan Sesi Jarak Jauh .

\n\n

Tetapkan kebijakan berikut ke Dinonaktifkan,\nuntuk menonaktifkan penggunaan codec AVC / H.264:

\n\n
    \n
  • Konfigurasikan pengkodean perangkat keras H.264 / AVC untuk koneksi Desktop Jarak Jauh
    • \n
  • Prioritaskan mode Grafik H.264 / AVC 444 untuk koneksi Remote Desktop
    • \n
\n\n

Bagaimanapun, sesi desktop yang tidak penuh seharusnya tidak saat ini\nmenggunakan GPU (tapi ini bisa berubah tanpa pemberitahuan).

\n\n

Referensi :

\n\n
    \n

  • Remote Desktop Protocol (RDP) 10 peningkatan AVC / H.264 di Windows 10 dan Pratinjau Teknis Windows Server 2016

    • \n

  • Konfigurasikan pengkodean perangkat keras H.264 / AVC untuk Koneksi Desktop Jarak Jauh

    • \n
\n\n

Referensi terakhir berisi teks ini:

\n\n
\n

Pengaturan kebijakan ini memungkinkan Anda mengaktifkan dukungan encoding perangkat keras H.264 / AVC untuk Koneksi Desktop Jarak Jauh. Saat Anda mengaktifkan pengkodean perangkat keras, jika terjadi kesalahan, kami akan mencoba menggunakan pengkodean perangkat lunak. Jika Anda menonaktifkan atau tidak mengonfigurasi kebijakan ini, kami akan selalu menggunakan encoding perangkat lunak.

Desktop Windows, terlepas dari itu digunakan oleh RDP atau secara lokal selalu menggunakan beberapa sumber daya GPU pada GPU utama.
Itu hanya dirancang seperti itu.
Satu-satunya cara yang saya tahu untuk menggunakan kedua GPU Anda yang didedikasikan untuk proyek pembelajaran mesin Anda adalah dengan menambahkan GPU ke-3 dan memastikan bahwa itu adalah yang utama. Ini bisa menjadi GPU dasar yang sangat murah atau grafis Intel HD bawaan jika CPU Anda kebetulan memilikinya.
Anda mungkin harus mengutak-atik pengaturan Bios dan / atau urutan GPU di slot PCIe untuk mendapatkan efek yang diinginkan terkait urutan kartu. Jika menggunakan grafis Intel HD, biasanya ini secara otomatis menjadi GPU utama, tetapi beberapa motherboard tidak akan mengaktifkan GPU Intel sama sekali jika ada GPU lain.
