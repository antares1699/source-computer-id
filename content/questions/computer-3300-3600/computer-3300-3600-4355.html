Apakah Windows XP melakukan validasi checksum setelah salinan file? Jika saya menyalin semua gambar dan video saya dari satu disk ke disk lain menggunakan penyalinan file Windows, apakah praktik terbaik untuk memverifikasi checksum file baru terhadap yang lama untuk memverifikasi tidak ada korupsi?

Jika Anda khawatir tentang itu, coba teracopy, secara otomatis melakukan ini, ditambah lebih cepat dari windows 'dan mendukung resume, dapat melewati file yang rusak (dan melaporkannya, tidak berhenti seperti windows) dan juga gratis, dan itu akan digunakan sebagai manajer penyalinan default jika Anda mau.

Ada perhitungan checksum di sana-sini saat menyalin file (paket IP memiliki checksum, perangkat blok tertentu seperti CD atau DVD memiliki ECC) tetapi sebenarnya tidak ada yang akan memberikan perlindungan ujung ke ujung. Bahkan ZFS (yang memiliki verifikasi CRC built-in) rentan terhadap kegagalan ketika sistem Anda membaca data yang benar kemudian menulis data yang salah (bug perangkat lunak, kesalahan bit cache, dll). Saya pernah memiliki HDD yang memiliki dua bit buruk dalam RAM-nya, menyebabkan masalah korupsi data yang sangat langka tetapi sangat parah. Anda melakukan kebaikan bagi diri Anda sendiri jika Anda memelihara database checksum (atau lebih tepatnya, CRC atau hash) untuk file yang ingin Anda simpan untuk waktu yang lama.
