Anda mendapatkan kesalahan ini karena git tidak diinstal pada /usr/local/git. Cari tahu di mana letaknya dengan which git, dan rm dengan lokasi yang benar.

\n\n

Jika Anda memiliki manajer paket (brew, fink atau macports), Anda akan ingin menggunakan antarmukanya untuk menghapusnya, jika itu digunakan untuk instalasi.

\n\n

EDIT setelah komentar Anda

\n\n

/opt/local/bin adalah lokasi perangkat lunak yang diinstal oleh macports , jadi Anda ingin menjalankan ini: 

sudo port uninstall git\n

Bahkan jika Anda telah menghapus dengan tangan (yang seharusnya tidak Anda miliki, halaman asli menyesatkan), itu harus dicopot sepenuhnya

Anda tidak perlu menghapus versi lama. Ambil tarbal sumbernya, keluarkan ke dalam direktori home Anda, dan bangunlah. Sebagai contoh: 

\n$ cd $HOME\n$ tar zxf git-x.y.z.tar.gz\n$ cd git-x.y.z\n$ ./configure --prefix=$HOME\n$ make && make install  # (Can probably just do \"make install\")\n

Itu memberi Anda versi yang sangat bagus yang dipasang di $ HOME. Taruh $ HOME / bin\ndi PATH Anda dan Anda siap melakukannya.
