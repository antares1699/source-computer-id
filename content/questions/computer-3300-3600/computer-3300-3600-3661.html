\"repositories\": [\n  \"type\": \"package\",\n    \"package\": {\n        \"name\": \"nodejs/nodejs\",\n        \"version\": \"0.12.6\",\n        \"dist\": {\n        \"type\": \"gzip\",\n        \"url\": \"https://nodejs.org/dist/v0.12.6/node-v0.12.6-linux-x64.tar.gz\"\n      }\n    }\n  }\n],\n\"require\": {\n  \"nodejs/nodejs\": \"0.12.*\"\n}\n

Masalahnya, saya bisa menggunakan tar atau gzip untuk membongkar archieve. 

Available types: git, svn, hg, perforce, zip, rar, tar, gzip, phar, file.\n

Itu "ter" Pilihan memahami keduanya telanjang .tar, .tar.gz, dan bahkan .tar.bz2 jadi cukup tentukan \"type\": \"tar\" dan tidak perlu menentukan \"gz\". Ini menghitung dekompresi itu sendiri.

\n\n

Saya mengalami masalah ini, dan awalnya juga bingung, mencoba menerapkan otomatis aplikasi php yang menggunakan pustaka berpemilik. Namun saya hanya mencoba "tar", itu berhasil! Bagian repositories JSON saya terlihat sedikit konyol karena struktur arsip tar yang tidak biasa, tetapi berfungsi. Itu terlihat seperti ini: 

\"repositories\": [\n  {\n    \"type\": \"package\",\n    \"package\": {\n      \"name\": \"RocketShipIt/RocketShipIt\",\n      \"version\": \"1.4.9.3\",\n      \"dist\": {\n        \"url\": \"https://secret-deployment-server/RocketShipIt.tar.gz\",\n        \"type\": \"tar\"\n      },\n      \"autoload\": {\n        \"files\": [\n          \"RocketShipIt/RocketShipIt.php\"\n        ],\n        \"psr-0\": {\n          \"RocketShipIt\\\\\": \"RocketShipIt/RocketShipIt/RocketShipIt/\"\n        }\n      }\n    }\n  }\n]\n

Anda dapat menemukan, agak tidak jelas di file Dokumentasi Composer API (bukan manual pengguna akhir) bahwa .tar, .tar.gz dan .tar.bz2 ketiganya secara resmi didukung oleh \"type\": \"tar\". Rupanya, ini ditangani secara internal oleh komposer oleh kelas Composer\\Downloader\\TarDownloader.
