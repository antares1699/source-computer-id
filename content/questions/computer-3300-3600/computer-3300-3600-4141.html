Perusahaan saya mengamanatkan Enkripsi Perangkat Lunak untuk semua laptop.

\n\n

Saya menyeret kaki saya karena saya tidak ingin ada apa pun antara saya dan data saya. Tapi saya akhirnya didorong ke dalamnya.

\n\n

Saya menginstalnya dan saya belum melihat banyak perbedaan (yay!). Saya ingin mengukur perbedaan sebenarnya (jika ada) yang ada.

\n\n

Jadi saya mencatat indeks "Kecepatan transfer data disk" hard disk Primer saya dari Windows Experience sebelum dan setelah saya menginstal enkripsi.

\n\n

Yang mengejutkan saya jumlahnya sama (5.9).

\n\n

Jadi saya bertanya-tanya apakah memang tidak ada perbedaan, atau apakah "Laju data" sama karena mengukur setelah enkripsi selesai.

\n\n

Adakah yang tahu mengapa nilainya akan sama?

Jika Anda memiliki CPU terbaru (dan perangkat lunak enkripsi ditulis dengan cerdas (sebagaimana mestinya)) itu harus menggunakan ekstensi cpu Anda (tidak tahu yang mana, tapi saya tahu Intel Core i3, i5, i7 memilikinya) untuk memudahkan enkripsi proses. (Tanpa ekstensi itu berfungsi sama juga, tetapi kinerjanya turun karena CPU memiliki lebih banyak pekerjaan yang harus dilakukan, tetapi tidak terlalu banyak).\nTapi ... HDD klasik lambat, CPU cepat, itu berarti kinerja tertinggi yang dicapai ada di disk, bukan di CPU, bahkan ketika Anda menyalin file X GB dan mengenkripsinya sekaligus.

\n\n

Ketika Anda memiliki disk SSD, casingnya sangat berbeda. Penurunan kinerja lebih tinggi dibandingkan dengan throughput baca / tulis SSD mentah.

\n\n

[Saya telah membaca ini ketika saya mempelajari cara kerja FileVault 2 di Mac saya dan, seperti Anda, saya mengkhawatirkan kinerja.]

\n\n

Juga, itu harus melakukan enkripsi ketika CPU dalam keadaan idle. Jadi ketika Anda melakukan pekerjaan normal (dan tidak menyalin file X GB misalnya) Anda seharusnya tidak melihat penurunan kinerja (bahkan jika Anda tidak memiliki ekstensi pendukung tersebut) sama sekali.

Harap dicatat bahwa 5,9 saat ini adalah nilai semaksimal mungkin untuk Windows Experience Index. Misalnya. 5.9 hanya berarti "lebih cepat dari XXX" untuk beberapa nilai XXX. Anda dapat mengalami penurunan kinerja yang nyata, dan masih memiliki indeks disk 5,9 jika kinerja Anda masih di atas XXX.

\n\n

Selain itu, enkripsi perangkat lunak berarti penggunaan CPU yang lebih tinggi selama penulisan dan pembacaan disk (biasanya data perlu dienkripsi / didekripsi). Jadi, jika CPU Anda sedang / tidak aktif, mungkin ada overhead enkripsi yang sangat sedikit, tetapi jika prosesor Anda terisi penuh, kinerja disk mungkin turun secara signifikan.

\n\n

Menurut pengalaman saya, enkripsi disk penuh yang dilakukan dengan benar tidak mempengaruhi sistem laptop modern secara signifikan, ketika bebannya adalah program kantor biasa.

\n\n

Tetapi jika Anda membangun proyek perangkat lunak besar, beberapa kompilasi paralel, file keluaran besar, dll., Waktu pembuatan jauh lebih lama.
