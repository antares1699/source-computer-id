Saya memiliki pengguna yang melakukan perjalanan ke China baru-baru ini. Karena mereka kembali, mencoba menavigasi ke salah satu bookmark mereka akan membawa mereka ke url ini:

\n\n

http://nfdnserror1.wo.com.cn:8080/issueunziped/nf20140811/index.html?UserUrl=<the URL>

\n\n

Halaman ini pada dasarnya hanyalah mesin pencari Cina Baidu, dengan bidang pencarian diisi dengan string kueri UserUrl. URL sepertinya seharusnya halaman kegagalan pencarian DNS kustom.

\n\n

Penanda tampaknya belum dimodifikasi. Menavigasi langsung ke URL juga mengarahkan ke halaman ini. Sepertinya hanya URL di bookmark yang terpengaruh, seperti yang diilustrasikan di bawah ini:

\n\n
\n\n

Tidak OK (ada di bookmark)

\n\n

http://<internal server name>/<subsite name>/

\n\n

baik

\n\n

http://<internal server name>/

\n\n

http://<internal server FQDN>/<subsite name>/

\n\n
\n\n

Masalahnya diisolasi ke IE11 dan akun pengguna tertentu itu. Chrome dan Firefox tidak memiliki masalah sama sekali, dan IE11 di akun lokal terpisah juga tidak mengalami masalah.

\n\n

OS adalah Windows 7 Pro x64.

\n\n

Saya telah memeriksa dan melakukan hal berikut:

\n\n
    \n
  • Pengaturan DNS sudah benar
    • \n
  • Hapus cache DNS
    • \n
  • File host baik-baik saja
    • \n
  • Tidak ada plugin IE tambahan
    • \n
  • Setel ulang IE (Opsi Internet -> Lanjutan -> Setel ulang IE)
    • \n
  • HijackThis tidak menangkap apa pun yang berhubungan dengan ini
    • \n
  • Malwarebytes mengambil beberapa kunci registri yang tampaknya tersisa dari beberapa bilah alat yang dipasang secara tidak sengaja, tetapi mengkarantina mereka tidak melakukan apa-apa
    • \n
  • Bookmark baru tidak mengalami masalah ini
    • \n
  • Menghapus bookmark lama dan menavigasi ke URL masih menghasilkan masalah
    • \n
  • Tidak ada proses mencurigakan yang sedang berjalan atau layanan baru terpasang
    • \n
  • Tidak ada folder Baidu di salah satu folder Program Files
    • \n
  • Baidu toolbar tidak pernah diinstal
    • \n
  • Memeriksa bahwa tidak ada kumpulan server proxy
    • \n
  • Memeriksa MSconfig, tidak ada program atau layanan startup yang tidak diharapkan
    • \n
  • Menjalankan Autoruns Sysinternals, tetapi tidak ditemukan sesuatu yang mencurigakan
    • \n
\n\n
