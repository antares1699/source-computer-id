Mengenai cara Firefox menghasilkan pratinjau Situs Teratas, MDN negara bagian :

\n\n
\n
    \n
  1. Daftar "situs teratas" global dicentang. Daftar ini dibuat oleh layanan Mozilla bernama Tippy Top . Penyimpanan data disediakan oleh\n Tippy Top menyertakan ikon yang dioptimalkan untuk setiap situs di\n daftar; jika situs ada di daftar ini, ikon itu digunakan.
    2. \n

  2. Jika Tippy Top tidak menyertakan situs tersebut, Firefox akan mencari di penyimpanan data Places untuk ikon yang ditentukan dalam elemen halaman <head>;

    \n \n

    Sebuah. Jika ikon SVG tersedia, ikon itu dipilih.

    \n \n

    b. Jika tidak, ikon yang dipetakan dengan lebar setidaknya 96 piksel berukuran\n dipertimbangkan. Jika ditemukan memiliki lebar persis 96 piksel, itu\n digunakan. Sebaliknya, gambar terkecil yang lebih besar dari 96 piksel adalah\n terpilih.

    \n \n

    c. Jika tidak ada ikon resolusi tinggi yang ditemukan, tangkapan layar diambil dari\n situs; tangkapan layar ini kemudian diskalakan dan dipotong agar pas dengan ruangnya\n dialokasikan untuk ikon. Jika situs tersebut juga memiliki favicon, mungkin saja\n terhampar di sudut gambar.

    3. \n
\n
\n\n
\n\n
\n

Ikon resolusi tinggi — juga dikenal sebagai ikon kaya — ditentukan oleh\n mengatur atribut rel ke salah satu dari:

\n \n
    \n
  • apple-touch-icon
    • \n
  • apple-touch-icon-precomposed
    • \n
  • fluid-icon
    • \n
\n \n

Ukuran setiap ikon diambil dari atribut size yang ditentukan pada\n <link>, jika disediakan. apple-touch-icon dan\n Ikon apple-touch-icon-precomposed tanpa atribut size adalah\n diasumsikan sebagai ukuran iPhone klasik standar 57x57 piksel.

\n \n

Ikon yang ukurannya paling dekat dengan 96 piksel dipilih. Firefox melakukannya\n terbaik untuk menghindari keharusan menskalakan gambar, yang dapat mengakibatkan kabur atau\n gambar terdistorsi, terutama pada ukuran yang lebih kecil.

\n
\n\n

Ada catatan yang membantu saya memperbaiki masalah saya dengan ikon besar yang tidak ditampilkan, yang disebabkan oleh hanya memiliki favicon.ico meskipun itu termasuk ukuran dari 16x16 hingga 256x256:

\n\n
\n

Catatan : Hanya ikon "kaya" (apple-touch-icon dan fluid-icon) yang mendukung\n sizes atribut. Anda tidak dapat menentukan ukuran untuk rel=\"icon\" menggunakan\n sizes atribut.

Firefox menggunakan ikon perangkat seluler untuk situs web untuk menampilkan ikon besar, dan ini ditetapkan oleh perancang / pemilik situs web di tajuk (bagian kepala) situs web melalui tag meta - awalnya ditujukan untuk saat Anda menyimpan / menandai situs web di perangkat seluler, untuk ditampilkan sebagai ikon aplikasi.

\n\n

Mengapa tim pengembang Firefox memutuskan ini adalah cara terbaik untuk mewakili situs web, saya tidak akan pernah tahu. Tetapi hasilnya adalah situs web yang tidak menampilkan ikon aplikasi seluler ini ditampilkan sebagai cuplikan situs web, dengan favicon situs web di kanan bawah gambar mini snapshot.

\n\n

Jadi semuanya tergantung pada apakah situs web telah dirancang untuk ditampilkan sebagai aplikasi dalam penggunaan seluler atau tidak. Semoga ini bisa membantu untuk memahaminya - meskipun itu adalah pilihan desain yang konyol.
