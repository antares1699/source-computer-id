Setelah memutakhirkan ke windows 10 v1511, sakelar Hyper-V yang saya miliki telah pergi ke mana-mana.

\n\n

Selain dinonaktifkan, dan tidak dapat memulai, manajer koneksi jaringan menunjukkan sakelar dengan nama berbeda "Ethernet 2" (deskripsi mengatakan itu adalah sakelar hyper-v). Ketika saya mencoba untuk menghapus sakelar melalui manajer Hyper-V, saya mendapatkan Kesalahan: 

Failed while removing virtual Ethernet switch. Internal miniport delete  failed, internal NIC = 'somenicidthatstoolong': Invalid parameter (0x80041008)\". The switches name is \"vEthernet\".\n

Karena itu tidak berhasil, saya mencoba menghapus vmswitch melalui PowerShell. 

remove-vmswitch -Name \"vEthernet\" -force\nremove-vmswitch -Name \"Ethernet 2\" -force\n

Keduanya memberi saya kesalahan lain: 

remove-vmswitch : Hyper-V was unable to find a virtual switch with name \"vEthernet\".\nremove-vmswitch : Hyper-V was unable to find a virtual switch with name \"Ethernet 2\".\n

Karena tidak ada yang berfungsi, saya mencoba membuat daftar semua VMSwitch menggunakan get-vmswitch ... yang tidak menghasilkan apa-apa.

\n\n

Jadi pada akhirnya kita memiliki vmswitch yang tidak ada, yang masih ada di manajer hyper-v, dan juga di Control Panel \\ Network dan Internet \\ Network Connections dengan nama yang berbeda.

\n\n

Bagaimana cara menghapus entri ini sehingga saya dapat membuat VMSwitch baru?

Baiklah, jadi bagi siapa pun yang memiliki masalah ini, inilah solusinya.

\n\n

PENAFIAN: KEMUNGKINAN ANDA AKAN KEHILANGAN KONEKTIVITAS JARINGAN PADA PC / SERVER HOST HINGGA LANGKAH AKHIR

\n\n

Karena Anda tidak dapat menghapus / menghapus pengalih Hyper-V 

Control Panel\\Network and Internet\\Network Connections\n

Satu-satunya pilihan yang tersisa adalah membuka pengelola perangkat. 

Device Manager > Your PC > Network adapters\n

Dalam daftar ini Anda akan menemukan Hyper-V Switch, tetapi kemungkinan besar TIDAK memiliki nama yang Anda berikan. Tentukan sakelar jaringan mana yang merupakan Sakelar Hyper-V, klik kanan dan hapus instalannya. Di sinilah konektivitas jaringan akan hilang.

\n\n

Setelah menghapus pengalih dari pengelola perangkat, catatan akan tetap tersisa di Pengelola Pengalihan Virtual Hyper-V Manager Anda. Namun, bersukacitalah, karena sekarang perangkat telah dihapus secara manual, Anda sekarang dapat menghapus sakelar virtual dari Virtual Switch Manager tanpa kesalahan.

\n\n

Sekarang, saat mengkonfigurasi Hyper-V Switch, adaptor fisik sebenarnya disetel sebagai Hyper-V Extensible Switch. Karena kami melepas adaptor secara manual, Virtual Switch Manager TIDAK mengembalikan sakelar ke pengaturan aslinya. Kembali ke 

Control Panel\\Network and Internet\\Network Connections\n

dan klik kanan pada adaptor jaringan utama Anda, tekan properti, dan hapus centang pada "Hyper-V Extensible Virtual Switch". Setelah melakukannya, Anda akan melihat kotak centang lain terisi secara otomatis, jika tidak karena suatu alasan, periksa dan periksa apa yang Anda butuhkan.

\n\n

Tekan OK, konfirmasikan pemberitahuan yang akan mengatakan Anda mungkin mengalami gangguan jaringan, dan Anda kembali memiliki internet di PC / Server Anda dan sekarang dapat membuat Ulang Sakelar Virtual Anda untuk Mesin Virtual Hyper-V Anda.
