Dengan asumsi itu Nomor Rekam MFT , Anda dapat menggunakan alat MS bernama nfi untuk membuang informasi ntfs dan kemudian utilitas seperti grep atau awk ( perlu diinstal / bukan kemenangan asli ) untuk memfilter hasil yang Anda minati saja.
nfi dapat diunduh sebagai bagian dari Alat OEM

nfi c: |awk \"/File 72\\r/{c=4;{print}next}c-->0\" menghasilkan output berikut di laptop saya:

\n\n
\n

Berkas 72
\\ WINDOWS \\ Temp
$ STANDARD_INFORMATION (penduduk)
$ FILE_NAME (penduduk)
$ INDEX_ROOT $ I30 (penduduk)

\n
\n\n

bagi mereka yang mungkin mencari tahu tentang ini di masa mendatang: Saya baru tahu bahwa di Linux, ada yang bisa 

ntfsls -iaR /dev/sdb1\n

-saya = show inode (mis., record MFT no.), -Sebuah = tampilkan semua (tidak tahu apa fungsinya tetapi tidak membahayakan), -R = recurse ke subdirektori, / dev / sdb1 adalah file perangkat partisi atau file image NTFS.

\n\n

Ini menghasilkan daftar file yang bagus dan panjang serta nomor catatan MFT mereka.

\n\n

ntfsls ada dalam paket ntfsprogs (setidaknya di Fedora.)

\n\n
