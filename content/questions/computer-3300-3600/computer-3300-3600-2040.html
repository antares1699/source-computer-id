Jika setidaknya satu kartu jaringan komputer mendukung penyeberangan otomatis, Anda dapat menghubungkan keduanya dengan kabel langsung. (Auto-crossover, juga disebut "auto-MDIX", adalah bagian dari standar Gigabit Ethernet, tetapi sangat sering didukung oleh kartu 100 Mbps juga.) Jika tidak, Anda memerlukan kabel cross-over, hub, atau sebuah saklar.

\n\n

Setelah mengatur koneksi fisik, konfigurasikan jaringan di OS:

\n\n
    \n

  • Jika kedua ujungnya punya IPv6 , Anda cukup menyambungkan menggunakan komputer lain alamat link-local ; sebagai contoh, sshÂ fe80::4a5d:60ff:fee8:658f%eth0 . (Di Linux, string setelah % adalah nama kartu Ethernet Anda, sedangkan Windows menggunakan ID antarmuka numerik dari netsh interface ipv6 show interface.)

    \n\n

    Anda juga dapat menambahkan alamat yang lebih pendek secara manual, seperti fc00::1 dan fc00::2 (dari fc00::/7 jaringan ).

    • \n

  • Untuk Khusus IPv4 tuan rumah, tetapkan alamat IP secara manual dari subnet yang sama - misalnya, 10.0.0.1/8 dan 10.0.0.2/8.

    \n\n

    Beberapa sistem operasi akan secara otomatis menetapkan alamat IPv4 tautan-lokal dalam kisaran 169.254.0.0/16.

    • \n

Saya akan merekomendasikan Anda menggunakan file rsync server di salah satu mesin untuk menghindari pemrosesan enkripsi ssh yang mungkin memberi Anda throughput yang lebih cepat, terutama pada CPU berkecepatan rendah:

\n\n

Di mesin "server", buat sendiri file rsyncd.conf yang berisi sesuatu seperti: 

[download]\ncomment =       Storage general files\npath =          /home/user/Download\nread only =     false\nwrite only =    false\nlist =          true\nincoming chmod = Du=rwx,go=rx,Fu=rw,go=r\nuid =           user\ngid =           user\n

Kemudian jalankan rsync dalam mode "server": 

sudo rsync --daemon --config=rsyncd.conf  --no-detach --log-file=/dev/tty\n

Kemudian pada klien, untuk menerima: 

rsync rsync://host/download/file-name local-file-name\n

atau untuk mengirim: 

rsync local-file-name rsync://host/download/file-name\n

Setelah selesai, cukup CTRL-C proses server.

Versi singkatnya adalah:

\n\n
    \n
  • Jika setidaknya salah satu komputer memiliki antarmuka gigabit, atau salah satu yang mendukung MDI / MDX tidak diperlukan router. Colokkan CAT5E biasa atau kabel yang lebih baik.
    • \n
  • Dari keduanya memiliki kartu 10 / 100mbit menggunakan kabel silang atau dua kabel biasa dan satu saklar.
    • \n
\n\n

Selanjutnya konfigurasikan kedua komputer untuk berada di subnet yang sama dan atur perutean. (Lihat ini di situs saudara kami: bagaimana-cara-ipv4-subnetting-kerja untuk latar belakang).

\n\n

Anda tidak menyebutkan OS mana yang Anda gunakan, tetapi dengan asumsi unix seperti:

\n\n
    \n
  • Komputer 1: ifconfig em0 192.168.0.1 netmask 255.255.255.0 (ganti em0 dengan nama antarmuka).
    • \n
  • Komputer 2: ifconfig em0 192.168.0.2 netmask 255.255.255.0
    Kedua kartu jaringan sekarang dikonfigurasi untuk LAN yang sama.
    • \n
  • OS Anda kemungkinan akan 'secara otomatis' menambahkan rute ke jaringan lokal. Jika tidak menggunakan route add -net 192.168.0.0 mask 255.255.255.0 192.168.0.1 di komputer 1. Lakukan hal yang sama di komputer 2 dengan route add -net 192.168.0.0 mask 255.255.255.0 192.168.0.2
    • \n
\n\n

Uji koneksi dengan ping atau salin dengan scp.

\n\n

(Klien: ~ homedir> scp myfile 192.1.68.0.2:Destination_folder)

Anda harus menyiapkan jembatan di server (S) dengan jaringan biasa dan kemudian menghubungkan klien (C) ke jaringan tersebut (berdasarkan IP) atau menggunakan subnet khusus. Yang terakhir cukup mudah (dengan eth0 menjadi antarmuka yang terhubung di klien dan server): 

C # ip addr add 10.0.1.2/24 dev eth0\nS # ip addr add 10.0.1.1/24 dev eth0\n

pada titik ini, C # ping 10.0.1.1 seharusnya berfungsi. Jika server SSH di S dikonfigurasi untuk mendengarkan di semua antarmuka (yaitu 0.0.0.0), Anda harus siap untuk pergi, jika tidak, Anda harus membuatnya mendengarkan di sana (dengan mengadaptasi /etc/ssh/sshd_config).

\n\n

Jika Anda tidak mendapatkan kecepatan yang Anda harapkan, pastikan (melalui ethtool eth0) bahwa kecepatan koneksi telah terdeteksi dengan benar. Anda dapat mengubahnya menggunakan alat yang sama.
