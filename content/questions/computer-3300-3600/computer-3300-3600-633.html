Ini dari manual untuk GNU find: 

   -exec command ;\n          Execute command; true if 0 status is returned.\n          All following arguments to find are  taken  to\n          be  arguments to the command until an argument\n          consisting of `;'\n

Dipotong oleh saya. Tapi kami juga punya: 

   -exec command {} +\n          This variant of  the  -exec  action  runs  the\n          specified  command  on the selected files, but\n          the command line is built  by  appending  each\n          selected  file name at the end; the total num‚Äê\n          ber of invocations of the command will be much\n          less  than  the  number of matched files.  The\n          command line is built in  much  the  same  way\n          that xargs builds its command lines.\n

Dipotong oleh saya.

\n\n

Jadi, ; dan + melakukan hal yang berbeda. Contoh singkat dalam kode untuk didemonstrasikan: 

$ mkdir test\n$ touch test/{a,b}\n$ find test -exec echo foo {} \\;\nfoo test\nfoo test/a\nfoo test/b\n$ find test -exec echo foo {} +\nfoo test test/a test/b\n

+ hanya menghasilkan satu pemanggilan dengan semua file yang cocok diberikan sebagai daftar argumen (dalam praktiknya berperilaku seperti xargs). ; menjalankan perintah satu kali untuk setiap file yang cocok (; harus di-escape di shell; oleh karena itu \\ sebelumnya).

\n\n

Perhatikan bahwa beberapa versi ; tidak memiliki opsi +, tetapi versi GNU kemungkinan besar diinstal pada Linux yang tidak tertanam.

TL; DR:

\n\n

Menggunakan + dalam -exec sama dengan -exec, kecuali bahwa {} diganti dengan nama jalur sebanyak mungkin untuk setiap pemanggilan utilitas .

\n\n


\n\n
