Saya telah menggunakan FFmpeg untuk mengekstrak semua I-frame dari file MKV / MP4 tetapi FFmpeg tampaknya memecahkan kode semua frame untuk melakukannya, jadi dibutuhkan waktu yang sangat lama jika file videonya 1080p dan lebih lama dari 10 menit. Saya hanya ingin itu melompat dari I-frame ke I-frame dan membuangnya ke file JPG / PNG. Selain itu, saya perlu mengetahui stempel waktu frame-I.

\n\n

Pilihan lainnya adalah menggunakan FFprobe untuk mendapatkan cap waktu untuk semua I-frame, tetapi itu juga menerjemahkan seluruh file.

\n\n

Saya mencoba melakukan sesuatu yang mirip dengan Avidemux. Di Avidemux Anda dapat melangkah ke setiap frame I dengan sangat cepat hanya dengan menekan tombol panah atas atau bawah, tetapi ia melakukannya tanpa mendekode semua frame B / P.

","source":"https://superuser.com/questions/1421133/extract-i-frames-to-images-quickly","author":"Hind-D","replies":[],"answers":[{"answer":"

Menggunakan 

ffmpeg -skip_frame nokey -i file -vsync 0 -frame_pts true out%d.png\n

skip_frame memberi tahu decoder untuk hanya memproses keyframe. -vsync 0 (dalam perintah ini) mempertahankan stempel waktu. -frame_pts menyetel bagian angka dari nama file gambar keluaran untuk mewakili stempel waktu. Penafsiran angka mengharuskan Anda mengetahui framerate, mis. jika framerate adalah 30 maka nama gambar dari 75 sesuai dengan stempel waktu 75/30 = 2,50 detik. Anda dapat menambahkan -r 1000 jika Anda ingin angka mewakili milidetik.
