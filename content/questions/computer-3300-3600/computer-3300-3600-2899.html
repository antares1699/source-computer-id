Saya telah melihat ini tapi itu tidak berhasil untuk saya. Saya tidak menggunakan bash versi Git tetapi yang disertakan dengan cygwin, jadi itu bisa menjadi bagian dari masalah. Berikut adalah cara saya mengatur tab saya di Console2: 

Title: bash.exe\nShell: C:\\cygwin\\bin\\bash.exe --login -i\nStartup dir: %HOMEDRIVE%%HOMEPATH%\n

Ini adalah entri registri saya: 

Directory\n    shell\n        Console2\n            Open Console2 Here\n            command\n                "C:\\Console2\\Console.exe" -d "%1"\n

bash selalu dimulai di direktori home saya alih-alih direktori yang saya klik, maka saya harus cd /cygdrive/d/code ... dll. Saya mencoba menghapus nilai di bidang "Startup dir", mengubahnya menjadi% 1,% 1% dengan tidak beruntung.

\n

Saya juga mencoba solusi ini tetapi itu juga tidak berhasil untuk saya: 

Title: bash.exe\nShell: C:\\cygwin\\bin\\bash.exe -l\n

Registri: 

Directory\n    shell\n        Console2\n            Open Console2 Here\n            command\n                "C:\\Console2\\Console.exe" -d "%V"\n

Solusi di atas berfungsi untuk versi bash.exe dan sh.exe yang disertakan dengan Git, tetapi bukan versi yang disertakan dengan cygwin karena alasan tertentu. Masih mencari alasan untuk ini.

Dari manual ,

\n\n
\n

Saat Bash dipanggil sebagai shell login interaktif, atau sebagai shell non-interaktif dengan opsi --login, Bash akan membaca dan menjalankan perintah dari file / etc / profile, jika file itu ada.

\n
\n\n

/etc/profile secara default berisi kode berikut: 

# Make sure we start in home unless invoked by CHERE\nif [ ! -z \"${CHERE_INVOKING}\" ]; then\n  unset CHERE_INVOKING\nelse\n  cd \"${HOME}\" || echo \"WARNING: Failed attempt to cd into ${HOME}!\"\nfi\n

Jadi salah satu solusinya adalah membuat file batch Windows yang menetapkan variabel lingkungan CHERE_INVOKING sebelum memulai bash. Itu tidak terlalu rapi.

\n\n

Lanjut,

\n\n
\n

Setelah membaca file itu, ia mencari ~ / .bash_profile, ~ / .bash_login, dan ~ / .profile, dalam urutan itu, dan membaca serta menjalankan perintah dari yang pertama ada dan dapat dibaca.

\n
\n\n

Skrip profil ini secara default menjalankan skrip ~/.bashrc Anda jika ada. Menempatkan cd - di akhir .bashrc saya berfungsi untuk saya (dengan konsol2 terbaru dan pesta Cygwin): 

echo 'cd -' >> ~/.bashrc\n

BTW, saya memanggil Console2 (Console.exe) tanpa argumen - Windows Explorer memastikan bahwa itu dimulai di direktori yang saya klik kanan. Saya menggunakan XP tetapi saya ragu MS akan mengubah perilaku ini di versi Windows yang lebih baru. Saya juga membiarkan opsi "startup dir" kosong untuk tab Bash saya.
