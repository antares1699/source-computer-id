Saya menjalankan VirtualBox di Mac (host), VM yang saya gunakan adalah windows 7 (guest).\nVirtualBox diatur untuk menggunakan adaptor jaringan NAT, dan saya bisa mengakses internet dengan baik (google, msn, semuanya) namun saya menghosting situs di Mac (host) dan saya tidak dapat mengaksesnya dari VM .

\n\n

Tamu diberi IP 10.0.2.10 dan IP host saya adalah 10.0.2.100, saya pikir masalahnya mungkin karena ini adalah jaringan yang terpisah, tetapi saya tidak tahu. Saya hanya ingin menguji situs di IE (saya tidak memiliki server terpisah / jarak jauh untuk menghostingnya)

Anda harus benar-benar mempertimbangkan untuk beralih dari NAT ke mode bridge. Dengan cara ini kedua "sistem" akan berada di jaringan yang sama. Saat Anda menjalankan dengan NAT, jaringan akan di-subnet dan VM Anda secara efektif ada di jaringan lain. Untuk membuatnya berfungsi dengan NAT Anda perlu mengkonfigurasi (melalui file konfigurasi, bukan opsi GUI) perutean NAT untuk membuka koneksi / port antar jaringan, dan juga mengatur tabel perutean.

\n\n

Jika Anda menyetel mode jembatan, VM meminta IP dari router yang sama dengan yang dilakukan host Anda, bukan dari sistem VM pada host. Pada gilirannya, ini akan semudah membuka IP host Anda.

Secara default, adaptor jaringan adalah mode "NAT". Anda perlu menempatkan adaptor jaringan Anda dalam mode jembatan. Berikut langkah-langkahnya.

\n\n

1> Pada kotak virtual pilih vm yang Anda inginkan "Mis. Win7"\n 2> Buka pengaturan -> Jaringan -> Adapater1 

 Change that from NAT -> Bridged Adapter \n

3> Mulai ulang VM tamu Anda

\n\n

Semoga itu bisa menyelesaikan masalah Anda.
