Telnet dapat digunakan untuk melakukan hal-hal yang dilakukan program secara otomatis secara manual. Ketika saya terhubung ke server saya perhatikan bahwa itu secara otomatis melakukan jabat tangan 3 arah untuk membuat koneksi TCP. Apakah bisa dilakukan secara manual di telnet? Jika tidak, lalu program apa yang bisa melakukannya?

Jika dengan "secara manual" yang Anda maksud "menginstruksikan telnet untuk mengirim paket SYN dan ACK", maka tidak. Ini dilakukan oleh sistem operasi, yang perlu melacak semua parameter TCP untuk koneksi - nomor urut, ukuran jendela, dll.

\n\n

Saya t akan menjadi mungkin bagi program untuk menggunakan soket IP mentah dan mengelola lapisan TCP dengan sendirinya. Tapi itu umumnya hal yang sangat tidak berguna untuk dilakukan - itu akan menyalin banyak kode dari kernel.

\n\n

Jika Anda mencoba mempelajari cara kerja TCP, coba Scapy . Itu dapat membuat dan mengirim paket menggunakan parameter yang diberikan, termasuk TCP. (Anda mungkin harus menggunakan packet sniffer untuk memperhatikan hal-hal seperti SYN + ACK.)

Buku "TCP / IP Illustrated, Volume 1" oleh Stevens menjelaskan apa yang terjadi dalam membuat dan menggunakan koneksi TCP. Ya, sudah ada beberapa saat ((c) 1994) tetapi kadang-kadang penjelasan yang diberikan ketika suatu teknologi baru memiliki lebih dalam daripada apa pun yang muncul setelahnya.

\n\n

Untuk melihat detail tentang apa yang terjadi di kawat, salinan Wireshark ( http://www.wireshark.org ) adalah yang Anda butuhkan. Jika Anda memahami bagaimana header dan pesan terstruktur dan dikemas, wireshark menunjukkan strukturnya dan Anda dapat menelusuri ke bidang bit.
