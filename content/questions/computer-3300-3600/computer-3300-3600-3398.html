Diantara masalah lainnya , setelah memutakhirkan ke Thunderbird 78, sekarang gagal terhubung ke akun email pribadi saya. Akun email pribadi saya berbasis Exchange - saya tidak memiliki kendali atasnya - tetapi berfungsi dengan baik sebagai akun IMAP biasa di Thunderbird 68 dan di ponsel saya, misalnya. Apakah ada orang lain yang beruntung dengan ini?

Thunderbird 78 sekarang menggunakan pengaturan default yang sama seperti Firefox untuk koneksi TLS: Hanya TLS 1.2+ yang diizinkan secara default.

\n

Jika server Exchange Anda sedikit lebih lama dan hanya mendukung hingga TLS 1.1, maka Thunderbird tidak dapat terhubung ke server IMAP.

\n

Cara agar Thunderbird terhubung lagi ke server itu adalah dengan mengizinkan TLS 1.0 / 1.1 lagi dengan mengatur entri konfigurasi Thunderbird 

security.tls.version.min = 1 \n
    \n
  • 1 = TLS 1.0-1.3 diizinkan
    • \n
  • 2 = TLS 1.1-1.3 diizinkan
    • \n
  • 3 = TLS 1.2-1.3 diizinkan (nilai default baru sejak Thunderbird 78)
    • \n
\n

Dokumentasi Mozilla pada security.tls.version

\n

Tambahan: Perhatikan bahwa karena server SMTP adalah bagian dari Windows dan bukan Exchange, TLS1.2 di SMTP biasanya didukung (SMTP menggunakan pustaka TLS Windows tetapi Exchange memiliki pustaka TLS sendiri yang lebih lama).
