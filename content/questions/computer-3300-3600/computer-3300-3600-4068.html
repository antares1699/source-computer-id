Apakah mungkin untuk memalsukan atau memanipulasi tanggal pembuatan? Dan jika demikian, bagaimana saya bisa "membuktikan" bahwa file-file tersebut benar-benar setua itu?

\n\n

Untungnya, saya menyimpan proyek saya di GitHub, yang mengkonfirmasi fakta bahwa file-file tersebut berasal dari era itu, tetapi bukan itu intinya.

\n\n

Saya hanya menjalankan Apple OS X.

Jika Anda memiliki akses ke disk maka Anda dapat mengubah informasi pada disk tersebut. Ini termasuk informasi dengan tanggal pembuatan. Jadi ya, Anda bisa memalsukannya.

\n\n

Satu-satunya cara untuk membuktikan bahwa Anda memilikinya terlebih dahulu adalah jika orang lain dapat memastikannya. GitHub mungkin bisa. Tanggal file pada disk lokal Anda yang sepenuhnya di bawah kendali Anda tidak akan berfungsi.

Ya ... IANAL, tetapi stempel tanggal / waktu pada file yang dibuat pada mesin tertentu mendapatkan informasi itu dari jam CMOS, yang dapat Anda atur ke waktu dan tanggal apa pun yang Anda inginkan. Jadi, Anda benar-benar tidak dapat mengatakan bahwa file dengan tanggal tersebut ada anda komputer membuktikan apa saja. Namun, file yang disimpan oleh orang lain, jika disimpan oleh orang lain seperti yang Anda katakan, buktikan bahwa file itu memang ada sebelum Anda bekerja di perusahaan ini.

\n\n

Jadi proyek yang disimpan di GitHub yang mengonfirmasi fakta bahwa file dari era itu tidak salah, itu akan menjadi "" intinya ... atau lynchpin dalam kasus ini. Apakah pengadilan menerima atau tidak keabsahan informasi mereka akan memutuskan apakah Anda membuat perangkat lunak ini sebelum Anda bekerja untuk perusahaan ini.

\n\n

Intinya mungkin bisa diperdebatkan. Jika Anda mengerjakan perangkat lunak pada waktu perusahaan, Anda mungkin masih diwajibkan secara kontrak untuk menyerahkannya kepada mereka. Itu semua tergantung pada apakah sesuatu yang Anda tanda tangani sebelum Anda bekerja di sana mengatakan bahwa apa pun yang Anda KEMBANGKAN saat bekerja untuk perusahaan adalah milik mereka ... karena DEVELOP tidak selalu berarti membuat. Bisa diartikan sebagai perbaikan, atau pengerjaan.
