Saya memiliki laptop yang menjalankan Lubuntu 13.04. Saya menggunakan ssh -X untuk masuk ke server saya dan memulai program di sana yang saya lihat di laptop saya. Ketika saya menarik kabel ethernet, laptop beralih ke WLAN tetapi jendela ssh -X saya menjadi hitam. IP saya berubah karena sakelar, dan ini mengganggu koneksi ssh.

\n\n

Router saya tidak dapat memberikan dua MAC berbeda dengan IP yang sama.

\n\n

Jadi, apakah mungkin menyetel IP perbaikan untuk wlan0 dan eth0?

","source":"https://superuser.com/questions/639069/same-ip-for-wlan-and-ethernet-on-linux","author":"Torsten Bronger","replies":[],"answers":[{"answer":"

Tidak mungkin memiliki alamat IP yang sama pada dua antarmuka berbeda pada perangkat jaringan apa pun (komputer, router, ...) dalam mode operasi L3 (mode operasi default antarmuka pada komputer adalah mode L3, sebagai antarmuka router).

\n\n

Untuk memiliki IP yang sama, Anda harus menjembatani antarmuka Anda.

Saya tidak yakin menyimpan IP yang sama akan membantu tetapi ini mungkin berhasil:

\n\n
    \n
  1. Hubungkan dengan kabel dan atur alamat IP Anda secara manual.
    2. \n
  2. Tarik kabel dan atur alamat IP yang sama secara manual dan juga ubah alamat MAC ke MAC Ethernet Anda.
    3. \n
  3. Nonaktifkan DHCP pada router WiFi (Saya berasumsi bahwa Anda terhubung ke router yang sama melalui kabel dan nirkabel).
    4. \n
\n\n

Mengatur alamat IP dapat dilakukan di pengelola jaringan jika Anda menggunakan pengaturan Gnome biasa.

\n\n

Anda tidak perlu menonaktifkan DHCP, router saya memiliki opsi untuk memberikan alamat DHCP, mis. hanya di atas 192.168.0.10. Jadi saya dapat mengatur alamat manual 192.168.1. [2-9] dan alamat DHCP akan dimulai dari 192.168.1.10. Dalam situasi seperti itu, Anda dapat mencoba mengatur IP kabel dan nirkabel Anda ke 192.168.1.5.
