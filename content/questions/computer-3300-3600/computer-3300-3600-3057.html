Sesuatu yang sama sekali berbeda.

\n\n

Lion Review Ars Technica membahas ini secara rinci. ( Lihat disini )

\n\n

Pada dasarnya ini memotong file, mencari potongan yang berbeda, dan menyimpannya dalam database SQLite. (CoreData menggunakan SQLite) DB terpisah digunakan untuk melacak semua revisi yang berbeda dan bagian mana yang sesuai dengan revisi mana.

\n\n

Mesin yang menangani chunking juga semi-cerdas. Itu dapat memecahkan beberapa format file (misalnya gambar JPEG, file PDF, file audio / video MPEG, dll.) Dan memisahkan berbagai blok di dalamnya, kemudian mencari perubahan di dalamnya. Itu mencegah perubahan di header video menghasilkan serangkaian perubahan yang membengkak penyimpanan delta secara tidak perlu.

\n\n

Yang juga perlu diperhatikan: Aplikasi yang ingin menggunakan sistem pembuatan versi ini harus ditulis untuk melakukannya. Ini tidak otomatis secara default, juga tidak setiap file di sistem file diversi dengan kerangka ini.

\n\n

Tinjauan Ars menunjukkan bahwa kerangka penyimpanan gernerasional Lion secara otomatis meniadakan revisi lama. Adakah cara untuk memberi tahu OS bahwa versi tertentu cukup penting untuk disimpan tanpa batas?
