Kerentanan Router

\n\n

Periksa apakah router memiliki kata sandi dan apakah itu diatur untuk dapat dikonfigurasi secara nirkabel dan / atau dari sisi WAN (di luar LAN). Sebagian besar perute dilengkapi dengan kata sandi default (yang dapat dengan mudah dicari) atau tanpa kata sandi sama sekali. Lebih buruk lagi, beberapa router sudah diatur sebelumnya sehingga mereka dapat dikonfigurasi secara nirkabel dan / atau dari WAN. Ini memudahkan pemindaian probe jaringan sederhana untuk menemukan router yang rentan.

\n\n

Contoh Serangan Pseudo-code

\n\n

Misalnya, pseudo-code mungkin termasuk: 

for (IP=0.0.0.0 to 255.255.255.255) do { //scan through all possible IPs\n  if (canConnect(IP)) {                  //test for connection\n    if (login(IP, \"admin\", \"\")) {        //try logging in as admin with blank password\n      doBadThings();                     //hack router\n    }\n    else {                               //router has a password, try defaults\n      foreach (knownpasswords) {         //try each known password\n        if (login(IP, \"admin\", pass) {   //try logging in with a default password\n          doBadThings();                 //hack router\n        }\n      }\n    }\n  }\n}\n

Tentu saja kode sebenarnya akan sedikit berbeda (alih-alih menguji kata sandi kosong secara terpisah, itu hanya akan menjadi kata sandi pertama dalam basis data kata sandi yang diketahui, basis data akan menyertakan nama login default serta kata sandi karena tidak selalu admin , rentang IP kemungkinan akan berbeda, akan ada pengoptimalan, dan seterusnya), tetapi itulah intinya secara umum.

\n\n

Kompromisasi Router

\n\n

Setelah malware menemukan router yang dapat dimasuki, malware tersebut dapat “pwn” router, dan dengan ekstensi, seluruh LAN dan perangkat yang terhubung dengannya.

\n\n

Spoofing DNS dan Keracunan Cache

\n\n
\n

Semua perangkat seluler dan laptop di jaringan rumah saya mengalami peringatan yang sama saat mencoba menyambung ke google.com. Bagaimana hal seperti ini bisa terjadi di semua platform di semua perangkat dengan cara yang sama? \n
\n\n

Dalam kasus Anda, kemungkinan besar yang terjadi adalah sederhana Serangan pengalihan DNS di mana server DNS pada router ditetapkan ke server DNS yang disusupi / terinfeksi.

\n\n

Saat Anda mengunjungi halaman web, komputer menghubungi server DNS untuk mencari alamat IP situs. Dengan spoofing DNS, server DNS yang disusupi mengembalikan alamat IP palsu ke beberapa situs web lain (kemungkinan besar menghosting malware), atau situs phishing yang terlihat seperti situs asli (kemungkinan besar menghosting malware), atau mengalihkan ke situs asli setelah melakukan beberapa hal buruk lainnya (mencatat informasi Anda, mencoba menginstal malware drive-by , mencuri kata sandi / cookie / dll., dan sebagainya).
