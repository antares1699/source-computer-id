Progresif Pengkodean JPEG mengatur data sedemikian rupa sehingga gambar dapat didekodekan dengan kualitas rendah terlebih dahulu, dan kemudian detail ditambahkan saat file lengkap tersedia. Oleh karena itu, saat mengunduh gambar, Anda sudah dapat melihat "Pratinjau" gambar.

\n\n

Dioptimalkan JPEG berarti bahwa tabel yang digunakan untuk pengkodean lossless Huffman dihitung secara adaptif untuk file tersebut. Ini tidak berdampak sama sekali pada kualitas file (tahap pengkodean Huffman adalah lossless, tidak mempengaruhi piksel) tetapi dapat mengurangi ukuran file hingga beberapa persen.

Untuk penggunaan praktis, JPEG progresif dapat digunakan mengoptimalkan ukuran file :

\n\n
\n

... JPEG progresif rata-rata lebih kecil. Tapi itu hanya\n rata-rata, itu bukan aturan yang sulit. Faktanya di lebih dari 15% kasus\n (1611 dari 10360 gambar) adalah versi JPEG progresif\n lebih besar.

\n
\n\n

\n\n
\n

Pesan yang dibawa pulang setelah melihat grafik di atas:

\n \n
    \n
  • ketika gambar JPEG Anda di bawah 10K, lebih baik disimpan sebagai JPEG baseline (diperkirakan 75% kemungkinan akan lebih kecil)
    • \n
  • untuk file di atas 10K, JPEG progresif akan memberi Anda kompresi yang lebih baik (dalam 94% kasus) Jadi, jika tujuan Anda adalah memeras setiap\n byte (dan konsistensi bukanlah masalah), hal terbaik yang harus dilakukan adalah mencoba\n baik baseline maupun progresif dan pilih yang lebih kecil.
    • \n
\n \n

Opsi lainnya adalah memiliki semua gambar yang lebih kecil dari 10K sebagai baseline dan\n sisanya sebagai progresif. Atau cukup gunakan garis dasar untuk jempol,\n progresif untuk yang lainnya.
