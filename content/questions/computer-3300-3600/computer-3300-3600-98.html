Server web berulir adalah server yang menangani setiap permintaan dengan utas baru, bukan menangani setiap permintaan dengan proses baru.

\n\n

Multi-threading lebih kompleks daripada konkurensi dengan proses, jadi ada peningkatan risiko server web mogok jika salah satu modulnya tidak berfungsi dengan benar. Juga, jika itu macet, itu lebih mungkin untuk merusak semua utas (mis. Semua permintaan yang sedang ditangani pada saat itu). Namun, server multi-utas mengkonsumsi lebih sedikit memori dan umumnya lebih cepat.

\n\n

Berikut adalah tolok ukur yang membandingkan dua modul apache (saling eksklusif): prefork dan worker.

Ini adalah server web yang memiliki antrian untuk permintaan HTTP yang masuk dan kumpulan utas untuk memuaskan mereka. Server mengambil permintaan pertama dari antrian dan menetapkan utas dari kumpulan untuk melakukan pekerjaan. Permintaan berbaris dalam antrean saat permintaan masuk lebih cepat daripada yang dapat diproses oleh utas atau jika kumpulan kehabisan utas.
