Adakah yang tahu kesalahan apa yang mungkin terjadi ketika saya dapat terhubung ke server MySQL saya melalui localhost tetapi tidak melalui 127.0.0.1? AFAIK, ini harus dirutekan secara identik dan tidak ada lapisan TCP tambahan yang harus dibuat saat menggunakan IP. 

$ mysql -u root -h localhost -D test # works\n$ mysql -u root -h 127.0.0.1 -D test # does not work\n(127.0.0.1 does map to localhost/vice versa)\n

Izin tabel secara eksplisit diatur untuk menyertakan kedua metode.

Pustaka klien MySQL menggantikan localhost untuk menggunakan soket Unix alih-alih koneksi TCP / IP ke 127.0.0.1. Jika Anda dapat terhubung ke localhost tetapi tidak 127.0.0.1, itu menunjukkan bahwa itu firewall atau tidak mendengarkan 127.0.0.1.

Klien MySQL menyimpulkan protokol dari host yang disediakan.

\n
\n

Untuk koneksi tertentu, jika protokol transport tidak ditentukan secara eksplisit, itu ditentukan secara implisit. Misalnya, koneksi ke localhost menghasilkan koneksi file soket pada sistem mirip Unix dan Unix, dan koneksi TCP / IP ke 127.0.0.1 sebaliknya.

\n
\n

Sumber: Dokumen MySQL

\n

Jika Anda tidak dapat terhubung ke server menggunakan 127.0.0.1 sebagai tuan rumah , ini mungkin berarti bahwa server tidak berjalan pada port TCP atau port 3306 (yang merupakan default jika tidak tersedia). Dalam hal ini, Anda hanya dapat menghubungkannya melalui file soket.
