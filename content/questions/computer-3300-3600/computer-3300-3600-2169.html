Saya telah menggunakan VMWARE cukup lama. Untuk mengunjungi mesin tamu dari mesin host, biasanya saya akan membuat antarmuka jembatan di mesin tamu dan menggunakan ssh untuk masuk ke mesin tamu. Masalah dari metode ini adalah bahwa setelah setiap mesin tamu dihidupkan ulang, saya perlu memeriksa ulang alamat ip mesin tamu karena saya hanya dapat menggunakan DHCP di antarmuka yang dijembatani.

\n\n

jadi saya bertanya-tanya apakah ada cara agar saya dapat masuk ke mesin tamu dengan nama host alih-alih dengan alamat IP kosong? kemudian saya bisa memberi nama untuk sesi dempul saya dan menggunakan sesi simpanan ini secara sederhana setiap kali saya ingin masuk ke mesin tamu.

MAC dari kartu jaringan virtual tidak akan berubah di antara boot, jadi jika pengaturan DHCP Anda memungkinkan Anda untuk menetapkan alamat tetap ke antarmuka dengan cara ini, itu adalah rute yang mudah. Jika Anda menggunakan fitur DHCP router murah, Anda mungkin tidak memiliki opsi ini, tetapi router yang baik akan melakukannya dan jika server DHCP adalah kotak Linux atau Windows maka Anda pasti bisa. Akankah server dhcp paling umum yang ditemukan di distribusi Linux seperti ini di file konfigurasi akan melakukan trik: 

host somevirtualmachine {\n  hardware ethernet 00:0C:29:36:BC:91;\n  fixed-address 192.168.39.240;\n}\n

Untuk menemukan MAC dari antarmuka jaringan di Linux, jalankan ifconfig dan cari HWaddr untuk antarmuka. VMWare juga dapat menyajikan informasi di layar manajemennya.

\n\n

Jika Anda tidak dapat mengatur alamat tetap di server DHCP maka Anda dapat mengatur alamat tetap di VM jika ada bagian dari rentang Anda yang tidak diberikan alamat oleh server DHCP (biasanya membiarkan DHCP menetapkan alamat untuk bagian dari rentang seperti 192.168.X.2-128 dan menyimpan sisanya untuk perangkat yang membutuhkan alamat tetap ). Cara melakukannya tergantung pada OS Anda (mengedit /etc/network/interfaces pada pengaturan Linux gaya Debian).

\n\n

Anda juga dapat melihat ke DNS dinamis, meskipun jika Anda tidak diatur untuk dapat memberikan alamat tetap melalui DHCP maka Anda mungkin juga tidak diatur untuk mengkonfigurasi DNS dinamis secara lokal.

\n\n

CATATAN: Anda harus memberikan lebih banyak detail dalam pertanyaan Anda tentang pengaturan Anda, terutama OS apa yang berjalan di VM dan perangkat yang bertanggung jawab untuk membagikan alamat melalui DHCP. Saya telah mengasumsikan Linux dalam contoh di atas tetapi semuanya akan bekerja dengan Windows juga (Anda hanya perlu bertanya kepada admin Windows untuk detailnya).

Ini tergantung pada host dan sistem operasi tamu, tetapi saya menduga Anda memiliki Windows sebagai host dan beberapa bentuk Linux sebagai guest.

\n\n

Protokol paling umum untuk resolusi nama lokal adalah NBNS (NetBIOS) - digunakan oleh semua versi Windows, sedikit tidak efisien karena penggunaan siaran tetapi kompatibel dengan hampir semua hal.

\n\n

Jika tamunya adalah Linux, instal Samba dan mulai komponen nmbd (smbd tidak diperlukan di sini), ini akan memungkinkan tamu dapat dihubungi dengan namanya.

\n\n

Untuk menghubungi tuan rumah dari tamu, tambahkan hosts: ..... wins hingga /etc/nsswitch.conf.

\n\n

Alternatifnya adalah mDNS - digunakan terutama oleh Mac OS X, tetapi juga tersedia untuk Windows (bagian dari "Bonjour Print Services" Apple) serta Linux (dalam bentuk avahi-daemon dan nss_mdns).
