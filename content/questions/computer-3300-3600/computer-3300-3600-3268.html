Saya ingin dapat menghubungkan komputer desktop yang tidak memiliki adaptor nirkabel ke jaringan nirkabel saya.

\n\n

Saya bisa saja menjalankan kabel jaringan dari ADSL / router nirkabel saya ke komputer desktop tetapi terkadang ini tidak praktis.

\n\n

Yang benar-benar ingin saya lakukan adalah menjembatani adaptor nirkabel dan kabel laptop saya sedemikian rupa sehingga saya dapat menjalankan kabel jaringan dari laptop saya ke sakelar dan kabel jaringan lain dari sakelar ke komputer desktop sehingga komputer desktop dapat mengakses Internet melalui ADSL / router nirkabel saya melalui latop saya:

\n\n

\n\n

\n\n

\n\n

Ini tidak terjadi.

Sekali lagi, desktop tidak dapat menjangkau ADSL / router nirkabel.

Saya lebih memilih untuk mendapatkan jembatan bekerja di bawah Ubuntu jadi saya masih mencari jawaban, tetapi saya bisa mendapatkan jembatan untuk bekerja pada Windows XP.

\n\n

Setelah menjembatani Local Area Connection dan Wireless Network Connection saya membuka command prompt dan mengetik yang berikut ini: 

netsh bridge show a\n

Ini mencantumkan Koneksi Area Lokal saya dengan ID 1 dan Koneksi Jaringan Nirkabel saya dengan ID 2. Mode ForceCompatibilityMode dari Koneksi Jaringan Nirkabel saya ditampilkan sebagai Tidak Dikenal, jadi saya mengetik yang berikut ini: 

netsh bridge set a 2 e\n

Ini mengaktifkan Koneksi Jaringan Nirkabel saya. Komputer desktop bisa mendapatkan sendiri alamat IP dari server DHCP di router. Ternyata, alamat IP yang didapatnya sama dengan alamat IP yang biasanya didapat Koneksi Jaringan Nirkabel saya yang menyebabkan Windows mengeluh tentang konflik alamat IP di jaringan. Saya memberi komputer desktop saya alamat IP statis dan semuanya baik-baik saja. Komputer desktop dan laptop sekarang dapat menggunakan Internet.

Saya telah memutuskan untuk menggunakan NAT di laptop Ubuntu saya daripada menjembatani yang berfungsi dengan baik kecuali karena tidak dapat menggunakan server DHCP router saya. Saya mungkin menginstal server DHCP di laptop saya jika saya tidak dapat diganggu secara manual mengkonfigurasi jaringan desktop saya. Terlepas dari itu, beginilah cara saya melakukan nirkabel ke kabel NAT ...

\n\n

Jaringan router saya (dan karena itu koneksi nirkabel laptop saya) adalah 192.168.1.0, saya akan menggunakan 192.168.0.0 untuk jaringan pada antarmuka eth0 dan desktop laptop saya.

\n\n

Untuk mulai dengan saya mematikan Manajer Jaringan dengan mengklik kanan pada ikon jaringan di baki sistem dan menghapus centang dari Aktifkan Jaringan.

\n\n

Saya membuat file bernama /etc/wpa_supplicant.conf (sebagai root) yang berisi berikut ini: 

ctrl_interface=/var/run/wpa_supplicant\nnetwork={\n    ssid=\"myssid\"\n    scan_ssid=1\n    key_mgmt=WPA-PSK\n    psk=\"mypresharedkey\"\n}\n

Di mana "myssid" adalah SSID router saya dan "mypresharedkey" adalah kunci pra-berbagi WPA router saya.

\n\n

Saya kemudian menjalankan jaringan nirkabel saya dengan: 

sudo iwconfig wlan0 essid myssid\nsudo wpa_supplicant -iwlan0 -c/etc/wpa_supplicant.conf -B\n

Dan dapatkan alamat IP, nameserver dan gateway default dengan: 

sudo dhclient wlan0\n

Itu bagian pertama yang dilakukan. Sekarang saya perlu membuat NAT berfungsi sehingga lalu lintas untuk eth0 bisa masuk ke / dari router.

\n\n

Pertama, saya perlu memberi tahu Ubuntu bahwa saya ingin mengizinkan penerusan: 

sudo bash\necho 1 > /proc/sys/net/ipv4/ip_forward\nexit\n

Saya akan berpikir saya bisa menggemakan 1 ke / proc / sys / net / ipv4 / ip_forward menggunakan sudo tetapi untuk beberapa alasan saya selalu mendapatkan pesan izin ditolak, jadi saya sudo bash dan echo 1 dari shell root.

\n\n

Lalu saya menyiapkan beberapa aturan penerusan: 

sudo iptables -t nat -A POSTROUTING -o wlan0 -j MASQUERADE\nsudo iptables -A FORWARD -i wlan0 -o eth0 -m state --state RELATED,ESTABLISHED -j ACCEPT\nsudo iptables -A FORWARD -i eth0 -o wlan0 -j ACCEPT\n

Dan terakhir atur port LAN, eth0: 

sudo ifconfig eth0 192.168.0.254\n

Semua selesai di laptop saya, sekarang untuk mengkonfigurasi jaringan desktop: 

sudo ifconfig eth0 192.168.0.1\nsudo route add default gw 192.168.0.254\nsudo bash\necho \"nameserver 192.168.1.254\" > /etc/resolv.conf\nexit\n

Sekali lagi, saya sepertinya tidak dapat menggunakan echo untuk menulis file kecuali saya sudo bash terlebih dahulu.

\n\n

Dan itu dia. Sekarang saya dapat mengakses Internet dari desktop saya, melalui koneksi nirkabel laptop saya.
