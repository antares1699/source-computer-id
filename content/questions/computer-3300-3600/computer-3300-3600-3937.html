Saya sedang menyiapkan sistem dengan dua hard drive 3TB sebagai pendukung penyimpanan. Karena ini adalah stasiun desktop, saya berpikir tentang pengaturan disk berikut: 

sda\n+ /boot with 500MB\n+ btrfs for the remaining GBytes\nsdb\n+ btrfs for all the 3000GB\n

Saya kemudian berpikir untuk menggunakan mode single untuk data pada / dan membuat subvolume untuk /home (dan kemungkinan lainnya) menggunakan raid1 sebagai mode data. Apakah itu mungkin dengan btrfs (kernel 3.7.X) saat ini? Apakah ada yang punya sumber untuk itu, saya tidak dapat menemukan apa pun.

\n\n

MEMPERBARUI: Saya membuat pengaturan pengujian kecil yang terdiri dari dua file biasa yang dihubungkan sebagai perangkat loopback untuk bermain-main. Pada file tersebut, saya membuat btrfs biasa. Kemudian saya melakukan hal berikut: 

btrfs subvolume create test\ndd if=/dev/urandom of=foo bs=1024 count=1  # use random data to prevent dedup\ndd if=/dev/urandom of=test/bar bs=1024 count=1  # use random data to prevent dedup\nbtrfs balance start -dconvert=raid1 test\nbtrfs balance start -dconvert=single .\nbtrfs filesystem df .\n

Perintah btrfs balance berlaku untuk sistem file seperti yang ditunjukkan oleh bantuan meskipun parameter "sistem file" bersifat opsional: 

# btrfs balance start --help\nusage: btrfs [filesystem] balance start [options] <path>\n

Sampai ada "btrfs subvolume keseimbangan "maka saya curiga fitur ini tidak ada.
