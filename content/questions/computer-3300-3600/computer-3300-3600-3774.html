Cygwin bekerja dengan menyediakan pustaka, cygwin1.dll, yang bertindak sebagai lapisan terjemahan antara API POSIX dan Linux, dan fitur Windows yang sesuai. Dengan cara ini tidak perlu memodifikasi OS sama sekali. (Program Cygwin, tentu saja, harus dikompilasi sebagai program Win32 .exe asli, cukup ditautkan ke pustaka yang berbeda.)

\n\n

Program berbasis terminal dapat dijalankan dengan dua cara: mereka dapat dijalankan di konsol Windows (di mana cygwin1.dll secara otomatis menerjemahkan urutan seperti VT100 yang dicetak ke dalam fungsi API konsol Windows), atau mereka dapat dijalankan di emulator terminal seperti VT100 seperti MinTTY , URxvt, atau PuTTYcyg. (Cygwin biasanya hadir dengan yang pertama secara default.)

\n\n

Biasanya layanan tidak diperlukan untuk operasi biasa, kecuali jika Anda menginstal program itu adalah dirancang sebagai layanan (misalnya daemon server OpenSSH).

\n\n
\n\n

[Sekadar perbandingan: Interix, alias Services for Unix, bekerja di lapisan bawah, dengan menyediakan "posix" subsistem yang berjalan di samping subsistem Win32 biasa, langsung di atas API "Asli" atau "NT" tingkat rendah.

\n\n

Namun, baik kernel maupun komponen OS lainnya tidak perlu dimodifikasi atau driver khusus diinstal, karena kedua metode hanya menambahkan lapisan terjemahan di ruang pengguna.]
