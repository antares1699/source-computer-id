Saya menggunakan VMWare Workstation 12, memiliki mesin host Windows 10, dan beberapa mesin tamu Windows. Saya ingin semua mesin fisik dan virtual dapat berkomunikasi satu sama lain dan saya ingin semua mesin tamu memiliki alamat IP statis sehingga saya dapat membuat peta jaringan alamat IP dan memastikan ke mesin mana alamat IP tertentu ditetapkan .

\n\n

Saya mencoba menggunakan NAT tetapi mesin tamu saya tidak dapat dilihat oleh mesin fisik lain di jaringan saya selain host.

\n\n

Bagaimana cara membuat semua VM muncul di jaringan saya ke semua mesin fisik lain di jaringan saya?

Yang ingin Anda lakukan adalah menggunakan jaringan Bridged - bukan NAT.

\n\n

Anggaplah Anda memiliki host Windows 10 dan VMWare Workstation 12 dengan beberapa mesin tamu Windows (pada prinsipnya, saran di sini harus berlaku untuk OS tamu non-Windows lainnya dan ke versi terbaru VMWare di bawah VMWare 12). Selain itu, saya akan berasumsi bahwa Anda ingin setiap mesin virtual memiliki alamat IP sendiri sehingga dapat ditemukan oleh mesin Windows lainnya di jaringan Anda.

\n\n

Untuk mewujudkannya:

\n\n

Luncurkan Editor Jaringan Virtual dari menu Edit VMWare Workstation. Di luar ruang lingkup untuk mencakup utilitas ini sepenuhnya dalam jawaban ini, tetapi pastikan Anda bekerja sebagai Administrator; kemudian pastikan bahwa VMnet 0 berjenis Bridged dan Koneksi Eksternal-nya disetel ke adaptor fisik host Anda.

\n\n

Setelah itu, edit properti mesin virtual dari OS Windows tamu Anda dan buka bagian Adaptor Jaringan. Pastikan untuk memilih Kustom dan tentukan VMnet0 sebagai jaringan virtual spesifik Anda. Selain itu, pastikan untuk memeriksa Hubungkan saat dihidupkan.

\n\n
    \n
  • Boot ke OS tamu Anda, buka Pusat Jaringan dan Berbagi, dan buka Ubah pengaturan adaptor.
    • \n
  • Klik kanan untuk membuka properti adaptor Ethernet Anda, dan gulir ke Internet Protocol Version 4.
    • \n
  • Pilih itu dan klik Properties. Gunakan tab Umum untuk menentukan alamat IP tertentu, Subnet mask, dan Gateway default.
    • \n
  • Pastikan bahwa alamat IP berada di subnet yang sama dengan host Anda dan Anda menggunakan alamat IP gateway yang sama dengan mesin host. (Jika Anda tidak yakin apa ini, buka host Anda, buka prompt perintah, dan ketik IPConfig, dan lihat alamat gateway untuk adaptor yang ditetapkan VMnet0.)
    • \n
  • Tentukan IP gateway Anda sebagai server DNS Pilihan Anda (server alternatif tidak perlu ditentukan)
    • \n

  • jika Anda memiliki masalah DNS, buka host, buka prompt perintah, dan ketik NSLookup dan lihat alamat IP yang dilaporkan: itu adalah alamat IP DNS yang mungkin ingin Anda gunakan. Saat masih dalam dialog properti adaptor jaringan, klik Lanjutan, buka tab MENANG, dan pastikan untuk memilih "Aktifkan NetBIOS melalui TCP / IP".

    • \n

  • Ini akan memastikan Anda dapat menjangkau mesin tamu dengan Nama Komputernya dari host atau dari VM lain yang Anda konfigurasikan sebagai VM yang dijembatani.

    • \n
  • Anda dapat mengklik kanan adaptor Ethernet di OS tamu, menonaktifkan lalu mengaktifkannya untuk melihat apakah perubahan Anda berhasil (tidak perlu di-boot ulang).
    • \n
\n\n

Saya telah membaca postingan online yang tak terhitung jumlahnya tentang masalah jaringan dan ada banyak sekali masalah yang dapat Anda hadapi, jadi petunjuk ini tidak akan bekerja secara ajaib untuk semua orang karena ada banyak tautan dalam rantai jaringan ...

\n\n

Karena itu, langkah-langkah ini seharusnya berfungsi selama tidak ada firewall atau masalah router yang tidak biasa. Berbicara tentang router, Anda ingin memastikan bahwa alamat IP statis yang Anda tetapkan berada di luar jangkauan server DHCP yang ditawarkan router Anda.

\n\n

Dengan begitu Anda dapat yakin bahwa alamat IP statis yang Anda gunakan tidak akan bertentangan dengan alamat yang ditetapkan DHCP yang mungkin ditetapkan ke perangkat lain di jaringan Anda.

\n\n

Bersulang.

Inti dari NAT adalah membuat alamat mesin lokal ke jaringan internal dan tidak dapat dialamatkan secara publik. Dalam kasus mesin virtual, jaringan internal tersebut adalah jaringan virtual yang menghubungkan mesin host fisik dan mesin virtualnya (berkemampuan NAT). Setiap VM yang dikonfigurasi untuk menggunakan NAT membagikan alamat host fisiknya.

\n\n

Jika Anda ingin mesin virtual dapat dialamatkan oleh mesin fisik lain di jaringan fisik Anda, Anda harus mengonfigurasinya untuk menggunakan jaringan Bridged. Ini akan membuat mesin virtual muncul di jaringan seolah-olah terpisah, mesin fisik.
