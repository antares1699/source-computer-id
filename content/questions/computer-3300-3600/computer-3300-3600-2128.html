Jika Anda benar-benar ingin menggunakan VPN, mungkin lebih mudah untuk mengatur router sebagai server VPN dan mengatur klien VPN di VPS. Misalnya, router DD-WRT dapat dengan mudah ditetapkan sebagai server VPN , dan Anda dapat menginstal klien OpenVPN di VPS.

\n\n

Pilihan lain yang mungkin lebih mudah:

\n\n

Anda dapat mengubah port server di PC Anda, atau mengubahnya di router jika Anda memilikinya. ISP Anda tidak mungkin memblokir semua port, kemungkinan besar hanya memblokir port yang umum. Jelas Anda mungkin melanggar TOS ISP Anda.

\n\n

Jika perangkat lunak server berjalan di Linux, Anda dapat menginstalnya di VPS. Kemungkinannya adalah itu akan memiliki uplink yang lebih baik daripada koneksi rumah Anda, dan tergantung pada permainannya ini mungkin dan keuntungannya.

\n\n

Jika perangkat lunak server berjalan di Windows, Anda bisa mendapatkan VPS Windows. Pertimbangan kecepatan yang sama seperti di atas.

\n\n

Banyak ISP menawarkan koneksi bisnis dengan harga lebih tinggi. Biasanya Anda dapat menjalankan server dengan koneksi seperti itu.

Hal seperti itu dapat dilakukan melalui SSH Remote tunnel, tetapi mungkin membingungkan klien penghubung atau server itu sendiri. Idenya, bagaimanapun, adalah untuk:

\n\n
    \n
  • Jalankan, di komputer lokal Anda, ssh -R :<<port to listen>>:localhost:<<same port>> <<your vps' address>>. Perhatikan : terkemuka. Kecuali diblokir oleh pengaturan GatewayPorts sshd Anda, ini akan menyiapkan port mendengarkan di server, yang ketika dihubungi akan diteruskan di dalam koneksi ssh ke komputer lokal Anda (karenanya localhost ), menghubungi apa pun yang mendengarkan di <<same port>> (yang akan menjadi game Anda).
      \n
    • Jika Anda menggunakan PuTTY sebagai klien SSH, baca http://oldsite.precedence.co.uk/nc/putty.html untuk mengetahui cara menyiapkannya. Centang kotak "Port jarak jauh melakukan hal yang sama" (: terdepan), letakkan <<port to listen>> di bidang Source port, dan localhost:<<same port>> di bidang Destination. Pastikan untuk memilih radio button Remote dan klik Add.
      • \n
    • \n
  • Minta server Anda "mengira" itu berjalan di alamat IP VPS Anda, jika memungkinkan. Ini akan membantu mengurangi kebingungan klien jika ini berhasil dan server mengiklankan ke beberapa daftar server.
    • \n
  • Cobalah untuk terhubung ke server dengan meminta klien terhubung ke alamat IP VPS. Jika berhasil, Anda telah mencapai kesuksesan.
    • \n
\n\n

EDIT: ini mungkin hanya berfungsi dengan port TCP
