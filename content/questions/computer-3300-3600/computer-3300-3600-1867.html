Dengan menggunakan ssh-keygen, saya perlu membuat file kunci pribadi SSH di mana algoritme enkripsi yang digunakan saat menggunakan frasa sandi adalah AES-128 dengan mode CBC.

\n\n

Saya tahu ini adalah default untuk OpenSSH> = 5,4 sesuai catatan rilis dimana mereka menyebutkan:

\n\n
\n

Kunci privat protokol SSH 2 yang dilindungi frasa sandi sekarang dilindungi\n dengan AES-128, bukan 3DES. Ini diterapkan ke kunci yang baru dibuat\n serta kunci yang dienkripsi ulang (misalnya dengan mengubah\n frasa sandi).

\n
\n\n

Namun, persyaratan saya adalah menggunakan algoritma enkripsi ini dengan OpenSSH 5.3p1 (Saya tidak dapat mengubah versi OpenSSH).

\n\n

Bagaimana cara mengubah algoritma enkripsi ke AES-128 tanpa memutakhirkan ke OpenSSH 5.4 atau yang lebih baru?

","source":"https://superuser.com/questions/961682/how-to-change-encryption-algorithm-for-private-key-file-using-openssh-5-3","author":"Tunaki","replies":[],"answers":[{"answer":"

ssh-keygen tidak memiliki opsi untuk menentukan enkripsi frasa sandi, jadi Anda tidak bisa begitu saja membuat ssh-keygen lama berperilaku seperti versi baru dengan mengubah file konfigurasi atau memberikan opsi padanya, tetapi Anda dapat menggunakan OpenSSL untuk menghasilkan atau mengubah kunci.

\n\n

Konversi

\n\n

Dengan asumsi kunci Anda adalah RSA 

openssl rsa -aes128 -in <3des_protected> -out <aes128_protected>\n

akan mengubah kunci 3des_protected menjadi AES128. Ini akan meminta frasa sandi tiga kali: sekali untuk membaca kunci lama, dua kali untuk menyetel yang baru.

\n\n

Penciptaan

\n\n

Pertama buat bagian pribadi 

openssl genrsa -aes128 -out <newkey>\n

kemudian ubah izinnya sehingga ssh tidak mengeluh dengan chmod 600 <newkey>

\n\n

Kunci publik kemudian diturunkan dari kunci privat dengan menerbitkan 

ssh-keygen -e -f newkey > newkey.pub\n
