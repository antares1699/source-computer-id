Satu hal yang membuat frustrasi yang sangat sering terjadi pada saya adalah saya mengetikkan perintah panjang ke dalam antarmuka baris perintah (CLI) dan kemudian menyadari bahwa saya perlu menjalankan perintah lain sebelum itu. Terkadang saya membuka CLI kedua untuk menjalankan perintah itu, tetapi di lain waktu itu tidak memungkinkan. Jadi saya akhirnya menghapus teks yang sudah saya ketik, mengetik dan menjalankan perintah preseden, dan akhirnya mengetik ulang dan menjalankan perintah asli.

\n\n

Berikut adalah alternatif terbaik yang saya ketahui:

\n\n

Dalam Terminal Linux , jika saya memiliki mouse, saya dapat dengan hati-hati menyorot teks dengan mouse, klik kanan -> Salin, dan hapus baris dengan Ctrl + U. Jika saya ingin teks yang disalin lagi, saya dapat Ctrl + Shift + V, klik tengah, atau klik kanan -> Tempel. Tanpa tikus, saya tersesat.

\n\n

Dalam Prompt Perintah Windows , jika saya memiliki mouse, saya dapat mengklik kanan -> Tandai, sorot teks dengan hati-hati menggunakan mouse, Ctrl + C untuk menyalin, dan tahan Backspace untuk menghapus garis. Ketika saya ingin teks yang disalin lagi, saya dapat mengklik kanan -> Tempel. Tanpa tikus, saya tersesat.

\n\n

Saya merasa harus ada cara yang lebih baik. Solusi ideal bagi saya adalah memiliki semacam perintah keyboard yang menyimpan teks yang telah saya ketikkan pada baris saat ini untuk nanti dan menghapusnya dari baris pada saat yang sama, serta perintah kedua yang akan dimasukkan kembali teks pada baris perintah. Solusi semacam itu akan lebih cepat dan nyaman karena hanya memiliki langkah yang lebih sedikit, tidak perlu beralih antara mouse dan keyboard, dan tidak perlu khawatir tentang menyorot jumlah teks yang tepat.

bash (Unix / Linux / Mac OS X)

\n\n

(juga MSYS, termasuk Git Bash untuk Windows)

\n\n

Metode Kill Ring

\n\n
    \n
  1. Jika Anda tidak berada di awal atau akhir baris, tekan salah satunya Rumah atau Akhir untuk memindahkan kursor ke salah satu ujung.
    2. \n
  2. tekan Ctrl + U untuk membunuh (memotong) semuanya ke kiri kursor atau Ctrl + K untuk membunuh semuanya ke Baik kursor.
    3. \n
  3. tekan Ctrl + Y untuk "menarik" teks kembali ke baris perintah.
    4. \n
\n\n

Jika Ctrl + U , Ctrl + K , Ctrl + W atau serupa ditekan selama mengetik baris baru (demikian pembunuhan teks baru), teks lama tidak hilang. Itu membunuh cincin dapat diputar setelah sentakan ( Ctrl + Y ) menggunakan Meta + Y (atau Alt + Y ). Dengan cara ini, Anda dapat menyimpan banyak perintah dan mengambilnya sesuka hati.

\n\n

Metode Komentar

\n\n

Menyimpan perintah:

\n\n
    \n
  1. tekan Rumah atau Ctrl + SEBUAH untuk memindahkan kursor ke awal baris saat ini.
    2. \n
  2. Ketik # untuk mengomentari baris.
    3. \n
  3. tekan Memasukkan (masukkan komentar ke dalam riwayat Anda).
    4. \n
\n\n

Mengambil perintah:

\n\n
    \n
  1. tekan â†‘ sampai Anda mencapai perintah berkomentar.
    2. \n
  2. tekan Rumah atau Ctrl + SEBUAH untuk memindahkan kursor ke awal baris.
    3. \n
  3. tekan Menghapus untuk menghapus #. Sekarang Anda memiliki perintah asli.
    4. \n
\n\n
\n\n

Prompt Perintah Windows

\n\n

Metode QuickEdit

\n\n

(Masih melibatkan mousework):

\n\n

Prasyarat: Pada bilah judul prompt perintah, klik kanan -> Properties -> centang "QuickEdit Mode" -> OK.

\n\n
    \n
  1. Sorot teks dengan menyeret menggunakan tombol kiri mouse.
    2. \n
  2. Klik kanan di dalam jendela prompt perintah untuk menyalin.
    3. \n
  3. tekan Ctrl + Rumah untuk menghapus semuanya ke kiri kursor atau Ctrl + Akhir untuk menghapus semuanya di sebelah kanan kursor.
    4. \n
  4. Klik kanan di dalam jendela prompt perintah untuk menempel.
    5. \n
\n\n

catatan: Solusi ini kurang dari ideal karena tidak berfungsi jika perintah Anda menggunakan lebih dari satu baris (seperti yang ditampilkan); pemisahan antar baris akan diartikan seolah-olah Anda telah menekan Memasukkan pada saat itu.
