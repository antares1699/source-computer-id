Saya memiliki server Linux Synergy (Si-Linux) yang hanya melayani satu klien Mac (Macbook Pro UK) (SiBook-Pro.local).

\n\n

Di server Linux saya, saya menggunakan keyboard Apple USB dengan ekstensi tepat tata letak keyboard laptop (keyboard aluminium UK yang ringkas).

\n\n

Saya ingin mengirim kunci media ke klien Mac setiap saat dan saya telah mencoba yang berikut ini di synergy.conf saya: 

keystroke(AudioPlay) = keystroke(AudioPlay,SiBook-Pro.local)\n

Ini sepertinya tidak berhasil jadi saya menjalankan server dan klien sebagai proses latar depan dan dengan debugging diaktifkan dan mengamati hal berikut:

\n\n

Log Server: 

DEBUG1: activate actions\nDEBUG1: hotkey: keyDown(AudioPlay,SiBook-Pro.local)\nDEBUG1: onKeyDown id=57523 mask=0x0000 button=0x0000\nDEBUG1: send key down to \"SiBook-Pro.local\" id=57523, mask=0x0000, button=0x0000\nDEBUG1: deactivate actions\nDEBUG1: hotkey: keyUp(AudioPlay,SiBook-Pro.local)\nDEBUG1: onKeyUp id=57523 mask=0x0000 button=0x0000\nDEBUG1: send key up to \"SiBook-Pro.local\" id=57523, mask=0x0000, button=0x0000\n

Log Klien: 

DEBUG1: recv key down id=0x0000e0b3, mask=0x0000, button=0x0000\nDEBUG1: mapKey e0b3 (57523) with mask 0000, start state: 0000\nDEBUG1: key e0b3 is not on keyboard\nDEBUG1: recv key up id=0x0000e0b3, mask=0x0000, button=0x0000\nDEBUG1: recv enter, 1279,386 5 2000\n

Seperti yang Anda lihat, klien mengklaim bahwa kunci yang diterima adalah bukan di keyboard . Saya tidak mengerti karena ini adalah tombol yang sama dengan yang ada di keyboard Macbook.

\n\n

Saya mencoba membalikkan konfigurasi klien-server untuk melihat apakah saya dapat menangkap kunci yang dikirim jika saya menekan tombol Putar di Macbook tetapi kuncinya tampaknya bahkan tidak masuk ke Synergy. Hampir semua penekanan keyboard dicatat tetapi tombol media tampaknya melewati log dan hanya menjalankan fungsinya secara lokal. Misalnya. Saya menekan putar di Macbook (dengan Macbook sebagai server) dan tombol memutar musik di Macbook dan kuncinya tidak masuk ke log debug.

Ini adalah batasan di Mac Synergy Client. Ada dua hal yang mencegah Mac Synergy menangani tombol multimedia:

\n\n
    \n
  1. Mac Synergy membangun peta papan ketiknya dari tata letak papan ketik Mac (misalnya, bahasa Inggris Amerika, Prancis, dll.) Dan ini tidak berisi tombol media
    2. \n
  2. Mac API tidak memungkinkan Anda membuat penekanan tombol sintetis yang sesuai dengan penekanan tombol multimedia.
    3. \n
\n\n

Saya telah menulis patch untuk Mac Synergy (dan mengirimkannya ke proyek di http://synergy-foss.org/spit/issues/details/3375/ ) yang mengatasi kedua masalah ini, dengan memasukkan tombol media ke dalam tata letak secara manual, dan dengan menggunakan penekanan tombol media serial yang direkam selama penekanan tombol sebenarnya.

\n\n

Anda dapat mengunduh tambalan itu sendiri dan menyusun sinergi, atau menambahkan catatan ke masalah yang memintanya untuk diintegrasikan. Jika tidak ada yang berfungsi, hubungi saya, dan saya dapat mengirimkan biner yang saya gunakan ...
