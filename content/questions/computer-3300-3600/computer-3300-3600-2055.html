Saya punya Laptop Sony Vaio . Sampai kemarin saya sudah menginstal Windows 7 dan Ubuntu 12.04 (Precise Pangolin). Hari ini saya meningkatkan ke Windows 8, yang mematikan layar pemilihan OS saya. Jadi saya boot ke Ubuntu 12.10 dari stik USB dan menjalankan Perbaikan Boot. Ini memulihkan GRUB layar dan biarkan saya boot ke instalasi Ubuntu 12.04 saya, tetapi sekarang saya dapat 'mem-boot Windows 8.

\n\n

Ketika saya memilih opsi yang mem-boot Windows 7 sebelumnya, sepertinya ia ingin memulai kembali Windows 7 (sic) (layar hitam dengan teks "Windows sedang memuat" atau semacamnya), tetapi kemudian me-reboot mesin setelah beberapa detik.

\n\n

Saya yakin Perbaikan Boot dapat memperbaiki ini, tetapi saya tahu terlalu sedikit tentang keseluruhan hal boot untuk mengetahui pengaturan yang tepat untuk itu.

\n\n

Inilah laporan Perbaikan Boot saya: http://paste.ubuntu.com/1308117/

\n\n

Pembaruan 1:

\n\n

Saya berhasil kembali ke Windows lagi. Saya mengubah garis 

chainloader (${root})/EFI/Microsoft/Boot/bootmgfw.efi.bkp\n

di /etc/grub.d/25_custom (jika Anda membaca ini mencari bantuan untuk masalah yang sama: file Anda mungkin disebut sesuatu yang berbeda) ke 

chainloader (${root})/EFI/Microsoft/Boot/bootmgfw.efi\n

dan menyalin file bootmgfw.efi dari [Windows partition]/Windows/Boot/EFI ke /boot/efi/EFI/Microsoft/Boot. (File sudah ada, jadi saya mengganti namanya menjadi bootmgfw.efi.old untuk mencadangkannya.) Terakhir, saya menjalankan sudo update-grub.

\n\n

Ketika saya me-restart komputer, Windows 8 sedang boot lagi, tetapi tanpa layar GRUB. Jadi coba tebak .. Saya tidak bisa boot ke Ubuntu sekarang.

\n\n

Di Windows, saya menginstal EasyBCD dan menambahkan opsi boot Ubuntu. Ketika saya mencoba memilihnya setelah reboot lagi, gagal karena tidak dapat menemukan beberapa file. Saya mencoba ini dengan opsi EasyBCD untuk menemukan instalasi Linux saya secara otomatis, dan saya juga mencoba memilih partisi secara manual. Keduanya memiliki hasil yang sama.

\n\n

Jadi saya sekali lagi mencoba boot dari stik USB Ubuntu saya dan menjalankan Perbaikan Boot. Kali ini saya mendapat layar pemilihan OS Windows (yang saya siapkan di EasyBCD). Ketika saya memilih Windows 8 di sana, itu mem-boot ulang komputer.

\n\n

Bagaimana cara mengatasi masalah ini?

\n\n

Perbarui 2:

\n\n

Saya dapat "beralih" antara Ubuntu dan Windows 8 sekarang. Namun tidak melalui menu boot. (Semua jawaban yang saya dapatkan sejauh ini tidak menyelesaikan masalah khusus saya.)

\n\n

Saat saya berada di Ubuntu dan ingin mem-boot Windows 8, saya melakukan apa yang saya jelaskan di pembaruan pertama (salin bootmgfw.efi dari direktori Windows dan ubah satu baris di /etc/gib.d/25_custom). Kemudian Windows 8 dimulai tanpa menu boot.

\n\n

Saat saya menggunakan Windows 8 dan ingin mem-boot Ubuntu, saya memulai Ubuntu 12.10 dari stik USB dan menjalankan Boot-Repair. Kemudian GRUB 1.99 dimulai dan ketika saya memilih Ubuntu, itu berhasil. (Opsi Windows tidak berfungsi.)

\n\n

Jadi itulah keadaan saat ini - I bisa berganti OS, tapi itu sangat merepotkan.

Saya memiliki masalah yang sama persis dengan Anda, dan saya menemukan solusinya di forum lain (sebenarnya, saya pikir mungkin Anda yang memposting solusi itu). Saya memperbaikinya dengan cara yang sedikit berbeda.

\n\n

Saya mengikuti instruksi yang Anda gunakan, menyalin bootmgw.efi dari partisi Windows ke partisi Ubuntu, dan saya cukup yakin saya juga melakukan sudo update-grub. Namun, alih-alih mengubah file /etc/grub.d/25_custom secara manual seperti yang Anda lakukan pada langkah pertama, saya masuk ke Grub Customizer (alat di Ubuntu yang mungkin atau mungkin belum Anda miliki), mengklik kanan opsi boot Windows yang sesuai (milik saya mengatakan "Windows UEFI loader , "milik Anda akan menjadi opsi yang mem-boot Windows 7 sebelum seluruh bencana ini, seperti yang Anda katakan), lalu saya pergi ke tab" sumber ", dan saya membuat perubahan di sana. (Saya baru saja menghapus .bkp, seperti yang Anda lakukan saat melakukannya secara manual.) Kemudian saya menyimpan semua perubahan, dan itu memperbaiki masalah.

\n\n

Jika Anda penasaran, saya menggunakan Grub Customizer daripada mengedit file secara manual karena dua alasan: 1. Saya tidak bisa mendapatkan akses ke file dan terlalu malas untuk mencoba dan mendapatkannya menggunakan terminal Linux; dan 2. Saya takut bahwa hanya dengan mengubah file mungkin tidak benar-benar mempengaruhi perubahan pada file boot (meskipun ini murni spekulasi; saya juga tidak tahu cara kerja booting). Saya pikir Grub Customizer akan benar-benar membuat perubahan yang benar-benar terlihat dalam file boot.

\n\n

Satu-satunya hal adalah, "lingkaran pemuatan" Windows 8 kecil (saya tidak tahu harus menyebutnya apa lagi) muncul di latar belakang ungu yang digunakan Grub; itu terlihat agak jelek. Saya membayangkan ini juga bisa diperbaiki melalui Grub Customizer.

\n\n

Memperbarui

\n\n

Jika Anda melakukan boot langsung ke Windows, maka Anda harus memastikan bahwa GRUB adalah opsi boot pertama di BIOS Anda. Saya tidak dapat memberi tahu Anda cara melakukan ini kecuali saya tahu utilitas BIOS apa yang Anda gunakan, tetapi saya dapat memberi Anda beberapa langkah umum.

\n\n

1. Nyalakan komputer Anda dan akses menu BIOS Anda. Biasanya ini dilakukan dengan menumbuk F8 setelah komputer dihidupkan, tetapi itu bervariasi dari komputer ke komputer (bagi saya itu F2 ). Pencarian Google cepat tentang mesin khusus Anda akan mengungkapkan tombol apa yang harus Anda tekan.

\n\n

2. Temukan bagian yang memungkinkan Anda untuk mengubah urutan boot, dan pindahkan Ubuntu / Linux / Grub (Saya tidak yakin dengan label yang tepat, tetapi harus jelas) ke atas. Jika Anda memiliki beberapa opsi boot ini, cukup pilih salah satu.

\n\n

3. Simpan dan mulai ulang PC Anda, dan itu harus boot ke GRUB mulai sekarang.
