Apa yang diperlihatkan oleh perintah Anda adalah sebagai berikut:

\n\n
    \n
  1. Komputer Anda telah menentukan alamat www.facebook.com menjadi\n69.171.228.14
    2. \n
  2. Itu mencoba mengirim paket ping ke sana
    3. \n
  3. Dalam perjalanan paket, router (mungkin milik Anda, di alamat 204.15.23.57) tidak dapat menemukan rute ke 69.171.228.14
    4. \n
  4. Agar Anda tahu tentang ini, router 204.15.23.57 memberi tahu Anda tentang hal ini dengan pesan balasan ICMP Destination net unreachable.
    5. \n
\n\n

Dalam perintah kedua Anda, Anda melakukan ping ke router yang memberi Anda balasan bersih yang tidak dapat dijangkau, bukan host facebook, oleh karena itu tidak mengherankan Anda mendapatkan balasan darinya.

\n\n

Seperti yang telah Anda tunjukkan dengan membantu - ini adalah hasil dari file host yang kacau. Anda juga bisa melihat masalahnya jika Anda menggunakan nslookup untuk menanyakan server DNS secara langsung: 

nslookup\nserver 8.8.4.4\nwww.facebook.com\n

Ini akan menunjukkan kepada Anda alamat sebenarnya dari host facebook dan Anda akan memperhatikan bahwa itu berbeda dari yang diberikan kepada Anda (69.171.228.14), jadi Anda akan tahu untuk melihat di file host saat komputer menyelesaikan alamat IP melalui file host atau permintaan DNS.

\n\n

Karena itu tidak baik bahwa virus dapat memperbarui file host, karena itu seharusnya hanya dapat ditulis oleh superuser / sistem (Administrator, SYSTEM atau root di Linux). Oleh karena itu implikasinya adalah virus / trojan mempunyai akses tingkat tinggi ke sistem.

Jika saya melakukan "ping google.com -4"\nSaya akan mendapat balasan. Itu memaksanya untuk menggunakan IPv4\nJika saya melakukan ping tanpa -4, itu tidak akan berhasil dan begitu pula yang lainnya (browser, email, dll).

\n\n
