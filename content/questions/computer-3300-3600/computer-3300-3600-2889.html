Ini tentang pengembangan Bash dan pengkodean skrip Bash portabel yang menggunakan RegEx .

\n\n

Menggunakan Bash RegEx, di Mac, saya dapat melakukan ini: 

coconut-mac$ a='bananacoconutman'; [[ \"$a\" =~ banana(.*?)man ]] && echo FOUND ${BASH_REMATCH[1]}\nFOUND coconut\n

Bagus. Berguna di banyak tempat. Suka.

\n\n

Ketika saya mencoba melakukan ini, gagal: 

coconut-mac$ a='<title>coconut</title>'; [[ \"$a\" =~ \\<title\\>(.*?)\\</title\\> ]] && echo FOUND ${BASH_REMATCH[1]}\n

Itu tepat perintah yang sama dijalankan sempurna di penguin: 

coconut-linux$ a='<title>coconut</title>'; [[ \"$a\" =~ \\<title\\>(.*?)\\</title\\> ]] && echo FOUND ${BASH_REMATCH[1]}\nFOUND coconut\n

Di Mac saya, info bash / =~ RET mengatakan:

\n\n
\n

Operator biner tambahan, =~', is available, with the same\n precedence as == 'dan `! ='. Saat digunakan, string ke\n kanan operator dianggap sebagai ekspresi reguler yang diperluas\n dan dicocokkan dengan semestinya (seperti pada regex3)).

\n
\n\n

man 3 regex mengatakan:

\n\n
\n

Operator pengulangan (?', * ', +', or bounds) cannot follow another repetition\n operator. A repetition operator cannot begin an expression or subexpression or\n follow ^' atau `| '.

\n
\n\n

Saya tidak melihat dokumentasi analog di GNU regex's man 3 regex atau info regex.

\n\n

Jika saya menghapus ? dari (.*?) Anda dan melakukan hal berikut, ini berfungsi pada kedua OS: 

$ a='<title>coconut</title>'; [[ \"$a\" =~ \\<title\\>(.*)\\</title\\> ]] && echo FOUND ${BASH_REMATCH[1]}\nFOUND coconut\n
