\n\n

Cara menyalurkan terminal / program saat ini dan masih menampilkannya di konsol saat ini.

\n\n

Ketika saya menggunakan perintah di bawah ini, saya dapat menyimpan keluaran program, tetapi saya tidak dapat melihatnya di konsol.

Saya bisa menggunakan opsi keluaran telnet, saya sedang mencari cara perintah independen untuk melakukannya.

\n\n

telnet ADDRESS PORT | tee -a myoutputfile\n

Ini menyalin output ke output standar dan nama file yang ditentukan, dan opsi -a ditambahkan, seperti yang dilakukan pengalihan >> Anda.

Ada trik untuk menggunakan keluaran kesalahan sebagai penyangga kedua untuk menghasilkan keluaran ke 2 perangkat.

\n\n

Perintah Anda adalah: 

telnet ADDRESS PORT >&2 2>>myoutputfile\n

Ini menambahkan output ke myoutputfile dan juga menulis apa pun yang keluar ke konsol.

\n\n

>&2: Ini menyalin output dari aliran 1 (konsol) ke aliran 2 (kesalahan)

\n\n

2>>: Ini memastikan segala sesuatu dari aliran 2 dikirim ke file.

\n\n

Jika Anda tidak ingin mengambil risiko mengalami errorlogging yang ditambahkan ke file, ubah 2 menjadi 3 di kedua baris. (>&3 3>>file)

\n\n

EDIT: Saya baru saja melihat tag linux. Ini ditulis untuk Windows, tetapi mungkin masih berfungsi.
