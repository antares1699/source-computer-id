Saya mencoba memahami tujuan server nama DNS publik (publik bukan SOA; seperti 8.8.8.8).

\n\n

Saya menjalankan perintah dig +trace, dan, sejauh yang saya tahu, satu-satunya saat server nama digunakan adalah ketika menemukan SOA domain root .. Setelah itu, SOA setiap domain di hierachy dipertanyakan.

\n\n

Asumsi saya adalah bahwa server nama publik digunakan untuk:

\n\n
    \n
  • mengidentifikasi SOA domain root, sehingga SOA tidak harus di-hardcode pada klien.
    • \n
  • mendesentralisasikan proses dan cache. Jadi jika SOA domain turun, server lain dapat memandu klien.
    • \n
  • mengurangi beban pada SOA. Jika server lain mengetahui recored, tidak perlu query SOA.
    • \n
\n\n

\n\n
    \n

  • Apakah asumsi saya benar?

    • \n

    • \n
  • Dapatkah / Apakah SOA menolak untuk menjawab pertanyaan dari nameserver tertentu jika ia melakukan terlalu banyak permintaan?
    • \n

Sebagai tambahan kecil Jawaban Mike Scott ...

\n\n

Tujuan utama server nama publik (seperti yang Anda tentukan) adalah untuk:

\n\n
    \n

  1. Cegah individu dari keharusan menjalankan server nama mereka sendiri.

    2. \n

  2. Membantu (berpotensi) mempercepat proses penyelesaian domain.

    3. \n
\n\n

Beberapa catatan tentang "tiga asumsi" Anda

\n\n
    \n

  • Beberapa server nama hanya meneruskan pencarian untuk permintaan yang tidak diketahui ke resolver rekursif lainnya (seperti server nama publik). Karena server nama publik menyimpan file file petunjuk root untuk melakukan pencarian rekursif, ini berarti file ini tidak diperlukan di sini hanya penerusan server nama.\nNamun, ini tidak ada hubungannya dengan "klien" (sejauh yang saya mengerti maksud Anda).

    • \n

  • Server nama publik dapat (bisa dibilang) membantu dengan desentralisasi dan caching. Selain itu, mereka berpotensi memberikan ketahanan pada sistem DNS secara keseluruhan. Tetapi ini adalah manfaat sampingan.

    • \n

  • Server DNS publik dapat mengurangi beban pada server nama otoritatif, tetapi sekali lagi, itu bukan alasan utama untuk melewati server tersebut. Idenya adalah bahwa informasi yang di-cache dikembalikan lebih cepat dibandingkan meminta informasi itu dari server nama yang berwenang.

    • \n
