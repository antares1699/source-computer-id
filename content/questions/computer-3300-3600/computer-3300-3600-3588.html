Aplikasi Windows Explorer serta .NET Framework bermasalah dengan jalur yang melebihi 260 karakter (lihat di sini: https://stackoverflow.com/questions/1880321/why-does-the-260-character-path-length-limit-exist-in-windows )

\n\n

Apa yang dapat Anda lakukan untuk menghindari ini hanyalah mempersingkat jalur dengan cara memetakannya ke huruf drive dengan subst.exe seperti ini: 

subst.exe <DriveLetter>: <super long path without trailing backslash>\n

Contoh: 

subst.exe A: C:\\users\\%username%\\appdata\\local\\temp\n

Ini memetakan jalur super panjang saya (C:\\users\\%username%\\appdata\\local\\temp dalam kasus ini) ke huruf drive A: sehingga mengurangi panjang jalur lebih dari 20 karakter. Ini juga berfungsi dengan jalur UNC di NAS Anda.

\n\n

Drive terlihat di Windows Explorer sehingga Anda hanya perlu membuka baris perintah untuk memetakan drive. Ini akan secara otomatis tidak dipetakan ketika Anda log off / reboot PC.
