Tidak yakin apakah Anda menjalankan ini pada apa pun sebelum Windows 10, tetapi ketika saya memutakhirkan dari Windows 7 saya menemukan bahwa beberapa skrip AutoHotKey saya berhenti berfungsi saat memuat. Saya akhirnya mengetahuinya karena mereka sekarang membutuhkan hak admin (saya rasa mereka tidak membutuhkannya di Win7), tetapi Win10 memblokir item di lokasi shell:startup dan shell:common startup agar tidak berjalan sebagai admin.

\n\n

Solusi saya sedikit berbelit-belit tetapi berfungsi dengan baik. Pada dasarnya, saya meninggalkan skrip saya di lokasi normal (non-startup), seperti C:\\Users\\xyz\\Documents dan kemudian membuat pintasan ke skrip di lokasi yang sama. Anda kemudian dapat mengatur properti pintasan untuk dijalankan sebagai admin. Akhirnya, di lokasi shell:startup, saya membuat file batch yang akan dijalankan secara otomatis dan memanggil pintasan: 

start \"MyScript\" /D \"C:\\Users\\xyz\\Documents\\\" /B MyScript.ahk.lnk\n

Saya menemukan solusi lain yang mungkin tampak lebih baik dari sebelumnya.

\n\n

Ketika saya mulai menggunakan AHK, saya membuat semua file .ahk dalam satu folder. Namun, menurut saya, semua orang melakukannya. Dalam hal ini sangat mudah untuk menjalankan semua file ini dengan file bat. Seperti ini: 

@echo off\ncd f:/hotkeys\nfor /f  %%f in ('dir /b /s .') do start %%f\n

Buat file dengan konten seperti itu dan taruh di direktori startup: Win+R> shell:startup. Itu saja.

\n\n
