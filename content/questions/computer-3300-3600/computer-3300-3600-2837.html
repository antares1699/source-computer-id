Saya memiliki server Linux / Debian yang berjalan: postgresql, tomcat. Terkadang, saya tidak dapat mengakses server dengan ssh.

\n\n

Ketika saya membuka koneksi ssh dengan server saya, saya dapat mengetikkan ID masuk (root) dan kata sandi saya, tetapi ketika saya mengetikkan kata sandi saya dan menekan enter, tidak ada tanggapan apa pun.

\n\n

Dengan fungsi 'sysinfo', saya menemukan bahwa ketika server menghadapi situasi ini, ada sekitar 1600 proses (nilai ini dari sysinfo.procs), dan rata-rata beban hampir 300 (dalam kasus biasa, rata-rata beban antara 0 dan 1)

\n\n

Apa yang saya coba adalah, mendaftar proses TOP 5 dengan jumlah utas yang dibuatnya. Namun saya tidak dapat menemukan metode yang mudah. Ada beberapa cara yang saya temukan.

\n\n
    \n
  1. Catat nilai NLWP ps -efL, jadi saya bisa tahu proses mana yang membuat terlalu banyak utas secara tidak normal.
    2. \n
  2. Ketika rata-rata beban semakin besar, salin semua file /procs/*/status ke direktori log, dan saya dapat mengetahui proses mana yang memiliki nilai 'Utas' terbesar nanti.
    3. \n
\n\n

Jika ada saran lain, beri tahu saya. Ini membuat saya gila karena server Linux ini seharusnya tidak gagal layanan
