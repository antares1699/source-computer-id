Secara umum, "Sesuaikan waktu musim panas secara otomatis" harus selalu On. Menonaktifkannya adalah untuk beberapa kasus edge lama dan beberapa tempat di dunia yang saat ini tidak memiliki entri yang sesuai dalam daftar zona waktu.

\n\n

Kemungkinan besar, jam sistem tidak disetel ke waktu yang benar. Mungkin ada masalah saat mendapatkan waktu yang benar dari Internet, yang diaktifkan atau dinonaktifkan dari opsi pertama "Setel waktu secara otomatis".

\n\n

Saya merekomendasi:

\n\n
    \n

  1. Periksa itu milik Anda Server Waktu Internet pengaturan sudah benar. Coba server waktu yang berbeda jika perlu.

    2. \n

  2. Aktifkan semua opsi, pastikan untuk mengaktifkan "Sesuaikan waktu musim panas secara otomatis" sebelum mengaktifkan "Setel zona waktu secara otomatis".

    3. \n

Ini baru saja terjadi pada saya minggu ini pada mesin Windows 10, saat kami memasuki Waktu Musim Panas di beberapa wilayah di sini di Brasil, bahkan dengan "Sesuaikan untuk waktu musim panas secara otomatis" diaktifkan. Windows memang mengenali DST, tetapi jam terus bergerak ke zona waktu yang salah: UTC, bukan UTC-2 (DST dari UTC-3 Brasil).

\n\n

Ini telah diselesaikan dengan mengubah server NTP ke pool.ntp.org, seperti yang ditunjukkan @MattJohnson dan @stevemidgley di atas. Pada awalnya, tampaknya server "time.windows.com" mungkin tidak diperbarui dalam pengaturan zona waktunya, tetapi konfigurasi waktu tetap OK setelah mengembalikan server ke "time.windows.com".

\n\n

Yang juga saya perhatikan adalah pengaturan zona waktu sering macet di Windows 10, mengharuskan menggunakan Task Manager untuk menghentikan prosesnya.
