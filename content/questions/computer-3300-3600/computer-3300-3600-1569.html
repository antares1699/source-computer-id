Saya memecahkan masalah dengan cara yang tidak terlalu ortodoks. Saya mengunduh Eclipse Graphical Modeling Framework SDK 2.0 dari versi arsip. Dari penelitian saya, saya menemukan artikel tentang menginstal Apollo untuk Eclipse, yang merupakan alat UML lainnya. Artikel ini menunjukkan bahwa versi SDK 2.0 berisi file org.apache.batik.dom.svg 1.6.0.jar.

\n\n

http://archive.eclipse.org/modeling/gmf/downloads/drops/R-2.0-200706210000/

\n\n

Setelah diunduh, saya membuka ritsleting file ke dalam folder penyimpanan. Saya kemudian menyalin seluruh konten folder "plugin" dan menggabungkannya ke folder "plugin" di foler eUML2 yang diekstrak. Saya kemudian menempatkan folder eUML2 ke dalam folder "dropins" di Eclipse dan memulai ulang Eclipse. Untuk beberapa alasan itu mengambil toples dokumentasi, tetapi tidak mengambil file jar lainnya. Saya kemudian menggunakan fitur "instal perangkat lunak baru" dan menunjuk ke folder di folder dropins. Ini berhasil, tetapi kemudian muncul kesenangan saat mencoba mengaturnya untuk melakukan apa yang saya inginkan, yaitu rekayasa balik. Semoga berhasil!!
