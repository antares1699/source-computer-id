Kemungkinan Duplikat:
\n Biarkan skrip bash berjalan di terminal jarak jauh saat tidak masuk?

\n
\n\n\n\n

Baru-baru ini, saya mencoba mengekstrak file zip berukuran 30 pertunjukan di sistem jarak jauh menggunakan Putty. Saat proses unzip yang lama berlanjut, saya menutup Putty, dengan asumsi bahwa proses tersebut akan terus berjalan di mesin jarak jauh.

\n\n

Ketika saya kembali lagi nanti dan masuk kembali ke mesin lagi, saya menyadari bahwa prosesnya pasti berhenti hanya sebagian ketika saya menutup Putty. Saya tidak mengharapkan itu terjadi.

\n\n

Pertanyaan saya adalah, bagaimana cara mencegah masalah ini? Dapatkah saya menjalankan proses di latar belakang? Atau sebaiknya atur saja cronjob satu kali yang akan menjalankan prosesnya untuk saya?
