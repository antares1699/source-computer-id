Saya sering menggunakan grep dengan opsi -n untuk mencantumkan nomor baris, misalnya 

$ grep -n xyz .*\n/ext1/acheong/foo/bar/a.cc:42:    inline bool abc(int xyz);\n/ext1/acheong/foo/bar/a.cc:43:    inline bool def(int xyz);\n/ext1/acheong/foo/bar/b.cc:415:            int xyz = result;\n/ext1/acheong/foo/bar/b.cc:490:                xyz += result;\n$\n

Apakah mungkin untuk alias vim sedemikian rupa 

vim /ext1/acheong/foo/bar/b.cc:415\n

akan membuka file dan mencari baris 415 secara otomatis?

Nah, ini bukan alias, Anda bisa menggunakan vim seperti ini:

\n\n

vim +<LineNumberHere> fileName

\n\n

Jadi misalnya 

vim +150 .bash_history\n

membuka file .bash_history Anda (untuk bash), dan menavigasi ke baris 150.

\n\n

Secara kebetulan, Anda juga dapat melakukan ini dengan istilah penelusuran. Sebagai contoh 

vim +/MyTerm MyFile\n

membuka MyFile dan menavigasi ke kemunculan pertama MyTerm dari atas.

\n\n

Cara paling sederhana dan tangguh:

\n\n
    \n
  1. Buat file baru ~/.bin/vim dan tempel kode di bawah ini
    2. \n
  2. Buat alias shell: di ~/.bashrc tambahkan baris alias vi=~/.bin/vim
    3. \n
\n\n
\n

Ini disebut substitusi parameter dan pada dasarnya akan memisahkan file\n parameter yang disediakan oleh ' : ' (usus besar).

\n
#!/usr/bin/env bash\n\nif [[ ${1} == *\":\"* ]]; then\n    vim ${1/:*/} -c \"${1/*:/}\"\nelse\n    vim ${1}\nfi\n

Kemudian Anda bisa menggunakannya sebagai:\nvim /ext1/acheong/foo/bar/b.cc:415
