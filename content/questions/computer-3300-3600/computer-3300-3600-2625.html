Mulai Manual:

\n

Anda dapat memulai cmd dan meneruskan parameter untuk memulai file.\nDari http://ss64.com/nt/cmd.html :

\n
\n

Pilihan:

\n

/ C Jalankan Perintah dan kemudian hentikan
Perintah: Perintah, program atau skrip batch yang akan dijalankan ...

\n
\n

Buka perintah run (WIN + R) dan ketik

\n

cmd /C "Path_to_your_file"

\n

Sebagai contoh: 

cmd /C "C:\\Users\\name\\Desktop\\script.cmd"\n

Setel ulang .cmd ke asosiasi cmdfile

\n

Anda juga dapat mengatur ulang asosiasi file dari file .cmd ke cmdfile, dengan asumsi inilah yang menyebabkan file .cmd Anda dibuka di vim:\nMulai cmd sebagai administrator dan ketik 

assoc.cmd=cmdfile\n

Setel ulang registri

\n

Yang ini dapat membahayakan komputer Anda jika Anda melakukan sesuatu yang salah karena Anda sedang mengerjakan registri.

\n

Buka regedit (WIN + R: regedit)

\n

Pertama kami ingin memeriksa apakah seni .cmd terkait dengan cmdfiles.

\n

Goto: HKEY_CLASSES_ROOT\\.cmd dan periksa apakah String (Default) berisi data cmdfile

\n

Juga periksa apakah HKEY_CLASSES_ROOT\\cmdfile\\shell\\open\\command berisi (Default) String "%1" %*.

\n

Jika demikian, lanjutkan ke HKEY_CURRENT_USER\\Software\\Microsoft\\Windows\\CurrentVersion\\Explorer\\FileExts\\.cmd

\n

.cmd harus berisi 2 Kunci: OpenWithList dan OpenWithProgids. Jika ada kunci UserChoice, hapus dengan semua isinya.

\n

OpenWithList seharusnya hanya berisi String (Default) dengan nilai kosong. Ini harus mengatakan (nilai tidak ditetapkan). Hapus nilai lainnya dalam OpenWithList.

\n

OpenWithProgids hanya boleh berisi String (Default) dengan nilai kosong dan nilai biner dengan nama cmdfile. Sekali lagi, hapus nilai lain dalam OpenWithProgids.
