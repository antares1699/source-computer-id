Jawaban lain mungkin bagus untuk Anda memiliki instance Chrome, tetapi setiap pengguna lain akan memiliki masalah yang sama. Masalahnya adalah Chrome perlu memiliki must-revalidate di header Cache-Control` untuk memeriksa ulang file untuk melihat apakah mereka perlu diambil ulang.

\n\n

Untuk memperbaikinya sekali dan untuk semua bagi semua pengguna, sertakan header tanggapan berikut: 

Cache-Control: must-validate\n

Ini memberi tahu Chrome untuk memeriksa dengan server, dan melihat apakah ada file yang lebih baru. Jika ada file yang lebih baru, itu akan menerimanya sebagai tanggapan. Jika tidak, itu akan menerima respons 304, dan jaminan bahwa yang ada di cache adalah yang terbaru.

\n\n

Jika Anda TIDAK menyetel tajuk ini, maka jika tidak ada pengaturan lain yang membuat file tidak valid, Chrome akan melakukannya tidak pernah periksa dengan server untuk melihat apakah ada versi yang lebih baru.

\n\n

Ini dia posting blog yang membahas masalah ini lebih lanjut.
