Saya memiliki dua mac yang menjalankan OS X 10.10.3. Satu memiliki Server.app terinstal, terhubung ke internet melalui ethernet, dan juga memiliki disk 2 TB yang terpasang melalui USB, dengan dua partisi - "Cadangan" dan "Penyimpanan". Mac saya yang lain adalah laptop yang digunakan untuk tugas sekolah.

\n\n

\n\n

Dengan begitu, cadangan jaringan tambahan akan jauh lebih kecil dari 300 GB dan dapat dilakukan dalam jangka waktu yang wajar, mis. semalam.

\n\n

Lebih baik nanti daripada tidak sama sekali, berikut adalah cara untuk mencapai ini:

\n

sudo ifconfig lo0 alias 127.0.0.2/32

\n

Colokkan usb drive anda, lalu via System Preferences/Sharing tambahkan share smb ke folder time-machine-macbook di drive time-machine-usb\nKemudian, tambahkan cadangan tujuan (Time Machine akan melihatnya sebagai berbagi jaringan)

\n

sudo tmutil setdestination -a "smb://user:password@127.0.0.2/time-machine-macbook"

\n

Anda dapat melihat apakah tidak masalah dengan:

Kemudian mulai pencadangan dengan tmutil startbackup dan mount akan menampilkan: 

/dev/disk3s1 on /Volumes/time-machine-usb (apfs, local, nodev, nosuid, journaled, noowners)\n//user@127.0.0.2/time-machine-macbook on /Volumes/time-machine-macbook (smbfs, nobrowse)\n

Anda dapat melihat kemajuan mendetail dengan tmutil status: 

Backup session status:\n{\n    BackupPhase = Copying;\n    ClientID = "com.apple.backupd";\n    DateOfStateChange = "2018-06-28 17:38:21 +0000";\n    DestinationID = "F707BD0B-57DC-4DB6-A3B7-824470FB5EB2";\n    DestinationMountPoint = "/Volumes/Time Machine Backups";\n    FirstBackup = 1;\n    Percent = "0.08711567546702646";\n    Progress =     {\n        TimeRemaining = 32679;\n        "_raw_totalBytes" = 355694600192;\n        bytes = 34429528173;\n        files = 887351;\n        totalBytes = 391264060211;\n        totalFiles = 2922384;\n    };\n    Running = 1;\n    Stopping = 0;\n    "_raw_Percent" = "0.09679519496336274";\n}\n

Dalam kasus saya, hampir 1000000 file selesai dalam waktu sekitar dua jam (penalti disk USB2, dibandingkan lebih dari 8 jam melalui Wi-Fi), saya menghitung persentase dengan skrip kecil: 

tm-progress.sh\nFiles : 918702 / 2922384 (31.43%) - Bytes : 32.21 GiB / 364.39 GiB (8.84%)\n

Tentu saja, Time Machine seharusnya dengan disabilitas selama langkah ini untuk mencegah pemasangan paket sparsebundle yang disalin sebagian (dan mungkin merusaknya)

Saya telah menggunakan drive apfs untuk memanfaatkan snapshot (untuk dapat kembali ke "backupdb" sebelumnya jika terjadi korupsi sparsebundle, belum diuji )
