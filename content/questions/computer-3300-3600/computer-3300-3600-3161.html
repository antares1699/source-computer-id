nohup menjalankan program dengan input standar yang dialihkan dari /dev/null (dengan asumsi bahwa Anda tidak mengalihkan perintah nohup itu sendiri). Jadi tidak, Anda tidak dapat mengirim masukan ke program ini.

\n\n

Jika Anda ingin mengirim masukan ke program, alihkan masukan saat Anda memulainya: 

nohup program <input-file.txt &  # input from a file\nnohup data-producer | nohup program &  # input from another program\nmkfifo program.pipe; nohup program <program.pipe &  # input from a named pipe, feed it what you want later\n

(Sebenarnya, dimungkinkan untuk menghubungkan kembali input standar program ke sumber lain, dengan menggunakan ptrace, yaitu debugger atau peretasan lainnya. Ini dapat merusak program jika terus melacak dari mana inputnya berasal. Lihat Bagaimana saya bisa menjeda proses yang berjalan melalui ssh, menolaknya, mengaitkannya ke shell layar baru dan membatalkan jeda? ; ada pertanyaan lain di jaringan SE tentang topik ini.)
