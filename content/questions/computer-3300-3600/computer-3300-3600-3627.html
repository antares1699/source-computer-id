Systemd versi 233 (Maret 2017) menambahkan dukungan untuk mengatur variabel lingkungan pada ~/.config/environment.d/*.conf. Lihat halaman manual environment.d dan diskusi yang mengarahkan ke fitur tersebut PR awal ini dan yang terakhir ini .

Cara yang berhasil untuk saya adalah menggunakan ~/.pam_environment.

\n\n

Gunakan man pam_env.conf untuk referensi.

\n\n

Saya hanya perlu menambahkan beberapa jalur pengguna ke PATH dalam sesi GNOME / Wayland dan menambahkan baris berikut ke ~/.pam_environment. 

PATH        OVERRIDE=${HOME}/bin:${HOME}/.local/bin:${PATH}\n

Belum ada cara untuk melakukan ini di Wayland.

\n\n

File dot seperti .pam_environment, .profile, dan jelas .xinitrc akan diabaikan.

\n\n

Cara terdekat untuk melakukannya adalah dengan menambahkan baris seperti di bawah ini ke bagian bawah file ~ / .bashrc Anda (dengan asumsi Anda menggunakan bash) dan kemudian menyetel emulator terminal Anda untuk diluncurkan saat startup. 

[[ $ENVSETUP != 0 ]] && export ENVSETUP=0 && exit\n

Ini akan membuat terminal Anda terbuka saat Anda masuk, tetapi segera menutup. Kemudian semua variabel lingkungan Anda yang ditentukan dalam .bashrc akan disiapkan.
