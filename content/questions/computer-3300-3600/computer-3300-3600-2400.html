Jadi saya mencoba mencari di seluruh memori saya untuk string tertentu, tetapi saya tidak tahu bagaimana cara mendapatkan akses ke konten memori saya.

\n\n

Saya mencoba menjalankan ini tetapi mendapatkan pesan kesalahan: 

~$ sudo cat /dev/mem | strings | grep findme\n[sudo] password for user:\ncat: /dev/mem: Permission denied\n

Saya menjalankan Ubuntu 9.04 64-bit dengan kernel 2.6.28-15-generik. Dari pemahaman saya, ini semacam perlindungan bawaan. Bagaimana saya bisa menyiasati ini jika sudo tidak cukup?

","source":"https://superuser.com/questions/39583/command-line-access-to-dev-mem-in-ubuntu","author":"Shane","replies":[],"answers":[{"answer":"

Akses ke memori "nyata" melalui / dev / mem telah dinonaktifkan oleh patch kernel ini . Jadi pilihan Anda adalah mengkompilasi ulang kernel Anda dengan opsi NONPROMISC_DEVMEM dinonaktifkan atau Anda mencoba rute lain, seperti menulis modul kernel Anda sendiri. Selamat meretas kernel!
