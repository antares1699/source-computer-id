Saya ingin menggunakan perintah 'which' untuk melihat eksekusi mana yang sebenarnya dipanggil, tetapi alias 'noglob' defensif tetap menghalangi. Apakah ada pintasan yang dapat saya gunakan untuk menemukan file yang dapat dieksekusi yang dipanggil saat perintah dialias?

\n\n

Saya pikir ini harus agnostik shell, tetapi saya menggunakan zsh jika itu penting.

Anda dapat menggunakan which -a COMMAND (atau where COMMAND atau whence -ca COMMAND) untuk menemukan semua kemunculan COMMAND di jalur perintah.

\n\n

Sebagai contoh: 

% alias ls='noglob ls'\n% ls () /bin/ls\n% which ls\nls: aliased to noglob ls\n% which -a ls\nls: aliased to noglob ls\nls () {\n        /bin/ls\n}\n/bin/ls\n

Karena alias diganti di baris perintah sebelum apa pun dieksekusi, perintah kedua dalam daftar adalah yang Anda cari (dengan asumsi tentu saja bahwa baris pertama entah bagaimana memiliki alias dengan nama yang sama)
