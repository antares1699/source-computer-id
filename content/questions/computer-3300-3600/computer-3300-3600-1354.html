Cara Mengkonfigurasi WAMPServer untuk menggunakan HTTPS SSL

\n\n

* Ini bukan proses yang sepele. Mudah-mudahan, tutorial ini akan membuat SSL berfungsi untuk Anda.\nNamun mendapatkan itu dikonfigurasi dengan benar setelah itu bekerja BENAR-BENAR DIKENAKAN UNTUK ANDA.

\n\n

Bacaan tambahan untuk semua orang yang melakukan perjalanan jalan ini

Saya mendasarkan tutorial ini pada pembuatan situs bernama www.wamphelpers.dev Jadi, di mana pun Anda melihat nama itu, ubahlah menjadi nama situs yang Anda coba amankan.

\n\n

Saya mulai dengan membuat situs tidak aman, di \\ wamp \\ www \\ wamphelpers

\n\n

menambahkan Host Virtual untuk situs itu, di httpd-vhosts.conf 

<VirtualHost *:80>\n    DocumentRoot \"c:/wamp/www\"\n    ServerName localhost\n    ServerAlias localhost\n    <Directory  \"c:/wamp/www\">\n        AllowOverride All\n        Require local\n    </Directory>\n</VirtualHost>\n\n<VirtualHost *:80>\n    DocumentRoot \"c:/wamp/www/wamphelpers\"\n    ServerName wamphelpers.dev\n    ServerAlias www.wamphelpers.dev\n    <Directory  \"c:/wamp/www/wamphelpers\">\n        AllowOverride All\n        Require local\n    </Directory>\n</VirtualHost>\n

Menambahkan namanya ke C: \\ windows \\ system32 \\ drivers \\ etc \\ hosts 

127.0.0.1 wamphelpers.dev www.wamphelpers.dev\n::1       wamphelpers.dev www.wamphelpers.dev\n

Sekarang restart dnscache sebagai berikut dari jendela perintah yang diluncurkan menggunakan 'Run as Administrator' 

net stop dnscache\nnet start dnscache\n

Kemudian buat script sederhana di \\ wamp \\ www \\ wamphelpers \\ index.php 

<?php\n    echo 'Hello, this is the WAMPHELPERS.DEV site homepage';\n?>\n

Sekarang restart Apache dan pastikan bahwa situs sederhana tanpa jaminan Anda berfungsi before continuing

Hasilkan kunci dan Sertifikat.

LANGKAH 1: Buat Kunci Pribadi RSA

\n\n

Pertama kita perlu membuat sertifikat untuk diri kita sendiri.\nProses normal (dibayar) adalah membuat sertifikat Anda dan kemudian meneruskannya ke otoritas penandatanganan.\nInilah sebabnya mengapa membutuhkan uang, sebagaimana yang harus mereka lakukan, uji tuntas, untuk memeriksa bahwa Anda adalah yang Anda katakan dan bahwa situs tempat Anda akan menggunakan sertifikat itu nyata dan sah.

\n\n

Toolkit openssl digunakan untuk menghasilkan Kunci Pribadi RSA dan CSR (Permintaan Penandatanganan Sertifikat) untuk digunakan pada Sertifikat kami.\nLangkah pertama adalah membuat Kunci Pribadi RSA Anda.\nKunci ini adalah kunci RSA 1024 bit yang dienkripsi menggunakan Triple-DES dan disimpan dalam format PEM sehingga dapat dibaca sebagai teks ASCII.

\n\n

Buka jendela Command (kotak Dos) menggunakan [b] Run as Administrator [/ b]\nUbah Direktori ke tempat Anda menginstal OpenSSL Toolkit di atas.\n Dalam kasus saya ini 

CD c:\\apps\\OpenSSL-Win32\\bin\n

Buat folder untuk output yang akan dimasukkan (untuk menjaga folder bin rapi) Saya menggunakan situs web 

md website\n

Sekarang masukkan perintah ini: 

openssl genrsa -out website\\server.key 2048\n

Ini seharusnya membuat file di folder situs web bernama privkey.pem, tanpa kunci frase sandi, periksa keberadaannya.

\n\n

Langkah 2: Buat CSR (Permintaan Penandatanganan Sertifikat)

\n\n

Selama pembuatan CSR, Anda akan diminta memberikan beberapa informasi.\nIni adalah atribut X.509 dari sertifikat tersebut.\nSalah satu petunjuknya adalah untuk "Nama Umum (mis. FQDN server atau nama ANDA) []:". Kolom ini harus diisi dengan nama domain server yang memenuhi syarat untuk dilindungi oleh SSL. Jadi jika situs web yang akan dilindungi adalah https://www.wamphelpers.dev, kemudian masukkan www.wampheplers.dev pada prompt ini.

\n\n

Jangan masukkan apa pun ke pertanyaan: Sandi tantangan []: ] Cukup tekan Enter.\nJika Anda memasukkan kata sandi di sini ketika Anda memulai Apache dengan SSL dikonfigurasi Apache tidak akan mulai dan akan memberikan pesan kesalahan ini: -

\n\n

[error] Init: SSLPassPhraseDialog builtin tidak didukung di Win32

\n\n

Pada dasarnya jika Anda memasukkan frasa sandi, Apache seharusnya menantang Anda untuk frasa sandi itu setiap kali dimulai.\nIni jelas tidak akan membuat hidup Anda lebih mudah tetapi terutama di windows itu tidak benar-benar berfungsi dan akan\nmenyebabkan Apache macet ketika mencoba meminta frasa sandi, dengan kesalahan di atas.

\n\n

Perintah untuk menghasilkan CSR adalah sebagai berikut: 

openssl req -new -key website\\server.key -out website\\server.csr\n\n\nExample question and answers:\nCountry Name (2 letter code) [AU]:GB\nState or Province Name (full name) [Some-State]: Hampshire\nLocality Name (eg, city) []: Portsmouth\nOrganization Name (eg, company) [Internet Widgits Pty Ltd]: Wamp Helpers Ltd\nOrganizational Unit Name (eg, section) []: Information Technology\nCommon Name (e.g. server FQDN or YOUR name) []: www.wamphelpers.dev\nEmail Address []: me@wamphelpers.dev\n\nPlease enter the following 'extra' attributes\nto be sent with your certificate request\nA challenge password []: ( leave blank just hit the enter key )\nAn optional company name []: ( leave blank just hit the enter key )\n

Langkah 3: Membuat Sertifikat yang Ditandatangani Sendiri

\n\n

Pada tahap ini Anda perlu membuat sertifikat yang ditandatangani sendiri karena\nAnda juga tidak berencana agar sertifikat Anda ditandatangani oleh CA, atau Anda ingin mengujinya\nimplementasi SSL baru Anda saat CA menandatangani sertifikat Anda.

\n\n

PRA - PERINGATAN Sertifikat ini akan menghasilkan kesalahan di browser klien yang menyebabkan itu\notoritas sertifikat penandatanganan tidak dikenal dan tidak dipercaya.\nIni tidak dapat dihindari karena kami sendiri yang menandatangani sertifikat, tetapi tentu saja web kepercayaan tidak mengetahui siapa kami. Lihat contoh nanti dalam dokumen ini yang menunjukkan cara memberi tahu browser Anda bahwa Anda benar-benar mempercayai sertifikat ini 

openssl x509 -req -days 365 -in website\\server.csr -signkey website\\server.key -out website\\server.crt\n\n\nExample output:\nLoading 'screen' into random state - done\nSignature ok\nsubject=/C=GB/ST=Hampshire/L=Portsmouth/O=WampHelpers Ltd/OU=Information Technology/CN=www.wamphelpers.dev/emailAddress=riggsfolly@wamphelpers.dev\nGetting Private key\n

Langkah 4: Memasang Kunci Pribadi dan Sertifikat

\n\n

Buat 2 direktori ini di bawah versi Apache yang Anda gunakan. 

md c:\\wamp\\bin\\apache\\apachex.y.z\\conf\\ssl.key\nmd c:\\wamp\\bin\\apache\\apachex.y.z\\conf\\ssl.crt\n

Dan salin file yang baru saja kita buat ke dalamnya seperti ini: 

copy website\\server.crt c:\\wamp\\bin\\apache\\apachex.y.z\\conf\\ssl.crt\ncopy website\\server.key c:\\wamp\\bin\\apache\\apachex.y.z\\conf\\ssl.key\n

Langkah 5: Konfigurasi Apache untuk mengaktifkan SSL

\n\n

Edit httpd.conf, Periksa apakah baris ini tidak diberi komentar 

LoadModule ssl_module modules/mod_ssl.so\n

Hapus juga komentar '#' dari baris ini 

Include conf/extra/httpd-ssl.conf\n

Kemudian pindahkan garis itu setelah blok ini .... seperti itu 

<IfModule ssl_module>\nSSLRandomSeed startup builtin\nSSLRandomSeed connect builtin\n</IfModule>\n\n# Secure (SSL/TLS) connections\nInclude conf/extra/httpd-ssl.conf\n

Langkah 6: Konfigurasi PHP untuk mengaktifkan SSL

\n\n

Edit php.ini Anda (gunakan menu wampmanager sehingga Anda mengedit yang benar)

\n\n

Hapus komentar ';' dari garis ini 

extension=php_openssl.dll\n

Langkah 7: Konfigurasikan Host Virtual situs aman Anda

File ini dirilis oleh Apache dan berisi beberapa lokasi file default.\nKita dapat membiarkan sebagian besar file ini apa adanya, tetapi kita perlu mengkonfigurasi host virtual di sini agar sesuai dengan lokasi situs kita yang sebenarnya dan beberapa hal lain jadi:

\n\n

temukan baris ini 

DocumentRoot \"c:/Apache2/htdocs\"\nServerName www.example.com:443\nServerAdmin admin@example.com\nErrorLog \"c:/Apache2/logs/error.log\"\nTransferLog \"c:/Apache2/logs/access.log\"\n

dan mengubahnya menjadi 

DocumentRoot \"c:/wamp/www/wamphelpers\"\nServerName wamphelpers.dev:443\nErrorLog \"c:/wamp/logs/ssl_error.log\"\nTransferLog \"c:/wamp/logs/ssl_access.log\"\n

Temukan 

SSLCertificateFile \"c:/Apache2/conf/server.crt\"\n

dan ubah menjadi 

SSLCertificateFile \"c:/wamp/bin/apache/apachex.y.x/conf/ssl.crt/server.crt\"\n

Temukan 

SSLCertificateKeyFile \"c:/Apache2/conf/server.key\"\n

dan ubah menjadi 

SSLCertificateKeyFile \"c:/wamp/bin/apache/apache2.2.26/conf/ssl.key/server.key\"\n

Temukan 

<Directory \"c:/Apache2/cgi-bin\">\n    SSLOptions +StdEnvVars\n</Directory>\n

dan ubah menjadi 

<Directory \"c:/wamp/www/wamphelpers\">\n    SSLOptions +StdEnvVars\n    Options Indexes FollowSymLinks MultiViews\n    AllowOverride All\n    Order Deny,Allow\n    Deny from all\n    Allow from 127.0.0.1 localhost ::1\n</Directory>\n

Temukan 

SSLSessionCache        \"shmcb:c:/Apache2/logs/ssl_scache(512000)\"\nSSLSessionCacheTimeout  300\n

dan mengubahnya menjadi 

SSLSessionCache        \"shmcb:c:/wamp/logs/ssl_scache(512000)\"\nSSLSessionCacheTimeout  300\n

Temukan 

CustomLog \"c:/Apache24/logs/ssl_request.log\" \\\n      \"%t %h %{SSL_PROTOCOL}x %{SSL_CIPHER}x \\\"%r\\\" %b\"\n

dan ubah menjadi 

CustomLog \"c:/wamp/logs/ssl_request.log\" \\\n          \"%t %h %{SSL_PROTOCOL}x %{SSL_CIPHER}x \\\"%r\\\" %b\"\n

Pada dasarnya lihat melalui file conf dan perintah apa pun yang tidak diberi komentar, tetapi memiliki referensi ke file atau folder\nharus diubah untuk mereferensikan struktur folder WAMPServer dan bukan 'C: / Apache2 ....'
