Saya ingin membuat folder bersama di server Debian saya sehingga pengguna dapat membuat dan mengedit file, tetapi tidak dapat saling menghapus file di direktori itu. 

/home/sharedfolder\n

Pengguna yang dapat membaca dan menulis file di peta berada dalam grup: work

\n\n

Saya mencoba beberapa perintah izin seperti ini, tetapi masih tidak berhasil: 

setfacl -dm u::rwx,g::rwx,o::r /shared/directory\n

Bagaimana saya bisa mencapai apa yang saya inginkan?

","source":"https://superuser.com/questions/1069880/linux-group-permission-user-have-read-and-write-but-cant-delete-each-other-fil","author":"m00p","replies":[{"reply":"Coba berikan izin penuh menggunakan perintah chmod 777 filename.txt","author":"manjesh23"},{"reply":"Apakah Anda mengetahui hak akses mana dan di mana yang bertanggung jawab untuk mengizinkan atau menolak penghapusan file?","author":"Run CMD"}],"answers":[{"answer":"

Solusi klasik untuk folder bersama adalah sticky bit, yang mencegah penghapusan file dari pengguna lain. Contoh terbaiknya adalah / tmp dir.

\n\n

Oleh karena itu, tetapkan kepemilikan berikut: 

chown root:work /shared/directory\n

dan izin: 

chmod 3775 /shared/directory\n

Saya akan merekomendasikan POSIX ACL berikut: 

setfacl -dm u::rwX,g::rwX,o::rX /shared/directory\n

Hanya pengguna grup work yang dapat mengedit file dan direktori yang baru dibuat tetapi tidak dapat menghapus file satu sama lain.
