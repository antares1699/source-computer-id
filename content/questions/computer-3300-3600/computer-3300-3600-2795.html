Saya mencoba menggunakan perintah grub-mkrescue untuk menghasilkan disk penyelamat. Saya memintanya seperti ini: 

grub-mkrescue -o grub.iso\n

Ini menghasilkan file .iso, tetapi ketika saya mencoba untuk mem-boot dari itu menggunakan VirtualBox, dikatakan bahwa media boot tidak dapat dibaca. Ketika saya membuka file dengan pengarsipan, .iso ternyata hanya memiliki direktori / boot / grub yang kosong.

\n\n

Saya juga memperhatikan beberapa orang menggunakan perintah seperti ini: 

grub-mkrescue -o grub.iso /boot/grub\n

Tapi kemudian mengeluh tentang direktori itu tidak ada. (Mungkin relevan bahwa saya memproduksinya di Debian VPS, karena saya tidak memiliki akses ke PC Linux yang sebenarnya saat ini.)

\n\n

Apa yang menyebabkan masalah ini? Apakah file GRUB hilang untuk disimpan ke dalam gambar yang dapat di-boot atau apakah saya menggunakan perintah dengan cara yang salah?

Langkah-langkah berikut menghasilkan gambar yang berfungsi untuk saya:

\n\n
    \n

  • buat file Multiboot hello world main.elf. GRUB tahu cara mem-boot file-file itu (GRUB juga tahu cara mem-boot kernel Linux, meskipun itu bukan Multiboot)

    • \n

  • buat file iso/boot/grub/grub.cfg yang berisi: 

    menuentry \"main\" {\n    multiboot /boot/main.elf\n}\n

    Tempatkan main.elf di bawah iso/boot/

    • \n

  • Di Ubuntu 17.10, di mesin UEFI: 

    sudo apt-get install grub-pc-bin\n

    seperti yang disebutkan di https://wiki.osdev.org/Bare_Bones Kalau tidak, itu tidak akan berhasil dan Anda tidak tahu caranya.

    \n\n

    TODO OMG benarkah ?! Saya harus belajar membangun GRUB dari sumber untuk menghilangkan jenis kegilaan ini.

    • \n

  • Hasilkan gambar dan jalankan: 

    grub-mkrescue -o main.img iso\nqemu-system-x86_64 -hda main.img\n
    • \n

Ternyata, ini terjadi ketika sistem tempat Anda menjalankan perintah ini tidak memiliki direktori /boot/grub (dapat diakses). Ini diam-diam akan gagal dan menghasilkan file ISO kosong.

\n\n

Saat menjalankan perintah yang sama pada penginstalan desktop normal Linux, perintah tersebut menghasilkan gambar bootable yang berfungsi.
