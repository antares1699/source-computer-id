Mengosongkan Sampah saya di Mac OS X 10.7 (Lion) (tidak aman) membutuhkan waktu sekitar tiga jam (sekitar 1,5 juta file, dari cadangan Time Machine). Saya harus menghentikan proses beberapa kali, karena saya tidak dapat memindahkan laptop saya dengan harddisk eksternal tempat file-file tersebut aktif. Ini juga menjadi masalah karena Pengosongan Sampah dimulai ulang dari awal setiap kali saya mengosongkan Sampah lagi (yaitu, file tidak dihapus saat Pengosongan Sampah dibatalkan).

\n\n

Saya membaca bahwa lebih cepat menggunakan rm -rf pada ~/.Trash, dalam kasus ini. Namun, apakah ini aman? (Saya khawatir Mac OS X 10.7 (Lion) melakukan tugas di belakang layar — yang akan menjelaskan kelambatannya — bahwa rm -r tidak melakukannya, yang dapat menyebabkan masalah di masa mendatang.)

","source":"https://superuser.com/questions/506370/emptying-the-trash-takes-forever-is-using-rm-r-safe-to-use-in-mac-os-x-10-7","author":"Eric O Lebigot","replies":[],"answers":[{"answer":"

Itu aman. OS X mungkin hanya mendeteksi file yang sedang digunakan oleh beberapa program, atau sedang berjalan. Jika Anda tahu Anda benar-benar menginginkannya hilang, rm itu. Reboot berikutnya, proses apa pun yang digunakan akan hilang.

\n\n

HATI-HATI menggunakan rm -r Ini secara rekursif menghapus semua subdirektori di bawah direktori yang ditentukan. Jika Anda tidak terbiasa dengan shell, saya sarankan untuk menggunakan rm dan secara eksplisit menentukan file yang ingin Anda hapus (meskipun Anda mengatakan ada banyak file).

\n\n

Coba ini:\nrm -r ~/.Trash/*

\n\n

Itu akan menghapus semua file dan folder di dalam folder .Trash, tanpa menghapus folder .Trash.
