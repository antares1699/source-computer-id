Setelah menghabiskan 2-3 hari untuk ini, saya menemukan solusi untuk ini.\nBenar-benar perangkat lunak sumber terbuka perlu mengembangkan dokumentasi yang lebih baik tentang cara menggunakan perangkat lunak !!

\n\n

Peringatan ketidakcocokan kecepatan usb di atas muncul karena kami memuat driver pengontrol host usb yang lebih lama (usb-bus = usb1.x), sedangkan perangkatnya adalah usb2.x / usb-ehci atau usb3.x / usb-xhci. Oleh karena itu terjadi ketidakcocokan kecepatan usb.

\n\n
    \n

  1. Muat pengontrol host usb bersama dengan perintah qemu.

    \n\n

    qemu-system-x86_64 --enable-kvm -m 4G -monitor stdio -usb -device nec-usb-xhci,id=xhci -device usb-ehci,id=ehci /pathtovirtdiskfile.vmdk

    2. \n

  2. Lihat perangkat host usb di terminal qemu dengan menjalankan perintah di bawah ini.

    \n\n

    info usbhost

    \n\n

    perangkat apple akan terdaftar di sini seperti di bawah ini:

    \n\n

    Bus 3, Addr 8, Port 3, Speed 480 Mb/s\n Class 00: USB device, iPhone

    3. \n

  3. Tambahkan perangkat host usb yang dikembalikan dari perintah di atas ke sistem tamu.

    \n\n

    device_add usb-host,hostbus=3,hostaddr=8,id=usb2.0,bus=ehci.0,port=3

    4. \n
\n\n

Sekarang perangkat apple akan muncul di sistem tamu.

Saya ingin memperbaiki solusi sccott.

\n\n

Saya menambahkan ini pada parameter boot qemu saya, dan keuntungannya adalah cocok dengan vendorid / productid, jadi tidak peduli di mana / kapan usb adalah plugin, itu akan bekerja secara otomatis. 

-device usb-ehci,id=ehci -device usb-host,vendorid=0x05ac,productid=0x12a8,bus=ehci.0,port=1\n

Penjelasan singkat tentang setiap bendera.

\n\n
    \n
  • -device usb-ehci,id=ehci membuat pengontrol usb ehci yang dapat dipasang oleh perangkat apple.
    • \n
  • -device usb-host,vendorid=0x05ac,productid=0x12a8 melewati perangkat usb dengan ID vendor dan ID produknya. (Yang ini rumit, karena ada sintaks yang mirip -usbdevice host:05ac:12a8 tetapi tidak mengambil parameter selanjutnya
    • \n
  • bus=ehci.0 menyambungkan perangkat apple ke hub USB2.0. Seperti yang dikatakan Sccott, ini akan menghilangkan peringatan ketidakcocokan kecepatan.
    • \n
  • Perangkat apple port=1 memiliki beberapa port USB dari satu alamat (kabel usb), jadi kita perlu memilih port pertama untuk keperluan sinkronisasi.
    • \n
