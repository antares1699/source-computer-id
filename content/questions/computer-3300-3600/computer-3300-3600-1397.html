Chrome berfungsi dengan baik saat saya menggunakan komputer saya secara fisik, tetapi saat saya menyambungkannya melalui RDP (Desktop Jarak Jauh), halaman tidak dimuat. Baik IE dan Firefox berfungsi dengan baik di semua situasi.

\n\n

Apakah ini bug bug yang dikenal di Chrome atau apa? Saya tidak menemukan informasi apa pun di Google. Perhatikan bahwa ini tidak ada hubungannya dengan ekstensi "Chrome Desktop Jarak Jauh".

Ini hanya tebakan, tapi mungkin ada hubungannya dengan akselerasi GPU yang tidak berfungsi melalui RDP. Bisakah Anda mencoba menambahkan bendera --disable-gpu setelah chrome.exe di pintasan?

\n\n

Atau mungkin mematikan item terkait GPU di chrome://flags/?

Terima kasih atas jawaban yang benar, Louis, berikut beberapa petunjuk:

\n\n

Memulai Chrome dengan opsi --disable-gpu berfungsi untuk saya. Untuk melakukannya, buat file jalan pintas dengan target sebagai berikut: 

\"C:\\Program Files (x86)\\Google\\Chrome\\Application\\chrome.exe\" --disable-gpu\n

Menonaktifkan penggunaan akselerasi grafis di chrome: // flags tidak berhasil untuk saya.

\n\n

Juga, pastikan bahwa Anda hapus centang "Biarkan Google Chrome berjalan di latar belakang" di bilah tugas. Jika tidak, opsi --disable-gpu tidak berguna.
