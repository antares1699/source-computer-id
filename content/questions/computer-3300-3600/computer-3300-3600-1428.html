Cara yang sama seperti di Linux: 

echo \"Hello\" >text.output\n

Kesalahan biasanya dicetak ke aliran stderr (ini # 2). Anda dapat merekam streaming stderr: 

del 1.txt 2>text.output\n

Jika file 1.txt tidak ada, text.output akan berisi Could not find 1.txt.

\n\n
\n\n

Anda dapat mengalihkan stdout ke stderr: 

echo \"Hello\" 2>&1\n

Atau stderr hingga stdout: 

echo \"Hello\" 1>&2\n
\n\n

P.S. Saya tidak yakin perintah Anda akan berfungsi di Linux. Pokoknya echo \"hello\" >& text.output menghasilkan kesalahan sintaks: >& was unexpected at this time.

Shell di Windows sekarang sangat mirip dengan BASH di Linux.\nJadi Anda bisa melakukannya 

echo \"Hello\" > text.txt\n

Atau, jalankan program Anda, dan biarkan keluaran sistem menulis ke text.txt: 

run_program > text.txt\n

Jika Anda perlu melacak kesalahan, gunakan yang berikut ini: 

run_program 2> error.txt\n
