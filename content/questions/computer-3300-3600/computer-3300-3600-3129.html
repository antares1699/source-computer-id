Saya telah menyiapkan reverse proxy di LAN tempat kerja saya sehingga beberapa situs dapat diakses dari luar LAN kami. Saya telah menyertakan mod-proxy-HTML, sehingga menangani penulisan ulang URL dalam HTML yang dikembalikan.

\n\n

Namun Firefox dan Chromium mengeluhkan masalah encoding.

\n\n

Jika saya mengedit opsi network.http.accept-encoding dari Firefox dan menghapus gzip;deflate default; maka situs tersebut terlihat baik-baik saja.

Konfigurasi Apache seperti ini: 

SetOutputFilter INFLATE;proxy-html;DEFLATE\n

memecahkan masalah saya.

Gunakan ini:

\n\n

ProxyHTMLCharsetOut *

\n\n

* ProxyHTMLCharsetOut\n Nama Syntax ProxyHTMLCharsetOut\n Ini memilih encoding untuk keluaran mod_proxy_html. Biasanya tidak boleh digunakan, karena setiap perubahan dari UTF-8 default (Unicode - seperti yang digunakan secara internal oleh libxml2) akan memberlakukan overhead pemrosesan tambahan. Token khusus ProxyHTMLCharsetOut * akan menghasilkan keluaran menggunakan pengkodean yang sama sebagai masukan. *
