Saya bekerja sebagai Arsitek OS Utama untuk Jolicloud (Joli OS) selama beberapa tahun terakhir, dan juga telah melakukan cukup banyak pekerjaan menyesuaikan OS Chromium untuk proyek lain. Untuk menjawab pertanyaan Anda, saya pasti bisa mengatakan itu yang terakhir untuk kedua OS:

\n\n
\n

Struktur OS berbasis browser web adalah browser yang terus-menerus dijalankan dipreteli OS berbasis Linux.

\n
\n\n

Joli OS dan Chrome OS memiliki beberapa kesamaan dalam desain inti, dan beberapa perbedaan utama antara satu sama lain dan jika dibandingkan dengan OS Linux yang lebih tradisional. Biar saya jelaskan :)

\n\n

Joli OS menggunakan fondasi Linux berbasis Ubuntu (yang memasok kernel, X.org, driver ruang pengguna, UI, dll.) Saya bekerja untuk menghapus sebanyak mungkin UI berbasis Gnome dan aplikasi lokal, menyediakan sebanyak mungkin UX melalui antarmuka aplikasi web HTML5. Jadi, dari perspektif komputer, ketika Joli OS memulai Kernel, yang memuat driver sistem, driver ruang pengguna, dan skrip init ruang pengguna. Skrip ini memulai X.org, yang memuat pengelola sesi kami, yang pada gilirannya memuat shell Gnome yang dilucuti, dan akhirnya Chromium ke http://my.jolicloud.com halaman awal.

\n\n

Chrome OS sangat mirip dalam desain ini, kecuali beberapa perbedaan utama: Pertama, ia menggunakan fondasi berbasis Gentoo (Google pernah menggunakan Ubuntu, tetapi beralih darinya pada awal 2010 ). Kedua, setelah Kernel, driver, dan X.org dimulai, Chrome sendiri menangani manajer sesi yang sebenarnya "memasukkan" pengguna. (Langkah ini juga mendekripsi direktori beranda pengguna, dan menyediakan fitur keamanan lainnya.) Selain itu, selain shell Gnome, Chrome juga menangani fitur sistem pengelola jendela tradisional seperti konfigurasi wifi, akun pengguna, dan memulai penutupan dan penangguhan acara, dll. Terakhir, halaman awal lokal yang menampilkan aplikasi web Chrome yang diinstal dimuat di samping UI browser web tradisional.

\n\n

Singkatnya, perbedaan utama dalam struktur Chrome OS adalah bahwa Chrome sendiri memenuhi lebih banyak tugas UX yang secara tradisional disediakan oleh pengelola sesi dan jendela seperti Gnome. Dengan melakukan semuanya di browser, dan menghapus aplikasi lokal sepenuhnya (Kebetulan, Joli OS berusaha untuk mempertahankan aplikasi lokal lama) pengalaman boot-up yang lebih cepat dan lebih efisien dapat disajikan kepada pengguna. Perbedaan tambahan ada melalui, yaitu, Chrome OS memodifikasi Kernel, driver, skrip-startup, dan X.org itu sendiri untuk bekerja lebih efisien ke perangkat keras target yang dirancang untuk Chrome OS.

\n\n

Jangan lupa bahwa Ubuntu, Gentoo, dan bahkan Joli OS semuanya dirancang untuk menjalankan satu ISO build pada sebanyak mungkin varian perangkat keras, tetapi Chrome OS dirancang untuk hanya berjalan pada profil perangkat keras yang sangat spesifik. Ini adalah perbedaan utama lainnya: Chrome OS dapat menghapus dukungan untuk profil perangkat keras non-target pada waktu pembuatan, juga menghasilkan UX yang lebih efisien dengan distribusi dan partisi boot yang lebih kecil.

\n\n

Meskipun tidak ada yang akan mengklaim bahwa chrome dapat melakukan semua yang dapat dilakukan bash, bagi sebagian besar pengguna, browser dapat menangani semua yang mereka inginkan.

\n\n
