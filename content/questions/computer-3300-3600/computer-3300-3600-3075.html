Sayangnya, server kami cukup terkunci sehingga jika Anda ingin melakukan sesuatu, Anda harus memiliki akses root. Jelas (meskipun ini jelas tidak disukai), kami bosan mengetik sudo sebelum setiap perintah yang kami ketik, yang mengharuskan kami terus-menerus mengetikkan kata sandi yang sangat rumit yang diamanatkan kepada kami berulang kali, jadi tentu saja kami semua hanya mengeksekusi perintah sudo su - saat login untuk menghindari semua ini.

\n\n

gunakan sudo -i untuk mendapatkan shell root dengan lingkungan pengguna saat ini masih dimuat.

\n\n

man sudo :

\n\n
\n

Opsi -i (simulasikan login awal) menjalankan shell yang ditentukan di entri passwd (5) pengguna tempat perintah dijalankan. Argumen nama perintah yang diberikan ke shell dimulai dengan - untuk memberi tahu shell agar dijalankan sebagai shell login. sudo mencoba mengubah ke direktori home pengguna itu sebelum menjalankan shell. Ini juga menginisialisasi lingkungan, membiarkan TERM tidak berubah, menyetel HOME, SHELL, USER, LOGNAME, dan PATH, dan menghapus semua variabel lingkungan lainnya. Perhatikan bahwa karena shell yang akan digunakan ditentukan sebelum file sudoers diurai, pengaturan runas_default di sudoers akan menentukan pengguna untuk menjalankan shell tetapi tidak akan mempengaruhi shell mana yang sebenarnya dijalankan.
