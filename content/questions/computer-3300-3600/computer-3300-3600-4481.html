Berdasarkan FAQ :

\n\n
\n

Sejak versi 3.7, untuk terminal baru untuk memulai di direktori kerja dari terminal saat ini, shell yang berjalan di terminal harus bekerja sama. Untuk ini, vte sejak versi 0.34.5 menginstal skrip /etc/profile.d/vte.sh yang ketika bersumber akan mengaturnya secara otomatis. Ini mendukung bash(1) dan zsh(1).

\n
\n\n

Saya melihat /etc/profile.d/vte.sh, tapi sejujurnya saya tidak tahu di mana dalam skrip itu hal ini terjadi. Dengan mengabaikan itu, Anda dapat mencapai hasil yang Anda inginkan dengan mengubah preferensi gnome-terminal. Bukan solusi yang sangat elegan, tetapi berhasil untuk saya yang menggunakan zsh dan Debian 10.

Sepertinya tidak ada cara untuk menimpa perilaku ini dari dalam gnome-terminal itu sendiri. Solusi termudah yang saya temukan adalah menambahkan cd eksplisit ke file ~/.bashrc Anda. 

echo 'cd $HOME' >> ~/.bashrc\n

Kemudian ketika Anda membuka tab baru, itu akan mengubah direktori saat ini kembali ke direktori home Anda.

Ini adalah fitur gnome-terminal yang sepertinya disukai semua orang kecuali saya dan thillai-selvan. Ada banyak dokumentasi tentang cara memperbaikinya jika tidak berfungsi dengan benar (yaitu, saat tab baru dibuka di direktori yang sama dengan tab sebelumnya). Namun, ada sedikit informasi tentang cara menonaktifkan fitur tersebut. Itu tidak dapat dinonaktifkan di gnome-terminal UI atau file konfigurasi apa pun sejauh yang saya tahu. Jadi, Anda perlu memecahkan atau menghindarinya.

\n\n

Untuk kasus penggunaan saya, echo 'cd $HOME' >> ~/.bashrc bukanlah solusi yang baik, karena dalam beberapa aplikasi saya ingin mereka dapat dibuka di direktori yang berbeda, terutama terminal tertanam di vscode, dan baris di bashrc akan mencegahnya.

\n\n

Fitur ini diimplementasikan dengan menyediakan PROMPT_COMMAND yang mencatat direktori saat ini setiap kali prompt dibuat. Versi gnome saat ini menerapkan ini dengan menginstal skrip vte ke /etc/profile.d. Anda harus dapat menghapus file itu (buat cadangannya sehingga Anda dapat memulihkannya nanti jika Anda mau) dan menonaktifkan fitur ini.
