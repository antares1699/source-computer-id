Saya mencoba membuat aplikasi Astrill VPN 32-bit v3.2 berfungsi di Ubuntu xenial 16.04.3.

\n\n

Saat mencoba masuk, saya mendapatkan pesan yang mengatakan "Tidak dapat memuat pustaka SSL".

\n\n

Satu-satunya saran dukungan Astrill adalah mencopot, me-reboot, dan menginstal ulang aplikasi, yang tidak membantu.

\n\n

Saya juga mencoba menginstal opensl versi i386, yang juga tidak membantu.

\n\n

Apa lagi yang bisa saya lihat untuk mencoba dan menyelesaikan ini?

Pertama, pastikan Anda telah menginstal openssl. Anda memerlukan versi 1.0 karena Astrill tidak mendukung 1.1. Untuk menginstal 1.0.2, jalankan 

sudo apt install openssl=1.0.2g-1ubuntu4.8\n

Kemudian instal pustaka bersama, periksa dulu versi mana yang Anda inginkan dengan menjalankan apt-cache search libssl | grep SSL. Contoh keluaran: 

libssl-dev - SSL development libraries, header files and documentation\nlibssl-doc - SSL development documentation documentation\nlibssl1.0.0 - SSL shared libraries  <-- version number\n

Instal perpustakaan dengan versi yang benar: 

sudo apt-get install libssl1.0.0\n

Dari percakapan di komentar, Anda juga perlu menginstal libssl-dev, sekali lagi, pastikan versinya 1.0. 

sudo apt install libssl-dev=1.0.2g-1ubuntu4.8\n

Mungkin Anda perlu menginstal paket ca-certificates juga.

\n\n
\n\n

Jika hal di atas tidak menyelesaikan masalah, buka file /etc/ld.so.conf sebagai sudo dan tambahkan baris baru di akhir 

/usr/lib/x86_64-linux-gnu/openssl-1.0.0\n

Bergantung pada sistem Anda, jalurnya bisa berbeda. Dua lokasi lain untuk dicoba adalah /usr/local/lib dan /usr/lib/openssl-1.0, lihat sini dan sini .

\n\n

Kemudian jalankan sudo ldconfig.
