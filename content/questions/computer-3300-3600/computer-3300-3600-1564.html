Booting jaringan berbeda dengan boot dari hard drive, dan Anda tidak dapat benar-benar mem-boot dari jaringan bersama Windows.

\n\n

Komputer yang ingin Anda boot dari jaringan harus mendukung PXE, yang paling banyak dilakukan saat ini. PXE bekerja saat Anda menyalakan komputer dan mengambil alamat dari server DHCP. Server DHCP juga harus dikonfigurasi dengan opsi yang memberitahukan ke mana harus pergi untuk mendapatkan bootloader. Bootloader tersebut kemudian diambil menggunakan tftp. Anda juga harus menjalankan server tftpd yang menjadi host bootloader tersebut.

\n\n

Tftpd32 adalah program Windows bagus yang dapat bertindak sebagai server DHCP dan TFTP jika Anda ingin bereksperimen.

\n\n

Bootloader kemudian bertanggung jawab untuk memuat dan meluncurkan OS. Bootloader tersebut harus memiliki kemampuan untuk menginisialisasi OS yang sadar "drive jaringan". Linux mendukung menjalankan langsung dari drive NFS yang terpasang, tetapi Windows tidak benar-benar mendukung itu sejauh yang saya tahu. Dengan Windows, Anda memiliki Remote Installation Services (RIS) dan sekarang, Windows Deployment Services. Ini adalah lingkungan minimal yang dirancang untuk mendukung pencitraan dan penginstalan ulang jarak jauh, jadi saya tidak tahu bagaimana itu akan berjalan sejauh hanya menjalankan OS dari jaringan berbagi.
