Saya mengembangkan perangkat pemantauan jarak jauh menggunakan Raspberry Pi. Untuk memberi saya peringatan, ada dongle 3G di dalamnya. Namun, karena ketidakstabilan catu daya, dongle terkadang akan mengubah nomor perangkat. Jadi, misalnya, saat boot, dongle 3G mungkin ditampilkan sebagai (menggunakan lsusb):

\n\n

Bus 001 Perangkat 005: ID 12d1: 1464 Huawei Technologies Co., Ltd

\n\n

Tetapi setelah beberapa jam berlalu, jumlah perangkat akan meningkat menjadi, katakanlah:

\n\n

Bus 001 Perangkat 007: ID 12d1: 1464 Huawei Technologies Co., Ltd

\n\n

Jadi pertanyaan saya adalah berapa batasan nomor perangkat? Apakah bisa terus meningkat hingga 999 atau bahkan lebih atau ada batas yang lebih rendah, seperti 256 atau 128? Dengan informasi ini saya dapat mengkodekan cek pada nomor perangkat dan memaksa reboot jika mendekati batas.

USB menggunakan pengalamatan 7bit sehingga Anda dapat menangani 127 perangkat. Tentu saja masalah yang lebih besar adalah daya saat menggunakan perangkat USB, jika tidak ada cukup daya, mereka tidak akan berfungsi.

Penomoran perangkat Linux pada bus USB menyimpan informasi tentang "nomor perangkat berikutnya yang akan ditetapkan". Ini secara alami diperbarui setiap kali perangkat baru terhubung.

\n\n

Selama nomor perangkat dari perangkat yang terhubung saat ini kurang dari 127, "nomor perangkat berikutnya" yang baru hanya akan menjadi nomor perangkat saat ini ditambah satu. Pada "nomor perangkat saat ini" 127, "nomor perangkat berikutnya" disetel ulang ke 1.

\n\n

Alokasi perangkat juga memeriksa apakah nomor yang diusulkan untuk ditetapkan sudah digunakan, dan jika demikian, akan kembali memeriksa nomor perangkat berturut-turut sampai yang gratis ditemukan.

\n\n

Jadi, secara keseluruhan, ada batas nomor perangkat, tetapi setelah mencapainya, penomoran akan bergulir dengan anggun.
