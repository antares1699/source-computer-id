Salah satu cara untuk melakukannya adalah buat Symbolic Links ke file (program) yang dimaksud.\nMisalkan Anda memiliki satu folder (C:\\bin) yang Anda tambahkan ke jalur sistem Anda. Lalu di folder itu\nAnda dapat membuat Symbolic Links ke file executable yang ingin Anda akses dengan cepat.

\n\n

Sekarang, anggaplah ada beberapa aplikasi yang ingin Anda akses cepat, baik dari\nbaris perintah atau dari kotak Windows / Run. Misalnya, daftar program Anda mencakup: 

  \"C:\\Program Files\\7-Zip\\7z.exe\"\n  \"C:\\Program Files\\AutoHotkey\\AutoHotkey.exe\"\n  \"C:\\Program Files\\iTunes\\iTunes.exe\"\n  \"C:\\Program Files (x86)\\Microsoft Visual Studio 11.0\\Common7\\IDE\\devenv.exe\"\n  \"C:\\Program Files (x86)\\Notepad++\\notepad++.exe\"\n

Untuk membuat tautan simbolik di atas, Anda dapat: 

  mkdir c:\\bin\n  cd /d c:\\bin\n\n  mklink 7z.exe \"C:\\Program Files\\7-Zip\\7z.exe\"\n  mklink ahk.exe \"C:\\Program Files\\AutoHotkey\\AutoHotkey.exe\"\n  mklink iTunes.exe \"C:\\Program Files\\iTunes\\iTunes.exe\"\n  mklink devenv.exe \"C:\\Program Files (x86)\\Microsoft Visual Studio 11.0\\Common7\\IDE\\devenv.exe\"\n  mklink npp.exe \"C:\\Program Files (x86)\\Notepad++\\notepad++.exe\"\n

Anda akan melihat bahwa saya telah "mengganti nama" AutoHotKey.exe menjadi ahk.exe dan Notepad++.exe menjadi npp.exe dalam contoh ini, sehingga memberikan nama singkatan untuk mereka.

\n\n

Seperti banyak hal di Windows, ada cara lain yang benar-benar valid untuk mencapai hal yang sama. Satu akan dibuat Windows\nAlias ​​Baris Perintah . Yang lain akan digunakan AutoHotKey . Yang lain lagi adalah membuat Pintasan Windows\ndan tambahkan ekstensi .lnk ke variabel lingkungan PATHEXT sebagai @Pengen N menjelaskan dalam jawabannya. Atau, Anda bahkan bisa menggunakan doskey.exe untuk mencapainya, seperti yang dijelaskan dalam ini menempatkan pada StackOverflow .

\n\n

Secara pribadi, saya terutama menggunakan alias karena saya cenderung sering menggunakan Windows Command Line. Saya juga menggunakan AutoHotKey, bagaimanapun, ketika situasi membutuhkannya.

Anda perlu memperbarui variabel lingkungan sistem PATHEXT. Variabel tersebut mencantumkan jenis file yang dianggap dapat dieksekusi untuk tujuan pencarian jalur. Tambahkan ;.LNK ke nilai variabel itu, dengan demikian membuat pintasan (.lnk) menjadi sesuatu yang akan dicari di PATH Anda. Perhatikan bahwa Anda mungkin perlu memulai ulang perintah perintah yang berjalan agar perubahan diterapkan.
