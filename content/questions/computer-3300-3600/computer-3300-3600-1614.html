Apakah mungkin, dengan menggunakan ps, untuk menentukan dari mana suatu proses berjalan?

\n\n

Saya memiliki dua aplikasi, keduanya identik dan berjalan di direktori paralel, seperti /app1/start.sh dan /app2/start.sh. Jika saya menjalankan ps -ef maka saya tidak dapat membedakan keduanya

\n\n

Menggunakan ps (atau alternatif), bagaimana cara mengetahui bahwa PID 123 adalah milik app1?

","source":"https://superuser.com/questions/209272/linux-ps-show-where-process-is-running-from","author":"user155695","replies":[],"answers":[{"answer":"

Beberapa contoh:

\n\n

Daftar proses saat ini disajikan dalam hierarki hierarki dan keluaran lebar (menampilkan argumen baris perintah lengkap dan tidak memotongnya setelah karakter X) ps -auxwwwf

\n\n

Command pstree, tampilkan semua proses dan id prosesnya: pstree -ap

\n\n

Atau, jika Anda hanya tertarik pada PID: pgrep -f app1 akan mencantumkan semua PID untuk proses yang cocok dengan pola app1 di suatu tempat di sepanjang nama, argumen baris perintah, atau jalurnya.

Lihat juga filesystem /proc/. Pada /proc/*pid* Anda memiliki:

\n\n
    \n
  • exe yang merupakan symlink ke file\nbiner yang dapat dieksekusi. Dalam kasus a\nskrip, ini adalah juru bahasa (/ bin / bash) bukan skrip.
    • \n
  • cwd\nyang merupakan symlink ke arus\ndirektori proses
    • \n
  • cmdline\nyang dipisahkan oleh NUL\nrepresentasi dari baris perintah
    • \n
\n\n

Anda juga memiliki daftar deskriptor file saat ini. Saya menemukan /proc/*pid*/fd/255 berguna dalam skrip bash - ini adalah symlink ke skrip itu sendiri.

\n\n

ps melihat / proc / untuk mencari info, jadi mungkin ada beberapa mantera ajaib dari bendera yang menunjukkan beberapa di antaranya - lihat ps -o
