Benar, saya cukup baru di linux 'Ubuntu 10.10' dan saya mengalami kesulitan mencoba mengimpor kunci dan sertifikat untuk koneksi vpn ke server kantor kantor saya.

\n\n

\n\n

Bagaimana saya mengimpor file-file ini untuk mendapatkan koneksi dan berjalan.

Untuk melengkapi jawaban @barbaz:

\n\n

Anda perlu mengonversi file .p12 menjadi sertifikat CA, sertifikat klien, dan kunci pribadi klien.

\n\n

Perintah baris perintah untuk tindakan ini adalah: 

openssl pkcs12 -nocerts -in default.p12 -out userkey.pem\nopenssl pkcs12 -nokeys -clcerts -in default.p12 -out usercert.pem\nopenssl pkcs12 -nokeys -cacerts -in default.p12 -out userca.pem\n

Ini akan meminta kata sandi untuk kunci pribadi klien Anda, Anda menulis apa yang Anda inginkan di sana, dan mengingat kata sandi itu untuk nanti.

\n\n

Kemudian Anda memilih opsi 'Sertifikat (TLS)', dan pilih tiga file, dan tambahkan kata sandi yang Anda tambahkan di baris perintah.

Jika Anda tidak terbiasa dengan openvpn, Anda mungkin ingin mencoba menggunakan NetworkManager untuk mengkonfigurasinya. Afaik ada paket bernama network-manager-openvpn yang akan mengaktifkan openvpn di bagian VPN NetworkManager Anda. File .p12 harus sudah berisi sertifikat CA, sertifikat klien dan kunci pribadi klien. Jika NetworkManager tidak dapat menggunakan file itu (tetapi hanya menerima masing-masing dari tiga file satu per satu dalam format PEM), Anda mungkin perlu mengonversi file Anda.

\n\n

Jika Anda memutuskan untuk menjalankan openvpn sendiri sebagai root, petunjuk konfigurasi untuk menggunakan file Anda adalah 

pkcs12 client.p12\n

sebagaimana didokumentasikan pada openvpn(8).
