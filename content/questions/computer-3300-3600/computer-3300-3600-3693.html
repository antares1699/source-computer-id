MAC tidak menggunakan PXE yang "persis"; mereka menggunakan lingkungan serupa yang disebut Boot Server Discovery Protocol (BSDP).

\n\n

Server OS X menyertakan alat sistem yang disebut NetBoot. Klien NetBoot menggunakan BSDP untuk secara dinamis memperoleh sumber daya yang memungkinkannya untuk boot sistem operasi yang sesuai. BSDP dibuat di atas DHCP menggunakan informasi khusus vendor untuk menyediakan fungsionalitas NetBoot tambahan yang tidak ada di DHCP standar. Protokol diimplementasikan dalam firmware klien. Pada saat boot, klien mendapatkan alamat IP melalui DHCP kemudian menemukan server boot menggunakan BSDP. Setiap server BSDP merespon dengan informasi boot yang terdiri dari:

\n\n
    \n
  1. Daftar image sistem operasi yang dapat di-boot
    2. \n
  2. Gambar sistem operasi default
    3. \n
  3. Gambar sistem operasi klien yang saat ini dipilih (jika ditentukan)
    4. \n
\n\n

Klien memilih sistem operasi dari daftar dan mengirimkan pesan ke server yang menunjukkan pilihannya. Server boot yang dipilih merespons dengan menyediakan file boot dan image booting, dan informasi lain yang diperlukan untuk mengunduh dan menjalankan sistem operasi yang dipilih.

\n\n

Mungkin untuk PXEboot Intel Mac dengan terlebih dahulu mem-boot utilitas seperti firmware PXE pengganti IPXE:

\n\n

http://ipxe.org/

\n\n

Jika mac Anda memiliki drive CD / DVD, ini tidak terlalu buruk, Anda cukup membakar file ISO IPXE ke CD dan menempelkannya di drive CD / DVD kapan pun Anda ingin boot PXE.

\n\n

Jika mac Anda cukup baru sehingga tidak memiliki drive CD / DVD maka saya pikir Anda dapat membuat drive USB yang dapat di-boot menggunakan versi EFI dari IPXE. Namun, ini jauh lebih terlibat. Mac tampaknya tidak mau mem-boot stik USB berformat MBR, jadi Anda tidak bisa begitu saja mendapatkan image USB Langsung yang mendukung netboot. Saya sebenarnya telah mencoba mencari cara untuk melakukan ini sendiri dan belum cukup menemukan apa yang diperlukan.
