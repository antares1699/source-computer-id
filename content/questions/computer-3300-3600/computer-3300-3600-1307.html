Membaca halaman manual find memberikan sedikit informasi berguna:

\n\n
\n

Ekspresi reguler yang dipahami oleh find secara default adalah Emacs\n Ekspresi Reguler.

\n
\n\n

Jadi, lihat sintaks dari Emacs Regex

\n\n

\n\n

Tampaknya semua karakter khusus harus di-escape dengan \\. Dengan mempertimbangkan hal ini, hasil yang diharapkan diperoleh setelah mengubah ekspresi reguler menjadi berikut: 

find /path/to/images/ -regex \".*\\(26[2-7]\\|27[0-2]\\).*\" -exec echo {} \\;\n

Saya berkonsultasi dengan halaman manual sebelum memposting pertanyaan ini, tetapi tampaknya tidak cukup teliti.

Saya menyadari Anda memposting jawaban Anda sendiri, tetapi jika Anda ingin menggunakan sintaks REGEX, Anda dapat mengubah jenis REGEX menggunakan perintah ini:

\n\n

-regextype TYPE

\n\n

Jadi saya yakin untuk membuat Anda berhasil:

\n\n

find /path/to/images/ -regextype sed -regex '.*(26[2-7]|27[0-2]).*' -exec echo {} \\;

\n\n

mungkin berhasil.

\n\n

Selain itu, hanya catatan, regex Anda akan cocok dengan apa pun yang memiliki 262-267 atau 270-272 di dalamnya. Itu berarti akan menangkap IZ262561.BMP, serta IZ134267.JPG, dan ASDNASDNOANFOIAJFOIJDIO000262.EXE misalnya. Saya tidak yakin file apa yang sedang Anda kerjakan, tetapi Anda mungkin perlu mempersempit regex Anda karena cukup longgar pada apa yang akan diterima.

Ada juga operator -or (atau -o) ke perintah find. Anda dapat membagi ekspresi Anda menjadi dua subekspresi: 

find /path/to/images/ -regex \".*(26[2-7]).*\" -o -regex \".*(27[0-2]).*\" -exec echo {} \\;\n

-o sesuai dengan POSIX, sedangkan -or tidak:

\n\n

Sesuai halaman manual find:

\n\n
   expr1 -o expr2\n          Or; expr2 is not evaluated if expr1 is true.\n\n   expr1 -or expr2\n          Same as expr1 -o expr2, but not POSIX compliant.\n
