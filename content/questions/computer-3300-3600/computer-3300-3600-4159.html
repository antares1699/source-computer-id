Saya menyadari ini adalah pertanyaan lama, tapi ...

\n\n

Saya percaya Gargoyle (dan banyak firwares pihak ketiga lainnya untuk router) digunakan dnsmasq.dll untuk layanan DNS / DHCP mereka, yang secara default memberikan IP berdasarkan hash dari alamat MAC perangkat. Dokumentasi memberikan sedikit lebih banyak detail:

\n\n
\n

--dhcp-sequential-ip

\n \n

Dnsmasq dirancang untuk memilih alamat IP untuk klien DHCP menggunakan hash\n dari alamat MAC klien. Ini biasanya memungkinkan klien\n alamat agar tetap stabil dalam jangka panjang, bahkan jika klien kadang-kadang\n mengizinkan sewa DHCP-nya kedaluwarsa. Dalam mode default ini, alamat IP adalah\n didistribusikan secara semu secara acak di seluruh rentang alamat yang tersedia.\n Terkadang ada keadaan (biasanya penerapan server) di mana\n akan lebih mudah jika alamat IP dialokasikan secara berurutan,\n mulai dari alamat terendah yang tersedia, dan pengaturan bendera ini\n mengaktifkan mode ini. Perhatikan bahwa dalam mode sekuensial, klien yang\n memungkinkan sewa untuk kedaluwarsa jauh lebih mungkin untuk memindahkan alamat IP; untuk\n alasan ini sebaiknya tidak digunakan secara umum.
