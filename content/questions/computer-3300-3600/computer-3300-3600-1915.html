Gunakan utilitas kecil HotSwap! .

\n\n

Ini menempatkan ikon di System Tray Anda seperti Keluarkan Media yang Dapat Dilepas ikon dan bekerja dengan cara yang sama. Saat Anda mengeluarkan disk hot-swap, HotSwap! perintahkan Windows untuk menyiapkan disk untuk penghapusan yang aman. Jika disk masih digunakan atau tidak siap untuk dihapus, HotSwap! akan mengingatkan Anda.

\n\n

Anda juga dapat menjalankan HotSwap! dari baris perintah, mengaktifkan penggunaannya dalam skrip. Saya menggunakannya bersama dengan hard drive hot-swap yang diputar keluar dari server untuk tujuan pencadangan dan menjalankannya pada server produksi Hyper-V Host selama beberapa tahun tanpa masalah.

\n\n

Keindahan alat ini adalah mengotomatiskan proses menggunakan alat MMC Manajemen Disk untuk menandai disk Offline sebelum memutusnya. Dengan kata lain, ini memanfaatkan fungsionalitas bawaan standar dalam Windows untuk mempersiapkan disk hot-swap Anda agar dapat dihapus dengan aman.

Windows mendukung SATA hot-plugging. Itu hanya tidak mendukung hot un-plugging.

\n\n

Masalah dengan Windows adalah bahwa drive SATA diperlakukan sebagai disk tetap dan tidak dapat dilepas. Tidak seperti sistem operasi berbasis * nix, Windows tidak menawarkan cara untuk menjamin unmount volume yang bersih.

\n\n

Jika Anda menghapus huruf drive dari Manajemen Disk, yang Anda lakukan hanyalah menghapus pegangan ke volume. Windows masih dapat mempertahankan kunci terbuka ke disk tanpa titik pemasangan. Jika Anda menonaktifkannya di Device Manager, itu tidak menjamin bahwa buffer tulis akan dibersihkan dengan benar. Windows tidak mendukung penghapusan disk tetap saat sistem sedang berjalan. Itu tidak berarti itu tidak bisa; t dilakukan. Artinya, tidak ada jaminan keamanan.
