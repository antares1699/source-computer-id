Di mana uTorrent menyimpan file yang dilewati / pengaturan prioritas?

\n\n

Dari apa yang saya lihat dengan BEncode Editor, pilihan tersebut tampaknya tidak disimpan di resume.dat, settings.dat, atau file torrent itu sendiri.

\n\n

Bisa jadi dalam .part atau .ut! file, tetapi dengan unduhan baru yang belum dimulai, file-file itu belum ada dan masih pilihannya disimpan.

\n\n

\n\n

Saya tidak dapat mengatakan dengan pasti file mana informasi tersebut disimpan (saya akan menyarankan resume.dat jika Anda belum mengecualikannya), tetapi jalankan cepat dengan Monitor Proses menunjukkan banyak janji.

\n\n

edit : Menanggapi komentar Anda, saya berasumsi Anda belum pernah bekerja dengan Monitor Proses sebelumnya. Jadi izinkan saya memberi sedikit detail tentang apa yang saya lakukan.

\n\n

Pertama-tama, mulai Monitor Proses. Kemudian mulai µTorrent dan lakukan perubahan pada prioritas file (atau perubahan apa pun yang ingin Anda ketahui lebih detailnya). Kemudian tutup µTorrent.
Anda sekarang akan menangkap ribuan peristiwa di Monitor Proses.
Cobalah untuk mencari contoh uTorrent dalam daftar, klik kanan dan pilih "Include uTorrent.exe". Ini akan membuat filter baru, yang akan menyembunyikan setiap acara yang tidak dibuat oleh uTorrent.exe.

\n\n

Sekarang Anda dapat melanjutkan untuk menyembunyikan tindakan tertentu. Misalnya, jika Anda mengklik kanan beberapa peristiwa "TCP Send" atau "UDP Receive", Anda dapat memilih untuk menyembunyikannya (perhatikan bahwa kolom tempat Anda melakukan klik kanan berpengaruh pada konten menu konteks Anda ).

\n\n

Membuat filter tersebut akan memungkinkan Anda dengan cepat menelusuri tindakan yang Anda minati.

\n\n

Saya, secara pribadi, dibiarkan dengan beberapa panggilan CreateFile ke file terkait µTorrent di profil roaming saya dan beberapa tindakan akses ke cache Internet Explorer (yang disebabkan oleh fungsionalitas terkait web di µTorrent yang saya asumsikan).

\n\n

Saya akan berasumsi Anda memiliki pengetahuan yang jauh lebih dalam tentang apa yang Anda harapkan untuk dilihat dan apa yang Anda minati, tetapi saya yakin Process Monitor akan mengungkapkan informasi yang Anda cari.

Saya salah, setelah tidak menemukan apa pun yang berguna dalam monitor proses, saya membandingkan resume.dats setelah mengubah prioritas dan memperhatikan bahwa itu memang disimpan dalam file resume.dat, di bawah kunci PRIO (data biner).

\n\n

Jumlah byte sebanyak jumlah file, dan urutannya sesuai dengan urutan default di uTorrent (yang saya asumsikan adalah urutan file muncul di file .torrent). Untuk setiap byte, nilai heksadesimal dapat berupa:

\n\n
    \n
  • 04: Prioritas rendah
    • \n
  • 08: Normal
    • \n
  • 0C: Prioritas Tinggi
    • \n
  • 00: Jangan unduh
    • \n
\n\n

Edit: Dikonfirmasi : Sebuah topeng byte berisi menunjukkan file mana yang memiliki prioritas apa, setiap byte mewakili satu file. Urutan byte sama dengan urutan yang ditemukan di file .torrent.

\n\n
