Jalankan perintah berikut: 

sudo rm /var/lib/dpkg/lock\n

Buat folder yang hilang (gunakan perintah ls -l /var/lib/dpkg/ terlebih dahulu untuk menampilkan folder yang ada, jika ada floder tidak membuatnya kembali lagi): 

sudo mkdir -p /var/lib/dpkg/{alternatives,info,parts,triggers,updates,methods,available,diversions}\n

Pindahkan dpkg.status.0 dan perbaiki file base: 

sudo cp /var/backups/dpkg.status.0 /var/lib/dpkg/status\nsudo apt-get download dpkg\nsudo dpkg -i dpkg*.deb\nsudo apt-get download base-files\nsudo dpkg -i base-files*.deb\nsudo apt update && sudo apt upgrade\n

Periksa dan perbaiki dependensi yang rusak: 

sudo apt-get check\nsudo apt -f install\n

Jalankan perintah di bawah ini untuk menghasilkan daftar semua proses yang namanya terdiri dari kata apt, Anda akan mendapatkan daftar yang mencakup semua proses apt atau apt-get dengan menggunakan ps dan perintah grep bersama dengan pipa. 

ps -A | grep apt\n

Untuk setiap proses apt-get atau apt yang dapat Anda lihat pada output dari perintah di atas, bunuh setiap proses menggunakan perintah di bawah ini.

\n\n

ID proses (PID) ditemukan di kolom pertama dari tangkapan layar di atas. 

sudo kill -9 processnumber\nor\nsudo kill -SIGKILL processnumber\n

Misalnya, dalam perintah di bawah ini di mana 9 adalah nomor sinyal untuk sinyal SIGKILL, akan menghentikan proses apt pertama: 

sudo kill -9 13431\nor\nsudo kill -SIGKILL 13431\n

Pertama-tama jalankan perintah di bawah ini untuk menghapus file kunci di direktori /var/lib/dpkg/: 

sudo rm /var/lib/dpkg/lock\n

Setelah itu paksa paket untuk mengkonfigurasi ulang seperti ini: 

sudo dpkg --configure -a\n

Atau, hapus file kunci di /var/lib/apt/lists/ dan direktori cache seperti di bawah ini: 

sudo rm /var/lib/apt/lists/lock\nsudo rm /var/cache/apt/archives/lock\n

Selanjutnya, perbarui daftar sumber paket Anda sebagai berikut: 

sudo apt update\nor\nsudo apt-get update\n
