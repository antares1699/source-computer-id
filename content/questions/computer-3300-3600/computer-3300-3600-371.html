Saya memiliki klien yang memberlakukan kebijakan di IE bahwa semua situs intranet lokal dimuat di Tampilan Kompatibilitas.

\n\n

Sayangnya aplikasi web kami tidak mendukung ini dan memberikan banyak masalah di comp. melihat.\nAdakah cara untuk secara khusus mengecualikan situs dari kebijakan ini tanpa harus memasukkannya ke dalam kode kami?

Dalam kebijakan grup, Anda bisa menggunakan daftar penetapan situs ke zona untuk menempatkan situs tersebut ke dalam zona situs tepercaya sebagai gantinya. Tampilan kompatibilitas dinonaktifkan secara default di zona itu.

Untuk memberi tahu Internet Explorer agar tidak menggunakan Mode Kompatibilitas di situs Anda, gunakan tag meta berikut (atau variasinya) di bagian head situs Anda: 

<meta http-equiv=\"X-UA-Compatible\" content=\"IE=9; IE=8; IE=EDGE\" />\n

Perhatikan bahwa tag ini harus jadilah tag pertama di dalam tag head. Untuk informasi lebih lanjut, kunjungi: Menentukan mode dokumen warisan

Di ASP.Net Anda juga dapat memasukkan yang berikut ini ke file web.config situs web: 

<system.webServer>\n<httpProtocol>\n    <customHeaders>\n        <clear/>\n        <add name=\"X-UA-Compatible\" value=\"IE=8; IE=9; IE=EDGE\" />\n    </customHeaders>\n</httpProtocol>\n
