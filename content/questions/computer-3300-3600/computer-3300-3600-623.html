Saya memiliki pengaturan yang relatif lengkap untuk ~/.ssh/config saya. Inilah yang ada di atas: 

Host *\n   ControlMaster auto\n   ControlPath /tmp/ssh_mux_%h_%p_%r\n   ControlPersist 24h\n   BatchMode yes\n   Ciphers blowfish-cbc\n   Compression yes\n   VisualHostKey yes\n

Saya ingin membuatnya sedemikian rupa sehingga ControlMaster dll tidak berlaku untuk github; mereka\ntampaknya secara aktif memutuskan koneksi persisten ini, yang saya mengerti\ndan hormat.

\n\n

Jadi untuk lebih jelasnya: 

 $ rm /tmp/ssh_mux_*\n $ ssh git@github.com\n   PTY allocation request failed\n   Hi frioux! You've successfully authenticated, but GitHub does not provide shell access.\n   Shared connection to github.com closed.\n\n $ ls /tmp/ssh_mux_*\n   /tmp/ssh_mux_github.com_22_git\n

Jadi saya membaca beberapa dokumen, saya bertanya di IRC, dan inilah yang akhirnya saya dapatkan: 

Host !github.com\n   ControlMaster auto\n   ControlPath /tmp/ssh_mux_%h_%p_%r\n   ControlPersist 24h\n\nHost *\n   BatchMode yes\n   Ciphers blowfish-cbc\n   Compression yes\n   VisualHostKey yes\n

Jadi sekarang github tidak menggunakan ControlMaster dengan benar: 

 $ rm /tmp/ssh_mux_*\n $ ssh git@github.com\n   Host key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48\n   +--[ RSA 2048]----+\n   |        .        |\n   |       + .       |\n   |      . B .      |\n   |     o * +       |\n   |    X * S        |\n   |   + O o . .     |\n   |    .   E . o    |\n   |       . . o     |\n   |        . .      |\n   +-----------------+\n\n   PTY allocation request failed\n   Hi frioux! You've successfully authenticated, but GitHub does not provide shell access.\n   Shared connection to github.com closed.\n\n $ ls /tmp/ssh_mux_*\n

Tapi begitu juga yang lainnya: 

 $ rm /tmp/ssh_mux_*\n $ ssh cs ls\n   Host key fingerprint is 89:d1:40:7f:0d:11:28:10:ce:23:e6:a9:12:9d:a1:5b\n   +--[ RSA 2048]----+\n   |    o+o  .+o     |\n   |   o  .+.  o     |\n   |  + + ..o . .    |\n   | = = . o o       |\n   |o E   . Saya mencoba membuat baris host Host !github.com, * dan Host *, !github.com tetapi\nsejauh yang saya tahu ketika saya melakukannya, itu selalu berlaku untuk github.

\n\n

Bagaimana saya bisa melakukan apa yang saya inginkan?
