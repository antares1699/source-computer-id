Saya menemukan solusinya! Di Windows 10 Anda tidak dapat menggunakan GUI hanya PowerShell.

\n\n

Kuncinya adalah nomor metrik antarmuka. Angka terendah adalah yang pertama dalam urutan. Masalah saya adalah: jika saya terhubung ke VPN di Jaringan kabel maka adaptor mendapatkan DNS dari luar karena mungkin nomor metrik adaptor VPN lebih tinggi dari adaptor Ethernet ...

\n\n

Larutan:

\n\n

Jalankan PowerSell yang ditinggikan (jalankan sebagai administrator)

\n\n

Dapatkan-NetIPInterface

\n\n

Kemudian Anda akan melihat semua antarmuka Anda dengan semua informasi. Periksa nomor metrik dan periksa indeks antarmuka.

\n\n

Set-NetIPInterface -InterfaceIndex (jumlah antarmuka yang dipilih) -InterfaceMetric (jumlah metrik)

\n\n

Ini adalah pengaturan saya:

\n\n

Set-NetIPInterface -InterfaceIndex 6 -InterfaceMetric 60

\n\n

Setel wifi ke indeks terendah. (Saya dapat memverifikasi ini berfungsi.) Tidak perlu GPO.

Ini terjadi pada saya dan sangat mengganggu saya untuk waktu yang lama.
Laptop saya adalah HP Pro Book 450, dan menjalankan Windows 10 build 1903, ini adalah laptop pribadi saya, tetapi saya menggunakannya di jaringan tempat kerja saya.

\n\n

Saya memecahkan masalah dengan cara ini:

\n\n
    \n
  1. Buka kotak pencarian di bilah start Windows dan ketik command prompt, klik kanan pada hasilnya dan klik "run as administrator".
    2. \n
  2. Ketik netcfg -d dan tekan enter, ini akan mengatur ulang semua konfigurasi jaringan pada mesin Anda (termasuk driver untuk adaptor Ethernet dan WiFi).
    3. \n
  3. Buka Panel Kontrol ---> Pengelola Perangkat ---> klik "Pindai perubahan perangkat keras", ini akan menyegarkan pengaturan dan konfigurasi semua adaptor jaringan Anda (Ethernet, dan WiFi).
    4. \n
\n\n

Jika solusi ini gagal, maka masalahnya bukan pada salinan Windows Anda, melainkan pada BIOS (pengalihan LAN / WAN), atau server perusahaan.

\n\n
