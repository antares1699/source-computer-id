7z x d:\\migration\\mongo\\mongodb.7z o:f:\\data *.* -r\n

Saya menggunakan perintah ini sebagai bagian dari batch untuk mengekstrak konten file 7z dari satu drive (D) ke drive lain (F). Struktur folder itu penting jadi saya menggunakan perintah x dengan rekursi.

\n\n

Yang seharusnya terjadi adalah isi arsip didekompresi menjadi f:\\data.

\n\n

Apa sebenarnya terjadi adalah konten yang sedang didekompresi ke direktori kerja direktori file batch (f:\\migration\\). Menentukan direktori kerja (-w:) dalam perintah tidak berpengaruh.

\n\n

Bagaimana saya dapat membuat perintah saya berfungsi sebagaimana mestinya?

\n\n

Saya menggunakan 7zip x64 9.22b di Windows Server 2012 R2.

\n\n

Anda melihat bagaimana jelas disebutkan bahwa sakelar adalah -o dan -w? Seperti dalam, ada a tanda penghubung mendahului sakelar tetapi bukan perintahnya. Juga usus besar aku s tidak bagian dari sakelar itu sendiri. Jika ya, maka Anda seharusnya menggunakan x: dengan cara yang sama, bukan hanya x untuk mengekstraksi dengan jalur. Gunakan sesuatu seperti ini untuk mengompresi folder secara rekursif dan menyimpan jalur relatif: 

7za a -r Archive.7z C:\\InputFolder\n

Gunakan perintah berikut untuk mengekstrak ke direktori tertentu: 

7za x -oD:\\OutputFolder Archive.7z\n

Tentunya jika nama folder Anda memiliki spasi di dalamnya gunakan tanda kutip ganda.
