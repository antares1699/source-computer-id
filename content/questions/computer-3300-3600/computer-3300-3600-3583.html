Ada beberapa area untuk meningkatkan koneksi SSL (dan latensi lalu lintas SSL umum). Beberapa di antaranya dapat memengaruhi keamanan dalam beberapa cara kecil atau hebat.

\n\n

(Ini untuk ssl.conf menggunakan Apache)

\n\n

Nonaktifkan pencarian Domain, dan izinkan Non-SNI menjangkau domain primer

\n\n
    \n
  • SSLStrictSNIVHostCheck mati
    • \n
\n\n

Alih-alih host bernama, gunakan alamat IP untuk entri: 443 VHOST

\n\n

Cache Sesi SSL

\n\n
    \n
  • SSLSessionCache shmcb: / run / httpd / sslcache (512000)
    • \n
  • SSLSessionCacheTimeout 300
    • \n
\n\n

Diizinkan untuk pesanan Protokol SSL lebih cepat , kompatibilitas umum, keamanan sedang:

\n\n
    \n
  • SSLProtokol semua -SSLv3
    • \n
  • SSLCipherSuite ALL: + HIGH: + TLSv1:! ADH:! EXP:! SSLv2:! MEDIUM:! LOW:! NULL:! ANULL
    • \n
  • SSLHonorCipherOrder aktif
    • \n
\n\n

OCSP Stapling (mempercepat SSL)

\n\n
    \n
  • SSLUseStapling aktif
    • \n
  • SSLStaplingResponderTimeout 5
    • \n
  • SSLStaplingReturnResponderErrors mati
    • \n
  • SSLStaplingCache shmcb: / run / ocsp (128000)
    • \n
\n\n

HSTS (dapat melakukan preload sehingga permintaan http: // menjadi https: // permintaan di browser

\n\n
    \n
  • Header selalu menyetel Strict-Transport-Security "max-age = 63072000; includeSubdomains; preload"
    • \n
\n\n

Kirimkan untuk pramuat di sini: https://hstspreload.appspot.com/

\n\n

Pertimbangkan penerapan lebih lanjut berikut ini:

\n\n
    \n
  • HTTP / 2 dengan HPACK
    • \n
  • Kompresi Brotli
    • \n
\n\n

Masalah non-SSL lainnya yang dapat membantu mempercepat koneksi

\n\n

Direktif lain untuk Apache di httpd.conf

\n\n
    \n
  • HostnameLookups Off
    • \n
