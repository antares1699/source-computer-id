Lama membaca manual untuk man, less, groff dan grotty akhirnya memberi saya jawaban

\n\n

Penyorotan secara default dibuat menggunakan urutan spasi mundur: c\\bc => tebal c, _\\bc => garis bawah c. Tetapi jika keluaran seperti menggunakan cat sebagai pager hanya mengeluarkan c biasa dalam kedua kasus. Juga baris kosong diperas, jadi untuk melakukan semua ini, pager harus disetel ke ul | cat -s.

\n\n

Pager dapat diatur dengan banyak cara:

\n\n
    \n

  1. menggunakan variabel MANPAGER atau PAGER (MANPAGER lebih baik karena PAGER tidak hanya mempengaruhi perintah man) 

    export MANPAGER='ul | cat -s'\n
    2. \n

  2. di man.conf 

    PAGER       ul | cat -s\n
    3. \n

  3. menggunakan parameter -P 

    cat -P 'ul | cat -s' …\n

    atau 

    alias man='man -P \"ul | cat -s\"'\n
    4. \n

Ini bukan yang Anda inginkan (Anda tidak akan mendapatkan output di konsol) tetapi Anda dapat membuat file dvi dengan konten manual seperti yang dijelaskan di man man:

\n\n
man -l -Tdvi ./foo.1x.gz > ./foo.1x.dvi\n

Perintah ini akan mendekompresi dan memformat manual sumber nroff\n halaman ./foo.1x.gz ke dalam file (dvi) yang tidak bergantung perangkat. Redi‐\n rection diperlukan karena flag -T menyebabkan keluaran diarahkan ke\n stdout tanpa pager. Outputnya bisa dilihat dengan program\n seperti xdvi atau diproses lebih lanjut menjadi PostScript menggunakan program\n seperti dvips.

\n
\n\n

Saya baru saja menguji ini dan membuka file dvi dengan evince: sorotan tidak hilang.
