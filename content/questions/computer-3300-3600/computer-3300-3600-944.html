Mengapa tidak memperkenalkan sumber baterai ke RAM untuk mempertahankan isinya di semua boot?

\n
\n\n

Sebenarnya, komputer Anda sudah menggunakan memori bertenaga baterai. Pernahkah Anda mengatur ulang pengaturan BIOS Anda ketika Anda tidak dapat mem-boot komputer? Ada dua cara untuk melakukannya: dengan jumper atau dengan melepas baterai CMOS. Baterai itu digunakan untuk memberi daya pada sejumlah kecil memori yang digunakan untuk menyimpan pengaturan BIOS. Ini bukan RAM, tetapi mendekati apa yang Anda minta - seperti yang Anda lihat, kami sudah menggunakan memori bertenaga.

\n\n

Sampai sekarang kami tidak menggunakan RAM bertenaga karena kami memiliki SSD. SSD tingkat konsumen memungkinkan Anda mem-boot OS Anda dalam hitungan detik dan harganya jauh lebih murah daripada RAM. Itu saja saat membaca file dari disk.

\n\n

Tetapi SSD adalah pengubah permainan total. Jadi mengapa kita tidak menggunakan RAM bertenaga sebelum Era SSD ? Karena 20 GB (ukuran folder Windows saya) ruang HDD berharga $ 2,2 dan 20 GB DDR3 berharga $ 220 (penawaran terbaik dari Newegg). Jika Anda mampu membelinya, ada beberapa pilihan tersedia .

\n\n

Saya hanya berbicara tentang menyimpan file dalam RAM karena dengan desain perangkat keras dan perangkat lunak saat ini, menjaga status saat reboot tidak mungkin. Keltari punya sudah diposting bahwa misalnya mengganti kartu grafis dapat merusak perangkat keras atau perangkat lunak. Hal yang sama berlaku untuk melakukan modifikasi disk.

\n\n

Misalnya file yang sama di RAM dan di hard disk bisa kehilangan sinkronisasi di beberapa titik jika Anda mengeluarkan drive (atau RAM) dan mengacaukan beberapa file). Memeriksa apakah file RAM cocok dengan file disk sama lambatnya dengan membaca dari disk, jadi itu akan menggagalkan tujuannya. Anda harus ingat untuk mengatur ulang RAM (seperti baterai CMOS) atau membuat sistem perangkat keras yang akan melakukannya, tetapi mereka juga membutuhkan sumber daya. Membangun sistem seperti itu sulit, jika bukan tidak mungkin.

\n\n

Dan akhirnya, negara dimaksudkan untuk menjadi tidak stabil - sebagai kebalikan dari file , yang harus tetap ada. Begitulah cara komputer dirancang. Untuk mengubahnya, Anda harus memikirkan kembali semua yang kami miliki sejak RAM volatile pertama kali digunakan di komputer.
