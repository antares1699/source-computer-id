Ada kesalahan ketik yang menyebabkan pesan kesalahan saat masuk (seseorang mengetik lias alih-alih alias). Saya telah memeriksa /etc/bashrc serta ~/.bashrc dan /etc/profile serta ~/.bash_profile untuk pengguna individu, tetapi tidak satu pun dari file ini berisi alias khusus untuk sistem. File alternatif apa yang dapat berisi alias di seluruh sistem?

","source":"https://superuser.com/questions/591265/where-are-aliases-set-in-centos","author":"Shouvik Sayef","replies":[],"answers":[{"answer":"

Senang Anda menemukannya, tetapi berikut adalah cara untuk melihat semua gambar file login secara umum: 

# run strace, see every syscall\nstrace -o /tmp/bash.out bash --login\n

(Keluar dari bash shell) 

# filter out opens that returned a descriptor, then use sed to get the file\n< /tmp/bash.out grep -o 'open(\"[^,]*,[^)]*)[ \\t]=[ \\t][0-9]' | sed -e 's/^[^\"]*\"//' -e  's/\".*$//' | sort -u > /tmp/openedfiles.txt\n\n# grep for the broken alias, or whatever\n< /tmp/openedfiles.txt xargs grep '^[ \\t]*lias'\n

strace adalah salah satu perintah yang bisa menjadi sihir jika Anda tahu cara menggunakannya.

Dan untuk kerumunan cepat dan kotor, saya akan ...

\n\n

grep ^lias ~/.* ; grep ^lias /etc/* /etc/*/*

\n\n

dan jika itu tidak menunjukkan saya ke arah yang benar, saya akan menggunakan ...

\n\n

grep alias ~/.* > /tmp/alias.txt ; grep alias /etc/* /etc/*/* >> /tmp/alias.txt ; less /tmp/alias.txt dan mengoreksi semua baris alias.
