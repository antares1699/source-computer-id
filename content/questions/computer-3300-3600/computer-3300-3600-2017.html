Bagaimana cara menemukan file yang didefinisikan variabel bash di sana?

\n\n

Asumsikan perintah echo ditampilkan 

$ echo $DB_ROOT\n/home/mahmood/db-4.0\n

Sekarang, bagaimana saya dapat menemukan file yang variabel ini telah didefinisikan di sana? Saya harus mengatakan bahwa saya belum mendefinisikannya pada tahun ~/.bashrc.

Ada banyak tempat di mana itu bisa didefinisikan.

\n\n

Untuk penjelasan lengkap, Anda perlu melihat halaman manual bash, di bawah INVOCATION, tetapi saya harus memperingatkan Anda bahwa itu mungkin akan membuat Anda pusing sebelum Anda mengetahui semuanya :-)

\n\n

Bergantung pada bagaimana shell dipanggil dan file apa yang ada, itu mungkin satu atau lebih dari:

\n\n
    \n
  • /etc/profile.
    • \n
  • /etc/bash.bashrc
    • \n
  • ~/.bash_profile.
    • \n
  • ~/.bash_login.
    • \n
  • ~/.profile.
    • \n
  • ~/.bashrc.
    • \n
  • apa pun yang ditentukan dengan opsi --rcfile.
    • \n
  • apa pun yang ditentukan dengan variabel lingkungan BASH_ENV.
    • \n
  • apa pun yang ditentukan dengan variabel lingkungan ENV.
    • \n
  • file apa pun yang dipanggil dari file sebelumnya.
    • \n
\n\n
\n\n

Untuk apa nilainya, saya biasanya memastikan saya memiliki ~/.bash_profile (karena itu adalah prioritas tertinggi dari set {~/.bash_profile, ~/.bash_login, ~/.profile} - itu hanya akan memproses pertama yang ditemukan di grup itu).

\n\n

Saya juga memastikan bahwa saya memiliki ~/.bashrc dan ~/.bash_profile bersumber di bagian akhir.

\n\n

Dengan begitu, ~/.bash_profile hanya menyimpan barang-barang yang saya butuhkan di shell login dan ~/.bashrc menyimpan barang-barang yang saya inginkan. setiap kulit. Anda hanya perlu memastikan bahwa yang terakhir tidak memiliki perintah "kumulatif" (seperti menambahkan sesuatu ke akhir jalur).

\n\n

Ada cara lain untuk mengaturnya, terutama jika Anda menggunakan shell yang berbeda dan harus berbagi profil di antara mereka, tetapi skema di atas adalah yang termudah yang saya temukan dalam pengaturan bash saja.
