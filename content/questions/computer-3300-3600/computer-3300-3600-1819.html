Lihat proyeknya Bashhub . Ini secara efektif persis seperti yang Anda minta - dropbox untuk riwayat bash - tetapi mereka telah menambahkan alat untuk merujuk riwayat dari semua mesin Anda, dengan asumsi mereka memiliki akses ke internet. Ada kekhawatiran yang valid tentang mengirim interaksi shell Anda ke pihak ketiga (mis. Pemberi kerja, privasi, mengerjakan materi sensitif), tetapi jika itu bukan faktor signifikan, ini sepertinya solusi yang bagus. Tampaknya mereka tidak memiliki sumber terbuka komponen server dari kode, sayangnya, yang akan bagus untuk mengurangi masalah yang disebutkan sebelumnya.

\n\n

Ada banyak teknik yang kurang canggih yang dapat Anda gunakan, terutama jika setiap mesin memiliki akses ssh satu sama lain, atau setidaknya masing-masing dapat berbicara ke satu server pusat melalui ssh. Solusi apa pun yang melibatkan beberapa mesin yang dapat digunakan secara bersamaan memperbarui file riwayat umum (baik di satu lokasi tertaut atau dalam pembaruan terdistribusi) akan merusak kemampuan perluasan riwayat bash - mis. mengingat perintah terakhir dengan !! bisa membuat Anda salah perintah. Ini hanyalah beberapa gagasan yang perlu disempurnakan:

\n\n
    \n
  • Konfigurasikan berbagi NFS dari server, pasang di setiap mesin, tautkan file .bash_history Anda darinya
    • \n

  • Gunakan kemampuan PROMPT_COMMAND bash untuk menambahkan file jarak jauh dengan perintah terakhir Anda (misalnya di .bashrc PROMPT_COMMAND=\"append_history\" lalu tambahkan fungsi append_history () { ( ssh you@yourserver \"echo !! >> ~/common_bash_history\" ) & }. Saya meletakkan perintah ssh di subkulit dan melatarbelakanginya untuk menghapus penundaan setelah memasukkan perintah. Anda kemudian harus untuk secara berkala mendorong perubahan dari file common_bash_history ke setiap mesin (Hanya menambahkan entri unik).

    • \n

  • Tugas cron periodik yang berjalan di salah satu mesin ('server') yang menarik histori dari masing-masing mesin, membuat master dan mendorongnya keluar.

    • \n
\n\n

Dalam kasus saya, Bashhub tidak akan berfungsi, karena alasan yang disebutkan di atas. Saya telah mencari solusi 'sempurna' untuk ini dan belum menemukannya - apa yang telah saya tetapkan untuk saat ini adalah menggabungkan semua riwayat saya dari setiap mesin ke dalam file riwayat utama di satu server. Tapi saya tidak mendorong hasil agregat kembali ke file sejarah masing-masing mesin; Saya menggunakan file terpisah untuk ini, yang memungkinkan saya menambahkan hal-hal seperti mesin mana asalnya, dan saya menyiapkan fungsi pencarian dan penyelesaian baru untuk file agregat (yang saya gunakan untuk riwayat 'jangka panjang') dan meninggalkan riwayat bashs ekspansi / penyelesaian utuh (yang saya gunakan untuk riwayat 'jangka pendek'). Jika bashhub membuat perangkat lunak server tersedia serta perangkat lunak klien, itu akan menjadi alternatif solusi yang sangat halus (peretasan) yang telah saya lakukan bersama.
