\n\n

Pertanyaan: Jika saya menjalankan perintah rsync dengan -v flag, itu akan mencetak banyak pesan di konsol. Apakah ada cara. Jadi, saya dapat mengarahkan log ini ke file?

Anda dapat memanggil skrip Anda dan mengarahkan "output standar" (AKA STDOUT) ke file log. Gunakan '>>' untuk menambahkan file log atau '>' untuk menimpa file log. File akan dibuat (jika izin pada direktorinya memungkinkan) jika belum ada. Jika file sudah ada, pastikan itu dapat ditulisi oleh Anda (atau siapa pun yang menjalankan skrip Anda).

\n\n

Jika Anda selalu menambahkan file log daripada menulis di atasnya, sebaiknya pastikan sejak awal bahwa sesuatu akan "memutar" file log Anda, yaitu memindahkannya ke nama lain dan menghapus file log yang lebih lama. Jika file log unik untuk skrip Anda, skrip Anda dapat mengatur rotasi file log.

\n\n

Biasanya, cara yang disukai adalah memanggil skrip Anda dengan pengalihan STDOUT, tetapi ada kalanya Anda ingin melakukan pengalihan dalam skrip itu sendiri untuk satu atau beberapa perintah. Anda dapat menambahkan ">> logfile" setelah pernyataan apa pun dalam skrip shell untuk menambahkan STDOUT dari perintah tersebut ke logfile. Anda dapat mengubahnya menjadi "2> & 1 >> logfile" untuk menambahkan "kesalahan standar" (AKA STDERR) dan STDOUT ke logfile. Anda biasanya tidak ingin STDERR juga masuk ke file log karena STDERR biasanya digunakan untuk memberi tahu siapa pun / siapa pun yang menjalankan skrip suatu masalah. Tetapi ada saat dan situasi di mana Anda memang menginginkannya. Jika mau, Anda bisa mengelompokkan perintah dalam skrip Anda di dalam {and} dan mengarahkan STDOUT seluruh grup sekaligus dengan meletakkan pengalihan setelah kurung kurawal tutup.
