Dari Ubuntu, pertama Anda harus menjalankan ntfsfix pada partisi ntfs Anda kemudian gunakan opsi remove_hiberfile untuk memasang partisi Anda:

\n\n

man ntfsfix:

\n\n
\n

DESKRIPSI

\n \n

ntfsfix adalah utilitas yang memperbaiki beberapa masalah NTFS yang umum. ntfsfix BUKANLAH versi Linux dari chkdsk. Ini hanya memperbaiki beberapa ketidakkonsistenan NTFS mendasar, mengatur ulang file jurnal NTFS dan menjadwalkan pemeriksaan konsistensi NTFS untuk boot pertama ke Windows.

\n \n

Anda dapat menjalankan ntfsfix pada volume NTFS jika menurut Anda itu rusak oleh Windows atau cara lain dan tidak dapat dipasang.

\n
\n\n

pria ntfs-3g:

\n\n
\n

remove_hiberfile

\n \n

Ketika volume NTFS di-hibernasi, pemasangan baca-tulis ditolak dan pemasangan hanya-baca dipaksa. Seseorang perlu melanjutkan Windows dan mematikannya dengan benar, atau menggunakan opsi ini yang akan menghapus Windows\n file hibernasi. Harap dicatat, ini berarti sesi Windows disimpan\n akan benar-benar hilang. Gunakan opsi ini di bawah tanggung jawab Anda sendiri.

\n
\n\n

misalnya: 

sudo ntfsfix /dev/sdaX\nsudo mount -t ntfs-3g -o remove_hiberfile /dev/sdaX /mnt/your_mount_point\n

Jika Anda memiliki media Pengaturan Windows (versi yang cukup baru akan dilakukan), Anda dapat chkdsk volume dari sana. Gunakan saja Bergeser F10 untuk menampilkan Command Prompt.

\n\n

Dengan begitu, Windows Anda tidak perlu melakukan booting. Anda mungkin juga memulihkan cadangan registri dari sana saat Anda melakukannya.

\n\n

Atau, jika Anda setidaknya dapat mencapai layar masuk, Anda dapat melakukan boot ulang dari tombol daya di sana. Seperti yang disebutkan @ Run5k dalam komentarnya, mematikan tidak dimatikan secara default. Jadi pilih untuk reboot.

\n\n

Untuk menutup "benar-benar", tahan Bergeser saat memilih untuk mematikan.

\n\n

Anda mungkin juga ingin mencoba me-reboot terlebih dahulu sebelum mencoba perbaikan apa pun.
