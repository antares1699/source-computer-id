Saya memiliki Modem / Router ADSL dari ISP, dan PC saya terhubung dengannya. Cukup satu PC yang terhubung ke Router.

\n\n

Router memiliki [PPPoE] mode dan a [Jembatan] mode.

\n\n

Mode mana yang memberi saya internet cepat dan latensi rendah ?

","source":"https://superuser.com/questions/1347005/bridge-vs-pppoe-mode","author":"amy programmer","replies":[],"answers":[{"answer":"

Dengan casing Anda, mode Bridge akan selalu lebih cepat.

\n\n

Mode PPPoE mengharuskan Modem / Router ADSL Anda melakukan tiga hal terpisah ke Frame Ethernet yang keluar dari modem ADSL.

\n\n
    \n

  • Enkapsulasi (PPPoE)

    • \n

  • Perutean IP

    • \n

  • TCP / UDP SNAT

    • \n
\n\n

Dua yang terakhir sama-sama mahal komputasi dan sama sekali tidak diperlukan untuk satu komputer untuk mengakses internet.

\n\n

Dalam mode Bridge frame PPPoE mentah lepas dari modem ADSL, penggunaan CPU nol.

\n\n

PC Anda dapat menangani PPPoE secara langsung dan bergabung dengan jaringan ISP (segmen / subnet) secara langsung. Dengan PC klien Anda langsung di jaringan ISP, tidak perlu perutean atau NATing.

\n\n

Jika Anda tidak percaya, coba buka koneksi 10k TCP saat menggunakan Router dalam mode PPPoE (misalnya dengan menjalankan beberapa torrent). Router akan dengan cepat menghabiskan memorinya pada tabel NAT.

\n\n

Namun. Satu efek samping dari menjalankan NAT adalah Anda memiliki pengaturan firewall implisit. Jika Anda langsung menghubungkan PC Anda ke internet menggunakan mode jembatan, Anda harus memasang firewall yang baik pada PC klien Anda.

Kedua opsi tersebut hampir sama dengan internet cepat dan latensi rendah, perbedaannya adalah pada Mode PPPoE router akan menjadi Router (secara harfiah ...) sementara jika masuk Mode jembatan perangkat (bukan-router) akan mengharapkan perangkat lain menjadi router (atau Anda harus mengkonfigurasi kredensial PPPoE pada PC itu sendiri yang tidak ideal).

\n\n

Jadi jika Anda tidak memiliki router lain di sana, saya akan mengatakan bahwa mode PPPoE jelas merupakan cara yang tepat.

\n\n
