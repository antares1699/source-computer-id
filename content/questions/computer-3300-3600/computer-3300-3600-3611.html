Beberapa metode sederhana:

\n
\n
    \n
  1. ImageMagick - Pada baris perintah yang dijalankan:
    identify -verbose mystery.jpg | grep Interlace
    Outputnya adalah "Interlace: JPEG" atau "Interlace: None."
    2. \n
  2. Photoshop - Membuka file. Pilih File -> Save for Web & Devices.
    Jika itu adalah jpeg progresif, kotak centang Progresif akan dipilih.
    3. \n
  3. Browser apa pun - Baseline jpegs akan dimuat dari atas ke bawah, dan jpeg progresif akan melakukan hal lain. Jika file dimuat terlalu cepat, Anda mungkin perlu menambahkan pembatasan bandwidth. Saya menggunakan ipfw di Mac saya.
    4. \n

Beberapa byte pengenal dapat digunakan untuk mengenali file. JPEG berbasis DCT progresif dapat diidentifikasi dengan byte “0xFF, 0xC2 ″. Selain itu, gambar JPEG progresif biasanya berisi beberapa pemindaian sehingga Anda dapat mencari beberapa kecocokan "Mulai Pemindaian" (byte: "0xFF, 0xDA"). Penulis menggunakan perintah Bash sederhana untuk mendapatkan hasil yang akurat: 

>grep -c -P “xffxc2″ progressive1.jpg\n>1\n\n>grep -c -P “xffxc2″ progressive2.jpg\n>1\n\n>grep -c -P “xffxc2″ baseline1.jpg\n>0\n\n>grep -c -P “xffxc2″ baseline2.jpg\n>0\n\n>grep -c -P “xffxDA” progressive1.jpg\n>6\n\n>grep -c -P “xffxDA” progressive2.jpg\n>12\n

Tautan juga berisi sejumlah alat berguna lainnya.

Plotnya semakin tebal (terima kasih kepada pengguna PussInBoots karena telah mengingatkan saya untuk menggali lebih dalam). Pertanyaannya menanyakan bagaimana cara mengetahui apakah JPEG itu diselamatkan sebagai Progresif.

\n
    \n

  • "Disimpan sebagai" tampaknya memiliki arti yang fleksibel. Saya bereksperimen dan menemukan bahwa dalam beberapa kasus, menyimpan sebagai progresif hanya berarti bahwa parameter meta data (Interlace) diubah, tetapi file tersebut tidak benar-benar diubah untuk memuat pemindaian progresif. Jadi file tersebut berisi satu gambar dan perangkat lunak rendering diperintahkan untuk memperlakukannya seperti gambar progresif.

    \n

    Cara file progresif "palsu" ini terwujud dalam pengujian yang dijelaskan di atas:

    \n
      \n

    • InageMagick: Parameter Interlace dibaca sebagai "JPEG" bukan "None".

      \n
      • \n

    • Photoshop: Satu-satunya versi Photoshop yang saya miliki adalah versi kuno (versi 7.0). Dalam dialog Simpan untuk Web, kotak Progressive telah diperiksa sebelumnya pada file-file ini, tetapi tidak untuk JPEG standar.

      \n
      • \n

    • Browser: Sistem dan koneksi internet saya terlalu cepat untuk menjadi tes yang berguna. Dalam kondisi normal, semuanya cukup dekat untuk seketika sehingga tidak dapat membedakan perbedaan apa pun dengan andal.

      \n
      • \n

    • Mengidentifikasi Bytes: Ini tampaknya menjadi indikator yang lebih andal. JPEG progresif "palsu" tidak memiliki salah satu pola byte pengidentifikasi.

      \n
      • \n
    \n
    • \n

  • Jadi untuk pertanyaan awal, jika ada tes di atas, menunjukkan bahwa file tersebut adalah tidak JPEG progresif, Anda dapat mengandalkan fakta bahwa sebenarnya tidak. Namun, "metode sederhana" di atas, tampaknya tidak dapat diandalkan untuk membedakan JPEG progresif yang sebenarnya dari yang "palsu", jadi Anda tidak dapat mengandalkannya untuk mengetahui apakah JPEG aku s sangat progresif. Faktanya, PussInBoots melaporkan dalam komentar bahwa Photoshop CS6 tampaknya memiliki pilihan default sebagai "progresif", jadi pengaturan awalnya tidak memberi tahu Anda tentang file yang ada.
