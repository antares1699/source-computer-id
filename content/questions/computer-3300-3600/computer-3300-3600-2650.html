Di mana Cygwin menyetel lokal ($ LANG env var)?

\n\n
\n

Saat memulai aplikasi, lokal aplikasi disetel ke lokal default "C" atau "POSIX". Di bawah Cygwin 1.7.2 dan yang lebih baru, lokal ini secara default ke set karakter ASCII pada tingkat aplikasi.

\n
\n\n

...

\n\n

Cara mengatur lokal

\n\n
\n

Asumsikan bahwa Anda telah menetapkan salah satu variabel lingkungan yang disebutkan di atas ke beberapa nilai lokal POSIX yang valid, selain "C" dan "POSIX". Asumsikan lebih jauh bahwa Anda tinggal di Jepang. Anda mungkin ingin menggunakan kode bahasa "ja" dan teritori "JP", sehingga menyetel, katakanlah, LANG ke "ja_JP".

\n \n

Anda tidak menetapkan set karakter, jadi apa yang akan digunakan Cygwin sekarang? Dimulai dengan Cygwin 1.7.2, kumpulan karakter default ditentukan oleh halaman kode ANSI Windows default untuk bahasa dan wilayah ini. Cygwin menggunakan set karakter yang merupakan kesamaan Unix dengan codepage Windows ANSI.

\n \n

Anda tidak ingin menggunakan kumpulan karakter default? Dalam hal ini Anda harus menentukan rangkaian karakter secara eksplisit. Misalnya, anggap Anda berasal dari Jepang dan tidak ingin menggunakan charset default Jepang EUC-JP, tetapi charset default Windows SJIS.

\n \n

Apa yang dapat Anda lakukan, misalnya, adalah menyetel variabel LANG di Terminal Cygwin mintty di bagian "Teks" dari dialog "Opsi".

\n
\n\n

\n\n
\n

Jika Anda memulai sesi Cygwin Anda melalui file batch atau pintasan ke file batch, Anda juga dapat mengatur LANG di sana:

\n
  @echo off\n\n  C:\n  chdir C:\\cygwin\\bin\n  set LANG=ja_JP.SJIS\n  bash --login -i\n
