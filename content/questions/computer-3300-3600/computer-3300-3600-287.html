Saya sangat baru mengenal Ubuntu, berjuang untuk menemukan perintah yang tepat untuk menginstal Zend Server di Server Ubuntu 9.10 saya.

\n\n

Setelah mengunduh Server ZF dan mengekstraknya, saya mencoba menjalankan perintah ini: 

install_zs.sh 5.3\n

Ini adalah ls dari arsip ZF yang diekstrak: 

install_zs.sh  README  upgrade_zs_php.sh  zend.deb.repo  zend.rpm.repo\n

Tapi tertulis command not found. Ada ide?

","source":"https://superuser.com/questions/134523/how-to-solve-command-not-found-on-ubuntu-bash-shell","author":"redcoder","replies":[{"reply":"Protip yang bagus di sini adalah dengan menekan tab untuk melengkapi nama file secara otomatis di terminal direktori Anda saat ini.","author":"Jonno_FTW"},{"reply":"Pada dasarnya, ketika Anda mencoba menjalankan perintah, shell mencoba menambahkan jalur yang diketahui ke sana, sehingga Anda dapat menjalankan perintah seperti pwd dari direktori mana pun tanpa harus menentukan yang mana. Shell akan menjalankan program pertama yang ditemukannya yang cocok dengan nama Anda. Jika Anda menjalankan sesuatu yang TIDAK di jalur yang diketahui, Anda harus menentukan jalurnya. . adalah direktori saat ini, jadi ./file secara manual menambahkan direktori saat ini ke panggilan ke file sehingga shell tidak perlu mencoba dan mencari file mana yang Anda inginkan.","author":"Ritwik Bose"}],"answers":[{"answer":"

Saat menjalankan eksekusi atau skrip yang tidak ada dalam variabel lingkungan "PATH" sistem Anda, Anda harus memberikan jalur lengkap ke sana. "." adalah alias untuk jalur lengkap direktori kerja Anda saat ini, jadi ./install_zs.sh 5.3 seharusnya berfungsi.

\n\n

aku suka untuk dengan kuat menyarankan Anda memberi ini , atau beberapa dokumentasi shell * nix serupa yang dapat dibaca, karena bermanuver di sekitar shell secara membabi buta tidak pernah, pernah sebuah ide bagus.
