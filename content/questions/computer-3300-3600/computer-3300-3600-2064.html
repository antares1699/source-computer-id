rm tidak selalu bekerja sesuai abjad, tetapi bekerja sesuai urutan yang Anda berikan argumennya (dengan -r menjadi lebih rumit, tetapi itu tidak masalah di sini). Jika Anda menulis persis seperti yang Anda katakan, * akan memperluas ke semua nama file menurut abjad.

\n\n

Begitu Iya , Anda telah menghapus file menurut abjad.

\n\n

Coba jalankan 

echo *\n

di direktori untuk melihat urutan kemunculannya. Ini adalah urutan yang sama dengan rm * akan menghapus file.

Ya, mereka bekerja menurut abjad ..

\n\n

Anda dapat menggunakan perintah strace untuk memeriksa urutan dan bagaimana kernel Anda bekerja secara internal. Di bawah ini adalah perintah strace yang saya gunakan untuk perintah rm yang menerjemahkannya * ke urutan abjad.

\n\n

surendra@linuxnix.com: ~ / scripts / temp / temp $ strace rm -rf *

\n\n

execve ("/ bin / rm", ["rm", "-rf", "a.txt", "b.txt", "c.txt", "d.txt"], [/ * 43 vars * /]) = 0

\n\n

<- Saya telah memotong hasilnya ->

\n\n

Di mana pun * ada, bash shell Anda akan mencoba memberikan semua kombinasi dalam urutan ini. Beberapa perintah lain yang mendukung ini adalah cp, mv, find dll.
