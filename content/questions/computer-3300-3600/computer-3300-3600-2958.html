Menggunakan Cygwin di Win7x32, ketika saya menggunakan sed untuk mengganti beberapa nilai dalam file, git kemudian menganggap seluruh file akan sepenuhnya ditulis ulang (yaitu, 500 penyisipan, 500 penghapusan). Apa yang terjadi di sini? Hanya satu baris dalam file yang benar-benar diubah, dan nilai penggantinya sama panjang dengan aslinya. Saya berharap git hanya mengenali garis-garis yang berubah. Satu-satunya hal yang tampaknya telah berubah sama sekali adalah nilai inode dan waktu yang dibaca dari stat some/file.txt. Adakah cara untuk memberi tahu git agar mengabaikan nilai perubahan apa pun yang relevan selama panggilan sed saya, atau memberi tahu sed untuk tidak mengubah apa pun? nilai-nilai yang menyebabkan git melihat file dengan konten yang sama sekali baru?

Juga, Anda dapat menjalankan cat -e sebelum dan sesudah Anda menelepon sed; jika file dikodekan untuk Windows, setiap baris akan diakhiri dengan ^M.
