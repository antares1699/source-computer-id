Saya memiliki dua file log yang dihasilkan dari data biner yang diterjemahkan. Dekodernya sedikit berbeda, dan saya mencoba mengisolasi perbedaan dalam output. Untuk melakukan ini, saya membedakan dua file log, yang berfungsi cukup baik kecuali stempel waktu berbeda untuk setiap baris. Untuk alasan tertentu, perbedaan stempel waktu tidak relevan, jadi saya ingin diff untuk mengabaikannya.

\n\n

Karena file log mengikuti format tertentu, saya cukup mengecualikan ~ 40 karakter terakhir dari setiap baris untuk mengabaikan stempel waktu. EX:

\n\n

Garis A: 

[T9] | ENTRY NAME                       varA             = 0000012B  varB             = 00000000 | 000015.508.107.113s | file.cpp              :738\n

Garis B: 

[T9] | ENTRY NAME                       varA             = 0000012B  varB             = 00000000 | 000015.508.107.163s | file.cpp              :738\n

Garis-garis ini harus diperlakukan sama dalam kasus saya.

\n\n

Bagaimana cara mengetahui perbedaan untuk hanya menyertakan n karakter pertama dari setiap baris, atau mengecualikan m karakter terakhir dari setiap baris?

","source":"https://superuser.com/questions/1115042/diff-how-do-you-only-compare-the-first-n-characters-in-each-line","author":"TheBat","replies":[],"answers":[{"answer":"

Di bash, Anda dapat menggunakan substitusi proses.

\n\n

Untuk menghapus 40 karakter terakhir, Anda dapat menggunakan 

diff <(sed 's/.\\{40\\}$//' file1) \\\n     <(sed 's/.\\{40\\}$//' file1)\n

Untuk memilih 40 pertama, Anda dapat menggunakan 

cut -c1-40 file\n
