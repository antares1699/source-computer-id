\n\n

Windows 7 menggunakan pendekatan kepemilikan "baru" dalam mengelola hard disk, Dynamic Disk (DD).\nSistem baru ini adalah simulasi RAID (jadi tidak baru sama sekali) ditambah beberapa gadget lain yang sebagian besar pengguna normal tidak akan pernah mengerti penggunaannya.\nSalah satu perbedaan yang terlibat adalah penggunaan skema blok deskripsi partisi yang berbeda dari MBR standar, yang disebut Microsoft GPT.\nDengan kata lain, indeks disk yang diperkenalkan oleh Microsoft secara substansial berbeda dari yang digunakan oleh standar MBR. Konsekuensi dari ini tidak lain adalah alat Microsoft dapat mengenali dan memahami dengan benar di mana partisi itu berada dan apa jenisnya.

\n\n

Karena alat partisi disk Linux bekerja pada sistem MBR standar (seperti kebanyakan jika tidak semua sistem operasi lain), mereka tidak dapat menafsirkan DD Microsoft dan indeks GPT-nya.\nKonsekuensinya adalah disk yang dipartisi DD dikenali sebagai bebas karena tidak ada data deskriptor MBR.\nHanya untuk memberikan kesempatan untuk memulihkan dokumen dan file lain dari partisi DD, beberapa penambahan telah dibuat ke kernel Linux, untuk memungkinkannya menangani zona hard disk tersebut.\nAFAIK kode kompatibilitas lebih lanjut di jalan.
Oleh karena itu, jangan lupa bahwa Linux tidak perlu terus mencoba agar kompatibel dengan nonstandar Microsoft ...

\n\n

Untuk saat ini, untuk menjalankan dualbooting Linux, Anda harus memaksa Windows untuk menggunakan standar MBR dan bukan DD Microsoft sendiri.\nAlat partisi Linux kemudian dapat mengenali partisi MBR Windows saat ini dan kemudian dapat menginstal Linux dengan benar dan kode boot (GRUB ...) untuk dualbooting.\nOleh karena itu, strategi ini sebagian besar valid jika Windows 7 telah diinstal dengan cara ini.

\n\n

Waspadalah terhadap kemungkinan situasi berisiko yang mencurigakan!\nJika Anda memiliki beberapa hard disk pada sistem Anda yang dipartisi dengan Windows 7 DD dalam salah satu mode GPT, saat menginstal Linux pada satu HD yang berbeda dari yang pertama (C: pada Windows), Anda mungkin berhasil melakukan dualbooting.\nMasalahnya adalah bahwa Windows 7, tergantung pada bagaimana Anda mengkonfigurasi hard disk Anda, dapat mendeteksi kegagalan volume jika volume seperti itu direntangkan pada HD fisik yang sekarang digunakan oleh Linux.\nAnda mungkin kehilangan banyak data Windows dengan cara ini.\nFaktanya adalah seluruh volume, jadi tidak hanya bagian yang secara fisik terkandung oleh hard disk tersebut yang dapat hilang dengan semua datanya.

\n\n

Jadi satu-satunya aturan yang harus diikuti, untuk saat ini, adalah perencanaan yang tepat dari Windows 7 dari konfigurasi partisi MBR, menghindari cara ini menggunakan multi-hard-disk spanned volume, menyisakan ruang untuk instalasi Linux dan kemudian melanjutkan dengan Linux Install.

\n\n

Masalah besarnya adalah pengguna Windows harus merasa nyaman (!) Dengan sistem Dynamic Disc sebelum dapat mengubah pengaturan partisi dan membuat ruang disk yang diperlukan untuk instalasi Linux.

\n\n

Kesimpulannya, pengenalan sistem Dynamic-Disk oleh Microsoft, sebenarnya bukan masalah teknis untuk koeksistensi Linux-Windows. Saat ini komunitas Linux sedang mengerjakan kebaruan Microsoft untuk menyelamatkan pengguna Windows dari mengacaukan sistem Windows mereka saat mencoba menginstal Linux.\nAgar strategi tambahan ini berfungsi sepenuhnya, diperlukan alat pemartisi Linux untuk mengetahui skema pemartisian non-standar.\nIni mungkin diperkenalkan segera karena ini adalah modifikasi sederhana bahkan jika struktur partisi Windows yang berbeda mungkin memerlukan pendekatan yang berbeda untuk mencapai penyiapan yang benar untuk penginstalan bersama sistem operasi baru.
