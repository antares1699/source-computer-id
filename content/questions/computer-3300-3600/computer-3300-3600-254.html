Saya berharap bahwa Anda perlu mengatur bit "yang dapat dieksekusi" pada file yang di-output oleh kompiler Linux C ++, saya rasa ini tidak dilakukan secara default karena alasan keamanan.

Cukup "cd" ke direktori tempat file executable yang Anda buat berada di dalamnya 

chmod +x myexecutable\n

Mengganti "myexecutable" dengan nama keluaran kompilator Anda.

\n\n

Kemudian coba jalankan dengan ./myexecutable

\n\n

Saya yakin ini juga harus membuat program "dapat dijalankan" menggunakan klik mouse dan juga dengan konsol.

\n\n

Demikian pula jika Anda perlu menjalankan "skrip" dari perintah, Anda dapat mencoba yang berikut ini dalam file skrip, simpan ini sebagai "myexe.sh" atau yang serupa: 

#!/bin/sh\ncommand1\ncommand2\n

Kemudian gunakan lagi 

chmod +x myexe.sh \n

untuk membuatnya sehingga Anda dapat menjalankan skrip dengan mengkliknya.

Saya pikir akar masalah Anda adalah Anda tidak benar-benar memahami perbedaan antara Terminal Emulator dan Shell dan bagaimana Linux mengetahui cara memulai proses.

\n\n

Pertama, ada "Terminal Emulator", di lingkungan berbasis Gnome ini biasanya Terminal Gnome.

\n\n

Lalu ada "Shell", di Linux ini biasanya disebut bash, meskipun shell lain juga memungkinkan.

\n\n

Sebuah "Shell" berjalan di dalam "Terminal Emulator". Perbedaan ini berasal dari usia terminal fisik, di mana Terminal fisik adalah perangkat keras yang mengambil input, menulis teks dalam warna, dll dan Shell adalah perangkat lunak yang memproses perintah pengguna dan mengelola proses lain berdasarkan perintah yang diberikan.

\n\n

Saat ini, kami memiliki layar tujuan umum yang dapat menampilkan gambar apa pun, jadi kami tidak lagi menggunakan Terminal fisik, tetapi memiliki "Terminal Emulator", perangkat lunak yang meniru pekerjaan terminal fisik, dan Shell, yang masih merupakan shell ole yang sama seperti sebelumnya (well, shell modern memang memanfaatkan berada di emulator terminal perangkat lunak seperti tidak dibatasi oleh batasan fisik kertas, dll tetapi pembagiannya tetap).

\n\n

"Shell" tidak selalu berjalan di dalam Terminal Emulator; ada juga Graphical Shell, seperti Nautilus (petunjuk petunjuk, Nautilus adalah nama makhluk laut dengan cangkang besar) atau Windows Explorer (jangan disamakan dengan Internet Explorer).

\n\n

Di kedua shell baris perintah, mis. bash, dan shell grafis, mis. Nautilus, sebuah executable ditandai dengan set execute bitnya.

\n\n

Di shell baris perintah, Anda dapat menggunakan ls -l untuk melihat bit izin file, mis. rwxrwxrwx berarti setiap orang dapat membaca / menulis / menjalankan program; rwxr-xr-- berarti pemilik memiliki izin penuh, orang dalam grup file dapat membaca dan mengeksekusi tetapi tidak dapat menulis, dan orang lain hanya dapat membaca file. Di Nautilus, Anda dapat mengklik kanan pada file> Properties> tab Permission. Di halaman properti Izin, Anda dapat memiliki izin file yang mirip dengan di shell baris perintah.

\n\n

File yang memiliki set bit eksekusi diperlakukan sebagai dapat dieksekusi, dan dapat dieksekusi dengan melakukan ./filename (shell baris perintah) atau mengklik ganda (shell grafis).

\n\n

Terakhir, ada beberapa seluk-beluk lain tentang bagaimana shell mengeksekusi file. Di sebagian besar shell Linux, Anda dapat "mengeksekusi" skrip yang ditulis dalam python / perl / php / bash yang bukan file yang dapat dieksekusi terkompilasi. Karena file-file ini tidak dapat dieksekusi secara native, mereka membutuhkan interpreter (misalnya interpreter python) untuk dieksekusi. Tidak seperti di shell Windows (Explorer), yang menemukan interpreter yang akan dipanggil melalui ekstensi file; Kerang Linux menemukan interpreter yang benar dengan melihat baris "hashbang" yang terlihat seperti ini 

#!/usr/bin/python\n

ketika bit eksekusi file diatur dan file memiliki baris hashbang ini, shell akan memanggil interpreter / usr / bin / python dengan file saat ini sebagai argumen.

\n\n

Nautilus juga dapat mengenali jika suatu program adalah aplikasi baris perintah, dan akan menawarkan Anda untuk menjalankan aplikasi di dalam Terminal. Saat Anda mengklik dua kali skrip yang dapat dieksekusi, Nautilus akan menanyakan apakah Anda ingin menjalankannya di Terminal, berjalan tanpa Terminal, atau mengedit file di Editor Teks.

Dengan kata sederhana:

\n\n
    \n
  1. File biner yang dapat dieksekusi yang Anda buat mungkin kekurangan dapat dieksekusi sedikit. Sistem berbasis Unix menggunakan fitur ini untuk alasan keamanan. Setel dengan: chmod +x yourbinary.
    2. \n
  2. Biner sebenarnya dapat dijalankan, tetapi karena linux tidak menjalankan emulator terminal (yang mirip dengan file antarmuka antara Anda dan stdout) untuk menjalankannya secara default, Anda tidak dapat melihat apa pun.
    3. \n
\n\n

Untuk kasus terakhir, Anda dapat:

\n\n
    \n

  1. Buat skrip yang menjalankan executable Anda melalui emulator terminal, seperti: 

    exo-open --launch TerminalEmulator -e 'yourbinary'\n

    Untuk menjalankannya di terminal default Anda (jika Anda memiliki perintah exo), atau secara eksplisit gunakan salah satu.

    2. \n

  2. Membuat desktop file untuk menjalankannya.

    3. \n

  3. Beri mereka ekstensi khusus dan khusus seperti .exec dan konfigurasikan agar berjalan menggunakan perintah pada saran 1. (atau serupa) pada pengaturan ekstensi file Anda (jika os Anda mendukung fitur itu).

    4. \n
