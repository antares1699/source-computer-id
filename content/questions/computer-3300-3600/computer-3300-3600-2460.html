Saya berasumsi wget digunakan sebagai alias dari cmdlet PowerShell Invoke-WebRequest. Maka ini seharusnya berhasil: 

wget -v  http://mysystem/image.jpg -outfile c:\\temp\\image.jpg\n

Sangat menyenangkan bahwa beberapa cmdlet PS memiliki alias yang sama dengan perintah * nix yang menyediakan fungsionalitas yang sama. Namun terkadang hal itu menimbulkan kebingungan seperti di sini. Anda tidak menjalankan wget di sini, itu adalah Invoke-WebRequest yang sangat mirip tetapi tidak sama.

EDIT

\n\n

Menanggapi komentar Anda: Kode di postingan Anda tidak mengunduh file, tetapi menyimpannya dalam memori dalam properti .content dari WebResponseObject yang dikembalikan. Bahkan dimungkinkan untuk menyimpan file dari objek yang dikembalikan. Seperti ini: 

$ret = wget -v  http://mysystem/image.jpg\n$file = [System.IO.FileStream]::new('C:\\temp\\image.jpg', [System.IO.FileMode]::Create)\n$file.write($ret.Content, 0, $ret.Content.Length)\n$file.close()\n

Tergantung dari apa yang diunduh, tipe $ret.Content dapat bervariasi. Kode di atas berfungsi untuk file biner. Secara keseluruhan, metode pertama menggunakan -Output lebih mudah dan menyelesaikan masalah format biner / teks untuk Anda.
