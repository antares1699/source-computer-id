Apakah saya benar memahami izin ke direktori di Linux / Unix?

\n\n
    \n

  1. Jika direktori Anda hanya memiliki r (baca) izin, Anda diperbolehkan untuk melihat isi direktori (file mana yang terletak di sana) tetapi Anda tidak dapat melakukannya karena Anda tidak dapat masuk (cd) ke direktori ini (karena tidak adanya izin x ). Anda juga tidak dapat melihat konten direktori (file mana yang terletak di sana) dari luar direktori (misalnya pada ls directoryname/*). Anda juga tidak akan dapat membaca (melihat) konten file yang terletak di direktori tersebut menggunakan perintah cat dan more (meskipun Anda memiliki izin untuk membaca file ini). Anda juga tidak akan dapat mengubah (menulis) file (bahkan jika Anda memiliki izin menulis padanya) jika file ini terletak di direktori seperti itu (apa pun yang Anda coba cat >>, echo >>, cp atau beberapa editor teks). Jadi, dari sudut pandang saya, memiliki hanya r izin ke direktori sama dengan tidak memiliki izin ke direktori sama sekali.

    2. \n

  2. Jika direktori Anda hanya memiliki x (mengeksekusi) izin, Anda diizinkan untuk masuk (cd) ke dalam direktori tetapi Anda tidak diizinkan untuk melihat (ls) isi direktori (karena Anda tidak memiliki izin untuk membaca direktori). Jika direktori hanya memiliki izin x dan berisi file yang izinnya Anda miliki r (baca) dan w (tulis), Anda masih tidak dapat membuka file ini dengan (setidaknya beberapa) editor teks (misalnya mcedit). Tetapi Anda akan dapat membaca konteks file menggunakan perintah seperti cp atau more. Anda semua akan dapat mengubah konten file menggunakan echo >> atau cat >>. Jadi, menurut saya, ini adalah x yang memungkinkan pengguna untuk "membaca" dan "menulis" file yang ada di direktori (jika file juga memiliki izin yang sesuai).

    3. \n

  3. Jika direktori memiliki izin r dan x tetapi tidak ada izin w (tulis), Anda tidak dapat mengubah konten direktori (kumpulan file yang terletak di sana). Misalnya Anda tidak dapat membuat file baru di sana atau menghapus yang ada di direktori. Tapi Anda tetap diperbolehkan mengubah konten file yang sudah ada. Jadi, Anda memerlukan izin w untuk membuat atau menghapus file di direktori.

    \n\n

    Ditambahkan:

    4. \n

  4. Menarik juga untuk menyebutkan bahwa izin w ke direktori diperlukan tetapi tidak cukup untuk membuat dan menghapus file di direktori. Jika direktori hanya memiliki w izin Anda tidak akan dapat menambah / menghapus file dari / ke direktori. Untuk dapat melakukannya, Anda harus memiliki izin x ke direktori (selain izin w).

    5. \n

Izin r memberikan akses ke DAFTAR nama file (tetapi bukan metadata)

\n\n

Izin x memberikan akses ke metadata file (inode, ukuran, pemilik, grup, perm, dll.) (Tetapi tidak ada akses ke DAFTAR nama file)

\n\n
\n\n

ls pertama menghitung DAFTAR nama file, dan kemudian mengakses metadata file untuk setiap file. Metadata diakses melalui stat.

\n\n

Berikut ini contoh, bayangkan direktori "contoh" yang berisi file "data".

\n\n

Dengan izin r:

\n\n
    \n

  • cat /example/data akan gagal (tidak ada akses ke file metadata)

    • \n

  • ls -lA example sebagian akan berhasil (akses diberikan ke DAFTAR nama file, tetapi bukan metadata)

    • \n

  • cd example akan gagal (direktori saat ini menyiratkan akses ke metadata yang tidak tersedia)

    • \n
\n\n

Dengan izin x:

\n\n
    \n

  • cat /example/data akan berhasil (akses ke metadata file diberikan) (akses ke LIST tidak diperlukan)

    • \n

  • ls -lA example akan gagal (tidak ada akses ke DAFTAR nama file)

    • \n

  • cd example akan berhasil

    • \n

  • ls akan gagal (tidak ada akses ke DAFTAR nama file)

    • \n

  • ls -l data akan berhasil (akses ke metadata diberikan)

    • \n
