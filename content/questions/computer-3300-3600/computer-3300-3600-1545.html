","source":"https://superuser.com/questions/311418/how-to-set-static-ip-in-laptop-to-be-used-with-wireless-network","author":"Muhammed Rauf K","replies":[],"answers":[{"answer":"

Pertama, saya akan berasumsi bahwa Anda berada di jaringan rumah dan router Anda membagikan alamat IP DHCP (otomatis). Jika tidak, Anda HARUS menghubungi administrator jaringan Anda, atau Anda akan menimbulkan masalah jika Anda memasukkan alamat IP duplikat di jaringan.

\n\n

Kedua, ini sepertinya pertanyaan yang mudah, tetapi ada beberapa hal yang perlu Anda lakukan untuk benar-benar memastikan Anda melakukannya dengan benar. Dalam praktiknya, bahkan jika tidak, di jaringan rumah, Anda mungkin tidak akan mengalami masalah karena kemungkinan IP duplikat rendah, tetapi masih sangat mungkin, jadi lebih mudah melakukannya dengan benar. Saya menyadari hal berikut tampaknya agak panjang, tetapi cukup mudah diikuti, namun, jika Anda memiliki pertanyaan, silakan kirim kembali dan saya akan menjawabnya.

\n\n

Selanjutnya, Anda harus mencoba mencari tahu alamat IP apa yang diberikan oleh server DHCP Anda sehingga Anda memilih IP yang tidak akan diberikannya ke komputer lain nanti. Cara terbaik adalah masuk ke router Anda karena dari sinilah Anda mungkin mendapatkan alamat IP DHCP. Biasanya itu ada di suatu tempat dalam pengaturan LAN-nya, atau di dekat mereka (periksa manual Anda, atau ajukan pertanyaan terpisah di sini dengan model router Anda terdaftar), Anda dapat menemukan kisaran yang dibagikan. Seringkali x.x.x.100-x.x.x.254.

\n\n

Jika itu tidak mudah bagi Anda, atau Anda tidak mengetahui kata sandi router, kami dapat membuat tebakan: Anda dapat membuka kotak perintah, dan melakukan IPCONFIG /ALL. Itu akan memberi Anda alamat IP dan gateway default Anda (biasanya router). Perhatikan juga server DNS Anda, karena Anda juga perlu memasukkannya di bawah.

\n\n

Pilih alamat IP yang bukan alamat router Anda, dan tambahkan 20. Jadi, jika router Anda adalah 192.168.1.1, pilih 192.168.1.20. Subnet mask untuk jaringan rumah hampir selalu 255.255.255.0 (sama dengan IPCONFIG / ALL Anda), dan gateway yang Anda masukkan saat menyetel statis akan sama seperti di IPCONFIG / ALL. Satu-satunya peringatan adalah jika alamat IP Anda ketika Anda melakukan IPCONFIG / ALL sangat rendah, seperti 192.168.1.2 atau di dekatnya (kami hanya benar-benar melihat angka terakhir), maka rentang DHCP yang diberikan rendah, jadi pilih sesuatu seperti 192.168.1.250. IP apa pun yang Anda pilih, buka kotak perintah, dan lakukan PING 192.168.1.20 (atau IP apa pun yang Anda pilih untuk dijadikan statis).

\n\n

Jadi sekarang setelah Anda mengetahui cara mengetahui IP apa yang akan digunakan, berikut adalah cara mengimplementasikannya. Buka Control Panel>Network and Internet>Network and Sharing Center>Change adapter settings dan klik kanan koneksi Jaringan Nirkabel Anda. Klik dua kali Internet Protocol Version 4 (TCP/IPv4), dan masukkan pengaturan Anda seperti pada contoh tangkapan layar berdasarkan apa yang Anda temukan di bagian pertama pertanyaan. Saya menambahkan 8.8.8.8 sebagai server DNS lain, yang dapat Anda gunakan jika IPCONFIG Anda hanya menampilkan router, jika tidak, jika Anda memiliki dua server DNS yang terdaftar, gunakanlah.

Setiap router yang setengah layak yang bertindak sebagai server DHCP (bagian yang mengeluarkan alamat IP) harus memiliki beberapa opsi konfigurasi yang memungkinkan Anda untuk memesan alamat IP tertentu untuk alamat MAC tertentu (MAC = Kode Akses Media, yaitu pengenal heksadesimal unik untuk setiap bagian dari perangkat keras jaringan, misalnya 01-23-45-67-89-AB).

\n\n

Anda dapat menemukan MAC untuk adaptor Nirkabel Anda dengan memeriksa keluaran 'ipconfig / all'.

\n\n

Router keluarga NetGear DG834 yang disediakan oleh ISP yang cukup mendasar memiliki kemampuan ini.

\n\n

Keuntungan dari metode ini adalah adaptor WiFi Anda akan terus berfungsi secara normal dengan jaringan DHCP lain, mis. kedai kopi abadi.

\n\n

Pertimbangan yang sama seperti yang disebutkan KCotreau dalam jawabannya berlaku, yaitu mencoba dan mengalokasikan rentang alamat IP terpisah untuk server DHCP Anda dan IP yang ditetapkan secara statis.
