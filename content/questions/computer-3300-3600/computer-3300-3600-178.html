Deskripsi masalah : Saya tidak dapat terhubung melalui Remote Desktop\nKoneksi untuk mengakses Komputer A dari Komputer B . Lebih khusus lagi, ketika saya menggunakan Komputer B, saya tidak dapat mengakses Komputer A melalui Sambungan Desktop Jarak Jauh. Saya diminta untuk menyediakan\nnama pengguna dan sandi, tetapi saya terus diberitahu oleh jendela "Keamanan Windows"\nyang mengatakan: "Kredensial Anda tidak berfungsi."

\n\n

\n\n

Pengamatan 1 : Saya dapat mengakses Komputer B dari Komputer A . (Jalan lain\nsekitar.)

\n\n

Pengamatan 2 : Satu-satunya perbedaan yang saya tahu, adalah: Saya menggunakan Microsoft\nAkun untuk masuk (secara lokal dan asli) Komputer B , dan menggunakan nama pengguna\nsendiri untuk masuk Komputer A . Nama pengguna di dua komputer (secara lokal\nberbicara) adalah:

\n\n
    \n

  • Komputer A : DESKTOP-XXX / nama pengguna

    • \n

  • Komputer B : namapengguna@hotmail.com

    • \n
\n\n

Kedua komputer menjalankan Windows 10 Pro.

\n\n

Pengamatan 3 : Saya telah memastikan bahwa tidak ada firewall\nantara dua komputer. (Mereka terhubung melalui sakelar Ethernet di file\njaringan rumah, menghubungkan ke dunia luar melalui router rumah.)

\n\n

Pertanyaan: apa yang salah? Apakah saya harus menetapkan dan menggunakan yang sama\nAkun Microsoft aktif Komputer A ? Saya telah memastikan bahwa kata sandinya telah\nperbaiki setiap saat, tetapi masalahnya tetap ada: Saya tidak dapat terhubung dari Komputer B untuk Komputer A , namun saya dapat terhubung dari Komputer A untuk Komputer B .

\n\n

PS: Saya juga menanyakan pertanyaan yang sama di forum TechNet, namun saya tidak menghitung\ndi atasnya cukup banyak. Ref url .

\n\n

informasi tambahan

\n\n

Alamat Komputer A dan B

\n\n

Karena saya hanya memiliki IP dinamis pada kedua mesin, saya menggunakan layanan No-IP. Jadi, untuk kedua komputer tersebut, misalnya:\n* Alamat untuk Komputer A: computer-A.ddns.net\n* Alamat untuk Komputer B: computer-B.ddns.net

\n\n

Eksperimen Tambahan

\n\n

Mematikan Komputer B

\n\n

Ketika saya mematikan Komputer B, menggunakan alamat computer-B.ddns.net untuk terhubung masih memberi saya pesan kesalahan yang sama (seperti pada gambar di atas).

\n\n

\n\n

Om mesin jarak jauh saya, saya memiliki dua akun: Saya memiliki akun administrator lokal dan saya memiliki MicrosoftAccount sebagai pengguna standar.

\n

Untuk menyinkronkan ulang kata sandi: masuk dengan akun administrator lokal, saya membuka prompt perintah dan ketik: 

runas /u:MicrosoftAccount\\[my account] cmd.exe\n

atau 

runas /u:[my account]@outlook.com cmd.exe\n

mengganti [akun saya] dengan nama akun sebenarnya dari Akun Microsoft

\n

Jika Anda baru saja mengubah kata sandi Anda, coba dengan yang lama, jika belum disinkronkan, itu akan berhasil.

\n

Ini akan memaksa mesin untuk menyinkronkan ulang kata sandi sehingga ketika Anda diminta, Anda dapat mengetikkan kata sandi terbaru. Sejak saat itu, RDP akan mengenali kata sandi baru Anda.

\n

Saya juga mencoba perintah di atas saat masuk sebagai pengguna standar. Sepertinya Anda dapat melakukan perintah "runas" untuk pengguna saat ini, jadi Anda mungkin tidak perlu menggunakan akun lain.

\n
