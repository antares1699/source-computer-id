Saya masih dapat memasang partisi ini menggunakan truecrypt setelah memulihkan header volume tetapi tidak dapat mengaksesnya.\nApakah ada cara untuk mendapatkan kembali data saya termasuk struktur folder tanpa disk penyelamat?

Memulihkan tata letak direktori

\n

Saya akan merekomendasikan untuk mencoba TestDisk , alat gratis dan sumber terbuka yang dirancang untuk membantu memulihkan sistem file yang rusak dan sektor boot yang rusak. Pasang drive di TrueCrypt terlebih dahulu, dan lakukan semua pekerjaan Anda pada drive yang didekripsi.

\n

Sepertinya taruhan terbaik Anda adalah mencoba memulihkan MFT partisi, atau Tabel File Master (setara kasar dari superblok sistem file Linux). Cadangan disimpan; namun, saya tidak tahu apakah format cepat juga menghapus cadangan.

\n

Mendapatkan kembali file satu per satu

\n

Jika struktur direktori bersulang, Anda mungkin masih dapat menggunakan alat bantu hapus seperti Recuva , yang merupakan freeware, untuk memulihkan file Anda. Anda tidak akan mendapatkan struktur direktori, tetapi jika Anda memiliki sesuatu yang sangat penting yang Anda perlukan kembali, Anda dapat menyaring apa pun yang ditemukan Recuva.

\n

Mencegah hal ini terjadi

\n

Saya yakin saya tahu bagaimana ini terjadi ... Anda memasang drive dan Windows mengklaim bahwa itu tidak diformat (karena dienkripsi dan terlihat seperti omong kosong untuk Windows), dan Anda tidak sengaja membiarkannya. Terlepas dari apakah ini yang terjadi atau tidak, ini masih sangat mengganggu saya.

\n

Anda perlu mencegah Windows mengaitkan partisi itu dengan huruf drive, yang akan mencegahnya mencoba memformatnya. Dengan disk Anda dicolokkan (tetapi tidak dipasang di TrueCrypt), jalankan editor partisi Windows bawaan (tekan Mulai, ketik partition, dan pilih "Buat dan format partisi hard disk"). Pilih partisi terenkripsi Anda dalam diagram, dan klik kanan partisi tersebut dan pilih "Ubah Huruf dan Jalur Drive". Hapus semua huruf yang terkait dengan partisi, konfirmasi, dan Anda harus mengatur!

Sebagai tambahan, jika Anda ingin mencegah partisi terenkripsi ini dipasang ke huruf drive di komputer windows lain, ada atribut GPT yang dapat Anda atur untuk mencegah hal ini:

\n\n
\n

0x8000000000000000

\n \n

GPT_BASIC_DATA_ATTRIBUTE_NO_DRIVE_LETTER

\n \n

Mencegah partisi memiliki huruf drive secara otomatis\n ditugaskan. Secara default, setiap partisi diberi huruf drive baru\n dan nama jalur GUID volume. Baik huruf kandar dan volume GUID\n pathname dapat digunakan untuk membuka volume menggunakan Win32 API. Mengatur ini\n atribut memastikan bahwa ketika disk dipindahkan ke komputer baru, baru\n huruf drive tidak akan dibuat secara otomatis. Sebaliknya, pengguna\n dapat menetapkan huruf drive secara manual.

\n
\n\n

Jadi di cmd prompt, jalankan DISKPART, pilih disk yang Anda miliki, pilih partisi yang dienkripsi, dan jalankan perintah ini: 

GPT ATTRIBUTES=0x8000000000000000\n

Dengan asumsi, tentu saja, itu adalah disk GPT
