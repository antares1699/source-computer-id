Saya baru saja mendapatkan Macbook Air yang telah dikonfigurasi sebelumnya (dengan 10.8.3) yang menolak untuk terhubung ke WiFi rumah saya dan memberikan pesan "batas waktu koneksi" yang tidak informatif. Router / modem berfungsi, setiap perangkat lain di rumah dapat terhubung menggunakan nama jaringan dan kata sandi yang sama. Saya memverifikasi bahwa saya memasukkan kata sandi yang benar dengan memeriksa halaman admin router.

\n\n

Jika ada yang namanya "terlalu banyak perangkat" yang menyambungkan perute, saya mematikan semua peralatan / gadget jaringan lain di rumah dan mencoba menyambungkan lagi dengan Air. Masih belum berhasil.

\n\n

Saya juga mencoba menghapus com.apple.airport.preferences.plist dari / Library / Preferences / SystemConfiguration dan memulai ulang seperti yang disarankan dalam posting MacRumors. Tidak melakukan apapun.

\n\n

Hal aneh lainnya adalah Macbook Air ini tidak memiliki masalah koneksi ke WiFi terbuka yang dimiliki oleh tetangga saya (ya, orang-orang seperti itu ada). Saya menggunakan WPA2 dan tidak ingin WiFi saya terbuka.

Saya dapat menjawab pertanyaan Anda yang kedua (sekarang, mungkin, sudah lama diperbaiki), tentang diagnostik wifi verbose, jika orang lain memiliki masalah serupa.

\n\n

Ada program OSX yang tidak begitu dikenal bernama /System/Library/PrivateFrameworks/Apple80211.framework/Versions/Current/Resources/airport yang dapat melakukan hal-hal luar biasa. Itu adalah perintah yang panjang untuk diketik setiap kali, jadi saya menambahkan dua baris ke .bash_profile saya. Alias ​​pertama alias saya "wifi" untuk perintah ini dan yang kedua memungkinkan alias untuk bekerja dengan perintah sudo: 

alias wifi='/System/Library/PrivateFrameworks/Apple80211.framework/Versions/A/Resources/airport'\nalias sudo='sudo '\n

Perintah bandara memiliki beberapa opsi. Ketik (menggunakan alias saya) wifi -h untuk mendapatkan laporan singkat atau wifi -H untuk melihat semuanya. Di antara fungsinya adalah opsi untuk terhubung secara terprogram ke router wifi, mengaktifkan diagnostik tingkat lanjut, dan bahkan mengaktifkan sniffing. Berikut ini beberapa yang berguna untuk masalah Anda:

\n\n

Informasi tentang status wifi saat ini: 

wifi -I\n

Pindai wifis yang tersedia dan berikan laporan lengkap: 

wifi -s\n

Aktifkan debug fungsi tingkat pengguna: 

sudo wifi en1 debug +AllUserLand\n

Aktifkan debug fungsi level driver: 

sudo wifi en1 debug +AllDriver\n

Aktifkan fungsi tingkat vendor: 

sudo wifi en1 debug +AllVendor\n

Arahkan output ke /var/log/wifi.log: 

sudo wifi en1 debug +LogFile\n

Mulai konsol dan arahkan ke /var/log/wifi.log (log dicantumkan di kolom sebelah kiri saat Anda mengklik ikon pertama di baris atas: Tampilkan Daftar Log ).
