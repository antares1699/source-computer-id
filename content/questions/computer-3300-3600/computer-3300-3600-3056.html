Jika Anda menginstal program Anda di HD, Anda akan kehilangan keuntungan akses acak dari SSD. Sebagian besar file OS yang umum digunakan (kernel dan DLL) harus berada dalam memori saat Anda memulai aplikasi, jadi akan lebih berguna untuk menginstal program pada SSD juga.

\n\n

Anda dapat menginstal program individu pada HD dengan mengubah direktori instalasi selama proses pengaturan. Jika program sudah diinstal, Anda dapat memindahkan direktorinya ke HD saat program tidak berjalan dan membuat sambungan (NTFS 5 pada Windows 2000 dan yang lebih baru) atau tautan simbolis (NTFS 6 pada Vista dan di atasnya) di tempat aslinya ke arahkan ke lokasi baru. Persimpangan (atau tautan simbolis) seperti jalan pintas, tetapi berfungsi pada tingkat yang lebih rendah. Tautan Ekstensi Shell dapat membantu Anda membuatnya (dan memindahkan file).

\n\n

Anda juga dapat memindahkan seluruh folder Program Files Anda ke partisi lain dan memasang yang terakhir sebagai folder "Program Files". Tetapi ini agak berisiko dan lebih sulit untuk dipertahankan.

Jika Anda memutuskan untuk mengabaikan jalur tautan simbolik, dan hanya ingin memindahkan sebagian besar data Anda ke hard drive, ada cara yang lebih mudah. Untuk direktori berikut di bawah pengguna Anda, Anda dapat mengklik kanan dan mengubah lokasi.

\n\n

Kontak, Desktop, Unduhan, Favorit, Tautan, Dokumen Saya, Musik Saya, Gambar Saya, Video Saya, Permainan dan Pencarian Tersimpan. Ini akan memindahkan sebagian besar data Anda dengan cara yang sangat mudah (Ini juga akan menanyakan apakah Anda ingin memindahkan data Anda saat ini secara otomatis ketika Anda mengubah lokasi).

\n\n

\n

\n\n

Tangkapan layar kedua ini sebenarnya adalah "Dokumen Saya" yang diarahkan ke drive jaringan.

\n\n
