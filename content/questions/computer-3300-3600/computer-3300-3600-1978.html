Kecepatan transfer untuk Sata III adalah 600MBs (mempertimbangkan pengkodean 10b/8b). Jadi SSD mungkin memaksimalkannya (SSD biasanya berkinerja lebih baik dengan kecepatan baca daripada tulis).

\n\n

Adapun melihat kecepatan ini di hard drive: alasannya adalah karena bagian dari file yang Anda transfer masuk ke HD penyangga (biasanya di mana saja dari 16MB hingga 64MB), yang merupakan memori solid-state. Setelah buffer terisi, buffer harus mulai menulis ke disk fisik (saat itulah Anda mulai melihat kecepatan yang lebih lambat).

\n\n

Anda mungkin juga memperhatikan bahwa ini tidak sering terjadi selama operasi baca. Satu-satunya kesempatan Anda dapat membaca file dari HD dengan kecepatan ini adalah jika file tersebut sudah ada dalam buffer HD. Selama menulis operasi, meskipun, ini tidak pernah menjadi masalah.

\n\n

Mengapa Anda mendapatkan nilai transfer di atas 600MBs?

\n\n

Kemungkinan besar karena ada bagian dari file yang sudah disimpan di RAM. Ini bisa jadi karena Anda baru saja membaca file ini menjadi ram (gambar itu mungkin bukan yang pertama kali, kan?).

\n\n

Atau, mungkin OS mulai memuatnya ke ram sebelum Anda memulai operasi, seperti setelah Anda mengklik file tersebut. Saya telah melihat ini, serupa, di Google Chrome - setelah Anda mengetik URL yang telah Anda kunjungi, permintaan HTTP itu dimulai bahkan sebelum Anda menekan enter. Banyak aplikasi yang mencoba melakukannya Tebak apa yang akan Anda lakukan, untuk memulai operasi intensif.

\n\n

Jika tidak, kecepatan transfer tidak dapat melebihi kecepatan bus maksimum.

\n\n

Ada apa dengan slop ke atas pada grafik?

\n\n

Saya tidak bisa mengatakan, dengan keyakinan apa pun, mengapa Anda melihat grafik seperti slop ke atas. Ini bisa menjadi efek prediksi cabang, tetapi sulit untuk mengatakannya. Namun, ini adalah pertanyaan yang bagus untuk ditanyakan.
