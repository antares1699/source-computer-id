Anda dapat menggunakan scp jika Anda memiliki server ssh yang berjalan pada mesin jarak jauh (tetap bagus untuk dimiliki). Telnet tidak bagus, tidak terenkripsi dan mengganggu.

Edit:\nSobat akan kesulitan mengcopy file lewat telnet. Mengapa Anda bersikeras menggunakan terminal?

\n\n

Artinya, Anda menjalankan Windows, jadi mungkin memiliki akses ke lingkungan grafis. Jadi apakah menggunakan file sharing windows / browser web / ftp client / ssh client / etc dikesampingkan karena mereka grafis? Dan jika ya, mengapa?

\n\n

Apakah Anda memiliki akses root pada mesin Ubuntu? Saya menyarankan Anda menginstal server ssh: 

sudo apt-get install openssh-server\n

Kemudian ambil klien Windows SSH / SCP, misalnya. WinSCP, dan gunakan untuk masuk ke server dan menyalin file Anda.

\n\n

Atau jika Anda benar-benar ingin melakukannya melalui terminal karena suatu alasan, ambil Cygwin dan instal terbuka, lalu gunakan perintah 'scp' yang disebutkan di bawah ini.

\n\n

Anda harus lebih spesifik tentang situasi Anda.

\n\n
    \n
  • OS apa yang dijalankan klien?
    • \n
  • OS apa yang dijalankan server?
    • \n
  • layanan apa yang dijalankan server, jika Anda tahu (FTP, HTTP, NFS, SMB, SSH, dll)
    • \n
\n\n

Misalnya, jika Anda memiliki akses SSH ke server, sesuatu seperti ini akan berfungsi: 

scp user@server.mydomain.com:/remote/path/file.txt /local/path/\n

Jika Anda memiliki akses HTTP ke file (http://server.mydomain.com/path/file.txt) maka seperti yang disarankan poster di atas, wget: 

wget http://server.mydomain.com/path/file.txt\n

Jawabannya sangat bergantung pada situasi Anda.

Jika Anda menggunakan Windows untuk ini, dan Anda benar-benar tidak dapat menggunakan HTTP dengan cara apa pun untuk mendapatkan file, Anda dapat menggunakan transfer file sekolah lama melalui protokol ASCII 7-bit. Terakhir kali melihat penggunaan utama melalui internet dial-up, khususnya di dunia BBS, dengan garis samping dalam transfer unix-to-unix ketika FTP tidak tersedia (umumnya melalui rsh, tapi saya menyimpang).

    2. \n
  2. Tar / Zip direktori yang Anda minati.
    3. \n
  3. Dari baris perintah, 
    sz filename.tar.gz
    4. \n
  4. Dari Hyperterm, pilih Transfer -> Terima File.
    5. \n
  5. Tentukan jalur penyimpanan Anda, dan "Zmodem" sebagai protokol penerima.
    6. \n

Jika ssh / scp tidak tersedia, coba netcat.

\n\n

http://nakkaya.com/2009/04/15/using-netcat-for-file-transfers/

\n\n

Netcat pada dasarnya adalah file dump sederhana ke port jaringan. Ini berguna untuk mentransfer ke host barebone ketika ssh tidak tersedia, atau mentransfer file yang sangat besar melalui LAN / internet di mana overhead enkripsi SSH menjadi hambatan.

\n\n

Meskipun demikian, scp masih merupakan cara standar untuk melakukan transfer file dari host jarak jauh. Dan jika Anda mencoba mentransfer seluruh direktori, yang terbaik adalah menggunakan rsync melalui ssh: 

rsync -avz -e ssh qa@ubuntu:~/www/html/js  ~/home/qa/html\n

Scp melintasi direktori saat ditransfer, ini bisa sangat lambat dengan folder besar. Rsync melintasi seluruh direktori dan menerapkan kompresi gzip sebelum transfer dimulai. Jika salinan lokal ada, rsync dengan membandingkan keduanya dan transfer hanya delta: bagian file yang diubah.

\n\n

Dan koreksi tambahan untuk saran scp sebelumnya: scp seluruh direktori membutuhkan opsi -r: 

scp -r qa@ubuntu:~/www/html/js  ~/home/qa/html\n
