Saya tidak bisa mendapatkan milik saya YubiKey Neo U2F bekerja di VMWare Workstation Pro 12.5. Mengujinya terhadap Situs uji Yubico atau melawan Situs uji Akisec keduanya gagal di sistem operasi tamu (menjalankan Debian Linux 9), tetapi keduanya berfungsi di sistem operasi host (menjalankan Windows 10).

\n\n

Pencarian google menunjukkan saya ingin menambahkan yang berikut ini ke file .vmx tamu VMWare saya: 

usb.generic.allowHID = \"TRUE\"\nusb.generic.allowLastHID = \"TRUE\"\n

Namun, tidak ada bedanya. Spoiler: Saya berspekulasi ini karena mesin virtual saya dienkripsi.

Setelah berbicara dengan arsitek solusi di Yubico, kami menentukannya File konfigurasi VMWare Workstation .vmx tidak dapat langsung diedit jika mesin virtual dienkripsi .

\n\n

Agar yubikey berfungsi di dalam mesin virtual, Anda harus mengikuti langkah-langkah berikut:

\n\n
    \n
  1. Dekripsi mesin virtual jika dienkripsi. Edit pengaturan mesin virtual,\npilih tab Options, pilih Access Control, lalu klik Remove\nTombol enkripsi.
    2. \n

  2. Sekarang, edit file .vmx. Ini adalah\nfile konfigurasi untuk mesin virtual Anda. Arahkan ke\ndirektori yang berisi mesin virtual Anda, temukan file .vmx (di sana\nharus persis satu), lalu buka di notepad. Cari garis\nyang mengatakan usb.present = "TRUE" dan masukkan dua baris berikut: 

    usb.generic.allowHID = \"TRUE\"\nusb.generic.allowLastHID = \"TRUE\"\n
    3. \n

  3. Perhatikan bahwa Yubico merekomendasikan juga menambahkan: 

    usb.generic.autoconnect = \"FALSE\"\n

    Mereka juga merekomendasikan untuk menghapus setiap contoh dari: 

    usb.autoConnect.device0 = ...\n
    4. \n

  4. Simpan file .vmx Anda.

    5. \n
  5. Matikan dan mulai ulang VMWare Workstation.
    6. \n
  6. Mulai mesin virtual Anda.
    7. \n
  7. Hubungkan yubikey Anda. Pada menu VMWare, tarik ke bawah opsi VM, lalu Perangkat Yang Dapat Dilepas. Pilih Yubico.com Yubikey NEO OTP + U2F + CCID. Hubungkan ke mesin virtual Anda. Catatan! Anda mungkin melihat opsi kedua untuk "Shared Yubikey". Ini bukan opsi yang Anda inginkan.
    8. \n
  8. Ketika ditantang untuk otentikasi u2f Anda, token Anda harus berkedip. Tekan untuk mengotentikasi.
    9. \n
  9. Jika Anda mendekripsi mesin virtual Anda, jangan lupa untuk mengenkripsi ulang. Edit pengaturan mesin virtual,\npilih tab Opsi, pilih Kontrol Akses, lalu klik tombol Enkripsi.
    10. \n

Dengan VMware Fusion 11.0 (dan mungkin rilis produk lain dari jangka waktu yang sama), tidak hanya emulasi HID (dan blok yang sesuai pada passthrough langsung), tetapi juga emulasi smartcard dan blok tambahan untuk itu.

\n\n

Untuk menonaktifkannya, diperlukan beberapa tanda tambahan: 

usb.ccid.disable = \"TRUE\"\nusb.generic.allowCCID = \"TRUE\"\n

... membuat serangkaian opsi lengkap yang perlu diubah agar berhasil melewati YubiKey: 

usb.ccid.disable = \"TRUE\"\nusb.generic.allowCCID = \"TRUE\"\nusb.generic.allowHID = \"TRUE\"\nusb.generic.allowLastHID = \"TRUE\"\n
