Jadi saya menggunakan distro linux tin-foil-hat ini Qubes OS . Satu hal yang mengganggu saya adalah bagaimana mendapatkan statistik beban sistem secara keseluruhan.

\n\n

Saya dapat melihat penggunaan dasar CPU / Mem / Disk setiap VM di jendela hypervisor:

\n\n

https://csgergo.blogs.balabit.com/files/2012/12/snapshot3.png (untuk beberapa alasan, popup foto berputar selamanya, saya belum pernah melihatnya di stackexchange sebelumnya ...)

\n\n

Jadi jika saya mengetahui jumlah memori fisik / disk, maka saya dapat melakukan perhitungan saya sendiri, tetapi Anda akan berpikir bahwa xen hypervisor memiliki cara untuk melihat statistik sistem secara keseluruhan, bukan?

\n\n
\n\n

Dari arsitektur Qubes, Anda akan berpikir dom0 akan memiliki akses ke ini, tetapi top hanya menunjukkan sebagian kecil dari dom0s 2 gb RAM yang digunakannya.

\n\n

Saya melihat perintah qvm-top, yang menunjukkan ini kepada saya: 

--------------------+-----+------+\n               name | cpu |  mem |\n--------------------+-----+------+\n untrusted-browsing | 1.8 | 1722 |\n              disp3 | 1.2 | 2601 |\n       sys-firewall | 0.1 |  660 |\n            sys-net | 0.1 |  301 |\n               dom0 | 0.0 | 2967 |\n

tetapi sekali lagi, tidak ada tentang total sistem.

\n\n

Saya juga menemukan halaman ini menjelaskan detail qmemman yang menangani manajemen memori antar VM, tetapi (sesuai tingkat keahlian saya) saya tidak melihat apa pun di sana yang memungkinkan saya mendapatkan hasil statistik yang cantik.
