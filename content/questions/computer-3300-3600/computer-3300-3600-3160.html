Saya belum begitu paham dengan fstab, jadi saya punya beberapa pertanyaan. Ketika saya menggunakan perintah ini untuk memasang perangkat: 

sudo mount -t cifs -o username=admin //192.168.1.134/share_name /mnt/share_name\n

Dengan kata sandi: sudo = "kata sandi pengguna lokal" kata sandi = "kata sandi untuk perangkat"

\n\n

Bagaimana cara menerjemahkannya menjadi entri fstab? Sejauh ini, saya telah mencoba ini di fstab dan pemasangan gagal: 

//192.168.1.134/share_name  /mnt/share_name  cifs  default  0  0\n

Di sinilah pertanyaannya masuk. Di mana saya memiliki default, haruskah ada sesuatu yang menunjukkan username=admin, dll?

","source":"https://superuser.com/questions/226561/how-to-set-options-in-fstab-manual-mounting-is-successfull","author":"nicorellius","replies":[],"answers":[{"answer":"

Anda harus memiliki sesuatu di sepanjang baris: 

//192.168.1.134/share_name /mnt/share_name cifs username=server_user,password=server_password,_netdev 0 0\n^ Path to your share       ^ Mountpoint                  ^ Username           ^ Password      ^ See note A\n

Catatan A:
Opsi _netdev memastikan pemasangan hanya dilakukan SETELAH koneksi jaringan dibuat.

Urutan file FSTab adalah: device name mount point fs-type options dump-freq pass-num; setiap item dipisahkan oleh spasi.

\n\n

Untuk contoh yang Anda berikan, entri FSTab adalah: 

//192.168.1.134/share_name /mnt/share_name cifs username=admin,password=? 0 0\n

(jelas mengganti ? dengan kata sandi yang benar)

\n\n

Jika Anda ingin mempelajari lebih lanjut tentang FSTab, lihat Artikel Wikipedia FSTab .
