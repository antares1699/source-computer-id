find -iname \"*.zip\" -size +$((60*1024*1024))c -size -$((70*1024*1024))c\n

JANGAN gunakan abreviations 60M dan 70M karena ini juga akan mengecualikan semua file dengan ukuran lebih besar dari 69MB termasuk 69.001MB !!!

\n\n

dari bagian dokumentasi info 2.4 Ukuran 

-- Test: -size n[bckwMG]\n    True if the file uses N units of space, rounding up.\n    ...\n

jadi 69.001 mendapat tercakup menjadi 70 dan dengan demikian mendapat pengecualian !

\n\n

contoh sempurna adalah find . -size -1M yang mana hanya cocokkan ukuran file nol .
