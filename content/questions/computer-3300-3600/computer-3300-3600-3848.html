Saya mengalami masalah saat membuat tugas terjadwal di Windows7 Pro SP1. Tugas ini seharusnya menjalankan file .bat pada startup sistem. Ini terjadi sebelum pengguna logon. Ketika saya mengklik 'OK' untuk menyelesaikan pembuatan tugas, ini adalah kesalahan yang saya dapatkan: 

Error has occurred for task 'TaskName'. Error message: User account restriction error. The possible reasons are that blank passwords not allowed, or that a policy restriction has been enforced.\n

Komputer berada di lokasi yang jauh dan sudah dikirim ke klien jadi saya tidak memiliki akses penuh. Satu-satunya cara saya mengaksesnya adalah melalui TeamViewer. Pengguna yang masuk ke komputer pada saat saya mengaksesnya melalui TeamViewer adalah 'admin'. Saya yakin bahwa pengguna tidak memiliki kata sandi. Itu mengarahkan saya ke arah ini: Aktifkan Run As di Home Premium tanpa kata sandi?

\n\n

Namun, ketika saya membuat tugas yang sama dengan opsi 'Jalankan hanya saat pengguna masuk', tugas dibuat dengan sukses. Itu membawa saya pada kesimpulan bahwa tautan atas mungkin tidak begitu relevan.

\n\n

Saya ingin mencoba dan menghindari pengeditan registri seperti yang diusulkan dalam tautan yang disebutkan di atas karena komputer itu praktis bukan milik saya. Tetapi jika itu adalah solusi dan pilihan terakhir, saya akan menggunakannya.

\n\n

Apakah Anda memiliki solusi yang berbeda untuk masalah ini atau apakah mengedit registri satu-satunya solusi (jika sebenarnya itu adalah solusi)?
