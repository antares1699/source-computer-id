Saya mencoba mengkonfigurasi Ubuntu 12.04 VPS (OpenVZ) saya untuk reboot ketika kehabisan memori. Untuk melakukan ini saya telah menambahkan baris berikut ke file "/etc/sysctl.conf" saya. 

vm.panic_on_oom = 1\nkernel.panic = 14\n

Tetapi ketika saya menjalankan konfigurasi menggunakan perintah "sudo sysctl -p" saya mendapatkan kesalahan berikut. 

error: permission denied on key 'vm.panic_on_oom'\nerror: permission denied on key 'kernel.panic'\n

Saya telah mencoba menjalankan perintah yang sama dengan root dan menemukan kesalahan yang sama.

\n\n

Adakah yang bisa menunjukkan kesalahan apa yang saya lakukan? atau bagaimana cara memperbaikinya?

","source":"https://superuser.com/questions/659236/permission-denied-when-setting-values-in-sysctl-on-ubuntu-12-04","author":"Jay Bhatt","replies":[],"answers":[{"answer":"

OpenVZ bukanlah mesin virtual, ini lebih seperti sebuah wadah - dengan kata lain, hanya ada satu kernel yang berjalan untuk host dan semua "VPS". Ini berarti kepanikan kernel akan menjatuhkan seluruh sistem, termasuk setiap VPS yang dihosting di dalamnya, dan banyak pengaturan kernel dibagikan antara host dan semua tamu.

\n\n

Jadi Anda tidak diperbolehkan untuk mengubah pengaturan tersebut karena tidak masuk akal untuk melakukannya.
