Ada beberapa perbedaan antara BitLocker Drive Encryption dan\n Sistem File Enkripsi. BitLocker dirancang untuk membantu melindungi semua\n dari file pribadi dan sistem pada drive tempat Windows diinstal\n jika komputer Anda dicuri, atau jika pengguna yang tidak sah mencoba mengakses\n komputer. EFS digunakan untuk membantu melindungi file individu di drive mana pun\n basis per pengguna. Tabel di bawah ini menunjukkan perbedaan utama antara\n Enkripsi Drive BitLocker dan EFS.

\n \n

BitLocker vs. Encrypting File System (EFS)

\n \n
    \n
  • BitLocker mengenkripsi semua file pribadi dan sistem di drive tempat Windows diinstal, atau di drive data di komputer yang sama.
    • \n
  • EFS mengenkripsi file individu di drive mana pun.
    • \n
  • BitLocker tidak bergantung pada akun pengguna individu yang terkait dengan file. BitLocker aktif atau nonaktif, untuk semua pengguna atau grup.
    • \n
  • EFS mengenkripsi file berdasarkan akun pengguna yang terkait dengannya. Jika komputer memiliki banyak pengguna atau grup, masing-masing dapat mengenkripsi miliknya sendiri\n file secara mandiri.
    • \n
  • BitLocker menggunakan Trusted Platform Module (TPM), microchip khusus di beberapa komputer baru yang mendukung fitur keamanan tingkat lanjut.
    • \n
  • EFS tidak memerlukan atau menggunakan perangkat keras khusus apa pun.
    • \n
  • Anda harus menjadi administrator untuk mengaktifkan atau menonaktifkan enkripsi BitLocker setelah diaktifkan.
    • \n
  • Anda tidak harus menjadi administrator untuk menggunakan EFS.
    • \n
  • Anda dapat menggunakan BitLocker Drive Encryption dan Encrypting File System bersama-sama untuk mendapatkan perlindungan yang ditawarkan oleh kedua fitur tersebut. Kapan\n menggunakan EFS, kunci enkripsi disimpan dengan operasi komputer\n sistem. Meskipun ini dienkripsi, tingkat keamanan itu bisa\n berpotensi disusupi jika peretas dapat mem-boot atau mengakses file\n penggerak sistem. Menggunakan BitLocker untuk mengenkripsi drive sistem dapat membantu\n lindungi kunci-kunci ini dengan mencegah drive sistem dari boot atau\n sedang diakses jika dipasang ke komputer lain.
    • \n
