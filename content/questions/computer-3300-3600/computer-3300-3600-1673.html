Saya ingin mencadangkan data pribadi sensitif ke S3 melalui antarmuka gaya rsync. Saat ini saya menggunakan s3cmd - alat yang hebat - tetapi belum mendukung sinkronisasi terenkripsi. Ini berarti bahwa sementara data saya dienkripsi (melalui SSL) selama transfer, data disimpan di ujungnya tidak terenkripsi.

\n\n

Saya ingin tahu apakah ini masalah besar.

\n\n

FAQ S3 mengatakan "Amazon S3 menggunakan metode kriptografi yang telah terbukti untuk mengautentikasi pengguna ... Jika Anda menginginkan keamanan ekstra, tidak ada batasan untuk mengenkripsi data Anda sebelum menyimpannya di Amazon S3."

\n\n

Mengapa saya suka tambahan keamanan? Adakah cara agar ember saya bisa dibuka untuk mengintip tanpa sepengetahuan saya? Atau apakah mereka hanya mencoba menyelamatkan Anda ketika Anda secara tidak sengaja mengubah ACL Anda dan membuat bucket Anda dapat dibaca di seluruh dunia?

","source":"https://superuser.com/questions/113082/can-i-rely-on-s3-to-keep-my-data-secure","author":"Jamie Hale","replies":[],"answers":[{"answer":"

Dengan data yang tidak terenkripsi, Anda tidak akan pernah 100% yakin bahwa beberapa sysadmin nakal di Amazon tidak mengintip data Anda. Dan Anda dapat yakin bahwa mereka akan menyerahkan data Anda jika pihak berwenang memintanya.

\n\n

Jika Anda ingin 100% aman, enkripsi secara lokal sebelum mengunggah.

Lihat ke sinkronisasi file super fleksibel. Ini adalah salah satu aplikasi bagus yang merupakan aplikasi file / sinkronisasi dan cadangan. Bekerja sangat baik dengan S3 juga.

\n\n

Ini akan memampatkan (menggunakan standar zip) dan mengenkripsi (AES 256) dengan cepat sebelum mengunggah ke ember.

\n\n

Seperti yang diketahui semua orang, mengunggah ke layanan cloud apa pun mengubah tanggal modifikasi file. SUpeflexible "menyimpan" informasi tanggal diubah sebagai bagian dari nama file sehingga ketika Anda mengembalikan file tanggal asli juga dipulihkan.

\n\n

Ini menangani pembuatan versi dan pembaruan sebagian (blok) dari file besar. Sangat bagus untuk database SQL dan file Outlook.pst (belum teruji).

\n\n

Juga memecah file besar dalam potongan yang dapat dikelola.

\n\n

Wizard pemulihan menyembunyikan sebagian besar informasi yang berlebihan (potongan & info tanggal dalam nama file).

\n\n

Ini bukan browser.

\n\n

Ini sangat kuat sehingga bukan untuk orang yang lemah hati. UI bagus dan penerapan profil mereka adalah salah satu yang terbaik yang pernah saya lihat.

\n\n

Yang terbaik dari semuanya, Anda tidak perlu Superflexible untuk mendekripsi file Anda dalam beberapa tahun seperti yang dikatakan Winrar (diuji), Winzip atau serupa (tidak diuji) / harus mendekripsi secara independen.

\n\n

Program ini juga menangani backup di seluruh LAN, usb, situs ftp lain, dll. Sangat cepat membandingkan file jarak jauh / lokal. Saya telah menggunakannya selama sekitar empat tahun untuk backup lokal dan sekarang baru-baru ini untuk S3.
