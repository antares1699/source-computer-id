Sepertinya gnome-terminal telah mengubah perilaku startupnya dan sekarang shell baru yang dibuka melalui CTRL-SHIFT-N telah ~ ditetapkan sebagai direktori kerja mereka. Sebelum perubahan ini, shell baru dibuka di directoy saat ini.

\n\n

Bagaimana cara mengembalikan perubahan ini?

Jika Anda menggunakan Debian dan tidak ingin menunggu pengelola yang berbeda akhirnya memutuskan di mana "posisi terbaik untuk skrip", Anda dapat melakukan hal berikut:

\n\n

Edit ~/.bashrc, cari baris berikut (mungkin terlihat sedikit berbeda): 

if [ \"$color_prompt\" = yes ]; then\n  PS1='${debian_chroot:+($debian_chroot)}\\[\\033[01;32m\\]\\u@\\h\\[\\033[00m\\]:\\[\\033[01;m\\]\\w\\[\\033[00m\\]\\$ '\nelse\n  PS1='${debian_chroot:+($debian_chroot)}\\u@\\h:\\w\\$ '\nfi\n

dan tambahkan ke cabang lain: 

  # XXX hack to fix ctrl + shift + N\n  . /etc/profile.d/vte.sh\n

Yang menghasilkan: 

if [ \"$color_prompt\" = yes ]; then\n  PS1='${debian_chroot:+($debian_chroot)}\\[\\033[01;32m\\]\\u@\\h\\[\\033[00m\\]:\\[\\033[01;m\\]\\w\\[\\033[00m\\]\\$ '\nelse\n  PS1='${debian_chroot:+($debian_chroot)}\\u@\\h:\\w\\$ '\n  # XXX hack to fix ctrl + shift + N\n  . /etc/profile.d/vte.sh\nfi\n

Untuk shell Bash dan Z, tambahkan ini ke file rc shell Anda (yaitu, ~/.bashrc atau /.zshrc): 

source /etc/profile.d/vte.sh\n

(Solusi ini dipromosikan dari komentar ke jawaban lain.)
