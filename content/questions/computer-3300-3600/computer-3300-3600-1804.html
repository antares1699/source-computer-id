Masalah ini tampaknya terjadi saat komputer Anda tidak tersambung ke Internet saat penginstalan sedang berjalan. Penginstal tidak akan dapat memverifikasi sumber apt-get yang ada, dan akan menonaktifkannya.

\n

Dalam kasus saya, server DNS tidak diatur (jadi ping domain apa pun tidak akan berfungsi), jadi saya pertama-tama menjalankan sudo nano /etc/resolv.conf, dan menambahkan beberapa server DNS:

\n
\n

nameserver 8.8.8.8

\n

nameserver 8.8.4.4

\n
\n

pinging server sekarang berhasil.

\n

Kemudian, untuk mengaktifkan sumber apt-get, saya menjalankan sudo nano /etc/apt/sources.list, dan menambahkan repositori bergulir Kali (dari http: //docs.kali.org/general-use/kali-linux-sources-list-repositories):

\n
\n

deb http://http.kali.org/kali kontribusi utama kali-rolling tidak gratis

\n

#Untuk akses paket sumber, hapus komentar pada baris berikut

\n

deb-src http://http.kali.org/kali kontribusi utama kali-rolling tidak gratis

\n
\n

Tutup nano, dan keluar dari cangkangnya. Jalankan apt-get update, lalu apt-get install grub-pc.\nKeduanya berhasil, dan saya keluar dari shell untuk menyelesaikan instalasi GRUB melalui GUI.
