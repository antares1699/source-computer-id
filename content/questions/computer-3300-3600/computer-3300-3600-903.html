Sebagai root saya menambahkan menggunakan crontab -e untuk menambahkan skrip saya. Namun skrip menggunakan biner pada /usr/sbin/ dan saya mendapatkan kesalahan tentang itu tidak ada ketika skrip dijalankan dari cron. Bagaimana cara mengatasinya?

","source":"https://superuser.com/questions/917855/how-do-i-run-a-usr-sbin-binary-in-cron","author":"user3109","replies":[],"answers":[{"answer":"

Anda hanya harus menggunakan jalur lengkap untuk biner.

\n\n

Jadi katakanlah Anda ingin menggunakan ipconfig dalam tugas cron. Daripada crontab Anda memiliki perintah seperti ini: 

ipconfig\n

Anda akan mengubahnya menjadi: 

/usr/sbin/ipconfig\n

Ingat: Jalur shell hanya memberi tahu sistem tempat mencari binari / executable saat Anda memanggilnya dari baris perintah untuk membuat hidup Anda mengetik perintah lebih mudah. Menyediakan jalur lengkap untuk biner / executable berfungsi 100% sama untuk penggunaan skrip.
