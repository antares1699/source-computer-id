Inilah idenya:

\n\n

1) Instal beberapa distribusi Linux ringan (seperti Debian) sebagai OS host.

\n\n

2) Instal produk virtualisasi pilihan Anda; untuk saat ini saya akan melanjutkan dengan VirtualBox.

\n\n

3) Instal mesin virtual yang Anda inginkan di VirtualBox.

\n\n

4) Di OS host Anda, buat akun pengguna baru untuk setiap mesin; misalnya, akun bernama win7 untuk mesin virtual Windows 7. Letakkan perintah seperti vboxmanage start win7 di pengguna tersebut ~ / .bashrc mengajukan.

\n\n

5) Di OS host Anda, konfigurasikan manajer login Anda untuk menampilkan semua akun pengguna aktif, jadi Anda cukup mengklik mesin virtual yang ingin Anda mulai. Bergantung pada kebutuhan keamanan Anda, Anda mungkin juga mengaktifkan login tanpa kata sandi untuk akun tersebut, jadi Anda bahkan tidak perlu memasukkan kata sandi untuk memulai VM.

Inilah yang dilakukan oleh sistem VM "bare metal" berkinerja tinggi seperti VMWare ESX / ESXi. Mereka memuat kernel sistem operasi yang sangat, sangat kecil, yang tidak dapat melakukan apa pun kecuali menjalankan satu atau lebih VM.

\n\n

Hypervisor logam kosong lainnya termasuk Microsoft HyperV, VM Integritas HP, dan proyek Xen gratis (digunakan oleh banyak layanan hosting).

Mesin virtual dijalankan dari dalam sistem operasi jadi apa yang Anda inginkan tidak mungkin dilakukan. Anda harus mem-boot mesin dengan OS yang diinstal (Windows, Lunix, OSX, dll) kemudian mulai mesin virtual.

\n\n

Sepertinya Anda menginginkan sistem boot ganda (multi) yang memungkinkan.
