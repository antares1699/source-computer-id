Ini mungkin pertanyaan yang sangat mendasar, tetapi saya memiliki perangkat USB yang tetap menggunakan nama lama setelah memformat ulang dengan mkfs dan bahkan setelah mempartisi ulang dengan fdisk.

Itu tergantung pada sistem file mana yang ada di partisi.

\n\n

Jika fs adalah ext2 / ext3 / ext4, e2label seharusnya berfungsi: 

e2label  /dev/sda5 my_label\n

Jika itu NTFS, pilih ntfslabel: 

ntfslabel  /dev/sda9  my_label\n

Jika itu FAT, pilih mtools yang sudah Anda dapatkan: 

mlabel -i /dev/sda4 ::my_label\n

Jika nama yang Anda lihat bersifat deskriptif (seperti "volume 20 GB"), mungkin itu adalah sesuatu yang dibuat oleh pengelola file Anda jika tidak ada label sistem file yang benar. Namanya "tetap" hanya karena pengelola file membuat deskripsi yang sama saat Anda membuat sistem file baru tanpa nama. Jika itu masalahnya, maka mengganti nama sistem file dengan salah satu perintah yang disebutkan Hans Chen, atau dengan GParted, harus melakukan triknya.

\n\n

Itu Sistem Partisi MBR tidak mendukung label partisi; Namun, itu Sistem GPT tidak. Sepengetahuan saya, tidak ada browser file Linux yang menampilkan label partisi GPT. Jika Anda ingin mengeditnya, Anda dapat menggunakan GPT fdisk (gdisk) alat - tetapi Anda harus tidak gunakan gdisk pada disk MBR, karena ini akan mengubahnya menjadi bentuk GPT, yang akan membuat disk tidak dapat di-boot sampai Anda menginstal ulang boot loader Anda. (gdisk memperingatkan Anda dan memberi Anda kesempatan untuk membatalkan saat Anda meluncurkannya pada disk MBR, jadi tetaplah waspada terhadap peringatan semacam itu.)
