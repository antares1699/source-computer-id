Saya menjadwalkan skrip batch DOS untuk dijalankan melalui Penjadwal Tugas. Apakah ada cara untuk mengkonfigurasi tugas / pekerjaan untuk menulis semua output (menghasilkan output menggunakan perintah ECHO) ke file log? Dapatkah logging dikonfigurasi dalam Penjadwal Tugas (muncul v1.0 diinstal).

Gunakan kelompok starter yang melakukan logging

\n\n

Anda dapat mengedit skrip untuk menampilkan semua ke file alih-alih layar atau Anda bisa membuat file batch untuk menjalankan file batch dan menjadwalkan file batch lainnya ... (maaf, terdengar membingungkan, saya tahu).

\n\n

Pada dasarnya, dengan asumsi file batch yang Anda jalankan disebut myscript.cmd buat file batch bernama myscript-logging.cmd. Dalam file batch itu masukkan satu perintah sebagai berikut: 

myscript > c:\\myscript.%date:~-4%%date:~4,2%%date:~7,2%.%time::=%.log 2>&1\n

Di atas harus membuat file log di root drive C: dengan nama berdasarkan tanggal dan waktu dijalankan (jadi tidak pernah menimpanya dan Anda memiliki log untuk setiap kali skrip dijalankan).

\n\n

Contoh

\n\n

Contoh di atas, jika skrip dijalankan pada jam 10 malam pada tanggal 22 Juni 2010, akan berakhir dengan nama file yang terlihat seperti ini: 

myscript.20100622.220000.00.log\n

Formatnya adalah ini: 

myscript.YYYYMMDD.HHMMSS.hundredths of a second.log\n

Perhatian: Perhatikan ruang disk Anda!

\n\n

PERHATIAN: ini berpotensi mengisi hard drive Anda karena tidak pernah menghapus satu pun file log dan terus menambahkan file.

\n\n

Jadi, jika Anda berencana menggunakan ini untuk waktu yang lama, Anda harus menginvestasikan upaya tambahan untuk memantau dan mungkin membersihkan ruang disk Anda secara otomatis.
