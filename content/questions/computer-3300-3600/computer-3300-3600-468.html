Apakah Windows 7 menentukan variabel lingkungan untuk perpustakaan yang sudah ada sebelumnya? Saya penasaran karena saya sering menggunakan ini saat mengetik jalur dalam dialog Buka / Simpan atau dari prompt Run. Misalnya, untuk membuka direktori pengaturan Firefox pengguna saat ini, run: 

%APPDATA%\\Mozilla\\Firefox\n

Ini jalan pintas yang berguna, dan saya berharap menemukan sesuatu yang serupa untuk jalur perpustakaan. Sebagai contoh: 

%LIBRARIES%\\Music\\\n%MUSIC%\\\n

Saya memahami bahwa perpustakaan menggabungkan satu atau lebih folder tingkat atas, jadi mungkin ada ambiguitas. Microsoft memiliki daftar variabel lingkungan, tetapi sebagian besar tidak dapat diakses melalui UI Windows. Pertanyaan ini juga menautkan ke daftar yang bagus, tetapi tidak menyertakan folder Documents pengguna, folder Music, dll.

Perpustakaan Windows sebenarnya hanya xml file yang menentukan cara menampilkan kumpulan folder. Mereka ditampilkan menggunakan ekstensi shell khusus yang dibangun di Windows. Karena bukan folder nyata, file dan sub-folder tidak dapat diakses secara langsung melalui jalur.

\n\n

Dengan sebuah Buka jendela explorer, Anda dapat mengetik di bilah alamat: 

libraries\\music\n

Ini bisa tidak tampaknya bekerja dari kotak File Name dialog buka / simpan. Juga tidak bekerja dengan perintah run.

\n\n

Apa yang dapat Anda lakukan:

\n\n

Windows Vista memperkenalkan " Folder yang Dikenal "konsep. Ini menggantikan" daftar ID item khusus konstan "yang lebih lama ( CSIDL ) metodologi yang digunakan di XP. Ada posting yang bagus di Blog Windows; ia membahas perubahan ini dan memberikan gambaran umum tentang struktur file *.library-ms.

\n\n

Folder yang Dikenal memiliki GUID. Untuk mengaksesnya melalui baris perintah, awali URI Anda dengan knownfolder:. Ada lusinan ditentukan . Untuk mengakses folder Music pengguna saat ini misalnya, run perintah berikut: 

knownfolder:{4BD8D571-6D19-48D3-BE97-422220080E43}\n

Ini akan membawa Anda ke yang telah ditentukan sebelumnya default folder musik. Jadi, jika Anda telah mengubah folder perpustakaan, sayangnya ini tidak akan berfungsi.

\n\n

Folder default perpustakaan bisa jadi diakses secara terprogram dengan menggunakan IShellLibrary antarmuka, tapi itu di luar cakupan pertanyaan awal saya. Tampaknya pendekatan terbaik adalah dengan:

\n\n
    \n
  1. Petakan variabel lingkungan secara manual ke jalurnya yang diketahui
    2. \n
  2. Atau secara manual memetakan variabel lingkungan menggunakan GUID-nya. Keuntungannya di sini adalah Anda menjadikannya variabel sistem dan membuatnya berfungsi untuk semua pengguna).
    3. \n

Jika Anda ingin %music% mengarahkan ke folder musik Anda, maka Anda cukup menambahkan variabel lingkungan baru ke sistem Anda.

\n\n

\n\n

Jika Anda ingin %Libraries% menunjuk ke perpustakaan Anda lalu buat pintasan pada c:\\, misalnya, dan kemudian buat variabel lingkungan baru yang menunjuk ke pintasan c:\\libraries Anda. (tetapi memasukkan langsung ke sub-folder tidak akan berfungsi, tetapi Anda dapat membuat variabel baru untuk yang ada di Perpustakaan)

\n\n
