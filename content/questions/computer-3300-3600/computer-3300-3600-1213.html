Saya telah mengacaukan pemasangan perl saya di Mac OS X 10.6. Saya akan menjelaskan caranya:
Saya ingin menambahkan paket cpan untuk Bundle :: DBD :: mysql, cpan menyuruh saya memperbarui Bundle :: CPAN terlebih dahulu. Saya tidak bisa karena izin. Jadi saya sudoed perl -MCPAN -e shell dan memperbarui cpan, kemudian saya mendapatkan bundel mysql, tetapi gagal dalam beberapa tes, yang kemudian saya paksa instal (berfungsi btw). Kemudian karena alasan tertentu saya memutuskan untuk membangun kembali bundel mysql. Ini butuh waktu lama. Akhirnya saya menyadari itu telah banyak dibangun kembali, seperti perldoc.

\n\n

Sekarang setiap kali saya mencoba dan menjalankan perldoc saya tidak memiliki izin untuk melakukannya, tetapi ketika saya sudo perldoc memberi tahu saya, itu tidak dapat menemukannya. 

dlamblins-macbook:~ dlamblin$ perldoc\n-bash: /usr/bin/perldoc: Permission denied\ndlamblins-macbook:~ dlamblin$ sudo perldoc\nPassword:\nsudo: perldoc: command not found\ndlamblins-macbook:~ dlamblin$ sudo /usr/bin/perldoc\nPassword:\nsudo: /usr/bin/perldoc: command not found\n

Saya ingin menginstal ulang perl secara nyata seperti Apple menginstalnya ketika saya memperbarui ke Snow Leopard (meskipun menyimpan DBD :: mysql akan menyenangkan), dan saya tidak tahu harus mulai dari mana. Saya tidak ingin menggunakan beberapa paket fink atau sistem bergaya port. Mendapatkan perl 5.10 terbaru akan menjadi nilai tambah.
