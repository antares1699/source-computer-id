Jadi, saya memiliki laptop dengan salinan OEM Windows 8 yang jelas diinstal dalam konfigurasi UEFI dengan partisi GPT. Beberapa saat yang lalu saya menginstal Ubuntu 13.xx (yang terbaru) di sampingnya dan GRUB untungnya mendeteksi Windows dan semuanya baik-baik saja sampai sekarang.

\n\n

Jadi kemarin, saya memperbarui ke Windows 8.1 dan tampaknya menulis ulang boot-loader dan secara alami saya tidak bisa boot ke Ubuntu. Ini membawa saya langsung ke Windows.

\n\n

Bagaimana cara memperbaiki GRUB? Sebagian besar halaman bantuan / artikel berbicara tentang partisi MBR yang kasusnya sangat berbeda dari UEFI + GPT AFAIK. Dan artikel yang berbicara tentang UEFI adalah tentang memperbaiki instalasi Windows yang tidak dapat di-boot setelah menginstal linux yang bukan kasus saya.

\n\n

Adakah metode yang telah dicoba dan diuji untuk mengaktifkan kembali Ubuntu?

\n\n

PS: Sebelum merekomendasikannya Tidak; EasyBCD atau alat serupa tidak dapat menambahkan linux ke bootloader Windows jika Windows diinstal dengan UEFI.

Menggunakan Perbaikan Boot, seperti yang disarankan MariusMatutiae, mungkin berhasil; Namun, program itu terkadang melakukan lebih dari yang bijaksana, jadi saya lebih suka menghindarinya. Setidaknya ada tiga solusi yang kurang radikal:

\n\n

Solusi 1: Gunakan Firmware

\n\n

Banyak EFI menyediakan manajer boot internal yang memungkinkan Anda menyesuaikan urutan boot. Entri Ubuntu / GRUB Anda mungkin masih ada, jadi yang perlu Anda lakukan hanyalah menyesuaikan urutan boot menggunakan firmware. Masalah dengan pendekatan ini adalah bahwa utilitas pengaturan EFI sangat bervariasi sehingga tidak mungkin memberikan instruksi yang berlaku secara universal tentang cara melakukan ini. Namun, jika firmware Anda mendukung fitur ini, kemungkinan itu adalah cara termudah untuk melakukannya - setelah Anda mengetahui cara mendapatkan opsi tersebut!

\n\n

Solusi 2: Gunakan bcdedit di Windows

\n\n

Alat Windows bcdedit dapat menambahkan boot loader non-Windows ke daftar boot. Triknya adalah mencari tahu apa file tersebut. Anda bisa melakukannya dengan cara ini:

\n\n
    \n
  1. Boot ke Windows
    2. \n
  2. Buka Administrator Jendela Command Prompt. (Jangan gunakan shell pihak ketiga untuk ini, baik; Saya telah melihat laporan bahwa bcdedit tidak akan berfungsi dengan benar dengan beberapa di antaranya.)
    3. \n
  3. Ketik mountvol S: /S untuk memasang ESP sebagai S:. (Anda dapat mengubah S: ke yang lain jika Anda mau.)
    4. \n
  4. Menggunakan Command Prompt, periksa S: untuk menemukan boot loader Ubuntu Anda. Mungkin S:\\EFI\\ubuntu\\grubx64.efi atau S:\\ubuntu\\shimx64.efi. Jika Anda melihat yang terakhir, seharusnya aman untuk menggunakannya, dan mungkin perlu menggunakannya - shim adalah cara Ubuntu menangani Secure Boot (SB), tetapi pada komputer non-SB , itu hanya akan berdampak kecil. Jika Secure Boot tidak aktif, maka shim mungkin diinstal atau tidak, jadi Anda mungkin perlu merujuk ke grubx64.efi secara langsung.
    5. \n
  5. Ketik bcdedit /set {bootmgr} path \\EFI\\ubuntu\\shimx64.efi, ubah shimx64.efi menjadi grubx64.efi jika shimx64.efi tidak ada. Ubah jalur jika itu sesuatu yang lain, yang tidak mungkin.
    6. \n
  6. Secara opsional, ketik bcdedit /set {bootmgr} description \"Ubuntu\" untuk mengatur nama yang muncul di daftar manajer boot EFI sendiri. Ubah Ubuntu menjadi apa pun yang Anda suka.
    7. \n
\n\n

Jika Anda sudah mengetahui nama file untuk boot loader Anda, Anda dapat melewati langkah # 3 dan # 4. (ESP tidak perlu dipasang untuk menggunakan bcdedit dengan cara ini.)

\n\n

Metode ini memiliki keuntungan yaitu mencegah Windows mengotak-atik urutan boot - terkadang Windows akan mencoba menyesuaikan urutan boot tanpa larangan. Saya tidak tahu apakah ini akan mencegah terulangnya masalah ini jika / ketika Anda meningkatkan ke apa pun yang datang setelah Windows 8.1.

\n\n

Solusi 3: Boot ke Linux dan Gunakan efibootmgr

\n\n

Anda mungkin dapat mem-boot ke Linux dengan menggunakan manajer boot firmware itu sendiri, yang dapat Anda akses di sebagian besar komputer dengan menekan Esc atau tombol fungsi pada saat boot, meskipun tombol mana yang bervariasi dari satu komputer ke komputer lainnya. Atau, Anda mungkin bisa menggunakan menemukan kembali pada USB flash drive atau CD-R sebagai boot manager jika milik Anda tidak memadai. Anda juga dapat melakukan booting menggunakan live CD Linux atau disk darurat, tetapi pastikan Anda melakukan boot dalam mode EFI - boot mode BIOS tidak akan memadai. Setelah Anda berada di Linux, Anda dapat menggunakan efibootmgr untuk menyesuaikan urutan boot:

\n\n
    \n
  1. Buka jendela Terminal.
    2. \n
  2. Ketik sudo efibootmgr -v untuk mendapatkan daftar program boot. Satu untuk Linux, dan akan meluncurkan shim atau GRUB. Perhatikan daftar BootOrder. Kemungkinannya adalah entri Windows sekarang menjadi yang pertama, dan entri Ubuntu muncul kemudian dalam daftar. Beberapa entri mungkin membingungkan. Abaikan saja; fokus untuk menemukan entri Ubuntu dan mengidentifikasi nomornya (dalam entri Boot#### di awal baris).
    3. \n
  3. Ketik sudo efibootmgr -o {list}, ubah {list} menjadi daftar nomor boot yang dipisahkan koma, seperti pada sudo efibootmgr -o 5,0 jika Boot0005 untuk Ubuntu dan Boot0000 untuk Windows. Anda dapat menambahkan lebih banyak entri jika Anda suka, tetapi yang pertama adalah yang paling penting, karena itulah yang akan di-boot terlebih dahulu.
    4. \n
\n\n

Jika entri Ubuntu melakukannya tidak ada, Anda dapat membuatnya dengan efibootmgr, seperti di: 

efibootmgr -c -d /dev/sda -p 1 -l '\\EFI\\ubuntu\\shimx64.efi' -L \"Ubuntu\"\n

Ubah -d /dev/sda untuk menunjuk ke seluruh perangkat disk Anda dan -c 1 untuk menentukan nomor partisi. (Faktanya, /dev/sda dan 1 adalah default, jadi Anda benar-benar membutuhkan ini hanya jika ESP Anda bukan /dev/sda1.)

Ada banyak solusi berbeda untuk ini. Satu set solusi pasti membutuhkan penggunaan distro live Linux, seperti misalnya Ubuntu. Bakar gambar Ubuntu ke stik USB, boot dari itu dengan memilih Coba Ubuntu tanpa menginstal ... , buka terminal, lalu instal dan gunakan Perbaikan Boot , mengikuti instruksi di halaman Web yang merupakan milik Dokumentasi Resmi Ubuntu. Ini umumnya cukup untuk memperbaiki sebagian besar masalah booting.

\n\n

Sebagai alternatif, alih-alih mengunduh Boot-Repair, Anda dapat memasang disk HDD Anda dari terminal, misalnya dengan menerbitkan 

   sudo mount /dev/sdaX /mnt\n

di mana X adalah nomor partisi tempat Anda meletakkan /. Jika Anda tidak yakin, sebelum perintah di atas, 

   sudo fdisk -l\n

akan memberi tahu Anda partisi mana. Kemudian keluarkan perintah ini: 

   cd /mnt\n   sudo chroot .\n   sudo update-grub\n

dan, setelah ini selesai, Ctrl + D.
