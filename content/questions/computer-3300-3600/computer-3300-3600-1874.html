Jika saya menggunakan fitur Teks ke Kolom pada data berikut, menggunakan ";" sebagai pembatas:

\n\n
\n

foo; bar; qux; baz; roti panggang;
quux; selai; kacang;

\n
\n\n

Saya akan mendapatkan hasil "rata kiri" di kisi sel yang dihasilkan: 

 |foo   |bar   |qux   |baz   |toast |\n |quux  |jam   |beans |      |      |\n

Namun, saya ingin mereka "rata kanan": 

 |foo   |bar   |qux   |baz   |toast |\n |      |      |quux  |jam   |beans |\n

Bagaimana saya bisa melakukan ini?

\n\n

\n\n

Rumus berikut akan memungkinkan konversi cepat data Anda ke formulir yang Teks-ke-Kolom akan langsung diurai dengan rata kanan seperti yang Anda gambarkan:

\n\n

\n\n

Rumus D5 (menambahkan titik koma jika tidak ada): 

=IF(RIGHT(B5,1)<>\";\",B5&\";\",B5)\n

Rumus G5 (menambahkan jumlah titik koma yang diperlukan): 

=REPT(\";\",5-(LEN(D5)-LEN(SUBSTITUTE(D5,\";\",\"\"))))&D5\n

Menyalin hasil yang diikuti dengan Tempel-Khusus-sebagai-Nilai harus mampu menghasilkan bahan mentah yang sesuai untuk konversi Teks-ke-Kolom.

\n\n

Solusinya tergantung pada jumlah kolom maksimum yang tetap; di sini, lima. Rumus G5 dapat digeneralisasikan dengan menambahkan sel 'sejumlah kolom untuk menghasilkan' di tempat lain pada lembar dan merujuk sel baru ini alih-alih nilai 5 yang di-hardcode.

\n\n

Selain itu, jika Anda dijamin bahwa data akan selalu memiliki tanda titik koma, langkah perantara dari =IF(RIGHT(B5,1)<>\";\",B5&\";\",B5)\n tidak berguna.

\n\n

EDIT: Berdasarkan pengamatan Some_Guy di komentar, metode ini juga akan berfungsi jika semua baris dikonstruksikan kekurangan tanda titik koma.

Berikut ini rutinitas VBA lain untuk melakukannya.\nLakukan Teks Anda ke Kolom,\nlalu pilih rentang persegi panjang tempat Anda memasukkan data\n(mis., kolom A- (bidang maks) × baris) dan jalankan makro ini.\nLihat Bagaimana cara menambahkan VBA di MS Office? untuk bahan ajar. 

Sub Copy_Right()\n    For Each rr In Selection.Rows\n        For cn = Selection.Columns.Count To 1 Step -1\n            If Len(rr.Cells(1, cn)) > 0 Then Exit For\n        Next cn\n        ' cn is now the (relative) column number of the last cell in this row\n        ' that contains (non-blank) data.\n        my_offset = Selection.Columns.Count - cn\n        ' my_offset is how many columns to the right we need to move.\n        ' If my_offset = 0, the row is full of data (or, at least,\n        ' the last column contains data; there may be blank cells\n        ' to its left), so there’s nowhere to move it.\n        ' If cn = 0, the row is empty, so there’s nothing to move.\n        If cn = 0 Or my_offset = 0 Then\n            ' Nothing to do.\n        Else\n            For cn = Selection.Columns.Count To 1 Step -1\n                If cn > my_offset Then\n                    ' Copy data to the right.\n                    rr.Cells(1, cn) = rr.Cells(1, cn - my_offset)\n                Else\n                    ' Set the cells on the left to blank.\n                    rr.Cells(1, cn) = \"\"\n                End If\n            Next cn\n        End If\n    Next rr\nEnd Sub\n

Ini akan menangani sel kosong tertanam (misalnya, the;quick;;fox;) dengan benar.\nJika tidak, perbedaan antara jawaban ini dan yang lainnya\nhanyalah preferensi pribadi yang sewenang-wenang,\ndan yang lainnya mungkin lebih unggul dalam hal-hal yang saya tidak mengerti.
