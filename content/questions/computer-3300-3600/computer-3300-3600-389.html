Saya menjalankan proses panjang pada shell jarak jauh. Saya ingin memicu pemberitahuan saat proses selesai. Jawaban ini tidak cukup, karena perintah long ada di server jauh, sehingga tidak dapat memanggil terminal-notifier.

\n\n

Saya bisa mendekati hasil yang saya inginkan dengan menggunakan pemicu iTerm2. Misalnya jika saya menjalankan perintah berikut di server jarak jauh dan mengatur pemicu iTerm2 untuk __FINISHED__: 

./long_process && echo \"\\__FINISHED__\"\n

Ini memiliki fitur yang tidak diinginkan bahwa setiap kali saya menggulir kembali ke perintah ini di editor saya atau kode saya (saya menjalankan kode di shell emacs), notifikasi terpicu.

\n\n

Salah satu solusinya mungkin semacam pemberitahuan teks yang akan dikenali oleh iTerm2, tetapi itu tidak akan muncul di buffer gulir balik emacs.

Anda belum menentukan jenis notifikasi yang Anda inginkan atau OS apa yang dijalankan server jarak jauh, jadi saya harus membuat beberapa asumsi di sini. Saya akan menganggap Anda tidak terlalu peduli jenis notifikasi apa selama Anda diberi tahu dan bahwa server jarak jauh menjalankan beberapa rasa * nix.

\n\n
    \n

  1. Kirim email ke diri Anda sendiri. Jika sendmail dikonfigurasi di server, Anda bisa melakukannya 

    ./long_process && echo \"Job done\" | sendmail pnj@yourdomain.com\n
    2. \n

  2. ssh kembali ke mesin lokal Anda (dengan asumsi ini mungkin) dan membuatnya berbicara dengan Anda. Lihat sini untuk cara yang lebih keren dalam membuat OSX berbunyi bip pada Anda. 

    ./long_process && ssh you@local.ip say \"Yo! All done\"\n

    atau 

    ./long_process && ssh you@local.ip terminal-notifier -message \"Job finished!\" -title \"Info\"\n

    Jika Anda menghubungkan dari IP dinamis dan Anda telah mengkonfigurasi router Anda sehingga Anda dapat ssh ke IP dinamis itu, Anda dapat melakukan ini (dengan asumsi Anda saat ini hanya terhubung dari mesin jarak jauh Anda): 

    ip=$(who | grep $USER | perl -lne 's/\\\\((.+?)\\\\)\\s*$//; print \"$1\"' | tail -n 1) &&\n ./long_process && ssh you@$ip terminal-notifier -message \"Job finished!\" -title \"Info\"\n

    Anda dapat mengatur ssh tanpa kata sandi di cara normal . Ini seharusnya tidak terpengaruh oleh IP dinamis. Setelah Anda melakukannya, kode di atas akan berfungsi.

    3. \n

  3. Menggunakan penurut dan mengirimkan pemberitahuan ke perangkat Android atau iOS Anda (jika ada) 

    ./long_process && pushover.pl \"All done\"\n
    4. \n
