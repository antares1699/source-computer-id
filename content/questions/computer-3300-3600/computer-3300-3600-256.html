Windows XP 32 bit memiliki dua batas memori yang terkenal. Setiap proses dibatasi hingga 2GB memori (atau 3GB jika Anda mengubah pengaturan). Total memori maksimum yang akan digunakan Windows XP adalah 3,25 GB.

\n\n

Tidak ada batas dasar 4GB untuk memori dalam sistem operasi 32 bit - Windows Server 2003 dapat menggunakan lebih dari 4GB. Batas kunci yang menentukan sistem 32-bit adalah per-proses (ruang alamat virtual untuk satu aplikasi tertentu). Inilah alasan batas 2GB / 3GB per-proses di Windows XP, yang juga digunakan bersama oleh Windows 2003 Server.

\n\n

Saya dual-boot, dengan 64-bit OpenSUSE 11.3 Linux dan Windows XP. Linux menjadi sistem operasi utama saya lebih dan lebih dari waktu ke waktu, tetapi saya memiliki terlalu banyak aplikasi Windows XP yang saya gunakan secara teratur untuk dibuang begitu saja.

\n\n

Karena itu, yang ingin saya ketahui adalah apakah ada apa saja cara mendapatkan memori di atas 3.25GB menjadi praktis digunakan di Windows XP 32 bit. Jenis kemungkinan yang ada dalam pikiran saya adalah ...

\n\n
    \n
  • Cara membuat Windows XP berperilaku lebih seperti Windows 2003 Server.
    • \n
  • Cara mengganti penanganan cache hard disk di Windows XP dengan alternatif yang dapat menggunakan memori ekstra.
    • \n
  • Disk RAM yang dapat menggunakan memori non-Windows.
    • \n
\n\n

Dua kemungkinan yang mungkin layak disebutkan, tetapi yang akan saya tolak untuk saat ini, adalah ...

\n\n
    \n
  • Gunakan Windows 2003 Server 32 bit - masalah lisensi.
    • \n
  • Gunakan Reactos - masih alfa, dengan masalah yang signifikan, dan saya tidak yakin apakah itu dapat menggunakan lebih dari 3.25GB memori.
    • \n
