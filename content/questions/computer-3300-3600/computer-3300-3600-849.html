Debian Buster dhclient secara acak menggunakan alamat MAC-nya atau DUID-LLT yang dihasilkan sebagai ID klien DHCP-nya.

\n\n

Dapatkah saya memaksanya untuk selalu menggunakan alamat MAC alih-alih DUID-LLT yang dihasilkan?

\n\n

Saya tahu opsi send dhcp-client-identifier = xx:xx:xx:xx:xx:xx;, tetapi itu memaksa saya untuk mengonfigurasi setiap antarmuka secara manual dengan alamat MAC-nya sendiri, sesuatu yang saya lebih suka hindari jika memungkinkan.

\n\n

Seperti yang saya katakan sebelumnya, itu sudah melakukannya, tetapi secara acak.

\n\n

Bisakah itu dipaksakan untuk selalu melakukannya?

Solusinya sederhana, tetapi tidak didokumentasikan (setidaknya tidak pada halaman manual dhclient.conf). Anda dapat mengatur opsi send dhcp-client-identifier = hardware daripada meng-hardcode alamat MAC tertentu.

\n

Contoh lengkapnya: 

$ echo "send dhcp-client-identifier = hardware;" >>/etc/dhcp/dhclient.conf\n$ rm /var/lib/dhcp/*\n$ systemctl restart networking\n

Ini mengkonfigurasi dhclient untuk mengirim alamat perangkat keras sebagai pengenal klien, menghapus sewa sebelumnya dan memulai ulang antarmuka dengan pengaturan baru. Setelah perubahan ini, alamat MAC akan digunakan sebagai ID klien dari setiap antarmuka, secara otomatis.
