Jika OS itu sendiri menunjukkan sektor buruk, maka drive Anda akan menurun. Perangkat keras harus menghapus sektor buruk dari layanan secara transparan, dan mungkin kehabisan sektor cadangan untuk dipetakan ulang. Jika demikian, data Anda sangat berisiko.

\n\n

Saya akan mengingat saran David dan memigrasikan data Anda SECEPATNYA. Anda mungkin ingin menginstal "smartmontools" dan menjalankan smartctl untuk melihat apa yang dilaporkan oleh perangkat itu sendiri.

\n\n

Anda dapat mencoba menggunakan utilitas "badblocks" pada perangkat mentah yang menampung drive NTFS Anda, karena ia bekerja pada perangkat dan tidak bergantung pada sistem file. Pastikan Anda membaca halaman manual DENGAN HATI-HATI dan gunakan mode non-destruktif. Ini harus dapat membaca setiap sektor, kemudian menulis ulang setiap sektor. Melakukan ini akan memaksa drive itu sendiri untuk memetakan ulang sektor yang bermasalah. Uji prosedur ini pada stik USB atau media lain terlebih dahulu.

\n\n

Saya sarankan Anda memberi tahu badblock untuk memproses sejumlah besar sektor sekaligus, karena drive 1TB akan membutuhkan waktu yang sangat lama untuk dipindai.

\n\n

Sekali lagi ... pastikan Anda tahu apa yang Anda lakukan. Satu slip dan data Anda terpanggang.

\n\n

Jika datanya berharga bagi Anda dan Anda tidak yakin dengan kemampuan Anda, Anda mungkin ingin menggunakan solusi komersial seperti SpinRite (meskipun, harganya mahal dan akan membutuhkan mesin Windows). Perhatikan, saya tidak pernah menggunakan SpinRite, tetapi saya telah mendengar hal-hal baik tentangnya secara online.

Itu tergantung pada sistem file yang Anda gunakan. Jika Anda menggunakan ext2, etx3 atau ext4 (IIRC Ubuntu 9.10 default ke ext4 untuk instalasi baru, versi sebelumnya default ke ext3) maka Anda dapat menjalankan 

e2fsck -c -c -k -C 0 /dev/device\n

yaitu 

e2fsck -c -c -k -C 0 /dev/sdc1\n

untuk sistem berkas pada partisi pertama dari drive sdc. Sistem berkas tidak boleh di-mount saat proses ini berlangsung, jadi jika ini diperlukan untuk sistem Anda yang sedang berjalan, Anda harus menjalankan e2fsck saat boot dari "live CD" atau yang serupa.

\n\n

Opsi -c adalah apa yang menyebabkan pemindaian permukaan ditambahkan ke apa yang dilakukan pemeriksa, kemudian -c kedua membuatnya melakukan tes baca + tulis non-destruktif, -k memberi tahu itu untuk menyimpan daftar badblock yang ada alih-alih menguji ulang mereka dan -C 0 hanya membuat keluaran e2fsck lebih banyak informasi kemajuan saat ia melakukan tugasnya. Lihat man e2fsck untuk detail lebih lanjut.

\n\n

Di atas akan menandai blok buruk sehingga tidak digunakan di masa depan. Anda perlu menjalankannya kembali jika Anda pernah memformat ulang drive atau partisi individu tersebut. Anda juga harus menjalankannya kembali sesekali, jika ada sektor "hampir buruk" yang semakin menurun. Tidak ada cara untuk "memulihkan" bad sector.

\n\n

Saya sarankan Anda mempertimbangkan untuk memigrasi data Anda ke drive lain sebagai gantinya, kecuali jika tidak ada yang benar-benar penting di sana (yaitu tidak akan terlalu merepotkan Anda jika drive tiba-tiba menjadi jauh lebih buruk dan data menjadi praktis tidak dapat dibaca atau rusak). Jika Anda tidak mengganti drive, pastikan Anda memiliki cadangan yang baik untuk apa pun yang Anda pedulikan (meskipun Anda tetap harus memiliki rezim pencadangan yang baik).

Jika Anda berharap untuk menyimpan data di partisi eksternal itu, Anda tidak ingin menulis ke drive itu lagi, yang diusulkan beberapa jawaban ini.\nSebagai gantinya, gunakan alat pemulihan forensik untuk mem-image drive. ddrescue adalah alat favorit saya untuk pekerjaan semacam itu.

\n\n

Inilah "catatan untuk diri sendiri" saya, yang khusus untuk mac, tetapi mencakup poin-poin utama: Mengidentifikasi nama sumber dan target pemasangan perangkat keras / dev, melepasnya, lalu menyapu sumber ke target, dengan coba baca-ulang dinonaktifkan, pembacaan log kesalahan, tetapi melewatkan blok yang gagal untuk yang terakhir.

\n\n

Pencitraan forensik disk dan / atau menyelamatkan disk yang gagal: Gnu ddrescue\n(tidak sama dengan dd_rescue, meskipun serupa) Semua perlu dilakukan sebagai root.
Pertama, konfirmasikan perangkat Anda terpasang:
daftar diskutil
Lepaskan sumber (mis. Disk1) (dan tujuan jika mengirim ke partisi fisik, bukan file gambar mentah):
diskutil unmount disk1
Lakukan salinan gambar blok-demi-blok, dari disk1 ke file gambar disk yang dapat dipasang, buat log dan pertama-tama fokuslah pada bagian-bagian yang dapat dibaca:
ddrescue -v / dev / disk1 myrescueddisk.dmg rescue.log
