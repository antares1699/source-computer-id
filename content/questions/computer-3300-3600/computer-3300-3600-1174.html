Coba pasang sebagai drive budak (non-booting) di komputer lain yang memiliki sistem operasinya sendiri.

\n\n

Kegagalan utama Windows tidak menyediakan alat untuk menangani masalah seperti ini; Sering kali, hal terbaik yang dapat dilakukan adalah meminta Anda untuk memformatnya ( jangan lakukan ini! ), jadi saya sarankan menggunakan alat yang berbeda untuk mencoba memperbaiki kesalahan.

\n\n

Karena Windows mengenalinya sebagai partisi RAW, itu berarti tabel partisi mungkin ditulis berlebihan atau rusak. Jangan khawatir; ada cara untuk memperbaikinya tanpa mematikan data Anda.

\n\n

Coba boot ke Knoppix dan menggunakan cfdisk untuk memeriksa dan menulis ulang tabel partisi (juga, pelajari Linux). Ini akan menyelesaikan sebagian besar masalah dan tidak akan menyentuh data Anda. Ada juga alat lain untuk memeriksa sistem file pada disk dari Knoppix.

\n\n

Pilihan lainnya adalah dengan membeli alat sejenis SpinRite dan menginspeksi permukaan data dari kerusakan.

\n\n

Juga, jika menghubungkan hard drive ke komputer menyebabkannya boot-hang, mengapa tidak menunggu sampai komputer sudah di-boot? Baik spesifikasi USB dan SATA adalah hot-swappable.

Saya memiliki masalah serupa yang disebabkan oleh beberapa sektor buruk.

\n\n

Jika Anda mengunduh SeaTools untuk DOS (unduh image ISO yang dapat di-boot) dan reboot ke sana, Anda dapat menjalankan pemindaian penuh yang akan mencoba dan mengidentifikasi serta menonaktifkan bad sector.

\n\n

Ini akan mencegah bad sector dilihat oleh OS. Kapasitas drive akan berkurang dengan jumlah bad sector.

\n\n

Ini akan menyebabkan hilangnya data, jadi pastikan Anda telah menjelajahi semua jalan untuk memulihkan data apa pun yang Anda butuhkan sebelumnya.
