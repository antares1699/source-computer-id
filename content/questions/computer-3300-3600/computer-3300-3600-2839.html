DOS dan lainnya mode nyata sistem operasi banyak menggunakan layanan BIOS. Karena mode nyata prosesor x86 terikat ke dalam ruang alamat 2 ^ 20 = 1Mb, vendor sistem i8086 memutuskan bahwa BIOS harus dipetakan ke ruang alamat kelas atas (seperti saat ini kernel setengah lebih tinggi memetakan memori virtual mereka di akhir rentang): dari 0xA0000 dan lebih tinggi.

\n\n

Mode terproteksi membuat memori yang dicadangkan ini menjadi celah dalam ruang alamat: program mode terlindungi tidak dapat memanggil fungsi dari BIOS tanpa peretasan yang signifikan, menjadikannya tidak dapat digunakan dalam mode terlindungi dan panjang (yang dijalankan oleh setiap OS modern), tetapi mentransfer perangkat lunak dari RM ke PM tidak cepat, sehingga insinyur Intel meninggalkan jalan keluar, mode v8086, yang memungkinkan untuk menjalankan program mode-nyata lama di kotak pasir di dalam lingkungan mode terlindungi (misalnya, OS / 2 menggunakan mode v8086 secara intens dan emulasi DOS di sistem Windows juga mengandalkan fitur ini), masih membutuhkan layanan BIOS untuk kompatibilitas.

\n\n

Memori ini masih digunakan dalam mode terlindung untuk beberapa tugas, seperti terminal VGA / output grafis (memori terminal terletak pada 0xB8000, font konsol juga dapat disesuaikan di sana, framebuffer grafis VGA pada 0xA8000, iirc). Terkadang memori ini bahkan digunakan untuk API seperti VBE 3 , tapi ya, pemetaan BIOS pada 0xA0000 sebagian besar adalah masalah kompatibilitas ke belakang. Ada upaya untuk menggantinya: dengan pemikiran ulang modern tentang BIOS, UEFI sekarang terus mendapatkan popularitas.

\n\n

Kesimpulan: Pemetaan BIOS pada 0xA0000 sebagian besar adalah masalah kompatibilitas ke belakang dan ada upaya untuk menggantinya dengan lingkungan boot yang lebih modern seperti UEFI.

AFAIK, ini sebagian besar karena alasan sejarah. Program disimpan pada chip, tetapi untuk menjalankannya, kode tersebut dimuat ke dalam RAM (BIOS shadow), di mana CPU dapat langsung mengambil instruksi. Ini terutama diperlukan untuk image BIOS yang dikompresi pada chip yang lebih murah dengan kapasitas minimal.

\n\n

Menonaktifkan opsi 'BIOS shadow' dapat memperlambat proses boot atau jarang menyebabkan masalah yang lebih buruk.\nSelain itu, sistem operasi seperti MS DOS juga menggunakan rutinitas BIOS untuk IO, yang semacam HAL. Ini akan sangat lambat dan tidak aman untuk digunakan dalam OS multithread saat ini, jadi setelah menyerahkan kendali ke OS, bagian RAM itu tetap tidak aktif.

\n\n

Sistem lain bahkan memiliki kernel di chip itu, mis. Commodore Amiga Kickstart, yang juga tersedia sebagai Floppy.
