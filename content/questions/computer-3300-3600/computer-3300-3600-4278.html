Saya bertanya-tanya bagaimana Torbutton dan Tor mengubah IP saat menjelajah internet menggunakan Firefox?

\n
\n\n

Torbutton tidak mengubah apapun. Itu hanya memberi tahu Firefox untuk menggunakan Tor.

\n\n

Saat Anda menggunakan server proxy, browser mengirimkan semua permintaan HTTP Anda ke server itu, yang meneruskannya ke tujuan akhirnya.

\n\n
    \n
  • browser ↔ server proxy ↔ server web
    • \n
\n\n

Alamat Anda yang sebenarnya tidak berubah, itu hanya disamarkan, karena server web hanya melihat apa yang terhubung dengannya langsung - dalam hal ini, proxy.

\n\n

(Namun, jika seseorang mengambil kendali atas proxy, mereka dapat memantau semua yang dikirim melaluinya (seperti siapa yang mengunggah file ke Situs X). Seringkali, itu adalah pemilik server itu sendiri.)

\n\n

Dengan Tor, permintaan dialihkan melalui beberapa server, dan dienkripsi secara terpisah untuk masing-masing:

\n\n
    \n

  • browser ↔ Tor di komputer Anda

    • \n

  • Tor di komputer Anda [[[↔]]] node A [[↔]] node B [↔] node C ↔ server web

    \n\n

    (Setiap [] mewakili lapisan enkripsi.)

    • \n
\n\n

Dengan cara ini, web server hanya melihat koneksi yang berasal dari node C, sedangkan C hanya melihat koneksi dari B. Hanya node A yang tahu di mana pengguna berada, tetapi tidak tahu apa yang ditransfer oleh pengguna.

\n\n

(Ini disebut perutean bawang . Lihat juga Ikhtisar Tor .)

\n\n
\n\n
\n

Saya belajar bahwa menggunakan proxy dapat mengubah IP menjadi IP server proxy. Namun apakah cara yang dilakukan Torbutton dan Tor sama dengan menggunakan proxy?

\n
\n\n

Tor aku s proxy, tetapi cara kerjanya berbeda dengan "server proxy" yang biasa Anda gunakan. Lihat uraian di atas.

\n\n

Torbutton, sekali lagi, hanyalah sebuah tombol yang mengkonfigurasi ulang Firefox.

\n\n
\n\n
\n

Dengan Torbutton dan Tor, mengapa IP hanya berubah saat browsing internet menggunakan Firefox, dan tidak diubah di luar Firefox, seperti menggunakan aplikasi browser yang berbeda?

\n
\n\n

Torbutton adalah Firefox add-on dan hanya dapat dikonfigurasi ulang Firefox . Tor tidak mengubah pengaturannya sendiri - saat Anda memulainya, ia hanya menunggu koneksi. Anda sendiri harus memberi tahu program untuk terhubung melaluinya.

\n\n

Jika program mendukung penggunaan server proxy SOCKS 5, arahkan ke 127.0.0.1:9050. Jika hanya mendukung proxy HTTP, Anda memerlukan Polipo, yang disertakan dengan Tor Bundle, dan berjalan pada 127.0.0.1:8123.

Situs Tor sebenarnya memiliki deskripsi yang cukup bagus tentang cara kerja perutean bawang. Pada dasarnya setiap node Tor merangkum permintaan yang diterimanya dan meneruskan permintaan yang dienkapsulasi ke node berikutnya.

\n\n

Kemampuan Tor untuk menyembunyikan alamat sumber yang berasal dari server tujuan bergantung pada lalu lintas yang sedang dilalui. Secara umum, server tujuan akan melihat alamat IP dari simpul Tor hop terakhir; Namun hal ini tidak selalu terjadi. Misalnya, situs web mungkin menyematkan beberapa Javascript yang mengirimkan alamat IP lokal ke dalam beberapa data formulir.

\n\n

Hanya aplikasi yang dikonfigurasi untuk menggunakan proxy Tor yang akan mendapatkan keuntungan.
