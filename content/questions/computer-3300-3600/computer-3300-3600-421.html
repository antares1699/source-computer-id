Saya menjalankan server Ubuntu 12.04 di dalam jaringan perusahaan saya. Ini membutuhkan proxy untuk mengakses jaringan di luar. Saya mengkonfigurasi proxy selama instalasi. Setelah itu, saya menemukan pengaturan proxy saya disimpan di /etc/apt/apt.conf. Tapi itu tidak dalam variabel lingkungan. Namun, wget juga dapat mengunduh halaman web melalui proxy. Di mana wget (dan program lain) membaca pengaturan proxy saya?

","source":"https://superuser.com/questions/417903/where-is-proxy-setting-saved-on-ubuntu-12-04","author":"woodings","replies":[],"answers":[{"answer":"

Seperti biasa saya kira tidak ada yang peduli untuk mengintegrasikan semua pengaturan komunikasi untuk program yang digunakan di Linux / Ubuntu, yang sangat disayangkan.

\n\n

Di 11.10 Anda memiliki pengaturan luas sistem di beberapa lokasi:

\n\n

/ etc / environment 

...\nhttp_proxy=\"user:pass@my.proxy:port\n...\n

/etc/apt/apt.conf.d/02/proxy

\n\n

Perintah apt memiliki pengaturannya di file ini: 

Acquire::http::proxy \"http:/user:pass@my.proxy:port\"\n

/ etc / wgetrc

\n\n

Perintah wget memiliki pengaturan dalam file ini mengikuti format yang sama dengan file /etc/environment yang disebutkan di atas. 

http_proxy=\"user:pass@my.proxy:port\n

Asalkan 12.04 memiliki konfigurasi file yang sama dengan 11.10 mengenai ketiga program ini, Anda dapat mengedit file-file ini untuk server.

\n\n

Saya memiliki file skrip untuk mengubah semua hal ini di 11,10 bolak-balik antara pekerjaan dan rumah saya, tetapi sekarang saya tidak menggunakan skrip itu sampai saya tahu lebih banyak detail tentang bagaimana 12,04 bekerja dengan masalah ini.

export http_proxy=http://proxyserver:port/\nexport ftp_proxy=http://proxyserver:port/\n
