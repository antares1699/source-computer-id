Saya suka penyorotan sintaks vim. Ini bekerja dengan sangat baik pada unified diffs, jadi saya sering menggunakannya seperti itu: 

diff -pu old new | vim -\n

Saya mengetahui satu tip untuk gunakan vim sebagai lebih baik kurang , tapi saya benar-benar ingin cara untuk hanya mengeluarkan keluaran berwarna (tanpa paging). Saya yakin itu bisa dilakukan, tapi bagaimana caranya?

","source":"https://superuser.com/questions/361874/using-vim-as-a-filter-to-colorize-command-line-output","author":"F'x","replies":[],"answers":[{"answer":"

Jika Anda menginginkan keluaran ANSI, saya tidak tahu cara untuk mencapainya, tetapi Vim memiliki perintah: TOhtml yang telah saya gunakan di salah satu Makefiles saya (disederhanakan di sini): 

vim -gf --noplugin -c 'if has(\"gui_running\") | stop | endif' \\\n    -c 'runtime syntax/2html.vim' \\\n    -c 'w foo.html' -c 'qa!' foo\n

Baris pertama yang menjalankan "stop" jika GUI tidak benar-benar menghentikan gVim, itu hanya meminimalkannya. Baris kedua dengan perintah "runtime" pada dasarnya adalah: TOhtml â€” Saya tidak ingat lagi mengapa saya memilih melakukannya dengan cara itu daripada menggunakan: TOhtml. Baris ketiga menulis "foo.html" dan berhenti, dan juga menyertakan file asli pada baris perintah, yaitu "foo" dalam kasus ini, dan Anda mungkin dapat mengubahnya menjadi - sebagai gantinya.
