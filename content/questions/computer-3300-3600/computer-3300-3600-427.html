Di suatu tempat saya membaca bahwa untuk mengakses mesin jarak jauh melalui SSH kita perlu menyalin ~/.ssh/id_rsa.pub dari mesin lokal ke ~/.ssh/authorized_keys dari mesin jarak jauh. Saya melakukan ini dan saya dapat mengakses mesin jarak jauh melalui ssh.

\n\n

Saya ingin mesin jarak jauh dapat diakses dari beberapa server, seperti: Mesin A dapat mengakses mesin B, dan ada mesin C lain yang juga dapat mengakses mesin B.

\n\n

Saya menyalin mesin A ~/.ssh/id_rsa.pub ke mesin B ~/.ssh/authorized_keys dan saya dapat mengakses mesin B.

\n\n

Tetapi saya juga ingin mengakses mesin B dari mesin C lalu apa yang harus saya lakukan? Maksud saya, di mana saya harus menyalin ~/.ssh/id_rsa.pub mesin C ke mesin B sehingga saya dapat mengakses mesin B dari mesin C?

File authorized_keys pada host jarak jauh dapat berisi lebih dari satu kunci publik. Tambahkan saja - pastikan setiap kunci mendapat baris baru.

\n

Untuk menyederhanakan proses ini, ada ssh-copy-id(1) . 

ssh-copy-id user@machine-B\n

Anda dapat menjalankan perintah di atas pada Mesin C. Ini akan menyalin kunci publik default ke Mesin B dan menambahkannya ke authorized_keys secara otomatis. Anda juga dapat menentukan kunci publik lain untuk disalin dengan menggunakan opsi -i jika Anda mau.

\n
\n

Berikut referensinya di manual :

\n
\n

Pengguna membuat pasangan kuncinya dengan menjalankan ssh-keygen(1). Ini menyimpan kunci pribadi di […] ~/.ssh/id_rsa.pub (protokol 2 RSA) di direktori home pengguna. Pengguna kemudian harus menyalin kunci publik ke ~/.ssh/authorized_keys di direktori home-nya di mesin jarak jauh.

\n

File authorized_keys […] memiliki satu kunci per baris, meskipun barisnya bisa sangat panjang. Setelah ini, pengguna dapat masuk tanpa memberikan kata sandi.
