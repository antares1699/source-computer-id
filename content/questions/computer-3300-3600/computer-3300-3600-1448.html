Setelah mengkloning VM CentOS saya di Virtualbox (saya menggunakan opsi Reinitialize the MAC address of all network cards), VM asli dan klon keduanya sekarang memiliki 127.0.0.1 sebagai IP mereka.

\n\n

Ada yang tahu bagaimana cara memperbaikinya?\nIP vbox dulu mengikuti IP jaringan saya (yaitu, jika host adalah 192.168.0.2 maka vboxnya adalah 192.168.0.3 tetapi sekarang vboxnya adalah 127.0.0.1)\nSaya menggunakan Bridged Adapter BTW.

\n\n

EDIT:\nOke, sekarang saya menyadari eth0 tidak berjalan saat start up. Pertanyaan saya sekarang adalah bagaimana cara mengaktifkan eth0 saat startup? Perhatikan bahwa ini digunakan untuk memulai secara otomatis sebelum saya melakukan klon.

CentOS mengikat skrip konfigurasi jaringan ke alamat MAC tertentu, jadi ketika Anda mengubah alamat MAC Anda, pada dasarnya Anda merusak skrip tersebut. Anda dapat memperbaikinya dengan mengedit skrip sebagai root: 

sudoedit /etc/sysconfig/network-scripts/ifcfg-eth0\n

Dalam skrip cari baris HWADDR baris. Perbarui nilainya agar sesuai dengan alamat MAC baru Anda dan kemudian reboot.
