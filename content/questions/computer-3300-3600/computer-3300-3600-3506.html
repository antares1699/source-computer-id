Saya menyadari bahwa saya tidak lagi dapat terhubung ke server web di x.x.202.50. Kemarin saya telah mengubah file konfigurasi ssh berikut: /share/homes/admin/.ssh/config dengan menambahkan pengaturan berikut: 

# ssh (secure shell) configuration file\nHost webserver\n    HostName x.x.212.50\n    User user1\n    IdentityFile ~/.ssh/id_rsa `\n

Alasan saya melakukan ini adalah untuk mengaktifkan login kunci untuk tujuan sinkronisasi (dengan Unison).

\n\n

Sekarang, ketika saya mencoba untuk menyambung ke server saya menerima kesalahan berikut:

\n\n
\n

Pemilik atau izin buruk di /share/homes/admin/.ssh/config

\n
\n\n

Saya membuat koneksi dengan Putty dari Windows 10 dan dari server Linux ke server Linux lain.

\n\n

Saya perlu terhubung ke server karena saya terlibat dalam sebuah proyek, dan saya tidak tahu bagaimana melakukannya. Apakah ada yang tahu bagaimana melakukan ini?

Aturan umum untuk file yang dapat mempengaruhi keamanan (kunci pribadi, file konfigurasi, kunci resmi) adalah bahwa mereka tidak boleh ditulis oleh orang lain selain pemiliknya (kunci pribadi tidak boleh dapat diakses!).

\n\n

Kesalahan berasal dari kode openssh di bawah: 

if (((sb.st_uid != 0 && sb.st_uid != getuid()) ||\n    (sb.st_mode & 022) != 0))\n      fatal(\"Bad owner or permissions on %s\", filename);\n

Jadi menerjemahkannya ke dalam bahasa Inggris, berarti file konfigurasi harus dimiliki oleh root atau oleh pengguna yang menjalankan ssh dan tidak dapat ditulis oleh grup mana pun atau pengguna lain.

\n\n

Seperti yang telah ditunjukkan di komentar, Anda mungkin memberikan izin ini kepada seseorang, jadi menghapus izin ini akan memperbaiki masalah itu: 

chmod go-w /share/homes/admin/.ssh/config\n
