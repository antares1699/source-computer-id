Lihat konfigurasi Anda jika variabel bash atau opsi shell disetel:

\n

Variabel Bash

\n

( Bagian 5.2 di Bash Reference Manual)

\n
\n

IGNOREEOF

\n

Mengontrol tindakan shell saat menerima karakter EOF sebagai input tunggal. Jika disetel, nilainya menunjukkan jumlah karakter EOF berturut-turut yang dapat dibaca sebagai karakter pertama pada baris masukan sebelum shell akan keluar. Jika variabel ada tetapi tidak memiliki nilai numerik (atau tidak memiliki nilai) maka default adalah 10 . Jika variabel tidak ada, maka EOF menandakan akhir dari input ke shell. Ini hanya berlaku untuk shell interaktif.

\n
\n

Misalnya IGNOREEOF=2

\n
\n

Memodifikasi Perilaku Shell: Set bawaan

\n
\n

set memungkinkan Anda untuk mengubah nilai opsi shell

\n
\n

( Bagian 4.3.1 di Bash Reference Manual)

\n
\n

-o ignoreeof

\n

Shell interaktif tidak akan keluar setelah membaca EOF.

\n
\n
\n

Mendapatkan pertanyaan Anda

\n
\n

Saya ingin mengontrol keduanya secara independen, jika memungkinkan (nonaktifkan fungsi ini untuk shell non-login, misalnya). Apakah itu mungkin?

\n
\n

Iya. Skrip yang harus Anda sumber periksa apakah shell adalah shell login dan atur variabel sesuai, seperti ini: 

if shopt -q login_shell; then IGNOREEOF=10; else IGNOREEOF=0; fi\n

Perhatikan nilai 10 masih memungkinkan Anda untuk meninggalkan shell Ctrl + D ; Anda hanya perlu 11 kali berturut-turut.
