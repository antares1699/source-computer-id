Kamu tidak bisa melakukan itu.

\n\n

Harus terlihat sangat jelas bahwa aplikasi pra-boot apa pun, seperti program boot loader untuk sistem operasi, terkait erat dengan firmware mesin; seperti program aplikasi biasa yang terikat dengan sistem operasi yang layanannya digunakan. Program boot loader EFI tidak dapat dijalankan pada mesin kecuali mesin tersebut menyediakan layanan firmware EFI. Program bootstrap loader gaya PC / AT lama tidak dapat dijalankan pada mesin kecuali mesin tersebut menyediakan layanan firmware PC / AT yang lama.

\n\n

Kata kerja chainloader di GRUB mewujudkan ini. Itu tidak didokumentasikan, tetapi itu melakukan hal-hal yang berbeda tergantung dari bagaimana GRUB dihosting - yaitu firmware mesin apa yang GRUB (harapkan akan) berjalan. Pada versi GRUB yang dihosting pada sistem PC / AT lama, GRUB mengharapkan untuk diberikan kumpulan blok disk atau file, dan memuat dan menjalankan (sektor pertama) apa yang diberikan dengan cara PC lama / AT VBR program bootstrap. Pada versi GRUB yang dihosting EFI, ia mengharapkan untuk diberi nama file, dan memuat serta menjalankan file sebagai aplikasi EFI biasa.

\n\n

Anda memiliki versi GRUB yang dihosting EFI. Anda telah meneruskan +1 hingga chainloader sebagai nama file, tetapi itu bukan jalur yang valid ke file gambar aplikasi EFI pra-boot. GRUB tidak dapat membuat file Jalur perangkat EFI untuk file gambar, karena Anda telah memberinya jalur yang salah secara sintaksis. Karenanya pesan kesalahannya.

\n\n

Penggunaan chainloader yang benar dalam GRUB yang dihosting EFI akan menamai file. Untuk Microsoft Windows NT 6.1, ini akan menjadi versi EFI dari Microsoft's Boot Manager yang ada di samping penyimpanan BCD di Partisi Sistem EFI: 

chainload (hd1,gpt1)/EFI/Microsoft/Boot/bootmgfw.efi

\n\n

Tentu saja, ini tidak akan berhasil untuk Anda, karena Anda tidak menginstal Windows NT 6.1 pada mesin ini - tetapi pada mesin lain dan mentransfer hard disk ke yang ini - atau tidak menginstal versi EFI-bootstrappable darinya , entah bagaimana membujuk Windows NT 6.1 untuk menjalankan versi non-EFI dari utilitas penginstalannya pada mesin EFI Anda. Bagaimana aku tahu? Karena Microsoft tidak akan membiarkan Anda menginstalnya ke hard disk yang tidak dipartisi EFI pada mesin EFI seperti yang telah Anda lakukan, dan karena Anda memiliki keduanya yang tepat Partisi Sistem EFI (partisi FAT pada hard disk kedua Anda) dan setara dengan Poor Man ketika Anda hanya membutuhkan yang pertama jika tidak.

\n\n

Dengan demikian, Windows NT 6.1 belum diinstal dengan versi Boot Manager Microsoft yang dihosting EFI, penyimpanan BCD berada di tempat yang salah, dan akan ada beberapa masalah lain - terkait dengan Windows NT yang mengharapkan satu kombinasi perangkat keras / firmware dan telah dikonfigurasi dengan driver dan pengaturan yang sesuai untuk itu, dan tiba-tiba di-bootstrap pada kombinasi perangkat keras / firmware lain - disimpan untuk Anda nanti.
