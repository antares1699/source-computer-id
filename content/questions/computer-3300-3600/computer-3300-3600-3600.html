Galat WMI adalah masalah yang diketahui:

\n\n

Dari Microsoft KB2545227 :

\n\n
\n

Sebab:

\n \n

Ini berasal dari proses pembuatan DVD / ISO Windows 7 SP1. Sana\n adalah masalah dalam proses pembuatan yang menyebabkan pendaftaran WMI ke\n tetap di DVD / ISO. Karena pendaftaran dirancang hanya untuk bekerja\n selama proses pembuatan DVD / ISO, gagal dijalankan pada sistem langsung\n dan menyebabkan peristiwa ini. Peristiwa ini tidak menunjukkan masalah apa pun\n dalam sistem dan dapat diabaikan dengan aman. Bagaimanapun Anda ingin\n mencegah acara ini dibuat dan ingin menghapusnya\n pendaftaran WMI tertentu secara manual, silakan ikuti langkah-langkah yang disebutkan\n di artikel ini untuk menjalankan skrip solusi.

\n
\n\n

Mereka menawarkan FixIt untuk memperbaiki masalah, ditambah instruksi manual:

\n\n
    \n

  1. Di Notepad buat dokumen baru bernama Workaround.txt

    2. \n

  2. Salin skrip berikut ke notepad:

    3. \n
\n\n
strComputer = \".\"\n\nSet objWMIService = GetObject(\"winmgmts:\" _\n& \"{impersonationLevel=impersonate}!\\\\\" _\n& strComputer & \"\\root\\subscription\")\n\nSet obj1 = objWMIService.ExecQuery(\"select * from __eventfilter where name='BVTFilter' and query='SELECT * FROM __InstanceModificationEvent WITHIN 60 WHERE TargetInstance ISA \"\"Win32_Processor\"\" AND TargetInstance.LoadPercentage > 99'\")\n\nFor Each obj1elem in obj1\n\nset obj2set = obj1elem.Associators_(\"__FilterToConsumerBinding\")\n\nset obj3set = obj1elem.References_(\"__FilterToConsumerBinding\")\n\nFor each obj2 in obj2set\n\n WScript.echo \"Deleting the object\"\n\n WScript.echo obj2.GetObjectText_\n\n obj2.Delete_\n\nnext\n\nFor each obj3 in obj3set\n\n WScript.echo \"Deleting the object\"\n\n WScript.echo obj3.GetObjectText_\n\n obj3.Delete_\n\nnext\n\nWScript.echo \"Deleting the object\"\n\nWScript.echo obj1elem.GetObjectText_\n\nobj1elem.Delete_\n\nNext\n
    \n

  1. Simpan teks sebagai Workaround.vbs

    2. \n

  2. Tutup Notepad

    3. \n

  3. Buka prompt perintah yang ditinggikan:

    \n\n

    Sebuah. Klik mulai

    \n\n

    b. Klik Program

    \n\n

    c. Klik kanan pada Prompt Perintah

    \n\n

    d. Pilih jalankan sebagai administrator

    4. \n

  4. Ubah Direktori ke yang berisi workaround.vbs (Contoh CD c:\\users\\%username%)

    5. \n

  5. Jalankan cscript workaround.vbs

    6. \n
\n\n

Setelah menjalankan skrip, kesalahan ID Peristiwa 10 yang terkait dengan peristiwa ini akan berhenti terjadi. Ini tidak menghapus salah satu entri yang ada di log peristiwa, mereka harus dibersihkan secara manual dari log peristiwa aplikasi.

\n\n

Untuk masalah "Layanan NEWDRIVER gagal":

\n\n

Anda memiliki layanan bernama "NEWDRIVER" yang terdaftar dengan sistem, tetapi file yang dibutuhkan untuk menjalankannya hilang. Karena file hilang, dan komputer Anda berperilaku baik, Anda dapat menghapus referensi ke layanan tersebut.

\n\n

Anda harus dapat melakukan ini dari prompt perintah yang ditinggikan, dengan perintah sc (Kontrol Layanan). Misalnya.:

\n\n

sc delete NEWDRIVER
