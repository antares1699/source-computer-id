Ini bekerja seperti yang diharapkan 

$ echo \"foo\" > foo\n$ cat foo                # foo\n$ echo \"updated\" > foo\n$ cat foo                # updated\n
\n\n

Dari man set 

-C      If  set, bash does not overwrite an existing file with the\n        >, >&, and <> redirection operators.  This may be overridden\n        when creating output files by using the redirection operator\n        >| instead of >.\n

$ set -C\n$ echo \"bar\" > bar\n$ cat bar                # bar\n$ echo \"updated\" > bar\nfile exists: bar\n

Ini semua baik dan bagus, tapi sekarang saatnya mematikan set -C.

Sepertinya Anda menggunakan Z shell , tapi kutip pesta halaman manual.

\n\n

Sekadar klarifikasi, perintah set -C dan set +C juga berfungsi di zsh, tetapi yang lebih mudah diingat adalah IMHO opsi shell : 

setopt no_clobber\n

Ini sama dengan set -C. Dan, untuk menonaktifkan opsi ini, memang ada unsetopt bawaan: 

unsetopt no_clobber\n

Semua opsi terdaftar dengan man zshoptions. Opsi agak simetris, jadi setopt no_clobber setara dengan unsetopt clobber. Itulah sebabnya, di halaman manual, MENGKRITIK opsi dijelaskan:

\n\n
\n

CLOBBER (+ C, ksh: + C)

\n \n

Izinkan pengalihan > untuk memotong file yang ada, dan >> untuk membuat file. Jika tidak, >! atau >| harus\n digunakan untuk memotong file, dan >>! atau >>| untuk membuat file.

\n
\n\n

Opsi terkait (IMHO sangat berguna) adalah

\n\n
\n

HIST_ALLOW_CLOBBER

\n \n

Tambahkan | ke pengalihan keluaran dalam riwayat. Hal ini memungkinkan referensi riwayat ke file clobber bahkan saat CLOBBER tidak disetel.

\n
\n\n

Demo: 

zsh$ setopt no_clobber hist_allowclobber\nzsh$ echo foo > baz\nzsh$ echo bar > baz\nzsh: file exists: baz\nzsh$ [Arrow UP]\nzsh$ echo bar >| baz\nzsh$\n

Dan sebagai komentar terakhir, opsi tidak peka huruf besar / kecil dan garis bawah diabaikan, karenanya NOCLOBBER, NO_clobber, >>| diperlakukan dengan cara yang sama.
