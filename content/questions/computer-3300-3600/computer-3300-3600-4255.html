Dugaan saya adalah ketika Anda memilih lagu untuk diputar di iTunes, iTunes akan mendorong hampir semua data mentah untuk lagu tersebut ke dalam DMA. Karena cara perangkat I / O berbicara dengan CPU, setiap perangkat memiliki alamat memorinya sendiri. Setelah CPU selesai mendorong data ke alamat itu, perangkat I / O dapat menggunakan data di sana tanpa Anda harus dirawat oleh CPU. Jadi iTunes memindahkan data untuk lagu tersebut ke alamat memori perangkat suara dan perangkat suara mulai memutarnya. Saat CPU terkunci saat Anda memainkan lagu, perangkat suara tidak menyadari bahwa CPU berhenti, dan tetap menggunakan data dari alamat memorinya, memutar lagu.
