Saya mencoba membuat lab untuk mendemonstrasikan mekanisme serangan dan deteksi ipv6 di Macbook saya. Ini adalah pengaturan saya dengan virtualbox:

\n\n
    \n

  • Host OS: OS X 10.8.2 IPv6 secara otomatis, koneksi nirkabel Geust

    • \n

  • OS: Ubuntu 8.04 IPv6 aktifkan, jaringan yang dijembatani

    • \n
\n\n

Semuanya bekerja. Saya menguji ping6 untuk melihat apakah tuan rumah dan tamu dapat dihubungi satu sama lain. Jadi saya mengaktifkan wireshark di OS host, dan melihat apakah saya dapat menangkap lalu lintas saat OS tamu melakukan ping ke sana. Sayangnya, saya tidak melihat lalu lintas yang datang dari OS tamu dan tidak yakin mengapa. Karena OS tamu menggunakan jaringan yang dijembatani, ini seharusnya menanyakan rute untuk menemukan siapa yang memiliki alamat ipv6, jadi saya yakin kartu jaringan fisik terlibat di dalamnya. Tetapi saya tidak yakin mengapa wireshark tidak dapat menangkap lalu lintas apa pun

Tampaknya, VirtualBox sendiri dapat menulis semua paket ke atau dari tamu ke file pcap , jadi itu mungkin salah satu cara untuk melakukan ini.

\n\n

Di Jawaban Wes Morgan untuk pertanyaan VirtualBox serupa , dia mengatakan bahwa mode yang dijembatani adalah "melewati tumpukan jaringan OS host sepenuhnya", yang mungkin berarti bahwa paket antara host dan tamu tidak dilihat oleh BPF di OS X dan karenanya tidak tersedia untuk Wireshark (yang menggunakan libpcap, yang menggunakan BPF di OS X).
