1) Instal CNTLM ke direktori default (di Windows, C: \\ Program Files (x86) \\ cntlm \\).

\n\n

2) Jalankan cmd.exe

\n\n

3) Jenis: cd "C: \\ Program Files (x86) \\ cntlm \\"

\n\n

4) Jenis: cntlm.exe -H -d your_domain -u your_username

\n\n

Ini akan menanyakan kata sandi Anda. Masukkan kata sandi Anda dan cntlm akan memberi Anda beberapa hash.\nSesuatu seperti ini: 

c:\\Program Files (x86)\\Cntlm>cntlm -H -d your_domain -u your_username\nPassword:\nPassLM          4E9C185900C7CF0B6FFCB2044F81920C\nPassNT          6E9F120B83EEA0E875CE8E6F9730EC9A\nPassNTLMv2      2A0B7C2457FB7DD8DA4EB737C4FA224F  \n

Sekarang Anda memiliki hash kata sandi. Simpan ke editor teks.

\n\n

5) Jenis: cntlm -M http://www.google.com

\n\n

Sekali lagi masukkan kata sandi Anda.\nIni akan memberi Anda sesuatu seperti itu 

c:\\Program Files (x86)\\Cntlm>cntlm -M http://www.google.com\nPassword:\nConfig profile  1/4... Credentials rejected\nConfig profile  2/4... OK (HTTP code: 302)\n----------------------------[ Profile  1 ]------\nAuth            NTLM\nPassNT          6E9F120B83EEA0E875CE8E6F9730EC9A\nPassLM          4E9C185900C7CF0B6FFCB2044F81920C\n------------------------------------------------\n

Sekarang Anda melihat bahwa profil 2 berhasil. Karena dikatakan OK untuk profil 2. Ini mungkin berbeda pada sistem Anda.

\n\n

Triknya adalah,

\n\n
    \n
  • jika Auth adalah NT, maka Anda harus menggunakan PassNT saja
    • \n
  • jika Auth adalah LM, maka Anda harus menggunakan PassLM saja
    • \n
  • jika Auth adalah NTLM, maka Anda harus menggunakan PassLM dan PassNT
    • \n
  • jika Auth adalah NTLMv2, maka Anda harus menggunakan PassNTLMv2 saja
    • \n
\n\n

Sekarang kami mendapatkan semua yang kami inginkan.\nUntuk konfigurasi saya, Auth mengatakan NTLM jadi saya akan menggunakan PassNT dan PassLM di file konfigurasi cntlm.ini.

\n\n

Ini adalah contoh file konfigurasi sesuai dengan jawaban yang diberikan: 

#\n# Cntlm Authentication Proxy Configuration File\n#\n\nUsername yourusername\nDomain yourdomain\n\nAuth NTLM\nPassNT 6E9F120B83EEA0E875CE8E6F9730EC9A\nPassLM 4E9C185900C7CF0B6FFCB2044F81920C\n\nWorkstation yourhostname.yourdomain\n\n# Most probably proxy.yourdomain:8080\nProxy  yourProxyIP:yourProxyPort\n\nNoProxy  localhost, 127.0.0.*, 10.*, 192.168.*\n\nListen  3132\n\nGateway yes\n

Sekarang Anda dapat menggunakan alamat IP dan porta 3132 komputer Anda sebagai proxy.

\n\n

Catatan 1: Jangan gunakan domain \\ nama pengguna sebagai nama pengguna\nNote 2: Jika ada maka jangan lupa gunakan extension '.intra' setelah domain anda (yourdomain.intra)

sudo cntlm -M http://www.google.com\n

Auth         LM\nFlags        0x06820000\n

sudo service cntlm restart\n
