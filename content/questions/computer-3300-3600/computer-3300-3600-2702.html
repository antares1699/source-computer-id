Saya telah mengunduh pemasang luring (konsumen, bukan paket msi) untuk Google Chrome. Tetapi ketika saya menjalankannya, ia mencoba menghubungi server Google untuk memeriksa versi yang lebih baru. Jika menemukannya, itu akan mengunduh yang pertama, dan kemudian menginstal versi itu. Itu tidak menggunakan versi yang sudah ada, sebagai bagian dari file EXE. Bagi saya, itu mengalahkan poin memiliki penginstal offline. Saya menyimpan gudang semua perangkat lunak gratis dan yang dibeli secara lokal, sehingga saya dapat menginstalnya dengan cepat kapan pun saya perlukan. Tetapi karena semakin banyak orang mendapatkan akses ke koneksi Internet broadband, banyak pengembang dan perusahaan mendorong pemasang offline dengan rilis baru. Banyak judul perangkat lunak, seperti Google Chrome, hampir secara eksklusif didistribusikan dengan pemasang online.

Bagaimana cara memasang Chrome sebagaimana adanya, tanpa memeriksa dan mendapatkan versi yang lebih baru secara otomatis?

\n\n

Akan Sakelar perintah Chromium bekerja dengan pemasang Chrome? Apakah ada tombol untuk menonaktifkan pemeriksaan versi saat menjalankan penginstal?

\n\n

Memperbarui:

\n\n

Pertanyaan "duplikat" yang ditautkan ke bawah menunjukkan File Hippo sebagai kemungkinan tempat penyimpanan di mana Anda bisa mendapatkan versi Chrome sebelumnya. Mereka memang memiliki daftar yang sehat dari hampir setiap versi Chrome yang pernah dirilis di Windows, baik versi beta, dev, dan stabil. Sayangnya mereka tidak menghosting file pemasang, mereka hanya mengarahkan ke server Google untuk mengunduh, yang hanya mengeluarkan pemasang untuk versi terbaru Chrome. Bukan hanya File Hippo, saya juga mencoba mengunduh Chrome versi lama dari Unduhan CNET, dan mereka juga mengarahkan Anda ke server Google untuk diunduh.

\n\n

Untuk alasan apa pun (mungkin terkait keamanan), Google tidak ingin siapa pun menginstal Chrome versi lama di sistem apa pun. Tampaknya bahkan bukan sumber umum Anda untuk mengunduh perangkat lunak versi lama, seperti Unduhan CNET, bahkan mereka tidak diizinkan untuk menghosting file pemasang Chrome di server mereka sendiri. Pengalihan ke Google mungkin ditentukan oleh Google.

\n\n
\n

Google adalah seorang diktator!

\n
\n\n

Mereka bahkan tidak mengizinkan penginstal offline mandiri lengkap untuk diinstal apa adanya, tanpa memeriksa versi yang lebih baru. Saya mencoba menjalankan pemasang dari cmd dengan beberapa sakelar baris perintah yang digunakan untuk Chromium. Saya hanya berakhir dengan kesalahan.

\n\n

Saya belum menemukan daftar lengkap sakelar yang dapat Anda gunakan. Satu-satunya yang saya temukan sejauh ini adalah /update tetapi itu hanya melakukan hal yang sebaliknya. Jadi saya akan menyerah di sini dan kembali ke Firefox yang terpercaya.

Menonaktifkan keinginan Google untuk memperbarui sebenarnya jauh lebih kompleks dari yang seharusnya. Langkah-langkah yang harus Anda ambil:

\n\n
    \n
  • Instal MSI dengan parameter "NOGOOGLEUPDATING", jika memungkinkan
    • \n
  • Hentikan tugas pembaruan yang sedang berjalan (yang biasanya dimulai selama penginstalan) dan JANGAN membuka Chrome sampai Anda menyelesaikan langkah-langkah berikut.
    • \n
  • Hapus folder update secara manual dan hapus "GoogleUpdate.exe"
    • \n
  • Setel layanan Google Update "gupdate" dan "gupdatem" ke "nonaktif"
    • \n
  • Nonaktifkan tugas terjadwal untuk GoogleUpdate (perhatikan jika Anda menghapusnya, tugas tersebut kemungkinan besar akan diganti; menonaktifkannya akan menghalangi penggantian yang coba didorong oleh Google)
    • \n
  • Setel kunci registri untuk menonaktifkan pembaruan (atau unduh template admin Google GPO dan setel kunci melalui GPO.)
    • \n
  • Periksa kunci registri tambahan di bawah HKLM | Perangkat Lunak | Google | Perbarui dan hapus semua entri yang terkait dengan "klien"
    • \n
\n\n

Ada dokumen NSA yang merinci bagaimana benar-benar menonaktifkan pembaruan, di https://www.iad.gov/iad/library/ia-guidance/security-configuration/applications/deploying-and-securing-google-chrome-in-a-windows-enterprise.cfm tetapi tidak termasuk hasil edit registri terakhir yang saya sebutkan di atas.

\n\n

Harap perhatikan bahwa jika Anda memasang Chrome dan menjalankannya sebelum menonaktifkan semua tempat yang diperlukan untuk menonaktifkan pembaruan, itu sudah ada di versi baru sebelum Anda mendapat kesempatan untuk melakukan apa pun.

\n\n

Selain itu, Google telah berusaha keras untuk menghapus semua kecuali versi terbaru dari semua repositori, jadi Anda harus mencari seseorang yang memiliki MSI yang lebih lama.
