Karena saya tidak memiliki mesin fotokopi atau pemindai, saya menggunakan kamera 8 megapiksel untuk menyalin dokumen. Ini bekerja dengan cukup baik kecuali mereka membutuhkan banyak pemrosesan sesudahnya. Saya ingin beralih dari foto ke bitmap, tetapi menggunakan 

djpeg -grayscale -pnm photo.jpg |\npgmtopbm -threshold -value XXX\n

tidak bekerja dengan baik, karena dua alasan:

\n\n
    \n

  1. Sulit menebak XXX seharusnya, dan XXX berbeda untuk foto yang berbeda.

    2. \n

  2. Iluminasi bervariasi, dan terkadang satu ambang batas bukanlah yang tepat untuk gambar.

    3. \n
\n\n

Bagaimana saya bisa lebih baik? Solusi ideal adalah program baris perintah otomatis yang dapat saya jalankan di Linux. (Saya telah menulis program untuk menghapus piksel gelap dari tepi gambar.)

\n\n

CATATAN: Saya sangat menginginkan file bitmap , itu hanya piksel hitam dan putih. Tidak ada skala abu-abu, tidak ada dithering.

-monochrome

\n\n

Opsi ini menggunakan beberapa dithering cerdas dan menghasilkan keluaran yang sangat terlihat: 

convert -monochrome in.png out.png\n

Dokumentasi: http://www.imagemagick.org/Usage/quantize/#monochrome

\n\n

Bandingkan itu dengan transformasi -threshold 50 yang lebih sederhana: 

convert -threshold 50 in.png out.png\n

yang kehilangan sebagian besar citra.

\n\n

Contoh konkrit dari: https://www.nasa.gov/mission_pages/galex/pia15416.html 

wget -O orig.jpg http://www.nasa.gov/images/content/650137main_pia15416b-43_full.jpg\n# Downsize to 400 height to have a reasonable file size for upload here.\nconvert orig.jpg -resize x400 in.jpg\nconvert -monochrome in.jpg out.jpg\nconvert -threshold 50 in.jpg threshold-50.jpg\n

in.jpg

\n\n

\n\n

\n\n
