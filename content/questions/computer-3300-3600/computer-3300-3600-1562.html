NTFS telah menjadi lebih dari sebuah filesystem "journal" yang tampaknya selama masa pakai Windows XP dan telah mengadopsi banyak fitur yang membuat chkdsk tidak diperlukan. Bersama-sama, teknologi ini berarti bahwa menjalankan chkdsk sebagian besar tidak diperlukan karena status sistem file jauh lebih mudah dan lebih cepat (misalnya, dapat dilakukan dengan cepat selama boot) untuk menentukan dan memulihkan ke status "yang diketahui baik".

\n

Windows Vista memperkenalkan Transactional NTFS yang tampaknya pada prinsipnya serupa dengan filesystem journal di mana kejadian jenis check-disk tertentu seharusnya jarang atau tidak perlu. Pada dasarnya, ketika sistem file dipasang, mudah bagi pengemudi untuk mengetahui apa yang terjadi sebelum listrik mati dan dengan sangat cepat menentukan apakah akan memutar mundur, memutar maju atau hanya membuang perubahan.

\n

(Saya yakin perubahan besar muncul di Vista karena saya belum melihat chkdsk berjalan saat boot sejak pindah dari XP)

\n

Seperti yang disebutkan dalam komentar ada kasus di mana sistem ini tidak melindungi Anda dan sangat mungkin sistem Anda menjadi korup ke titik di mana chkdsk aku s diperlukan, terutama saat menggunakan sistem file yang tidak mendukung fitur lanjutan seperti Sistem file FAT digunakan pada memory stick Flash. Fitur-fitur ini ada untuk mengurangi seberapa sering chkdsk diperlukan selama boot dengan membuatnya lebih mudah untuk menentukan status sistem file sebelum mati tiba-tiba.

Log NTFS

\n

NTFS adalah sistem file Penjurnalan dan menggunakan NTFS Log ($ LogFile) untuk merekam perubahan metadata ke volume.

\n

Ini adalah fungsionalitas penting NTFS (fitur yang tidak disediakan FAT / FAT32) untuk memastikan bahwa struktur data kompleks internal (terutama bitmap alokasi volume, atau pemindahan data yang dilakukan oleh API defragmentasi, modifikasi pada catatan MFT seperti pemindahan) dari beberapa atribut dengan panjang variabel yang disimpan dalam catatan MFT dan daftar atribut), dan indeks (untuk direktori dan deskriptor keamanan) akan tetap konsisten jika terjadi kerusakan sistem, dan memungkinkan rollback yang mudah dari perubahan yang tidak terikat pada struktur data penting ini ketika volume dipasang kembali .

\n

NTFS transaksional

\n

Pada Windows Vista, aplikasi dapat menggunakan Transactional NTFS untuk mengelompokkan perubahan file menjadi satu transaksi. Transaksi akan menjamin bahwa semua perubahan terjadi, atau tidak ada yang terjadi, dan itu akan menjamin bahwa aplikasi di luar transaksi tidak akan melihat perubahan sampai mereka berkomitmen.

\n

Ini menggunakan teknik yang sama seperti yang digunakan untuk Volume Shadow Copies (yaitu copy-on-write) untuk memastikan bahwa data yang ditimpa dapat diputar kembali dengan aman, dan log CLFS untuk menandai transaksi yang masih belum dilakukan, atau yang telah dilakukan. berkomitmen tetapi masih belum diterapkan sepenuhnya (jika terjadi kerusakan sistem selama komit oleh salah satu peserta).
