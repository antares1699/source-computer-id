Saya telah menerima baris ini: 

root:$1$25A116L54$7/IbiL1ZsUyPwYBx/zuNv/\n

dan diminta untuk memecahkan kata sandi. Jadi, saya membuat file crack.hash dan meletakkan yang berikut ini di dalamnya: 

$1$25A116L54$7/IbiL1ZsUyPwYBx/zuNv/\n

Dan kemudian jalankan perintah berikut di terminal: 

hashcat -m 0 -a 0 -o /crackResult.txt --remove /crack.hash /passwords.txt\n

Dan hasilnya adalah: 

skip line: $1$25A116L54$7/IbiL1ZsUyPwYBx/zuNv/ (line length exception)\n

Apa kesalahan saya?

","source":"https://superuser.com/questions/657659/line-length-exception-in-hashcat-kali-linux","author":"Amin Saqi","replies":[{"reply":"coba dengan -m 500","author":"Jor-el"},{"reply":"Juga kesalahan ejaan dalam kesalahan mereka (panjang harus panjang) XD","author":"DarkMantis"}],"answers":[{"answer":"

Ini terlihat seperti string kata sandi unix. 

-m 0 is for MD5 hashes\n

Menggunakan 

-m 500 for Unix hashes\n

Lihat https://hashcat.net/wiki/doku.php?id=hashcat
