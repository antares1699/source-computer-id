Saya memiliki beberapa file log yang berisi beberapa byte yang bukan karakter yang dapat ditampilkan (karena tanda tangan biner pada permintaan).

\n\n

Setiap kali saya membukanya dengan lebih sedikit saya ditanya: 

$ less /var/log/file.log       \n\"/var/log/file.log\" may be a binary file.  See it anyway? \n

Bagaimana cara mengurangi lewati langkah ini dengan opsi atau konfigurasi baris perintah?

","source":"https://superuser.com/questions/805711/how-do-i-make-less-open-binary-files-without-confirmation","author":"giorgiosironi","replies":[],"answers":[{"answer":"

Gunakan opsi -f: 

-f or --force\n              Forces non-regular files to be opened.  (A non-regular file is a directory or\n              a device special file.)  Also suppresses the warning message  when  a  binary\n              file  is  opened.   By  default,  less will refuse to open non-regular files.\n              Note that some operating systems will not allow directories to be read,  even\n              if -f is set.\n

Anda dapat menambahkan opsi ini ke variabel lingkungan LESS di .bashrc untuk menjalankan less secara default dengan -f. 

export LESS='-f'\n
