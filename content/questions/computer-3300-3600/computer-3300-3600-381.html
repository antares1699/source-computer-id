Bukan ide yang baik untuk menonaktifkan memori virtual. Manajemen virtual adalah proses yang kompleks tetapi secara sederhana Windows hanya menggunakan memori virtual jika tidak ada memori "nyata" untuk dialokasikan, kemudian windows akan menukar item yang kurang digunakan dalam memori ke memori virtual.

\n\n

Jadi jika Anda memiliki cukup memori windows tidak akan menggunakan memori virtual, dan akan menggunakannya hanya jika perlu digunakan. (Ini tidak berarti bahwa itu tidak akan menulis apa pun di memori virtual, tetapi tidak akan menyebabkan penurunan kinerja.)

\n\n

Tetapi ketika Anda menonaktifkan memori virtual dan tidak ada cukup memori untuk aplikasi, maka Anda akan mengalami aplikasi mogok karena tidak cukup memori.

Anda tidak dapat menonaktifkan memori virtual. Windows dan aplikasi Anda selalu berjalan di memori virtual. Alamat yang disajikan ke CPU oleh program dan OS adalah alamat virtual, bukan fisik. Selalu.

\n\n

Menghapus file halaman tidak menonaktifkan memori virtual. (Terminologi dalam dialog Windows menyesatkan.) Yang dilakukannya adalah menonaktifkan paging ke disk untuk jenis memori virtual tertentu yang disebut memori "komitmen pribadi". Hal ini kemungkinan besar akan menyebabkan tipe utama lainnya, memori yang dipetakan, dihalaman lebih dari yang lain. Ini tidak akan meningkatkan kinerja.

\n\n

Saya bukan ahli Windows, dan saya mengatakan ini murni sebagai perspektif pengguna lama Windows.

\n\n

Jika Anda mengaktifkan memori virtual:

\n\n
    \n
  1. File besar (biasanya 1,5 kali ukuran memori fisik) dibuat di direktori root sistem (biasanya, c) drive. File ini tidak dapat dipindahkan ke drive lain. Jika drive sistem Anda adalah SSD kecil, dan mengatakan Anda memiliki memori 16GB, ini akan sangat mengganggu Anda.
    2. \n
  2. Jika Anda meminimalkan program dan memulihkannya setelah waktu yang lama (seperti beberapa jam kemudian), Windows akan banyak membaca drive, dan program menjadi tidak responsif untuk sementara waktu. Mungkin, Windows telah memasukkan semua memori program ke dalam memori virtual dan mencoba memulihkan memori saat diaktifkan.
    3. \n
  3. Memori nyata yang disimpan dari 1 dan 2 di atas dapat digunakan untuk cache file. File yang pernah dibaca dapat dibaca dengan super cepat dari waktu berikutnya. Perhatikan bahwa, Windows melakukan ini apakah memori virtual digunakan atau tidak. Hanya saja akan ada RAM yang lebih nyata untuk ini, jika memori virtual diaktifkan. Anda mungkin atau mungkin tidak memperhatikan efeknya.
    4. \n
\n\n

Jika Anda menonaktifkan memori virtual:

\n\n
    \n
  1. Anda tidak akan mengalami dua gangguan pertama di bagian "aktifkan".
    2. \n
  2. Jika penggunaan RAM Anda mendekati ukuran memori fisik, ya "hampir" tidak sama, Windows akan terus-menerus menampilkan pesan pop-up yang mengatakan bahwa sistem kehabisan memori. Misalnya, jika Anda memiliki RAM fisik 1GB dan Task Manager menunjukkan sekitar 0,5GB RAM digunakan, tetapi Windows mungkin tetap menampilkan peringatan pop-up.
    3. \n
  3. Jika penggunaan RAM Anda benar-benar mendekati ukuran memori fisik, seluruh sistem mungkin menjadi sangat lambat dan tidak responsif. Aplikasi mungkin macet tanpa alasan yang jelas.
    4. \n
\n\n

Selain itu, saya belum mengalami kerugian apa pun dalam menonaktifkan memori virtual. Menurut saya, teknologi memori virtual telah ditemukan sejak lama ketika PC memiliki RAM 8MB atau 16MB (MB bukan GB). Ketika saya masih mahasiswa saya mengambil kelas Sistem Operasi, dan belajar bagaimana memori virtual bekerja. Itu adalah mekanisme umum, bukan khusus Windows. Ini mungkin berguna ketika PC memiliki jumlah RAM yang sangat kecil, tetapi sekarang siapa pun dapat dengan mudah membeli 16GB di desktop seseorang. Saya pikir memori virtual tidak begitu berguna lagi, kecuali jika Anda menggunakan laptop yang memiliki RAM 4GB yang tidak dapat diperluas. Saya pribadi telah menonaktifkannya di PC saya yang memiliki RAM 16GB.

\n\n

Dalam kasus Anda, jika Anda 100% yakin bahwa penggunaan RAM Anda tidak akan pernah melebihi 5GB (karena Anda memiliki 6GB), lanjutkan dan nonaktifkan.
