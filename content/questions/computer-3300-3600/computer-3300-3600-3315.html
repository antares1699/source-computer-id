Firefox biasanya membeku sekitar satu detik setiap kali saya mencari sesuatu dari kotak pencarian toolbar, saat mengirimkan formulir, atau saat mengosongkan riwayat kotak pencarian. Saya menduga itu ada hubungannya dengan fitur pelengkapan otomatis. Menggunakan Resource Monitor Windows 7, masalahnya tampaknya berasal dari file: 

C:\\Users\\<username>\\AppData\\Roaming\\Mozilla\\Firefox\\Profiles\\<profile>\\formhistory.sqlite-journal\n

Saya yakin ini adalah file sementara yang menulis cache database. Tangkapan layar berikut menunjukkan waktu respons yang sangat tinggi dari enam pencarian berbeda, dan panjang antrian di drive C meleset dari skala:

\n\n

\n\n

Profil Firefox saya menggunakan SSD Intel X25-M G2. Masalahnya sepertinya tidak terjadi jika saya membuat profil baru di hard disk drive. Namun, saya ingin tahu mengapa masalah ada pada SSD di tempat pertama (karena ini adalah masalah yang mengganggu yang bertentangan dengan alasan saya membeli SSD, dan itu mungkin terjadi dengan aplikasi lain juga), dan bagaimana mencegahnya. Ini masih terjadi jika Firefox dimulai dalam mode aman, dan dengan versi beta terbaru.

\n\n

Anda mungkin mendapatkan keuntungan dari menyedot debu database sqlite. Perintah ini harus melakukannya (di Linux): 

cd ~/.mozilla/firefox/dasda418.default\nfor i in *.sqlite\ndo\necho 'vacuum;' | sqlite3 $i\ndone\n

dengan dasda418.default adalah direktori profil Firefox Anda yang sebenarnya. Tentunya Anda harus melakukan ini saat Firefox tidak berjalan. Di Windows perintahnya adalah : 

for %i in (*.sqlite) do @echo VACUUM; | sqlite3 %i\n

Ini akan memadatkan database, membuatnya lebih kecil dan mungkin menyelesaikan masalah Anda.
