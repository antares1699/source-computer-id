Cukup edit file /etc/profile Anda dengan ie, nano: 

sudo nano /etc/profile\n

dan di akhir file tambahkan 

http_proxy=\"http://user:password@foo_server:port\"\nhttps_proxy=\"http://user:password@foo_server:port\"\nftp_proxy=\"http://user:password@foo_server:port\"\nexport http_proxy ftp_proxy https_proxy\n

dimana foo_server adalah server proxy Anda, dan Pelabuhan adalah port yang Anda butuhkan untuk terhubung ke server proxy Anda.

\n\n

Untuk memeriksa apakah pengaturan baru sedang digunakan, gunakan tipe source /etc/profile dan Anda dapat memeriksa apakah server sedang digunakan dengan (instal curl jika Anda belum memilikinya) 

curl icanhazip.com\n

yang seharusnya mengembalikan alamat server proxy Anda.

\n\n

Perangkat lunak seperti git, ssh, telnet, akan cenderung mengabaikan pengaturan tersebut dan menggunakan milik mereka sendiri, saya tahu itu there are several utilities untuk ssh yang memungkinkan Anda melakukan itu. Tidak akan ada yang unik, bekerja untuk semua cara melakukannya.

\n\n

Mengenai Lingkungan Konsol

\n\n

Menetapkan berbagai variabel Lingkungan, seperti yang disarankan jawaban lain, adalah cara yang harus dilakukan. Saya juga akan mendefinisikan pasangan huruf besar dari semua variabel karena beberapa aplikasi hanya mengenali satu atau yang lain. Selain itu, saya akan mendefinisikan NO_PROXY untuk host yang tidak ingin Anda proxy, misalnya: 

\n  export no_proxy=\"localhost,127.0.0.1,localaddress,.corporate.com\"\n

Proksi SSH

\n\n

Ini membutuhkan sedikit lebih banyak penyesuaian. Ada alat pihak ketiga yang disebut 'sambungkan' tersedia sini . Homepage-nya adalah sini . Ini akan memungkinkan Anda untuk membuat proxy koneksi SSH melalui proxy HTTPS dan SOCKS. Namun, ini hanya mendukung metode otentikasi dasar.

\n\n

Agar ini berfungsi, Anda harus memodifikasi file ~ / .ssh / config Anda untuk menggunakan perintah 'sambungkan' sebagai perintah proxy. Dengan asumsi server SOCKS perusahaan Anda berjalan di socks.corporate.com pada port 1080, Anda dapat menambahkan opsi ProxyCommand di ~ / .ssh / config, seperti ini: 

\nHost remote.outside.net\n  ProxyCommand connect -S socks.corporate.com %h %p\n

Variabel% h dan% p akan diganti saat menjalankan perintah proxy dengan nama host target dan port yang ditentukan ke perintah SSH.

\n\n

Jika Anda tidak ingin menambahkan entri HOST terpisah untuk setiap host target, Anda juga dapat menggunakan karakter pengganti: 

\n## Outside of the firewall, use connect command with SOCKS conenction.\nHost *\n  ProxyCommand connect -S socks.corporate.com %h %p\n\n## Inside of the firewall, use connect command with direct connection.\nHost *.socks.coporate.com\n  ProxyCommand connect %h %p\n

Demikian pula, jika Anda ingin menggunakan proxy https, gunakan opsi -H daripada opsi -S pada contoh di atas, seperti ini: 

\n## Outside of the firewall, with HTTP proxy\nHost *\n  ProxyCommand connect -H proxy.corporate.com:8080 %h %p\n\n## Inside of the firewall, direct\nHost *.corporate.com\n  ProxyCommand connect %h %p\n

Karena Anda berada dalam jaringan perusahaan, mungkin ide yang bagus untuk memastikan departemen TI Anda setuju dengan Anda membuat proxy SSH. Mungkin mereka bahkan memiliki solusi yang lebih baik dari saya. Jika tidak, Anda mungkin berakhir di daftar hitam dan tidak ada konektivitas sama sekali :)
