Anda dapat melakukan ini menggunakan file Automator alur kerja. Ini sedikit lebih kompleks daripada kebanyakan, jadi berhati-hatilah saat menerapkannya.

\n\n

Posting ini berisi dua versi: Satu lebih pendek dan menyimpan output sebagai Processed PDF File.pdf di desktop, yang lain lebih panjang dan menyimpan file sebagai (Edited) InputFileName.pdf di direktori yang sama. Langkah-langkah yang diperlukan untuk versi yang lebih panjang hanya ditandai (pilihan) .

\n\n

Buka Automator dan pilih untuk membuat yang baru Layanan yang menerima File PDF sebagai masukan Aplikasi apa saja .

\n\n
    \n
  1. (pilihan) Tambah sebuah Tetapkan Nilai Variabel action dan beri nama variabel FilePath.
    2. \n

  2. (pilihan) Tambah sebuah Jalankan AppleScript tindakan dan gunakan kode skrip berikut untuk mendapatkan nama folder tempat file berada: 

    on run {input, parameters}\n    tell application \"Finder\" to return (container of first item of input) as alias\nend run\n
    3. \n

  3. (pilihan) Tambah sebuah Tetapkan Nilai Variabel action dan beri nama variabel Folder.

    4. \n
  4. (pilihan) Tambah sebuah Dapatkan Nilai Variabel tindakan dan mengembalikan variabel FilePath. Abaikan masukan tindakan ini di Pilihan .
    5. \n

  5. (pilihan) Tambah sebuah Jalankan Shell Script tindakan dan masukan lulus sebagai argumen . Gunakan skrip berikut untuk mengekstrak nama dasar file: 

    echo \"$( basename \"$1\" )\"\n
    6. \n

  6. (pilihan) Tambah sebuah Tetapkan Nilai Variabel action dan beri nama variabel FileName.

    7. \n

  7. (pilihan) Tambah sebuah Dapatkan Nilai Variabel action dan beri nama variabel FilePath. Abaikan input tindakan ini di dalamnya Pilihan .

    8. \n

  8. Tambah sebuah PDF ke Gambar tindakan, menyimpan output ke Desktop atau folder apa pun yang dapat menyimpan file sementara. Beri nama sesuka Anda.

    9. \n
  9. Tambah sebuah Tetapkan Nilai Variabel tindakan, jadi kita tahu nanti file sementara mana yang harus dihapus. Beri nama variabel TempFiles.
    10. \n

  10. Tambah sebuah Jalankan AppleScript tindakan, dan gunakan kode skrip berikut untuk memfilter daftar file sementara (ini adalah tempat kami menghapus halaman pertama): 

    on run {input, parameters}\n    return rest of input\nend run\n
    11. \n

  11. Tambah sebuah Gabungkan halaman PDF tindakan untuk menyatukan potongan lagi, dengan menambahkan halaman .

    12. \n
  12. Menggunakan Pindahkan Item Penemu tindakan untuk memindahkan file yang dihasilkan (PDF gabungan) ke Desktop, atau folder mana pun yang Anda inginkan. Jika Anda memutuskan untuk menghitung folder induk file masukan, di sinilah Anda menarik & melepas referensi ke variabel Folder.
    13. \n
  13. Itu Ubah Nama Item Penemu tindakan dapat memberi file ini nama file yang lebih baik daripada mis. zOpY3O.pdf, yang merupakan nama file yang ditetapkan secara otomatis untuk file PDF gabungan. Gunakan mis. Beri nama Item Tunggal dan berikan nama dasar Processed PDF File. Jika Anda memilih untuk menggunakan varian yang lebih panjang, seret FileName dari daftar variabel ke bidang teks, dan tambahkan (Edited) tepat sebelumnya. Sekarang pada dasarnya kita sudah selesai, hanya perlu bersih-bersih.
    14. \n
  14. Tambahkan Dapatkan Nilai Variabel tindakan dan dapatkan nilai TempFiles. Abaikan masukan tindakan ini di dalamnya Pilihan .
    15. \n
  15. Tambah sebuah Pindahkan Item Penemu ke Sampah tindakan untuk menghapus file halaman tunggal sementara.
    16. \n
\n\n

Menggunakan file alat PDF baris perintah , Saya dapat menyusun alur kerja yang melakukan hal berikut:

\n\n
    \n
  1. Ambil PDF input (atau PDF) dan buat file baru tanpa halaman pertama
    2. \n
  2. Pindahkan PDF asli ke tempat sampah
    3. \n
  3. Ubah nama file baru agar sesuai dengan file aslinya
    4. \n
\n\n

Pertama saya menginstal alat PDF seperti yang diinstruksikan. Alat utama dalam hal ini adalah pdfsplit.

\n\n

Di Automator, saya membuat layanan baru untuk menerima file PDF yang dipilih di Finder.

\n\n

Saya menambahkan tindakan "Run Shell Script", dengan shell sebagai "/ bin / bash" dan "pass input" disetel ke "sebagai argumen." Saya kemudian menulis skrip sederhana berikut: 

for f in \"$@\"\ndo\n        /usr/local/bin/pdfsplit \"$f\" 2- > \"$f\".tmp\ndone\n

Saya menambahkan tindakan "Pindahkan Item Penemu ke Sampah" untuk file asli dan tindakan "Ganti Teks" untuk menghapus ekstensi .tmp dari file baru.

\n\n

Untuk menjalankan proses dengan input folder, skripnya akan menjadi seperti ini: 

cd \"$@\"\nfor f in *pdf\ndo\n        /usr/local/bin/pdfsplit \"$f\" 2- > \"$f\".tmp\ndone\n

Saya kira saya bisa melakukan semuanya di skrip shell, termasuk menghapus dan mengganti nama. Tetapi perintah rm bisa berbahaya, dan saya lebih suka memindahkan file asli ke tempat sampah.

\n\n

Skrip dapat dimodifikasi untuk melakukan lebih dari sekadar menghapus x jumlah halaman. Saya telah mengembangkan program serupa dengan batch crop dan menggabungkan PDF, misalnya. Lihat manual di pdfsplit dan alat yang menyertainya untuk info lebih lanjut.
