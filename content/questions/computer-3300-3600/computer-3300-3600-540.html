Saya akan mempertimbangkan untuk menggunakan ext4 pada instalasi baru karena peningkatan kinerja cukup signifikan. Saya percaya bahwa ext4 adalah default untuk Ubuntu pada rilis berikutnya (9.10), tetapi tersedia tetapi tidak default di rilis saat ini (9.04).

\n\n

Anda dapat mengonversi sistem file saat ini sebagai ext3 ke ext4, tetapi peningkatan kinerja mungkin tidak berlaku untuk data yang ada karena beberapa perubahan arsitektur internal tidak berlaku untuk data tersebut. Setiap data baru, Anda akan mendapat manfaat dari luasan dan semacamnya.

\n\n

ext4 ditandai stabil, jadi rilis kernel baru tidak akan sepenuhnya merusak sistem file Anda yang sudah ada seperti sistem file yang tidak stabil seperti yang dilakukan btrfs.

\n\n

Tentu saja, saya akan mendukung semuanya sebelum mengacaukan apa pun, untuk berjaga-jaga .... Anda tidak pernah tahu.

Alokasi tingkat ext4 tidak heran ketika Anda menggunakan BitTorrent. Ketika klien BT mengalokasikan sebagian besar (katakanlah, saat Anda mengunduh iso DVD) disk untuk menghindari fragmentasi, pada ext4 itu hampir seketika, sedangkan pada ext3 dapat memakan waktu puluhan detik.

\n\n

Saya memiliki kernel oopses dengan ext4 pada Fedora 10 awal (tetapi tidak ada kehilangan data), tetapi saat ini tidak ada satu masalah pun dengannya. Ini terlihat cukup solid sejauh yang saya ketahui.
