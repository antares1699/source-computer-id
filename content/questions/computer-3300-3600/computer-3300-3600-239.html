Sistem Anda kemungkinan besar sudah disiapkan untuk debugging postmortem .

\n\n

Saya tidak begitu yakin apa yang membuat Windows memberikan opsi untuk men-debug proses yang gagal. Memiliki sebuah debugger dipasang pasti bisa menjadi bagian dari keputusan. Mungkin suatu proses bahkan dapat memaksa untuk menampilkan opsi debug setelah gagal.

\n\n

Biasanya opsi ini hanya ditujukan untuk pengembang.

Jika Anda memiliki JIT debugging diaktifkan di Visual Studio, atau debugger lainnya, itu akan memberi Anda kesempatan untuk melampirkan debugger ke proses yang lumpuh sehingga Anda dapat melakukan debugging post-mortem. Jika Anda tidak ingin JIT men-debug, Anda dapat menonaktifkannya, lalu Anda harus memasang debugger secara manual ke proses yang ingin Anda debug sebelum macet.
