Di mesin Windows 7 Enterprise x64 saya di tempat kerja, "Tampilkan konten jendela sambil menyeret" secara teratur dinonaktifkan.

\n\n

Saat ini saya memiliki Visual Effects yang disetel ke "Sesuaikan untuk penampilan terbaik", namun, sesekali, saya hanya menyeret batas jendela ke sekeliling - bukan konten.

\n\n

\n\n

(klik gambar untuk memperbesar)

\n\n

Ketika saya mengalami ini, kotak centang "Tampilkan isi jendela sambil menyeret" masih dicentang. Menghapus centang pada kotak, menerapkan pengaturan, mencentang kotak dan menerapkan pengaturan memulihkan fungsionalitas.

\n\n

Saya sebelumnya menggunakan "Kustom" sebagai lawan "Sesuaikan untuk penampilan terbaik". Ketika saya mengalami masalah saat itu, "Tampilkan isi jendela sambil menyeret" tidak akan dicentang. Saya bisa memeriksanya, menerapkan pengaturan dan "beberapa saat" nanti, masalah akan kembali.

\n\n

Saya tidak mengetahui pengaturan apa pun di skrip logon atau GPO yang akan menyebabkan ini. Juga log off dan kembali, atau bahkan me-reboot komputer, tidak mereproduksi masalah (setidaknya tidak secara konsisten).

\n\n

Saya jarang keluar atau reboot, tetapi saya sering terhubung melalui RDP dari rumah. Saya sudah mencoba menyambung ke sesi saya dari mesin Windows XP, keduanya dengan "Tampilkan isi jendela sambil menyeret" dinonaktifkan dan diaktifkan di klien RDP. Dalam kedua kasus tersebut, "Tampilkan konten jendela sambil menyeret" diaktifkan dan berfungsi saat masuk secara lokal - seperti yang diharapkan.

\n\n

Untuk apa nilainya, komputer saya dilengkapi dengan kartu grafis nVidia NVS 4200M dan Intel HD 3000.

\n\n

Adakah yang mengalami masalah ini atau yang serupa? Adakah ide apa yang dapat menyebabkan ini atau bagaimana menentukan?

\n\n

Terima kasih!

Ada solusi sederhana dan efektif untuk masalah ini. Di Notepad, ketikkan baris berikut:

\n\n

Windows Registry Editor Versi 5.00

\n\n

[HKEY_CURRENT_USER \\ Control Panel \\ Desktop] "DragFullWindows" = "1"

\n\n

[HKEY_LOCAL_MACHINE \\ SOFTWARE \\ Policies \\ Microsoft \\ Windows \\ Control Panel \\ Desktop]\n"DragFullWindows" = "1"

\n\n

Simpan file sebagai "Show Window Contents.reg." Mengklik dua kali file ini dan memulai ulang akan memperbaiki masalah secara permanen. Kirim pesan jika Anda merasa ini berguna.

Dalam kasus saya, Citrix Receiver yang menonaktifkan pengaturan tersebut, setiap kali saya meluncurkan aplikasi yang diterbitkan.

\n\n

Saya belum menemukan cara untuk menonaktifkan perilaku ini. Sebagai gantinya, saya telah membuat aplikasi konsol C ++ kecil yang mengaktifkan kembali pengaturan tersebut. Ini berjalan melalui tugas terjadwal setiap 5 menit.

\n\n

Seluruh kode sumber, yang dapat dikompilasi menggunakan VS Studio Community Edition: 

#include <windows.h>\n#include <tchar.h>\n\nint _tWinMain(HINSTANCE hInstance,\n    HINSTANCE hPrevInstance,\n    LPTSTR    lpCmdLine,\n    int       cmdShow)\n{\n    BOOL enabled;\n\n    SystemParametersInfo(SPI_GETDRAGFULLWINDOWS,\n        NULL, &enabled, 0);\n\n    //printf(\"Current 'dragfullwindow' option status: %d\\n\", enabled);\n\n    if (!enabled)\n    {\n        SystemParametersInfo(SPI_SETDRAGFULLWINDOWS,\n            TRUE,\n            NULL,\n            SPIF_UPDATEINIFILE | SPIF_SENDCHANGE);\n    }\n    return 0;\n}\n
