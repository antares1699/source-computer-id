Setelah menemukan posting Pengguna Super lain tentang penangkapan paket loopback di Windows, saya diarahkan ke RawCap , sebuah utilitas yang menyediakan pseudo-capture dari paket loopback. RawCap menulis ke file libpcap yang dapat dibaca oleh Wireshark. Ini bagus untuk menangkap dan kemudian menampilkan data yang diambil, tetapi saya memiliki kasus penggunaan di mana saya ingin melihat paket-paket ini diambil secara real time.

\n\n

Wireshark mendukung paket streaming dari STDIN , tetapi RawCap tidak mendukung pengarahan outputnya ke STDOUT. Itu hanya dapat menulis ke file. RawCap mendukung opsi untuk menonaktifkan buffering, menulis setiap paket ke file saat masuk.

\n\n

Ternyata saya sebenarnya bisa menggunakan tail untuk ini. Ini tidak ideal karena mungkin mencari \\n dalam data sebelum dikeluarkan, tetapi ada cukup data dalam penangkapan paket untuk membuat ini berfungsi. Inilah yang saya lakukan:

\n\n
    \n
  1. Install UnixUtilities untuk mendapatkan port of tail untuk Win32.
    2. \n
  2. Install RawCap
    3. \n
  3. Install Wireshark
    4. \n
  4. rawcap -f 127.0.0.1 localhost.pcap
    5. \n
  5. tail -c +1 -f localhost.pcap | wireshark -k -i -
    6. \n
\n\n

Karena RawCap membutuhkan terminal awal Anda, Anda harus memulai terminal kedua untuk bagian ekor. Juga, jelas gantikan di jalur apa pun yang Anda butuhkan.

\n\n

Sedikit hack, tapi berhasil! Saya hanya bisa berharap bahwa Wireshark akan menyertakan menembaki RawCap sebagai metode yang lebih langsung di rilis mendatang.
