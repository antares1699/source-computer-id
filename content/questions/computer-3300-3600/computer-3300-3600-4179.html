Saya telah melakukan ini sekali, di Debian; pengalaman saya juga harus diterapkan ke Ubuntu. Tidak seperti Anda, saya memiliki peningkatan terencana, jadi selalu memiliki opsi untuk kembali ke perangkat keras lama jika perlu.

\n\n

Jawaban singkatnya: ya, Anda akan dapat boot, dengan asumsi bahwa Anda memiliki sesuatu seperti kernel stok pada disk, dan driver yang cukup built-in (atau dalam initrd) untuk dapat me-mount disk. Dalam praktiknya, itu berarti driver SATA atau SCSI yang sesuai harus disertakan.

\n\n

Setelah Anda aktif dan berjalan, Anda harus dapat menggunakan dpkg --add-architecture untuk memberi tahu sistem Anda bahwa Anda menginginkan paket amd64 serta paket i386, dan apt-get update berikutnya (atau aptitude update) akan menawarkan versi paket i386 dan amd64. Instal kernel 64-bit dan tidak ada lagi .

\n\n

Boot ulang ke kernel baru. Verifikasi (menggunakan uname -m) bahwa Anda benar-benar menjalankan kernel baru. Jika booting tetapi gagal untuk memulai userland, mungkin Anda tidak mengkompilasi untuk mendukung objek ELF 32-bit - Anda akan membutuhkannya sampai Anda sepenuhnya dialihkan. (Anda tidak menghapus kernel 32-bit yang ada, bukan?)

\n\n

Sistem ini akan dengan senang hati menjalankan binari 32-bit Anda yang sudah ada. Anda sekarang dapat menginstal versi 64-bit dpkg; ini sekarang akan membuat amd64 menjadi arsitektur default Anda.

\n\n

Kemudian tingkatkan, beberapa paket sekaligus (kecuali Anda bodoh cukup berani untuk mengambil risiko peningkatan sekaligus), dan Anda akan memiliki sistem 64-bit sepenuhnya.

\n\n

Itu sedikit lebih sulit ketika saya melakukan peralihan, sebelum multiarch - saya harus menerima pukulan besar karena dependensi perpustakaan, dan itu menjadi sangat salah di tengah jalan. Anda mungkin ingin pindah ke Ubuntu yang lebih baru (mungkin 14,04 LTS) sambil tetap menjalankan 32-bit semuanya, dan kemudian bermigrasi. Itu lebih mungkin berhasil daripada sebaliknya.
