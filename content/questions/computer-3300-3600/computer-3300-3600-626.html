Anda dapat mencoba melakukan chroot ke sistem itu dari Linux atau LiveCD lain. Anggaplah distro Anda yang mati ada di sda2. Pertama, pasang partisi: 

$ mount /dev/sda2 /mnt\n

Sekarang ikat folder dengan binari penting untuk menggantikan yang mungkin mati. mount -B memasang satu direktori di atas yang lain secara virtual mengganti isinya, tetapi tanpa menyentuhnya secara fisik: 

$ mount -B /bin /mnt/bin\n$ mount -B /sbin /mnt/sbin\n$ mount -B /usr/bin /mnt/usr/bin\n

Sekarang chroot ke distro lama. chroot akan berpura-pura bahwa direktori yang diberikan adalah / (jadi dalam kasus kami, kami akan memiliki shell yang berjalan di distro offline, mengerjakan file-nya kecuali untuk binari yang telah kami jilid sebelumnya) 

$ chroot /mnt\n

Sekarang perintah apa pun yang Anda keluarkan akan berfungsi seolah-olah partisi itu adalah root sistem file Anda. lsb_release -a akan dieksekusi dari binari Anda, tetapi akan membaca dari file distro yang sudah mati.

\n\n

Anda dapat menggunakan chroot untuk banyak hal lainnya. Misalnya Anda dapat memutakhirkan Ubuntu tanpa mem-boot-nya: cukup boot LiveCD, pasang partisi Ubuntu, ikat direktori penting seperti /proc dll. (JANGAN lakukan binari - ini akan memperbarui binari LiveCD) dan menjalankan do-release-upgrade di shell chroot.
