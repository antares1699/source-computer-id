Ringkasan tujuan : agar mesin virtual Windows XP SP3 Hyper-V berfungsi dengan benar pada host Windows 10 Pro.

\n\n

Cakupan dan kendala :

\n\n
    \n
  • OS host harus Windows 10 Pro. Sistem operasi atau versi hosting lain berada di luar cakupan ini.
    • \n
  • Hypervisor virtualisasi harus Microsoft Hyper-V. Hypervisor lain seperti VMWare atau Virtual Box berada di luar cakupan ini.
    • \n
  • Mesin virtual Windows XP diharapkan bekerja dengan benar. Windows XP dengan driver yang hilang tidak dapat diterima.
    • \n
\n\n

Pertanyaan ini bukan duplikat di komunitas Pengguna Super.

\n\n

Saya mulai dengan rekomendasi yang disebutkan di Pertanyaan "Dapatkan mesin virtual Windows XP resmi untuk Hyper-V" . Saya bisa mendapatkan instalasi mesin virtual Windows XP SP3 baru. Namun, ada masalah dengan penginstalan ini, karena tidak dapat menyelesaikannya dengan sukses. Beberapa driver penting hilang, dan ini membuat mesin virtual tidak dapat digunakan.

\n\n

Kemudian, saya menemukan masalah driver perangkat lunak yang hilang dalam bentuk file Pertanyaan "Tidak ada driver VGA dan suara yang diinstal di mesin tamu XP dalam virtualisasi Hyper-v" . Sayangnya, pertanyaan ini masih belum terpecahkan. Tanggapan yang diberikan oleh komunitas sejauh ini tidak lengkap, tidak meyakinkan atau di luar jangkauan.

\n\n

Semakin banyak penelitian yang saya lakukan tentang topik ini, saya semakin dekat dengan kesimpulan itu ini tidak akan bisa dicapai . Saya ingin umpan balik dari komunitas untuk mengkonfirmasi atau menyangkal tesis ini.

\n\n

Terima kasih.

Iya. DVD / ISO instalasi harus memiliki SP3 yang sudah di-slipstream di dalamnya. Versi lama tanpa SP3 tidak dapat dipasang.

\n\n
    \n
  • Anda harus menggunakan VM Gen 1, perangkat keras dasar akan berfungsi dengan baik dengan XP.
    • \n
  • Nonaktifkan memori dinamis
    • \n
  • Versi rumah tidak akan berfungsi di bawah Hyper-V, Anda memerlukan versi Pro.
    • \n
  • Batasi RAM Anda untuk penginstalan hingga 1 hingga 2 GB, Anda dapat meningkatkan hingga 4 GB setelah penginstalan selesai
    • \n
  • Batasi vCPU Anda menjadi 1 atau 2 untuk penginstalan.
    • \n
\n\n

Komponen Integrasi

\n\n

Setelah penginstalan selesai, Anda harus menginstal Komponen Integrasi (IC) secara manual. (Sama seperti jawaban yang saya berikan di posting lain yang Anda tautkan)\nAnda perlu mendapatkan IC dari versi Hyper-V yang lebih lama. Saya memilikinya di situs wordpress saya di sini: https://smudj.wordpress.com/2017/03/02/vmguest-iso-for-older-windows-oses-in-win102016/

\n\n

Anda juga dapat mengunduh server Hyper-V dari Microsoft dan mengekstraknya, jika Anda menginginkannya dari sumber tepercaya.

\n\n

Versi IC 2012R2 harus diinstal otomatis setelah Anda memasang ISO ke drive dvd dan menginstal semua file perlu pengemudi.

\n\n

Perangkat Tidak Dikenal Anda akan melihat 2 perangkat yang tidak dikenal. Menurut Microsoft, ini diharapkan dan dapat diabaikan untuk OS yang lebih lama dari Windows Server 2012 R2. ( https://support.microsoft.com/en-us/help/2925727/unknown-device-vmbus-in-device-manager-in-virtual-machine-for-avma )

\n\n

XP pada Hyper-V jelas dapat dicapai dan biasanya mudah untuk diinstal.

\n\n

Satu catatan lain, audio akan mengharuskan Anda menggunakan RDP untuk menyambung ke XP dan sistem host harus memiliki audio yang berfungsi.
