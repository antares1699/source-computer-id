Saya memiliki dua perangkat lunak virtualisasi di salah satu laptop saya yang didedikasikan untuk pengembangan Windows 8.
VMWare untuk virtualisasi umum dan Windows 8 seperti Hyper-V untuk WP8 SDK.

\n\n

Sayangnya perangkat lunak ini tidak saling menyukai.
Apakah ada cara untuk membuat mereka ikut bermain?

\n\n

Masalah sebenarnya adalah jika saya menginstal SDK, VMWare bahkan tidak akan memperbarui atau menjalankan.
Hanya ada satu cara untuk membuatnya berhasil. Aktifkan dan nonaktifkan Hyper-V. Yang mana satu restart sepanjang waktu.

Inilah cara Anda melakukannya:

\n\n

SDK Windows Phone 8 di Mesin Virtual dengan Working Emulator

\n\n

Ini mengasumsikan bahwa Anda ingin menjalankan Hyper-V dalam VMware. Rupanya VMware, menggunakan terjemahan biner thingamajig, dapat sepenuhnya memvirtualisasikan CPU x86, meskipun lambat.

\n\n

Saya tidak berpikir Anda dapat menjalankan VMware di Hyper-V VM (yang merupakan Windows ketika Hyper-V diinstal dan Anda belum mengubah menu boot).

\n\n

Namun, Anda dapat mengubah menu boot, boot ke Windows tanpa Hyper-V dan menjalankan VMware dengan cara itu.

\n\n

Memperbarui: Menemukan situs yang menjelaskan bagaimana melakukan itu.

Dari Catatan Rilis VMWare Workstation 9 :

\n\n
\n

Virtualisasi Bersarang

\n \n

Workstation 9 meningkatkan implementasi virtual Intel VT-x / EPT atau\n Ekstensi AMD-V / RVI. Ini memungkinkan pengguna untuk menjalankan ESX sebagai OS tamu dan\n menjalankan sistem operasi 64-bit yang bersarang di ESX menggunakan lebih sedikit sistem\n sumber daya.

\n \n

Catatan: Jika Anda mengaktifkan ekstensi virtualisasi secara virtual\n mesin yang berjalan di Workstation 8, Anda mungkin perlu menonaktifkan file\n ekstensi, tingkatkan mesin virtual ke perangkat keras virtual terbaru\n versi (kompatibel dengan Workstation 9), lalu aktifkan kembali\n ekstensi.

\n \n

Hyper-V

\n \n

Hyper-V telah ditambahkan ke sistem operasi guest Workstation 9\n daftar. Ini memungkinkan pelanggan untuk menjalankan Windows 8 dengan Hyper-V diaktifkan, atau\n instal Server Hyper-V. Ini dapat digunakan untuk tujuan pendidikan atau\n untuk membangun prototipe Awan Hibrida. Fitur ini TIDAK DIDUKUNG\n dan mungkin tidak akan pernah. Microsoft tidak mendukung pengumpulan\n hypervisor mereka yang membuatnya sangat sulit - jika tidak\n VMware tidak mungkin memperbaiki masalah yang mungkin terjadi dalam hal ini\n konfigurasi. Untuk itulah kapabilitas ini telah diimplementasikan\n semata-mata untuk melihat apakah kami bisa melakukannya!

VMWare dan Hyper-V keduanya merupakan implementasi dari konstruksi perangkat lunak yang dikenal sebagai a hypervisor . Perangkat keras komputer modern memiliki fitur pengoptimalan khusus untuk hypervisor untuk mendukung peningkatan kinerja. Ini adalah sirkuit khusus yang dibangun ke dalam cpu dan motherboard komputer Anda.

\n\n

Perangkat lunak hypervisor modern, seperti VMWare dan Hyper-V, bergantung dan mengandalkan perangkat keras khusus ini. Jika satu sistem menggunakan perangkat keras itu, yang lain tidak bisa.

\n\n

Singkatnya, tidak ada cara untuk menggunakan VMWare dan Hyper-V secara bersamaan.
