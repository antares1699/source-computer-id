Saya telah menemukan sistem lisensi yang entah bagaimana terkait dengan media penyimpanan. Meskipun mendasarkan ini pada UUID pada filesystem cukup mudah, saya tidak percaya ini masalahnya (lihat di bawah). Contoh tipik dari perilaku ini adalah sebagai berikut:

\n\n
    \n
  1. Pada setiap boot, file id dibuat.
    2. \n
  2. Sistem akan memeriksa file kunci untuk lisensi. Jika kuncinya cocok dengan id, maka semuanya baik-baik saja.
    3. \n
  3. Jika tidak ada kunci, atau jika kunci tidak cocok dengan id (seperti, disalin dari file\nsistem yang berbeda), lisensi tidak valid.
    4. \n
\n\n

Saya telah mengamati perilaku berikut:

\n\n
    \n
  • Menghapus sepenuhnya kartu CF dan menginstal ulang menghasilkan file id yang sama, sehingga tidak diperlukan file kunci baru.
    • \n
  • Menggunakan kunci yang sama pada CF yang berbeda tidak akan bekerja, karena sebenarnya tidak\ncocok dengan konten file id baru.
    • \n
\n\n

Ini membuat saya menyimpulkan bahwa ada cara untuk mendapatkan id unik dari kartu CF entah bagaimana caranya. Bagaimana seseorang bisa mencapai ini di linux?

Kartu tersebut akan memiliki banyak informasi yang terdaftar dengan cara hanya-baca.

\n

Artikel Membaca Informasi Kartu SD Internal mengatakan ini:

\n
\n

Informasi Kartu SD Internal

\n

Informasi tentang kartu SD dikodekan di kartu internalnya\npendaftar. Salah satunya adalah Identifikasi Kartu (CID) Daftar, a\nKode 16 byte yang berisi informasi yang secara unik mengidentifikasi SD\nkartu, termasuk nomor seri kartu (PSN), nomor ID produsen\n(MID) dan tanggal pembuatan (MDT). Register CID diset ketika\nkartu diproduksi dan tidak dapat diubah setelah disetel. (Menurut\nuntuk spesifikasi kartu SD informasi hanya untuk ditulis sekali,\nNamun jika kartu tidak sesuai dengan spesifikasi ini\ninformasi dapat diubah!)

\n

Cara membaca CID dari kartu SD

\n

Salah satu cara membaca CID adalah dengan menggunakan laptop dengan slot kartu SD. Kartu\npembaca di laptop biasanya terhubung langsung melalui bus PCI\n(atau bus IDE). Ini tidak akan bekerja melalui pembaca kartu USB karena\nperintah untuk mengambil informasi kartu dicegat dan tidak\ndipahami oleh pembaca kartu. Tidak semua komputer dengan slot kartu built-in\nakan berfungsi, beberapa pembaca kartu internal terhubung melalui bus USB.

\n

Dengan asumsi Anda memiliki perangkat keras yang tepat, ada beberapa metode untuk Anda\ndapat digunakan untuk mendapatkan informasi kartu. Dengan Linux, membaca internal\nInformasi kartu SD sederhana. Masukkan kartu dan lihat di bawah\n/sys/block/mmcblk0/device/ (lokasi ini dapat berubah tergantung pada Anda\nalat). Anda akan melihat beberapa atribut yang tersedia yang mencakup\nCID register dan informasi di dalamnya.

\n

Seperti yang dikomentari orang lain, ini mungkin terkait dengan cara kartu CF telah dihapus, tetapi kemungkinan besar perangkat lunak mengidentifikasi kartu CF berdasarkan firmware di kartu -

\n\n

Kabin informasi ini biasanya dibaca menggunakan "udevinfo" - misalnya, jika kartu CF adalah / dev / sdb udevinfo -a -p $ (udevinfo -q path -n / dev / sdb) - ini kemungkinan besar akan menyertakan informasi yang cukup untuk mengidentifikasi kartu secara unik.

\n\n

Catatan : pada distro berbasis debian udevadm info menggantikan udevinfo:

\n\n

udevadm info -a -p $( udevadm info -q path -n /dev/sdb )
