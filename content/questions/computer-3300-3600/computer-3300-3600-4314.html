Saya telah menginstal docutils di sistem Mac OS saya mengikuti instruksi docutils s.

\n\n

Saya juga telah melakukan: 

$ sudo cp tools/buildhtml.py tools/rst2html.py /usr/local/bin\n

Namun, jika saya membuka baris perintah dan mengetik rst2html, saya mendapatkan 'perintah tidak ditemukan'.

\n\n

Jika saya mengetik rst2html.py, tidak ada masalah, jadi file Python itu sendiri ada di jalur saya.

\n\n

Bagaimana cara agar sistem saya mengenali rst2html sebagai perintah yang valid?

Saat Anda mengetik perintah rst2html pada baris perintah, itu akan mencari program bernama rst2html (bukan rst2html.somethingorother) di direktori di PATH Anda. Jika Anda ingin rst2html berfungsi, Anda memerlukan file bernama rst2html. Ada beberapa cara untuk melakukannya:

\n\n
    \n

  1. Ubah nama program. Sebenarnya tidak diperlukan ekstensi .py, karena sistem tahu itu adalah skrip Python karena itu peristiwa garis. 

    sudo mv /usr/local/bin/rst2html.py /usr/local/bin/rst2html\n
    2. \n

  2. Hubungkan program dengan keras. Anda dapat membuat program terdaftar di / usr / local / bin dengan dua nama berbeda, rst2html.py dan rst2html, dan salah satunya dapat digunakan untuk meluncurkannya. 

    sudo ln /usr/local/bin/rst2html.py /usr/local/bin/rst2html\n
    3. \n

  3. Tautan simbolik program. Mirip dengan hard-linking, kecuali bahwa hanya satu yang merupakan file "asli", yang lainnya hanyalah alias unix-style. Ini mungkin sedikit lebih mudah dikelola daripada tautan langsung, karena jika Anda pernah memasang versi yang diperbarui, tautan tersebut akan secara otomatis menggunakan versi baru (sementara dengan tautan keras, Anda perlu memperbarui tautan secara manual). 

    sudo ln -s rst2html.py /usr/local/bin/rst2html\n
    4. \n
\n\n

Secara pribadi, saya baru saja mengganti namanya (opsi 1), tetapi salah satu opsi lain juga berfungsi dengan baik.
