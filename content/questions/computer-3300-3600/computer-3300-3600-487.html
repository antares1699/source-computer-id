Gnu adalah userland jadi tidak ada hubungannya dengan jumlah CPU. Batas CPU dan memori bergantung pada kernel dan arsitekturnya.

\n\n

Saya berasumsi Anda bertanya tentang arsitektur x86 yang berjalan pada mode 64 bit.

\n\n

Jumlah maksimum CPU (lebih tepatnya inti ((bahkan lebih tepatnya utas perangkat keras dalam kasus Chip Multi-Threading / HyperThreading))) yang didukung oleh kernel Linux adalah parameter yang ditetapkan pada waktu kompilasi. Nilai umum adalah 8 (smp default), 32 untuk perangkat keras smp yang lebih besar. Pengaturan maksimum adalah 512, meskipun Anda akan kesulitan untuk mengetahui perangkat keras x86 yang sebenarnya dengan jumlah CPU sebanyak itu.

\n\n

Anda mungkin mendapatkan nilai yang ditetapkan untuk kernel Anda dengan melihat nilai NR_CPUS yang dilaporkan dalam file /boot/config-$(uname -r).

\n\n

Maksimum (percobaan) adalah 4096 [Sunting: mungkin 8192 sejak tambalan 2013] tetapi AFAIK hanya digunakan untuk tujuan pengujian kode.

\n\n

Tentang memori, kernel Linux dapat mendukung sekitar 64 TB RAM dan setiap proses sekitar 128 TB memori virtual. Di sini juga, batasan ini cukup besar karena tidak dapat dijangkau oleh perangkat keras yang ada.

Saya pikir Anda perlu lebih spesifik untuk pertanyaan Anda. Distribusi apa? Beberapa dari jawaban ini akan bergantung pada kernel yang Anda gunakan. Jika Anda menginginkan nomor tertentu, Anda dapat mencocokkannya dengan distribusi tertentu.

\n\n

Jumlah CPU dapat memiliki dua jawaban - berapa jumlah maksimum CPU yang dapat digunakan OS, titik. Dan apa yang bisa digunakannya secara efisien. Terkadang ini bisa sangat berbeda.

\n\n

Mengenai memori, Anda akan mendapatkan batasan kernel, tetapi juga batasan perangkat keras. Motherboard / chipset tertentu akan memiliki batasan praktis, baik karena pengalamatan, atau karena jumlah slot RAM yang terbatas.
