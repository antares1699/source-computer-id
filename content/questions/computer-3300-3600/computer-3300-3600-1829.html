Apakah ada opsi konfigurasi di Cygwin sehingga ketika Bash melengkapi nama perintah, akhiran .exe nama file diabaikan (tidak disertakan)?

\n\n
\n\n

Penjelasan masalah

\n\n

Saat menggunakan penyelesaian tab di Bash pada sistem Cygwin untuk melengkapi nama perintah, ekstensi .exe ditambahkan ke nama perintah, misalnya, mengetik opens dan menekan Tab menyelesaikan perintah ke openssl.exe.

\n\n

Perintah berjalan dengan baik (dengan MS Windows ekstensi .exe adalah opsional saat menjalankan perintah) tetapi masalahnya adalah bahwa fungsi penyelesaian _openssl() disediakan oleh bash-completion paket hanya dikonfigurasi untuk memberikan penyelesaian argumen untuk openssl - bukan openssl.exe. Contoh: 

$ complete -p openssl openssl.exe\ncomplete -o default -F _openssl openssl\nbash: complete: openssl.exe: no completion specification\n

Masalah yang sama muncul saat mencoba melengkapi argumen untuk semua perintah yang dapat dieksekusi.

\n\n

Saat ini saya menggunakan mode Bash dengan Emacs yang dikonfigurasi untuk pengeditan Readline sehingga saya dapat menekan ESC diikuti oleh dua Menghapus menekan untuk menghapus sufiks .exe sebelum saya mulai mengetik argumen untuk perintah tersebut. Idealnya, saya ingin menghindari keharusan melakukan ini setiap kali saya menjalankan perintah.

\n\n
\n\n

Apa yang sudah saya coba / teliti

\n\n

Saya pikir itu mungkin tidak mungkin tanpa memodifikasi kode sumber dari Cygwin DLL atau penyelesaian perintah Bash (pcomplete.c). Namun, saya perhatikan bahwa Bash builtins, type dan command secara otomatis menghapus akhiran .exe dari nama file yang dapat dieksekusi, misalnya, 

$ type -a openssl\nopenssl is /usr/bin/openssl\n\n$ command -v openssl\n/usr/bin/openssl\n

Tampaknya Bash yang berjalan di Cygwin memiliki beberapa mekanisme untuk menyediakan nama perintah kosong (tanpa ekstensi .exe). Namun, saya bingung bagaimana - atau jika - ini dapat digunakan untuk menghilangkan ekstensi file saat menyelesaikan perintah.

Ternyata disana aku s opsi konfigurasi di Cygwin yang mengonfigurasi Bash untuk tidak menyertakan ekstensi .exe nama file ketika itu melengkapi nama perintah.

\n\n

Mengaktifkan opsi completion_strip_exe (khusus untuk port Cygwin dari Bash) melakukan apa yang saya inginkan: 

shopt -s completion_strip_exe\n
\n\n

Fitur ini tidak didokumentasikan dengan jelas: fitur ini disebutkan sepintas lalu di Perluasan Pathname bagian dari halaman manual Cygwin untuk Bash (tidak disertakan dalam kode sumber hulu sehingga tidak didokumentasikan di halaman manual resmi atau dokumentasi untuk Bash ). Saya menemukannya saat membaca dengan teliti /usr/share/doc/Cygwin/bash.README (sekitar 4 bulan setelah menanyakan pertanyaan ini):

\n\n
\n

7b. menggunakan 'shopt -s complete_strip_exe' untuk melengkapi sufiks .exe.

\n
\n\n

Tampaknya opsi ini telah tersedia di Cygwin Bash selama lebih dari 5 tahun:

\n\n
\n

----- versi 4.1.9-1 - 2010-12-29 -----
Tambahkan EXECIGNORE dan patch complete_strip_exe dari Dan Colascione.

\n
\n\n

Penelitian lebih lanjut menunjukkan bahwa tambalan untuk fitur ini adalah diajukan oleh Dan Colascione kembali pada November 2010 dengan uraian berikut :

\n\n
\n

completion_strip_exe adalah opsi shell baru. Saat diaktifkan, bash mencoba menggunakan nama pendek dari sebuah program daripada nama yang lebih panjang ".-Exe" -suffixed. Dengan ini, pin selesai untuk "ping".
