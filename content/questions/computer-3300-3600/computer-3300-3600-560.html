Ini adalah salah satu perilaku paling menjengkelkan yang pernah saya alami.

\n\n

Anda mengambil file JPG dari Web, tetapi Chrome memaksa Anda untuk mendownloadnya dalam format WEBP. Konversi tidak masuk akal, terutama jika tidak diinginkan (dan ada banyak, banyak alasan untuk ini: menginginkan file asli yang tepat, mengambil metadata, masalah menggunakan kembali file webp di perangkat lunak lain, dll).

\n\n

Adakah cara untuk menonaktifkan perilaku yang mengganggu ini?

Cara paling serbaguna adalah menggunakan ModHeader untuk mengubah header permintaan "Terima" Anda ke nilai "image / avif, image / apng, image / svg + xml, image / , /*;q=0.8 ".

\n

Situs apa pun yang masih menyajikan WebP meskipun header Terima permintaan sekarang kehilangan penerimaan untuk WebP dapat diakali dengan mengubah header Agen-Pengguna. Lagi, ModHeader akan dapat melakukannya, meskipun itu perlu sedikit lebih berhati-hati, karena banyak situs web akan menyajikan konten yang sama sekali berbeda berdasarkan Agen-Pengguna.

\n

curl -O https://www.predictiveanalyticstoday.com/wp-content/uploads/2016/09/Stardog.jpg

\n

Ini tersedia secara default di OS X, Linux, dan Windows 10. Jika Anda memiliki versi Windows yang lebih lama, Anda dapat menginstal curl .

\n

Anda dapat melakukan hal yang sama dengan wget jika Anda mau:

\n
