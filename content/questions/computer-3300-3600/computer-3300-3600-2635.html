Saya mencoba untuk mencapai "Enkripsi Disk Penuh" di sistem Linux yang sudah terinstal. Saya telah menemukan LUKS dengan dm-crypt, itu dapat mengenkripsi drive / seluruh disk, tetapi sebelum mengenkripsi itu memformat drive terlebih dahulu dan semua data dihapus secara permanen.

\n\n

Karena saya perlu melakukan "Enkripsi Disk Penuh" untuk mesin Linux yang sudah ada & dewasa (misalnya Ubuntu, centOS), saya tidak dapat membayar pemformatan drive / partisi. Apakah ada metode yang memungkinkan saya untuk melakukan "Enkripsi Disk Penuh" tanpa kehilangan konten yang ada di sistem.

\n\n

Saya sangat baru dalam konsep ini sehingga panduan yang tepat akan sangat membantu.\nTerima kasih!!

Pertama, Anda perlu mengecilkan partisi Anda sebesar 1028 KB (sedikit lebih dari 1 MB). Saya akan merekomendasikan untuk mengecilkannya sebesar 4 MB. Lepaskan partisi Anda sebelum operasi.

\n\n

Untuk ext4: 

sudo dumpe2fs /dev/sdd1|grep 'Block count'\n

kurangi 1024 dari nilai ini (dengan asumsi Anda memiliki blok 4KB), kemudian lakukan: 

sudo resize2fs /dev/sdd1 new_value\n

Kemudian enkripsi: 

sudo cryptsetup-reencrypt -N --reduce-device-size 4M /dev/sdd1\n
