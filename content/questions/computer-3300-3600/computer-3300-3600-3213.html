Pendekatan biasa untuk ini adalah dengan melakukan hal berikut.

\n\n
    \n

  1. Pastikan kedua router menggunakan sama subnet. Karena Anda mengatakan router terintegrasi Thompson & modem DSL harus menggunakan alamat IP lokal 10.0.0.139, tampaknya Anda harus mengatur WRT54G untuk menggunakan jaringan 10.0.0.0 dengan subnet mask yang sama 255.255.255.0 yang digunakan oleh router Thompson.

    2. \n

  2. Seperti yang telah Anda ketahui, hanya satu router yang dapat disetel sebagai server DHCP. Penting juga agar pengaturan DHCP Anda dilakukan TIDAK tumpang tindih / konflik dengan salah satu alamat tetap / statis di LAN lokal Anda. Jadi pilihlah IP untuk WRT54G yang tidak dalam rentang DHCP Anda. (Anda tidak menyebutkan kisaran alamat IP yang digunakan DHCP Anda saat ini).

    3. \n

  3. Pastikan Anda menghubungkan kedua router melalui kabel ethernet di salah satu port LAN lokal WRT54G. Melakukan tidak gunakan port WLAN di WRT54G. (Ini adalah cara termudah untuk menghindari penggunaan NAT di WRT54G IMO).

Jika Anda tidak dapat mengubah ini dengan mengubah pengaturan untuk DHCP router Thompson, maka menggunakan IP yang tidak mungkin ditetapkan oleh DHCP seperti 10.0.0.250 adalah yang akan saya coba.

\n\n
\n

Tidak yakin apa yang Anda maksud dengan "rute default".

\n
\n\n

Ada tiga nilai konfigurasi dasar untuk setiap sistem dalam jaringan IPv4.

\n\n
    \n
  1. Alamat IP unik. Alamat IP digunakan untuk mengidentifikasi sistem atau perangkat di jaringan.
    2. \n
  2. Subnet mask yang akan digunakan. Subnet mask digunakan oleh sistem atau perangkat untuk memutuskan apakah alamat IP tujuan ada di LAN lokal Anda atau "remote" (mis. Di tempat lain di Internet).
    3. \n
  3. Rute default. Ini hanyalah alamat IP perangkat yang dikirimi paket data ketika IP tujuan dari paket tersebut tidak di LAN lokal Anda.
    4. \n
\n\n

Setiap PC di LAN Anda harus memiliki tiga pengaturan konfigurasi ini. Dan agar berhasil berkomunikasi dengan perangkat lain di LAN lokal Anda atau di Internet, pengaturan harus konsisten dengan pengaturan perangkat lain di LAN Anda.

\n\n

Anggaplah laptop Anda mencoba mengirim paket IP ke alamat 10.0.0.250. Hal pertama yang dilakukan adalah subnet mask di laptop pengaturan IP diterapkan ke alamat IP laptop dan IP tujuan.

\n\n

Jika subnet mask adalah 255.255.255.0, maka 3 angka pertama dari alamat IP sumber & tujuan (yaitu 10.0.0) dibandingkan. Jika keduanya identik, maka tujuannya ada di LAN lokal dan paket dikirim langsung ke ethernet Anda.

\n\n

Tetapi jika IP sumber & tujuan berbeda, maka paket akan dikirim ke alamat IP dari pengaturan konfigurasi "rute default". Perangkat di IP itu kemudian harus mencari tahu ke mana harus mengirim paket selanjutnya sehingga pada akhirnya mencapai tujuan akhirnya.

\n\n

Yang penting di sini adalah bahwa ketiga pengaturan ini (bagian dari) yang diberikan DHCP ke laptop Anda dan komputer lain. Jika laptop Anda menggunakan Windows, salah satu cara Anda dapat menampilkan nilai yang ditetapkan untuk laptop Anda adalah dengan mengeluarkan perintah ipconfig /all di jendela prompt perintah. Nilai untuk rute default harus selalu berupa alamat router Thompson Anda karena router tersebut adalah satu-satunya perangkat di sistem Anda yang terhubung ke Internet.

\n\n

Saya cukup yakin Anda harus selalu menggunakan server DHCP di Thompson. Untuk menggunakan DHCP pada WRT54G, Anda juga harus dapat menentukan "rute default" yang akan diinstruksikan oleh DHCP untuk digunakan oleh PC Anda. Saya tidak berpikir ini bisa dilakukan.

\n\n

DHCP di router rumah yang saya kenal biasanya memberikan alamat IP LAN lokalnya sendiri sebagai rute default. Ketika hanya ada satu router dan itu adalah satu-satunya cara untuk mencapai Internet, asumsi ini berhasil. Tetapi ketika Anda menambahkan router kedua sebagai Titik Akses Nirkabel alternatif, rute default-nya masih harus berupa IP LAN lokal dari router yang terhubung ke Internet yang akan menjadi Thompson Anda.
