Kemungkinan Duplikat:
\n Bagaimana cara membagi proses yang tidak mati saat shell keluar?

\n
\n\n\n\n

Terkadang saya terhubung dari jarak jauh ke Ubuntu saya menggunakan NX. Saya kemudian menjalankan beberapa pekerjaan di latar belakang, misalnya: 

$ /path/to/script.pl &\n

Namun, ketika saya keluar, pekerjaan berhenti berjalan.

\n\n

Bagaimana saya bisa membuatnya berlanjut bahkan setelah saya keluar?

Shell membunuh semua proses dalam grup prosesnya saat berakhir, dengan mengirimkan SIGHUP.

\n\n

Jika Anda menjalankan bash shell, Anda dapat mengetik disown agar tetap berjalan setelah Anda keluar. Ini menghapusnya dari daftar proses yang akan dikirimi sinyal.

\n\n

Atau Anda dapat meluncurkan skrip dengan nohup, tetapi kemudian Anda harus mengingatnya saat menjalankan perintah. Ini memberi tahu perintah Anda untuk mengabaikan sinyal SIGHUP yang akan dikirim oleh shell. Ini akan bekerja pada shell apa pun.

Lihat "Tetap jalankan skrip linux setelah Anda menutup shell jarak jauh" untuk cara melakukan ini menggunakan perintah layar.

\n
\n

Jawabannya terletak pada alat baris perintah yang disebut layar.

\n

Layar memungkinkan Anda untuk memulai proses pada layar virtual, kemudian melepaskan layar itu dan melakukan> sesuatu yang lain (termasuk keluar). Anda juga dapat memasang kembali layar Anda setelah keluar> dan masuk lagi.

\n

Jika Anda tidak memiliki perintah layar pada kotak linux jarak jauh Anda, instal dulu dari sumber atau menggunakan pengelola paket favorit Anda. Kemudian login ke kotak remote Anda dan jalankan perintah yang Anda inginkan diawali dengan "layar". Sebagai contoh:

\n

layar atas

\n

Sekarang untuk melepaskan layar gunakan CTRL + a diikuti oleh d. Ini akan melepaskan layar Anda dan Anda dapat melakukan bisnis lain, termasuk keluar dari sesi jarak jauh Anda.

\n

Menyambungkan kembali layar Anda kapan saja semudah menjalankan perintah:

\n

layar -r
