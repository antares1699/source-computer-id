Jika emulator QEMU untuk komputer ARM terlalu lambat,\nAnda bisa mencoba Microsoft Device Emulator 3.0 - Rilis Mandiri .\nAnda dapat menemukan beberapa informasi di artikel Panduan Pengembang untuk Emulator ARM ,\nmeskipun itu berasal dari tahun 2005 dan tentang Microsoft Device Emulator 2.0.

\n\n

Itu Proyek Xen juga telah diumumkan Dukungan untuk CPU ARM ,\nmeskipun saya tidak tahu seberapa maju perkembangan itu dan tidak jelas\napakah itu dapat meniru ARM atau hanya berjalan di CPU fisik ARM.\nIni adalah emulator lain yang bisa Anda coba.

\n\n

Jika Anda ingin menggunakan arsitektur ARM dengan kecepatan asli, satu-satunya solusi\nadalah mendapatkan perangkat ARM.\nRaspberry Pi adalah kemungkinan murah di sini.

\n\n

Untuk mendapatkan gambar siap pakai dengan Raspberry Pi di QEMU, lihat proyeknya Emulasi Raspberry Pi untuk Windows .\nItu datang dengan file run.bat dan Anda harus menjalankannya dan mengikuti instruksi.

\n\n

Jika semua yang Anda inginkan adalah bermain dengan program assembler ARM, Anda dapat menggunakan CEMU ,\nmendukung Arm / AArch64, dan dijelaskan sebagai:

\n\n
\n

aplikasi GUI yang dibundel yang memberdayakan pengguna untuk menulis assembly dan mengujinya dengan mengompilasinya menjadi bytecode dan mengeksekusinya dalam emulator berbasis QEMU.
