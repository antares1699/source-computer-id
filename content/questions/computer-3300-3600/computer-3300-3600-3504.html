Setelah beberapa jam debugging, saya menemukan pelakunya. Sepertinya itu memang sebagian rollout / tes AB dari facebook.

\n\n

Lebih tepatnya itu disebut bendera disableDuplicatePlayingRequest dari VideoPlayerExperiments yang dimuat di sumber halaman untuk profil Anda.\nSetelah disetel ke true mengharuskan Anda untuk menjeda video sebelum mengubah waktu, jika tidak, video akan macet / macet.

\n\n

Nah untuk solusinya, tidak ada yang sederhana. Tampaknya digunakan dalam closure jadi tidak ada cara mudah untuk mengubahnya. Setidaknya dari apa yang saya lihat / uji.

\n\n

Satu-satunya cara saya bisa melewatinya adalah dengan mengganti https://www.facebook.com/rsrc.php/v3iTQQ4/yd/l/fb_LT/JTeC8QI2Wlf.js ( kemudian edit : nama / lokasi skrip sering berubah, saya telah menambahkan tutorial untuk menemukan yang tepat) skrip dari facebook dan memuat versi skrip yang disesuaikan dengan bantuan ekstensi yang mengabaikan pemeriksaan dari file: if (c('VideoPlayerExperiments').disableDuplicatePlayingRequest && this.isState('playing')) return;

\n\n

Jika ada yang tertarik dengan ekstensi ini, namanya Resource Override , tapi tidak semudah itu untuk digunakan. Anda memerlukan server untuk menghosting file js yang dimodifikasi dan beberapa aturan Header untuk melewati CORS dan CSP.

\n\n

P.S. Saya membuat pertanyaan komunitas di sini untuk mendapatkan jawaban dari Facebook. Siapa pun dapat membantu dengan memberikan suara (atau menyukai seluler) pertanyaan.

\n\n

P.S.2: Oke, jadi untuk benar-benar memperbaiki masalah ini, Anda perlu menggunakan Chrome dan mengikuti langkah selanjutnya:

\n\n
    \n

  1. Buka FB dan login. Cari semua sumber js untuk flag tersebut disableDuplicatePlayingRequest . Jika Anda tidak tahu bagaimana melakukannya, Anda bisa menggunakan tutorial ini

    2. \n

  2. Setelah Anda menemukan file yang berisi file tersebut, gunakan tombol {} (Pretty Print) lalu salin seluruh konten dari file tersebut ke editor teks. Pergi ke baris yang memiliki sesuatu seperti if (c('VideoPlayerExperiments').disableDuplicatePlayingRequest && this.isState('playing')) return; dan hapus atau komentari. Anda sekarang perlu menyimpan file ini dan mengunggahnya ke server. Saya tidak dapat membagikan file saya karena FB mungkin menganggapnya sebagai pelanggaran persyaratan, jadi Anda harus melakukannya sendiri. Ingat url akses publik untuk file tersebut.

    3. \n

  3. Instal Resource Override ekstensi dari toko chrome

    4. \n

  4. Buka opsi ekstensi. Di Tab URL masukkan *facebook.com*

    5. \n

  5. Tambahkan aturan ubah tajuk. Di For: masukkan https://www.facebook.com*. Sekarang masuk ke Edit headers. Kita hanya akan bekerja dengan header respon, jadi pastikan untuk hanya menggunakan bagian kedua. Tambahkan 2 set aturan header untuk Content-Security-Policy dan X-Content-Security-Policy. Sekarang kita perlu menambahkan server Anda ke kebijakan yang ada yang digunakan FB. Untuk melakukan itu kembali ke halaman facebook dan masuk ke Konsol Pengembang> tab Jaringan. Pilih salah satu permintaan file .js. Dalam header respon Anda akan melihat content-security-policy. Salin semuanya setelah : ke editor teks. Ini harus dimulai dengan default-src.... Sekarang temukan blob:;script-src dan setelah itu tambahkan domain yang Anda gunakan untuk meng-host file js Anda. Misalnya xyz.com atau *.xyz.com (jika Anda menggunakan subdomain). Pastikan untuk meninggalkan spasi sebelum dan sesudahnya sehingga tidak memengaruhi rangkaian kebijakan yang ada. Sekarang gunakan baris baru ini dan letakkan di 2 bidang nilai dari bidang tajuk yang ditetapkan dari ekstensi. Keduanya harus identik. Sekarang kita membutuhkan satu lagi aturan tajuk yang ditetapkan untuk Access-Control-Allow-Origin dengan nilai *. Klik tutup untuk kembali ke menu utama.

    6. \n

  6. Tambahkan satu lagi aturan tajuk perubahan. Kali ini kami akan menggunakannya untuk domain Anda untuk memastikan Anda memiliki tajuk yang tepat di sana juga. Di kolom For: masukkan link yang tepat bahwa file Anda yang dimodifikasi dapat diakses di internet. Https yang lebih disukai. Menggunakan http atau file lokal dapat menyebabkan masalah. Sekarang masuk ke bagian tajuk perubahan aturan. Kami akan menggunakan di sini bagian tanggapan juga. Dalam preset, pastikan untuk memilih opsi Allow outside content agar otomatis menambahkan 2 aturan untuk Anda. Setelah 2 itu, kita membutuhkan satu set aturan header lagi untuk Access-Control-Allow-Origin dengan nilai https://www.facebook.com. Jangan klik tutup dan kembali ke menu utama.

    7. \n

  7. Langkah terakhir adalah benar-benar mengganti file FB js dengan file yang sudah dimodifikasi. Untuk melakukan itu tambahkan aturan untuk URL -> URL. Di From: tambahkan file dari langkah (1). Sebagai catatan tambahan, link ini akan berubah secara berkala karena facebook cenderung mengupdatenya setiap 1-3 hari sekali. Anda juga perlu mengubahnya ketika Anda melihat video akan mulai membeku lagi. Anda dapat mengikuti langkah (1) untuk mendapatkan tautan baru untuk itu. Sekarang di bidang To: Anda harus meletakkan tautan Anda ke file yang dimodifikasi. Hal terakhir adalah memeriksa bahwa semua 3 aturan ini disetel ke ON. Itu saja, video akan berfungsi kembali seperti dulu.

    8. \n
\n\n

P.S.3: Anda dapat menemukan aturan ekstensi yang disimpan di sini . Anda perlu mengganti nama xyz.co dan https://xyz.co/modified.js dengan nama Anda sendiri. Mereka masing-masing muncul 2 kali. Setelah itu, pastikan bahwa tautan file yang berisi disableDuplicatePlayingRequest sudah benar.
