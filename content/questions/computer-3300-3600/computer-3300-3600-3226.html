Jika saya mengetikkan kata sandi yang salah saat masuk, maka sistem memaksa saya menunggu sekitar satu detik sebelum saya dapat mencoba lagi. Adakah cara untuk mengurangi batas waktu ini? Selain itu, apakah ada pengaturan batas waktu global untuk su dan sudo atau apakah saya harus mengubah batas waktu tersebut menggunakan metode yang berbeda?

ubah baris FAIL_DELAY pada /etc/login.defs. Itu akan mempengaruhi baik login maupun su. Tetapi mengapa Anda ingin melakukan itu?

Memeriksa /etc/login.defs di Ubuntu 11.10 Saya melihat bahwa opsi konfigurasi yang disebutkan b0fh telah dipindahkan ke file /etc/pam.d/login sebagai: 

auth       optional   pam_faildelay.so  delay=3000000\n

yang saya ubah dari 3 detik. menjadi setengah detik untuk mengurangi efek dari kebiasaan buruk saya yang sering salah memasukkan sandi saat pertama kali masuk. (Saya menganggap risiko tambahan dari serangan brute force yang memakan waktu seperenam dari waktu yang diperlukan jika tidak adalah faktor yang dapat diabaikan)

Untuk beberapa alasan mengedit penundaan pada /etc/pam.d/login tidak berpengaruh untuk Ubuntu 12.04 saya.

\n\n

Akan lebih baik jika ada penundaan kecil tapi bukan nol (seperti setengah detik); Saya tidak dapat melakukannya, tetapi saya dapat menonaktifkan penundaan dengan mengedit /etc/pam.d/common-auth dari 

auth    [success=1 default=ignore]      pam_unix.so nullok_secure\n

untuk 

auth    [success=1 default=ignore]      pam_unix.so nullok_secure nodelay\n

Tidak perlu reboot.
