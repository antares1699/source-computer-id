Ini cukup standar untuk ls untuk menunjukkan ukuran file yang sebenarnya secara default, dan saya baru saja memverifikasi itu. Begitu: 

ls -l <yourfiles>\n

Untuk menghitung total: 

ls -l <yourfiles> | awk '{sum+=$5} END {print sum}'\n

Jika penghitungan Anda perlu menyertakan file dalam subdirektori, solusi yang umum adalah menggunakan find untuk memanggil ls: 

find <yourdir>/. -type f -exec ls -l '{}' \\; | awk '{sum+=$5} END {print sum}'\n

Untuk melihat total hanya satu penggunaan folder tertentu:

\n\n
\n

du -s / path_to / specific_folder /

\n
\n\n

Untuk melihat total setiap folder dalam satu folder tertentu, gunakan tanda bintang:

\n\n
\n

du -s / path_to / specific_folder / *

\n
\n\n

Jika Anda tidak memiliki semua file di direktori tersebut, gunakan sudo untuk memastikan semuanya dihitung.

\n\n

Edit: Ah, saya mengerti apa yang Anda tanyakan sekarang. Kamu ingin:

\n\n
\n

du -sk / path_to / specific_folder /

\n

Anda dapat meminta total dunia Mac (termasuk garpu sumber daya) seperti ini: 

# Put this in a shell function or script, 'macTotal'\nosascript - \"${1:-.}\" <<\\EOF | perl -Mbignum -lpe '$_+=0,\"\\n\"'\non run {arg}\n    alias POSIX file arg\n    tell application \"System Events\" to get size of result\nend run\nEOF\n\n$ macTotal ~/Library\n4465742628\n

AppleScript mencetak nomor dalam notasi ilmiah. Kode Perl adalah cara ceroboh untuk memperluas notasi ilmiah.

\n\n

Jika Anda tidak keberatan membaca angka dari GUI, cukup buka jendela Info folder di Penemu . Ukuran yang dilaporkan sama dengan apa Peristiwa Sistem memberikan di AppleScript.

\n\n
\n\n

Jika Anda hanya peduli tentang garpu data, saya akan menggunakan yang mirip dengan\njawaban pra, tetapi menggunakan stat alih-alih ls dan xargs alih-alih -exec untuk efisiensi yang sedikit lebih. 

$ find . -type f -print0 | xargs -0 stat -f %z | awk '{t+=$1}END{print t}'\n4461971024\n
