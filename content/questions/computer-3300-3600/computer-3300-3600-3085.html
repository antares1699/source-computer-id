Gunakan kode pengembalian svn info. Ini 0 jika di bawah kontrol versi, bukan nol sebaliknya.

\n\n

Shell saya menampilkan kode pengembalian bukan nol setelah perintah, jadi terlihat seperti ini: 

$ svn info trunk\nPath: trunk\nURL: https://(...)/trunk\nRepository Root: https://(...)\nRepository UUID: 651713a4-5a46-7e42-a99e-f31e79777eab\nRevision: 213\nNode Kind: directory\n(...)\n\n$ touch foo\n$ svn info foo \nfoo:  (Not a versioned resource)\n\nsvn: A problem occurred; see other errors for details\n\nrc: 1\n

Bagaimana Anda melakukannya dalam skrip bash (serupa di shell lain), dengan menekan keluaran info: 

svn info <filename> 1>/dev/null 2>&1\necho $?\n

Saya menggunakan ini: 

if [ $(svn status $file | awk '{ print $1 }') == \"?\" ]; then\n    echo \"File is not under version control\"\nfi\n

Dengan cara ini Anda juga dapat menanyakan kode status svn yang berbeda yang dapat dilihat di sini: http://svnbook.red-bean.com/en/1.7/svn.ref.svn.c.status.html

Saya ingin memecahkan masalah serupa: Sebagai bagian dari skrip yang menghasilkan file, secara otomatis menambahkannya ke subversi jika ini belum pernah dilakukan sebelumnya.

\n\n

Saya menemukan yang berikut berhasil: 

if ! svn info ${file} 1>/dev/null 2>&1 ; then\n  svn add ${file}\nfi\n

Menguji nilai keluar di Bash membawa saya ke jawaban ini.
