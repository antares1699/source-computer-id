\n\n

Situs web yang menyediakan skrip yang berjalan di mesin klien (JavaScript menjadi teknologi umum) memungkinkan konten untuk diproses dan diubah tanpa memerlukan perjalanan bolak-balik server. Di sisi lain, skrip sisi server (seperti bahasa seperti PHP) diproses di server, oleh karena itu perlu memuat ulang. Permintaan dikirim ke server, diproses di sana dan halaman baru dikirim kembali ke klien di mana ia "di-refresh".

Pemrograman AJAX memungkinkan Anda memperbarui elemen halaman secara dinamis. Namun, AJAX memiliki sejumlah kelemahan dibandingkan hanya menampilkan halaman web. Ini termasuk:

\n\n
    \n
  • Mereka lebih sulit untuk diprogram. Jauh lebih sulit untuk melakukan pemrograman AJAX, dan sebagai konsekuensinya, orang akan menghindarinya kecuali mereka harus melakukannya.
    • \n
  • Mereka memiliki "Sisi Klien" dan "Kode Sisi Server". Selain menulis kode di sisi server untuk menampilkan halaman, Anda juga harus menulis kode yang berjalan di browser (Internet Explorer, dll) untuk melakukan pembaruan dinamis. Ini membutuhkan waktu lebih lama, dan lebih sulit untuk dipahami.
    • \n
  • Standar Browser. Browser web tertentu (seperti Internet Explorer yang disebutkan sebelumnya) berperilaku berbeda dengan standar yang ditentukan dan browser web lainnya (seperti firefox). Browser lama mungkin tidak bekerja dengan AJAX sama sekali. Browser seluler (di ponsel Anda) mungkin juga tidak bekerja dengan AJAX, atau mungkin bekerja dengan subset dari AJAX. Karena itu, mungkin lebih baik membuat laman web Anda dengan cara yang berfungsi di semua browser dan tidak menggunakan AJAX.
    • \n
  • AJAX masih cukup baru. Karena itu, banyak programmer tidak tahu cara membuat halaman web dinamis. Juga - laman web / situs web lama tidak memiliki AJAX di dalamnya. Terakhir, saat ini terdapat kekurangan pelatihan dan dokumentasi berkenaan dengan cara AJAX melakukan sesuatu.
    • \n

DOM: Model Objek Dokumen. Secara longgar, ini mendefinisikan bagaimana program AJAX menemukan tempat di halaman untuk meletakkan / mengubah elemen baru (mis. Komentar). Sekali lagi, DOM dan cara Anda mengaksesnya berbeda dari browser ke browser. Ketika Anda mendengar orang-orang mengeluh tentang betapa buruknya browser IE6, itu karena akses ke DOM-nya (a) rusak (b) sulit digunakan dan (c) berbeda dengan setiap browser lain di luar sana.
