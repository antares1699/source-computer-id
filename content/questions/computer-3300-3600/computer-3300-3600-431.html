TL; DR

\n\n

Saya mencoba untuk mencari tahu apa yang saya katakan dahulu kala dalam banyak kata.

\n\n

Kedengarannya seperti, tidak ada harapan, mungkin mencoba sakelar /U, bahkan pengalihan shell tidak akan menyelamatkan Anda, mungkin hanya mengubahnya menggunakan program lain.

\n\n

Asli

\n\n

Di Notepad ++, saya bisa membacanya ketika memilih "Encoding" di bawah "Character Sets"> "Western European"> "OEM-US".

\n\n

Setelah itu saya bisa memilih "Encoding"> "Convert to UTF-8", jadi saya bisa mentransfernya dalam format yang mudah-mudahan semua orang bisa membacanya.

\n\n

Sebagai referensi, sistem saya adalah Windows 7 Professional SP1 64-bit (x64), dengan sistem lokal saya saat ini disetel ke "Inggris (Amerika Serikat)".

\n\n

Selain itu, saya yakin pertanyaan ini memiliki lebih banyak informasi dan berperingkat lebih tinggi: Menyimpan hasil 'tree / f / a "ke textfile dengan dukungan unicode

\n\n

Saya juga menjalankan Command Prompt saya dengan opsi "/ U" (melalui ConEmu jika itu penting), dan jawaban di halaman itu menyatakan: Entah itu tidak berfungsi seperti yang diharapkan, atau perintah pohon tidak menghormatinya karena alasan tertentu.

\n\n

Saya sekarang telah menemukan kutipan ini dari sumber berikut, yang mungkin menjelaskan mengapa perintah tree tidak menulis teks yang dikodekan Unicode ke file:

\n\n
\n

Mengeluarkan karakter UNICODE (UCS-2 le)
Pilihan ini akan mempengaruhi perpipaan atau pengalihan ke file.
File teks yang paling umum adalah ANSI, gunakan sakelar ini
ketika Anda perlu mengubah himpunan karakter.

\n
\n\n
