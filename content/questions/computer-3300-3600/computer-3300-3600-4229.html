Nima,\nApakah Anda menggunakan versi 64 bit dari JDK dan versi 64 bit dari Helios? .\nJika tidak, cobalah memiliki keduanya pada 64 bit, sehingga Anda dapat memanfaatkan OS Anda.\nJuga, edit file eclipse.ini Anda untuk meluncurkannya dengan menunjuk ke vm tertentu.\nmisal: 

-vm\nmyjdk/1.6.0_22/bin/javaw.exe\n

[ASK] Masalah diselesaikan dengan menghapus file .snap dari yourworkspace\\.metadata\\.plugins\\org.eclipse.core.resources

\n\n

Penjelasan untuk file snap:\nSaat Eclipse berjalan, informasi tentang apa yang telah berubah di file\nruang kerja secara bertahap masuk ke berbagai file "snapshot"\n(termasuk .snap). Pada shutdown Eclipse normal, ruang kerja lengkap\nstatus disimpan dan file .snap dihapus. Saat Eclipse mogok,\nfile snapshot digunakan selama startup berikutnya untuk memulihkan\njatuh.

\n\n

Jadi, tampaknya terjadi beberapa kesalahan yang menyebabkan Eclipse\njatuh. Kemudian, terjadi kesalahan saat memulihkan dari kemacetan berikutnya\nmemulai. Ini adalah masalah yang sangat mungkin tidak terkait, tetapi secara manual\nmenghapus file snapshot Anda menipu Eclipse agar berpikir bahwa a\ncrash tidak terjadi, sehingga rutinitas pemulihan crash tidak berjalan.

\n\n
