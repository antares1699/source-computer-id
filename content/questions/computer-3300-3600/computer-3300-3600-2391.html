Saya ingin membuat kunci ssh (tanpa kata sandi) yang dapat saya berikan kepada pengguna jarak jauh sehingga mereka dapat melakukan hg pull atau hg clone dari repositori saya.\nSaya telah membuat kunci ssh yang hanya menggunakan perintah / usr / bin / hg tetapi itu tidak berfungsi saat menggunakan perintah klon atau tarik, karena string perintah tidak cocok dengan apa pun panggilan perintah jarak jauh lincah digunakan secara internal.

\n\n

Apakah mungkin untuk mendapatkan perintah yang dijalankan dengan cepat pada host jarak jauh ketika Anda melakukan: 

hg clone ssh://user@host//path-to-repo\nhg pull ssh://user@host//path-to-repo\n

Dan dapatkah saya dengan andal menghasilkan beberapa string perintah yang akan mencakup semua basis saya? Jika mereka akan menarik dengan revisi tertentu, apakah itu akan menggunakan perintah jarak jauh yang berbeda melalui ssh?

","source":"https://superuser.com/questions/47502/creating-an-ssh-key-just-for-mercurial-pulls-clones","author":"goathens","replies":[],"answers":[{"answer":"

Mercurial hadir dengan skrip praktis yang sudah disiapkan untuk jenis akses terbatas ini. Skrip ini mungkin sudah ada di server Anda dan diberi nama hg-ssh, tetapi Anda juga dapat menemukannya di sini: http://www.selenic.com/repo/hg-stable/raw-file/tip/contrib/hg-ssh

\n\n

Untuk menggunakannya, cukup letakkan baris seperti ini di file /home/user/.ssh/authorized_keys Anda: 

command=\"hg-ssh path/to/repo1 /path/to/repo2 ~/repo3 ~user/repo4\" ssh-dss ...\n

yang akan memastikan bahwa orang dengan kunci itu hanya dapat menggunakan mercurial (bukan shell) dan hanya pada repo yang Anda daftarkan.
