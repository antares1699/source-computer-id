Kemungkinan Duplikat:
\n Biarkan skrip bash berjalan di terminal jarak jauh saat tidak masuk?
\n Bagaimana cara melepaskan proses dari Terminal, seluruhnya?

\n
\n\n\n\n

Saya menjalankan program di server linux yang membutuhkan waktu berhari-hari untuk menyelesaikannya.

\n\n

Saya meluncurkannya dari stasiun kerja saya dari terminal SSH, karena program ini hanya baris perintah.

\n\n

Saya ingin bisa melakukan semua ini:

\n\n
    \n
  • luncurkan program itu,
    • \n
  • mengarahkan keluaran standar ke file,
    • \n
  • keluar dari sesi SSH saya tanpa membuat ini menghentikan proses.
    • \n
\n\n

Saya berpikir tentang $ ./MyProg.csh -params -foo -bar </dev/null 1>~/out.log 2>~/err.log &

\n\n

Namun, prosesnya dihentikan saat saya menutup sesi SSH saya. Stasiun kerja saya menjalankan Windows XP, dan saya tidak dapat menjamin waktu aktifnya selama beberapa hari, yang diperlukan untuk pemrosesan data saya di server Linux.

\n\n

Seperti yang telah Anda catat, program saya harus diluncurkan dari CSH.

\n\n

Apakah mungkin melakukan ini?

\n\n

\n\n

Buat pembungkus cangkang seperti ini: 

#!/bin/sh\nexec </dev/null\nexec >> ~/out.log\nexec 2>> ~/err.log\nexec setsid ./MyProg.csh -params -foo -bar\n

Jika Anda membutuhkan file PID juga, kirimkan skrip shell ini ke bg dan baca $! (PID dari proses bg terakhir). Anda dapat melakukannya menggunakan subkulit dengan () dalam skrip yang sama seperti ini: 

#!/bin/sh\n( exec </dev/null\n  exec >> ~/out.log\n  exec 2>> ~/err.log\n  exec setsid ./MyProg.csh -params -foo -bar\n) &\nkill -0 $! > /dev/null 2>&1 || exit 1\necho $! > program.pid\nexit 0\n
