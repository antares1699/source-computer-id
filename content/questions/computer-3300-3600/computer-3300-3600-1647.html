Saya perlu mengekstrak satu folder dan subfoldernya dari tarball (.tar.gz) di server CentOS. Saya sama sekali tidak tahu bagaimana melakukannya menggunakan terminal SSH.

\n\n

Saya mencoba: 

gunzip -c files_20100623.0110.tar.gz | tar -xvf home/bsisplas/public_html/staging/template/*.*\n

Namun tampaknya tidak dapat menemukan file di arsip. Juga, di manakah dalam sintaks gunzip Anda menentukan ke mana file yang diekstrak harus disimpan? Saya seorang noob terminal Linux untuk sedikitnya.

Pertama, saat Anda menggunakan opsi -f hingga tar, Anda perlu memberinya argumen yang memberitahukan nama file arsip. Karena Anda mengumpankannya dari pipa dari gunzip, kami menggunakan - yang berarti input standar: 

gunzip -c files_20100623.0110.tar.gz | tar -xvf - home/bsisplas/public_html/staging/template/*.*\n

Poin saya selanjutnya adalah Anda hanya perlu memberikan nama direktori. (Juga: *.* biasanya adalah DOS-isme. Jika yang Anda maksud adalah "semua file" di Unix, cukup tulis *. Jika Anda menulis *.* Anda mengatakan "semua file dengan titik di suatu tempat di namanya" yang dapat mengecualikan file penting tanpa titik, seperti Makefile atau README): 

gunzip -c files_20100623.0110.tar.gz | tar -xvf - home/bsisplas/public_html/staging/template\n

Seharusnya itu berhasil. Tetapi Anda dapat membuatnya sedikit lebih mudah dengan menggunakan opsi -z gunzip -c files_20100623.0110.tar.gz | tar -xvf - home/bsisplas/public_html/staging/template\n, yang memerintahkannya untuk melakukan gunzip itu sendiri. Kami menggunakannya, dan mengganti nama file masukan - dengan nama file arsip: 

tar -xvzf files_20100623.0110.tar.gz home/bsisplas/public_html/staging/template\n

Bagaimana cara kerjanya?
