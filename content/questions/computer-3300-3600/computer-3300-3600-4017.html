\n\n

Tetapi ketika saya menyimpannya sebagai CSV, itu sebenarnya menimpa data di sel tersebut dengan #######

\n\n

\n\n

Mengklik ganda panjang sel untuk memperlebar itu tidak berfungsi, karena sel tersebut sangat besar bahkan Excel tidak mengembangkannya cukup lama. Memperluas baris dan / atau kolom juga tidak membantu, masih muncul sebagai ###### karena tidak ada cukup ruang untuk menampilkannya di layar / sel.

Masalahnya mungkin Anda memiliki format teks yang diterapkan ke sel yang menyinggung. Dalam sel Excel dengan format Teks, terdapat file membatasi dari 1.024 karakter. Misalnya, dua sel di bawah ini berisi teks yang sama, tetapi A1 diformat sebagai Umum dan B1 diformat sebagai Teks. Saat disimpan sebagai CSV, B1 disimpan sebagai tanda pagar.

\n\n

\n\n

Coba hapus format teks jika tidak benar-benar diperlukan untuk menyajikan data dengan benar, lalu simpan sebagai CSV. Itu seharusnya berhasil.

\n\n

Jika Anda memiliki data panjang yang memerlukan format Teks agar Excel tidak mengubah data Anda (misalnya, urutan panjang angka dengan nol di depan), Anda dapat memasukkan apostrof sebelum teks di bilah rumus, lalu mengonversi format sel menjadi Umum.
