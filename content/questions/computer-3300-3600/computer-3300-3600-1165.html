Saya telah menjalankan beberapa versi terbaru Inkscape (saat ini menggunakan "0.47 + devel"), dan mengalami masalah dengan menu Ekstensi. Sejauh ini, di setiap versi Inkscape yang saya coba, hampir setiap ekstensi menghasilkan kesalahan berikut:

\n\n
\n

Pembungkus lxml yang fantastis untuk libxml2 dibutuhkan oleh inkex.py dan oleh karena itu ekstensi ini. Harap unduh dan instal versi terbaru dari http://cheeseshop.python.org/pypi/lxml/ , atau instal melalui manajer paket Anda dengan perintah seperti: sudo apt-get install python-lxml

\n
\n\n

Saya sudah mencoba instruksi yang tercantum di sana, tentu saja, tidak berhasil. Saya juga menemukan banyak referensi untuk masalah ini di forum, di pelacak bug, dll., Dan dengan demikian juga mencoba: 

sudo easy_install lxml\ncd /Applications/Inkscape.app/Contents/Resources/lib\nmv libxml2.2.dylib libxml2.2.dylib.old\nln -s /usr/lib/libxml2.dylib\n

dan beberapa solusi serupa. Tidak ada yang menghasilkan perubahan apapun dalam perilaku Inkscape.

\n\n

Adakah yang tahu A) apa yang sebenarnya terjadi di sini? Karena dari apa yang saya kumpulkan kesalahannya tidak menggambarkan masalah yang sebenarnya. Dan tentu saja B) solusi sederhana? Saya membutuhkan fitur-fitur itu! :)

Saya mengalami masalah ini dan berhasil menyingkirkannya dengan menginstal lxml secara eksternal (saya kira itulah yang dilakukan oleh perintah easy_install lxml Anda, meskipun saya menggunakan sudo port install py26-lxml sebagai gantinya).

\n\n

Lalu saya pergi ke 

/Applications/Inkscape.app/Contents/Resources/python/site-packages/i386\n

Di bawah ini, ada beberapa direktori, satu untuk setiap versi Python, dan masing-masing berisi salinan lxml. Saya mengganti namanya, memulai ulang inkscape dan ekstensi mulai berfungsi.

\n\n

Jadi 

sudo port install py26-lxml\ncd /Applications/Inkscape.app/Contents/Resources/python/site-packages/i386\nmv 2.3/lxml 2.3/_lxml\nmv 2.4/lxml 2.4/_lxml\nmv 2.5/lxml 2.5/_lxml\nmv 2.6/lxml 2.6/_lxml\n

Bekerja untuk saya.

Saya telah menginstal Inkscape melalui homebrew, jadi ini mungkin tidak berfungsi dengan binari situs web, lihat sini untuk solusi yang mungkin.

\n\n

TL; DR memperbaiki:\nPastikan Anda menggunakan versi sistem python, periksa dengan menjalankan 

which python\n❯ /usr/bin/python\n

Instal pip pengelola paket python, jika belum 

sudo easy_install pip\n❯\n

Gunakan pip untuk menginstal libxml ke dalam sistem Anda python 

sudo pip install lxml\n❯ Collecting lxml…\n

Sirip.

\n\n

Pada dasarnya masalahnya adalah Inkscape di-hardcode untuk menggunakan sistem build python, untuk menghindari konflik dengan build python lainnya. Tapi saya berasumsi dalam pembaruan OSX terbaru mereka telah menghapus perpustakaan libxml dari sistem build.

\n\n

Pertama saya harus menghapus instalasi homebrew saya dari python, itu adalah versi yang sama (2.7.10) dengan sistem yang dibangun sehingga tidak ada kerugian. Kedua, saya perlu menginstal pip ke sistem menggunakan python 

❯ sudo easy_install pip\n

Akhirnya, saya menginstal libxml ke sistem instalasi python 

❯ sudo pip install lxml\n
