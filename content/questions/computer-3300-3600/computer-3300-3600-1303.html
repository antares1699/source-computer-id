Debian menjatuhkan dukungan LSB , jadi lsb-core tidak ada lagi di Debian> = Stretch.

\n\n

Memperbarui

\n\n

Sebagaimana dicatat dengan benar oleh @tknv, beberapa bulan setelah jawaban ini lsb-compat baru telah tersedia. Jadi solusi terbaik untuk masalah ini adalah menginstalnya: 

sudo apt install lsb-compat

Cara merusak (tidak digunakan lagi):

\n\n

Agar berfungsi, Anda juga dapat:

\n\n
    \n
  1. Ubah metadata paket yang menghapus ketergantungan lsb-core
    2. \n
  2. Pastikan keberadaan linker ld-lsb ELF (pernah disediakan oleh lsb-core)
    3. \n
\n\n

Untuk melakukannya, Anda dapat:

\n\n
    \n
  1. Ikuti instruksi yang sudah dilaporkan sini
    2. \n

  2. Periksa nama ld-lsb ELF linker yang diperlukan (tergantung arsitektur): 

    ldd /opt/google/earth/free/googleearth-bin |grep ld
    3. \n

  3. Menggunakan informasi yang dikumpulkan di 2, buat symlink secara manual. Untuk arsitektur 64-bit, perintahnya adalah: 

    sudo ln -s /lib64/ld-linux-x86-64.so.2 /lib64/ld-lsb-x86-64.so.3

    Untuk arsitektur 32-bit I Tebak perintah yang benar adalah: 

    sudo ln -s /lib/ld-linux-x86.so.2 /lib/ld-lsb-x86.so.3
    4. \n
