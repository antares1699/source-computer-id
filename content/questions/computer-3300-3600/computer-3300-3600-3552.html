Saya menemukan bukti paling meyakinkan tentang gejala dan penyebabnya sini :

\n\n

Pada dasarnya, jika server SMB memiliki bagian anonim, tampaknya patch KB3189866 menyebabkan Windows tidak meminta kredensial Anda dan gagal begitu saja. Anda bisa mendapatkan sejumlah pesan kesalahan yang berbeda untuk ini. Dalam kasus saya, itu adalah: 

The error you may get is \"\\\\SERVERNAME\" is not accessible. You might not have \npermission to use this network resource. Contact the administrator of this server \nto find out if you have access permissions. A specified logon session does not \nexist. It may already have been terminated\n

Ada dua solusi, baik:

\n\n
    \n
  1. Copot pemasangan KB3189866 (berguna untuk membuktikan penyebabnya)
    2. \n
\n\n

ATAU

\n\n
    \n

  1. Tambahkan kredensial untuk bagian tersebut ke dalam Windows Credential Manager:

    \n\n

    Sebuah. Buka Panel Kontrol -> Akun Pengguna -> Kelola Kredensial Anda\n-> Kredensial Windows

    \n\n

    b. Untuk setiap server yang ingin Anda akses, tambahkan entri (menggunakan "Tambahkan kredensial Windows") dengan:

    \n\n
      \n
    • Alamat internet atau jaringan: \\\\ SERVERNAME
      • \n
    • Nama pengguna: tamu
      • \n
    • Kata sandi: tamu
      • \n
    2. \n

Sepertinya ada masalah dengan prompt kredensial karena jika saya menulis pernyataan "penggunaan bersih" secara lengkap dan menentukan juga kata sandi, itu berfungsi.

\n\n

net use I: \\\\network_name\\share /USER:domain\\username password\n

Jika saya menghapus kata sandi, itu malah membuat kesalahan.
