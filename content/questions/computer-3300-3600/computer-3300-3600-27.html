Langkah-langkah ini tampaknya berhasil:

\n\n
    \n
  • aktifkan pengguna tamu
    • \n
  • jalankan gpedit.msc, masuk ke Computer Configuration \\ Windows Settings \\ Security Settings \\ Local Policies \\ Security Options, ubah nilai Network access: Sharing and security model for local accounts menjadi guest only
    • \n
\n\n

Mengenai poin ke-2, saya membaca dokumen Microsoft berikut ini:

\n\n

https://docs.microsoft.com/en-us/windows/security/threat-protection/security-policy-settings/network-access-sharing-and-security-model-for-local-accounts

\n\n

Kesimpulan saya setelah membacanya adalah, jika Anda menjalankan server jaringan, Anda harus menggunakan "klasik", jika tidak, tidak masalah menggunakan guest only karena Anda mungkin tidak menginginkan kontrol akses sama sekali, seperti saya.

Berikut adalah metode alternatif yang saya gunakan untuk melakukannya di Windows 10 Pro. Metode ini melibatkan pengaktifan fungsionalitas berbagi folder Publik yang dibangun ke dalam Windows, membuat folder Bersama baru dan mengatur izin Berbagi dan NTFS yang identik dengan folder Publik di bawah direktori Pengguna. Kemudian nonaktifkan share Publik. Metode ini tidak mengubah kebijakan keamanan lokal atau pengaturan registri (yang telah saya lihat diposting di seluruh Internet)

\n\n
    \n
  1. Buka "Jaringan dan Pusat Berbagi" dan klik tautan "Pengaturan berbagi lanjutan".
    2. \n
  2. Luaskan "Semua Jaringan".
    3. \n
  3. Centang "Aktifkan berbagi sehingga siapa pun dengan akses jaringan dapat membaca dan menulis file di folder Publik"
    4. \n
  4. Klik "Matikan berbagi yang dilindungi kata sandi".
    5. \n
  5. Buat folder "Bersama" di drive pilihan Anda di drive apa pun yang Anda pilih.
    6. \n
  6. Aktifkan share dengan mengklik tombol “Advanced Sharing…”.
    7. \n
  7. Setel izin Berbagi ke "Semua Orang", "Kontrol Penuh".
    8. \n
  8. Tetapkan izin Keamanan [NTFS] sama dengan folder "Publik" di bawah direktori C: \\ Users.
    9. \n
  9. Untuk "Interaktif", "Layanan" dan "Batch", setel 2 izin khusus untuk mencocokkan izin di Pengaturan Keamanan Lanjutan,\nTunjukkan Izin Lanjutan.
    10. \n
  10. Opsional: Nonaktifkan berbagi di direktori "Pengguna" yang diaktifkan saat berbagi Folder Publik diaktifkan.
    11. \n
  11. Opsional: Jika beberapa subnet / VLAN perlu mengakses fileshare, masuk ke Windows Defender Firewall, Advanced\n FirewallSettings, klik "Inbound Rules" dan filter berdasarkan File\n andPrinter sharing group dan tipe profil. Di bawah tab Scope,\n modifyeach Inbound firewall rule dan ubah "localesubnet" di bawah\n "RemoteIP Address" untuk menyertakan subnet tambahan yang dibutuhkan\n akses untuk berbagi.
    12. \n
  12. Uji akses ke folder "Bersama" yang baru dibuat.
    13. \n
  13. Periksa dalam "Manajemen Komputer" untuk status Sesi. Ini menunjukkan sebagai "Tamu" adalah akun yang digunakan untuk mengotentikasi.
    14. \n
  14. Ulangi proses untuk beberapa folder bersama dengan akses anonim / tamu.
    15. \n
\n\n
