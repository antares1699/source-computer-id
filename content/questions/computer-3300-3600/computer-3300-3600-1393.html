Tujuan saya adalah memiliki file dengan ukuran asli yang tidak dapat dibuka (atau dipulihkan) di perangkat lunaknya masing-masing seperti MS Office Word. Tujuan saya adalah untuk mensimulasikan keberadaan file sementara saya tidak ingin membagikan konten file apa pun. Idealnya konten file harus sepenuhnya ditimpa dengan byte acak sedangkan ukuran file harus tetap sama seperti aslinya.

\n\n

Saya tahu bahwa terkadang ini dapat dilakukan dengan hex editor atau bahkan notepad di mana seseorang dapat mengubah hanya beberapa byte dan file tersebut menjadi tidak dapat digunakan. Tetapi ini mungkin tidak selalu berfungsi untuk beberapa format seperti gambar dan audio yang masih dapat dibuka tetapi dengan artefak.

\n\n

Untuk pengguna yang paling ingin tahu masih menuntut saya untuk menjawab pertanyaan itu "Mengapa Anda membutuhkan ini?" - Saya memerlukan ini karena saya tidak ingin membagikan perkembangan saya dengan beberapa orang yang memaksa saya untuk membagikan file. Terkadang Anda hanya perlu. Anda dapat memikirkan banyak kasus penggunaan lainnya - untuk mensimulasikan payload untuk penyalinan file atau operasi pencadangan, atau untuk menguji perangkat lunak yang Anda kembangkan dapat menangani sampah dengan baik.

\n\n

Saya telah menguji sejumlah utilitas rusak aman yang dapat menimpa file dengan data acak tetapi setelah itu file tersebut dihapus. Saya belum melihat opsi untuk mematikan langkah penghapusan. Jika opsi seperti itu ada, itu akan menjadi solusi.

Saya telah menulis pendekatan cepat menggunakan Pembuat File Data Acak .

\n\n
\n

Mike mencatat bahwa RDFC tidak mengisi file dengan data acak dengan benar. Saya telah mengirim email kepada penulis dan melaporkan masalah ini. Untuk saat ini, saya sudah menerapkan pengganti drop-in untuk rdfc.exe . Jika Anda ingin mengkompilasi dan menggunakannya, silakan lihat Bagaimana Anda menyusun proyek dari sumber?

\n
\n\n

Skrip baris perintah Windows ini akan melakukan apa yang Anda inginkan: 

@ECHO OFF\nSETLOCAL\nIF \"%1\"==\"\" GOTO missingParam\nGOTO :begin\n\n:missingParam\nECHO Missing target. Use %~nx0 [DRIVE:\\]DIRECTORY\nGOTO :eof\n\n:begin\nSET RDFC=%CD%\\rdfc.exe\nPUSHD %1\nCALL :treeProcess\nPOPD\nENDLOCAL\nGOTO :eof\n\n:treeProcess\nFOR %%f IN (*.*) DO (\n  ECHO | SET /P=Shredding %%f... \n  %RDFC% %%f %%~zf B overwrite > NUL\n  ECHO %%~zf random bytes written.\n)\nFOR /D %%d IN (*) DO (\n    CD %%d\n    CALL :treeProcess\n    CD ..\n)\nEXIT /B\n

Ini mengharuskan rdfc.exe ditempatkan di direktori yang sama dengan skrip. Skrip akan membutuhkan 1 parameter, direktori target. Setiap file di direktori target akan diganti dengan file dengan ukuran yang persis sama, tetapi diisi dengan data acak.

Tidak membutuhkan alat eksternal. Sebagai bonus, ini mempertahankan waktu LastWrite. Waktu lain tidak diubah. 

# Get file as item to access content and file dates\n$file = Get-Item \"C:\\Users\\Nixda\\Desktop\\input.txt\" \n\n# Save write time to temp variable for later use\n$oldWriteTime = $file.LastWriteTime \n\n# Create an empty array of the same size as your file size in bytes\n$content = New-Object Byte[] $file.length \n\n# Fill the array using a System.Random object. Here is the magic\n(New-Object Random).NextBytes($content)\n\n# Write the array to the same file with System.IO.File\n[IO.File]::WriteAllBytes($file,$content)\n\n# Set back old write time\n$file.LastWriteTime = $oldWriteTime\n
