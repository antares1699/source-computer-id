Ini benar-benar membuat frustrasi. Saya memiliki mesin virtual Windows 7 di dalam host Windows 7. Saya dapat menampilkannya dengan baik dan berfungsi dengan baik, tetapi salin dan tempel antara tamu dan tuan rumah tidak berfungsi.

\n\n

Jadi, masuklah ke menu Tools di jendela Windows Virtual PC dan klik Enable Integration Features, karena saya berasumsi begitulah cara Anda mengaktifkan salin / tempel. Ini menampilkan dialog yang meminta saya untuk memasukkan kredensial saya. Secara default, ini meminta saya untuk memasukkan kata sandi ke akun domain yang saya gunakan untuk masuk ke PC host. Saya memasukkan kata sandi dan dikatakan 'Upaya masuk gagal'.

\n\n

Kemudian saya berpikir mungkin itu meminta saya untuk akun di mesin virtual yang sebenarnya. Jadi saya mencoba masuk dengan akun mesin virtual lokal. Kali ini, ia memikirkannya sekitar 10-20 detik, lalu memberi saya hasil `Upaya masuk gagal 'yang sama.

\n\n

Fakta yang relevan:

\n\n
    \n
  • Host ada di domain.
    • \n
  • Tamu tidak digabungkan ke dalam domain
    • \n
  • Tuan rumah adalah Windows 7 64 bit
    • \n
  • Tamu adalah Windows 7 Professional N (32 bit)
    • \n
\n\n

Apa yang saya lewatkan?

\n\n

EDIT: Saya menemukan jawabannya.

\n\n

Kredensial yang diinginkannya adalah untuk OS tamu. Namun, Pengguna yang saya buat tidak memiliki kata sandi. Saya memberinya kata sandi, memasukkannya dan semuanya baik-baik saja sekarang.

\n\n

Jenis gotcha yang buruk. Saya berharap itu didokumentasikan di suatu tempat.

* Saya tahu Anda sudah mengetahuinya. :)

\n\n

Saat Anda mengaktifkan 'Fitur Integrasi', cara Anda berinteraksi dengan VM akan berubah.

\n\n

Ketika mereka diinstal dan dinyalakan itu memulai mesin di latar belakang dan kemudian memasukkan Anda ke dalamnya melalui RDP.

\n\n

Jadi seperti penggunaan RDP 'tradisional', Anda memerlukan kredensial untuk membuat sesi, dan tidak dapat melakukan remote desktop ke akun yang tidak memiliki kata sandi.

\n\n

Selain itu, hal-hal seperti pengaturan resolusi, salin / tempel, koneksi drive, dll. Bekerja sama seperti saat Anda melakukan RDP ke mesin jarak jauh (karena memang demikian. :)).

\n\n

Ini cukup keren ketika Anda memiliki beberapa RDP yang terbuka dan satu atau dua VM, karena Anda tidak dapat membedakannya dengan interaksi, itu menjadi sangat mulus.

\n\n
