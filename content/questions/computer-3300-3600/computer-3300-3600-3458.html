Selama beberapa minggu terakhir, saya telah memperhatikan kecepatan penjelajahan web saya terhenti, dan punya alasan kuat untuk mencurigai Firefox adalah penyebabnya. Ini meskipun dalam edisi Rilis (yaitu stabil) dan selalu memperbaruinya ke versi terbaru.

\n

Firefox merekomendasikan Refresh untuk masalah seperti ini - masalah dengan Refresh, bagaimanapun, adalah bahwa itu berarti semua ekstensi dan preferensi yang telah saya kumpulkan, sesuaikan, dan perbaiki selama beberapa tahun akan hilang:

\n
\n

Item dan pengaturan ini akan dihapus:

\n

Ekstensi dan tema, izin situs web, preferensi yang diubah, mesin pencari yang ditambahkan, penyimpanan DOM, sertifikat keamanan dan pengaturan perangkat, tindakan unduhan, pengaturan plugin, penyesuaian bilah alat, gaya pengguna, dan fitur sosial akan dihapus.

\n
\n

Dari: Segarkan Firefox

\n

Bagaimana cara melakukan Penyegaran Firefox tanpa kehilangan semua data ini, dan dengan itu, menjadi bagian penting dari pengalaman penjelajahan saya?

Pertama-tama, perhatikan bahwa ada alasan bagus Firefox Refresh menghapus ekstensi dan preferensi Anda - ini biasanya merupakan penyebab yang paling mungkin dari masalah seperti kelambatan dan bug yang ditemukan di Firefox.

\n\n

Artinya, Menyegarkan Firefox dan segera memulihkan semuanya dari profil pengguna lama Anda yang terpengaruh dapat membuat profil Anda diperbarui dan mengalami masalah yang sama. Oleh karena itu, jika Anda mampu mengatasi ketidaknyamanan ini, membuat Firefox dari awal lagi adalah pilihan yang lebih unggul dan lebih ramping untuk mengimpor data lama.

\n\n

Karena itu, Refresh setelah semua data Anda dicadangkan tidak ada salahnya, dan bisa menjadi metode tercepat untuk menghilangkan masalah. Jika, setelah memulihkan data lama Anda setelah Refresh, Anda masih melihat masalah yang sama terjadi, cukup coba Refresh lain tanpa menyalin data, atau dengan menyalinnya sedikit demi sedikit, untuk mengisolasi penyebab sebenarnya dari masalah tersebut.

\n\n
\n\n

Mencadangkan profil Firefox Anda

\n\n

Firefox Refresh membuat profil pengguna baru dengan hanya data yang paling penting, dan secara default, memindahkan profil pengguna lama Anda (dan dengan itu, semua data Anda) ke folder bernama Data Firefox Lama di Desktop Anda. Ini hampir selalu cukup, tetapi jika folder ini tidak dibuat, Anda dapat melanjutkan dan membuat cadangan sekunder dengan mengikuti langkah-langkah di bawah ini.

\n\n

Anda harus membuka folder tempat profil Firefox Anda saat ini disimpan. Untuk melakukan ini, klik Tolong > Informasi pemecahan masalah dari bilah menu Firefox. Cari Folder Profil di bawah tajuk Dasar-Dasar Aplikasi, dan klik Folder terbuka .

\n\n

Di folder yang terbuka, catat nama folder pengguna tempat Anda berada, lalu gunakan pengelola file Anda untuk naik satu direktori. Di sini, akan ada dua atau lebih folder - satu untuk setiap profil pengguna yang saat ini ada untuk instalasi Firefox Anda, ditambah satu folder default. Seringkali, jika Anda sudah memiliki Firefox untuk waktu yang cukup lama, itu dapat mengakumulasi beberapa profil pengguna. Salin folder pengguna yang sebelumnya Anda masuki ke Desktop atau di tempat lain di dalam folder pengguna Anda untuk mencadangkannya.

\n\n

Menyegarkan Firefox

\n\n

Ikuti instruksi di sini untuk menyegarkan Firefox.

\n\n

Setelah Refresh selesai, Firefox akan memberi tahu Anda apa yang diimpor. Ini harus menyertakan anda:

\n\n
    \n
  • Bookmark (disimpan dalam file places.sqlite)
    • \n
  • Browsing dan download history (disimpan di file places.sqlite)
    • \n
  • Kata sandi dan login (disimpan di file key4.db dan logins.json)
    • \n
  • Buka jendela dan tab (disimpan dalam file sessionstore.jsonlz4)
    • \n
  • Cookies (disimpan di file sessionstore.jsonlz4)
    • \n
  • Informasi pengisian otomatis formulir web (disimpan dalam file formhistory.sqlite)
    • \n
  • Kamus pribadi (disimpan dalam file persdict.dat)
    • \n
\n\n

Segala sesuatu yang tidak terlihat di atas sekarang akan hilang. Jika Anda telah menghabiskan banyak waktu untuk menyesuaikan pengaturan penjelajahan Anda, Firefox sekarang akan terasa asing.

\n\n

Memulihkan ekstensi, preferensi, dan data lama

\n\n

Anda harus memulihkan semua yang lain agar Firefox berperilaku seperti yang Anda harapkan lagi.

\n\n

Arahkan ke Desktop Anda. Jika Anda melihat folder bernama Data Firefox Lama , Anda tidak memerlukan cadangan sekunder dari profil pengguna yang Anda buat sebelumnya, dan dapat menghapusnya dengan aman. Buka Data Firefox Lama folder, dan kemudian satu-satunya folder profil pengguna di dalamnya. Anda sekarang harus melihat konten lengkap dari profil Firefox Anda sebelumnya.

\n\n

\n\n

Sekarang Anda harus menavigasi ke profil pengguna yang baru dibuat oleh Refresh.

\n\n

Buka Firefox, dan dari Menu Bar, klik Tolong > Informasi pemecahan masalah . Di Jendela yang terbuka, klik Folder terbuka lanjut ke Folder Profil bidang.

\n\n

Anda sekarang harus mentransfer file dan folder sesuai kebutuhan dari profil pengguna lama di Data Firefox Lama untuk yang baru ini digunakan oleh Firefox.

\n\n

Apa yang Anda salin bergantung pada data apa yang ingin Anda pulihkan dari profil lama Anda. File dan folder kunci yang berisi data belum disimpan oleh Refresh ditunjukkan di bawah ini.

\n\n
    \n

  • Itu bookmarkbackups folder berisi cadangan bookmark harian yang dibuat secara otomatis oleh Firefox, hingga maksimal 15 cadangan. Salin folder ini jika Anda ingin memulihkan bookmark dari beberapa minggu terakhir.

    • \n

  • Itu browser-ekstensi-data dan ekstensi-data folder berisi data yang disimpan dan digunakan oleh ekstensi. Saya membayangkan banyak, jika tidak sebagian besar, ekstensi browser akan sangat tidak berguna tanpa data yang disimpan dalam folder ini, jadi jika Anda berencana untuk mempertahankan ekstensi, Anda mungkin harus mempertahankan folder ini.

    • \n

  • Itu chrome folder berisi userChrome.css mengajukan. Kecuali kalau Anda telah mengedit anda userChrome.css dan Anda ingin menyimpan hasil edit yang telah Anda buat, Anda tidak perlu menyalin folder ini. Perhatikan bahwa "chrome" di sini, dan di tempat lain di bagian dalam Firefox, merujuk pada istilah tersebut browser chrome , bukan Google Chrome.

    • \n

  • Itu content-prefs.sqlite dan izin.sqlite file menyimpan izin Firefox, seperti situs mana yang diizinkan untuk menampilkan sembulan, atau tingkat zoom kustom apa pun yang telah ditetapkan di situs.

    • \n

  • Itu ekstensi folder berisi semua ekstensi yang dipasang untuk Firefox, termasuk yang ditandai sebagai warisan setelah Firefox 57 (Quantum).

    • \n

  • Itu handlers.json preferensi penyimpanan file yang memberi tahu Firefox apa yang harus dilakukan saat menemukan jenis file tertentu, seperti membuka file PDF dengan Acrobat Reader saat Anda mengekliknya.

    • \n

  • Itu prefs.js dan user.js (jika ada) file berisi sebagian besar preferensi pengguna, termasuk semua pengaturan Opsi Firefox yang disimpan.

    • \n

  • Itu search.json.mozlz4 file menyimpan semua mesin telusur ubahsuaian yang Anda tambahkan ke bilah Pencarian Firefox.

    • \n

  • Itu xulstore.json file menyimpan toolbar dan pengaturan ukuran / posisi jendela.

    • \n
\n\n

Memastikan Firefox ditutup, salin salah satu dari yang di atas sesuai kebutuhan - dengan menggunakan Bergeser + Klik untuk memilih beberapa item sekaligus - rekatkan ke profil pengguna Firefox baru, dan konfirmasikan semua timpaan atau penggabungan saat diminta.

\n\n

Setelah selesai, dan Firefox dibuka kembali, itu akan terlihat dan berperilaku persis seperti sebelum Refresh - mudah-mudahan tanpa bug yang mengharuskannya di tempat pertama.
