Untuk RHEL / Fedora, Anda mungkin ingin melihat booting PXE dan menggunakan kickstart untuk menginstal. Menggunakan "gambar emas" (yaitu melakukan apa yang dilakukan SysPrep) sebenarnya bukan cara terbaik untuk menerapkan mesin baru di RHEL / Fedora (menurut saya). Ini (Kickstart) memungkinkan Anda mendistribusikan gambar yang identik tanpa harus mengambil ruang untuk gambar itu - dan memungkinkan Anda memperbarui mesin baru dengan hanya mengedit file / template kickstart alih-alih harus memperbarui gambar emas, lalu menyiapkannya kembali, lalu verifikasi ulang itu. Saya diberitahu Ubuntu memiliki fungsionalitas "pra-benih" yang secara substansial mirip, tetapi saya tidak terbiasa dengannya jadi saya tidak bisa mengatakan apa-apa selain "Saya tahu namanya".
