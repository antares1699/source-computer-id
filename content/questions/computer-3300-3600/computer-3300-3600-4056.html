Saya memiliki mesin windows 7 yang saya gunakan terutama sebagai server media (NAS). Untuk alasan ini, saya tidak ingin meminta kata sandi saat mesin melakukan booting. Jadi, saya telah menonaktifkan kata sandi untuk akun utama, dan komputer melakukan booting tanpa hambatan. Namun, untuk alasan keamanan, saya telah menonaktifkan Remote Desktop dan telah menggunakan LogMeIn untuk mengakses komputer dari jarak jauh. LogMeIn memiliki kekurangannya, dan saya lebih suka melakukan remote ke mesin secara langsung menggunakan koneksi desktop jarak jauh (MSTSC).

\n\n

Adakah cara agar saya dapat meminta kata sandi saat membuat koneksi desktop jarak jauh, tetapi masih mengizinkan komputer untuk boot tanpa menyediakannya?

","source":"https://superuser.com/questions/441308/only-require-password-for-remote-desktop","author":"Nick Heidke","replies":[],"answers":[{"answer":"

Anda harus mengatur kata sandi untuk pengguna.
Setelah selesai, cukup konfigurasikan Windows untuk masuk secara otomatis di konsol (sangat lokal).
Untuk melakukannya, Anda dapat mengedit registri: 

[HKEY_LOCAL_MACHINE\\SOFTWARE\\Microsoft\\Windows NT\\CurrentVersion\\Winlogon]\nDefaultUserName = user_name\nAutoAdminLogon = 1\nDefaultPassword = xxxxx\n

Anda juga bisa melihat-lihat di sini: Apakah mungkin untuk masuk secara otomatis di Windows XP?

\n\n

Ini hanya berfungsi di konsol (keyboard dan layar lokal). Jadi desktop jarak jauh akan meminta kata sandi.
