Versi OpenOffice, yang semestinya mudah diadaptasi ke Excel (menurut saya satu-satunya perbedaan adalah OO menggunakan titik koma untuk memisahkan argumen fungsi, dan Excel menggunakan koma):

\n\n

Diberikan dua blok data berlabel "Kumpulan Data 1" (ditampilkan di bawah ini di sel A3: B6) dan "Kumpulan Data 2" (ditampilkan di bawah di sel D3: E6):

\n\n
    \n
  1. Salin Kumpulan Data 1 ke rentang baru (diperlihatkan di bawah ini di sel A10: B13).
    2. \n

  2. Di sebelah kanan Kumpulan Data 1 (diperlihatkan di sel D10), masukkan rumus berikut: 

    =IF(ISNA(VLOOKUP($A10;$D$3:$E$6;1;0));\"\";VLOOKUP($A10;$D$3:$E$6;1;0))\n
    3. \n

  3. Berdekatan dengan sel ini (diperlihatkan di sel E10, masukkan rumus berikut: 

    =IF(ISNA(VLOOKUP($A10;$D$3:$E$6;2;0));\"\";VLOOKUP($A10;$D$3:$E$6;2;0))\n
    4. \n

  4. Salin dan tempel sel D10: E10 ke sel D11: E13.

    5. \n
\n\n

Ide di balik ini adalah menggunakan VLOOKUP untuk menemukan sel yang cocok dengan nilai di kolom A. Jika sel yang cocok adalah tidak ditemukan (mis., fungsi VLOOKUP mengembalikan nilai N / A), letakkan string kosong ke dalam konten sel. Jika sel yang cocok adalah ditemukan , letakkan hasil VLOOKUP ke dalam konten sel.

\n\n

Beginilah cara saya melakukannya di bawah Excel, berdasarkan jawaban Mike Renfro:

\n\n

Diberikan dua blok data berlabel "Kumpulan Data 1" (ditampilkan di bawah ini di sel A3: B6) dan "Kumpulan Data 2" (ditampilkan di bawah di sel D3: E6):

\n\n
    \n
  1. Salin Kumpulan Data 1 ke rentang baru (diperlihatkan di bawah ini di sel A10: B13).
    2. \n

  2. Di sebelah kanan Kumpulan Data 1 (diperlihatkan di sel D10), masukkan rumus berikut: 

    =IFERROR(VLOOKUP($A10,$D$3:$E$6,COLUMN()-COLUMN($D10)+1,0),\"\")\n
    3. \n

  3. Salin dan tempel rumus ini ke D10: E13

    4. \n
\n\n

Perbedaan dari jawaban Mike:

\n\n
    \n
  1. Daripada memasukkan nomor kolom secara manual, saya menggunakan rumus COLUMN.
    2. \n
  2. Daripada melakukan VLOOKUP dua kali, saya melakukannya sekali, dan kemudian menggunakan IFERROR jika tidak dapat menemukan apa pun.
    3. \n
  3. Saya menggunakan koma dan bukan titik koma, seperti yang dikatakan Mike.
    4. \n
