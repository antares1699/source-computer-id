Saya ingin dapat terhubung dari Windows 7 Explorer ke saham SMB di sistem yang berbeda, menggunakan kredensial yang berbeda pada waktu yang berbeda.

\n\n

Bagaimana Anda membuat Explorer menawarkan prompt untuk menggunakan kredensial yang berbeda? Tampaknya jika mengetahui kredensial untuk jalur berbagi jaringan, ia akan menggunakannya.

\n\n

Atau sebaliknya - jika ia mengetahui beberapa kredensial untuk server itu, tetapi Anda memilih bagian yang tidak dimiliki kredensial tersebut, maka Explorer hanya akan mengatakan "Anda tidak memiliki izin".

\n\n

Dan "Disconnect Drive" tampaknya tidak berlaku di sini - untuk satu hal, drive jaringan tidak muncul di daftar itu kecuali dipetakan. Dan jika mereka dipetakan, Anda dapat memutuskannya, tetapi ketika Anda menjelajah kembali ke sana ... Explorer menggunakan kredensial yang diingatnya dan masuk langsung tanpa prompt.

\n\n

Dalam satu kasus yang menempatkan item kembali ke daftar penggunaan bersih, untuk bagian lain tidak. Bagaimanapun, tetap tidak dapat mengubah ke kredensial lain.

Saya kemudian menemukan ini terjadi jika (a) beberapa jendela Explorer sedang menjelajahi share, atau (b) saya kebetulan menjalankan perangkat lunak VPN (dalam kasus saya NetExtender).

Di command prompt windows Anda dapat menggunakan yang berikut ini daftar kredensial yang diingat: 

net use\n

Outputnya akan mirip dengan: 

New connections will be remembered.\n\n\nStatus       Local     Remote                    Network\n\n-------------------------------------------------------------------------------\nOK                     \\\\123.123.123.123\\shared      Microsoft Windows Network\nThe command completed successfully.\n

Kemudian Anda dapat menggunakan perintah berikut untuk menghapus kredensial: 

net use <path> /delete\n

Untuk contoh di atas akan menjadi: 

net use \\\\123.123.123.123\\shared /delete\n
