Mengapa Excel bisa mengabaikan celah data dan menghubungkan titik dengan garis?

\n\n

Saya memiliki tabel ini yang dipetakan sebagai grafik ini. Apakah celah di tabel kosong (""), atau "# N / A # atau" #VALUE! ", Excel memplotnya sebagai nol pada sumbu X. Saya mencoba menyuruhnya untuk menghubungkan data dengan garis di menu untuk sel tersembunyi dan kosong. Itu tidak membuat perbedaan.

\n\n

Bagaimana cara memperbaiki grafik ini?

\n\n

Memiliki pengembalian IFERROR # N / A tidak membuat perbedaan. Jika saya membuat kode keras # N / A, menulisnya dengan tangan, saya bisa membuatnya berfungsi. Tapi ketika dikembalikan oleh fungsi IFERROR dengan argumen INDIRECT, Excel masih memplot celah tersebut sebagai nol. Lihat gambarnya.

Dalam contoh Anda, Anda menggunakan = IFERROR (bagus!) Untuk mengembalikan nilai "#VALUE!" (buruk!). Rumus Anda memberi tahu Excel untuk memasukkan nilai teks "#VALUE!", Yang bukan 0, # N / A, atau "" dan karenanya memplot 0 dan mengabaikan permintaan Anda untuk Menyambungkan Data dengan Garis.

\n\n

Perbarui pernyataan = IFERROR Anda untuk mengembalikan NA () sebagai argumen keduanya. Excel akan mengenali itu sebagai kesalahan "yang dapat diabaikan" saat membuat bagan dan membiarkan poin kosong.

\n\n

\n\n

FWIW, saya tidak tahu untuk memaksa kesalahan apa pun selain # N / A berdasarkan fungsi-sisanya harus terjadi karena kondisi kesalahan yang dihitung.
