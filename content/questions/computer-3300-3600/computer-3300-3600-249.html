Secara teori, sebaiknya aman untuk hanya menghentikan program (dalam kasus Anda, Anda dapat menggunakan CTRL-C) karena upaya dibuat dalam desain aplikasi Microsoft Disk Defragmenter untuk mencegah kehilangan data jika terjadi gangguan yang tidak terduga. Ini didokumentasikan di situs web Microsoft sebagai berikut: 

The following steps are the processes by which Disk Defragmenter and\nfile system defragment an NTFS volume and update the MFT.\n\n   1. Locate the fragments of each file throughout the disk.\n   2. Copy the fragments to contiguous locations on the disk.\n   3. Verify that the copy is an exact duplicate of the original file.\n   4. Update the MFT in NTFS with the new location of the beginning of\n      the file.\n   5. Free up the space previously used by the file.\n

Microsoft - Cara Kerja Disk Defragmenter
http://technet.microsoft.com/en-us/library/cc778290(WS.10).aspx
(teks di atas dapat ditemukan di bawah " Memperbarui Tabel File Master ")

\n\n

Pada dasarnya, datanya adalah disalin ke sektor baru, penunjuk ke data yang dipindahkan diperbarui untuk mengarah ke sektor baru ini, dan sebagai hasilnya, sektor lama dibebaskan. Ini adalah pendekatan yang sangat aman untuk digunakan jika terjadi crash karena dalam skenario terburuk (seperti tabrakan ke layar biru) Anda akan memiliki dua salinan dari data yang sama, satu di sektor yang tidak terisi, dan yang lainnya di sektor yang dialokasikan. Dari perspektif membaca data, itu akan tampak sama.

\n\n

Selain itu, ada banyak perbaikan yang dapat dilakukan pada teknik Defragmentasi Disk yang digunakan Microsoft (yang sudah lebih dari cukup untuk banyak orang), dan berbagai produk alternatif diperiksa dan dibandingkan dengan sangat rinci di sini:

\n\n

Defragger (dari Daftar Istilah Java Roedy Green)
http://www.mindprod.com/jgloss/defragger.html
