Apakah mungkin untuk menginstal sistem operasi ke drive USB, dan kemudian boot ke sana, semuanya di dalam VirtualBox? Tidak seperti meletakkan disk virtual pada drive USB. Menginstal seluruh OS di sana agar dapat di-boot dari komputer lain juga. Untuk lebih spesifik, OS yang saya rencanakan untuk dipasang adalah Lengkungan .

\n\n

Edit, untuk kejelasan:

\n\n

Saya memiliki Mac sebagai tuan rumah. Dan saya berencana untuk menggunakan Arch sebagai host sementara. Saya berencana untuk menjalankan LiveCD di VirtualBox dan kemudian menginstal Arch bukan ke drive virtual VirtualBox, tetapi ke drive USB yang kemudian saya dapat mem-boot komputer lain. VirtualBox akan menjadi cara sederhana untuk menjalankan LiveCD. Setelah ini, saya ingin melakukan beberapa konfigurasi / pengujian di VirtualBox, dengan drive USB itu. Apakah saya dapat melakukan booting dari drive USB, seolah-olah itu adalah LiveCD?

\n\n

Perhatikan bahwa saya juga ingin disk ini dapat di-boot dengan sempurna dari komputer lain, seperti distro LiveUSB.

VirtualBox BIOS tidak memungkinkan Anda untuk boot dari USB.

\n\n

Namun, Anda dapat menggunakan produk yang disebut Plop Boot Manager untuk mengatasi ini. Plop dilengkapi dengan image iso USB yang dapat di-boot yang akan mendeteksi perangkat USB yang dapat di-boot di Vbox vm.

\n\n

Dengan menggunakan metode ini, saya dapat mem-boot stik USB OpenBSD yang saya miliki, tetapi saya perlu mengkonfigurasi Vbox untuk USB v1.1, bukan 2.0. Mungkin saja untuk mengatasi ini ... Saya hanya tidak perlu melakukannya.

Saya masih merasa belum sepenuhnya jelas apa yang Anda coba lakukan. Akan sangat membantu jika Anda menggunakan kata-kata guest dan host dalam pertanyaan Anda untuk membantu memperjelas apa yang sebenarnya Anda bicarakan.

\n\n

Pertama - baca tentang cara kerja penyimpanan virtual dan opsi apa yang tersedia untuk Anda di manual online, bab 5: http://www.virtualbox.org/manual/ch05.html

\n\n

Kedua - jelas Anda ingin menginstal setidaknya satu sistem operasi ke media USB Anda, dan sungguh jika Anda menginstal host dan tamu ke media USB, itu sedikit di luar cakupan pertanyaan ini. Tetapi begitu Anda menginstal host (di mana pun itu akan berada) - tampaknya tamu pasti akan diinstal pada drive USB.

\n\n

Anda telah menyebutkan secara khusus bahwa Anda tidak ingin memiliki file disk virtual yang ada di drive, jadi saya rasa ingin seluruh drive berfungsi sebagai drive virtual. Ada jenis khusus dari disk virtual yang disebut "hard disk mentah". Anda masih memerlukan file VMDK di suatu tempat yang memberi tahu VirtualBox partisi mana yang akan digunakan dalam mode RAW, jadi Anda mungkin ingin benar-benar mempartisi flash drive. Ini mungkin juga bermasalah tergantung pada host Anda karena jalur fisik ke drive / partisi dapat berubah tergantung pada bagaimana penyimpanan Anda diinisialisasi (khususnya, urutannya) - ketika Anda mengatur ini, flash drive Anda mungkin / dev / sdc tetapi lain kali Anda mem-boot (atau menggunakannya di komputer lain) drive Anda mungkin muncul sebagai / dev / sdb. Bab 9 berbicara tentang format mentah secara lebih rinci: http://www.virtualbox.org/manual/ch09.html#rawdisk

\n\n

Akan sangat membantu untuk membantu menyatakan tujuan Anda atau kendala apa yang sedang Anda tangani untuk mendorong Anda menginginkan jenis penyiapan ini. Saat ini, hal terbaik yang dapat kami tawarkan adalah beberapa nasihat umum, membuat asumsi tentang hal itu, dan mudah-mudahan mengarahkan Anda ke jalan yang benar. Saya mempertimbangkan untuk menandai pertanyaan ini karena seperti yang dinyatakan pada awalnya, pertanyaan ini tidak dapat dijawab - tetapi menurut saya pertanyaan ini dapat diselamatkan dengan sedikit lebih banyak masukan dari Anda.
