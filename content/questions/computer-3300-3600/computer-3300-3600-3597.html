Registri (HKLM \\ SOFTWARE \\ Microsoft \\ Windows \\ CurrentVersion \\ Explorer \\ ShellIconOverlayIdentifiers), namun, hanya mencantumkan: OneDrive1, OneDrive2, OneDrive3, OneDrive4, OneDrive5. Setelah beberapa kali mengaduk-aduk, berikut hasilnya:

\n\n
    \n
  1. Tidak menyinkronkan (x merah)
    2. \n
  2. Disinkronkan - Dibagikan
    3. \n
  3. Sinkronisasi - Berbagi
    4. \n
  4. Disinkronkan
    5. \n
  5. Menyinkronkan
    6. \n
\n\n

Bagi yang penasaran, registri memiliki CLSID dari masing-masing kunci di atas. Anda kemudian dapat memuat objek COM itu dan menanyakan di mana letak ikon (file plus nomor gambar). Kemudian Anda dapat melihat di file untuk menentukan ikon (menggunakan misalnya, IconsExtract Nirsoft). Berikut beberapa contoh kode dari proyek VS2013: 

    #include \"stdafx.h\"\n    #include <Shlobj.h>\n    #include <iostream> \n    #include <tchar.h>\n\n    int _tmain(int argc, _TCHAR* argv[]){\n      LPCOLESTR str_clsid = L\"{FB314EE0-A251-47B7-93E1-CDD82E34AF8B}\";\n      HRESULT hr;\n      CLSID clsid;\n      IShellIconOverlayIdentifier* pISIOI;\n      DWORD dwFlags;\n      int Index;\n      wchar_t* wszIconFile = new wchar_t[128];\n\n      hr = CLSIDFromString(str_clsid, (LPCLSID)&clsid);\n      CoInitialize(NULL);\n      hr = CoCreateInstance(clsid, NULL, CLSCTX_INPROC_SERVER, IID_IShellIconOverlayIdentifier, (void**)&pISIOI);\n      hr = pISIOI->GetOverlayInfo(wszIconFile, 128, &Index, &dwFlags);\n      std::wcout << L\"Icon Path:\\n    \" << wszIconFile << L\" \" << Index << std::endl;\n      pISIOI->Release();\n      CoUninitialize();\n      return 0;\n    }\n

\n\n
    \n
  • OneDrive1 - Kelas ErrorOverlayHandler
    • \n
  • OneDrive2 - Kelas SharedOverlayHandler
    • \n
  • OneDrive3 - Kelas UpToDateCloudOverlayHandler
    • \n
  • OneDrive4 - Kelas UpToDatePinnedOverlayHandler
    • \n
  • OneDrive5 - Kelas SyncingOverlayHandler
    • \n
  • OneDrive6 - ReadOnlyOverlayHandler Class
    • \n
  • OneDrive7 - Kelas UpToDateUnpinnedOverlayHandler
    • \n
\n\n
