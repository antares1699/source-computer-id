Saya cukup yakin Anda tidak dapat membuat tautan keras ke folder, hanya file. Tautan simbolik /D dan persimpangan /J akan berfungsi untuk folder.

\n\n

Juga saat membuat tautan keras, perlu diingat bahwa Anda tidak dapat membuat tautan antara 2 drive yang berbeda (bahkan pada hard drive fisik yang sama).

\n\n

Penjelasan singkat dari SU lainnya menjawab :

\n\n
\n

Tautan keras adalah fitur sistem file yang tidak dapat melewati batas sistem file. Anda tidak dapat menautkan file di C: ke D: karena mereka adalah sistem file terpisah. Masing-masing mungkin berisi jenis sistem file yang sama (mis. NTFS) tetapi mereka adalah sistem file yang terpisah.

Tidak, Anda BISA membuat hard link ke folder / direktori. Ini disebut persimpangan.\nmklink / J

\n\n

https://msdn.microsoft.com/en-us/library/windows/desktop/aa365006(v=vs.85).aspx \n http://www.sevenforums.com/tutorials/278262-mklink-create-use-links-windows.html

\n\n

Selain itu, mklink / D membuat tautan simbolis direktori, bukan tautan keras. Dalam praktiknya, symbolic link adalah "jalan pintas" ke file dan folder, sedangkan hard link adalah semacam "sinkronisasi file" untuk file saja, di mana "pintasan" adalah "kembaran" dari file target; buat perubahan pada salah satunya, dan Anda membuat perubahan pada keduanya ... tetapi kenyataannya hanya ada satu file secara fisik di drive, jadi jika file target berukuran 500MB, hanya 500MB yang digunakan. JUNCTIONS adalah hard link untuk folder dan fungsinya sama seperti hard link.
