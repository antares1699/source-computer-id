Saya memiliki VM Ubuntu 64bit yang saya peroleh dari file ova ~ 1GB, saya menggunakannya di bawah VMware vSphere untuk menginstal lebih banyak perangkat lunak di dalamnya dan menggunakan sistem terhadap beberapa kumpulan data yang membutuhkan ruang sekitar 2-3GB, tetapi sekarang bahwa saya menghapus file-file besar ini, saya berharap bahwa file ova yang baru dibuat darinya juga akan memakan sekitar ~ 1,1GB ruang, tetapi malah membutuhkan sekitar 3GB. Jika saya melihat detail file .vmdk di virtualbox, saya melihat yang berikut ini:

Ukuran sel telur saat saya mengekspor mesin ini adalah ~ 3GB.

\n\n

Ada ide bagaimana saya bisa mengecilkannya?

Ruang dalam .vmdk dialokasikan saat OS melakukan penulisan pertama ke sektor tertentu. Setelah ditulis, itu dialokasikan selamanya.

\n\n

Jadi, Anda perlu menambahkan HDD virtual kedua berukuran dinamis ke mesin virtual Anda, lalu boot VM dari LiveCD, buat partisi dengan fdisk dan salin semua file (cp -R). Kemudian matikan VM, tukar HDD virtual, boot lagi dan verifikasi semuanya berfungsi, lalu lepaskan HDD virtual asli dari VM dan hapus sebagai file pada disk sistem host.

\n\n

Penyalinan langsung dengan dd di VM tidak akan berfungsi, karena ini akan menulis setiap sektor ke dalam file .vmdk target, sehingga akan diperluas ke ukuran penuhnya, bukan menyusut.
