Anda boot dari flashdrive untuk memulai proses penginstalan. Dari perspektif penginstal (yang merupakan program yang berjalan pada versi Windows "live-in-the-sense-of-live-CD" yang disebut Windows PE), drive C: mungkin adalah drive yang dapat dilepas (flash) ( pesan tersebut bahkan menyebutkan bahwa itu adalah drive yang dapat dilepas). Saya t aku s aneh bahwa itu masih akan disebut C: pada saat itu dalam proses instalasi, karena setelah bagian pertama dari proses (di mana file disalin), sistem biasanya reboot dan selesai menjalankan penginstal dari hard disk. Pada saat itu, sangat aman untuk menghapus flashdrive, tetapi saya tidak mengharapkan penginstal menyebutnya drive C :.

\n\n

Sekarang, saya telah menginstal Windows dari flashdisk berkali-kali, dan saya belum pernah melihat pesan itu. Saya tidak menggunakan alat Microsoft, saya hanya membuat flashdisk dapat di-boot menggunakan diskpart, list volume, select volume, active, exit dan kemudian menyalin konten media penginstalan ke root drive. Saya berharap hanya itu yang dilakukan alat tersebut, tetapi saya tidak yakin. Saya juga tidak pernah perlu menghapus instalasi flashdrive, tetapi itu mungkin karena saya hanya memberi tahu BIOS / EFI untuk mem-boot flashdrive sebagai hal satu kali; itu boot dari HD secara default.

\n\n

Sudahkah Anda mencoba menghapus flashdrive dan mem-boot sistem (atau secara eksplisit mem-boot dari hard disk)? Itu akan memulai pengalaman normal pasca-pemasangan di luar kotak, jika penginstalan benar-benar selesai. Meskipun tidak berhasil, kasus terburuk Anda hanya perlu menginstal ulang Windows.

\n\n

Layar yang Anda lihat di gambar kedua adalah tata letak partisi default yang dihasilkan oleh edisi Win8.1 yang lebih tinggi saat diinstal pada disk GPT yang bersih. Saya tidak 100% yakin apa tujuan masing-masing partisi, tetapi satu atau kedua partisi pemulihan mungkin terkait dengan fitur "reset" / "refresh" di Win8 +, dan partisi Sistem berisi bootloader, dll. Dan terpisah dari main (OS) partisi sehingga Anda dapat menggunakan BitLocker untuk mengenkripsi partisi utama dan masih dapat melakukan boot. Jika Anda suka, Anda dapat menghapus semua partisi tersebut dan memulai lagi dari disk kosong, meskipun Windows mungkin akan membuatnya kembali sebagaimana adanya.
