URL hanyalah alamat, referensi. Jadi begini caranya:

\n\n
    \n
  1. pengguna memasukkan URL di browser. Ini ditangani oleh driver keyboard yang akan mengirim acara keyboard ke browser, yang akan merespons dengan memperbarui bilah alamat dengan karakter yang diketik
    2. \n
  2. browser mengurai URL di berbagai bagian seperti yang ditentukan oleh RFC 3986
    3. \n
  3. browser pilih bagian host dan itu akan menggunakan perpustakaan resolver internal untuk menemukan alamat IP. Penyelesai yang paling sering digunakan adalah DNS . Untuk ini pustaka resolver yang dipetakan di dalam aplikasi akan membuat permintaan UDP pada port 53 di server DNS yang dikonfigurasi untuk mesin klien. Server DNS akan mencari permintaan di cache internal dan jika kadaluwarsa atau hilang itu akan membuat permintaan rekursif mencoba menemukan alamatnya.
    4. \n
  4. browser terhubung ke server HTTP pada port default (TCP / 80).
    5. \n
  5. browser mengirimkan permintaan GET dan juga menentukan header HOST sesuai HTTP / 1.1 RFC 2616
    6. \n
  6. server http akan mengurai permintaan dan mengirimkan permintaan ke host virtual yang ditentukan dalam permintaan
    7. \n
  7. aplikasi mengembalikan konten yang dihasilkan (biasanya file HTML ) ke browser
    8. \n
  8. browser mengurai hasilnya, menghasilkan beberapa JavaScript / Peristiwa DOM , dan merender halaman (mis. Tokek mesin di FireFox).
    9. \n
\n\n

Ini adalah penjelasan yang sangat padat. Saya tidak berbicara tentang SSL, kompresi, bahasa dan negosiasi pengkodean karakter, otentikasi, CGI, pipa-lining dan keep-alives.
