Ya, itu cukup dapat diandalkan (sebanyak yang Anda percayai MS), rekan kerja menjalankan pengaturan yang sama untuk proyek VS 2003 dan 2005, pengembangan bekerja dengan baik.

\n\n

Mode XP pada dasarnya adalah Microsoft Virtual PC yang telah diinstal sebelumnya dengan XP SP3, dibundel dan dikonfigurasi dengan Windows 7. Jadi ya, Anda harus dapat menyambung ke mode XP SQL Server dari host, karena VPC memungkinkan jaringan host-tamu.

Sejauh yang aku tahu:

\n\n

jika komputer Anda cukup kuat untuk menjalankan perangkat lunak itu di VM lain, mode XP sama baiknya dengan yang lain (secara teoritis)

\n\n

Aplikasi yang diinstal di bawah Mode XP Anda akan berjalan "mulus" (jadi tidak ada Jendela di sekitarnya), tetapi tentu saja akan lumpuh karena mereka tidak secara asli ada di sistem Anda. Mereka juga tidak akan mendukung eye-candy Windows 7 sejauh yang saya bisa mengujinya sekarang.

\n\n

Untuk memastikan: buat gambar atau vhd dari instalasi XP Anda saat ini, kemudian beralihlah ke Windows 7. Jika semuanya tidak berhasil, Anda dapat dengan mudah beralih kembali!

Dengan asumsi Anda memiliki perangkat keras yang sesuai, ini seharusnya berfungsi dengan baik. Mode XP dapat menjalankan aplikasi tanpa terlihat dan terlihat seperti aplikasi Win7 asli, atau Anda dapat membuka gambar VirtualPC secara langsung dan berinteraksi dengannya di lingkungan asalnya.

\n\n

Salah satu kegunaan hebat dari mode XP adalah dapat menginstal aplikasi berdampingan yang jika tidak, tidak akan hidup berdampingan pada sistem. Ini adalah penggunaan yang bagus dari kemampuan itu.
