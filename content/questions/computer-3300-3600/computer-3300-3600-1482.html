kemungkinan besar tidak terkait dengan scp itu sendiri, tetapi semacam resolusi nama dari satu host ke host lainnya. hal lain yang perlu dipertimbangkan adalah otentikasi. sebaiknya Anda memiliki kunci autentikasi tanpa frasa sandi dari A ke B. di mana host .ssh / config Anda mendefinisikan host ini? scp tidak menemukan definisi itu. coba fqdn sebenarnya dari host dan seharusnya berfungsi dengan baik, jika definisi host masalahnya. juga, gunakan -v untuk melihat apakah lebih banyak detail dapat ditemukan untuk kesalahan tersebut.

Bagi mereka yang datang ke sini karena scp menuduh file lokal sebagai nama host yang tidak dapat diselesaikan (sumber lain Nama atau layanan tidak diketahui ), seperti: 

> scp archive_2017-03-01-09:54:02.tar.gz backupserver:/var/backups/\nssh: Could not resolve hostname archive_2017-03-01-09:54:02.tar.gz: Name or service not known\n

> scp ./archive_2017-03-01-09:54:02.tar.gz backupserver:/var/backups/\n
