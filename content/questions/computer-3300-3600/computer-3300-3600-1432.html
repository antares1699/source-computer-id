Apakah ada cara untuk menghitung, dan menguraikan, ukuran file rata-rata untuk direktori (atau drive) tertentu?

\n\n

Sesuatu yang dapat memberikan detail seperti: |

\n\n
    \n
  • File 4KB - 22KB: mencapai 20% dari semua file
    • \n
  • File berukuran 32KB - 64KB: mencakup 10% dari semua file
    • \n
  • File berukuran 1MB - 20MB: mencakup 70% dari semua file
    • \n
\n\n

Dan seterusnya.

\n\n

Mungkin Anda mengetahui utilitas yang dapat melakukan itu?

Anda dapat melakukan ini dengan mudah di Powershell 

$foo = (Get-ChildItem -path \"c:\\your\\file\\path\" -recurse | measure-object | select -expand Count)\n$bar = ((Get-ChildItem -path \"c:\\your\\file\\path\" -recurse | Measure-Object -property length -sum).sum /1MB)\n$avrg = $bar / $foo\n$avrg\n

Contoh keluaran: 

PS X:\\> $avrg\n1.03643417358398\n

Saya baru saja menggunakan total commander (shareware) dan microsoft excel untuk melakukannya dalam 2 menit.

\n\n

Di Total Commander, buka root C: (atau drive apa pun yang Anda suka) klik Ctrl + B, yang mencantumkan semua file dalam semua subdirektori di satu tempat. Dari menu Tandai item, klik "Salin ke clipboard dengan semua detail" dan tempel ke excel ... Anda sekarang memiliki daftar panjang item dengan ukuran file mereka, urutkan dan hitung sesuka hati Anda. Anda juga dapat Google "histogram di Excel" untuk hasil yang lebih baik.

\n\n

Apa yang saya dapatkan pada sistem Windows 8.1 yang berusia satu tahun ditunjukkan di bawah ini (ukuran dalam byte, dan angka sumbu x adalah kisaran yang saya tentukan). Seperti yang bisa dilihat, sebagian besar file saya berkisar sekitar 1k.
