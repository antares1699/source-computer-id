Saya sedang menulis geektool 3 skrip untuk menunjukkan ukuran mesin virtual VMware Fusion tertentu. "File" .vmwarevm tersebut sebenarnya adalah direktori yang dikemas.

\n\n

Mendapatkan informasi di Finder mengatakan file tersebut "52.91 GB". Saya menjalankan perintah du berikut untuk mendapatkan ukuran file: 

> du -hs ~/Documents/Virtual\\ Machines.localized/MY-PRECIOUS-7.vmwarevm | awk '{print $1}'\n

Perintah du -hs ini mengembalikan ukuran file sebagai "49G". Apa yang menyebabkan perbedaan dari apa yang dilaporkan Finder?

\n\n

kalau tidak , Saya telah mencoba mengganti opsi -s dengan opsi -d seperti: 

du -hd ~/Documents/Virtual\\ Machines.localized/MY-PRECIOUS-7.vmwarevm | awk '{print $1}'\n

Perintah du -hd ini mengembalikan ukuran file sebagai "59G". Apa yang menyebabkan perbedaan antara Finder, du -hd, dan du -hs?

\n\n

Juga, perintah du -hd ini tidak menghasilkan keluaran di geektool 3. Apa penyebabnya?

Bukan pengguna Mac OS X tetapi saya membaca di suatu tempat bahwa Finder menggunakan basis 10 saat ini.

\n\n

Mungkinkah perbedaannya karena du masih menggunakan basis 2? 

49.0 GiB * (1024^3 bytes / GB) = 52,613,349,376 bytes = 52.6 GB\n

(perbedaan kecilnya adalah karena du dibulatkan ke GB terdekat)

Argumen -d mengharuskan Anda menentukan kedalaman. Dari percobaan barusan, tampaknya jika Anda memberikan sesuatu selain angka untuk kedalaman, itu memakan argumen yang Anda tentukan dan berperilaku seolah-olah Anda melakukannya -d 0. 59G yang Anda dapatkan adalah ukuran direktori saat ini, bukan ukuran vmwarevm.

\n\n

Ini eksperimen yang saya jalankan: 

[~]$ ls -d Code\nCode/\n\n[~]$ du -h -s Code\n8.3M    Code\n\n[~]$ du -h -d 0 Code\n8.3M    Code\n\n[~]$ du -h -d Code\n 38G    .           <-- this is the size of ~/\n\n[~]$ du -h -d 0\n 38G    .\n

Anda mungkin ingin membaca Halaman manual Mac OS X du (1) untuk informasi lebih lanjut.
