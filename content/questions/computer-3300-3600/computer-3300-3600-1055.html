Saya memiliki file teks biasa 5 juta plus baris. Windows Notepad tidak dapat melakukan apa-apa dan macet saat mengklik dua kali file tersebut. Notepad ++ dapat membuka file meskipun dengan 5 juta baris.\nSaya ingin membagi file menjadi 10 file teks tunggal - yaitu 50K baris pertama akan berada dalam file individual, dan selanjutnya dan berikutnya.

\n\n

Masalahnya adalah ketika saya Ctrl + G dan pergi ke baris 50k, saya harus menahan shift, menekan panah atas sampai garis 50k pertama dipilih. Ini membutuhkan waktu beberapa menit. Apakah ada cara yang lebih mudah untuk membagi file?

Shift-control-home akan menyorot dari posisi saat ini ke bagian atas file.

Anda mungkin memberi MEMBAGI di GnuWin32 mencoba. Secara sederhana: 

split --lines=50000 YourInputFile.dat OutputFile.\n

akan menghasilkan: 

OutputFile.aa\nOutputFile.ab\nOutputFile.ac\nOutputFile.ad\n .\n .\n

masing-masing dengan 50.000 rekaman. File terakhir akan berisi sisanya.
