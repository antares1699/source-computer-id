Hari ini saya perhatikan bahwa Dropbox menghabiskan hampir setengah dari waktu CPU yang dikonsumsi oleh kernel_task pada OS X 10.8.3, yang merupakan jumlah penggunaan CPU yang lumayan. Jadi saya menghentikannya dan melihat apa yang akan terjadi.

\n\n

Anehnya, bahkan setelah saya menghentikan sinkronisasi, Dropbox membutuhkan jumlah CPU yang kira-kira sama (dan tidak ada yang diubah).

\n\n

Jadi, Dropbox terus mengindeks meski saya perintahkan untuk "jeda"? Lalu "jeda" hanya berlaku untuk penggunaan jaringan? Ini sepertinya sangat bodoh?

Fitur jeda hanya dimaksudkan untuk menghentikan sinkronisasi dengan server. Ini berguna jika Anda akan mematikan komputer atau memutuskan koneksi dari internet dan akan mencegah kerusakan file apa pun. Ini juga akan berguna jika Anda bermain game online karena dengan game online lebih sering daripada tidak, koneksi jaringan Anda menjadi penghambat daripada penggunaan CPU.

\n\n

Penggunaan CPU akan tetap konstan atau hanya menunjukkan sedikit penurunan saat sinkronisasi dihentikan karena proses latar belakang masih mengindeks dan mempertahankan widget di bilah menu.

Pengindeksan tidak bergantung pada sinkronisasi file dari akun Dropbox Anda. Dropbox mengandalkan pengetahuan tentang konten file yang diperoleh melalui pengindeksan untuk menyinkronkan data Anda secara efisien, sehingga Dropbox hanya dapat mengunggah bagian file yang telah berubah, bukan seluruh file saat tidak diperlukan. Jika Dropbox tidak mengindeks file, itu tidak akan bisa melakukan ini.
