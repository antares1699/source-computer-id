Saya berkomentar bahwa NT AUTHORITY \\ SYSTEM tidak memiliki kata sandi , jadi kata sandi kosong adalah\nentri yang benar untuk prompt itu.

\n\n

Saya hanya dapat memastikan bahwa akun domain Anda bukan anggota\ngrup lokal Pengguna Log Kinerja dan / atau Pengguna Pemantau Kinerja,\njadi mungkin bukan Administrator lokal penuh.

\n\n

Grup ini didefinisikan sebagai:

\n\n
    \n

  • Pengguna Log Kinerja
    Anggota grup ini dapat mengelola penghitung kinerja, log, dan peringatan di komputer - baik secara lokal maupun dari klien jarak jauh - tanpa menjadi anggota grup Administrator.

    • \n

  • Pengguna Monitor Kinerja
    Anggota grup ini dapat memantau penghitung kinerja di komputer - secara lokal dan dari klien jarak jauh - tanpa menjadi anggota grup Administrator atau grup Pengguna Log Kinerja.

    • \n
\n\n

Masalah yang sama ini dibahas di pos Masalah Izin dengan File yang Dihasilkan oleh PerfMon ,\ndimana dikatakan:

\n\n
\n

Kumpulan Pengumpul Data dapat berisi informasi sensitif tentang komputer, sehingga akses ke sana biasanya mengharuskan pengguna setidaknya menjadi anggota grup Pengguna Log Kinerja.

\n
\n\n

Solusi di sana, sebenarnya solusi, adalah membuat tugas terjadwal\nakan aktif saat Kumpulan Pengumpul Data selesai berjalan,\nuntuk mengubah ACL dari struktur direktori secara rekursif menjadi "Semua Orang Kontrol Penuh".

\n\n

Ada masalah saat membuat pemicu untuk pekerjaan itu, yang kemudian diperlukan\npemicu Kustom dimasukkan secara manual sebagai XML: 

<QueryList>\n  <Query Id=\"0\" Path=\"Microsoft-Windows-TaskScheduler/Operational\">\n    <Select Path=\"Microsoft-Windows-TaskScheduler/Operational\">\n        *[System[TimeCreated[timediff(@SystemTime) &lt;= 3600000]]]\n         and\n        *[System[(EventID='102')]]\n         and\n        *[EventData[Data and (Data='YOUR DATA COLLECTOR SET NAME')]] \n    </Select>\n  </Query>\n</QueryList>\n

Pemicu meluncurkan skrip PowerShell ini: 

$Path = \"C:\\PerfLogs\\Admin\\New Data Collector Set\"\n$ACL  = (Get-Item $Path).GetAccessControl(\"Access\")\n$ACE  = New-Object System.Security.AccessControl.FileSystemAccessRule(\"Everyone\", \"FullControl\", \"ContainerInherit,ObjectInherit\", \"None\", \"Allow\")\n$ACL.AddAccessRule($ACE)\nForEach($_ In Get-ChildItem $Path -Recurse)\n{\n    Set-Acl -ACLObject $ACL $_.FullName\n}\n

Solusi ini rumit, jadi mungkin lebih sederhana untuk melanjutkan\nmemasukkan kata sandi kosong saat diminta.
