Apa itu string build Windows?

\n\n

Ini adalah sekumpulan karakter yang secara unik mengidentifikasi build Windows.

\n\n

Apa yang membangun? ...

\n\n

Build mengacu pada membangun solusi. Yaitu: menyusun kode sumber menjadi kode objek dan menghubungkan kode objek ke dalam kode mesin yang dapat dieksekusi. Ini adalah produk akhir dari upaya analisis, desain, dan pemrograman. Namun, ini mungkin bukan produk akhir, karena mungkin direvisi beberapa kali sebelum itu terjadi. Jika ya, itu dikenal sebagai rilis final, rilis emas, atau dalam kasus perangkat lunak Microsoft: RTM (rilis ke manufaktur).

\n\n

Pada akhirnya, build adalah solusi untuk masalah yang dapat diselesaikan dalam perangkat lunak! Oleh karena itu lebih dikenal hanya sebagai perangkat lunak. Dalam hal ini adalah perangkat lunak sistem Windows - sistem operasi yang kemungkinan besar berjalan di komputer Anda sekarang dan membuatnya berguna. Anak-anak masa kini menyebutnya semua "aplikasi", awalnya merupakan bentuk singkat dari "aplikasi" atau "perangkat lunak aplikasi", tetapi sekarang juga memiliki arti lain. Grup hanya menyebutnya "program". Untuk menggunakan kembali pepatah Swedia, "anak yang dicintai memiliki banyak nama."

\n\n
\n

Itu juga merupakan nama konferensi pengembang Microsoft tahunan.

\n
\n\n

Bagaimana build string dapat dibaca di Windows?

\n\n

String build disimpan secara internal sebagai nilai string di registri Windows. Ada dua jenis string build yang harus diperhatikan. Satu hanya disebut string build dan hanya menyertakan nomor build, jadi ini hanya numerik. Yang lainnya terkadang disebut string build lengkap, karena ini juga menyertakan nama cabang dan waktu kompilasi, atau lebih tepatnya build cap waktu.

\n\n

Kunci berikut di registri Windows menampung string build lengkap dari sistem yang sedang berjalan. 

HKLM\\Software\\Microsoft\\Windows NT\\CurrentVersion\n

Ada dua variabel yang menarik di sini: 

BuildLab:   10240.th1.150729-1800\nBuildLabEx: 10240.16412.amd64fre.th1.150729-1800\n

Apa arti dari build string?

\n\n

Secara umum, string build berisi segmen berikut: versi mayor, versi minor, nomor build, nama cabang, tanggal pembuatan, waktu pembuatan.

\n\n

Secara umum, skemanya terlihat seperti ini:

\n\n

[major].[minor].[build].[branch].[date]-[time]

\n\n

winmain_prs adalah cabang "Windows main". Ini adalah cabang utama, stabil dan siap untuk rilis yang dikembangkan dari semua versi Windows yang lalu dan yang akan datang (setidaknya dalam kasus keluarga Windows NT). "Prs" mungkin berarti mitra, atau produksi siap dan stabil. Saya benar-benar hanya menebak-nebak! Tidak banyak informasi tentang ini di web.

\n\n

Saya cukup yakin bahwa "fbl" adalah singkatan dari "build masa depan". Ini sedang dalam proses, cabang yang tidak stabil. Mereka menggunakan nama-nama lucu seperti "mengesankan", "luar biasa" dan "rilis" (yang terakhir tidak terlalu lucu). Menurut saya, "mengesankan" dan "mengagumkan" digunakan untuk menambahkan fitur baru, sedangkan "rilis" digunakan untuk men-debug dan menstabilkan sebelum diintegrasikan secara terbalik ke cabang utama.

\n\n

Cabang "th1" baru. Saya tidak yakin apa artinya. Tetapi saya tahu bahwa hanya build RTM yang tergantung di cabang ini. Jadi ini mungkin semacam indikasi perubahan model rilis Windows. Saya mengacu pada pergeseran ke Windows sebagai layanan.

\n\n

Pembaruan: Build 10525 baru-baru ini dirilis di cabang "th2". The "th" tampaknya merupakan kependekan dari "threshold" yang merupakan codename untuk Windows 10. "th1" tampaknya merupakan rilis publik umum, sedangkan "th2" adalah cabang rilis pratinjau orang dalam.

\n\n

Mengapa string build berguna?

\n\n

Mereka memungkinkan identifikasi yang tepat dari build Windows, hingga ke cabang dan waktu kompilasi. Hal ini membuat pelacakan perubahan kecil menjadi sangat mudah, karena setiap perubahan pada kode, betapapun kecilnya, yang dikompilasi, ditautkan, dibuat, dan dirilis juga mendapatkan string build baru yang mengidentifikasinya. String build sebagian besar berguna untuk penguji Beta. Karena perubahan kecil dan sering pada kode dilakukan selama fase pengujian siklus pengembangan.

\n\n

Setelah rilis final dibuat, string build lengkap menjadi sedikit menarik. Sebaliknya, hal utama yang menarik adalah nomor mayor, minor, dan build. Ini membantu mengidentifikasi "versi" dari sistem Windows yang sedang berjalan. Inilah yang digunakan aplikasi Windows pihak ketiga untuk mengidentifikasi versi Windows tempat mereka diinstal atau dijalankan. Ini berbeda dari nama pemasaran Windows, seperti "Windows XP" atau "Windows Vista".
