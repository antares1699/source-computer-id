\n

apakah berbahaya mengunduh sesuatu saat menggunakan boot usb?

\n
\n\n

Secara umum, ini tidak lebih berbahaya daripada mengunduh data saat boot dari HDD biasa. Tindakan pencegahan apa pun yang Anda lakukan saat boot dari HDD juga akan berlaku saat boot dari USB.

\n\n
\n

Di mana sistem menyimpan unduhan saya?

\n
\n\n

Biasanya, dengan penginstalan USB, seluruh sistem akan menjalankan RAM Anda, yang berarti file tidak diunduh ke lokasi "persisten" mana pun (ini disebut mode Live), atau stik USB itu sendiri akan memiliki lokasi tetap / partisi untuk menyimpan file, dan modifikasi yang akan disimpan (mode Penginstalan Penuh). Jadi, secara default, file akan diunduh ke RAM, atau ke stik USB. Kecuali Anda secara khusus menyuruhnya mengirimnya ke HDD Anda. Mengarah ke pertanyaan ketiga Anda ...

\n\n
\n

Bisakah itu menimpa data di hdd saya?

\n
\n\n

Tidak dengan cara yang mungkin Anda pikirkan. Sebagian besar penginstalan USB biasanya memiliki kemampuan untuk mengakses HDD Anda, terutama jika HDD diformat dalam salah satu jenis sistem file standar yang umum (ext3 / 4, NTFS, FAT, HFS +, dll). Inilah mengapa penginstalan langsung biasanya digunakan untuk memecahkan masalah penginstalan HDD. Namun, ini tidak berarti ia akan menulis secara sembarangan ke HDD tanpa memperhatikan konten HDD. Jika Anda menyimpan file yang diunduh ke HDD, "tulis" ini akan bekerja dengan cara yang sama seperti penulisan normal mana pun, saat di-boot dari HDD. Artinya, file akan ditulis ke ruang kosong, dan di dalam sistem file yang ada di HDD. Anda akan melihat ini karena Anda masih harus menavigasi sistem file yang ada di HDD, jadi jika saat boot ke HDD Anda membuat folder bernama / berbagai Hal, saat boot ke USB dan mengakses HDD, Anda masih akan melihat dan dapat tulis file ke / berbagai Hal.

\n\n

Jadi, singkatnya, tidak, boot dari USB tidak akan merusak HDD Anda kecuali Anda secara khusus melakukannya.

Persistent Live USB dan Full Linux Instal di USB

\n\n
\n

Live USB adalah cara yang baik untuk menguji distro Linux tanpa membuat perubahan pada komputer Anda. Tidak diketahui banyak orang, ada mode persistensi data dalam sesi Live di mana Anda dapat membuat perubahan dan menyimpan file ke drive USB Anda dan data akan tetap ada bahkan setelah sesi live dimatikan. Setelah menguji sesi langsung, Anda juga dapat menginstal distro ke drive USB Anda alih-alih hard drive internal. Kedua metode ini memungkinkan Anda untuk mem-boot Linux dari drive USB dan menyimpan data ke dalamnya. Jadi apa perbedaan antara keduanya dan mana yang harus Anda gunakan?

\n
\n\n

\n\n
