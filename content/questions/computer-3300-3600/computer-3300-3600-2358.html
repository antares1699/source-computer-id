Izinkan saya mengubah kalimat: laptop Anda menggunakan Win-XP 32-bit dan Anda telah menyalin instalasi itu ke mesin virtual. Laptop Anda sekarang menggunakan Windows 7 tetapi itu tidak masalah untuk keseluruhan pertanyaan, bukan? Kemudian lagi, laptop Anda tampaknya cukup kuat untuk menjalankan sistem Windows 7 64-bit sehingga host VMWare harus cukup kuat untuk menampung semuanya.

\n\n

Pertama-tama, Anda harus selalu mengharapkan kinerja turun jika host sekuat sistem aslinya. Ada lapisan virtualisasi di antaranya yang menambahkan beberapa penundaan. Saya melakukan hal yang sama dengan laptop saya, tetapi mesin virtual yang saya buat di-host pada sistem yang jauh lebih bertenaga, sehingga kinerjanya meningkat.

\n\n

Kemacetan yang mungkin terjadi dapat berupa penginstalan SQL Server atau database lain di dalam mesin virtual. Ini cenderung memakan banyak sumber daya. Mungkin lebih baik menginstal SQL Server pada host dan membuat VM terhubung ke database "eksternal" ini.\nMasalah lain bisa jadi adalah jumlah RAM yang dimiliki mesin virtual. Meskipun XP bekerja dengan baik dengan 512 MB, saya cenderung memberikan VM saya hingga 2 GB RAM. Dengan begitu, VM akan menggunakan lebih sedikit swapfile miliknya, karena ia memiliki lebih banyak memori yang tersedia.

\n\n

Hambatan lain bisa jadi harddisk yang berisi VM. Jika ini terhubung ke sistem Anda melalui kabel USB maka pasti akan lambat. Saya cenderung memberikan dua disk VM. Satu dengan sistem operasi yang akan ditempatkan pada disk tercepat di host saya dan yang lainnya untuk data yang dapat berupa disk apa pun. Seringkali saya menggunakan berbagi jaringan untuk ini, dan berbagi disk / folder di host. Dengan cara ini, data tersedia dari dalam dan luar VM.

\n\n

Disk sistem akan berukuran hingga 8 GB dan saya cenderung mengklaim ruang disk ini segera, alih-alih meminta VMWare meningkatkan ukuran disk sedikit demi sedikit. Itu memang meningkatkan kinerja tetapi memakan banyak ruang disk.

\n\n

Jangan letakkan mesin virtual ini pada disk terkompresi untuk menghemat ruang disk! Kompresi disk cenderung membuat akses disk sedikit lebih cepat karena sebenarnya harus membaca lebih sedikit data dari disk tetapi juga memberi beban lebih besar pada prosesor Anda karena perlu mendekompresi data ini lagi. Ini menempatkan permintaan yang lebih besar pada CPU Anda, yang bahkan dapat meningkatkan panas di dalam sistem Anda. Dengan laptop, sistem cenderung menurunkan kecepatan CPU ketika kipas tidak dapat mendinginkan sistem dengan cukup.
