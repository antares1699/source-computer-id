Apakah mungkin untuk mengatur satu folder desktop bersama untuk beberapa instalasi linux pada komputer dual-boot? Saya ingin beberapa penginstalan Linux di satu komputer untuk berbagi satu desktop. Saya juga sedang menggunakan Ubuntu 64 bit.

Saya yakin Anda kemungkinan besar ingin memiliki partisi /home yang dibagikan di antara semua instalasi Anda. Dengan begitu, semua penyesuaian per pengguna akan dibagikan, tetapi semua konfigurasi sistem khusus distro terpisah.

\n\n

Ketika Anda menginstal sistem Anda untuk pertama kali, Anda biasanya dapat memilih partisi kustom, dan di sana juga memilih partisi yang ada dan menentukan titik kait untuk itu. Itu akan menjadi pendekatan yang paling mudah. Jadi untuk sistem pertama Anda, Anda akan melakukan partisi khusus, membuat partisi baru sesuai keinginan Anda, memiliki salah satunya untuk linux / ini, satu untuk /home, dan keduanya baru diformat. Untuk penginstalan berikutnya, Anda akan menggunakan salah satu dari partisi lain untuk / setelah pemformatan, dan sistem file yang ada untuk /home tanpa pemformatan.

\n\n

Jika Anda ingin memulai dari sistem yang sudah ada, keadaan menjadi sulit dan kesalahan mungkin memiliki konsekuensi yang parah. Kamu harus

\n\n
    \n
  1. mengecilkan sistem file yang ada
    2. \n
  2. menyusutkan partisi penampung
    3. \n
  3. buat partisi baru
    4. \n
  4. membuat sistem file itu
    5. \n
  5. boot ke mode pengguna tunggal untuk langkah selanjutnya
    6. \n
  6. ganti nama rumah lama
    7. \n
  7. buat mountpoint untuk rumah baru
    8. \n
  8. buat entri fstab untuk pemasangan baru
    9. \n
  9. pasang rumah baru
    10. \n
  10. cp -a barang dari rumah lama ke rumah baru
    11. \n
  11. reboot lagi dalam mode multi-pengguna
    12. \n
\n\n

Beberapa aspek lebih lanjut yang harus Anda perhatikan:

\n\n
    \n
  • Informasi kepemilikan file Linux disimpan menggunakan numerik pengenal pengguna dan grup. File /etc/passwd dan /etc/group memetakannya ke nama. Anda harus memastikan bahwa pemetaan untuk pengguna sama di kedua sistem. Biasanya ada sejumlah besar akun sistem untuk berbagai layanan, tetapi hanya sedikit akun pengguna. Dan sebagian besar distro saat ini mungkin mengalokasikan id dengan cara yang sama. Jadi ini mungkin bekerja secara otomatis, tetapi jika file tiba-tiba menjadi tidak dapat diakses, ini kemungkinan penyebabnya.
    • \n
  • Pastikan Anda tidak memulai linux apa pun saat yang lain sedang hibernasi. Hibernasi akan membuat sistem file dalam keadaan tidak konsisten, dan lebih buruk lagi, kernel akan mengharapkan sistem tidak berubah selama hibernasi, dan mungkin menyimpan cache karena alasan ini. Jadi korupsi data besar dapat disebabkan oleh pola akses ini.
    • \n
