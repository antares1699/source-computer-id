Itu Ctrl + R fungsionalitas untuk mencari ke belakang melalui sejarah shell disediakan oleh Readline perpustakaan yang digunakan oleh Bash. Fungsi yang sesuai untuk mencari ke depan melalui sejarah, secara default, terikat Ctrl + S .

\n\n

Namun, masalahnya adalah driver terminal sudah menggunakan kombinasi tombol ini untuk Alur kontrol : mendesak Ctrl + S berhenti atau jeda teks yang sedang dicetak ke terminal dan Ctrl + Q resume aliran karakter yang dikirim ke perangkat terminal.

\n\n

Pengaturan ini diaktifkan secara default untuk sebagian besar perangkat terminal pada sistem GNU / Linux sehingga Bash / Readline tidak pernah melihat Ctrl + S urutan kunci karena ditangkap oleh driver terminal dan tidak pernah diteruskan ke shell.

\n\n

Perilaku ini (juga disebut Kontrol aliran XON / XOFF ) dapat dinonaktifkan dengan menjalankan: 

stty -ixon\n

Sekarang Ctrl + S kombinasi tombol akan diteruskan ke Bash / Readline.

\n\n

Kombinasi tombol alternatif

\n\n

Jika Anda lebih suka Readline menggunakan kombinasi tombol yang berbeda untuk penelusuran meneruskan , gunakan perintah bind untuk mengaktifkan pengikatan kunci yang berbeda, mis. Ctrl + F (untuk Maju). NB: semua kutipan diperlukan. 

bind '\"\\C-f\": forward-search-history'\n

Ubah konfigurasi untuk menyimpan perubahan ini

\n\n

Kontrol aliran terminal tidak terlalu berguna pada perangkat modern jadi saya akan menganjurkan menjalankan perintah berikut setiap kali terminal baru atau perangkat pseudo-terminal dimulai: 

stty -ixon\n

.bashrc mungkin akan menjadi file start-up yang baik untuk menjalankan perintah ini karena file tersebut bersumber untuk shell Bash interaktif. Jika .bashrc tidak bersumber dari .profile atau .bash_profle, perintah stty di atas harus ditambahkan ke file start-up mana pun yang digunakan untuk shell login.

\n\n

File konfigurasi Readline adalah .inputrc jika Anda lebih suka menggunakan pengikatan kunci yang berbeda seperti Ctrl + F untuk mencari meneruskan melalui sejarah: 

bind \"\\C-f\": forward-search-history\n
