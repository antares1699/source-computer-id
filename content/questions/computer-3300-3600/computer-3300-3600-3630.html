Singkatnya, ya. Awalnya (di 8086) itu disebut DMA, atau Direct Memory Access. Saat itu ada chip pengontrol DMA khusus yang harus diprogram oleh CPU untuk mentransfer data antara ram dan periferal, tetapi setidaknya sejauh hari-hari awal PCI (dan saya berpikir pada bus ISA asli juga, tetapi saya tidak tahu perangkat apa pun yang memilikinya), periferal itu sendiri juga dapat menjadi master bus, dan mentransfer datanya sendiri.

\n\n

Mereka biasanya masih membutuhkan cpu untuk memberi tahu mereka bagian memori mana yang harus mereka akses, tetapi beberapa perangkat berperilaku buruk, seperti pengontrol firewire, membuka akses DMA ke semua ram sistem atas perintah beberapa perangkat eksternal di bus firewire. Beberapa sistem PCIe modern memiliki jembatan PCIe cerdas yang dapat diprogram untuk melakukan terjemahan alamat dan pembatasan akses sehingga CPU dapat mencegah perangkat yang terhubung di belakangnya mengakses memori yang seharusnya tidak mereka akses, tetapi saat ini tampaknya terbatas pada motherboard jenis server besar, bukan tipikal Anda. Desktop.

Mengenai caranya, bus PCI (e) memungkinkan perangkat apa pun untuk melakukan baca / tulis ke alamat apa pun, baik itu di ram sistem utama (biasanya dikontrol langsung oleh CPU saat ini), atau di perangkat lain di bus. Artinya, satu perangkat PCI (e) dapat langsung mengakses memori atau register perangkat PCI (e) lain, atau ram sistem utama, tanpa memperhatikan lokasinya. Jembatan PCI (e) merutekan permintaan ke perangkat yang sesuai, karena dikonfigurasi (oleh firmware / bios sistem, dan mungkin kemudian oleh kernel OS) untuk mengetahui kisaran alamat apa yang "dimiliki" oleh perangkat apa.
