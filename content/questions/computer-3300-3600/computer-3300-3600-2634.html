Saya memiliki skrip Batch pada dasarnya saya ingin menampilkan jendela notepad kepada pengguna yang dibuka dengan file konfigurasi untuk diedit pengguna setiap kali mereka menjalankan skrip, dan saya ingin Command Prompt menunggu hingga program ini ditutup untuk melanjutkan eksekusi, dan jalankan program yang menggunakan file konfigurasi ini.

\n\n

Saya ingin melakukan ini karena saya tidak ingin mengambil input pengguna dalam skrip Batch dengan gaya perintah "set / P", dan harus menyiapkan pembacaan dan penulisan parsing ke file konfigurasi yang memiliki banyak nilai untuk diperiksa.

\n\n

Saya percaya perilaku standar pada Windows saat menjalankan program yang dapat dieksekusi, untuk file "exe" itu akan membuka program dalam prosesnya sendiri dan segera mengontrol kembali ke skrip / prompt perintah. Untuk file skrip "bat" dan "cmd", ia akan menjalankan skrip dan menunggu hingga kontrol dikembalikan ke skrip pertama atau prompt perintah. Ini harus menjadi perilaku ketika setidaknya memanggil skrip lain dengan perintah "CALL" atau memanggilnya secara langsung.

\n\n

Sekarang di Linux saya percaya perilaku default untuk konten yang dapat dieksekusi yang merupakan kode mesin asli yang dapat datang tanpa ekstensi, dan skrip seperti file "sh", dijalankan dengan cara memblokir, kecuali Anda menentukan karakter ampersand ("&") di suatu tempat di jalur masukan.

\n\n

Anda bisa menggunakan keduanya start perintah (perhatikan urutan /B /WAIT sakelar) 

start \"\" /B /WAIT notepad.exe \"somefile\"\n

atau sederhananya 

notepad.exe \"somefile\"\n

Sesuai tautan di atas:

\n\n
\n

Saat Anda menjalankan aplikasi antarmuka pengguna grafis 32-bit (GUI),\n cmd tidak menunggu aplikasi ditutup sebelum kembali ke\n prompt perintah. Perilaku ini tidak terjadi jika Anda menjalankan file\n aplikasi dari skrip perintah .

\n
\n\n

Sunting sehubungan dengan komentar Karan: Mengapa / B diperlukan?

\n\n
\n

/B memulai aplikasi tanpa membuka Command Prompt baru\n jendela.

\n
\n\n
    \n
  • Sejujurnya: jika Anda menjalankan file aplikasi antarmuka pengguna grafis kemudian tidak Jendela Command Prompt baru terbuka meskipun tombol /B dihilangkan.
    • \n
  • Di sisi lain, jika Anda menjalankan file alat baris perintah , menghilangkan saklar /B menyebabkan menjalankan alat itu di jendela Command Prompt baru yang akan segera ditutup setelah itu berakhir dan semua hasilnya hilang .
    • \n

notepad.exe | echo > NUL\n

Apa yang dilakukannya adalah, ini membuka notepad.exe dan semua outputnya diarahkan ke perintah echo > NUL. Untuk mengarahkan ulang seluruh output, program harus selesai. Oleh karena itu kita harus menunggu sampai notepad.exe selesai.\nOutputnya bergema. Echo tidak menggunakan input dengan cara apa pun dan hanya mengeluarkan sesuatu ke NUL. NUL adalah aliran jendela standar yang tidak mengarah ke mana pun.
