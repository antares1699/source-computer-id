Postfix cukup mudah dikonfigurasi, terutama untuk menyampaikan melalui "smarthost" (Gmail atau ISP atau apa pun).

\n\n
\n\n

Konfigurasi Postfix untuk Gmail:

\n\n
    \n

  1. Perbarui /etc/postfix/main.cf: 

    compatibility_level = 2\ninet_interfaces = loopback-only\nrelayhost = [smtp.gmail.com]:587\n# NOTE: the CAfile/CApath is distro-dependent!\nsmtp_tls_CApath = /etc/ssl/certs\nsmtp_tls_loglevel = 1\nsmtp_tls_security_level = verify\nsmtp_sasl_auth_enable = yes\nsmtp_sasl_password_maps = hash:/etc/postfix/sasl_passwd\nsmtp_sasl_security_options = noanonymous\n
    2. \n

  2. Buat /etc/postfix/sasl_passwd: 

    [smtp.gmail.com]:587     joe@gmail.com:JoesSekritPassword\n
    3. \n

  3. Jalankan sebagai root: 

    postmap /etc/postfix/sasl_passwd\nchmod 640 /etc/postfix/sasl_passwd*\nchgrp postfix /etc/postfix/sasl_passwd*\n
    4. \n

  4. Beri tahu Postfix untuk memuat ulang konfigurasi: 

    postfix reload\n
    5. \n
\n\n
\n\n

Exim juga tidak buruk, selama Anda mengabaikan semua tutorial khusus Debian dan makronya yang tidak masuk akal.

\n\n

Ada juga msmtp dan esmtp yang bisa hanya gunakan smarthost sehingga tidak perlu dijalankan sebagai daemon. Mereka juga lebih mudah dikonfigurasi, terutama untuk lingkungan multi-pengguna.

\n\n
\n\n

Jika Anda mencari perintah postmap /etc/postfix/sasl_passwd\nchmod 640 /etc/postfix/sasl_passwd*\nchgrp postfix /etc/postfix/sasl_passwd*\n, pastikan Anda mendapatkan heirloom-mailx atau GNU mailutils. (Klasik bsd-mailx berfungsi, tetapi tidak memiliki banyak fitur seperti dukungan Maildir atau MIME.) Saya masih lebih suka mutt.

Ada tiga hal yang Anda perlukan untuk email dasar:

\n\n
    \n
  • sebuah program untuk format email RFC 822 yang akan Anda kirim
    • \n
  • sebuah program untuk send (RFC 822) email berformat ke sistem pengiriman / pengiriman email Anda melalui SMTP (agen pengiriman email - MSA, atau agen transfer email - MTA)
    • \n
  • sebuah program untuk mengakses kotak surat jarak jauh Anda untuk hal-hal seperti draf yang disimpan dari jarak jauh, kotak masuk Anda, dll. dengan menggunakan IMAP, POP, SMAP, rsyncing ke mbox atau direktori surat lokal, atau bahkan hanya menggunakan sshfs untuk membuat mbox atau direktori surat tersebut tersedia di sistem file (agen pengguna surat - MUA , biasanya menggabungkan komponen pertama)
    • \n
\n\n

Jika Anda melihat istilah 'agen pengiriman surat' (MDA), abaikan. Itu adalah program program sisi server bagi mereka yang membuat layanan mereka sendiri.

\n\n

Anda kemungkinan besar sudah memiliki MUA (mutt, mailx, alpine, dll.), Serta komponen ketiga. MSA mungkin juga dibangun ke dalam program yang sama dengan MUA Anda. Simak berikut ini.

\n\n
    \n

  • Misalnya, untuk paket Arch Linux mutt resmi, dalam konfigurasi yang paling sederhana (lihat muttrc(5)): 

    # MUA part\nset folder = \"imaps://foo@example.com\" # shortcut so that I can use relative\n                                   # names for $spoolfile, etc.\nset spoolfile = \"=INBOX\"\nset record = \"=Sent Messages\"\nset postponed = \"=Drafts\"\n\nset from = \"foo@example.com\"\n\n# MSA/MTA part\nset smtp_url = \"smtp://foo@example.com\" # or `foo@example.com@example.com`\n                                        # if the server requires a fully\n                                        # qualified user name, say because it\n                                        # serves multiple domains\n
    • \n

  • Saya tidak pernah menggunakan alpine tetapi Arch Wiki memiliki halaman di atasnya .

    • \n

  • Untuk mailx (paket heirloom-mailx, saya yakin ~/.mailrc berikut (lihat mailx(1)) bisa digunakan. Gunakan mailx -A example.com untuk memberi tahu mailx untuk menggunakan akun itu. 

    account example.com {\n    set folder=imaps://foo@example.com\n    set imap-auth=login\n    set record=+Sent\n        set \n\n    set smtp=example.com\n    set smtp-auth=plain # or whatever your server uses\n    set smtp-auth-user=foo\n\n    set from=\"foo@example.com (Foo Bar)\"\n    set hostname=example.com\n}\n
    • \n
\n\n

Sekarang, baca terus jika Anda masih ingin menggunakan metode MTA mandiri tradisional. Karena kebanyakan MUA tradisional seperti mutt atau mailx dapat menggunakan antarmuka sendmail (yaitu, mereka hanya mengirimkan email ke biner sendmail, dan Anda hanya mengirim email, bukan menyiapkan server email lengkap, gunakan Agen Surat Capung , daripada mengunduh server lengkap (seperti Postfix atau sendmail). Ada di AUR. Ini hanya berfungsi dan hanya berjalan ketika Anda benar-benar mengirim email (yaitu, ini bukan deamon). Ini menyediakan biner pembungkus pada /usr/bin/sendmail. Tidak ada konfigurasi, dengan asumsi Anda menggunakan relai terbuka. Jika Anda menggunakan relai yang membutuhkan SMTP AUTH (Anda mungkin sering mendengar istilah SASL, yang sebagian akurat), Anda dapat melakukan hal seperti berikut:

\n\n

File /etc/dma/auth.conf: 

some user|some mail relay or submission system:password\n

File /etc/dma/dma.conf (selain yang sudah ada di sana secara default): 

SMARTHOST=hostname or IP address of smarthost or submission system or mail relay\n

Anda mungkin harus mengkonfigurasi MUA Anda dengan benar. Jika saya ingat dengan benar, mailx dan mutt tidak memerlukan konfigurasi tambahan.
