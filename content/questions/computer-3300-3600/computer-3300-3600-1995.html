Saya telah mengalami masalah dengan Raspberry Pi saya: itu terus memasang hard drive USB saya sebagai hanya-baca. Saya telah memasangnya dengan sudo mount -t ntfs -o umask=000 /dev/sda1 \"/media/My Book\" tetapi untuk beberapa alasan sekarang dikatakan mount: warning: /media/My Book/ seems to be mounted read-only ketika saya melakukannya, dan ls -l menunjukkan itu benar. Drive memiliki catu daya sendiri dan berfungsi dengan baik dengan laptop saya, jadi saya pikir masalahnya pasti ada pada Pi. Apa yang saya lakukan salah?

\n\n

Hanya untuk memperjelas, perintah itu dulu berfungsi, tetapi sekarang tidak. Dan sekarang hanya-baca untuk SEMUA ORANG, bahkan root: 

$ ls -l /media\ntotal 8\ndr-xr-xr-x 1 root root 4096 Apr 28 11:11 My Book\n

Isinya muncul dengan cara yang sama.

\n\n

Pembaruan: Mungkin menemukan penyebabnya. Dari /var/log/syslog: 

May 17 19:37:52 raspberrypi kernel: [   64.688724] NTFS driver 2.1.30 [Flags: R/W MODULE].\nMay 17 19:37:52 raspberrypi kernel: [   65.195397] NTFS volume version 3.1.\nMay 17 19:37:52 raspberrypi kernel: [   65.195439] NTFS-fs error (device sda1): load_system_files(): Volume is dirty.  Mounting read-only.  Run chkdsk and mount in Windows.\n

Mungkin seharusnya sudah mencoba mencari di sana sebelumnya ...
