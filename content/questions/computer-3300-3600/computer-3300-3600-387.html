Saya telah menginstal Jenkins melalui brew, dan sekarang berjalan pada port 8080 pada Machine A (dengan IP 10.0.6.6): 10.0.6.6:8080 memberi saya Jenkins ketika pada Machine A.

\n\n

Di Mesin B, jika saya mengetik 10.0.6.6:8080, halaman tidak dapat dimuat. Saya telah memastikan bahwa port 8080 memang terbuka, dan saya membukanya secara manual dengan: 

sudo ipfw add 8080 allow tcp from any to any dst-port 8080\n

Saya menjalankan Apache dan mendengarkan pada port 80 pada Machine A, dan saya dapat mengaksesnya dengan baik dari Mesin B dengan 10.0.6.6:80

\n\n

Menjalankan sudo lsof -i:8080 untuk melihat apa yang sedang didengarkan pada port 8080 memberikan: 

java      67405            root  168u  IPv6 0x8559ae29226e47f7      0t0  TCP localhost:http-alt (LISTEN)\njava      67405            root  276u  IPv6 0x8559ae2927c6b397      0t0  TCP localhost:http-alt->localhost:52067 (ESTABLISHED)\n

Machine A ada di Mac OSX 10.8.5. Tahu apa yang menyebabkan ini?

\n\n

Detail tambahan yang bisa saya berikan:

\n\n

Saya menginstal Jenkins dengan: 

brew install jenkins\n

Saya memulai Jenkins dengan: 

sudo launchctl load ~/Library/LaunchAgents/homebrew.mxcl.jenkins.plist\n
","source":"https://superuser.com/questions/650420/jenkins-not-accessible-through-lan","author":"Snowman","replies":[],"answers":[{"answer":"

Homebrew.mxcl.jenkins.plist default dari homebrew memiliki yang berikut ini: 

<string>--httpListenAddress=127.0.0.1</string>\n

Ini membuatnya hanya mendengarkan di localhost. Untuk membuatnya mendengarkan di semua antarmuka Anda dapat menggantinya dengan: 

<string>--httpListenAddress=0.0.0.0</string>\n

Maka saya pikir Anda perlu launchctl unload / load untuk memulai kembali. Anda juga bisa memasukkan alamat mendengarkan yang lebih spesifik jika Anda mau, tentunya.
