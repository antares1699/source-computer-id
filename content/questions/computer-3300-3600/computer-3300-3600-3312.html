Ada tutorial di Forum Ubuntu: tautan

\n\n
\n

Ini adalah cara singkat dan manis yang tetap sangat penting. Saya t\n didasarkan pada pengalaman saya saat mengubah nama host suatu sistem. Itu\n nama host secara efektif hanya nama sistem. Anda tidak\n biasanya perlu mengubahnya, tetapi jika Anda harus melakukannya karena alasan apa pun,\n itu tidak terlalu sulit sama sekali.

\n \n

Meskipun mengubah nama host bukanlah tugas yang sulit, itu bisa\n berbahaya karena jika sistem tidak dapat menyelesaikan nama hostnya sendiri,\n tidak mungkin sudo. Jadi, jika dilakukan dengan tidak benar, mengubah a\n nama host dapat mengunci Anda dari sistem Anda sendiri. Pertama, dan paling banyak\n yang terpenting, kami membuat cadangan. Buka aksesori / terminal dan tempel\n perintah ini masuk, lalu tekan enter: 

sudo cp /etc/hosts /etc/hosts.bak && sudo cp /etc/hostname /etc/hostname.bak\n

Anda HARUS membuat cadangan, untuk berjaga-jaga. Jika Anda tidak bisa boot ke file yang bisa digunakan\n sistem setelah menyelesaikan cara ini, atau sudo tidak berfungsi, lihat\n instruksi di akhir panduan ini. Jika Anda tidak sepenuhnya\n akrab dengan terminal linux, mungkin ada baiknya untuk menulis\n ini ke bawah atau cetak.

\n \n

Ok, sekarang kita sudah melakukan backup, kita bisa edit / etc / hosts: 

sudo nano /etc/hosts\n

Anda akan melihat apa yang Anda cari dengan cukup cepat, di bagian atas\n file itu akan terlihat seperti: Kode: 

127.0.0.1       localhost\n127.0.1.1       old_hostname\n

Ubah old_hostname menjadi hostname baru pilihan Anda. Sekarang simpan dan\n keluar.

\n \n

File kedua yang akan diedit adalah / etc / hostname. Ini melakukan apa yang tertulis di\n timah, file tersebut akan memiliki satu baris di dalamnya dan baris itu adalah nama host Anda.\n Ubah ke nama host baru Anda, simpan dan keluar.

\n \n

Sekarang reboot, Anda harus memiliki sistem yang berfungsi dengan baik di mana sudo\n bekerja. Anda dapat menguji sudo dan nama host baru Anda dengan melakukan, cukup: 

sudo hostname\n

CATATAN: Perintah ini biasanya tidak perlu dijalankan sudo, kita cukup\n menguji nama host yang diubah DAN sudo secara bersamaan.

\n \n

Jika tidak berhasil, lihat di bawah. Anda DID mengambil cadangan, bukan?

\n \n
    \n
  • Masuk ke menu grub dengan menekan esc saat bootup, atau izinkan untuk ditampilkan secara otomatis jika Anda telah mengaturnya seperti ini
    • \n
  • Pilih mode pemulihan
    • \n
  • Jika diminta, pilih "drop to single user mode" atau serupa (catatan: ini hanya berlaku pada hardy menurut saya)
    • \n

  • Saat Anda berada di shell root, ketik perintah berikut: 

    cp /etc/hosts.bak /etc/hosts && cp /etc/hostname.bak /etc/hostname\n
    • \n
\n \n

Kemudian tekan ctrl-alt-del dan Anda harus mem-boot ulang kembali ke sistem yang berfungsi sempurna.
