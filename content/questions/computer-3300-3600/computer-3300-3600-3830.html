Apakah Anda mencari sesuatu seperti perintah tr? Ini dia manpages . Ini disertakan dengan msysgit, cygwin, dan gnuwin32 tools sejauh yang saya tahu.

\n\n

Jadi Anda dapat menghapus semua spasi sebelum diffing dengan melakukan sesuatu seperti: 

tr --delete '[:space:]' <filename.txt\n

Anda kemudian dapat memberi makan output dari perintah itu ke diff dan membuatnya berfungsi tanpa spasi.

\n\n

Misalnya, saya memiliki file bernama HelloWorldApp.java. Izinkan saya menunjukkan kepada Anda bagaimana cygwin memprosesnya: 

C:\\temp>cat HelloWorldApp.java\nclass HelloWorldApp {\n    public static void main(String[] args) {\n        System.out.println(\"Hello World!\"); // Display the string.\n    }\n}\nC:\\temp>tr -d '[:space:]' <HelloWorldApp.java\nclassHelloWorldApp{publicstaticvoidmain(String[]args){System.out.println(\"HelloWorld!\");//Displaythestring.}}\n

Meskipun berorientasi pada garis, diff bisa dibuat untuk melakukan apa yang ada\ndiminta. Secara kasar, Anda dapat melakukan diff -w dan kemudian memposting proses\nkeluaran diff. \n\n

Sepertinya itu mungkin O (n ^ 2) atau hal buruk lainnya, tapi tetap saja\nsangat membantu jika membatasi diri pada (katakanlah) tidak bergabung, gabung 2 baris, gabung 3 baris pada file 1\nX no join, join 2 lines, join 3 lines on file 2 on a moving "window" of the diff output.\n(dengan X adalah ~ "produk silang")

\n\n
