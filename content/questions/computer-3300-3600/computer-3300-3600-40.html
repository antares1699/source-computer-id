Anda mencari file konfigurasi ~/.ssh/config. Misalnya, tambahkan entri ini di file ini. 

Host host\n    HostName host.example.com\n    User johndoe\n    IdentityFile ~/.ssh/host.key\n

Sekarang Anda bisa mengetik 

ssh host\n

Dan ssh secara otomatis akan menggunakan nama pengguna dan kunci yang Anda tentukan oleh parameter IdentityFile. Oleh karena itu, perintah itu setara dengan: 

ssh johndoe@host.example.com -i ~/.ssh/host.key\n

Lihat http://www.openbsd.org/cgi-bin/man.cgi/OpenBSD-current/man5/ssh_config.5 untuk semua detail konfigurasi untuk file itu.

Jika Anda berada di * nix, Anda dapat menggunakan ssh-agent / ssh-add untuk menyimpan identitas Anda di memori untuk sesi Anda.

\n\n
    \n
  1. Jalankan ssh-agent dari terminal
    2. \n
\n\n

pengguna @ mesin $ ssh-agent bash

\n\n

Ini akan membuat proses bash dengan ssh-agent berjalan

\n\n

Jika Anda tidak ingin membuat proses lain, maka

\n\n

pengguna @ mesin $ ssh-agent -s

\n\n

SSH_AUTH_SOCK = / tmp / ssh-shtzxe4347 / agent.4347; ekspor SSH_AUTH_SOCK;\nSSH_AGENT_PID = 4348; ekspor SSH_AGENT_PID; echo Agen pid 4348;

\n\n

Anda harus menjalankan perintah dengan huruf tebal untuk memperbarui variabel env Anda.

\n\n

2. Setelah salah satu langkah sebelumnya

\n\n

pengguna @ mesin $ ssh-add

\n\n

untuk menambahkan .ssh / id_rsa Anda ke agen, jika kunci Anda memiliki kata sandi, itu akan menanyakannya

\n\n

http://linux.die.net/man/1/ssh-agent \n http://linux.die.net/man/1/ssh-add

\n\n
\n\n

Desktop Linux modern memiliki GUI untuk ini, dan akan memulai agen sejak login.

\n\n

Di Windows, Anda dapat menggunakan Pageant (bagian dari Putty) sebagai ganti ssh-agent.
