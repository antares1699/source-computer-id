Setiap kali saya meluncurkan aplikasi KDE dari baris perintah, misalnya okular untuk melihat pdf, saya mendapatkan spam dengan banyak peringatan dan berbagai pesan yang tidak saya minati. Bahkan beberapa jam setelah saya menutup program, saya mendapatkan pesan acak di terminal itu . Berikut ini contohnya: 

kded(26751)/kdecore (KConfigSkeleton) KCoreConfigSkeleton::writeConfig:\nkded(26751)/kdecore (KConfigSkeleton) KCoreConfigSkeleton::writeConfig:\nkded(26751)/kdecore (KConfigSkeleton) KCoreConfigSkeleton::writeConfig:\nkded(26751)/kdecore (KConfigSkeleton) KCoreConfigSkeleton::writeConfig:\n

Anda mengerti. Bagaimana cara menonaktifkan perilaku ini semua Aplikasi KDE tanpa harus menambahkan sesuatu seperti >/dev/null setiap saat?

\n\n

Saya menggunakan KDE 4.4.5.

Secara pribadi, saya baru saja menutup program pada 2>/dev/null. Anda harus membuangnya semua pesan kesalahan untuk semua aplikasi, atau simpan daftar nama biner di suatu tempat dengan program yang ingin Anda sembunyikan.

\n\n

Untuk menyimpan daftarnya, Anda bisa mis. letakkan skrip overloading di PATH Anda dengan nama yang sesuai dengan binari, mis. okular: 

#!/bin/sh\n/usr/bin/okular \"${@}\" 2>/dev/null\n

Atau, Anda dapat membuat skrip bernama mis. q (untuk diam) dan letakkan di jalur Anda dengan isinya: 

#!/bin/sh\n\"${@}\" 2>/dev/null\n

lalu tambahkan alias seperti 

alias okular=\"q okular\"\n

dan seterusnya.
