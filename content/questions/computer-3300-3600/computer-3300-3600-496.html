Ini benar-benar terdengar seperti bug bagi saya, karena subnet mask dan alamat IP saya seharusnya valid oleh semua akun.

\n\n

ISP saya telah memberi saya alamat IP tetap 65.110.7.20, dengan subnet mask 255.255.255.252. Kedengarannya benar, karena kami mencoba menyiapkan router dengan jaringan / 31 - hanya 4 alamat ip.

\n\n

\n\n

"Kombinasi alamat IP dan subnet mask tidak valid. Semua bit di bagian alamat host dari alamat IP disetel ke 0."

\n\n

Tapi mereka tidak semua diatur ke 0. Itu akan menjadi subnet mask 255.255.255.255. Bahkan kemudian, itu juga merupakan subnet mask yang valid, yang terdiri dari jaringan dengan 1 alamat IP.

\n\n

Apakah ada jalan keluarnya? Mungkin peretasan registri?

Alamat pertama dalam jaringan tidak valid untuk host, karena digunakan untuk mengidentifikasi jaringan.

\n\n

65.110.7.20/30 rusak seperti ini

\n\n
\n\n

65.110.7.20 - Pengidentifikasi jaringan - tidak dapat digunakan untuk host
65.110.7.21 - OK untuk digunakan oleh tuan rumah
65.110.7.22 - OK untuk digunakan oleh tuan rumah
65.110.7.23 - Alamat siaran untuk jaringan - Tidak dapat digunakan untuk host

\n\n
\n\n

Pada kenyataannya, Anda hanya memiliki 2 alamat yang dapat digunakan. Saya yakin jika Anda mengatur IP ke alamat host yang valid, Anda tidak akan memiliki masalah.
