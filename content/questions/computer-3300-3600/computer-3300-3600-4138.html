Saya meninggalkan driver adaptor ethernet di make menuconfig saya - apakah saya harus membangun kernel lagi (gentoo distro) atau adakah cara untuk membangun dukungan setelah os diinstal dan dijalankan?

","source":"https://superuser.com/questions/369614/if-i-miss-something-in-make-menuconfig-do-i-need-to-rebuild-the-kernel","author":"rutherford","replies":[],"answers":[{"answer":"

Jika Anda tidak menjalankan "make clean" (atau yang serupa) setelah kompilasi, Anda hanya mengompilasi perbedaannya saat menjalankan "make" lagi.

\n\n

Contoh: 

# make menuconfig\n# make \n# make modules_install\n# make install\n

"Oh, saya lupa driver ethernet-nya": 

# make menuconfig (change the old config)\n# make (now, you're only compiling the changes)\n# make modules_install\n# make install\n

Jika Anda menghapus direktori dengan kernel, ya, Anda perlu mengkompilasi semuanya lagi.

Itu terutama tergantung pada cara Anda menggunakan kernel. Dimungkinkan untuk hanya menautkan modul.

\n\n

Dari pengalaman saya, kecuali jika Anda memiliki alasan yang sangat bagus mengapa tidak membangun kembali kernel, yang terbaik adalah membangunnya kembali. Sistem Anda harus berjalan dengan baik (cukup baik sehingga Anda dapat membangun kembali kernel) bahkan tanpa dukungan Ethernet dan semua pengaturan yang Anda buat saat Anda membangun kernel untuk pertama kali akan disimpan sehingga Anda hanya perlu mengaktifkan opsi yang Anda butuhkan dan menjalankan\nmake && make modules_install dan kemudian salin kernel baru sebagai pengganti kernel lama.
