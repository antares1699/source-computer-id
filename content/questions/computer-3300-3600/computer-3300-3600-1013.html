Saya mengonfigurasi sistem Lenovo U410 baru dengan Windows 7 Home Premium untuk pengguna. Saya menerima sistem langsung dari toko. Sebagai bagian dari konfigurasi, saya menginstal Java menggunakan penginstal online. Ini bekerja dengan benar.

\n\n

Kemudian, karena kesalahan yang saya buat, saya perlu mengembalikan sistem ke default pabrik. FORMAT C default pabrik: \\ dan mengembalikan (seharusnya) konfigurasi pabrik yang tepat. Namun, setelah melakukan ini, saya tidak lagi dapat menginstal Java dengan sukses menggunakan metode yang sama yang saya gunakan sebelumnya.

\n\n

Sekarang, setiap kali saya mencoba menggunakan penginstal Java online, hal berikut terjadi. Pertama-tama, selalu muncul jendela "Selamat datang di Java", "Download Penginstal Java ...". Setelah beberapa saat, jendela ini menghilang dan kemudian salah satu dari tiga hal terjadi:

Saya telah mencoba mengunduh dengan Chrome dan Internet Explorer 9. Saya telah mencoba mencopot semua perangkat lunak anti-virus dan menonaktifkan firewall windows sepenuhnya. Satu-satunya hal yang membuat perbedaan adalah menjalankan penginstal dalam mode kompatibilitas Windows XP, yang memungkinkan penginstalan selesai.

Saya tahu saya dapat mengatasi kesalahan ini dengan menggunakan penginstal offline jadi tolong jangan posting itu sebagai jawaban. Saya mencari penjelasan tentang akar masalahnya. Selain itu, jika saya menggunakan penginstal offline, pembaru tidak berfungsi. Updater juga tidak berfungsi jika saya menginstal dalam mode XP. Pembaru gagal karena bekerja dengan hanya mengunduh pengaturan online terbaru dan menjalankannya.

\n\n

Juga ingat bahwa installernya adalah ditandatangani secara digital. Tanda-tanda memverifikasi dengan benar sehingga tidak mungkin ini disebabkan oleh unduhan yang rusak.

Beberapa teori yang saya miliki:

\n\n
    \n

  • File pengaturan Java di java.com sebenarnya berubah di antara pemasangan pertama yang berhasil dan upaya saya selanjutnya. Sepertinya tidak mungkin karena tidak ada nomor versi yang berubah. Namun, saya telah melihat beberapa laporan tentang kesalahan ini yang muncul dalam 24 jam terakhir. Ini sepertinya penjelasan yang paling mungkin saat ini : http://www.oracle.com/us/corporate/press/1735645 - Oracle merilis 7 update 6 dua hari lalu. Pemeriksaan hati-hati terhadap pemasang mengungkapkan bahwa mereka sebenarnya mencoba mengunduh 0,6, bukan 0,5 seperti yang diklaim laman unduhan. Sebenarnya tidak benar. Hanya alat pembaruan yang mencoba menginstal 7u6. Penginstal online masih mencoba 7u5. Namun, 7u6 yang dirilis dua hari lalu terlalu kebetulan untuk diabaikan. Pembaruan: Penginstal online 7u6 tersedia dari technetwork Oracle. Itu crash dengan cara yang persis sama.

    • \n

  • Perangkat lunak reset pabrik menggunakan GMT-8 dan saya menggunakan GMT-1. Akibatnya, setelah pengaturan ulang pabrik, perangkat lunak apa pun yang peduli untuk diperiksa akan berpikir bahwa sistem dipulihkan 7 jam di masa depan, karena kebijakan buruk Window dalam menyimpan waktu lokal di jam sistem. Ini bisa membingungkan pemeriksaan sertifikat atau serupa. Pembaruan: Saya menemukan bahwa ini menyebabkan Pembaruan Windows gagal. Solusinya, menyetel jam kembali sebelum memulai reset pabrik, tidak memungkinkan Java untuk menginstal dengan benar.

    • \n

  • Gambar reset pabrik tidak benar-benar sama dengan yang dipasang di partisi utama saat Anda membeli sistem. Lenovo yang nakal.

    • \n

  • Penginstal tampaknya macet saat menginstal atau menampilkan sesuatu yang berkaitan dengan bilah alat Ask.com. Sepertinya itulah yang dilakukan javaic.dll.

    • \n

  • Microsoft Tuesday adalah tanggal 14. Beberapa pembaruan di dalamnya dapat menyebabkan ini. Namun, saya mengatur ulang mesin setiap saat, jadi kecuali tambalan disisipkan ke dalam citra pemulihan, atau ada beberapa mekanisme yang dengannya mereka dipasang secara diam-diam bahkan jika pembaruan dinonaktifkan, maka saya tidak melihat bagaimana ini bisa terjadi. penyebab.

Terobosan besar:

\n\n

Browser default pada sistem Lenovo adalah Google Chrome. Saya perhatikan bahwa "cek sponsor" JavaIC.dll sebenarnya melakukan pemeriksaan pada browser default Anda untuk memutuskan iklan sponsor mana yang akan ditampilkan. Biasanya itu akan memberi Anda Ask toolbar di IE9. Tetapi bilah alat itu tidak berfungsi di Chrome, sehingga pemasang mencoba menampilkan iklan yang berbeda. Iklan yang berbeda inilah yang menyebabkan crash. Mengubah browser default ke IE9 memungkinkan penginstal berjalan dengan benar. Jadi ini terlihat seperti bug asli pada kode iklan sponsor di penginstal, yang disebabkan oleh kombinasi browser default Google Chrome dan tidak ada di AS. (Pemasang juga memeriksa lokasi Anda menggunakan layanan geolokasi IP dan menampilkan iklan yang berbeda berdasarkan itu.)
