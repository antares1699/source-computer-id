Saya mencoba menulis FreeBSD-10.3-RELEASE-amd64-memstick.img ke USB flash drive, dengan harapan menginstal FreeBSD pada komputer x64 di rumah saya. Ini gambarnya: ftp://ftp.freebsd.org/pub/FreeBSD/releases/amd64/amd64/ISO-IMAGES/10.3/.

\n\n

Saya telah mencari cara menulis file .img ke USB flash drive, dengan sebagian besar situs mengatakan untuk menggunakan Unetbootin , Pembuat Disk Mulai , atau perintah dd . Ketika saya menghubungkan dan memilih flash drive setelah menggunakan Unetbootin, saya menerima pesan "Sistem operasi hilang". Dengan Disk Creator, saya menerima pesan yang mengatakan bahwa Disk Creator tidak dapat menulis gambar ke flash drive.

\n\n

Adapun perintah dd , Saya menggunakan ini: dd if=FreeBSD-10.3-RELEASE-amd64-memstick.img of=/dev/sdb1 bs=10240 conv=sync, yang saya dapatkan dari sini dan sini . Perintah tersebut menulis banyak file sampah ke flash drive saya, alih-alih konten gambar FreeBSD (yang saya asumsikan bukan file sampah).

\n\n

Bagaimana cara saya menulis FreeBSD ke flash drive dan menginstalnya ke komputer saya, mengingat opsi ini tidak berfungsi?

Dengan asumsi Anda menggunakan dd dalam FreeBSD atau Linux:

\n\n

Menggunakan Linux:

\n\n

1) umount /dev/sdb

\n\n

Jika stik USB Anda di bawah / dev / sdb

\n\n

2) dd if=FreeBSD-10.3-RELEASE-amd64-memstick.img of=/dev/sdb bs=10240

\n\n

Menggunakan (Gratis) BSD:

\n\n

1) umount /dev/da0

\n\n

Jika stik USB Anda di bawah / dev / da0

\n\n

2) dd if=FreeBSD-10.3-RELEASE-amd64-memstick.img of=/dev/da0 bs=1m

\n\n

Harap perhatikan argumen of=/dev/sdb karena Anda seharusnya menulis ke seluruh tongkat alih-alih partisi pertamanya.

\n\n

Hal yang sama juga terjadi pada of=/dev/da0. Namun perlu diketahui bahwa da adalah khas awalan untuk perangkat port USB di (Gratis) BSD. Yang berarti mungkin saja da1. Jadi pastikan untuk memeriksa terlebih dahulu. :)

\n\n

Cara termudah adalah membandingkan hasil ls /dev sebelum dan sesudah mencolokkan / mencabut flash drive / stik USB Anda.

\n\n

3) Sekarang reboot dan pilih USB stick sebagai opsi boot.

Jika Anda mencoba menulis gambar ke USB di Windows, coba Rufus . Pilih drive USB Anda di bawah Device, periksa Create a bootable disk using dan pilih DD Image dari drop down, telusuri ke file gambar dan tekan Start.

\n\n

Tidak jelas dari uraian Anda apakah masalah Anda semata-mata dengan pembuatan drive atau dengan mem-boot dari kunci USB yang mungkin bagus (versi dd harus baik dilihat dari perintah dan dengan asumsi sdb adalah perangkat yang tepat)
