Macports benar-benar tidak suka berada di / usr / local. Dari Pertanyaan Umum MacPorts

\n\n

/ opt / local dipilih untuk menghindari menginjak berbagai instalasi lainnya (misalnya fink menggunakan / sw untuk melakukan hal yang sama); / usr / local bukanlah pilihan yang tepat karena beberapa alasan:

\n\n
    \n
  1. Beberapa perangkat lunak (terutama alat * otomatis dari Gnu) mencari di / usr / local sebagai lokasi default, yang berarti MacPorts tidak dapat dengan mudah diisolasi saat diperlukan
    2. \n
  2. / usr / local biasanya disediakan untuk admin sistem yang diberikan untuk menginstal item lokal ke sistem itu, dan cenderung menjadi pilihan yang buruk untuk diambil alih oleh sistem port non-sistem
    3. \n
  3. gcc menganggap / usr / local sebagai direktori sistem standar, menyebabkan (setidaknya) direktori include tidak dapat muncul di awal daftar direktori include, dan karenanya menyebabkan item yang diinstal MacPorts sulit digunakan dengan benar untuk item yang membutuhkannya (di mana versi lain diinstal di tempat lain, seperti / usr / X11R6)
    4. \n
\n\n

Juga jika Anda memiliki penginstalan dari sumber yang berbeda, Anda yakin bahwa versi tersebut persis sama. Jika tidak, Anda akan mengalami masalah build dan runtime yang aneh.

\n\n

Jika setelah semua itu Anda masih ingin mencoba kami metode yang tidak didukung ini dan Anda tidak dapat menggunakan jalur untuk menemukan hal-hal yang saya duga akan lebih mudah untuk membuat kode Anda mencari di / opt / local / bin lalu / us / local / bin lalu pilih salah satu yang ditemukannya (juga apa yang terjadi jika keduanya ada)
