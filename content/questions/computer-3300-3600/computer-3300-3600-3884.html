Saya memiliki file teks download-list.txt yang terlihat seperti ini: 

http://www.example.com/file/image1.png\nhttp://www.example.com/file/image2.png\nhttp://www.example.com/file/image3.png\n.\n.\n.\n

Untuk mendownload file dalam daftar yang saya jalankan 

wget -i download-list.txt\n

Apakah ada sihir wget yang dapat saya gunakan yang tidak mengharuskan saya membuat daftar seperti itu?

","source":"https://superuser.com/questions/593433/more-optimal-method-to-wget-several-images-that-differ-only-by-image-number-in-t","author":"hpesoj626","replies":[],"answers":[{"answer":"

Jika Anda menggunakan bash yang cukup modern, Anda dapat menggunakan ekspansi shell seperti ini: 

wget http://www.example.com/file/image{1..99}.png\n

Di atas akan mencoba untuk mendapatkan image1.png hingga image99.png. Jelas, pilih kisaran yang sesuai. File-file yang tidak ada itu seharusnya memunculkan kesalahan, tetapi tidak menyebabkan masalah sebenarnya. Jika mau, Anda mungkin dapat menekan kesalahan dengan beberapa opsi wget, atau dengan mengarahkan STDERR ke / dev / null: 

wget http://www.example.com/file/image{1..99}.png 2> /dev/null\n

Untuk jumlah file yang sangat besar, Anda mungkin menemui batasan baris perintah - ada jumlah maksimum argumen yang dapat berada di baris perintah. Anda sangat tidak mungkin mengalami masalah ini, tetapi jika Anda berurusan dengan file dalam jumlah yang sangat besar, loop for mungkin lebih aman: 

for f in {1..1000}; do wget http://www.example.com/file/image$f.png; done\n##  and the version sending STDERR to /dev/null:\nfor f in {1..1000}; do wget http://www.example.com/file/image$f.png 2> /dev/null; done\n
