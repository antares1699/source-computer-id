Lihat tautan ini jika Anda menggunakan fitur pengujian unit Visual Studio 2010

\n\n

Pengaturan Server Proxy diatur secara otomatis yang menyebabkan saya tidak dapat menjalankan pengujian manual.

\n\n
\n

Dapatkah Anda menonaktifkan Diagnostic Data Adapter "ASP.NET Client Proxy untuk IntelliTrace dan Test Impact" dan memeriksa apakah Anda masih menghadapi masalah ini? Untuk melakukan ini lakukan langkah-langkah berikut.

\n \n

Rencanakan-> Properti-> Proses Manual: Pengaturan Tes -> untuk membuka wizard pengaturan tes.

\n \n

Di wisaya pengaturan pengujian, buka "Tab Data dan Diagnostik" dan matikan "Proxy Klien ASP.NET ...". Klik Selesai dan ingat untuk menyimpan rencananya. Sekarang jalankan tes dan lihat apakah Anda masih menghadapi masalah ini.

Masalah asli saya ditemukan ketika Fungsi pencarian Google dengan lingkungan Chrome dengan Window 8.1 menjadi tidak dapat diandalkan karena sering membalas pesan "Halaman web tidak tersedia dengan kode kesalahan: ERR_CONTENT_DECORDING_FAILED" . Beberapa pencarian pertama baik-baik saja tetapi setelah pesan kesalahan ini muncul, saya tidak dapat lagi menggunakan pencarian Google. Pesan kesalahan ini tidak tercantum dalam dukungan teknis Chrome.

\n\n

Namun, tampaknya ini disebabkan oleh pengiriman kode yang salah dari PC saya ke mesin pencari di situs Google. Setelah berjuang, saya menemukan itu mungkin disebabkan oleh pengaturan LAN yang salah di opsi Internet: yaitu (1) "gunakan server proxy untuk LAN Anda" dicentang, (2) "Deteksi pengaturan secara otomatis" tidak dicentang. Seharusnya ada cara yang salah, sehingga saya reset ke kondisi default: (1) gunakan server proxy ... tidak dicentang, (2) Deteksi otomatis ..... Centang. Fungsi pencarian Google menjadi lebih baik tetapi masih mengembalikan situasi kesalahan setelah beberapa pencarian tes. Saya perhatikan itu memiliki waktu bergantung untuk kemunculan kembali masalah.

\n\n

Saya memeriksa pengaturan LAN setiap kali dan menemukan bahwa pengaturan itu otomatis berubah kembali . Akhirnya saya menggunakan "Task Manager" untuk memeriksa perangkat lunak yang sedang berjalan yang berhubungan dengan koneksi Internet dan menemukan perangkat lunak yang disebut "BrouserSafeguard dengan Tab roket" seperti yang dinyatakan di salah satu jawaban Anda. Ini sepertinya mengubah kembali pengaturan secara otomatis saat memindai setiap menit. Hal ini menyebabkan mesin pencari Google tetapi mesin pencari lain seperti "Bing" dan "Ask Jeeves" tidak terpengaruh oleh pengaturan di Chrome. Ini bukan malware atau virus, jadi anti-virus Norton saya tidak mendeteksinya. Saya mencopotnya dari "Control Panel" secara manual dan masalah telah terpecahkan. Saya berterima kasih atas jawaban Anda yang paling berguna di situs ini.
