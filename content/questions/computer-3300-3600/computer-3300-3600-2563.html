Larutan

\n\n

Ini harus melakukan trik: 

PS4='+%x:%I>' zsh -i -x -c '' |& grep subl\n

Ini harus mencetak sesuatu seperti 

+/Users/brandon/.zshrc.d/aliases.zsh:32>alias 'subl=/Applications/Sublime\\ Text\\ 2.app/Contents/SharedSupport/bin/subl'\n

Artinya pada file /Users/brandon/.zshrc.d/aliases.zsh (contoh saja) pada baris 32 perintah alias 'subl=/Applications/Sublime\\ Text\\ 2.app/Contents/SharedSupport/bin/subl' dijalankan. Perhatikan bahwa baris perintah yang ditampilkan adalah hasil penguraian perintah yang ditulis dalam file, karena itu mungkin tidak bersifat indentikal.

\n\n
\n\n

Penjelasan

\n\n

Konfigurasi zsh pada dasarnya adalah daftar perintah shell. Informasi dari mana perintah dijalankan biasanya hanya tersedia pada saat perintah benar-benar dijalankan. Setelah perintah selesai, perubahan apa pun di lingkungan atau pengaturan shell mungkin tetap ada, tetapi informasi bagaimana perubahan ini terjadi telah hilang. Jadi mencari file konfigurasi atau menelusuri informasi selama runtime adalah satu-satunya pilihan.

\n\n

Pencarian file konfigurasi - seperti yang Anda perhatikan - rentan terhadap kesalahan, karena mudah untuk mengabaikan beberapa file konfigurasi; zsh langsung memuat hingga delapan file konfigurasi berbeda saat memulai shell interaktif dan sejumlah file tambahan dapat diambil dari sana.

\n\n

Disinilah tempatnya zsh ini XTRACE (-x) mulai dimainkan. Mirip dengan VERBOSE (-v) XTRACE mencetak baris perintah yang dijalankan, tetapi juga mendahului setiap baris dengan prompt PS4. PS4 dapat dikonfigurasi untuk menunjukkan file dan nomor baris yang sedang dijalankan. (Secara default PS4 menunjukkan nama fungsi atau file script dan nomor baris yang sedang dieksekusi. Jadi, jika sebuah baris kode berada di dalam suatu fungsi, itu tidak akan langsung menunjukkan di mana ia didefinisikan.).

\n\n

Secara rinci perintah di atas melakukan hal berikut:

\n\n
    \n

  • Tetapkan variabel lingkungan PS4 ke +%x:%I> (hanya untuk perintah ini). %x diganti dengan nama file yang berisi kode sumber yang sedang dijalankan. %I adalah nomor baris dari kode tersebut di dalam file. + dan > adalah dekorasi untuk membedakan keluaran XTRACE dari keluaran apa pun yang mungkin dihasilkan oleh skrip.

    \n\n

    catatan: Ini mungkin tidak bekerja dengan benar, jika PS4 disetel ke nilai yang berbeda di suatu tempat dalam konfigurasi. Jika keluaran tidak memiliki format yang diinginkan, mungkin perlu terlebih dahulu menghapus / mengomentari pengaturan PS4 atau PROMPT4.

    • \n

  • zsh dimulai zsh

    \n\n
      \n
    • dengan mode interaktif diaktifkan (-i)
      • \n
    • dengan XTRACE diaktifkan (-x)
      • \n
    • dan hanya menjalankan baris perintah kosong (-c ''), segera keluar setelahnya.
      • \n
    \n\n

    Artinya, hanya file inisialisasi untuk mode interaktif yang dijalankan dan tidak ada kekacauan dari pembuatan prompt atau perintah hook.

    • \n
  • |& adalah singkatan dari 2>&1 |, yang menghubungkan keluaran standar dan kesalahan standar dari satu perintah ke masukan standar berikutnya.
    • \n
  • grep subl mencari "subl". Saya sarankan mencari nama alias saja, karena baris perintah yang tepat mungkin bukan alias subl=\"...\" karena penguraian. (Di zsh Anda bahkan dapat menentukan alias tanpa menggunakan perintah alias dengan menetapkan pasangan kunci-nilai dalam larik assoziatif aliases: aliases[subl]=\"...\")
