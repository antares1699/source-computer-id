Perintah bash saya menunjukkan nama host saya. Jika saya mengubah nama host saya dengan perintah hostname, pada /etc/hosts, /etc/hostname dan $HOSTNAME, prompt masih menggunakan nama lama. Apakah mungkin untuk memperbarui ini tanpa keluar?

","source":"https://superuser.com/questions/200057/how-to-make-bash-update-its-idea-of-hostname","author":"Sam Brightman","replies":[],"answers":[{"answer":"

Jika Anda memiliki \\h atau \\H dalam pengaturan prompt Anda, perluasannya diinisialisasi saat shell dimulai, jadi Anda harus memulai ulang bash (exec bash). Jika itu terlalu mengganggu Anda, ubah pengaturan prompt Anda untuk mencetak $HOSTNAME.

Jika Anda ingin prompt bash Anda diperbarui secara dinamis karena nama host (seperti yang dilaporkan oleh perintah /bin/hostname) diperbarui, Anda dapat melakukan ini: 

[root@foo ~]# PS1='[\\u@`/bin/hostname` \\W]\\$ '\n[root@foo ~]# hostname\nfoo\n[root@foo ~]# hostname bar\n[root@bar ~]# hostname baz\n[root@baz ~]#\n

Itu boros (menjalankan subkulit setiap kali Anda menekan kembali - tidak mengerikan tetapi tidak elegan,) jadi saya hanya akan menggunakan ini di mana situasinya membutuhkannya. Saya menggunakannya saat bekerja pada template VM dasar yang digunakan untuk menghasilkan beberapa instance VM target di mana metadata tentang instance tersebut ditunjukkan dalam nama host. Dan meskipun demikian, PS1 khusus hanya disetel sementara, sebagai bagian dari sesi masuk awal saat instance VM peran sedang diterapkan.
