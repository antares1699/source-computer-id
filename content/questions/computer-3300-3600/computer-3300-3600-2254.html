Minta administrator sistem Anda untuk menginstal klien bagian dari CUPS. (Anda tidak mengatakannya yang Linux yang Anda gunakan, jadi saya tidak dapat memberi tahu Anda nama paket mana yang akan menjadi ...)

\n\n

Ini akan memungkinkan Anda mencetak tanpa spooling lokal, asalkan server cetak CUPS jarak jauh memungkinkan Anda mengakses: 

lpstat -h remote.cups.host -p\n

kemudian akan mengembalikan Anda nama-nama printer yang tersedia pada remote.cups.host. 

lpoptions -h remote.cups.host -l -p printer33\n

akan menunjukkan kepada Anda opsi pekerjaan cetak mana yang printer33 tawarkan pada tuan rumah itu. 

lp -h remote.cups.host -d printer33 -o [your options go here] filename\n

akan mencetak nama file .

\n\n

Anda juga dapat membuat file ~/.cups/client.conf dengan konten ini: 

ServerName remote.cups.host\n

Dengan cara ini semua dialog cetak GUI akan tahu ke mana harus mencari printer dan opsi cetak, dan ke mana harus menggulung pekerjaan mereka.

Saya menyarankan Anda mulai dengan menanyakan administrator sistem Anda (atau pengguna sistem yang lebih berpengalaman) bagaimana cara mencetak. Jika ternyata pencetakan benar-benar belum disiapkan, tanyakan kepada mereka dengan sangat baik apakah mereka dapat memeriksanya.

\n\n

(Saya berasumsi, tentu saja, pertanyaan Anda berarti "Adakah cara bagi pengguna tanpa akses root untuk menyiapkan pencetakan?")

\n\n

Jika pencetakan tidak diatur dan sysadmin Anda tidak dapat menemukan waktu untuk mengaturnya, maka mungkin printer yang ingin Anda cetak ada di jaringan - tidak ada gunanya menghubungkan printer langsung ke server dan kemudian tidak mengkonfigurasi server untuk mencetaknya - dan Anda bisa mungkin menginstal semua yang diperlukan untuk mencetaknya di bawah direktori home Anda, tetapi mungkin akan cukup banyak pekerjaan untuk membangun semuanya, dan itu mungkin akan sedikit rapuh. (Karenanya, itu harus menjadi pilihan terakhir Anda.)

Saya tidak sepenuhnya yakin apa pengaturan Anda. Anda tidak perlu spooler (atau print server mis. CUPS ) pada mesin yang tidak memiliki printer yang terpasang (Anda cukup mengirimkan pekerjaan ke server cetak yang sebenarnya melalui sesuatu seperti Protokol Pencetakan Internet ), dan mesin dengan printer yang terpasang tidak akan berguna tanpa spooler. Apakah server memiliki printer yang terpasang atau di tempat lain di jaringan?

\n\n

Yang mengatakan, jika sysadmin Anda secara eksplisit mengatakan untuk tidak melakukan sesuatu, jangan mencoba melakukan sesuatu di belakang punggung mereka - itu cara yang bagus untuk mengaktifkan mode BOFH. Anda harus membicarakannya. Jelaskan kepada mereka mengapa Anda harus dapat mencetak di server itu (saya tidak bermaksud "melakukan pekerjaan saya" - sesuatu yang lebih spesifik seperti "Saya hanya bisa mendapatkan keluaran dari program X dengan mencetak"). Saat ini Anda sedang meminta sesuatu larutan (yaitu mengaktifkan pencetakan). Cobalah untuk langsung ke akar Anda masalah - Apa yang menghalangi Anda untuk tidak dapat mencetak, dan mengapa itu buruk? Jika Anda mempresentasikan ini masalah kepada administrator sistem Anda, mereka mungkin dapat menyarankan solusi berbeda yang menyelesaikan masalah Anda tanpa menyebabkan mereka sakit kepala tambahan seperti mencetak. Atau, ini dapat membantu mereka melihat bahwa pencetakan memang satu-satunya solusi, dan menyebabkan mereka mencari perbaikan untuk masalah yang mereka hadapi sebelumnya.
