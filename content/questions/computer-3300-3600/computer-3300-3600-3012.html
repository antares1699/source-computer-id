Setiap kali pemilihan visual berubah di MacVim, itu disalin ke papan klip sistem. Apakah ini normal dan dapatkah itu dinonaktifkan?

","source":"https://superuser.com/questions/348207/prevent-macvim-copying-every-visual-selection-to-clipboard","author":"DavidNorth","replies":[],"answers":[{"answer":"

Ini bukan perilaku default MacVim tetapi dapat diaktifkan dengan menggunakan set guioptions+=a atau set go+=a. Kemudian semua yang Anda pilih akan disalin ke clipboard Anda. Mungkin Anda memiliki baris itu di .vimrc Anda.

\n\n

Untuk menonaktifkannya, cukup gunakan set go-=a atau tambahkan ke .vimrc Anda.

Apakah Anda memilih foo di foo bar dan dapatkan foo di clipboard lalu ubah pikiran Anda dan pilih batang bukan dan menemukan bar di tempat foo di clipboard?

\n\n

Jika ya, saya tidak pernah menemukan perilaku ini di MacVim atau aplikasi MacOS X asli lainnya.

\n\n

Ini mungkin terjadi di GVim di Linux di mana ada lebih dari satu clipboard (salah satunya berisi teks yang dipilih) tetapi bahkan di sana itu akan tergantung pada bagaimana ia dikompilasi.

\n\n

Apakah Anda menemukan foo di clipboard setelah mengubahnya menjadi bar?

\n\n

Jika ya itu benar-benar normal: mengubah string dengan p atau c selalu menempatkan aslinya di register default yang sesuai dengan clipboard sistem di MacVim.

\n\n

Salah satu cara untuk melewati "masalah" ini adalah dengan menggunakan \"_dP daripada p. Artinya "hapus teks yang dipilih ke dalam 'register lubang hitam' (\"_d) lalu letakkan konten register default sebelum kursor".

\n\n

Anda dapat menggunakan strategi serupa karena c: \"_di akan melakukan hal yang sama tetapi menempatkan Anda dalam mode INSERT sebagai gantinya.
