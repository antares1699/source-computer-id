Saya menginstal Ubuntu-Server di Oracle Virtual Machine dan gagal menginstal langkah grub. Di akhir penginstalan, saya mendapat pesan: 

You need to manually boot linux\n/vmlinuz kernel on partition /dev/sda1 and\nroot=/dev/sda1 passed as kernel argument\n

Saya benar-benar ingin tahu cara mem-boot Linux tanpa boot loader.

","source":"https://superuser.com/questions/638433/booting-linux-without-bootloader","author":"Xk0nSid","replies":[{"reply":"Boot menggunakan live cd, pasang partisi root. instal grub","author":"hek2mgl"}],"answers":[{"answer":"

Untuk mem-boot linux secara manual, Anda masih harus memiliki bootloader. Biasanya Anda cukup menekan enter atau menunggu waktu tunggu, dan gambar default dimuat. Tetapi Anda dapat menentukan image kernel untuk dimuat dan beberapa parameter.

\n\n

Pada prompt grub, ketik 'vmlinuz root = / dev / sda1' (tidak yakin tentang leading / on vmlinuz). Istilah "manual" berarti Anda harus mengetikkan hal ini secara manual, daripada membiarkannya boot secara otomatis.

\n\n

Namun, karena langkah instalasi grub gagal, tidak jelas apakah Anda akan melihat prompt.
