bagaimana cara repo sinkronisasi berdasarkan tanggal?

Bisa saya gunakan repo untuk menyinkronkan menurut tanggal? (mungkin dalam git )

Misalnya, saya ingin mengunduh sumber mulai 1 Maret jam 3 sore. Saya perlu cara untuk mengambil basis tertentu dari CyanogenMod9 sumber. Sebaiknya gunakan tag atau commit ID, tetapi sumbernya diatur dalam beberapa sub-repositori yang tidak memiliki tag umum. Saya pikir saya bisa checkout berdasarkan tanggal. Saran?

6

Sepertinya saya harus menggunakan repo forall. Ini memerlukan sinkronisasi semuanya terlebih dahulu (tidak dapat melakukan pembayaran kode lama hingga repositori diunduh sepenuhnya).

repo forall -c 'git checkout `git rev-list --all -n1 --before="2012-03-01 15:00"`'

  • Ini mengasumsikan bahwa manifes tidak berubah sama sekali. Jika manifes saat ini berbeda dari manifes pada tanggal yang diinginkan, semua jenis masalah dapat mengganggu Anda --- berbagai cabang, jalur, perintah penyalinan file, dll.
    — Jeremy Frank
  • Anda seharusnya tidak menggunakan --all di sini. Ini akan mencantumkan komit dari cabang mana pun. Pastinya, Anda tidak ingin beranjak dari cabang saat ini. Anda hanya ingin pergi ke komit pertama dalam sejarah cabang saat ini, yang memenuhi persyaratan "sebelum": repo forall -c 'git checkout `git rev-list -n1 --before="2012-03-01 15:00"` HEAD'
    — schlimmchen
