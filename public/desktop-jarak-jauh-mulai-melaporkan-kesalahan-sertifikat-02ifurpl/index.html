Desktop jarak jauh mulai melaporkan kesalahan sertifikat

Saya terhubung ke beberapa mesin Windows 10 dengan alamat IP statis melalui RDP dari mesin Windows 10 lain untuk beberapa periode waktu mungkin sekitar beberapa bulan. Pada beberapa saat RDP mulai melaporkan kesalahan sertifikat:

1) Apa yang bisa menyebabkan ini?

2) Di mana saya dapat menemukan sertifikat sebelumnya untuk memeriksa apakah sudah kedaluwarsa atau tidak?

Ini agak normal, karena sistem Windows selalu membuat sertifikat TLS yang ditandatangani sendiri untuk Desktop Jarak Jauh (kecuali mereka ada di Active Directory dengan AD CS tersedia, dalam hal ini mereka mungkin menggunakan CA internal domain).

Meskipun dimungkinkan untuk memasang sertifikat tepercaya secara global, secara praktis tidak ada yang melakukannya, jadi pada sistem mandiri Anda akan melakukannya hampir selalu lihat peringatan "Bukan CA tepercaya", dan Anda akhirnya akan mencentang kotak untuk mengingat sertifikat individu ini.

Sistem Windows 10 juga cenderung meregenerasi sertifikat Remote Desktop mereka bahkan jika itu belum kadaluarsa - kemungkinan besar terjadi selama peningkatan OS penuh, yaitu sertifikat lama mungkin dibuang setiap kali rilis Windows baru sedang diinstal.

Anda tidak dapat melihat sertifikat sebelumnya pada klien Anda, karena hanya mengingat sidik jari SHA-1, bukan sertifikat lengkapnya. Kemungkinan besar Anda juga tidak dapat melihat sertifikat lama di server, karena sering berubah disebabkan oleh sertifikat lama telah dibuang.

Namun, Anda dapat melihat file arus sertifikat pada server dengan menjalankan certlm.msc (pada layar fisik, tentu saja, bukan melalui RDP) dan membandingkan sidik jarinya dengan apa yang ditampilkan oleh klien RDP Anda.
