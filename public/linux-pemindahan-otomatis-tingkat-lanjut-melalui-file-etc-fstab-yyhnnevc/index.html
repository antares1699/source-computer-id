Linux - Pemindahan otomatis tingkat lanjut melalui file / etc / fstab

Saya memiliki server sederhana dengan 6 hard disk drive (semua SATA) di dalamnya. Mereka berada dalam mode JBOD sederhana (hanya sekelompok disk), jadi Linux hanya menampilkan perangkat berikut (asumsikan masing-masing memiliki satu partisi ext2).

/dev/sda
/dev/sda1
/dev/sdb
/dev/sdb1
/dev/sdc
/dev/sdc1
/dev/sdd
/dev/sdd1
/dev/sde
/dev/sde1
/dev/sdf
/dev/sdf1

Sekarang, jika saya menukar dua disk secara tidak sengaja selama pemeliharaan rutin, pemetaan antara nama perangkat (yaitu: /dev/sdx) dan nama disk yang sebenarnya ditukar.

Apakah ada cara agar fstab memerlukan argumen tambahan (yaitu: semacam VOLUME / DISK UUID, atau semacam nomor seri drive) sehingga saya dapat menerapkan pemetaan nama-perangkat-ke-fisik-perangkat yang tidak berubah jika drive tidak terhubung dengan benar di dalam server?

Terima kasih.

Alih-alih /dev/sdX Anda dapat menggunakan UID= seperti dalam contoh ini:

UUID=81658fcf-1777-457a-934e-5d9a399fdf26 /  ext4  errors=remount-ro   0   1

Maka urutan perangkat seharusnya tidak lagi menjadi masalah.
