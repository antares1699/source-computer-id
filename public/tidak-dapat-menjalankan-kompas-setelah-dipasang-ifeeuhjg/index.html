Tidak dapat menjalankan kompas setelah dipasang

Ini adalah kesalahan pemula saya yakin karena saya baru mengenal terminal, dll. Saya mencoba untuk mendapatkan kompas bekerja di mac saya, saya sudah menginstal sass dan membuatnya berfungsi tetapi ketika saya mencoba menginstal kompas meskipun berfungsi, saya tidak dapat menjalankannya ...

computer-name:~ me$ sudo gem install compass
Password:
Successfully installed compass-0.12.2
Parsing documentation for compass-0.12.2
1 gem installed
computer-name:~ me$ compass -v
-bash: compass: command not found
computer-name:~ me$ sass -v
Sass 3.2.12 (Media Mark)

Memasuki:

gem list

Juga mengembalikan:

compass (0.12.2)

Beberapa hal yang saya pikirkan adalah mungkin saya mengalami konflik, saya awalnya mencoba menginstal tanpa sudo dan berhasil, saya mencoba lagi dengan sudo tetapi masih tidak berhasil.

$ PATH saya memiliki:

computer-name:~ me$ echo $PATH
/usr/local/bin:/usr/local/sbin:~/bin:/opt/local/bin:/opt/local/sbin:/usr/bin:/bin:/usr/sbin:/sbin:/opt/X11/bin:/Users/me/.rvm/bin

Yang terakhir .rvm / bin tampaknya ruby ​​tetapi tidak mengandung kompas (meskipun sass juga tidak ada ...)

Catatan: sass ada di folder yang sama dan sass -v menunjukkan versinya jadi saya asumsikan ada di jalur bagaimana (mungkin ruby ​​menemukannya ...)

Berikut adalah hal yang sangat penting tentang menjalankan dua atau lebih versi Ruby di OS X: Jika Anda menggunakan sudo gem, Anda akan menggunakan default OS X gem dari /usr/bin/gem. Ini bukan yang Anda inginkan.

Jika Anda menggunakan gem saja (tanpa sudo), biasanya Anda harus menggunakan gem yang baru diinstal (mis. Yang dari penginstal Ruby OS X, atau yang dari RVM, rbenv, metode apa pun yang Anda gunakan untuk menjalankan versi Ruby lain). Konfirmasikan ini dengan menjalankan which -a gem. Seharusnya mengembalikan sesuatu seperti:

/Users/your-username/.rvm/bin/gem
/usr/bin/gem

Jika gem baru Anda dari RVM bukan default (yaitu, yang pertama dalam daftar itu), instalasi RVM Anda rusak. Jika tidak, panggilan apa pun ke gem, bahkan tanpa sudo, akan tetap menggunakan /usr/bin/gem dan kemudian gagal karena izin yang tidak memadai.

Jadi, pastikan bahwa semua panggilan ke ruby dan gem adalah versi RVM, dan bukan versi sistem. Dan tidak pernah gunakan sudo lagi untuk menginstal sesuatu, karena itu akan mendarat di OS X Ruby default Anda.

Khusus untuk RVM, periksa apakah itu bersumber setelah semua manipulasi PATH Anda dilakukan di file konfigurasi shell (biasanya .profile atau .bash_profile di OS X, tidak yakin mana yang dimodifikasi RVM).
