File test.txt yang aneh selalu muncul di folder dev

Dalam ribuan folder dan subfolder, saya memiliki file bernama "test.txt", yang selalu persis 48 byte, terlihat seperti ini:

2
NUL
8
T
2
DLE
0
B
2
NUL
8
H
2
NUL 
0
P

Atau dalam hex:

320D0A200D0A380D0A540D0A320D0A10
0D0A300D0A420D0A320D0A200D0A380D
0A480D0A320D0A200D0A300D0A500D0A

Terkadang file di folder yang saya kerjakan berubah, biasanya sekali sehari. Hanya angka yang pernah berubah, huruf (T, B, H, P) dan karakter NULL dan DLE selalu tetap sama.

Saya sudah mengalami ini selama beberapa waktu di banyak komputer, tetapi rekan-rekan saya tidak. File secara eksklusif tampaknya muncul di folder yang merupakan bagian dari beberapa proyek perangkat lunak, tetapi tidak terbatas pada bahasa pemrograman tertentu (Python, Java, C #), IDE (Eclipse, Unity) atau sistem kontrol versi (Git, SVN).

Akhirnya, saya menemukan masalah ini tidak mungkin untuk Google, karena setiap contoh pengkodean lain menggunakan 'test.txt' dan menambahkan 'di setiap folder', 'muncul' atau istilah pencarian lainnya sepertinya tidak membantu sedikit pun :) Saya sangat yakin bahwa saya sendiri tidak pernah membuat layanan apa pun yang terus-menerus memantau sistem file saya dan meninggalkan potongan teks misterius di mana-mana.

Jadi saya tidak tahu apa-apa tentang apa yang menyebabkan file-file ini, dan agak khawatir bahwa itu mungkin tanda dari beberapa malware. Apakah ada yang tahu apa yang membuat file-file ini di komputer saya? Apakah Windows, pemindai virus, atau perangkat lunak resmi apa pun yang diketahui melakukan ini? Ada saran untuk mengetahui proses apa yang memodifikasi file?

Strukturnya terlihat persis seperti sistem antarmuka / tertanam waktu nyata yang menggunakan mikrokontroler 9512 (dan lainnya).

Ada beberapa aplikasi yang mungkin menggunakan kode seperti itu, termasuk berbasis java ... jadi jika Anda memiliki perangkat pemrograman besar dengan banyak alat pemrograman yang terpasang, akan sulit untuk menentukan apa yang menghasilkannya.

Untuk mendapatkan petunjuk, hapus file tersebut dan ketika muncul kembali lihat tanggal / waktu pembuatan. Ini akan memberi Anda petunjuk tentang apa yang membuatnya.

Fungsionalitas audit di Windows juga dapat membantu dengan situasi seperti ini, tetapi ini menghasilkan volume besar data yang hampir tidak berguna sehingga, secara praktis, tidak layak untuk digali.

Anda juga bisa mendapatkan Monitor Proses (gratis) dan memantau proses pada saat muncul. Ini akan memudahkan Anda untuk mengidentifikasi apa yang membuatnya.
