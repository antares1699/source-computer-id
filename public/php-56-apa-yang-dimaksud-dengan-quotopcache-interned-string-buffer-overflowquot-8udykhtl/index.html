PHP 5.6: Apa yang dimaksud dengan "opcache: interned string buffer overflow"?

Saya menjalankan PHP 5.6 dan mengaktifkan opcache modul yang dikompilasi. Sekarang ketika saya membaca peringatan di dalam log kesalahan saya, saya mendapatkan pesan ini setiap beberapa milidetik:

Warning Interned String buffer overflow

Tidak ada deskripsi untuk kesalahan ini menggunakan pencarian google ini . Namun, kode sumber mengatakan, di sana mungkin tidak cukup memori ? Apakah saya membacanya dengan benar?

Jadi, menurut pemahaman saya, tidak masalah jika cache habis dan akan digunakan kembali / didaur ulang untuk string baru. Jika tidak, apa yang saya lewatkan?

Saya menetapkan opcache.interned_string_buffer dari 4 menjadi 8 MiB, tetapi belum berhasil.

opcache.interned_strings_buffer - Pengaturan yang cukup rapi dengan dokumentasi seperti 0. PHP menggunakan teknik yang disebut string interning untuk meningkatkan kinerja — jadi, misalnya, jika Anda memiliki string "foobar" 1000 kali dalam kode Anda, secara internal PHP akan menyimpan 1 variabel yang tidak dapat diubah untuk string ini dan hanya menggunakan penunjuk untuk yang lain. 999 kali Anda menggunakannya. Keren. Pengaturan ini membawanya ke tingkat berikutnya— alih-alih memiliki kumpulan string yang tidak dapat diubah ini untuk setiap proses php-fpm TUNGGAL, pengaturan ini membagikannya ke SEMUA proses php-fpm Anda. Ini menghemat memori dan meningkatkan kinerja, terutama dalam aplikasi besar.

https://www.scalingphpbook.com/best-zend-opcache-settings-tuning-config/

Ada bug yang sedang berlangsung yang mempengaruhi beberapa pengguna ketika opcache.interned_string_buffer tidak disetel cukup tinggi. Bug menyebabkan Apache crash sehingga Anda pasti tahu jika Anda mengalami masalah.

Saya sarankan untuk sementara membuat file php seperti info.php dan menambahkan

<?php phpinfo(); ?>

lihat file di browser Anda untuk dengan mudah melihat "Memori yang Digunakan String Interned" dan pastikan Anda memiliki beberapa memori kosong. Saya memiliki situs wordpress yang cukup besar dan saya memiliki batas memori yang ditetapkan pada 32. Saya secara konsisten memiliki sekitar 14mb memori string internal yang digunakan.
