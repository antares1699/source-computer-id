Di mana Less.app menyimpan file konfigurasinya?

Less.app adalah alat yang sangat keren untuk menggunakan LESS / CSS - tetapi saya tidak tahu di mana ia menyimpan file konfigurasinya.

Saya pikir mungkin itu menyimpannya di file ini:

~/Library/Application Support/Less app/lessDataFile.lessData

Namun, tampaknya bukan itu masalahnya. Adakah yang tahu di mana file konfigurasi yang benar? Untuk memperjelas, saya mencari yang menyimpan daftar folder dan jalur yang digunakan oleh aplikasi.

Bryan di sini. Orang di belakang Less.app. Lokasi di atas untuk file data dan preferensi sudah benar. Alasan aplikasi menyimpan informasi dalam format "jangan main-main dengan ini" adalah karena tidak pernah dirancang untuk digunakan di seluruh mesin.

Inilah alasannya: katakanlah Anda memiliki dua komputer. Keduanya memiliki proyek situs web yang sama persis di lokasi yang sama persis. Katakanlah, ~ / Desktop / Project / main.less ada di kedua komputer. Anda mungkin berpikir Anda dapat menambahkan proyek ini ke Less.app di satu komputer dan "berbagi" file data antara dua komputer. Namun di balik layar, OS X melihat file yang "sama persis" ini sangat berbeda. Mereka memiliki pengidentifikasi file yang berbeda, yang merupakan cara utama Less.app "mengetahui" file mana.

Ini adalah masalah yang sangat menantang untuk dikelola, karena Less.app tidak dapat mengetahui apakah ada dua proyek di dua mesin Betulkah proyek yang sama atau jika mereka hanya memiliki struktur dan nama file yang sama.

Dan, sejak aplikasi mulai hidup tiga tahun lalu sebagai alat pribadi saya dan karena saya tidak perlu melakukan sinkronisasi di beberapa mesin, saya tidak pernah merancangnya untuk digunakan seperti itu.

Anda benar-benar menebak dengan benar. Daftar folder dan jalur yang digunakan oleh aplikasi ada di:

~/Library/Application Support/Less app/lessDataFile.lessData

Ini adalah file plist biner (Daftar Properti), dan dari tampilan struktur file, itu tidak dimaksudkan untuk dikacaukan. Sepertinya serialisasi / dump yang tidak intuitif dari status internal aplikasi.

Sepertinya menyimpannya preferensi di file plist biner lainnya ini:

~/Library/Preferences/com.incident57.Less.plist

... tetapi file ini tidak berisi jalur folder apa pun yang digunakan oleh aplikasi. Itu semua dalam file lessDataFile.lessData yang sudah Anda temukan.

Ini praktik standar untuk aplikasi Mac OS X untuk menyimpan preferensi mereka dalam plist seperti itu (lokasi itu, skema penamaan itu).

Anda dapat membaca dan mengedit file plist biner dengan Xcode (dan banyak editor teks berorientasi pemrograman lainnya untuk Mac OS X), atau dengan alat baris perintah PlistBuddy(8), atau defaults(1). Atau Anda dapat mengonversinya menjadi file teks XML dengan plutil(1). Beberapa alat untuk bekerja dengan plists mengharapkan nama file berakhir pada .plist, jadi ingatlah itu jika Anda kesulitan mendapatkan alat untuk membuka file .LessData.
