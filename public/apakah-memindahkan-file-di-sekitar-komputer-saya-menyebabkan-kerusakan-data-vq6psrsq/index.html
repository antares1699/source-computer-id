Apakah memindahkan file di sekitar komputer saya menyebabkan kerusakan data?

Saya punya pertanyaan konyol yang melayang di kepala saya: Jika saya memiliki file di C: / test1 dan memindahkannya ke C: / test2 lalu ke D: / Other dan seterusnya ... lagi dan lagi, mungkinkah file tersebut rusak? Bagaimana dengan membuat salinan file lagi dan lagi (salinan salinan salinan)?

  • jawaban singkatnya: tidak. file menjadi rusak jika tidak ditransfer secara penuh atau sebagian menjadi tidak dapat dibaca / diubah karena kerusakan hard drive / kebiasaan buruk yang melibatkan flash drive.
    — Lorenzo Von Matterhorn
  • Jawaban singkat lainnya: mungkin (tapi tidak mungkin). Setiap kali Anda membaca dari disk, ada kemungkinan perangkat keras akan mengalami gangguan dan memberikan data yang salah; setiap kali Anda menulis ke disk, ada kemungkinan perangkat keras akan melakukan kesalahan dan menyimpan data yang salah. Tetapi ada mekanisme untuk membantu komputer mendeteksi bahwa ini telah terjadi dan mengambil tindakan perbaikan (sebagai upaya terakhir, melaporkan kesalahan).
    — Scott

Mereka seharusnya tidak melakukannya; file adalah pola hidup / mati, dan semua operasi seperti memindahkan dan menyalin dirancang untuk memindahkan seluruh data, tanpa kehilangan apa pun. Ini tidak seperti permainan manusia "telepon" di mana menyalin file berulang kali membuatnya lebih bergumam dan membingungkan setiap kali sampai yang tersisa hanyalah omong kosong.

Tapi , setiap kali Anda melakukan sesuatu, kecil kemungkinannya terjadi kesalahan. Semakin banyak hal yang Anda lakukan, semakin besar kemungkinan Anda mengalami kesalahan. Salah satunya, misalnya, adalah pemadaman listrik di tengah gerakan - apa yang akan Anda dapatkan? File parsial? Tidak ada dokumen? Dua file?

Tidak ada yang sempurna; di sini, pada tahun 2005, sebuah pelajaran melihat tingkat kesalahan drive disk, dan menemukan:

Kami mengamati 3 peristiwa kerugian saat membaca 1.4 PB, Ini adalah 3 file yang hilang. Di lingkungan pusat data kami kehilangan 2 dari 32.000 file 10GB, di pengaturan kantor (Sistem 1) kami kehilangan satu file 10GB dalam 35.000 percobaan, dan tidak ada file 100GB dalam 7.560 percobaan.

Itu hanya kesalahan disk, mereka juga "experienced at least 4 controller firmware or driver errors".

Selain itu, ada kesalahan memori - kemungkinan kerusakan antara memuat informasi ke memori komputer dan kemudian menulisnya lagi. Ada studi Google yang terkenal di mana mereka melacak kesalahan dalam memori di ribuan server mereka, yang dikutip di Wikipedia sini yang menyatakan:

"Studi berskala sangat besar berdasarkan jumlah server Google yang sangat besar telah dipresentasikan pada konferensi SIGMETRICS / Performance'09. [4] Tingkat kesalahan aktual [..] 25.000 hingga 70.000 kesalahan per miliar jam perangkat per megabit [.. ] lebih dari 8% modul memori DIMM dipengaruhi oleh kesalahan per tahun. "

Dan tentu saja ada kemungkinan kesalahan, kesalahan membaca, kerusakan oleh lonjakan listrik atau sinar kosmik, cacat desain, mendorong toleransi rendah dari perangkat keras murah, di media magnetis pada disk, dalam transfer melalui kabel, di motherboard, chipset, prosesor, memori, firmware atau perangkat lunak, dan semuanya kembali lagi.

Kebetulan, berikut adalah artikel yang fantastis berdasarkan gagasan bahwa jika perangkat komputasi merusak data dalam memori saat mencari alamat internet, perangkat tersebut mungkin mencari alamat yang salah. Jadi penulis mendaftarkan beberapa "alamat yang salah" (apa yang akan Anda dapatkan jika Anda merusak sedikit nama situs internet populer) untuk melihat apakah ada perangkat yang terhubung dengannya - dia memiliki 52.000 permintaan dalam hampir 6 bulan.

Studi tentang bagaimana membuat sesuatu / lebih / dapat diandalkan dalam komputasi adalah bidang yang sibuk. Dari CRC dan checksum, ke RAID dengan paritas, sistem file ZFS berbasis pohon checksum, hingga replikasi otomatis, ke memori ECC, dan seterusnya.
