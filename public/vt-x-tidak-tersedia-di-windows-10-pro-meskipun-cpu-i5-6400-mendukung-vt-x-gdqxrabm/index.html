VT-x tidak tersedia di Windows 10 Pro meskipun CPU i5-6400 mendukung VT-X

Saya memiliki Intel Core i5-6400 CPU diinstal pada mainboard ASUS Z-170A. Menurut dokumentasi saya CPU , CPU ini mendukung Intel Virtualization Technology (VT-X) dan memang begitu terdaftar sebagai kompatibel dengan HAXM untuk Android . Saya memiliki Windows 10 Pro (64-bit build 1803) dan VT-x adalah diaktifkan dalam pengaturan BIOS, dan Memory Integrity adalah dengan disabilitas di Windows Defender. Saya tidak dapat menjalankan mesin virtual di VMWARE karena meminta kesalahan:

VMware Workstation dan Device / Credential Guard tidak kompatibel. VMware Workstation dapat dijalankan setelah menonaktifkan Device / Credential Guard.

Namun, saat saya gunakan Utilitas Identifikasi Prosesor Intel (R) , ia melaporkan bahwa CPU saya tidak mendukung VT-X. Ini mencegah pemasangan HAXM untuk Android. Bagaimana cara mengatasi ini?

Sayangnya VMWare prompt dan dokumen resolusinya menyesatkan dan tidak cukup menjelaskan untuk membantu saya.

Untuk mengaktifkan VT-x selain mengaktifkannya di BIOS, penggunaan harus Nonaktifkan Penjaga Kredensial Windows Defender seperti di bawah ini:

  1. Jalankan gpedit.msc dan kemudian pergi ke:

Computer Configuration -> Administrative Templates -> System -> Device Guard -> Turn on Virtualization Based Security

dan pilih opsi Disabled.

  1. Kemudian jalankan regedit dan hapus kunci ini jika ada di register:

HKEY_LOCAL_MACHINE \ System \ CurrentControlSet \ Control \ LSA \ LsaCfgFlags HKEY_LOCAL_MACHINE \ Software \ Policies \ Microsoft \ Windows \ DeviceGuard \ EnableVirtualizationBasedSecurity HKEY_LOCAL_MACHINE \ Software \ Policies \ Microsoft \ Windows \ DeviceGuard \ RequirePlatformSecurityFeatures

  1. Jalankan cmd sebagai administrator dan jalankan perintah di bawah ini untuk mengaktifkan bit NX:

bcdedit.exe / setel {current} nx AlwaysOn

  1. Bagi saya, bagian ajaibnya adalah ini. jalankan regedit dan di folder:

HKEY_LOCAL_MACHINE \ SYSTEM \ CurrentControlSet \ Control \ DeviceGuard \ Scenarios \ HypervisorEnforcedCodeIntegrity

setel Enabled ke 0.
