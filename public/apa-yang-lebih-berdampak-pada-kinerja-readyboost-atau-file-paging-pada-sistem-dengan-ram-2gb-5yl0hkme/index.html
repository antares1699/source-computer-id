Saya telah membaca tentang Readyboost dan kedengarannya seperti caching dengan manfaat memori flash (latensi rendah, dll.)

Manakah dari berikut ini yang lebih baik saya gunakan?

Mengenai file halaman, Mark Russinovich (hampir semua ahli Windows dalam segala hal) menulis sebuah artikel yang dapat ditemukan di sini: http://blogs.technet.com/markrussinovich/archive/2008/11/17/3155406.aspx .

Dia menemukan bahwa mematikan pagefile adalah kesalahan besar. Kutipan kuncinya mungkin:

Mungkin salah satu yang paling sering ditanyakan pertanyaan yang berhubungan dengan memori virtual adalah, seberapa besar saya harus membuat paging mengajukan? Konyol tidak ada habisnya saran di web dan di majalah kios yang meliput Windows, dan bahkan Microsoft menerbitkan rekomendasi yang menyesatkan. Hampir semua saran didasarkan tentang mengalikan ukuran RAM dengan beberapa faktor, dengan nilai umum menjadi 1.2, 1.5 dan 2. Sekarang Anda telah memahami peran yang dimainkan oleh file paging mendefinisikan batas komit sistem dan bagaimana proses berkontribusi pada komit mengisi daya, Anda berada di posisi yang tepat untuk melihatnya betapa tidak bergunanya formula seperti itu sebenarnya.

Karena batas komit menetapkan batas atas terikat pada seberapa pribadi dan memori virtual yang didukung pagefile dapat dialokasikan secara bersamaan dengan menjalankan proses, satu-satunya cara untuk masuk akal ukuran file paging adalah untuk mengetahui total biaya komit maksimum untuk program yang ingin Anda jalankan waktu yang sama. Jika batas komit adalah lebih kecil dari angka itu, Anda program tidak akan dapat mengalokasikan memori virtual yang mereka inginkan dan akan gagal untuk berjalan dengan baik.

Jadi, bagaimana Anda tahu berapa banyak komitmen mengisi beban kerja Anda membutuhkan? Kamu mungkin telah memperhatikan di tangkapan layar bahwa Windows melacak nomor itu dan Process Explorer menunjukkannya: Peak Commit Biaya. Untuk mengukur paging Anda secara optimal file Anda harus memulai semua aplikasi yang Anda jalankan pada saat bersamaan, memuat kumpulan data tipikal, lalu catat puncak biaya komit (atau lihat nilai ini setelah jangka waktu tertentu di mana Anda tahu beban maksimum itu tercapai). Tetapkan file halaman minimum menjadi nilai itu dikurangi jumlah RAM di sistem Anda (jika nilainya adalah negatif, pilih ukuran minimum ke izinkan Anda jenis kecelakaan sampah dikonfigurasi untuk). Jika Anda ingin memiliki beberapa ruang bernapas untuk berpotensi tuntutan komit besar, tetapkan maksimum untuk menggandakan angka itu.

Jika Anda memiliki> = 2GB maka Anda tidak membutuhkan Readyboost, kecuali Anda menggunakan aplikasi dengan permintaan yang berlebihan. Jika Anda memiliki 4GB maka jangan pernah memikirkannya, karena Vista akan menggunakan hampir 2GB untuk cache sistem (minus cache video). Dalam penggunaan normal Anda tidak akan melihat peningkatan kinerja apa pun dalam kecepatan I / O, dan pemrosesan tambahan Readyboost bahkan mungkin menurunkan kinerja CPU Anda.

ReadyBoost memang menghasilkan peningkatan kinerja yang sangat besar terlepas dari berapa banyak RAM yang telah Anda instal di sistem Anda. ReadyBoost bekerja dengan menyimpan data yang sering digunakan ke memori flash perangkat yang dialokasikan untuk menggunakan ReadyBoost. Cache ReadyBoost bekerja bersama-sama dengan file halaman sistem.

File halaman di Windows sebenarnya adalah area hard disk yang dialokasikan seolah-olah itu adalah RAM. Oleh karena itu, Windows menyimpan data dan menjalankan program dari area hard disk yang dialokasikan ini dengan cara yang sama seperti RAM. Ketika ReadyBoost berfungsi dan diaktifkan (menggunakan perangkat yang cukup cepat seperti kartu SD SDHC ultra Sandisk 15MB / dtk, atau flash drive berkinerja tinggi), ini bekerja dengan file halaman dengan memindahkan data yang jika tidak akan dimasukkan ke harddisk. disk ke perangkat memori flash yang dialokasikan oleh ReadyBoost.

File halaman yang lebih besar sebenarnya menurunkan kinerja dengan memaksa drive mekanis untuk membaca dan menulis data lebih sering. Kami memiliki RAM di komputer kami karena ketika kami meluncurkan program atau file, windows mengambil data dari hard drive (program, file, dll) dan memindahkannya ke RAM, tempat data diproses secara tradisional. Gunakan kartu Sandisk SD 15MB / s atau flashdrive, atur file halaman Anda ke 2GB atau lebih jika Anda mau.

Apa pun itu, Windows akan menyimpan data ini ke cache ReadyBoost alih-alih hard disk Anda. Anda akan melihat peningkatan kinerja yang sangat besar. Selain itu, gunakan hanya hard drive 7200 RPM dan RAM tercepat yang didukung sistem Anda. Selain itu, sebaiknya gunakan minimal RAM 4 GB.

Saya harap penjelasan saya membantu menjelaskan semuanya di sini. Banyak informasi menyesatkan di web akhir-akhir ini.
