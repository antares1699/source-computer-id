wget memunculkan 'url yang hilang' saat mencoba memasang kunci tanda tangan repositori

Saya sedang menginstal INetSim di Ubuntu dan saat menjalankan perintah ini untuk menginstal kunci tanda tangan untuk program:

sudo wget -O - http://www.inetsim.org/inetsim.org-archive-signing-key.asc | apt-key add -

Saya mendapatkan pesan:

wget: missing URL
Usage: wget [OPTION]... [URL]...

Try `wget --help' for more options.

Mengapa saya mendapatkan kesalahan ini?

Itu wget perintah dalam teks pertanyaan Anda tidak cocok dengan perintah di tangkapan layar Anda. Berikut adalah perintah di tangkapan layar Anda:

sudo wget -O http://www.inetsim.org/inetsim.org-archive-signing-key.asc | apt-key add -

Berikut adalah perintah dalam teks contoh Anda; periksa juga perintah yang digunakan di petunjuk penginstalan INetSim resmi :

sudo wget -O - http://www.inetsim.org/inetsim.org-archive-signing-key.asc | apt-key add -

Perhatikan perbedaan halusnya? Salah satu yang gagal kehilangan - antara sudo wget -O dan URL yang mengikutinya. Cobalah dengan - dan seharusnya berfungsi dengan baik.

Supaya Anda memahami apa yang terjadi — atau tidak — terjadi, -O (tanda --output-document mengharuskan - memberi tahu wget untuk mengalirkan konten URL tersebut ke keluaran standar. Jika Anda hanya menjalankan perintah seperti ini:

wget http://www.inetsim.org/inetsim.org-archive-signing-key.asc

Semua wget akan lakukan adalah menyimpan file itu secara lokal di sistem Anda ke file bernama inetsim.org-archive-signing-key.asc. Tetapi pengaturan -O - dan kemudian memiliki pipa itu (|) ke | apt-key add - perintah pada dasarnya mengatakan, streaming konten file itu ke put standar, lalu pipa itu output ke apt-key add sehingga konten kunci dapat ditambahkan ke daftar repositori apt Anda.

Selain itu, tangkapan layar Anda menunjukkan kesalahan tambahan:

EROR: Perintah ini hanya dapat digunakan oleh root.

Itu terkait langsung dengan kurangnya tanda hubung dalam perintah Anda; itu menyebabkan segalanya gagal juga.

Jika pipa perintah rusak, seluruh perintah gagal dan tidak akan pernah sampai ke titik di mana Anda diminta untuk memasukkan kata sandi sudo. Namun, sisa perintah lainnya masih diurai.

Jadi meskipun perintah lengkap gagal, ia masih mencoba menjalankan apt-key. Dan karena bagian perintah satu baris yang tersisa pada dasarnya terlepas dari perintah sudo yang dipanggil sebelumnya — yang tidak pernah dipanggil sebagai sudo karena kesalahan— itu mencoba untuk menjalankan apt-key sebagai pengguna biasa kemudian berkata:

EROR: Perintah ini hanya dapat digunakan oleh root.

Karena, ya… Mencoba menjalankan apt-key sebagai pengguna biasa / tidak berhak memang merupakan kesalahan.
