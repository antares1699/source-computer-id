Bagaimana cara menguji apakah denyhosts dapat mengirimi saya email?

Itu denyhosts layanan seharusnya menganalisis log saya dan memberi tahu saya jika seseorang mencoba melakukan brute force sshd saya. Itu harus menolak akses mereka dan mengirimi saya email. Bagaimana saya tahu apakah itu berfungsi? Secara khusus, dapatkah saya membuatnya mengirimi saya email percobaan sehingga saya tahu bahwa itu sebenarnya memiliki alamat dan konfigurasi yang benar.

Saya sangat curiga tidak bisa kirimi saya email, karena tidak ada sendmail di kotak ini.

Tampaknya denyhosts memerlukan server SMTP untuk dikonfigurasi pada denyhosts.cfg - tidak mendukung antarmuka standar /usr/sbin/sendmail.

Coba jalankan denyhosts --foreground, periksa juga syslog dan / atau file log yang dikonfigurasi.

Anda juga dapat menggunakan skrip ini untuk mengirim email percobaan; jalankan menggunakan sudo -u denyhosts:

#!/usr/bin/env python
import sys
sys.path.insert(0, '/usr/share/denyhosts')
from DenyHosts import constants, prefs, util
prefs = prefs.Prefs(constants.CONFIG_FILE)
util.setup_logging(prefs, True, False, False)
util.send_email(prefs, "This is a test.")

FYI: di sini "sendmail" tidak merujuk ke Sendmail MTA secara khusus, melainkan program yang dapat dijalankan pada /usr/sbin (terkadang /usr/lib), yang diinstal oleh apa saja MTA - tidak hanya Sendmail, tetapi juga Postfix, Exim, dan lainnya.

Agar DenyHosts mengirimi Anda email percobaan.

  1. Masuk ke sistem Linux Anda yang berisi server sshd dan beralihlah ke pengguna root.
  2. Tambahkan baris berikut ke file /etc/hosts.deny - sshd: 127.0.0.1
  3. Simpan file
  4. Mencoba menyambung ke server sshd lokal: - $ ssh localhost (jika Anda menghubungkan pada port selain 22, gunakan sakelar -p)
  5. Anda akan menerima pesan galat berikut: - ssh_exchange_identification: Sambungan ditutup oleh host jarak jauh
  6. Anda seharusnya juga telah dikirimi email dari DenyHosts [Nobody @ yourserver]
  7. Ingatlah untuk menghapus baris yang Anda tambahkan ke file /etc/hosts.deny - simpan file.
