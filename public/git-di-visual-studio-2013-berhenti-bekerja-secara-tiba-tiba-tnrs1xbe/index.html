Git di Visual Studio 2013 berhenti bekerja secara tiba-tiba

Kemarin, saya pulang dengan Git bekerja dengan sangat baik di dalam Visual Studio 2013. Saya bisa 'Bandingkan dengan yang tidak dimodifikasi' untuk file sumber, saya akan melihat status kontrol versi dari file sumber saya, dan semua barang Git di dalam Visual Studio. Saya mematikan komputer saya ketika saya pulang kerja kemarin.

Namun, ketika saya datang bekerja pagi ini, segera setelah saya meluncurkan VS, saya mendapat pesan aneh dari Git Version control yang mengatakan,

'Path 'C:\' doesn't point at a valid Git repository or workdir.'

Saya melanjutkan untuk membuka proyek saya yang berada di bawah kendali versi Git, dan anehnya, Git tidak mengenali proyek saya - Sepertinya tidak ada file sumber yang ditambahkan ke kontrol versi, semua opsi seperti membandingkan dengan versi file yang tidak dimodifikasi ada hilang, seolah-olah proyek tersebut tidak pernah berada di bawah kendali versi Git.

Jadi, saya melakukan pemulihan sistem, dan itu membuat semuanya berfungsi kembali. Sekarang, ketika saya meluncurkan Visual Studio, saya mendapatkan pesan berikut dari Git Version control

Opening repository: C:\Workspace\Dev\

Jalur ini adalah jalur di mana kode sumber saya berada.

Bisakah Anda memikirkan apa yang bisa mengacaukan integrasi Git-Visual Studio secara tiba-tiba?

P.S. Saya menggunakan VS 2013, jadi saya tidak menginstal ekstensi Git eksternal, saya menggunakan ekstensi yang disertakan dengan VS 2013. Masalah yang sama persis telah terjadi dua kali sekarang. Pada kedua kesempatan tersebut, saya harus melakukan pemulihan sistem agar semuanya berfungsi kembali.

Tidak tahu apakah itu akan membantu Anda tetapi saya baru saja mengalami masalah ini sendiri. Yang saya lakukan adalah, di Visual Studio, buka Tools -> Options -> Source Control -> Plug-in Selection dan ubah dari Microsoft Git Provider ke Visual Studio Team Foundation Server, klik OK, lalu kembali ke sana dan ubah kembali ke Penyedia Git Microsoft.

Pesan itu hilang setelah itu dan sekarang saya melihat "kunci" kecil kembali di penjelajah sumber (untuk file yang tidak berubah).
