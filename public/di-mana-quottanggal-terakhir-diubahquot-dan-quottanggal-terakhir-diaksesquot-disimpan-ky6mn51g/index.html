Di mana "tanggal terakhir diubah" dan "tanggal terakhir diakses" disimpan?

Judulnya menjelaskan semuanya. Di mana tanggal yang disebutkan (dan file lainnya meta data) disimpan?

Apakah mereka bagian dari file atau terpisah? Jika saya memiliki file, ubahlah sehingga isinya tetap sama (hapus surat, lalu tulis kembali) ... mengubah tanggal yang diubah. Apakah kedua file yang dihasilkan masih sama persis dengan isinya (biner sama)?

Kedua cap waktu terpisah dari konten file. Mereka disimpan bersama dengan metadata lain di inode (kebanyakan sistem file Unix) atau yang setara dengan Windows - entri file di "tabel file utama" . Secara umum, ini sangat spesifik untuk sistem file yang digunakan - FAT dan ext4 dan btrfs dan NTFS melakukannya dengan sangat berbeda.

Waktu modifikasi pada Windows diperbarui saat menutup file, jika setidaknya satu byte telah diubah. Apakah data baru sama atau berbeda tidak masalah, selama WriteFile () atau fungsi terkait telah dipanggil; konten baru bahkan bisa sama persis.

Waktu akses diperbarui saat file ditutup setelah dibuka untuk membaca atau menulis - meskipun belum diubah dengan cara apa pun. (Ini tidak termasuk pembacaan metadata, karena metadata berada di luar file. Pada Windows, program cadangan juga dapat meminta atime untuk tetap tidak dimodifikasi.) Biasanya ada batasan tambahan tentang kapan waktu ini diperbarui: misalnya, hanya jika atime lama sebelumnya waktu modifikasi (Linux relatime ), atau hanya jika atime lama lebih dari satu jam sebelumnya (Windows lama), atau tidak diperbarui sama sekali (Windows lebih baru).
