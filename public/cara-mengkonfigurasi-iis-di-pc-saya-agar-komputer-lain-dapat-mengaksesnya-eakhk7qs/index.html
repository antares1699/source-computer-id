Cara mengkonfigurasi IIS di PC saya agar komputer lain dapat mengaksesnya

Saya memiliki mesin yang menjalankan Windows 7 Home Premium x64.

IIS7 aktif dan berjalan; Saya dapat menavigasi ke localhost: 80 dan layar selamat datang IIS7 yang keren muncul.

Ketika saya mencoba ini di salah satu mesin saya yang lain, waktu permintaan habis. Dan, ya, saya berhasil melakukan ping ke mesin host IIS7 dari komputer lain.

Saya sudah mencoba melihat-lihat dokumentasi IIS7, dan jawabannya mungkin ada tetapi saya tidak dapat menemukannya.

Atau mungkin ada hubungannya dengan keamanan Windows 7?

Apakah Anda mencoba menyambung ke server web dengan alamat IP? Untuk mengosongkan DNS, Anda mungkin ingin mencoba menghubungkan dengan alamat IP.

Dan saya setuju bahwa Windows Firewall adalah penyebab yang paling mungkin di sini. Jika Anda dapat mengesampingkan firewall, Anda mungkin ingin memverifikasi pengikatan situs baik dari Manajer IIS dan melalui sesuatu seperti:

NETSTAT -noab > out.txt && out.txt

telnet SERVER_IP 80

80 pada akhirnya penting karena memberitahu telnet untuk terhubung ke port 80. Jika Anda tidak dapat terhubung maka saya akan memeriksa firewall windows dan konfigurasi IIS. Dalam urutan itu.

Tidak diperlukan perubahan pada firewall. Ikuti langkah-langkah di bawah ini:

  1. Buka Manajer IIS
  2. Di bawah ‘Situs’, pilih situs web yang ingin Anda akses dari jarak jauh (Situs web default).
  3. Klik kanan padanya dan pilih 'Edit Bindings'.
  4. Pilih entri dengan port no 80 dan klik edit.
  5. Ubah nama host 'localhost' menjadi alamat IP-nya sendiri (alamat IP server).
  6. Simpan pengaturannya.
  7. Akses situs web menggunakan tautan berikut: http: // server_IP_address / Website_Name
