227 Memasuki Mode Pasif (0,0,0,0,…) saat menghubungkan ke server FTP vsftpd

Seperti yang dikatakan dalam judul, saya tidak berhasil menghubungkan host Windows 10 saya ke server FTP saya di sisi tamu VirtualBox (Ubuntu 18.04). Di sisi tamu, server vsftpd berfungsi dengan baik, saya dapat terhubung tanpa masalah.

Jadi saya mencoba meneruskan port terkait ke host (NAT) dan mencoba terhubung ke server melalui FileZilla. Itu masuk, tetapi selalu gagal untuk mengambil daftar direktori. Saya mencoba menonaktifkan firewall, memberikan semua izin akses ke folder tetapi tidak mengubah apa pun.

Saya membaca bahwa itu bisa menjadi masalah mode pasif di sana:
Masalah dengan akses file FTP ke tamu VirtualBox yang menjalankan Windows 2008 Server R2 x64

Jadi saya menambahkan baris berikut ke saya vsftpd.conf :

pasv_enable=YES
pasv_min_port=8020
pasv_max_port=8020
port_enable=YES
pasv_address=127.0.0.1

Saya meneruskan port 8020 tetapi sekarang saya mendapatkan kesalahan berikut di FileZilla:

Sambungan data tidak dapat dibuat: WSAEADDRNOTAVAIL - Tidak dapat menetapkan alamat yang diminta

Tanggapan FileZilla lengkap:

Status: Connection established, waiting for welcome message...

Command:    PWD

Saya agak bingung dengan ini, apakah ada yang punya saran? Terima kasih sebelumnya

EDIT1:
Tanpa harapan apa pun, saya juga mencoba mengakses dari Command Prompt Windows. Itu terhubung tetapi ketika saya ls folder saat ini saya mendapatkan kesalahan berikut:

500 illegal PORT command.
425 use PORT or PASV first.
But I guess that this is because ftp.exe doesn't support passive mode.

Untuk berjaga-jaga jika itu dapat membantu, Anda dapat menemukannya di sini seluruh file konfigurasi vsftpd saya saat ini . Juga, saya dapat dengan mulus mengakses ke server SSH tamu dari host dengan Putty.

Respon: 227 Memasuki Mode Pasif (0,0,0,0,31,84).

0,0,0,0 kemungkinan besar karena bug yang diketahui di vsftpd.

Jika demikian, untuk memperbaikinya, pastikan server tidak mendengarkan alamat IPv6:

listen_ipv6=NO
listen=YES

Untuk detailnya, lihat pertanyaan serupa di Server Fault:
Vsftpd balasan pasif dengan alamat 0,0,0,0 bahkan dengan pasv_address yang benar
