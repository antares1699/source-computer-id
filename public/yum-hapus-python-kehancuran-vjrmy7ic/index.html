yum hapus python = kehancuran

Saya yakin pertanyaan ini akan membuat banyak orang tertawa, tapi yang pasti bukan saya.

Saya mencoba menginstal beberapa paket untuk python hari ini dan juga mencoba mengubah kompiler pydev di AptanaStudio, dan ada sesuatu yang tidak berhasil. Jadi saya memutuskan untuk sepenuhnya menghapus python dari komputer (saya menjalankan Fedora 10) dengan menjalankan

sudo yum erase python

Saya pasti seharusnya tidak melakukan ini. Bahkan pesan bahwa 788 paket akan dihapus tidak menghentikan saya. Akibatnya, saya sepertinya telah menghapus seluruh antarmuka Fedora. Setelah banyak kesakitan, saya berhasil menyalin semua hal penting ke hard drive eksternal (tampaknya hard drive komputer tetap utuh), tetapi Fedora tidak mau memulai. Jadi, inilah yang ingin saya ketahui:

  1. Apa sebenarnya yang terjadi dan mengapa?

  2. Apakah mungkin untuk memulihkan antarmuka entah bagaimana?

Seperti yang dicatat Kimvas, Anda bisa mendapatkan daftar semua paket yang dihapus dari /var/log/yum.log. Menginstal ulang dengan yum sepertinya tidak akan berhasil karena yum adalah skrip python dan python telah dihapus dari sistem Anda.

Apa yang dapat Anda coba adalah mengunduh rpm yang dihapus dari mirror Fedora kemudian boot dari beberapa disk yang dapat di-boot Fedora, melakukan chroot ke partisi sistem dan menginstal rpm tersebut menggunakan rpm.

Merupakan ide yang baik untuk memisahkan partisi /home dan sistem /, sehingga ketika / dihapus atau diinstal ulang, semua data Anda di /home tetap utuh.

Saya pikir taruhan terbaik Anda adalah menginstal ulang seluruh sistem dengan mengingat hal-hal berikut:

  1. cadangkan direktori /etc/ Anda - penginstalan ulang akan menimpa file konfigurasi yang tersisa.
  2. Kumpulkan daftar paket yang tidak sengaja Anda hapus seperti yang diinstruksikan di bawah ini sebelum melakukan instalasi ulang (karena file log akan ditimpa)
  3. Melakukan TIDAK format / inisialisasi / mkfs sistem file apa pun saat melakukan instal ulang
  4. Setelah melakukan a minimal instal ulang sistem, instal ulang paket yang dihapus yang Anda kumpulkan dari yum log.
  5. terakhir, ganti konfigurasi default dari paket yang Anda instal dari / etc / backup. Anda mungkin perlu melakukan beberapa langkah dengan tangan, seperti membuat ulang akun dan grup pengguna.

Untuk melihat apa yang Anda unistall, lihat file /var/log/yum.log mis. sudo cat /var/log/yum.log | grep Erased untuk mendaftar semua paket yang telah dicopot (sejak pembuatan file log itu!)

Untuk mencoba menginstal ulang semuanya, Anda dapat mencoba sesuatu seperti sudo cat /var/log/yum.log | grep Erased | awk '{print $5}'|xargs sudo yum install
