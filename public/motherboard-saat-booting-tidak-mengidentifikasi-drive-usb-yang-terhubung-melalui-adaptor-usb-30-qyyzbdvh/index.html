Motherboard saat booting, tidak mengidentifikasi drive USB yang terhubung melalui adaptor usb-3.0

Saya memiliki PC berbasis motherboard Intel DP55KG "Kenigsberg". Itu tidak memiliki bus usb 3.0 (xHCI). Saya memasang kartu adaptor usb 3.0 (Inateck KT4006) (berfungsi dengan baik, baik di Windows dan Ubuntu 18); tetapi, saya ingin mem-boot dari sistem Ubuntu yang berada di HDD USB; dan, sebelum boot, BIOS tidak mengenali bus (dan perangkat) usb 3.0 tambahan yang dibuat oleh adaptor.

Jadi saya tidak dapat memilih HDD USB untuk boot, jika terhubung melalui adaptor usb 3.0. (HDD USB yang sama berfungsi dengan baik setelah boot, terhubung ke adaptor usb 3.0).

Jika motherboard tidak dilengkapi dengan pengontrol xHCI, kemungkinan besar firmware-nya juga tidak memiliki driver xHCI bawaan. Tanpa itu, firmware tidak tahu cara menghitung perangkat USB melalui pengontrol ini.

(Sebenarnya motherboard Anda mendahului rilis pertama xHCI dalam beberapa bulan, jadi firmware defaultnya tidak mungkin memiliki driver untuk itu.)

Sementara itu, OS dapat menggunakan USB 3.0 dengan baik karena tidak bergantung pada fungsionalitas firmware - baik Windows dan Linux memiliki driver xHCI (dan EHCI, OHCI) sendiri yang langsung berbicara dengan pengontrol melalui bus PCI.


Untuk mengatasi ini, Anda dapat meletakkan GRUB dan partisi Ubuntu /boot pada penyimpanan yang berbeda (misalnya SATA HDD atau USB 2.0 flash drive). Ini akan memungkinkan motherboard untuk memulai GRUB, dan segera setelah GRUB memuat kernel + initrd Ubuntu, pengontrol xHCI akan dapat digunakan melalui driver Linux dan sistem akan boot secara normal.

(Sebenarnya itulah inti dari memiliki initrd - initrd berisi semua driver yang diperlukan kernel untuk mengakses sistem file root.)
