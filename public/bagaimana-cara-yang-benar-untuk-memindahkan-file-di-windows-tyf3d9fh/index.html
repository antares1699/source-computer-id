Bagaimana cara yang benar untuk memindahkan file di Windows?

Kami memiliki masalah dalam pekerjaan saya saat saya memotong dan menempelkan beberapa file. Segera ketika saya melakukannya, banyak sekali file yang hilang. Saya telah bekerja di IT selama 10+ tahun. Saya tahu cara memotong dan menempelkan file. Nah, ketika sampai ke manajer saya mengapa file-file itu hilang, mereka mengatakan saya menggunakan potong dan tempel menyebabkan semua masalah dan bertanya mengapa di dunia ini seseorang yang berpengetahuan luas seperti saya pernah memotong dan menempelkan file, dan tidakkah saya tahu bahwa itu adalah cara yang benar-benar salah untuk memindahkan file? Mereka mengklaim,

"Cara yang benar untuk memindahkan file adalah dengan menyeret file. Saat memotong dan menempelkan, itu memindahkan file 1+ GB (di server) ke clipboard (di PC saya), yang tentunya akan menimbulkan masalah. Tidak pernah menyeret file menyentuh papan klip. "

Sejujurnya, saya tidak percaya itu sebentar. Saya percaya ketika saya memotong dan menempelkan teks, itu masuk ke clipboard. Saya telah melihatnya di versi lama windows. Tetapi ketika mengklik kanan pada 100+ file yang sama dengan 1+ GB, saya tidak percaya bahwa semua data itu disalin segera dari bagian apa pun yang saya miliki di server di seluruh nirkabel saya di laptop saya ke clipboard lokal saya untuk pergi kembali ke server untuk berbagi lain. Tampaknya mereka akan membangun beberapa logika di OS server atau OS lokal saya (lebih mungkin OS lokal saya) yang akan mengatakan saat menyalin file, jangan melakukan tindakan pemindahan sampai saya mengklik tempel dan jika file tetap lokal ke tempat mereka sebelumnya, pindahkan saja.

Jadi siapa yang benar?

Kamu benar

Ketika pengguna memotong dan menempel file menggunakan Windows Explorer, apa yang disimpan di clipboard bukanlah konten file yang sebenarnya, melainkan lokasi file. Saat Anda memulai penyisipan, OS cukup pintar untuk memulai pemindahan, atau lebih tepatnya, Windows akan menyalin file ke atas dan kemudian menghapus file asli (melalui jaringan, drive lokal bergerak dengan memperbarui penunjuk jika berada di drive yang sama ).

Saya tidak terlalu yakin langkah lain apa yang terjadi sementara, tetapi dari sudut pandang admin jaringan dan sistem, kami TIDAK PERNAH pernah memotong dan menempelkan dari satu drive jaringan ke drive lain. Kami akan menyalin, dan kemudian setelah memverifikasi file dipindahkan dengan aman, kami kemudian menghapus yang asli. Praktik buruk untuk memotong dan menempel.

Semoga Anda menemukan file yang hilang itu, jalankan program pemulihan file di server - mungkin masih ada di sana.

Memotong file tidak menghapusnya dari lokasi aslinya di Windows Explorer. Itu akan terlalu berbahaya. Cukup salin dua file secara berurutan, dan yang pertama akan dihapus ...

Dari Mengelola File (artikel serupa juga tersedia untuk versi Windows yang lebih baru):

Catatan : Saat Anda menggunakan perintah CUT dan PASTE, Windows 2000 tidak segera menghapus item dari lokasi aslinya. Perintah CUT hanya menempatkan salinan item di clipboard. Setelah Anda menggunakan perintah PASTE untuk menempelkan file ke lokasi baru, file tersebut dihapus dari lokasi lama.

Itu sebabnya tidak ada Memotong untuk file di pengelola file OS X: ini keliru. Ini pada dasarnya Tandai file untuk dipindahkan nanti .

Meskipun mungkin ada masalah selama transfer yang mengakibatkan penghapusan data di kedua sisi, sesuatu yang dengan mudah mungkin dilakukan melalui jaringan yang tidak dapat diandalkan, memindahkan file dengan drag & drop akan sama risikonya.

Jika Anda ingin memastikan, selalu salinan pertama, kemudian verifikasi bahwa itu berfungsi sebagaimana mestinya (tidak perlu kesalahan transfer nyata, sesuatu seperti nama jalur yang terlalu panjang akan merusak bagian transfer seperti yang akan terjadi), lalu hapus dari lokasi sumber.
