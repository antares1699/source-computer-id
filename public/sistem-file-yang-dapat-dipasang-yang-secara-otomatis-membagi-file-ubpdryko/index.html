Sistem file yang dapat dipasang yang secara otomatis membagi file

Apakah ada sistem file [virtual-] yang dapat secara otomatis membagi file dalam penyimpanan tetapi menyajikannya kepada pengguna sebagai file tunggal?

Sebagai contoh:

The user sees: /user/files/10TBfile
But it is stored as: /backingstorage/user/files/10TBfile.{1..100}

Pada dasarnya cara kerja arsip yang sama tetapi dalam waktu nyata. Saya merasa itu harus dimungkinkan karena pada dasarnya menyimpan setiap blok disk virtual sebagai file terpisah tetapi saya tidak tahu ada solusi yang ada untuk itu.

Jika Anda penasaran, tujuan akhirnya mungkin mirip dengan pertanyaan ini: Bagilah file lokal di server berbeda dengan ruang terbatas dengan rsync kecuali bahwa saya memiliki satu file besar yang perlu dipecah dan file-file tersebut perlu diperbarui secara real-time sehingga cron / rsync harian dan tar terpisah tidak mungkin dilakukan. Saya sudah memasang drive jarak jauh jadi saya hanya perlu cara untuk membagi file dan menyajikannya sebagai satu file kepada pengguna.

Terima kasih!

Apa yang kamu inginkan adalah chunkfs :

ChunkFS adalah sistem file berbasis FUSE yang memungkinkan Anda memasang file arbitrer atau memblokir perangkat sebagai pohon direktori file yang masing-masing mewakili potongan ukuran yang ditentukan pengguna dari file yang dipasang.

Itu ditulis untuk tujuan yang sama seperti milik Anda:

ChunkFS awalnya ditulis untuk membuat cadangan tambahan yang hemat ruang dari gambar sistem file terenkripsi menggunakan rsync. Dengan menggunakan opsi --link-dest dari rsync, Anda dapat membuat cadangan tambahan dari gambar yang dipasang di ChunkFS di mana potongan apa pun yang tidak berubah sejak pencadangan terakhir akan menjadi tautan keras ke potongan yang sesuai dari cadangan sebelumnya.

Biasanya Anda akan melakukan ini di level blok. Beberapa solusi untuk ini termasuk:

  • Serangan 0
  • DRBD (lebih dari itu untuk mirroring, jaringan OK)
  • ZFS (tingkat abstraksi yang lebih tinggi)

Dari perspektif sistem file:

  • Simpan separuh file secara manual pada satu sistem file dan separuh lainnya di sistem file lainnya, lalu sajikan melalui sesuatu seperti sistem file FUSE kustom (Pikirkan kode kustom yang rumit).
  • Sebagian besar solusi sistem file difokuskan pada sinkronisasi, bukan pada partisi data.
  • Hadoop (Data Sharding, bukan FS tradisional)
