Macvim tertinggal sementara Vim di terminal sangat halus

Saya menjalankan OS X Lion 10.7.3 dan Macvim berjalan jauh lebih lambat daripada vim pada terminal untuk saya. Semua perintah gerakan di Macvim jauh lebih lambat. Pindah ke atas dan ke bawah dalam mode visual sama lambannya. Saya tidak melihat ada jeda ini saat menggunakan vim dari terminal. Adakah yang tahu apa alasannya? Saya menjalankan NERDtree pada setiap tab yang terbuka, dan saya tahu ini berkontribusi pada beberapa overhead memori dan berpotensi melambat; tetapi bahkan ketika saya tidak menjalankan NERDtree Macvim berjalan jauh lebih lambat dari vim dari terminal. Bantuan apa pun dalam memecahkan masalah ini akan sangat dihargai.

Vim yang Anda jalankan di Terminal.app mungkin adalah Vim default yang disediakan oleh Apple. Itu dibangun dengan fitur yang jauh lebih sedikit daripada MacVim sehingga mulai dan bekerja lebih cepat.

Apakah MacVim lambat di setiap kesempatan? Saat mengedit jenis file apa pun? Hanya untuk beberapa tipe file?

Vim terkenal rentan terhadap perlambatan saat penyorotan sintaks baris yang sangat panjang, apakah itu yang terjadi di sini? Apakah vim masih lambat setelah :syntax off?

Beberapa hal lain yang diketahui memperlambat Vim adalah :set cursorline dan :set cursorcolumn.

Masalah saya utamanya adalah MacVim membuka, membaca, dan menulis dengan sangat lambat (terkadang hingga 15 detik). mvim --startuptime mengungkapkan bahwa sumber file dari .vimrc dan runtime / direktori memakan sebagian besar waktu, dan masalah diselesaikan dengan menambahkan baris:

set rtp+=/usr/local/Cellar/macvim/7.4-73/MacVim.app/Contents/Resources/vim

Seperti yang Anda lihat, saya menggunakan penginstalan Homebrew dari MacVim, dan karenanya Anda harus memastikan Anda memasukkan jalur dan nomor versi yang benar.
