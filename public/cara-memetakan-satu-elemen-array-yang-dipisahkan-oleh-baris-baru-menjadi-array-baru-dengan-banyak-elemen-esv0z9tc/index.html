Cara memetakan satu elemen array yang dipisahkan oleh baris baru menjadi array baru dengan banyak elemen

Saya memilih 4 file di Nemo di jalur /home/myUsername/.local/share/nemo/scripts/Folder with spaces/. Nemo menyimpan jalur file dalam variabel lingkungan NEMO_SCRIPT_SELECTED_FILE_PATHS.

(Seperti yang Anda lihat dalam jawaban dari glenn jackman di bawah, NEMO_SCRIPT_SELECTED_FILE_PATHS berisi daftar jalur file yang dipisahkan baris baru.)

Mengapa "Ya!" tidak dicetak setelah setiap baris jalur file? Bagaimana saya bisa mencapai itu? (Pertanyaan ini mungkin menjengkelkan, karena awalnya saya mengira daftar jalur file ini adalah array, padahal sebenarnya bukan.)

NASKAH

#!/bin/bash

for i in "${NEMO_SCRIPT_SELECTED_FILE_PATHS[@]}"
do
    echo "$i"
    echo "Yeah!"
done

KELUARAN

/home/myUsername/.local/share/nemo/scripts/Folder with spaces/script1
/home/myUsername/.local/share/nemo/scripts/Folder with spaces/script2
/home/myUsername/.local/share/nemo/scripts/Folder with spaces/script3
/home/myUsername/.local/share/nemo/scripts/Folder with spaces/script4

Yeah!

Asumsi:

  • variabel lingkungan $NEMO_SCRIPT_SELECTED_FILE_PATHS entah bagaimana secara ajaib ditetapkan untuk Anda oleh nemo, dan
  • itu berisi daftar nama file yang dipisahkan baris baru,

Anda dapat menguraikannya menjadi bash array seperti ini:

$ NEMO_SCRIPT_SELECTED_FILE_PATHS="file one
file two
file three"

$ mapfile -t files <<<"$NEMO_SCRIPT_SELECTED_FILE_PATHS"

$ echo ${#files[@]}
3

$ printf ">>%s\n" "${files[@]}"
>>file one
>>file two
>>file three

mapfile adalah perintah bash builtin yang membaca input standar, membagi baris baru dan menyimpan baris dalam array bernama.

Ini rusak jika salah satu nama file Anda berisi baris baru (yang merupakan karakter nama file resmi).
