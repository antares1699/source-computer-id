Mungkinkah untuk mem-boot kernel Linux tanpa initrd?

Apakah mungkin untuk mem-boot kernel Linux (katakanlah 2.6.35) tanpa initrd hari ini? Saya sepertinya ingat bahwa ini mungkin sebelumnya (awal 2.6.x dan 2.4). Maksud saya, secara teori, jika kernel memiliki semua dukungan perangkat keras bawaan (tanpa modul), itu tidak memerlukan ruang pengguna awal, bukan?

Saya tidak menggunakan initrd. Semua yang perlu terjadi adalah Anda memiliki semua driver yang diperlukan untuk mem-boot sistem Anda yang dikompilasi ke dalam kernel dan bukan sebagai modul.

Jika Anda menggunakan gentoo , kemudian membangun kernel dan driver Anda adalah bagian dari proses instalasi, itu menjadi kebiasaan jika Anda akan menambahkan perangkat keras yang bergantung pada boot ke dalam sistem Anda yang Anda tambahkan driver ke kernel terlebih dahulu. Ini juga menjadi kebiasaan untuk menonaktifkan apa pun yang tidak Anda perlukan - karena tidak ada gunanya mengumpulkan banyak driver bahkan sebagai modul jika Anda tidak akan pernah menggunakannya.

Proses ini secara konseptual lebih sulit pada distro seperti ubuntu, di mana kernel dikelola sebagai sebuah paket dan memiliki semua yang termodulasi (itu harus, untuk bekerja dengan sukses pada banyak sistem yang berbeda di luar kotak). Seringkali tidak sepadan dengan repotnya.
