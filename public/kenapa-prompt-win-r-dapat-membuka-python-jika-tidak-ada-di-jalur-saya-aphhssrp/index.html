Kenapa prompt Win + R dapat membuka Python jika tidak ada di jalur saya?

Ketika saya menggunakan run prompt di Windows XP Professional (Win + R), dan mengetik python.exe atau python, itu bekerja dan menyapa saya dengan prompt python.

Namun, saat saya memulai jendela cmd, lalu mengetik python.exe atau python, jendela tidak menemukannya.

Ini yang saya harapkan, karena direktori Python (bagi saya, I:\Python31\) tidak ada di PATH saya.

Mengapa, kemudian, jika saya mengetik python.exe di prompt Win + R, itu berhasil?

Sunting: ini adalah sebagian keluaran SET, saya menghapus sebagian besar entri yang tidak relevan, saya tidak yakin mengapa ini berguna, selain dari variabel PATH yang sudah saya katakan tidak termasuk direktori Python. Jika Anda membutuhkan variabel tertentu selain ini, silakan tanyakan.

CLIENTNAME=Console
CommonProgramFiles=I:\Program Files\Common Files
ComSpec=I:\WINDOWS\system32\cmd.exe
FP_NO_HOST_CHECK=NO
OS=Windows_NT
Path=I:\WINDOWS\system32;I:\WINDOWS;I:\WINDOWS\system32\WBEM;I:\WINDOWS\system32\WindowsPowerShell\v1.0;I:\Qt\2010.05\mingw\bin;I:\Program Files\CMake 2.8\bin
PATHEXT=.COM;.EXE;.BAT;.CMD;.VBS;.VBE;.JS;.JSE;.WSF;.WSH;.PSC1
ProgramFiles=I:\Program Files
PROMPT=$P$G
SESSIONNAME=Console
SystemDrive=I:

windir=I:\WINDOWS

Perintah Jalankan mencari jalur registri untuk program yang akan diluncurkan serta% jalur%. Lihatlah HKLM\Software\Microsoft\Windows\CurrentVersion\App Paths - Anda mungkin akan menemukan node bernama "Python.exe" dengan entri default yang mengarah ke Python yang dapat dieksekusi. Pemasang Sebaiknya tambahkan entri registri untuk semua executable utama di sini, tetapi perintah Run adalah fitur yang kurang dihargai, jadi banyak pengembang lupa untuk menambahkan langkah sederhana ini ke penginstal mereka.
