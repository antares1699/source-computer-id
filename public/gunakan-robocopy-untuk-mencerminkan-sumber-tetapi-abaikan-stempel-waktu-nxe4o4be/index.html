Gunakan robocopy untuk mencerminkan sumber tetapi abaikan stempel waktu

Saya ingin membuat cadangan folder untuk dibagikan di linux nas dengan robocopy.

Saya menggunakan commandline robocopy /mir <source> <dest>

Setelah salinan berhasil di explorer, cap waktu sama tetapi entah bagaimana robocopy menyalin file yang sama saat berikutnya berjalan dan mengklaim file sumber lebih baru. Tampaknya menjadi masalah dengan versi seseorang di nas.

Sekarang saya dapat menggunakan sakelar /xn untuk mencegah penyalinan file tetapi itu juga akan mencegah file disalin saat saya menjalankan pekerjaan yang sama di lain waktu.

Jadi pertanyaan saya adalah: Bagaimana cara menggunakan robocopy untuk mengabaikan stempel waktu tetapi hanya menyalin file jika ukuran file berubah (saya tahu ini bukan solusi 100%) tetapi karena saya ingin membuat cadangan dokumen dan gambar kantor, ini akan menjadi OK buat saya.

Setelah salinan berhasil di explorer, stempel waktunya sama tetapi entah bagaimana Robocopy menyalin file yang sama saat berjalan dan mengklaimnya lagi file sumber lebih baru. Tampaknya ada masalah dengan versi seseorang nas.

Tambahkan sakelar /FFT dengan Robocopy:

ROBOCOPY /MIR /FFT <source> <dest>

Sakelar Robocopy

/FFT : Assume FAT File Times (2-second date/time granularity).

Deskripsi Lebih Baik

/ FFT menggunakan pengaturan waktu file lemak, bukan NTFS. Artinya, perincian> sedikit kurang tepat. Untuk operasi berbagi lintas jaringan ini tampaknya jauh lebih dapat diandalkan - jangan mengandalkan file tersebut pengaturan waktu agar benar-benar tepat ke detik.
