Mengapa nslookup menampilkan "permintaan DNS habis waktu"

Saya menggunakan komputer umum di beberapa jaringan publik dan ingin melihat bagaimana nslookup bekerja di sana. Saya mencoba set type=ns untuk beberapa situs web dan mendapatkan pesan ini:

Permintaan DNS kehabisan waktu. batas waktu adalah 2 detik.

Kemudian saya mencoba set ti=30 dan beberapa nomor lain yang lebih rendah, tetapi mendapat pesan yang sama.

Tidak ada batas waktu seperti itu ketika saya mencoba hal yang sama di rumah.

Untuk alasan apa pesan ini muncul? Bagaimana cara memperbaikinya?

Permintaan DNS kehabisan waktu berarti NSLookup mengirimkan kueri ke server DNS, tetapi tidak mendapat tanggapan.

Mungkin server DNS yang Anda tanyakan mengalami masalah dan tidak dapat membalas. Kesalahan jaringan juga bisa menjadi penyebabnya.

Namun, mengingat Anda melakukan ini dari "komputer publik", penjelasan yang paling mungkin adalah bahwa pencarian DNS Anda diblokir oleh firewall jaringan. Administrator jaringan biasanya mengharuskan pencarian DNS yang berasal dari node di jaringan internal dilakukan menggunakan server DNS yang disetujui. Ini bisa berupa server DNS di bawah kendali administrator, atau server DNS publik tertentu yang dipilih oleh admin.

Memaksa klien untuk menggunakan server DNS di bawah kendali admin (atau menahan pembatasan atau tidak ada kemampuan untuk menavigasi Internet) biasanya dilakukan karena berbagai alasan:

  • Untuk memastikan klien LAN menggunakan server DNS internal yang benar, yang mungkin diperlukan untuk memastikan fungsi yang tepat dari domain DNS internal (misalnya, Active Directory).

  • Untuk mencegah informasi nama domain internal bocor ke Internet melalui pencarian terhadap server DNS publik (yang tidak terenkripsi).

  • Untuk mengizinkan logging mudah dari permintaan DNS yang dibuat oleh semua klien LAN.

  • Untuk memungkinkan admin mengubah hasil kueri tertentu, misalnya ketika admin ingin mengalihkan nama host yang dapat diselesaikan secara publik ke alamat IP pribadi internal (misalnya sebagai alternatif untuk menerapkan jepit rambut NAT ).

  • Untuk memeriksa dan berpotensi memblokir pencarian tertentu, seperti yang dibuat oleh malware ke server C&C-nya.

Bagaimana cara memperbaikinya?

Tindakan terbaik adalah menggunakan server DNS yang benar. Jika komputer yang Anda gunakan biasanya memiliki akses Internet, maka server DNS yang dikonfigurasi untuk digunakan secara default adalah yang benar.

Alternatifnya, jika Anda dapat membuat koneksi VPN dari mesin, Anda dapat menggunakannya untuk menyalurkan permintaan DNS melewati firewall jaringan. Jelas Anda tidak ingin melakukan ini jika itu bertentangan dengan kebijakan penggunaan yang dapat diterima untuk mesin yang Anda gunakan.
