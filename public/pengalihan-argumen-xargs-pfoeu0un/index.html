pengalihan argumen xargs

Ini tampaknya menjadi masalah yang sepele, tetapi solusinya membuatku sangat berhati-hati.

Saya ingin menghapus banyak file teks; file log. Alasan? Untuk menghemat ruang disk.

Dengan satu file teks, ini semudah echo '' > path/to/file.txt

Namun, filenya banyak. Saya terpaksa menggunakan find dan xargs. Tetapi saya tidak tahu bagaimana cara menyiasatinya "pengalihan masukan" .

Saya telah mencoba find . -name <regex> | xargs -I target echo '' > target dan echo '' > { find . -name <regex> | xargs -I target target }; tidak ada yang berhasil

Saya tidak mahir dalam pembuatan skrip shell, apakah bantuan apa pun dihargai.

Terima kasih.

Apa yang telah Anda lakukan di kedua contoh Anda adalah meletakkan > di mana arus shell dapat melihatnya, jadi pengalihan dilakukan hanya sekali, sebelum perintah find dan xargs dijalankan. Itu masalah pertamamu.

Masalah kedua Anda adalah jika Anda mengutip > sehingga diteruskan ke xargs, itu masih tidak berfungsi karena xargs tidak meneruskan perintah Anda melalui shell kecuali Anda memintanya.

Masalah ketiga Anda adalah jika Anda memberi tahu xargs untuk menggunakan shell untuk menjalankan perintah, shell akan melakukan hal yang salah jika ada nama file yang memiliki karakter lucu di dalamnya. (Juga xargs melakukan hal yang salah dengan karakter lucu itu sendiri, tetapi itu dapat diperbaiki dengan -0.)

Masalah lain, yang mungkin belum Anda pedulikan, termasuk:

  • echo '' tidak membuat file kosong, tetapi file yang berisi baris baru.
  • opsi -name menggunakan glob, bukan regex.
  • Anda mungkin ingin menambahkan -type f kalau-kalau ada direktori yang cocok dengan glob.

Berikut adalah versi perintah yang diperbaiki sebagian:

find . -name '*thisisaglob*' -type f -print0 |
xargs -0 -I target sh -c ': > target'

Itu memperbaiki sebagian besar masalah yang saya sebutkan. Masih ada masalah shell salah menafsirkan nama file yang mengandung metakarakter shell. Untuk memperbaikinya, Anda harus memberikan nama file ke shell sebagai parameter, bukan sebagai bagian dari perintah -c. Itu akan terlihat seperti ini:

find . -name '*thisisaglob*' -type f -print0 |
xargs -0 -I target sh -c ': > "$1"' fnord target

"Fnord" adalah placeholder. Ini menjadi $0 yang tidak kita butuhkan.

Sekarang setelah mencapai tujuan menggunakan xargs dan pengalihan bersama-sama dengan aman, saya akan menunjukkan cara mencapai tujuan Anda dengan tidak menggunakan keduanya.

find . -name '*thisisaglob*' -type f -exec truncate -s 0 '{}' +

Ini memerlukan perintah truncate, yang merupakan bagian dari GNU coreutils dan bukan utilitas standar unix, jadi kurang portabel, tetapi jauh lebih mudah dibaca, bukan?

Meskipun jawaban Alan Curry lengkap, tepat, dan sangat luas, saya punya satu pertanyaan. Mengapa Anda ingin file tetap ada, bahkan kosong?

Rekomendasi alami saya adalah:

find . -name <glob> -print0 | xargs -0 rm

Jika Anda membutuhkan file tersebut karena suatu alasan, Anda dapat melakukan serangkaian perintah:

find . -name <glob> -print0 | tee /tmp/filelist | xargs -0 rm
cat /tmp/filelist | xargs -0 touch

Jika Anda hanya ingin mengompres file (misalnya jika sebagian besar adalah informasi yang berlebihan / berulang, tetapi Anda ingin menghemat ruang dan simpan file), Anda dapat mencoba:

find . -name <glob> -print0 | xargs -0 tar -xzf /tmp/logfiles.tgz

Jika Anda memiliki GNU Parallel http://www.gnu.org/software/parallel/ terinstal Anda dapat melakukan ini:

find . -name '*thisisaglob*' -type f | parallel '>'

Anda dapat menginstal GNU Parallel hanya dengan:

wget http://git.savannah.gnu.org/cgit/parallel.git/plain/src/parallel
chmod 755 parallel
cp parallel sem

Tonton video intro untuk GNU Parallel untuk mempelajari lebih lanjut: https://www.youtube.com/playlist?list=PL284C9FF2488BC6D1
