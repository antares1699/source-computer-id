Apa yang dimaksud dengan 'proxy'?

Ketika saya membaca teks jaringan, saya telah melihat istilah alamat 'proxy'.

Adakah yang bisa memberikan rincian tentang apa itu proxy dan mengapa kami menggunakannya?

Pada dasarnya, ketika Anda menyiapkan server proxy, semua koneksi internet Anda (atau hanya HTTP, tergantung) melalui proxy terlebih dahulu sebelum pergi ke salah satu ujung baris. Ini berguna ketika koneksi langsung diblokir atau Anda ingin melakukan sesuatu seperti melacak riwayat Anda untuk penelitian atau sesuatu.

Proksi dapat melakukan apa pun yang diinginkan dengan data yang melewatinya, yang keduanya baik ( Prioksi ) atau buruk ( Man in the Middle Attacks / episode Hak5 ini ).

Proksi adalah server yang Anda sambungkan, yang kemudian terhubung ke internet. Jika Anda menggunakan proxy, komputer Anda tidak berkomunikasi secara langsung dengan server situs web yang Anda lihat, tetapi berkomunikasi melalui proxy.

Beberapa orang menggunakannya untuk privasi sehingga server web tidak dapat melihat siapa Anda, tetapi hanya melihat proxy.

Wikipedia

Dalam jaringan komputer, server proxy adalah server (sistem komputer atau aplikasi program) yang bertindak sebagai perantara permintaan dari klien yang mencari sumber daya server lain. Klien terhubung ke server proxy, meminta beberapa layanan, seperti file, koneksi, halaman web, atau sumber daya lain, yang tersedia dari server yang berbeda. Server proxy mengevaluasi permintaan sesuai dengan aturan pemfilterannya. Misalnya, ini mungkin memfilter lalu lintas berdasarkan alamat IP atau protokol. Jika permintaan divalidasi oleh filter, proxy menyediakan sumber daya dengan menghubungkan ke server yang relevan dan meminta layanan atas nama klien. Server proxy dapat secara opsional mengubah permintaan klien atau respons server, dan terkadang dapat melayani permintaan tanpa menghubungi server yang ditentukan. Dalam hal ini, respons 'cache' dari server jarak jauh, dan mengembalikan permintaan berikutnya untuk konten yang sama secara langsung.

Server proxy memiliki berbagai macam tujuan potensial, termasuk:

  • Untuk menjaga agar mesin di belakangnya tetap anonim (terutama untuk keamanan).
  • Untuk mempercepat akses ke sumber daya (menggunakan caching). Proksi web biasanya digunakan untuk menyimpan halaman web dari server web.
  • Untuk menerapkan kebijakan akses ke layanan jaringan atau konten, mis. untuk memblokir situs yang tidak diinginkan.
  • Untuk mencatat / mengaudit penggunaan, yaitu untuk menyediakan pelaporan penggunaan Internet karyawan perusahaan.
  • Untuk melewati keamanan / kontrol orang tua.
  • Untuk memindai malware pada konten yang dikirimkan sebelum pengiriman.
  • Untuk memindai konten keluar, misalnya, untuk perlindungan kebocoran data.
  • Untuk menghindari pembatasan regional.

Server proxy yang meneruskan permintaan dan balasan tanpa modifikasi biasanya disebut gateway atau terkadang proxy tunneling.

Sebuah server proxy dapat ditempatkan di komputer lokal pengguna atau di berbagai titik antara pengguna dan server tujuan di Internet.

Reverse proxy adalah (biasanya) proxy yang menghadap ke Internet yang digunakan sebagai front-end untuk mengontrol dan melindungi akses ke server di jaringan pribadi, biasanya juga melakukan tugas-tugas seperti load-balancing, autentikasi, dekripsi, atau caching.

Sederhananya, a server proxy adalah perangkat yang bertindak sebagai tipuan antara klien dan lokasi jaringan lainnya.

Jadi, alih-alih terhubung langsung ke situs web atau layanan, proxy akan terhubung atas nama klien.

Ini berarti klien akan meminta server proxy untuk sumber daya jaringan tertentu (file, situs web, dll), dan server proxy akan terhubung ke, mengunduh, dan kemudian mengembalikan sumber daya kembali ke klien - klien tidak pernah terhubung ke yang lain server yang memegang sumber daya.
