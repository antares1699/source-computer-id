sed baik-baik saja dengan ` atau ' di regex-nya. Di mana Anda mengalami masalah adalah mendapatkan karakter tersebut melalui interpreter shell karena kutipan tunggal dan backtick memiliki arti khusus untuk shell, dan harus dikutip jika mereka ingin membuatnya menjadi sed tanpa bash membuat kesalahan sintaks atau merusak input.

Ada tiga cara untuk mengutip karakter khusus: petik tunggal ', petik ganda ", dan garis miring terbalik \. Semuanya berperilaku sedikit berbeda. ' mengutip semuanya kecuali dirinya sendiri, jadi Anda dapat menggunakan ini untuk mengutip ` tetapi tidak '. " mengutip semuanya kecuali \, `, dan $ (tidak relevan di sini), jadi Anda dapat menggunakan sed untuk mengutip ' tetapi tidak `. \ mengutip semuanya sehingga Anda bisa menggunakannya untuk keduanya. Dimungkinkan untuk menyarangkan metode ini selama Anda menjaga aturan kutipan tetap lurus (mereka ditafsirkan dari kiri ke kanan).

Untuk membuatnya lebih konkret, misalkan Anda menginginkan s/`'/replacement/ dalam regex sed Anda. Ada banyak cara untuk melakukannya. Dua contoh berbeda diberikan di bawah ini.

# concatenation of single-quoted s/`, backslash-quoted ', and 
# single-quoted remainder of the command
sed -e 's/`'\''/replacement/'

# one double-quoted string quotes the single-quote, but an 
# additional backslash is needed to quote the backtick 
sed -e "s/\`'/replacement/"

Ingatlah aturan mengutip dan gunakan versi yang paling mudah dibaca yang menyelesaikan pekerjaan. Lihat bagian QUOTING di manual bash untuk penjelasan dan referensi lengkap.

Sebagai catatan terakhir, saya akan menghindari penggunaan khusus \d, \x, ` dll. Lolos karena

  • Mereka adalah ekstensi GNU hingga sed yang mungkin kurang portabel dibandingkan pola yang dikutip dengan benar.
  • Sedikit lebih sulit untuk dibaca. Berapa banyak orang yang mengetahui nilai ASCII untuk ' dan `?

Salah satu cara untuk melakukannya adalah dengan menggunakan [] untuk menyertakan karakter yang ingin Anda cari

ini berfungsi, meskipun saya masih memiliki masalah dengan bash catching the `tilde char.

sed -n '/ [`] / p'

['] kutipan tidak berfungsi bagaimanapun dan terjebak oleh bash pada perluasan baris perintah.

$ sed -n '/[']/p' ./final/kh_elec_main.db.fin.dump  
> ^C

Ini bagaimana bisa berhasil. Sekarang saya bisa mengganti '= ascii 96 dengan' = ascii 39

$ printf "% d% d \ n" "''" "'` "
39 96
sed -ne 's / \ d96 / \ d39 / gp'
