Menghosting sertifikat HTTPS .p7b di nginX

Kami memiliki persyaratan untuk mengubah layanan web dari HTTP ke HTTPS, jadi kami membuat .csr dan memberikannya kepada klien kami dan mendapatkan kembali file .p7b dan .cer sebagai imbalan untuk menggunakannya di nginX yang dihosting di server linux.

Saya baru mengenal nginX dan sesuai pemahaman saya, .p7b dan .cer keduanya adalah file sertifikat tetapi saya bingung mana yang harus saya gunakan di sertifikat_sl parameter atau apakah saya perlu menggabungkan kedua file dan meneruskan file yang dihasilkan ke sertifikat_sl untuk melayani tujuan tersebut?

server {

listen   443;
ssl    on;
ssl_certificate    /etc/ssl/your_domain_name.pem; (or bundle.crt)
ssl_certificate_key    /etc/ssl/your_domain_name.key;

server_name your.domain.com;
access_log /var/log/nginx/nginx.vhost.access.log;
error_log /var/log/nginx/nginx.vhost.error.log;
location / {
root   /home/www/public_html/your.domain.com/public/;
index  index.html;
}

}

Saya mengikuti tautan berbeda di sepanjang tautan ini tetapi tidak dapat menemukan jawaban yang tepat.

Adakah yang bisa memberi tahu saya secara rinci bagaimana saya harus menggunakannya?

Keduanya berisi data yang sama (kebanyakan), dalam format yang berbeda. Nginx akan menginginkan format .crt "biasa", tetapi tidak semua CA menyertakan data yang diperlukan di dalamnya, jadi mungkin saja Anda perlu mengonversi dari .p7b. (Lihat juga https://superuser.com/a/123327/1686 .)

File ssl_certificate memiliki dua persyaratan:

  • Untuk nginx (dan banyak layanan lainnya), itu harus dalam format tekstual alias "berenkode PEM", dengan header BEGIN CERTIFICATE.

    Terkadang .cer file dalam format biner. Periksa dengan editor teks; Anda dapat menggunakan openssl x509 -inform DER < thing.cer > thing.crt untuk mengonversi ke format tekstual.

  • File sertifikat juga harus berisi rantai sertifikat perantara - sekali lagi, buka dengan editor teks (atau gunakan certtool -i).

    Beberapa CA menyertakannya sebagai file terpisah, dalam hal ini mereka harus digabungkan bersama (sertifikat Anda terlebih dahulu, diikuti oleh perantara).

Karena Anda juga mendapatkan sertifikat yang sama dalam file .p7b, mungkin paling mudah untuk mengonversinya bahwa ke file sertifikat PEM:

openssl pkcs7 [-inform DER] -print_certs < thing.p7b > fullchain.crt

File .crt yang dihasilkan harus memiliki semua yang diperlukan.
