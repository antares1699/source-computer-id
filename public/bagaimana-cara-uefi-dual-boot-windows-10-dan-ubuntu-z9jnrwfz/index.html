Bagaimana cara UEFI dual boot Windows 10 dan Ubuntu?

TL; DR : Di Windows 10 / Ubuntu UEFI skenario dual-boot, mem-boot Windows 10 dari Grub menyebabkan Windows Boot Manager menimpa / menghapus Grub. Bagaimana saya bisa menghentikan ini terjadi?

PC saya dengan senang hati telah menjalankan UEFI single-boot Ubuntu 18.04 untuk waktu yang lama. Saya baru-baru ini memutuskan untuk menginstal Windows 10 dan mem-boot ganda mereka. Saya telah melakukan ini berkali-kali di hari-hari BIOS / MBR lama jadi tidak mengantisipasi masalah besar.

Proses Instalasi

Proses instalasi saya adalah:

  • Instal Windows 10. Seperti yang diharapkan, ini menimpa Grub dengan Windows Boot Manager.
  • Gunakan live-usb dan Perbaikan Boot untuk menginstal ulang Grub.
  • Boot ke instalasi Ubuntu 18.04 saya yang sudah ada dan jalankan update-grub yang secara otomatis mengambil instalasi Windows baru.
  • Gunakan Grub untuk mem-boot Windows 10

Masalah

Di sinilah ada yang salah. Setelah menghabiskan beberapa waktu di Windows saya perlu boot kembali ke Ubuntu. Saya me-restart mesin dan, yang mengejutkan saya, menemukan bahwa itu boot langsung kembali ke Windows 10 tanpa tanda-tanda Grub di mana pun.

Saya memulihkan Grub seperti sebelumnya, tetapi menemukan bahwa boot ke Windows menyebabkan Grub "menghilang" setiap saat.

Pengaturan

  • HP Z420 : BIOS terbaru
  • Pengaturan Boot:
    • Mode boot UEFI diaktifkan
    • Mode boot lama dinonaktifkan
    • Secureboot dinonaktifkan
    • Fastboot diaktifkan
  • SSD tunggal (sda) dengan Tabel GPT
    • sda1: Partisi Sistem EFI
    • sda2: Ubuntu 18.04 (memasang sda1 di bawah /boot/efi)
    • sda3: Windows 10

Hal-Hal yang Telah Saya Coba

  • Banyak jawaban yang muncul di sekitar ini tampaknya menyarankan dual-boot Legacy / MBR. Saya ingin menghindari kerumitan dan potensi kehilangan data saat mengganti GPT ke tabel partisi DOS dan juga ... ini tahun 2020 rasanya seperti ini seharusnya tidak menjadi masalah!
  • Saya mencoba menonaktifkan dan menurunkan prioritas Windows Boot Manager menggunakan efibootmgr seperti yang disarankan sini dan sini tetapi Windows Boot Manager baru saja mengaktifkan kembali dirinya sendiri saat saya mem-boot Windows!

Saya akan sangat menghargai bantuan apapun. Saya merasa ini seharusnya tidak sulit!

Grub menggunakan efibootmgr untuk membuatnya menjadi yang pertama dalam urutan boot dengan pembaruan atau Perbaikan-Boot. Permintaan maaf telah menjawab pertanyaan saya sendiri tetapi jika itu membantu orang lain di masa depan.

Seperti yang disarankan dalam komentar, saya mencoba yang berikut:

  • Menonaktifkan Hibernate di Windows
  • Menonaktifkan Fastboot di BIOS

tapi tak satu pun dari ini berhasil.

Saya akan mencoba mengganti nama boot manager asli ketika saya menemukan ini menjawab. Menjalankan perintah:

bcdedit /set "{bootmgr}" path \EFI\ubuntu\grubx64.efi

telah mencegah Windows Boot Manager mengganti Grub. Saya bahkan berhasil mengaktifkan kembali Hibernate dan Fastboot tanpa kehilangan Grub.

NB: Penting untuk mendapatkan jalur EFI yang benar. Setelah menjalankan boot-repair dari LiveUSB untuk mendapatkan grub kembali, ini akan memberi tahu Anda jalur lengkap untuk digunakan karena tidak selalu identik dengan yang tercantum di atas.
