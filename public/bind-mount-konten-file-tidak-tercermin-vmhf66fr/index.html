bind mount: konten file tidak tercermin

Saya mengikat memasang satu file seperti ini:

mount --bind a b

Setelah itu, saya dapat melihat konten a jika saya membaca b. Namun, jika saya sekarang membuat perubahan ke a, perubahan itu tidak tercermin dengan b, yang tidak terduga.

Apakah saya salah paham tentang apa yang dilakukan bind mount?

6
6

Ketika Anda mengikat-mount ke sebuah file, ia akan terhubung ke inode sebenarnya dari sebuah file (seperti tautan keras). Jadi jika Anda menggunakan editor yang menghasilkan file baru, Anda tidak akan melihat perubahannya.

Saat Anda memodifikasi file di tempat (seperti echo text >> alias), perubahannya akan terlihat.

Sekadar pengingat, Anda dapat melihat nomor inode jika file dengan ls -li.

Beberapa editor mengizinkan untuk mematikan swapping di file sementara baru. Ini sedikit lebih berisiko tetapi tetap mempertahankan identitas file. (Saya pikir masuk vim Anda memerlukan opsi backupcopy=yes.)

Dan jika Anda bertanya-tanya apa alternatif yang baik, tautan simbolis menunjuk ke file alias dengan nama atau untuk mengikat-mount seluruh direktori.
