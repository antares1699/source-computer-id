Aktivasi diperlukan setelah mengembalikan gambar

Saya memiliki sekitar 40 desktop Lenovo baru dan yang saya butuhkan adalah menghasilkan gambar (model) dan mengkloningnya ke 39 lainnya. Tugas sederhana. Saya menggunakan acronis trueimage home 2009.

Saya melakukan gambar bersih saya tetapi setiap kali saya memuat gambar di PC baru, saya harus mengaktifkan kembali Windows 7 dan Office 2010.

Windows 7 dan Office 2010 telah diaktifkan dengan kunci MAK saya di PC "model".

Adakah yang tahu bagaimana menghindari memasukkan kunci di setiap klon?

Jawabannya bisa jadi sesimpel itu , tapi saya tidak tahu cara kerjanya:

Secara berkala, Windows 7 dan Windows Server 2008 R2 memeriksa konfigurasi perangkat keras komputer tempat sistem operasi diinstal. Jika sistem operasi mendeteksi bahwa perangkat keras telah berubah secara substansial, diperlukan pengaktifan kembali.

Anda selalu dapat mengaktifkan klien MAK menggunakan metode lain selain memasukkan kunci di setiap klon. Lihat Dokumen Technet Microsoft ini untuk informasi lebih lanjut, terutama VAMT, yang dapat digunakan untuk mengaktifkan klien dari jarak jauh. Kamu bisa Unduh di sini .

Lenovo biasanya menjual salinan OEM Windows 7 dan salinan "eceran" Office 2010. Untuk mencitrakan di seluruh sistem baru dan mengaktifkan aktivasi melibatkan pelestarian pra-aktivasi OEM untuk Windows 7 dan tidak mungkin dilakukan secara langsung dengan lisensi "eceran" dari kantor.

Untuk OS, Anda memerlukan tiga bahan: BIOS Lenovo, kunci produk OEM Lenovo generik, dan file lenovo.xrm-ms. Google cara mengaktifkan OEM Windows secara permanen dengan bahan-bahannya.

Untuk Kantor: VAMT 2.0 dapat melakukan aktivasi proxy untuk "retail" Office dan dapat menerapkan kembali CIL offline ke Office reaktif tanpa menggunakan jumlah aktivasi untuk retail.

Alternatifnya, lisensi volume tunggal Office 2010 harus dapat digunakan dalam gambar dan tidak akan diaktifkan ketika dipindahkan ke perangkat keras baru. Ini mungkin yang Anda inginkan.

Cukup rumit untuk membuat hari yang menyedihkan bagi Amerika.

Saya telah melakukan ini berkali-kali di masa lalu, dan bahkan tidak menyadari sampai saat ini bahwa prosesnya bergantung pada kunci lisensi yang benar untuk Office dan Windows, yaitu kunci volume, bukan ritel, atau MAK / KMS.

Aktivasi Windows menggunakan pengenal unik dari perangkat keras yang mendasarinya. Inilah sebabnya mengapa jika Anda mengganti papan sistem, hard drive, dll. Anda mungkin diminta untuk mengaktifkan kembali Windows. Ini melarang pengaktifan Windows sekali dan kemudian mengkloningnya ke beberapa komputer lain ketika Anda hanya memiliki satu kunci yang valid. Itu adalah desain. Mengelaknya berarti menghindari aktivasi Windows.

Saya akan menggunakan slmgr.vbs untuk Windows 7 dan ospp.vbs untuk Office 2010 untuk membuat file batch yang akan mengaktifkan perangkat lunak. Alternatifnya adalah alat Manajemen Aktivasi Volume (VAMT).

Alternatif lain adalah dengan menggunakan Windows Deployment Services. Anda dapat menggunakan ImageX untuk menentukan MAK untuk Windows 7, meskipun Anda harus menulis penginstal untuk Office 2010 yang kemudian akan diaktifkan dengan ospp.vbs.

Saya tidak memiliki pengalaman dengan MAK, tetapi dari membaca dokumen di Mengonversi Klien KMS menjadi Aktivasi MAK , menurut saya Anda mungkin telah menginstal kunci MAK di komputer referensi hanya dengan memasukkannya melalui Panel Kontrol -> Sistem -> Ubah kunci produk Anda.

Menurut dokumen tersebut, klien kemudian akan mengaktifkan dirinya sendiri melalui Internet satu kali . Jika Anda kemudian telah mengambil gambar dan memindahkannya ke komputer lain, maka sebenarnya Anda telah mentransplantasikan sistem yang diaktifkan ke perangkat keras yang berbeda, oleh karena itu membutuhkan aktivasi kedua.

Kemungkinan yang paling sederhana adalah dengan memasukkan kunci MAK pada komputer referensi yang tidak diaktifkan saat diputuskan dari Internet. Dalam hal ini, saat gambar dipasang, gambar akan aktif dengan sendirinya setelah tersambung ke Internet.

Solusi lain mungkin mengaktifkan komputer referensi menggunakan metode kedua:

slmgr.vbs /ipk <MultipleActivationKey>

Klien MAK kemudian tidak mencoba mengaktifkan dirinya sendiri secara otomatis, sehingga Anda dapat mengambil gambar yang tidak diaktifkan. Mudah-mudahan, setelah instalasi terkadang image akan aktif dengan sendirinya, tetapi dokumentasinya tidak menentukan kapan dan jika, jadi masih ada sedikit keraguan apakah aktivasi harus dilakukan secara manual.
