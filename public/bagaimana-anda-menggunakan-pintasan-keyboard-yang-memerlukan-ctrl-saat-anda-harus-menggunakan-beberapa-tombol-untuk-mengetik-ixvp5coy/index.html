Default di sebagian besar terminal, Anda dapat menekan ctrl \ untuk mengirim sinyal SIGQUIT ke proses.

Saya tidak tahu bagaimana Anda bisa mengetik Ctrl + \ , tetapi saya dapat memberi Anda cara alternatif untuk menghasilkan SIGQUIT dari keyboard. Temukan karakter yang dapat Anda ketik sebagai kombinasi tombol sederhana. Ini mungkin perlu dipetakan ke satu karakter ASCII (yaitu, satu byte); misalnya., Ctrl + SEBUAH . Lalu ketik

stty quit ^A

Bisa juga s t t y   q u saya t   Ctrl + SEBUAH Memasukkan (jika Anda mendapatkan pesan kesalahan, coba ketik Ctrl + V. sebelum Ctrl + SEBUAH ) atau secara harfiah s t t y q u saya t ^ SEBUAH Memasukkan (jika Anda mendapatkan pesan kesalahan, coba masukkan ^ SEBUAH dalam tanda kutip). Kemudian ketikkan perintah seperti sleep 9, tekan kombinasi tombol Anda (mis., Ctrl + SEBUAH ), dan lihat apakah program berhenti seperti yang Anda harapkan Ctrl + \ .

Jika Anda menemukan kombinasi yang berfungsi, letakkan (perintah stty quit) ke .login, .profile, .bashrc Anda, atau file perintah inisialisasi apa pun yang Anda gunakan.

Saya memiliki masalah yang hampir sama dengan bahasa Prancis " azerty " papan ketik.

Untuk mendapatkan karakter backslash \, saya harus menekan alt bergeser / .

Tapi menekan ctrl alt bergeser / tidak mengirim sinyal apa pun, itu hanya memasukkan titik . di Terminal.

Saya menemukan bahwa menekan ctrl ` (tombol backtick atau backquote) mengirimkan sinyal SIGQUIT:

$ sleep 9
^\Quit: 3

Mendesak ctrl bergeser ` juga bekerja.

Perhatikan bahwa, pada keyboard Prancis, "menggeser" karakter backtick (mis. Menekan bergeser ` ) menghasilkan tanda Pound £ .

Anda bisa mendapatkan daftar semua sinyal dengan perintah kill -l:

Saya harus mencari tahu hal yang persis sama hari ini, dengan keyboard Jerman dan di Ubuntu. Jadi saya telah membuat skrip di bawah ini. Menemukan kunci yang tepat masih membutuhkan trial and error, tetapi cara itu cukup sederhana.

Untuk catatan, Ctrl - 4 melakukan trik dalam kasus saya.

#!/usr/bin/bash

# Shows character codes while typing. More to the point, it also captures signals.

# Trap function
sig() {
    printf '\t\t\t%s\t(re-enabled now, works next time)\n' "$1"
    # A trap disables itself after being called once.
    trap - $1
}

# Setting traps for all signals
mapfile -t signals < <(trap -l | grep -Eo SIG[-+A-Z0-9]+)

for signal in "${signals[@]}"; do
    trap "sig $signal" "$signal"
done

# Display characters
printf "Start typing to examine characters and signals. Exit with Ctrl-C (twice).\n\n"
printf "Char\tDec\t Hex\tSignal\n"

while IFS= read -rN 1 char; do 
    LC_CTYPE=C printf '\t%3d\t0x%02X\n' "'$char" "'$char"
done

Ctrl - \ kemungkinan besar akan menghasilkan kode ASCII 28. Coba lakukan hal lain yang mengirimkan kode ASCII 28.

Misalnya, coba ini: tahan Alt , lalu tekan 2 di numpad, lalu 8 di numpad, lalu melepaskan file Alt kunci. (Memang, jika Anda mengalami masalah saat menekan garis miring terbalik, mungkin Anda mengalami masalah saat menggunakan tombol numpad.) Saya tidak memiliki Mac oleh saya, tetapi itu berfungsi pada banyak sistem Unix-ish (dan DOS / Microsoft Windows), jadi mungkin bermanfaat memberikan tembakan.

Jika Anda berinteraksi dengan sistem jarak jauh, Anda dapat meletakkan karakter di file dan kemudian beberapa perangkat lunak terminal akan mengizinkan Anda menggunakan "unggahan ASCII" untuk mengirim penekanan tombol.

Sumber daya lain yang mungkin: AppleScript: mengirim kunci untuk OSX
