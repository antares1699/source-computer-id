Login sidik jari Windows 10 mengubah metode login default kembali ke kata sandi

Saya memiliki laptop Lenovo yang menjalankan Windows 10 yang saya gunakan secara mandiri saat dalam perjalanan, dan terhubung melalui dok ke monitor, keyboard, dan mouse saat saya di rumah.

Saat saya keluar, saya menggunakan pembaca sidik jari untuk masuk. Namun, saat saya menggunakannya di rumah, penutupnya tertutup, sehingga pembaca sidik jari tidak mudah diakses, jadi saya menggunakan keyboard untuk masuk.

Saat menggunakan keyboard untuk masuk, saya lebih suka masuk melalui PIN, daripada kata sandi. Jika saya masuk melalui PIN beberapa kali saat berada di rumah, metode ini mengingat metode masuk terakhir saya setiap kali, dan menjadikan PIN sebagai opsi masuk default. Namun, jika suatu saat, saya menggunakan login sidik jari, ini mengubah metode login default kembali ke Keyboard, jadi lain kali saya menggunakannya di rumah, saya harus melakukan dua klik mouse untuk mengubahnya kembali menjadi PIN lagi. Saya tahu ini bukan masalah besar, tapi menjengkelkan harus melakukan itu.

Adakah cara untuk mencegahnya mengalihkan metode login default dari PIN kembali ke Kata sandi setiap kali saya menggunakan pembaca sidik jari untuk login?

Memeriksa artikel ini tentang cara mengatur opsi masuk default Anda di Windows 10 menggunakan tweak registri.

  1. tekan Windows + R kunci bersama, lalu ketik regedit dan tekan Enter untuk mengakses Peninjau Suntingan Registri.

  2. Arahkan ke kunci registri: HKEY_LOCAL_MACHINE \ SOFTWARE \ Microsoft \ Windows \ CurrentVersion \ Authentication \ LogonUI \ UserTile

  3. Temukan SID akun Anda saat ini di panel kanan, dan klik dua kali di atasnya untuk memodifikasi.

  4. Tipe {D6886603-9D2F-4EB2-B667-1971041FA96B} di bidang Data nilai dan klik baik .

  5. Nyalakan ulang komputer Anda dan PIN akan ditampilkan sebagai proses masuk default Anda opsi di layar masuk.

GUID untuk setiap opsi masuk:

  • PIN: {D6886603-9D2F-4EB2-B667-1971041FA96B}
  • Gambar Logon: {2135F72A-90B5-4ED3-A7F1-8BB705AC276A}
  • Sandi: {60B78E88-EAD8-445C-9CFD-0B87F74EA6CD}
  • Masuk Sidik Jari: {BEC09223-B018-416D-A0AC-523971B639F5}

Ada cara untuk memaksa Windows 10 ke default ke layar Login Sidik Jari Windows Hello, terlepas dari metode login sebelumnya (kata sandi atau PIN). Idenya adalah untuk mencegah sistem mengubah metode login yang terkait dengan akun pengguna Anda di registri pada setiap boot.

PENTING : Tetapkan titik Pemulihan Sistem terlebih dahulu, untuk berjaga-jaga jika terjadi kesalahan. Lebih baik aman!

  1. Temukan SID nama pengguna Anda dengan memasukkan yang berikut ini di jendela prompt perintah

    wmic useraccount get name,sid

    Catat SID agar Anda dapat menemukannya di langkah berikutnya.

  2. Menggunakan RegEdit, navigasikan ke HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Authentication\LogonUI\UserTile

  3. Temukan SID kunci pengguna Anda dan ubah nilainya menjadi {BEC09223-B018-416D-A0AC-523971B639F5} (termasuk tanda kurung) yang merupakan metode masuk sidik jari. Mungkin sudah ada, tergantung pada metode login yang digunakan saat Anda terakhir kali boot. Jika demikian, biarkan apa adanya.

  4. Klik kanan folder UserTile dan klik Permissions

  5. Klik Advanced dan di jendela baru, klik Disable inheritance dan pilih Convert inherited permission into explicit permission on this object di jendela popup, lalu klik Apply.

  6. Klik dua kali pada "Administrator", klik "Edit", dan pastikan bahwa "Type" disetel ke "Allow" dan "Full Control" dicentang, yang seharusnya menjadi default. Dengan asumsi itu, klik "Batal".

  7. Klik dua kali pada "Sistem", klik 'Edit ", setel" Jenis "ke" Tolak ", klik" tampilkan izin lanjutan "dan hapus centang semua kotak kecuali untuk kotak" Setel nilai ". Itulah satu-satunya kotak yang seharusnya diperiksa.

  8. Centang kotak yang bertuliskan "Hanya terapkan izin ini ke objek dan / atau wadah dalam wadah ini" jika belum dicentang.

  9. Klik OK pada semua kotak sampai semua jendela perizinan ditutup untuk menyimpan pengaturan Anda.

  10. Hanya untuk memastikan, klik kanan tombol UserTile lagi, pilih "Izin", klik "Lanjutan" dan buka tab "Akses Efektif" di jendela.

  11. Klik "Pilih pengguna", ketik "SISTEM" (tanpa tanda kutip "), klik" OK ", dan klik" Lihat Akses Efektif ". Konfirmasikan bahwa ada" X "di samping" Kontrol Penuh "dan" Set Nilai ", dan bahwa semua entri lainnya masih memiliki tanda centang hijau. Jika demikian, Anda dapat membatalkan semua jendela perizinan dan menutup editor registri. Pada setiap boot ulang berikutnya, Anda sekarang akan melihat layar masuk sidik jari Windows Hello.

Metode ini bekerja dengan sempurna untuk saya. Satu-satunya masalah yang mungkin saya lihat adalah jika Anda memutuskan untuk menambahkan pengguna baru karena ketidakmampuan SISTEM untuk mengubah nilai registri kunci ini mungkin bermasalah, tetapi saya belum mencobanya. Dalam hal ini, Anda harus dapat memberikan kontrol penuh kembali ke izin SISTEM untuk sementara, menambahkan pengguna, dan mengulangi langkah-langkah di atas untuk mencegah SISTEM mengubah nilai dalam kunci registri UserTile ini untuk kedua pengguna. Perhatikan bahwa Anda dapat menyetel metode masuk default untuk setiap pengguna satu per satu.

Jika ada yang masih mengikuti utas ini dan ingin mencobanya, beri tahu saya jika berhasil.

Untuk menambahkan ke jawaban Durfee, berikut adalah baris yang Anda butuhkan untuk file batch untuk terus mengubah pengaturan menjadi apa yang Anda inginkan.

reg add HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Authentication\LogonUI\UserTile /v Enter_your_current_account_SID_here /t REG_SZ /d {60B78E88-EAD8-445C-9CFD-0B87F74EA6CD} /f

Jangan lupa untuk mengubah kode tersebut sehingga memiliki SID Anda, yang dapat ditemukan sebagai nama kunci yang sudah ada di sana.

Sekarang Anda dapat membuat tugas di Penjadwal Tugas yang menjalankan skrip di setiap membuka kunci stasiun kerja. Dengan begitu Anda dapat terus menggunakan buka kunci sidik jari dan apa pun yang terjadi, itu akan selalu menampilkan kata sandi (atau perintah apa pun yang Anda tentukan dengan nilainya) di layar kunci.

Saya terkejut tentang semua jawaban dengan solusi registri. Tetapi Anda hanya memerlukan registri untuk penyedia kredensial yang tepat dan CLSID-nya.

Jauh lebih baik dan lebih mudah menggunakan editor kebijakan grup lokal untuk ini.

Pilih penyedia kredensial yang tepat dari sini: HKEY_LOCAL_MACHINE \ SOFTWARE \ Microsoft \ Windows \ CurrentVersion \ Authentication \ Credential Providers

Dalam hal ini adalah Penyedia Kredensial WinBio: {BEC09223-B018-416D-A0AC-523971B639F5}

Di jendela Editor Kebijakan Grup Lokal, buka: Konfigurasi Komputer -> Template Administratif -> Sistem -> Logon

Di panel kanan, cari pengaturan kebijakan bernama "Assign a default credential provider". Edit kebijakan ini dan ubah ke status Diaktifkan. Di kotak input "Tetapkan penyedia kredensial berikut sebagai penyedia kredensial default", ketik CLSID {BEC09223-B018-416D-A0AC-523971B639F5}.

Klik Apply diikuti dengan OK. Anda dapat menutup Editor Kebijakan Grup dan melakukan boot ulang untuk membuat perubahan efektif.
