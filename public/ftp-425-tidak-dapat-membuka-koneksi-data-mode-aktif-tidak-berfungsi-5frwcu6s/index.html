FTP - 425 Tidak dapat membuka koneksi data (Mode Aktif tidak berfungsi)

Saya telah meneliti topik ini secara menyeluruh tentang apa itu mode Aktif dan mengapa saya mengalami kesalahan 425 Can't open data connection. yang terkenal tetapi setelah semua dokumentasi dan tutorial dan forum di internet tidak menghasilkan solusi, saya memutuskan untuk datang ke sini untuk satu celah terakhir. .

Dengan berbagai cara (dijelaskan di bawah), saya telah dapat terhubung ke server ini dari jarak jauh tetapi hanya melalui koneksi Pasif. Saya tidak dapat terhubung menggunakan mode Aktif - yang merupakan default klien FileZilla (setidaknya dalam kasus saya).

Server menjalankan Windows 8 dan FileZilla Server (bagian dari paket XAMPP) dan menggunakan porta:

  • 21 untuk mendengarkan
  • 990 untuk FTP implisit melalui TLS
  • 14147 untuk manajemen jarak jauh
  • 6000-7000 untuk Mode Pasif

Porta ini dibuka di NAT router dan Windows Firewall.

Ini adalah pengaturan untuk server

dan ini adalah pengaturan untuk klien

di bawah mode pasif, fallback ke mode aktif juga dipilih

Untuk alasan yang jelas, koneksi lokal berfungsi tetapi koneksi jarak jauh tidak.

Layak disebut - ftptest.net melaporkan koneksi sebagai KEBERHASILAN , namun saya menemukan bahwa ia menggunakan mode PASV untuk menyambung ke server. Saya telah meletakkan log dari sambungan yang berhasil ini pada pastebin sebagai referensi. Sekali lagi ini menggunakan FTP Implisit melalui TLS. Koneksi FTP standar berfungsi menggunakan situs web ini (dan bukan melalui cara menghubungkan saya sendiri), tetapi saya menonaktifkannya di dalam server yang memaksa koneksi SSL / TLS saja.

FTP memiliki koneksi kontrol dan koneksi data. Koneksi kontrol dimulai dari klien ke server dan dengan demikian biasanya tidak ada masalah, tetapi koneksi datanya berbeda:

  • dalam mode aktif server mencoba untuk menyambung ke klien. Pengaturan IP dan port yang diperlukan dikirim dalam perintah PORT atau EPRT di dalam saluran kontrol. Jika klien berada di belakang mode aktif tidak akan berfungsi, karena klien menggunakan alamat IP yang tidak dapat dijangkau oleh server. Masalah serupa adalah jika klien berada di belakang firewall, karena koneksi dari luar akan diblokir begitu saja. Hanya firewall yang sadar FTP mungkin menggunakan pembantu khusus untuk mengelola koneksi aktif (dan ini tidak akan bekerja dengan FTPS).
  • dalam mode pasif klien terhubung ke server. IP dan port yang diperlukan dikirim dalam respons ke perintah PASV atau EPSV. Tidak ada masalah dengan NAT di sisi klien dan biasanya juga tidak ada masalah dengan firewall. Tetapi akan ada masalah jika server itu sendiri berada di belakang firewall atau NAT.

Singkatnya: Mode pasif berfungsi hampir sepanjang waktu, sedangkan mode aktif hanya berfungsi jika klien memiliki alamat publik. Tetapi ini tidak terjadi untuk klien di belakang router (di rumah, hotspot publik ...) dan biasanya juga tidak di jaringan seluler.

Menjalankan klien FTP di perangkat jaringan lokal mengalami masalah saat menggunakan alamat IP eksternal, seperti: 92.646.178.168. Log server akan menunjukkan IP ini.

Dengan mengonfigurasi 'nama atau alamat host' klien, jadi ini bukan alamat IP, gunakan nama seperti "Pembuat". Router dengan benar mengirimkan paket data ke FZserver. Jadi saat menguji koneksi, saya menyarankan perubahan sederhana ini. Kemudian gunakan perangkat jaringan lain dalam kasus saya perangkat nirkabel hanya untuk menggunakan IP eksternal di sini.
