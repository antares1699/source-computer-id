mengoreksi izin di Mac setelah penginstalan Mountain Lion

Mengupgrade ke Mountain Lion dari Snow Leopard telah memengaruhi banyak perangkat lunak yang saya gunakan. Saat mencoba memperbarui dan menginstal beberapa permata Ruby di sistem saya serta perangkat lunak lain (menggunakan Homebrew), saya mendapat pesan kesalahan ini

Warning: The /usr/local directory is not writable.
Even if this directory was writable when you installed Homebrew, other
software may change permissions on this directory. Some versions of the
"InstantOn" component of Airfoil are known to do this.

You should probably change the ownership and permissions of /usr/local
back to your user account.

Adakah yang tahu bagaimana saya dapat mengubah kepemilikan dan izin / usr / local kembali ke akun pengguna saya?

Homebrew memberi tahu Anda apa yang harus dilakukan:

Anda mungkin harus mengubah kepemilikan dan izin / usr / local kembali ke akun pengguna Anda.

Untuk itu kami akan menggunakan chown dan (secara rekursif) menetapkan pengguna Anda sebagai pemilik untuk /usr/local:

sudo chown -R $(whoami) /usr/local

GitHub Gist ini melakukan hal yang sama dan sedikit lebih banyak, yang mungkin diperlukan atau mungkin tidak diperlukan untuk mendapatkan Homebrew yang berfungsi penuh lagi: Perbaiki izin di / usr / local untuk Homebrew
