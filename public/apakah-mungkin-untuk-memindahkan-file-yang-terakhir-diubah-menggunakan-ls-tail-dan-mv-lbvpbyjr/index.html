Apakah mungkin untuk memindahkan file yang terakhir diubah menggunakan ls, tail dan mv?

Saya ingin memindahkan 3 file terakhir yang dimodifikasi dari direktori menggunakan perintah bash. Namun, saya perhatikan bahwa saya dapat menggunakan find dengan cara berikut:

find . -type f -mtime -0.5 -print -exec mv {} /home/user/Desktop \;

Tetapi saya belum menemukan cara melakukan hal yang sama dengan ls -tr | tail -n 3. Misalnya, ini tidak berhasil:

ls -tr | tail -n 3 -exec mv {} /home/user/Desktop             
tail: invalid option -- 'e'

Satu-satunya alasan saya lebih suka menggunakan opsi kedua adalah untuk menentukan jumlah file daripada perkiraan waktu. Apakah mungkin membuatnya bekerja dengan ls dan tail

Perintah yang Anda cari adalah xargs, karena tail tidak memiliki kemampuan asli untuk menjalankan program.

Perintah lengkapnya adalah:

ls -tr | tail -n 3 | xargs -I{} mv {} /home/user/Desktop

Memecahnya:

  • ls -tr mencantumkan file yang diurutkan berdasarkan tanggal / waktu modifikasi (-t). Yang terbaru adalah yang pertama, secara default; itu terbalik (file terakhir diubah terakhir) jika Anda menambahkan -r.
  • tail -n 3 memfilternya hingga tiga entri terakhir.
  • xargs -I{} mv {} /home/user/Desktop menjalankan mv {} /home/user/Desktop untuk setiap baris yang diterima dari tail. Perhatikan bahwa {} diganti dengan keluaran dari tail.

Perhatikan bahwa Anda mungkin perlu keluar dari tanda kurung kurawal dalam panggilan ke xargs.

xargs -I\{\} mv \{\} /home/user/Desktop

Bergantung pada nama file, solusi yang mengandalkan output ls mungkin bermasalah. Dengan menggunakan zsh, seseorang dapat mengandalkan flag globbing untuk menyelesaikan pekerjaan:

mv *(.om[1,3]) ${somewhere}

'Ajaib' ada di antara dua tanda kurung setelah *:

  • . hanya memilih file biasa;
  • om mengurutkan hasil glob berdasarkan mtime;
  • [1,3] memilih tiga hasil pertama

Saya menyadari bahwa pertanyaan tersebut secara khusus menanyakan solusi pesta, jadi sebenarnya jawaban ini tidak akan memenuhi syarat. Saya tetap mempostingnya untuk orang lain yang mungkin memiliki tugas serupa dan dapat menggunakan shell lain.
