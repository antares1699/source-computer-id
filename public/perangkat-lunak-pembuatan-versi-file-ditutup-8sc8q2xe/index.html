aku akan mencoba Git . Dan untuk bantuan dalam memulai, GitCasts sangat bagus untuk saya. Secara khusus, Anda mungkin ingin membayar Git di Windows .

Tapi untuk pembuatan versi semua file di sistem Anda, saya tidak yakin seberapa baik itu akan bekerja. Git dirancang dengan mengutamakan pembuatan versi file kode sumber (teks), bukan keseluruhan sistem. Ini berfungsi dengan baik untuk saya dengan jenis file lain, tetapi saya belum menggunakannya sebesar itu. Saya akan menunggu untuk melihat apakah orang lain dapat memberi Anda beberapa ide yang dirancang untuk apa yang ada dalam pikiran Anda sebelum mengotak-atik ini. Saya yakin ini juga akan berlaku untuk Subversion.

Juga, seperti yang ditunjukkan Shevek tentang Subversion, ini akan membutuhkan beberapa pekerjaan untuk membuatnya secara otomatis versi file Anda. Namun, yang diperlukan hanyalah menjalankan git add -A dan kemudian git commit -m "Some message" setiap kali Anda ingin menyimpan versi. Tentu saja, Anda mungkin bisa menulis skrip untuk melakukan ini untuk Anda.
