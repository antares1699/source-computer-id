Turunkan versi PHP dari 5.5.3 menjadi 5.4.x di Ubuntu 13.10

Selama pembaruan dari Ubuntu 13.04 ke 13.10, PHP ditingkatkan ke 5.5.3.

Bagaimana saya bisa menurunkan versi ke 5.4.x?

  1. Sebagai medigeek disarankan : Anda bisa mencoba menggunakan ondrej php5-oldstable paket, tapi saya pikir saat ini mereka tidak berfungsi pada Saucy Salamander.
  2. Anda dapat mengkompilasi PHP Anda sendiri.
  3. Solusi mudahnya adalah dengan beralih sementara ke repositori lama (jarang), menginstal PHP (5.4), Apache (2.2), dll dan mengunci versi paket (sehingga tidak akan diupgrade lagi):

Harap pertimbangkan ini sebagai operasi yang berisiko, karena Anda mungkin kehilangan pengaturan dan merusak sistem Anda. Saya tidak menguji semua garis, jadi tolong beri tahu saya jika saya harus menyesuaikan sesuatu

# upgrade system, so you can add to ignore all updates later
sudo apt-get update
sudo apt-get upgrade

# remove your php, apache, etc
sudo apt-get purge apache2 php5 libapache2-mod-php5 # add here your server packages

# change repositories to raring  (with backup)
sudo sed -i.bak "s/saucy/raring/g" /etc/apt/sources.list

# update and install server packages
sudo apt-get update
sudo apt-get install apache2 php5 libapache2-mod-php5 phpmyadmin #add here packages you need and make sure you install php5.4 and apache2.2

# change repositories back to saucy
sudo sed -i "s/raring/saucy/g" /etc/apt/sources.list

# ignore all current upgrades (package hold)
sudo apt-mark hold `aptitude -F%p --disable-columns search ~U`

Saya bisa mendapatkan ini bekerja pada 13.10 dengan menggunakan PHPBrew - https://github.com/c9s/phpbrew

Setelah Anda mengaktifkan dan menjalankan PHPBrew, saya harus menginstal APXS dengan:

sudo apt-get install apache2-threaded-dev

Kemudian saya bisa menjalankan perintah PHPBrew berikut:

sudo phpbrew install php-5.4.20 +mysql+apxs2-/user/bin/apxs

Ini adalah parameter apxs2 yang memungkinkan PHPBrew membuat file php5.load baru untuk Apache (file Anda saat ini dicadangkan) yang mereferensikan file php5_module baru /usr/lib/apache2/modules/libphp5.4.20.so

Langkah terakhir bagi saya untuk mengaktifkan dan menjalankan MySQL adalah mengedit file php.ini baru - yang terletak di tempat PHPBrew menginstal sumber php-5.4.20 baru, dan secara eksplisit memanggil file mysqld.sock. Untuk beberapa alasan ini tidak berhasil di luar kotak untuk saya.

Semoga langkah-langkah ini membantu orang lain yang ingin menurunkan versi dari PHP 5.5 di Ubuntu 13.10
