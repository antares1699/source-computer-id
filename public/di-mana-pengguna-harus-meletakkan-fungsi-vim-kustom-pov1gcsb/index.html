Di mana pengguna harus meletakkan fungsi vim kustom?

Beberapa fungsi spesifik jenis file, dan saya bertanya-tanya apa praktik terbaik untuk mengatur fungsi ini. Haruskah saya mendefinisikannya pada ftplugin sebagai fungsi script-local atau meletakkan semuanya pada autoload?

Tergantung.

Pertama, bersikaplah pragmatis. Jika itu hanya fungsi singkat, dan hanya digunakan secara lokal, saya tidak ragu untuk memasukkannya langsung ke dalam skrip ~/.vimrc saya, atau ke dalam skrip ~/.vim/ftplugin/*.vim. Yang terakhir ini agak tidak efisien, karena fungsinya didefinisikan ulang untuk setiap file yang dibuka yang memiliki tipe file yang sama.

Anda sudah tahu tentang fungsi muat otomatis , dan tentu saja, ini adalah tempat kanonik untuk meletakkan fungsi kustom. Orang dogmatis akan bersikeras bahwa plugin (ft-) hanya berisi konfigurasi, pemetaan, perintah kustom, dan :autocmd, dan semua fungsi ditempatkan di ~/.vim/autoload/. Saya akan merekomendasikan bahwa untuk fungsi yang lebih lama (atau set fungsi yang memanggil satu sama lain), atau apa pun yang ingin Anda publikasikan untuk digunakan kembali oleh orang lain.

Untuk plugin tipe file, saya telah mengadopsi konvensi penempatan fungsi seperti itu ke ~/.vim/autoload/ft/{filetype}.vim, atau ~/.vim/autoload/ft/{filetype}/{submodule}.vim.
