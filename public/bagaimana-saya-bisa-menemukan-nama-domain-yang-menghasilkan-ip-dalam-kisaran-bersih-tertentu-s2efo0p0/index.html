Bagaimana saya bisa menemukan nama domain yang menghasilkan IP dalam kisaran bersih tertentu?

Misalnya, kisarannya:

172.128.0.0 - 172.191.255.255

Saya perlu menemukan beberapa domain yang mengarah ke alamat IP dalam jangkauan. Apa itu mungkin ? Saya menggunakan sistem Linux.

Nah, itu Utilitas DiG ( man dig ) dapat melakukan pencarian DNS terbalik untuk melihat apakah alamat IP tertentu memiliki data DNS terkait. AFAIK hanya menerima satu alamat pada satu waktu, tetapi Anda dapat menggunakan beberapa pesta perulangan untuk menghasilkan perintah untuk memeriksa semua alamat.

for i in $(seq 128 191) ; do 
  for j in $(seq 1 255) ; do
    for k in $(seq 1 255) ; do
      dig -x 172.$i.$j.$k
      sleep 5
    done
  done
done

Dalam satu baris:

for i in $(seq 128 191) ; do for j in $(seq 1 255) ; do for k in $(seq 1 255) ; do dig -x 172.$i.$j.$k ; sleep 5 ; done ; done ; done

Anda mungkin menginginkan opsi +short (atau opsi lain) untuk membantu mendapatkan keluaran menjadi beberapa bentuk yang berarti. Lihat man dig untuk opsi yang memungkinkan; inilah sintaksnya:

dig +short -x 172.$i.$j.$k

Ada situs yang menawarkan apa yang dikenal sebagai pencarian DNS pasif. Mereka membangun database dengan memantau lalu lintas DNS, dan melacak hasilnya.

Biasanya, pencarian DNS hanya memberikan IP terbatas ke layanan pencarian nama host. Jika pemilik ruang IP repot-repot mengatur DNS terbalik (rDNS), ia dapat memberi tahu Anda bahwa ada satu pemetaan dari 78.56.34.12.in-addr.arpa ke satu nama host, tetapi ini belum tentu keseluruhan kebenaran, atau bahkan kebenaran sama sekali. Dalam praktiknya, sangatlah biasa jika tidak ada informasi DNS terbalik, atau memberikan hasil yang tidak cocok dengan informasi DNS penerusan yang sesuai. Dan biasanya, layanan web hosting memiliki ratusan atau ribuan situs web yang hidup di alamat IP yang sama; tidak ada cara biasa untuk mengetahui apa itu dari DNS, mengingat alamat IP sebagai masukan.

(Reverse DNS adalah jawaban yang diberikan kode dig untuk Anda.)

DNS pasif melihat hasil dari kueri DNS, biasanya dengan tinggal di server DNS dengan lalu lintas tinggi dan mengamati hasil yang diperolehnya. Jadi jika seseorang - yang entah bagaimana dilacak oleh layanan ini - mencari wild.example.com dan mendapatkan alamat IP sebagai hasilnya, hasilnya disimpan dalam database pemetaan dari alamat IP ke nama host, yang kemudian dapat ditanyakan.

Lihat mis. http://www.bfk.de/bfk_dnslogger.html dan / atau https://www.dnsdb.info/
