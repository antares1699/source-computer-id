Apakah ada cara untuk menyalin dan menempel secara terprogram ke RDP?

Saya ingin menulis skrip cepat untuk mengirim satu file (yang sering saya ubah) dari workstation saya (OS windows7) ke server jauh (windows server 2012) melalui klien RDP.

Idealnya, saya mencari untuk membuat contoh proses rdpclip.exe dengan beberapa argumen. Sesuatu seperti: rdpclip.exe -f [fileToCopyFromLocalToRemote] tetapi sepengetahuan saya (dan pencarian singkat di Google) tidak ada yang setara seperti itu.

Juga, saya tidak mencari alat pihak ketiga untuk melakukan ini!

Anda dapat menggunakan PowerShell untuk meletakkan file ke clipboard:

Add-Type -AssemblyName System.Windows.Forms
$f = New-Object System.Collections.Specialized.StringCollection
$f.Add("D:\tmp\mypic1.bmp")
[System.Windows.Forms.Clipboard]::SetFileDropList($f)

Jika Anda berniat untuk tetap terhubung ke sesi RDP, maka Anda bisa memetakan perjalanan pada koneksi Anda yang dapat membuat drive lokal Anda tersedia untuk komputer jarak jauh. Ini ada di Remote Desktop Sumber Daya Lokal tab di bawah Lebih... tombol.

Setelah diaktifkan, Anda dapat membuat tugas terjadwal untuk menyalin file. Anda dapat mengakses drive lokal dari nama berbagi khusus \\tsclient\c atau drive mana pun yang Anda coba akses, seperti yang dijelaskan di artikel TechNet ini . Sekian contoh script yang akan Anda jalankan di komputer jarak jauh mungkin terlihat seperti ini:

IF EXIST "\\tsclient\c\Mydir\MyFile" copy /Y "\\tsclient\c\Mydir\MyFile" "C:\Syncrhonized\MyDir\MyFile"

Anda dapat memasukkan ini sebagai tugas terjadwal untuk dijalankan setiap beberapa menit atau lebih. Waspadai masalah potensial jika orang lain masuk ke komputer ini dengan pemetaan drive dan struktur file. Skrip Anda tidak akan tahu komputer mana yang benar, kecuali Anda memasukkan semacam logika untuk memeriksanya.

Menurut penelitian saya, rdpclip.exe tidak menerima argumen baris perintah apa pun. Untuk menggunakannya untuk menyalin file secara terprogram tampaknya tidak dimaksudkan.

Sejak Windows Vista robocopy adalah bagian dari setiap instalasi Windows. Saya tidak tahu bagaimana Anda ingin menggunakan rdpclip.exe untuk mentransfer file ke host jarak jauh. Tapi pikiran pertama saya adalah memasang netshare, menggunakan robocopy untuk mentransfer file dan turun dari netshare. Dalam sebuah skrip akan terlihat seperti:

net use \\Server\Sharename /USER:[username] [passwort]
robocopy [Source] [Destination]
net use \\Servername\Sharename /del

Seperti yang Anda katakan bahwa Anda tidak dapat berbagi hard disk lokal Anda dengan server, satu-satunya mekanisme berbagi yang tersisa adalah papan klip.

Jika papan klip dapat dibagikan melalui koneksi RDP, Anda dapat menggunakan sumber terbuka proyek Clipboard RDC .

Proyek ini terdiri dari program Java yang sangat kecil yang diluncurkan pada keduanya sisi koneksi. Program sisi klien digunakan untuk menyandikan file dan memuatnya ke clipboard, sementara program sisi server menerjemahkan dan menyimpannya dalam file lokal. Batasannya di sini adalah bahwa file harus cukup kecil agar muat di papan klip.

Seperti berdiri, ini lebih banyak pekerjaan daripada yang ingin Anda lakukan. Tetapi Anda dapat memodifikasi program sederhana ini dan memisahkannya menjadi dua bagian:

  1. Program sisi klien yang akan menerima jalur file sebagai parameter dan letakkan file yang disandikan di clipboard.
  2. Program sisi server yang akan berulang, menguji setiap beberapa detik untuk file keberadaan file tersebut di clipboard, mendekode dan menyimpannya di folder.

Karena ini akan menjadi program Anda sendiri yang Anda kontrol sepenuhnya, Anda bisa menambahkan fitur seperti itu seperti menambahkan jalur tujuan ke informasi yang Anda tempatkan di clipboard, sehingga ini menjadi alat tujuan umum untuk mentransfer secara otomatis beberapa file berukuran cukup kecil dari klien ke server.
