Distro linux mana yang berjalan pada SSD yang lambat?

Saya membeli Eee PC 1000, model Linux SSD, beberapa tahun lalu. Saya akhirnya menempatkan Easy Peasy (kemudian disebut Ubuntu Eee) ke atasnya, hanya untuk tidak puas dengan kecepatannya. Kemudian saya meletakkan Windows XP di atasnya, dan dengan BANYAK tweaker itu berjalan baik-baik saja. Sekarang saya menariknya keluar dan membersihkannya tetapi saya ingin menjalankan Linux.

Ini memiliki prosesor Intel Atom 1.6ghz dan 2gb RAM (saya memutakhirkannya) jadi tidak ada kekurangan daya di sana, tetapi SSD-nya sangat lambat; ia memiliki buffer tulis kecil, tetapi ketika Anda melakukan sesuatu yang sedikit signifikan, Anda dapat merasakan sistem tersendat setiap satu atau dua detik saat SSD menghentikan semuanya sementara ia membuang cache lengkapnya ke disk. Saya berbicara tentang gagap yang serius, dan cache-nya tidak terlalu besar; agar Firefox tidak gagap, saya harus memindahkan semua cache ke RAM dan menonaktifkan riwayat (bahkan hanya menulis log riwayat ke disk yang membekukan sistem dengan setiap halaman web).

Bagaimanapun, saya harap saya telah memberi Anda gambaran yang layak tentang seberapa lambat SSD ini. Dengan itu, apakah ada distro Linux yang dioptimalkan untuk hard drive yang sangat lambat tetapi sistem bertenaga yang layak? Saya tidak mencari sesuatu yang kurang bertenaga karena prosesor dan RAM sangat bertenaga, saya hanya ingin sesuatu yang mungkin dioptimalkan untuk tidak sering menulis ke disk.

Dengan mesin serupa, saya menjalankan Ubuntu dengan sedikit penyesuaian.

  • Tidak ada swap, meskipun saya tidak yakin ini benar-benar menghasilkan apa-apa. (Tanpa swap berarti aplikasi tidak membeku karena halaman datanya dimuat dari swap, tetapi itu juga berarti aplikasi dibekukan karena kode atau data hanya-baca sedang dimuat ulang dari file.)

  • Sistem file dipasang dengan opsi relatime (sudah default dengan Ubuntu).

  • /tmp adalah tmpfs, yaitu, garis pada /etc/fstab terlihat seperti ini

    tmpfs /tmp tmpfs mode=1777

  • Saya menggunakan Chromium, bukan Firefox sebagai browser web grafis. ~/.cache adalah tautan simbolis ke direktori di bawah /tmp, yang secara radikal meningkatkan daya tanggap Chromium dengan SSD yang lambat.

Naluri saya mengatakan cache Firefox bisa jadi itu. Firefox menulis banyak hal acak di drive seperti yang tidak Anda bayangkan. Ketika saya menjalankan FF dari flash drive di Windows, saya tidak dapat melakukan apa pun dengan drive tersebut sampai saya mematikan FF dari Task Manager.

Tombuntu bagus daftar dari beberapa (4) hal yang dapat Anda lakukan untuk mengoptimalkan Linux untuk SSD Anda.

Anda mungkin juga ingin berinvestasi di Windows 7 (meskipun biaya Windows 7 mungkin cukup untuk SSD baru: P). Ada banyak sekali panduan di luar sana untuk mengoptimalkan Windows untuk SSD, tetapi tidak begitu banyak untuk Linux.

Jika saya ingat dengan benar beberapa versi Knoppix awal (mungkin 3,6 atau lebih?) Memiliki kemampuan untuk memuat CD keluar ke RAM dan dijalankan dari sana. Saya mencobanya beberapa kali dan seingat saya butuh beberapa menit untuk memuat tetapi cukup bersemangat setelah itu. Dengan RAM 2 GB kedengarannya memang layak, tergantung seberapa sering kamu perlu boot up gadget ini. Anda dapat menyalin Knoppix dari internet ke HD dan kemudian menyalin OS ke RAM selama boot selanjutnya. Boot memakan waktu tetapi berhasil.

Jika Anda siap untuk solusi perangkat keras, Anda selalu dapat kembali ke stok 160 GB HD.

Gagal keduanya, saya mendukung solusi kartu SD yang disebutkan di atas - jika itu akan boot dari SD.
