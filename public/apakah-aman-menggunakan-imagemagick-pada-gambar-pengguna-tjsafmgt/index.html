Apakah aman menggunakan ImageMagick pada gambar pengguna?

Saya pernah mendengar mungkin ada masalah keamanan saat menggunakan ImageMagick pada gambar sewenang-wenang yang disediakan pengguna. Misalnya, tautan di bawah ini memberikan bukti bahwa dulu ada beberapa eksploitasi buffer overflow. Adakah yang tahu jika saya sekarang dapat menggunakan ini dengan aman sebagai bagian dari layanan web yang menangani gambar pengguna?

Ya, ada risiko keamanan - perangkat lunak memiliki bug dan beberapa di antaranya dapat digunakan untuk menyerang sistem yang menjalankannya.

Tapi Anda bisa meminimalkan risikonya:

  1. Buat akun pengguna yang memiliki izin minimum sehingga cukup untuk menjalankan ImageMagick. Sekarang gunakan hanya akun itu untuk menjalankan ImageMagick

  2. Aktifkan pembaruan keamanan otomatis: Sistem Linux seperti Ubuntu dapat dikonfigurasi untuk menginstal perbaikan keamanan secara otomatis. Ini mengurangi waktu sistem Anda rentan jika kerentanan akan ditemukan. Sistem Windows juga memiliki sistem pembaruan otomatis tetapi itu tidak termasuk perangkat lunak pihak ketiga seperti ImageMagick.

Tidak.

ImageMagick dirancang untuk fleksibilitas. Ini tidak dikeraskan terhadap input yang dibuat seperti perpustakaan pemrosesan gambar browser. Mungkin akan jauh lebih mudah untuk menemukan bug yang dapat dieksploitasi di perpustakaan seperti ImageMagick daripada di browser.

Penyerang yang menemukan buffer overflow di ImageMagick kemudian dapat menggunakannya untuk menjalankan kode arbitrer dengan izin proses yang menjalankan ImageMagick.

Hal ini dapat memberikan landasan bagi penyerang yang dapat mereka gunakan untuk meningkatkan hak lebih jauh; misalnya melalui PATH dan serangan kuda trojan , dengan mengacaukan backend lain atau dengan mengeksploitasi layanan lain yang belum ditambal di dalam lingkungan produksi Anda.

Jika Anda benar-benar harus gunakan ImageMagick pada konten pihak ketiga, lihat "Pertimbangan Keamanan Lainnya" di Dokumen kebijakan keamanan ImageMagick :

Selain kebijakan keamanan, Anda dapat membuat ImageMagick lebih aman dengan ...

  • menjaga ImageMagick tetap mutakhir. Rilis terbaru memiliki perbaikan untuk setiap kelemahan keamanan yang kami temukan di masa lalu.
  • membersihkan nama file atau opsi baris perintah yang Anda berikan ke ImageMagick.
  • menjalankan ImageMagick dalam wadah perangkat lunak yang sudah dibersihkan seperti Docker.
  • menjalankan ImageMagick sebagai pengguna dengan hak paling rendah (mis. 'tidak ada'). secara eksplisit mengatur jenis file gambar. Misalnya, gunakan nama file png: image.png daripada image.png. Tanpa jenis gambar eksplisit dalam nama file, ImageMagick menebak jenis gambar tersebut.
