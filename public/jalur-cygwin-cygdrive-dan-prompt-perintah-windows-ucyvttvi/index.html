jalur cygwin cygdrive dan Prompt Perintah Windows

Saya mengalami masalah aneh dengan cygwin yang bertindak tidak konsisten di antara penginstalan, khususnya scp. Saya memiliki c: \ cygwin \ bin di Windows PATH saya dalam kedua kasus. Ketika saya menjalankan perintah berikut dari a Prompt Perintah Windows , namun, saya mendapatkan hasil yang sangat berbeda antara kedua penginstalan:

scp /cygdrive/c/something.txt User@server:${HOME}/something.txt

Di satu mesin itu mentransfer file dengan baik, tetapi di mesin lain saya mendapatkan kesalahan:

/cygdrive/c/something.txt: No such file or directory

Namun, jika saya menjalankan perintah dengan cara ini pada mesin yang memberi saya kesalahan, itu mentransfer dengan baik:

scp /c/something.txt User@server:${HOME}/something.txt

Mengapa ada perbedaan? Apakah ada sesuatu yang perlu saya konfigurasikan di dalam cygwin agar ini berfungsi dengan / cygdrive / c?

PEMBARUAN: Ini sesuatu yang lebih menarik. Jika saya melakukan ls /c dari prompt perintah Windows saya mendapatkan apa yang Anda harapkan, sebagai daftar semuanya di C :. Namun, ls /cygdrive/c mengatakan bahwa itu tidak ada. Menjalankan perintah tersebut dari cygwin bash menghasilkan perilaku yang berlawanan.

Masalahnya adalah prioritas PATH saya. Dulu

PATH=%PATH%;C:\cygwin\bin

tapi itu perlu

PATH=C:\cygwin\bin;%PATH%

Setelah memperbaiki PATH /cygdrive/c diselesaikan seperti yang diharapkan.

Saya mengalami masalah ini setelah saya menginstal Git, menambahkan binnya ke PATH saya, kemudian menginstal Cygwin dan menjalankan ssh-host-config untuk menyiapkan layanan Windows untuk sshd. Masalahnya tampaknya cara ssh-host-config mendaftarkan layanan menyebabkan sshd menjalankan perintah bash (atau scp, atau ssh-disediakan) Git yang dapat dieksekusi daripada Cygwin dalam proses menyiapkan koneksi scp. Untuk memperbaikinya, saya menghapus layanan seperti yang diinstal oleh ssh-host-config. Dari Cygwin bash, saya mengeksekusi cygrunsrv -R sshd. Kemudian saya mendaftarkan ulang layanan dengan penggantian variabel PATH,

cygrunsrv -I sshd -d "CYGWIN sshd" -p "/usr/sbin/sshd" \
  --env "PATH=/usr/bin:/bin:/cygdrive/c/WINDOWS/system32:/cygdrive/c/WINDOWS:$PATH" \
  -a "-D" -u <priv_user> -w <password>

di mana priv_user mungkin cyg_server. Lihat http://cygwin.com/ml/cygwin/2005-05/msg00012.html .
