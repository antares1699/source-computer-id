Cara mengonfigurasi mod-proxy-HTML tanpa mendapatkan kesalahan encoding konten

Saya telah menyiapkan reverse proxy di LAN tempat kerja saya sehingga beberapa situs dapat diakses dari luar LAN kami. Saya telah menyertakan mod-proxy-HTML, sehingga menangani penulisan ulang URL dalam HTML yang dikembalikan.

Namun Firefox dan Chromium mengeluhkan masalah encoding.

Jika saya mengedit opsi network.http.accept-encoding dari Firefox dan menghapus gzip;deflate default; maka situs tersebut terlihat baik-baik saja.

Konfigurasi Apache seperti ini:

SetOutputFilter INFLATE;proxy-html;DEFLATE

memecahkan masalah saya.

Gunakan ini:

ProxyHTMLCharsetOut *

* ProxyHTMLCharsetOut Nama Syntax ProxyHTMLCharsetOut Ini memilih encoding untuk keluaran mod_proxy_html. Biasanya tidak boleh digunakan, karena setiap perubahan dari UTF-8 default (Unicode - seperti yang digunakan secara internal oleh libxml2) akan memberlakukan overhead pemrosesan tambahan. Token khusus ProxyHTMLCharsetOut * akan menghasilkan keluaran menggunakan pengkodean yang sama sebagai masukan. *
