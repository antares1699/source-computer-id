Masalah dengan kunci ssh bersama

Mengikuti proses yang saya gunakan lingkungan lain , Saya sudah mencoba mengatur kunci bersama antara Mac saya dan server web CentOS 4 saya. Saya telah melihat masalah yang sama dengan workstation Ubuntu 7.10 lama saya yang mencoba terhubung melalui kunci ke server web yang sama.

Saya telah mencoba kedua jenis tombol dsa dan rsa (sshkeygen -t <type>).

File sshd_config di server web saya tampaknya mengizinkan login berbasis kunci:

RSAAuthentication yes
PubkeyAuthentication yes
AuthorizedKeysFile      .ssh/authorized_keys

Dan .ssh / authorized_keys saya telah menambahkan kunci dsa dan rsa saya.

Di mana saya harus mencari apa yang harus diubah selanjutnya untuk membuat login berbasis kunci "Just Work ™"?

Apakah itu terkait dengan garis,

#UseDNS yes

dan sshd mencoba melakukan pencarian balik pada IP saya, tetapi tidak bisa karena itu NAT?
