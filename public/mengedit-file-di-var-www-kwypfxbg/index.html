Mengedit file di / var / www

Saya tahu bahwa pemilik file pada /var/www harus www-data agar dapat dijalankan dengan benar oleh apache. Tetapi ketika saya menjalankan perintah chown -R www-data:www-data /var/www saya tidak dapat menyimpan file php di editor saya.

Bagaimana saya bisa mengedit file tanpa mengacaukan izin?

chown -hR www-data:www-data /var/www

Kepemilikan yang tepat.

chmod  -R g+rw /var/www

Ini memastikan grup dapat membaca dan menulis file, tetapi tidak mengubah izin lainnya.

find /var/www -type d -ls -exec chmod g+x {} ';'

Ini memastikan grup dapat mencantumkan direktori, tetapi tidak mengubah bit x untuk file.

usermod -aG www-data your-user

Tambahkan diri Anda ke grup www-data sehingga Anda dapat menggunakan izin grup ini.

Pemilik file tidak harus www-data agar apache berfungsi dengan baik. Persyaratannya adalah bahwa server apache yang berjalan sebagai

sudo chmod g+s /var/www
sudo usermod -aG www-data $LOGNAME
sudo chown -R $LOGNAME /var/www
harus dapat membaca berkas. Kebanyakan sistem secara default mengizinkan akses baca grup, jadi memiliki grup pada file berukuran www-data sudah cukup.

Server apache harus dapat menulis konten dinamis seperti wiki, direktori unggahan konten, dan cache. Jika tidak, akan lebih aman untuk mencegah server apache menulis direktori konten dan file-nya.

Pertimbangkan untuk mengubah kepemilikan kepada pengguna yang perlu mengubah konten. Tambahkan mereka ke grup www-data. Anda juga dapat mengatur grup sticky bit pada direktori / var / www. Jika Anda adalah pengguna, Anda dapat menjalankan perintah ini.

sudo chmod g+s /var/www
sudo usermod -aG www-data $LOGNAME
sudo chown -R $LOGNAME /var/www

Keluar dan masuk lagi untuk mendapatkan akses ke grup data www. Anda kemudian dapat mengubah konten.
