AutoHotKey dan Windows 8: Perintah Alt + Tab tidak berfungsi

Saya tidak bisa membuatnya berfungsi baik saya menggunakan

AltTab

atau

Kirim {Alt down} {tab} {Alt up}

Saya dapat memverifikasi bahwa perintah lain berfungsi karena saya bisa mendapatkan hotkey saya untuk menampilkan MessageBox. Sepertinya W8 telah mengubah cukup banyak hal untuk merusak AHK Alt + Tab kait.

Mencari solusi. Saya mencoba untuk memetakan LCtrl + Tab untuk Alt + Tab (dan versi Shift dari mereka).

Menyetel UIAccess = true di manifes tersemat yang dapat dieksekusi tampaknya menyelesaikan masalah ini. Eksekusi juga harus berada di lokasi tepercaya, seperti folder Program Files. Untuk informasi lebih lanjut dan skrip yang membuat modifikasi yang diperlukan untuk Anda, temukan EnableUIAccess di forum AutoHotkey.

Terima kasih kepada Cliff Smolinsky untuk posting berikut ini di Pelacak masalah sinergi :

Sejumlah perubahan keamanan telah diterapkan di Windows 8 yang mencegah program yang bukan aplikasi Metro untuk memasukkan kunci yang akan menghapus Anda dari lingkungan Metro. Cara kerjanya adalah pintasan keyboard diabaikan jika tidak berasal dari keyboard fisik atau program yang memiliki izin yang sesuai. [...] Untuk mendapatkan izin yang tepat, program harus dibangun dengan UIAccess

Saya tidak dapat mengonfirmasi ini melalui sumber resmi apa pun, selain dari komentar tidak jelas berikut di MSDN:

Aplikasi aksesibilitas dapat menggunakan SendInput untuk memasukkan penekanan tombol yang sesuai dengan tombol pintasan peluncuran aplikasi yang ditangani oleh shell. Fungsionalitas ini tidak dijamin berfungsi untuk jenis aplikasi lain.

Jika Anda menggunakan AutoHotkey (Basic), basis kode belum diperbarui sejak September 2009. Jelas, setiap perubahan mendasar di Windows 8 belum ditangani. AutoHotkey_L jauh lebih mutakhir, yang terakhir diperbarui bulan ini. Namun, melihat file changelog , lebih lanjut dukungan Windows 8 minimal.

Jika skrip Anda masih tidak berfungsi bahkan setelah menginstal AutoHotkey_L, Anda dapat mencoba yang lain Kirim perintah:

  • SendRaw
  • SendInput
  • SendPlay
  • SendEvent

Jika tidak, Anda hanya perlu bersabar dan menunggu pengembang bekerja menambahkan dukungan untuk Windows 8.
