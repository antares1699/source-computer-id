Penggunaan operator Regex “ATAU” dengan perintah find

Ada begitu banyak pertanyaan ini di internet, tetapi saya tidak dapat memecahkan masalah sederhana ini.

Saya memiliki direktori dengan banyak gambar, tetapi saya hanya ingin menyalin sebagian dari mereka yang diidentifikasi oleh rentang angka. Setiap gambar memiliki format: "[random characters][capture number].BMP"

Misalnya: IZ000561.BMP

Saya menggunakan ini sehubungan dengan opsi find dan -regex. Ekspresi reguler yang saya pikir untuk digunakan ternyata sederhana:

.*(26[2-7]|27[0-2]).*

jika saya ingin mencocokkan gambar dengan tag [262-267], [270-272]. Namun pendekatan ini gagal. Saya pergi ke internet Regex penguji dan semua cocok seperti yang diharapkan menggunakan ekspresi ini. Pasti mesin regex find mengharapkan format yang berbeda untuk jenis pemfilteran ini.

Perintah lengkap yang saya gunakan:

find /path/to/images/ -regex ".*(26[2-7]|27[0-2]).*" -exec echo {} \;

Apa ekspresi yang valid untuk apa yang saya coba lakukan?

Membaca halaman manual find memberikan sedikit informasi berguna:

Ekspresi reguler yang dipahami oleh find secara default adalah Emacs Ekspresi Reguler.

Jadi, lihat sintaks dari Emacs Regex

Tampaknya semua karakter khusus harus di-escape dengan \. Dengan mempertimbangkan hal ini, hasil yang diharapkan diperoleh setelah mengubah ekspresi reguler menjadi berikut:

find /path/to/images/ -regex ".*\(26[2-7]\|27[0-2]\).*" -exec echo {} \;

Saya berkonsultasi dengan halaman manual sebelum memposting pertanyaan ini, tetapi tampaknya tidak cukup teliti.

Saya menyadari Anda memposting jawaban Anda sendiri, tetapi jika Anda ingin menggunakan sintaks REGEX, Anda dapat mengubah jenis REGEX menggunakan perintah ini:

-regextype TYPE

Jadi saya yakin untuk membuat Anda berhasil:

find /path/to/images/ -regextype sed -regex '.*(26[2-7]|27[0-2]).*' -exec echo {} \;

mungkin berhasil.

Selain itu, hanya catatan, regex Anda akan cocok dengan apa pun yang memiliki 262-267 atau 270-272 di dalamnya. Itu berarti akan menangkap IZ262561.BMP, serta IZ134267.JPG, dan ASDNASDNOANFOIAJFOIJDIO000262.EXE misalnya. Saya tidak yakin file apa yang sedang Anda kerjakan, tetapi Anda mungkin perlu mempersempit regex Anda karena cukup longgar pada apa yang akan diterima.

Ada juga operator -or (atau -o) ke perintah find. Anda dapat membagi ekspresi Anda menjadi dua subekspresi:

find /path/to/images/ -regex ".*(26[2-7]).*" -o -regex ".*(27[0-2]).*" -exec echo {} \;

-o sesuai dengan POSIX, sedangkan -or tidak:

Sesuai halaman manual find:

   expr1 -o expr2
          Or; expr2 is not evaluated if expr1 is true.

   expr1 -or expr2
          Same as expr1 -o expr2, but not POSIX compliant.
