Mengapa Windows File Explorer membutuhkan lebih banyak waktu untuk membuka dari bilah tugas daripada dari makro keyboard?

Jika saya membuka jendela File Explorer dengan mengkliknya di bilah tugas (baik klik kiri jika tidak ada yang terbuka atau klik tengah jika sudah terbuka), dibutuhkan sekitar satu atau dua detik hingga jendela terbuka. Ini tidak banyak, tetapi ini adalah perlambatan yang nyata.

Saya menggunakan Windows 10 dan File Explorer diatur untuk membuka PC Ini, bukan Akses Cepat. Saya menggunakan laptop i5 dengan SSD. Saya telah memutakhirkan dari penginstalan Windows 8.1, yang memiliki perilaku yang sama sejak awal.

Jika saya menggunakan makro keyboard untuk keyboard Logitech G710 + yang disetel untuk membuka "Komputer Saya", yang memiliki efek membuka PC Ini, jendela akan langsung muncul setelah saya menekan tombol yang ditetapkan. Saya tidak yakin apa yang makro ini lakukan secara internal, karena ini adalah preset yang baru saja saya pilih di perangkat lunak Logitech.

Bisakah saya membuat File Explorer terbuka lebih cepat saat mengkliknya?

Mengapa ada perbedaan mencolok antara kedua kecepatan pengaktifan?

Dalam mode aman sepertinya baik-baik saja. Sepertinya ada masalah dengan kartu grafis nvidia. Saya menemukan solusi, masalah hilang jika Anda menonaktifkan opsi nvidia menu kontekstual.

Buka Kontrol nVidia (klik kanan pada desktop -> panel kontrol nvidia). Kemudian di menu bar pilih Desktop dan UNPILIH "Add Desktop Context Menu" dan "Add run with Graphics Processor". Setelah ini, bahkan tanpa reboot, masalah tampaknya hilang dan file explorer jauh lebih cepat dari taskbar ...

Sumber: http://www.neowin.net/forum/topic/1139638-file-explorer-slow-to-open-from-taskbar/?do=findComment&comment=596672310
