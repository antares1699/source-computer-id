grep tidak berfungsi dengan benar

Saya telah dengan senang hati menggunakan grep selama bertahun-tahun tanpa masalah apa pun, tetapi sejak hari ini grep berhenti berfungsi. Selama satu jam terakhir saya mencoba ini dan itu, tetapi cukup sudah, saya memposting bajingan itu di sini:

Pada perintah paling sederhana seperti

grep 'aaa' file.txt

Saya mendapatkan ini: grep: aaa: No such file or directory

Jadi grep tidak menafsirkan argumen pertama sebagai pola sebagaimana mestinya, tetapi memperlakukannya sebagai jalur.

Kamu mungkin memiliki alias untuk grep. Sebuah contoh:

$ grep aaa qq.s

$ alias grep='grep zzz'

$ grep aaa qq.s
grep: aaa: No such file or directory

Saya memiliki masalah yang sama. Jika Anda mengubah kutipan tunggal menjadi tanda kutip ganda di baris GREP_OPTIONS .bash_profile Anda, Anda akan mengatasi masalah tersebut. Ini bekerja:

 export GREP_OPTIONS="--color=auto"

Ekspor yang sama dengan tanda kutip tunggal tidak berfungsi. Ini ada hubungannya dengan cara bash menafsirkan tanda kutip tunggal vs tanda kutip ganda. Ini mungkin memberikan panduan tambahan: http://www.gnu.org/software/bash/manual/bashref.html#Double-Quotes
