Sepengetahuan saya, OpenWRT tabel perangkat keras yang didukung (atau versi lama ) adalah tempat untuk memulai. Ada juga a Versi DD-WRT jika Anda ingin membandingkan. Tabel ini mencantumkan perangkat yang didukung dan merinci kemampuan perangkat kerasnya, termasuk platform CPU, frekuensi CPU, jumlah RAM, dan kapasitas memori flash.

Sebagian besar perangkat dalam tabel ini lebih tua dan akan memiliki kecepatan CPU ~ 200MHz. Melihat sekilas tabel OpenWRT, beberapa perangkat yang lebih baru menganggap saya memiliki potensi. Ini adalah tampilan sepintas, dan sulit untuk mengatakan bagaimana satu CPU bertumpuk dengan yang lain tanpa penelitian lebih lanjut.

  • D-Link DIR-825 , CPU 680MHz (Atheros AR7161), flash 8MB, RAM 64MB
  • Linksys WRT350N v2 , CPU 500MHz (Marvell 88F5181L), flash 8MB, RAM 32MB
  • Netgear WNR854T , CPU 500MHz (Marvell 88F5180NB1), flash 8MB, RAM 32MB
  • RouterStation Ubiquiti dan RouterStation Pro , 680 MHz CPU (Atheros AR7161), flash 16MB, RAM 64MB (128MB pada Pro) (mungkin harus menambahkan miniPCIe NIC nirkabel Anda sendiri)

Selain itu, bagian Work-in-Progress dari tabel mencantumkan beberapa perangkat menarik lainnya:

  • Linksys WRT400N , 680MHz CPU (Atheros AR7161), 8MB flash, 32MB RAM (hati-hati untuk info template yang tidak berguna di tautan)
  • Linksys WRT610N, 300MHz CPU (Broadcom BCM4705), flash 8MB, RAM 64MB ( WIP )
  • Mikrotik RouterBoard RB-450G , CPU 680MHz (Atheros AR7161), flash 512MB, RAM 256 MB (!! ... awww, tanpa nirkabel ...)
  • Netgear WNDR3700 , CPU 680MHz (Atheros AR7161), flash 8MB, RAM 64MB

Itu tampaknya menjadi perangkat paling besar yang terdaftar, tetapi saya tidak terbiasa dengan salah satu dari mereka. Ini adalah awal untuk penelitian Anda. Dari sini, Anda dapat melihat kemampuan CPU Atheros yang paling cepat digunakan perangkat ini, dan melihat perbandingannya dengan CPU Marvell yang digunakan oleh perangkat lain.

Router wifi OpenWRT tercepat dalam hal apa?

Waktu mulai? Seberapa cepat ia melatih sinyal di telepon? Mengalihkan lalu lintas? Kekuatan pemrosesan? Mengenkripsi data untuk VPN?

Jika Anda seorang teknisi internet yang berpindah dari pengujian koneksi ADSL dari rumah ke rumah, maka Anda tidak ingin sesuatu yang melatih ke jalur selama 5 menit.

Jika Anda menggunakannya untuk menyalin file besar antar komputer di jaringan Anda, Anda mungkin menginginkan sesuatu yang dapat mengalihkan lalu lintas dengan cepat.

Apakah Anda menginginkan OpenWRT sehingga Anda dapat menjalankan server web di router Anda? Dalam hal ini Anda mungkin menginginkan kekuatan pemrosesan.

Apakah router ini akan menjadi bagian dari hub dan spoke VPN dan harus mengenkripsi lalu lintas? Dalam hal ini Anda mungkin menginginkan sesuatu dengan perangkat keras yang dapat mempercepat enkripsi.
