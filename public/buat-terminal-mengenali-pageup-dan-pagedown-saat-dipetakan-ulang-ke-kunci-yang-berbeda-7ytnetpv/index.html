Buat Terminal mengenali PageUp dan PageDown saat dipetakan ulang ke kunci yang berbeda

Saya memahami Terminal secara default tidak mendukung PgDn dan PgUp . saya tidak punya PgDn dan PgUp pada keyboard saya, tetapi telah memetakannya kembali ke tombol yang berbeda saat ini dan karenanya Bergeser + PgDn tidak bekerja. Saya juga sudah mencoba Fn + Naik . Bagaimana saya dapat membuat Terminal mengenali kunci paging saya yang dipetakan ulang?

(komentar: Anda telah menemukan XKB! selamat, ini adalah lubang kelinci yang sangat besar untuk jatuh. perhatikan solusi Anda akan terhapus saat berikutnya paket xkb-data ubuntu memperbarui file itu, dan akan mempengaruhi keyboard lain yang Anda pasang yang menggunakan konfigurasi yang sama.)

XKB mengikuti model yang disebut RMLVO - Rules, Model, Layout, Variant, Options. Solusi Anda mengubah file Model sistem, tetapi solusi yang lebih baik adalah menemukan atau membuat Opsi untuk menambahkan penggantian Anda. (evdev adalah Aturan default untuk sebagian besar distribusi, dan sebagian besar keyboard eksternal akan menggunakan model pc, meskipun model internal laptop mungkin memiliki variasi tertentu yang sudah dekat dengan kasus penggunaan Anda. Keyboard QWERTY AS standar akan menggunakan tata letak us tanpa varian, meskipun Anda mungkin memilih varian intl atau altgr-intl jika Anda bekerja dengan bahasa lain.)

Opsi adalah pengesampingan khusus untuk fungsionalitas dasar. Jika Anda ingin mematikan CapsLock karena tombol itu payah, sudah ada opsi untuk itu (caps:none), atau Anda dapat mengaturnya menjadi tombol Escape tambahan (caps:escape). Anda dapat melihat penggantian spesifik di /usr/share/X11/xkb/symbols/caps; setiap bait adalah opsi berbeda yang dapat Anda aktifkan saat runtime.

Jika Anda melakukan setxkbmap -print Anda harus mendapatkan daftar pengaturan XKB Anda saat ini:

$ setxkbmap -print
xkb_keymap {
    xkb_keycodes  { include "evdev+aliases(qwerty)" };
    xkb_types     { include "complete"  };
    xkb_compat    { include "complete"};
    xkb_symbols   { include "pc+us+inet(evdev)" };
    xkb_geometry  { include "pc(pc105)" };
};

Buat file timpa Anda:

$ mkdir -p .config/xkb/symbols
$ touch .config/xkb/symbols/pgupdown

Sekarang edit file baru dengan editor favorit Anda dan masukkan yang berikut ini:

// mapping PgUp and PgDown to other keys
partial alphanumeric_keys
xkb_symbols "rwinmenu" {
    key <RWIN> { [ Prior     ]   };
    key <MENU> { [ Next      ]   };
};

// same but switched
partial alphanumeric_keys
xkb_symbols "menurwin" {
    key <MENU> { [ Prior     ]   };
    key <RWIN> { [ Next      ]   };
};

Sekarang buat peta kunci dasar:

$ setxkbmap -print > ~/.config/xkb/mymap

Edit file itu, dan tambahkan opsi baru Anda ke dalam daftar simbol dalam format +optionfile(function):

xkb_keymap {
    xkb_keycodes  { include "evdev+aliases(qwerty)" };
    xkb_types     { include "complete" };
    xkb_compat    { include "complete"};
    xkb_symbols   { include "pc+us+inet(evdev)+pgupdown(rwinmenu)" };
    xkb_geometry  { include "pc(pc105)" };
};

Jika nanti Anda memutuskan bahwa Anda lebih suka PgUp pada tombol Menu, cukup ganti rwinmenu dengan menurwin di sana dan Anda siap melakukannya.

Sekarang Anda dapat memuat file itu dengan xkbcomp:

$ xkbcomp -I$HOME/.config/xkb $HOME/.config/xkb/mymap $DISPLAY

Luncurkan xev (dari paket x11-utils) dan tekan tombol untuk memastikan perubahan dimuat:

# sed to scrape out non-kbd events; xev is pretty spammy
$ xev | sed -ne '/^KeyPress/,/^$/p'

Jika Anda sudah mendapatkan semuanya di tempat yang tepat, tombol Menu Anda harus mengirim acara Berikutnya, dan kunci Win_R mengirim Sebelumnya. Jika berhasil, Anda dapat menambahkan perintah xkbcomp di atas ke .xinitrc atau .xprofile Anda sehingga perintah tersebut berjalan saat Anda masuk atau memulai X.

... kecuali Anda menjalankan Gnome. (dan / atau mungkin KDE.) Kemudian kita melangkah lebih jauh ke lubang kelinci, dan mencari cara untuk memasukkan opsi baru ini ke dalam file sistem sehingga DE dapat mengambilnya sebagai opsi standar. Pada dasarnya, itu akan melibatkan

  1. menempatkan file opsi Anda di /usr/share/X11/xkb/symbols/ (inilah mengapa pgupdown adalah pilihan opsi yang baik, karena tidak ada file simbol yang sudah ada sebelumnya dengan nama itu)
  2. lalu menambahkan baris pada $ setxkbmap -print > ~/.config/xkb/mymap (untuk memetakan format optionsfile:function dari baris perintah ke +optionsfile(function) yang digunakan oleh xkbcomp)
  3. dan terakhir menyetel DE untuk menggunakan tata letak dan opsi pilihan Anda.

Langkah 2: Jika Anda perlu pergi ke rute ini, Anda mungkin perlu melakukan langkah 2 lagi setelah peningkatan terkait xkb.

# in /usr/share/X11/xkb/rules/evdev
# find the line that starts "! option = symbols" and add new lines:
  pgupdown:rwinmenu = +pgupdown(rwinmenu)
  pgupdown:menurwin = +pgupdown(menurwin)

# in /usr/share/X11/xkb/rules/evdev.lst
# find the line that starts "! option" and add a new line:
  pgupdown:rwinmenu          PgUp on RWin, PgDown on Menu
  pgupdown:menurwin          PgUp on Menu, PgDown on RWin

# in /usr/share/X11/xkb/rules/evdev.xml
# .... ummm .... ¯\_(ツ)_/¯  .... sorry! ....

Langkah 3: Periksa apakah localectl tersedia di sistem Anda. localectl dapat mengkonfigurasi keyboard konsol virtual serta mengatur keyboard X11 default sistem. Karena kedua opsi disiapkan di file aturan, Anda dapat menggunakan rwinmenu atau menurwin dalam perintah ini.

$ localectl set-x11-keymap us pc "" pgupdown:rwinmenu
#  format               [layout] [model] [variant] [option1,...,optionN]

Untuk Gnome, Anda harus menggunakan dconf-editor atau gsettings:

# set layout/variant
$ gsettings set org.gnome.desktop.input-sources sources "[('xkb', 'us')]"

# set options
$ gsettings set org.gnome.desktop.input-sources xkb-options "['pgupdown:rwinmenu']"

Pilihan lain untuk Gnome mungkin menonaktifkan pengaturan keyboard-nya; ini akan memungkinkan opsi localectl atau opsi user-config tetap berlaku:

$ gsettings set org.gnome.settings-daemon.plugins.keyboard active false

Penyelesaian masalah: Jika perubahan Anda tampaknya tidak diambil dan Anda yakin telah mendapatkan format file yang benar, hapus file *.xkm dari /var/lib/xkb/. Ini adalah peta kunci yang telah dikompilasi dalam cache dan mungkin menghalangi Anda.
