Dapatkah Raspberry PI menangkap input HDMI

Apakah mungkin untuk menangkap input HDMI dengan papan Raspberry PI (atau alternatif)? Tujuan saya adalah menangkap data HDMI yang dikirim dari TV saya dan bekerja dengannya dari papan Raspberry PI.

Menangkap HDMI dimungkinkan tetapi membutuhkan perangkat keras khusus yang disebut digitiser atau perangkat penangkap video. Ini tidak akan bekerja dengan HDMI keluaran . Ini tidak akan berfungsi dengan Raspberry Pi kecuali Anda menambahkan perangkat keras ini.

Karena itu, Pi memang memiliki port USB dan Anda dapat dengan mudah menemukan HDMI to USB video capture dongle berbasis USB. ( Google itu )

Pastikan bahwa perangkat apa pun yang akan dibeli dapat bekerja dengan USB2 (masalah daya dan bandwidth!) Dan ada dukungan driver untuk itu.

Menindaklanjuti rekaman pertarungan Keltari dalam resolusi penuh:

HDMI 1.0 menentukan hingga 3,96 Gbit / dtk bandwidth video (1080p / 60 Hz atau UXGA) dan audio 8 saluran LPCM / 192 kHz / 24-bit. (Sumber: Wikipedia )

Bandingkan ini dengan bandwidth maks teoretis USB2. yaitu 480mbit / detik.

Bahkan dengan kompresi pintar pada digitiser Anda, Anda cenderung mengalami masalah kecuali Anda benar-benar menurunkan resolusi. Dan untuk memperburuk keadaan model rPi B + (dan saya pikir juga model Pi lainnya) hanya memiliki satu port USB. Di B + saya, port tunggal itu (dan bandwidth-nya) diumpankan ke chip yang mengubahnya menjadi 4x USB dan Ethernet. Namun hasil maksimal masih terbatas. Jika Anda menulis ke harddisk USB eksternal (kemungkinan besar karena rekaman tidak muat di kartu SD) maka Anda harus berbagi bandwidth ini.
