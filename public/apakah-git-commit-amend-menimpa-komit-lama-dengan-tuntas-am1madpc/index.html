Apakah git commit --amend menimpa komit lama dengan tuntas?

Ke repositori lokal saya secara tidak sengaja memasukkan file dengan kata sandi di dalamnya yang tidak boleh dipublikasikan (itu ada di sana untuk tujuan pengujian yang malas ...).

Saya menyimpan file itu lagi tanpa kata sandi dan menggunakan git commit --amend untuk memasukkan file itu lagi. Apakah kata sandi sekarang hilang untuk selamanya atau masih dapat ditemukan di suatu tempat di repositori?

Jika Anda belum menjalankan git push ke repositori jarak jauh sebelum melakukan perubahan, maka komit tidak tersedia di repositori jarak jauh itu. Itu juga tidak akan didorong di masa depan.

Namun, Anda lokal repositori masih berisi komit lama yang dapat Anda lihat dengan menjalankan git reflog.

Komit aktual (lama) kemudian dapat ditampilkan dengan perintah seperti:

git show HEAD@{1}

(dengan asumsi bahwa Anda melakukan sesuatu dan kemudian mengubahnya tanpa komit lebih lanjut / peralihan cabang)

Untuk menyingkirkan komit tersebut secara lokal, Anda harus melihat git gc. Ketika Anda menjalankannya sebagaimana adanya, ini akan menghapus komit lama yang menggantung dengan usia tertentu.
