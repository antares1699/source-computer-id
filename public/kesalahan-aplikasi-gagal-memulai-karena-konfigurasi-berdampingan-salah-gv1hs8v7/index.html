Ini terjadi ketika Visual C ++ Runtime yang diperlukan hilang. Memulai Prompt perintah dengan hak admin dan jalankan perintah ini:

SxsTrace Trace -logfile:SxsTrace.etl

Sekarang jalankan Glary Utilities dan kembali ke command prompt tekan ENTER untuk menghasilkan SxsTrace.etl. Sekarang ketik ini:

sxstrace Parse -logfile:SxSTrace.etl -outfile:SxSTrace.txt

Sekarang buka pelacakan SxSTrace.txt dan lihat runtime VC ++ mana yang hilang.

Bagikan

Saya menghadapi masalah ini saat mencoba meluncurkan alat winMerge diff. Saya memeriksa solusinya sini :

Metode 1:

Unduh dan instal Microsoft Visual C ++ 2013 Redistributable Paket dari bawah diberikan tautan dan periksa apakah itu membantu:

Paket Visual C ++ yang Dapat Didistribusikan Ulang untuk Visual Studio 2013

http://www.microsoft.com/en-in/download/details.aspx?id=40784

Tetapi ketika saya menjalankan penginstal, saya menyadari bahwa Microsoft Visual C++ 2013 Redistributable Package telah diinstal di komputer saya dan sebagai gantinya ia meminta perbaikan. Untuk amannya, saya hanya menjalankan wizard perbaikan sekali.

Kemudian saya menghapus dan menginstal ulang alat WinMerge seperti yang disebutkan di ini video Youtube. Setelah itu saya reboot komputer saya lalu akhirnya winMerge bekerja dengan lancar.
