hostapd: jalankan perintah ketika ada koneksi baru yang dibuat

Saya perlu memulai daemon sewenang-wenang ketika setidaknya ada satu koneksi jaringan WiFi (dari klien). Dan hentikan daemon sewenang-wenang saat tidak ada lagi koneksi ke AP ini. Bagaimana melakukannya secara efisien? Untuk saat ini saya mempertimbangkan hasil polling hostapd_cli all_sta, tetapi tampaknya aneh. Saya yakin pasti ada beberapa peristiwa sistem yang dapat dikaitkan.

Seperti yang dikatakan @beemaster, jalankan ini:

hostapd_cli -a '/onHostapdChange.sh'

Dengan script: onHostapdChange.sh

#!/bin/bash
if [[ $2 == "AP-STA-CONNECTED" ]]
then
  echo "someone has connected with mac id $3 on $1"
fi

if [[ $2 == "AP-STA-DISCONNECTED" ]]
then
  echo "someone has disconnected with mac id $3 on $1"
fi

Untuk saat ini saya telah menemukan solusi berikut:

hostapd_cli -a 'some_script'

Di mana skrip menerima nama antarmuka jaringan dan jenis peristiwa sebagai argumen. Ini harus dimulai setelah daemon hostapd. Juga membutuhkan ctrl_interface untuk disetel pada hostapd.conf.
