cat "file input adalah file output" ketika mencoba menggabungkan beberapa file menjadi satu

Rencananya adalah mengumpulkan semua baris sumber Java menjadi satu file, secara rekursif:

$ find . -name '*.java' | xargs cat >> all.java

Tapi ada kesalahan:

cat: ./all.java: input file is output file

File all.java tidak ada sebelum perintah ini, jadi saya pikir mungkin xargs mencoba menjalankan cat >> all.java file1 file2 file3 ...

Pertama-tama, Anda dapat mengabaikan kesalahan itu dengan aman. Perintah akan berjalan dengan sukses dan itu akan, dengan benar, mengabaikan all.java itu sendiri. Ini hanya memberi tahu Anda bahwa itu terjadi.

Bagaimanapun, untuk menghindari kesalahan, Anda dapat menggunakan opsi tee dan find's exec:

$ find . -name '*.java' -exec cat {} + | tee all.java

Dari man find:

   -exec command ;
          Execute  command;  true  if 0 status is returned.  All following
          arguments to find are taken to be arguments to the command until
          an  argument  consisting of `;' is encountered.  The string `{}'
          is replaced by the current file name being processed  everywhere
          it occurs in the arguments to the command, not just in arguments
          where it is alone, as in some versions of find. 

   -exec command {} +
          This  variant  of the -exec action runs the specified command on
          the selected files, but the command line is built  by  appending
          each  selected file name at the end; the total number of invoca‐
          tions of the command will  be  much  less  than  the  number  of
          matched  files.

Jadi, Anda dapat menggunakan -exec untuk memberi tahu find untuk menjalankan perintah pada setiap hasilnya. {} diganti dengan nama file / direktori sebenarnya yang ditemukan. + hanyalah penanda yang memberi tahu find bahwa perintah berakhir di sini. Saya menggunakannya daripada \; karena ini akan menjalankan lebih sedikit perintah karena akan mencoba menggabungkannya menjadi sesedikit mungkin proses.

Atau gunakan find ! untuk mengecualikan all.java:

$ find . -name '*.java' ! -name all.java -exec cat {} +  >> all.java

Atau Globbing :

$ shopt -s globstar
$ cat **/*.java > all.java
Bagikan

Masalahnya adalah itu pertama file all.java dibuat, kemudian find menemukannya dan memasukkannya ke cat, yang tidak menyukainya karena dikeluarkan di sana.

Solusi yang mungkin:

$ find . -name '*.java' | xargs cat >> all.j

Atau:

$ find . -name '*.java' | xargs cat >> ../all.java
