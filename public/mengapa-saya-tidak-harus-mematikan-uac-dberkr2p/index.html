Mengapa saya tidak harus mematikan UAC?

Saya telah membaca di beberapa tempat bahwa Anda tidak boleh mematikan Kontrol Akun Pengguna Windows (UAC), tetapi ada panduan di seluruh internet yang menjelaskan cara melakukannya, dan beberapa orang mengatakan saya tidak perlu khawatir.

Saya bahkan tidak benar-benar tahu mengapa UAC ada di sana, dan itu mengganggu saya ketika muncul, jadi mengapa harus dinyalakan?

Saya tidak bertanya apakah risikonya lebih besar daripada manfaatnya, atau sebaliknya. Saya juga tidak bertanya apakah saya pribadi harus menyalakan atau mematikannya:

Cara lain untuk menanyakan pertanyaan ini:

  • Apa potensi risiko mematikannya?
  • Apa manfaat membiarkannya aktif?

Anda tidak boleh mematikannya karena jika Anda mematikannya tidak akan lagi meminta izin Anda ketika sebuah program ingin melakukan hal-hal berbahaya.

Misalnya. Saya membuat game. Anda mendownloadnya dan menjalankannya. Saya memutuskan bahwa jika Anda kalah, saya akan menghapus harddisk Anda.

Jika Anda menjalankan ini tanpa UAC maka game akan berjalan dan saya dapat menghapus disk (atau melakukan hal lain).
Jika UAC dibiarkan hidup maka permainan akan tetap berjalan, namun meminta ijin terlebih dahulu sebelum melakukan hal yang berbahaya. Ini bisa menjengkelkan seperti "Saya ingin menginstal directX untuk yang umtieth kali", tetapi juga akan memperingatkan untuk hal-hal yang lebih serius.

Oleh karena itu, tindakan terbaik adalah membiarkan keamanan UAC tetap aktif.

UAC seperti sistem deteksi intrusi tertanam di Windows. Ketika sebuah program perlu melakukan beberapa operasi dengan hak istimewa tinggi (yaitu Memodifikasi file sistem Windows yang sangat penting), UAC mencegat permintaan dan memberi pengguna prompt untuk memutuskan apakah program harus diizinkan atau tidak. itu akan ada gunanya bagimu. Meskipun dapat mengganggu pada tingkat tertentu, ini memastikan bahwa pengguna mengetahui program apa yang sedang berjalan di komputernya dan operasi apa yang mereka lakukan.
