Bagaimana saya bisa memanggil fungsi dalam skrip bash shell

Saya hanya ingin tahu perbedaan yang memanggil fungsi antara $(one_function) dan one_function dalam skrip bash shell.

Ketika saya menetapkan variabel PS1 pada ~/.bashrc, saya tidak dapat menjalankan fungsi pada one_func ex:

export PS1="\n\[\e[31m\] \$(one_func)  # it works 

export PS1="\n\[\e[31m\] one_func      # it doesn't work

Berlawanan dengan bagaimana variabel diakses, fungsi dipanggil oleh nama tanpa mendahului nama dengan '$'.

Anda mungkin bingung tentang bagaimana pada baris perintah, Anda dapat mendefinisikan fungsi dan memanggil fungsi itu dengan nama, tetapi di PS1 Anda, Anda harus meletakkan perintah dalam tanda kurung diawali dengan '\ $'. Mengapit nama fungsi dalam '$ (' dan ')' menyebabkan seluruh '$ (function)' diganti dengan apa pun keluaran standar dari fungsi itu. Menempatkan garis miring terbalik di depannya menyebabkan shell Anda mengevaluasi / menjalankan fungsi itu setiap kali ingin mengeluarkan $ PS1. Jika Anda telah meninggalkan garis miring terbalik, fungsi tersebut akan dipanggil hanya sekali, saat Anda pertama kali mendefinisikan PS1, dan apa pun output dari fungsi tersebut yang pertama kali, akan selamanya ada dalam prompt PS1 Anda sejak saat itu.

Ketika Anda memanggil $(one_func), itu akan menjalankan fungsi, dan mengembalikan output. Jadi jika Anda berkata, misalnya:

var=$(ls)

itu akan menyimpan output dari perintah ls (yaitu daftar file di direktori saat ini) ke dalam variabel $var. Sedangkan perintah:

var=ls

hanya akan menyetel nilai $var menjadi "ls".

Ngomong-ngomong, memanggil fungsi di bash bekerja dengan cara yang sama seperti menjalankan perintah.
