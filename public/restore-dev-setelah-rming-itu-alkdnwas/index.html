restore / dev setelah `rm`ing itu

selama instalasi gentoo dari LiveVD, ada satu langkah untuk membuat direktori dev untuk digunakan nanti di chroot target untuk instalasi.

itu dilakukan seperti itu

mount --rbind /dev /mnt/gentoo/dev

di tengah instalasi saya memutuskan untuk memulai kembali ... jadi saya baru saja melakukannya

rm -fr /mnt/gentoo/*

tidak berpikir apa-apa tentang itu ("hanya akan membongkar bola tar stage3 lagi") saya menghapus sebagian besar file perangkat dari / dev saya yang sebenarnya karena yang ada di chroot hanya akan diikat ke perangkat asli.

sekarang, solusi mudahnya adalah: reboot. Saya tahu bahwa jika saya reboot mereka akan ada lagi. Tapi ini adalah sebuah livev tanpa ketekunan. Jadi jika saya reboot saya akan kehilangan ~ 1 jam persiapan dengan firmware dan yang lainnya,

untuk saat ini saya akan reboot. Tetapi saya ingin belajar bagaimana memulihkan a / dev jika kasus seperti ini terjadi lagi (dan karena sekarang saya perlu tahu bahkan jika saya tidak akan pernah menggunakannya lagi)

memperbarui: baru ingat tentang LFS. membaca linux dari dokumen awal, saya pikir perbaikan paling sederhana adalah menelepon

 /etc/init.d/udev restart

sebagian besar file tampaknya ada di sana sekarang.

Pembaruan 2014: Versi udev terbaru tidak membuat node perangkat sama sekali, dan mengandalkan devtmpf kernel untuk melakukannya. Oleh karena itu, jawaban lama akan berhenti berfungsi jika distribusi Anda memiliki udev terbaru. Dengan devtmpfs, satu-satunya metode otomatis mungkin adalah reboot.

/ dev dikelola oleh udev di sebagian besar distro Linux, jadi

udevadm trigger

harus membantu.

Memulai ulang udev sepenuhnya juga akan berfungsi, tetapi itu tidak perlu, dan perintahnya sangat bergantung pada distro yang digunakan.

CATATAN: Anda harus membuat beberapa item secara manual sebelum perintah itu berfungsi. / dev / {nol, null}
