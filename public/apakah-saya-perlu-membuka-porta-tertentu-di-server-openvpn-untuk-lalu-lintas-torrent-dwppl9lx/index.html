Apakah saya perlu membuka porta tertentu di server OpenVPN untuk lalu lintas torrent

Jika saya terhubung ke server openvpn pada port 1194 dari Tunnelblick pada Mac (OSX 10.10), bagaimana saya dapat memastikan bahwa port yang digunakan oleh Transmisi (misalnya 66887) akan terbuka untuk koneksi masuk?

Setup saat ini:

Terhubung ke Wifi Hotspot (di luar kendali saya, tidak ada akses ke pengaturan router) Terhubung ke server OpenVPN (Saya memiliki kendali penuh atas konfigurasi)

Transmisi 'berfungsi' tetapi lambat dan port ditampilkan sebagai tertutup dalam preferensi jaringan Transmisi. Sambungan VPN openvpn bekerja secara normal untuk semua lalu lintas http standar.

Openvpn berjalan di server Ubuntu jarak jauh. Saya mencoba menggunakan ufw untuk membuka port di atas tetapi ini tidak berpengaruh dan saya menduga bahwa diperlukan konfigurasi jaringan yang lebih dalam.

versi openvpn adalah OpenVPN 2.3.2 x86_64.

Apakah mungkin untuk meneruskan port 66887 dalam skenario ini atau apakah saya memerlukan akses ke router LAN lokal agar ini berfungsi?

Saya mencoba mengubah alamat mengikat - BindAddressIPv4 - untuk Transmisi melalui file preferensi plist tetapi melakukan ini secara tidak benar atau tidak ada bedanya.

Terkejut tidak ada yang memperhatikan, tetapi 66887 bukanlah nomor port yang valid.

https://stackoverflow.com/questions/113224/what-is-the-largest-tcp-ip-network-port-number-allowable-for-ipv4#113228

Nomor port adalah bilangan bulat 16-bit unsigned, jadi 65535.

Kisaran yang valid untuk port adalah 0-65535.
Karena Anda menentukan porta 66887 yang tidak valid, sebagian besar sistem operasi akan memotongnya menjadi 1351:

[root@f ~]# tcpdump -qnn host 8.8.8.8 & telnet 8.8.8.8 66887
[1] 4054
Trying 8.8.8.8...
tcpdump: verbose output suppressed, use -v or -vv for full protocol decode
listening on eth0, link-type EN10MB (Ethernet), capture size 65535 bytes
15:30:56.175482 IP 10.0.0.42.60280 > 8.8.8.8.1351: tcp 0

Atau di C:

[root@f ~]# cat > 16.c << EOF
> #include <stdio.h>
> #include <stdint.h>
> int main(void) {
>  uint16_t port=66887;
>  printf("%d\n",port);
>  return 0;
> }
> EOF
[root@f ~]# gcc -o 16 16.c
16.c: In function ‘main’:
16.c:4: warning: large integer implicitly truncated to unsigned type
[root@f ~]# ./16
1351

Anda perlu meneruskan port di server openvpn Anda.

Saya tidak tahu perintah ufw, tetapi menggunakan iptables, perintahnya adalah (dengan 10.8.0.6 IP mesin Anda saat terhubung ke VPN dan 66887 port untuk meneruskan):

iptables -t nat -A PREROUTING -p tcp --dport 66887 -j DNAT --to 10.8.0.6:66887
iptables -t nat -A PREROUTING -p udp --dport 66887 -j DNAT --to 10.8.0.6:66887
