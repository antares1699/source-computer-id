Tar - file split

saya menggunakan perintah berikut untuk membuat cadangan dan file sql:

tar -zcvf "$BACKUP_DST/$FILE_NAME.tgz" "$BACKUP_DST/$FILE_NAME.sql"

Saya ingin memastikan file yang dikompresi tidak akan lebih besar dari 300mb, jika melebihi 300mb, bagi menjadi beberapa file.

ada pemikiran?

Saya tidak berpikir tar memiliki fungsionalitas built-in untuk dibagi pada ukuran yang sewenang-wenang (ada -M untuk multi-volume, tetapi ini bergantung pada media tujuan yang mendeteksi kondisi di luar ruang), tetapi ada dua hal yang dapat Anda lakukan. melakukan:

  1. Buat satu file tar.gz dengan cara biasa, kemudian gunakan perintah split untuk membaginya menjadi beberapa bagian (dan cat untuk menggabungkannya kembali untuk dekompresi).

  2. Menggunakan dar bukannya tar, karena ini memiliki fungsionalitas pemisahan bawaan (di antara banyak fitur lain yang tidak dimiliki tar). Sebagai contoh

    $ dar -c "$BACKUPNAME" -g "$INPUTFILE.sql" -s300M -z
