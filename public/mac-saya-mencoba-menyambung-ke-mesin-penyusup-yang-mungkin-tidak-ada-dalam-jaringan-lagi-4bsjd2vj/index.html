Mac saya mencoba menyambung ke mesin penyusup yang mungkin tidak ada dalam jaringan lagi

Saya baru saja mengajukan pertanyaan Saya rasa orang lain memiliki akses ke jaringan nirkabel saya. Apa selanjutnya? . Saya telah mengubah keamanan nirkabel ke WPA2, mengubah kata sandi, menonaktifkan berbagi file, dll seperti yang disarankan dalam jawaban. Saya bahkan me-restart modem saya sehingga kontrak DCHP kedaluwarsa. (Menjadi modem lama, tidak ada pilihan, setidaknya dari apa yang bisa dilihat, untuk mengatur ulang sewa)

Sekarang, karena penasaran saya unduh Wireshark untuk melihat apa yang sebenarnya terjadi di jaringan saya.

Saya masih melihat 192.168.1.6 melakukan sesuatu di jaringan. Lebih buruk lagi, saya melihat mesin saya, 192.168.1.2 berbicara dengannya!

  3. Mengapa mac saya terhubung sama sekali? Berbagi file tidak aktif. Berbagi web tidak aktif. Apakah ini sesuatu yang perlu dikhawatirkan atau saya panik tanpa alasan?

  1. Bagaimana ini mungkin? Sewa DCHP modem mengatakan tidak ada 192.168.1.6.

DHCP hanya memberikan alamat IP. Ini sepenuhnya nasihat, dan setiap host dapat memilih alamatnya sendiri dengan atau tanpa DHCP.

2. Mungkinkah Mac saya mengingat alamat lama dan mencoba menyambungkannya? Lalu kenapa ada respon? 3. Mengapa mac saya terhubung sama sekali? Berbagi file tidak aktif. Berbagi web tidak aktif. Apakah ini sesuatu yang perlu dikhawatirkan atau saya panik tanpa alasan?

Dari apa yang dapat kita lihat di bagian ini, Mac Anda memulai koneksi TCP dan ingin menggunakan NetBIOS. NetBIOS sering digunakan untuk resolusi nama, tidak harus berbagi file. Mac Anda memang mengingat titik akhir jarak jauh NetBIOS dan sekarang terhubung dengannya. 192.168.1.6 berbicara NetBIOS, tetapi menolak koneksi. Lalu lintas itu sendiri tidak menjadi masalah.

kemungkinan mesin penyusup

Pertama, cari tahu alamat IPv4 router. Mungkin 192.168.1.1, tetapi jika itu adalah 0,6, misterinya terpecahkan. Kemudian, nonaktifkan WLAN untuk sementara dan hubungkan melalui kabel. Pastikan tidak ada komputer lain yang terhubung, baik melalui kabel maupun WLAN (LED WiFi harus mati). Kemudian, tetapkan kata sandi WPA2 baru, pertama di router, dan kemudian di semua mesin. Periksa alamat IP pada semua mesin yang terhubung ke jaringan untuk memastikan 192.168.1.6 bukan file-server kuno di lemari Anda. Jika Anda masih dapat mendeteksi penyusup (dan saya berani bertaruh Anda menghadapi kesalahan konfigurasi, bukan penyusup - penyusup berpengetahuan mungkin tidak akan berbicara NetBIOS), ajukan pertanyaan baru dan pastikan komputer Anda tidak terganggu.
