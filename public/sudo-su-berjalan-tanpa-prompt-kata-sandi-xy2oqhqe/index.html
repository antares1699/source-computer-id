sudo su berjalan tanpa prompt kata sandi?

Jadi, inilah masalahnya. Kami telah menyiapkan file / etc / sudoers sehingga pengguna dapat menjalankan perintah dari / bin seperti "cat" atau "mkdir" tanpa memasukkan kata sandi. Masalahnya adalah bahwa perintah "su" juga ada di / bin, jadi jika mereka memasukkan "sudo su", itu memberi mereka akses root tanpa kata sandi. Ini file / etc / sudoers:

Defaults targetpw    
%users ALL=(ALL) ALL 

root    ALL=(ALL) ALL

support ALL=(ALL) NOPASSWD: /sbin/, /bin/, /opt/, /etc/init.d/, /elo/
support ALL=(ALL) NOPASSWD: /usr/bin/mysql

Apakah ada cara agar saya dapat menolak / bin / su sambil tetap mengizinkan perintah / bin lainnya?
