Upaya CLONE Mac OS X HFS + drive utama gagal dengan "Kembalikan Kegagalan Terjadi kesalahan (32) saat menyalin (Pipa Rusak)

Selama bertahun-tahun saya telah mengkloning drive utama di MacBook saya untuk membuat drive eksternal yang dapat di-boot. Dengan cara ini saya memiliki drive eksternal portabel yang (cukup) up-to-date, dapat di-boot, dan portabel yang dapat saya colokkan ke mac apa pun dan boot untuk alasan apa (misalnya, drive utama gagal dan saya perlu memulihkan).

Kemarin, ini gagal dengan kesalahan yang diberikan pada judul.

Konfigurasi dan prosedur: MacBook Pro (awal 2011) dengan RAM 8 GB, Lion (server). Drive utama berukuran 750 GB

Ini adalah konfigurasi MacBrook Pro yang cukup standar.

Drive eksternal adalah LaCie 2 TB. Sambungannya adalah Firewire 800. LaCie dibagi menjadi 2 partisi: 1. Boot => 750 GB 2. Data RAID Set Backup => 1,25 TB

Semua orang diformat HFS +

LaCie diformat menggunakan SoftRaid 4.3.3

  1. Nyalakan ulang dan tekan tombol opsi untuk mendapatkan daftar perangkat yang dapat di-boot.
  2. Pilih untuk mem-boot disk pemulihan Lion
  3. Pilih untuk masuk ke Disk Utility
  4. Gunakan "Restore" di MacIntosh HD. MacIntosh HD adalah sumber dan "Boot" adalah tujuannya.
  5. Tekan Pulihkan. Dan dalam hal ini mendapatkan kesalahan di atas.

Adakah orang lain yang melihat ini? Saya mencari-cari di Google dan melihat satu kasus di mana seseorang melihat kesalahan ini - tetapi mereka bekerja dengan Boot Camp (jadi itu perbedaan) dan terlebih lagi saya tidak yakin bahwa pertanyaan yang ditanyakan benar-benar terjawab.

Oke, saya telah bereksperimen sedikit dan mencoba beberapa "alur kerja" baru yang berbeda dan memiliki satu yang berfungsi dalam kasus ini. Ini dia - sangat sederhana.

Menggunakan sama utilitas disk untuk memformat dan mempartisi disk tujuan yang Anda gunakan untuk pemulihan itu sendiri (dalam hal ini adalah utilitas disk di disk pemulihan Lion). Itu dia.

Secara terperinci:

  1. Mulai ulang, tekan tombol opsi. Boot ke disk Pemulihan. Pilih Utilitas Disk.
  2. hapus dan partisi disk tujuan. Harus GPT (Tabel partisi GUID)
  3. kembalikan drive boot utama ke partisi boot baru pada disk tujuan.

Jika Anda (seperti saya) menggunakan SoftRaid, Anda dapat mengonversi ke SoftRaid setelah Anda membuat disk booting belakang yang baru.

Saya lakukan tidak tahu mengapa mempartisi menggunakan utilitas disk atau utilitas SoftRaid pada boot drive normal (yang hanya merupakan versi terbaru dari Lion) akan gagal dengan kesalahan mereka dan mungkin itu hanya kekhasan sistem saya).

Terlepas dari itu, selain menjadi pembuang waktu, masalah ini tidak berakhir menjadi penghenti. Jika karena alasan apa pun Anda mengalami masalah ini saat mengkloning drive boot utama di laptop Anda, coba gunakan utilitas disk yang sama untuk memformat dan mempartisi disk tujuan yang Anda gunakan untuk pemulihan itu sendiri dan saya pikir Anda akan menghindari jenis ini. masalah yang mengganggu.
