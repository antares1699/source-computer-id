Mencari pemeriksa file .WAV baris perintah Linux atau Windows

Saya mencoba mencari program baris perintah (Windows atau Linux) yang dapat memeriksa file .WAV, untuk melihat apakah sudah lengkap, tanpa kerusakan atau berakhir tiba-tiba.

Meskipun saya telah menemukan beberapa program yang dapat melakukan ini dengan file .mp3, saya tidak dapat menemukan apa pun yang dapat melakukannya dengan file .WAV, selain dari beberapa program Windows komersial yang hanya menggunakan GUI sehingga tidak dapat digunakan dalam skrip.

Berikut artikel tentang tampilan file PCM wav:

https://ccrma.stanford.edu/courses/422/projects/WaveFormat/

Harus dapat menulis semacam skrip untuk memeriksa header dan memastikannya terlihat baik-baik saja.

Memperbarui:

http://www.sendspace.com/file/cdy1hk

Berikut adalah utilitas kecil yang mungkin sesuai dengan kebutuhan Anda. Ia memeriksa file, mengeluarkan beberapa informasi pada file, dan keluar. Kode yang dikembalikan adalah 0 jika berhasil, bukan nol jika ada masalah dengan file tersebut.

Itu ditulis dengan Python dan bergantung pada pustaka gelombang Python. Anda dapat mengekstrak file .py dari zip dan memeriksa / menjalankan secara independen, atau menggunakan exe yang telah dikompilasi bersama dengan file pendukung yang disertakan.

Tampaknya berfungsi dengan baik untuk mis. output file oleh lame --decode, dan menangkap ketika saya memotong WAV (periksa untuk memastikan ukuran file waras).

Tidak ada cara untuk menentukan apakah data PCM telah rusak tanpa mis. checksum eksternal, dll.

Memperbarui:

Menambahkan pemeriksaan kesalahan yang sedikit lebih baik. Versi exe yang dikompilasi menjadi aneh ketika wavcheck.py ada di folder yang sama, jadi saya memasukkannya ke dalam subfolder (/ src). Lol. Tautan yang diperbarui di atas ke file baru.

Memperbarui:

Saya melihat shntool seperti yang disarankan oleh @boehj - sepertinya perangkat lunak yang bagus untuk memeriksa file WAV, dan memiliki hasil detail yang bagus. Outputnya dapat diurai dengan mudah atau kode sumbernya dimodifikasi untuk memungkinkan status keluar semua OK dalam mode 'info' dan / atau 'len'. Seperti berdiri, ia memiliki kode keluar 0 bahkan ketika melaporkan masalah dengan file tersebut.

Beranda untuk shntool: http://www.etree.org/shnutils/shntool/

Saya tahu ini adalah utas lama, tetapi untuk berjaga-jaga jika ada gunanya:

Itu JHOVE alat (yang cukup banyak digunakan dalam domain pengarsipan / perpustakaan digital) termasuk modul WAV yang dapat memvalidasi file WAV. Tersedia disini:

http://jhove.openpreservation.org/

Tidak tahu bagaimana caranya JHOVE Validasi dibandingkan dengan pemeriksaan itu shntool tampil.

MEMPERBARUI : jadi setelah memposting jawaban saya, saya membantai beberapa WAV sendiri dan kemudian menjalankannya melalui sejumlah alat. Tulisan lengkap dengan tautan untuk menguji data dan skrip di sini:

http://openpreservation.org/blog/2017/01/04/breaking-waves-and-some-flacs/

Hasil utama: lebih baik tetap menggunakan Shntool tua yang bagus! JHOVE ternyata tidak berguna untuk hal semacam ini (yang juga mengejutkanku). (Kebetulan blog juga mendeteksi file FLAC yang rusak, meskipun itu bukan pertanyaannya di sini.)
