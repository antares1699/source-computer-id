Mesin Virtual lebih cepat dari komputer host?

Sepertinya tidak mungkin, saya kaget.
Saya menginstal Windows 7 di PC saya, butuh sekitar 30-40 menit untuk menginstal, dan boot normal (ketika tidak ada program yang diinstal) membutuhkan waktu sekitar 30 detik. Saya menjalankan VM pada PC yang sama, menginstal Windows yang sama pada VM dan instalasi membutuhkan waktu sekitar 20 menit (!) Dan boot dalam 10-15 detik setiap saat.
Saya melihat smtg seperti itu ketika saya melakukan hal yang sama dengan Windows XP (sekitar setahun yang lalu) tetapi saya pikir itu adalah kesalahan.
Saya tidak akan percaya juga, tetapi kenyataannya - sekarang saya menulis pertanyaan saya dari PC itu.

Jadi pertanyaan saya adalah - Bagaimana mungkin?

Catatan 1: Pada mesin host saya menggunakan image ISO untuk menginstal dan menginstalnya tanpa DVD atau USB, sehingga DVD dan USB tidak dapat menyebabkan penurunan performa.
Catatan 2: Saya menjalankan Windows 7, VMWare Workstation 9, VM memiliki ukuran tetap VMDK (15GB) dan menggunakan kedua inti prosesor saya.

Saya dapat memikirkan setidaknya satu alasan:

Host fisik memiliki perangkat keras nyata yang perlu menjalankan tes aktual dan merespons dengan status OS.

Ini mungkin paling menonjol saat driver diinisialisasi; katakanlah, selama boot atau instal.

Mesin virtual memiliki perangkat keras buatan (virtual) yang merespons dalam waktu singkat. Pasangkan dengan sumber daya host yang sudah "panas" (drive, memori, dll.), Dan semuanya menjadi lebih cepat.

Misalnya, bandingkan kecepatan menyalakan, menginisialisasi, dan menggunakan drive DVD aktual, vs. yang terpasang ke VM.
