Mengapa dimungkinkan untuk membuat file tanpa nama yaitu .htaccess menggunakan notepad ++, tetapi ketika menggunakan windows explorer itu membutuhkan nama file?

Ini ada hubungannya dengan cara Microsoft mengizinkan Explorer.exe bekerja dengan menyimpan file tanpa nama file dan hanya ekstensi file yang ditentukan.

Tampaknya beberapa aplikasi seperti Notepad dan Notepad ++ tidak memiliki batasan ini yang diprogram ke dalam logikanya, dan mereka mengizinkan penyimpanan file hanya dengan ekstensi — jadi ini adalah fitur atau fungsi keamanan Windows Explorer yang memaksakan hal ini agar tidak terjadi secara default di tingkat program.

SOLUSI

Solusinya adalah dengan menambahkan titik ekstra di akhir ekstensi dan Windows Explorer akan memungkinkannya untuk disimpan tetapi itu akan mengurai tanda titik dan menyimpannya tanpa nama file dan tanpa titik tambahan di akhir.

Jadi ketika Anda membuatnya dengan Windows Explorer sebagai .htaccess. itu tidak akan memberikan pesan kesalahan di bawah ini, dan memungkinkan Anda untuk menyimpannya dengan baik.

SUMBER: Bagaimana cara mengganti nama file menjadi .htaccess di Windows 7?

Secara teknis, file seperti .htaccess memiliki nama .htaccess dan tidak ada ekstensi. Ini mengikuti dari dokumentasi resmi "Naming Files, Paths, dan Namespaces" di https://msdn.microsoft.com/en-us/library/windows/desktop/aa365247.aspx .

Nama File dan Direktori

Semua sistem file mengikuti konvensi penamaan umum yang sama untuk file individual: nama file dasar dan ekstensi opsional, dipisahkan oleh titik.

Perhatikan bahwa ada selalu sebuah nama, yang opsional hanyalah ekstensi. Jadi jika nama file lengkap dimulai dengan titik, dan tidak memiliki titik lain di dalamnya, maka itulah nama dan ada tidak perpanjangan. Bahasa spesifikasi tidak menyisakan ruang untuk nama file yang memiliki ekstensi tetapi tidak memiliki nama.

Ini dapat diverifikasi dengan membuat beberapa file bernama .htaccess dan .htaccess.bak di direktori kosong, kemudian menjalankan perintah berikut pada prompt cmd.

C:\etc>dir /a-d /b *.*
.htaccess
.htaccess.bak

C:\etc>dir /a-d /b *.
.htaccess

dir pertama mencantumkan kedua file seperti yang diharapkan, yang kedua (di mana *. mencocokkan file dengan tidak ekstensi) hanya mencantumkan .htaccess.

Adapun Explorer, tampaknya mengabaikan spesifikasi dan mengikuti aturannya sendiri. File seperti .htaccess ditampilkan dengan kedua nama = .htaccess dan ketik = HTACCESS file selama ekstensi tersebut tidak terdaftar. Jika namanya cocok dengan a dikenal ekstensi maka nama dikosongkan dan ekstensi dianggap cocok mis. file .cmd ditampilkan dengan type = Windows Command Script dan tanpa nama (saat menggunakan default ' sembunyikan ekstensi untuk jenis file yang dikenal 'pengaturan).

Karena menampilkan nama kosong terlihat buruk, Penjelajah pasti telah memutuskan untuk memblokir nama tersebut secara langsung agar tidak dibuat dengan sendirinya. Sejauh itu terlihat dari http://blogs.msdn.com/b/oldnewthing/archive/2008/04/14/8389268.aspx .
