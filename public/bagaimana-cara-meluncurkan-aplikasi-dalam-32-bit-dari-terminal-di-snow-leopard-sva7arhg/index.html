Bagaimana cara meluncurkan aplikasi dalam 32 bit dari Terminal di Snow Leopard?

Seperti kebanyakan orang, saya memiliki beberapa masalah dengan meluncurkan aplikasi saya dalam mode 32 bit, karena driver LCC 3.1 yang jauh dari sempurna untuk keyboard tepi Logitech di novo saya. Pada dasarnya, saya tidak dapat menggunakan tata letak keyboard yang benar ketika dalam mode 64 bit (tata letak saya berubah ketika saya mengganti aplikasi yang sangat mengganggu).

Jadi saya terpaksa meluncurkan aplikasi saya dalam 32 bit.

Dapatkah saya melakukan ini dari terminal dengan perintah exec? Memeriksa 'aplikasi peluncuran dalam mode 32 bit' tampaknya tidak berpengaruh (tetapi berfungsi dari gui).

Terima kasih sebelumnya.

EDIT Saya seharusnya sudah mempersiapkannya. Saya mencoba membuat skrip mvim (diberikan dalam macvim) meluncurkan macvim dalam 32 bit. Jadi solusi berdasarkan open, meskipun meluncurkan aplikasi dalam 32 bit, bukanlah yang saya cari, karena tampaknya tidak menangani pipa dan pengalihan argumen dengan benar.

Jika hanya tentang MacVim, Anda dapat menghapus kode 64-bit dari biner dengan ditto.

ditto --arch i386 /Applications/Utilities/MacVim.app/Contents/MacOS/MacVim /Applications/Utilities/MacVim.app/Contents/MacOS/MacVim_i386
exec /Applications/Utilities/MacVim.app/Contents/MacOS/MacVim_i386

Saya baru saja mencobanya dan sepertinya berhasil. Mungkin Anda bahkan dapat menyalurkan melalui ditto, tetapi saya ragu.

Bagikan
Debilski — Sumber
  • Saya juga tidak bisa membuatnya bekerja: tampaknya ketika Anda memberikan opsi -g ke Vim (exec /Applications/MacVim.app/Contents/MacOS/Vim_i386 -g), itu bercabang dalam mode 64 bit.
    — LeMiz
