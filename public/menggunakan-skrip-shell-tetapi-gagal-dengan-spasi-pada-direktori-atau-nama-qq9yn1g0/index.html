Menggunakan skrip shell tetapi gagal dengan spasi pada direktori atau nama

Saya relatif baru mengenal skrip Linux dan saya mengalami masalah dengan skrip yang saya coba gunakan ketika ada spasi di nama direktori atau nama file. Skrip mengecilkan file PDF sehingga saya dapat memasukkannya ke dalam email.

gs  -q -dNOPAUSE -dBATCH -dSAFER \
    -sDEVICE=pdfwrite \
    -dCompatibilityLevel=1.3 \
    -dPDFSETTINGS=/screen \
    -dEmbedAllFonts=true \
    -dSubsetFonts=true \
    -dColorImageDownsampleType=/Bicubic \
    -dColorImageResolution=72 \
    -dGrayImageDownsampleType=/Bicubic \
    -dGrayImageResolution=72 \
    -dMonoImageDownsampleType=/Bicubic \
    -dMonoImageResolution=72 \
    -sOutputFile=Compressed$1 \
     "$1"

jadi saya menggunakan perintah

find * -type f -name "*.pdf" -exec sudo sh ~/shrinkpdf.sh {} \;

Ini akan bekerja pada PDF di folder yang sama yang tidak memiliki spasi di dalamnya, tetapi ketika mencoba mengakses folder di dalamnya yang memiliki spasi, itu kesalahan dan membuat file hanya menggunakan kata pertama dari nama folder.

Bagaimana saya bisa mendapatkan ini untuk menerima file dan folder dengan spasi di namanya?

Anda mengutip salah satu $ 1 tetapi tidak yang lain. Jika jalur Anda berisi spasi, kutipan itu penting. Untuk amannya, sebaiknya asumsikan bahwa setiap jalur yang diterima skrip berisi karakter bermasalah, dan mengutip setiap jalur.

Inilah saran saya:

gs  -q -dNOPAUSE -dBATCH -dSAFER \
    -sDEVICE=pdfwrite \
    -dCompatibilityLevel=1.3 \
    -dPDFSETTINGS=/screen \
    -dEmbedAllFonts=true \
    -dSubsetFonts=true \
    -dColorImageDownsampleType=/Bicubic \
    -dColorImageResolution=72 \
    -dGrayImageDownsampleType=/Bicubic \
    -dGrayImageResolution=72 \
    -dMonoImageDownsampleType=/Bicubic \
    -dMonoImageResolution=72 \
    "-sOutputFile=Compressed$1" \
    "$1"

Jika Anda bertanya-tanya, tidak masalah untuk mengutip opsi seperti ini; tidak ada bedanya dengan mengutip jenis argumen lain. gs -q dan gs "-q" terlihat persis sama dengan gs ketika gs menerimanya.

Kami hanya menggunakannya di sini untuk memastikan seluruh jalur dijadikan bagian dari opsi -sOutputFile, daripada yang terjadi sebelumnya: fragmen sebelum spasi ditetapkan dengan benar ke argumen opsi, dan fragmen setelah spasi adalah argumen independen , bahwa gs mungkin diperlakukan seolah-olah itu adalah argumen file masukan.
