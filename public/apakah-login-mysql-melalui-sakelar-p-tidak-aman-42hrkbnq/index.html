Apakah login MySQL melalui sakelar -p tidak aman?

Saya ingin masuk ke database MySQL dari shell.

Biasanya saya gunakan

mysql -p

Tapi saya baru saja memperhatikan yang berikut di bantuan MySQL:

-p, --password[=name] 
                      Password to use when connecting to server. If password is
                      not given it's asked from the tty. WARNING: This is
                      insecure as the password is visible for anyone through
                      /proc for a short time.

Apakah ini pepatah bahwa berikut ini tidak aman?

mysql -p=mypassword

Atau bahwa semua penggunaan -p tidak aman?

Hanya ini yang tidak aman:

mysql -p=mypassword

karena dengan cara ini dapat terlihat ketika ps digunakan untuk melihat argumen baris perintah. Singkatnya, selalu gunakan opsi -p, dan ketika otomatisasi diinginkan, gunakan "file opsi"

Mengacu pada http://dev.mysql.com/doc/refman/5.0/en/password-security-user.html untuk informasi lebih lanjut:

Ini nyaman tetapi tidak aman, karena sandi Anda dapat dilihat oleh program status sistem seperti ps yang mungkin dipanggil oleh pengguna lain untuk menampilkan baris perintah. Klien MySQL biasanya menimpa argumen kata sandi baris perintah dengan nol selama urutan inisialisasi mereka. Namun, masih ada interval singkat di mana nilainya terlihat. Selain itu, pada beberapa sistem, strategi penimpaan ini tidak efektif dan sandi tetap terlihat oleh ps. (Sistem SystemV Unix dan mungkin yang lainnya mengalami masalah ini.)
