Server Virtualisasi apa yang mendukung Klien Windows dengan Direct X?

Sebelumnya saya ingin menyatakan pertanyaan ini mirip dengan pendahulunya sini dan sini tapi, pertanyaan-pertanyaan itu sudah lama dan saya ingin jawaban yang up-to-date berdasarkan teknologi terkini.

Saya ingin meng-host beberapa desktop sederhana untuk penggunaan sehari-hari keluarga saya dan beberapa contoh sementara lainnya untuk saya mainkan. Beberapa anggota keluarga saya memerlukan Direct X untuk akselerasi grafis.

Penawaran Server Virtualisasi apa saat ini yang mendukung Direct X untuk klien? Perhatian sekunder adalah biaya dan keandalan serta kinerja.

Saya belum membeli perangkat keras hosting, rekomendasi Anda sangat kami hargai serta jawaban langsung atas pertanyaan tersebut.

Pertama-tama, kecuali jika keluarga Anda sudah cukup paham teknologi, sepertinya bukan ide yang baik untuk beralih ke mereka. Saat ini, 3D tervirtualisasi tidak sebagus aslinya. AFAIK, beberapa game ringan dapat dilakukan, tetapi pengalaman bermain game yang berat masih di bawah standar. Itu tanpa membahas keseluruhan masalah dari pengalaman penggunaan yang sebenarnya - membiarkan anggota keluarga Anda menggunakan VM "mereka", dll. Saya hanya tidak melihat ini sebagai ide atau pengalaman yang sangat bagus untuk disebarkan ke orang lain.

Semua yang dikatakan, jika Anda bertekad untuk melakukan ini, inilah beberapa yang perlu Anda ketahui.

Perangkat lunak

Di antara hypervisor Tipe-2 Anda, VMware Workstation dan Oracle's VirtualBox akan mempercepat Direct3D dan OpenGL tamu dengan berbagai tingkat keberhasilan. Saya yakin bahwa produk Parallels Workstation juga berfungsi dengan baik. Salah satu dari produk ini akan memerlukan OS host yang dijalankan terlebih dahulu, dengan mesin virtual dimulai sesudahnya. Semua tolok ukur yang saya lihat dari program-program ini menunjukkan bahwa ada banyak masalah kompatibilitas, dan kinerja mengalami pukulan yang parah dibandingkan dengan aslinya menggunakan GPU yang sama.

Jika Anda memutuskan untuk menggunakan hypervisor Tipe-1, Microsoft Hyper-V mungkin akan menjadi pilihan terbaik Anda (setidaknya dengan tamu Windows), karena itu RemoteFX teknologi akan memungkinkan Anda menampilkan GPU virtual untuk tamu Windows 7. Saya belum melihat tolok ukur apa pun, tapi posting blog ini memiliki tangkapan layar Crysis. Saya memahami bahwa KVM memungkinkan VGA Passthrough juga, tetapi saya memahami bahwa mendedikasikan GPU untuk VM, yang mungkin tidak persis seperti yang Anda cari. (Beberapa hypervisor Tipe-2 juga dapat memberikan kartu grafis sekunder ke VM juga.)

Perangkat keras

Persyaratannya relatif mudah, tetapi menemukan suku cadang (terutama dari Intel) yang mendukungnya bisa ... rumit.

Dari CPU, Anda akan membutuhkannya Terjemahan Alamat Tingkat Kedua (SLAT) dukung. Intel menyebutnya Extended Page Tables dan AMD menyebutnya Rapid Virtualization Indexing. Tabel Halaman Bersarang adalah nama lain yang mungkin Anda lihat. Ini pertama kali terlihat dengan arsitektur Nehalem (Seri Core-i5 / i7 Generasi Pertama) pada Intel, dan CPU AMD Opteron Generasi ke-3. Menurut AMD , hampir semua Athlon II, Phenom, atau CPU yang lebih baru seharusnya memilikinya. Intel memiliki file Daftar VT pada basis pengetahuan ARK mereka. Namun, itu hanya mencakup VT-x, bukan VT-d, dan Intel menggunakan dukungan VT-d untuk menyegmentasikan pasar mereka. (Misalnya, CPU Sandy Bridge "K", misalnya Core i5-2500K, lakukan tidak mendukung VT-d. Pastikan solusi virtualisasi Anda tidak memerlukan VT-d sebelum membeli.)

GPU harus mendukung DirectX 9.0c dan 10.0. Dengan kata lain, hampir semua GPU AMD atau nVidia modern. Untuk RemoteFX, meskipun Microsoft anemia HCL , Pihak ke-3 telah menunjukkan bahwa hampir semua GPU saat ini berfungsi. Ini adalah kasus di mana memori GPU tambahan bermanfaat, jadi saya pasti akan menggunakan setidaknya 1 GB RAM pada kartu video.

Selain itu, Anda berbicara tentang host VM. RAM dan I / O hampir selalu menjadi masalah dalam segala jenis beban. Untungnya RAM relatif murah - tidak ada alasan Anda tidak dapat memiliki 16 GB RAM pada sistem desktop yang relatif murah. I / O adalah masalah lain - jika Anda ingin menjalankan beberapa VM sekaligus, menempatkannya di hard drive mereka sendiri sangat bermanfaat. SSD juga merupakan pilihan, jika Anda mampu membeli kapasitas yang Anda butuhkan.

Ya, versi terbaru Hyper-V memiliki fitur yang disebut RemoteFX. Anda perlu memasang kartu 3D asli di host Hyper-V Anda (yang mungkin berarti kartu tersebut harus berada dalam sasis 4U) dan memasang driver untuknya.

Kemudian Anda dapat menambahkan kartu 3D virtual ke VM Anda dengan cara yang sama seperti Anda menambahkan perangkat keras virtual baru ke dalamnya.
