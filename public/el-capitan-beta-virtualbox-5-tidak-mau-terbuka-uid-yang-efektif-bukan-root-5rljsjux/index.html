El Capitan beta - VirtualBox 5 tidak mau terbuka: UID yang efektif bukan root

Saya memiliki VB 4.3 yang berjalan di OSX El Capitan beta 1-3. Saya menginstal VB 5 dan itu tidak diluncurkan dan memberikan pesan UID efektif bukan root.

Saya menemukan kiriman di VirtualBox tidak dimulai di Mac OS X . Metode yang diposting oleh Deborah Speece berfungsi untuk memulainya melalui Terminal tetapi postingan dari realityloop merusaknya lagi dan saya perlu menginstal ulang.

Adakah yang punya cara untuk meluncurkan VB 5 dari ikon aplikasi di OSX El Capitan?

Saya baru saja menginstal VirtualBox 5 di OSX Yosemite tanpa masalah. Masalah ini sepertinya khusus untuk El Capitan yang dalam versi beta. Mudah-mudahan ini akan diselesaikan dalam versi yang didukung.

Saya juga memiliki penyelesaian dengan menggunakan aplikasi alur kerja Automator dengan beberapa skrip.

Saya telah berhasil memecahkan masalah.

Pertama, Anda harus ingin menghapus aplikasi dengan AppCleaner (brew cask install appcleaner). Ini log terminal saya setelah itu. Menggunakan homebrew.dll untuk instalasi.

rm -rf ~/VirtualBox\ VMs ~/Library/VirtualBox
brew cask uninstall virtualbox
brew cask install virtualbox
for bin in VirtualBox VirtualBoxVM VBoxNetAdpCtl VBoxNetDHCP VBoxNetNAT VBoxHeadless; do
    sudo chmod u+s "/Applications/VirtualBox.app/Contents/MacOS/${bin}"
done
