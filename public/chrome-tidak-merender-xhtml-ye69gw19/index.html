Chrome tidak merender xhtml?

Buka file xhtml lokal di chrome, namun, chrome memperlakukannya sebagai file XML, dan tidak akan merendernya sebagai HTML.

Saya tidak ingin mengubah semua ekstensi file * .xhtml saya menjadi .html, jadi apakah ada solusi alternatif?

Karena Anda menyatakan XHTML-1.0 ketat, tag html Anda memerlukan ruang nama xml:

<html xmlns="http://www.w3.org/1999/xhtml">

Beberapa masalah validasi lainnya: Anda memerlukan deklarasi kumpulan karakter dan teks Anda memerlukan <p> di sekitarnya.

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN"
  "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">
    <head>
        <title>Example</title>
    </head>
    <body>
      <p>
        <b>Hello</b>, 
        <i>World</i>!
      </p>
    </body>
</html>

OmnipotentEntity — Sumber
  • Meskipun solusinya benar, alasannya sedikit berbeda. Browser biasanya tidak memuat entitas eksternal. DTD akan default atribut xmlns ke namespace XHTML, tetapi entitas eksternal DTD tidak dimuat. Xmlns diperlukan untuk XHTML secara umum, bukan karena doctype XHTML 1.0 Strict. Kecuali Anda menggunakan namespace itu, browser tidak akan tahu bahwa elemen XML Anda sebenarnya adalah HTML.
    — Christian Hujer
  • Dan deklarasi encoding tidak diperlukan. XML memiliki mekanisme deteksi dan secara eksplisit mengatakan bahwa di Deklarasi XML , pengkodeannya opsional.
    — Christian Hujer
