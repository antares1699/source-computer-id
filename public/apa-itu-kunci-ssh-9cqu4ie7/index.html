Apa itu kunci SSH?

Saya mendaftar ke github dan melihat opsi kunci ssh yang terlihat menarik. Saya awalnya mengharapkan sesuatu seperti kunci ssl (nama, nama rekan, dll). Setelah melalui itu saya perhatikan saya hanya memasukkan kata sandi dan selalu myuser @ comp-name (ini windows). Mengapa? Saya pikir itu adalah id pengguna / sandi dan saya dapat membuat kunci terpisah untuk tujuan terpisah untuk alasan privasi.

Sekarang saya lihat saya diharuskan menggunakan satu untuk membuat repositori. Saya juga melihat sesuatu tentang 'file kunci pribadi' saat melihat opsi. Apa sebenarnya Kunci SSH dan bagaimana saya bisa membuat pengguna terpisah tanpa membuat login terpisah di windows.

Untuk menambah jawaban luar biasa David:

Untuk membuat kunci SSH (atau lebih tepatnya: pasangan kunci publik / pribadi) di Windows, Anda dapat menggunakan PuttyGen (gratis). Ini adalah tambahan untuk klien PuTTY SSH yang populer.

Instruksi mis. sini: http://theillustratednetwork.mvps.org/Ssh/Private-publicKey.html

Tentu saja Anda juga dapat membuat kunci menggunakan alat OpenSSH; ini hanya kurang umum di Windows.

Pernahkah Anda mendengar tentang enkripsi asimetris (juga disebut kriptografi kunci publik)? Itulah teknologi yang menjadi dasar kunci SSH.

Konsep dasar enkripsi asimetris adalah Anda memiliki dua kunci kriptografi, yang biasanya berupa angka yang sangat besar (atau sama, string panjang data biner). Pesan apa pun yang dienkripsi dengan salah satunya dapat didekripsi hanya dengan yang lain - bahkan dengan kunci penyandian asli itu sendiri! Ini berguna karena Anda dapat menyimpan salah satu kunci, file kunci pribadi , untuk diri Anda sendiri, dan publikasikan yang lainnya, file kunci publik , di suatu tempat secara online atau di mana pun Anda suka. Orang-orang dapat mengenkripsi pesan menggunakan kunci publik dan yakin bahwa hanya Anda, pemegang kunci pribadi yang sesuai, yang dapat mendekripsi dan membaca isinya. (Beginilah cara kerja GPG dan PGP dan program serupa, jika Anda pernah mendengarnya)

Ternyata prosesnya juga bekerja secara terbalik, dengan tombol yang diaktifkan. Jika Anda mengenkripsi sesuatu menggunakan kunci pribadi Anda, itu hanya dapat didekripsi menggunakan kunci publik yang sesuai. Ini mungkin tampak tidak berguna karena siapa pun di dunia bisa mendapatkan kunci publik Anda dan mendekripsi pesan, tetapi ini memiliki satu efek samping yang sangat berguna: itu menegaskan bahwa pesan itu berasal dari Anda . Karena tidak ada orang lain di dunia yang memiliki kunci pribadi Anda, tidak ada orang lain yang dapat membuat pesan yang dapat didekripsi dengan kunci publik Anda.

Jika Anda memikirkannya, dapat memverifikasi bahwa sebuah pesan datang dari Anda dan bukan dari beberapa penipu adalah apa sebenarnya otentikasi itu. Setiap kali Anda masuk ke situs web menggunakan nama pengguna dan kata sandi, Anda memberikan beberapa informasi rahasia yang seharusnya hanya Anda ketahui, untuk membuktikan bahwa permintaan masuk tersebut berasal dari Anda. Nah, Anda dapat melakukan hal yang sama dengan kunci pribadi Anda: mengenkripsi permintaan login Anda untuk membuktikan bahwa itu berasal dari Anda. Dan begitulah cara kerja otentikasi pasangan kunci SSH. Alih-alih Anda memberikan nama pengguna dan kata sandi, server SSH mengirim pesan kepada klien SSH Anda untuk dienkripsi dengan kunci pribadi Anda. Klien mengenkripsinya, mengirim kembali pesan terenkripsi, dan server mendekripsinya menggunakan kunci publik Anda untuk memeriksanya. Jika lolos cek, selamat, Anda sudah login!

Saya sendiri belum mendaftar ke GitHub, jadi saya tidak paham dengan prosedur pastinya, tetapi saya curiga ketika mereka meminta username@host, itu hanya sebagai komentar untuk mengidentifikasi kunci tertentu. Jika Anda masuk ke GitHub dari komputer yang berbeda, Anda akan memiliki kunci pribadi SSH yang berbeda untuk masing-masing komputer tersebut, dan GitHub perlu melacak semua kunci publik yang sesuai. username@host adalah cara yang cukup standar untuk memberi label kunci publik SSH yang berasal dari orang dan / atau host berbeda.

Omong-omong, perlu disebutkan bahwa otentikasi pasangan kunci SSH umumnya dianggap lebih aman daripada otentikasi nama pengguna / kata sandi. Alasannya adalah, ketika Anda menggunakan nama pengguna dan kata sandi untuk masuk ke situs web, Anda harus berbagi informasi itu dengan setidaknya satu komputer lain: server web yang menjalankan situs tersebut. Sering kali, tidak apa-apa, tapi selalu begitu bisa jadi bahwa siapa pun yang bertanggung jawab atas situs web, atau bahkan hanya seseorang yang memiliki akses ke server, bisa merekam sandi Anda dan menggunakannya di belakang Anda untuk menyamar sebagai Anda. Menggunakan pasangan kunci privat / publik menghilangkan kemungkinan itu, karena Anda tidak pernah berbagi kunci privat dengan siapa saja sama sekali, bahkan bukan server yang Anda masuki. Jadi, meskipun server (mis. GitHub) dapat dengan sempurna memeriksa apakah itu benar-benar Anda yang masuk, itu masih tidak memiliki cukup informasi untuk berpura-pura masuk sebagai Anda.

Mengenai cara Anda membuat pasangan kunci SSH: Saya sendiri pengguna Linux, jadi saya tidak yakin bagaimana hal itu akan dilakukan dengan perangkat lunak yang Anda gunakan di Windows. Program klien SSH apa pun yang Anda gunakan (PuTTY?) Akan memberi Anda beberapa cara untuk membuat pasangan kunci.
