Saya perlu bantuan menginstal Linux Mint 18 di komputer dengan Windows 10

Inilah yang terjadi.

  1. Saya mematikan komputer saya.
  2. Saya mematikan Secure Boot
  3. Saya memasukkan disk instalasi Linux Mint (asli) saya ke dalam drive disk saya
  4. Saya memeriksa integritas media. Tidak ada kesalahan yang ditemukan.
  5. Saya memulai Linux Mint.
  6. Linux membutuhkan waktu lama untuk memulai, dan kemudian membekukan saya bahkan sebelum selesai.
  7. Saya masuk ke mode kompatibilitas untuk memeriksa apa yang salah.
  8. Komputer memberi tahu saya bahwa metadata hard drive saya ada di cache Windows.
  9. Saya mengecilkan partisi utama hard drive saya.
  10. Saya mematikan komputer lagi. (Saya tidak mematikannya begitu saja.)
  11. Saya boot ke disk lagi.
  12. Mencoba memulai Linux dalam Mode Kompatibilitas.
  13. Memberitahu saya partisi NTFS yang saya buka dengan mengecilkan partisi utama sistem berada dalam keadaan tidak aman karena saya perlu melanjutkan Windows dan mematikannya sepenuhnya.
  14. Saya mencoba banyak hal lagi dan lagi dengan sedikit keberhasilan.

Apa yang salah?

Anda perlu mematikan opsi yang disebut Startup Cepat .

  1. Buka Panel Kontrol.
  2. Buka opsi Daya.
  3. Klik pada opsi "Pilih apa yang dilakukan tombol daya" di sudut kiri atas.
  4. Klik pada "Ubah pengaturan yang saat ini tidak tersedia pilihan".
  5. Hapus centang "Turn on Fast startup".

Apa itu Fast startup?

Dimulai dengan Windows 8, mode startup cepat tersedia untuk memulai a komputer dalam waktu kurang dari yang biasanya dibutuhkan untuk komputer tradisional, startup dingin. Startup cepat adalah kombinasi hybrid dari startup dingin dan startup bangun-dari-hibernasi.

Startup cepat memungkinkan untuk memulai windows lebih cepat dengan menggunakan teknologi seperti hibernasi. Tetapi ini berarti Windows tidak dimatikan dengan benar. Ini lebih atau seperti hibernasi. Karenanya pesan kesalahan "Lanjutkan Windows dan matikan sepenuhnya". Nonaktifkan startup cepat dan Anda akan baik-baik saja.

Ketika Windows mengaktifkan Hibernate, ia menulis file ke HDD, hiberfil.sys, yang berisi status sistem saat dimatikan, dan mungkin menandai disk sebagai sedang digunakan. Sebagai pengaman untuk mencegah hilangnya data, Linux menolak untuk melakukan booting.

Untuk menyingkirkan hiberfil.sys dengan aman , restart Windows 10 dan matikan sepenuhnya hibernasi dengan perintah berikut di prompt CMD yang dimulai dengan hak administratif:

powercfg -h mati

Anda mungkin mengatakan itu, dan sebaliknya,

powercfg -h aktif

menjadi dua skrip batch, dan membuat pintasan untuk menjalankannya sebagai Administrator, agar mengaktifkan dan menonaktifkan Hibernasi lebih nyaman dalam sistem booting biasa.

Setelah mode Hibernate dinonaktifkan, Anda dapat melanjutkan instalasi distro Linux.
