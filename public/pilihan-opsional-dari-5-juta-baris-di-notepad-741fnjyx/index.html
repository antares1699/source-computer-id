Pilihan opsional dari 5 juta + baris di Notepad ++

Saya memiliki file teks biasa 5 juta plus baris. Windows Notepad tidak dapat melakukan apa-apa dan macet saat mengklik dua kali file tersebut. Notepad ++ dapat membuka file meskipun dengan 5 juta baris. Saya ingin membagi file menjadi 10 file teks tunggal - yaitu 50K baris pertama akan berada dalam file individual, dan selanjutnya dan berikutnya.

Masalahnya adalah ketika saya Ctrl + G dan pergi ke baris 50k, saya harus menahan shift, menekan panah atas sampai garis 50k pertama dipilih. Ini membutuhkan waktu beberapa menit. Apakah ada cara yang lebih mudah untuk membagi file?

Anda mungkin memberi MEMBAGI di GnuWin32 mencoba. Secara sederhana:

split --lines=50000 YourInputFile.dat OutputFile.

akan menghasilkan:

OutputFile.aa
OutputFile.ab
OutputFile.ac
OutputFile.ad
 .
 .

masing-masing dengan 50.000 rekaman. File terakhir akan berisi sisanya.
