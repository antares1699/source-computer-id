Saat menjalankan mesin virtual, apakah dukungan VT-x atau kecepatan clock yang lebih tinggi lebih baik?

Saya sedang membangun workstation desktop baru. Saya akan menggunakan VMWare Workstation untuk menjalankan mesin virtual, mungkin hanya satu atau dua dalam satu waktu. Saat ini saya sedang memutuskan CPU dan opsi yang saya pertimbangkan baik i7 2600 atau i7 2600K. 2600 mendukung instruksi VT-x dan VT-d tetapi tidak dapat di-overclock. 2600K dapat di-overclock (dari stok 3.4GHz menjadi 4.5 GHz dengan mudah) tetapi tidak memiliki dukungan untuk VT-x dan VT-d.

Mana yang lebih berguna untuk pengembangan umum, dan VM secara khusus?

Yah, kebanyakan orang tidak benar-benar melakukan overclock server mereka karena mereka menginginkan lingkungan yang stabil.

Juga VT-X adalah yang Anda inginkan dengan server, Anda akan memiliki lebih sedikit overhead, karena mesin virtual Anda akan dapat mengakses beberapa instruksi pada CPU secara langsung, menyebabkan lebih sedikit overhead dan dengan demikian meningkatkan kinerja.

Juga beberapa operasi tidak berjalan tanpa VT (Windows 2008). Beberapa sistem host secara eksplisit membutuhkan VT (ESX terbaru).

Apakah Anda akan menjalankan OS berbasis Windows di Box Anda? Anda mungkin ingin mencoba Distribusi Linux dengan Implementasi KVM terbaru seperti OpenSUSE (+ Virtualization Repos) atau Fedora (Bloody Edge and Development Platform). Mesin Virtual berbasis Kernel (-KVM-) menggunakan Virtualisasi berbasis Perangkat Keras berdasarkan VT-x (Intel Vanderpool) atau VMX (ADM Pacifica). Anda hampir dapat memvirtualisasikan Platform apa pun yang Anda inginkan. Alternatif untuk ini akan menggunakan VirtualBox yang tampaknya juga mampu memanfaatkan Virtualisasi Perangkat Keras. HTH
