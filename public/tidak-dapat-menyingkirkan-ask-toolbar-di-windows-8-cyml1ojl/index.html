Bagaimana cara menghapus penginstalan Toolbar?

Anda dapat dengan mudah menghapus Toolbar menggunakan petunjuk di bawah ini:

Windows 7:

  1. Dari tombol 'Start' di komputer Anda, pilih 'Control Panel'
  2. Klik opsi "Uninstall a Program" yang ada di bawah kategori "Program"
  3. Pilih program dengan logo Ask dan teks "Ask Toolbar" (atau merek mitra kami untuk Toolbar kustom)
  4. Klik 'Hapus'

Anda juga dapat menghapus penginstalan toolbar langsung dari toolbar

  1. Klik pada chevron (panah bawah) pada tombol Option pada toolbar
  2. Pilih "Uninstall" dari menu drop down
  3. Klik "Ya" ketika ditanya "Apakah Anda yakin ingin mencopot pemasangan produk ini"
  4. Tutup semua browser web yang terbuka
  5. Klik "OK" di jendela popup

Ekstensi Toolbar kami harus dihapus dari browser Chrome Anda.

Setel ulang halaman beranda:

  • Klik pada ikon tiga garis horizontal di pojok kanan atas browser Chrome Anda.
  • Di menu yang muncul - di bawah "Alat" - pilih "Ekstensi".
  • Dari sini, klik "Setting" di sisi kiri halaman.
  • Di bawah "Tampilkan Tombol Beranda" di bagian "Tampilan", klik "Ubah"
  • Masukkan URL yang ingin Anda gunakan sebagai beranda Anda dan kemudian klik "OK".

Setel ulang pencarian default:

  • Klik ikon tiga garis horizontal di pojok kanan atas browser Chrome Anda.
  • Di menu yang muncul, klik "Setting" di sisi kiri halaman.
  • Di bawah "Penelusuran", klik "Kelola mesin telusur ..."
  • Klik "Jadikan Default" di samping penyedia pencarian yang Anda inginkan dan kemudian klik "OK".

Saya ingin menunjukkan cara memperbaikinya masalah . Masalah ini adalah kesalahan Chrome.

Fakta bahwa bilah alat Ask.com dikunci bukanlah sihir malware - ini mendokumentasikan perilaku Google Chrome. Sementara solusi lain mungkin bekerja sedikit demi sedikit untuk malware tertentu tertentu, perbaikan yang mendasarinya adalah menghapusnya dari Chrome.

Google membuat Kebijakan Grup untuk Chrome yang memungkinkan administrator TI untuk memaksa memasang ekstensi :

ExtensionInstallForcelist

Konfigurasikan daftar aplikasi dan ekstensi yang dipasang secara paksa

Tipe data: Daftar string
Lokasi registri Windows: Software\Policies\Chromium\ExtensionInstallForcelist

Nama preferensi Mac / Linux: ExtensionInstallForcelist
Didukung di:
- Chromium (Linux, Mac, Windows) sejak versi 9
- Chromium OS (Chromium OS) sejak versi 11

Fitur yang didukung:
Penyegaran Kebijakan Dinamis: Ya, Per Profil: Ya
Deskripsi:

Menentukan daftar aplikasi dan ekstensi yang dipasang secara diam-diam, tanpa interaksi pengguna, dan yang tidak dapat dicopot pemasangannya oleh pengguna. Semua izin yang diminta oleh aplikasi / ekstensi diberikan secara implisit, tanpa interaksi pengguna, termasuk izin tambahan apa pun yang diminta oleh versi aplikasi / ekstensi yang akan datang. Selain itu, izin diberikan untuk API ekstensi enterprise.deviceAttributes dan enterprise.platformKeys. (Kedua API ini tidak tersedia untuk aplikasi / ekstensi yang tidak dipasang secara paksa.)

Kebijakan ini lebih diutamakan daripada kebijakan ExtensionsInstallBlacklist yang berpotensi bentrok. Jika aplikasi atau ekstensi yang sebelumnya dipasang secara paksa dihapus dari daftar ini, aplikasi atau ekstensi tersebut dicopot secara otomatis oleh Chromium.

Untuk instance Windows yang tidak tergabung dengan domain Active Directory, penginstalan paksa terbatas pada aplikasi dan ekstensi yang tercantum di Toko Web Chrome.

Perhatikan bahwa kode sumber ekstensi apa pun dapat diubah oleh pengguna melalui Alat Pengembang (berpotensi membuat ekstensi tidak berfungsi). Jika ini menjadi perhatian, kebijakan DeveloperToolsDisabled harus disetel.

Setiap item daftar kebijakan adalah string yang berisi ID ekstensi dan URL "pembaruan" yang dipisahkan dengan titik koma (;). ID ekstensi adalah string 32 huruf yang ditemukan, mis. di chrome: // extensions saat dalam mode pengembang. URL "perbarui" harus mengarah ke dokumen XML Manifes Pembaruan seperti yang dijelaskan di https://developer.chrome.com/extensions/autoupdate . Perhatikan bahwa URL "update" yang disetel dalam kebijakan ini hanya digunakan untuk penginstalan awal; pembaruan ekstensi selanjutnya menggunakan URL pembaruan yang ditunjukkan dalam manifes ekstensi.

Misalnya, gbchcmhmhahfdphkhkmpfmihenigjmpp; https://clients2.google.com/service/update2/crx memasang aplikasi Chrome Desktop Jarak Jauh dari URL "pembaruan" Toko Web Chrome standar. Untuk informasi lebih lanjut tentang ekstensi hosting, lihat: https://developer.chrome.com/extensions/hosting .

Jika kebijakan ini dibiarkan tidak disetel, tidak ada aplikasi atau ekstensi yang dipasang secara otomatis dan pengguna dapat mencopot pemasangan aplikasi atau ekstensi apa pun di Chromium.

Kebijakan ini seharusnya tidak ada. Browser milik pengguna - bukan tempat bermain IT untuk menginstal sesuatu.

Ekstensi apa pun yang dipasang menggunakan kebijakan grup ini adalah perangkat lunak perusak. Untuk menghapusnya, Anda perlu menghapus file ExtensionInstallForecelist kunci registri dari:

HKEY_LOCAL_MACHINE/Software/Policies/Google/Chrome/ExtensionInstallForcelist

Setelah seluruh daftar ekstensi paksa dihapus, sekarang saatnya untuk mencegah siapa pun memasukkan kebijakan grup apa pun ke Chrome. Anda melakukan ini dengan menolak akses semua orang untuk menulis ke node kebijakan Chrome.

Arahkan ke node HKEY_LOCAL_MACHINE\Software\Policies\Google\Chrome di RegEdit, klik kanan, dan pilih Izin :

Klik Menambahkan , masuk Semua orang sebagai wali untuk menambahkan, dan klik baik :

Pilih Semua orang wali, dan klik Maju . Pilih Semua orang wali, dan klik Sunting :

Dari sana Anda mau Menyangkal :

  • Tetapkan nilai : menyangkal
  • Buat subkunci : Tolak

dan klik baik sepanjang jalan.

Sekarang Oracle tidak bisa lagi memaksa file Toolbar Ask.com .

Langkah-langkah di atas akan berfungsi untuk ekstensi paksa apa pun; tidak peduli perusahaan mana yang memasangnya.
