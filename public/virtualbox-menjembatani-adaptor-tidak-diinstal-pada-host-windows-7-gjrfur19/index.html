VirtualBox menjembatani adaptor tidak diinstal pada host Windows 7

Setelah menginstal VirtualBox 4.1.8r75467 pada host Windows 7 64 bit, adaptor jembatan tampaknya belum diinstal. Itu tidak ditampilkan di pengelola perangkat, dan di VB, mengubah Pengaturan> Jaringan> Terlampir ke Adaptor yang Dijembatani hanya menampilkan Tidak Dipilih di menu tarik-turun Nama. Tidak ada pesan kesalahan selama penginstalan, dan ya, saya telah mengonfirmasi selama penginstalan.

Hal-hal yang saya coba:

  • Menggunakan penginstal untuk memperbaiki penginstalan
  • Selesaikan uninstall dan instal ulang
  • Instal ulang dalam mode aman
  • Instal ulang dengan UAC dan anti-virus (Kaspersky) dinonaktifkan
  • Seperti yang direkomendasikan dalam edisi lama sini , mencoba menghapus file ini, tetapi akses ditolak, bahkan dalam mode aman:% SystemRoot% \ System32 \ DriverStore \ INFCACHE.1

Ini adalah masalah pemblokiran, dan saya tidak bisa mencoba. Setiap saran akan sangat dihargai.

Tanggapan ini sini tentang nilai registri MaxNumFilters memperbaiki masalah untuk saya:

Menyelesaikan masalah saya tidak yakin apakah itu menyelesaikan masalah untuk orang lain, Windows 7 saya telah kehabisan filter jaringan, mengubah entri registri untuk pelapor jaringan dari 8 ke 14 memungkinkan menjembatani ke adaptor untuk ditampilkan

HKEY_LOCAL_MACHINE \ SYSTEM \ CurrentControlSet \ Control \ Network \ MaxNumFilters

Saya melihat banyak entri di pengelola perangkat ketika saya menunjukkan yang tersembunyi, jadi saya menetapkan batas maksimal 24, dan penginstalan tampaknya berhasil.

Saya mengalami masalah yang sama, dan sementara beberapa ide di sini pada akhirnya membantu, masalahnya adalah VirtualBox terputus pada adaptor jaringan saya yang sudah dijembatani. Anda tidak menyebutkan detail tentang konfigurasi adaptor Anda.

Inilah yang berhasil untuk saya - YMMV.

  1. Sebelum melakukan setiap langkah, nonaktifkan perlindungan virus. Setelah setiap langkah, reboot.
  2. Tingkatkan jumlah filter bersih: HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Network\MaxNumFilters Saya menaikkan angka ini menjadi 32.
  3. Di Pengelola Perangkat, pilih 'Lihat Adaptor Tersembunyi', dan hapus semua adaptor jaringan "Perangkat tidak dikenal"
  4. Copot pemasangan VB
  5. Di Device Manager, hapus adapter jaringan (tersembunyi) yang tersisa dengan kata "VirtualBox" di judulnya
  6. Langkah kritis: Di Control Panel / Network and Internet / Network Connections klik kanan pada Network Bridge dan buka properti. Dibawah 'Adapters' membatalkan pilihan semuanya. Terapkan, OK, tutup
  7. Instal VB (biarkan pilihan default, yaitu segalanya)

Saya tidak berpikir adaptor yang dijembatani seharusnya muncul di pengelola perangkat, hanya VirtualBox Host-Only Ethernet Adapter - Saya rasa itu juga tidak muncul?

Saya menyarankan untuk mencoba beberapa hal:

Pertama, apakah Anda memverifikasi hash MD5SUM / SHA256 unduhan Anda dengan versi yang disimpan di laman unduhan: https://www.virtualbox.org/wiki/Downloads

Kedua, dapatkah Anda menggunakan Pemulihan Sistem untuk kembali ke keadaan baik yang diketahui sebelum mencoba menginstal VirtualBox?

Ketiga, gunakan snap-in Sertifikat MMC untuk mencari di toko pengguna dan toko komputer Anda untuk sertifikat dari Oracle atau Sun - dan hapus mereka. Ini harus memastikan bahwa Anda mendapatkan prompt saat menginstal menanyakan apakah Anda ingin mempercayai perangkat / vendor. Tentu Anda ingin mempercayai mereka untuk mengaktifkan semua fitur VirtualBox.
