ffmpeg mengedit metadata avi dan penamaan trek audio

Saya memiliki banyak film dalam format avi di disk saya. Namun, mereka tidak mengandung metadata seperti judul dll.

Saya menemukan perintah yang akan melakukannya untuk saya:

ffmpeg -i input.avi -metadata title="Moonshine" -metadata author="Moonshine" -metadata copyright="2009" -metadata comment="foo" -acodec copy -vcodec copy output.avi

Ini berfungsi dengan baik TAPI itu hanya akan menyalin SATU trek audio. File saya berisi lebih dari satu trek audio. Saya ingin ffmpeg menyalin semuanya dan hanya memberi judul pada file tersebut.

Selain itu, adakah cara untuk memberi nama bahasa pada trek audio? Misalnya file berisi 2 track audio. Yang pertama adalah bahasa Inggris dan yang kedua adalah bahasa Rusia. Bagaimana cara menamainya dengan benar sehingga ketika saya menggunakan pemutar video, saya akan melihat di menu bahasa bahwa yang pertama adalah bahasa Inggris dan yang kedua adalah bahasa Rusia?

pemilihan aliran

Aliran default perilaku seleksi adalah memilih satu jenis aliran per input. Misalnya, seperti pada masukan Anda, jika berisi dua aliran audio, maka hanya aliran dengan jumlah saluran terbanyak yang akan dipilih.

Anda dapat memberi tahu ffmpeg untuk menyertakan semua aliran dengan opsi -map.

Karena opsi -map mulai menghitung pada 0, 0 dalam contoh ini merujuk ke masukan pertama (input.avi: satu-satunya masukan Anda dalam hal ini). Jika Anda memiliki masukan tambahan maka -map 1 akan merujuk ke masukan kedua.

mengatur metadata bahasa

Untuk menyetel bahasa streaming audio pertama dan kedua, gunakan opsi -metadata dengan penentu aliran :

ffmpeg -i input.avi -map 0 -codec copy -metadata:s:a:0 language=eng -metadata:s:a:1 language=rus output.avi

Lihat dokumentasi untuk info lebih lanjut.

Butuh waktu cukup lama bagiku untuk memikirkannya. Saya mempostingnya di sini, karena bahkan setelah 3 tahun, utas ini adalah salah satu hit pertama di pencarian google: AVI (lebih spesifik: RIFF) mendukung nama bahasa tetapi berlawanan dengan mkv, metadata tidak disimpan di aliran tetapi di header menggunakan tag IAS1-9 untuk hingga 9 aliran audio yang berbeda.

ffmpeg -i input.avi -map 0 -codec copy -metadata IAS1=eng -metadata IAS2=ger output.avi

VLC cukup toleran. Jika Anda memasukkan kode ISO "ger", VLC menerjemahkannya menjadi "Deutsch", jika Anda memasukkan "MyLang" sebagai gantinya, VLC menampilkan "MyLang". Perangkat lunak lain seperti Kodi membutuhkan kode ISO yang benar. Itu akan membaca "MyL" saja dan kemudian menampilkan bahasa sebagai tidak dikenal.

Namun, perlu diketahui bahwa ffmpeg tidak hanya menambahkan bahasa tetapi juga mengubah metadata lainnya. Misalnya, audio interleave dan pramuat data berbeda di output.avi, tidak tahu apakah ini bagus atau mungkin menyebabkan audio tidak sinkron. Periksa dengan MediaInfo atau serupa.
