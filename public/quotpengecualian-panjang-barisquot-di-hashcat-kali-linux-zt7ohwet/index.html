"Pengecualian panjang baris" di hashcat (kali linux)

Saya telah menerima baris ini:

root:$1$25A116L54$7/IbiL1ZsUyPwYBx/zuNv/

dan diminta untuk memecahkan kata sandi. Jadi, saya membuat file crack.hash dan meletakkan yang berikut ini di dalamnya:

$1$25A116L54$7/IbiL1ZsUyPwYBx/zuNv/

Dan kemudian jalankan perintah berikut di terminal:

hashcat -m 0 -a 0 -o /crackResult.txt --remove /crack.hash /passwords.txt

Dan hasilnya adalah:

skip line: $1$25A116L54$7/IbiL1ZsUyPwYBx/zuNv/ (line length exception)

Apa kesalahan saya?

  • Ini benar-benar menjawab pertanyaan Anda. Kesalahan hashcat 'line-length exception' agak tidak jelas; artinya adalah mengharapkan hash dengan panjang yang berbeda (karena Anda telah memintanya untuk memecahkan mentah MD5 (-m 0) ketika Anda memiliki MD5crypt (-m 500) hash. Ubah sintaks Anda untuk menggunakan -m 500, seperti yang dikatakan mbs dan Jor-el.
