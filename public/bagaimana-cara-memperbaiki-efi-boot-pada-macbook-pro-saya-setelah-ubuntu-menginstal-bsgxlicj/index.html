Bagaimana cara memperbaiki EFI-boot pada Macbook Pro saya setelah Ubuntu menginstal?

MacBook tidak bisa boot. Layar tetap kosong (hitam) - bukan abu-abu, layar aktif seperti biasa. Untuk mematikan, saya menahan tombol daya.

Setelah 5 upaya boot, MacBook akhirnya menampilkan abu-abu, layar aktif dan boot OSX.

Ini dimulai setelah saya menginstal Ubuntu 11.04 pada partisi ke-2.

Berikut ini urutan kejadian dan upaya saya untuk memperbaiki masalah:

  1. OS X pada partisi 1.
  2. rEFIt diinstal dan diatur untuk muncul di setiap boot - semuanya masih berfungsi dengan baik.
  3. Menginstal Ubuntu 11.04 pada partisi 2 - pada akhirnya dikatakan bahwa Grub sedang menginstal pada hard-drive.
  4. MacBook, setelah shutdown, dan restart terdengar nada panjang dan memiliki layar kosong (hitam), tidak menyala (yaitu tidak abu-abu / aktif). Itu tidak bisa boot atau melakukan apapun. Holding Pilihan kunci tidak berfungsi. Setelah 5 siklus daya, MacBook akhirnya melakukan boot ke OSX.
  5. Saya menyinkronkan tabel partisi dengan rEFIt dan shutdown.
  6. Sama seperti langkah 4, setelah banyak reboot saya dapat memuat rEFIt kemudian OS X atau Ubuntu.
  7. Saya menjalankan skrip rEFIt enable-always.sh untuk memberkatinya lagi - hasilnya sama dengan 4.
  8. Saya menggunakan utilitas Disk Startup OS X untuk memilih hard-drive OSX - hasilnya sama dengan 4.
  9. Saya kemudian menggunakan OS X install-DVD untuk memformat hard drive sepenuhnya dan menginstal ulang OS X - hasilnya sama dengan 4.
  10. Banyak upaya lagi dari langkah 5 hingga 9 - hasilnya sama dengan 4.

Rincian lebih lanjut tentang nomor 4 (di atas):

Catatan, Pilihan (AKA Alt ) berfungsi pada boot normal, tetapi setelah / di luar masalah saya.

Sebuah "boot normal" dapat dijelaskan sebagai:

  • OSX dimuat
  • atau, memegang Pilihan tombol menunjukkan pemilihan boot EFI
  • atau, memegang Perintah + Pilihan + P. + R tombol mengatur ulang NVRAM.

Namun, masalah yang saya hadapi adalah "boot normal" gagal untuk 5 siklus daya pertama. (Setelah boot gagal, saya dapat mematikan daya dengan menahan tombol daya.) Selama boot gagal, TIDAK ADA yang berfungsi (tidak ada tombol, tidak ada OSX) - hanya layar kosong.

Ini seperti kode EFI macet atau macet. Tapi pada power-cycle ke-5, "boot normal" selalu berfungsi, seperti beberapa fail-safe yang dipicu. Kemudian setelah saya mematikan OSX, masalah booting berulang - yaitu 5 siklus daya lagi sebelum saya dapat kembali ke OSX.

Perhatikan, pada boot gagal pertama (hanya yang pertama) (satu dari lima) (setelah shutdown) MacBook Pro mengeluarkan nada (menurut saya nada kesalahan).

Ini sistem saya:

  • Pengidentifikasi Model: MacBookPro5,4
  • Nama Prosesor: Intel Core 2 Duo
  • Kecepatan Prosesor: 2,53 GHz
  • Kecepatan Bus: 1,07 GHz
  • Versi Boot ROM: MBP53.00AC.B03
  • Versi SMC (sistem): 1.49f2

Kurangnya penjelasan rinci tentang penginstalan membuat tidak jelas apakah perangkat keras melakukan booting dalam mode EFI atau menggunakan CSM. Jika CD / DVD instalasi dimasukkan, komputer menyala, dan Anda menahan tombol opsi untuk membuka menu disk startup, jika ikon CD bertuliskan "Windows" maka itu akan mengaktifkan boot CSM (emulasi BIOS) dan penginstal akan menginstal bootloader yang bergantung pada BIOS. Jika ikon CD berlabel EFI Boot maka itu adalah mode booting EFI dan penginstal akan menginstal bootloader yang bergantung pada EFI: kemungkinan besar grub.efi, pada partisi Sistem EFI.

Di Macbook Pro 4,1 saya, saya hanya dapat melakukan booting EFI dengan konsol teks. Nouveau dan driver nvidia berpemilik menggantung tampilan grafis saat diaktifkan, meskipun mungkin untuk ssh ke dalam sistem dan berfungsi dengan baik jika tidak. Menggunakan parameter kernel 'nomodeset' Saya berhasil mem-boot teks saja dan menggunakan tampilan bawaan. Menurut bug berikut, pengembang baru mengatakan bahwa mereka tidak dapat memperoleh informasi yang diperlukan dari boot mode EFI, dari implementasi EFI Apple. (Komentar 34).

https://bugzilla.redhat.com/show_bug.cgi?id=650949

Di Macbook Pro 8,2 saya, boot mode EFI menghasilkan teks dan video grafik yang tidak dapat digunakan dan saya tidak dapat melakukan pengujian lebih lanjut.

Dengan kedua model perangkat keras, boot mode CSM berfungsi cukup baik di luar ketersediaan dan penyempurnaan driver perangkat keras tertentu. Tetapi tidak jelas bagaimana pengguna dapat mengetahui apakah mode EFI atau mode CSM boot sedang terjadi. Ini adalah salah satu alasan mengapa saya memutuskan untuk meninggalkan rEFIt karena saya tidak bisa benar-benar tahu apa yang sedang di-boot atau bagaimana, dan itu akan menyajikan opsi untuk boot yang tidak akan berfungsi.

Untuk menghasilkan instalasi distro linux yang menggunakan CSM tanpa bergantung pada rEFIt memerlukan penanganan khusus agar EFI Apple dapat mengaktifkan CSM. Saya telah mengajukannya dalam bug ini.

https://bugzilla.redhat.com/show_bug.cgi?id=746901

Sungguh kita harus melobi Apple untuk dukungan penuh UEFI 2.x, daripada firmware mereka saat ini yang sebagian besar masih didasarkan pada Intel EFI 1.10 (dengan beberapa hal dari spesifikasi UEFI). Menurut pendapat saya, tidak masuk akal, meskipun akan menyenangkan, mengharapkan distribusi linux mendukung implementasi EFI non-standar Apple. Jadi untuk saat ini, saya menggunakan CSM, tapi saya berharap Apple akan menyelesaikan masalah ini.

Bagi mereka yang menggunakan Lion, Anda mungkin akan memiliki partisi "Recovery HD" sebagai partisi ke-3 Anda, jadi partisi 3, 4, dan 5 pada bug di atas akan menjadi 4, 5, dan 6.

Cara terbaik untuk memperbaiki masalah boot dengan rilis final Ubuntu 14.04.

Dengan o tanpa prainstal Ubuntu.

Bekerja dengan rilis final Ubuntu 14.04

Tekan Tombol Daya dan Boot dengan DVD atau USB Instalasi MACOSX.

Buka terminal dan tempel skrip:

bless --device /dev/disk0s1 --setBoot --legacy

Dan kemudian reboot.

Setelah 30 detik, grub 2.0 dari Ubuntu boot, tidak ada masalah.

Versi en español:

Tambahkan masalah ini ke masalah yang ada di MacBook sederhana atau PRO.

Con ubuntu ya instalado, corriendo solo NO dual boot.

Inicie la MacBook con un disco de instalación MACOSX or un USB Booteable.

Escoga el lenguaje y seguido Abra la terminal y ingrese el siguiente script.

bless --device /dev/disk0s1 --setBoot --legacy

Reinicie y espere 30 segundos, el Grub 2.0 de Ubuntu arranca sin problemas.

Saludes y suerte.
