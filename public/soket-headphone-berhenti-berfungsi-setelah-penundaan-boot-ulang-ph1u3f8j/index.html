Soket headphone berhenti berfungsi setelah penundaan / boot ulang

Saya mengalami masalah yang sangat menjengkelkan ini, di mana jack headphone akan berfungsi dengan baik setelah start yang dingin. Tetapi setelah penangguhan atau boot ulang dingin (nyalakan ulang komputer tanpa mematikannya sepenuhnya) jack headphone akan berhenti berfungsi.

Sistem mendeteksi, saat headphone dicolokkan masuk dan keluar, tetapi tidak ada lagi suara yang keluar dari headphone.

Saya mengalami ini hanya di bawah Ubuntu 14.04 (di bawah partisi Windows saya berfungsi dengan baik) pada Laptop XMG A305. Sepertinya masalah ini terkait dengan driver yang digunakan dengan kartu suara saya (Kartu: HDA Intel PCH, Chip: VIA VT1802) atau interaksi antara driver yang berbeda.

Juga, ada beberapa orang lain yang mengalami ini di komputer lain: Laporan bug untuk Laptop Clevo

Setelah mencari cukup lama, saya telah menemukan solusi yang bekerja dengan baik untuk saya dan mungkin juga membantu Anda:

Kode tersebut sebagian besar diambil dari sini .

Dapatkan kodenya

Ada skrip python yang dapat mengaktifkan kembali jack headphone. Kode ini diambil dari ektor5 di GitHub . Anda perlu mengunduh skrip, meletakkannya di suatu tempat di mana sistem Anda menemukannya dan membuatnya dapat dieksekusi. Anda dapat melakukan semua ini dengan baris ini:

sudo wget https://raw.githubusercontent.com/ektor5/init-headphone/master/init-headphone -O /usr/local/sbin/init-headphone && sudo chmod +x /usr/local/sbin/init-headphone

Instal dependensi

Skrip init-headphone membutuhkan dependensi yang dapat diinstal dengan:

sudo apt-get install python-smbus

Tambahkan bendera grub

Selain itu, kernel Anda harus dijalankan dengan mengaktifkan flag tambahan. Anda dapat melakukan ini dengan mengedit file '' / etc / default / grub ''. Ubah baris ini

GRUB_CMDLINE_LINUX=""

untuk ini

GRUB_CMDLINE_LINUX="acpi_enforce_resources=lax"

Setelah itu jalankan

sudo update-grub

Muat modul saat memulai

Selain itu, skrip perlu memuat modul. Yaitu ´i2c_dev´ dan ´i2c_i801´. Ini dapat dimuat saat runtime dengan

modprobe i2c_dev
modprobe i2c_i801

Sekarang Anda dapat menguji apakah ini berfungsi, dengan menjalankan ´sudo init-headphone´.

Untuk memuat modul secara otomatis saat startup, tambahkan baris berikut ke '' / etc / modules '':

i2c_dev
i2c_i801

Buat skrip startup

Terakhir, kami ingin skrip berjalan secara otomatis, ketika komputer kami reboot atau kembali dari penangguhan. Untuk melakukannya, tempatkan skrip berikut di ´ / etc / pm / sleep.d / ´.

sudo gedit /etc/pm/sleep.d/init-headphone

Tambahkan baris berikut

#!/bin/sh

if [ ! -x /usr/local/sbin/init-headphone ]; then
    exit 0
fi

case $1 in
     resume|thaw)
        /usr/local/sbin/init-headphone
       ;;
esac

Dan jangan lupa untuk membuat file tersebut dapat dieksekusi

sudo chmod +x /etc/pm/sleep.d/init-headphone

Saya harap ini membantu, beri tahu saya jika Anda memiliki masalah.

Mengalami masalah yang sama baru-baru ini saat menggunakan Fedora 32. Skrip yang disebutkan @cbandera berfungsi dengan baik. Untuk OS saya, bagaimanapun, otomatisasi perlu dilakukan melalui systemd.

Saya membuat file layanan

sudo gedit /etc/systemd/system/init-headphone.service

dan menempatkan yang berikut ini di file

[Unit]
Description=Initialize headphones upon resume
After=suspend.target hibernate.target hybrid-sleep.target suspsend-then-hibernate.target

[Service]
ExecStart=/usr/local/sbin/init-headphone

[Install]
WantedBy=suspend.target hibernate.target hybrid-sleep.target suspend-then-hibernate.target

Kemudian saya mengaktifkan layanan tersebut

sudo systemctl enable init-headphone

Setelah itu semuanya berhasil.

#!/bin/sh

[ -x /usr/local/sbin/init-headphone ] || exit 0

[ "$1" = "post" ] && /usr/local/sbin/init-headphone
