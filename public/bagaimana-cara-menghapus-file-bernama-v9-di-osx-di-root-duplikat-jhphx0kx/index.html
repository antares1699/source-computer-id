Bagaimana cara menghapus file bernama -v9 di OSX di / root? [duplikat]

Kemungkinan Duplikat:
Unix: File dimulai dengan tanda hubung, -

Pada mesin OSX (10.6.4) saya, entah bagaimana saya berhasil membuat direktori bernama "-v9" di direktori / var / root. Bagaimana cara menghapus direktori ini?

Menggunakan garis miring terbalik untuk menghindari minus sepertinya tidak berhasil:

# rm -r \-v9
rm: illegal option -- 9

Bahkan menggunakan wildcard tidak akan berhasil:

# rm -ri *9
rm: illegal option -- 9
usage: rm [-f | -i] [-dPRrvW] file ...
       unlink file

Saya mencoba menghapusnya menggunakan Finder, tetapi mengalami masalah yang berbeda. Karena berada di direktori / var / root, Finder tidak akan menampilkan konten folder. Setidaknya, itu tidak melakukan ini secara default, dan saya tidak tahu cara mengaktifkannya (ditto untuk Path Finder).

Bagaimana dengan sudo rm -rf /var/root/-v9?

Atau, lihat HiddenFiles Widget, yang memungkinkan Anda memeriksa konten folder tersembunyi.

Edit:

  1. Saya baru tahu bahwa Finder tidak mengizinkan Anda melihat di /var/root, karena izin - ini dimiliki oleh Sistem.
  2. /var sebenarnya adalah jalan pintas ke /private/var, jadi perintah bash di atas mungkin harus membaca sudo rm -rf /private/var/root/-v9
  3. Omong-omong, gunakan saran ini dengan risiko Anda sendiri ;-)

Sunting # 2: Jika Anda menggunakan jalur absolut ke folder Anda, tidak masalah bahwa itu dimulai dengan '-' (setidaknya di mac saya). Namun, jika Anda terlebih dahulu cd ke direktori yang relevan dan mengetik rm -rf -v9 kemudian rm menafsirkan bagian -v9 sebagai argumen (ilegal) yang diteruskan kepadanya.
