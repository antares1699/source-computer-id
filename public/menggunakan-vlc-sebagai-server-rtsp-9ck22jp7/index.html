Menggunakan VLC sebagai server RTSP

Saya mencoba mencari cara untuk menggunakan kapabilitas server VLC. Lebih khusus lagi, cara mengekspor file SDP saat streaming RTP. Di Bab 4 di bagian yang terkait dengan RTP Streaming contoh untuk server dan klien diberikan:

vlc -vvv input_stream --sout '#rtp{dst=192.168.0.12,port=1234,sdp=rtsp://server.example.org:8080/test.sdp}'
vlc rtsp://server.example.org:8080/test.sdp

Tidak begitu jelas bagi saya bagaimana membuatnya benar-benar berhasil. Saya telah mencoba dua perintah ini untuk server dan klien menggunakan dua contoh cmd:

vlc -I rc screen:// --sout=#rtp{dst=127.0.0.1,port=4444,sdp=rtsp://localhost:8080/test.sdp} 
vlc -I rc rtsp://localhost:8080/test.sdp

Memanggil perintah kedua menyebabkan yang pertama macet. Perintah kedua menunjukkan pesan kesalahan "tidak dapat terhubung ke localhost: 8080".

Masalah Anda adalah kemungkinan besar contoh yang Anda lihat adalah untuk streaming file dan perintah contoh Anda menggunakan input screen://. Agar perangkat screen:// berfungsi, Anda perlu menyandikan video terlebih dahulu. Jadi cobalah sesuatu seperti ini:

vlc -I rc screen:// --sout=#transcode{vcodec=h264,vb=800,scale=0.25,fps=10}:rtp{dst=127.0.0.1,port=4444,sdp=rtsp://localhost:8080/test.sdp}
vlc -I rc rtsp://localhost:8080/test.sdp
