Mengetik dokumen secara terprogram - Pilihan selain LaTeX dan Apache FOP

Apakah ada bahasa markup lain yang tidak memiliki kurva belajar yang sulit (saya berhasil mendapatkan kecepatan dalam Lateks setelah sekitar 2 bulan dan masih membuat kesalahan) dan dapat dengan mudah diprogram. Saya melihat Apache FOP, tetapi tampaknya agak menakutkan sekarang. Dan Lateks tidak lagi dipertanyakan.

Yang saya inginkan adalah memberikan input teks biasa dan / atau gambar ke bagian yang telah ditentukan dari dokumen saya dan memiliki program (Java atau C #) untuk melanjutkan dan memformat teks itu sesuai template yang telah saya tentukan dan kemudian mengkompilasi dokumen, ke dalam PostScript atau PDF . Saya tidak menulis versi lain dari Mahabharata atau Odyssey dan tidak memerlukan opsi pemformatan yang rumit. Dokumen-dokumen tersebut dimaksudkan untuk menjadi sangat sederhana dan maksimal 15-20 halaman.

Penerapan ide ini, misalnya, adalah pembuat resume. Saya mendefinisikan format output saya dalam program ini. Kemudian kapan pun saya menginginkan salinan resume saya, saya cukup memilih keahlian, proyek, dll. Yang mana yang harus muncul dan voila, program akan memberi saya PDF persis seperti yang saya inginkan, disesuaikan dengan kebutuhan saya.

Lout sering digambarkan sebagai lebih mudah mempelajari Latex, meskipun YMMV.

Lateks, yang didasarkan pada TEX asli, memiliki beberapa keunikan. Lout adalah awal yang baru dalam merancang sistem seperti itu. Hal-hal yang lebih mudah ditentukan pengguna, lebih masuk akal, mungkin lebih cepat, dll. OTOH, pengguna Latex selama bertahun-tahun telah membangun banyak koleksi tipografi, grafik dan paket plot, paket khusus untuk hukum, sejarah, kimia, musik, dll. . dan beberapa orang mengatakan bahwa penyusunan huruf matematika Latex / TEX sedikit lebih baik daripada Lout, meskipun perlu perhatian khusus dalam penyusunan huruf untuk diperhatikan.

Skrip Python atau Ruby (atau bahasa favorit Anda) dapat mengedit atau membuat file lout.

http://lout.wiki.sourceforge.net/

http://snark.ptc.spbu.ru/~uwe/lout/essays/essays.html

sumber dan file lainnya: ftp://ftp.cs.usyd.edu.au/jeff/lout/

Anda mungkin mempertimbangkan doxygen . Memiliki sangat markup sederhana, penggunaan kombinasi apa pun dari markup lateks, javadoc atau HTML dalam dokumen Anda dan dapat menghasilkan PDF, HTML, RTF, lateks, dan lain-lain. Ini dirancang untuk mendokumentasikan kode sumber, tetapi Anda dapat menginstruksikannya untuk menarik dokumentasi dari file teks yang diformat dengan benar. Anda dapat menyematkan matematika lateks, tetapi Anda memerlukan kompiler lateks yang diinstal untuk memprosesnya.

Kamu bisa gunakan graphviz.dll untuk menambahkan diagram menggunakan markup teks juga.
