Beberapa kode alternatif Unicode tidak berfungsi

Berikut adalah daftar semua karakter khusus Hongaria:

ALT-0193 Á   ALT-0225 á
ALT-0201 É   ALT-0233 é
ALT-0205 Í   ALT-0237 í
ALT-0211 Ó   ALT-0214 Ö   ALT-0336 Ő
ALT-0243 ó   ALT-0246 ö   ALT-0337 ő
ALT-0218 Ú   ALT-0220 Ü   ALT+0368 Ű
ALT-0250 ú   ALT-0252 ü   ALT+0369 ű

Semua bekerja dengan Alt + NUMPAD kecuali kolom ketiga. Saya mendapatkan surat biasa P, Q, p dan q, bukan surat yang diberikan. Apa alasannya ini? Kode apa yang harus saya masukkan jika saya ingin menghubungi mereka?

Jawaban untuk bagian kedua dari pertanyaan Anda adalah, sebagai Microsoft dan George Hernandez menjelaskan, mengetik + (pada keypad numerik) dan kemudian heksadesimal nilai karakter Unicode (UCS-16), semua sambil menahan Alt kunci bawah. Saya tidak memilikinya dari sumber bersertifikat - tapi saya pikir itu Alt + Numpad hanya berfungsi untuk nilai dalam kisaran 0..255. Jadi, semua yang Anda coba dengan nilai yang lebih tinggi 'dilemparkan' ke kisaran ini.
Jadi misal: 336-256 = 80. Karena itu -> Alt + 336 sama dengan Alt + 80 . Yaitu 'P'.

Edit: Itu Alt + Numpad metode ini hanya berfungsi pada halaman kode aktif Anda, jadi metode ini sebenarnya mengetik karakter dalam pengkodean lokal Anda (mis. ISO 8859-16 untuk Hongaria). Jadi pada dasarnya keying Alt + 193 akan mengetik karakter yang panjangnya hanya 1 byte dan memiliki nilai 0xC3 (193 dalam HEX). Secara kebetulan, 193 dalam ISO 8859-16 memiliki nilai tampilan yang sama dengan \u0193 adalah peta Unicode. Di halaman kode lokal saya Alt + 193 menghasilkan karakter ┴ yaitu \u2534 di peta unicode.

Rekomendasi saya, jika Anda perlu menampilkan nilai-nilai ini di beberapa aplikasi, gunakan konvensi unicode alih-alih memasukkannya sebagai Alt + Numpad kombinasi. Kamu bisa mengunduh BableMap untuk ini - ini adalah alat yang hebat untuk tujuan ini dan gratis.
