Dapatkah saya menggunakan sistem file ReFS baru di Windows 8?

Windows Server 2012 hadir dengan Sistem File baru ReFS , yang merupakan peningkatan Microsoft ke NTFS. Apakah ada cara untuk menggunakan ini untuk partisi Windows 8 saya?

Microsoft berencana untuk memutakhirkan ReFS agar mampu menjadi volume boot, tetapi belum ada rentang waktu yang diberikan.

Dengan pemikiran ini, kami akan mengimplementasikan ReFS dalam evolusi bertahap dari Fitur: pertama sebagai sistem penyimpanan untuk Windows Server, kemudian sebagai penyimpanan untuk klien, dan akhirnya sebagai volume boot. Ini sama saja pendekatan yang kami gunakan dengan sistem file baru di masa lalu.

Dalam iterasi saat ini, ini bukan upgrade ke NTFS.

Berdasarkan artikel ini oleh Denny Cherry:

Fitur NTFS yang kami pilih untuk tidak didukung di ReFS adalah: bernama aliran, ID objek, nama pendek, kompresi, enkripsi tingkat file (EFS), transaksi data pengguna, jarang, tautan keras, diperpanjang atribut, dan kuota.

Dalam pikiran saya, ini adalah daftar alasan penting mengapa Anda bahkan tidak perlu mempertimbangkan ReFS untuk waktu dekat.

Jika ini berubah dan mereka mendukung fitur tersebut, maka saya mungkin mempertimbangkan untuk berubah pikiran. Tapi ini adalah model penyimpanan pertama saat ini, dan saya tidak akan menyentuhnya untuk lingkungan desktop.

Dengan Windows 8.1, dimungkinkan untuk mengaktifkan dukungan seluruh sistem untuk ReFS (termasuk manajer disk dan alat format) dengan kunci registri berikut. Ini akan memungkinkan Anda untuk memformat partisi yang sudah ada ke ReFS.

Windows Registry Editor Versi 5.00 [HKEY_LOCAL_MACHINE \ SYSTEM \ CurrentControlSet \ Control \ MiniNT] “AllowRefsFormatOverNonmirrorVolume” = dword: 00000001
