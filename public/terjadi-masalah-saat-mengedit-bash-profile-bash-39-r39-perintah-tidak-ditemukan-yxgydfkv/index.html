Terjadi masalah saat mengedit ~ / .bash_profile: -bash: $ '\ r': perintah tidak ditemukan

Saya menginstal CygWin di Windows 7. Menggunakan Notepad, saya mengedit file ~ / .bash_profile saya untuk ditambahkan ke variabel PATH…

PATH="${PATH}:/cygdrive/c/apache-ant-1.8.2/bin"

Sekarang, ketika saya SSH masuk ke mesin Windows saya, saya mendapatkan kesalahan ini…

-bash: $'\r': command not found
-bash: $'\r': command not found
-bash: $'\r': command not found
-bash: $'\r': command not found
-bash: $'\r': command not found
-bash: /home/dev/.bash_profile: line 39: syntax error: unexpected end of file

dan PATH saya tidak disetel. Adakah yang tahu bagaimana saya bisa memperbaiki ini?

Mungkin notepad telah menambahkan beberapa $'\r', i. e. CR karakter - seperti biasa pada Windows yang memiliki CRLF sebagai terminator baris.

Solusinya adalah dengan menggunakan editor lain yang mampu menjaga akhir baris sebagaimana adanya, e. g. vim untuk Windows, atau Notepad2.

Bergantung pada lingkungan yang Anda gunakan, Anda mungkin terjebak dengan skrip yang berisi CRLF, yaitu saat menggunakan Visual SourceSafe, Perforce untuk Windows, dll. Untuk menyimpan file Anda. Namun, bash memiliki cara universal untuk mengatasi masalah ini:

export SHELLOPTS
set -o igncr

Jika Anda meletakkan ini di awal .bash_profile Anda atau skrip startup lain yang mungkin Anda gunakan, Anda keluar dari masalah, karena urutan CRLF sebelum perintah pertama dalam skrip startup Anda tidak merugikan.
