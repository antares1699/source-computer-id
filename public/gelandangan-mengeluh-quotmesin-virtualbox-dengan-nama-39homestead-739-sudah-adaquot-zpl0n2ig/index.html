Gelandangan mengeluh "Mesin VirtualBox dengan nama 'homestead-7' sudah ada"

Setiap kali saya mencoba menjalankan perintah ini vagrant up, saya menghadapi kesalahan berikut:

VirtualBox machine with the name 'homestead-7' already exists.
Please use another name or delete the machine with the existing name, and try again.

Apa yang saya lakukan, untuk memulai, saya mengkonfigurasi semuanya dengan sempurna, berarti gelandangan saya berfungsi dengan baik. Saya mengatur semuanya di C: drive dengan struktur direktori berikut.

C:/rec (contains my development Laravel code)
C:/recordings/Homestead (contains homestead files)

pada saat itu konfigurasi Yaml saya seperti ini .homestead/Homestead.yaml

---
ip: "192.168.10.10"
memory: 2048
cpus: 1
provider: virtualbox

authorize: ~/.ssh/id_rsa.pub    
keys:
    - ~/.ssh/id_rsa
folders:
    - map: C:\rec
      to: /home/vagrant/Code
sites:
    - map: homestead.app
      to: /home/vagrant/Code/Laravel/public

Sampai keadaan yang disebutkan di atas, semuanya bekerja dengan baik. Tetapi saya memutuskan untuk mengubah folder Homestead ke folder dokumen saya, saya melakukan ini pada my document

git clone https://github.com/laravel/homestead.git Homestead

Ini membuat direktori Homestead dalam dokumen saya seperti ini C:\Users\SweetHome\Homestead.

Saya membuka baris perintah untuk direktori ini dan menjalankan perintah ini ./init.bat Saya membuat file yang diperlukan di direktori .homestead. Tetapi setelah semua ini, ketika saya menjalankan perintah vagrant up di direktori Homestead yang baru dibuat, itu memberi saya kesalahan

VirtualBox machine with the name 'homestead-7' already exists.
Please use another name or delete the machine with the existing name, and try again.

Jika saya menjalankan vagrant up di direktori sebelumnya / lama C:/recordings/Homestead semuanya bekerja dengan baik. Tapi di Homestead baru saya menghadapi kesalahan.

Pandu saya tentang bagaimana saya dapat menyingkirkan Homestead lama dan harus mulai bekerja di Homestead baru. Saya juga mencoba menghapus direktori Homestead yang lebih lama dan menjalankan vagrant up tetapi kesalahan yang sama

Itu Vagrantfile skrip upaya Homestead (dengan memanggil scripts / homestead.rb script) untuk membuat mesin dengan nama default 'homestead-7'. Itu sebabnya gagal, jika kotak dengan nama itu sudah ada.

Anda memiliki 2 cara untuk menyelesaikan ini:

  1. Buka Oracle VirtualBox dan ganti nama mesin yang ada (melalui pengaturannya) dari 'homestead-7' ke nama lain, lalu jalankan kembali perintah vagrant up.
  2. Letakkan baris name: SomeOtherName (ganti 'SomeOtherName' dengan yang Anda inginkan) di Homestead.yaml file mesin baru, dan skrip akan menggunakan nama itu alih-alih 'homestead-7' (tidak didokumentasikan, tetapi lihat ke dalam scripts / homestead.rb menunjukkan bahwa ia pertama kali memeriksa nama yang ditentukan pengguna dan menggunakannya, jika tidak, ia menggunakan nama 'homestead-7' secara default: config.vm.define settings["name"] ||= "homestead-7")
Bagikan
