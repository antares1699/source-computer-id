perfmon meminta saya untuk kredensial NT AUTHORITY \ SYSTEM

Saya telah menyiapkan pengumpul data penghitung kinerja saya sendiri. Itu berjalan sebentar dan sekarang saya ingin mengeditnya. Setelah mengklik OK, saya diminta memasukkan kredensial untuk NT AUTHORITY \ SYSTEM! WTH? Jelas saya tidak tahu ini dan saya pikir bahkan tidak ada kata sandi!

Saya dapat membuat kumpulan pengumpul data baru tetapi setiap kali saya mencoba untuk mengedit beberapa kumpulan meminta saya untuk ini dan saya tidak dapat menyimpan perubahannya.

EDIT: Saya berbicara tentang laptop kerja saya dengan Windows 10 Enterprise build 1903. Saya menggunakannya sebagai orang lain - memiliki akun AzureAD yang juga merupakan anggota grup administrator lokal. PC rumah saya di mana saya melihat perilaku yang sama adalah Windows 10 Pro build 1903. Di sana saya masuk dengan akun pengguna lokal yang merupakan anggota grup Administrator.

Saya berkomentar bahwa NT AUTHORITY \ SYSTEM tidak memiliki kata sandi , jadi kata sandi kosong adalah entri yang benar untuk prompt itu.

Saya hanya dapat memastikan bahwa akun domain Anda bukan anggota grup lokal Pengguna Log Kinerja dan / atau Pengguna Pemantau Kinerja, jadi mungkin bukan Administrator lokal penuh.

Grup ini didefinisikan sebagai:

  • Pengguna Log Kinerja
    Anggota grup ini dapat mengelola penghitung kinerja, log, dan peringatan di komputer - baik secara lokal maupun dari klien jarak jauh - tanpa menjadi anggota grup Administrator.

  • Pengguna Monitor Kinerja
    Anggota grup ini dapat memantau penghitung kinerja di komputer - secara lokal dan dari klien jarak jauh - tanpa menjadi anggota grup Administrator atau grup Pengguna Log Kinerja.

Masalah yang sama ini dibahas di pos Masalah Izin dengan File yang Dihasilkan oleh PerfMon , dimana dikatakan:

Kumpulan Pengumpul Data dapat berisi informasi sensitif tentang komputer, sehingga akses ke sana biasanya mengharuskan pengguna setidaknya menjadi anggota grup Pengguna Log Kinerja.

Solusi di sana, sebenarnya solusi, adalah membuat tugas terjadwal akan aktif saat Kumpulan Pengumpul Data selesai berjalan, untuk mengubah ACL dari struktur direktori secara rekursif menjadi "Semua Orang Kontrol Penuh".

Ada masalah saat membuat pemicu untuk pekerjaan itu, yang kemudian diperlukan pemicu Kustom dimasukkan secara manual sebagai XML:

<QueryList>
  <Query Id="0" Path="Microsoft-Windows-TaskScheduler/Operational">
    <Select Path="Microsoft-Windows-TaskScheduler/Operational">
        *[System[TimeCreated[timediff(@SystemTime) &lt;= 3600000]]]
         and
        *[System[(EventID='102')]]
         and
        *[EventData[Data and (Data='YOUR DATA COLLECTOR SET NAME')]] 
    </Select>
  </Query>
</QueryList>

Pemicu meluncurkan skrip PowerShell ini:

$Path = "C:\PerfLogs\Admin\New Data Collector Set"
$ACL  = (Get-Item $Path).GetAccessControl("Access")
$ACE  = New-Object System.Security.AccessControl.FileSystemAccessRule("Everyone", "FullControl", "ContainerInherit,ObjectInherit", "None", "Allow")
$ACL.AddAccessRule($ACE)
ForEach($_ In Get-ChildItem $Path -Recurse)
{
    Set-Acl -ACLObject $ACL $_.FullName
}

Solusi ini rumit, jadi mungkin lebih sederhana untuk melanjutkan memasukkan kata sandi kosong saat diminta.
