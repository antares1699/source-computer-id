Jalankan perintah berikut:

sudo rm /var/lib/dpkg/lock

Buat folder yang hilang (gunakan perintah ls -l /var/lib/dpkg/ terlebih dahulu untuk menampilkan folder yang ada, jika ada floder tidak membuatnya kembali lagi):

sudo mkdir -p /var/lib/dpkg/{alternatives,info,parts,triggers,updates,methods,available,diversions}

Pindahkan dpkg.status.0 dan perbaiki file base:

sudo cp /var/backups/dpkg.status.0 /var/lib/dpkg/status
sudo apt-get download dpkg
sudo dpkg -i dpkg*.deb
sudo apt-get download base-files
sudo dpkg -i base-files*.deb
sudo apt update && sudo apt upgrade

Periksa dan perbaiki dependensi yang rusak:

sudo apt-get check
sudo apt -f install

  • Pesan kesalahan menanyakan apakah Anda root agar diizinkan menjalankan perintah untuk menginstal perangkat lunak dari terminal. Gunakan sudo sebelum perintah untuk menjalankan perintah dengan hak akses root, seperti ini:

    sudo apt install example-package

    Perintah contoh ini menginstal paket bernama contoh-paket .

  • Jika Anda masih mendapatkan kesalahan penguncian, tutup Pusat Perangkat Lunak Ubuntu (Ubuntu 14.04) atau aplikasi Perangkat Lunak (Ubuntu 16.04) dan terminal untuk membuka kunci /var/lib/dpkg/, buka kembali terminal dan coba perintah lagi.

  • Jika Anda masih mendapatkan kesalahan kunci, jalankan perintah ini untuk menampilkan detail tentang proses penguncian.

    ps -e | grep -e apt -e adept | grep -v grep

    Kemudian buka aplikasi System Monitor dan matikan proses yang memiliki ID proses yang sama seperti hasil dari perintah sebelumnya.

Untuk kesalahan yang disebutkan dalam komentar:

Errors were encountered while processing: /var/cache/apt/archives/libgtk-3-bin_3.18.9-1ubuntu3.1_amd64.deb   
E: Sub-process /usr/bin/dpkg returned an error code (1)

Coba perbaiki dengan menjalankan perintah ini:

sudo apt-get clean  
sudo apt update  
sudo apt upgrade  
sudo dpkg --configure -a  
sudo apt-get -f install

Kemudian coba jalankan kembali perintah yang menghasilkan pesan kesalahan.

Jalankan perintah di bawah ini untuk menghasilkan daftar semua proses yang namanya terdiri dari kata apt, Anda akan mendapatkan daftar yang mencakup semua proses apt atau apt-get dengan menggunakan ps dan perintah grep bersama dengan pipa.

ps -A | grep apt

Untuk setiap proses apt-get atau apt yang dapat Anda lihat pada output dari perintah di atas, bunuh setiap proses menggunakan perintah di bawah ini.

ID proses (PID) ditemukan di kolom pertama dari tangkapan layar di atas.

sudo kill -9 processnumber
or
sudo kill -SIGKILL processnumber

Misalnya, dalam perintah di bawah ini di mana 9 adalah nomor sinyal untuk sinyal SIGKILL, akan menghentikan proses apt pertama:

sudo kill -9 13431
or
sudo kill -SIGKILL 13431

Pertama-tama jalankan perintah di bawah ini untuk menghapus file kunci di direktori /var/lib/dpkg/:

sudo rm /var/lib/dpkg/lock

Setelah itu paksa paket untuk mengkonfigurasi ulang seperti ini:

sudo dpkg --configure -a

Atau, hapus file kunci di /var/lib/apt/lists/ dan direktori cache seperti di bawah ini:

sudo rm /var/lib/apt/lists/lock
sudo rm /var/cache/apt/archives/lock

Selanjutnya, perbarui daftar sumber paket Anda sebagai berikut:

sudo apt update
or
sudo apt-get update
