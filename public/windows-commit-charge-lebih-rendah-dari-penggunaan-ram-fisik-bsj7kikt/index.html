Windows Commit Charge lebih rendah dari penggunaan RAM Fisik?

saya sudah melihat pertanyaan ini dan ini bukan duplikat, terlepas dari judulnya, karena jawaban yang diberikan tidak menjawab pertanyaan saya.

Saat ini saya memiliki sistem Windows XP yang melaporkan penggunaan Komit sistem kurang dari penggunaan Memori Fisik. Pemahaman saya adalah bahwa System Commit adalah jumlah total pagefile ditambah penggunaan RAM. Seperti yang diutarakan oleh Wikipedia:

Jumlah file halaman yang akan digunakan jika semua konten RAM saat ini harus dihapus.

Jadi bagaimana mungkin memiliki penggunaan RAM lebih tinggi daripada jumlah RAM dan penggunaan pagefile?

Jumlah biaya komit tidak termasuk semua memori fisik; itu hanya menghitung semua memori fisik yang dapat dipindahkan ke disk.

Dari Menekan Batas Windows - Memori Virtual (Saya akan merekomendasikan membaca seluruh artikel)

Seperti yang Anda harapkan dari deskripsi jaminan commit, file batas komit adalah jumlah memori fisik dan ukuran paging file. Pada kenyataannya, tidak semua memori fisik diperhitungkan dalam batas komit karena sistem operasi mencadangkan bagian fisik memori untuk digunakan sendiri.

dan

Tidak semua memori virtual yang dialokasikan oleh suatu proses dihitung ke dalam file batas komit. Seperti yang Anda lihat, memori virtual yang dipesan tidak. Virtual memori yang mewakili file pada disk, disebut tampilan pemetaan file, juga tidak dihitung dalam batas tersebut kecuali jika diminta oleh aplikasi copy-on-write semantik, karena Windows dapat membuang data apa pun terkait dengan tampilan dari memori fisik dan kemudian mengambilnya dari file. Memori virtual di ruang alamat Testlimit di mana gambar DLL yang dapat dijalankan dan sistemnya dipetakan karena itu tidak dihitung menuju batas komit. Ada dua jenis memori virtual proses yang dihitung terhadap batas komit: pribadi dan didukung oleh berkas halaman.

Jadi beberapa penggunaan memori OS tidak dihitung untuk batas itu serta file yang dipetakan memori. Di Windows semua file EXE dan DLL dimuat sebagai file yang dipetakan memori. Mereka dimuat ke dalam memori fisik, tetapi karena mereka tidak dimodifikasi setelah memuat (biasanya) Windows tidak mendukung mereka dengan file halaman, karena ia tahu bahwa itu hanya dapat membacanya kembali dari file disk mereka - itulah sebabnya mereka tidak tidak dihitung terhadap batas komit.

  • Satu koreksi kecil, shf301: Anda menulis "commit charge [...] hanya menghitung semua memori fisik yang dapat dipindahkan ke disk." Ini harus ditulis "hanya menghitung semua ruang alamat virtual yang dapat disambungkan ke file halaman." Pertama, commit charge adalah penghitungan memori virtual, bukan fisik. Kedua, adalah mungkin, bahkan umum, untuk file yang dipetakan memori untuk dibaca / ditulis dipetakan. Jika halaman yang berasal dari file tersebut dimodifikasi, maka jika halaman itu harus di-paged ke disk, itu ditulis kembali ke file asalnya, bukan ke pagefile.
    — Jamie Hanrahan
