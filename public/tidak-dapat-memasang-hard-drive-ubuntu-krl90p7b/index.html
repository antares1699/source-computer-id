Tidak dapat memasang hard drive. Ubuntu

Saya tidak dapat lagi memasang hard drive internal ekstra pada sistem Ubuntu 11.04 64-bit saya. Saya bisa sampai beberapa hari yang lalu.

Saya mengalami masalah ini baru-baru ini, dan dapat mengatasinya. Karena sepertinya tidak ada jawaban yang jelas di sini, saya pikir saya akan berusaha memperbaikinya.

Pertama, penting untuk memahami bahwa Anda bisa telah meletakkan sistem file secara langsung di perangkat blok. Dalam hal ini, Anda akan memasang / fsck / etc perangkat, seperti:

/ dev / sda

BUKAN / dev / sda1

Untuk lebih jelasnya, ini berarti Anda dapat memiliki sistem file tanpa tabel partisi. Ini disebut sistem file tanpa partisi, dan sudah ada sejak lama. Berikut adalah utas di stackexchange yang memperdebatkan manfaatnya: https://unix.stackexchange.com/questions/14010/the-merits-of-a-partitionless-filesystem

Selanjutnya, ini tampaknya menjadi masalah yang agak umum, terutama karena Ubuntu memutuskan untuk menyertakan opsi penginstalan langsung ke perangkat.

Masalah muncul karena dalam beberapa kasus penginstalan tanpa partisi merusak rumus yang digunakan EXT4 untuk menghitung ukuran disk. Anda dapat mengetahui apakah Anda mengalami masalah ini jika perbedaan antara ukuran yang diharapkan dan sebenarnya adalah 265 byte. (dan tidak, itu tidak seharusnya 256) Google baru saja memberi tahu saya bahwa 265 adalah ukuran struktur inode EXT4. Sejak menempatkan sistem file pada perangkat kosong didukung, dan dalam beberapa kasus bahkan didorong, saya akan menganggap ini adalah bug. Saya menggunakan kernel versi 2.6.39 ketika itu terjadi pada saya.

Akhirnya, solusinya. Sangat mudah kok.

Pertama, Anda perlu memaksa fsck dari sistem berkas:

fsck.ext4 -f /dev/(your device)

Sekali lagi, untuk perangkat tanpa partisi, Anda akan menggunakan misalnya / dev / sda, bukan / dev / sda1 atau semacamnya.

Selanjutnya Anda perlu mengubah ukuran perangkat agar sesuai dengan apa yang diinginkan ext4:

resize2fs /dev/(your device) ####

Di mana <####> adalah nilai dari pesan kesalahan:

EXT4-fs (sda): geometri buruk: jumlah blok melebihi ukuran perangkat (#### blok)

HARAP DICATAT: penafian biasa berlaku. Mengacaukan filesystem berbahaya, dll. Hati-hati! Ini dimaksudkan sebagai upaya terakhir untuk memperbaiki sistem file, ketika satu-satunya alternatif lain adalah memformat ulang. Saya bukan ahli sistem file, dan tidak tahu apakah perbaikan di atas akan berhasil pada sistem Anda, atau menimbulkan masalah di kemudian hari. YMMV.

Setelah Anda menyelamatkan data Anda, pendekatan teraman mungkin adalah memformat ulang dan membuat tabel partisi standar, jika Anda bisa.

Sangat legal untuk memiliki filesystem seperti ext4 tanpa partisi apapun. Saya punya masalah yang sama dengan Anda.

Jika Anda menjalankan e2fsck pada drive Anda, ia akan mengatakan bahwa ia mungkin memiliki superblok yang buruk dan akan mengusulkan untuk dibatalkan. tekan n untuk melanjutkan, dan itu akan memeriksa sistem file. Jika masalah Anda sama dengan masalah saya, Anda akan melihat bahwa sistem file ada di sana dan sangat bersih.

Anda juga dapat menjalankan tune2fs -l pada /dev/sda dan Anda akan memiliki banyak detail yang menunjukkan bahwa sistem file ada dan sehat, terlepas dari perbedaan ukuran blokir itu.

Saya tidak punya jawaban untuk saat ini. Kemungkinan besar Anda dapat memasang filsystem di Ubuntu 10.10. Saya belum mencobanya, tetapi Anda dapat mencoba memulihkan data menggunakan 10.10 Live CD misalnya.

Cara lain, yang belum saya coba, akan menggunakan resize2fs untuk mengubah ukuran sistem file sehingga ukuran blokirnya sama dengan perangkat.

Pertama, Anda perlu membuka Terminal dan mengedit file / etc / fstab menggunakan perintah berikut:

sudo gedit /etc/fstab

Kemudian tambahkan baris ini di akhir file untuk filesystem ext3:

/dev/sdb1 /media/yourdrive ext3

Tambahkan baris ini untuk sistem file FAT32:

 /dev/sdb1 /media/yourdrive vfat

Sekarang reboot komputer Anda agar perubahan diterapkan.

Perintah mount yang Anda daftarkan tidak lengkap, yaitu:

$ sudo mount /dev/sda /media/openSpaceI

Ini harus seperti (mungkin):

$ sudo mount /dev/sda1 /media/openSpaceI

Karena disk tidak dipasang sekarang, mengapa tidak melakukannya:

$ sudo e2fsck /dev/sdaX

untuk melihat apakah ada kesalahan yang ditemukan dan dapat diperbaiki (di mana X adalah nomor partisi pada /dev/sda).
