Solusi kotor saya adalah chmod +s /sbin/shutdown. Ini berfungsi tetapi ini mungkin bukan praktik yang baik dan tidak aman. Selain itu, setelah beberapa pembaruan sistem, bit suid disetel ulang. Apa cara yang benar untuk melakukannya?

Jika shutdown Anda menerima sakelar -a (periksa dengan shutdown --help), Anda dapat melakukan hal berikut:

  • Tambahkan pengguna yang seharusnya dapat mematikan sistem ke /etc/shutdown.allow.

    Contoh:

    userA
userB

  • Matikan sistem menggunakan sakelar -a.

    Contoh:

    shutdown -a -h now

Sumber: Halaman manual UNIX: shutdown (8)

Saya menggunakan SL 6.4. Ini memiliki versi pengguna poweroff, halt, reboot yang disediakan oleh paket kode pengguna. Saya bisa mematikan, reboot sebagai pengguna biasa (dari baris perintah juga)

    $ which {poweroff,reboot,halt}
    /usr/bin/poweroff
    /usr/bin/reboot
    /usr/bin/halt

    rpm -qf $(which poweroff reboot halt)
    usermode-1.102-3.el6.x86_64
    usermode-1.102-3.el6.x86_64
    usermode-1.102-3.el6.x86_64

Sebagai root

    # which {poweroff,reboot,halt}
    /sbin/poweroff
    /sbin/reboot
    /sbin/halt

    # rpm -qf $(which poweroff reboot halt)
    upstart-0.6.5-12.el6.x86_64
    upstart-0.6.5-12.el6.x86_64
    upstart-0.6.5-12.el6.x86_64
