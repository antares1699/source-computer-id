Bagaimana cara menyimpan perubahan pada pengaturan Panel Kontrol Java?

Saya baru saja mengunduh JRE terbaru di komputer yang menjalankan instalasi Windows 7. Sekarang saya ingin melakukan tiga hal:

  1. nonaktifkan pembaruan otomatis
  2. hapus ikon dari baki sistem
  3. mengurangi jumlah ruang yang digunakan Java untuk file sementara

Saya dapat melakukan 2 dan 3 dengan membuka Panel Kontrol Java (ikon systray> properti, atau Panel Kontrol> Java), menghapus centang pada kotak / mengubah nomor, dan memilih "Terapkan" dan "OK." Namun, saya tidak bisa melakukan 1.

Langkah saya sejauh ini: buka tab Perbarui, hapus pilihan "Periksa Pembaruan Secara Otomatis", klik "Jangan Pernah Periksa" alih-alih "Periksa Bulanan" dan lihat tombol "Lanjutan" berwarna abu-abu. Klik "Terapkan" dan "OK" dan mulai ulang. Setelah mesin di-boot ulang, buka Panel Kontrol Java ... dan "Periksa Pembaruan Secara Otomatis" diaktifkan lagi.

Beberapa tahun yang lalu saya mengalami masalah yang sama dengan versi Java yang jauh lebih awal. Itu akhirnya terselesaikan dengan sendirinya, saya berasumsi karena sudah diperbaiki di rilis selanjutnya. Apakah saya terjebak menunggu perbaikan lagi, atau adakah yang bisa saya lakukan?

Saya tidak benar-benar melihat perangkat lunak saya yang lain mengganggu, tetapi untuk kelengkapan, inilah semua yang telah saya instal sejauh ini:

  • ThinkVantage Toolbox (Saya menggunakan ThinkPad)
  • Symantec Endpoint Protection
  • Firefox 3.6
  • IE 9 beta
  • Pemutar VLC
  • Pidgin

Ini adalah masalah izin pengguna dan tampaknya telah (kembali) diperkenalkan beberapa pembaruan JRE yang lalu. Untuk menyimpan pengaturan Anda, Anda harus menjalankan Panel Kontrol Java dalam mode administrator.

Temukan javacpl.exe di subdirektori bin dari instalasi JRE Anda, dan gunakan menu konteks untuk menjalankannya sebagai administrator. Setiap perubahan yang Anda buat pada pengaturan sekarang harus disimpan saat Anda menerapkan dan menutup jendela.

Simpan ini sebagai Java32_Fix.reg dan jalankan dan itu akan memperbaiki javacpl.exe sebagai masalah Administrator / Panel Kontrol sekali dan untuk semua:

Windows Registry Editor Versi 5.00

[HKEY_LOCAL_MACHINE \ SOFTWARE \ Microsoft \ Windows NT \ CurrentVersion \ AppCompatFlags \ Layers] "C: \ Program Files \ Java \ jre6 \ bin \ javacpl.exe" = "RUNASADMIN"

Bergantian, ini untuk 64bit OS yang menjalankan 32bit Java:

Windows Registry Editor Versi 5.00

[HKEY_LOCAL_MACHINE \ SOFTWARE \ Microsoft \ Windows NT \ CurrentVersion \ AppCompatFlags \ Layers] "C: \ Program Files (x86) \ Java \ jre6 \ bin \ javacpl.exe" = "RUNASADMIN"
