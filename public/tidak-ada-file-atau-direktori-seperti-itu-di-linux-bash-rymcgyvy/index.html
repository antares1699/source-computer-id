:~/Desktop$ xle
bash: /home/yasaman/Desktop/XLE/bin/xle: No such file or directory

Ini dengan jelas menunjukkan bahwa file tersebut pada PATH Anda, dan memiliki izin eksekusi.

Penyebab paling sering dari kesalahan ENOENT dalam kondisi ini adalah loader dinamis yang hilang atau rusak. Kesalahan not a dynamic executable dari ldd adalah indikasi lain bahwa itulah yang terjadi.

Ketika kernel execve adalah sebuah eksekusi dinamis, itu tidak benar-benar menjalankan eksekusi itu sendiri. Sebaliknya, ini menjalankan pemuat dinamis, dan pemuat benar-benar mengatur agar dapat dieksekusi untuk memulai, setelah memetakan semua pustaka bersama yang diperlukan.

Apa yang readelf -l xle katakan? Apakah jalur yang ditentukan dalam segmen INTERP ada? Apakah itu bisa dieksekusi?

Karena file tersebut adalah 32-bit ELF yang dapat dieksekusi, apakah Anda mungkin menggunakan 64-bit hanya sistem? Jika demikian, Anda harus memasang paket libc6-i386.
