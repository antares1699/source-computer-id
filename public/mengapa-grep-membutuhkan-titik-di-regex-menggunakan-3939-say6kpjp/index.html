Mengapa grep membutuhkan titik di regex menggunakan '$'?

Saya pikir saya tahu grep, tapi mungkin tidak.

Saya ingin menemukan semua baris dalam file yang diakhiri dengan ':' If I run

grep :$ ~/greptester.txt

tapi yang mengejutkan saya, tidak ada hasil. Kadang-kadang saya bingung dengan '^' dan '$', harus menebak mana yang dimulai dan mana yang merupakan akhir baris, tapi saya memeriksa dan $ memang akhir baris.

Setelah banyak mengacau, saya tidak sengaja menemukan itu berjalan

grep :.$ ~/greptester.txt

memberikan hasil yang diharapkan. Mengapa?

Ini file teksnya:

test line one
1 line with a colon:
ignore this line
3456 some stuff:
cat: meow; dog: bark; horse: four (4) legs.
goat, 7 elephants

Ini terjadi baik di Ubuntu maupun di mesin Windows dengan Cygwin.

File Anda memiliki akhiran baris \r\n (CR + LF) (kemungkinan besar, dibuat di windows?), Sedangkan kebanyakan file UNIX hanya berakhiran \n (LF).

Jadi, sebelum grep melihat \n setelah karakter :, ada \r yang harus dicocokkan dengan karakter pengganti ..
