Bisakah OS tamu merusak OS host?

Misalkan, saya menggunakan Debian versi tidak stabil atau eksperimental sebagai OS tamu. Setelah menginstal perangkat lunak baru, itu benar-benar merusak sistem. Bisakah itu juga mempengaruhi OS host?

Bergantung pada mesin virtual, itu bisa dimungkinkan.

Misalnya, jika perangkat lunak VM mengaktifkan akselerasi 3D, maka itu memberi tamu lebih banyak akses ke kartu grafis Anda daripada yang Anda harapkan. Ada juga beberapa bug dalam berbagai perangkat lunak virtualisasi yang memungkinkan tamu mendapatkan hak istimewa yang sama dengan tuan rumah.

Tetapi meskipun ini dapat digunakan oleh perangkat lunak berbahaya yang berjalan di tamu, hal ini sangat tidak mungkin terjadi secara tidak sengaja. Jika Anda hanya menghancurkan OS tamu dengan cara tertentu, tidak mungkin hal itu dapat memengaruhi host.

Sebagian besar jawaban di sini membahas mesin virtual itu sendiri (seperti halnya pertanyaan Anda) - tetapi perhatian sebenarnya terletak pada perangkat lunak virtualisasi. Kekhawatirannya adalah bahwa perangkat lunak virtualisasi memiliki kerentanan dalam lingkungan tervirtualisasi, atau tidak melindungi sistem operasi host. Sebagian besar produk virtualisasi yang tersedia cukup solid dan memiliki rekam jejak stabilitas dan keamanan yang baik.

Dalam keadaan normal, tidak. Mesin virtual benar-benar terpisah dari mesin host.

Tetapi terkadang dimungkinkan untuk berbagi folder antara tuan rumah dan tamu, dan dengan cara itu dimungkinkan untuk melakukan hal-hal berbahaya kepada tuan rumah. Atau Anda dapat menghubungkan tamu ke tuan rumah melalui internet dan melakukan hal-hal buruk dengan cara itu. Tetapi sebaliknya, cukup aman untuk bereksperimen di mesin virtual.

Catatan: tergantung pada perangkat lunak virtualisasi, sistem file tamu disimpan dalam satu file di suatu tempat di host. Tetapi tidak mungkin bagi tamu untuk dengan mudah keluar dari kotak pasir yang disediakan perangkat lunak virtualisasi.
