Tab "Kebijakan" Driver tidak muncul di Windows

Saya memiliki driver Marvell yang di versi Windows sebelumnya menunjukkan tab "Kebijakan" di Properties-nya di Device Manager, tetapi tidak lagi di Windows 8 / 8.1.

Ini seperti ini:

Meskipun merender dengan baik di Windows 7 dan Vista, di Windows 8 tab baru ini tidak ada. Saya menggali dan penyebab utama (?) Adalah mv91xxm.dll, yang menangani kotak dialog. Sebuah decompiler (NikPEViewer) menunjukkan kepada saya a Ikon MFC , sebuah ekspor disebut ESATAPortPropPageProvider, dan nama aslinya adalah hotplug.exe sekali. Para programmer malas di Marvell lupa memperbarui file ini sejak 2009.

Kegunaan utama dialog ini adalah untuk mengaktifkan Mode Cache Driver di Marvell Controller dan menonaktifkan perilaku "Safely Removal" di Windows (untuk drive yang terhubung ke pengontrol).

Saya tidak tahu cara memperbaiki masalah, juga tidak tahu apakah memperbarui atau membuat kunci registri (saya kira ada di HKLM\System\CurrentControlSet\Services\mvs91xx) atau mengedit beberapa file apa pun itu.

Segala jenis bantuan sangat dihargai.

catatan: Saya mengupload driver ke Mega , jika Anda ingin menggali.

[Perbarui 01-05-2014]

Saya mengirim email ke Marvell melalui situs resminya , dan meskipun mereka adalah pembuat driver, mereka tidak menyediakan dukungan untuk driver yang tidak tersedia untuk diunduh di situs web mereka

Kepada pelanggan yang terhormat, Satu-satunya driver yang disetujui yang dirilis Marvell untuk pengguna akhir terletak di situs web kami:

http://www.marvell.com/support/downloads/search.do

Jika driver yang anda cari tidak ada disini, maka kami tidak menyediakannya.

Driver yang terdapat di situs web kami adalah satu-satunya dukungan yang kami sediakan untuk pengguna akhir> (pelanggan yang tidak memiliki NDA dengan Marvell).

Jika Anda memiliki kebutuhan dukungan lainnya, silakan hubungi pabrik komputer Anda. Terima kasih

Menggunakan trik untuk menambahkan kunci registri TreatAsInternalPort tidak berfungsi, pada dasarnya karena port tidak dikelola oleh driver Microsoft default untuk AHCI. Naluri saya mengatakan itu juga karena Microsoft mengubah nilai konfigurasi driver ini di Windows 8.

Informasi perpustakaan

Meskipun informasi hak cipta tidak diperbarui sejak 2009, file tersebut terakhir diubah pada Oktober 2013 menurut stempel waktunya. Ini tampaknya dikonfirmasi oleh direktori build yang tertanam dalam file:

d:\project\2013_10\magni_svn\windows\win\sys\amd64\mv91xxm.pdb

Ekstensi file .pdb adalah singkatan dari Database Program , format milik Microsoft yang menyimpan informasi debugging. Bahkan ada juga string msvcrt.dll, yang berarti pustaka tersebut dikompilasi menggunakan beberapa versi Microsoft Visual C ++ .

Sebagai catatan tambahan, mereka menggunakan (d) Apache Subversion sebagai versi perangkat lunak dan sistem kontrol revisi.

Kompatibilitas Windows 8.x.

Karena pustaka diperbarui pada Oktober 2013, masuk akal untuk berpikir bahwa mereka akan menyediakan kompatibilitas dengan Windows 8, dan mungkin juga Windows 8.1. Memeriksa mvs91xx.cat file katalog , berikut detail yang terkait dengan file library mvs91xx.dll:

Tag                     9B1349794EE18ECCCBBD7F38FEEA9D44CC42746E
Thumbprint algorithm    sha1
Thumbprint              9b 13 49 79 4e e1 8e cc cb bd 7f 38 fe ea 9d 44 cc 42 74 6e
OSAttr                  2:6.1,2:6.2,2:6.3
File                    mvs91xx.dll

OSAttr:2:6.0

Atribut OSAttr menentukan versi Windows target yang persyaratan penandatanganannya kompatibel dengan paket pengemudi . Nilai atribut menentukan yang berikut:

  • Nilai 2 menentukan file katalog kompatibel dengan versi sistem operasi Windows berbasis NT.
  • Nilai 6.0 menentukan file katalog kompatibel dengan Windows Vista.

Catatan Jika paket pengemudi kompatibel dengan beberapa versi Windows, Anda harus menggunakan opsi CATATTR terpisah untuk menentukan atribut OSAttr untuk setiap versi Windows.

Sumber: Membuat File Katalog untuk Paket Driver Non-PnP

Dalam hal ini, nilai OSAttr mencakup tiga versi Windows berbasis NT: Windows 7 (6.1), Windows 8 (6.2), dan Windows 8.1 (6.3). Ini berarti paket driver tersebut kompatibel dengan Windows 8 / 8.1, setidaknya di atas kertas; masih Kebijakan panel tidak tersedia di Windows 8.x. Apakah sistem operasi atau file pustaka yang harus disalahkan, saya tidak akan tahu.

Fungsi registri

Pengaturan driver biasanya disimpan di registri, dan perpustakaan menyertakan beberapa Windows fungsi registri :

RegSetValueExA
RegQueryValueExA
RegCloseKey

Mencari string terkait kebijakan, saya menemukan DriverPolicySet. Dengan rekayasa balik perpustakaan saya mengkonfirmasi bahwa fungsi registri memang digunakan, dan bahwa string yang disebutkan di atas adalah bagian dari jalur registri yang dibaca atau ditulis.

Perubahan kebijakan manual

Mencari "DriverPolicySet" di web memberikan hasil yang sedikit, namun menarik. Secara khusus a posting di papan Jepang dan a membalas di forum Rusia . Dengan menerjemahkan dan mengumpulkan potongan-potongan informasi, saya bisa mendapatkan gambaran yang lebih jelas - secara harfiah.

Nilai DriverPolicySet adalah string (REG_SZ) yang dapat disetel ke:

  • 0 - Nonaktifkan semua kebijakan.
  • 1 - Aktifkan hanya cache driver.
  • 2 - Aktifkan penghapusan yang aman saja.
  • 3 - Aktifkan cache driver dan penghapusan aman.

Biasanya disimpan di kunci registri berikut:

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Class\{4D36E97B-E325-11CE-BFC1-08002BE10318}\0001

Catatan

  • Jika nilai registri tidak ada, Anda harus membuatnya sendiri.

  • Anda harus memulai ulang sistem untuk menerapkan perubahan.

  • {4D36E97B-E325-11CE-BFC1-08002BE10318} adalah kelas perangkat GUID ditugaskan ke pengontrol SCSI dan RAID. Ini didefinisikan dalam file mvs91xx.inf.

  • Bagian penutup 0001 mungkin sedikit berbeda tergantung pada jumlah pengontrol yang tersedia. Misalnya, mungkin 0000, atau 0002, dll. Sebagai gantinya.

Penjelasan

Seperti yang Anda katakan, mereka tidak memperbaruinya ketika mereka mem-porting driver ke Windows 8. Meskipun demikian, itu juga mungkin (bahkan mungkin mungkin ) bahwa mereka memilih untuk menghapus dari driver Windows 8 karena bug dan / atau ketidakcocokan dengan Windows 8 dan / atau karena desain antarmuka Windows 8 yang berbeda (biasanya Anda tidak melihat ikon area pemberitahuan secara default).

Larutan

Taruhan terbaik Anda adalah dengan sederhana kontak Marvell dan beri tahu mereka bahwa tab tersebut hilang dan minta mereka memperbarui driver untuk memperbaikinya. Entah mereka akan melakukannya, atau mereka akan menjelaskan secara spesifik mengapa mereka menghapusnya.

Bekerja di sekitar

Sementara itu, jika Anda memerlukan fungsionalitas halaman, Anda dapat menentukan dengan tepat entri registri mana yang disetel jika Anda memiliki akses ke sistem yang tersedia (mis., Di Windows 7):

  1. Lari Monitor Proses
  2. Setel untuk menyaring semuanya kecuali registri
  3. Hentikan pengambilan dan hapus hasilnya
  4. Buka Kebijakan tab
  5. Alihkan kotak centang
  6. Beralih ke ProcMon dan mulai tangkap
  7. Cepat beralih kembali ke tab
  8. Klik [Apply]
  9. Cepat beralih kembali ke ProcMon dan hentikan penangkapan
  10. Periksa perubahan registri untuk menemukan yang relevan

Solusi Alternatif

Jika Anda tidak lagi memiliki akses ke sistem yang kompatibel (yaitu, Anda tidak dapat menginstal salinan sementara Windows 7 di sistem Anda), maka Anda dapat mencoba alternatif (dan vendor-independen ) solusi yang disebutkan di halaman tempat Anda menemukan tangkapan layar yang Anda poskan di pertanyaan Anda (orang yang mengeposkan belum mengujinya di Windows 8, tetapi ada baiknya jika Anda tidak sabar menunggu tanggapan dari Marvell):

  1. Arahkan ke HKLM\SYSTEM\CurrentControlSet\services\msahci
  2. Buat kunci bernama Controller0
  3. Buat subkunci bernama Channel0
  4. Buat DWORD bernama TreatAsInternalPort dan setel ke 1
  5. Ulangi untuk Channel1, Channel2, dll., Dan jika perlu untuk Controller1, dll.
  6. Mulai ulang

Berikut ini skrip registri dengan setelan default yang dikatakan oleh orang-orang di untaian tersebut berfungsi untuk mereka:

REGEDIT4
[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\services\msahci\Controller0\Channel0]
"TreatAsInternalPort"=dword:00000001
[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\services\msahci\Controller0\Channel1]
"TreatAsInternalPort"=dword:00000001
[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\services\msahci\Controller0\Channel2]
"TreatAsInternalPort"=dword:00000001
[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\services\msahci\Controller0\Channel3]
"TreatAsInternalPort"=dword:00000001
[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\services\msahci\Controller0\Channel4]
"TreatAsInternalPort"=dword:00000001
[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\services\msahci\Controller0\Channel5]
"TreatAsInternalPort"=dword:00000001
