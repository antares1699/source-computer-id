Ini bukan jawaban otoritatif, tetapi tampaknya tidak ada cara untuk membedakan antara Ctrl + Shift + <letter> dan Ctrl + <letter> di gvim.

Saya menguji ini menggunakan gvim 7.3.138 untuk Windows dan gvim 7.3.189 untuk X di Linux. Saya memasuki mode penyisipan, mengetik Ctrl-V untuk memasukkan kunci berikutnya secara harfiah (lihat ": help i_CTRL-V"), lalu mengetik Ctrl-V. Hasilnya muncul sebagai ^V. Saya mengulangi percobaan, mengetik Ctrl-Shift-V di akhir. Hasilnya muncul lagi sebagai ^V.

Ada juga item daftar tugas (": help todo.txt") yang menyarankan bahwa menangani dua kasus secara terpisah tidak diterapkan tetapi sedang dipertimbangkan.
