"Mengikat" server DNS dan mempertimbangkan "/ etc / hosts"

di LAN saya, saya menggunakan mesin kecil sebagai server / gateway (dengan Ubuntu 10.04). Di server ini berjalan bind9 Server DNS dan semua klien LAN menggunakannya sebagai DNS unik. Semuanya bekerja dengan baik, kecuali saya ingin menemukan metode untuk dipertimbangkan server /etc/hosts file.

Misalnya, jika file /etc/hosts server saya berisi baris:

192.168.10.5    testserver05.lan

Saya ingin mencerminkan jawabannya mengikat . Nyatanya sekarang saya mengerti tidak ditemukan kesalahan:

root@nasinsu:/etc/bind# nslookup testserver05.lan
Server:     127.0.0.1
Address:    127.0.0.1#53

** server can't find testserver05.lan: NXDOMAIN

Apakah ini mungkin?

Apakah Anda benar-benar melayani zona nyata di bind, atau apakah itu hanya bertindak sebagai cache / forwarder? Jika ini hanya cache / forwarder, maka Anda mungkin ingin melihatnya DNSMASQ . DNSMASQ memiliki server DNS caching sederhana, dan dapat merujuk file host lokal, dan file lain untuk menjawab permintaan.
