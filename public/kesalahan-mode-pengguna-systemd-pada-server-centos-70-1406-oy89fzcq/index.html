kesalahan mode pengguna systemd pada server CentOS 7.0 (1406)

Saya mencoba menjalankan mode pengguna systemd di server CentOS 7.0 (tidak ada X11 yang diinstal). misalnya

systemctl --user start hw.service

itu bereaksi:

Failed to get D-Bus connection: Unable to autolaunch a dbus-daemon without a $DISPLAY for X11

Jika saya menambahkan

export DISPLAY=:0

dan jalankan lagi, itu melaporkan:

Failed to get D-Bus connection: /bin/dbus-launch terminated abnormally without any error message

Adakah yang bisa menjelaskan ini? btw, systemctl - laporan versi:

systemd 208
+PAM +LIBWRAP +AUDIT +SELINUX +IMA +SYSVINIT +LIBCRYPTSETUP +GCRYPT +ACL +XZ

Terima kasih!

CentOS tidak mendukung "systemd --user". Paket mereka langsung menghapus bagian systemd itu. (Perhatikan bagaimana tidak ada user@.service, antara lain.)

Meskipun demikian, pesan kesalahan peluncuran otomatis libdbus menyesatkan, dan $DISPLAY Anda salah.

Pesan kesalahan benar-benar mengatakan "[systemctl tidak dapat menjangkau systemd di alamat pribadi, jadi ia meminta libdbus untuk menggunakan bus sesi, tetapi] tidak ada alamat bus sesi yang ditetapkan secara eksplisit, jadi libdbus mencoba meluncurkannya secara otomatis, tetapi tidak bisa lakukan itu juga ".

Sekarang, jika Anda itu mencoba menggunakan peluncuran otomatis sesi D-Bus, lalu hanya menyetel $ DISPLAY tidak akan cukup - server X11 seperti Xorg akan diharapkan sudah berjalan di: 1 atau semacamnya (dalam hal ini, $ DISPLAY sudah akan disetel).

Tetapi systemctl --user tidak benar-benar menginginkan peluncuran otomatis - ia mengharapkan untuk menemukan systemd di lokasi tertentu, /run/user/$UID/bus dan /run/user/$UID/systemd/private - jadi meluncurkan bus baru akan sia-sia karena tidak akan ada systemd di sana. Jadi versi terbaru tidak lagi menggunakan peluncuran otomatis (atau libdbus dalam hal ini).
