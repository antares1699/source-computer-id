snap=snapshot.file   
touch snapshot.file-1

$ [ -a $snap-1 ] && echo yes

Jika Anda menjalankan test atau [ pada bash, itu sebenarnya mungkin versi built-in, dan bukan versi coreutils pada /usr/bin:

$ type test
test is a shell builtin
$ type [
[ is a shell builtin

Meskipun demikian, tampaknya versi coreutils mengimplementasikan -a dan -e, dengan perilaku yang persis sama. Mungkin -a tidak tercermin di halaman manual karena memang tidak standar , jadi mungkin itu ditambahkan kemudian dan orang itu lalai memperbarui halaman manual yang sesuai. Tapi saya tidak bisa mengatakan saya tahu sejarah di balik mengapa itu ditambahkan (atau bahkan apa singkatan dari a).

-a adalah operator AND untuk menggabungkan kondisi, jadi berikut ini menunjukkan ya jika kedua direktori ada:

[-e / root -a -e / usr] && echo ya

Penggunaannya dengan satu kondisi yang secara default untuk uji keberadaan sepertinya mempertahankan perilaku lama tetapi saya tidak dapat menemukannya di halaman manual lama.

Referensi (gulir ke bawah beberapa setelah info coreutils 'test invocation'):

16.3.6 Konektor untuk `tes '

Penghubung logis biasa.

`! EXPR ' Benar jika EXPR salah.

`EXPR1 -a EXPR2 ' Benar jika EXPR1 dan EXPR2 adalah benar.

`EXPR1 -o EXPR2 ' Benar jika EXPR1 atau EXPR2 benar.
