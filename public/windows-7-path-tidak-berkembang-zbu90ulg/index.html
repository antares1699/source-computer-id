Windows 7 PATH tidak berkembang

Saya menggunakan berikut ini untuk membuat dan mengedit variabel lingkungan untuk Windows 7.

Control Panel\All Control Panel Items\System -> Advanced system settings -> Environment Variables

Di bawah System variables saya memiliki variabel terkait berikut:

PROG32=C:\Program Files (x86)
REALDWG_SDK_DIR=%PROG32%\Autodesk\RealDWG 2011
Path=%REALDWG_SDK_DIR%;%PROG32%\Haskell\bin

Namun, hal berikut terjadi:

C:\>echo %PROG32%
C:\Program Files (x86)

C:\>echo %Path%
%REALDWG_SDK_DIR%;C:\Program Files (x86)\Haskell\bin

Apakah mungkin untuk mengembangkan rantai variabel?

Jika saya mengganti nama Path menjadi yang lain, terkadang saya mendapatkan masalah, dan terkadang tidak.

Edit: Setelah bereksperimen dengan ini sedikit lagi, masalah sebenarnya tampaknya terkait dengan nama variabel - mungkin semacam bug di parser yang memperluas variabel bersarang. Tampaknya variabel bersarang yang Anda tambahkan harus datang sebelum variabel tingkat atas jika diurutkan menurut abjad.

Sebagai contoh, mengubah REALDWG_SDK_DIR menjadi OEALDWG_SDK_DIR akan berfungsi, seperti halnya BEALDWG_SDK_DIR, tetapi PEALDWG_SDK_DIR tidak akan berfungsi, begitu pula ZEALDWG_SDK_DIR. Alternatifnya, mengganti nama PATH menjadi SATH akan berfungsi, tetapi RATH tidak.

Jadi, solusi untuk masalah Anda adalah dengan menggunakan sesuatu yang dimulai dengan huruf sebelum P, atau secara manual ketikkan bagian pertama jalur.

Saya ingin tahu apakah ini mungkin terkait dengan kerusakan ini di Pangkalan Pengetahuan Microsoft:

http://support.microsoft.com/kb/329308

Saya mengalami masalah yang sama dengan variabel lingkungan pada beberapa sistem XP (tidak hanya% APPDATA%, seperti artikel KB tentang). Menggunakan berbagai metode untuk mengatur variabel lingkungan, termasuk keduanya yang disebutkan di artikel KB, tidak membantu ... begitu juga dengan setx (alat Kit Sumber Daya Windows).

Namun, metode yang sama persis bekerja pada sistem lain. Ini sangat membuat frustrasi - satu-satunya solusi yang saya temukan tampaknya menggunakan jalur eksplisit daripada referensi variabel lingkungan (yang menyebalkan, tetapi berfungsi).

Misalnya,

MYVAR=C:\something
PATH=C:\something\bin

daripada:

MYVAR=C:\something
PATH=%MYVAR%\bin

EDIT: Sebelumnya, saya baru saja menggunakan mklink ( http://technet.microsoft.com/en-us/library/cc753194.aspx ) untuk membuat Persimpangan (pada dasarnya tautan simbolis) daripada memikirkan variabel lingkungan dan buggieness mereka. Misalnya, daripada MYVAR sebagai env var, saya hanya membuat c:\myvar - ini dapat mengarah ke E:\some\really\long\and\hard\to\remember\let\alone\type\path atau apa pun yang Anda inginkan. mklink konsisten, dapat dibuat melalui baris perintah, dan tetap ada di antara boot ulang. Misalnya:

mklink /j c:\myvar c:\something
mklink /j c:\myvar E:\some\really\long\and\hard\to\remember\let\alone\type\path
