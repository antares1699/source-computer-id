Setiap kali saya melakukan hibernasi Windows 7 dan kemudian melanjutkannya nanti, seluruh sistem tampak baik-baik saja untuk sementara waktu, kemudian semuanya menjadi lambat hingga sistem membutuhkan waktu sekitar 10 detik untuk memuat halaman web yang dimuat secara instan sebelumnya.

Lihat LED HDD di sistem Anda. Saya berani bertaruh bahwa selama beberapa menit setelah melanjutkan dari hibernasi, LED HDD berkedip dengan cepat (atau bahkan menjadi solid).

Ini normal. Ketika Windows melanjutkan dari hibernasi, itu tidak membaca seluruh file hibernasi (yang bisa berukuran beberapa gigabyte) ke dalam memori. Sebaliknya, ia memuat bagian-bagian yang penting untuk bangkit dan berjalan kembali (kernel, memori driver, layanan, program aktif, dll.) Dan memuat sisanya di latar belakang. Dengan cara ini, Anda dapat langsung mulai bekerja lagi alih-alih menunggu satu atau dua menit.

Secara alami ini berarti bahwa sistem akan macet sementara Windows terus memuat sisa sistem di latar belakang. Tentu saja ia berusaha sebaik mungkin untuk bersikap responsif mungkin, tetapi tidak mengherankan, ini lebih mudah diucapkan daripada dilakukan. Ini terutama benar karena aktivitas terus memaksanya ke jalur samping dari pemuatan latar belakang untuk memprioritaskan sesuatu yang lain, jadi misalnya, jika Anda tidak meminimalkan program, Windows harus melakukannya. mencoba untuk berhenti memuat apa pun yang sedang dikerjakannya dan dengan cepat memulihkan memori untuk program lain itu, tetapi mungkin harus menyelesaikan operasi saat ini sebelum pelacakan samping.

Saya sudah mencoba sfc / scannow, tidak ada bedanya.

Karena tidak ada yang salah dengan drive.

Adakah yang bisa dilakukan untuk menghentikan hal ini terjadi?

Selain mendapatkan lebih banyak memori dan drive yang lebih cepat (mis., SSD), hal terbaik yang dapat Anda lakukan untuk meningkatkan kinerja pasca-hibernasi adalah keluar dari sebanyak mungkin program dan mengosongkan memori sebanyak mungkin sebelum hibernasi. Jelas ini agak mengalahkan tujuan hibernasi, tetapi tidak ada yang gratis, dan jika Anda menginginkan kinerja di tujuan ini, maka Anda harus membayar untuk itu, dan sebaliknya.

Sebagai bonus, jika Anda mengurangi jumlah program yang sedang berjalan dan menggunakan memori sebelum hibernasi, Anda juga mendapatkan keuntungan bahwa Windows melakukan hibernasi lebih cepat karena tidak perlu menyimpan banyak memori ke disk.

Juga Artikel MSDN Bob itu tersebut menjelaskan alasan lain mengapa melanjutkan lambat (lebih daripada hibernasi):

Peningkatan boot loader juga memberikan peningkatan serupa dalam waktu resume hibernasi, terutama dengan menyederhanakan jalur I / O yang digunakan oleh Ntldr untuk membaca gambar hibernasi. File hibernasi dikompresi saat ditulis, dan untuk efisiensi, algoritme kompresi tumpang tindih dengan file I / O. Namun, saat melanjutkan dari hibernasi, Ntldr menggunakan BIOS untuk melakukan I / O; Oleh karena itu, tidak layak untuk tumpang tindih disk I / O dibaca dengan dekompresi.

…

Untuk mengoptimalkan proses hibernasi di Windows XP, beberapa perbaikan telah diterapkan. Algoritme kompresi telah dioptimalkan untuk mengompresi dan mendekompresi blok besar (64K) data. Selain itu, kompresi tumpang tindih dengan penulisan disk. Saat blok data saat ini sedang ditransfer ke disk, blok data berikutnya sedang dikompresi. Tumpang tindih kompresi dengan penulisan disk membuat waktu kompresi hampir bebas. Selain itu, file hibernasi ditulis menggunakan IDE DMA, bukan mode PIO. Sebagian besar pengontrol dan disk IDE modern mencapai kinerja terbaiknya hanya dalam mode DMA.

Dengan kata lain, proses hibernasi terjadi di Windows dengan semua driver dan sejenisnya dimuat dan siap sehingga Windows dapat mengakses perangkat keras (yaitu, drive) dengan kecepatan maksimum. Namun ketika Anda melanjutkan, Windows tidak dimuat, sehingga menggunakan BIOS yang hanya menyediakan layanan tingkat dasar — ​​bahkan jika sebanyak itu — bagaimanapun juga, B di BIOS adalah singkatan dari Basic.

Situasi untuk Windows 7 pada sistem EFI sedikit lebih baik, tetapi pada dasarnya masih sama. Tidak mungkin bagi EFI untuk memiliki “driver” untuk setiap perangkat keras yang memungkinkan, sehingga sistem EFI sekalipun tidak dapat menyediakan akses penuh dengan kinerja maksimum.


Oleh karena itu, melanjutkan dari hibernasi akan selalu agak lambat (setidaknya bagian yang mengaktifkan dan menjalankan Windows; pemuatan latar belakang sisanya akan sedikit lebih cepat). Nyatanya, semakin banyak memori yang telah Anda instal, semakin lama waktu yang dibutuhkan karena ada lebih banyak data yang harus dibaca dari disk — hibernasi adalah satu-satunya kelemahan dari memiliki lebih banyak memori.
