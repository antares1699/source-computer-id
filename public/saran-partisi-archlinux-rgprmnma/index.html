Saran Partisi ArchLinux

Saya mencoba menginstal ArchLinux dan saya mengalami sesuatu yang sangat aneh.

Nya panduan menyarankan Anda untuk membuat 2 partisi:

  1. ~ Partisi root 15gb
  2. Partisi rumah [mengisi sisa ruang].

Saya punya beberapa masalah dengan ini:

  1. Di mana partisi boot [dan flag bootable, yang sepertinya tidak dapat saya temukan di cgdisk]?
  2. Di mana rekomendasi swap-nya?
  3. Mengapa memisahkan ruang root [yang mencakup aplikasi terinstal] dari direktori home?

Apakah instruksinya salah tentang ini?

'Salah' mungkin istilah yang kuat, tetapi pengalaman saya selama beberapa tahun terakhir adalah bahwa membuat banyak partisi cenderung menjadi kesalahan - khususnya, nanti, saya ingin /home atau /var atau apa-apa-Anda berisi lebih banyak byte daripada yang diizinkan ukuran partisi, yang menyebabkan rasa sakit yang besar di leher tidak peduli bagaimana pemecahannya. Saya lupa berapa kali saya harus melakukan ini, dan itu tidak pernah menjadi lebih mudah.

Inilah yang terjadi, kecuali Anda memiliki alasan yang sangat kuat untuk melakukan sebaliknya (atau Anda benar-benar suka melakukan pekerjaan sysadmin, yang saya lakukan sampai saya mengisinya), saya akan merekomendasikan untuk membuat satu yang besar / partisi pada disk, bersama dengan partisi swap jika Anda merasa perlu. (Saya baru saja menukar ke file; lihat 9.2 sini untuk cara melakukannya. Anda mungkin perlu menambahkan perintah swapon di sana ke /etc/rc.local atau tempat serupa yang cocok untuk memastikannya tetap ada di semua boot.)

Memang ada sedikit risiko tambahan di sini bahwa /home akan tumbuh terlalu besar dan membuat sistem kelaparan, atau serupa, tetapi selama Anda tetap memperhatikan berapa banyak ruang yang Anda gunakan, Anda tidak perlu terlalu khawatir tentang itu. skor - dan kenyamanan tambahan karena tidak harus, misalnya, secara berkala mentransfer /var ke disk lain, mengacaukan /etc/fstab, dan reboot, jauh lebih besar daripada risiko kecil dalam pengalaman saya.

Saya biasanya menggunakan yang berikut ini:

/ 30 ~ 40 GB
/boot 256 MB
swap    4 GB (might need more if you want to use hibernation)
/home   20 GB
/mnt/data  the rest

Pada /home saya biasanya hanya memiliki dotfiles; semua data saya disimpan di /mnt/data - ini sebagian besar adalah pilihan pribadi; juga, jika Anda mem-boot ganda Windows / Linux, Anda dapat memiliki partisi ini sebagai NTFS untuk mengakses dari kedua OS.

Inti dari memiliki partisi /home yang terpisah adalah:

  1. jika Anda menghapus partisi root Anda, Anda tidak kehilangan pengaturan / file pribadi
  2. jika Anda mem-boot ganda distribusi Linux, Anda dapat menggunakan /home yang sama di mana saja

Sedangkan untuk swap, saya memiliki RAM 6 GB dan partisi swap 6 GB yang saya miliki hampir tidak tersentuh.
