Kapan fsck akan berjalan jika jumlah pemasangan maksimum adalah -1?

Saya mencoba untuk menentukan apakah fsck akan berjalan saat reboot. Saya menjalankan perintah ini dumpe2fs -h /dev/(diskname) dan mendapatkan hasil ini:

Mount count:              13
Maximum mount count:      -1

AFAIK fsck akan berjalan jika jumlah pemasangan sama atau lebih besar dari jumlah pemasangan maksimum. Dalam hal ini jumlah pemasangan maksimum adalah -1 sehingga fsck tidak akan pernah berjalan. Apakah ini normal?

Dari man tune2fs

-c max-mount-counts

Sesuaikan jumlah mount yang setelah itu sistem berkas akan diperiksa oleh e2fsck(8). Jika max-mount-counts adalah 0 atau -1, berapa kali sistem berkas dipasang akan diabaikan oleh e2fsck(8) dan kernel.

Dalam hal ini "diabaikan" berarti ya, -1 tidak akan pernah berjalan karena alasan ini dan ini normal. Bagaimanapun juga ada

-i interval-between-checks[d|m|w]

Sesuaikan waktu maksimal antara dua pemeriksaan sistem file. Tidak ada sufiks atau d yang akan menafsirkan interval angka-antara-cek sebagai hari, m sebagai bulan, dan w sebagai minggu. Nilai nol akan menonaktifkan pemeriksaan tergantung waktu.

Jadi fsck mungkin masih berjalan karena alasan lain ini, tergantung pada apakah interval disetel atau tidak.

Sangat disarankan agar pemeriksaan -c (tergantung jumlah mount) atau -i (tergantung waktu) diaktifkan untuk memaksa pemeriksaan sistem file e2fsck(8) secara berkala. Kegagalan untuk melakukannya dapat menyebabkan kerusakan sistem file (karena disk, kabel, memori, atau bug kernel yang buruk) tidak diketahui, yang pada akhirnya mengakibatkan hilangnya atau rusaknya data.
