Isi baris terakhir hanya dengan kekosongan pada posisi tertentu

Saya perlu mengisi baris terakhir file dengan spasi kosong hingga posisi 80, dan saya juga perlu memastikan itu selalu diakhiri dengan tidak ada yang baru garis (CR / LF)

Informasi

Di Windows 10 di bawah SED digunakan untuk memformat ulang file sehingga CRLF berada pada posisi baris 80 dari setiap baris hingga EOF dan memperbarui file di tempat tetapi baris terakhir tidak pernah memiliki CR atau LF .

File asli adalah string besar karakter yang dapat dicetak tanpa carriage return atau line feed di file ini sehingga semuanya dimulai pada baris nomor satu dan di bawahnya adalah SED Saya gunakan untuk memformatnya.

sed -i -e "s/.\{80\}/&\r\n/g" "C:\Folder\test.txt"

Masalah Saya

Saya tidak tahu bagaimana membuat baris terakhir dari file ini diisi hingga posisi 80 dengan spasi kosong dan tidak ada jalur kembali.

Saya sudah mencoba beberapa hal tetapi saya jauh dan lebih suka tidak mencantumkan semua hal yang telah saya coba jadi saya menunjukkan sintaks sed yang saya jalankan di atas untuk bagian itu. Saya yakin ada sesuatu yang sederhana yang saya abaikan. . .

Pilihan

Saya lebih suka menggunakan SED untuk ini dan saya harus menjaga CRLF di baris lain di atas baris terakhir dalam format yang sama ini tetapi saya ingin melihat solusi Windows lainnya juga jika tidak terlalu rumit.

Pertimbangan

Karakter baris terakhir dapat berupa 1 karakter, 79 karakter, hingga mungkin berpotensi 80 karakter di mana saya perlu memastikan tidak ada carriage return atau feed baris pada baris terakhir jika spasi non-putih mengisi ke posisi baris 80 di baris terakhir.

Meskipun cuplikan layar berisi 91 baris, file ini dapat berisi lebih banyak baris atau lebih sedikit baris tetapi aturannya harus selalu sama; carriage return line feed di posisi 80 pada setiap baris dan baris terakhir harus tepat 80 posisi di atas mengisi dengan ruang putih jika diperlukan dan tidak ada carriage return atau line feed di atasnya.

Coba ini:

sed -i -e 's/.\{80\}/&\r\n/g' \
  -e 's/$/################################################################################/' \
  -e 's/\(\r\n.\{80\}\)#*$/\1/' "C:\Folder\test.txt"

(Saya telah memecah perintah menjadi beberapa baris hanya untuk keterbacaan dalam presentasi; Anda dapat melakukan semuanya dalam satu baris.) s/$/###…###/ menambahkan 80 # karakter, dan kemudian s/\(\r\n.\{80\}\)#*$/\1/ mencari urutan ini:

  • \r\n
  • 80 karakter apa pun
  • sejumlah #-an
  • hingga akhir buffer

dan menggantinya dengan grup yang dibentuk oleh dua peluru pertama. Dengan kata lain, itu membuat n karakter yang merupakan baris terakhir file dan 80- n # karakter yang mengikutinya, dan membuang n # karakter yang mengikuti itu.

Setelah Anda memverifikasi bahwa ini berfungsi, cukup ubah semua kemunculan # pada perintah di atas menjadi spasi.

CATATAN: Garis SET s10= memiliki 10 ruang kosong di sebelah kanan tanda sama dengan.

Skrip Batch

@ECHO ON

SET File=C:\Folder\test.txt
::: The below s10 variable equals ten white\empty spaces - so it has 10 blank spaces in it
SET s10=          
SET s40=%s10%%s10%%s10%%s10%
SET s80=%s40%%s40%

sed -i -e "s/.\{80\}/&\r\n/g" -e "s/$/%s80%/" -e "s/\(\r\n.\{80\}\)\s*$/\1/" "%File%"
GOTO :EOF

Kami dapat membuat ini sedikit lebih mudah dibaca dan tidak terlalu membosankan untuk diketik.

  1. Sed murni

    Jika teks Anda semuanya alfanumerik, dan ada beberapa karakter (seperti @) yang tidak akan pernah muncul di data Anda, maka Anda dapat melakukan sesuatu seperti ini:

      -e 's/$/@@@@@@@@@@/' \        # Append 10 @ characters to the line.
  -e 's/@/        /g' \         # Change each @ to eight spaces.

    Jika Anda ingin membuat skrip Anda dapat dibaca dengan jelas, dengan biaya membuatnya lebih bertele-tele, ambillah satu langkah lebih jauh:

      -e 's/$/@@@@@@@@@@/' \        # Append 10 @ characters to the line.
  -e 's/@/@@@@@@@@/g' \         # Change each @ to eight @s (there are now 80 @s).
  -e 's/@/ /g'        \         # Change each of the 80 @s to a space.

    Jika tidak ada karakter tunggal yang tidak pernah muncul di data Anda, tetapi ada beberapa karakter (pendek) urutan , seperti (), yang tidak pernah muncul, Anda dapat menggunakan prinsip yang sama:

      -e 's/$/()()()()()()()()()()/' \  # Append 10 () pairs to the line.
  -e 's/()/        /g' \            # Change each () to eight spaces.

    Hati-hati saat berhadapan dengan karakter itu istimewa dalam ekspresi reguler; misalnya., . , *, [, ^, &, dll. (Dan jangan menyertakan #-komentar dalam skrip Anda di tempat-tempat itu; untuk menghentikan perintah sederhana di beberapa baris, \ harus menjadi karakter terakhir di baris (sebelum baris baru) dan bukan dalam komentar.)

  2. Menggunakan cangkang

    Prinsip yang sama:

    #!/bin/sh
      ︙
s="          "                  # Set $s to 10 spaces.
s80="$s$s$s$s$s$s$s$s"          # Set $s80 to 8 copies of $s; i.e., 80 spaces.
      ︙
sed -i -e 's/.\{80\}/&\r\n/g' \
  -e 's/$/'"$s80"'/' \
  -e 's/\(\r\n.\{80\}\)\s*$/\1/' "C:\Folder\test.txt"
