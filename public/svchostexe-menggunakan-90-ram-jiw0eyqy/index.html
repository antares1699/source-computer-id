SVCHost TIDAK mengambil 90% dari RAM.

Saya menyadari bahwa Anda mungkin memiliki beberapa perangkat lunak yang mengatakan demikian. Namun, saya jamin, BUKAN.

SVCHost pada dasarnya adalah trik mewah (terdokumentasi) yang digunakan Microsoft agar beberapa "layanan" terlihat seperti satu proses. Sumber masalah sebenarnya hampir tidak pernah sebenarnya SVCHost, tetapi ini adalah salah satu proses yang disamarkan sebagai SVCHost. Ya, ini memperumit pemecahan masalah. Namun, ada beberapa keuntungan kecepatan, jadi menggabungkan beberapa perangkat lunak menjadi satu "proses" memang memiliki tujuan yang sah. (Jadi ini belum tentu menunjukkan adanya virus.)

Mencoba: TASKLIST /SVC Kemudian Anda dapat mencocokkan PID ke beberapa layanan.

Atau, coba Resource Monitor (Task Manager, tab Performance), CPU. Di bagian "layanan" (yang mungkin perlu Anda kembangkan), Anda dapat mengurutkan berdasarkan PID untuk melihat layanan apa yang menggunakannya.

Saya telah menemukan bahwa di dunia bisnis, penyebab umum adalah perangkat lunak yang banyak menggunakan disk (mis., Perangkat lunak cadangan). Saya kira perangkat lunak semacam itu sering bekerja dengan driver yang mungkin menangani hal-hal pada tingkat rendah, dan tindakan seperti itu mungkin sering melibatkan perangkat lunak yang dirancang untuk kecepatan daripada interaktivitas pengguna (seperti menunjukkan nama "proses" yang tepat).

Semua ini mengatakan, virus juga dapat menyebabkan masalah seperti penggunaan CPU yang tinggi, dan telah dikenal bersembunyi sebagai SVCHost.exe. Jadi, Anda mungkin berurusan dengan virus. Namun, Anda mungkin juga tidak. Dalam pengalaman saya memecahkan masalah banyak komputer (untuk beberapa bisnis yang dikelola secara profesional), perangkat lunak berbahaya adalah kasus yang kurang umum. Pemecahan masalah menyeluruh umumnya menemukan penyebab yang dapat diperbaiki selain aktivitas virus.

Coba gunakan svchost viewer. https://svchostviewer.codeplex.com/

Dan dari situs web lain (tidak ingat di mana):

Aplikasi Mengakses Internet Dari Komputer Windows Anda

Ada banyak alat pihak ketiga yang membantu Anda mencapai ini. Contoh populer dari alat tersebut adalah CurrPorts by Nirsoft, yang merupakan aplikasi portabel, menampilkan semua koneksi terbuka dan port mendengarkan.

Tetapi bagaimana jika Anda ingin memeriksa ini secara native tanpa menggunakan opsi pihak ketiga apa pun? Untuk melakukan ini, Anda dapat menggunakan Windows PowerShell atau Command Prompt yang sangat populer dan mudah digunakan. Langkah-langkah berikut akan membantu Anda mendeteksi aplikasi yang menggunakan bandwidth Internet Anda dengan Command Prompt. Ini adalah salah satu cara termudah dan termudah untuk melacak koneksi Anda.

Langkah 1: Tekan tombol Windows pada keyboard Anda, dan cari cmd.exe dengan mengetiknya.

Langkah 2: Anda perlu menjalankan cmd.exe sebagai administrator. Untuk melakukan itu, klik kanan padanya, dan pilih 'Run as administrator'.

Langkah 3: Ketik perintah berikut dan tekan enter:

netstat -bona

Ini akan memberi Anda daftar semua koneksi aktif Anda. Gunakan daftar itu untuk mencocokkan dengan svchost viewer dan Anda dapat menghilangkan item dengan proses yang semestinya.

Ini adalah metode terbaik yang saya temukan untuk memvalidasi keabsahan semua item svchost.exe di Task Manager

Semoga berhasil
