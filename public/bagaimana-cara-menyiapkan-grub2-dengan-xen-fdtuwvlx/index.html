Bagaimana cara menyiapkan grub2 dengan Xen?

Saya sedang mem-boot kernel Xen baru dan semuanya berjalan dengan baik sampai beberapa titik selama proses boot ketika mesin melakukan boot ulang tanpa peringatan. Sayangnya, saya tidak melihat kesalahan muncul sebelum itu terjadi.

Apakah ada cara untuk mem-boot mesin dengan kernel yang berfungsi dan melihat log dari boot sebelumnya atau semacamnya?

EDIT: Yang tidak saya perhatikan adalah sebenarnya ada kesalahan di layar. Itu hanya berkedip sangat cepat sebelum reboot. Kesalahan ini terkait dengan tidak dapat memasang sistem file root karena ada yang salah dengan konfigurasi grub saya.

Entri normal di grub.cfg terlihat seperti ini:

menuentry 'Ubuntu, with Linux 2.6.32-33-server' --class ubuntu --class gnu-linux --class gnu --class os {
        recordfail
        insmod ext2
        set root='(hd0,1)'
        search --no-floppy --fs-uuid --set 75475e50-82e2-4f74-b860-6cf92c91b42e
        linux   /vmlinuz-2.6.32-33-server root=/dev/mapper/xenserver-root ro   crashkernel=384M-2G:64M,2G-:128M 
        initrd  /initrd.img-2.6.32-33-server
}

Saya menambahkan yang berikut ini di /etc/grub.d/40_default

menuentry 'Xen 4 with Linux 2.6.32.45-xen' --class ubuntu --class gnu-linux --class gnu --class os --class xen {
        recordfail
        insmod ext2
        set root='(hd0,1)'
        search --no-floppy --fs-uuid --set=root 75475e50-82e2-4f74-b860-6cf92c91b42e
        echo 'Loading Xen 4 ...'
        multiboot /xen.gz placeholder
        echo 'Loading Linux 2.6.32.45-xen ...'
        module  /vmlinuz-2.6.32.45-xen placeholder root=/dev/mapper/xenserver-root ro iommu=1
        echo 'Loading initial ramdisk ...'
        module  /initrd.img-2.6.32.45-xen
}

Tapi itu tidak berhasil karena suatu alasan. Ada ide?

EDIT: Saya juga mencoba skrip 20_linux_xen. Sayangnya itu tidak berhasil. Lihat kesalahan ini:

Found linux image: /boot/vmlinuz-2.6.32.45-xen
Found initrd image: /boot/initrd.img-2.6.32.45-xen
Found linux image: /boot/vmlinuz-2.6.32-33-server
Found initrd image: /boot/initrd.img-2.6.32-33-server
/etc/grub.d/20_linux_xen: 57: uses_abstraction: not found

Found initrd image: /boot/initrd.img-2.6.32.45-xen
/etc/grub.d/20_linux_xen: 156: gettext_printf: not found

gettext sudah diinstal.
Saya menggunakan ubuntu 10.04 LTS.

Kesalahan yang muncul saat boot adalah seperti "harap masukkan opsi root = boot kernel yang benar ..."

Mengapa ini sangat sulit dan mengapa ada begitu banyak contoh grub2 xen? beberapa meletakkan / boot di depan beberapa hanya /. Saya mencoba meletakkan / boot di depan tetapi itu tidak membantu. Juga beberapa memiliki --set = root uuid, namun semua entri saya yang lain hanya memiliki --set uuid. apakah ini ada hubungannya dengan versi grub2 yang berbeda?

Agar grub2-mkconfig dapat mendeteksi kernel Linux Anda, mereka harus diberi nama vmlinuz-* atau kernel-*. Dalam beberapa skrip, Anda juga dapat melihat vmlinux-*. Misalnya (...- xen di bagian akhir hanyalah catatan saya):

/boot/kernel-3.6.2-xen

Agar skrip 20_linux_xen menyertakan kernel Xen Anda, /boot harus berisi file konfigurasi kernel dengan nama persis config-* yang sesuai dengan file kernel Anda. Sebagai contoh:

/boot/config-3.6.2-xen

Untuk membuat file ini cukup jalankan zcat /proc/config.gz > /boot/config-[your_version], dan Anda harus menjalankan grub2-mkconfig lagi. Pada saat ini script mungkin akan menghasilkan record duplikat untuk kernel Xen, karena (mungkin) Anda memiliki beberapa symlink ke kernel xen. Jadi hapuslah dan tinggalkan hanya file asli (xen-4.2.0.gz). (Saya harap ini aman, jika menghapus symlink ini salah, harap simpan komentar Anda di bawah.)

Situasi yang sama terjadi pada file initrd / initramfs, tetapi banyak variasinya.

Menambahkan parameter kernel telah dijelaskan sini .

sumber: penamaan kernel , 20_linux_xen:147, 10_linux:194-201
