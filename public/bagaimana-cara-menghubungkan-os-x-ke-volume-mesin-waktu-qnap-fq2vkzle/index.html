Bagaimana cara menghubungkan OS X ke volume Mesin Waktu QNAP?

Saya memiliki QNAP TS-212 NAS yang menjalankan firmware 4.0.2 terbaru. Firmware ini memiliki file Stasiun Cadangan opsi yang membuat volume cadangan Time Machine untuk digunakan OS X. Dalam kasus saya OS X 10.8.5.

Saat saya mengaktifkan Time Machine dari OS X, saya sebenarnya dapat melihat volume cadangan yang disebut TMBackup dalam daftar volume yang dapat dipilih.

Namun, memilihnya, mencoba menghubungkan, dan kemudian memberitahu saya bahwa koneksi gagal. Saya dapat terhubung ke share AFP lainnya di NAS tanpa masalah, tetapi tidak akan terhubung ke TMBackup.

Apa yang dapat saya lakukan agar dapat membuat cadangan lagi?

Saya memiliki masalah yang sama dengan macbook pro baru saya yang menjalankan OSX Mavericks 10.9.1 dan QNAP TS-219P dengan firmware 4.0.2.

Pada akhirnya saya harus melewati GUI Time Machine dan mengatur volume cadangan melalui baris perintah menggunakan perintah ini:

sudo tmutil setdestination -ap afp://TimeMachine@NAS.local/TMBackup

di mana NAS.local adalah nama jaringan QNAP saya. Setelah itu, setel cadangan berfungsi dengan baik dan GUI juga merespons dengan baik.

*** bagian afp: // dari perintah di atas hanya dapat digunakan dalam item menu "Hubungkan ke Server" dari Finder. Saya menemukan bahwa penggunaan nama pengguna "TimeMachine" membuat semua perbedaan, mengingat saya juga menggunakan server jaringan yang sama untuk jaringan normal saya.

Apa yang saya lakukan dan yang secara mengejutkan berhasil adalah memaksa OS X untuk terhubung ke share sekali. Buka Terminal.app dan masukkan berikut ini, ganti qnap.local dengan nama host NAS Anda yang sebenarnya:

ssh TimeMachine@qnap.local

Masalahnya, volume Time Machine perlu diakses oleh akun pengguna yang berbeda, yaitu TimeMachine. Dapat mengakses NAS melalui SSH dan AFP. Kata sandi adalah yang Anda atur di Stasiun Cadangan panel kendali. Saham lain mungkin hanya untuk tamu atau dapat diakses dari pengguna lain, tetapi saya percaya bahwa OS X menolak koneksi ke TMBackup sepenuhnya jika belum melihat mesin itu.

Anda harus bisa memasukkan kata sandi Anda. Kemudian Anda akan diusir lagi. Setelah itu, setelah Anda mencoba menyambungkan ke volume Time Machine dari Preferensi Sistem OS X, ini berfungsi tanpa masalah.

Saya juga mengalami masalah yang sama dengan Yosemite dan QNAP HS-251 NAS dengan firmware 4.1.3. Tak satu pun dari solusi ini berhasil untuk saya.

Saya menemukan peningkatan berdasarkan jawaban Phil:

sudo tmutil setdestination afp://TimeMachine:yourpassword@yourIPaddress/TMBackup

di mana "TimeMachine" adalah pengguna, "kata sandi Anda" adalah kata sandi TimeMachine yang diisi di antarmuka NAS, "alamat IPAnda" adalah alamat IP NAS dan "TMBackup" adalah nama tampilan.

Disk muncul secara ajaib di daftar Time Machine dan saya dapat menyambungkannya.

Saya punya banyak masalah dengan ini. Tidak ada solusi di atas yang berhasil untuk saya, meskipun saya pikir masalahnya ada pada kata sandi.

Jika saya menggunakan kata sandi yang panjang (64 karakter), saya tidak dapat menghubungkan / memasang di Finder atau Terminal. Saya kesal dengan kata sandi yang panjang jadi hanya untuk pemecahan masalah, diubah menjadi kata sandi 4 karakter. Terminal masih gagal, begitu pula Finder - sampai saya mencoba terhubung dengan Finder menggunakan yang berikut ini:

afp://timemachine:PASSWORD@SERVER(TimeMachine)._afpovertcp._tcp.local/TMBackup

catatan:

  • Nama pengguna adalah "mesin waktu" huruf kecil (meskipun di QNAP menggunakan CamelCase)
  • PASSWORD adalah kata sandi yang Anda atur di QNAP Backup Station dan bersifat opsional tetapi dengannya Anda dapat melewati layar login (ingat saja untuk menghapus riwayat koneksi Finder)
  • SERVER adalah nama server Anda, tetapi tinggalkan "(TimeMachine)" dan semuanya setelah itu

Setelah ini berhasil, saya kembali ke QNAP dan mengubah kata sandi saya menjadi 64 karakter lagi. Saya mengulangi langkah-langkah di atas dan selalu gagal. Saya mengubah kata sandi 25 karakter tanpa karakter khusus (yang mungkin menjadi masalah sebenarnya) dan berhasil lagi.

Hasil dapat bervariasi.
