crypt32.dll rusak untuk Virtualbox sejak patch Selasa 9-Des-2014?

Dalam 18 jam terakhir, mesin Windows 7 Pro 64-bit saya telah di-boot ulang sebagai tanggapan atas tambalan penerapan Pembaruan Windows.

Sekarang saya menemukan ketika saya mencoba untuk memulai VirtualBox Manager (bukan VM sebenarnya yang dikelola manajer, saya tidak sampai sejauh itu) Saya mendapatkan pesan "\. \ GLOBALROOT \ SystemRoot \ System32.dll tidak dirancang untuk dijalankan di Windows atau ada kesalahan "

Ketika saya mencari file ini di% systemroot% \ System32 \ crypy32.dll itu tidak ada.

Tampaknya penambalan telah menghapus file ini?

Saya telah mengikuti saran dari kotak dialog untuk menginstal ulang VirtualBox (baik versi yang awalnya saya jalankan dan beberapa versi yang lebih baru) tetapi tidak ada yang membantu (sebenarnya saya mendapatkan kesalahan yang berbeda dari versi yang lebih baru tetapi saya lebih suka mengatasi masalah ini daripada mengarungi kesalahan VirtualBox lainnya).

Bagaimana cara paling mudah mendapatkan salinan crypt32.dll yang relevan? Saya hanya memiliki partisi pemulihan, bukan "media instalasi asli".

Sepertinya pembaruan KB3004394 disadap di Windows 7 (baik-baik saja di Windows 8):

  • https://www.virtualbox.org/ticket/13677
  • https://forums.virtualbox.org/viewtopic.php?f=6&t=65056
  • https://answers.microsoft.com/en-us/windows/forum/windows_7-windows_update/windows-update-kb3004394-issues/ace25277-7f65-4486-bc44-c1b106907a18

Anda dapat mengatasi masalah ini untuk sementara dengan menghapus pembaruan tersebut.

Bahkan, mungkin tidak cukup hanya dengan menghapus pembaruan - Anda mungkin harus menggunakan Pemulihan Sistem untuk kembali ke titik sebelum pembaruan (pembaruan itu sendiri seharusnya membuat titik pemulihan, sehingga Anda dapat menggunakannya).
