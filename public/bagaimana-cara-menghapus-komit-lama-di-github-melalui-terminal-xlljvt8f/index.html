Menghapus folder .git dapat menyebabkan masalah pada repositori git Anda. Jika Anda ingin menghapus semua riwayat komit Anda tetapi menyimpan kode dalam keadaannya saat ini, sangat aman untuk melakukannya seperti berikut ini:

Periksa

git checkout --orphan latest_branch

Tambahkan semua file

git add -A

Lakukan perubahan

git commit -am "commit message"

Hapus cabangnya

git branch -D master

Ubah nama cabang saat ini menjadi master

git branch -m master

Terakhir, paksa perbarui repositori Anda

git push -f origin master

PS: ini tidak akan menyimpan riwayat commit lama Anda.

Anda memiliki jawaban yang sangat bagus untuk itu di sini: https://stackoverflow.com/questions/448919/how-can-i-remove-a-commit-on-github

Dan saya mengutip

Catatan: silakan lihat alternatif ke git rebase -i di komentar di bawah—

git reset --soft HEAD^

Pertama, hapus komit di repositori lokal Anda. Kamu bisa melakukan ini menggunakan git rebase -i. Misalnya, jika itu adalah komit terakhir Anda, Anda dapat melakukannya git rebase -i HEAD~2 dan hapus baris kedua di dalam editor jendela yang muncul.

Lalu, paksa push ke GitHub dengan menggunakan git push origin +master.

Lihat Git Magic Bab 5: Pelajaran Sejarah - Dan Kemudian Beberapa untuk lebih informasi (mis. jika Anda ingin menghapus komit lama).

Oh, dan jika pohon kerja Anda kotor, Anda harus melakukan git rebase -i HEAD~2 pertama, dan kemudian git stash apply setelahnya.

Semoga ini membantu Anda :)
