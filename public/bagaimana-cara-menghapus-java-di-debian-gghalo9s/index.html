Bagaimana cara menghapus Java di Debian?

java -version
java version "1.6.0_18"
OpenJDK Runtime Environment (IcedTea6 1.8.13) (6b18-1.8.13-0+squeeze2)
OpenJDK 64-Bit Server VM (build 14.0-b16, mixed mode)

Sepertinya tidak ada cara mudah untuk mencari paket di apt jadi saya menggunakan dpkg untuk mencari paket yang berhubungan dengan java:

# dpkg --get-selections | less | grep java
ca-certificates-java            install
java-common                     install
javascript-common               install
libaccess-bridge-java           install
libaccess-bridge-java-jni       install
libdb4.7-java-gcj               deinstall
libhsqldb-java-gcj              deinstall
sun-java6-bin                   deinstall
sun-java6-jre                   deinstall
tzdata-java                     install

Saya melanjutkan untuk menghapus java-common, namun setelah selesai java masih terinstal dan java -versi masih menunjukkan hal yang sama.

Tidak ada yang berubah, bahkan paket umum java masih muncul di pencarian dpkg. Apa yang saya lewatkan?

Saya juga menghapus OpenJDK, yang tampaknya menyetelnya ke "deinstall":

# dpkg --get-selections | less | grep openjdk
openjdk-6-jre                   deinstall
openjdk-6-jre-headless          install
openjdk-6-jre-lib               install

Jawa masih ada.

# java -version
java version "1.6.0_18"
OpenJDK Runtime Environment (IcedTea6 1.8.13) (6b18-1.8.13-0+squeeze2)
OpenJDK 64-Bit Server VM (build 14.0-b16, mixed mode)

Saya menemukan ini karena saya sedang mengerjakan buku petunjuk untuk pemrograman android yang ingin menggunakan Java SE dan JRE (8u121) tertentu dan meminta agar semua paket java lainnya dihapus sebelum melanjutkan instalasi. Jadi setelah membaca ide di atas saya pergi dengan:

sudo apt-get remove openjdk*

dan hasilnya adalah:

bryan@kali:~$ java -version
bash: /usr/bin/java: No such file or directory

Jadi di atas menunjukkan bahwa tidak ada java yang ditemukan dalam versi apa pun. Yang OP ingin lakukan (menurut saya).

Jika seseorang menemukan utas ini karena mereka mencopot pemasangan Java sehingga mereka dapat menginstal Versi Lama, saya telah menyertakan panduan yang saya temukan. Dua kode pertama membantu Anda menginstalnya jika Anda tidak memiliki GUI. Jika Anda memiliki tarball, lewati saja. Blok kode terakhir memiliki beberapa bagian yang perlu Anda edit jika Anda tidak memasang versi saya yang sama.

Jika Anda mencoba menginstal Java JDK dari jarak jauh dan sama sekali tidak memiliki akses ke Antarmuka Pengguna Grafis dan browser web, gunakan metode curl berikut untuk mengunduh Java JDK menggunakan baris perintah.

Pertama, dapatkan URL unduhan yang benar dengan menggunakan perintah curl:

$ curl -s http://www.oracle.com/technetwork/java/javase/downloads/jdk8-downloads-2133151.html | grep "otn-pub" | cut -d \" -f12

Perintah di atas mengeluarkan banyak URL untuk pilihan Anda. Untuk memulai pengunduhan file Java yang diinginkan, salin URL-nya dan mulai pengunduhan menggunakan perintah curl sambil menerima lisensi Jave yang diminta. Sebagai contoh:

$ curl -LOb "oraclelicense=a" http://download.oracle.com/otn-pub/java/jdk/8u121-b13/e9e7ea248e2c4826b92b3f075a80e441/jdk-8u121-linux-x64.tar.gz

Instal Java JDK Pada tahap ini, kita harus memiliki tarbal JDK Java yang sesuai di dalam direktori kerja kita saat ini:

$ ls
jdk-8u121-linux-x64.tar.gz

Buat direktori instalasi JDK Java target:

# mkdir /opt/java-jdk

Ekstrak tarbal JDK Java yang diunduh sebelumnya:

# tar -C /opt/java-jdk -zxf jdk-8u121-linux-x64.tar.gz

Tetapkan Oracle Java sebagai default Saat ini, sistem tidak mengenali instalasi Java JDK kami:

$ update-alternatives --list java
update-alternatives: error: no alternatives for java
$ java
bash: java: command not found

Gunakan perintah perbarui-alternatif untuk memasukkan keduanya, java dan javac sebagai bagian dari lingkungan Java sistem. Harap ganti jalur di bawah ini ke biner java jika sesuai untuk mencerminkan versi java yang Anda unduh:

# update-alternatives --install /usr/bin/java java /opt/java-jdk/jdk1.8.0_121/bin/java 1   
update-alternatives: using /opt/java-jdk/jdk1.8.0_121/bin/java to provide /usr/bin/java (java) in auto mode
# update-alternatives --install /usr/bin/javac javac /opt/java-jdk/jdk1.8.0_121/bin/javac 1
update-alternatives: using /opt/java-jdk/jdk1.8.0_121/bin/javac to provide /usr/bin/javac (javac) in auto mode
