Menemukan penggunaan CPU dari suatu layanan

Karena kipas laptop saya berputar tanpa alasan, saya melihat penggunaan CPU dan saya melihat svchost dengan CPU 12,5%, yang pada dasarnya adalah inti penuh.

Sekarang, saya sadar bahwa svchost menjalankan layanan dan saya tahu cara menampilkan layanan yang dijalankannya di Task Manager atau Process Explorer ( seperti yang dijelaskan di sini ).

Sekarang saya tahu layanan mana yang dijalankan, tetapi saya tidak tahu layanan mana yang memakan waktu CPU. Saya dapat menonaktifkan satu layanan demi layanan, tetapi saya tidak terlalu suka melakukan ini (ini dapat merusak hal-hal yang sedang digunakan dan saya bisa lupa mengaktifkan kembali layanan nanti).

Oleh karena itu saya ingin tahu apakah mungkin untuk menginstruksikan Windows menjalankan hanya satu layanan per svchost lain kali saya mem-boot PC saya. Meskipun saya mungkin memiliki sedikit kelemahan dari beberapa proses, saya dapat segera mengetahui proses mana itu dan akan menjadi keputusan yang jauh lebih mudah untuk menghentikan atau memulai ulang layanan atau tidak.

Disalin dari http://windowsitpro.com/windows-server/configure-service-run-its-own-svchost-process

Jalankan perintah berikut di Command prompt administrator (perintah ditulis untuk layanan pembaruan windows)

sc config wuauserv type = own

untuk membuat layanan pembaruan windows berjalan dalam proses svchost terpisah.

CATATAN : perlu spasi antara tanda sama dengan dan nilainya.

MEMPERBARUI : Menjalankan sc config wuauserv type= own di Windows 10 menghentikan Pembaruan Windows agar tidak berfungsi dengan benar.

Kamu bisa gunakan ProcessExplorer atau ProcessHacker menunjukkan properti dari proses svchost. Pergi ke tab Threads dan di sini kedua alat menunjukkan kolom dari mana layanan panggilan tersebut berasal.

Dalam gambar ini beberapa penggunaan CPU berasal dari layanan IP Helper.

Dalam Windows 10 RS2 Dibangun sejak Build 14942 Microsoft mengisolasi setiap layanan lagi jika Anda menggunakan PC dengan RAM lebih dari 3,5GB sehingga Anda dapat dengan mudah melihat layanan mana yang termasuk dalam proses svchost:

Jika PC Anda memiliki memori 3,5+ GB, Anda mungkin melihat peningkatan jumlah proses di Task Manager. Sementara perubahan ini mungkin terlihat mengkhawatirkan Sekilas, banyak yang akan bersemangat untuk mengetahui motivasi di baliknya perubahan ini. Seiring bertambahnya jumlah layanan prainstal, mereka mulai untuk dikelompokkan ke dalam proses yang dikenal sebagai host layanan (svchost.exe's) dengan Windows 2000. Perhatikan bahwa RAM yang disarankan untuk PC untuk ini rilis 256 MB, sedangkan RAM minimum 64 MB. Karena peningkatan dramatis dalam memori yang tersedia selama bertahun-tahun, file Keuntungan penghematan memori dari host layanan telah berkurang.
