Seberapa baik SVN memfasilitasi pengeditan kolaboratif dalam hal ini?

Buruk, karena, seperti yang telah disebutkan, file MS-Office lama adalah "hanya biner" untuk subversi, penggabungan otomatis biner dapat memberikan hasil yang tidak dapat diprediksi, untuk penggabungan manual yang digerakkan secara manual, ini lebih menjadi pertanyaan bagi admin Subversion, yang dapat (harus) menyediakan dan konfigurasikan alat penggabung diff | khusus untuk * .doc dalam repositori untuk memungkinkan pengguna akhir melakukan penggabungan dengan cara WYSIWYG biasa

Untuk situasi baru, Office berbasis XML menjadi lebih buruk. AFAICR, docx | File .xlsx sebenarnya adalah arsip zip multi-file dan menggabungkan, secara teoritis, dapat mempengaruhi dan mengubah lebih dari satu file. Saya tidak tahu bagaimana menangani kasus ini

SVN tidak benar-benar cocok dengan versi file biner (yang mana .doc dan .xls), karena file tidak berisi teks, secara praktis tidak mungkin untuk membeda-bedakan file atau menggabungkan perubahan. Format yang lebih baru (.docx dan .xlsx) sebenarnya adalah dokumen XML yang dapat membantu sedikit operasi diff dan penggabungan tetapi masih akan cukup sulit.

Rekomendasi saya adalah menggunakan sesuatu seperti Google Docs yang memungkinkan untuk membuat versi dan mengedit secara bersamaan hingga 50 pengguna (juga spreadsheet). Jika Anda sudah mati saat menggunakan SVN, saya akan merekomendasikan untuk tetap menggunakan dokumen teks biasa, saya tahu beberapa orang yang pernah menggunakannya Getah dan SVN untuk beberapa tugas, yang memungkinkan mereka melakukan pemformatan.
