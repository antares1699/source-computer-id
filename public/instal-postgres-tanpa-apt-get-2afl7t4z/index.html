Instal postgres tanpa apt-get

Saya memiliki akses ke mesin di hpc ilmiah, dan saya menyiapkan lingkungan untuk membuat aplikasi web, yang pada dasarnya terdiri dari KUCING JANTAN dan postgresql.dll . Saya tidak dapat menggunakan server web lain.

OS adalah linux dengan Debian (pemerasan) dan saya tidak punya akar akses ke mesin.

Setelah googling sedikit, saya hanya menemukan metode yang membutuhkan akses root ke mesin untuk menginstal postgresql (apt-get ....).

Apakah mungkin untuk menginstal postgresql di direktori lokal di mana saya memiliki izin penuh? Atau Apakah postgresql membutuhkan akses root untuk diinstal? Mengapa?

Ini akan menghemat banyak waktu saya, mengunduh biner, meletakkannya di berbagi file atau cloud dan kemudian:

Linux:

aws s3 cp 's3://yourAwsBucket/postgresql-9.6.21-1-linux-x64-binaries.tar.gz' .
tar xvzf postgresql-9.6.21-1-linux-x64-binaries.tar.gz
cd pgsql/bin
psql -c 'Select version();' "postgresql://<user>:<password>@<dbendpoint>/<dbName>"

atau jika Anda mendapatkan file tidak ada file atau direktori seperti itu 'psql' menggunakan subproses, coba letakkan pesta sebagai argumen pertama:

print(subprocess.run(['bash','psql','-c','Select version();','postgresql://<user>:<password>@<dbendpoint>/<dbName>'],shell=isWindows, stdout=subprocess.PIPE, stderr=subprocess.PIPE).stdout.decode('ascii'))

Windows:

https://stackoverflow.com/questions/33854798/how-do-i-install-just-the-client-tools-for-postgresql-on-windows
