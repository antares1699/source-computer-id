Bagaimana cara ssh-login ke Mac saya sebagai root?

Untuk beberapa alasan saya tidak bisa login sebagai root ke Mountain Lion 10.8.2 saya.

Saya sudah mengedit file /private/etc/sshd_config dengan menghapus komentar dari

PermitRootLogin yes

dan memulai ulang Mac. Masih dikatakan "Akses ditolak" saat saya mencoba ssh ke Mac saya.

Saya juga mencoba pergi Preferensi Sistem> Berbagi> Login Jarak Jauh dan memilih "Semua pengguna" untuk opsi "Izinkan akses untuk", tetapi saya masih mendapatkan "Akses ditolak". Login Jarak Jauh sudah dihidupkan.

Kata sandi yang saya gunakan adalah kata sandi sudoing yang valid dan sama dengan yang digunakan untuk mengedit file /private/etc/sshd_config.

Program yang saya coba sambungkan ke Mac saya adalah WinSCP, yang pada dasarnya adalah manajer file, jadi saya tidak dapat melakukan sudoing dengannya dan karenanya benar-benar perlu menjadi root untuk memiliki kendali penuh atas file Mac saya. .

Bermigrasi ke jawaban:

Per apel :

  By default, the root user is not enabled [in Mountain Lion].

Artinya, kecuali Anda telah mengaktifkan akun dan menyetel kata sandi - Anda mungkin bahkan tidak mengaktifkan pengguna root. Sesuai tautan di atas:

Enable the root user
   1. Choose Apple menu > System Preferences, and then click Users & Groups.
   2. Click the lock icon to unlock it, and then type an administrator name and password.
   3. In the Network Account Server section, click Join or Edit.
   4. Click Open Directory Utility.
   5. Click the lock icon to unlock it, and then enter your administrator name and password.
   6. Choose Edit > Enable Root User, and then enter a root user password in the Password and Verify fields.
 Be sure to specify a secure password.

Kemudian Anda dapat menguji apakah itu berfungsi melalui SSH dan WinSCP.

Cara Unix yang lebih pendek

Buka Terminal dan jalankan

sudo passwd

masukkan kata sandi sudoing Anda jika perlu, lalu berikan kata sandi untuk akun root saat diminta.

Dan jangan lupa untuk menambahkan "Administrator" ke daftar pengguna yang diizinkan untuk login jarak jauh.

Satu opsi yang saat ini saya gunakan dengan instalasi Debian saya:

  1. Gunakan WinSCP untuk menyalin file ke lokasi yang dapat ditulis (mis. Direktori home Anda)
  2. Terhubung melalui SSH menggunakan program seperti Baki PuTTY
  3. Tingkatkan diri Anda ke root
  4. Pindahkan / Salin file ke lokasi yang benar

Salah satu alasan saya memilih PuTTY Tray daripada PuTTY adalah fitur yang diperluas, seperti menu konteks klik kanan.

Saya tahu ini tidak secara langsung menjawab pertanyaan Anda, tetapi saya sendiri tidak beruntung mendapatkan WinSCP untuk bekerja seperti yang Anda tanyakan.
