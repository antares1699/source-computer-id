  • @Oliver, ya, persis seperti itulah cara kerjanya. Ketika terjadi kemacetan, Windows menggunakan debugger apa pun yang ditunjukkan di kunci registri AeDebug. Secara default, ini diatur ke Dr. Watson yang hanya membuat crash dump dan mencatatnya, tetapi jika Anda menginstal debugger lain (yang sebagian besar lingkungan pemrograman akan menginstal untuk alasan yang jelas), ia menggunakannya sebagai gantinya. Dalam hal ini, ketika program macet dan Windows melihat bahwa Anda memiliki debugger yang diinstal, ia meminta Anda menangani macet karena tidak tahu apakah itu program Anda sendiri yang Anda uji yang macet atau program pihak ketiga.
    — Synetech

Jika Anda memiliki JIT debugging diaktifkan di Visual Studio, atau debugger lainnya, itu akan memberi Anda kesempatan untuk melampirkan debugger ke proses yang lumpuh sehingga Anda dapat melakukan debugging post-mortem. Jika Anda tidak ingin JIT men-debug, Anda dapat menonaktifkannya, lalu Anda harus memasang debugger secara manual ke proses yang ingin Anda debug sebelum macet.
