Definisikan ulang asosiasi ekstensi file di Sublime text 2

Saya perhatikan bahwa Sublime text 2 secara default mengaitkan file dengan ekstensi *.scm dan *.ss dengan bahasa lisp. Tapi sebenarnya scheme. Apakah mungkin untuk mendefinisikan kembali asosiasi ini (mungkin dalam konfigurasi)?

Saat ini, untuk mendapatkan penyorotan sintaks yang benar, saya hanya membuka View -> Syntax -> Scheme setiap kali saya membuka file scheme.

Asosiasi ekstensi didasarkan pada skema penyorotan sintaks.

Untuk menghapus asosiasi yang tidak diinginkan, buka ST2 dan pilih Preferensi → Jelajahi Paket ... , folder dengan paket akan terbuka. Temukan paket yang salah terdeteksi (dalam kasus ini Pelat ) dan buka a .tmLanguage file ( Lisp.tmLanguage ) di ST2. Baris pertamanya terlihat seperti ini:

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple Computer//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>comment</key>
    <string></string>
    <key>fileTypes</key>
    <array>
        <string>lisp</string>
        <string>cl</string>
        <string>l</string>
        <string>mud</string>
        <string>el</string>
    </array>
(...)

String tersebut pada baris 9-13 adalah ekstensi file yang akan diasosiasikan oleh ST2 dengan Lisp. Anda dapat menghapus beberapa baris tersebut (letakkan kursor pada baris yang tidak diinginkan dan tekan Ctrl + Bergeser + K ), perubahan akan segera diterapkan saat Anda menyimpan file.

Untuk menambahkan asosiasi baru, buka a .tmLanguage file dari paket dan menambahkan string baru di antara yang sudah ditentukan.

Ini sudah dijawab di StackOverflow: https://stackoverflow.com/a/8014142/1346682

Dalam versi Sublime Text 2 (Build: 2139) saat ini, Anda dapat mengatur sintaks untuk semua file dari ekstensi file tertentu menggunakan opsi di bilah menu. Buka file dengan ekstensi yang ingin Anda setel defaultnya dan navigasikan melalui menu berikut: View -> Syntax -> Open all with current extension as... ->[your syntax choice].

Berlaku juga untuk versi terbaru saat ini (Build: 3065).
