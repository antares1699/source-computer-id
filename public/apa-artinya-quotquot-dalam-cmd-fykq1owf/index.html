() sebenarnya bukan perintah tetapi operator pengelompokan yang memungkinkan Anda untuk mengelompokkan urutan perintah bersama-sama untuk menggunakannya di tempat di mana shell CMD biasanya mengharapkan satu perintah untuk muncul.
Perintah antara () sebenarnya dijalankan dalam contoh CMD yang terpisah. Ketika Anda mengetik "(" secara manual Anda memasukkan contoh baru itu dan menunggu perintah Anda.

if not exist c:\myfile (
  echo File doesn't exist
) else (
  echo File DOES exists
  del c:\myfile
)

Perhatikan bahwa biasanya IF ELSE harus berada di baris yang sama dan Anda hanya dapat menjalankan satu perintah di klausa THEN dan ELSE.
Menggunakan "()" akan memperlakukan seluruh blok (xxxxx) sebagai satu perintah dan bahkan seolah-olah itu muncul di baris yang sama.

2. Bagaimana ( dan ) mempengaruhi arti dari sebuah perintah?

Seperti yang ditunjukkan orang lain, ada perintah seperti

C:\> for %f in (\Windows\*.dll) do @echo %f

di mana (… ) adalah bagian dari sintaks perintah itu sendiri (FOR, dalam kasus ini). Jika Anda menghapus tanda kurung, Anda mendapatkan kesalahan sintaks. Saya tidak membicarakan tentang itu.

Seperti yang dinyatakan sampai membosankan , tanda kurung kelompok perintah. Pertimbangkan baris perintah ini:

echo cat & echo dog | findstr /n a

Itu berarti

  1. Lakukan echo cat, lalu
  2. Lakukan echo dog | findstr /n a
    Karena kata "anjing" tidak mengandung huruf "a", perintah ini tidak akan menghasilkan keluaran.

Dengan demikian, keluaran untuk di atas adalah

C:\> echo cat & echo dog | findstr /n a
cat

Demikian pula,

C:\> echo cat & echo dog | findstr /n o
cat
1:dog

di mana kami mendapatkan keluaran 1:dog tambahan, karena "anjing" tidak mengandung "o", dan "anjing" adalah yang pertama ( 1 st) baris masukan ke perintah findstr.

Sekarang pertimbangkan

(echo cat & echo dog) | findstr /n a

(sama seperti di atas, tetapi dengan tambahan tanda kurung), yang artinya

  1. Lakukan echo cat & echo dog | findstr /n a ,
    lalu
    lakukan echo dog, dan
  2. Ambil output dari atas dan pipa ke findstr /n a

Begitu,

C:\> (echo cat & echo dog) | findstr /n a
1:cat

C:\> (echo cat & echo dog) | findstr /n o
2:dog

karena keluaran dari sisi kiri (# 1) dari baris perintah adalah

cat
dog

dan "a" ditemukan di baris 1 dari teks itu, dan "o" ditemukan di baris 2 . Kami tidak lagi mendapatkan jalur keluaran yang mengatakan hanya cat (tanpa nomor baris) karena

C:\> (
More? date /t
More? time /t
More? )
Tue 04/11/2017
06:18 PM

C:\> (
More? date /t
More? time /t)
Tue 04/11/2017
06:18 PM

C:\>
bukan lagi perintah yang berdiri sendiri, tetapi merupakan bagian dari kelompok yang membentuk sisi kiri pipa.

Berikut contoh pengelompokan perintah yang lebih sederhana:

C:\Temp> echo cat & echo dog > animals
cat

C:\Temp> type animals
dog

C:\Temp> (echo cat & echo dog) > animals

C:\Temp> type animals
cat
dog

Dan, tentu saja, seperti yang ditunjukkan di atas dan di jawaban lainnya, Anda dapat menambahkan jeda baris dalam grup:

C:\> (
More? echo cat
More? echo dog
More? ) | findstr /n a
1:cat

3. Apakah tanda kurung membuat instance CMD baru?

Saya kira tidak; tidak dengan cara yang jelas dan bermakna:

C:\> (cd windows)

C:\Windows> set a=aardvark

C:\Windows> (set a=bat)

C:\Windows> echo %a%
bat

Perhatikan bahwa cd dalam tanda kurung mengubah direktori kerja dari proses CMD utama, dan set dalam tanda kurung menimpa nilai variabel yang ditetapkan di luar tanda kurung.

(Dan) seperti begin block dan end block. Apa pun yang dapat Anda masukkan ke dalam file batch (.bat) dapat diketik ke dalam baris perintah.

Misalnya saya dapat menulis file batch dengan dapat meletakkan blok pernyataan ke dalam if:

if "%1"=="" (
  echo Multiple
  echo Lines when no parameter
)
echo END

Sekarang ketika saya menjalankan batch tanpa parameter saya mendapatkan semua perintah antara ( dan ) untuk dijalankan. Ketika parameter pertama diberikan, keluaran satu-satunya adalah 'END'.

Sekarang kembali ke baris perintah: Saya tahu tidak ada trik atau teknik untuk benar-benar menggunakan ini dengan langsung mengetik di baris perintah. Tapi bagaimanapun itu mengikuti aturan yang sama. Pada baris perintah jika masukkan berikut ini:

(echo "1" & echo "2")

Outputnya adalah:

1
2
