Bisakah nama domain DNS mengarah ke folder di server?

Jadi saya memiliki nama domain DNS untuk komputer saya sehingga saya dapat terhubung dengan mudah tanpa mengingat alamat IP saya setiap saat dan itu berfungsi dengan baik. Saya juga memiliki situs web yang diinangi dari komputer saya yang nama domainnya saya miliki. Nama domain memiliki setelan penerusan URL Frame(cloaking) yang mengarah ke folder situs web saya.

Saya memiliki bagian login ke situs saya yang menggunakan sesi php. Ketika saya pergi ke localhost untuk menguji sesi, itu berfungsi dengan baik. Tetapi ketika saya menggunakan nama domain saya, itu tidak berfungsi sama sekali. Jadi saya mencoba menggunakan nama DNS saya dan meletakkan folder setelahnya example.tk/MySite. Yang mengejutkan saya ternyata berhasil. Tetapi jika saya tidak mengetikkan /MySite setelah nama DNS, ini akan masuk ke MAMP Index of /. Saya sama sekali tidak menginginkannya. Saya bertanya-tanya apakah mungkin untuk meneruskan nama domain DNS ke folder. Saya mencarinya dan orang-orang berkata tidak, Anda harus menyiapkan "Hosting Virtual" atau apa pun. Sekarang saya tahu sangat sedikit tentang semua hal tentang Domain ini, jadi bantuan apa pun akan sangat kami hargai.

Terima kasih sebelumnya!

Di mana saya mengetahui tentang hosting virtual.

Perilaku yang Anda amati persis seperti yang diharapkan. Perhatikan bahwa URL seperti http://example.tk:80/MySite berarti bahwa klien harus menghubungi host example.tk (dan hanya untuk mengetahui mengapa host mana yang merujuk pada DNS digunakan untuk konteks ini), kemudian memulai koneksi TCP ke proses server mendengarkan di port 80 dari host tersebut, lalu gunakan protokol http untuk meminta URI /MySite dari example.tk. Server web biasa akan menafsirkan URI yang mirip jalur tersebut sebagai jalur yang berhubungan dengan root dokumennya dan sebagai contoh mengembalikan konten file dengan nama tersebut ( atau jalankan skrip php atau daftar isi direktori [sebenarnya setelah pengalihan] atau apa pun yang Anda konfigurasikan).

Jadi karena Anda hampir puas dengan hasilnya - tidak jauh dari level direktori - saya sarankan Anda mengkonfigurasi ulang server web Anda untuk melayani dari root dokumen yang berbeda. Di sisi lain, laporan Anda tentang http://localhost melakukan hal yang benar menunjukkan bahwa Anda sudah melakukannya memiliki virtual hosting set uop, yaitu vhost localhost dikonfigurasi untuk menggunakan folder MySite itu sebagai root dokumennya. Harus dimungkinkan untuk mengubah konfigurasi host example.tk Anda untuk melayani dari root dokumen yang sama. Mungkin mulai membaca file konfigurasi khusus vhost dari server web Anda (mungkin terletak di /etc/apache2/sites-enabled atau serupa). Perhatikan terutama untuk opsi DocumentRoot, tentu saja.

DNS tidak dapat meneruskan ke folder.

Ada beberapa opsi untuk menyelesaikan ini dengan cara berbeda.

Pertama, banyak penyedia DNS mengizinkan Anda menyiapkan "pengalihan HTTP" untuk nama DNS. Apa yang sebenarnya terjadi dengan fitur ini adalah bahwa DNS mengarah ke server web penyedia, di mana mereka memiliki konfigurasi server web sederhana yang disiapkan untuk menyediakan bagian pengalihan HTTP.

Anda dapat menggunakan pendekatan ini sendiri: Arahkan DNS ke server tempat Anda menjalankan server web yang dikonfigurasi untuk merespons nama domain yang sama, dengan menampilkan atau mengalihkan ke folder. Detailnya sangat bervariasi berdasarkan detail situasi Anda.
