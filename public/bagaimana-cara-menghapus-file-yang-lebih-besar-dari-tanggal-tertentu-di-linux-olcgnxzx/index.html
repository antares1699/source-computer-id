Bagaimana cara menghapus file yang lebih besar dari tanggal tertentu di linux

Terkadang saya mengekstrak arsip ke folder yang salah dan ingin memindahkan atau menghapus file yang baru diekstrak.

Apa cara termudah untuk melakukan ini melalui baris perintah? Dapatkah saya mencari semua file yang lebih baru dari waktu ekstraksi dan menyalurkannya ke rm?

Edit: Seperti dicatat dalam komentar, tar memodifikasi mtime dan ctime dari file yang diekstrak agar sesuai dengan tanggal dalam arsip, jadi metode pertama ini tidak akan berfungsi kecuali jika flag -m digunakan selama ekstraksi. Metode terakhir adalah optimal, tetapi dapat mengakibatkan penghapusan file yang Anda inginkan jika nama file bertabrakan.

find mendukung -newer file bendera, menentukannya harus menemukan file yang lebih baru dari mengajukan . touch memiliki argumen -t untuk mengubah waktu akses / modifikasi pada file. Jadi untuk memperbaiki oops yang terjadi sekitar pukul 19.25.30:

$ tar xzf whoops.tar.gz
$ touch -t 200909261925.30 whoops-timestamp
$ find . -newer whoops-timestamp

Dan jika Anda yakin itu menampilkan file yang benar:

$ find . -newer whoops-timestamp -print0 | xargs -0 rm -v

Alternatifnya adalah menghapus semua file yang tercantum dalam arsip yang baru saja Anda ekstrak:

$ tar tfz whoops.tar.gz | xargs rm -v

Alternatif lain dengan find:

$ find "/path/to_clean/" -type f -mtime +30 -print0 | xargs -0 rm -f

di mana +30 adalah jumlah hari yang ingin Anda pertahankan.
