Mengapa variabel lingkungan saya tidak disetel?

Saya harus menetapkan variabel lingkungan yang disebut GDK_NATIVE_WINDOWS untuk diperbaiki masalah dengan tombol Eclipse di Ubuntu . Untuk mengatur variabel lingkungan, saya menambahkan baris berikut ke ~/.pam_environment.

GDK_NATIVE_WINDOWS DEFAULT = benar

Anehnya, variabel lingkungan tidak disetel ketika saya echo $GDK_NATIVE_WINDOWS di terminal. Namun, semua variabel lingkungan lain yang saya daftarkan di ~/.pam_environment disetel dengan benar. Selain itu, ketika saya beralih ke tty, mis. Alt+Ctrl+F1, variabel lingkungan disetel dengan benar. Apa yang salah dengan menyetel variabel lingkungan ini pada ~/.pam_environment?

Tetap berpegang pada sintaks "key = value" sederhana dalam file ~ / .pam_environment. Tanpa DEFAULT, OVERRIDE, $ {HOME}, tidak ada apa-apa. Hanya kunci = nilai. Halaman manual yang Anda tautkan hanya untuk pam_env.conf. Lihat "man 8 pam_env" yang tidak menjanjikan hal lain "Modul ini juga dapat mengurai file dengan pasangan KEY = VAL sederhana pada baris terpisah.".

Ngomong-ngomong, meletakkan sesuatu seperti PATH = $ {PATH}: more / paths in ./pam_environment adalah cara yang bagus untuk menghentikan hampir semua upaya untuk masuk, karena PATH disetel ke $ {PATH}: more / paths secara harfiah (tidak banyak yang bisa ditemukan di sana). Untuk menyelamatkan, "/ bin / mv .pam_environment out_of_the_way" dan login ulang.

Seperti yang telah Anda identifikasi, variabel lingkungan Anda harus disetel pada ~/.pam_environment seperti yang direkomendasikan pada https://help.ubuntu.com/community/EnvironmentVariables . Lebih mudah diucapkan daripada dilakukan;)

Mungkin saja Anda mengalami celah konfigurasi yang sama dengan yang saya alami. Lihat solusi untuk rumah terenkripsi di bawah.

~/.pam_environment saya:

PATH            DEFAULT=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/games:${HOME}/bin
IDEA_JDK        DEFAULT=${HOME}/Applications/jdk

Mengapa jalur statis yang jelek? ${PATH} tidak akan berhasil untuk saya. Saya mem-bata login saya beberapa kali mencoba mengatasinya jadi saya tetap menggunakan salinan statis default yang jelek :)

Solusi untuk folder utama terenkripsi

Di Ubuntu, rilis hingga dan termasuk Ubuntu 12.04 (Precise Pangolin) Beta 2, jika Anda menggunakan direktori home terenkripsi, Anda perlu memodifikasi /etc/pam.d/common-session agar memuat ~/.pam_environment. Solusi ini tampaknya berfungsi untuk rilis sebelumnya, tetapi saya belum mengujinya.

Guenther Montag (g-montag) menulis pada 2010-08-19:

Ini tampaknya menjadi masalah dengan direktori beranda yang dienkripsi. saya tambahkan

sesi diperlukan pam_env.so

di akhir /etc/pam.d/common-session dan sekarang ~ / .pam_environment dibaca. Pada sistem lain tanpa direktori home terenkripsi (juga 10,04) penyelesaian tidak diperlukan. Mungkin dalam kasus saya, sistem mencoba membaca ~ / .pam_environment sebelum didekripsi.
