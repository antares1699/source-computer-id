Bagaimana cara mengubah format tanggal / waktu melalui command prompt / PowerShell?

Saya mencoba mengotomatiskan pengaturan PC saya sebanyak mungkin dan saya bertanya-tanya apakah mungkin untuk mengubah format tanggal / waktu melalui baris perintah.

Jika ya, saya ingin tahu perintahnya.

Format tanggal / waktu dikodekan sebagai bagian dari wilayah . Anda dapat mengatur ini dari CLI melalui PowerShell Set-Culture cmdlet. Ini adalah bagian dari International modul tersedia sejak Windows 8 / Server 2012. Pada dasarnya Anda mencari yang setara dengan applet Wilayah Panel Kontrol GUI (yang dapat ditampilkan dengan control /name Microsoft.RegionAndLanguage).

Contoh

Untuk yang tidak sabar, perintah PowerShell dasar adalah:

$culture = Get-Culture
$culture.DateTimeFormat.ShortDatePattern = 'd/MM/yyyy'
Set-Culture $culture

Ada beberapa pola tanggal / waktu lain yang telah saya cantumkan di bagian bawah yang mungkin ingin Anda ubah.

Detail

Pertama, jika Anda menggunakan Windows 8, Anda harus mengimpor modul. Ini tidak perlu di Windows 10.

Import-Module International

Sekarang, jika Anda ingin mengubah seluruh wilayah Anda, Anda dapat melakukannya dalam satu perintah, misalnya:

Set-Culture 'en-AU'

Kamu bisa menggunakan apapun yang valid BCP-47 tandai di sini. Perhatikan bahwa ini juga akan mengubah pengaturan terkait kawasan lainnya, seperti pemisah desimal.

Ini adalah pendekatan yang sering saya rekomendasikan, kecuali jika Anda memiliki kebutuhan khusus.

Jika Anda ingin mengontrol format tanggal / waktu secara lebih spesifik, Anda harus membuatnya sendiri CultureInfo menolak atau mengambil budaya saat ini dan memodifikasinya:

$culture = Get-Culture
$culture.DateTimeFormat.ShortDatePattern = 'd/MM/yyyy'
Set-Culture $culture

Properti tanggal / waktu yang relevan

Anda dapat melihat daftar opsi yang ada hanya dengan menjalankan $culture.DateTimeFormat (atau Get-Culture.DateTimeFormat). Yang mungkin ingin Anda ubah adalah:

FullDateTimePattern
LongDatePattern
LongTimePattern
ShortDatePattern
ShortTimePattern

Saya akan merekomendasikan untuk tidak menyentuh pola RFC1123 dan pola yang dapat diurutkan.
