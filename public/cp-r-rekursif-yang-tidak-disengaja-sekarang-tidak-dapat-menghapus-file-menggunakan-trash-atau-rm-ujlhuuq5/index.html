Cp -R rekursif yang tidak disengaja, sekarang tidak dapat menghapus file menggunakan Trash atau rm

Setelah salah ketik di jalur saat menggunakan cp, sepertinya saya secara tidak sengaja menyalin folder ke folder itu sendiri secara rekursif, dan sekarang saya tidak dapat menghapus folder itu menggunakan rm -rf, atau Sampah di Mac OS X Finder. File tersebut terletak di startup (karena itu HFS + J) disk.

Output dari rm -rf hanya Directory not empty diulang, seperti:

rm: o/old/old/old/old/old/old/old/old: Directory not empty
rm: o/old/old/old/old/old/old/old: Directory not empty
rm: o/old/old/old/old/old/old: Directory not empty
rm: o/old/old/old/old/old: Directory not empty
rm: o/old/old/old/old: Directory not empty
rm: o/old/old/old: Directory not empty
rm: o/old/old: Directory not empty
rm: o/old: Directory not empty
rm: o: Directory not empty

Apa yang harus saya lakukan untuk menghapus folder ini?

Saya sudah mencoba mengganti nama folder pertama, seperti yang Anda lihat, untuk mempersingkat jalur file. Folder tersebut dipindahkan ke denda Trash, tetapi tidak akan dihapus karena dialog peringatan berikut:

The operation can’t be completed because the item is in use.

Mencoba

find o -delete

atau

find o -depth -delete

-depth memberitahu find untuk pergi ke bagian terdalam dari pohon direktori terlebih dahulu, dan kemudian bekerja kembali. Halaman manual mengatakan, "Penggunaan -delete secara otomatis mengaktifkan opsi -depth." Jika versi find Anda tidak mendukung -delete, coba

find o -depth -exec rm -rf {} ";"

Jika Anda mengerti, itu sudah jelas. Karena Anda tidak dapat mengulang jauh di lubuk hati, Anda harus memindahkan direktori yang satu tingkat lebih dalam hanya satu tingkat ke atas. Setelah itu direktori dengan kedalaman "satu" tetap ada yang bisa dihapus.

Anggaplah Anda memiliki direktori "tak terbatas" a / a / a / a / a / a / a / ....

lalu Anda melakukan:

mv a/a b
rm -r a
mv b/a a
rm -r b

Ulangi sampai tidak ada direktori yang tersisa.

Skrip berikut melakukannya: (Anda harus menekan ctrl-c setelah selesai)

while [ true ]; do mv a/a b; rm -r a; mv b/a a; rm -r b; done

Karena namanya identik, skrip 'membalik' di antara dua nama sementara.
