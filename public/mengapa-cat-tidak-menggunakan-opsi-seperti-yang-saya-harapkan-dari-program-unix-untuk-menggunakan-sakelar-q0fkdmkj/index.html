Mengapa cat tidak menggunakan opsi seperti yang saya harapkan dari program UNIX untuk menggunakan sakelar?

Saya telah menjadi pengguna UNIX selama bertahun-tahun daripada yang saya pikirkan, dan pada saat itu saya telah dilatih untuk mengharapkan bahwa ketika sakelar yang kontradiktif diberikan ke suatu program, yang terakhir menang. Baru-baru ini saya memperhatikan itu

cat -bn file

dan

cat -nb file

keduanya menggunakan opsi -b (nomor baris tidak kosong) di atas opsi -n (nomor semua baris). Saya mendapatkan perilaku ini di BSD dan Linux, jadi menurut saya ini bukan quirk implementasi. Apakah ini sesuatu yang ditentukan di suatu tempat dan apakah saya gila karena mengharapkan contoh pertama memberi nomor pada semua baris?

Saya melihat di Kode sumber FreeBSD untuk cat (1) , dan baris sumber yang relevan adalah:

case 'b':
    bflag = nflag = 1;  /* -b implies -n */

Jadi ini terlihat seperti keputusan desain yang disengaja; interpretasi dari -b adalah bahwa ia mengubah perilaku -n, daripada -b dan -n menjadi dua alternatif yang saling eksklusif.

Bagikan
coneslayer — Sumber
  • Itu adalah keputusan yang aneh karena mereka didokumentasikan berperilaku berbeda (nomor semua baris vs nomor baris kosong). Jika -n didokumentasikan menjadi "garis bilangan" tanpa kata "semua", saya setuju dengan kodenya. Hmm, tapi melihat halaman manual BSD tertulis "Beri nomor baris keluaran, mulai dari 1", jadi ini benar-benar hanya masalah dengan dokumentasi GNU cat.
    — Chas. Owens
