Google Chrome berhenti tiba-tiba

Beberapa jam yang lalu Chrome ditutup di MacBook Air saya dengan OS X Yosemite 10.10.2 dan tidak ada cara untuk memulihkannya. Setiap kali saya mencoba meluncurkannya, pesan "Google Chrome berhenti tiba-tiba", memberi saya opsi untuk mengirim laporan ke Apple, Abaikan, atau mulai ulang Chrome. Dan hal yang sama terjadi setelahnya.

Saya telah mencoba meluncurkan sesi aman Chrome dengan opsi "--safe-plugins" namun tidak berhasil, berikut ini yang terjadi saat saya melakukan ini dari terminal:

Tebak saja:

Ini mungkin masalah dengan profil pengguna Anda di Chrome. Saya tahu saya pernah mengalami masalah ini sebelumnya di PC Windows. Ini masuk akal karena menghapus dan menginstal ulang aplikasi di Mac tidak akan menghapus file pengguna ini.

Mungkin saja mereka telah rusak atau rusak.

Navigasi ke:

~/Library/Application Support/Google/Chrome/

Ubah nama folder Default menjadi seperti Default-Backup dan luncurkan kembali Chrome. Ini harus menghasilkan profil default baru dan semoga masalah Anda akan diperbaiki.

Namun, melakukan ini akan menyebabkan Anda kehilangan data. Aktifkan sinkronisasi Chrome sebelum Anda memulai proses ini jika Anda khawatir kehilangan apa pun.

Saya kira jika perbaikan berfungsi, Anda dapat 'memilih ceri' file dan folder dari direktori cadangan sampai mulai mengalami masalah lagi untuk mempersempit file yang mungkin menjadi akar penyebab crash.

Edit:

Masalah:

Dilihat dari jumlah perhatiannya masalah kromium telah diatasi selama beberapa hari terakhir, sepertinya masalah yang Anda gambarkan disebabkan oleh masalah dengan driver panel sentuh Mac .

Tetapi ini adalah bug yang aneh karena jelas tidak memengaruhi semua orang, bahkan orang-orang dengan versi Mac OS X (10.10.2) yang paling baru. Tapi menurut ini , masalahnya berasal dari Apple depresiasi pemilih isResting dan kemudian menghapusnya sepenuhnya di pembaruan terkini.

Sekali lagi, kesalahan yang dilaporkan orang terlihat seperti ini:

[NSTouch _initWithPreviousTouch:newPhase:position:isResting:]: unrecognized selector sent to instance

Solusinya:

Sepertinya beberapa orang hanya memasang mouse eksternal dan jika mereka tidak menyentuh trackpad, tidak apa-apa. Jika ini terdengar seperti solusi yang baik, pastikan Anda juga mengaktifkan "Abaikan trackpad saat mouse". di pengaturan Aksesibilitas.

Anda dapat mencoba menurunkan versi Chrome jika opsi sebelumnya terdengar merepotkan. Tidak yakin apakah ini akan berhasil, tetapi ini mungkin usaha yang bagus.

Saya membayangkan tambalan akan didorong keluar ke canary build selama beberapa hari ke depan atau Apple akan mengirimkan pembaruan driver baru (tetapi mungkin tidak karena mereka bekerja untuk menghapus isResting). Saya akan terus mewaspadai hal itu.

Opsi 2 dari jawaban Pippyn berfungsi sehingga memberikan perbaikan sementara penuh (saya tidak bisa begitu saja memberi suara positif pada jawabannya):

[1] Masukkan baris berikut ke dalam file "patch.m":

#import <AppKit/AppKit.h>

__attribute((constructor)) void Patch_10_10_2_entry()
{
    NSLog(@"10.10.2 patch loaded");
}

@interface NSTouch ()
- (id)_initWithPreviousTouch:(NSTouch *)touch newPhase:(NSTouchPhase)phase position:(CGPoint)position isResting:(BOOL)isResting force:(double)force;
@end

@implementation NSTouch (Patch_10_10_2)
- (id)_initWithPreviousTouch:(NSTouch *)touch newPhase:(NSTouchPhase)phase position:(CGPoint)position isResting:(BOOL)isResting
{
    return [self _initWithPreviousTouch:touch newPhase:phase position:position isResting:isResting force:0];
}
@end

Kompilasi:

clang -dynamiclib -framework AppKit patch.m -o patch.dylib

Gunakan:

env DYLD_INSERT_LIBRARIES=/path/to/patch.dylib "/path/to/Google Chrome.app/Contents/MacOS/Google Chrome"

Ada masalah dengan driver trackpad baru di OSX 10.10.2.

PILIHAN 1: Anda dapat menggunakan mouse eksternal dan menonaktifkan penggunaan trackpad saat mouse USB terhubung. (Aksesibilitas> Mouse & Trackpad> Abaikan trackpad internal saat mouse atau trackpad nirkabel tersedia)

PILIHAN 2: (ditemukan di Reddit ) Perbaikan ini bukan solusi terbaik, tetapi berhasil. Saat Anda mengikuti petunjuk, Anda harus menjalankan perintah terminal terakhir setiap kali Anda ingin menjalankan Chrome. Tetapi Anda juga dapat membuat peluncur chrome dengan aplikasi automator dan perintah terminal.
