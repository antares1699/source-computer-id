Mengedit file /etc/launchd.conf di OS X

Saya mencoba menjalankan proyek Android menggunakan Maven in Mac 10.7.4 , untuk itu saya perlu mengatur variabel lingkungan dan jalur.

Ini adalah langkah-langkah yang harus saya ikuti untuk menjalankan android menggunakan Maven

Maven 3.0.3+ diinstal,

Setel variabel lingkungan ANDROID_HOME ke jalur SDK Android yang Anda pasang dan tambahkan $ ANDROID_HOME / tools serta $ ANDROID_HOME / platform-tools ke $ PATH Anda. (atau di Windows% ANDROID_HOME% \ tools dan% ANDROID_HOME% \ platform-tools).

Pengguna MacOS: Perhatikan bahwa agar jalur berfungsi pada baris perintah dan di IDE yang dimulai oleh launchd Anda harus mengaturnya di /etc/launchd.conf dan BUKAN di .bashrc atau yang lainnya

Tapi saya sangat baru mengenal OS X jadi saya tidak tahu bagaimana mengaturnya. Saya melihat ini pos tetapi saya terjebak di Langkah 4, menyimpan file etc/launchd.conf setelah memasuki jalur.

  • Bagaimana cara mengatur variabel lingkungan secara global?

  • Apa yang saya perlukan untuk membuka, mengedit, menyimpan file etc/launchd.conf?

Apakah Anda benar-benar membutuhkan variabel di seluruh sistem? Jika variabel di lingkungan pengguna Anda mencukupi, Anda dapat membuat direktori ~/.MacOSX dan file environment.plist di dalamnya, jika salah satunya tidak ada, lalu tambahkan variabel Anda ke environment.plist.

mkdir ~/.MacOSX
touch ~/.MacOSX/environment.plist
open ~/.MacOSX/environment.plist

Keluar dan kembali agar perubahan menjadi aktif.

Anda dapat menggunakan editor apa pun untuk mengubah file /etc/launchd.conf, Anda tidak harus menggunakan vi atau vim.

Karena Anda memerlukan hak pengguna super untuk mengedit file, Anda tidak bisa begitu saja menjalankan TextEdit. Namun apa yang dapat Anda lakukan adalah membuka Terminal dan masuk:

sudo touch /etc/launchd.conf
sudo /Applications/TextEdit.app/Contents/MacOS/TextEdit /etc/launchd.conf

Sekarang, TextEdit akan membuka /etc/launchd.conf. Lakukan perubahan seperti yang dijelaskan di jawaban Stack Overflow ini Anda sebutkan. Pastikan untuk mengganti bagian yang disorot dengan jalur sebenarnya ke Android SDK Anda, juga mengubah nama pengguna Anda.

setenv ANDROID_HOME /Users/your-username/Documents/android-sdk/

Sekarang simpan file seperti biasa melalui File »Simpan atau Cmd S .

Reboot mungkin diperlukan.
