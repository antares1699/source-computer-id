Bagaimana cara menafsirkan keluaran w32tm?

08:38:08 h: + 00.0311860s o: + 00.2011040s

  • 08:38:08 adalah waktu setempat

  • d:+00.0311860s adalah internal delay (perbedaan waktu antara paket udp yang diterima dan paket udp yang dikirim di sisi server

  • o:+00.2011040s sebenarnya adalah offset antara waktu lokal dan waktu server

Itu masih menyisakan d: ... apakah itu latensi ping?

Apakah saya perlu menambahkan detik-detik tersebut dengan o: untuk mendapatkan waktu terakhir dari server waktu jarak jauh?

o: adalah offset sebenarnya, jadi Anda tidak perlu menambahkan d: untuk mendapatkan waktu terakhir.

Aplikasi konsol w32tm.exe memungkinkan pemantauan offset waktu lokal ke waktu server waktu internet.

Cara termudah untuk melakukannya adalah dari jendela konsol dengan berikut ini set parameter:

w32tm /stripchart /computer:time.windows.com /period:120

Akibatnya, waktu sistem dan offsetnya ke server waktu dibuang ke konsol setiap 120 detik:

Tracking time.windows.com [65.55.21.14.123]

08:38:57 d:+00.0417301s o:+00.1024506s [ * ]

Setiap baris terdiri dari waktu lokal (08:38:57), penundaan internal (perbedaan waktu antara paket udp yang diterima dan paket udp yang dikirim di sisi server, yaitu, d: + 00.0419394s), offset aktual di antaranya waktu lokal dan waktu server (o: + 00.1024506s) dan yang sangat mendasar stripchart dari offset.
