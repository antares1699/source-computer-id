Di Linux, apakah urutan perangkat konstan?

Misalnya akankah sda2 menjadi sda2 pada setiap boot? Bagaimana dengan NIC? Akankah eth0 selalu menjadi kartu yang sama?

Kebanyakan sistem linux / Unix menggunakan udev untuk mengambil perangkat yang dimasukkan kernel / sys dan membuat symlink di / dev berdasarkan berbagai aturan.

Aturan Udev masuk /etc/udev/rules.d dan terkadang di /lib/udev/rules.d - Anda dapat mengubahnya agar sesuai dengan kebutuhan Anda (misalnya, mengubah eth0 dan eth1) dengan mengubah file aturan yang relevan.

Sunting: Saya harus menambahkan itu, itu mungkin untuk berjalan tanpa udev, dan sebagai gantinya membuat symlink statis untuk perangkat (yang mungkin diinginkan dalam konfigurasi server di mana perangkat keras tidak mungkin sering berubah). Namun demikian, jenis sistem ini lebih sulit untuk dipelihara, karena Anda secara manual mengambil peran mengisi / dev dengan perangkat keras baru di masa mendatang.

Saya percaya mereka biasanya begitu. Angka 2 di sda2 berarti itu adalah partisi kedua dan itu tergantung pada tabel partisi dan biasanya tidak berubah.

Huruf a, b, c pada nama sda, sdb, sdc memiliki urutan yang sama sesuai dengan urutan penyambungan secara fisik ke pengontrol.

Eth0 mungkin akan tetap eth0 juga. Dan saya tidak tahu tentang perangkat lain, tetapi saya tidak melihat alasan mengapa pesanan mereka juga tidak berubah.

Bagikan
robo — Sumber
  • The letters a, b, c in the names sda, sdb, sdc are in the same order according to the order in which they are physically connected to the controller. - kadang-kadang. Mereka diurutkan dalam urutan mana pun yang dideteksi kernel terlebih dahulu (sama seperti perangkat jaringan) yang mana bisa berubah di setiap boot.
    — Xyon
