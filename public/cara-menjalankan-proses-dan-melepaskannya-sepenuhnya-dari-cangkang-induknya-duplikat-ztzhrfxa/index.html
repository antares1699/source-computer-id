Cara menjalankan proses dan melepaskannya sepenuhnya dari cangkang induknya [duplikat]

Kemungkinan Duplikat:
Biarkan skrip bash berjalan di terminal jarak jauh saat tidak masuk?
Bagaimana cara melepaskan proses dari Terminal, seluruhnya?

Saya menjalankan program di server linux yang membutuhkan waktu berhari-hari untuk menyelesaikannya.

Saya meluncurkannya dari stasiun kerja saya dari terminal SSH, karena program ini hanya baris perintah.

Saya ingin bisa melakukan semua ini:

  • luncurkan program itu,
  • mengarahkan keluaran standar ke file,
  • keluar dari sesi SSH saya tanpa membuat ini menghentikan proses.

Saya berpikir tentang $ ./MyProg.csh -params -foo -bar </dev/null 1>~/out.log 2>~/err.log &

Namun, prosesnya dihentikan saat saya menutup sesi SSH saya. Stasiun kerja saya menjalankan Windows XP, dan saya tidak dapat menjamin waktu aktifnya selama beberapa hari, yang diperlukan untuk pemrosesan data saya di server Linux.

Seperti yang telah Anda catat, program saya harus diluncurkan dari CSH.

Apakah mungkin melakukan ini?

Terima kasih.

Anda dapat menggunakan screen atau nohup

jika Anda memilih screen:

luncurkan screen sebelum Anda menjalankan layanan Anda:

$ screen

dan kemudian, jalankan layanan Anda di dalam screen:

./MyProg.sh > myprog.log (atau apapun yang Anda inginkan di sini).

lalu Ctrl + Sebuah d

ketika Anda kembali, cukup:

$ screen -r

untuk informasi lebih lanjut: http://www.gnu.org/s/screen/

tentang nohup: hanya:

nohup ./MyProg.sh > myprog.log &

Buat pembungkus cangkang seperti ini:

#!/bin/sh
exec </dev/null
exec >> ~/out.log
exec 2>> ~/err.log
exec setsid ./MyProg.csh -params -foo -bar

Jika Anda membutuhkan file PID juga, kirimkan skrip shell ini ke bg dan baca $! (PID dari proses bg terakhir). Anda dapat melakukannya menggunakan subkulit dengan () dalam skrip yang sama seperti ini:

#!/bin/sh
( exec </dev/null
  exec >> ~/out.log
  exec 2>> ~/err.log
  exec setsid ./MyProg.csh -params -foo -bar
) &
kill -0 $! > /dev/null 2>&1 || exit 1
echo $! > program.pid
exit 0

Sambungan terputus karena shell mengirimkan SIGHUP ke semua turunan saat keluar. Jika Anda menggunakan bash atau zsh, Anda dapat menggunakan disown dan shell tidak akan lagi mengirim SIGHUP dan prosesnya akan tetap berjalan.

screen dan setsid di atas berfungsi, tetapi jawaban ini memiliki keuntungan karena tidak harus mengingat untuk melakukan layar sebelum Anda memulai apa pun. screen memiliki banyak kekuatan selain ini, jadi Anda mungkin ingin membiasakan diri menggunakannya.
