Memindahkan direktori ProgramData di Windows 10 saat menginstal

AS dirinci dalam panduan ini: http://www.tenforums.com/tutorials/1964-users-folder-move-location-windows-10-a.html , Saya telah memindahkan direktori Users saya dari default C:\Users ke D:\Users. Saya menggunakan file jawaban ini, yang saya simpan sebagai relocate.xml:

<xml version="1.0" encoding="utf-8"\>

<unattend xmlns="urn:schemas-microsoft-com:unattend">

<settings pass="oobeSystem">

<component name="Microsoft-Windows-Shell-Setup" processorArchitecture="amd64" publicKeyToken="31bf3856ad364e35" language="neutral" versionScope="nonSxS" xmlns:wcm="http://schemas.microsoft.com/WMIConfig/2002/State" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">

<FolderLocations>

<ProfilesDirectory>D:\Users</ProfilesDirectory>

</FolderLocations>

</component>

</settings>

<cpi:offlineImage cpi:source="wim:X:\sources\install.wim#Windows 10 Pro" xmlns:cpi="urn:schemas-microsoft-com:cpi" />

</unattend>

Tapi saya juga berpikir bahwa saya perlu memindahkan ProgramData juga. Jadi ke relocate.xml, saya menambahkan:

<ProgramData>D:\ProgramData</ProgramData>

di baris 7.

Ketika saya melakukan perintah ini:

% windir% \ system32 \ sysprep \ sysprep.exe / oobe / reboot /unattend:Y:\relocate.xml

Ini menjalankan Sysprep, tetapi setelah meminta saya untuk memasukkan pengaturan saya, sistem sepertinya terjebak dalam satu lingkaran. Tidak seperti saat saya menggunakan relocate.xml asli seperti di panduan, yang melengkapi dan memasukkan saya ke akun baru.

Pencarian umum tentang merelokasi direktori ProgramData tampaknya tidak mengarah ke panduan apa pun. Jadi apa alasan tidak memiliki cara langsung untuk merelokasi ProgramData?

Saya telah menemukan masalah yang sama bahkan ketika didokumentasikan sebagai parameter yang valid tampaknya tidak.

Ada solusi. Anda dapat menyalin seluruh folder (programdata) ke tujuan baru Anda (saya akan menggunakan boot winpe dan menggunakan robocopy untuk mempertahankan semuanya persis sama)

Sesuatu seperti

robocopy "C:\ProgramData" "D:\ProgramData" /E /ZB /DCOPY:T /COPYALL /R:1 /W:1 /V /TEE

Setelah itu Anda dapat menghapus folder programdata dan menambahkan junction ke dalamnya

mklink /j "C:\ProgramData" "D:\ProgramData"

Ingatlah bahwa ini tidak dapat berfungsi dalam beberapa skenario, saya akan mencoba instalasi bersih atau mesin virtual terlebih dahulu.

Karena pertanyaan ini muncul tinggi di Google saat mencari informasi tentang file jawaban tanpa pengawasan, saya memutuskan untuk menambahkan balasan lain meskipun pertanyaannya sudah lama.

Itu Dokumentasi Resmi tentang File Jawaban Tanpa Pengawasan dengan jelas menyatakan:

Data program - Menentukan jalur ke folder data program.

Peringatan Gunakan pengaturan ini hanya di lingkungan pengujian. Jika Anda mengubah lokasi default folder data program ke volume selain volume sistem, Anda tidak dapat memperbaiki gambar Anda. Setiap pembaruan, perbaikan, atau paket layanan mungkin tidak diterapkan ke penginstalan.

Oleh karena itu saya berasumsi (meskipun saya belum mengujinya) Anda terjebak dalam loop boot saat Anda mencoba menginstal semua pembaruan terbaru selama proses isntallation, yang terus gagal dan windows terus mencoba kembali.

Karena perilaku ini dikenal dan didokumentasikan, satu-satunya solusi untuk ini adalah merelokasi folder dengan cara lain (lihat balasan @ Varacolaci sebagai contoh) atau simpan di lokasi default.
