Pulseaudio: tunggu perangkat dimulai?

Berdasarkan panduan ini , Saya menyiapkan pulseaudio di Raspberry Pi saya untuk berjalan dalam mode sistem. Beginilah tampilan file layanan systemd saya:

[Unit]
Description=PulseAudio Daemon
After=sound.target
Requires=sound.target

[Install]
WantedBy=default.target

[Service]
Type=notify
ExecStart=/usr/bin/pulseaudio --disallow-exit --no-cpu-limit --daemonize=no --system --realtime --disallow-module-loading --log-target=journal -vvvv

Namun, saya tidak mendapatkan suara apa pun - menurut log, saat pulseaudio dimulai, udev-detect melaporkan "Ditemukan 0 kartu." dan saya hanya mendapatkan wastafel nol. Menjalankan systemctl --system restart pulseaudio.service memperbaiki ini dan perangkat audio saya terdeteksi dengan benar.

Adakah cara untuk memberi tahu pulseaudio agar menunggu hingga perangkat terdeteksi?

Mengatakan bahwa PulseAudio tidak akan berguna. Ia sudah dapat mendeteksi perangkat hotplugged dengan sendirinya; namun, karena --disallow-module-loading itu tidak lagi mampu menggunakan perangkat, karena dilarang memuat module-alsa-card setelah perangkat terdeteksi. (Bisa dibilang module-udev-detect harus dibebaskan dari pembatasan ini…)

Kamu dapat katakan systemd untuk menunda memulai layanan pulseaudio.sampai perangkat tertentu siap, dengan mencantumkan nama unit .device yang sesuai di After= unit ini, misalnya:

[Unit]
After=sys-devices-pci0000:00-0000:00:1b.0-sound-card0.device

  • Tidak ada gunanya menunggu ALSA "secara umum". (Semua inisialisasi perangkat terjadi di kernel; alsa-restore.service hanya memuat level volume dari disk ke sudah terdeteksi perangkat audio.) Jika Anda mengkonfigurasi PA module-alsa-card secara manual, yang perlu Anda tunggu hanyalah perangkat kartu itu sendiri.
