Mengapa IIS dapat menerima lalu lintas HTTP 80, meskipun aturan firewall dinonaktifkan?

Menggunakan Windows7 Home Premium. Ini adalah mesin pengembangan, tetapi itu tidak masalah.

Saya telah menginstal IIS dan dapat menjelajah ke http: // localhost situs web dari mesin lokal. Situs web default disetel untuk menggunakan port 80.

Saya menambahkan situs web lain ke IIS, menyetelnya untuk menggunakan kumpulan aplikasi 32-bit, dan port 81. ketika saya mencoba menjelajahinya, melalui localhost, saya mendapatkan "Tidak ada koneksi yang dapat dibuat karena mesin target secara aktif menolaknya 127.0.0.1 : 81 "

Ok, ini cukup mudah, bukan? Cukup aktifkan HTTP melalui port 81 di firewall Windows, bukan? Saya pikir untuk menggunakan aturan port-80 di firewall sebagai pola.

Yang mengejutkan saya, ada aturan port 80 di Windows Firewall, tapi itu tidak diaktifkan . Ini adalah snip dari Windows Firewall MMC:

Namun saya dapat terhubung ke localhost melalui port 80.
dan saya tidak dapat terhubung ke port localhost 81.

Saya cukup yakin firewall tidak memblokir / memeriksa alamat localhost / loopback (127.0.0.1) karena itu adalah komputer Anda. Jadi karena target dan sumbernya sama, sebenarnya tidak ada apa pun untuk firewall.

Ketika Anda mengaktifkan port 81, atau menonaktifkan firewall sepenuhnya, dapatkah Anda mengakses situs ke-2? Jika tidak, Anda mungkin mengalami masalah konfigurasi IIS, dan bukan masalah firewall.
