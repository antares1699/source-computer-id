Cara Mengkonfigurasi WAMPServer untuk menggunakan HTTPS SSL

* Ini bukan proses yang sepele. Mudah-mudahan, tutorial ini akan membuat SSL berfungsi untuk Anda. Namun mendapatkan itu dikonfigurasi dengan benar setelah itu bekerja BENAR-BENAR DIKENAKAN UNTUK ANDA.

Bacaan tambahan untuk semua orang yang melakukan perjalanan jalan ini

Baik,

Saya mendasarkan tutorial ini pada pembuatan situs bernama www.wamphelpers.dev Jadi, di mana pun Anda melihat nama itu, ubahlah menjadi nama situs yang Anda coba amankan.

Saya mulai dengan membuat situs tidak aman, di \ wamp \ www \ wamphelpers

menambahkan Host Virtual untuk situs itu, di httpd-vhosts.conf

<VirtualHost *:80>
    DocumentRoot "c:/wamp/www"
    ServerName localhost
    ServerAlias localhost
    <Directory  "c:/wamp/www">
        AllowOverride All
        Require local
    </Directory>
</VirtualHost>

<VirtualHost *:80>
    DocumentRoot "c:/wamp/www/wamphelpers"
    ServerName wamphelpers.dev
    ServerAlias www.wamphelpers.dev
    <Directory  "c:/wamp/www/wamphelpers">
        AllowOverride All
        Require local
    </Directory>
</VirtualHost>

Menambahkan namanya ke C: \ windows \ system32 \ drivers \ etc \ hosts

127.0.0.1 wamphelpers.dev www.wamphelpers.dev
::1       wamphelpers.dev www.wamphelpers.dev

Sekarang restart dnscache sebagai berikut dari jendela perintah yang diluncurkan menggunakan 'Run as Administrator'

net stop dnscache
net start dnscache

Kemudian buat script sederhana di \ wamp \ www \ wamphelpers \ index.php

<?php
    echo 'Hello, this is the WAMPHELPERS.DEV site homepage';
?>

Sekarang restart Apache dan pastikan bahwa situs sederhana tanpa jaminan Anda berfungsi before continuing

Toolkit openssl. Openssl.exe, ssleay32.dll dan libeay32.dll disertakan dengan, dan terletak di, folder C: \ wamp \ bin \ apache \ apachex.y.z \ bin Ini harus menjadi semua yang Anda butuhkan untuk membuat sertifikat yang ditandatangani sendiri !!

NAMUN: Ini tidak berfungsi untuk saya pada salah satu versi Apache yang telah saya instal. Saya selalu mendapat pesan kesalahan ini.

Di mana nomor urut berubah tergantung pada folder versi apache tempat saya berada.

Jika Anda mendapatkan kesalahan ini jangan khawatir, inilah yang perlu Anda lakukan.

instal OPENSSL TOOLKIT versi terbaru

Ini bisa didapatkan dari sini

Pilih versi terbaru dari 'Win32 OpenSSLv xxx Light' karena hanya ini yang Anda butuhkan.

Ini akan mengunduh file .exe yang dapat Anda jalankan untuk menginstal toolkit ini.

Ini akan menanyakan pertanyaan berikut, saya sarankan Anda menjawabnya seperti ini sehingga Anda tidak menginstal sesuatu ke C: \ windows \ system32. Lagipula ini adalah toolkit dan sering berubah. Yang terbaik adalah menjaga agar hal-hal ini tetap terpisah dan tidak menjadikannya sistem global.

Setelah itu diinstal (ke folder mana pun yang Anda tentukan dalam penginstalan), Anda harus siap untuk memulai proses pembuatan kunci dan sertifikat!

Hasilkan kunci dan Sertifikat.

LANGKAH 1: Buat Kunci Pribadi RSA

Pertama kita perlu membuat sertifikat untuk diri kita sendiri. Proses normal (dibayar) adalah membuat sertifikat Anda dan kemudian meneruskannya ke otoritas penandatanganan. Inilah sebabnya mengapa membutuhkan uang, sebagaimana yang harus mereka lakukan, uji tuntas, untuk memeriksa bahwa Anda adalah yang Anda katakan dan bahwa situs tempat Anda akan menggunakan sertifikat itu nyata dan sah.

Toolkit openssl digunakan untuk menghasilkan Kunci Pribadi RSA dan CSR (Permintaan Penandatanganan Sertifikat) untuk digunakan pada Sertifikat kami. Langkah pertama adalah membuat Kunci Pribadi RSA Anda. Kunci ini adalah kunci RSA 1024 bit yang dienkripsi menggunakan Triple-DES dan disimpan dalam format PEM sehingga dapat dibaca sebagai teks ASCII.

Buka jendela Command (kotak Dos) menggunakan [b] Run as Administrator [/ b] Ubah Direktori ke tempat Anda menginstal OpenSSL Toolkit di atas. Dalam kasus saya ini

CD c:\apps\OpenSSL-Win32\bin

Buat folder untuk output yang akan dimasukkan (untuk menjaga folder bin rapi) Saya menggunakan situs web

md website

Sekarang masukkan perintah ini:

openssl genrsa -out website\server.key 2048

Ini seharusnya membuat file di folder situs web bernama privkey.pem, tanpa kunci frase sandi, periksa keberadaannya.

Langkah 2: Buat CSR (Permintaan Penandatanganan Sertifikat)

Selama pembuatan CSR, Anda akan diminta memberikan beberapa informasi. Ini adalah atribut X.509 dari sertifikat tersebut. Salah satu petunjuknya adalah untuk "Nama Umum (mis. FQDN server atau nama ANDA) []:". Kolom ini harus diisi dengan nama domain server yang memenuhi syarat untuk dilindungi oleh SSL. Jadi jika situs web yang akan dilindungi adalah https://www.wamphelpers.dev, kemudian masukkan www.wampheplers.dev pada prompt ini.

Jangan masukkan apa pun ke pertanyaan: Sandi tantangan []: ] Cukup tekan Enter. Jika Anda memasukkan kata sandi di sini ketika Anda memulai Apache dengan SSL dikonfigurasi Apache tidak akan mulai dan akan memberikan pesan kesalahan ini: -

[error] Init: SSLPassPhraseDialog builtin tidak didukung di Win32

Pada dasarnya jika Anda memasukkan frasa sandi, Apache seharusnya menantang Anda untuk frasa sandi itu setiap kali dimulai. Ini jelas tidak akan membuat hidup Anda lebih mudah tetapi terutama di windows itu tidak benar-benar berfungsi dan akan menyebabkan Apache macet ketika mencoba meminta frasa sandi, dengan kesalahan di atas.

Perintah untuk menghasilkan CSR adalah sebagai berikut:

openssl req -new -key website\server.key -out website\server.csr


Example question and answers:
Country Name (2 letter code) [AU]:GB
State or Province Name (full name) [Some-State]: Hampshire
Locality Name (eg, city) []: Portsmouth
Organization Name (eg, company) [Internet Widgits Pty Ltd]: Wamp Helpers Ltd
Organizational Unit Name (eg, section) []: Information Technology
Common Name (e.g. server FQDN or YOUR name) []: www.wamphelpers.dev
Email Address []: me@wamphelpers.dev

Please enter the following 'extra' attributes
to be sent with your certificate request
A challenge password []: ( leave blank just hit the enter key )
An optional company name []: ( leave blank just hit the enter key )

Langkah 3: Membuat Sertifikat yang Ditandatangani Sendiri

Pada tahap ini Anda perlu membuat sertifikat yang ditandatangani sendiri karena Anda juga tidak berencana agar sertifikat Anda ditandatangani oleh CA, atau Anda ingin mengujinya implementasi SSL baru Anda saat CA menandatangani sertifikat Anda.

PRA - PERINGATAN Sertifikat ini akan menghasilkan kesalahan di browser klien yang menyebabkan itu otoritas sertifikat penandatanganan tidak dikenal dan tidak dipercaya. Ini tidak dapat dihindari karena kami sendiri yang menandatangani sertifikat, tetapi tentu saja web kepercayaan tidak mengetahui siapa kami. Lihat contoh nanti dalam dokumen ini yang menunjukkan cara memberi tahu browser Anda bahwa Anda benar-benar mempercayai sertifikat ini

openssl x509 -req -days 365 -in website\server.csr -signkey website\server.key -out website\server.crt


Example output:
Loading 'screen' into random state - done
Signature ok
subject=/C=GB/ST=Hampshire/L=Portsmouth/O=WampHelpers Ltd/OU=Information Technology/CN=www.wamphelpers.dev/emailAddress=riggsfolly@wamphelpers.dev
Getting Private key

Langkah 4: Memasang Kunci Pribadi dan Sertifikat

Buat 2 direktori ini di bawah versi Apache yang Anda gunakan.

md c:\wamp\bin\apache\apachex.y.z\conf\ssl.key
md c:\wamp\bin\apache\apachex.y.z\conf\ssl.crt

Dan salin file yang baru saja kita buat ke dalamnya seperti ini:

copy website\server.crt c:\wamp\bin\apache\apachex.y.z\conf\ssl.crt
copy website\server.key c:\wamp\bin\apache\apachex.y.z\conf\ssl.key

Langkah 5: Konfigurasi Apache untuk mengaktifkan SSL

Edit httpd.conf, Periksa apakah baris ini tidak diberi komentar

LoadModule ssl_module modules/mod_ssl.so

Hapus juga komentar '#' dari baris ini

Include conf/extra/httpd-ssl.conf

Kemudian pindahkan garis itu setelah blok ini .... seperti itu

<IfModule ssl_module>
SSLRandomSeed startup builtin
SSLRandomSeed connect builtin
</IfModule>

# Secure (SSL/TLS) connections
Include conf/extra/httpd-ssl.conf

Langkah 6: Konfigurasi PHP untuk mengaktifkan SSL

Edit php.ini Anda (gunakan menu wampmanager sehingga Anda mengedit yang benar)

Hapus komentar ';' dari garis ini

extension=php_openssl.dll

Langkah 7: Konfigurasikan Host Virtual situs aman Anda

Yup untuk semua yang Anda Virtual Host dan katakan, sekarang Anda tidak dapat menghindari proses tersebut.

Edit \wamp\bin\apache\apachex.y.z\conf\httpd-ssl.conf

File ini dirilis oleh Apache dan berisi beberapa lokasi file default. Kita dapat membiarkan sebagian besar file ini apa adanya, tetapi kita perlu mengkonfigurasi host virtual di sini agar sesuai dengan lokasi situs kita yang sebenarnya dan beberapa hal lain jadi:

temukan baris ini

DocumentRoot "c:/Apache2/htdocs"
ServerName www.example.com:443
ServerAdmin admin@example.com
ErrorLog "c:/Apache2/logs/error.log"
TransferLog "c:/Apache2/logs/access.log"

dan mengubahnya menjadi

DocumentRoot "c:/wamp/www/wamphelpers"
ServerName wamphelpers.dev:443
ErrorLog "c:/wamp/logs/ssl_error.log"
TransferLog "c:/wamp/logs/ssl_access.log"

Temukan

SSLCertificateFile "c:/Apache2/conf/server.crt"

dan ubah menjadi

SSLCertificateFile "c:/wamp/bin/apache/apachex.y.x/conf/ssl.crt/server.crt"

Temukan

SSLCertificateKeyFile "c:/Apache2/conf/server.key"

dan ubah menjadi

SSLCertificateKeyFile "c:/wamp/bin/apache/apache2.2.26/conf/ssl.key/server.key"

Temukan

<Directory "c:/Apache2/cgi-bin">
    SSLOptions +StdEnvVars
</Directory>

dan ubah menjadi

<Directory "c:/wamp/www/wamphelpers">
    SSLOptions +StdEnvVars
    Options Indexes FollowSymLinks MultiViews
    AllowOverride All
    Order Deny,Allow
    Deny from all
    Allow from 127.0.0.1 localhost ::1
</Directory>

Temukan

SSLSessionCache        "shmcb:c:/Apache2/logs/ssl_scache(512000)"
SSLSessionCacheTimeout  300

dan mengubahnya menjadi

SSLSessionCache        "shmcb:c:/wamp/logs/ssl_scache(512000)"
SSLSessionCacheTimeout  300

Temukan

CustomLog "c:/Apache24/logs/ssl_request.log" \
      "%t %h %{SSL_PROTOCOL}x %{SSL_CIPHER}x \"%r\" %b"

dan ubah menjadi

CustomLog "c:/wamp/logs/ssl_request.log" \
          "%t %h %{SSL_PROTOCOL}x %{SSL_CIPHER}x \"%r\" %b"

Pada dasarnya lihat melalui file conf dan perintah apa pun yang tidak diberi komentar, tetapi memiliki referensi ke file atau folder harus diubah untuk mereferensikan struktur folder WAMPServer dan bukan 'C: / Apache2 ....'

Sekarang pastikan semua file yang telah kita ubah ini disimpan, dan restart Apache menggunakan menu wampmanager.

Uji pertama apakah situs yang tidak dilindungi masih berfungsi.

Kemudian coba gunakan situs baru Anda yang dilindungi dengan menambahkan 'https: //' di depan nama domain yaitu https://www.wamphelpers.dev tanpa tanda kutip tunggal tentunya.

Jika Apache tidak restart Anda mungkin salah mengeja. Uji konfigurasi seperti ini: -

Buka jendela perintah

cd \wamp\bin\apache\apachex.y.z\bin
httpd -t

Ini akan mengurai semua file konfigurasi dan akan memberi Anda nama file dan nomor baris di mana kesalahan telah ditemukan.
