Beralih dari Windows Virtual PC ke VirtualBox tanpa pemformatan

Saya memiliki mesin virtual Win7 yang berjalan di Windows Virtual PC yang saat ini saya kembangkan. Saya menemukan bahwa saya tidak menyukai WVPC, dan menginstal VirtualBox, berharap untuk kinerja yang lebih baik.

Namun, mengimpor VHD yang ada ke VM baru tampaknya tidak berfungsi, karena meskipun saya melihat layar boot Windows, OS akan macet pada BSOD dan memerlukan alat pemulihan untuk dijalankan. Alat itu tidak menemukan masalah, reboot tetapi BSOD masih ada.

Saya tidak ingin memformat VM baru jika memungkinkan.

Apakah mungkin melakukan peralihan seperti itu?

Saya melihat pesan kesalahan dan mencari di Google:

stop: 0x0000007b (0x80786b50,0xc0000034,0x00000000,0x00000000)

Ini berarti sistem mengalami masalah dengan driver perangkat boot-nya. Jadi karena saya tahu bahwa Virtual PC menggunakan drive IDE sebagai perangkat boot sistem, saya baru saja mengganti VirtualBox untuk menggunakan perangkat IDE, bukan SATA. Kemudian, itu berhasil.

Saya memiliki masalah serupa yang datang dari VirtualPC ke VirtualBox dengan beberapa VM Linux.

Saya menyiapkan mesin virtual menggunakan Linux VHD dan memiliki perilaku dan penguncian yang agak tidak stabil meskipun ada dukungan file VirtualPC VHD.

Jadi saya mengkloning VM (Machine-> Clone). VM kloning berjalan dengan baik tanpa keanehan. Saya tidak yakin bagaimana cara memperbaikinya, tetapi saya telah melakukan ini dengan file VHD lainnya dan semuanya tampaknya berjalan cukup baik setelah kloning.

Saya belum melakukan ini dengan VM Windows. Mengingat penderitaan yang saya alami dengan memindahkan hard drive Windows kehidupan nyata dari satu mesin ke mesin lain, ada beberapa pekerjaan persiapan serius yang diperlukan untuk membuat Windows memindai ulang perangkat kerasnya dan muncul.

Saya menyadari ini adalah jenis jawaban di pinggiran, tetapi perpindahan ke VirtualBox sangat berharga.

Bagikan
