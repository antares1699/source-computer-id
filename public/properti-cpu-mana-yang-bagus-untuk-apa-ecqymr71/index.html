Frekuensi jam

Setiap siklus jam, CPU mencoba melakukan pekerjaan. Semakin cepat jam, semakin banyak pekerjaan yang dapat dilakukan. Namun, CPU sering menunggu hal-hal untuk dikerjakan yang datang dari RAM atau perangkat I / O, jadi ini sering kali menjadi faktor pembatas dalam kinerja daripada kecepatan jam. Jika apa yang Anda lakukan melibatkan penghitungan matematika yang berat atau banyak enkripsi, maka frekuensi clock yang lebih cepat menguntungkan.

Jika jenis pekerjaan yang Anda lakukan bukan matematika yang berat, jam yang lebih lambat akan menghemat daya. Begitu banyak CPU hari ini akan memungkinkan OS untuk mengatur kecepatan clock dengan beberapa cara karena alasan ini.

Cache

Apa pun yang dapat dikerjakan oleh CPU harus hidup dalam RAM. RAM eksternal di DIMM lebih lambat daripada CPU - artinya hal-hal yang menempati RAM eksternal akan membuat CPU menunggu, tetapi RAM cache berada pada atau mendekati kecepatan yang sama dengan CPU. Jadi pengontrol memori CPU akan mencoba menyimpan hal-hal yang sering diakses di cache.

Cache RAM akan sangat memengaruhi kinerja karena apa pun yang digunakan CPU harus ada dalam RAM, setidaknya untuk sementara. Anda menginginkannya sebanyak mungkin untuk sebagian besar aplikasi dan sistem operasi.

Core

Jika Anda memiliki 2 core, maka pada setiap siklus clock. pada dasarnya setara dengan 2 CPU dapat mencoba melakukan pekerjaan independen pada saat yang bersamaan. Mereka masih dibatasi oleh RAM dan I / O. Program dan sistem operasi harus ditulis untuk mendistribusikan pekerjaan di antara banyak inti (semua yang modern sekarang).

Banyak program akan menggunakan utas untuk mengerjakan sesuatu - utas menjadi bagian dari program yang dapat diselesaikan atau bekerja secara independen dari program utama. Pada sistem multi-inti, lebih dari satu utas dapat dijalankan sekaligus, yang berarti pekerjaan selesai lebih cepat. Program yang tidak membagi pekerjaan menjadi beberapa utas (program yang lebih lama, beberapa permainan) tidak akan mendapat manfaat, tetapi jika Anda menjalankan dua program seperti itu, Anda masih akan mendapat manfaat karena setiap program semacam itu dapat berada di inti CPU-nya sendiri.

Mengingat bahwa OS modern melakukan banyak hal di latar belakang dan bersaing untuk sumber daya CPU terhadap program sampai batas tertentu, hampir selalu bermanfaat untuk memiliki banyak inti, dan semakin banyak semakin baik. Mungkin tidak terlalu menjadi masalah untuk memiliki lebih dari 2 jika Anda kebanyakan menjalankan program lama atau satu program non-threaded sepanjang waktu.
