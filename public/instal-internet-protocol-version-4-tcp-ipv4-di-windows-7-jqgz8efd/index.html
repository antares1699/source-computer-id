Instal Internet Protocol Version 4 (TCP / IPv4) di Windows 7

Saya tidak bisa menjelaskan apa yang telah terjadi, tiba-tiba saya kehilangan koneksi internet. Pada properti koneksi area lokal, IPv4 tidak diinstal, ISP saya mengatakan bahwa mereka hanya mendukung IPv4. Maka saya harus memilikinya karena saya memiliki koneksi sebelumnya. Apa yang bisa terjadi jika IPv4 hilang? Dan bagaimana cara menginstalnya kembali?

cara 1:

Jalankan cmd sebagai Administrator. Jalankan perintah:

netsh interface IPV4 uninstall

Tunggu 2-3 menit, jalankan:

netsh interface IPV4 install

cara 2:

Jalankan cmd sebagai Administrator. Jalankan perintah:

mengatur ulang entri WINSOCK ke default instalasi

netsh winsock reset catalog

mengatur ulang tumpukan IPv4 TCP / IP ke default penginstalan:

netsh int ipv4 reset reset.log

atau

netsh int ip reset reset.log

perlu reboot

cara 3:

Jalankan ncpa.cpl sebagai Administrator.

Klik kanan koneksi jaringan yang Anda gunakan, dan kemudian klik Properti.

Klik pada Internet Protocol TCP / IPv4.

Klik Instal lalu.

Klik Protokol, lalu klik Tambahkan.

Klik Punya Disk.

Ketik lokasi %windir\inf%, contoh: C: \ windows \ inf lalu klik OK.

Di daftar protokol yang tersedia, klik Protokol Internet (TCP / IPv4), dan kemudian klik OK.

cara 4:

wmic nicconfig get MACAddress, IPAddress, ServiceName, Caption, IPEnabled, SettingID | find /I "Realtek"

Dapatkan NIC GUID dan del HKEY_LOCAL_MACHINE\SYSTEM\ControlSet00x\Services\{GUID} di registri.

wmic nic get Caption, PNPDeviceID, ServiceName, NetConnectionStatus | find /I "Realtek"

Dapatkan NIC PCI ID: PCI \ VEN_XXXX & DEV_YYYY & SUBSYS_ZZZZZZZ & REV_TT dan del HKEY_LOCAL_MACHINE\SYSTEM\ControlSet00x\Enum\PCI\VEN_{PCI_ID} di registri.

Jika sulit untuk memperbaiki langkah-langkah hukum, entri registri, dan kemudian menimpa pemiliknya dapat menggunakan Windows Driver Kit (WDK) 7.1.0.

Jalankan regedit dengan hak SISTEM. Perlu Windows Driver Kit (WDK) 7.1.0 :

ntsd -c qd regedit

Scott Chamberlain (c), komentar bagus !:

Anda dapat menjalankan regedit dengan hak SISTEM dengan menggunakan PSExec juga, unduhan yang jauh lebih kecil. Salah satu contoh yang mereka posting adalah menjalankan regedit sebagai SYSTEM.

psexec -i -d -s c:\windows\regedit.exe

Mulai ulang!

cara 5 (terapkan jika dalam keadaan darurat, jika metode sebelumnya tidak membantu):

/ 1. Cadangkan Registry / 2. Edit file% windir% \ inf \ Nettcpip.inf pada [MS_TCPIP.PrimaryInstall]

string komentar ;:

;    Characteristics = 0xA0 ; NCF_HAS_UI | NCF_NOT_USER_REMOVABLE

dan tambahkan

;    Characteristics = 0xA0 ; NCF_HAS_UI | NCF_NOT_USER_REMOVABLE

/ 3. Panel Kontrol-> Koneksi Jaringan-> Koneksi Area Lokal-> Properti

Instal protokol. Tekan tombol Install, pilih protokol, ketik Windows Inf path.

Pilih protokol IPv4. (Buka blokir tombol IPv4 uninstall.)

Copot IPv4.

/ 4. hapus kunci registri:

HKEY_LOCAL_MACHINE\System\CurrentControlSet\Services\Winsock
HKEY_LOCAL_MACHINE\System\CurrentControlSet\Services\Winsock2

jika perlu, hapus kunci registri:

HKEY_LOCAL_MACHINE\System\CurrentControlSet\Services\tcpip
HKEY_LOCAL_MACHINE\System\CurrentControlSet\Services\dhcp
HKEY_LOCAL_MACHINE\System\CurrentControlSet\Services\dnscache
HKEY_LOCAL_MACHINE\System\CurrentControlSet\Services\ipsec
HKEY_LOCAL_MACHINE\System\CurrentControlSet\Services\policyagent
HKEY_LOCAL_MACHINE\System\CurrentControlSet\Services\atmarpc
HKEY_LOCAL_MACHINE\System\CurrentControlSet\Services\nla

/5. Boot ulang

/ 6. Instal IPv4

/ 7. Mulai ulang!

/ 8. Pulihkan% windir% \ inf \ Nettcpip.inf

/ *. Lihat juga Menginstal ulang perangkat miniport WAN
