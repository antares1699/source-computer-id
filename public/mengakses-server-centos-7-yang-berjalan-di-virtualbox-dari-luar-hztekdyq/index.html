Mengakses server CentOS 7 yang berjalan di VirtualBox dari luar

Apakah mungkin untuk mengakses server Apache saya dari luar VirtualBox di browser Google Chrome di komputer host saya? Ini berjalan di CentOS 7 di VirtualBox. Saya telah mencoba menghubungkan ke alamat ip mesin virtual CentOS tetapi tidak berhasil. Ini menggunakan jaringan 'Bridged Adapter' dalam pengaturan VM dan saya memeriksa alamat ip menggunakan perintah 'ip addr'. Terima kasih.

Saya bertemu masalah yang sama. Saya memperbaikinya sebagai berikut:

  2. Nyalakan port CenOS7: menggunakan perintah berikut di terminal (misalnya port 8080): $ firewall-cmd --permanent --add-port=8080/tcp $ firewall-cmd --reload

Anda dapat menggunakan satu NAT (tanpa adaptor kedua):

Setel adaptor No1 ke NAT, lalu klik "Penerusan port", ketik (misalnya) 8080 di porta Host, dan 80 di port Tamu (tidak perlu alamat ip). Anda mungkin juga (opsional) ingin memetakan SSH di sini ke port yang berbeda.

Kemudian di VM CentOS 7, setelah server Apache atau NGINX Anda diinstal dan berjalan (misalnya: yum install nginx, systemctl start nginx, systemctl aktifkan nginx), konfigurasikan firewall seperti yang disarankan niaomingjian, tetapi mungkin lebih tepat:

sudo firewall-cmd --zone=public --add-service=http --permanent
sudo firewall-cmd --reload

Kemudian navigasikan di mesin host Anda ke http://127.0.0.1:8080 dan periksa apakah Anda mendapatkan halaman selamat datang. Jika ini tidak langsung berhasil, coba hentikan dan mulai layanan firewalld, atau boot ulang VM.

Anda dapat membagi masalah menjadi beberapa bagian yang lebih kecil yang akan membantu pemecahan masalah:

  • Gunakan 'ping' dari tuan rumah ke tamu untuk menentukan apakah Anda dapat menjangkaunya.
  • Periksa apakah ada firewall yang berjalan pada tamu yang memblokir port ping dan / atau www (mis. 'Iptables -L'.
  • Buka halaman web dengan browser di dalam tamu untuk memeriksa apakah itu benar-benar berjalan dan dapat diakses.
