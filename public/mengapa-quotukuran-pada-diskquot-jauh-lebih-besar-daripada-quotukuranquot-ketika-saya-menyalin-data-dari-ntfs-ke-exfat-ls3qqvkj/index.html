Mengapa "ukuran pada disk" jauh lebih besar daripada "ukuran" ketika saya menyalin data dari NTFS ke exFAT?

Saya mencoba mencadangkan dari hard drive eksternal NTFS saya ke hard drive eksternal exFAT yang baru. Ketika penyalinan selesai, saya melihat bahwa saya hanya memiliki sedikit ruang kosong yang tersisa. Saya memeriksa ukuran file saya dan saya melihat bahwa ukuran sebenarnya dari file saya adalah "30,4 GB" tapi ukuran disknya "396 GB" .

Saya harus menyebutkan bahwa jumlah file sangat tinggi (sekitar 96110 File, 10807 Folder).

Mengapa ini terjadi? Bagaimana saya harus memperbaikinya? Saya memformat hard drive baru saya dengan filesystem exFAT dengan A ukuran unit lokasi dari 4096 kilobyte .

Jika perhitungan saya benar dari ukuran file rata-rata di exFAT Anda adalah 331,7KB, tetapi Anda telah menetapkan ukuran unit minimal menjadi 4096 KB, yang berarti ada 92% ruang yang tidak terpakai di setiap unit, jadi 96110 (file) x 3768, 32 (ruang bebas) = ​​362173235,2 KB = 345,40 GB (gratis) + 30,4 (nyata) = 375,8 GB, masih kehilangan beberapa pertunjukan di suatu tempat, mungkin karena ukuran rata-rata itu.

Dengan kata lain, setel ukuran unit sekecil mungkin atau gunakan ".tar" atau semacamnya.
