Konfigurasikan izin titik pemasangan CIFS

Pada mesin Ubuntu saya, saya memasang share CIFS dengan memiliki yang berikut di /etc/fstab saya

//netgear0/photos /media/netgear0-photos  cifs  credentials=/home/bob/passwd/netgear0-smb-cred,iocharset=utf8,uid=1000,gid=1000,rw  0  0

Ini memasang share dengan benar dan muncul seperti ini:

drwxr-xr-x  1 bob  bob    0 2011-10-10 07:25 netgear0-photos

Namun saya ingin mengubah hak akses hanya pada direktori titik pemasangan ke 700 sehingga hanya bob yang dapat melihat file di titik pemasangan. Saya telah mencoba mengatur izin pada direktori sebelum memasang tetapi ini diganti selama pemasangan. Saya dapat menemukan opsi untuk mengubah izin di dalam titik pemasangan, tetapi tidak untuk mengubah titik pemasangan itu sendiri.

Jika server Samba Anda mengizinkan ekstensi CIFS Unix, Anda mungkin perlu menambahkan nounix ke jawaban micke: ,dir_mode=0700,file_mode=0700,nounix

Ini karena dengan ekstensi CIFS Unix mode tidak dapat diganti oleh klien. Meskipun tidak terlalu jelas, ada referensi tentang file halaman manual mount.cifs pepatah:

   If the uid's and gid's being used do not match on the client and
   server, the forceuid and forcegid options may be helpful. Note however,
   that there is no corresponding option to override the mode. Permissions
   assigned to a file when forceuid or forcegid are in effect may not
   reflect the the real permissions.
