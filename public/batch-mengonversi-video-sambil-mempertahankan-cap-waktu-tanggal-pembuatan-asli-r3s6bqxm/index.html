(Batch) mengonversi video sambil mempertahankan cap waktu / tanggal pembuatan asli

Saya memerlukan cara mudah untuk mengonversi video ke h264 sambil tetap mempertahankan tanggal pembuatan asli.

Saat ini saya melakukannya secara manual (menyandikan, lalu mengedit tanggal), tetapi itu tugas yang sangat membosankan untuk dilakukan ketika saya memiliki 10+ video.

Apakah ada cara untuk melakukannya secara otomatis?

Program konversi dengan opsi "pertahankan tanggal file asli", batch, skrip: saya terbuka untuk solusi apa pun di OS apa pun.

PENGUJIAN Salin direktori video Anda ke direktori baru, lalu ikuti petunjuk di bawah ini.

Skrip PowerShell

PowerShell hadir dengan Windows 7, tidak perlu mengunduh. (Mulai-> Semua Program-> Aksesori-> Windows PowerShell).

Mulai PowerShell sebagai administrator.

Anda harus melakukan hal berikut untuk pertama kalinya sebelum Anda dapat menjalankan skrip apa pun (Anda hanya perlu melakukannya sekali)

Set-ExecutionPolicy RemoteSigned

Jawab "Y".

Letakkan 2 skrip berikut di direktori video Anda. Buka PowerShell dan cd ke direktori video Anda. .ps1 adalah ekstensi skrip PowerShell.

SaveTime.ps1 (sebelum konversi video)

Simpan skrip berikut sebagai SaveTime.ps1 di direktori video Anda. Jalankan skrip ini PowerShell sebelum konversi video Anda.

Skrip akan dibuat OldTime-record.ps1 di direktori yang sama.

Script TIDAK AKAN MENYELESAIKAN OldTime-record.ps1 . Ini hanya akan ditambahkan ke bagian akhir jika file sudah ada.

Jadi tidak perlu khawatir jika Anda menjalankannya secara tidak sengaja dan kehilangan stempel waktu asli Anda. Cukup gunakan notepad untuk menghapus baris ekstra. Saya menambahkan contoh situasi ini di bagian akhir.

# SaveTime.ps1 - Start 
$file = get-item *

write-output "`$file = New-Object string`[`] $($file.count)" >> OldTime-record.ps1
write-output "`$time = New-Object string`[`] $($file.count)" >> OldTime-record.ps1

$time = New-Object string[] $file.count

for ($i = 0; $i -lt $file.count; $i++) {
    write-output "`$file`[$i`]=`'$($file[$i].fullname)`'" >> OldTime-record.ps1
    write-output "`$time`[$i`]=`'$($file[$i].CreationTimeUTC.tostring('o'))`'" >> OldTime-record.ps1
}
# SaveTime.ps1 - End

OldTime.ps1 (setelah konversi video)

Simpan skrip berikut sebagai OldTime.ps1 di direktori video Anda. Jalankan skrip ini PowerShell setelah konversi video. Skrip ini akan terbaca OldTime-record.ps1 dan ubah waktu pembuatan file yang sesuai.

# OldTime.ps1 - Start
. .\OldTime-record.ps1

for($i = 0; $i -lt $file.count; $i++) {
    write-output "$($file[$i])"
    write-output "$($time[$i])"
    Set-ItemProperty -Path $($file[$i]) -Name CreationTimeUTC -Value $($time[$i])
}
# OldTime.ps1 - End

Solusi lain untuk Windows yang baru saya temukan untuk mereka yang tidak ingin berurusan dengan banyak skrip:

Konversi dan Pertahankan Tanggal

Setelah Anda mengunduhnya, program itu sendiri terletak di MultipleAviConvDatePreserve \ bin \ Release \ MultipleAviConvDatePreserve.exe

Dan Anda dapat menggunakannya dengan konverter apa pun yang memungkinkan baris perintah (seperti rem tangan). Bukan program termudah untuk digunakan, tetapi lebih mudah daripada solusi yang diposting.

Karena saya mengalami kesulitan untuk menjalankan program sendiri, saya pikir beberapa arahan khusus tentang cara menggunakan program mungkin bisa membantu.

Buka Rem tangan (atau beberapa pembuat enkode video yang diaktifkan dengan baris perintah) dan menyiapkan file untuk disandikan seperti biasa (dengan semua preferensi yang Anda inginkan) dan kemudian menambahkannya ke antrean. Kemudian buka antrian dan klik tombol "Queue" dan "Generate Batch Script" seperti yang ditunjukkan pada tangkapan layar di bawah ini:
Kemudian buka skrip batch di notepad dan salin isinya (kita akan menggunakannya nanti). Selanjutnya buka MultipleAviConvDatePreserve.exe Dan klik tombol "Set SCRIPT"
Kemudian tempel konten skrip batch yang dihasilkan oleh Handbrake ke dalam kotak teks skrip tetapi ganti sumber dan tujuan di skrip ke% source dan% dest:
Saya tidak bisa menjalankan RunAsDate.exe, tetapi Anda dapat mencobanya - itu seharusnya membuatnya sehingga tanggal sistem tidak perlu diubah. Setelah ini, klik OK dan kembali ke jendela utama. Kemudian klik "Pilih DIR" dan pilih direktori dengan file video yang ingin Anda ubah dengan tetap mempertahankan tanggal pembuatan asli. Kemudian masukkan ekstensi file EXACT (case-sensitive) untuk sumber di bawah tombol radio "source Extension" setelah itu Anda akan melihat beberapa file muncul dalam daftar (jika Anda tidak melihat file apa pun, maka ekstensinya salah). Kemudian pilih file yang ingin Anda konversi (pilih semuanya dengan mengklik yang pertama dan kemudian klik-shift file terakhir). Lalu tekan "GO!" dan semuanya harus cukup jelas dari sana.

Di Windows dengan Powershell diaktifkan, Anda dapat menggunakan skrip .BAT ini untuk mengonversi secara rekursif semua file .AVI dan .MP4 di pohon folder saat ini. Dengan asumsi Anda melakukan encoding dengan ffmpeg dari FF Prompt dengan parameter yang sama; tetap merasa bebas untuk mengedit perintah pengkodean di sebelah baris terakhir. Perhatikan penggunaan variabel% 1 dan% 2 untuk menampung nama setiap asli (mis. "Name.ext") dan file yang dikonversi (mis. "Name_.ext").

FOR /r %%I in (*.avi, *.mp4) DO CALL :loopbody "%%~fI" "%%~dpnI_%%~xI" "%%~tI"
GOTO :EOF

:loopbody
   ffmpeg -i %1 -c:v libx264 -crf 18 -maxrate 4000k -bufsize 4000k -c:a libvo_aacenc -q:a 100 -map_metadata 0 -preset veryslow -tune film -movflags +faststart %2
   powershell Set-ItemProperty -LiteralPath '%2' -Name CreationTimeUtc -Value ('%3' -as [Datetime])

Anda mengatakan "tanggal pembuatan". Jika "tanggal modifikasi" cukup baik untuk Anda, baca terus:

Anda tidak mengatakan OS apa yang Anda gunakan. Jika itu * nix, gunakan perintah touch:

touch –r old_file new_file ...

menetapkan tanggal / waktu modifikasi new_file menjadi sama dengan old_file.

Ini tersedia di beberapa paket emulasi Unix, seperti Subsistem untuk Aplikasi berbasis UNIX (SUA). ISTR ini adalah download gratis dari Microsoft.

Saya akhirnya menemukan cara yang cukup sederhana! Diuji pada Windows XP, mungkin akan bekerja pada sistem yang lebih tinggi.

  1. Konversikan file di salah satu pengonversi Batch favorit Anda. Saya dulu Beralih Perangkat Lunak Pengonversi File Audio . Saya tidak mengerti mengapa semua alat ini tidak memiliki ekstensi pertahankan cap waktu feauture! Itu sangat mendasar. Dan kadang-kadang mereka bahkan dengan bodohnya berdebat mengapa opsi itu tidak diperlukan (lihat 1 dan 2 )
  2. Unduh utilitas touch untuk Windows. Dalam kasus saya, saya memilikinya di dalam RTools dan QGis, tetapi Anda juga dapat mengunduhnya di mis. itu Utilitas GNU untuk Win32 .

  3. Jalankan perintah cmd di direktori, di mana file video asli (*.MOV dalam kasus saya) dan file keluaran yang dihasilkan (dalam kasus saya itu adalah file *.wav). Masukkan perintah berikut (ubah ekstensi dengan benar):

    for %f in (*.wav) do (touch %f -r %~nf.MOV)

    berikut adalah hasilnya:

HEUREKA! Semua file keluaran sekarang memiliki stempel waktu yang sesuai dengan file asli!
