Temukan | grep tidak berfungsi di Cygwin

Saya telah menginstal cygwin dan ketika saya mencari kode sumber dengan perintah ini tidak ada yang muncul, bahkan saya memiliki string itu dalam sebuah file

$ echo foo > bar
$ find . -name '*' | grep foo

Tidak ada yang muncul. Perintah ini berfungsi dengan baik di GNU + Linux.

$ grep foo bar

Seperti yang disebutkan, find menghasilkan daftar file yang ditemukan ke stdout, grep biasanya mengharapkan untuk mencari melalui stdout jika dipanggil dengan cara ini.

Bisa juga menemukan pipa ke xargs, dan itu akan " membangun dan menjalankan baris perintah dari input standar ", seperti dalam

$ find . | xargs grep foo

Jika Anda memiliki nama file yang gila, dengan baris baru dan yang lainnya, maka ini akan lebih baik:

$ find . -type f -print0 | xargs -0 grep foo

dan -type f hanya akan menemukan file biasa, jadi tidak ada upaya untuk melakukan grep melalui direktori . atau .. atau direktori atau file "lucu".

Ketika saya mencari kode sumber dengan perintah ini tidak ada yang muncul

$ echo foo > bar
$ find . -name '*' | grep foo

find mencari nama file yang memenuhi kriteria yang diinginkan: Nama, Ukuran, Jenis File dan mengembalikan daftar nama file yang cocok. Itu tidak mengembalikan isi dari file yang cocok.

Ketika daftar nama file itu diteruskan ke grep menggunakan operator pipa grep akan melihat string bar (nama file) dan tidak foo (isi file bar).

grep oleh karena itu tidak menemukan kecocokan sehingga tidak ada output.

catatan:

  • grep dapat mencari file secara langsung. Anda tidak perlu menggunakan find. pipa |, atau $ echo foo > bar $ find . -name '*' | grep foo seperti yang disarankan dalam jawaban lain.

Perintah berikut berfungsi di Cygwin:

grep --exclude-dir=* "foo" ./*

Contoh:

DavidPostill@Hal /f/test
$ cat bar
foo

DavidPostill@Hal /f/test
$ grep --exclude-dir=* "foo" ./*
./bar:foo

Bacaan lebih lanjut

  • Indeks A-Z dari baris perintah Bash untuk Linux - Referensi yang sangat baik untuk semua hal yang terkait dengan baris perintah Bash.
  • Temukan - Cari nama file di hierarki folder yang memenuhi kriteria yang diinginkan: Nama, Ukuran, Jenis File.
  • grep - Cari file untuk teks tertentu.

Saya pikir Anda menginginkan ini.

find . -name '*' -type f -exec grep foo /dev/null {} \;

Anda hanya ingin grep file, bukan direktori atau file khusus. / Dev / null membuat grep menampilkan nama file. Anda mungkin ingin menghilangkan '-name' * ', tetapi jika Anda melakukannya, maka file titik akan di-grep.

Anda juga bisa lari

grep -r foo .
