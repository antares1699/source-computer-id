Mengapa perintah ls memerlukan tanda hubung sebelum parameternya, tetapi untuk perintah tar, tanda hubung bersifat opsional?

Saya baru mengenal Ubuntu dan saya berusaha keras untuk memahami baris perintah.

Ketika saya ingin melihat isi file tar yang saya gunakan:

tar tf tarfile.tar

Tetapi ketika saya ingin melihat daftar semua file jika saya menggunakan:

adalah nama file

Ia mengatakan:

ls: tidak dapat mengakses: Tidak ada file atau direktori seperti itu

Mengapa demikian?

Mengapa tanda minus "-" diperlukan untuk satu perintah dan bukan untuk perintah lainnya?

Perintah tar adalah salah satu perintah tertua di * nix. Itu dibuat sejak lama untuk melakukan arsip rekaman di mana data disimpan di tape drive.

Ketika utilitas ditulis, standar saat ini untuk memberikan tanda hubung untuk menunjukkan parameter belum ditemukan.

Ini opsional untuk tar, tetapi wajib untuk hampir semua perintah lainnya.
