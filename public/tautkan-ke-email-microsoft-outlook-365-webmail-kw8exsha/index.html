Tautkan ke email Microsoft Outlook 365 Webmail

Saya menggunakan email Microsoft di tempat kerja. Saya menggunakan antarmuka webmail. (Saya rasa ini disebut webmail Outlook 365). Saya perhatikan bahwa ketika Anda membaca email, URL unik muncul di bagian atas layar. Yang ingin saya lakukan adalah menyalin dan menempelkan URL tersebut ke perangkat lunak organisasi saya (saya menggunakan "Checkvist" yang merupakan outliner online yang secara samar-samar mirip dengan Mode Org). Ini agar saya dapat menautkan tugas tertentu ke email tertentu. Namun, ketika saya menyalin URL dan menempelkannya ke jendela baru, tidak terjadi apa-apa (ini membawa saya ke aplikasi web, tetapi tidak membuka email yang relevan). Adakah yang tahu cara membuka email yang relevan menggunakan URL?

Solusi 1 - Buat URL menggunakan ID dari URL email

Pertama agar ini bekerja, Anda perlu menonaktifkan mode percakapan di OWA, Anda dapat melakukannya dengan mengklik roda gigi di kanan atas saat berada di kotak surat Anda lalu di bawah "Tampilan Percakapan", setel ini ke nonaktif. (Ini agar URL ketika Anda memilih email akan memberi Anda ID pesan / item dan bukan ID percakapan.)

Kemudian pilih email yang ingin Anda buat tautannya, URL akan terlihat seperti di bawah ini tetapi dengan [MESSAGE_ID] menunjukkan ID pesan lengkap dalam format yang dikodekan URL.

https://outlook.office.com/mail/inbox/id/[MESSAGE_ID]

Salin [MESSAGE_ID] lengkap dan masukkan seperti yang ditunjukkan pada URL berikut:

https://outlook.office.com/owa/?ItemID=[MESSAGE_ID]&viewmodel=ReadMessageItem&path=&exvsurl=1

Saat membuka tautan ini, Anda akan langsung dibawa ke email.

Silakan lihat skrip PowerShell di bawah ini yang akan membuka formulir, Anda memasukkan URL yang disalin dari browser ke dalam kotak teks pertama dan klik "Ubah", ini kemudian akan mengembalikan URL yang akan mengarah ke email di O365 dan menghapus bidang pertama sehingga cepat untuk mengubah beberapa baris, juga jendela harus tetap paling atas. Ini masih memerlukan mode konversi untuk disetel ke nonaktif seperti yang disebutkan di paragraf pertama. Saya ingin menambahkan bahwa ini telah diperbaiki dengan sangat cepat menggunakan editor PoshGUI dan jangkar dll belum diatur sehingga memperluas jendela mungkin tidak memiliki efek yang diinginkan.

Function Convert-URL{
Param(
    [String]$O365_URL
)
    $inputURL = $O365_URL
    $returnURL = "https://outlook.office.com/owa/?ItemID=" + ($inputURL | Split-Path -Leaf) + "&viewmodel=ReadMessageItem&path=&exvsurl=1"
    $returnURL
}

Solusi 2 - API Grafik Office 365

Properti "Tautan Web" dikembalikan dari permintaan API "Dapatkan Pesan".

Silakan lihat sumber di bawah ini yang menunjukkan properti ini:

Di bawah tanggapan, Anda akan melihat properti "tautan web", URL dibuat sama dengan solusi di atas.

https://docs.microsoft.com/en-us/graph/api/message-get?view=graph-rest-1.0&tabs=http

Tautan berikut adalah ke penjelajah grafik, di mana Anda dapat masuk dan berinteraksi dengan API grafik untuk melihat contoh yang berfungsi, setelah masuk klik "Dapatkan Surat Saya" di panel kiri ". Ini akan mengembalikan tanggapan API di kanan bawah semua email Anda dan Anda akan mendapatkan tautan web kembali untuk setiap email di jendela ini.

https://developer.microsoft.com/en-us/graph/graph-explorer

Namun sementara cara di atas tidak terlalu praktis untuk digunakan secara manual, di sinilah saya akan mulai membuat skrip / program untuk mengambil URL.

Anda bisa mendapatkan URL email dengan:

  1. Klik kanan email dan / atau utas dan "Buat Tugas"
  2. Beralih ke To-Do / Tasks (kiri bawah menu antarmuka OWA, ikon tanda centang)
  3. Klik tugasnya
  4. Panel info sisi kanan terbuka
  5. Salin tautan dari teks "Buka di Outlook"

Ini adalah tautan ke pesan dan / atau percakapan. Saya belum menguji secara khusus tetapi setidaknya akan membawa Anda ke pesan dari utas yang Anda cari.

Ini sedikit merepotkan untuk penggunaan tingkat tinggi tetapi integrasi biasa dengan program tugas akan baik-baik saja.

Ini membuka pesan di tab baru, yang tidak ideal; tetapi sekali lagi, ini mungkin lebih untuk memori / tindak lanjut daripada untuk barang-barang GTD cepat-api.
