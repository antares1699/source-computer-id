Bingung tentang pos pemeriksaan Hyper-V

Saya menggunakan Hyper-V di windows 8.1 untuk mengatur lingkungan pengujian.

Pohon pos pemeriksaan saya terlihat seperti ini

Base
  +Test1
    >Now
  +Test2
  +Test3

Karena ini adalah lingkungan pengujian, saya biasanya tidak ingin menyimpan perubahan saat berpindah dari satu lingkungan ke lingkungan lain. yaitu saya hanya Apply pos pemeriksaan tempat saya ingin pindah.

Kebingungan muncul saat saya ingin menyimpan perubahan di satu lingkungan. Katakanlah pohon saya terlihat seperti ini:

Base
  +Test1
    >Now
  +Test2
    +Changes
  +Test3

Pertanyaan saya adalah:

  • Bagaimana cara menggabungkan pos pemeriksaan Changes menjadi Test2? Apakah saya hanya Delete saja?
  • Katakanlah saya tidak membutuhkan lingkungan Test3 lagi. Bagaimana cara menghapusnya? Akan Delete menggabungkannya dengan Base

Saat ini saya menggabungkan perubahan seperti ini tetapi tampaknya terlalu rumit:

  1. Apply pos pemeriksaan Changes

    Base
  +Test1
  +Test2
    +Changes
      >Now
  +Test3

  2. Delete Checkpoint Subtree pada Test2

    Base
  >Now
  +Test1
  +Test3

  3. Buat pos pemeriksaan baru dan ganti namanya Test2

    Base
  +Test2
    >Now
  +Test1
  +Test3

Saya selalu membuat dan Apply pos pemeriksaan saat VM dimatikan. Delete dan Apply mengacu pada pilihan menu konteks.

saya menemukan ini artikel dan sekarang saya memahaminya.

Poin penting untuk pemahaman saya adalah bahwa pos pemeriksaan selalu digabungkan hilir (induk digabung menjadi anak).

Jika tidak ada pos pemeriksaan di hilir (Now dianggap sebagai pos pemeriksaan dalam konteks ini), pos pemeriksaan hanya dihapus tanpa penggabungan.

Yang membuat saya bingung adalah bahwa disk diferensial, dan undo-disk di Virtual PC, bergabung ke hulu (anak digabung menjadi induk).

Jadi jawaban atas pertanyaan saya di atas adalah:

  • Bagaimana cara menggabungkan checkpoint Perubahan ke Test2?

Tidak mungkin karena Anda tidak dapat menggabungkan upstream!
Yang mungkin adalah menggabungkan Test2 menjadi Changes dengan Deleting Test2 !.

Base
  +Test1
    >Now
  +Changes
  +Test3
  • Katakanlah saya tidak membutuhkan lingkungan Test3 lagi. Bagaimana cara menghapusnya?

Test3 tidak memiliki pos pemeriksaan hilir sehingga hanya bisa menjadi Deleted Jika memiliki pos pemeriksaan hilir yang juga harus dihapus, gunakan Delete Chekpoint subtree.
