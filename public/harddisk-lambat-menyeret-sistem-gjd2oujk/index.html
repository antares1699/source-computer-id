Harddisk Lambat Menyeret Sistem

Saya punya HP Pro 3000 SFF menjalankan Window 8.1 Update 1 & Ubuntu 14.04 LTS. Ini menggunakan Chipset Intel G45s , Sebuah Hitachi HDS721050CLA362 500 GB harddisk, sebuah Prosesor Intel Core 2 Duo E8400 , dan memiliki 4 GB (sekarang 8 GB) RAM terpasang. [Spesifikasi mesin lainnya di kolom kiri di sini , HDD info patokan di sini .]

Saya perhatikan bahwa setiap kali mesin saya memiliki aktivitas disk, seluruh sistem akan mengalami draaaaag. Saya akan membuka Task Manager selama proses ini, dan saya melihat bahwa CPU saya biasanya menggunakan sekitar 20% - 30% penggunaan, dengan RAM biasanya sekitar 30% - 40% penggunaan, tetapi aktivitas disk membanting ke langit-langit sampai sistem menjadi lebih responsif. Setelah aktivitas disk mereda, kinerjanya bagus lagi.

Saya membeli komputer ini digunakan (mohon tidak ada komentar tentang ini: D), dan menerima mesin dengan HDD kosong. Saya langsung menginstal Windows 8.1, dan saya telah memperhatikan masalah ini sejak hari pertama.

Jadi, ini tampaknya bukan masalah prosesor atau RAM, karena tidak memerlukan aplikasi berat untuk menghasilkan aktivitas disk yang tampaknya ekstrem.

Saya ingin membeli HDD baru, karena saya hanya memiliki pengalaman biasa-biasa saja dengan SSD. Menurut pengalaman Anda, mungkinkah hard drive yang lebih baru akan memperbaiki masalah aktivitas disk ini?

Terima kasih atas waktu Anda.

Sebelum Anda menyalahkan disk drive Anda, Anda harus melihat jumlah ram yang Anda miliki.

Dalam komputer yang sehat dengan banyak ram, sejumlah besar itu akan didedikasikan untuk cache disk (sehingga mencegah pembacaan ulang data). Demikian juga di komputer "lambat", disk akan digunakan untuk menukar memori ke disk atau perlu melakukan pembacaan ulang data secara signifikan untuk memuat aplikasi dari disk ke memori.

Beberapa hal yang dapat Anda coba adalah menutup dan membuka browser web Anda berulang kali. Atau tutup sakelar browser web Anda ke beberapa aplikasi lain, lalu kembali ke aplikasi asli.

Jika Anda menemukan bahwa Anda masih memuat dengan lambat, komputer Anda setidaknya sebagian terkena kekurangan ram. Secara umum lebih banyak ram bagus, dan sekarang 8GB adalah jumlah yang bagus (meskipun 4GB seharusnya OK).

Itu semua dikatakan, disk laptop 5400 RPM bisa sangat lambat. Grafik kinerja yang Anda anggap benar, dan tidak menunjukkan apa pun yang bermasalah. Jika Anda menggunakan SSD yang bagus, bahkan jika Anda perlu menukar ke disk sesekali, semuanya akan terasa dan terasa lebih cepat.

Hal pertama yang akan saya periksa adalah kesehatan Hard Drive. (CATATAN: jika Anda memiliki drive yang buruk mendefragmentasi sebelum memeriksa status SMART pada dasarnya dapat merusak data Anda). Pergi ke http://www.smartmontools.org/ dan unduh versi windows. Kemudian buka prompt CMD dan periksa status SMART.

C: \ Program Files \ smartmontools \ bin \

smartctl -a / dev / sda /

Jika Anda melihat status kesehatan yang buruk - Anda menemukan masalah Anda. Ganti drive.

Jika statusnya bagus:

Pindah ke pemecahan masalah lainnya. Karena Ubuntu dan Windows mengalami masalah - driver tidak salah. Periksa BIOS untuk mode AHCI - ya, Anda dapat mengubahnya setelah OS diinstal. Jika drive dalam mode IDE, semuanya akan merangkak terlepas dari driver yang diinstal.

Anda akan menemukan pengaturan di bawah Advanced di menu pengaturan komputer BIOS.

Lihat Victoria HDD untuk Memeriksa Kesehatan HDD, ini adalah alat Tingkat Rendah, jadi berhati-hatilah saat Anda menggunakannya

Hal lain yang sebenarnya saya miliki Pembaruan Windows 8.1 1 Apakah itu Superfetch layanan terlalu banyak membunuh HDD saya hingga 100% hampir sepanjang waktu

Anda dapat mematikan layanan itu dengan Menjalankan perintah berikut di Command prompt, Anda dapat membuka prompt perintah dengan menekan Windows + X dan kemudian Mengklik Command Prompt (Admin) , berjalan tanpa izin administrator tidak akan berfungsi:

net stop superfetch

Layanan ini dimulai setiap saat saat boot, jadi jika Anda menyelesaikan masalah, Anda mungkin ingin menonaktifkannya secara permanen, buka:

services.msc

Dan cari Superfetch > Klik Kanan> Properti > dan setel ke Dengan disabilitas

Periksa aktivitas intensif disk latar belakang Anda (berbagi / penyajian file, pengindeksan disk - terutama pengindeksan koleksi gambar) untuk menghindari beban berlebihan pada subsistem disk; mungkin, lakukan pemindaian melalui malware untuk melihat apakah itu adalah hama kecil yang berjalan dan menghambat kinerja disk Anda.

Periksa statistik SMART disk Anda untuk memeriksa apakah sudah sehat - apa pun kinerjanya, sesekali menjaga data Anda merupakan praktik yang baik.

Gunakan SSD untuk sistem, perangkat lunak, swap (hanya jika Anda berencana untuk menjalankan aplikasi intensif memori, jika tidak, RAM mesin modern lebih dari cukup untuk menghindari penggunaan swap sama sekali), dan untuk data yang Anda akses lebih sering - yaitu direktori kerja - tetapi gunakan HDD untuk sebagian besar data karena SSD belum terlalu kompetitif dalam hal harga per GB.
