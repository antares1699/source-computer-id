Tooltip aplikasi GTK tidak dapat dibaca di dalam KDE

Sistem saya adalah openSUSE 42.1 dengan Plasma 5.

Saya juga menginstal beberapa aplikasi GTK, dan saya mengalami perilaku yang agak mengganggu darinya.

Yaitu, tooltips dari aplikasi ini hampir tidak dapat dibaca, menjadi teks putih di atas latar belakang abu-abu terang, seperti yang Anda lihat pada gambar.

Saya tidak dapat menemukan di mana dan bagaimana mengubah pengaturan ini; hanya aplikasi non-KDE yang terpengaruh, dan tidak semuanya: Gimp, Inkscape, Filezilla adalah di antaranya, sementara Firefox, Libreoffice, Handbrake menampilkan keterangan alat dengan cara yang dapat dibaca.

Masalah ini disebabkan oleh penggantian sebagian warna GTK melalui gaya KDE.

Buka System Settings > Colors, hapus pilihan Apply colors to non-Qt applications, mulai ulang aplikasi GTK (mis. Gimp). Ini mematahkan perasaan bahwa semua aplikasi berbagi gaya yang sama tetapi tooltips akan kembali dapat dibaca di aplikasi Qt dan GTK.

Saya memiliki masalah yang sama di edisi KDE Linuc Mint. Untungnya saya menemukan jawaban di wiki ArchLinux:

Untuk mengubah warna dalam aplikasi GTK2, cari bagian untuk tooltips di file .gtkrc-2.0 dan mengubahnya. Untuk aplikasi GTK3 dua file perlu diubah, gtk.css dan settings.ini.

Sumber: https://wiki.archlinux.org/index.php/KDE
