Mengakses file "tanpa nama"

Saya memiliki situasi ini di direktori tertentu:

$ ls -al
...
drwxr-xr-x   2 me users         8192 Juni 28 15:27 tmp
-rw-r--r--   1 me users          439 Jan.  3  2013
...

Bagaimana saya bisa mengakses entri kedua, yang tampaknya tidak memiliki nama file?

Contoh yang dijalankan untuk membuat file bernama "non breaking space" dan mengganti namanya:

$ ls -1 # Each file on a separate row
testfile
$ touch   # I'm actually writing the character u00a0 here
$ ls -1
testfile
 
$ ls -i # Print inodes
2031842 testfile
2023653  
$ find . -maxdepth 1 -inum 2023653 -exec mv {} hithere \;
$ ls -i1
2023653 hithere
2031842 testfile

exec dalam bentuknya hanya masuk akal jika hanya ada satu kecocokan, tetapi nomor inode harus unik pada partisi itu kecuali ada tautan keras ke file yang sama. Anda dapat mengujinya dengan menjalankan find tanpa argumen terlebih dahulu untuk melihat apakah Anda mendapatkan beberapa kecocokan. Jika demikian, Anda dapat mis. perbaiki pola find agar hanya cocok dengan file yang Anda inginkan.

Karena penggantian nama tidak bergantung pada karakter yang tepat dari nama file, seharusnya tidak ada masalah dengan memasukkan poin kode yang aneh.

Saya akan mencoba "ls -1a> / tmp / list" dan hexdump untuk melihat apakah ada sesuatu yang muncul. Mungkin saja nama file adalah urutan spasi dan tab. Jika hexdump menunjukkan nama file sebagai spasi (katakanlah 1 spasi), maka lakukan "mv \ somename"

ls -1

ls -1 > /tmp/list
cat /tmp/list

hexdump /tmp/list
          00 01 02 03 04 05 06 07 - 08 09 0A 0B 0C 0D 0E 0F  0123456789ABCDEF

00000000  20 0A     {------                                          .
mv \  new        {------
ls -1
new
