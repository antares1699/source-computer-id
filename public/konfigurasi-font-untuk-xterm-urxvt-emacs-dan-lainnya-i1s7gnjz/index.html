Konfigurasi font untuk xterm, urxvt, emacs dan lainnya

Saya baru-baru ini mulai menggunakan xmonad sebagai pengelola jendela saya. Saya juga memutuskan untuk menggunakan urxvt sebagai terminal default, karena seharusnya ringan.

Sebagian besar aplikasi yang memberikan opsi konfigurasi font memungkinkan saya untuk memilih jenis font (seperti Arial, Consolas, FreeMono, Dejavu Sans, dll.), Dan untuk menyetel ukuran dalam piksel (11px, 12px, atau apa pun). Ini mudah dimengerti.

Sekarang ketika mencoba mengatur font untuk aplikasi seperti xterm, urxvt dan emacs, saya harus menemukan string yang agak aneh yang terlihat seperti: "-*-Fixed-Bold-R-Normal-*-13-*-*-*-*-*-*-*" atau "-unknown-DejaVu Sans Mono-normal-normal-normal-*-14-*-*-*-m-0-iso10646-1"

Saya telah menggunakan program xfontsel untuk menghasilkan ini untuk saya, menggunakan trial and error. Tapi saya tidak begitu mengerti apa artinya, atau mengapa itu perlu.

Selain itu, saya juga pernah menggunakan barang-barang seperti ini: URxvt.font: 9x15bold.

Pertanyaan saya:

  • Apa string panjang yang aneh untuk mengkonfigurasi font?
  • Apa artinya 9x15bold? Bagaimana semuanya bisa bersatu?
  • Mengapa tidak tetap menggunakan cara konvensional untuk mengatur font seperti yang digunakan sebagian besar aplikasi lain?
  • Mengapa font xterm dan urxvt tidak terlihat sebagus, katakanlah, font gnome-terminal? Apakah ada cara untuk mencapai ini?

  1. Itu adalah X deskripsi font logis , digunakan sebagian besar oleh program pure-Xlib.

  2. "9x15bold" adalah alias praktis untuk font bitmap yang disertakan dengan X.org. Ini tidak berarti sesuatu yang spesifik, hanya ada tabel "fonts.alias" yang menerjemahkannya menjadi XLFD:

    ! $Xorg: fonts.alias,v 1.3 2000/08/21 16:42:31 coskrey Exp $
fixed        -misc-fixed-medium-r-semicondensed--13-120-75-75-c-60-iso8859-1
variable     -*-helvetica-bold-r-normal-*-*-120-*-*-*-*-iso8859-1
5x7          -misc-fixed-medium-r-normal--7-70-75-75-c-50-iso8859-1
...
8x16         -sony-fixed-medium-r-normal--16-120-100-100-c-80-iso8859-1
9x15         -misc-fixed-medium-r-normal--15-140-75-75-c-90-iso8859-1
9x15bold     -misc-fixed-bold-r-normal--15-140-75-75-c-90-iso8859-1
10x20        -misc-fixed-medium-r-normal--20-200-75-75-c-100-iso8859-1
...

  3. Xresources aku s cara konvensional. Atau setidaknya, dulu. Misalnya, seseorang dapat mengatur font untuk satu program dengan URxvt.font, atau untuk semuanya dengan *font, dan ini akan mempengaruhi semua program - termasuk yang berjalan di server jarak jauh. (X11 adalah protokol jaringan.)

    Kebanyakan program yang baru ditulis menggunakan sistem seperti dconf di GNOME, yang tidak masalah, karena dibuat khusus untuk GNOME. Xterm, bagaimanapun, adalah desktop-lingkungan-agnostik, karena - yang paling penting - itu dibuat jauh sebelum KDE dan GNOME dan Qt dan GTK; itu adalah bagian dari rangkaian perangkat lunak inti X11. Jadi ia menempel pada Xresources karena sederhana dan standar, tanpa ketergantungan pada pustaka atau layanan pihak ketiga. (Tentu saja ada kekurangannya, seperti kurangnya konfigurasi ulang waktu nyata.)

  4. Xterm adalah program Xlib murni dan menggunakan rendering font yang disediakan oleh X11, sedangkan Terminal GNOME, yang ditulis dengan GTK, menggunakan Freetype (melalui Pango).

    Namun, versi terbaru Xterm dan URxvt menyertakan dukungan untuk rendering font Freetype dasar melalui Xft (memiliki fitur yang lebih sedikit daripada Pango, misalnya tidak memiliki penggantian font, tetapi sering kali "cukup baik"). Anda dapat menggunakannya dengan menambahkan sumber daya berikut:

    URxvt.font: xft:DejaVu Sans Mono:size=9

XTerm*faceName: DejaVu Sans Mono
XTerm*faceSize: 9

    Ukuran font dalam poin (bukan piksel).
