Kemungkinan efek samping negatif dari peningkatan ruang alamat virtual untuk aplikasi?

Berikut adalah konfigurasi sistem saya:

  • Pratinjau Pengembang Windows 8 (32-bit)
  • RAM 2 GB
  • Nvidia GeForce 8400 M (dengan RAM 256 MB)

Untuk menjalankan aplikasi intensif memori, saya perlu menambah ruang alamat virtual dari 2 GB menjadi 3 GB. (Perhatikan bahwa ini tidak ada hubungannya dengan memori virtual - konsep yang sama sekali berbeda.)

Apa saja kemungkinan efek samping negatif dari peningkatan ruang alamat virtual sebesar 1 GB? Dugaan saya adalah bahwa Windows akan menyimpan lebih sedikit pembacaan disk, menggunakan lebih banyak file halaman, dll. Apakah ada hal lain yang lebih serius yang harus saya khawatirkan?

Dalam versi 32-bit, ruang alamat virtual Anda adalah 4GB; 2GB untuk ruang Pengguna, dan 2GB untuk Sistem. Saat Anda menggunakan 3GB untuk ruang Pengguna, Anda hanya menyisakan 1GB untuk ruang Sistem. Ini berarti menghabiskan ruang alamat Sistem dua kali lebih mudah, jadi Anda mungkin mengalami masalah driver tingkat kernel (hal-hal seperti pengontrol disk dan I / O lainnya).

Saya menemukan artikel bagus ini di TechNet yang menjelaskan beberapa jebakan: " Manajemen Memori - Demystifying / 3GB "...

Berikut uraian yang berlaku:

Ingatlah bahwa kami hanya memiliki total ruang alamat 4GB untuk digunakan. Jika kami harus mengalokasikan tambahan 1GB dari ruang alamat ini ke ruang mode pengguna, maka ruang Sistem dipotong menjadi dua. Driver, Heap, Paged & NonPaged Memory semuanya hanya memiliki setengah sumber daya untuk digunakan sekarang. Namun, karena cara kerja pemetaan memori, memotong file ruang kernel menjadi dua melakukan lebih dari sekadar mengurangi alamat ruang. Banyak struktur dalam ruang memori virtual kernel dipotong jauh lebih dari 50%. Misalnya, kami menggunakan Windows Mesin Server 2003 Enterprise R2 dengan RAM 1 GB terpasang dan membandingkan beberapa nilai dengan dan tanpa sakelar / 3GB diaktifkan.

Bangun OS Default:

Sistem Gratis PTE: 251.980 (1.007.920 kb)

NonPaged Pool Max: 206.848 kb

Dengan / 3GB diaktifkan:

Sistem Gratis PTE: 34.884 (139.536 kb)

NonPaged Pool Max: 129.312 kb

Seperti yang Anda lihat, Free System PTE turun lebih dari 200.000. Tetap Ingatlah bahwa ini hanya server uji yang tidak dalam jenis apa pun beban. Mesin dengan beban sedang hingga berat dapat dengan mudah habis PTE gratis - artinya sistem tidak dapat lagi memetakan halaman sistem seperti ruang I / O, tumpukan kernel, dan daftar deskriptor memori. Di Selain itu, lihat NonPaged Pool setelah parameter / 3GB diaktifkan.
