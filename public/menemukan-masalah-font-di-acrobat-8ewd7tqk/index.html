Menemukan Masalah Font di Acrobat

Ok biar saya coba lagi, jadi maaf belum jelas. Kami membuat PDF kami melalui Quark, lalu mengirim untuk dicetak. Saya biasanya membuat garis besar pada file EPS saya sebelum saya memuat di Quark tetapi kali ini saya lupa. Kami melewati kesalahan font yang diberikan Quark kepada kami secara tidak sengaja dan menemukan PDF kami sangat terlambat dan membutuhkan banyak uang untuk memperbaikinya.

Kami mencoba menemukan cara untuk memeriksa PDF kami untuk masalah font sebelum kami mengirimkannya untuk dicetak, jika masalah ini terjadi lagi. Kami hanya ingin lebih yakin bahwa kami telah mencoba segalanya.

Apa yang saya lihat di Quark adalah seperti apa font itu seharusnya. Saat saya melihat PDF saya, teksnya tercampur. Ini dapat dibaca tetapi tidak terlihat seperti yang seharusnya dan semua spasi di dalam teks.

Bos saya memberi tahu saya tentang preflight di Quark dan Struktur Internal untuk font. Dia bertanya padaku apakah ini akan membantu dan apa arti semua istilah itu. (Di situlah pertanyaan pertama saya dimulai)

Gambar di sebelah kiri adalah EPS saya yang benar, gambar di sebelah kanan adalah dari PDF. Teks putih di kanan atas dan situs web di kiri bawah adalah yang kacau.

Saya menjalankan Mac 10.5.8, Quark 7.5 dan Acrobat 8.3.1.

Terima kasih, Jayme

tl; dr

Selain bola mata manusia, saya tahu tidak ada alat yang dapat memeriksa PDF dan menyimpulkan bahwa program yang digunakan untuk menghasilkan PDF telah menggantikan font.

Anda bisa berasumsi bahwa jika font Courier ada di PDF, ada yang tidak beres. Pemeriksaan kasar dan siap akan dilakukan

strings filename.pdf | grep Courier

Secara umum, untuk mencegah masalah semacam ini, saya akan melakukannya

0) Pastikan objek EPS sumber memiliki semua font yang disematkan.

Ini penting jika Mac yang digunakan untuk proyek Quark tidak memiliki font apa pun.

Saya biasanya membuat garis besar pada file EPS saya sebelum saya memuat di Quark tetapi kali ini saya lupa.

Mengonversi karakter menjadi garis besar (yaitu ke kurva dan data titik kontrol) adalah cara lain untuk menghapus persyaratan apa pun bagi konsumen / penerima EPS agar font yang digunakan sudah terpasang.

1) Buat Quark menanamkan font

Pengaturan Font

Saat Anda mengekspor tata letak dalam format PDF, Anda dapat memilih untuk referensi atau menanamkan (unduh) font yang digunakan dalam tata letak itu.

  • Menyematkan berarti font itu sendiri sudah termasuk dalam file PDF. Ini meningkatkan ukuran file PDF, tetapi memastikan bahwa file akan ditampilkan atau dikeluarkan dengan benar.

2) Lihat daftar font di Acrobat

dan periksa kembali apakah itu menunjukkannya semua seperti yang disematkan (Menu: File -> Properties, tab Fonts)

Saya khawatir tentang font ke-5 dalam daftar ini.

Memperbarui:

File zip Jayme menunjukkan kotak dialog Quark yang bertuliskan

"Beberapa gambar EPS / PDF dalam dokumen ini menggunakan font layar yang tidak tersedia di sistem Anda, termasuk Univers-Condensed dan Univers-CondensedBold "

Jelas dari gambar PDF akhir bahwa Quark telah menggantikan Courier untuk font yang hilang tetapi telah menerapkan pemosisian huruf dari EPS yang akan sesuai untuk Univers-Condensed.

Salah satu solusinya adalah membeli dan menginstal Univers-Condensed dan Univers-CondensedBold di Mac di mana proyek Quark sedang di-output ke PDF.

Solusi lain adalah kembali ke aplikasi yang menghasilkan EPS yang telah ditempatkan dalam proyek ini dan mereproduksi EPS itu tetapi memaksanya untuk menyematkan font (subset bekas) ke dalam EPS, kemudian mengimpor kembali EPS itu ke dalam proyek akhir sebelum memproduksi PDF siap cetak akhir.

Mencoba memahami struktur internal PDF untuk masalah seperti ini adalah tugas yang bodoh. Saya akan merekomendasikan sebagai gantinya Anda membiarkan orang yang menjawab pertanyaan itu mengusulkan solusi, daripada bertanya mengapa solusi itu Anda berpikir harus bekerja tidak bekerja.

Bagaimanapun, beberapa informasi latar belakang.

Di setiap PDF, ada dua jenis font: font yang disematkan , dan font yang direferensikan .

Font yang disematkan terkandung secara keseluruhan di dalam PDF itu sendiri. Program pembaca PDF apa pun yang kompatibel Sebaiknya dapat mencetak setiap karakter dari font yang disematkan. Jika tidak bisa, itu masalah tersendiri. Keuntungannya adalah Anda dapat menggunakan font khusus dan orang yang melihat / mencetak dokumen akan dapat membaca font tersebut terlepas dari apakah font tersebut diinstal di komputer mereka atau tidak.

Font yang direferensikan adalah font yang hanya disebut "menurut nama" dalam dokumen PDF, tetapi PDF bergantung pada sistem operasi untuk memiliki file font yang ada pada saat rendering / pencetakan, atau Anda mendapatkan "masalah font" seperti yang Anda katakan . Keuntungannya adalah ukuran file kecil.

Masalah yang mungkin Anda hadapi ada banyak, dan saya tidak tahu yang masalah yang Anda alami dari pertanyaan Anda, jadi saya akan menghitungnya bersama dengan kemungkinan penyelesaian:

  • Font Anda dapat disematkan dalam PDF, tetapi printer / renderer PDF Anda mungkin mengalami masalah saat mengurai (menggunakan) font tersebut. Ini biasanya merupakan bug khusus untuk perangkat lunak, atau mungkin tidak kompatibel dengan format file font yang disematkan. Anda biasanya dapat mengatasi masalah ini dengan memutakhirkan perangkat lunak pencetakan PDF Anda, dan Anda harus memverifikasi bahwa PDF dirender dengan benar pada penampil PDF kanonik seperti Adobe Reader (versi terbaru) di Windows, sebelum mengambil tindakan korektif.

  • Font Anda dapat direferensikan dalam PDF, tetapi printer PDF Anda tidak memiliki font tersebut. Anda dapat mengatasi masalah ini dengan (a) menginstal font yang sesuai di komputer / perangkat yang mencetak atau merender PDF, atau (b) mengalihkan perangkat lunak pembuat PDF Anda untuk menyematkan font di PDF daripada mereferensikannya .

Kamu bisa mendeteksi fonta mana yang direferensikan dan font mana yang disematkan dengan berbagai cara (bergantung pada platform Anda, dll) - lihat pertanyaan StackOverflow ini . Hal pertama yang dapat Anda lakukan adalah mengonfigurasi perangkat lunak yang menghasilkan PDF untuk selalu menyematkan semua font. Ini akan meningkatkan ukuran file; jika Anda dapat menangani ukuran file yang dihasilkan, itu Sebaiknya membuat pencetakan jauh lebih andal.

Anda juga dapat menentukan apakah setiap font yang direferensikan tersedia di sistem, tetapi itu bergantung pada platform, yang berarti bahwa solusi untuk melakukannya akan bergantung pada sistem operasi yang Anda jalankan dan jenis lingkungan perangkat lunak apa yang tersedia untuk Anda, mis. untuk pemrograman.

Secara keseluruhan, ini adalah pertanyaan yang sangat samar dan menurut saya pertanyaan ini tidak dapat dijawab dalam bentuknya saat ini karena semua kemungkinan variabel yang mungkin menyebabkan masalah ini. Anda sepertinya bertanya tentang internal PDF dengan cara yang sangat abstrak tanpa mengacu pada perangkat lunak atau platform tertentu. Saya telah memberikan jawaban ini dalam upaya untuk membantu, tetapi perlu diketahui bahwa pertanyaan ini dapat ditutup jika tidak dibuat lebih spesifik.
