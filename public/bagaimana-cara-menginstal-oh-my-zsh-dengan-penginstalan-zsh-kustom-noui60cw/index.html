Bagaimana cara menginstal Oh My Zsh dengan penginstalan Zsh kustom?

Jadi, saya baru saja berhasil menginstal zsh pada mesin yang saya memiliki akses SSH (tetapi yang terpenting, tidak ada akses root ke) sesuai saran sini . Saya kemudian melanjutkan untuk mencoba menginstal Oh My Zsh, dan disambut oleh pesan ini:

Zsh is not installed! Please install zsh first!

Yang tidak terlalu mengejutkan. Setelah memeriksa skrip instal, kode untuk memeriksa apakah zsh diinstal adalah:

CHECK_ZSH_INSTALLED=$(grep /zsh$ /etc/shells | wc -l)
if [ ! $CHECK_ZSH_INSTALLED -ge 1 ]; then
    printf "${YELLOW}Zsh is not installed!${NORMAL} Please install zsh first!\n"
    exit
fi
unset CHECK_ZSH_INSTALLED

Jadi, tanpa akses root, saya tidak bisa mengedit / etc / shells. Penting juga untuk dicatat bahwa ketika saya SSH ke sistem ini, saya tidak perlu masuk ke mesin fisik yang sama; sebaliknya, saya login ke salah satu dari beberapa mesin, di mana direktori home saya disalin. Saat ini saya memiliki kode di .profile saya yang menambahkan ~/bin ke $PATH saya dan meluncurkan zsh saat masuk.

Pada dasarnya, apakah ada cara lain untuk mengatasi ini? Saya berasumsi bahwa skrip instal dapat dimodifikasi untuk mengakomodasi zsh yang diinstal ke ~ / bin / zsh, tetapi hanya menghapus logika pemeriksaan secara membabi buta tentu saja tidak terlihat apa-apa jika tidak salah.

Logika pemeriksaan hanya ada untuk mencegah chsh gagal di akhir penginstalan. Jika biner zsh Anda tidak terdaftar pada /etc/shells, Anda tidak akan dapat menggunakan chsh untuk mengubah shell login Anda. Dalam hal ini menjalankan ~/bin/zsh dari salah satu file startup shell login Anda saat ini adalah satu-satunya cara untuk meluncurkan zsh secara otomatis. Karena ~/.profile biasanya hanya digunakan untuk sesi login, seharusnya tidak masalah. Anda mungkin ingin menjalankan zsh -l dari sana untuk mendapatkan sesi login alih-alih hanya sesi interaktif (ini berbeda di mana file konfigurasi dimuat).

Untuk menginstal oh-my-zsh , Anda harus menghapus beberapa kode dari install.sh: yaitu centang yang disebutkan dalam pertanyaan dan di mana ia mencoba untuk menjalankan chsh secara otomatis. Atau Anda cukup menjalankan dua perintah berikut:

git clone --depth=1 https://github.com/robbyrussell/oh-my-zsh.git $HOME/.oh-my-zsh
cp $HOME/.oh-my-zsh/templates/zshrc.zsh-template ~/.zshrc

Kecuali jika Anda ingin menginstal oh-my-zsh di suatu tempat selain ~/.oh-my-zsh tidak ada lagi yang dibutuhkan. Segala sesuatu yang lain di install.sh hanya:

  • perlindungan terhadap penimpaan yang sudah ada oh-my-zsh instalasi di tempat yang sama, yang akan dicegah oleh git clone
  • mencadangkan ~/.zshrc, yang mungkin tidak Anda miliki pada instalasi baru
  • mengatur shell login, yang toh tidak berfungsi dalam kasus Anda
  • dan banyak hasil berwarna.
