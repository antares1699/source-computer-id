Bagaimana cara mengaktifkan respons untuk menyiarkan ping di Windows?

Saya memiliki banyak server Windows di lab. Saya ingin menggunakan siaran ping untuk menemukan mesin. Saya mencoba menghindari server pusat untuk mempertahankan daftar mesin karena ini tampak seperti satu titik kegagalan bagi saya.

ping 10.0.0.255 di jaringan 255.255.255.0 tidak akan menerima respons apa pun.

Bagaimana cara mengaktifkan respons ke ping siaran?

Program Windows ping tidak dapat mengirim permintaan ping ke alamat broadcast. Tetapi Anda dapat menggunakan IcmpSendEcho API untuk mengirim permintaan ping ke jaringan yang terhubung. Tetapi dari pengamatan saya, ini tidak berhasil dengan:

  • stasiun Windows yang terhubung langsung - tampaknya stasiun Win tidak menanggapi permintaan ICMP siaran
  • melalui sakelar - tampaknya sakelar memblokir permintaan ICMP siaran

Jika rentang alamat IP tidak terlalu besar, dimungkinkan untuk menggunakan:

  • Pemindaian ARP
  • Pemindaian ICMP
  • Pemindaian SNMP - jauh lebih lambat, tetapi berfungsi

ini metode yang saya gunakan:

Pertama ... di baris perintah gunakan ini ...

UNTUK / L% i di (1,1,255) lakukan @ping -n 1 192.168.1.% I | cari "Balas"

Tentu saja, Anda perlu memberikan alamat 'asli' untuk 192.168.1.% I, itu hanya sebagai contoh.

Anda tidak perlu menyalurkannya (-> | temukan "Balas") ke perintah find tetapi dikatakan untuk menyaring sampah; Saya telah melakukannya dengan dua cara dan tidak ingat melihat banyak perbedaan ...
