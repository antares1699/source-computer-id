Snapshot teraman dari harddisk yang gagal?

Saya memiliki mesin tanpa kepala yang berhenti melakukan booting, jadi saya mengeluarkannya untuk diagnosis dan mendapat pesan bahwa salah satu hard drive akan gagal, jadi saya menarik semuanya dan saya harus melepaskan semuanya, sebelum mencari tahu yang saya butuhkan untuk menyingkirkan. Saya tidak yakin drive mana yang gagal, karena hanya tertulis "Harddisk 1" dan saya tidak tahu yang mana yang dimaksud.

Saya bertanya-tanya cara terbaik untuk melepaskan semuanya. Saya khawatir jika saya menyalin semuanya, saya bisa mendapatkan data yang rusak dan tidak menyadari beberapa file salah sampai drive benar-benar tidak berfungsi. Apa pilihan terbaik saya untuk menghapus semuanya dengan cara yang dapat saya pindahkan dengan aman ke penyimpanan baru?

Pertama, Salin Data Paling Penting dengan Cepat

Jika Anda memiliki hard drive lain di sistem Anda, segera salin file terpenting Anda ke drive itu. Setidaknya Anda akan memiliki duplikat data penting Anda. Jika Anda mendapatkan data yang rusak, itu masih lebih baik daripada menunggu sementara situasinya bisa semakin memburuk.

Kedua, Lakukan Pencadangan Gambar Penuh

Kemudian, saya akan mendapatkan sebanyak mungkin informasi dari drive secepat mungkin. Saya akan mencoba program pencitraan disk (seperti Acronis True Image ). Itu adalah cara tercepat yang saya tahu untuk mendapatkan sebanyak mungkin informasi dari hard drive secepat mungkin. Jika ada korupsi, Anda akan mendapatkannya juga ... tetapi setidaknya Anda memiliki sesuatu untuk dikerjakan.

Ketiga, Temukan dan Pulihkan Data yang Rusak

Jika cadangannya bagus, Anda dapat mengembalikan gambar ke hard drive baru. Jika Anda menemukan korupsi , maka saya akan mencoba program seperti SpinRite . Program itu dirancang untuk bekerja tanpa lelah pada hard drive Anda pada level terendah (bahkan di bawah level sistem operasi) dan benar-benar menggali data hard drive Anda dan mengambil setiap bit terakhir data yang akan diambil.

Saya memiliki pengalaman terbatas dengan skenario ini, tetapi pendekatan tipikal saya adalah mengeluarkan sebanyak mungkin dari drive secepat mungkin; biasanya saya meletakkannya di kotak linux dan menjalankan gambar dd penuh dari drive.

Setelah saya mendapatkan gambar, saya bayi drive yang sebenarnya; taruh di tas antistatis dan simpan offline sampai saya dapat menentukan apakah gambar itu lengkap / rusak / dll ... jika ada yang salah dengan gambar, saya dapat memutuskan apakah saya perlu mengembalikan drive online untuk pergi setelah spesifik data atau apakah akan mencoba gambar lain dengan pendekatan berbeda.
