Tingkatkan Kecepatan Firefox?

Menemukan cara untuk benar-benar mempercepat Firefox - Saya telah menerapkannya dan melihat peningkatan kecepatan yang serius. Adakah yang bisa berkomentar?

  1. Ketik alamat Firefox about: config
  2. Terima pemberitahuan Peringatan dan lanjutkan
  3. Di bagian atas halaman akan ada filter - cari, dan sesuaikan, pengaturan berikut:

network.http.pipelining: true
network.http.proxy.pipelining: true
network.http.max-koneksi: 50
network.http.max-koneksi-per-server: 40
network.http.max-persistent-koneksi-per-proxy: 20
network.http.max-persistent-koneksi-per-server: 10
network.http.pipelining.maxrequests: 20

  1. Kemudian klik kanan di mana saja di ruang kosong dan pilih New -> Integer
  2. Beri nama integer ini "nglayout.initialpaint.delay" dan setel nilainya menjadi 0.
  3. Klik kanan lagi di manapun dalam ruang kosong dan pilih New -> Boolean
  4. Beri nama boolean ini "config.trim_on_minimize" dan setel nilainya ke True.
  5. Sekarang di Baki Ikon, klik kanan pada Ikon Mozilla Firefox dan pilih Properti
  6. Di bawah "Target", Anda akan melihat baris "C: \ Program Files \ Mozilla Firefox \ firefox.exe"
  7. Ubah baris ini menjadi “C: \ Program Files \ Mozilla Firefox \ firefox.exe” / Prefetch: 1 ″
  8. Pastikan ada spasi yaitu \ firefox ”(spasi) / Prefetch: 1 ″

Kelemahan pipelining adalah memberikan tekanan tambahan pada server web tempat Anda meminta halaman. Hanya satu orang yang melakukan ini bukanlah masalah besar, tetapi banyak orang menggunakan penyesuaian ini. Banyak orang diarahkan ke situs itu, dan bahkan jika 25% dari mereka menggunakan pipelining, ketegangan server mulai meningkat hampir secara eksponensial.

Selain itu, angka dalam instruksi tersebut terlalu tinggi. Firefox membatasi jumlah permintaan HTTP secara bersamaan. Saya ingat pernah membaca bahwa maksimumnya sekitar 8 atau 16, tetapi saya bisa saja salah.

Masalah utama di sini adalah pipelining. (Nglayout.initialpaint.delay itu adalah seberapa cepat Firefox menampilkan data yang telah diterimanya.)

Pada dasarnya, ketika Anda menyetel data pipelining Anda (non-proxy) ke sesuatu seperti 50 (yang berlebihan), Anda menggunakan kecepatan koneksi sisi server dari 50 orang yang berbeda. Tidak ada kerugian bagi Anda, tetapi tempat yang akan Anda datangi akan menderita.

Pipelining proxy sedikit berbeda, tetapi pada dasarnya masalahnya sama. Anda akan memberi tekanan lebih pada proxy (buruk jika itu adalah layanan proxy publik, oke jika proxy tersebut adalah komputer pribadi Anda atau hanya melayani sejumlah kecil orang).

Jika Anda benar-benar membutuhkan peningkatan kecepatan maka Anda mungkin dapat mengatur nilai pipelining Anda menjadi seperti 2 atau 3, tetapi sebenarnya Anda tidak memerlukan pipelining sama sekali. Permintaan tambahan tersebut pada akhirnya adalah data yang terbuang (sisi server), karena pada akhirnya akan mengirimi Anda banyak salinan data Anda.

Juga, koneksi internet Anda secara keseluruhan mungkin terpengaruh, karena Anda akan mentransfer lebih banyak data secara total. Melakukan torrent, mengunduh, memperbarui, dll. Anda mungkin terpengaruh.
