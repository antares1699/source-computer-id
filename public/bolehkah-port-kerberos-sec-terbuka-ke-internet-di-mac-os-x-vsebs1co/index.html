Bolehkah port kerberos-sec terbuka ke Internet di Mac OS X?

Saya baru saja membeli Mac pertama saya, saya ingin menghubungkannya langsung ke Internet di fasilitas colocation.

Menjalankan nmap melawannya menunjukkan:

Starting Nmap 6.40 ( http://nmap.org ) at 2015-02-18 21:54 GMT
Nmap scan report for foo.bar (123.45.67.89)
Host is up (0.00036s latency).
rDNS record for 123.45.67.89: 89.67.45.123.in-addr.arpa
Not shown: 997 closed ports
PORT     STATE SERVICE
22/tcp   open  ssh
88/tcp   open  kerberos-sec
5900/tcp open  vnc
MAC Address: 01:23:45:67:89:AB (Apple)

Saya sengaja memulai layanan SSH dan VNC.

Apa yang memulai kerberos?

Bolehkah porta itu dibuka ke Internet?

Jika tidak, bagaimana saya bisa menutupnya? Ketika saya mencoba mengatur firewall, satu-satunya aplikasi adalah Remote Management (VNC) dan Remote Login (SSH).

Tampaknya kerberos-sec pada port 88 adalah metode otentikasi yang terhubung ke berbagi file AFP (Apple Filing Protocol) atau SMB (Samba). Jika Anda ingin menonaktifkannya, Anda bisa sesuai instruksi di situs ini . Ini seharusnya berfungsi untuk Mac OS X 10.6.8 dan yang lebih lama:

sudo launchctl unload -w /System/Library/LaunchDaemons/com.apple.kdcmond.plist

Untuk Mac OS X 10.7 (Lion) dan di atasnya, perintahnya adalah:

sudo launchctl unload -w /System/Library/LaunchDaemons/com.apple.Kerberos.kdc.plist

Jika Anda ingin mengaktifkannya lagi, ubah saja unload menjadi load seperti ini:

sudo launchctl load -w /System/Library/LaunchDaemons/com.apple.Kerberos.kdc.plist
