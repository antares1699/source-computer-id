Proxifier yang mendukung UDP? [Tutup]

Saya mencari proxifier yang memaksa semua lalu lintas Windows melalui proxy tertentu. Saya memiliki server sendiri sehingga pada akhirnya saya dapat mengkonfigurasi semua jenis proxy (Socks, HTTPS, dll) jadi itu tidak menjadi masalah, meskipun dukungan HTTPS lebih disukai karena saya suka squid.

Salah satu masalah terbesar yang saya alami adalah menemukan program proxy yang mendukung koneksi UDP. Favorit saya mungkin Proxifier ( http://proxifier.com/ ) bekerja luar biasa, tetapi tidak mendukung UDP untuk jenis proxy APA PUN.

Secara khusus, saya mencoba membuat proxy berfungsi untuk game saya (yang jelas, menggunakan UDP)

Semua jenis program proxy injeksi tidak akan berfungsi - Saya memerlukan program yang mengalihkan semua lalu lintas umum menggunakan driver jaringan atau yang serupa. (kecuali Anda dapat menjamin bahwa program injeksi serupa akan berfungsi untuk sebagian besar 99% game FPS di PC)

Anda tidak dapat membuat proxy UDP, hanya karena tidak seperti TCP, UDP adalah protokol tanpa sambungan. Proksi bergantung pada Anda membuat koneksi ke server proksi, server proksi tersebut membuat permintaan atas nama Anda, dan kemudian mengembalikan data kembali kepada Anda.

Yang perlu Anda lakukan adalah menulis ulang paket saat mereka melewati firewall Anda. NAT biasanya melakukan ini untuk Anda. Firewall Anda juga harus mendukung SPI - Stateful Packet Inspection sehingga menyimpan catatan aliran UDP sehingga lalu lintas kembali kembali ke mesin yang tepat.

Jika Anda tidak dapat menyesuaikan firewall yang ada di antara Anda dan internet, maka VPN sejauh ini adalah cara termudah untuk digunakan. Ada banyak cara berbeda untuk membuat VPN menggunakan banyak teknologi berbeda. Yang benar bergantung sepenuhnya pada a) apa yang ingin Anda konfigurasikan, dan b) apa yang akan diizinkan oleh firewall Anda dan c) Apa yang dapat dijalankan oleh mesin klien Anda.

Di bawah bagian "Protokol dan metode proxy", lihat kolom 'UDP Associate'.

WideCap sekarang freeware dan berjalan di PC. URL: http://widecap.com/

Saya belum mengujinya seberapa baik menangani UDP.

EDIT: Saya menindaklanjuti dan menguji WideCap. Sistem konfigurasinya agak kikuk, tetapi sesuai dengan standar SOCKS5 termasuk fungsi asosiasi UDP. Terhubung ke server AntiNAT dalam mode Socks5, itu memungkinkan aplikasi yang tidak mendukung proxy konektivitas TCP dan UDP tanpa masalah selain karena agak lambat.

Catatan: Seperti yang saya pahami, ini hanya berfungsi untuk perangkat lunak klien. Sepertinya Anda tidak dapat menggantung server dengan LISTEN port di belakang Socks5 dan menerima koneksi masuk.
