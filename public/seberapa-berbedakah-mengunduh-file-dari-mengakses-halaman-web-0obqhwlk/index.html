Seberapa berbedakah mengunduh file dari mengakses halaman web?

Saya bertanya karena di sini saya memiliki kecepatan unduh yang seperti 1MBps sedangkan halaman web membutuhkan waktu yang sangat lama untuk memuat (pasti bukan 1 MBps).

Saya kira mengakses halaman web pada dasarnya memintanya dari server, dan kemudian dengan cara "mengunduh" halaman dan kemudian merendernya.

Apakah aku salah? Apa yang menjadi alasan saya memiliki perbedaan kecepatan unduh dan halaman web?

Beberapa halaman web memiliki banyak komponen individual, yang semuanya harus diunduh secara terpisah. Browser Anda menangani semua ini untuk Anda, tetapi seringkali dengan kinerja yang menurun, karena banyak koneksi yang berbeda harus dibuat, dan kemungkinan beberapa harus menunggu yang lain selesai. Selain itu, halaman web sering kali memiliki data dari berbagai server yang berbeda (iklan mungkin berasal dari server mereka sendiri), dan penundaan untuk elemen apa pun dapat memperlambat keseluruhan. File adalah unduhan sederhana dari satu file dari satu tempat.

Oleh karena itu, bandwidth (jumlah data yang dapat Anda unduh dalam satuan waktu) sangat penting dalam pengunduhan file. Latensi (waktu tambahan) lebih penting di web.

Ini mungkin disebabkan oleh salah satu / semua hal berikut:

  • Server lambat.
  • Server memiliki koneksi yang lambat.
  • Komputer Anda lambat. Proses rendering juga membutuhkan waktu.
  • Meskipun koneksi memungkinkan lalu lintas 1Mbps, namun memiliki latensi tinggi. Bandingkan ini dengan kapal. Anda dapat memuat banyak barang di dalam kapal, tetapi akan memakan waktu lama untuk mencapai tujuannya.

Penting untuk memahami bagaimana halaman web diunduh untuk memahami mengapa mengunduh file 1MB mungkin lebih cepat daripada 256kb yang terdiri dari sebuah situs web.

  1. Jika kompresi GZIP ada di server web untuk HTML / JavaScript / dll. (kemungkinan besar), browser Anda akan mengunduh konten terkompresi dan kemudian harus mendekompresinya.
  2. Gambar mungkin hanya berukuran 30-40k, tetapi setelah didekodekan jumlahnya dapat mencapai banyak megabyte dalam memori
  3. Jika situs web memiliki beberapa file eksternal (CSS / JavaScript / Gambar), masing-masing file tersebut dapat diunduh pada koneksi terpisah sebagai bagian dari browser Anda yang menyusun laman.
  4. Mengunduh file adalah operasi dasar dalam HTTP / FTP yang disederhanakan. Tidak ada konten untuk membedakan dan merender

Belum disebutkan - ada cara lain untuk mengunduh laman web biasa saat ini daripada teks yang Anda lihat di peramban. Jika halaman yang Anda kunjungi memiliki kemampuan untuk diedit, mungkin berisi banyak kode untuk menanganinya. TinyMCE adalah editor "Teks Kaya" javascript yang populer untuk web. Itu kecil tapi tidak bahwa mungil.

Ini hanya satu contoh, tetapi hal semacam ini membengkak semua jenis halaman web dan dapat membuat pengalaman browsing Anda tidak seperti yang Anda harapkan. Bergantung pada orang yang membuat halaman web atau aplikasi web, Anda mungkin akan mengunduh aplikasi seperti ini apakah Anda pernah menggunakannya atau tidak . Itu baru saja dimuat ke dalam memori, menunggu.

Aspek lainnya adalah psikologis. Saat Anda memilih file untuk diunduh, Anda akan menunggu sampai file itu tiba. Ketika Anda mengklik link ke sebuah halaman, Anda lebih cenderung mengharapkan kepuasan instan, jadi meskipun saya tidak mengatakan 'itu semua ada di kepala Anda', pasti ada harapan di sana.

Anda tidak menentukan browser mana yang Anda gunakan dan jenis koneksi internet yang Anda miliki.

Jika koneksi internet misalnya. tautan satelit, memiliki bandwidth unduhan tinggi dan bandwidth unggahan sangat lambat dan latensi tinggi. Hal serupa berlaku untuk koneksi seluler 3G / HSDPA, meskipun latensi lebih rendah daripada di satelit tetapi masih sering 10-30x dibandingkan dengan ADSL atau kabelodem.

Ini dapat menjadi lebih buruk dengan menggunakan browser yang buruk seperti Internet Explorer yang hanya membuka 2-3 koneksi bersamaan saat mengunduh file dari server web. Dengan koneksi jenis ponsel, koneksi HTTP yang melakukan jabat tangan TCP / IP 3-arah, satu file dapat memakan waktu hampir satu detik sebelum mulai diunduh, sekarang gabungkan ini dengan hanya beberapa koneksi simultan dari browser dan tiba-tiba membutuhkan puluhan detik untuk mengunduh halaman web kecil yang memiliki beberapa gambar dan stylesheet serta javascript yang ditautkan ke sana.

Saya menggunakan Opera yang memungkinkan saya untuk mengatur koneksi bersamaan ke beberapa nomor yang sangat besar, misalnya 32 atau lebih. Ini berarti bahwa ketika membuka situs web melalui tautan latensi tinggi, browser membutuhkan beberapa detik untuk melakukan jabat tangan dengan server dan kemudian mengunduh 32 item secara bersamaan dan seringkali dapat memaksimalkan bandwidth unduhan yang tidak akan pernah terjadi jika browser mengunduh satu / beberapa file sekaligus.

Perbedaan kecepatan juga dapat terjadi jika komputer Anda lambat atau tidak memiliki cukup memori dan browser kemudian dapat memperlambat mesin saat merender halaman. Hal ini dapat dihindari dengan menggunakan browser yang lebih cepat / lebih ringan seperti Opera atau jika memungkinkan, atur browser untuk tidak merender halaman hingga semua elemen yang diperlukan telah diunduh. Saat ini browser mencoba merender halaman secepat mungkin, yang biasanya menyebabkan mereka merender ulang halaman beberapa kali dan ini bisa lambat pada mesin yang lebih lama.
