Matplotlib font_manager.py kesalahan menemukan font

Pada mesin Ubuntu 12.04, setiap kali saya merencanakan sesuatu, saya mendapatkan kesalahan seperti:

/usr/lib/pymodules/python2.7/matplotlib/font_manager.py:1228: UserWarning: findfont: Tidak bisa cocok: family = Bitstream Vera Sans: style = normal: variant = normal: weight = medium: stretch = normal: size = x-large. Mengembalikan /usr/share/matplotlib/mpl-data/fonts/ttf/cmb10.ttf

yang saya tidak dapat menemukan solusi untuk online. Kode berikut mereproduksi kesalahan

import pylab as plt
plt.plot(np.random.random((128)))
plt.title("This should be monospaced")
plt.savefig("/tmp/testfig.pdf")

dan juga tes mandiri matplotlib memberikan kesalahan ini:

python -c "import matplotlib as m ; m.test(verbosity=1)"

Di ~/.matplotlib/matplotlibrc Saya sudah:

font.family         : monospace
font.style          : normal
font.variant        : normal
font.weight         : medium
font.stretch        : normal
font.size           : 10.0
font.serif          : Palatino, Bitstream Vera Serif, New Century Schoolbook, Century Schoolbook L, Utopia, ITC Bookman, Bookman, Nimbus Roman No9 L, Times New Roman, Times, Charter, serif
font.sans-serif     : Helvetica, Bitstream Vera Sans, sans-serif
font.monospace      : Monaco, Ubuntu Mono Regular, Bitstream Vera Sans Mono, Andale Mono, Nimbus Mono L, Fixed, Terminal, monospace

Beberapa hal yang tidak menyelesaikan masalah bagi saya:

  1. Hapus file di ~/.matplotlib/ (mis. matplotlibrc, fontList.cache, tex.cache/)
  2. Ubah properti font.family di ~/.matplotlib/matplotlibrc
  3. Instal ulang matplotlib setelah apt-get purge

Versi yang saya gunakan adalah:

  • matplotlib 1.1.1 ~ rc1 + git20120423-0ubuntu1
  • Python 2.7.3
  • IPython 0.12.1
  • Ubuntu 12.04

matplotlib selalu menggunakan kumpulan fonnya sendiri, terlepas dari fon lain yang diinstal pada sistem. Di Ubuntu 12.04 LTS, font ini terletak di /usr/share/matplotlib/mpl-data/fonts/.

Saat memanggil fonta dalam kode Anda yang tidak tersedia dalam direktori ini, matplotlib pertama-tama akan mencoba kembali ke Bitstream Vera Sans. Namun, tidak ada font yang tersedia pada .../mpl-data/fonts/ pada instalasi Ubuntu 12.04 LTS. Akhirnya, matplotlib kembali ke font cmb10.ttf, yang tersedia.

Menginstal paket font sistem seperti yang disarankan di atas tidak akan memperbaiki peringatan ini.

Namun, seseorang dapat menambahkan rcParams['mathtext.fontset'] = 'cm' ke kode untuk menghindari beberapa fallback dan peringatan.

Atau, tentukan fonta yang ada pada /usr/share/matplotlib/mpl-data/fonts/.

Bagi saya di OSX, instalasi Python saya ada di direktori home saya karena batasan admin yang menyebabkan kebingungan lokasi font. Saya bisa membuat Helvetica bekerja dengan mengikuti langkah-langkah ini: http://blog.olgabotvinnik.com/blog/2012/11/15/2012-11-15-how-to-set-helvetica-as-the-default-sans-serif-font-in/

  1. Unduh dan instal Fondu untuk mengubah Mac-Helvetica menjadi ttf-Helvetica. Dengan homebrew, ini dapat dilakukan dengan: brew install fondu.

  2. Temukan Helvetica di sistem Anda. Bagi saya, itu /System/Library/Fonts/Helvetica.dfont

  3. Temukan di mana matplotlib menyimpan datanya. Gunakan python import matplotlib; matplotlib.matplotlib_fname() untuk mencetak direktori.

  4. Letakkan .ttf di: your_matplotlib_dir/matplotlib/mpl-data/fonts/ttf lalu sudo fondu -show /System/Library/Fonts/Helvetica.dfont

  5. Edit file .matplotlibrc Anda. Salin file .matplotlibrc asli ke dalam direktori pribadi agar tidak selesai ditulis saat memperbarui matplotlib. $ cp your_matplotlib_dir/matplotlib/mpl-data/matplotlibrc ~/.matplotlib/matplotlibrc

  6. Temukan baris: #font.sans-serif : Bitstream Vera Sans, Lucida Grande, Verdana, Geneva, Lucid, Arial, Helvetica, Avant Garde, sans-serif lalu hapus komentar dan pindahkan Helvetica ke depan. font.sans-serif : Helvetica, Bitstream Vera Sans, Lucida Grande, Verdana, Geneva, Lucid, Arial, Helvetica, Avant Garde, sans-serif
