Bagaimana cara saya terhubung ke server IPv6 NAT-ed saya?

Sampai saat ini saya memiliki server yang berjalan di rumah sebagai DMZ (semua lalu lintas eksternal dialihkan ke sana) di belakang router / modem kabel. Ini bekerja dengan baik. Sekarang, setelah ISP saya menggunakan 100% IPv6, saya tidak lagi dapat terhubung dengannya.

Saya dulu memahami IPv4, segmen jaringan, dll, tetapi saya tidak dapat "membaca" IPv6 dengan cara yang sama jadi ketika saya sekarang tidak dapat ssh di saya tidak tahu mengapa. Mungkin karena yang saya yakini sebagai IP eksternal mungkin bukan ...

Saya mendapatkan alamat IP eksternal dengan membuka whatismyip.org, yang menghasilkan 2a01:79c:cebd:5bc:b0cc:d7d2:22d0:3369 muncul di layar. Ini tidak dilakukan pada server (yang tidak memiliki kepala), tetapi pada lain mesin di jaringan yang sama. Ketika saya mendengar sesuatu tentang IPv6 yang memungkinkan untuk langsung terhubung ke mesin di belakang NAT tanpa penerusan, ini membuat saya curiga bahwa alamat IPv6 yang saya lihat bukanlah IP router (yang biasanya ada di dunia IPv4) , tetapi alamat IPv6 komputer di jaringan lokal. Saya mungkin jauh dari sini, tetapi saya pikir itu mungkin layak untuk disebutkan.

Bagaimanapun, mencoba menyambung ke server web yang berjalan di server telah dicoba seperti ini: curl http://[2a01:79c:cebd:5bc:b0cc:d7d2:22d0:3369]. Saya tidak dapat tersambung.

Ketika saya mendengar sesuatu tentang IPv6 yang memungkinkan untuk langsung terhubung ke mesin di belakang NAT tanpa penerusan

Oh, begitu banyak kebingungan yang tidak perlu. Maafkan saya.

IPv6 pada dasarnya bekerja sama dengan IPv4. Jelas ada beberapa perbedaan, seperti IPv6 yang menggunakan lebih banyak bit per alamat.

Untuk sementara, DHCPv6 tidak berkembang seperti DHCP / IPv4. Orang-orang didorong untuk menggunakan protokol Permintaan Router / Iklan Router sebagai gantinya. (Saya pernah melihat ini disebut sebagai RS dan RA.) RS / RA tidak pernah memiliki banyak popularitas di IPv4, mungkin terutama karena DHCP / IPv4 sudah cukup populer. Dengan IPv6, proses yang disebut SLAAC menggunakan RS dan RA.

Jika Anda menggunakan SLAAC, beberapa implementasi mengharuskan Anda menggunakan / 64 ukuran subnet. Ini adalah perubahan penting dari DHCP / IPv4 di mana banyak ukuran subnet didukung, dan ukuran subnet yang berbeda cukup umum digunakan.

Dengan IPv6, standar awal menetapkan bahwa ISP harus memberikan setidaknya / 48 blok. (Setiap / 48 blok memiliki 2 ^ 16/64 subnet, alias 16.384 / 64 subnet.) Ini membiarkan orang subnet. Ini pada dasarnya menghilangkan kebutuhan untuk menggunakan NAT untuk membantu meminimalkan penggunaan alamat, karena orang mendapatkan begitu banyak alamat. Hal ini menyebabkan banyak orang menyatakan bahwa NAT tidak diperlukan.

Cara untuk "menghindari persyaratan untuk menggunakan NAT" adalah dengan mengatur subnetting IPv6. Namun, "menyiasati persyaratan NAT" secara signifikan berbeda dengan "menyiasati NAT". Anda dapat menggunakan NAT dengan IPv6, dan membuatnya secara teknologi bekerja dengan cara yang sama persis dengan NAT dengan IPv4, dengan perbedaan yang jelas seperti umumnya memiliki ukuran subnet yang lebih besar. Prinsipnya, bagaimanapun, akan bekerja sama.

ini membuat saya curiga bahwa alamat IPv6 yang saya lihat bukanlah IP router (yang biasanya ada di dunia IPv4), tetapi alamat IPv6 komputer di jaringan lokal.

Ya, itu mungkin. Dengan IPv4, Anda dapat menetapkan alamat IP publik ke mesin internal Anda. Hal ini biasanya tidak dilakukan karena kurangnya alamat IPv4 publik yang tersedia, tetapi dapat diatur. Dengan IPv6, skenario umum umumnya adalah tidak adanya kekurangan alamat publik yang tersedia, jadi jenis penyiapan ini lebih umum dengan IPv6.

Namun, mesin internal Anda akan memiliki gateway default, yang kemungkinan besar adalah router. Masuk ke router itu, dan Anda dapat melihat tabel perutean.

Anda menggunakan alamat IPv6 publik. Alamat IPv6 penggunaan pribadi dimulai dengan "fd" (seperti alamat IPv4 penggunaan pribadi dimulai dengan "192.168." Atau "172.16." Hingga "172.31." Atau "10.") Alamat yang dimulai dengan "fe80:" adalah "tautan alamat -lokal ", mirip dengan" 169.254 "IPv4. alamat tetapi dengan beberapa perbedaan. (Dalam IPv6, Melihat "fe80:" adalah suatu persyaratan; tidak melihatnya adalah masalah. Sebaliknya, melihat IPv4 "169.254." Sering menjadi masalah.)

Anda mungkin dapat menemukan alamat server Anda dengan NDP. mis., di Microsoft Windows:

netsh interface ipv6 show neighbor

Sistem operasi lain mungkin menggunakan perintah seperti ndp atau ip -6 untuk menampilkan tetangga IPv6.

Biasanya, hanya mengaktifkan IPv6 tidak akan memengaruhi IPv4. Jadi saya rasa maksud Anda bahwa ISP Anda sebenarnya memiliki dengan disabilitas konektivitas IPv4 asli (dan pergi dengan CGNAT)?

Bagaimanapun, pertama-tama mari kita perjelas apa yang membuat IPv6 berbeda:

Ketika saya mendengar sesuatu tentang IPv6 yang memungkinkan untuk langsung terhubung ke mesin di belakang NAT tanpa penerusan, ini membuat saya curiga bahwa alamat IPv6 yang saya lihat bukanlah IP router (yang biasanya ada di dunia IPv4) , tetapi alamat IPv6 komputer di jaringan lokal.

Alasan Anda tidak memerlukan penerusan port adalah itu Anda tidak berada di belakang NAT lagi. Sebagai gantinya, selain alamat globalnya sendiri, router Anda juga mendapatkan alamat lengkap jarak (awalan a / 64) untuk digunakan pada LAN, dan setiap perangkat memiliki alamat globalnya sendiri dari awalan itu. Jangkauan LAN Anda adalah 2a01:79c:cebd:5bc::/64, dan setiap alamat darinya dapat dijangkau secara eksternal.

Jadi, jika Anda mencoba menyambung ke alamat yang ditunjukkan oleh "whatismyip" di komputer Anda, sebenarnya Anda tersambung ke komputer yang sama - jika Anda ingin mengetahui alamat server Anda, Anda harus memeriksa saya t dari server itu. Anda sebenarnya tidak perlu mengunjungi situs - karena alamatnya sudah mendunia, Anda bisa mendapatkannya dari ip addr di Linux, atau ifconfig di BSD, atau ipconfig di Windows.

(Bahkan itu lebih baik karena banyak host mengonfigurasi sendiri banyak alamat - alamat tetap, ditambah alamat sementara berubah setiap hari. Situs web akan menunjukkan kepada Anda alamat sementara, tetapi untuk koneksi masuk Anda pasti menginginkan yang permanen. Perintah yang disebutkan di atas akan menunjukkan kepada Anda berdua.)

Namun, hanya karena tidak ada "penerusan port" tidak berarti Anda tidak memerlukan konfigurasi router lagi.

Di IPv4, menambahkan aturan "penerusan port" tidak hanya menyiapkan terjemahan NAT - tetapi juga membuka firewall untuk port tertentu tersebut. Meskipun NAT tidak lagi menjadi perhatian di IPv6, Anda masih memiliki firewall - dan jika ISP Anda memiliki rasa aman, firewall itu masih memblokir koneksi masuk secara default.

Jadi, jika Anda memiliki alamat yang benar tetapi masih tidak dapat terhubung, Anda dapat melakukannya masih perlu mengakses konfigurasi router Anda, menemukan pengaturan firewall, dan menambahkan aturan yang mengizinkan lalu lintas masuk ke server_ip.

IPv4 dan IPv6 benar-benar berbagi sebagian besar konsep yang sama - peruteannya sama, subnetnya sama, NAT adalah suatu hal, alamat pribadi masih ada, seperti yang disebutkan TOOGAM. Dua yang terakhir ini kurang lazim di IPv6 daripada di IPv4 (di mana mereka digunakan karena kekurangan alamat).
