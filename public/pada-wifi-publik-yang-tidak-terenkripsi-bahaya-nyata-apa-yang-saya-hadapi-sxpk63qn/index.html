Pada wifi publik yang tidak terenkripsi, bahaya nyata apa yang saya hadapi?

Pertama, penyiapan saya

  • Windows 7 Home Premium

  • gunakan firewall bawaan

  • UAC pada maks

  • di pengaturan jaringan nirkabel Windows untuk jaringan publik, selain pengaturan default, saya telah 1) mengatur untuk memblokir semua streaming media, dan 2) mematikan berbagi file publik. Ketika saya terhubung ke wifi gratis, saya selalu mengatur Windows untuk mengklasifikasikannya sebagai "Jaringan Publik".

  • sepenuhnya ditambal Windows, Office, Firefox, Flash Player, Java, dll, dll .... semuanya

  • Saya selalu memastikan 100% bahwa saya terhubung ke wifi yang diharapkan dan bukan jaringan wifi bajakan pencuri data palsu milik seseorang.

  • Setiap kali saya masuk ke situs yang saya pedulikan, saya selalu masuk menggunakan https. Semua akun email saya (gmail) menggunakan https. Saya benar-benar tidak peduli jika ada yang melihat websurfing saya (posting forum, pencarian google, dll).

Terlepas dari apakah AV saya akan mendeteksi dan menghentikan malware yang benar-benar masuk ke komputer saya, izinkan saya bertanya

Selanjutnya, dengan asumsi tidak ada malware yang ditempatkan di komputer saya dari lan / wifi, apa (dan seberapa hebat) bahayanya

q2) data saya di komputer saya (dari dicuri atau dilihat)?

q3) data sensitif yang saya kirimkan menggunakan https terlihat atau dicuri dan tidak dienkripsi?

Saya menanyakan bahaya dunia nyata ("di alam liar") di wifi gratis acak seperti Starbuck dll, tetapi bukan tentang setiap kemungkinan atau tentang metode yang membutuhkan keterampilan atau peralatan yang sangat tidak mungkin untuk ditemui.

btw, saya tahu ada langkah lebih lanjut yang dapat saya ambil untuk keamanan yang lebih baik daripada yang saya sebutkan di atas, namun, untuk saat ini, saya perlu mendapatkan gambaran yang akurat, tetapi tidak terlalu paranoid, tentang posisi saya dengan pengaturan saya saat ini. a1) Apa yang Anda maksud dengan "peluang apa saja"? Seberapa besar kemungkinan pemilik wifi jahat, atau seberapa besar kemungkinan mereka dapat melakukannya jika ya? Pertanyaan sebelumnya tidak ada datanya. Yang terakhir tergantung pada apa Anda menggunakan wifi mereka. Jika Anda mengunduh file yang dapat dieksekusi dan menjalankannya maka jelas sangat mudah bagi mereka untuk menaruh malware di komputer Anda. Vektor berikutnya yang paling mungkin adalah PDF, atau Java / Flash / skrip berbahaya di situs web, tetapi semua itu mengharuskan Anda menjalankan perangkat lunak yang rentan (meskipun dalam kasus Adobe Acrobat, itu aku s rentan bahkan jika Anda 100% up-to-date, kami tidak tahu apa yang salah dengannya namun ;)

Untuk menghindari hal ini, saya akan mengatakan, dalam urutan naik paranoia (yaitu 1 masuk akal, sisanya lebih paranoid):

  1. Jangan mengunduh file yang dapat dieksekusi melalui koneksi internet yang tidak Anda percayai
  2. Tidak mengatur browser Anda untuk membuka PDF di Acrobat (ada banyak alternatif yang lebih aman), Flash, atau applet Java tanpa meminta Anda
  3. Pertimbangkan untuk menggunakan NoScript

Tentu saja, jika Anda menggunakan situs web SSL, mereka tidak dapat mengubah data apa yang Anda dapatkan. Mungkin. Lihat jawaban 3.

a2) Dengan asumsi tidak ada malware yang ditanam di komputer Anda, dan Anda beroperasi menurut aturan di jawaban 1, efektif nol. Mungkin ada program yang membocorkan informasi, atau memiliki bug yang memungkinkan orang meletakkan sesuatu di komputer Anda, tetapi itu tidak benar-benar relevan dengan wifi. Meminimalkan jumlah aplikasi yang diizinkan untuk menggunakan internet (dalam pengaturan firewall) adalah ide yang bagus untuk alasan ini.

a3) Saat Anda menggunakan HTTP, browser Anda memverifikasi bahwa situs tersebut sesuai dengan yang mereka katakan dengan memeriksa sertifikat mereka. Hanya orang tertentu yang dapat memberikan sertifikat ini, dan browser Anda tahu cara memeriksa sertifikat mereka.

Apa artinya untuk keamanan? Pertama, itu berarti Anda mempercayai para penulis sertifikat itu. Ada serangan pada sistem mereka untuk menghasilkan sertifikat palsu di masa lalu, dan ada kasus browser mempercayai otoritas sertifikat sehingga tidak ada yang yakin siapa pemiliknya sekarang.

Apa yang bisa kau lakukan? Beberapa browser memiliki ekstensi untuk membantu Anda di sini. Yang Anda inginkan adalah sesuatu yang mengingat sertifikat apa yang dimiliki situs web terakhir kali Anda mengunjunginya, dan akan memberikan peringatan besar jika itu berubah. Ini berarti meskipun otoritas sertifikat dikompromikan dengan cara tertentu, Anda tetap tidak akan menyerahkan data Anda.

Ini adalah hasil yang sangat tidak mungkin, ngomong-ngomong - ini akan membutuhkan seseorang untuk mendapatkan sertifikat palsu DAN kemudian menargetkan orang yang menggunakan situs itu melalui wifi mereka ... Mengingat nilai sertifikat, dan upaya untuk mendapatkannya, itu banyak kemungkinan besar itu akan digunakan dalam serangan yang lebih luas. Tapi tidak ada ruginya melindungi diri Anda dari hal-hal seperti itu.

Oh dan tentu saja, situs yang menggunakan sertifikat yang ditandatangani sendiri mudah untuk disamarkan. Memiliki ekstensi yang membandingkan sertifikat terakhir kali Anda mengaksesnya akan mengingatkan Anda tentang man-in-the-middle yang terjadi.

Q1) Jika tidak ada masalah dengan tumpukan IP, risiko utama Anda adalah risiko tambahan adalah serangan pria tipe menengah di mana seseorang menyamar sebagai server, dan memasukkan kode ke dalam beberapa data yang Anda ambil. Ini adalah risiko setiap kali Anda tidak memiliki kendali atas jalur penuh. Apakah koneksi dienkripsi atau tidak kemungkinan besar tidak banyak mengubah risikonya. Serangan injeksi kode umumnya dilakukan dengan menginfeksi server dan tidak terkait dengan akses wi-fi.

Q2) Kecuali jika berbagi file dalam bentuk tertentu dihidupkan, data komputer Anda harus aman dari akses file jaringan. Masalah dengan tumpukan IP atau perangkat lunak lain mungkin memungkinkan seseorang untuk memecahkan sistem Anda. Selain kemudahan mengakses komputer Anda secara langsung melalui jaringan, menggunakan wi-fi publik tidak banyak mengubah risikonya. Jika Anda tetap menambal, semua vektor yang diketahui harus ditutup.

Q3) HTTPS cukup aman, dan tidak ada metode pemecahan praktis yang diketahui untuk sandi umum. Data yang dikirimkan melalui protokol ini dapat dianggap aman. Serangan orang di tengah mungkin dilakukan, tetapi umumnya memerlukan sertifikat yang cocok yang dikeluarkan oleh vendor sertifikat tepercaya. Ada beberapa serangan di masa lalu di mana bilah alamat ditimpa untuk menyembunyikan alamat asli. Ini seharusnya tidak mungkin lagi.

Anda tampaknya mengikuti prosedur yang baik untuk memastikan Anda aman. Jika Anda menjadi sasaran khusus, risiko Anda meningkat. Penggunaan wi-fi publik dapat meningkatkan risiko Anda untuk beberapa vektor serangan yang ditargetkan.

Itu Secunia PSI perangkat lunak dapat mengaudit perangkat lunak yang Anda instal untuk mengetahui risiko keamanan yang diketahui.

Dalam kebanyakan kasus, wifi publik seharusnya tidak lebih berisiko dari jenis malware yang mungkin dihentikan oleh perangkat lunak antivirus Anda daripada wifi lainnya. Namun, dimungkinkan untuk mengatur banyak titik akses untuk menggunakan apa yang disebut halaman "portal tawanan" - jenis hal yang mungkin Anda temukan di tempat yang membuat Anda masuk sebelum meneruskan lalu lintas ke web. Seorang peretas dapat menyiapkan salah satu laman ini untuk mengautentikasi Anda secara otomatis dan kemudian mengarahkan Anda ke situs web berbahaya apa pun yang mereka pilih.

Selain itu, ketika Anda menggunakan wifi tidak terenkripsi, semua lalu lintas non-https Anda disiarkan di tempat yang jelas untuk siapa saja yang mungkin mendengarkan. Mudah-mudahan satu-satunya "pendengar" adalah router / titik akses nirkabel Anda, tapi tidaklah sulit untuk menyiapkan komputer untuk mendengarkan lalu lintas ini lagi . Ini sangat berbahaya di tempat umum di mana ada kemungkinan orang lain menggunakan koneksi yang sama, tetapi bahkan rumah Anda sendiri tidak kebal jika Anda tidak mengenkripsi lalu lintas Anda.

Kecuali jika Anda juga membatasi penjelajahan Anda hanya ke jaringan terenkripsi atau hanya situs https (sebagai lawan dari situs http biasa), informasi siaran termasuk cookie, token otentikasi, kata sandi, dan banyak lagi. Jika Anda tidak mengenkripsi wifi Anda, pada dasarnya game sudah berakhir.
