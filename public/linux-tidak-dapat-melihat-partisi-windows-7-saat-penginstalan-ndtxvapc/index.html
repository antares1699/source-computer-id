Linux tidak dapat melihat partisi Windows 7 saat penginstalan

Saya telah mencoba menginstal Linux sebagai dual boot dengan Windows 7 di Dell latitude e6510 saya. Saat ini menjalankan Windows 7 dan saya telah menggunakan alat MS disk untuk mengecilkan partisi Win 7 NTFS untuk memberi ruang bagi Linux.

Masalah yang saya alami adalah ketika saya menjalankan penginstal Linux dengan boot dari CD, mereka melihat seluruh hard drive sebagai ruang yang tidak terisi. Saya telah mencoba Ubuntu 10.10, Kbuntu 10.10 dan Fedora 14 dan semuanya memiliki masalah yang sama.

Saya juga mencoba opsi "instal di Windows" Ubuntu dan tidak dapat menjalankannya.

EDIT:

Boot Gparted 0.8.0 dari drive usb tidak berfungsi. Ini melaporkan seluruh drive sebagai tidak dipartisi.

Windows 7 menggunakan pendekatan kepemilikan "baru" dalam mengelola hard disk, Dynamic Disk (DD). Sistem baru ini adalah simulasi RAID (jadi tidak baru sama sekali) ditambah beberapa gadget lain yang sebagian besar pengguna normal tidak akan pernah mengerti penggunaannya. Salah satu perbedaan yang terlibat adalah penggunaan skema blok deskripsi partisi yang berbeda dari MBR standar, yang disebut Microsoft GPT. Dengan kata lain, indeks disk yang diperkenalkan oleh Microsoft secara substansial berbeda dari yang digunakan oleh standar MBR. Konsekuensi dari ini tidak lain adalah alat Microsoft dapat mengenali dan memahami dengan benar di mana partisi itu berada dan apa jenisnya.

Karena alat partisi disk Linux bekerja pada sistem MBR standar (seperti kebanyakan jika tidak semua sistem operasi lain), mereka tidak dapat menafsirkan DD Microsoft dan indeks GPT-nya. Konsekuensinya adalah disk yang dipartisi DD dikenali sebagai bebas karena tidak ada data deskriptor MBR. Hanya untuk memberikan kesempatan untuk memulihkan dokumen dan file lain dari partisi DD, beberapa penambahan telah dibuat ke kernel Linux, untuk memungkinkannya menangani zona hard disk tersebut. AFAIK kode kompatibilitas lebih lanjut di jalan.
Oleh karena itu, jangan lupa bahwa Linux tidak perlu terus mencoba agar kompatibel dengan nonstandar Microsoft ...

Untuk saat ini, untuk menjalankan dualbooting Linux, Anda harus memaksa Windows untuk menggunakan standar MBR dan bukan DD Microsoft sendiri. Alat partisi Linux kemudian dapat mengenali partisi MBR Windows saat ini dan kemudian dapat menginstal Linux dengan benar dan kode boot (GRUB ...) untuk dualbooting. Oleh karena itu, strategi ini sebagian besar valid jika Windows 7 telah diinstal dengan cara ini.

Waspadalah terhadap kemungkinan situasi berisiko yang mencurigakan! Jika Anda memiliki beberapa hard disk pada sistem Anda yang dipartisi dengan Windows 7 DD dalam salah satu mode GPT, saat menginstal Linux pada satu HD yang berbeda dari yang pertama (C: pada Windows), Anda mungkin berhasil melakukan dualbooting. Masalahnya adalah bahwa Windows 7, tergantung pada bagaimana Anda mengkonfigurasi hard disk Anda, dapat mendeteksi kegagalan volume jika volume seperti itu direntangkan pada HD fisik yang sekarang digunakan oleh Linux. Anda mungkin kehilangan banyak data Windows dengan cara ini. Faktanya adalah seluruh volume, jadi tidak hanya bagian yang secara fisik terkandung oleh hard disk tersebut yang dapat hilang dengan semua datanya.

Jadi satu-satunya aturan yang harus diikuti, untuk saat ini, adalah perencanaan yang tepat dari Windows 7 dari konfigurasi partisi MBR, menghindari cara ini menggunakan multi-hard-disk spanned volume, menyisakan ruang untuk instalasi Linux dan kemudian melanjutkan dengan Linux Install.

Masalah besarnya adalah pengguna Windows harus merasa nyaman (!) Dengan sistem Dynamic Disc sebelum dapat mengubah pengaturan partisi dan membuat ruang disk yang diperlukan untuk instalasi Linux.

Kesimpulannya, pengenalan sistem Dynamic-Disk oleh Microsoft, sebenarnya bukan masalah teknis untuk koeksistensi Linux-Windows. Saat ini komunitas Linux sedang mengerjakan kebaruan Microsoft untuk menyelamatkan pengguna Windows dari mengacaukan sistem Windows mereka saat mencoba menginstal Linux. Agar strategi tambahan ini berfungsi sepenuhnya, diperlukan alat pemartisi Linux untuk mengetahui skema pemartisian non-standar. Ini mungkin diperkenalkan segera karena ini adalah modifikasi sederhana bahkan jika struktur partisi Windows yang berbeda mungkin memerlukan pendekatan yang berbeda untuk mencapai penyiapan yang benar untuk penginstalan bersama sistem operasi baru.
