Meng-hosting sendiri layanan dns dinamis?

Apakah ada cara untuk menghosting sendiri layanan DNS dinamis? Saya sangat menyukai fleksibilitas DynDNS dkk, tetapi saya ingin menghosting layanan DNS dinamis saya sendiri sehingga saya dapat memiliki kontrol yang lebih baik dan kemampuan untuk mengirim domain milik saya ke nameserver tanpa harus membayar biaya. Apakah implementasi DNS dinamis FOSS tersedia, atau adakah cara untuk melakukannya di BIND?

bind v9 memiliki dukungan untuk pembaruan dinamis:

zone "example.net" {
    type master;
    file "zones/example.net.db";

    update-policy local;

    auto-dnssec allow;
    key-directory "keys/example.net";
};

  • Kamu bisa gunakan nsupdate untuk mengelola zona secara lokal,

  • atau hubungi nsupdate dari skrip CGI (diakses oleh PC klien, seperti DynDNS updater),

  • atau gunakan nsupdate dari jarak jauh menggunakan TSIG .

    zone "example.net" {
    update-policy {
        grant <keyname> name <hostname> A AAAA TXT;
        //grant * self * A AAAA TXT;
    };
};

key <keyname> {
    algorithm HMAC-MD5;
    secret "<keydata>";
};

    (dokumen: update-policy )

  • Windows memiliki dukungan asli DDNS menggunakan GSS-TSIG , yang memerlukan penyiapan Kerberos yang berfungsi (tidak harus Active Directory).

  • Di jaringan lokal, pembaruan dapat dilakukan oleh klien DHCP (ISC dhclient) atau server (ISC dhcpd). Ini juga dijelaskan di Amankan DNS dinamis halaman.

  • bind kaleng menandatangani secara otomatis zona dinamis .
