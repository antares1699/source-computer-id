Jika Anda ingin melihat sertifikat menggunakan Firefox misalnya, Anda perlu mematikan pengalihan.

Itu https://www.truecrypt.org situs melakukan pengalihan 301 (permanen) ke http://www.truecrypt.org .

Browser / proxy tampaknya menangani ini dengan sangat serius, dan akan menyimpan 301 sedemikian rupa sehingga setiap kunjungan berikutnya langsung ke alamat yang dialihkan.

Di firefox bisa Anda lakukan

  1. Kosongkan cache
  2. Nonaktifkan cache di about: config, set browser.cache.memory.enable dan browser.cache.disk.enable untuk Salah
  3. Buka Opsi / Lanjutan / Umum dan periksa Peringatkan saat mengalihkan

Sekarang jika Anda melakukan semua ini, lalu pergi ke https://www.truecrypt.org , itu harus tetap di sana dengan peringatan yang ingin dialihkan. Tetapi kemudian Anda dapat melihat sertifikat dengan cara biasa.

Saya bisa mendapatkan info ini menggunakan keriting , tersedia untuk Windows / Linux / Unix / MacOS dll .:

curl -v https://www.truecrypt.org

* SSL connection using TLS_DHE_RSA_WITH_AES_256_CBC_SHA
* Server certificate:
*       subject: CN=www.truecrypt.org,OU=Domain Control Validated,O=www.truecrypt.org
*       start date: Apr 10 12:41:56 2009 GMT
*       expire date: Apr 10 12:41:56 2012 GMT
*       common name: www.truecrypt.org
*       issuer: serialNumber=07969287,CN=Go Daddy Secure Certification 
Authority,OU=http://certificates.godaddy.com/repository,O="GoDaddy.com, 
Inc.",L=Scottsdale,ST=Arizona,C=US

(sebagian besar keluaran dipotong agar singkat)

Saya menganggap masalah ini (bahwa Anda tidak dapat melihatnya menggunakan alat browser standar) sebagai masalah keamanan yang serius untuk situs web yang sebenarnya.

SSL melayani dua tujuan: enkripsi dan otentikasi. Dalam kasus khusus ini, enkripsi sama sekali tidak penting, dan otentikasi sangat penting. Dan mereka:

  1. Gunakan sertifikat tingkat terendah, yang benar-benar hanya membuktikan bahwa seseorang yang dapat mengakses admin@truecrypt.org atau alamat serupa membeli sertifikat itu.

  2. Tidak akan membiarkan pengguna melihat sertifikat dengan mudah, oleh karena itu memverifikasi sertifikat merupakan masalah bagi sebagian besar pengguna.
