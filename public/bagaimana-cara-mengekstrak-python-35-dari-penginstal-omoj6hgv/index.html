Bagaimana cara mengekstrak Python 3.5+ dari penginstal?

Dalam versi menurunkan dari 3.5.0 ada .MSI penginstal untuk Windows, yang memiliki opsi tersembunyi yang sangat bagus. Opsi itu terlihat seperti ini:

msiexec.exe /a "file.msi" /qb /L*V "file.log" ALLUSERS=0 TARGETDIR="target" CURRENTDIRECTORY="%~dp0" <additional options>

yang pada dasarnya memungkinkan saya untuk mengabaikan hak admin, karena MSI beralih ke semacam penginstal untuk seluruh jaringan, dengan demikian izin menjadi solusi.

Dengan Python 3.5.0 diperkenalkan penginstal .EXE baru, yang memiliki .MSI file yang dikemas sendiri, Anda dapat mengeluarkannya dengan:

python-3.5.0.exe /layout [optional target directory]

namun ada hal yang sangat mengganggu yang terjadi dengan solusi ini. Ketika saya melakukan ini, file .MSI memiliki akhiran -d.msi dan ketika saya membukanya secara manual dengan perintah msiexec di atas, setiap file memiliki sufiks itu juga, yang menjadikannya a rusak total instalasi. Mengganti nama file sebenarnya bukan pilihan karena setiap file memiliki akhiran -d.<file ext>, bukan -d.msi.<file ext>, jadi terlalu sulit untuk mengganti namanya dengan cara yang sederhana dengan alat seperti Batch, kecuali saya ingin memeriksa beberapa kasus (misalnya folder).

Apakah ada cara untuk mengekstrak konten penginstal ke folder terpisah, seperti sebelumnya tanpa barang tambahan apa saja yang dimasukkan ke Programs and features, seperti Python 3.5.0 (64bit) atau simlar?

Atau pertanyaan lain - adakah cara untuk melarang penginstal mengakses Programs and features, menutupnya dari meminta hak istimewa admin dan registri sama sekali?

Ini cukup berguna jika saya ingin memiliki beberapa instalasi python yang tidak terikat pada apa pun dengan pengujian sebagai tujuan utamanya. Perhatikan, bahwa saya tidak berniat untuk menggunakan python launcher (hal py.exe itu), virtualenv atau "solusi" alternatif lainnya karena semuanya memungkinkan saya untuk menginstal hanya tunggal Penginstalan Python dengan versi yang sama dan / atau terlalu besar untuk penggunaan cepat.

Rupanya penginstal Python biasa sejak 3.5 memiliki MSI yang saya butuhkan sematkan dan mereka tidak akan keluar. Namun ada penginstal web untuk setiap Python dan dengan itu Anda dapat melakukan hal yang sama dan mendapatkan instalasi Python yang berfungsi:

python-3.5.0-webinstall.exe /layout <folder>

Ini mengunduh file MSI Release (bukan hanya Debug atau PDB). Kemudian Anda harus memfilter file yang memiliki akhiran _d.msi dan _pdb.msi, yang sepele dengan Batch sekarang dan Anda berakhir dengan struktur ini:

core.msi
dev.msi
doc.msi
exe.msi
launcher.msi
lib.msi
path.msi
pip.msi
python-3.6.0-webinstall.exe
tcltk.msi
test.msi
tools.msi

dan cukup banyak VS yang dapat didistribusikan ulang. Tergantung pada kebutuhan Anda, Anda mungkin ingin menghapus file .exe, .msu (redists), _d.msi, _pdb.msi di direktori kerja Anda. Sisanya adalah Release file dan file yang memperluas fungsionalitas interpreter seperti launcher (py.exe), path (mungkin hanya menempatkan python secara permanen ke PATH), dll.

Jika Anda menjalankan masing-masing dengan ini:

msiexec.exe /a <file> targetdir=<folder>

Anda mendapatkan instalasi Python portabel yang berfungsi. Perhatikan bahwa hal seperti itu tidak didukung secara resmi .
