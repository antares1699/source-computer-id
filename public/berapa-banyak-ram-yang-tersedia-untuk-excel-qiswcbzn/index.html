Excel 2010, 2013 dan 2016 tersedia dalam 2 versi: 32-bit (2 Gigabyte memori virtual)

Kehabisan Memori, Batas Memori, Kebocoran Memori, Excel tidak akan mulai.

Bagaimana saya bisa melihat berapa banyak RAM yang tersedia untuk Excel?

Task Manager akan menunjukkan berapa banyak memori yang digunakan Excel.

Di sisi lain, Office edisi 32-bit dibatasi hingga 2 GB ruang alamat virtual, dan ruang ini digunakan bersama oleh Excel, file buku kerja, dan add-in yang berjalan dalam proses yang sama. (Lembar kerja lebih kecil dari 2 GB pada disk mungkin masih berisi cukup data untuk menempati 2 GB atau lebih dari memori yang dapat dialamatkan.)

Selain itu,

Batasan 2-GB adalah per contoh proses Windows dari Excel. Kamu bisa menjalankan banyak file dalam satu contoh. Namun, jika file tersebut benar-benar besar dan harus terbuka, pertimbangkan untuk membuka beberapa instance untuk file file lainnya. Untuk informasi tentang batasan yang mungkin Anda temui, buka ke situs web berikut:

Penggunaan memori di edisi 32-bit Excel 2013 dan 2016

Selain itu,

Versi 32-bit dari Microsoft Excel 2013 dan Excel 2016 dapat digunakan memanfaatkan fungsionalitas Large Address Aware (LAA) setelahnya penginstalan pembaruan terkini. (lihat bagian "Resolusi") Perubahan ini memungkinkan penginstalan 32-bit Excel 2016 menggunakan file memori saat pengguna bekerja pada OS Windows 64-bit. Sistem menyediakan kemampuan ini dengan meningkatkan memori virtual mode pengguna dari 2 gigabyte (GB) hingga 4 GB. Perubahan ini memberikan memori 50 persen lebih banyak (misalnya, dari 2 GB hingga 3 GB) saat pengguna bekerja pada sistem 32-bit.

Selain itu,

Jika Anda menjalankan Windows 64-bit, perubahan ini diterapkan secara otomatis. Anda tidak perlu melakukan apa pun. Memori yang tersedia untuk proses Excel secara otomatis digandakan dari 2 GB menjadi 4 GB. Ini meningkatkan dukungan untuk tindakan yang menggunakan banyak memori.
