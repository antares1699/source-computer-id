Menguji kondisi perangkat keras laptop di linux [ditutup]

Saya akan membeli laptop bekas dan saya perlu memastikan apakah ada masalah. Ini akan menjalankan windows. Saya adalah pengguna linux, jadi saya ingin menjalankan beberapa tes dengan live cd. Saya dapat menguji di windows tetapi Linux lebih disukai.

Tolong, beri tahu saya beberapa perangkat lunak / cara untuk menguji setiap perangkat keras yang mungkin.

Tidak terlalu sulit untuk mendapatkan informasi diagnostik pada perangkat keras. Saya akan memberi Anda beberapa instruksi baris perintah yang seharusnya berfungsi pada sebagian besar distribusi Linux apa pun, namun Anda mungkin perlu mengunduh beberapa utilitas (jika Anda tidak memiliki akses internet, pertimbangkan untuk menyimpan beberapa paket pada live flash drive yang dapat Anda gunakan ).

Hal pertama yang saya periksa adalah hard drive tidak melaporkan masalah apa pun. Sebagian besar hard drive mendukung SMART untuk melaporkan masalah, yang dapat Anda periksa dengan berikut ini:

sudo smartctl -d ata -H /dev/sda

Ini harus memberikan sesuatu seperti

=== START OF READ SMART DATA SECTION ===
SMART overall-health self-assessment test result: PASSED

jika semuanya baik-baik saja. Smartctl mungkin diinstal pada distribusi langsung Anda secara default, tetapi tidak ada salahnya untuk memeriksanya. Penjelasan singkat dari opsinya: -d adalah tipe disknya, mungkin akan ata, dan -H adalah drive - ini akan berbeda berdasarkan jenis drive dan mungkin juga / dev / hda. Perhatikan bahwa smartctl bukanlah ukuran yang sempurna untuk kinerja hard drive dan Anda cenderung berisiko lebih tinggi mengalami kegagalan hard drive jika Anda membeli hard drive bekas.

Setelah itu Anda bisa mendapatkan beberapa informasi umum tentang perangkat keras. Untuk ini saya akan merekomendasikan lshw - yang mungkin tidak akan diinstal secara default. Jalankan saja perintah tanpa argumen seperti

sudo lshw

dan Anda akan mendapatkan informasi mendetail tentang semua perangkat keras di mesin. Saya sarankan melakukannya di terminal layar penuh. Beri perhatian khusus pada * -memori bagian untuk memastikan sebanyak yang dikatakannya. Anda juga bisa menggunakan

lspci

untuk daftar perangkat keras internal yang lebih ringkas.

Anda mungkin juga ingin memeriksa bagaimana kinerja baterai. Ini cukup sederhana dengan perintah acpi, yang disertakan beberapa distribusi secara default tetapi hal lain yang ingin Anda periksa ulang. Untuk melihat kapasitas desain baterai dan persentase jenis itu

acpi -i

Anda akan melihat persentase di akhir baris kedua, ini adalah kapasitas maksimum baterai saat ini dibandingkan dengan kapasitas yang dirancang (menurun seiring waktu, biasanya Anda dapat mengharapkan 98-99% untuk yang baru). Jika Anda memiliki persentase rendah di sana, Anda dapat mengharapkan masa pakai baterai minimal dan kemungkinan perlu menggantinya jika Anda bermaksud menggunakannya dengan baterai.

Jika sesuatu akan segera terjadi, masalah yang jelas mungkin itu adalah hard drive. RAM adalah hal lain yang harus diperhatikan.

Itu saja yang bisa saya pikirkan di luar kepala saya, mungkin menambahkan lebih banyak nanti.

Kamu bisa gunakan Memtest86 + untuk memastikan memori bekerja dengan benar. Ini biasanya ditemukan dibundel dengan LiveCD dari sebagian besar distribusi, atau LiveCD yang dibuat khusus untuk tugas-tugas pemulihan seperti Knoppix atau SystemRescueCD. Anda biasanya mengaksesnya dari layar boot setelah CD dimuat. Misalnya, pada LiveCD Ubuntu, ini diberi label sebagai "Memori uji".

Ada utilitas serupa untuk menguji stres CPU seperti x86test dan mprime.
