Mengapa dua program tidak dapat mengakses webcam saya secara bersamaan?

Saya pertama kali meluncurkan cheese dan webcam saya menyala. Saya kemudian menjalankan vlc untuk mengambil output /dev/video0 tetapi gagal dengan:

[0x7f3ea40012e8] v4l2 demux error: cannot set input 0: Device or resource busy
[0x7f3ea40012e8] v4l2 demux error: cannot set input 0: Device or resource busy
[0x7f3ea4002168] v4l2 access error: cannot set input 0: Device or resource busy
[0x7f3ea4002168] v4l2 access error: cannot set input 0: Device or resource busy
[0x7f3eb4000b78] main input error: open of `v4l2:///dev/video0' failed

Apa pun pasangan program video yang saya jalankan (skype, cheese, vlc, dll.), Hasilnya selalu sama: program kedua tidak dapat lagi menggunakan webcam ketika yang pertama sudah mengambil output.

Namun saya merasa penasaran seperti video4linux menyatakan :

Secara umum, perangkat V4L2 dapat dibuka lebih dari satu kali. Saat ini didukung oleh pengemudi, pengguna dapat misalnya memulai "panel" aplikasi untuk mengubah kontrol seperti kecerahan atau volume audio, sementara aplikasi lain menangkap video dan audio.

Kamera web saya terlihat pada lspci sebagai 058f:a014 Alcor Micro Corp. Asus Integrated Webcam, tetapi saya bahkan tidak tahu apa penyebab utamanya, jadi saya tidak dapat memeriksa apakah masalah saya terkait dengan pengemudi atau tidak. Setiap masukan akan sangat disambut!

Ini sama di Windows dan mungkin semua sistem operasi. Kamera web bukanlah perangkat input sederhana: untuk membaca bingkai darinya Anda perlu secara aktif mengirim perintah sesuai dengan beberapa protokol. Ini membuatnya menjadi stateful dan oleh karena itu aplikasi yang mengelola statusnya harus memiliki akses eksklusif ke sana.

Di windows Anda dapat menggunakan splitcam untuk menduplikasi feed. Di Linux, Anda dapat menggunakan WebcamStudio.
