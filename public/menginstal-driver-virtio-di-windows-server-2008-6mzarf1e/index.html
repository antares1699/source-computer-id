Menginstal driver VirtIO di Windows Server 2008

Kami menjalankan sistem Windows Server 2008 sebagai "tamu" di server virtual Linux-KVM (SLES11, dengan dukungan VirtIO). Kami mengalami masalah dengan kinerja sistem dan ini mungkin karena tidak menggunakan driver VirtIO. Saya tidak memiliki banyak pengalaman baik dengan KVM maupun VirtIO. Baru saja mendengarnya ini bisa menjadi alasan masalah kita.

Pertanyaan:

  • Contoh penginstalan yang baru saya temukan menjelaskan cara menginstal driver selama pengaturan Windows. Apakah mungkin untuk menginstal driver VirtIO nanti? Kami telah menjalankan perangkat lunak pada sistem itu dan ingin menghindari penginstalan ulang / penyiapan semua ini.

  • Saya sudah menemukan file halaman menjelaskan cara menandatangani driver, yang tampaknya diperlukan. Halaman tutorial yang bagus (petunjuk langkah demi langkah) akan menyenangkan. Apakah ada yang seperti itu di luar sana?

Prosesnya dijelaskan dalam artikel Mengonversi Mesin Virtual Windows Vista KVM ke Redhat VirtIO Drivers .

Anda harus mengunduh dari linux-kvm.com Driver Tamu Windows dalam gambar cdrom (iso). Versi saat ini ada di Link ini .

Berikut ini kutipan dari artikel tersebut (baca "Windows 2008" untuk "Vista"):

  • Pastikan Vista VM sudah diperbarui pada tambalan dan disk bebas dari kesalahan.
  • Pasang file .iso sebagai perangkat CD-ROM.

Sekarang Anda mungkin berpikir Anda dapat menggunakan 'Add Hardware Wizard' di sini dan menambahkan driver, tambahkan perangkat keras, dan jadilah baik. Aku melakukannya. Saya berakhir dengan disk yang tidak dapat di-boot. Rupanya deteksi otomatis Vista diperlukan dalam hal ini proses. Begitu…

  1. Tambahkan perangkat jaringan baru dengan jenis 'virtio'. Vista akan melakukan rutinitas "Anda memiliki perangkat keras" dan menjalankan Anda melalui semua penyihirnya. Ketika meminta Anda untuk driver, arahkan ke direktori i386 / 2008 di gambar disk driver. Ya, Ya, Oke, Ya, Sungguh, Lanjutkan, dll.
  2. Matikan VM dan hapus pengontrol ethernet lama. Boot Vista dan pastikan jaringan berfungsi. Anda bisa melewatkan ini langkah sekarang jika Anda ingin membuat pemecahan masalah menjadi lebih sulit.
  3. Tambahkan pengontrol Penyimpanan baru. Biarkan yang ada apa adanya untuk saat ini. Anda harus memilih gambar disk yang tidak Anda gunakan sekarang, atau buat file baru. Apa pun baik-baik saja, kami tidak akan pernah menggunakannya di dalam Vista. Lakukan tarian supir lagi.
  4. Matikan Windows. Hapus pengontrol penyimpanan, dan tambahkan yang baru, ketik 'virtio', dengan gambar hard drive normal Anda. Mengurus pengontrol ethernet lama di sini juga, jika Anda mengabaikan sebelumnya nasihat.
  5. Boot Windows secara normal. Ini sekarang akan muncul pada disk VirtIO dan driver jaringan. Jika Anda mendapatkan layar biru atau permintaan untuk menggunakan RepairCD, ada yang tidak beres. Gunakan CD perbaikan untuk memulihkan titik pemulihan sebelumnya dan coba lagi.

Saya tidak bisa menjelaskan secara detail, karena saya belum melakukannya sendiri ... Tapi, inilah yang sebenarnya dilakukan oleh seorang rekan kerja:

Anda sebenarnya dapat menginstal driver di bawah sistem yang sedang berjalan. Sebelumnya perangkat harus diubah dalam konfigurasi XML untuk perangkat "virtio" saat dimatikan. Anda tidak dapat menginstal driver tanpa perangkat yang sudah ada. Ini terutama bermasalah untuk hard drive ... karena windows tidak mau memulai, ketika hanya mengkonfigurasinya untuk virtio. Solusinya adalah dengan meletakkan "dummy" virtio-harddisk ke dalam konfigurasi (atau hanya harddisk yang sama dua kali sebagai virtio dan harddisk normal; pastikan untuk tidak menulis akses keduanya!) Dan kemudian instal driver dan kemudian ganti konfigurasinya untuk hard drive utama.

Dia mengatakan kepada saya ada "paket driver redhat untuk windows" atau sesuatu seperti itu ... ini mungkin bekerja tanpa penandatanganan (sudah ditandatangani)? Saya sebenarnya tidak menanyakan detailnya ... Perhatikan bahwa mungkin ada sumber yang berbeda untuk driver.

Harapan yang membantu orang lain ...
