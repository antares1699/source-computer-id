Paksa koneksi MS-CHAPv2 VPN dari Mac

Jika Anda mencoba menyambung dari Mac ke VPN (PPTP) Windows Server, hanya menerima MS-CHAPv2, klien gagal dengan kesalahan berikut:

18/6/13 9:06:08.672 AM pppd[10247]: MS-CHAP authentication failed: E=691 Authentication failure

Solusi yang diusulkan ( Posting dari Macworld , Posting dari forum Apple ) adalah menonaktifkan Digitally sign communications (always) dan mengaktifkannya hanya if client agrees (tidak diuji, karena kami tidak dapat mengubah pengaturan di server).

Seringkali, Anda tidak ingin atau tidak dapat mengubah pengaturan apa pun di server dan dengan demikian mencari solusi di sisi klien.

Adakah yang menemukan baris perintah untuk memaksa penggunaan MS-CHAPv2, atau, dengan kata lain, untuk menghalangi penggunaan metode otentikasi lainnya?

MacOSX memiliki pppd tertanam dan Anda dapat terhubung melalui Terminal. Untuk terhubung melalui pppd dan hanya memiliki subset jaringan yang dirutekan melalui vpn, saya menggunakan baris perintah berikut (multiline untuk keterbacaan):

sudo pppd 
   remoteaddress *vpn_server_address* 
   pty "*vpn_server_address* --nolaunchpppd" 
   lock 
   user *DOMAIN\\username* 
   password *password* 
   nomppe 
   require-mschap-v2 
   noauth 
   nobsdcomp 
   nodeflate 
   nodetach 
   plugin pptp.ppp 
   noipdefault 
   refuse-pap 
   refuse-eap 
   refuse-chap 
   refuse-mschap 
   plugin /System/Library/SystemConfiguration/PPPController.bundle/Contents/PlugIns/PPPDialogs.ppp  
   logfile /path/to/your/log.log

Kemudian, saya menyatakan di terminal lain rute yang dibutuhkan, misalnya

sudo route add -net *distant_local_network* *gateway_ip* *mask*

sebagai contoh

sudo route add -net 192.192.0.0 192.192.0.1 255.255.255.0

Kemudian semua lalu lintas ke jaringan yang jauh menggunakan VPN dan sisa Internet lainnya masih dapat diakses.
