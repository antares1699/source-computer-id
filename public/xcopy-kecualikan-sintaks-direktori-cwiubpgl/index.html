XCOPY / Kecualikan Sintaks Direktori

Saya mencoba menggunakan XCOPY untuk menyalin direktori yang terlihat seperti ini:

-MainFolder
 -ManyFoldersIWantToCopy
 -ManyFoldersIWantToCopy
 -...
 -Source Code

Saya ingin menggunakan opsi /EXCLUDE untuk menyalin setiap folder kecuali Source Code.

Saya mencoba menggunakan yang berikut ini, tetapi saya tidak beruntung:

xcopy "Source" "Destination" /EXCLUDE:"\Source Code\" /E /C /R /I /K /Y

Pesan kesalahan yang saya terima mengatakan: Can't read file: "\Source Code\".

Saya sudah mencoba menghapus tanda kutip ganda sekaligus, menggunakan tanda kutip tunggal, dengan / tanpa \, tetapi saya mendapatkan pesan kesalahan yang sama.

Saya yakin kesalahan ini membutuhkan penjelasan lebih lanjut. aku menemukan ini posting blog tentang masalah ini sangat informatif.

Pada dasarnya, Anda perlu menggunakan 8.3 lama (nama file pendek DOS di jalur pengecualian). Anda dapat menentukan nama pendek dengan menjalankan perintah "DIR / X" dari prompt perintah setelah mengubah direktori ke induk folder (Anda ingin nama 8.3 untuk itu). Sayangnya MS belum memperbarui perintah ini untuk mendukung nama file yang panjang.

Saya akan menyarankan menggunakan robocopy sebagai gantinya jika Anda tidak ingin menggunakan jalur pendek / nama file. Tentu saja saya rasa Anda bisa meletakkan nama panjang di eksternal "excluded_files.txt". Ini sepertinya didukung oleh XCopy.

Saya menggunakan flag xcopy EXCLUDE dengan sukses tanpa meletakkan garis di tanda kutip, meskipun banyak jalur berisi spasi. Jadi bagi saya (Win7 Home Premium SP1 perbaikan menginstal DOS prompt) kembali kereta adalah batasan yang memadai. Saya baru saja menggunakan file teks biasa dalam format:

file 1.txt
path 2\file 2.text
and so on\and so forth.txt

lalu EXCLUDE: F: \ filename.txt sebagai flag xcopy. Saya tidak menyertakan huruf drive di baris di file kecualikan (seperti yang dikonfirmasi di /? Halaman itu hanya perlu sebagian yang cocok). Itu melewatkan file bernama dengan sangat cepat. Saya menggabungkan ini dengan / D: 09-21-2014 untuk hanya mendapatkan file terbaru dan sejumlah tag lainnya dengan sukses.

Untuk mengonfirmasi, saya tidak menggunakan nama pendek DOS yang lama, bertentangan dengan temuan Ken Richard di atas.
