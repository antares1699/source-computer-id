Konfigurasi jaringan VirtualBox

Saya memiliki mesin virtual CentOS 6.1, divirtualisasikan dengan VirtualBox di Windows XP. Mesin ini menghosting server MySQL pengembangan lokal saya.

Saya ingin menemukan konfigurasi jaringan yang benar, sehingga:

  • mesin tamu saya selalu memiliki IP yang sama, dan dapat diakses dari host
  • mesin tamu saya selalu dapat menggunakan akses internet host (baik kabel maupun nirkabel)

Sejauh ini saya telah mencoba solusi yang berbeda ini:

  • NAT
    • kelebihan: menyediakan akses internet, apapun koneksinya
    • kontra: tidak menyediakan koneksi yang berfungsi dari tuan rumah ke tamu
  • Jembatan
    • kelebihan: menyediakan koneksi jaringan dua arah, menyediakan koneksi internet
    • kontra: hanya berfungsi dengan satu adaptor, saya harus mengganti pengaturan setiap kali parameter koneksi saya berubah
  • Jaringan Khusus Host
    • pro: menyediakan koneksi jaringan dua arah
    • kontra: tidak ada akses internet dari tamu

Ada ide yang lebih baik untuk mencapai tujuan saya?

Pertimbangkan untuk mengonfigurasi mesin dengan dua antarmuka, bukan satu.

Pertama, Anda dapat menggunakan Host-only Adapter sehingga tuan rumah Anda selalu dapat berbicara langsung dengan tamu. Kemudian konfigurasikan antarmuka kedua sebagai NAT atau Bridged Adapter tergantung pada kebutuhan spesifik Anda untuk aplikasi Anda.

Jika tamu Anda berjalan sebagai server maka Anda akan menginginkan Bridged. Untuk antarmuka yang berjalan dalam mode Bridged, itu akan bertindak seperti antarmuka biasa, jadi jika Anda menginginkan alamat IP statis, Anda harus mengkonfigurasi server DHCP jaringan Anda untuk menetapkan alamat IP yang sama ke alamat MAC virtual Anda. Antarmuka yang dijembatani, atau Anda harus mengatur antarmuka Bridged itu secara statis (tidak disarankan jika jaringan Anda menggunakan DHCP kecuali Anda mengoordinasikannya dengan admin jaringan lokal Anda ... dalam hal ini Anda lebih baik meminta alamat IP yang sama dari server DHCP).

Ingatlah bahwa jika Anda menggunakan DHCP untuk distribusi alamat IP, Anda harus menambahkan pengecualian di kumpulan DHCP untuk berjaga-jaga, sehingga alamat IP yang ditetapkan tidak akan tumpang tindih dengan alamat yang disewakan lainnya.
