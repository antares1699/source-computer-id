Windows 7 kehilangan zona waktu yang benar saat reboot

Saya memiliki PC standar yang menjalankan Windows 7 Ultimate (64-bit) . Untuk beberapa alasan, ia menolak untuk menjaga zona waktu yang benar (baterai BIOS baik-baik saja) saat dihidupkan ulang.

Catatan 1) : Zona waktu benar. Tab "Waktu Internet" juga menunjukkan "komputer ini disetel untuk menyinkronkan secara otomatis dengan 'waktu.windows.com'. Saat saya mengeklik tombol 'Ubah setelan ...', kotak centang 'Sinkronkan dengan server waktu Internet' diperiksa .

Namun, saat booting ulang, waktu akan berubah menjadi 6 jam ... dan tidak terkoreksi sendiri bahkan setelah menunggu berjam-jam hingga "sinkronisasi otomatis" ini terjadi.

Catatan 2) : Waktu BIOS diatur ke lokal (yaitu, bukan UTC). Ketika saya me-restart Windows 7 tanpa mem-boot ke OS lain yang diinstal di konfigurasi dual-boot (Ubuntu Linux), sepertinya saya mengingat waktu dengan benar. Ini mungkin menjelaskan segera waktu saat reboot, tetapi tidak menjelaskan mengapa Windows 7 biasa secara otomatis 'Menyinkronkan dengan server waktu Internet' bahkan setelah satu jam.

Mengapa ini terjadi dan bagaimana cara memperbaikinya?

Karena cara kedua sistem operasi mengatur jam perangkat keras. secara default ubuntu menggunakan UTC, dan windows localtime.

Jadi, saat Anda mematikan, jam perangkat keras Anda disetel ke "13:00". Ketika Anda boot, windows melihat "13:00" sebagai waktu lokal, jadi 1 PM, tetapi Ubuntu melihatnya sebagai UTC dan mengubah waktu kembali dari UTC ke waktu lokal.

Anda dapat memperbaikinya dengan meminta windows untuk mengatur jam perangkat keras dengan UTC, atau Ubuntu untuk menggunakan waktu lokal.

untuk membuat perubahan di Ubuntu, edit edit / etc / default / rcS dan ubah

UTC=yes hingga no, seperti ini:

# assume that the BIOS clock is set to UTC time (recommended)
UTC=no

Lihat:

https://help.ubuntu.com/community/UbuntuTime#Multiple_Boot_Systems_Time_Conflicts
