Beberapa perintah meminta dengan satu koneksi SSH?

Saya memiliki kebutuhan untuk melakukan banyak hal di vim pada server jarak jauh yang saya perlukan untuk melakukan SSH. Saya menemukan bahwa yang terbaik adalah jika saya dapat melihat banyak dokumen di beberapa jendela di desktop saya, jadi untuk melakukan ini saya membuka dua atau tiga sesi SSH ke server jarak jauh. Ini menjadi agak membosankan, karena saya harus memasukkan kembali un / pw beberapa kali, dan saya harus memastikan bahwa setiap jendela tetap cukup aktif untuk mencegah log-off otomatis.

Apakah ada cara untuk masuk ke SSH sekali dan kemudian memboncengnya dengan jendela terminal yang berbeda? (Saya menggunakan OSX dan Ubuntu secara dominan).

Satu alternatif lain yang memungkinkan adalah menggunakan sshfs.

sshfs memungkinkan Anda memasang direktori dari jarak jauh yang dapat diakses pada mesin jarak jauh melalui ssh, sebagai volume pada mesin Anda. Bekerja dengan baik. Manfaatnya di sini adalah Anda dapat bekerja dengan semua file di mesin jarak jauh seolah-olah file itu lokal. Secara keseluruhan, ini akan lebih mudah digunakan daripada layar - Anda masuk sekali, maka hal lain yang Anda lakukan dengan file jarak jauh akan sama seperti jika Anda bekerja dengan file lokal.

Alternatif lainnya adalah menggunakan kemampuan vim untuk mengedit file jarak jauh melalui ssh. Lihat disini: http://vim.wikia.com/wiki/Editing_remote_files_via_scp_in_vim

tmux juga merupakan opsi (mirip dengan GNU screen). Basis kodenya lebih kecil dan lebih bersih, ini mendukung panel berdampingan tanpa ada tambahan main-main, dan menurut saya itu jauh lebih menyenangkan untuk digunakan. Layar jauh lebih umum, jadi sebaiknya Anda belajar menggunakannya.

Untuk menggunakan tmux, cukup instal di mesin yang Anda inginkan untuk menggunakan SSH. Kemudian SSH masuk, jalankan tmux, dan Anda sekarang dapat menjalankan beberapa terminal atau aplikasi konsol pada layar yang sama atau di 'jendela' yang berbeda (mirip dengan ruang kerja Ubuntu) pada terminal yang sama. Coba man tmux untuk kontrol.

Dan opsi lainnya adalah plugin netrw bawaan vim untuk membuka file jarak jauh. Berfungsi baik dengan plugin Project.

Tetapi untuk dapat bekerja dengan apa pun di atas ssh, hal pertama yang saya lakukan adalah mendaftarkan kunci ssh saya: ssh-copy-id user@host untuk mendaftarkan kunci dengan host dan memeriksa ssh-agent sedang berjalan dan menambahkan kunci yang digunakan dengan ssh-add.

Saya sering melakukan hal-hal seperti:

diffsplit scp://user@$REMOTE/path/to/dir/%

yang membedakan file pada host ssh-copy-id user@host pada ~user/path/to/dir dengan nama relatif yang sama dengan file yang terbuka saat ini. Saya suka vim, meskipun saya yakin bisa juga di emacs.

Anda dapat menjalankan beberapa sesi SSH melalui satu koneksi SSH dengan membuat login pertama (yang mengatur koneksi) sebagai master kontrol dan meminta pemanggilan SSH berikutnya menghubungi master kontrol dan sesi mereka diteruskan melalui koneksi master yang sudah diautentikasi.

Lakukan ini dengan yang berikut di file konfigurasi Anda:

Host somehost
    # ~/tmp/sc must exist beforehand and should have mode 0700.
    ControlPath ~/tmp/sc/%h.%p
    ControlMaster auto

Lihat ssh_config(5) halaman manual untuk beberapa detail lebih lanjut tentang cara kerjanya.
