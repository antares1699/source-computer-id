Tangani nama file dengan spasi di dalam skrip Bash

Dalam skrip Bash saya, saya harus menangani nama file dengan spasi. Ini adalah baris penting di dalam skrip saya:

mp3file="/media/d/Music/zz_Hardcore/Sampler/Punk-O-Rama\ Vol.5\ \[MP3PRO\]/01\ -\ Nofx\ -\ Pump\ up\ the\ Valium.mp3"
echo "Command: mp3info -x `echo $mp3file`"
mp3info -x `echo $mp3file`

Sayangnya, perintah tersebut tidak berfungsi, karena nama file dipisahkan:

mp3info: invalid option -- '\'
mp3info: invalid option -- '\'
Error opening MP3: /media/d/Music/zz_Hardcore/Sampler/Punk-O-Rama\: No such file or directory
Error opening MP3: Vol.5\: No such file or directory
Error opening MP3: \[MP3PRO\]/01\: No such file or directory
Error opening MP3: Nofx\: No such file or directory
Error opening MP3: Pump\: No such file or directory
Error opening MP3: up\: No such file or directory
Error opening MP3: the\: No such file or directory
Error opening MP3: Valium.mp3: No such file or directory

Saya juga mencoba menambahkan IFS khusus saat saya membaca di beberapa forum:

SAVEIFS=$IFS
IFS=$(echo -en "\n\b")
# Script like above
IFS=$SAVEIFS

Tapi dengan cara ini, saya mendapatkan kesalahannya

Error opening MP3: /media/d/Music/zz_Hardcore/Sampler/Punk-O-Rama\ Vol.5\ \[MP3PRO\]/01\ -\ Nofx\ -\ Pump\ up\ the\ Valium.mp3: No such file or directory

Saya sudah mencoba cukup lama sekarang tetapi saya tidak bisa membuat skrip saya berfungsi. Yang aneh adalah jika saya menjalankan perintah yang sama yang harus dibuat skrip saya secara manual (menggemakannya di dalam skrip saya) di Shell, itu benar-benar berfungsi. Tapi tidak di dalam skripku. Ada petunjuk?

  • Saya tidak tahu apa yang harus dilakukan backspace di IFS (apalagi tentang backspace, saya baru menyadari itu bisa berupa karakter apa saja dan itu ada untuk mempertahankan baris baru yang jika tidak akan dijatuhkan), tetapi Anda dapat mengatur IFS tanpa menggunakan echo: IFS=$'\n'. Selain itu, di satu tempat Anda menggunakan backticks dan di tempat lain Anda menggunakan $(). Bentuk $() lebih disukai karena lebih mudah dibaca, mudah disarangkan, dll.

Bungkus nama variabel dalam tanda kutip:

mp3file="/media/music/Punk-O-Rama Vol.5 \[MP3PRO\]/01 - Nofx - Pump up the Valium.mp3"
echo "Command: mp3info -x \"$mp3file\""
mp3info -x "$mp3file"

Saya juga menghapus perintah echo yang berlebihan dan melarikan diri yang tidak perlu.
