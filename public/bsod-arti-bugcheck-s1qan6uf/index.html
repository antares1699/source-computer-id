BSOD - arti bugcheck?

Saat masuk ke Windows 7 hari ini, PC saya langsung BSOD. Menggunakan WhoCrashed saya mendapatkan laporan berikut:

-

  • Pada Tue 12.02.2013 13:56:20 GMT komputer Anda crash
  • file dump kecelakaan: C: \ Windows \ Minidump \ 021213-27390-01.dmp
  • uptime: 00:00:25
  • Ini mungkin disebabkan oleh modul berikut: ntoskrnl.exe (nt + 0x1AA698)
  • Kode bugcheck: 0x1000007E (0xFFFFFFFFC0000096, 0xFFFFF80003610698, 0xFFFFF8800614C7B8, 0xFFFFF8800614C010)
  • Kesalahan: SYSTEM_THREAD_EXCEPTION_NOT_HANDLED_M
  • jalur file: C: \ Windows \ system32 \ ntoskrnl.exe
  • produk: Sistem Operasi Microsoft® Windows®
  • perusahaan: perusahaan Microsoft
  • deskripsi: Kernel & Sistem NT
  • Deskripsi pemeriksaan bug: Ini menunjukkan bahwa utas sistem menghasilkan pengecualian yang tidak ditangkap oleh penangan kesalahan.

Ini tampaknya merupakan bug driver perangkat lunak yang khas dan tidak mungkin disebabkan oleh masalah perangkat keras. Kerusakan terjadi di kernel Windows. Mungkin masalah ini disebabkan oleh driver lain yang tidak dapat diidentifikasi saat ini.

-

Sekarang, PC saya macet / beku kadang-kadang dan pada tugas-tugas berat dengan kinerja tertentu di masa lalu, tetapi penyebabnya (saya pikir) adalah slot RAM yang cacat di motherboard saya. Menjaga slot tetap kosong menghentikan crash.

Hari ini, itu macet lagi, dan saya tidak mengubah apa pun yang terkait dengan perangkat keras.

Saya tahu saya bisa berkeliling Google membaca apa arti kode bugcheck ini, tetapi akhir-akhir ini saya menyadari bahwa pengalaman pribadi dari seseorang (dengan bugcheck / masalah yang sama) jauh lebih berguna, khususnya karena orang ini mungkin datang ke larutan.

Terima kasih banyak!

Dalam kasus ini, thread mengalami pengecualian

C0000096: STATUS_PRIVILEGED_INSTRUCTION
          Executing an instruction not allowed in current machine mode.

Kesalahan ini dimunculkan oleh CPU itu sendiri. Beberapa kode mencoba menjalankan instruksi yang tidak diizinkan untuk dilakukan. Kemungkinan besar ini disebabkan oleh kerusakan memori; di mana kode kernel mencoba mengeksekusi data sampah.

Jenis kesalahan ini benar-benar tidak mungkin untuk ditentukan. Terjadi kesalahan "inti" kode yang seharusnya tidak terjadi. Sangat tidak mungkin ada bug perangkat lunak dalam kode Microsoft mana pun; yaitu saat Anda mulai mencari di tempat lain.

  • Pengemudi Pihak Ketiga . Drive mode kernel memiliki akses penuh ke perangkat keras fisik. Setiap bug yang tersesat di driver pihak ketiga mana pun (mis. Video, suara, jaringan, USB 3.0, SATA) dan dapat merusak kode atau data apa pun di sistem. Langkah selanjutnya : coba hapus perangkat keras yang baru ditambahkan (sehingga beberapa driver pihak ketiga tidak dimuat), coba boot dalam mode aman (sehingga beberapa driver pihak ketiga tidak dimuat), atau instal ulang Windows (sehingga beberapa driver pihak ketiga tidak dimuat)
  • RAM buruk . Jika sedikit dibalik, dan itu mengubah instruksi yang sangat jinak menjadi instruksi yang berbeda, tidak valid, Anda bisa mendapatkan kesalahan ini. Langkah selanjutnya : Lepaskan RAM stick, pindahkan RAM ke slot lain, unclock RAM, ubah catu daya

  • Overclocking . Terkadang hal-hal luar biasa aneh bisa terjadi saat Anda melakukan overclock. Mudah-mudahan setiap orang mengirimkan Microsoft crash dumps mereka; karena Microsoft memang menyelidikinya. Kesalahan umum yang akan mereka dapatkan adalah ketika CPU menjalankan instruksi:

    xor eax, eax;

    Ini adalah operasi yang sangat sederhana yang dapat dijalankan oleh CPU; itu hanya mengatur register CPU internal EAX ke nol. Tidak ada cara itu bisa gagal; kecuali saat Anda melakukan overclock - atau masalah fisik lainnya.

tl; dr: Jika Anda telah menghilangkan perangkat lunak, maka perangkat kerasnya.

Memperbarui : Metodologi Pemecahan Masalah

saya ingin menyebutkan detail yang saya alami, hampir tanpa berpikir saat melihat kesalahan ini.

Yang pertama adalah kode bugcheck yang sebenarnya:

0x1000007E - SYSTEM_THREAD_EXCEPTION_NOT_HANDLED_M

Binging itu di Google memberikan halaman dokumentasi Microsoft

Pemeriksaan Bug 0x1000007E: SYSTEM_THREAD_EXCEPTION_NOT_HANDLED_M

Ini menunjukkan bahwa utas sistem menghasilkan pengecualian yang tidak ditangkap oleh penangan kesalahan.

Saya tahu, dari pengalaman menjadi pengembang, bahwa jika aplikasi saya (atau salah satu utasnya) mengalami "pengecualian" , dan saya tidak "menangani "pengecualian, Windows pada akhirnya akan menanganinya dengan mematikan aplikasi. Jika pengecualian yang tidak tertangani terjadi saat dalam mode kernel, OS tidak punya pilihan selain menanganinya dengan mematikan kernel. Yang saya minati adalah yang pengecualian sedang dilempar. saya diasumsikan (salah, ternyata) itu adalah "Pelanggaran akses" .

Saya tahu bahwa semua pemeriksaan bug disertai dengan empat parameter yang menjelaskan apa yang sebenarnya terjadi:

  • Parameter 1: 0xFFFFFFFFC0000096
  • Parameter 2: 0xFFFFF80003610698
  • Parameter 3: 0xFFFFF8800614C7B8
  • Parameter 4: 0xFFFFF8800614C010

Tapi apa artinya ini ?! Saat itulah kita kembali ke halaman dokumentasi, yang tidak mendeskripsikannya. Tetapi tidak mengatakan:

Pemeriksaan bug 0x1000007E memiliki arti dan parameter yang sama seperti pemeriksaan bug 0x7E (SYSTEM_THREAD_EXCEPTION_NOT_HANDLED).

Luar biasa. Dan halaman lain ini mendokumentasikan parameter:

Parameter SYSTEM_THREAD_EXCEPTION_NOT_HANDLED

Parameter berikut muncul di layar biru.

  • Parameter 1: Kode pengecualian yang tidak ditangani
  • Parameter 2: Alamat tempat terjadinya pengecualian
  • Parameter 3: Alamat catatan pengecualian
  • Parameter 4: Alamat rekaman konteks

Inilah yang saya inginkan, kode pengecualian yang tidak ditangani . Dalam kasus Anda itu adalah kode pengecualian:

0xFFFFFFFFC0000096

saya tahu, dari pengalaman, bahwa Anda menjalankan Windows 64-bit, karena kode itu panjangnya 64-bit. Sungguh saya hanya menginginkan 32-bit yang lebih rendah:

0xC0000096

Biasanya saya berharap menemukan kode kesalahan itu pada winerror.h di direktori pengembangan saya; tapi itu tidak ada. Butuh beberapa Binging, tetapi saya menemukan bahwa mencari:

winerror C0000096

membawa saya ke halaman di winehq.dll , yang menyatakan konstanta:

STATUS_PRIVILEGED_INSTRUCTION = 0xC0000096

Binging untuk hal yang konstan membuat saya a halaman dokumentasi Microsoft kanonik :

Pengecualian Perangkat Keras

0xFFFFFFFFC0000096 : Menjalankan instruksi tidak diperbolehkan dalam mode mesin saat ini.

Saya juga tahu bahwa pengecualian ini dilemparkan oleh CPU diri . saya tahu itu karena "Petunjuk Istimewa" berarti Anda mencoba menjalankan instruksi CPU, Anda tidak diizinkan. Saya juga dapat mengetahui hal ini karena halaman tersebut dipanggil Pengecualian Perangkat Keras .

Jadi kami pada intinya adalah beberapa kode sedang berjalan yang mencoba mengeksekusi instruksi CPU yang tidak seharusnya. Ada dua kemungkinan:

  • memori rusak; perangkat lunak tidak ditulis untuk mencoba mengeksekusi kode itu, tetapi itulah yang hanya berakhir di RAM
  • itu benar-benar perangkat lunak buggy, dan mencoba melakukan sesuatu yang tidak diizinkan.

Mengingat bahwa kode Microsoft terus-menerus diuji di lapangan di jutaan mesin setiap hari, kemungkinan besar:

  • menjadi masalah dengan perangkat keras Anda
  • bug di kode orang lain yang menyebabkan masalah

Bagaimanapun, itu tadi bagaimana saya mengerjakan pemeriksaan bug itu. Mungkin dengan mengetahui bagaimana saya menjalaninya, ini dapat membantu Anda saat berikutnya Anda melakukan pemeriksaan bug.
