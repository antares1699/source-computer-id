Ini berhasil untuk saya: Saya tidak memiliki disk OS yang tersedia.

HATI-HATI - Mengubah file sistem secara manual sangat berisiko - buang semua cara lain terlebih dahulu.

Dari tautan http://www.tipandtrick.net/2008/endless-reboot-loop- While-configuring-updates-stage-3-of-3-during-vista-sp1-installation/

Saya bisa melihat solusi yang mungkin adalah menghapus atau mengganti nama file C: \ windows \ winsxs \ pending.xml

Ketika saya mencoba dengan masuk ke Perbaikan Sistem dan memilih opsi boot prompt perintah, tampaknya membiarkan saya mengganti nama file ini menjadi sesuatu yang lain - tetapi di latar belakang Vista tidak mengubah file untuk sistem secara keseluruhan - hanya untuk login pengguna saya.

Oleh karena itu saya menghubungkan hard disk dari komputer yang sakit ke PC Vista yang berfungsi. Saya mem-boot PC yang berfungsi - dan kemudian dapat mengikuti instruksi di tautan berikut: www.vistaheads.com/forums/microsoft-public-windowsupdate/181710-re-error-number-80073712-a.html

yaitu

jalankan command prompt dengan "Run as Administrator".

Ketik perintah berikut:

takeown /f C:\Windows\winsxs\pending.xml

cacls C:\Windows\winsxs\pending.xml /G <YourUsername>:F

Saya kemudian dapat menyalin pending.xml sebagai cadangan.

lalu saya hapus file tersebut.

Saya menjalankan checkdisk dan mengembalikan hard disk ke komputernya. Kali ini saat boot, Vista tidak terjebak pada siklus reboot tanpa akhir.

Ide untuk menghapus pending.xml dari c: \ windows \ winsxs adalah ide yang bagus, tetapi Anda juga harus memindahkan / menghapus file lain di folder tersebut yang dimulai dengan pending.xml. dan mengizinkan pembaruan Windows untuk membangun pemeriksaan pembaruan baru yang segar, pemeriksaan pertama membutuhkan waktu beberapa saat tetapi biasanya dilanjutkan dari mana pembaruan dilakukan tanpa masalah.

Dalam kasus saya, saya menggunakan Hirens Boot CD untuk mem-boot Mini Windows XP agar saya dapat mengakses c: \ windows \ winsxs

Membuat folder di root bernama c: \ pendings dan memindahkan semua file yang dimulai dengan pending.xml ke folder itu, jika Anda adalah penggemar kotak DOS maka berikut ini adalah perbaikan cepat yang bagus ...

Dengan asumsi C: adalah tempat folder windows Anda berada ...

md c: \ pendings

pindahkan c: \ windows \ winsxs \ pending.xml *. * c: \ pendings

shutdown -r -t 0
