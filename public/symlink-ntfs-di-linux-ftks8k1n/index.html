Symlink NTFS di linux

Saya memiliki drive berformat NTFS (dari instalasi windows sebelumnya) dan berisi beberapa symlink (titik reparse),

Untuk beberapa alasan, saya tidak dapat melakukan apa pun dengan symlink ini: Saya tidak dapat pergi ke tempat mereka tertaut, saya bahkan tidak dapat menghapusnya.

Saya kira alasannya pasti karena tautan tersebut berisi C:\ sebagai bagian darinya.

Bagaimana saya bisa mengatasi masalah ini?

MEMPERBARUI:

Untuk memperjelas, tautan ini adalah persimpangan / titik reparse.

Misalnya, saya memiliki C:\progs menjadi symlink ke C:\Program Files

Jika saya mencoba untuk menghapusnya, saya mendapatkan:

$ rm /media/disk/progs/ -rf
rm: cannot remove directory `/media/disk/progs': Operation not supported

Saya sebenarnya menghapus /media/disk/Program Files/.

UPDATE2:

Sistem saya adalah linux mint (berbasis ubuntu)

Seperti yang dijelaskan di pertanyaan pada bagian dukungan NTFS-3G, menghapus Titik Reparse tidak sepenuhnya didukung oleh driver NTFG-3G. Anda mungkin harus menginstal driver Advanced NTFS-3G:

Mengapa saya mendapatkan pesan "Operasi tidak didukung" saat menghapus file?

Driver NTFS-3G 1.5012 telah memecahkan masalah ini, atau Anda mencoba menghapus NTFS Reparse Point yang belum sepenuhnya didukung di NTFS-3G Stabil, hanya di NTFS-3G tingkat lanjut versi pengemudi .
