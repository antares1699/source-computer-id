Kesalahan “Folder terkompresi (zip) tidak valid atau rusak” setelah mengunduh file zip - terkadang, tetapi tidak selalu

Saya memiliki arsip zip ~ 250Mb yang terletak di server HTTP Apache. Beberapa pengguna mengeluhkan "folder zip rusak" setelah mengunduh file langsung dari situs web (meskipun situs tersebut dibangun di Drupal tetapi menggunakan metode pengunduhan publik sehingga harus mengunduh langsung)

Pesan kesalahan ketika mereka mencoba membukanya di Windows adalah:

Folder terkompresi (zip) tidak valid atau rusak

Saya telah mencoba ini sendiri di Windows XP dan menggunakan Firefox 14 dan IE8. Terkadang mengunduh dan membuka dengan benar, terkadang saya mendapatkan kesalahan.

Pengguna lain menggunakan Windows 7 dan menggunakan Internet Explorer dan mendapatkan pesan ini setelah beberapa kali mencoba mengunduh.

Ketika saya mengunduh melalui FTP menggunakan Filezilla saya dapat membuka file zip tanpa kesalahan.

Karena unduhan FTP dan HTTP kadang-kadang berfungsi, maka saya berasumsi file di server tidak rusak tetapi sesuatu yang terjadi selama mengunduh dan mengunggah ulang file tidak akan memperbaiki apa pun (dan saya telah mencoba ini).

Saya membuat file zip menggunakan IZArc tetapi saya tidak begitu yakin metode pembuatannya relevan dalam kasus ini karena terkadang berfungsi.

Apa penyebab munculnya pesan error di atas?

Unduhan mungkin rusak atau terpotong (tidak lengkap). Periksa apakah ukuran file yang diunduh sama dengan ukuran di server.

Mengapa hal itu terjadi setiap saat dan tidak setiap saat?

Unduhan dapat terpotong karena koneksi terputus, yang merupakan sesuatu yang acak - itu mungkin terjadi dan mungkin tidak terjadi. Jika kualitas koneksi Anda lebih baik, kemungkinan hal itu tidak akan terjadi.

Apa yang dapat saya lakukan untuk memastikan file zip terbuka tanpa kesalahan di masa mendatang?

Berikan checksum untuk semua unduhan Anda, dan minta pengguna Anda untuk memverifikasinya. Namun, Anda tidak dapat benar-benar menghindari masalah jaringan. Anda mungkin ingin berbicara dengan tuan rumah Anda.

Satu hal yang dapat Anda lakukan untuk memecahkan masalah apa yang terjadi adalah menggunakan hash MD5 atau SHA-1 dari file zip tersebut. Saya berasumsi bahwa Anda memiliki zip asli yang tersedia untuk Anda secara offline.

Menggunakan Cygwin , Anda dapat menghitung hash SHA-1 atau MD5 dari file zip Anda lalu mempublikasikan hasilnya. Hash sangat keren karena dapat digunakan untuk memverifikasi ke tingkat bit bahwa satu file sama persis dengan yang lain. Sebagai contoh: Hash untuk file yang berisi "0123456789" adalah 3749f52bb326ae96782b42dc0a97b4c1. Hash untuk file yang berisi "1123456789" adalah b0b7fa225796be83e7fea1345c13d3dd.

Jika saya mengingat ASCII saya dengan benar, hanya ada sedikit perbedaan antara kedua string tetapi seperti yang Anda lihat, ada perbedaan besar dalam hash yang dihasilkan. Menerbitkan hash MD5 atau SHA-1 adalah hal yang sangat umum dilakukan di dunia pengembangan perangkat lunak karena ini adalah cara yang sangat baik untuk memastikan bahwa file Anda tidak rusak. Setelah memublikasikan hash MD5 / SHA-1, Anda dapat meminta pelanggan Anda mengambil hash dari zip untuk melihat apakah itu benar-benar rusak atau tidak.

Ada banyak utilitas di luar sana untuk Windows, Linux, dan Mac yang akan menghasilkan hash MD5 dan SHA-1. Saya menggunakan Cygwin karena gratis.
