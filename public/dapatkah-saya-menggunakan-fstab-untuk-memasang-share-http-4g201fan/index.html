Dapatkah saya menggunakan fstab untuk memasang share HTTP?

Saya biasanya memasang "drive" yang dimaksud di Windows dengan alamat http://1.2.3.4/sharename. Di Linux (Ubuntu 10.04), saya mencoba menambahkan entri berikut ke / etc / fstab tidak berhasil:

http://1.2.3.4/sharename /mnt/sharename cifs username=myusername,password=mypassword 0 0

Ketika saya mount /mnt/sharename, saya melihat kesalahan ini:

mount, wrong fs type, bad option, bad superblock on http://1.2.3.4/sharename

Bagaimana saya bisa memasang jalur ini di Ubuntu 10.04?

Mungkin tidak. fstab bukanlah sihir, ini hanya daftar sistem file yang akan dicoba dipasang oleh komputer Anda pada waktu mulai (dan waktu lain, tetapi waktu mulai biasanya di tempat yang dibaca). File dibaca, dan kemudian berbagai utilitas pemasangan digunakan, bergantung pada jenis sistem file. Dan CIFS sebagai protokol tidak terlalu dekat dengan protokol http. Ini seperti Anda menulis 'pasang kartrid Nintendo 64 ke XBox360' dan anggap itu akan berfungsi. Anda membutuhkan sesuatu yang benar-benar berbicara http dalam hal ini.

Apakah Anda benar-benar memerlukan sistem file melalui http, atau Anda hanya ingin membaca / menulis file di server web yang Anda kontrol? Jika ini yang terakhir, Anda perlu mencari tahu bagaimana file-file itu disajikan pada server file (cifs, nfs) dan mengkonfigurasinya.

Apakah Anda benar-benar ingin melayani melalui http? Itu tidak dirancang (meskipun dapat dipaksakan) sebagai protokol sistem file. Lihatlah ke WebDav, dan ada beberapa ( http://savannah.nongnu.org/projects/davfs2 ) driver yang benar untuk webdav.
