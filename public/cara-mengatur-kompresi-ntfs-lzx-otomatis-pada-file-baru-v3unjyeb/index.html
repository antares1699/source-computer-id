cara mengatur kompresi NTFS LZX otomatis pada file baru

Saya tahu dimungkinkan untuk mengompres file pada commandline dengan LZX:

COMPACT /C /S:E:\Backup /EXE:LZX

Tetapi saya ingin melakukannya SEBELUM file ditulis di disk.

Jika saya mengaktifkan kompresi pada folder root melalui Explorer, file akan ditulis dengan metode kompresi yang lemah, sementara saya ingin menggunakan LZX.

Tidak boleh

Itu tidak bekerja dengan cara yang sama, tidak diwariskan, Anda tidak dapat mengompres folder (sendiri), Anda tidak dapat mengubah (atau menambahkan) informasi ke file LZX sambil mempertahankan kompresi, jika Anda melakukannya, file tersebut adalah didekompresi seluruhnya, seperti file baru, dan Anda perlu mengompresnya kembali.

Bantuan ringkas memberi:

"Direktori akan ditandai sehingga file yang ditambahkan sesudahnya akan dikompresi kecuali / EXE"

Menambahkan file dekompresi byte:

echo a >> test.txt

File LZX seperti "file zip" (lebih mirip GZ) dibuat transparan untuk OS, tetapi tidak sama dengan file terkompresi umum dengan cara NTFS lama.

Solusi yang saya usulkan adalah menggunakan tugas terjadwal untuk menjalankan perintah kompak saat komputer dalam keadaan idle (atau dijalankan setelah pencadangan), dan saya juga merekomendasikan untuk hanya memadatkan dengan file LZX: dll dan exe dalam program kerja, (jika mereka bukan cadangan sebagai contoh Anda), untuk mencegah potensi perlambatan dan masalah (jangan! kompres file boot di direktori root).

Jika file Anda sudah dikompresi dengan metode lama, Anda perlu menambahkan / F untuk memaksa kompresi ulang saat pertama kali Anda melakukannya, dan menghapus kompresi dari folder itu sendiri untuk mencegah "metode lemah" digunakan, atau buat program yang lebih kompleks untuk menemukan dan mengompres ulang file terkompresi yang lemah.

Ingat juga bahwa LZX tidak kompatibel dengan versi windows sebelumnya.
