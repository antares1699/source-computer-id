Gunakan dua antarmuka jaringan secara bersamaan

Saya sering menggunakan Wi-Fi dari beberapa jaringan publik yang tidak mengizinkan beberapa protokol (seperti torrent atau IRC - selanjutnya saya hanya akan berbicara tentang IRC). Jika saya ingin menggunakan salah satu protokol tersebut, saya harus menggunakan koneksi 3G ponsel saya (melalui antarmuka usb0).

Saya ingin tetap terhubung ke Wi-Fi untuk navigasi web saya (jadi gunakan antarmuka klasik wlan0), tetapi gunakan antarmuka lain (di sini usb0) untuk beberapa perangkat lunak (seringkali hanya untuk weechat, klien CLI IRC).

Jadi saya punya dua pertanyaan (terkait):

  • bagaimana cara terhubung secara bersamaan ke dua antarmuka tanpa semua mengacaukan semuanya?
  • Bagaimana cara mengkonfigurasi satu perangkat lunak untuk menggunakan antarmuka, dan perangkat lunak lain untuk menggunakan antarmuka lainnya? Karena weechat adalah perangkat lunak CLI, mungkin saya hanya perlu mendeklarasikan variabel lingkungan tertentu; tetapi bagaimana cara memastikan bahwa Firefox tidak menggunakan usb0 dan bukan wlan0?

Solusi paling sederhana adalah memiliki dua tabel perutean terpisah; salah satunya (wlan0, dalam kasus Anda) akan menjadi default, sementara kita dapat menggunakan trik rapi untuk memaksa aplikasi tertentu menggunakan tabel non-default.

Pertama, buat dua tabel (gantikan NAME1 dengan nama yang masuk akal bagi Anda, hal yang sama dengan IP1, DEV1, dan seterusnya):

echo 201 NAME2 >> /etc/iproute2/rt_tables

Tambahkan gateway ke setiap tabel perutean (jika perlu):

ip route add NET1 dev DEV1 src SRC1 table main
ip route add NET2 dev DEV2 src SRC2 table NAME2

Kemudian rute default:

ip route add default via IP1 table main
ip route add default via IP2 table NAME2

Kemudian aturan untuk memilih tabel rute berdasarkan alamat sumber:

ip rule add from IP1 table main
ip rule add from IP2 table NAME2

Sekarang kita perlu memaksa weechat untuk menggunakan IP2. Kita melakukannya dengan membangun modul bind baru . Ini sepele untuk melakukannya, ada instruksi sederhana dalam referensi. Setelah Anda menyusun modul dan menempatkannya di /usr/lib, Anda mengeluarkan perintah:

 BIND_ADDR=IP2 LD_PRELOAD=/usr/lib/bind.so weechat

dan kamu selesai. Keuntungannya adalah dapat dengan mudah dibuat skrip, dan dapat diulangi untuk aplikasi sebanyak yang Anda suka.

Jika Anda menentukan dua sub-jaringan, misalnya:

  • 192.168.1.x / 255.255.255.0 - wlan0
  • 192.168.2.x / 255.255.255.0 - usb0

Jika Anda menetapkan gateway hanya untuk wlan0, semua lalu lintas jaringan di samping yang dialamatkan langsung ke 192.168.2.x melewati wlan0.

Anda dapat merutekan lalu lintas jaringan ke alamat IP tertentu melalui antarmuka usb0 Anda. Anda juga bisa mengarahkan lalu lintas melalui antarmuka tertentu untuk proses di linux .
