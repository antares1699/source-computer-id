Berapa banyak informasi yang disertakan dengan tanda tangan GPG?

Katakanlah Anda memverifikasi tanda tangan seseorang dengan GPG:

$ gpg --decrypt statement.txt.sig 
"I am not a crook."
gpg: Signature made 17 Nov 1973 9:31:50 AM CDT using RSA key ID 92861D99
gpg: Good signature from "Richard Nixon <rnixon@whitehouse.gov>"
gpg:                 aka "[jpeg image of size 19022]"

File statement.txt.sig jelas menyertakan pernyataan / data yang ditandatangani serta kunci publik dari orang yang menandatanganinya. Namun, ada lebih banyak informasi disertakan dalam keluaran, seperti nama lengkap dan alamat email, serta gambar JPEG yang disertakan dengan kunci publik penandatangan.

Dari mana GPG mendapatkan data tambahan itu? Apakah itu termasuk dalam file statement.txt.sig, atau apakah itu melihat keyring Anda dan "mengisi" semua detail tambahan yang Anda miliki tentang orang itu?

Tidak, informasi tersebut berasal dari sertifikat yang disimpan di key ring publik Anda (atau diunduh dari server kunci). Tanda tangan hanya berisi beberapa informasi yang diperlukan untuk menemukan sertifikat yang tepat.

Gunakan gpg --list-packets atau pgpdump:

$ echo Foo. > foo.txt
$ gpg --detach-sign foo.txt
$ gpg --list-packets < foo.txt.sig
:signature packet: algo 1, keyid D24F6CB2C1B52632
    version 4, created 1410762670, md5len 0, sigclass 0x00
    digest algo 10, begin of digest 25 58
    hashed subpkt 2 len 4 (sig created 2014-09-15)
    subpkt 16 len 8 (issuer key ID D24F6CB2C1B52632)
    data: [4091 bits]

Tanda tangan yang terlepas memiliki satu paket "tanda tangan", yang berisi algoritme, stempel waktu penandatanganan, dan ID kunci 8-byte penanda tangan, yang digunakan untuk mencari sertifikat penanda tangan di keyring Anda. (Jika Anda tidak memiliki kunci publik dengan keyid tersebut, GnuPG akan mencoba menemukannya di server kunci publik.)

(Saya tidak tahu mengapa ia memiliki ID kunci dua kali.)

GnuPG memperoleh segala sesuatu tentang penanda tangan dengan mencari keyid - sebenarnya kunci publik untuk keduanya memverifikasi tanda tangan, serta bidang userid (nama, alamat, foto) yang digunakan untuk mendeskripsikan penanda tangan.

Selain: Perhatikan bahwa contoh Anda memiliki keyid 4-byte yang pendek, yang sangat buruk karena ada beberapa ratus tabrakan short-keyid yang diketahui, baik disengaja maupun tidak disengaja. Menggunakan keyid-format long di ~ / .gnupg / gnupg.conf Anda akan membuatnya menampilkan keyid yang lebih panjang, meskipun mereka masih mudah bertabrakan dengan sengaja, jadi Anda harus selalu periksa sidik jari saat mengimpor kunci baru.

(Namun, paket tanda tangan selalu menyimpan keyid 8-byte di dalamnya. Beberapa orang mengonfigurasi GnuPG untuk meletakkan sidik jari di bidang khusus (notasi), tetapi sayangnya itu tidak digunakan oleh perangkat lunak itu sendiri.)

Kembali ke tanda tangan. Jika Anda melakukan gpg --list-packets dengan tanda tangan biasa (sebaris), Anda akan melihat beberapa hal lagi:

:compressed packet: algo=1
:onepass_sig packet: keyid D24F6CB2C1B52632
    version 3, sigclass 0x00, digest 10, pubkey 1, last=1
:literal data packet:
    mode b (62), created 1410762587, name="",
    raw data: 5 bytes
:signature packet: algo 1, keyid D24F6CB2C1B52632
    version 4, created 1410762587, md5len 0, sigclass 0x00
    digest algo 10, begin of digest eb 31
    hashed subpkt 2 len 4 (sig created 2014-09-15)
    subpkt 16 len 8 (issuer key ID D24F6CB2C1B52632)
    data: [4095 bits]

Pesan yang ditandatangani sebenarnya ada dalam paket "data literal", biasanya dikompresi menggunakan DEFLATE (pgpdump akan menunjukkan algoritme sebenarnya yang sedang digunakan).

Ini didahului oleh paket "onepass_sig", yang tujuan satu-satunya adalah menyediakan keyid tanpa harus membaca sampai akhir seluruh pesan - sehingga GnuPG dapat mulai mencari keyid dan melanjutkan dengan memverifikasi pesan sekaligus. (Saat membaca dari pipa, misalnya cat|gpg, tidak mungkin untuk mencari ke depan dan ke belakang; semuanya harus dibaca dalam satu kali jalan.)

Jika mau, Anda juga dapat melakukan ini dengan kunci penanda tangan (sertifikat). Cukup ekspor ke file terlebih dahulu:

# gpg --export D24F6CB2C1B52632 | gpg --list-packets
:public key packet:
    version 4, algo 1, created 1256993643, expires 0
    pkey[0]: [4096 bits]
    pkey[1]: [17 bits]
    keyid: D24F6CB2C1B52632
:user ID packet: "Mantas Mikul\xc4\x97nas "
:signature packet: algo 1, keyid D24F6CB2C1B52632
    version 4, created 1256993643, md5len 0, sigclass 0x13
    digest algo 10, begin of digest 5a e2
[many more lines]

Sertifikat juga terdiri dari banyak paket, dimulai dengan kunci publik, ID pengguna (label teks) dengan tanda tangan sendiri untuk masing-masing (untuk melindungi seseorang yang melampirkan ID pengguna palsu ke kunci pub), lalu beberapa paket subkunci publik (sekali lagi dengan tanda tangan mandiri) ).
