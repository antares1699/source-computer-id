Bagaimana cara memeriksa versi Ubuntu secara manual? (misalnya dari hard drive)

Ada cara cepat untuk memeriksa versi sistem ubuntu:

$ lsb_release -a
No LSB modules are available.
Distributor ID: Ubuntu
Description:    Ubuntu 11.10
Release:        11.10
Codename:       oneiric

Tetapi file apa saja yang menyimpan informasi ini dan bagaimana saya dapat mengaksesnya? Secara khusus, saya memiliki partisi lama dengan Linux mati tergeletak di sana dan saya ingin memeriksa versi Ubuntu-nya. lsb_release -a menunjukkan versi Linux saya saat ini saja ...

Anda dapat mencoba melakukan chroot ke sistem itu dari Linux atau LiveCD lain. Anggaplah distro Anda yang mati ada di sda2. Pertama, pasang partisi:

$ mount /dev/sda2 /mnt

Sekarang ikat folder dengan binari penting untuk menggantikan yang mungkin mati. mount -B memasang satu direktori di atas yang lain secara virtual mengganti isinya, tetapi tanpa menyentuhnya secara fisik:

$ mount -B /bin /mnt/bin
$ mount -B /sbin /mnt/sbin
$ mount -B /usr/bin /mnt/usr/bin

Sekarang chroot ke distro lama. chroot akan berpura-pura bahwa direktori yang diberikan adalah / (jadi dalam kasus kami, kami akan memiliki shell yang berjalan di distro offline, mengerjakan file-nya kecuali untuk binari yang telah kami jilid sebelumnya)

$ chroot /mnt

Sekarang perintah apa pun yang Anda keluarkan akan berfungsi seolah-olah partisi itu adalah root sistem file Anda. lsb_release -a akan dieksekusi dari binari Anda, tetapi akan membaca dari file distro yang sudah mati.

Anda dapat menggunakan chroot untuk banyak hal lainnya. Misalnya Anda dapat memutakhirkan Ubuntu tanpa mem-boot-nya: cukup boot LiveCD, pasang partisi Ubuntu, ikat direktori penting seperti /proc dll. (JANGAN lakukan binari - ini akan memperbarui binari LiveCD) dan menjalankan do-release-upgrade di shell chroot.

ubuntu@ubuntu:/media/ubuntu/Boot18.04$ cat etc/lsb-release
DISTRIB_ID=Ubuntu

cat: etc/os-release: No such file or directory

ubuntu@ubuntu:/media/ubuntu/Boot18.04$ cat '/media/ubuntu/Boot18.04/etc/issue' 
Ubuntu 12.04 LTS \n \l

ubuntu@ubuntu:/media/ubuntu/Boot18.04$ cat '/media/ubuntu/Boot18.04/etc/issue.net' 
Ubuntu 12.04 LTS

tidak konsisten dengan versi boot sebenarnya yang pasti 18.04. "sesuatu". Apakah mungkin untuk secara pasti menentukan versi terinstal yang sebenarnya dengan mudah ? (terutama mengingat bahwa volume tersebut tidak lagi boot)
