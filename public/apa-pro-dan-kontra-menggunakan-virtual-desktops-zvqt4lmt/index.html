Apa Pro dan Kontra menggunakan Virtual Desktops?

Saya selalu menyukai gagasan tentang desktop virtual , tapi tidak pernah bisa merangkul mereka sepenuhnya. Setiap kali saya mencoba mengadopsi desktop virtual, saya cenderung menemukan peralihan konteks terlalu menggelegar dibandingkan dengan paradigma desktop tunggal. Saya pikir saya akan membawa ini ke komunitas.

  2. Bagaimana Anda mengatur desktop virtual Anda?
  3. Apakah Anda menggunakan desktop virtual untuk mengganti alt + tab di antara aplikasi?

Kelebihan: Manajemen visual yang mudah untuk tugas dan aplikasi Anda tanpa mengacaukan bilah tugas Anda. Meningkatkan produktivitas karena Anda dapat menyimpan semua item terkait tanpa harus menyaring berbagai jendela.

Kekurangan: Mudah bingung tentang apa yang sedang berjalan dan apa yang tidak karena mungkin bersembunyi di desktop virtual. Anda juga mungkin ingin memiliki aplikasi "lengket" (seperti pemutar musik Anda) yang tersedia di semua desktop yang mungkin / mungkin tidak tersedia. Dapat berpotensi memperlambat sistem Anda jika pengalih desktop bermasalah atau Anda memiliki terlalu banyak hal yang sedang berjalan.

Saya menjalankan 4 desktop:

  1. Desktop Utama: Digunakan untuk pekerjaan pengembangan utama saya (aplikasi web). Saya juga menggunakan pengaturan monitor ganda sehingga saya dapat memiliki Visual Studio di satu jendela dan browser atau editor DB di jendela lainnya.

  2. Desktop Penjelajahan Web: Buka sepanjang hari dengan blog berita saya, stack overflow, dan sejenisnya. Memungkinkan untuk "istirahat mental" saat saya membaca tentang berbagai hal karena saya tidak terganggu oleh kueri database yang sedang berjalan atau pembuatan aplikasi.

  3. Mail Desktop: Saya harus memantau dua akun di tempat kerja, jadi saya meninggalkan desktop lengkap untuk itu.

  4. Desktop "Sekunder": Terkadang ada email yang meminta saya melakukan sesuatu, atau rekan kerja memiliki permintaan, dan lebih mudah bagi saya untuk beralih ke desktop sekunder dan melakukan tugas yang diperlukan tanpa mengganggu pekerjaan saya yang lain.

Jangan gunakan untuk menggantikan ALT + Tabbing, karena saya akan memiliki banyak aplikasi di desktop tertentu.

Satu hal penting yang perlu diperhatikan tentang beradaptasi dengan lingkungan, memiliki hotkey atau metode cepat untuk memanipulasi desktop sangat penting. Saya menggunakan WIN + Numpad untuk beralih antara desktop dan WIN + Number Row untuk memindahkan jendela ke desktop tertentu. Jika tidak ada cara untuk berinteraksi dengan desktop dengan cepat, akan terlalu merepotkan untuk memindahkan mouse Anda ke bilah tugas / baki sistem / "zona panas" untuk bekerja dengan desktop dan itu mengalahkan tujuan.

Virtual Desktop pilihan (bukan dalam pertanyaan tapi saya pikir saya akan memasukkannya): Dexpot

Anehnya, saya tidak menggunakan desktop virtual di rumah, meskipun saya melakukan banyak hobi komputasi di sampingnya. 8 ^ D

Saya menggunakannya untuk tugas, dan saya banyak menggunakannya. Misalnya, di tempat kerja, saat ini saya memiliki dua belas (ditugaskan ke alt-f1 hingga alt-f12). Saya akan memiliki lebih banyak, kecuali saya kehabisan kunci-F.

  1. Saat ini, sedang menulis beberapa dokumentasi untuk sebuah proyek perangkat lunak. LyX, beberapa xterms, satu atau dua gvim, dll.
  2. Menulis kode untuk proyek tersebut, di bagasi. Beberapa xterms, banyak gvim, beberapa sesi layar.
  3. Proyek yang berbeda, tidak melakukan banyak pekerjaan aktif yang satu ini saat ini. Terkadang perlu melakukan tugas impor / ekspor, atau tugas kecil lainnya, jadi senang melakukannya. xterms dengan layar yang berjalan di dalamnya, juga beberapa xtail untuk melihat log.
  4. Cabang penerapan berikutnya dari proyek perangkat lunak pada 2.
  5. apapun (gunakan saat saya membutuhkan desktop untuk sesuatu yang lain)
  6. Masa bodo
  7. Masa bodo
  8. bugzilla
  9. bacula monitor / config (kebanyakan xterms, beberapa dengan sesi layar)
  10. database (xterms menjalankan klien baris perintah mysql, juga beberapa psql juga)
  11. email (icedove, mutt, dan xterms dengan log email lokal)
  12. misc (mis., penjelajahan web acak)

Di rumah, saya hanya memiliki enam desktop (ini adalah pengaturan monitor ganda, tidak seperti pekerjaan, jadi saya membutuhkan lebih sedikit), dan hanya dua di antaranya yang hanya digunakan sekali (satu untuk menangani video, yang lainnya untuk menangani gambar) . Satu untuk umum apa pun, dan tiga lainnya untuk proyek apa pun yang kebetulan sedang saya kerjakan.

Semua pengaturan dengan beberapa xterm, sesi layar, dll. Dimulai sepenuhnya secara otomatis (termasuk memposisikan jendela) menggunakan skrip shell.

Pengaturan ini mungkin membuat saya benar-benar gila.

saya menggunakan wm yang luar biasa , yang membawa metafora desktop virtual ke level berikutnya. Alih-alih desktop virtual Anda memiliki apa yang disebut "tag". Setiap tag dapat dilampirkan ke beberapa jendela dan Anda dapat menampilkan sejumlah tag pada saat yang bersamaan. Ini dapat dikontrol sepenuhnya oleh keyboard, atau Anda juga dapat menggunakan mouse. Itu dapat menyusun jendela yang Anda tampilkan untuk memaksimalkan penggunaan ruang layar. Wm mengagumkan sangat mudah dikonfigurasi. Versi 3 bahkan hadir dengan integrasi Lua, yang memberi Anda lebih banyak kemampuan penyesuaian.

Saya sekarang menggunakan pengaturan default 9 tag (dapat dipilih dengan super + 1-9), dan dapat diisi dengan mudah. Biasanya saya memiliki tag terpisah untuk windows (satu untuk terminal, satu untuk browser, satu untuk klien email, satu untuk terminal jarak jauh, satu untuk emacs dll ...), dan saya dapat dengan mudah memilih tag baru untuk ditampilkan dengan menggunakan super + ctrl + 1-9.

Itu tidak datang dengan animasi yang bagus seperti compiz, tetapi itu benar-benar meningkatkan efisiensi Anda.
