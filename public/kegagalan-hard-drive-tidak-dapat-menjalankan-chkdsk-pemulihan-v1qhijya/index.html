Kegagalan Hard Drive? Tidak dapat menjalankan CHKDSK. Pemulihan?

Saya memiliki drive eksternal WD My Book Essential 3TB selama lebih dari 2 tahun sekarang. Itu bekerja dengan baik sampai sekitar sebulan yang lalu ketika itu berhenti dikenali. Setelah googling ekstensif, saya menjalankan chkdsk / r di atasnya dan itu baik-baik saja. Lain kali saya menggunakannya, sekitar 2 minggu setelahnya, lagi-lagi tidak dikenali. Saya tidak dapat menggunakannya sampai kemarin. Setelah sekian lama, laptop saya bisa mengenalinya. Saya mencoba menyalin beberapa file saya ke hard drive lain tetapi kecepatan transfernya turun menjadi sekitar 700 kb / s. Karena saya memiliki setidaknya 450 GB file penting (ini tidak termasuk file 900+ GB lainnya), kecepatan transfer ini tidak akan berfungsi.

Saya menyalakan komputer saya sekarang dan ini adalah hasil dari Manajemen Disk:

Mencoba mengakses Drive F meskipun hampir tidak mungkin. Saya pergi ke command prompt untuk mengakses file tetapi saya hanya dapat mengakses direktori root. Saya tidak bisa pergi ke subdirektori. Menyegarkan Manajemen Disk, drive saya sudah hilang - Tidak Diketahui, Tidak Diinisialisasi.

Lucunya, tidak seperti semua waktu sebelumnya, saya dapat melihat perjalanan di penjelajah: (meskipun saya hanya dapat mengakses direktori root). Tetapi drive tidak diketahui dalam Manajemen.

Saya mencoba menjalankan chkdsk dari command prompt administrator tetapi terus berkata "Tidak dapat membuka volume untuk akses langsung."

Saya sudah mencoba menarik drive SATA 3,5 "dari enklosur dan menghubungkannya ke PC desktop, menukar hard drive dengan drive eksternal WD Essential lainnya (meskipun modelnya berbeda, 1TB) - yang semuanya tidak berfungsi mungkin karena enkripsi perangkat keras .

Saya benar-benar perlu memulihkan file-file itu sebelum saya mendapatkan enklosur / dok lain untuk memformat hard drive ini. Aku kehabisan rambut untuk dicabut. Seseorang tolong bantu.

Anda tidak menyebutkan banyak gejala, jadi sulit untuk mengatakan apa masalahnya.

Pertama-tama, "kecepatan transfer ini tidak akan berhasil" tidak relevan. Saat drive sedang sekarat setiap saat Anda memiliki akses ke sana harus digunakan untuk menyalin data Anda . Anggaplah diri Anda beruntung karena itu menjadi hidup sekali lagi alih-alih mengeluh tentang kecepatan transfer. Biarkan aktif, jangan reboot, dan terus menyalin data Anda dari itu, bahkan jika itu membutuhkan waktu berminggu-minggu. Itu hal pertama yang harus Anda lakukan. Bahkan menjalankan chkdsk adalah ide yang buruk.

Jika drive Anda tidak mengeluarkan bunyi klik, saya curiga permukaan drive. Prosedur yang saya rekomendasikan dalam kasus ini adalah:

1) Jangan mematikan / reboot lebih dari yang diperlukan. Jika titik buruk berada di sektor kritis di awal disk, sistem mungkin tidak bisa boot lagi. Proses penulisan saat mematikan / reboot juga dapat memperburuk situasi.

2) Hindari menulis ke disk. Menariknya keluar dari komputer Anda dan meletakkannya di komputer lain sebagai disk non-utama adalah cara yang baik untuk mencegah perangkat lunak (OS) secara otomatis menulis padanya. Anda sudah mencobanya.

3) Secara opsional, cobalah untuk membuat gambar dari seluruh disk atau dengan satu atau lain cara menarik semua data penting dari disk. Ada banyak variasi tentang cara melakukannya, tetapi metode apa pun yang Anda pilih, hanya baca! . Anda sedang melakukan itu sekarang.

4) Jalankan perangkat lunak yang akan memeriksa dan memperbaiki bad sector.

Dalam urutan ketelitian yang meningkat (yang umumnya juga menunjukkan waktu yang dibutuhkan untuk menjalankannya) saya sarankan:

Lari Seatools untuk DOS (bukan versi Windows; unduh ISO), dan pilih 'LONG Test'. Program ini gratis.

Lari HDD Regenerator . Program ini tidak gratis ($ 99 AS).

Lari SpinRite (menjalankan executable di bawah Windows akan mengekstrak ISO yang dapat Anda bakar). Pilih level 4 untuk pengujian menyeluruh. Program ini tidak gratis ($ 89 AS) dan merupakan satu-satunya yang saya tahu yang sering kali dapat memulihkan data dari sektor yang rusak; ini mungkin membutuhkan waktu lama. Itu datang dengan jaminan uang kembali.

Ketiga program memindai hard disk pada tingkat fisik, dan harus dijalankan dari CD yang dapat di-boot. Mereka tidak peduli sistem file OS apa yang digunakan.

5) Terakhir, jalankan perangkat lunak yang melakukan pemeriksaan logis pada struktur disk, seperti chkdsk. Ini bisa sangat baik tidak temukan kesalahan apa pun setelah langkah 4.

Ditambahkan 21 feb 2014:

1) Orang sering memiliki kecenderungan untuk melakukan langkah 5 sebelum 4: chkdsk.dll sudah dekat jadi mengapa tidak menggunakannya dengan cepat. Ini tidak optimal. Jika sektor disk buruk dan ini menyebabkan struktur alokasi file rusak, alat seperti chkdsk akan melakukan yang terbaik untuk memperbaiki logis struktur file. Apa yang mereka perbaiki di sini tidak dapat diubah. Mereka umumnya tidak dapat memulihkan semuanya. Jika kamu pertama lakukan langkah 4 dan ini hanya masalah sektor fisik yang rusak, program semacam itu mungkin dapat menghidupkan kembali atau mengalokasikan kembali sektor yang buruk dan menyimpan datanya. Jika itu terjadi, OS tidak akan lagi memiliki masalah dengan struktur alokasi file dan semua struktur data logis yang terkait akan baik-baik saja (itulah mengapa saya menulis "Ini tidak akan menemukan kesalahan apa pun").

2) Anda mungkin tidak memiliki disk cadangan atau mungkin tidak dapat membaca dari disk yang rusak untuk melakukan langkah 3. Tidak masalah, langkah 3 hanyalah tindakan pencegahan umum: "Sebelum Anda mulai melakukan sesuatu yang tidak dapat diubah (langkah 4 dan 5 ) mencoba untuk menghilangkan data sebanyak mungkin ".

Tapi, baik MS & Linux tidak suka mentransfer folder besar. Harus mengambil paling banyak 4 ~ 300mb video sekaligus jika tidak mereka rusak.
