Program (tugas) autorun dengan hak istimewa berhenti sendiri

Ada program yang saya ingin dijalankan secara otomatis saat permulaan, tetapi karena program ini membutuhkan hak istimewa yang lebih tinggi, UAC mengganggu itu berjalan otomatis.

Saya menggunakan Metode Penjadwal Tugas mengatasi masalah ini; Saya membuat tugas untuk itu, lalu membuat pintasan untuk menjalankan tugas, mengaturnya untuk diminimalkan, dan meletakkannya di Memulai folder di Menu Mulai .

Ini bekerja seperti yang diharapkan untuk sebagian besar; program berjalan secara otomatis saat startup tanpa prompt dan meminimalkan ke tray.

Namun ada masalah: tampaknya menghilang sesekali, tampaknya tanpa alasan. Mengapa‽

(Sebenarnya, saya telah melakukan ini untuk dua program; satu menghilang dan yang lainnya membuat crash-dump.)

Masalahnya mungkin karena pengaturan tugas default dan lebih sering muncul di laptop.

Ternyata saat Anda membuat tugas, beberapa pengaturan defaultnya adalah tidak yang Anda inginkan dan akan menyebabkan masalah untuk program yang dijalankan secara otomatis, jadi Anda harus memastikan untuk memeriksa semua pengaturan saat menggunakan solusi Penjadwal-Tugas.

Dalam Pengaturan tab (gambar 1), pastikan untuk menonaktifkan Hentikan tugas jika berjalan lebih lama dari… pengaturan. Membiarkan pengaturan ini (yang defaultnya tiga hari) akan menyebabkan program berjalan otomatis berakhir setelah beberapa hari, jadi jika Anda menggunakan siaga atau hibernasi alih-alih mematikan (umum dengan laptop), maka Anda dapat menemukan program telah berhenti berjalan tiga hari setelah Anda melakukan booting terakhir.

Dalam Kondisi tab (gambar 2), pastikan untuk menonaktifkan semua opsi di Kekuasaan bagian. Pengaturan default menyebabkan program tidak dapat dijalankan secara otomatis jika Anda menggunakan baterai saat booting. Selain itu, default membuatnya sehingga meskipun Anda terhubung saat boot dan program berjalan, segera setelah Anda mencabut kabel dan beralih ke baterai, Windows hentikan itu .

(Itu Bangunkan komputer untuk menjalankan tugas ini opsi dinonaktifkan secara default, dan Anda harus membiarkannya seperti itu karena program yang dijalankan otomatis tidak akan membutuhkan ini.)

Peringatan

Satu hal yang perlu diperhatikan adalah jika Anda mematikan pengaturan Start the only if the computer is on AC power, maka itu akan menonaktifkan pengaturan Stop if the computer switches to battery power (gambar 3). Implikasi yang jelas adalah bahwa terlepas dari apa yang ditampilkan kotak centang untuk pengaturan terakhir, jika Anda telah menonaktifkan yang sebelumnya, maka tugas tidak akan berakhir saat beralih ke daya baterai. Ini masuk akal dan hampir semua orang mengharapkan hal itu terjadi, terlebih lagi, begitulah perilaku semua program yang ditulis dengan benar. Sayangnya ada bug di Windows di mana Penjadwal Tugas mengikuti kotak centang kedua terlepas dari yang pertama. Oleh karena itu, jika Anda menonaktifkan pengaturan pertama tetapi yang kedua memiliki tanda centang, maka beralihlah ke daya baterai akan hentikan tugasnya. Untuk menghentikan ini, Anda harus mengaktifkan opsi pertama, menonaktifkan yang kedua, lalu menonaktifkan lagi yang pertama.
