Perlu hard reset router Dlink seminggu sekali

Saya mengalami Dlink DIR 655 router di rumah. Sekali seminggu secara acak, saya melihat koneksi saya menjadi lemah dan rusak. Saya harus pergi dan melakukan hard-reset pada router (matikan dan nyalakan), sehingga koneksi nirkabel saya dapat digunakan lagi. Mungkinkah masalah ini terjadi pada produk router yang buruk atau dapatkah saya melakukan sesuatu untuk memiliki koneksi nirkabel yang andal? Jika Anda menghadapi masalah ini, bagaimana Anda mengatasinya?

Jika sudah memperbarui firmware Anda dan tidak ada perbaikan, Anda mungkin ingin mengembalikannya, atau, jika sudah keluar dari masa garansi / pengembalian, periksa di halaman administrasi browser. Harus ada tombol untuk mengatur ulang router. Jika menekan tombol itu memperbaiki masalah dengan mengatur ulang router, maka Anda akan memiliki cara untuk mengotomatiskannya.

Saya mengalami masalah serupa dan cara saya menyelesaikannya seperti ini: Saya menggunakan alat pengembang di dalam Firefox (Anda juga dapat menggunakan Wireshark atau sebagian besar browser lain) untuk memeriksa data POST yang dikirim saat Anda mengekliknya. Setelah Anda memilikinya, Anda dapat dengan mudah mereplikasi ini dengan menggunakan cronjobs / tugas terjadwal dan curl.

Saya menggunakannya di server Windows untuk berjalan sekali sehari dan pada dasarnya terlihat seperti ini:

curl.exe --head -d "POST_DATA_HERE" username:pass@192.168.0.1/apply.cgi

Jelas Anda harus menemukan apa sebenarnya (jika ada) yang dikirim ke router Anda, dan apakah pengaturan ulang panel kontrol bahkan cukup.

Periksa ventilasi di sekitar router: pastikan mendapatkan sirkulasi udara yang cukup agar tidak terlalu panas. Banyak perangkat jaringan beroperasi di tepi batas termalnya dan yang diperlukan hanyalah sedikit debu, ditutupi oleh kertas, atau didorong ke bawah rak untuk mendorongnya ke tepi.

Perilaku buruk setelah berlari selama beberapa hari (atau bahkan hanya beberapa jam), diikuti oleh perilaku yang baik setelah memulai ulang adalah gejala klasik kerusakan panas. Ini juga merupakan gejala router buggy, jadi periksa juga firmware-nya. Sayangnya, setelah komponen rusak karena panas, komponen tersebut mungkin tidak pulih bahkan dengan pendinginan. Jadi, Anda mungkin perlu menggantinya. Pastikan semua perangkat jaringan Anda berventilasi baik.

Sejujurnya, DLink tidak memiliki reputasi yang baik dalam membangun produk berkualitas tinggi. Namun demikian saya telah melihat kasus bahwa NAT di router menyebabkan kebocoran memori dan penurunan kinerja dari waktu ke waktu.

Terkadang pembaruan firmware dapat memperbaikinya, terkadang lebih baik membeli produk yang berbeda.

Hal-hal semacam ini sulit untuk didiagnosis jika Anda tidak memiliki akses penuh ke firmware dan / atau router, tetapi jika Anda mendapatkan masalah setelah banyak lalu lintas yang menggunakan klien torrent (untuk menyebutkan contoh klasik) Anda mungkin mengalami masalah itu. . (Klien torrent membuat banyak koneksi ke banyak port berbeda dan karenanya menghabiskan banyak sumber daya router.)

Pastikan router tidak terlalu panas, pastikan Anda tidak memiliki tetangga yang melakukan transmisi pada saluran nirkabel yang sama seperti yang Anda lakukan (gunakan program seperti Insidder untuk memeriksanya.) Dan dengan semua ini Anda mungkin dapat mengetahui apa itu menyebabkan masalah khusus Anda.

Semoga berhasil dan semoga ini membantu!
