Itu tergantung distro-nya.

Debian dan kerabatnya menggunakan

     lsb_release -a
     cat /etc/lsb-release

salah satu harus baik-baik saja. Namun, meskipun ini terkait dengan LSB (Linux Standard Base), tidak semua distro memilikinya. Jika Anda tidak mendapatkan balasan dari perintah di atas, Anda harus mencoba

      ls /etc/*release

dan kemudian lihat ke dalam file mana pun yang Anda temukan. Anda harus berhati-hati dengan ini: sementara RedHat memang memiliki / etc / redhat-release, yang lain, seperti Arch Linux, memiliki kosong / etc / arch-release file, dan file yang berisi info yang Anda cari adalah / etc / os-release .

Jadi, YMMV.

Pada Fedora dan Debian modern (dan Arch, CentOS, CoreOS, Mageia, openSUSE, Red Hat Enterprise Linux, SUSE Linux Enterprise Server, Ubuntu, dan lainnya), cara termudah adalah dengan melihat /etc/os-release atau menjalankan hostnamectl status.

Dalam skrip, Anda bisa melakukannya

#!/bin/sh
source /etc/os-release
echo $PRETTY_NAME

Sebagai contoh:

$ source /etc/os-release
$ echo $PRETTY_NAME
Fedora 25 (Workstation Edition)

...

$ source /etc/os-release 
$ echo $PRETTY_NAME
Debian GNU/Linux 8 (jessie)

Tentu saja, jika Anda perlu mengidentifikasi versi lama atau juga distro non-systemd, Anda dapat memulai dengan pendekatan ini jika /etc/os-release ada, dan jika tidak, kembali mencari lsb_release, /etc/redhat-release, mencoba mencari tahu. dari /proc/version, atau heuristik lainnya.

Lihat Mendeteksi nama dan versi distribusi Linux tentang Stack Overflow Documentation untuk lebih lanjut.
