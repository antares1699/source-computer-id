mengembangkan Windows secara efisien di dalam Linux

Saya perlu memvirtualisasikan Windows di laptop Linux saya. Awalnya saya coba QEMU tapi ternyata terlalu terkelupas. Kemudian saya menggunakan Server VMWare untuk sementara dan berpikir itu cukup bagus. Baru-baru ini saya mencoba VirtualBox dan merasa lebih responsif daripada VMWare.

Menurut pengalaman Anda, apa yang terbaik untuk memvirtualisasikan Windows di dalam Linux? Saya mencari solusi gratis yang efisien dan terintegrasi dengan baik dengan desktop Linux.

Saya menjalankan "mesin" Windows yang bekerja di rumah di dalam VMWare Player terbaru di bawah Ubuntu (saya bukan dual boot hanya untuk menjalankan administrivia) dan sama sekali tidak ada masalah dengannya.

VMWare telah menginvestasikan banyak upaya untuk menjadikan ini solusi yang mulus. Versi sebelumnya Anda harus mengkompilasi ulang modul secara manual ketika Anda mengubah kernel tetapi tidak lebih. Semuanya terjadi secara otomatis.

Saya telah lama bermain dengan QEMU dan VirtualBox tetapi saya mendapat kesan bahwa mereka adalah produk yang belum selesai ketika ditempatkan di samping VMWare.

Dan semuanya bebas bir, yang terutama menjadi perhatian saya (bukan bebas bicara).

Tapi ini benar-benar pertanyaan superuser karena tidak benar-benar terkait dengan pemrograman jadi saya mendorongnya ke sana.

Tidak yakin mengapa Anda menemukan QEMU bersisik. Mungkin jika Anda menjelaskan beberapa masalah QEMU spesifik Anda, Anda bisa mendapatkan saran di sini.

Baik KVM dan VirtualBox seperti yang disarankan oleh orang lain juga didasarkan pada QEMU. Saya juga menemukan QEMU sebagai solusi virtualisasi tujuan umum yang sangat baik. Saya telah menggunakannya untuk memvirtualisasikan server untuk sistem komersial lawas sehingga dapat dijalankan di dalam perangkat keras modern karena perangkat keras lama tidak lagi tersedia.

sybreon — Sumber
  • KVM didasarkan pada QEMU tetapi tidak sama. KVM menyediakan 'virtualisasi asli' yang berarti dapat berinteraksi langsung dengan prosesor yang mendukung virtualisasi sedangkan QEMU menggunakan 'paravirtualisasi' yang berarti mengemulasi arsitektur prosesor murni dalam perangkat lunak. Karena itu, KVM memiliki keunggulan kinerja yang jelas sedangkan QEMU adalah platform hebat untuk menjalankan misi kritis berdasarkan arsitektur prosesor yang lebih lama pada sistem yang lebih baru. Peran QEMU dalam KVM (dan mungkin VirtualBox) adalah mendukung fasilitas virtualisasi perangkat keras.
    — Evan Plaice
