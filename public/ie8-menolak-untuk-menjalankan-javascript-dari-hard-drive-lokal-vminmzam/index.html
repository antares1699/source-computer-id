IE8 Menolak untuk menjalankan Javascript dari Hard Drive Lokal

Saya memiliki masalah yang baru saja mulai bekerja baru-baru ini dan manajer jaringan yakin dia tidak mengubah apa pun dengan kebijakan grup. Bagaimanapun, berikut adalah penjelasan rinci tentang masalahnya.

Mesin saya adalah Windows XP SP3, dan saya menggunakan IE8 untuk menjelajah. Kami memiliki perangkat lunak anti-virus McAffee yang tidak dapat saya konfigurasikan. Saya menggunakan file berikut untuk menguji ...

<!DOCTYPE html>
<html>
  <head>
    <title>Javascript Test</title>
  </head>
  <body>
    <script type="text/javascript">
      document.write("<h1>PASS</h1>");
    </script>
    <noscript>
      <h1>FAIL</h1>
    </noscript>
  </body>
</html>

Ketika saya membuka file ini dari drive C :, gagal setiap saat . Jika saya menjalankannya di tempat lain (server web lokal / jarak jauh atau pada drive jaringan yang dipetakan), ini berfungsi dengan baik. Ketika saya hanya menjelajah Internet, Javascript di situs web berfungsi dengan baik. ini hanya gagal pada file yang dijalankan dari drive C: saya. Selain itu, saya memiliki beberapa programmer lain di departemen yang mencoba file ini di drive C: mereka, dan itu berfungsi dengan baik untuk mereka. Jadi saya tidak percaya ini adalah kebijakan kelompok.

Saya perlu memperbaikinya karena saya melakukan pengujian ekstensif dari drive C: saya, dan saya sudah terbiasa melakukannya. Saya tidak ingin terbiasa memindahkan file ke drive lain hanya untuk mengujinya.

Hal-hal yang telah saya coba:

  • Mengaktifkan "Izinkan Konten Aktif untuk Menjalankan File di Komputer Saya" di Opsi | Lanjutan | Keamanan
  • Mengaktifkan "Izinkan Skrip Aktif" di Opsi | Keamanan | Tingkat Kustom
  • Memverifikasi bahwa "Skrip" tidak dicentang sebagai dinonaktifkan di Bilah Alat Pengembang
  • Menambahkan localhost ke Situs Tepercaya di Opsi
  • McAffee dinonaktifkan sepenuhnya (untuk sementara, dengan bantuan dari admin jaringan)
  • Menggunakan DOCTYPE lama di halaman HTML pengujian saya
  • Instal ulang IE8 sepenuhnya
  • Menjalankan regsvr32 di JScript.dll
  • Keyboard dibanting

Saya yakin ada pengaturan di suatu tempat yang akan memperbaiki masalah ini, mungkin di registri. Saya tidak akan terkejut jika itu terkait dengan toolbar pengembang. Pada titik ini saya tidak tahu harus mencari ke mana lagi.

Adakah yang bisa membantu saya menyelesaikan masalah ini?

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Internet Settings\Zones\n

Di mana n adalah indeks zona, di mana 0 berarti My Computer. Zona ini tidak tersedia di Opsi Internet. Tetapkan Nilai 1400 ke 0 untuk mengaktifkan Javascript.

Sudahkah Anda mencoba menambahkan "Mark of the Web" di sumber HTML Anda?

Tambahkan saja sesuatu seperti:

<!DOCTYPE html>
<!-- saved from url=(0021)http://www.google.com -->
<html>

Tanda Web berada setelah tag DOCTYPE, sendiri di baris terpisah, tepat sebelum <html>.

Lihat dokumentasi MSDN di Mark of the Web untuk bantuan penuh.

Anda dapat mencoba memverifikasi penginstalan IE8 dengan menggunakan alat Microsoft yang terdapat di:
Cara mengatasi masalah penginstalan Internet Explorer 8 .

Artikel ini berisi tombol Fix It yang melakukan pemecahan masalah umum untuk IE8.

Saya mengutip:

Masalah saat menginstal Internet Explorer 8 dapat disebabkan oleh perbedaan masalah. Itulah mengapa ada beberapa metode pemecahan masalah yang Anda bisa gunakan untuk mencoba menyelesaikan masalah Anda.

Untuk memperbaiki masalah ini secara otomatis, klik tautan Perbaiki masalah ini. Kemudian klik Jalankan dalam dialog Unduh Berkas kotak, dan ikuti langkah-langkah di sini Penyihir.

Meskipun ini segera menyelesaikan masalah Anda, Anda tetap harus melakukan boot ulang dan memeriksa lagi. Jika masalah telah kembali, maka ini mungkin masalah GPO.

Selain itu, saya menemukan utas menarik yang mungkin berlaku:
Setelah pembaruan IE8- Buka Peringatan Keamanan File untuk semua eksekusi lokal .

Saya mengutip:

Setelah menghabiskan banyak waktu dengan file registri, internet, dan beberapa lainnya alkohol (hanya bercanda), saya rasa saya menemukan sumber masalahnya. :) Saya Saya cukup yakin bahwa ini adalah sumbernya dari itu, karena saya dapat mereproduksi file masalah dengan mengubah nilai a kunci registri tertentu bolak-balik. Ingin tahu? :) Oke, ini dia:

Kunci registri berikut telah diidentifikasi menyebabkan masalah: [HKEY_CURRENT_USER \ Software \ Policies \ Microsoft \ Windows \ CurrentVersion \ Internet Pengaturan \ Lockdown_Zones \ 0]

Menurut KB182569 ( http://support.microsoft.com/kb/182569 ) kunci ini bertanggung jawab untuk: "Miscellaneous: Meluncurkan aplikasi dan file yang tidak aman. "

Artikel yang sama juga menjelaskan: "Catatan Kecuali dinyatakan lain, setiap DWORD nilainya sama dengan nol, satu, atau tiga. Biasanya, pengaturan nol menetapkan a tindakan spesifik yang diizinkan, a pengaturan salah satu menyebabkan prompt untuk muncul, dan pengaturan tiga melarang tindakan tertentu. "

Saya menemukan bahwa jika saya menyetel dword "1806" nilai menjadi "0" dan kemudian gunakan tugas manajer untuk menghentikan tugas "penjelajah" dan mulai lagi, masalahnya ada lenyap. Jika saya mengubah nilainya kembali ke "1" dan membunuh / memulai penjelajah lagi, lalu masalahnya kembali.

Sekarang saya mulai mencari sistem saya menerapkan GPO dan menemukan salah satunya mengatur nilai ini menjadi "1" yang artinya meminta pengguna. Ini diatur dalam GPO di bawah [User Konfigurasi / Administrasi Template / Komponen Windows / Internet Kontrol Explorer / Internet Panel / Halaman Keamanan / Kunci-Bawah Lokal Program Zona Mesin / Lauching dan file tidak aman].

Seperti saya katakan, ini adalah sumber saya masalah dan sekarang saya tahu cara memperbaikinya. Tapi yang aneh bagi saya adalah file GPO yang sama berlaku untuk mesin IE7 dan masalahnya tidak muncul di sana. Menurut deskripsi GPO, fitur ini didukung sejak IE7:

Meluncurkan program dan tidak aman filesSetting Path:

Konfigurasi Pengguna / Administratif Template / Komponen Windows / Internet Kontrol Explorer / Internet Panel / Halaman Keamanan / Lokal Terkunci Zona MesinDidukung Pada: Setidaknya Penjelasan Internet Explorer 7.0 pengaturan kebijakan mengontrol apakah atau tidak "Buka File - Peringatan Keamanan" prompt ditampilkan saat meluncurkan file yang dapat dieksekusi atau file tidak aman lainnya.

Misalnya, jika pengguna meluncurkan dieksekusi dari berbagi file Intranet menggunakan Windows Explorer, pengaturan ini mengontrol apakah prompt adalah atau tidak ditampilkan sebelum file dibuka.

Jika Anda mengaktifkan pengaturan kebijakan ini dan kotak dropdown diatur ke Aktifkan, file akan terbuka tanpa keamanan cepat. Jika kotak dropdown diatur ke Prompt, prompt keamanan akan ditampilkan sebelum membuka file.

Jika Anda menonaktifkan pengaturan kebijakan ini, file tidak akan dibuka.

Jika Anda tidak mengkonfigurasi kebijakan ini pengaturan, pengguna dapat mengkonfigurasi perilaku yang cepat. Secara default, eksekusi diblokir di Zona Terbatas, diaktifkan di Intranet dan Lokal Zona komputer, dan setel ke prompt Internet dan zona Tepercaya.

Jadi berdasarkan deskripsi dari fitur dan apa yang saya telah mengalami, saya telah datang ke kesimpulan bahwa IE7 sedang menangani ini fitur tidak benar dan IE8 melakukannya akhirnya menanganinya dengan benar. Ini akan menjelaskan mengapa pembaruan ke IE8 menyebabkan fitur tersebut muncul.
