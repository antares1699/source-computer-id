SCM mana yang kurang intensif disk? (GIT vs SVN vs lainnya)

Saya ingin tahu SCM pouplar mana (terutama saya tertarik dengan GIT, SVN) kurang intensif disk.

Masalahnya adalah, saya akan mendapatkan SSDisk (SSD)) dan mereka memiliki siklus penulisan yang terbatas. Dan saya berpikir tentang bahwa SCM memelihara banyak file dan oleh karena itu penggunaan yang intensif dari mereka dapat menyebabkan umur disk yang lebih pendek (:

Menurut saya, GIT lebih intensif disk karena mempertahankan riwayat lengkap dan beralih dari versi ke versi jauh lebih mudah.

Sementara Git bekerja dengan riwayat lengkap, Git terutama akan melakukan akses baca untuk menghitung perbedaan dengan riwayat tersebut.

Jika Anda khawatir dengan akses tulis pada SDD:

  • menginisialisasi repo DVCS Anda pada stik USB
  • simpan di logoff pengembangan Anda saat ini dengan membuat bundel di SSD lokal Anda (hanya satu file yang dibuat / diperbarui) di file Direktori Dropbox .

Dengan cara itu, Anda akan melakukannya Betulkah minimalkan operasi tulis tersebut;)

Karena git memampatkan file sesuai permintaan (versi file yang lebih lama tidak digunakan di head) dan menyimpan hash / konten yang sama hanya sekali, tidak akan ada banyak pemborosan ruang atau siklus tulis / hapus dibandingkan dengan sistem lain seperti svn.

Mengenakan blok pegangan leveling terhapus dengan cukup baik, pastikan Anda memiliki cukup ruang kosong sebagai cadangan di drive Anda.
