Mengapa mengubah kata sandi truecrypt membutuhkan waktu lama?

Saya mengubah kata sandi wadah file truecrypt. Ini membutuhkan waktu sekitar 1 menit. Mengapa?

time truecrypt --text --change /tmp/user1.tc --keyfiles= --new-keyfiles= --password=known --new-password=known --random-source=/dev/null"

Jika saya menggunakan strace, saya melihat bahwa itu pada dasarnya tidak melakukan apa-apa: ia hanya membaca banyak data acak dari / dev / urandom (bahkan jika saya menentukan / dev / null sebagai sumber acak) dan akhirnya mengubah kata sandi:

open("/dev/urandom", O_RDONLY)          = 6
read(6, "\36&{\351\212\212\343\202\34\313\242\312I\326\235\245\224\300\354O)\270Q\200 \201J\227\224\311_\212\367"..., 640) = 640
close(6)                                = 0

Truecrypt membuat kunci baru untuk dikaitkan dengan frasa sandi baru. Untuk mencapai sifat acak untuk menghasilkan kunci aman, truecrypt mengambil sampel dari banyak sumber berbeda dalam waktu singkat (1 menit). Ini diperlukan untuk kunci aman baru. Menggunakan / dev / null tidak direkomendasikan! Semoga ini menyelesaikannya.

Saya berhasil menggunakan ini untuk membuat sumber acak:

         dd if=/dev/urandom of=/root/rn bs=1b count=320

Saya memilih 320 karena truecrypt meminta 320 karakter acak jika Anda tidak menentukan sumber acak.

/dev/urandom agak lebih baik dari /dev/zero

truecrypt --text --change --keyfiles= --new-keyfiles= --password=pw --new-password=newpw --random-source=/root/rn /dev/xvgf1
