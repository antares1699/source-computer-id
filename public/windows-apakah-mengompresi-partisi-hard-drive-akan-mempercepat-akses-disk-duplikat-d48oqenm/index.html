Windows: Apakah mengompresi partisi hard drive akan mempercepat akses disk? [duplikat]

Kemungkinan Duplikat:
Bagaimana kompresi NTFS mempengaruhi kinerja?

Secara teori, pada PC yang cukup cepat, sebagian besar prosesor menganggur selama akses hard drive.

File terkompresi membutuhkan lebih sedikit ruang fisik, sehingga hard drive (menjadi bagian paling lambat dalam sistem) harus membaca lebih sedikit data fisik.

Orang dapat berargumen bahwa sedikit tambahan pekerjaan bagi prosesor untuk membuka kompresi data dengan cepat tidak akan mengakibatkan perlambatan operasi pembacaan, jadi oleh karena itu pertanyaan saya:

Jika saya mengaktifkan kompresi drive pada mesin Windows, dapatkah saya secara positif memengaruhi throughput data dengan mengorbankan penggunaan CPU yang agak lebih tinggi?

Ini sangat bergantung pada berapa biayanya melalui waktu untuk mengambil dua sektor disk vs mengambil satu sektor dan mendekompresi menjadi dua sektor data.

Pada HDD, penundaan terbesar terkait dengan revolusi disk, sehingga mendefrag disk (atau file tunggal) akan sangat mempercepat pembacaan sekuensial.

Untuk SSD , tidak ada penundaan pencarian, jadi defragmenting memiliki efek yang lebih kecil (tetapi tetap membuat petunjuk ke bagian file lebih kompak dalam MFT , sehingga membutuhkan satu bacaan ekstra untuk fragmen vs per seluruh file).

Contoh: Saya lari OpenBSD dari CF. kartu dan kernel berukuran 20 MB, yang dibaca dalam 10 detik. Saya kompres dan menjadi seperti 6 MB, dan membaca dalam 3 detik, satu detik lagi untuk dekompresi. Dalam kasus yang disematkan ini, saya menghemat 5 detik dari urutan boot, sehingga file hanya-baca dapat dikompresi untuk mendapatkan efek yang baik. (misalnya, a PXE installer yang dapat di-boot akan menjadi kandidat yang baik).

Masalah yang lebih besar muncul ketika Anda mengompres database sistem seperti pembaruan Windows atau SQL Server, di mana sebagian besar file dikompresi ulang (yaitu, membaca, mendekompresi, memodifikasi, mengompresi ulang, dan menulis) yang mengarah ke kecepatan yang buruk dan fragmentasi yang sangat besar.

Dengan harga disk modern, saya sarankan untuk membeli disk yang lebih cepat untuk kecepatan lokal, dan membatasi kompresi hanya untuk skenario hanya-baca seperti netboot atau menginstal DVD.
