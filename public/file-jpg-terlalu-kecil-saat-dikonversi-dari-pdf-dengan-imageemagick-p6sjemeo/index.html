File JPG terlalu kecil saat dikonversi dari PDF dengan imageemagick

Saya menggunakan Ubuntu 10.04 dengan imagemagick. Saya menggunakan perintah ini:

convert myfile.pdf -density 300 file.jpg

Ini berfungsi dengan baik, tetapi JPG hanya setinggi sekitar satu inci. Bagaimana cara mempertahankan ukuran kertas dalam proses konversi?

Anda dapat mengatur -densitas sebelum Anda membaca sumber pdf, jadi Anda bisa melakukannya

convert -density 300 myfile.pdf myfile.jpg

dan itu akan membaca pdf pada 300dpi, kemudian merender jpg dengan ukuran asli pdf, apa pun itu.

Saya tidak tahu apakah Imagemagick dapat merasakan ketinggian yang benar secara otomatis. Namun, jika Anda dapat menemukan tinggi yang benar dalam piksel, Anda dapat mengatur tingginya secara manual.

Berdasarkan http://www.indigorose.com/forums/archive/index.php/t-13334.html kertas A4 memiliki tinggi 3508 piksel dalam 300 dpi jadi Anda bisa melakukannya

convert myfile.pdf -density 300 -geometry x3508 file2.jpg

Perintah ini menyetel lebar ke 3508 piksel dan mempertahankan aspek radio. Lihat http://www.imagemagick.org/script/command-line-processing.php#geometry untuk detail tentang argumen -geometry.

  • Untuk kualitas yang lebih baik, Anda harus mengatur kepadatan SEBELUM nama file input, seperti "convert -density 300 myfile.pdf file2.jpg". Kemudian ImageMagick menggunakan resolusi ini untuk meraster PDF. Jika tidak, itu akan meraster PDF pada resolusi default (72 dpi) dan perintah -geometry menskalakan gambar yang sudah di-raster. Perintah -density setelah nama file input hanya mengubah properti EXIF ​​dari file output, bukan pikselnya. (Sebagai bonus, jika Anda menentukan -densitas sebelum nama file input, Anda tidak perlu -geometri lagi untuk mempertahankan ukuran gambar.)
    — Jaan
