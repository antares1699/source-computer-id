Apakah mungkin memiliki akun IMAP tetapi menggunakan server SMTP yang berbeda?

ISP saya memiliki masalah dengan server surat keluar mereka, mengakibatkan sebagian besar dari email keluar saya dibuang secara acak. Saya menggunakan IMAP.

Saya juga memiliki akun lain dengan ISP yang berbeda. Jika saya masih menggunakan POP3 lama yang baik, saya dapat menggunakan server SMTP ISP kedua sebagai server surat keluar untuk kedua akun, dan menggunakan yang pertama hanya untuk surat masuk.

Apakah ini mungkin dilakukan dengan IMAP? Dan jika demikian, bagaimana cara menyiapkannya di Mozilla Thunderbird? (Google gagal mencerahkan saya!)

Itu mungkin. Kedua protokol tersebut adalah sama sekali tidak terkait . IMAP tidak peduli bagaimana Anda mengirim email, dan SMTP tidak peduli bagaimana Anda menerima email. Di Thunderbird, pengaturan server SMTP bekerja sama dan bahkan harus ditempatkan di tempat yang sama.

Masalah sebenarnya adalah banyak domain menggunakan SPF untuk menentukan daftar putih kecil di mana server SMTP diizinkan untuk mengirim email sebagai domain itu. Jadi, jika domain ISP1 Anda menggunakan SPF, dan Anda mencoba mengirim email melalui server ISP2, sebagian besar penerima Anda akan memberikan skor "spam" yang tinggi karena pemeriksaan SPF gagal.

Dengan kata lain, tingkat penolakan Anda bahkan mungkin meningkat.
