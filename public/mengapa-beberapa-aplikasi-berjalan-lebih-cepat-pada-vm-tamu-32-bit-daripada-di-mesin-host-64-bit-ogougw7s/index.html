Karena Anda menyebutkan Java, apa versi JVM Anda dan apakah itu berjalan dalam mode 32-bit atau 64-bit pada host?

Ups terkompresi

Sebuah "oop", atau penunjuk objek biasa dalam bahasa Java Hotspot, adalah penunjuk yang dikelola ke suatu objek. OOP biasanya berukuran sama dengan penunjuk mesin asli, yang berarti 64 bit pada sistem LP64. Pada sistem ILP32, ukuran heap maksimum kurang dari 4 gigabyte, yang tidak cukup untuk banyak aplikasi. Pada sistem LP64, heap yang digunakan oleh program tertentu mungkin harus berukuran sekitar 1,5 kali lebih besar daripada saat dijalankan pada sistem ILP32. Persyaratan ini disebabkan oleh ukuran pointer terkelola yang diperluas. Memori tidak mahal, tetapi saat ini bandwidth dan cache kekurangan pasokan, sehingga meningkatkan ukuran heap secara signifikan dan hanya mendapatkan sedikit di atas batas 4 gigabyte tidak diinginkan.

Pointer yang dikelola di heap Java mengarah ke objek yang disejajarkan pada batas alamat 8 byte. Ups terkompresi mewakili pointer yang dikelola (di banyak tempat tetapi tidak di semua tempat di perangkat lunak JVM) sebagai offset objek 32-bit dari alamat basis heap Java 64-bit. Karena mereka objek offset dan bukan byte offset, mereka dapat digunakan untuk menangani hingga empat miliar benda (bukan byte), atau ukuran heap hingga sekitar 32 gigabyte. Untuk menggunakannya, mereka harus diskalakan dengan faktor 8 dan ditambahkan ke alamat dasar heap Java untuk menemukan objek yang dirujuknya. Ukuran objek yang menggunakan oops terkompresi sebanding dengan yang ada dalam mode ILP32.

Syarat membaca sandi digunakan untuk mengekspresikan operasi di mana oop terkompresi 32-bit diubah menjadi alamat asli 64-bit menjadi heap terkelola. Operasi terbalik disebut sebagai pengkodean .

Ups terkompresi didukung dan diaktifkan secara default di Java SE 6u23 dan yang lebih baru. Di Java SE 7, penggunaan oops terkompresi adalah default untuk proses JVM 64-bit saat -Xmx tidak ditentukan dan untuk nilai -Xmx kurang dari 32 gigabyte. Untuk JDK 6 sebelum rilis 6u23, gunakan flag -XX:+UseCompressedOops dengan perintah java untuk mengaktifkan fitur tersebut.

Jejak memori yang lebih besar dari 64-bit JVM memiliki implikasi kinerja yang sangat signifikan.
