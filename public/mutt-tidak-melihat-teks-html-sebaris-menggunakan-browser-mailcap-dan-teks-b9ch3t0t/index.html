mutt tidak melihat teks / html sebaris menggunakan browser mailcap dan teks

Saya menggunakan mutt di Linux. Ini telah bekerja dengan baik selama bertahun-tahun, namun awal tahun ini telah ditingkatkan dan "sesuatu" berubah.

Ketika saya mendapatkan email yang isi pesannya adalah html, dan headernya menunjukkan Content-Type: text/html, saya dulu bisa mengetik v untuk melihat bagian-bagiannya, lalu tekan Memasukkan dan konten HTML akan ditampilkan menggunakan $HOME/.mailcap dan browser teks pilihan saya. .mailcap memiliki entri text/html; w3m %s; nametemplate=%s.html.

Sekarang, ini tidak lagi berfungsi dan saya melihat kemuliaan email HTML dalam semua kengeriannya yang luar biasa. Saat saya menekan v , ini menunjukkan hanya ada satu bagian dan itu adalah teks / html. aku menekan Memasukkan dan menampilkan HTML mentah. Bagaimana saya membuatnya menampilkan HTML yang diformat dengan browser teks saya lagi?

Di sisi lain, jika email berisi teks / multi bagian dan salah satunya adalah HTML, atau HTML adalah lampiran, itu berfungsi dengan baik. Hanya jika tubuh pesan adalah HTML yang tidak berfungsi lagi. Ini seperti mutt mengabaikan .mailcap ketika teks / html adalah badan, dan bukan lampiran.

Mutt mengubah perilakunya di 1.5.21. Anda dapat menemukan detailnya di sini:

http://marc.info/?l=mutt-dev&m=128478045511699&w=2

Mutt sekarang akan menampilkan sebagian besar teks / * bagian secara internal melalui halaman. Saya m menebak sistem Anda / etc / mailcap memiliki entri teks / html tanpa Kata kunci "needterminal" yang digunakan Mutt untuk merender teks.

Anda masih bisa mendapatkan perilaku lama di menu lampiran dengan menekan 'm' (view-mailcap) yang akan selalu menggunakan mailcap untuk melihat, dan harus mengambil entri apa pun untuk pemirsa eksternal ("perluterminal").
