Macet kecil atau 'menonjol' saat komputer menjalankan program

Saya telah mengalami masalah ini selama beberapa minggu sekarang dan setelah mencari-cari posting yang tak terhitung jumlahnya sepertinya tidak ada yang membantu, jadi saya memposting pertanyaan saya, di sini untuk Anda semua.

Saat menjalankan program apa pun (dari game intensif grafis hingga iTunes hingga browsing youtube) saya mengalami jutters kecil atau freeze yang berlangsung sekitar setengah detik. Biasanya dalam semburan antara empat dan enam jutter selama sepuluh detik. Mereka akan berhenti selama tiga puluh detik, lalu kembali lagi.

Ini membuatku frustrasi tanpa akhir. Saya baru-baru ini membersihkan semua debu dari komputer saya, saya seorang pembuat komputer yang rajin jadi saya yakin semua perangkat keras sudah terpasang dengan benar, tidak terlalu panas dan tidak ada yang ditambahkan baru-baru ini.

Ketika masalah ini terjadi, semuanya macet selama setengah detik itu termasuk suara, menciptakan suara 'blargh'. Tidak yakin apakah itu membantu, tapi membuatku tertawa saat mengetiknya.

Semua driver telah diperbarui ke versi terbaru.

Salam sebelumnya.

Spesifikasi:

  • Win7 64bit berjalan pada Intel SSD
  • Intel i7 960 @ 3,2 GHz
  • RAM 12 GB
  • 950w Corsair PSU
  • G1.Guerrilla (rev. 1.0) Mobo
  • nVidia Geforce GTX 680
  • Drive 3x SATA (2x 1TB 1x 500GB)
  • G19 & G9 Logitech K / B dan Mouse.

Saya menduga bahwa masalahnya adalah sesuatu yang melekat pada Windows 7 (dan mungkin variasi lainnya juga). Persisnya, saya tidak tahu, tapi mungkin ini akan menjelaskan masalah ini.

Saya mengambil mesin Windows 7 Pro 64bit baru, dibangun dari awal tanpa aplikasi tambahan sama sekali. Bahkan tidak perangkat lunak anti-virus. Perangkat Keras adalah CPU i7-6700 pada motherboard Asus H270. Satu hard drive dan RAM 16 GB.

Saya mematikan sinkronisasi waktu Internet otomatis. dan mematikan pemeriksaan berkala untuk pembaruan.

Saya menginstal git-bash (git untuk Windows) jadi saya akan memiliki shell bash. Saya menulis skrip yang melakukan posting http ke server yang dijamin akan merespons dalam 31,25 msec ke posting itu. Jika Anda mengeluarkan posting segera setelah menerima tanggapan untuk posting sebelumnya, dibutuhkan sekitar 31 msec untuk mendapatkan posting berikutnya. Skrip mengatur waktu interval antara posting yang berjalan dalam putaran yang ketat.

Jika klien berjalan tanpa penundaan maka diharapkan interval rata-rata menjadi 31 msec. Jika ada beberapa penundaan dari waktu ke waktu yang mungkin mencapai 40 msec atau lebih, dan kemudian "mengejar" posting berikutnya dengan interval sekitar 20 msec. Faktanya, menggunakan skrip bash yang sama pada mesin CentOS 7 (dengan CPU yang jauh lebih lambat dan hanya 2 core) saya mengukur rata-rata 31 ms, minimal 13 dan maksimal 49 selama beberapa jam operasi.

Tetapi menggunakan mesin Windows (seperti dijelaskan di atas) sebagai klien, minimumnya adalah 31 msec, rata-rata 59 msec, dan maksimum 1092 msec. Sekitar sekali atau dua kali dalam satu jam (mungkin lebih?) Sistem Windows akan "menghilang" selama sekitar satu detik. (Tidak hanya itu, tetapi sistem Windows tidak dapat mengikuti tingkat 32 Hz penuh yang dapat disediakan oleh server.)

Saya pikir itu hanya sesuatu yang perlu kita jalani!

Masalah semacam ini bisa disebabkan oleh banyak hal, jadi saya akan mulai dengan beberapa diagnostik dasar.

Biarkan Windows Task Manager berjalan sehingga Anda memiliki sesuatu yang dapat Anda periksa saat Anda melihat perilaku ini di lain waktu. Monitor sumber daya memberikan lebih banyak informasi daripada aplikasi dasar, tab proses.

Muat Windows Task Manager (Ctrl + Alt + Escape)

Pergi ke Tab Kinerja

Klik Tombol Resource Monitor

Lain kali PC Anda membeku, periksa monitor sumber daya dan periksa tingkat CPU, Memori, dan Disk I / O, semoga Anda akan melihat proses yang terlihat seperti kemungkinan penyebabnya, dan Anda dapat menyelidiki perbaikan yang lebih spesifik dari sana.
