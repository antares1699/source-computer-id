Windows tetap menggunakan server DNS pilihan?

Saya mengalami masalah saat menggunakan WiFi Panera, dan akhirnya saya menemukan masalahnya:

Setiap kali saya reboot, Windows mengatur ulang pengaturan TCP / IP saya untuk adaptor nirkabel saya untuk menggunakan server DNS tertentu, daripada membiarkan DHCP menemukannya.

Bagaimana saya bisa menghentikan ini terjadi lagi dan lagi?

Saya memiliki kecurigaan bahwa Tor / Vidalia / Polipo melakukan ini, tetapi saya GUI baki sistem mereka tidak diatur untuk berjalan saat startup.

Pembaruan: Secara khusus, pengaturan DNS saya diubah setelah reboot ke yang berikut ini

Preferred DNS server: 127.0.0.1

Alternate DNS server: 129.174.97.3

Ini tampaknya archon.gmu.edu, layanan dari Universitas George Mason, tempat saya tinggal. Dengan asumsi ini hanya pengaturan Windows yang persisten dan bukan spyware dari perangkat lunak portal tawanan yang mengganggu (jangan tanya), bagaimana cara saya menghapus pengaturan Windows ini?

Saya akan mencoba penyetelan ulang yang lebih keras dari pengaturan TCP / IP.

Tor atau beberapa perangkat lunak dari bundel Tor tidak melakukan ini. Karena Tor bekerja pada level aplikasi (proxy), tidak perlu mengubah pengaturan adaptor.

Mungkin Anda juga menggunakan (atau telah menggunakan) beberapa perangkat lunak atau perangkat lunak VPN untuk pengelolaan pengaturan adaptor berbasis lokasi? Beberapa sistem dilengkapi dengan perangkat lunak seperti itu yang sudah diinstal sebelumnya. Mungkin data PTR atau WHOIS server DNS memberikan beberapa petunjuk perangkat lunak mana yang mengubah pengaturan.
