Artikel HOWTO perlbrew Perl 32-bit pada Centos 64-bit menjelaskan prosedur berikut:

A. Instal perlbrew

Anda perlu menginstal perlbrew dari CPAN, dan CPAN memiliki banyak dependensi. App :: cpanminus yang luar biasa membuat pengalaman ini semudah mungkin, jadi saya menginstalnya sebelum pindah ke perlbrew itu sendiri.

$ sudo yum install perl-CPAN
$ sudo cpan App::cpanminus
$ sudo cpanm install App::perlbrew

B. Inisialisasi perlbrew

$ perlbrew init
$ perlbrew install-patchperl

Perhatikan output dari langkah init - ini akan mengarahkan Anda untuk membuat perubahan pada konfigurasi shell Anda.

C. Instal Perpustakaan 32-bit

Menginstal kedua paket ini cukup untuk membangun inti perl 32-bit. Jika Anda membuat modul XS tambahan terhadap perl 32-bit, mereka mungkin memerlukan penginstalan pustaka 32-bit lainnya.

$ sudo yum install glibc-devel.i686 libgcc.i686

D. Membangun Perl

$ perlbrew install 5.8.9 -Accflags="-m32 -march=i686" -Aldflags="-m32 -march=i686" -Alddlflags="-shared -m32 -march=i686"
Fetching perl-5.8.9 as /home/zts/perl5/perlbrew/dists/perl-5.8.9.tar.bz2
Installing /home/zts/perl5/perlbrew/build/perl-5.8.9 into ~/perl5/perlbrew/perls/perl-5.8.9

This could take a while. You can run the following command on another shell to track the status:

  tail -f ~/perl5/perlbrew/build.perl-5.8.9.log

perl-5.8.9 is successfully installed.

Hanya itu saja, meskipun hasilnya tidak terlalu sempurna. Meskipun pemanggilan di atas membangun perl 32-bit, pemanggilan tersebut tidak menimpa archname sistem - sehingga @INC yang dihasilkan terlihat seperti ini:

@INC:
    /home/zts/perl5/perlbrew/perls/perl-5.8.9/lib/5.8.9/x86_64-linux
    /home/zts/perl5/perlbrew/perls/perl-5.8.9/lib/5.8.9
    /home/zts/perl5/perlbrew/perls/perl-5.8.9/lib/site_perl/5.8.9/x86_64-linux
    /home/zts/perl5/perlbrew/perls/perl-5.8.9/lib/site_perl/5.8.9
    .

Untuk tujuan saya, ini hanyalah masalah estetika - direktori x86_64-linux berisi objek bersama 32-bit.

Saya belum mencobanya (dan saya bertanya-tanya mengapa Anda membutuhkan perl 32-bit pada sistem 64-bit), tetapi mungkin sesuatu seperti ini akan berhasil. Sesuaikan jalur dan opsi konfigurasi sesuai kebutuhan. (gunakan ls dan cd untuk menemukan direktori yang tepat di sistem Anda, gunakan ./configure --help untuk melihat opsi konfigurasi yang tersedia):

export CC="gcc -m32"
export CXX="g++ -m32"
export PKG_CONFIG_PATH="/usr/lib32/pkgconfig"       
./configure --prefix=/usr --libdir=/usr/lib32 --enable-utf --enable-unicode-properties --enable-pcre16 --enable-pcre32 --enable-jit
make
Bagikan
