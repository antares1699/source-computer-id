Memformat ulang USB untuk mendukung file besar di Windows, Linux, dan Mac

Saya mencoba menyalin file 5,3 GB dari hard disk ke kunci USB 3.0 SanDisk 16 GB USB. Saya sudah mencoba ini baik dari OS X maupun dari Linux. Dalam kedua kasus, file tersebut adalah disalin sebagian dan kemudian operasi gagal. Di OS X itu gagal dengan pesan seperti "file terlalu besar". Saya harus menggunakan perintah split untuk membagi file menjadi beberapa potongan berukuran 100MB dan kemudian salin direktori yang memuatnya ke USB kunci untuk menyalin file.

Kemudian, di Linux saya memasang kembali file-file tersebut dengan perintah cat menjadi satu file, mengirimkan output ke file hard drive komputer. Namun, ketika saya mencoba memasang kembali file membagi file menjadi satu file dengan perintah cat dan mengirim file output ke kunci USB, kunci USB masih tidak menerima file yang dihasilkan, yang lagi-lagi, terlalu besar, dan dikeluhkan dengan pesan kesalahan

"cat: kesalahan tulis: Kesalahan protokol".

Ketika saya memeriksa hasil yang berhasil dengan:

$ ls -l --block-size=M MyFileName

ukuran file persis:

4096M (yaitu, 4 GB).

Ini sepertinya batasan yang terkait dengan sistem file default kunci USB yang diformat sebelum dijual.

Ketika saya memeriksa filesystem-nya di bawah Windows 7 dengan mengklik kanan file Disk USB dan memeriksa propertinya Saya dapat melihat bahwa filesystem tersebut adalah sistem file FAT32.

Dapatkah saya entah bagaimana, mulai dari Windows, memformat kunci USB sehingga itu menggunakan sistem file yang akan menerima file 5,3 GB saya, dan bisa di saat yang sama dibaca dari dan ditulis ke dari semua OS X, Linux, dan Windows 7? Sistem file apa yang harus saya format kunci USB secara berurutan untuk mencapai tujuan ini?

Tidak ada solusi sempurna, tetapi kandidat teratas Anda adalah NTFS dan exFAT :

  • Windows (versi terbaru) akan membaca dan menulis keduanya, secara native
  • OS X akan membaca NTFS dan membaca / menulis exFAT
  • Linux akan membaca NTFS (dukungan kernel)

Driver pihak ketiga dapat mengisi celah yang hilang (non-Windows); beberapa distro Linux menyertakannya. Jika penggunaan Anda sebenarnya hanya membaca dari Windows, maka NTFS akan berfungsi di luar kotak. Jika Anda terutama OS X dan bukan Linux, maka exFAT akan lebih baik.

Salah satunya dapat diformat langsung di Windows (File) Explorer GUI di desktop .
