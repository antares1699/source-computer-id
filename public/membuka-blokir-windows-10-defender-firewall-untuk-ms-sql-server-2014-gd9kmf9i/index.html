Membuka blokir Windows 10 Defender Firewall untuk MS SQL Server 2014

Saya memiliki contoh MS SQL Server 2014 (sebut saja MSSQL) pada mesin Windows 10 (sebut saja WINTEN).

Menjalankan SSMS di WINTEN, saya dapat menyambung ke SQL Server lokal dan juga ke instance SQL Server di komputer lain di jaringan pribadi.

Namun, saya tidak dapat membuat sambungan dari SSMS di komputer lain ke contoh SQL Server WINTEN\MSSQL kecuali kalau Saya benar-benar mematikan Windows Defender Firewall.

Browser SQL Server

SQL Server (WINTEN)

Masuk ke propertinya dan catat jalur ke file yang dapat dieksekusi.

Di Windows Firewall dengan Keamanan Lanjutan buat 4 aturan masuk baru yang memungkinkan koneksi kedua program ini untuk TCP dan UDP.

Saya biasanya melakukan ini melalui Windows Firewall standar (firewall.cpl) yang akan membuat aturan ini untuk Anda dengan sedikit pekerjaan. Buka firewall.cpl, lalu pilih "Izinkan aplikasi atau fitur melalui Windows Firewall". Pilih Izinkan aplikasi lain dan tambahkan dua jalur yang dapat dieksekusi yang ditemukan sebelumnya.

Alasan Anda perlu mengizinkan program daripada port melalui firewall adalah bahwa meskipun Browser SQL selalu pada port 1433, Mesin Virtual SQL Anda akan berada di port acak / dinamis yang akan diarahkan oleh SQL Browser untuk koneksi. Jika Anda ingin mengizinkan port daripada program, maka Anda perlu mengubah Instans SQL ke instans default dan memaksanya untuk menggunakan port tetap dengan browser SQL dinonaktifkan.

Harap periksa juga apakah Layanan Browser SQL Anda dikonfigurasi untuk memulai secara otomatis saat permulaan dan bahwa di Manajer Konfigurasi SQL bahwa TCP / IP diaktifkan untuk Mesin Virtual SQL Anda.

Buka Manajer Konfigurasi Server Sql di WINTEN.

Di Konfigurasi Jaringan SQL Server, pilih MSSQL Anda dan periksa pengaturan untuk TCP / IP.

Pastikan bahwa protokol diaktifkan dan server mendengarkan pada porta 1433. (Tab Alamat IP , bagian IPAll , bidang Port TCP )
