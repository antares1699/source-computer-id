Mengapa saya mendapatkan "Kata Sandi Salah" saat menghubungkan ke server Mumble? Saya bahkan tidak pernah menggunakan kata sandi!

Saya baru saja menginstal ulang Mumble, dan sekarang ketika saya mencoba masuk ke server favorit saya, saya mendapatkan «Kata Sandi Salah». Mengapa demikian?

Saat Anda menginstal Mumble untuk pertama kali, sertifikat telah dibuat untuk Anda. Bahwa adalah "kata sandi" Anda. Anda seharusnya menyimpannya di tempat yang aman, dan mengimpornya kembali saat memasang ulang Mumble. Anda tidak melakukannya.

Ketika Anda mendaftar, atau terdaftar dengan server favorit Anda, yang sebenarnya terjadi adalah pengikatan antara sertifikat dan nama panggilan Anda. Dengan sertifikat itu, Anda hanya dapat masuk dengan nama pengguna itu; tanpa sertifikat itu, Anda tidak dapat masuk dengan nama pengguna itu. Anda sekarang membutuhkan sertifikat itu.

Jika Anda masih menyimpannya di suatu tempat, impor: buka Konfigurasi → Wisaya sertifikat , lalu pilih Sertifikat impor . Pilih sertifikat Anda dan lanjutkan; Anda akan dapat masuk secara normal.

Jika Anda tidak memiliki sertifikat lagi, karena Anda mungkin tidak pernah tahu tentang semua hal iniadongdong, yang harus Anda lakukan adalah:

  1. Minta admin server Anda untuk Hapus akun anda. (Dialog terkait dapat dihubungi melalui Server → Pengguna terdaftar. )
  2. Saat dia melakukan itu, buka Konfigurasi → Wisaya sertifikat , lalu pilih Ekspor Sertifikat. Taruh di tempat yang aman (katakanlah, Dropbox) untuk diambil nanti.
  3. Saat admin memproses permintaan Anda, ubah nama panggilan Anda untuk tetap masuk jika Anda sedang terburu-buru. Jangan mendaftar!
  4. Saat admin memberi tahu Anda bahwa dia telah selesai menghapus akun Anda, keluar , lalu masuk dengan julukan "resmi" Anda.
  5. Daftar dengan server dan meminta admin untuk menyesali privasi yang Anda miliki sebelumnya.

... dan lain kali, ingatlah untuk impor sertifikat itu! Ini penting.
