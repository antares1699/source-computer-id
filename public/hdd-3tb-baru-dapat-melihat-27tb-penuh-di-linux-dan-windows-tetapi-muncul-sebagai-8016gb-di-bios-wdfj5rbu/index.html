HDD 3TB baru, dapat melihat 2,7TB penuh di Linux dan Windows, tetapi muncul sebagai 801,6GB di BIOS

Saya baru saja membeli file Seagate Barracuda 3TB drive (ST3000DM001). Setelah menginstalnya, BIOS saya mengenalinya tetapi melaporkan ukurannya sebagai 801.6gb. Saya melanjutkan dan boot ke Linux (Ubuntu 11.10 64-bit).

Linux melihatnya sebagai 2,7TB. Mengikuti beberapa petunjuk online (sayangnya, tidak memiliki tautannya), tampaknya disarankan untuk mengonversi drive ini ke GPT. Jadi saya menggunakan gparted untuk melakukan itu, lalu memformatnya ke NTFS juga menggunakan gparted. (Saya menggunakan NTFS karena mesin saya adalah daul-boot dan saya ingin memiliki akses ke drive di Windows juga).

Saya mem-boot ulang ke Windows (Windows 7 64-bit), dan Windows juga melihat drive dengan gratis 2,7TB. Semuanya sepertinya bekerja dengan baik.

Satu-satunya masalah adalah BIOS saya masih melaporkan drive sebagai 801.6GB. Motherboard saya adalah ASRock 770 Extreme3 dan BIOS adalah versi terbaru.

Karena semuanya tampaknya berfungsi dengan drive baru, saya berharap fakta bahwa BIOS melaporkan ukuran yang salah bukanlah masalah yang sebenarnya. Tapi sejujurnya, saya tidak begitu tahu. Adakah orang di luar sana yang lebih akrab dengan ini tahu apakah ini berpotensi menimbulkan masalah di masa depan? Adakah cara agar BIOS melaporkan ukuran yang benar?

Anda mungkin memeriksa apakah ada pembaruan BIOS untuk memperbaiki masalah, tetapi jika Anda tidak melakukan booting dari drive, itu tidak perlu dikhawatirkan. Jika Anda memang perlu mem-boot dari drive, dan tidak ada perbaikan bios, Anda masih dapat mengatasi masalah dengan menempatkan partisi boot di 800mb pertama, dan kemudian setelah OS aktif dan berjalan, Anda dapat menyimpan file data di sisa drive.
