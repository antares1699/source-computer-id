Printer label jaringan tersedia untuk dicetak melalui web?

MEMPERBARUI

Saat ini saya melihat printer label DYMO dan BROTHER untuk tujuan ini. Idealnya, mereka dapat mengklik tombol "cetak" di sebelah alamat di Aplikasi Web yang akan mencetak label tanpa perlu menyalin / menempel ke beberapa perangkat lunak berpemilik ...

Saya ingin dapat mengirim perintah cetak ke printer label jaringan lokal dari aplikasi web yang berjalan di server awan. Katakanlah saya ingin mengirim label ke rumah orang tua saya karena suatu alasan dan printer dipasang di jaringan rumah mereka.

Saya sudah bertanya bagaimana melakukan ini melalui kode di Stackoverflow sini dan pada Serverfault yang terkait dengan sisi jaringan sini , tetapi terkadang garis dapat dikaburkan dari admin jaringan (server default) dan masalah seperti printer (superuser) ...

Saya ingin tahu apakah ini mungkin dari sudut pandang jaringan / perangkat keras. Apakah ada printer "label" yang dapat saya tambahkan ke jaringan? Dapatkah saya menambahkan pencetak label sebagai perangkat cetak dari jaringan lain melalui web?

Ada beberapa opsi yang akan saya sarankan:

1 .. Siapkan VPN yang memungkinkan akses ke jaringan tempat printer disiapkan, dan mencetak dari sana.

  • Kelebihan:

    • Akses aman ke printer. Untuk mengakses printer, pengguna harus terhubung ke jaringan melalui VPN menggunakan nama pengguna dan kata sandi yang disiapkan oleh administrator jaringan. Anda bahkan dapat menyiapkan server cetak yang menangani semua tugas, dan meletakkan kedua perangkat (printer dan server) di jaringan yang sepenuhnya terpisah.

  • Kekurangan:

    • Pengguna untuk mengakses printer HARUS terhubung ke VPN, yang berarti ada potensi kebingungan dan kebutuhan bantuan dengan pengguna.

Untuk menyiapkan VPN, Anda memerlukan server yang menangani koneksi VPN. Menyiapkan VPN itu di server berbeda-beda menurut perangkat lunak server. Berikut beberapa panduan yang saya temukan untuk: " Windows Server 1 "," Windows Server 2 "," linux ", dan" DD-WRT "perangkat yang di-flash.

Saya pribadi telah menggunakan VPN dengan Universitas tempat saya bekerja dan berfungsi tanpa masalah. Teman saya bahkan dapat mencetak dari Droidnya, namun kami harus terhubung ke VPN untuk dapat mencetak.

2 .. Anda dapat mengkonfigurasi printer menggunakan IIS:

  • Kelebihan:

    • Printer dapat diakses dari alamat internet yang diteruskan dari server ke printer. Hal ini membuat akses ke printer jauh lebih mudah dan setelah printer disiapkan pada mesin pengguna, overhead TI akan berkurang.

  • Kekurangan:

    • Karena printer dapat diakses melalui internet, printer dan server / workstation yang terhubung langsung memiliki risiko gangguan yang lebih tinggi.

Saya tidak pernah menggunakan apa pun yang berhubungan dengan IIS, namun menurut saya artikel ini menarik dan mungkin berguna untuk mengembangkannya jika Anda menginginkannya: " Pencetakan Internet "," Cara menyambung ke Printer Internet ", dan" Bagikan Printer Secara Online "

Memperbarui Dengan DYMO label printer Anda dapat mengatur printer untuk mencetak melalui internet. Ini adalah situs web yang bagus yang menjelaskan bagaimana semua ini bekerja. Pencetak label jaringan BROTHER juga mendukung IIS.

3 .. Google Cloud Printing

Dengan OS berbasis online Google, mereka harus berurusan dengan pencetakan melalui jaringan cloud (internet). Mereka saat ini sedang dalam pengembangan pencetakan Cloud yang memungkinkan pekerjaan pencetakan dikirim melalui cloud untuk diselesaikan dengan sukses tanpa instalasi driver. Meskipun ini dalam tahap pengembangan, mereka sudah melakukannya kode yang dapat Anda lihat dan mainkan. Saya ragu ini yang sebenarnya Anda inginkan, tetapi saya pikir saya meletakkannya.

4 .. Sistem Cloud Printing HP

HP mengembangkan printer yang dapat mencetak melalui email! Lihat ini situs web tentang info lebih lanjut.

Anda dapat menggunakan produk gratis PrinterShare (diubah namanya dari PrintAnywhere), tersedia di Windows, Mac dan Linux.

Perangkat lunak harus diinstal pada mesin yang Anda inginkan untuk mencetak dan pada mesin yang Anda inginkan untuk mencetak. Prosesnya lebih baik dijelaskan dalam hal ini memanfaatkan artikel .

Inilah situs webnya Bagaimana itu bekerja mengatakan:

Jika printer bersama sedang online, dicetak dokumen diunggah langsung ke mesin target. Cepat dan aman. Jadi, biasanya Anda ingin menyimpan file printer online.

Jika printer bersama sedang offline, Anda masih bisa mencetak di atasnya. Dokumen akan disimpan sementara di PrinterShare print server, dan diunduh dengan menerima klien segera karena akan siap.

Ini seharusnya sangat mudah dilakukan, dan ada sejumlah cara untuk melakukannya.

Jika itu saya, dan saya mengontrol aplikasi ASP.Net, saya akan melakukan hal berikut:

Instal printer di server dan pastikan berfungsi kemudian buat aplikasi asp.net baru.

Sekarang hadir sedikit kesenangan bahwa seseorang di Stackoverflow dapat membantu Anda lebih dari saya ...

Bagian pertama aplikasi dapat berupa layanan web yang menautkan ke aplikasi Anda yang ada yang menerima berbagai variabel dari aplikasi pertama dan menyimpannya ke database.

Selanjutnya buatlah pada web service ini, buatlah interface yang pada dasarnya dapat menampilkan semua entri pada database sebelumnya.

Terakhir, buat aplikasi klien yang terhubung ke antarmuka ini di atas, dan jalankan di salah satu mesin di Office satu, itu dapat berjalan pada tugas terjadwal, atau bahkan Anda dapat mengatur semacam pemberi tahu sehingga akan mencetak secara real time.

Sekali lagi, saya yakin ada programmer yang jauh lebih baik daripada saya yang dapat memberi Anda gagasan yang jauh lebih baik tentang apa yang perlu dibuat, tetapi ketika Anda berbicara tentang server cloud dan banyak lagi, saya yakin Anda atau seseorang di tim memiliki teknis kemampuan untuk mencapai ini, dan saya yakin bahwa pendekatan semacam ini adalah solusi yang jauh lebih baik daripada apa pun karena Anda dapat menyelesaikan pencetakan dari mesin apa pun yang Anda butuhkan (tidak perlu menginstal driver printer, VPN, atau solusi lain) ... , ini bagus untuk masa depan, Anda mungkin ingin memutakhirkan printer label * dan Anda hanya perlu melakukan sedikit modifikasi pada aplikasi klien.

(*) Pengalaman saya dengan printer label kecil itu adalah bahwa mereka bagus tetapi jika Anda akan mencetak banyak, mereka bisa menjadi mahal dengan cepat dan Anda mungkin ingin meningkatkan ke printer Zebra yang serius atau printer serupa .... Off topic, tapi semoga anda menyukai nasehatnya!
