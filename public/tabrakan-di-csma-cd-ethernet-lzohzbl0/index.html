Pertama, berhati-hatilah karena LAN Ethernet yang dialihkan modern bukan lagi CSMA / CD. CSMA / CD adalah teknik yang diterapkan pada Ethernets 10mbit / detik dan 100mbit / detik yang menggunakan hub, bukan switch. Dan sejujurnya tidak pernah ada banyak hub 100BASE-TX; semua orang beralih ke switch sekitar waktu itu. Spesifikasi Gigabit Ethernet (1000BASE-T) membutuhkan sakelar; tidak ada yang namanya GigE hub.

Di Ethernets sakelar modern, Anda tidak memiliki media bersama lagi. Saat Anda terhubung ke sebuah sakelar, "domain tabrakan" hanya antara Anda dan port sakelar Anda. Dan jika Anda dalam mode dupleks penuh, yang hampir selalu benar dengan sakelar, maka Anda tidak memiliki kemungkinan tabrakan sama sekali. Jika Anda tidak bisa mengalami tabrakan, Anda tidak akan pernah mendeteksi tabrakan, jadi Anda tidak akan pernah punya alasan untuk mengirimkan sinyal macet.

Jadi kembali pada masa hub (dan kabel bersama seperti thinnet / cheapernet / 10BASE-2 coax dan thicknet 10BASE-5), berikut cara kerjanya:

Bayangkan Anda memiliki LAN 10BASE-T besar dengan banyak hub dan kabel panjang maksimal, memaksimalkan "aturan pengulang" dari maksimum 4 repeater (hub) antara dua perangkat di LAN. Karena penundaan propagasi sinyal di seluruh hub, diperlukan spesifikasi IEEE maksimum 232 bit kali untuk sinyal yang ditransmisikan dari Host A untuk mencapai host terjauh di jaringan (Host B).

Sekarang bayangkan Host A mulai mentransmisikan frame, dan sialnya, Host B, di ujung lain jaringan, 232 bit-kali, mulai mentransmisikan frame hanya 231 bit kali setelah Host A mulai mentransmisikan. Karena penundaan propagasi pada jaringan, Host B tidak tahu bahwa Host A sudah 231 bit ke dalam transmisinya ketika Host B mengirim bit pertama pembukaannya. Sekarang Host B akan mendeteksi tabrakan ini dalam yang pertama, katakanlah, 32 bit transmisi Host B, yang bisa sekitar 232 hingga 264 bit kali lebih cepat daripada Host A akan mendeteksinya. Jika Host B kebetulan mendeteksinya pada bit pertama transmisi, dan hanya berhenti mentransmisikan saat mendeteksi tabrakan ini, maka mungkin tidak tinggal di media cukup lama bagi Host A untuk juga mendeteksinya dan menyadari bahwa tabrakan terjadi ( Anda tidak dapat menjamin bahwa penerima akan mendeteksi tabrakan pada bit pertama tabrakan). Itu kemudian berarti bahwa Tuan Rumah A tidak akan tahu untuk melakukan prosedur penanganan tabrakan yang benar. Jadi daripada hanya berhenti mentransmisikan, Host B memancarkan sinyal Jam sehingga tetap berada di media cukup lama untuk memastikan Host A menyadari bahwa tabrakan telah terjadi.

Ini juga alasan mengapa frame Ethernet memiliki panjang minimum 64 Bytes. Ini menjamin bahwa Host A tetap berada di media cukup lama sehingga tabrakan dapat dideteksi sepenuhnya di sisi lain jaringan, dan sinyal Jam dari Host B dapat kembali ke seluruh jaringan, sehingga Host A masih berada di media saat sinyal Jam datang, sehingga dapat menyadari bahwa seseorang bertabrakan dengan transmisinya.
