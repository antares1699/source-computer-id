GRUB2: / boot sebagai subvolume BTRFS

Selain /boot, semua partisi saya (termasuk root, /) adalah subvolume BTRFS. Saya beralih ke BTRFS RAID1, dan demi integritas, ingin menyertakan /boot di dalam partisi BTRFS. Lebih disukai sebagai subvolume terpisah (dukungan kuota) tetapi mungkin sebagai bagian dari subvolume root. Saya menggunakan fedora terbaru, 22, yang cara :

  • grub2 2.02-0.16.fc22
  • btrfs-progs 4.0-1.fc22
  • kernel 4.0.4-303.fc22

Apakah ini mungkin (ya / tidak)? Harap berikan informasi tentang penyiapan khusus BTRFS + GRUB. Maksud saya, membuat subvolume baru dan menyalin data relatif mudah, tetapi mendapatkan seluruh alat untuk benar-benar boot adalah masalah lain sama sekali.

Iya. Saya sendiri berhasil memindahkan / boot ke sistem BTRFS sebagai subvolume. Tidak perlu pertimbangan khusus - jangan lupa untuk menginstal ulang grub dan membangun kembali initramfs.

edit:

Akan memerlukan menjalankan grub2-mkconfig -o /boot/grub2/grub.cfg setelah setiap pemutakhiran yang melibatkan kernel (linux, memtest, dll) atau selama / setelah pemutakhiran fedora (fedup) karena grubby tidak mendukung pemutakhiran grub.cfg pada subvolume btrfs. Tersedia tambalan [ # 864198 - lihat lampiran ] yang belum terintegrasi karena merupakan retasan. Dengan kernel-install dan BootLoaderSpec, grubby tampaknya mendekati akhir masa pakainya, jadi saya tidak merasa keberatan.

TLDR:

Memerlukan pemutakhiran grub.cfg secara manual setiap pemutakhiran pasca-kernel, yang tidak menimbulkan rasa sakit kecuali Anda telah mengeditnya secara langsung.
