OSX di kotak Virtual SerialNumber = 0

Saya telah menginstal OSX El Capitan yang berjalan di Virtualbox sehingga saya dapat menguji Xcode dan berpotensi mulai berkembang.

Sepertinya saya tidak bisa masuk ke App Store meskipun tampaknya nomor seri disetel ke 0.

Apakah ada cara untuk mengatur nomor seri di file konfigurasi Virtualbox?

Periksa nomor seri dalam tentang kami di VM. Jika tidak ada atau ada 0. Maka solusi di bawah ini pasti akan memperbaiki masalah Anda.

VirtualBox tidak secara default menyediakan nomor seri untuk tamu OS X. Tentu saja OS X berjalan dengan baik di dalam VirtualBox yang berjalan di Mac bahkan dalam kasus itu. Namun karena nomor serinya adalah "0", aplikasi App Store dan Server tidak akan berfungsi.

Solusinya adalah menentukan nomor seri Mac yang sebenarnya dengan membuka Tentang Mac Ini atau menjalankan Informasi Sistem. Ini adalah string alfanumerik. Di Mac Pro 5,1 saya, ini dimulai dengan huruf dan panjangnya 11 karakter.

Di Terminal, jalankan perintah berikut menggantikan nama mesin virtual VirtualBox Anda dan nomor seri Anda (semua satu baris):

VBoxManage setextradata “<your-vm-name>” VBoxInternal/Devices/efi/0/Config/DmiSystemSerial “<your-serial-number>“

Pastikan Anda menghapus tanda kutip saat memasukkan perintah di atas.

Memulai mesin setelah perubahan ini akan menampilkan nomor seri yang cocok dengan sistem dasar dan aplikasi akan berfungsi dengan benar.

Ini telah diuji pada Mac Pro 5,1 yang menjalankan 10.10.5 dengan VirtualBox 5.0.14. Tamu itu adalah 10.11.3.

Saya memiliki masalah yang sama dan menyelesaikannya setelah mengikuti tutorial ( Tautan sudah mati dari Youtube sekarang ) yang menggunakan bootloader semanggi. Sayangnya ada kendala. Anda harus melalui proses bootloader (menekan F12 dan memilih tombol peluncuran OS) setiap kali Anda menyalakan mesin. Jika Anda tidak mengikuti, Anda mendapatkan nomor seri 0. Saya tidak keberatan ini karena saya tidak memulai ulang tamu kecuali benar-benar diperlukan.
