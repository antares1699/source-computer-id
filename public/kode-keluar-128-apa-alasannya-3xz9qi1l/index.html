kode keluar 128, apa alasannya?

Sesuai definisi kode keluar 128 berarti 'argumen keluar tidak valid'. Tetapi saya selalu mendapatkan 255 (Status keluar di luar jangkauan) jika argumen tidak valid seperti nomor float.

Apakah ini implementasi berpemilik pada distribusi linux saya?

# exit 1.234
exit
bash: exit: 1.234: numeric argument required

$ echo $?
255   //this should be 128?

# exit -1
exit

$ echo $?
255   //this is okay

Tidak ada apa-apa di dalamnya Dokumentasi Bash yang mengatakan 128 adalah kode keluar tidak valid yang diperlukan.

Bash sendiri mengembalikan status keluar dari perintah terakhir yang dieksekusi, kecuali jika terjadi kesalahan sintaks, dalam hal ini ia keluar dengan nilai bukan nol.

Perintah terakhir adalah bash builtin exit (dari halaman manual)

keluar [n]

Menyebabkan shell keluar dengan status n. Jika n dihilangkan, status keluar adalah dari perintah terakhir yang dijalankan.

Dicentang spesifikasi untuk WEXITSTATUS.

WEXITSTATUS (stat_val)

Jika nilai WIFEXITED (stat_val) bukan nol, makro ini mengevaluasi ke 8 bit orde rendah dari argumen status yang diteruskan proses anak ke _exit () atau exit (), atau nilai proses anak yang dikembalikan dari main ().

Jadi keluar dibatasi ke integer 8 bit (0 - 255), jadi -1 akan menjadi 255. Exit hanya memahami argumen integer dan bukan float, jadi kemungkinan menendang -1 default.

bash$ echo $BASH_VERSION
4.1.10(4)-release

bash$ exit foo
exit
bash: exit: foo: numeric argument required
$ echo $?    
255

bash$ exit 2
exit
$ echo $?
2

bash$ exit -2
exit
$ echo $?
254

Ini khusus untuk varian shell Anda. Ini http://pubs.opengroup.org/onlinepubs/9699919799/utilities/V3_chap02.html#tag_18_08 atau http://pubs.opengroup.org/onlinepubs/9699919799/utilities/V3_chap02.html#exit jangan mengkonfirmasikan kesan Anda bahwa 128 adalah kode keluar khusus.

Untuk exit 3.45 versi saya dari ksh dan zsh return 3, tcsh mengembalikan 1 (sebenarnya tidak keluar), dan ash mengembalikan 2 (tetapi juga tidak benar-benar keluar).
