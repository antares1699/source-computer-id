Apa perbedaan antara "Rufus" dan "Rufus portable"

Di situs ini: https://rufus.akeo.ie/ , ada dua unduhan, satu berlabel "Rufus 2.8" dan yang lainnya berlabel "Rufus 2.8 portable" Saya telah mencoba keduanya, dan sejauh yang saya ketahui, keduanya tidak berbeda dalam fungsinya. Saya tidak tahu apa perbedaan antara keduanya, jika ada, karena yang tidak berlabel "Portable" juga portabel "(yaitu tidak memerlukan instalasi").

Sebagai pengembang utama Rufus, izinkan saya menjelaskan sedikit lebih banyak pertanyaan dengan mengutip dari apa yang menurut saya merupakan bagian yang lebih relevan dari FAQ resmi Rufus :

Pertama-tama, saya pikir saya perlu mendefinisikan apa itu portabilitas, karena banyak orang (termasuk Wikipedia) menggunakan definisi yang salah, dan benar-benar kehilangan inti dari apa sebenarnya aplikasi portabel.

Aplikasi portabel adalah aplikasi yang memberi Anda kemampuan untuk membawa dan memelihara pengaturan Anda saat berpindah dari satu komputer ke komputer berikutnya.

Itu dia. Hanya itu yang dilakukan oleh aplikasi portabel.

Jadi, jika Anda mengharapkan bahwa portabilitas menyiratkan sesuatu tentang TIDAK menulis ke dalam registri pada Windows, atau tidak disertai dengan penginstal, Anda salah besar. Sering kali, menjadi portabel berarti aplikasi akan menulis pengaturannya ke file teks (seperti file .ini pada Windows) yang dapat Anda bawa dengan perangkat lunak, saat Anda berpindah dari satu komputer ke komputer lain, alih-alih registri, dan ini mungkin alasan mengapa banyak orang mengacaukan "portabel" dengan "tidak pernah menulis ke registri" pada Windows, tetapi sebenarnya tidak ada janji dibuat dari aplikasi portabel yang bonafid bahwa ia akan membiarkan registri tidak tersentuh.

Jadi, dengan klarifikasi ini, saya dapat menjelaskan bahwa versi reguler Rufus sudah memenuhi syarat sebagai aplikasi portabel karena, jika Anda kebetulan memiliki rufus.ini di direktori yang sama dengan Rufus Anda yang dapat dieksekusi (meskipun itu adalah file kosong ), kemudian Rufus akan membaca dan menulis pengaturannya, seperti bahasa yang Anda inginkan untuk menjalankan aplikasi, atau opsi lain yang dipertahankan antar sesi, ke dalam file itu, dan jika Anda menyalin rufus.ini dan Rufus yang dapat dieksekusi ke komputer lain, Anda akan melihat bahwa pengaturan Anda telah disimpan dari komputer sebelumnya, karenanya "portabel". Dan pada tahap ini, saya juga harus menekankan bahwa, bahkan ketika Rufus berjalan dalam mode portabel, registri Anda akan dimodifikasi, karena ini BUKAN tentang portabilitas.

Lalu, kenapa harus menyediakan versi portabel, katamu? Nah, ini membawa kita pada HANYA perbedaan antara versi "portabel" dari Rufus dengan versi "biasa", yaitu versi "portabel" akan membuat rufus.ini secara default (sehingga Anda tidak perlu melakukan sendiri, jika Anda ingin menggunakan Rufus dalam mode portabel), sedangkan versi biasa tidak. Hanya itu yang ada di sana!

Ini juga alasan mengapa ketika Rufus mengunduh pembaruan, ia selalu memilih versi "biasa", meskipun versi yang Anda jalankan adalah versi "portabel", karena Anda sudah memiliki rufus.ini, oleh karena itu, versi baru " versi reguler "yang diunduh akan terus berjalan dalam mode portabel.

Sekarang, jika Anda masih bingung tentang apa artinya di atas, maka sebaiknya Anda menggunakan Rufus versi standar saja. Portabilitas benar-benar HANYA untuk orang yang perlu bekerja dengan aplikasi di banyak komputer, dan ingin mempertahankan setelan mereka saat melakukannya. Jika itu tidak berlaku untuk Anda, maka Anda harus mengunduh versi reguler.

Mengapa versi portabel dan reguler identik biner?

Itu karena cara Rufus mendeteksi apakah itu harus dijalankan secara portabel atau mode biasa adalah dengan memeriksa nama file yang dapat dieksekusi. Itu cara kerjanya seperti ini: jika nama file mengandung huruf p, maka kode akan berjalan dalam mode portabel. Dan jika tidak ada p, maka mode biasa digunakan. Faktanya, di server web, file mengunduh untuk versi portabel hanyalah tautan simbolis ke versi biasa, dengan p ditambahkan ke nama, jadi tentu saja binari akan selalu identik.

Tetapi tidak ada yang mewah atau misterius tentang metode ini - perangkat lunak seperti Busybox telah melakukan ini selama bertahun-tahun dan Anda tidak boleh panik keluar, atau beri tahu saya bahwa ada masalah dengan unduhan, di akun bahwa ukuran dan konten Rufus versi portabel dan reguler persis sama. Ada banyak cara untuk membuat hal yang sama persis executable berperilaku dengan cara yang sama sekali berbeda, melalui eksternal faktor, seperti nama file ...

Dari FAQ Rufus

Perbedaan satu-satunya yang disebutkan FAQ adalah di bawah ini:

Rufus terhubung ke internet, tetapi saya tidak pernah mengizinkannya - mengapa?

...

Bagaimana saya bisa mengatasi dilema ini? Sederhana: Jika Anda melihat http://rufus.akeo.ie/downloads/ Anda akan melihat sebenarnya ada 2 versi dari versi Rufus terbaru, yang disebut rufus - #. #. exe (as serta versi portabel yang sesuai) dan yang disebut lainnya rufus.exe. Mereka pada dasarnya adalah file biner yang sama (rufus.exe adalah sebenarnya hanya tautan simbolis ke yang pertama di server web). Namun, ketika Rufus mulai, ia memeriksa nama yang dapat dieksekusi, dan jika ditemukan bahwa itu disebut "rufus.exe", itu tidak menampilkan file pertanyaan apakah pengguna ingin memeriksa pembaruan, dan mengaktifkannya periksa secara otomatis.
