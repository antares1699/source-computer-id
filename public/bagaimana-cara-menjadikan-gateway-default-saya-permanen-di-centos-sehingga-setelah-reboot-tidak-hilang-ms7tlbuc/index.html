Bagaimana cara menjadikan gateway default saya permanen di CentOS? Sehingga setelah reboot tidak hilang

Saya menjalankan CentOS. Penting untuk menyediakan CentOS saya saat saya terhubung dari jarak jauh ke router VPN. Saya mencoba menambahkan gateway default ke router VPN, tetapi setelah reboot hilang. Bagaimana saya bisa menjadikannya permanen?

ip route add 192.168.2.3 via 10.0.0.1 dev eth0

Jika saya sudah membaca pertanyaan Anda dengan benar, saya rasa Anda ingin menambahkan

GATEWAY=192.168.2.3

ke / etc / sysconfig / network lalu reboot atau lakukan service network restart untuk mengujinya.

Meskipun saya tidak sepenuhnya yakin dari pertanyaan Anda bahwa Anda benar-benar menginginkan gateway default sama sekali. Saya pikir Anda mungkin hanya ingin membuat rute statis menjadi permanen?

Jika ada yang menjalankan CentOS dengan beberapa NIC, hanya 1 yang terhubung ke Internet dan Anda ingin membuat antarmuka khusus itu untuk mengambil semua lalu lintas Internet, lalu tambahkan baris berikut ke file /etc/sysconfig/network Anda:

GATEWAYDEV=network_interface
GATEWAY=default_gateway_ip

Simpan dan mulai ulang layanan jaringan Anda. Ini berhasil untuk saya dan tidak ada yang berhasil.

Anda mungkin dapat menambahkan baris itu ke /etc/ifconfig.INT yang sesuai (di mana "INT" mewakili antarmuka yang sesuai), seperti yang kadang-kadang kita lakukan dengan dhclient (perhatikan titik penjelasan utama yang dimasukkan sebagai karakter pertama):

!ip route add 192.168.2.3 via 10.0.0.1 dev eth0

(Saya belum mencoba CentOS, tetapi saya berasumsi bahwa mekanisme ini pada dasarnya harus sama seperti di NetBSD.)
