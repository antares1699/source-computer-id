Masalah dengan menggunakan alat sysprep untuk menjalankan Windows pada perangkat keras yang berbeda

Saya menggunakan alat sysprep untuk menjalankan Windows 7 pada perangkat keras yang berbeda. Apa yang saya lakukan adalah menjalankan sysprep di komputer, pilih Audit Sistem, centang kotak Generalize, pilih Shutdown, klik OK dan tunggu komputer untuk shutdown.

Ketika sistem dimatikan, saya menghapus hard disk dari komputer saya dan menyambungkannya ke komputer lain yang memiliki perangkat keras berbeda. Kemudian saya menyalakan komputer dan setelah serangkaian operasi (termasuk satu kali reboot), saya akhirnya sampai ke Desktop Windows pada komputer perangkat keras yang diubah TETAPI masalahnya adalah Alat Persiapan Sistem mulai secara otomatis. Saya mem-boot ulang komputer tetapi Alat Persiapan Sistem mulai setiap kali.

Satu hal lagi yang dicatat adalah bahwa komputer memberi pesan pada setiap reboot sebelum memuat Desktop bahwa "Sistem sekarang sedang mempersiapkan komputer Anda untuk penggunaan pertama".

Tahu bagaimana saya bisa mendapatkan Desktop yang bersih setelah melakukan sysprep? atau apakah ada langkah yang saya lewatkan?

Terima kasih banyak

Saya rasa masalahnya adalah Anda memilih Enter System Audit Mode ... Mode audit sistem adalah lingkungan khusus yang dirancang untuk menginstal program dan mengubah pengaturan sebelum pengguna akhir menerima sistem.

Selain itu, dengan memilih opsi Generalisasi, Anda menghapus pengaturan perangkat keras yang menyebabkan / memaksa Windows untuk memindai perangkat keras baru di setiap boot.

Anda hanya perlu memilih generalisasi saat Anda mengubah perangkat keras yang akan Anda gunakan untuk menggunakan Windows.

Terakhir, Juga, untuk keluar dari mode audit, pilih Enter system Out-Of-Box Experience (OOBE). Setiap kali Anda memilihnya, kali berikutnya Anda memulai Windows, Anda akan mendapatkan wizard, tetapi seharusnya cukup cepat.

Jadi - cukup pilih Enter system Out-Of-Box Experience (OOBE), dan centang Generalize tapi hanya jika Anda perlu mengganti komputer!
