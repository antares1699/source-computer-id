Vista (Bisnis) tidak akan mengunduh pembaruan Windows

Saya memiliki 3 PC Vista di kantor saya yang tidak dapat mengunduh Pembaruan Windows, baik secara otomatis maupun manual. Ketika saya mengklik "Periksa Pembaruan", saya mendapatkan pesan "Windows tidak dapat mencari pembaruan baru", kode kesalahan 80072EFD. Saya telah mencoba saran di artikel Bantuan dan KB yang ditautkan ke kesalahan (terutama mengenai firewall dan perangkat lunak antivirus), tetapi tidak berhasil. Saya menduga ada beberapa pengaturan di server saya (SBS2003) yang mencegah pembaruan ini, meskipun semua mesin XP di jaringan tidak memiliki masalah dengan pembaruan, hanya 3 Pemandangan. Adakah pemikiran tentang di mana saya dapat mencari pengaturan yang mungkin menyebabkan masalah ini? PC ini belum dapat diperbarui sejak pertama kali ditambahkan ke jaringan 2 tahun yang lalu (meskipun saya tidak menyadarinya hingga sekitar 6 bulan yang lalu, dan telah mencoba menyelesaikannya sejak saat itu). Terima kasih!

Microsoft memiliki artikel KB yang menjelaskan kesalahan ini serta beberapa kesalahan lainnya. Ini adalah KB 836941 Anda mungkin mengalami kesalahan terkait sambungan sementara saat menggunakan Pembaruan Windows atau Pembaruan Microsoft untuk menginstal pembaruan . Ada tujuh kemungkinan resolusi, jadi Anda mungkin perlu mencoba masing-masing hingga terselesaikan.

  • Aplikasi atau proses yang mengganggu komunikasi Internet
  • Masalah sumber daya di komputer Anda
  • Terlalu banyak aktivitas di situs pembaruan
  • Aktivitas Internet yang tinggi
  • Kesalahan database yang dapat dipulihkan
  • Masalah waktu habis server di Pembaruan Windows atau situs Pembaruan Microsoft

Saya memiliki masalah yang sama

Untuk memperbaikinya, saya membuka log pembaruan yang terletak di

%windir%\WindowsUpdate.log

Di sana saya menemukan

WARNING: SendRequest failed with hr = 80072efd. Proxy List used: 
         <127.0.0.1:8888> 
         Bypass List used : <(null)> Auth Schemes used : <>

Dan kemudian saya ingat saya telah menginstal Fiddler. Fidler digunakan sebagai proxy lokal untuk menampilkan lalu lintas HTTP, jadi ketika tidak dimulai, server proxy secara efektif tidak berfungsi dan itu mencegah Windows mengunduh pembaruan. Untuk menambah kebingungan, saya dapat mengakses situs web dan mengunduh sesuatu secara manual dari browser lokal. Hanya hal otomatis yang tidak berhasil dan saya salah menafsirkan ini sebagai semacam malware.

Saya memperbaiki Fiddler dengan menghapus centang pada kotak centang proxy
