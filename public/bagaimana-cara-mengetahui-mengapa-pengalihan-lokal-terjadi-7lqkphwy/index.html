Bagaimana cara mengetahui mengapa pengalihan lokal terjadi?

Beberapa waktu yang lalu saya yakin saya menambahkan pengalihan lokal untuk domain .dev ke localhost. Saya tidak dapat lagi mengunjungi domain .dev mana pun di browser mana pun pada mesin Mac saya. Namun sekarang saya ingin menghapus pengalihan ini, karena .dev adalah TLD asli.

Tapi saya tidak tahu dari mana logika pengalihan itu berasal.

File /etc/hosts saya tidak menyebutkannya:

127.0.0.1   localhost
255.255.255.255 broadcasthost
::1             localhost
fe80::1%lo0 localhost

127.0.0.1 figmadaemon.com

Dan /etc/resolv.conf saya juga tidak:

nameserver 192.168.1.1

Saya juga mencoba membersihkan cache DNS saya tanpa efek, dengan:

sudo killall -HUP mDNSResponder

Apakah ada tempat lain yang bisa membuatnya datang?

Salah satu cara umum di macOS untuk mengganti DNS biasa untuk satu (atau beberapa) domain level teratas adalah dengan membuat file di / etc / resolver / (misalnya / etc / resolver / dev), dan membuat daftar server alternatif sana. Lihat item Petunjuk Mac OS X ini dan jawaban ini tentang menggunakan dnsmasq dengan domain .dev . Jika di sanalah tempatnya, cukup hapus file (Anda dapat meninggalkan direktori atau menghapusnya jika Anda mau).
