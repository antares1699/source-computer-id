scp remoteA: file remoteB:

dengan scp saya bisa mendapatkan file

 scp remoteA:file .

dan meletakkan file

 scp file remoteB:

tetapi ketika saya mencoba

 scp remoteA:file remoteB:

saya mendapat

ssh: Could not resolve hostname remoteB: Name or service not known
lost connection

Mengapa?

EDIT:

deskripsi remoteA dan remoteB ada di ~ / .ssh / config

kemungkinan besar tidak terkait dengan scp itu sendiri, tetapi semacam resolusi nama dari satu host ke host lainnya. hal lain yang perlu dipertimbangkan adalah otentikasi. sebaiknya Anda memiliki kunci autentikasi tanpa frasa sandi dari A ke B. di mana host .ssh / config Anda mendefinisikan host ini? scp tidak menemukan definisi itu. coba fqdn sebenarnya dari host dan seharusnya berfungsi dengan baik, jika definisi host masalahnya. juga, gunakan -v untuk melihat apakah lebih banyak detail dapat ditemukan untuk kesalahan tersebut.

Anda perlu menentukan host yang dapat dijangkau oleh remoteA karena salinannya dibuat dari remoteA langsung ke remoteB.

Untuk menghindarinya, Anda dapat menggunakan opsi -3. man scp mengatakan apa yang dilakukannya:

Salinan antara dua host jarak jauh ditransfer melalui host lokal. Tanpa opsi ini, data akan disalin langsung antara dua host jarak jauh. Perhatikan bahwa opsi ini menonaktifkan pengukur kemajuan.

Bagi mereka yang datang ke sini karena scp menuduh file lokal sebagai nama host yang tidak dapat diselesaikan (sumber lain Nama atau layanan tidak diketahui ), seperti:

> scp archive_2017-03-01-09:54:02.tar.gz backupserver:/var/backups/
ssh: Could not resolve hostname archive_2017-03-01-09:54:02.tar.gz: Name or service not known

Solusi:

> scp ./archive_2017-03-01-09:54:02.tar.gz backupserver:/var/backups/
