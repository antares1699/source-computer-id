Pada Windows 7, apakah membantu menggunakan dua partisi pada hard drive untuk file halaman memori virtual?

Saya perhatikan kami dapat mengatur dua hard drive atau lebih untuk file halaman memori virtual. Tapi tebakan saya, jika itu adalah hard drive yang sama, hanya dua partisi yang berbeda, maka mungkin juga hanya satu partisi? (kami tidak ingin kepala hard drive bergerak melintasi dua partisi)?

Jika saya menggunakan dua hard drive ... satu internal dan satu eksternal, maka hard drive eksternal pada USB 2.0 mungkin juga tidak banyak membantu, karena kecepatan transfer hanya sekitar 30MB / s ... mungkin hard drive internal lain akan membantu, atau hard drive lain pada USB 3.0 mungkin membantu, yang saat ini tidak saya miliki, jadi mungkin tidak membantu?

Kecurigaan Anda benar: Menempatkan 2 file halaman pada dua partisi yang berbeda pada drive fisik yang sama menurunkan kinerja karena pencarian besar tambahan yang terlibat.

Saya tidak yakin apakah Windows akan membiarkan Anda meletakkan file halaman pada drive USB, tetapi USB 2.0 akan terlalu lambat.

File satu halaman per drive fisik paling baik untuk kinerja, dan Windows Vista dan yang lebih baru melakukan ini secara otomatis secara default, dan 2000 dan XP dapat dikonfigurasi untuk itu.

Semua ini mengatakan, saya akan melakukan cukup banyak swapping sehingga Anda khawatir tentang efek kinerja semacam ini, maka lebih banyak RAM akan menjadi solusi yang jauh lebih baik daripada menggunakan SSD kelas atas untuk penyimpanan file halaman.

Terkadang berguna untuk memindahkan file halaman ke partisi yang berbeda karena alasan ruang, tetapi bukan karena alasan kecepatan.

Memiliki pagefile di disk yang berbeda ke tempat sebagian besar aktivitas disk Anda terjadi adalah hal yang ideal. Seperti yang Anda harapkan, Anda ingin transfer cepat dari file halaman ke memori, jadi apa pun yang dapat Anda lakukan untuk mengurangi kemacetan akan membantu.

Menambahkan lebih banyak RAM pada akhirnya adalah cara terbaik untuk mengurangi masalah kinerja yang terkait dengan pertukaran file halaman.

Saya tidak bisa tidak setuju dengan alasan Anda. Drive internal atau USB 3 terpisah adalah cara yang tepat. (FYI, saya bertanya-tanya tentang latensi / kinerja IO acak melalui USB3 tetapi tampaknya baik-baik saja - tampaknya SSD dapat mencapai + 20MB / detik 4k acak melalui USB3, jadi jangan khawatir dengan HDD).

Tentu saja, cara yang tepat untuk menangani masalah pagefile adalah dengan mendapatkan lebih banyak RAM dan / atau meletakkannya di SSD.
