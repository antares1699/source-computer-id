Menangkan 7 tombol bilah tugas - gabungkan untuk beberapa aplikasi atau tidak untuk aplikasi lain

Apakah mungkin membiarkan tombol tugas dari beberapa aplikasi tidak pernah digabungkan dan yang lainnya selalu digabungkan?

Misalnya, jangan pernah menggabungkan untuk Outlook, Excel, dan Visual studio dan selalu gabungkan untuk yang lain.

Saya belum menemukan cara untuk mengotomatiskan ini, tetapi itu bisa dilakukan secara manual. Windows 7 dan yang lebih baru digunakan ID Model Pengguna Aplikasi (AppUserModelIDs) untuk menentukan jendela mana yang akan dikelompokkan / digabungkan dan mana yang akan diperlakukan secara terpisah. Anda dapat mengubah AppUserModelID (disingkat AppID) dari proses yang sedang berjalan, dan utilitas praktis yang dapat melakukan ini dan banyak lagi lainnya adalah 7+ Tweaker Taskbar (dapat dipasang dalam mode portabel).

  1. Katakanlah bilah tugas Anda terlihat seperti ini:

    Seperti yang Anda lihat, tombol Windows Explorer dikelompokkan / digabungkan secara default karena memiliki AppID yang sama.

  2. Sekarang mulailah 7+ Tweaker Taskbar dan klik Inspektur Bilah Tugas tombol untuk meluncurkannya:

  3. Seperti yang Anda lihat pada gambar di bawah, dua jendela Explorer dikelompokkan di bawah explorer.exe karena mereka berbagi AppID yang sama:

  4. Klik kanan salah satu entri Explorer, buka Change Application ID dan atur ke nilai acak yang ditampilkan (lihat gambar di atas).

  5. Voilà, Anda akan melihat bahwa tombol Penjelajah di bilah tugas segera dipisahkan:

    Jika Anda memiliki Aero Peek dihidupkan, Anda dapat mengonfirmasi dengan mengarahkan kursor ke salah satu tombol bilah tugas Explorer. Sekarang Anda akan melihat bahwa tidak ada lagi pratinjau yang menampilkan jendela lain.

Anda dapat menggunakan trik yang sama untuk mengelompokkan / menggabungkan secara sewenang-wenang apa saja kumpulan jendela aplikasi. Misalnya, Anda dapat mengelompokkan / menggabungkan jendela Explorer dengan jendela Outlook, jika diinginkan.

Sayangnya, belum ada cara untuk menyelamatkan negara, jadi seperti yang saya katakan di atas, melakukannya secara manual itu mungkin, tetapi menyakitkan jika harus dilakukan berulang kali. Saya kira ini harus menjadi isyarat bagi beberapa programmer yang giat untuk segera membangun utilitas otomatisasi seperti itu! :)
