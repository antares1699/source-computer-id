Mengapa kecepatan transfer file menurun saat menyalin beberapa file yang lebih kecil?

Katakanlah saya menyalin direktori aplikasi portabel dari PC ke HDD eksternal. File application.exe berukuran sekitar 4 megabyte, tetapi kebanyakan file lain hanya berukuran puluhan kilobyte, dan sebagian besar file tersebut membutuhkan waktu lebih lama untuk disalin / dipindahkan daripada file yang lebih besar yang memiliki 4 megabyte.

Kenapa ini?

Saat mentransfer data, jika Anda memindahkan satu file besar, Anda hanya perlu menulis entri direktori satu kali dan kemudian mengalirkan data dan menulis lebih atau kurang terus menerus. Ketika Anda memiliki banyak file, Anda menulis info direktori, pindah ke area file dan menulis datanya, lalu kembali dan tulis entri direktori untuk file berikutnya dan kemudian pindah kembali ke area data dan tulis data dan bolak-balik . Overhead dari gerakan bolak-balik bertambah.

Jika Anda mentransfer file, ada open + read + close di sumbernya, dan open + write + close di tujuan, dengan semua atribut terkait (ctime / atime / mtime dll) dibuat di tempat tujuan. Fragmentasi (waktu pencarian yang lebih tinggi) juga akan memengaruhi datarate. Kemacetan mis. saluran sata juga dapat menjadi masalah (gunakan port terpisah untuk setiap disk, beli kartu sata jika kehabisan). File yang lebih besar memiliki operasi baca atau tulis yang lebih lama, dan akan ditransfer lebih cepat. Saya yakin ada lebih dari itu. Apakah ini membantu Anda?

Begitulah cara kerja windows, dan itulah bagian yang buruk pada HDD. Biasanya SSD pandai menyalin file kecil. Anda dapat memeriksa pembandingan HDD dan SSD tentang kecepatan transfer file. Tetapi Anda harus tahu bahwa ada hal-hal lain yang ada, seperti windows membaca file terlebih dahulu, kemudian antivirus Anda memeriksanya, dan kemudian mereka menulis ke folder tujuan yang baru. Belum lagi ketika Anda memotong file, proses penghapusan ditambahkan. Jadi, lebih banyak file, lebih banyak waktu "nyata" ditambahkan untuk menyalin lebih banyak file.

Karena data yang ditulis ke (eksternal) hdd tidak di-cache, tetapi ditulis langsung ke hdd, dan drive usb juga harus menjalankan indeks (lemak, tabel alokasi) untuk setiap file, jadi harus menggaruk hdd kembali dan seterusnya dari data file ke data alokasi file. Karena jika gagal saat di-cache, file akan hilang, hdd rusak, dll. Drive internal dapat di-cache. Kemudian bisa seperti burst file ke hdd. Bahkan mereka melakukannya untuk memperlambat Anda, seperti seluruh bisnis komputer komersial. Driver lebih lanjut pada drive usb hdd tidak intelligen, juga bukan hdd tersebut. Orang ingin barang murah.
