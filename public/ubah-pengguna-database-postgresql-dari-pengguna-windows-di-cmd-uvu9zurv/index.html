Ubah pengguna database postgreSQL dari pengguna windows di CMD

Saya baru-baru ini menginstal PostgreSQL di windows 7, setiap kali saya mencoba menggunakan CMD untuk menjalankan perintah SQL, saya mendapatkan kesalahan ini

createdb: could not connect to database template1: FATAL: password authentication failed for user "R-61"

Di mana R-61 adalah nama pengguna mesin windows saya.

Berikut adalah cara saya menggunakan CMD dengan pgSQL

  • pertama saya cd ke folder bin pgSQL $ cd C:\xampp\pgsql\9.1\bin
  • saya mencoba membuat database $ createdb NEWDATABASE

  • saya mendapat prompt untuk memasukkan kata sandi. di sinilah saya bingung karena saya tidak tahu kata sandi apa. Saya telah mencoba kata sandi pengguna jendela saya, kata sandi untuk pengguna super postgres, sepertinya tidak ada yang berfungsi. ini adalah pesan kesalahan yang saya dapatkan createdb: could not connect to database template1: FATAL: password authentication failed for user "R-61"

    Ada yang tahu solusi untuk ini? Saya berpikir mengubah bentuk pengguna R-61 menjadi postgres superuser akan berhasil tetapi saya tidak tahu bagaimana melakukannya.

Bantuan Anda sangat dibutuhkan.

createdb adalah perintah yang harus terhubung ke server seperti perintah klien lainnya (bahkan ketika dijalankan di server itu sendiri), dan karena itu memerlukan pengguna database untuk terhubung.

Jika Anda tidak menentukan pengguna database dengan opsi -U username, ini mengambil nama pengguna OS Anda, maka referensi ke "R-61" dalam pesan kesalahan. Kata sandi yang diminta adalah kata sandi dari pengguna database R-61 yang tidak ada.

Ada juga opsi lain seperti host default ke localhost dan port default ke 5432, lihat halaman manual untuk semuanya.

Karena Anda menyebutkan bahwa Anda mengetahui kata sandi superuser postgres, solusi paling sederhana adalah menggunakan pengguna postgres untuk memulai perintah. Ini masuk akal juga, karena pada penginstalan baru, pengguna ini akan menjadi satu-satunya pengguna yang memiliki izin untuk membuat database.

Coba createdb -U postgres newdatabase

Itu harus meminta kata sandi, dan itu akan menjadi kata sandi dari pengguna database postgres.

Jika tidak meminta kata sandi, itu berarti kebijakan keamanan di pg_hba.conf dikonfigurasi seperti itu. Jika kata sandi hilang (atau tidak ada kata sandi yang pernah disetel, yang merupakan kasus untuk beberapa penginstalan otomatis, terutama di Unix), pg_hba.conf harus diubah dan layanan dimuat ulang untuk mengotorisasi koneksi tanpa kata sandi, setidaknya untuk sementara waktu untuk menyetel kata sandi baru .

Saya tidak memiliki pengalaman langsung dengan ini tetapi saya telah memeriksanya dan saya pikir inilah yang Anda cari. Saya menemukan masalah serupa yang menunjukkan bahwa PostgresSQL di Windows meminta kata sandi saat menginstal. Kata sandi itu mungkin kata sandi untuk akun administrator untuk PostgresSQL. Akun admin harus bekerja untuk menyambungkan ke database itu dan menyesuaikan izin untuk itu.

Atau, mengedit file pg_hba.conf untuk mempercayai koneksi pengguna dari pengguna dan lokasi tertentu akan memungkinkan Anda terhubung juga. Anda akan ingin memperbaiki masalah izin dan mengembalikan baris pg_hba.conf yang telah diedit.

Diskusi yang saya temukan ini di: Re: Kata sandi default Dokumentasi pada file pg_hba.conf: 19.1. File pg_hba.conf
