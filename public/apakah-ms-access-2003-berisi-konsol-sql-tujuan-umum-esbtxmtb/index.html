Apakah MS Access 2003 berisi konsol SQL tujuan umum

Saya tahu Anda bisa membuka konsol SQL di Access saat membuat kueri atau tampilan, tetapi apakah ada konsol 'semua tujuan' yang memungkinkan saya untuk mengeksekusi pernyataan seperti:

CREATE TABLE Foo
(
Foo_Id int NOT NULL,
Name varchar(255) NOT NULL,
CONSTRAINT pk_FooID PRIMARY KEY (Foo_Id,Name)
)

Saya tidak tahu apa yang Anda maksud dengan "konsol serba guna," tetapi mungkin di dalam Access, Jendela Segera yang akan melayani kebutuhan Anda. Jika Anda menekan Ctrl-G pada keyboard dari jendela Access utama, itu akan membawa Anda ke Jendela Segera di VBE dan di sana Anda dapat mengetik:

  CurrentDB.Execute "CREATE TABLE Foo (Foo_Id int NOT NULL, Name varchar(255) NOT NULL, CONSTRAINT pk_FooID PRIMARY KEY (Foo_Id,Name))"

Ini adalah antarmuka baris tunggal, jadi tidak ada pemformatan SQL, tetapi itu akan mengeksekusi SQL.

Masalah dengan menggunakan CurrentDB sebagai objek eksekusi adalah ia mengembalikan objek database baru setiap kali Anda memanggilnya (untuk memastikan bahwa semua koleksinya adalah yang terbaru). Alternatifnya adalah:

  DBEngine(0)(0).Execute "CREATE TABLE Foo (Foo_Id int NOT NULL, Name varchar(255) NOT NULL, CONSTRAINT pk_FooID PRIMARY KEY (Foo_Id,Name))"

Karena DBEngine (0) (0) selalu mengembalikan objek yang sama, Anda kemudian dapat memeriksa hal-hal seperti:

  DBEngine(0)(0).RecordsAffected

Masalahnya adalah jika Anda mengubah koleksi database, Anda harus ingat untuk menyegarkannya sebelum Anda menggunakannya. Misalnya, jika Anda menambahkan atau menghapus tabel, Anda harus merefresh koleksi TableDefs ke koleksi menjadi yang terbaru:

  DBEngine(0)(0).TableDefs.Refresh

Alternatif lain adalah dengan menggunakan objek database yang di-cache, seperti yang saya lakukan di semua aplikasi saya, tetapi itu tidak menghindari masalah penyegaran koleksi, tetapi hanya membuatnya lebih mudah untuk mengetik (saya merasa mengetik DBEngine (0) (0) menjadi cukup rawan kesalahan, dan merasa fungsi dbLocal saya jauh lebih mudah digunakan).
