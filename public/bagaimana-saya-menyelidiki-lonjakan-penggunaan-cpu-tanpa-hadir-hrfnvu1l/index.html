Bagaimana Saya Menyelidiki Lonjakan Penggunaan CPU Tanpa Hadir?

Saya memiliki server yang secara misterius menghabiskan banyak CPU di malam hari, padahal seharusnya tidak di bawah beban berat. Hal ini menyebabkan beberapa panggilan database ke waktu habis, dan saya telah menyiapkan beberapa peringatan di PerfMon untuk memicu jika penggunaan CPU melebihi 50% dan ini tampaknya bertepatan dengan lonjakan CPU yang bertepatan dengan waktu habis database.

Saya menduga bahwa ini adalah semacam tugas terjadwal, tetapi saya telah melihat daftar tugas terjadwal Windows dan SQL Server dan tidak menemukan apa pun. Log peristiwa Windows hanya berisi waktu habis database dan entri PerfMon yang menunjukkan bahwa penggunaan CPU telah melonjak.

Pertanyaan saya adalah: Bagaimana cara menentukan proses apa yang menyebabkan lonjakan CPU tanpa ada secara fisik? Saya benar-benar tidak ingin masuk ke server pada jam 2 pagi.

Memperbarui : Ternyata masalahnya ada pada produk yang sedang saya kerjakan. Masalahnya mulai terwujud selama jam kerja normal, dan saya tidak harus bergantung pada teknik investigasi kinerja yang licik.

Berikut beberapa opsi; yang paling mudah adalah Process Explorer jika penggunaan CPU signifikan, jika tidak, Anda kemungkinan besar ingin melihat grafik di Process Monitor. Saya merasa bahwa kemungkinan besar penyebabnya adalah cadangan atau pembaruan, jadi Anda mungkin ingin melihat kapan itu dijadwalkan. Penjadwal tugas tidak ditampilkan saat tugas terakhir dijalankan ...

Tanpa jejak

  • Proses Explorer , periksa siklus CPU yang dikonsumsi oleh proses Anda.

Berdasarkan suatu peristiwa

  • ProcDump dapat secara otomatis membuat file dump berdasarkan hal-hal seperti penggunaan CPU yang tinggi.

    Keuntungan memiliki dump adalah Anda bisa menjadi pengembang debug di WinDBG .

Dengan jejak

  • Monitor Proses , setel untuk menyimpan banyak riwayat dan biarkan hanya mencatat peristiwa pembuatan profil.

    1. Dalam Opsi, Anda dapat mengatur kedalaman riwayat dan dapat mengaktifkan pembuatan peristiwa profil.

    2. Kemudian, nonaktifkan tombol I / O, Registri, dan Jaringan.

  • Toolkit Kinerja Windows memiliki XPerf, yang memungkinkan Anda untuk mengambil jejak lanjutan.

    Ini mungkin sulit untuk pertama kalinya, jadi Anda mungkin ingin mencoba Monitor Proses terlebih dahulu ...
