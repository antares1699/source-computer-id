find -exec harus diakhiri dengan \; atau +?

Menurut opsi man find -exec harus diakhiri dengan ;.

Ekspresi harus diakhiri dengan titik koma (``; '').

Tetapi ketika saya menggunakan find-grep di Emacs, ekspresi diakhiri dengan +. Yang mana yang harus saya gunakan? Apakah ada perbedaan dua ekspresi?

Ini dari manual untuk GNU find:

   -exec command ;
          Execute command; true if 0 status is returned.
          All following arguments to find are  taken  to
          be  arguments to the command until an argument
          consisting of `;'

Dipotong oleh saya. Tapi kami juga punya:

   -exec command {} +
          This variant of  the  -exec  action  runs  the
          specified  command  on the selected files, but
          the command line is built  by  appending  each
          selected  file name at the end; the total num‐
          ber of invocations of the command will be much
          less  than  the  number of matched files.  The
          command line is built in  much  the  same  way
          that xargs builds its command lines.

Dipotong oleh saya.

Jadi, ; dan + melakukan hal yang berbeda. Contoh singkat dalam kode untuk didemonstrasikan:

$ mkdir test
$ touch test/{a,b}
$ find test -exec echo foo {} \;
foo test
foo test/a
foo test/b
$ find test -exec echo foo {} +
foo test test/a test/b

+ hanya menghasilkan satu pemanggilan dengan semua file yang cocok diberikan sebagai daftar argumen (dalam praktiknya berperilaku seperti xargs). ; menjalankan perintah satu kali untuk setiap file yang cocok (; harus di-escape di shell; oleh karena itu \ sebelumnya).

Perhatikan bahwa beberapa versi ; tidak memiliki opsi +, tetapi versi GNU kemungkinan besar diinstal pada Linux yang tidak tertanam.

TL; DR:

Menggunakan + dalam -exec sama dengan -exec, kecuali bahwa {} diganti dengan nama jalur sebanyak mungkin untuk setiap pemanggilan utilitas .


Sedikit lebih lama dari man find di komputer saya:

   -exec utility [argument ...] ;
             True if the program named utility returns a zero value
             as its exit status.  Optional arguments may be passed
             to the utility.
             The expression must be terminated by a semicolon.  If you
             invoke find from a shell you may need to quote the 
             semicolon if the shell would otherwise treat it as a control
             operator.  If the string ``{}'' appears anywhere in the 
             utility name or the arguments it is replaced by the pathname
             of the current file.
             Utility will be executed from the directory from which find was
             executed.  Utility and arguments are not subject to the further
             expansion of shell patterns and constructs.


     -exec utility [argument ...] {} +
             Same as -exec, except that `{}` is replaced with as many 
             pathnames as possible for each invocation of utility.  
             This behaviour is similar to that of xargs(1).

Perhatikan paragraf kedua. Jangan berhenti membaca setelah paragraf pertama menjelaskan -exec.
