Seberapa pentingkah dual-gigabit lan untuk NAS rumah pengguna super?

Singkat cerita : Saya membangun server rumah saya sendiri berdasarkan Ubuntu dengan 4 drive di RAID 10. Tujuan utamanya adalah NAS dan cadangan.

Apakah saya akan membuat kesalahan besar dengan membangun Server NAS dengan satu Gigabit NIC?

Cerita panjang : Saya tahu maks absolut yang bisa saya keluarkan dari satu port Gigabit adalah 125MB / s, dan saya ingin NAS ini dapat menangani hingga 6 komputer mengakses file secara bersamaan, dengan maksimal dua di antaranya streaming video .

Dengan Ubuntu NIC-bonding dan kinerja RAID 10, saya secara teoritis dapat menggandakan throughput saya dan mencapai 250MB / s (ok, tidak juga, tapi akan lebih cepat). Itu drive memiliki throughput baca rata-rata 83.87MB / s menurut Perangkat Keras Tom .

Unit itu sendiri akan didasarkan pada Casing Chenbro ES34069-BK-180 . Dengan pilihan perangkat keras saya saat ini, ia akan melakukannya motherboard ini dengan CPU Core i3 dan RAM 8GB. Berlebihan, saya tahu, tetapi server ini akan melakukan hal-hal lain juga (seperti video transcoding).

Sayangnya, satu-satunya papan Mini-ITX yang dapat saya temukan dengan dual-gigabit dan 6 port SATA adalah berbasis Intel Atom, dan saya membutuhkan lebih banyak daya pemrosesan daripada yang ditawarkan Atom.

saya akan cinta untuk menemukan papan dengan 6 port SATA dan dua port Gigabit LAN yang mendukung a CPU Core i3 . Sejauh ini, pencarian saya kosong. Jadi, dilema saya.

Haruskah saya bertahan untuk papan seperti itu, menggunakan solusi berbasis Atom, atau tetap menggunakan konfigurasi gigabit tunggal saya saat ini?

Saya tahu ada unit NAS konsumen dengan hanya satu antarmuka gigabit (mungkin sebagian besar), tetapi saya pikir saya akan menuntut lebih banyak dari server saya daripada pengguna rumahan rata-rata.

Setiap saran dihargai. Terima kasih.

Konfigurasi Anda kemungkinan tidak memerlukan dual-GbE, terutama jika Anda menggunakan drive 5,4K RPM seperti yang disarankan bagan. Anda akan melihat throughput maksimum saat mengambil file besar yang disimpan secara berurutan (tidak terfragmentasi). Kecuali Anda mengambil langkah untuk menjaga urutan file Anda, mungkin dengan menggunakan XFS, kecepatan GigE tidak mungkin dalam konfigurasi ini. Random I / O, yang lebih mungkin, bahkan tidak akan mendekati kecepatan GigE, itulah sebabnya hampir semua head Home / SOHO NAS hadir dengan satu port GigE.

Anda mengatakan Anda menginginkan dua NIC sehingga Anda dapat mendukung dua salinan kecepatan GigE pada saat yang bersamaan. Jika dua klien menyalin pada saat yang sama, drive harus menyajikan data dari dua bagian drive yang berbeda. Pola akses I / O akan 'acak' dalam kasus itu, dan kecepatan agregat yang dilihat oleh kedua klien, bahkan dalam skenario ikatan NIC, tidak akan mendekati GigE. Anda tidak memiliki tempat di dekat drive-count untuk itu.

Anda tidak memerlukan dua NIC, jadi gunakan motherboard Mini-ITX yang mendukung CPU yang Anda butuhkan.
