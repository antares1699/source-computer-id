Menggunakan SSD sebagai RAM Linux

Saya memiliki SSD OCZ Revo yang sangat cepat. Saya ingin menggunakan banyak RAM tetapi saya terbatas pada RAM 4GB untuk saat ini. Jika saya menggunakan SSD OCZ Revo saya sebagai swap (keseluruhannya, 160GB), apakah itu berarti saya memiliki total ukuran RAM 164GB secara efektif?

Pertanyaan saya adalah, bagaimana linux memperlakukan swap? Bagaimana saya bisa menggunakan SSD saya sebagai RAM dengan RAM yang sebenarnya bersama-sama? Ada yang tahu?

Anda dapat melakukan operasi apa pun yang dapat Anda lakukan dengan RAM 164GB. Tetapi karena SSD ratusan kali lebih lambat dari RAM, ini akan memakan waktu lebih lama.

Anda akan ingin mengubah sistem swappiness up sehingga Anda dapat menggunakan pertukaran SSD yang lebih cepat untuk memperluas ukuran cache halaman secara efektif. Jika tidak, sistem akan menganggap swap dan disk hampir sama cepatnya dan tidak memindahkan sesuatu ke swap ketika dapat membacanya dari disk, yang tidak masuk akal dalam situasi Anda yang tidak biasa.

Jika Anda menemukan bahwa Anda memiliki banyak I / O disk dan sangat sedikit swap yang digunakan, naikkan swappiness. Jika Anda merasa sedang "mengaduk" SSD, putar swappiness turun. Perhatikan bahwa ini akan memiliki beberapa efek negatif kecil pada masa pakai SSD Anda - semakin tinggi swappiness, semakin besar efeknya. (Dengan SSD modern, hampir tidak masalah. Tidak ada gunanya memiliki SSD jika Anda tidak menggunakannya.)

Linux menggunakan swap sebagai partisi. Saya memiliki partisi 18GB di HDD saya yang saya format dengan GParted menjadi "linux-swap" Saya menambahkannya ke / etc / fstab dan berfungsi dengan baik (hanya v. Lambat) itu akan menjadi banyak lebih cepat menggunakan SSD, terutama yang PCIe, namun menggunakan SSD sebagai swap dapat menurunkannya dan mempersingkat masa pakai SSD, karena SSD memiliki jumlah tulis yang terbatas ..

https://wiki.archlinux.org/index.php/Solid_State_Drives#Swap_Space_on_SSDs
