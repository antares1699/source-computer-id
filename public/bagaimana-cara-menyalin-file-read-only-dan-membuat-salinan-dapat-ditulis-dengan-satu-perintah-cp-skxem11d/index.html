Bagaimana cara menyalin file Read-Only dan membuat salinan dapat ditulis dengan satu perintah cp?

Bagaimana cara menyalin file Read-Only di Linux dan membuat salinan dapat ditulis dengan satu perintah cp di Linux (Ubuntu 10.04)?

--No-preservasi dan --preserve tampaknya menjadi kandidat yang baik, kecuali bahwa mereka harus "dan" menandai mode, sementara yang saya cari adalah sesuatu yang akan "atau" mereka (mode tambah + w).

Keterangan lebih lanjut:

Saya harus mengimpor repositori dari GIT ke Perforce. Saya ingin semua file depot Perforce adalah Read-Only (begitulah desain Perforce), sementara semua file lain yang diturunkan / disalin dari file depot dapat ditulis. Saat ini jika Makefile mencoba menyalin file Read-Only maka file turunan juga akan menjadi Read-only. Hal ini menyebabkan error build saat cp mencoba menimpa file Hanya-Baca untuk kedua kalinya. Tentu saja --force adalah solusi di sini, tetapi file turunannya juga menjadi Hanya-Baca. Juga saya tidak ingin mengacaukan "chmod" setelah setiap perintah "cp" - saya akan melakukannya hanya sebagai pilihan terakhir.

Anda harus melihat file Install perintah.

Dari man install

SYNOPSIS
       install [OPTION]... [-T] SOURCE DEST
       install [OPTION]... SOURCE... DIRECTORY
       install [OPTION]... -t DIRECTORY SOURCE...
       install [OPTION]... -d DIRECTORY...

DESCRIPTION
       This  install  program copies files (often just compiled) into 
       destination locations you choose.  If you want to download and install
       a ready-to-use package on a GNU/Linux system, you should instead be 
       using a package manager like yum(1) or apt-get(1).

       In the first three forms, copy SOURCE to DEST or multiple SOURCE(s) to
       the existing DIRECTORY, while setting permission modes and owner/group.
       In  the  4th form, create all components of the given DIRECTORY(ies).
...
       -m, --mode=MODE
          set permission mode (as in chmod), instead of rwxr-xr-x
