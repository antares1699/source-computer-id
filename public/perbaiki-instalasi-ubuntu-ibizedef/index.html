Perbaiki instalasi Ubuntu

Saya tidak sengaja menghapus (apt-get purge) banyak hal sistem penting, termasuk gnome dekstop dan apa pun yang bertanggung jawab untuk koneksi internet, antara lain. Apakah ada cara untuk memperbaikinya? Tanpa instal ulang lengkap?

Saya dapat mengaksesnya melalui baris perintah (terminal) lebih atau kurang normal. memindahkan, menyalin file, dll., tidak perlu menginstal apt-get karena tidak ada koneksi internet. Jadi saya kira memperbaiki internet akan menjadi prioritas pertama ... Adakah cara untuk memeriksa mengapa tidak berhasil dan apa yang perlu dilakukan?

Saya memiliki LiveCD tempat saya dapat boot. Jadi saya dapat menyalin file yang diperlukan secara manual darinya, jika ada.

Terima kasih sebelumnya atas bantuan Anda

Sunting: setelah saya boot dari LiveCD saya, saya dapat menggunakan internet, namun, segera setelah saya melakukan chroot ke sistem file nyata, saya tidak dapat menggunakan internet dari sana. Tebak itu cara yang diharapkan untuk bekerja ...

Edit 2 : bagaimana saya memperbaikinya: dengan bantuan orang-orang di sini, saya akhirnya berhasil. Saya mem-boot dari liveCD, menyalin /etc/resolv.conf ke instalasi asli, melakukan chroot di sana, apt-get install ubuntu-desktop. Meskipun penginstalan gagal karena banyak kesalahan, file tetap diunduh ke dalam penginstalan nyata. Jadi saya me-reboot mesin dari live cd ke ubuntu asli, dan kali ini ketika saya masih tidak memiliki internet, apt-get memiliki cache ubuntu-desktop. Jadi saya menginstal ubuntu-desktop dan mem-boot ulang dengan GUI kali ini. Setelah beberapa perbaikan manual kecil (jika updown berhasil) saya juga bekerja nirkabel, sekarang ini hanya masalah pembaruan otomatis, dll., Dengan sinaptik. Fiuh .... terima kasih sekali lagi semuanya :) Terima kasih khusus kepada ~ dukun yang tidak memberikan jawaban "formal" yang dapat saya terima secara resmi, tetapi paling membantu dengan komentar :)

jika Anda dapat menjalankan utilitas ifconfig (sebagai root), dan memiliki adaptor ethernet tunggal yang dapat menggunakan DHCP untuk mengakses jaringan:

sudo ifconfig eth0 up
sudo dhclient eth0 &

Itu harus mengaktifkan adaptor ethernet pertama Anda dan menjalankan dhclient untuk mendapatkan alamatnya melalui DHCP. Jika berhasil, Anda mungkin dapat melakukan perbaikan instalasi Anda dengan menjalankan:

sudo aptitude install ubuntu-desktop

Perintah itu akan menginstal paket yang disertakan dalam instalasi Ubuntu dasar. (Anda dapat mengganti "kubuntu-desktop" jika Anda menggunakan KDE atau "xubuntu-desktop" jika Anda menggunakan XFCE.

Jika Anda tidak bisa mendapatkan jaringan sama sekali, cara terbaik adalah seperti yang disarankan Justin di atas, gunakan apt-cdrom untuk menginstal paket ubuntu-desktop dari CD Anda.

Anda dapat mengkonfigurasi apt untuk menggunakan install cd sebagai repositori, dan kemudian menginstal apa yang Anda butuhkan untuk akses internet, dan gnome, dari cd.

sudo apt-cdrom -d <mountpoint> add adalah perintah yang Anda inginkan untuk ini. pastikan mountpoint adalah mountpoint cd-rom yang terdaftar di / etc / fstab Anda.
