Saya menyadari ini adalah pertanyaan lama, tapi ...

Saya percaya Gargoyle (dan banyak firwares pihak ketiga lainnya untuk router) digunakan dnsmasq.dll untuk layanan DNS / DHCP mereka, yang secara default memberikan IP berdasarkan hash dari alamat MAC perangkat. Dokumentasi memberikan sedikit lebih banyak detail:

--dhcp-sequential-ip

Dnsmasq dirancang untuk memilih alamat IP untuk klien DHCP menggunakan hash dari alamat MAC klien. Ini biasanya memungkinkan klien alamat agar tetap stabil dalam jangka panjang, bahkan jika klien kadang-kadang mengizinkan sewa DHCP-nya kedaluwarsa. Dalam mode default ini, alamat IP adalah didistribusikan secara semu secara acak di seluruh rentang alamat yang tersedia. Terkadang ada keadaan (biasanya penerapan server) di mana akan lebih mudah jika alamat IP dialokasikan secara berurutan, mulai dari alamat terendah yang tersedia, dan pengaturan bendera ini mengaktifkan mode ini. Perhatikan bahwa dalam mode sekuensial, klien yang memungkinkan sewa untuk kedaluwarsa jauh lebih mungkin untuk memindahkan alamat IP; untuk alasan ini sebaiknya tidak digunakan secara umum.
