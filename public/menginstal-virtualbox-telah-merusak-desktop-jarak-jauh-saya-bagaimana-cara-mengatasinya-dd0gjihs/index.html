Menginstal VirtualBox telah merusak desktop jarak jauh saya - bagaimana cara mengatasinya?

Baru-baru ini saya menginstal VirtualBox (dan dukungan jaringan yang menyertainya). Semuanya bekerja dengan baik, jauh lebih bagus daripada VM lama yang saya gunakan.

Namun, sejak menginstal, saya tidak dapat lagi melakukan remote desktop ke komputer saya!

Saya telah memeriksa untuk memastikan bahwa alamat IP jaringan saya tidak berubah, dan bahwa router saya masih diatur untuk mengizinkan desktop jarak jauh ke pc saya, dan, bagaimanapun, saya masih tidak dapat terhubung.

Pernahkah seseorang menjumpai masalah ini? Bagaimana cara mengatasinya?

Apakah ada VM Anda yang mengaktifkan tampilan jarak jauh? Ini mengikat secara default ke port desktop jarak jauh Windows standar (3389). Jika VM menggunakan port ini, Windows tidak akan dapat menggunakannya untuk desktop jarak jauh. Lihat manual untuk diskusi lebih lanjut tentang cara mengubah port.

Saya mengalami masalah yang sama - menambahkan jawaban di sini karena penyebab dan solusi bagi saya berbeda dengan yang tercantum di atas.

Saya menemukan bahwa "VirtualBox Host-Only Network Adapter" mencegah saya menggunakan remote desktop ke host. Ini juga mengganggu akses cadangan jaringan perusahaan ke host itu.

Solusi yang harus saya ikuti http://brianreiter.org/2010/09/18/fix-virtualbox-host-only-network-adapter-creates-a-virtual-public-network-connection-that-causes-windows-to-disable-services/

Artikel itu menjelaskan perubahan registri untuk menetapkan *NdisDeviceType ke DWORD nilai 1 di pengaturan adaptor untuk VirtualBox Host-Only Network Adapter. Pengaturan adaptor jaringan dapat ditemukan di bawah HKLM:\system\CurrentControlSet\control\class\{4D36E972-E325-11CE-BFC1-08002BE10318} di registri.

*NdisDeviceType didokumentasikan di msdn ( http://msdn.microsoft.com/en-us/library/ff557037%28VS.85%29.aspx ); nilai 1 yang diperlukan digunakan untuk menunjukkan bahwa adaptor jaringan tidak tersambung ke jaringan eksternal. Tanpa nilai yang ditetapkan, kehadiran 'jaringan tak dikenal' membuat Windows menjadi lebih agresif tentang firewall, oleh karena itu saya melihat gangguan pada desktop jarak jauh dan akses cadangan.

Perhatikan bahwa saya harus menerapkan kembali perubahan ini setiap kali saya memutakhirkan VirtualBox. Mendesah.

Bagikan

Saya juga menginstal VirtualBox dari jarak jauh dan itu memutuskan koneksi Remote Desktop saya.

Pengalaman saya begini: Firewall Windows sepertinya telah disetel ulang.

Satu solusi yang dapat saya berikan kepada Anda adalah ini: Hubungkan keyboard, mouse, dan layar Anda ke mesin jarak jauh dan atur ulang firewall lagi. Dalam kasus saya, HDTV saya tidak memiliki input VGA dan saya tidak memiliki monitor VGA, jadi saya harus memasukkan kartu video saya dengan adaptor DVI-ke-HDMI untuk mengontrol server saya secara langsung.

Ini menyebalkan, tetapi tampaknya itu satu-satunya cara KECUALI ada cara untuk mengkonfigurasi firewall dari jarak jauh (tetapi firewall adalah firewall dan saya ragu ini akan terjadi).

PS: Sementara dalam subjek firewall dan subjek yang berbeda dari Remote Desktop dan VirtualBox, dari komputer saya di jaringan internal, saya mencoba mengakses situs web saya di server saya dan port 80 telah diblokir. Ini setelah saya menginstal VirtualBox dengan dukungan jaringan.
