Menggunakan ADB untuk mengirim sentuhan acara ke telepon, tetapi tidak dapat melepaskan

Saya memiliki HTC 10 yang menjalankan CM13 Saya mencoba mengirim acara sentuh ke sana melalui ADB dan Kirim Acara Setelah mengirim ini:

adb shell
sendevent /dev/input/event6 3 57 0
sendevent /dev/input/event6 3 53 300
sendevent /dev/input/event6 3 54 400
sendevent /dev/input/event6 3 48 5
sendevent /dev/input/event6 3 58 50
sendevent /dev/input/event6 0 2 0
sendevent /dev/input/event6 0 0 0

Saya mendapatkan klik di telepon, dengan showtouches aktif, saya dapat melihat sentuhan telah dimulai, tetapi sentuhan tersebut masih aktif di telepon.

Berdasarkan bacaan saya sini dan sini seperti halnya tempat lain, sepertinya yang harus saya lakukan hanyalah:

sendevent /dev/input/event6 3 57 -1
sendevent /dev/input/event6 0 2 0
sendevent /dev/input/event6 0 0 0

atau bahkan mungkin hanya

sendevent /dev/input/event6 3 57 -1
sendevent /dev/input/event6 0 0 0

namun, tidak satu pun dari perintah ini yang menyebabkan sentuhan diakhiri di telepon.

Saya mencoba meletakkan semua ini pada skrip shell di ponsel itu sendiri, dan menjalankan skrip dari shell adb, namun itu menghasilkan hal yang sama. Awal dari sentuhan, dan bukan akhir.

Meskipun tidak memiliki lingkungan Anda, saya telah memeriksa masalah tersebut. Ada berbagai artikel yang bisa ditemukan, seperti yang Anda lakukan, yang sepertinya selalu begitu bekerja untuk beberapa orang tetapi tidak untuk orang lain. Ini adalah kesimpulan saya bahwa ada perbedaan antara kombinasi phone-hardware / Android-version / SDK-version yang membuatnya jadi tidak ada satu solusi untuk semua pengaturan.

Anda harus mencari tahu apa yang cocok untuk Anda. mengingat bahwa jurusan pembaruan perangkat lunak selalu dapat mengembalikan Anda ke titik awal.

Metode paling sederhana yang saya temukan adalah dengan menggunakan perintah input tap x y:

adb shell input tap x y

Tidak semua versi input memiliki parameter tap, tetapi memulainya tanpa argumen akan mencetak semua parameter yang tersedia dan sintaks:

shell@m0:/ $ input
input
usage: input ...
       input text <string>
       input keyevent <key code number or name>
       input [touchscreen|touchpad|touchnavigation] tap <x> <y>
       input [touchscreen|touchpad|touchnavigation] swipe <x1> <y1> <x2> <y2> [duration(ms)]
       input trackball press
       input trackball roll <dx> <dy>

Cara lain adalah dengan menggunakan Getevent untuk merekam peristiwa yang terjadi saat Anda menyentuh layar. Seperti yang diambil dari sini :

Rekam serangkaian acara

  1. Siapkan pc untuk merekam data dalam sebuah file (/tmp/android-touch-events.log)

    $ adb shell getevent | grep --line-buffered ^/ | tee /tmp/android-touch-events.log
  2. melakukan beberapa hal di telepon
  3. hentikan perintah adb pada pc dengan ctrl-c

Putar ulang acara yang direkam

Perintah ini akan melakukan konversi hex pada awk

$ awk '{printf "%s %d %d %d\n", substr($1, 1, length($1) -1), strtonum("0x"$2), strtonum("0x"$3), strtonum("0x"$4)}' /tmp/android-touch-events.log | xargs -l adb shell sendevent

Berhati-hatilah dengan gotcha bahwa Getevent menampilkan parameternya dalam hex tetapi hanya menerima masukan dalam desimal, yang coba diperbaiki oleh skrip di atas.

monkeyrunner adalah alat lain yang dapat Anda gunakan, seperti yang ditunjukkan sini :

Anda mungkin ingin menggunakan monkeyrunner seperti ini:

$ monkeyrunner
>>> from com.android.monkeyrunner import MonkeyRunner, MonkeyDevice
>>> device = MonkeyRunner.waitForConnection()
>>> device.touch(200, 400, MonkeyDevice.DOWN_AND_UP)

Anda juga dapat melakukan seret, memulai kegiatan, dll. Lihat apinya MonkeyDevice .
