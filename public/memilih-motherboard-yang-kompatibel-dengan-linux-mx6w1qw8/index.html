Memilih motherboard yang kompatibel dengan linux

Terakhir kali saya membeli komputer baru (saya membuatnya sendiri) saya mendapat motherboard yang memiliki dukungan linux yang sangat buruk untuk waktu yang lama. Khususnya audio. Saya harus menunggu berbulan-bulan sebelum kernel mendukung chipset audio on board. Persis seperti itulah situasi yang saya coba hindari kali ini.

Saya punya beberapa pertanyaan khusus tentang "motherboard server" sebenarnya. Saya melihat beberapa model motherboard server oleh intel, dan beberapa model acak di newegg. Saya tidak dapat melihat banyak perbedaan dari motherboard desktop biasa selain kebanyakan memiliki dua soket, dan dukungan untuk lebih banyak ram. Papan ini tampaknya lebih populer di kalangan pengguna Linux. Mengapa?

AMD dan Intel keduanya memiliki CPU server juga. Beberapa pertanyaan, apa bedanya?

Untuk membuat pertanyaan ini lebih konkret, saya melihat ini ini motherboard. Pertanyaan utama tentang itu yang tidak bisa saya jawab adalah:

  • Bisakah saya mendapatkan motherboard tanpa on board raid dan audio? Saya ingin mendapatkan pengontrol serangan perangkat keras dan kartu audio PCI. Saya pikir motherboard server akan lebih murah dan tidak memiliki "ekstra" ini, karena siapa yang menginginkan kartu audio di server?
  • Di mana saya dapat mengetahui tentang dukungan Linux untuk komponen di papan ini? "Intel ICH10R", "Realtek ALC889", "Marvell 88E8056"

Saya membeli komputer ini untuk bekerja sebagai desktop Linux untuk banyak pekerjaan kompilasi, pengkodean dan audio / video, tetapi saya tidak ingin mengesampingkan kemungkinan menginstal windows dan memainkan beberapa game pada satu titik. (bahkan jika game terakhir yang saya dapatkan telah duduk di kotaknya tanpa dibuka selama hampir setahun). Apakah ide yang baik untuk membeli "motherboard server" dan memainkan game di atasnya, atau apakah papan desktop lebih menghargai ini?

Solusi utama bagi saya adalah motherboard yang memiliki penyelam GPL untuk LAN onboard, satu soket CPU, banyak PCI express dan PCI. USB 3.0, dan tidak ada pengontrol hard disk yang mewah karena saya akan mendapatkan yang terpisah.

Papan ini tampaknya lebih populer dengan Pengguna Linux. Mengapa?

Vendor motherboard server dan server biasanya sangat berhati-hati untuk mendukung Linux, setidaknya Red Hat dan SuSE, karena itu hampir 1/4 dari pasar mereka. Persentase pasar desktop jauh lebih kecil (meskipun kuantitas keseluruhan mungkin jauh lebih tinggi).

AMD dan Intel keduanya memiliki CPU server juga. Beberapa pertanyaan, apa bedanya?

Hingga saat ini, sistem AMD menikmati antarmuka memori yang jauh lebih cepat yang lebih cocok untuk server. Sejak arsitektur Nehalem mencapai server, Intel telah melampaui AMD. CPU terbaru Intel lebih efisien dalam instruksi mereka per rasio jam. AMD mempertahankan posisinya melalui harga dan kinerja per watt yang cermat.

Kecuali Anda benar-benar membutuhkan prosesor server tercepat yang tersedia, sistem server AMD tetap kompetitif, tetapi Intel (untuk saat ini) menguasai kelas atas.

Bisakah saya mendapatkan motherboard tanpa serangan papan dan audio? Saya ingin mendapatkan pengontrol serangan perangkat keras dan PCI kartu audio. Saya pikir server motherboard akan lebih murah dan tidak memiliki "ekstra" ini, karena siapa yang mau kartu audio di server?

Motherboard server umumnya mendukung memori ECC (mengoreksi kesalahan sendiri), dan dapat mendukung hal-hal seperti catu daya ganda, dan biasanya memiliki setidaknya dua port Ethernet, dan garansi akhir yang lebih tinggi, yang semuanya menaikkan harga.

tetapi saya tidak ingin mengesampingkan kemungkinan menginstal windows

Beberapa motherboard server hanya memenuhi syarat untuk Windows Server 2003 dan yang lebih baru. Lainnya (lebih ditargetkan di pasar workstation CAD) divalidasi untuk Windows Vista dan / atau 7 64 bit.

Apakah ide yang baik untuk membeli "motherboard server" dan memainkan game di atasnya, atau apakah papan desktop lebih menghargai ini?

Ada begitu banyak jenis motherboard desktop, workstation, upright server, dan rack mount server di luar sana, kemungkinan besar Anda dapat menemukan kompromi apa pun yang Anda cari, kecuali untuk faktor harga. Peralatan yang lebih mahal harganya lebih mahal, kecuali Anda membeli bekas atau usang, itu tidak akan berubah.

Berikut adalah alternatif untuk membuat PC awal yang telah bekerja cukup baik untuk saya.

Saya telah menemukan bahwa kompromi yang baik antara membangun komputer homebrew dan membeli sampah konsumen off-the-shelf adalah dengan membeli mesin sekelas workstation bekas dari ebay. CPU pada mesin ini akan cukup cepat (jika tidak canggih) dan komponen cenderung memiliki kualitas yang cukup baik karena mesin dibuat untuk aplikasi yang memerlukan SLA.

Soket tunggal bekas dan dua kotak soket yang lebih kecil (seperti HP XW6400s) seringkali dapat dibeli hanya dengan beberapa ratus dolar. Mereka juga cenderung memiliki dukungan driver Linux yang baik karena pabrikan biasanya menawarkan Linux sebagai opsi pada jenis mesin ini.

Sebagian besar, saya telah membangun mesin yang lebih besar dari sasis XW9300 (juga murah dari Ebay akhir-akhir ini tetapi mungkin agak lama) untuk pekerjaan pengembangan basis data - yang melibatkan pembelian pengontrol RAID pihak ketiga dan satu set disk SCSI. Bahkan pada tahun 2007-2008 ketika saya membuat receiver ini (XW9300 masih merupakan model saat ini ketika saya pertama kali melakukan ini) yang bekas atau bekas demo biasanya lebih murah daripada disk yang saya masukkan ke dalamnya.

Fitur bagus lainnya dari receiver ini adalah yang keluar dari sewa sekarang dibangun selama kelebihan memori, jadi ada kelebihan komponen memori DDR1 / DDR2. Memori untuk mesin ini - bahkan modul ECC terdaftar - cukup murah. Sekali lagi, Ebay adalah temanmu.

Sebagai alternatif, sebagian besar produsen motherboard lebih baik seperti Tyan, Asus atau Supermicro memiliki jalur motherboard 'workstation' atau server (motherboard di XW9300 sebenarnya adalah Tyan S2895) yang menyertakan papan soket tunggal yang sesuai. Mereka juga cenderung secara eksplisit mempublikasikan dokumentasi kompatibilitas O / S seperti yang ditemukan sini (supermicro.com), sini (asus.com) atau sini (tyan.com)

Jika Anda mengikuti cara ini, coba situs web pabrikan, temukan model yang sesuai dengan spesifikasi yang sesuai dan kemudian cari tahu siapa yang mendapatkannya.
