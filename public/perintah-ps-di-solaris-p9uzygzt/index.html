Solaris 10 dan yang lebih baru tidak lagi membuat /usr/ucb/ps setuid-root secara default, dan Anda memerlukan hak akses root untuk masuk ke ruang alamat proses pengguna lain untuk mendapatkan set argumen lengkap.

Jalankan sebagai root atau pemilik proses untuk melihat lebih dari 80 karakter baris perintah.

Pada Solaris 11, jika Anda menggunakan opsi tanpa tanda hubung (seperti /usr/bin/ps auxwww), opsi tersebut akan diperlakukan sebagai opsi gaya UCB, dan hasilnya akan menampilkan garis ekstra panjang, bahkan saat Anda TIDAK menjalankan sebagai root. Ini tidak dipublikasikan dengan baik.

Untuk referensi lebih lanjut:

  • comp.unix.solaris: s10: / usr / ucb / ps output dipotong setelah 79 (80) char ketika digunakan oleh pengguna nonpriv?
  • comp.unix.solaris: / usr / ucb / ps auxwww - keluaran dipangkas untuk pengguna non-root di Sol10.
  • unix.stackexchange.com: Bagaimana saya bisa mendapatkan daftar proses lengkap di solaris, tanpa garis terpotong?

Memperbarui : Mulai Solaris 11.3 SRU 5.6 (dirilis Februari 2016), batas 80 karakter dihapus sepenuhnya dari ps - lihat Solaris 11.3 SRU 5.6: pembaruan di ps (1) dan / proc / <pid> / {cmdline, environment, execname} .
