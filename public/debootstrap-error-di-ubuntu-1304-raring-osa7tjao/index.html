debootstrap error di Ubuntu 13.04 raring

Saya mencoba membuat sistem file di Ubuntu dengan perintah:

sudo debootstrap --foreign --no-check-gpg --include=ca-certificates --arch=armhf testing rootfs http://debian.raspbian.com/debian

tetapi mendapat kesalahan berikut:

/usr/sbin/debootstrap: 1294: /usr/sbin/debootstrap: cannot create /home/.../rootfs/test-dev-null: Permission denied
E: Cannot install into target '/home/.../rootfs' mounted with noexec or nodev

ada yang bisa membantu saya di sini? Terima kasih!

Cari tahu filesystem mana lokasi /home/.../rootfs berada (mungkin merupakan partisi khusus atau LVM, Anda tidak memberikan spesifikasinya), dan, sebagai root (atau menggunakan sudo) pasang ulang dengan opsi exec dan dev.

# mount -o remount,exec,dev /home

Di sini, /home adalah contoh yang mengasumsikan partisi terpisah atau volume logis khusus.

Remount dengan exec dan dev adalah solusinya. Jika filesystem Anda adalah dienkripsi , Anda juga harus meneruskan -i untuk memasang.

 $ sudo mount -i -o remount,exec,dev /home/user

Di mana dalam kasus saya /home/user/ adalah direktori home terenkripsi saya. Setelah itu debootstrap tidak memiliki masalah lagi.
