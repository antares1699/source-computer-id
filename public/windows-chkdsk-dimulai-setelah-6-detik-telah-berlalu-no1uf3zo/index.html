Windows CHKDSK dimulai setelah 6 detik telah berlalu

Setelah membaca utas ini Saya mungkin sudah tahu mengapa prompt CHKDSK dimulai.

Tapi masalah terbesar adalah saat saya diminta dengan Press any key with in 10..0 seconds, tidak ada yang terjadi sampai timer mencapai 6, kemudian CHKDSK memulai pemeriksaan meskipun saya telah menekan tombol apa pun (namun tersisa 4 detik).

Pertanyaan ini menyatakan masalah seperti yang terlihat pada saya, meskipun jawabannya salah.

Mengapa pemeriksaan sebenarnya dimulai setelah 6 detik, dan adakah cara untuk menonaktifkannya dari dalam Windows?

Lakukan perintah ini:

bcdedit /set {current} bootstatuspolicy ignoreallfailures

Mirip

bcdedit /set {default} bootstatuspolicy ignoreallfailures

Mereka bisa sama jika Anda di-boot ke beban default, jadi default akan sama dengan arus.

Untuk informasi lebih lanjut, pencarian Google yang relevan adalah "nonaktifkan Windows Error Recovery" (tanpa tanda kutip).

http://msdn.microsoft.com/en-us/library/ff794974(v=winembedded.1001).aspx

Satu kemungkinan, tetapi setelah melihat tanggal artikel saya menganggap ini telah ditambal pada sistem yang sepenuhnya ditambal, tidak yakin, Anda dapat melihat versi apa dari autochk.exe Anda saat ini diinstal, lihat apakah cocok dengan versi tambalan yang tercantum di halaman.

Catatan: W7 64bit mungkin memiliki versi file autochk 64 dan x86, satu di folder System32 dan satu di folder SysWOW64, periksa keduanya.

Di komputer yang menjalankan Windows 7, Anda menggunakan program Chkdsk.exe untuk menjadwalkan pemeriksaan disk selama startup sistem berikutnya. Namun, pemeriksaan disk tidak dimulai dengan benar. Khususnya, Anda mungkin mengalami salah satu dari masalah berikut ini saat berikutnya Anda memulai komputer:

Penghitung waktu mundur ditampilkan dan Anda diminta untuk menekan sembarang tombol jika Anda ingin melewati pemeriksaan disk. Namun, setelah Anda menekan salah satu tombol, penghitung waktu mundur terus berlanjut. Akhirnya, sistem mungkin berhenti merespons pada hitungan 1, atau mulai tanpa melakukan pemeriksaan disk.

Penghitung waktu mundur tidak ditampilkan dan pemeriksaan disk segera dimulai.

Penghitung waktu mundur ditampilkan. Namun, pemeriksaan disk dimulai sebelum penghitung waktu mundur mencapai nol.

Masalah ini lebih mungkin terjadi pada komputer yang dilengkapi perangkat inframerah.

Jika Anda ingin menonaktifkannya, Anda mungkin perlu membersihkan kunci registri berikut: "HKLM \ SYSTEM \ CurrentControlSet \ Control \ Session Manager \ BootExecute" Anda akan menemukan "autocheck autochk *" pada kunci ini. Bersihkan saja dan itu akan dinonaktifkan.
