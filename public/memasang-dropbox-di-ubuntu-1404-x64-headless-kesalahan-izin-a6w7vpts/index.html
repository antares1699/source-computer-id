Memasang Dropbox di Ubuntu 14.04 x64 headless: kesalahan izin

Jadi saya mencoba memasang Dropbox (sebagai root) mengikuti petunjuk yang tampaknya mudah di dropbox.com:

cd ~ && wget -O - "https://www.dropbox.com/download?plat=lnx.x86_64" | tar xzf -
~/.dropbox-dist/dropboxd

Perintah pertama dijalankan tanpa masalah, namun perintah kedua memicu pesan kesalahan ini:

Tidak dapat memulai Dropbox. Ini biasanya karena kesalahan izin. Menyimpan folder rumah Anda di jaringan berbagi juga dapat menyebabkan kesalahan.

Dapatkan bantuan lebih lanjut di https://www.dropbox.com/c/help/permissions_error

Saya memiliki masalah yang sama saat menginstal drop box dari jarak jauh pada mesin ubuntu 14.04. Saya bisa mengatasinya dengan menjalankan dropboxd dengan izin root untuk pertama kalinya.

sudo ~/.dropbox-dist/dropboxd

Setelah diatur, tidak lagi membutuhkan hak root dan berjalan dengan benar di bawah akun pengguna saya. Saya baru saja menggunakan alat CLI dropbox untuk melakukannya:

~$ dropbox status
Up to date
~$ dropbox stop
Dropbox daemon stopped.
~$ dropbox start
Starting Dropbox...Dropbox isn't running!
Done!

HTH :)

dan tidak

Masalah serupa, menyiapkan dropbox pada instalasi linux headless. Satu-satunya perbedaan adalah saya tidak memiliki root. Ini adalah lingkungan cluster sehingga direktori home default berada di Luster FS. Saya menyalin .dropbox-dist ke jalur di disk lokal dan melakukan ini:

export HOME="/your/path/to"; .dropbox-dist/dropboxd

Selain jawaban @ Tom, ada kasus tepi lain di mana kesalahan yang sama di atas diberikan. Jika Anda terhubung ke mesin yang dimaksud melalui SSH dengan penerusan X11 diaktifkan, dropbox akan mencoba dan mencari berbagai pustaka terkait X11 yang mungkin tidak diinstal.

Saya memiliki masalah yang sama di OpenSUSE. Saya menyelesaikannya dengan menghapus file tersembunyi (tekan: Ctrl + h pada folder utama untuk melihatnya) terkait dengan dropbox, yaitu di /home/user/.dropbox*.

Di dalam file tersembunyi tersebut terdapat file yang terkait dengan konfigurasi dropbox. Setelah ini, coba instal kembali dropbox; yang mungkin memecahkan masalah.
