Ubah router menjadi sakelar?

Saya memiliki beberapa modem / router yang disertakan dengan layanan ISP yang tidak lagi digunakan. Saat ini saya memiliki router ISP saat ini yang menjalankan jaringan nirkabel dan sebagian kabel ke beberapa ruangan. Di salah satu ruangan itu saya memiliki sakelar untuk menghubungkan semua komputer di sana ke kabel yang berasal dari router.

Di ruangan lain saya memiliki beberapa kabel (sebenarnya satu kabel terbagi menjadi dua) tetapi saya membutuhkan setidaknya satu koneksi kabel tambahan di sana.

Bisakah saya mengonversi router yang ada sehingga saya harus berperilaku sebagai sakelar? Saya tidak ingin router ini mengontrol subnet mereka sendiri yang memberikan IP ke perangkat yang terhubung dengannya. Mereka harus transparan ke jaringan seolah-olah saya baru saja memiliki beberapa koneksi kabel tambahan yang tersedia di sana.

Itu sangat tergantung pada router.

Jika Anda dapat menonaktifkan server DHCP di router dan router memiliki sakelar bawaan, ya, Anda harus dapat menggunakannya tanpa masalah.

Jika Anda tidak dapat menonaktifkan server DHCP maka itu akan menyebabkan masalah pada jaringan Anda, jadi jangan gunakan.

Majenko — Sumber
  • Sebagian besar perangkat konsumen yang dijual sebagai "router" sebenarnya adalah router plus sakelar. Sebuah "router-only" tingkat konsumen yang sebenarnya umumnya hanya akan memiliki 2 port ethernet, LAN dan WAN. Matt Jenkins di atas benar, cukup banyak jika Anda menonaktifkan DHCP dan tidak memiliki hal lain yang berjalan di perangkat seperti RIP, server web, dll. Maka itu akan berfungsi.
    — LawrenceC
  • Berhasil. Saya menonaktifkan nirkabel dan DHCP dan menghubungkan LAN dengan koneksi jaringan ke salah satu port switch (bukan WAN) dan semua kabel ethernet perangkat lainnya di port switch yang tersisa dan semuanya berfungsi.
    — greye

  • Menonaktifkan DHCP adalah satu langkah
  • Langkah selanjutnya adalah tidak menggunakan port WAN yang biasanya terhubung ke hulu dan hanya menggunakan port LAN untuk upstream (terhubung ke router dll) serta downstream (klien) akan menyebabkan port LAN berperilaku sebagai sakelar dan kemudian perlu menarik DHCP IP dari perangkat upstream seperti Consumer / Internet Router atau DHCP Server (pada NAS atau Domain Controller).

Misalnya, Dalam Jaringan Media Rumah ini di bawah Router WR1 adalah penyedia DHCP untuk Subnet Lokal sementara Router WR2 terhubung melalui port LAN dengan WAN terputus. Dengan begitu mereka semua berada di Subnet yang sama dan semua IP dikelola melalui WR1 (Router berbasis firmware USB Tomat)

Jika saya menghubungkan WR2 menggunakan port WAN (dengan DHCP-nya yang diaktifkan) itu akan membuat Subnet-nya sendiri.
