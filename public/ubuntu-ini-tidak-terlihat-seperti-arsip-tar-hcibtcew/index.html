Ubuntu: Ini tidak terlihat seperti arsip tar

Saya mencoba menjalankan perintah berikut setelah saya mengunduh file tar menggunakan perintah sudo wget untuk menginstal CasperJS di kotak Ubuntu kami.

sudo tar -xvf casperjs.tar

This does not look like a tar archive
Skipping to next header
Exiting with failure status due to previous errors

Apakah saya perlu menjalankan perintah sudo tar asli secara berbeda?

  • Ngomong-ngomong, mengapa Anda menambahkan sudo di depan semua perintah? Dari sudut pandang keamanan Anda tentunya tidak boleh menjalankan klien Internet (wget) sebagai superuser.
    — pabouk
  • Untuk referensi di masa mendatang: Ada perintah bagus bernama file. Jika saya menggunakan file demo.tgz saya mendapatkan demo.tgz: gzip compressed data, from Unix, last modified: Fri Nov 8 13:31:16 2013. Menjalankan file pada file tar biasa akan mengembalikan: 'demo.tar: arsip tar POSIX'.
    — Hennes

untuk melihat apa yang Anda dapatkan:

file casperjs.tar

dan kemudian untuk melihat isinya, jika itu adalah file tar:

tar tvf casperjs.tar
tar tvzf casperjs.tar  #if that was a gzip-ed tar file
tar tvbf casperjs.tar  #if that was a bzip-ed tar file
tar tvZf casperjs.tar  #if that was a compress-ed tar file  #now very unlikely...
 #note that after the 'f', you need to have a SEPARATOR (space, or tab) followed by the FILENAME. 
 #ie, you can't place an option after the f
 #ex: "tar tvfz something.tar.gz", would try to open file "z" instead and find "something.tar.gz" inside it...

kemudian ubah 't' menjadi 'x' untuk mengekstrak (saya sarankan untuk menggunakan 't' terlebih dahulu, meskipun ...)

File yang Anda unduh adalah .tar.gz. (File .tar terkompresi)

Kamu bisa gunakan

tar -zxvf casperjs.tar

Tetapi sebenarnya Anda harus mengunduhnya sebagai .tgz seperti ini:

sudo wget -O casperjs.tar github.com/n1k0/casperjs/tarball/1.0.0 –O casperjc.tgz
sudo tar -zxvf casperjs.tgz

-z adalah singkatan dari:

-z, --gzip, --ungzip memfilter arsip melalui gzip
