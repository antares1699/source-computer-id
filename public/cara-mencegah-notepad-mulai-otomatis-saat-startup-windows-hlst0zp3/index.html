Cara mencegah Notepad ++ mulai otomatis saat startup Windows

Saya punya masalah dengan Notepad ++ dimulai dengan startup Windows. Jendela Notepad ++ tidak muncul. Hanya proses yang dimuat. Saya tidak dapat membuka file apa pun dengan Notepad ++ dari menu konteks hingga proses Notepad ++ dimatikan secara manual.

Solusinya akan sederhana untuk tidak memulai proses itu saat startup. Masalahnya adalah, saya tidak dapat menemukan proses starter baik di Autostart maupun melalui msconfig. Dan saya tidak dapat menemukan opsi untuk tidak memulai saat startup di preferensi Notepad ++.

Q: Mengapa proses Notepad ++ dimuat saat startup dan bagaimana cara mencegahnya?

Anda dapat memeriksa semua opsi mulai otomatis yang berbeda dengan Sysinternals Autoruns

Utilitas ini, yang memiliki pengetahuan paling komprehensif tentang lokasi mulai otomatis dari monitor startup mana pun, menunjukkan kepada Anda program apa yang dikonfigurasi untuk dijalankan selama boot sistem atau login, dan menampilkan entri dalam urutan proses Windows. Program-program ini termasuk yang ada di folder startup Anda, Run, RunOnce, dan kunci Registry lainnya.

Ada sejumlah tempat di registri tempat mulai otomatis dapat bersembunyi.

Periksa mis.

  • HKEY_CURRENT_USER \ Software \ Microsoft \ Windows \ CurrentVersion \ Run
  • HKEY_CURRENT_USER \ Software \ Microsoft \ Windows \ CurrentVersion \ RunOnce
  • HKEY_USERS \ << NAMA PENGGUNA >> \ Software \ Microsoft \ Windows \ CurrentVersion \ Run
  • HKEY_USERS \ << NAMA PENGGUNA >> \ Software \ Microsoft \ Windows \ CurrentVersion \ RunOnce

dengan regedit untuk referensi Notepad ++.

@saya kerusakan yang sama terjadi, solusi saya: Ada desktop.ini di folder startup di setiap akun:

desktop.iniX [.ShellClassInfo] LocalizedResourceName = @% SystemRoot% \ system32 \ shell32.dll, -21787

Mengganti nama membawa obatnya.
