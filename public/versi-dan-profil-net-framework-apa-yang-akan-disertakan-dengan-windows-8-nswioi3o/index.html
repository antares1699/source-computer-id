Berdasarkan artikel googled ini:

.NET Framework 4.5 RC disertakan dengan Windows 8 Release Preview dan diinstal di komputer Anda dengan sistem operasi. Namun, .NET Framework 3.5 tidak diinstal secara otomatis dengan Windows 8 Pratinjau Rilis. Untuk menjalankan aplikasi yang membutuhkan .NET Framework 3.5 pada Pratinjau Rilis Windows 8, Anda harus mengaktifkan versi 3.5 di komputer Anda. Ada dua cara untuk melakukannya: dengan menginstal atau menjalankan aplikasi yang membutuhkan .NETFramework 3.5 (yaitu, oleh menginstal .NETFramework 3.5 sesuai permintaan), atau dengan mengaktifkan .NET Kerangka 3.5 di Panel Kontrol. Kedua opsi tersebut membutuhkan Internet koneksi.

Apalagi menurut yang lain Artikel MSDN :

Dimulai dengan .NETFramework 4.5, Profil Klien telah dihentikan dan hanya paket lengkap yang dapat didistribusikan ulang yang tersedia. Pengoptimalan yang disediakan oleh .NET Framework 4.5, seperti yang lebih kecil ukuran unduhan dan penerapan yang lebih cepat, menghilangkan kebutuhan akan file paket penerapan terpisah. Garis arus tunggal yang dapat didistribusikan ulang proses penginstalan dan menyederhanakan opsi penerapan aplikasi Anda.

Ketika Anda menginstal .NETFramework 4.5 RC, .NETFramework 4 Profil Klien diperbarui ke versi lengkap .NETFramework.
