Untuk memperluas jawaban qasdfdsaq, ID yang Anda lihat haruslah GUID dari salah satu file partisi sistem yang dibuat Windows . Partisi lain harus ditampilkan seperti biasa dengan label dan huruf drive-nya. Lihat tabel di artikel Wikipedia jika ID Anda cocok dengan salah satunya dan Anda memiliki jawaban yang tepat.

Saya berharap bahwa ini sudah disaring untuk tidak muncul di Defrag sama sekali dan mungkin itulah masalahnya di sini bahwa ini adalah GUID yang dihasilkan yang tidak masuk daftar hitam di Defrag. Anda dapat membiarkan ini apa adanya atau cukup beri label pada partisi di Powershell atau manajer partisi yang dapat mengaksesnya agar terlihat lebih bagus. Defrag seharusnya tidak dapat bekerja pada partisi ini secara tidak sengaja. Jika Anda tidak memiliki banyak disk dan partisi dan Anda tidak ingin terlalu akurat, cukup pilih nama apa pun untuk labelnya, jika tidak buka diskmgmt.msc atau jalankan get-partition di Powershell dan lihat jenisnya. Saya dapat mengidentifikasi milik saya sebagai Recovery.

Jika itu tidak cukup akurat untuk Anda, karena Anda memiliki banyak partisi, maka Anda memerlukan program atau cuplikan kode yang menampilkan UUID / GUID partisi untuk setiap partisi untuk membuat asosiasi yang tepat yang memetakan GUID ke nomor partisi mana dan bagaimana itu. digunakan di Windows. Sayangnya saya tidak tahu solusi gratis dan cepat untuk itu di Windows. (Di Linux saya dapat mencarinya dengan ls di bawah /dev/disk-by-partuuid.) Gores itu, Windows menyebutnya DeviceID, cukup ganti GUID di bawah ini dengan milik Anda dan label pilihan Anda.

$guid = "12345678-abcd-abcd-abcd-1234567890ef"
$label = "System Recovery"

$drive = Get-WmiObject -Class win32_volume -Filter "DeviceID = '\\\\?\\Volume{$guid}\\'"
Set-WmiInstance -input $drive -Arguments @{Label="$label"}
