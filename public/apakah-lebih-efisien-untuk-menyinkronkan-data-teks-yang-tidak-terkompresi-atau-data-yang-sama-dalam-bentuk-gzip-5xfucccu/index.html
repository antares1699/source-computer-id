Apakah lebih efisien untuk menyinkronkan data teks yang tidak terkompresi atau data yang sama dalam bentuk gzip?

Rsync hanya menyalin bagian file yang diubah. Jadi jika saya memiliki data dump yang besar, misalnya database SQL dalam format teks, dan saya ingin menyalinnya ke komputer lokal saya dengan menggunakan bandwidth sesedikit mungkin, apa yang harus saya lakukan?

Saya dapat membiarkannya sebagai teks yang tidak terkompresi dan menggunakan rsync dengan sakelar -z, atau saya dapat melakukan gzip di server dan rsync file gzip.

Kecurigaan saya adalah bahwa perubahan kecil dalam file teks SQL sumber dapat mengakibatkan pengacakan data yang lebih besar dalam versi gzip terkompresi, jadi rsync harus melakukan lebih banyak pekerjaan karena lebih banyak file telah berubah.

Selain itu, karena sakelar -z melakukan kompresi, ini bisa menjadi yang paling efisien. Namun, saya tidak yakin.

Dalam kasus saya, tidak terkompresi file ditransfer paling cepat menggunakan rsync dengan kompresinya sendiri. Ukuran file biasanya 4-9 GB dari data yang dibuang dan beberapa di antaranya akan disinkronkan dalam beberapa menit.

Jika data dump dikompresi, rsync mentransfer seluruh file setiap saat dan ini membutuhkan waktu berjam-jam.

Ada baiknya melakukan beberapa tes dengan data Anda, tetapi saya curiga Anda akan menemukan yang terbaik untuk membiarkan file seperti itu untuk rsync untuk dipilah.
