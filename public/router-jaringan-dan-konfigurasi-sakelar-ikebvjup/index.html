Alasan saya ingin melakukan ini adalah karena router saya juga merupakan titik akses nirkabel saya. Modem kabel saya terletak di sudut terjauh dari rumah saya di mana panel akses telepon / tv / jaringan saya berada. Saya lebih suka menyimpan titik akses nirkabel di lokasi yang lebih sentral, tetapi saya tidak yakin apakah DHCP akan terus berfungsi dengan baik dengan konfigurasi itu.

Edit: Saya ingin terus dapat berbagi file dan mengakses drive jaringan dan printer saya dari setiap komputer di rumah.

DHCP tidak akan berfungsi dengan baik, tetapi bukan DHCP antara router Anda dan mesin di belakang router Anda yang tidak berfungsi, melainkan DHCP antara ISP dan router Anda.

Asumsikan Anda tidak mencoba untuk menekan tombol di tengah seperti itu. Router Anda mendapatkan IP publik melalui DHCP dari ISP Anda, pada antarmuka WAN-nya. Router Anda juga menjalankan server DHCP-nya sendiri, dan membagikan alamat mungkin di subnet 192.168.0.X ke mesin yang terhubung dengannya, di antarmuka LAN. Baca terus untuk mengetahui mengapa ini relevan.

Rintangan pertama adalah jika modem kabel Anda berfungsi seperti milik saya (yang, karena modem saya adalah modem Comcast, kemungkinan besar) ia tidak akan mengenali mesin kedua di belakangnya. Saya harus mengatur ulang modem kabel saya setiap kali saya mengubah apa yang disambungkan.

Hambatan kedua adalah bahwa ISP Anda sangat kecil kemungkinannya, bahkan jika modem kabel bekerja sama, bersedia memberi Anda dua alamat IP publik tanpa membayarnya.

Ada banyak cara untuk menyelesaikan masalah Anda, tetapi karena lokasi router / AP itu penting, menurut saya akan paling mudah jika Anda mendapatkan router kedua dan meletakkannya di tempat sakelar sekarang, atau mendapatkan dongle wifi untuk PC1 dan PC2.

Ini akan berfungsi, tetapi PC1 dan PC2 Anda tidak akan berada di belakang router / firewall.

EDIT: Jika Anda ingin PC1 dan PC2 Anda berada di belakang firewall, Anda memerlukan beberapa kabel panjang. Modem kabel Anda harus masuk ke port WAN router, dan PC ke port LAN.

Sebagai alternatif, Anda dapat menggunakan sakelar ke port LAN, dan kabel panjang terpisah untuk modem dan router melalui port WAN.

Modem kabel harus terhubung ke satu perangkat yang terhubung ke internet. Entah satu PC, dalam hal ini hanya PC itu yang dapat mengakses internet, atau ke router, dalam hal ini semua komputer yang terhubung ke router dapat mengakses internet.

Sambungan antara router dan modem kabel akan menjalankan PPPoE bukan TCP / IP, sehingga router tidak akan dapat berkomunikasi dengan komputer di sisi tersebut.

Apa yang ingin Anda lakukan pada dasarnya adalah menukar router dan beralih, jadi begini:

cable modem -> router ------------------> switch -> pcs
                   \-----> pcs
