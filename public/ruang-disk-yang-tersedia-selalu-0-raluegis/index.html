Ruang Disk yang Tersedia selalu 0%

Saya menjalankan centos 5. Sejauh ini tidak ada masalah tetapi baru kemarin, ketika dilaporkan "tidak ada ruang kosong" untuk penulisan file, saya mencoba menghapus beberapa file seperti biasa. Sayangnya kali ini tidak peduli berapa banyak file yang telah saya hapus, tetap saja tidak ada ruang yang tersedia untuk melakukannya.

Hasil dari df:

[root@LSMSVR ~]# df -h
Filesystem            Size  Used Avail Use% Mounted on
/dev/mapper/VolGroup00-LogVol00
                      1.2G  269M  879M  24% /
/dev/hda6             4.8G  138M  4.4G   4% /tmp
/dev/hda5              19G  2.4G   16G  14% /usr
/dev/hda3              48G   12G   34G  25% /var
/dev/hda2             379G  365G     0 100% /home
/dev/hda1              99M   15M   80M  16% /boot
tmpfs                 180M     0  180M   0% /dev/shm

Ada ide bagaimana memulihkan ruang yang hilang di / home? Terima kasih.

Default untuk sistem file linux adalah bahwa 5% dicadangkan untuk akses root saja - itulah mengapa Anda tidak melihatnya. Anda dapat melihat angka ini saat membandingkan "ukuran" dan "bekas"

Anda dapat mengubahnya menjadi 1% dengan menjalankan sudo tune2fs -m 1 /dev/hda2 Jelas Anda dapat mengubah parameter -m ke sejumlah persen.

Catatan! Mungkin ide yang bagus untuk menyetel parameter ini ke 0 untuk semua drive, tetapi sistem Anda mungkin mengunci Anda sama sekali jika hard drive terisi. (bukan masalah pada disk non-root)

Tidak ada "ruang yang hilang". Secara default, 5% ruang pada sistem file dicadangkan untuk root, sehingga pengguna tidak mengisinya sepenuhnya dan mematikan sistem. Terus hapus file hingga ada lebih dari 5% ruang kosong tersisa di sistem file. Anda memiliki setidaknya 5GB lagi atau lebih untuk digunakan.
