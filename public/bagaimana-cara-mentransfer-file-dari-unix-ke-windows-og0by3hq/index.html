Bagaimana cara mentransfer file dari UNIX ke Windows?

Saya ingin mentransfer file dari server UNIX ke Server Windows 2003, jadi saya mengunduh copssh ke server Windows, dan openssh sudah diinstal di server UNIX.

Ketika saya menjalankan perintah berikut (di server UNIX):

scp -r /file_in_unix/ user@windows_hostname:\c:\\\

Output berikut muncul (di layar server UNIX):

100% |***************************************************************|   562       00:00

Namun, ketika saya melihat file di drive C di server Windows, saya tidak menemukan apa pun. Mengapa file tersebut tidak muncul?

Jalur tujuan terlihat salah - untuk sebagian besar shell unix garis miring terbalik adalah karakter pelarian bukan penanda jalur, jadi saya menduga file tersebut telah jatuh ke direktori home pengguna SSH dengan nama file yang aneh.

IIRC copssh didasarkan pada cygwin, jadi yang mungkin Anda perlukan untuk menjalankannya adalah:

scp -r /file_in_unix/ user@windows_hostname:/cygdrive/c/

Alternatif untuk menyalin ke layanan SSH di mesin Windows adalah dengan menggunakan klien GUI seperti WinSCP di kotak Windows untuk masuk ke mesin Unix dan menarik file dengan cara itu - meskipun ini tidak cocok jika Anda mencoba mengotomatiskan prosesnya.

Jika Anda memiliki akses istimewa pada mesin unix (yaitu Anda, atau dapat melalui sudo atau serupa, root) dan memasang dukungan yang relevan, Anda dapat menyalin file ke share Windows. Anda tidak mengatakan apa Unix yang Anda gunakan. Untuk Ubuntu dan pemeriksaan serupa yang mendukung ada dan menginstalnya jika tidak dapat dilakukan dengan sudo aptitude install smbfs, Anda dapat memasang saham Windwos dengan sesuatu seperti sudo mount -tcifs //11.22.33.44//sharename /mnt/tmp -ousername=WindowsUserName (di mana 11.22.33.44 adalah alamat IP dari mesin windows, tergantung pada pengaturan jaringan Anda. Anda mungkin bisa merujuk ke mesin dengan nama daripada alamat). Setelah Anda selesai melakukannya, Anda dapat menggunakan alat manajemen file dasar (cp, mv, ...) untuk berinteraksi dengan berbagi Windows itu dan memanggil umount /mnt/tmp ketika Anda selesai. Anda mungkin ingin memilih nama titik pemasangan yang lebih bermakna daripada / mnt / tmp. Anda dapat membiarkan share tetap terpasang, tentu saja, jika transfer data akan diotomatiskan / dijadwalkan. Metode ini mengasumsikan bahwa mesin Unix dapat melihat fileshares mesin Windows melalui pengaturan firewall yang mungkin ada di antara mereka.
