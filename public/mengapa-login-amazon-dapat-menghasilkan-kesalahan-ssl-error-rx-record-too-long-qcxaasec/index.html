Mengapa login Amazon dapat menghasilkan kesalahan “SSL_ERROR_RX_RECORD_TOO_LONG”

Tadi malam saya memasukkan kredensial saya untuk masuk ke akun Amazon saya (menggunakan Firefox v.50.1.0) dan mendapatkan kesalahan yang ditunjukkan di bawah ini.

Terjadi kesalahan selama koneksi ke www.amazon.com:80. SSL menerima catatan yang melebihi panjang maksimum yang diizinkan. Kode kesalahan: SSL_ERROR_RX_RECORD_TOO_LONG

Setelah melihat kesalahan di atas, saya memasukkan amazon.com ke bidang alamat di tab baru, dan sepertinya saya sudah masuk. Karena saat itu Jumat larut malam, saya pikir mungkin Amazon membuat beberapa perubahan di situs mereka dan ini baru saja masalah sementara yang akan hilang dengan sendirinya.

Tetapi pagi ini saya mendapatkan kesalahan yang sama ketika saya masuk ke Amazon. Dan perilakunya masih sama, yaitu jika saya mengabaikannya dan menavigasi ke amazon.com setelahnya, sepertinya saya sudah masuk.

Namun saya hanya melihatnya di satu komputer di rumah.

Jadi pertanyaan yang ingin saya tanyakan adalah apakah kesalahan ini dapat disebabkan oleh beberapa konfigurasi pada mesin lokal saya?

[MEMPERBARUI:] Terima kasih atas penjelasan Anda. Kesalahan ini tampaknya terjadi karena port 80 ditentukan dalam alamat, meskipun saya tidak memasukkannya sendiri.

Setelah mencoba mengulangi ini beberapa kali, tampaknya ini terjadi pada akhir dari urutan langkah berikut:

  1. Arahkan ke amazon.com (browser harus diarahkan ke https: www.amazon.com)

[Perhatikan bahwa saya hanya dapat mereplikasi ini dengan versi tertentu dari laman beranda Amazon seperti yang dijelaskan lebih lanjut di bawah]

  1. Klik pada Akun & Daftar> Masuk. Amazon akan mengarahkan browser ke URL ini (URL ini berisi pengalihan tambahan yang akan selesai setelah Anda memasukkan kredensial dan klik Masuk Di).

  2. Masukkan kredensial dan klik Masuk.

  3. Amazon mengalihkan ke URL yang menentukan port tidak aman 80, menghasilkan pesan kesalahan.

Selama upaya saya untuk mereplikasi ini, saya perhatikan bahwa Amazon saat ini melayani setidaknya dua halaman beranda yang berbeda. Halaman beranda yang lebih sering memiliki opsi menu ini di sudut kanan atas:

Versi beranda kedua (lebih jarang) memiliki lebih banyak opsi di sudut kanan atas dan terlihat seperti ini:

Opsi masuk dalam versi terakhir ini mengalihkan ke URL pada langkah 2 di atas. URL itulah yang menyetel alamat ke amazon.com:80, yang mengarah ke pesan kesalahan.

Anda terhubung ke https://www.amazon.com:80/. Browser Anda mengharapkan handshake SSL terjadi karena bagian protokol https. Tetapi situs web sebenarnya menyajikan respons HTTP biasa karena Anda juga secara eksplisit menentukan port 80, port default untuk HTTP yang tidak terenkripsi.

Oleh karena itu, Firefox salah menafsirkan tanggapan HTTP sebagai lalu lintas SSL dalam format yang salah yang hanya secara tidak sengaja terlihat seperti "data RX terlalu panjang". Kesalahan seharusnya hilang jika Anda mengetik https://www.amazon.com/ atau https://www.amazon.com:443/. (Port 443 adalah default untuk HTTPS.)

Anda dapat memprovokasi perilaku serupa dengan banyak situs lain. Berikut adalah cara Chrome menghasilkan ERR_SSL_PROTOCOL_ERROR serupa saat mengunjungi https://google.com:80/:

Koneksi http biasanya menggunakan port 443, dan Http menggunakan 80. Ini terjadi saat Anda mencoba menyambung dengan koneksi yang tidak aman (tanpa SSL pada port 80), sedangkan pengaturan browser Anda hanya mengizinkan koneksi aman untuk situs ini (dengan SSL pada port 443). Menghapus ": 80" dari URL harus memperbaiki ini.
