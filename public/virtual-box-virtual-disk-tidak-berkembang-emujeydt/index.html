Komputer

Virtual Box Virtual Disk tidak berkembang

Saya memiliki komputer host iMac Core i5, Hard drive 1TB, Memori 8GB.

Saya telah membuat Virtual Box VM untuk menjalankan Fedora 14 64bit dengan 200 GB hard drive yang dialokasikan secara dinamis. Virtual Box versi 4.1.6 r74713

Saya telah mencoba dua kali untuk mengimpor file sql database 40+ GB dan saya terus kehabisan ruang hard drive.

Setiap kali saya mencoba ini, hard drive VM saya berhenti berkembang pada ~ 50 GB.

df -h keluaran:

/dev/mapper/VolGroup-lv_root
                       50G   47G   96M 100% /
tmpfs                 981M  260K  981M   1% /dev/shm
/dev/sda1             485M   30M  430M   7% /boot
/dev/mapper/VolGroup-lv_home
                      144G  213M  136G   1% /home
ds                    931G  169G  763G  19% /media/sf_ds
ds                    931G  169G  763G  19% /var/www/site-example.org

Saya memiliki banyak ruang di komputer host (800+ GB)

Virtual Box melaporkan ukuran sebenarnya dari disk adalah 46,88 GB Adakah ide tentang mengapa Virtual Box berhenti memperluas disk dan bagaimana cara memperbaikinya?

Memperbarui

Saya telah menjalankan GParted di VM dan dikatakan semua 200 GB digunakan meskipun df -h tidak melaporkannya.

Anda menetapkan sebagian besar ruang pada disk virtual Anda ke volume LVM /dev/mapper/VolGroup-lv_home. Sekarang Anda mencoba mengimpor database Anda ke dalam file yang (kemungkinan besar) tidak ada di direktori home Anda.

Jadi ruang dari volume root Anda digunakan. Volume itu hanya berukuran 50G.

Anda perlu mengubah ukuran volume LVM Anda.

Saya berasumsi bahwa Anda memilih untuk memiliki direktori home Anda di partisi terpisah selama instalasi Fedora.
Mengingat Anda hanya menggunakan 1% dari ruang kosong di partisi itu dan karena Anda kehabisan ruang di partisi root Anda, pilihan itu mungkin tidak optimal.

Anda akan ingin menggunakan lvresize untuk mengecilkan volume rumah Anda dan meningkatkan volume root Anda. Jika Anda tidak memiliki pengalaman dengan alat itu, sepertinya ada banyak panduan online. Seperti yang ini: Panduan Mengubah Ukuran LVM .

Di mana Anda mencoba mengimpornya? Sistem file root (/ dev / mapper / VolGroup-lv_root) sudah penuh. Jika Anda membutuhkan lebih banyak ruang pada root, Anda perlu mengubah ukuran partisi Anda untuk memindahkan ruang dari / dev / mapper / VolGroup-lv_home

Saya yakin Anda mungkin ingin memeriksa ukuran Hard Drive yang dialokasikan di dalam VM Fedora. Bergantung pada bagaimana fedora diatur, ukuran partisi maksimum yang dilihat Fedora bisa jadi 50G (seperti yang terlihat dari output df -h Anda). Meskipun dinamis di mata VirtualBox, Fedora tidak melihatnya dinamis karena partisi selama penginstalan berada pada nilai yang tetap.

Saya menemukan posting forum serupa tentang ini di Forum Kotak Virtual Lihat disini

Catatan cepat setelah membaca ulang output. Lihatlah alokasi ke / dev / mapper / VolGroup-lv_home Anda yang telah dialokasikan 144G untuknya. Jadi jika Anda mentransfernya, harus ada transfer. Masalah partisi di atas masih ada kecuali jika Anda mentransfernya ke tempat lain selain / dev / mapper / VolGroup-lv_home Anda akan kehabisan ruang.
