Pembaruan Windows mengubah pengaturan BIOS?

Ini mungkin hanya kebetulan, tetapi setelah batch pembaruan terbaru untuk Windows 10 opsi "Kata sandi saat boot" saya mati di BIOS - meskipun kata sandi masih ada karena saya membutuhkannya untuk mengakses BIOS dan mengembalikan pengaturan ke di.

Saya mendapat kesan bahwa BIOS tidak dapat diubah oleh sistem operasi apa pun, apakah saya salah saja?

Saya adalah satu-satunya pengguna sistem ini dan saya sendiri belum mengubah pengaturan itu ...

Apa - selain saya dan kasus kehilangan memori jangka pendek - yang dapat mengubah pengaturan itu?

Hal yang sama terjadi 4/4/2018 (m / d / yyyy) pada HP EliteDesk 705G3 SFF dengan Windows 10 v1703. Tetapi ini adalah mesin lab komputer yang terkunci dan kata sandi administratif BIOS-nya telah ditetapkan.

Butuh beberapa saat untuk mencari tahu mengapa tiba-tiba tidak bisa boot karena selama pengaturan awal musim panas lalu, seperti semua 40 PCS baru di lab ini, firmware telah diatur untuk mengaktifkan boot Legacy (BIOS) dan menonaktifkan boot Secure (UEFI) .

Setelah mengesampingkan semua kemungkinan yang biasa,
Saya memutuskan untuk memeriksa apakah firmware telah diubah. Saya harus menggunakan kata sandi pengawas BIOS untuk mengakses pengaturan dan menemukan bahwa pengaturan memang telah diatur ulang ke default mereka (Nonaktifkan Boot Legacy dan Aktifkan Boot Aman).

Saya mengubahnya kembali untuk mengaktifkan boot BIOS dan menonaktifkan boot UEFI. Kemudian saya melalui proses validasi perubahan firmware HP (ketik 4 digit angka untuk mencocokkan nomor yang dibuat secara acak yang diberikan kepada saya) dan kemudian reboot PC.

PC boot ke Windows 10 secara normal, tetapi kemudian saya mendengar suara startup Windows!

Ini seharusnya tidak terjadi karena selama pengaturan PC, speaker internal telah dinonaktifkan sesuai prosedur biasa kami untuk mencegah nada dan nada acak (atau mungkin tidak terlalu acak) dari lusinan PC yang mengganggu kelas.

Saya telah masuk ke mesin ini beberapa kali sejak pengaturan awal dan tidak pernah mengeluarkan suara startup Windows atau suara lainnya. Jadi saya memulai ulang dan kembali menggunakan akses supervisor admin BIOS - menemukan bahwa speaker internal diaktifkan - menonaktifkannya dan kembali melalui proses validasi perubahan.

Setelah boot ke Windows 10, semua bekerja secara normal dan diam-diam.

Pemeriksaan log Pengaturan Windows mengungkapkan bahwa pematian PC sebelumnya adalah pada 3/21/2018. Log penataan menunjukkan bahwa KB4074590 (Pembaruan Keamanan untuk Microsoft Windows) telah diinstal pada 3/8/2018 (reboot tidak terjadi hingga 3/16/2018).

Pada 3/19/2018, perubahan dimulai untuk mengubah KB4056887 dari Dipasang menjadi Absen untuk id klien CbsTask dan untuk mengubah KB4056890 dari digantikan ke Absen untuk id klien CbsTask, tetapi tampaknya tidak perlu reboot agar perubahan ini terjadi.

Kemudian saya menemukan " Cara mengakses pengaturan Firmware UEFI di Windows 10 / 8.1 "di situs web Windows Club. Artikel tersebut menunjukkan kepada saya cara mengubah pengaturan firmware melalui Pengaturan> Pembaruan dan Keamanan> Pemulihan> Pengaktifan lanjutan> Pemecahan Masalah> Pengaturan Firmware UEFI.

GARIS BAWAH : Ini rupanya aku s mungkin untuk Pembaruan Windows (atau kombinasinya) untuk mengesampingkan kata sandi administrator BIOS dan mengatur ulang firmware PC ke default - tanpa menonaktifkan atau mengubah kata sandi BIOS.

TINDAK LANJUT: Setelah ujian akhir semester Musim Semi, uji hipotesis baterai CMOS rendah dengan memutuskan sambungan mesin dari daya listrik selama tiga minggu. (Menurut log peristiwa Windows, ada dua minggu dari pematian 3/21/2018 hingga 4/4/2018 ketika masalah ditemukan, jadi dua minggu akan menjadi waktu maksimum mesin dapat diputuskan dari listrik power.) Namun, saat mesin dihubungkan kembali ke daya listrik dan dihidupkan setelah tiga minggu, mesin melakukan booting dalam mode BIOS seperti biasa. Karenanya, tampaknya masih mungkin bahwa kombinasi kejadian yang sangat tidak biasa (dan tidak diketahui saat ini) entah bagaimana memungkinkan proses Pembaruan Windows yang tidak berfungsi untuk menggunakan otoritas perubahan firmware Windows 10 untuk mengatur ulang firmware ke nilai defaultnya.

Saya memiliki SSD yang lebih lama pada 0: (terdaftar pertama di bios), dan SSD yang dapat di-boot pada 1 :, dan saya curiga bahwa pembaruan menyebabkan bios mereset SSD mana yang harus dilihat untuk boot.

Suatu hari saya berada di depan komputer bermain Elder Scrolls Online (permainan yang luar biasa) dan sebelum tidur, matikan, semuanya normal. Saya menduga bahwa pada suatu saat selama hari itu, pembaruan diterapkan karena saya mengaktifkan pembaruan otomatis dan karena keesokan paginya ketika saya menyalakan komputer, setelah memposting, ia pergi ke bsod yang mengatakan kode kesalahan 0000225.

Saya mencoba memulai ulang beberapa kali dan berpikir itu pasti kesalahan, tetapi hal yang sama terus terjadi, dan saya pikir saya mungkin perlu memformat ulang atau mengatur ulang OS sepenuhnya. Saya restart lagi dan saat memposting saya masuk ke pengaturan bios hanya untuk melihat apakah ada yang salah ... dan itu dia ... bios saya entah bagaimana telah diubah untuk melihat SSD yang salah saat boot.

Saya tidak tahu apakah itu akan membantu, tetapi saya meletakkan pw di bios sekarang jadi mungkin pembaruan atau apa pun yang dilakukannya sekarang akan dicegah untuk melakukannya. Saya melihat mungkin mengaktifkan opsi PCI Lock di msconfig, tetapi belum yakin. Kesimpulan besarnya, adalah sesuatu selain saya yang mengubah pengaturan bios saya.
