Ketika Windows XP tidak mau restart - bagaimana mendiagnosis, bagaimana cara mematikan tanpa power-cycle?

Terkadang Windows (XP sp3) tidak mau mendengarkan ketika saya memerintahkannya untuk Restart / Shutdown / Logoff. Segera, beberapa hal menghilang dari system tray, dan tidak ada yang berjalan di panel Program dari Task Manager; maka tidak ada yang terjadi. Bahkan jika saya menunggu satu jam (tidak hanya lambat - berhenti). Saya dapat meluncurkan program baru, mengutak-atik apa saja - sistemnya tidak lamban - ia tidak mau mati.

Bagaimana cara mengetahui proses mana yang tidak mendengarkan perintah shutdown?

Saya lebih suka tidak hanya memotong daya / menekan tombol reset. Apakah ada perintah mematikan yang lebih kuat? Saya sudah mencoba semua parameter baris perintah untuk "shutdown" dasar di shell perintah.

EDIT: Tampaknya ini dipengaruhi oleh ketersediaan koneksi jaringan - router perlu diatur ulang, dan ketika saya melakukannya mesin mati dengan cepat. Cabut kabel Ethernet, dan shutdown macet. Saya pikir beberapa proses sedang mencari sumber daya jaringan. Untuk saat ini, saya tidak akan meluangkan lebih banyak waktu untuk menemukan pelakunya - hanya menunjukkan bagaimana beberapa programmer menganggap jaringan akan selalu responsif. Terima kasih atas jawabannya!

Saya akan melihat ke Peraga Peristiwa, mungkin di log Sistem atau mungkin log Aplikasi.

Beberapa perangkat lunak akan mencegah shutdown, misalnya perangkat lunak Mesin Virtual (saya yakin VMWare melakukannya, setidaknya pada OS X) tidak akan memungkinkan Anda untuk mematikan / reboot saat OS tamu berjalan.

Ide lain adalah memuat setiap aplikasi yang biasanya Anda buka dan mencoba reboot setelah dimuat. Ini dapat membantu Anda melacak apa yang Anda buka saat Anda tidak dapat melakukan boot ulang. Ini akan sangat memakan waktu.
