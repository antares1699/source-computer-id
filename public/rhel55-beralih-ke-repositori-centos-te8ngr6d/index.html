RHEL5.5 beralih ke repositori CentOS?

Saya diberi VM untuk mesin RHEL5.5 tetapi saya tidak memiliki langganan RHN. Saya mendapatkan peringatan ini saat menggunakan yum:

bash-3.2$ sudo yum update
[sudo] password for streamsadmin: 
Loaded plugins: rhnplugin, security
This system is not registered with RHN.
RHN support will be disabled.
Skipping security plugin, no data
Setting up Update Process
No Packages marked for Update

Bisakah saya beralih menggunakan repositori CentOS?

Anda pasti bisa melakukan peralihan ini. Pada dasarnya Anda, kurang lebih, mengarahkan instalasi Anda ke repositori CentOS dan melanjutkan perjalanan Anda. Anda akan memiliki satu masalah besar, meskipun, semua repo CentOS tampaknya memiliki paket 5.x mereka pada rilis 5.9. Sekilas, tampaknya tidak ada lagi repo khusus 5,5.

Saya melakukan ini di pertanian web bertahun-tahun yang lalu (kira-kira era RHEL / CentOS 5.3) tetapi belum sejak .... Pastikan Anda memiliki cadangan sistem Anda saat ini terlebih dahulu, karena ini, seperti semua konversi, dapat mengakibatkan kerusakan lokal yang cukup besar jika terjadi kesulitan. Saya dapat melakukan cross-grade ke versi CentOS / RHEL yang sama ketika saya melakukan ini. Saya tidak yakin apa implikasi dari penilaian silang dan peningkatan secara bersamaan. Mungkin berantakan, jadi lanjutkan dengan hati-hati.

1) Bersihkan cache yum:

yum clean all

2) Unduh kunci CentOS (Saya baru saja memilih salah satu cermin CentOS, Anda dapat menggunakan siapa pun yang Anda suka http://mirrors.centos.org )

wget http://mirrors.cat.pdx.edu/centos/5/os/x86_64/RPM-GPG-KEY-CentOS-5

3) ... rilis rpm dan catatan:

wget http://mirrors.cat.pdx.edu/centos/5/os/x86_64/CentOS/centos-release-5-9.el5.centos.1.x86_64.rpm  
wget http://mirrors.cat.pdx.edu/centos/5/os/x86_64/CentOS/centos-release-notes-5.9-0.x86_64.rpm

4) ... nyam

wget http://mirrors.cat.pdx.edu/centos/5/os/x86_64/CentOS/yum-3.2.22-40.el5.centos.noarch.rpm  
wget http://mirrors.cat.pdx.edu/centos/5/os/x86_64/CentOS/yum-updatesd-0.9-5.el5.noarch.rpm

5) Impor KUNCI baru:

rpm --import RPM-GPG-KEY-CentOS-5

6) Hapus rilis redhat

rpm -e --nodeps redhat-release

7) ... dan plugin Redhat Network

rpm -e yum-rhn-plugin

8) Instal RPM yang baru saja Anda unduh (pastikan Anda tidak memiliki RPM lain di direktori Anda saat ini terlebih dahulu)

rpm -Uvh -force yum*.rpm centos*.rpm

9) Tingkatkan sistem dengan paket dari repositori baru:

yum upgrade

10) reboot.

Saya melakukan ini, sekali lagi, beberapa tahun yang lalu dan berhasil persis seperti yang diiklankan saat itu. Tapi dan itu besar tapi, itu antara rilis versi yang sama dari RHEL dan CentOS.

RHEL dan CentOS sangat stabil, jadi ini mungkin masih berfungsi. Jika ada rilis non-perusahaan lainnya, saya, tanpa ragu-ragu, merekomendasikan Anda untuk menginstal ulang. Namun, sebagaimana adanya .... itu mungkin saja berhasil.
