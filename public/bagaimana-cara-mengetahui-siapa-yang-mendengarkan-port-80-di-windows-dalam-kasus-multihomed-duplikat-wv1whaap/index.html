Baiklah, mari kita jadikan jawaban.

Apa yang Anda lihat kemungkinan besar adalah hasil dari program yang menggunakan http.sys Mesin HTTP Mode Kernel Windows yang diperkenalkan di Windows Server 2003 (dan XP?). Ini memungkinkan pengembang untuk menghindari batasan bahwa port pendengar hanya dapat dilayani oleh satu program dalam satu waktu. Dengan http.sys, program mendaftarkan namespace, mis. http://192.168.1.2:80/abc. Program ini sekarang menerima semua permintaan yang dimulai dengan namespace terdaftar. Pada saat yang sama, program lain dapat mendengarkan pada http://192.168.1.2:80/xyz.

Efek samping lainnya adalah bahkan program "non-root" dapat mendengarkan pada port 80, asalkan pengaturan satu kali telah selesai.

Secara alami, Microsoft tidak mengembangkan ini karena ini semua kesenangan dan permainan. Layanan Informasi Internet (IIS) menggunakan http.sys untuk mendengarkan. Jadi ini adalah kandidat yang mungkin dalam kasus Anda.

Jika tidak ada program yang saat ini memiliki ruang nama yang terdaftar, http.sys berhenti mendengarkan.

http.sys harus dapat menghitung semua ruang nama yang aktif.
