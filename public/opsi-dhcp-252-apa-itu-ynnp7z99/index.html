opsi dhcp 252 apa itu?

Di seluruh internet saya melihat contoh cara mengatur dhcpd dengan proxy WAPD konfigurasi otomatis menggunakan DHCP. Di file /etc/dhcpd.conf ada pernyataan seperti ini:

option wpad code 252;  
"http://wpad.url";  
option wpad code 252 = text;  
"http://wpad.url";

Apa itu kode 252, di mana itu didefinisikan dan apa artinya. Mengapa ini angka 252 dan bukan pernyataan yang berguna seperti wpad-proxy-url; Mengapa menggunakan 252 dan bukan angka lain (0, 5000). Saya mengerti bahwa itu berhasil, itu tidak masuk akal bagi manusia (saya). Mencari di Google apa opsi 252 tidak membuahkan hasil.

Opsi 252 menyediakan klien DHCP URL untuk digunakan untuk mengkonfigurasi pengaturan proxy-nya. Ini ditentukan dalam draft-ietf-wrec-wpad-01 .

Jika itu adalah pernyataan seperti "wpad-proxy-url" maka hanya sistem yang memahaminya yang dapat menggunakannya (mereka harus mengenali string itu dan tahu cara menanganinya). Karena ini adalah angka, tag sewenang-wenang, itu dapat diproses bahkan oleh server yang tidak memahaminya.

Misalnya, jika saya mengatakan "Ada apa di kepala Jessica?" Anda harus tahu siapa Jessica untuk menjawab saya. Tetapi jika saya mengatakan "Apa yang ada di kepala orang ketiga yang mengantre?" Anda tidak perlu tahu bahwa namanya adalah Jessica.

Ini 252 karena itu adalah nomor yang ditetapkan ke protokol oleh kelompok kerja IETF DHC. Lihat RFC 2132 .

Berikut ini tautan bagus yang mencantumkan arti dari banyak kode ### dan bagaimana mereka digunakan. http://www.iana.org/assignments/bootp-dhcp-parameters/bootp-dhcp-parameters.xml

252 adalah bagian dari bagian khusus penggunaan pribadi. Pada dasarnya untuk digunakan orang lain di server yang ada karena mereka merasa perlu.

Microsoft, memutuskan untuk menggunakan 252 untuk server dhcp mereka, dan kemudian menulis ke browser mereka kemampuan untuk berbicara dengan server dhcp dan meminta kode 252 untuk mendapatkan informasi tentang pengaturan proxy di jaringan dari url yang terdaftar dengan opsi.

Firefox, dan chrome masih memerlukan penyiapan dns ke server file .pac untuk konfigurasi proxy. Mereka saat ini tidak bekerja dengan opsi dhcp.
