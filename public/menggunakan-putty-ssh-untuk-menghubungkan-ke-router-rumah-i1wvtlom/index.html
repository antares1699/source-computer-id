Menggunakan PuTTY (SSH) untuk menghubungkan ke router rumah

Saya baru mengenal puTTY dan SSH. Saya mengalami kesulitan untuk menghubungkan ke router saya menggunakan SSH. Saya dapat terhubung dengan baik dengan Telnet. Apakah ada prasyarat yang dibutuhkan untuk SSH? Misalnya, apakah saya perlu menginstal sesuatu di router saya untuk menghubungkannya?

Menurut Rilis Manual untuk router itu, itu tidak dibangun di luar kotak.

Apakah ini berarti dengan pasti bahwa itu tidak mendukungnya? Belum tentu. Mereka mungkin merilis pembaruan untuk itu. Melihat beberapa perintah yang mereka minta Anda ketik dengan telnet, itu tampaknya seperti itu mungkin menjalankan Linux. Dapatkah Anda melakukan ls atau pwd untuk mendapatkan jalur atau file? Mungkin router mendukung Tomat atau DD-WRT , dan keduanya akan memberi Anda akses SSH.

Untuk mendapatkan akses SSH, router Anda harus mendukungnya, yang biasanya hanya merupakan layanan tambahan. Jika Anda khawatir tentang memiliki koneksi aman dari dunia luar ke dalamnya, saya sarankan memiliki kotak Linux kecil (misalnya, pada Raspberry Pi) yang menjalankan daemon SSHd, lalu telnet dari itu ke router. Jika Anda mencari koneksi terenkripsi dari workstation Anda dalam jaringan ke router, saya akan mencari tahu mengapa ada mata yang mengintip di jaringan Anda.

Saya menyadari ini mungkin tidak menjawab pertanyaan, tapi mudah-mudahan ini memberi Anda beberapa info, dan gagasan lain tentang ke mana harus mencari atau apa yang harus dilakukan.
