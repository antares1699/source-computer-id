Kecepatan lambat pada Samsung SSD PM800 256gb pada Lenovo W500

Saya baru saja membeli W500 dengan drive samsung ssd 256 GB. Sekarang tampaknya sangat lambat dalam menulis. Saya menyalin file sekitar 30mb / s, tetapi saya dapat membacanya sekitar 200mb / s.

Saya mengujinya dengan AS SSD Benchmark v1.4, dan mendapatkan kecepatan tulis berurutan 34.64mb / s dan kecepatan membaca 196.95mb / s

Ketika saya membelinya, drive tersebut hanya digunakan selama sekitar 240 jam, dan menurut aplikasi CrystalDiskInfo, drive tersebut memiliki 98% kesehatan.

Apakah ada hambatan di sini? Atau apakah drive benar-benar buruk. Saya sangat senang jika seseorang dapat membantu saya menemukan beberapa jawaban.

Spesifikasi utama yang relevan (saya harap) w500 untuk mesin ini adalah;

  • T9600 @ 2.8GHz
  • 4 GB DDR3

Anda menggunakan windows XP yang mengasumsikan drive memiliki sektor 512 byte dan yang mentransfer file dalam potongan 512 byte.

Namun hal ini tidak lagi terjadi pada drive modern. Banyak hard drive modern menggunakan ukuran sektor internal 4KB. Jika Anda hanya menulis 512 byte ke sektor 4096 byte (4KB), drive perlu membaca semua 4K, mengubah 512 byte, dan menulis ulang.

Itu banyak sekali. (seperti yang Anda lihat di bawah)

OLD DATA - OLD DATA-OLD DATA-OLD DATA-OLD DATA-OLD DATA-OLD DATA-OLD DATA
NEW DATA - OLD DATA-OLD DATA-OLD DATA-OLD DATA-OLD DATA-OLD DATA-OLD DATA

Dengan SSD bahkan lebih buruk lagi.

SSD tidak dapat menulis ke sektor yang sudah digunakan. Mereka hanya memiliki tiga opsi:

  1. Temukan bagian kosong yang sama sekali tidak terpakai dan tulis blok besar (kebanyakan kosong) untuk itu.
  2. Baca beberapa data dan tulis blok yang lebih lengkap, tandai data lama sebagai tidak digunakan.
  3. Bersihkan sebagian dari SSD. Biasanya dalam potongan minimal 1MB. (Halo overhead)

Dijelaskan secara grafis:

State before:  [empty][empty][empty][empty]    [empty][empty][empty][empty]  
Single write:  [data] [empty][empty][empty]    [empty][empty][empty][empty]   
               \                          /    \                          /
                 -------------------------      --------------------------
                  Flash 'sector' on disk                 Next cell

Kecepatan akan tetap tinggi, tetapi pada akhirnya Anda akan kehabisan sektor kosong. Ketika itu terjadi:

Sektor pertama diubah:

State before:  [used][used][used][used][used]    [empty][empty][empty][empty]  
State after:   [old] [old] [old ][old ][old]     [DATA] [used] [used] [used]
               \                          /      \                          /
                 -------------------------        --------------------------
                  Flash 'sector' on disk                   Next cell

Karena drive tidak dapat menulis ke sektor pertama, drive tersebut membaca semua data.
Itu kemudian menandai sektor lama sebagai tidak lagi relevan (tetapi tidak kosong) dan menulis ke sel bersih baru.

Sel lama tersisa dengan data yang tidak digunakan. Ini dapat dihapus (kembali ke [kosong]), tetapi itu biasanya hanya dapat dilakukan dengan kelompok sel senilai 1MB atau 2MB dan membutuhkan waktu lama.

Artinya:

  • Salah satu drive melakukan ini saat idle (benar untuk beberapa drive)
  • Atau setelah beberapa saat penulisan menjadi sangat lambat.

OS modern menyadari sifat SSD dan melakukan beberapa hal untuk membantunya (cari TRIM), sedangkan Windows XP tidak.

Itulah alasan pertama hal-hal menjadi lambat dengan SSD dan XP.

Ada item kedua yang juga memperlambat segalanya. NTFS (digunakan sebagai default oleh windows XP) memiliki ukuran sektor 4KB untuk sistem filenya.

Jika itu selaras dengan sel pada drive maka semuanya tidak baik-baik saja. Jika tidak maka Anda mendapatkan dua kali lipat masalah.

Unaligned example
4KB of data in a NTFS part:       [DATA] [DATA]     [DATA] [DATA]
Cells on the drive: [empty][empty][empty][empty]    [empty][empty][empty][empty]

Perhatikan tumpang tindih penulisan NTFS dan dua sel SSD? Tidak baik.

Partisi pintar dapat membantu dalam hal ini, tetapi Anda melakukannya selama instalasi XP, dan Anda harus memilih nilai yang baik secara manual.
