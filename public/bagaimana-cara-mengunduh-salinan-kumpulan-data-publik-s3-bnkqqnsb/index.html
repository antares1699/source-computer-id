Bagaimana cara mengunduh salinan kumpulan data publik S3?

saya dengan naif berasumsi saya bisa melakukan sesuatu seperti

s3cmd sync s3://snap-d203feb5 /var/tmp/copy

tapi sepertinya saya memiliki gagasan yang salah tentang bagaimana melakukan hal ini. Saya bahkan tidak bisa mendapatkan sesuatu yang sederhana untuk dikerjakan;

vnix$ s3cmd ls s3://snap-d203feb5
Bucket 'snap-d203feb5':
ERROR: Bucket 'snap-d203feb5' does not exist

Saya kira pengenal yang saya miliki bukan untuk "keranjang" tetapi untuk "kumpulan data publik". Bagaimana cara beralih dari satu ke yang lain? Apakah saya harus memulai instans EC2 dan membuat keranjang untuk ini? Bagaimana? Instruksi di http://docs.amazonwebservices.com/AWSEC2/latest/UserGuide/using-public-data-sets.html Sepertinya berasumsi saya ingin menggunakan data dalam contoh EC2, tetapi dalam kasus ini, saya hanya ingin menjelajah sedikit, setidaknya sebagai permulaan.

Oleh karena itu, salin / tempel "US Snapshot ID" menyebabkan pelacakan balik yang buruk dari Python; mereka mempublikasikan ID dengan tanda hubung Unicode (saya kira) aneh yang tidak dapat langsung disalin / ditempel. Apakah ada kesalahan saat saya menyalinnya? Dan apa pentingnya "AS" di sana? Tidak bisakah saya menggunakan data di luar Amerika Utara ??

Kumpulan data publik tidak dihosting di Amazon S3 dengan demikian, mereka disediakan sebagai Amazon Elastic Block Store (EBS) snapshot. Meskipun ini sebenarnya disimpan di S3, tidak mungkin untuk mengakses snapshot seperti itu secara langsung, Anda perlu membuat snapshot baru. Volume EBS darinya dan lampirkan ke file Amazon EC2 contoh untuk diproses lebih lanjut sesuai kebijaksanaan Anda.

Hanya menjelajahi kumpulan data tentu saja merupakan kasus penggunaan yang wajar, sayangnya saat ini Anda tidak dapat menghindari penggunaan instans EC2 dan volume EBS - lihat bagian Bagaimana itu bekerja untuk detailnya:

Kumpulan data publik tertentu dihosting di Amazon EC2 secara gratis sebagai Amazon Snapshot Elastic Block Store (Amazon EBS). Pelanggan Amazon EC2 bisa mengakses data ini dengan membuat volume Amazon EBS pribadi mereka sendiri, menggunakan snapshot kumpulan data publik sebagai titik awal. Mereka kemudian bisa mengakses, memodifikasi, dan melakukan penghitungan volume ini secara langsung menggunakan instans Amazon EC2 mereka [...]

Untuk mulai menggunakan Kumpulan Data Publik di AWS, cukup lakukan ini tiga langkah mudah:

  1. Mendaftar untuk akun Amazon EC2.
  2. Luncurkan instans Amazon EC2.
  3. Buat volume Amazon EBS menggunakan Snapshot ID yang tercantum dalam katalog di atas untuk snapshot pilihan Anda.

Bagaimana langkah-langkah ini dilakukan secara rinci dijelaskan dalam dokumentasi yang sudah Anda tautkan, yaitu. Meluncurkan sebuah Instance dan Membuat Volume Kumpulan Data Publik .

Setelah Anda membuatnya tersedia seperti itu, Anda mungkin menyimpan kumpulan data di bucket S3 Anda tentunya.
