ini tidak "ASCII" atau "ASCII Rusia".

Sebelum Unicode tersebar luas, sebagian besar sistem komputer menggunakan pengkodean karakter ISO-8859, yang ada 16, masing-masing untuk wilayah yang berbeda (Eropa Tengah, Sirilik, Yunani ...). Windows memiliki 'halaman kode' sendiri, sangat mirip tetapi dengan mesin terbang tambahan dalam rentang yang tidak digunakan. Semua pengkodean karakter ini adalah 8-bit dan hanya berbeda di paruh kedua (128-255).

Masalah dengan pengkodean ini adalah hampir tidak mungkin bagi program untuk menentukannya yang pengkodean digunakan untuk menyimpan file, kecuali jika ditentukan secara eksplisit (seperti di halaman HTML; namun, file teks biasa tidak memiliki tag metadata seperti itu). Baca artikel Wikipedia di Mojibake untuk penjelasan yang lebih detail.

Dalam contoh Anda, dokumen disimpan menggunakan Windows-1251 (Sirilik), tetapi program Anda membacanya seolah-olah Windows-1252 (Eropa Barat), yang memiliki karakter yang sangat berbeda pada posisi yang sama. Untuk komputer, ini terlihat baik-baik saja - tidak mengerti bahasa atau skrip. (Ada beberapa program yang melakukan analisis statistik untuk menentukan pengkodean yang benar, meskipun - beberapa browser web memiliki fungsi seperti itu.)

Ada beberapa cara untuk mengonversi teks seperti itu ke Unicode:

  • Menggunakan alat online seperti yang ini atau yang ini .

  • Gunakan browser web Anda:

    1. Seret file .txt ke browser.

    2. Dari Lihat → Pengkodean Karakter (atau Firefox → Pengembang Web → Pengkodean Karakter , atau Kunci Pas → Alat → Pengkodean ), pilih pengkodean asli yang benar: "Sirilik (Windows-1251)" dalam kasus Anda.

  • Menggunakan Notepad2 editor teks:

    1. Buka file.

    2. Dari File → Pengkodean → Pengodean Ulang ... , pilih pengkodean asli yang benar.

  • Gunakan GNU iconv, dengan binari Windows dari GnuWin32 atau Gettext untuk Win32 .

    iconv -f cp1251 -t utf-8 < myfile.txt > myfile.fixed.txt

    Windows Notepad akan membaca teks berenkode UTF-8 dan UTF-16 dengan benar.
