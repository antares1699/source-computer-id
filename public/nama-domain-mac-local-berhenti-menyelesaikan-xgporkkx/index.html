Nama domain Mac .local berhenti menyelesaikan

Laptop saya memiliki nama host Michaels-MacBook-Pro.local, dan saya dapat melakukan ping dengan ping Michaels-MacBook-Pro.local.

Hari ini, itu berhenti begitu saja. Saya dapat mengakses laptop rekan kerja dengan ping Name-MacBook-Pro.local, dengan nama mereka, tetapi milik saya, saya (dan semua orang di kantor) baru mendapatkan ping: cannot resolve Michaels-MacBook-Pro.local: Unknown host

Anehnya, ini terjadi sekitar satu jam setelah saya berhasil mengupgrade OS ke Sierra. Ini berhasil beberapa saat, lalu berhenti.

Jika seseorang bahkan bisa memberi saya petunjuk ke dokumentasi tentang bagaimana itu seharusnya untuk bekerja, saya akan berterima kasih.

Di Sierra, komputer Anda tidak akan mendaftarkan versi .local dari namanya kecuali Anda memulai salah satu layanan Berbagi (seperti Berbagi File, misalnya).

Saya tidak yakin mengapa itu terjadi dan jika ada cara yang lebih baik untuk menghindari itu, tetapi mengaktifkan Berbagi File (atau layanan Berbagi lainnya) akan membuat entri untuk Michaels-MacBook-Pro.local di Penemuan Layanan DNS (Anda dapat memeriksanya melalui: dns-sd -G v4 Michaels-MacBook-Pro.local, dan bandingkan dengan versi non = lokal: dns-sd -G v4 Michaels-MacBook-Pro).

Melengkapi saran tnarik untuk mengaktifkan layanan berbagi, yang merupakan langkah penting yang sangat membantu saya di Sierra, v10.12.6:

Saya juga harus menghapus centang "Aktifkan mode siluman" di preferensi firewall saya untuk menerima balasan ping melalui .local.

Kotak centang ada di "System Preferences" -> "Security & Privacy" -> "Firewall" -> "Firewall Options ..." -> "Enable stealth mode" [dengan deskripsi "Jangan tanggapi atau akui upaya untuk mengakses komputer ini dari jaringan dengan menguji aplikasi menggunakan ICMP, seperti Ping "]
