Setel ulang lebar scroll bar horizontal di Notepad ++

Saya memiliki dokumen terbuka di Notepad ++, dan untuk beberapa alasan scrollbar horizontal kecil. Sepertinya lebar area teks melebihi 7000 karakter. Dokumen tersebut memang memiliki satu baris panjang, sekitar 1000 karakter, tetapi tentu saja jauh lebih kecil dari area yang dicadangkan untuk itu.

Apakah ada cara untuk mengatur ulang ukuran horizontal di Notepad ++? Sangat merepotkan untuk menggulir dengan bilah yang sangat kecil, terutama karena area yang digunakan hanya 10-20% dari total.

Ini tidak mungkin.

Notepad ++ menggunakan komponen editor Scintilla yang tidak menyediakan fitur ini. Ya, itu akan meningkatkan rentang gulir horizontal tetapi tidak mengurangi saya t.

Inilah bagian yang relevan dari dokumentasi (penekanan saya):

Jika pelacakan lebar gulir diaktifkan maka lebar gulir disesuaikan untuk memastikan bahwa semua garis yang saat ini ditampilkan dapat digulir sepenuhnya. Mode ini tidak pernah menyesuaikan lebar gulir menjadi lebih sempit .

Pengembang Scintilla Neil Hodgson telah berulang kali mengatakan bahwa fitur ini tidak akan diterapkan karena biaya kinerja untuk menentukan jalur terluas saat ini secara dinamis.

Jika baris terpanjang Anda sekitar 1000 karakter tetapi rentang pengguliran Anda lebih besar, Anda selalu dapat menutup dan membuka kembali file untuk menyetel ulang rentang ke baris terpanjang Anda.

Tidak ada cara untuk mengatur ulang secara otomatis, tetapi dimungkinkan untuk mengatur ulang secara manual. Berdasarkan tanggapan terhadap masalah yang dilaporkan ini , Anda dapat menggunakan plugin scripting untuk mengubah lebar scroll horizontal scintilla. Saya bisa membuatnya bekerja menggunakan Skrip Python plugin dengan skrip 1 baris berikut:

editor.setScrollWidth(1)

Jika bilah gulir perlu lebih panjang, Anda akan melihatnya menciut terlalu banyak sehingga pelacakan gulir horizontal akan segera meluas ke lebar yang diperlukan.
