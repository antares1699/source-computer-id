Bisakah / usr / bin / sftp melanjutkan transfer file?

Saya memiliki mesin dengan hanya akses sftp. Tanpa ssh, tanpa shell, tanpa rsync. Ketika saya mendapatkan file menggunakan / usr / bin / sftp, dapatkah saya membuatnya melanjutkan transfer yang dibatalkan?

Saya menggunakan lftp untuk menyinkronkan semua foto saya dari NAS ke server web murah saya secara otomatis. Jika ada foto berubah maka secara otomatis akan memperbaruinya.

#!/bin/ash
SOURCE=/volume2/PHOTO/2011
BACKUPDIR=/foto.whatever.com/albums/2011

lftp -u username,password ftp.whatever.com << EOF
mirror -R -n -I *.jpg -I *.JPG -X @eaDir/ -X Collage/ -X ‘whatever/’ -X .piccache/ -X .recent/ -X Originals/ -X *.Db $SOURCE $BACKUPDIR
quit

Anda dapat menggunakan opsi -a dalam perintah get pada sftp untuk melanjutkan unduhan yang sebelumnya dibatalkan.

Contoh:

echo -e "lcd ~/localFolder\nget -ar *" | sftp username@host.com:/some/path

Atau ditulis sebagai

sftp username@host.com:/some/path <<EOF
lcd ~/localFolder
get -ar *
EOF

Namun ini tidak secara otomatis terhubung kembali ke server jika koneksi terputus. lftp adalah alternatif yang lebih baik untuk masalah ini.
