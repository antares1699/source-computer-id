Ekstensi pgcrypto diinstal tetapi fungsinya tidak tersedia

Pada server klien saya mengalami masalah - saya tidak memiliki izin untuk membuat ekstensi pgcrypto. Jadi admin server melakukannya untuk saya, dan sekarang saya dapat melihat di pg_available_extensions bahwa pgcrypto diinstal (1.1), namun fungsinya tidak tersedia untuk saya:

=> select gen_random_uuid();
ERROR:  function gen_random_uuid() does not exist
LINE 1: select gen_random_uuid();
               ^
HINT:  No function matches the given name and argument types. You might need to add explicit type casts.

Juga saya tidak memiliki izin untuk melepaskan ekstensi (yang mungkin menjadi petunjuk apa yang salah?):

=> drop extension pgcrypto;
ERROR:  must be owner of extension pgcrypto

Tahu apa yang salah dan mengapa saya tidak bisa melihat fungsi pgcrypto? Sesuatu dengan izin?

Beberapa penjelasan yang mungkin:

  • Versi PostgreSQL lebih lama dari 9.4. Fungsi ini gen_random_uuid() tidak ada sebelumnya http://www.postgresql.org/docs/9.4/static/release-9-4.html :

    Tambahkan UUID generator nomor acak gen_random_uuid () ke pgcrypto (Oskari Saarenmaa)

    Gunakan select version(); untuk memeriksa versi.

  • atau 9.4+, tetapi ekstensinya dibuat dalam skema yang tidak ada dalam sesi SQL search_path Anda. Gunakan perintah meta \dx di psql untuk mendaftar ekstensi yang diinstal, dan centang kolom Schema.

  • atau ekstensi berada dalam skema yang disertakan dalam search_path, tetapi pengguna db tidak memiliki izin untuk memeriksanya. Berikan dengan: GRANT USAGE ON SCHEMA schema_name TO role_name.
