Cara menggunakan openvpn untuk menjelajah

Saya baru memulai dengan VPN dan OpenVPN. Apa yang saya pahami adalah bahwa saya ketika saya memiliki koneksi vpn, itu seperti saya berada di jaringan yang sama dengan server. Maka saya harus dapat terhubung ke server melalui alamat lokal (10.8.x.x). Selanjutnya saya harus bisa browsing internet melalui server.

Saya memiliki server sendiri di cloud untuk hal-hal pribadi. Saya bisa menggunakannya untuk apapun yang saya suka. Ini berjalan di Debian 7. Saya menginstal openvpn dan dapat terhubung dari laptop saya. Saya pikir ini akan mengubah alamat IP saya ke salah satu server, tetapi ketika saya membuka whatismyip.com di browser, saya masih melihat IP normal saya.

Alih-alih menggunakan OpenVPN, gunakan tunneling SSH. Ini bekerja lebih baik dan lebih mudah untuk mengatur dan juga dienkripsi. Salah satu bonus tambahannya adalah tidak seperti OpenVPN, jika koneksi ditutup, Anda tidak dapat lagi menjelajah internet. Dengan OpenVPN, jika koneksi ditutup, Anda akan secara otomatis dikembalikan ke jaringan Anda sendiri dan diizinkan untuk melanjutkan penjelajahan tanpa peringatan. Tidak ada yang perlu diubah di server Anda.

Jika Anda menjalankan Windows di komputer Anda:

  1. Unduh PuTTY (jika Anda belum memilikinya)
  2. Di panel samping Kategori, buka Koneksi> SSH dan centang Aktifkan Kompresi
  3. Di panel samping Kategori, buka Connection> SSH> Tunnels dan masukkan nomor di bidang port sumber. Saya biasanya menggunakan 3456. Kemudian pilih Dynamic dan terakhir klik Add.
  4. Di panel samping Kategori, kembali ke Sesi. Masukkan username @ hostname Anda (ganti your_username dengan nama pengguna Anda pada sistem dan nama host dengan domain atau alamat IP server Anda) di bidang nama host dan kemudian simpan sesi.
  5. Klik Buka, lalu masukkan kata sandi Anda.
  6. Unduh Firefox jika Anda belum memilikinya.
  7. Di Firefox, buka Firefox> Opsi lalu Lanjutan> Jaringan> Pengaturan.
  8. Pilih Konfigurasi proxy manual dan kemudian hapus semua yang ada di semua kotak yang dapat diedit.
  9. Masukkan localhost sebagai SOCKS Host dan masukkan 3456 (atau port apa pun yang Anda gunakan pada langkah 3) sebagai Port.
  10. Klik Ok untuk kedua jendela konfigurasi Firefox yang terbuka.
  11. Masukkan about: config di bilah lokasi Firefox dan tekan enter. Beri tahu Firefox bahwa Anda akan berhati-hati.
  12. Cari network.proxy.socks_remote_dns dan klik dua kali untuk menyetelnya ke true.

Jika Anda menjalankan Linux atau Mac OS di komputer rumah:

  1. Buka terminal dan ketik ssh -CD: 3456 your_username @ hostname (ganti your_username dengan nama pengguna Anda pada sistem dan nama host dengan domain atau alamat IP server Anda) dan tekan enter.
  2. Unduh Firefox jika Anda belum memilikinya.
  3. Di Firefox, buka Edit> Preferensi, lalu Lanjutan> Jaringan> Pengaturan.
  4. Pilih Konfigurasi proxy manual dan kemudian hapus semua yang ada di semua kotak yang dapat diedit.
  5. Masukkan localhost sebagai SOCKS Host dan masukkan 3456 (atau port apa pun yang Anda gunakan pada langkah 3) sebagai Port.
  6. Klik Ok untuk kedua jendela konfigurasi Firefox yang terbuka.
  7. Masukkan about: config di bilah lokasi Firefox dan tekan enter. Beri tahu Firefox bahwa Anda akan berhati-hati.
  8. Cari network.proxy.socks_remote_dns dan klik dua kali untuk menyetelnya ke true.

Setelah Anda selesai, tutup PuTTY atau terminal dan setel Firefox untuk menggunakan Tanpa proxy alih-alih yang manual yang Anda konfigurasikan. Lain kali Anda ingin menyambungkan, jalankan saja sesi PuTTY yang disimpan atau jalankan perintah baris perintah dan setel Firefox untuk menggunakan opsi konfigurasi proxy Manual. Firefox akan mengingat pengaturan proxy Anda.

(Perhatikan bahwa di mana saya telah memberikan contoh, mereka minimal, dan Anda perlu memuaskan diri Anda sendiri, Anda memahami cara kerjanya dan menyempurnakannya, Anda dapat mengunci diri Anda sendiri di luar kotak Anda. Selain itu, petunjuk ini tidak permanen - jadi Anda setidaknya dapat melakukan boot ulang untuk menghilangkan masalah jika Anda melakukan sesuatu yang bodoh.) Saya tidak yakin bahwa proxy seperti ini adalah ide yang bagus sebagai solusi yang selalu aktif karena kerumitan, kecepatan yang berkurang, dan masalah curveball yang akan dilontarkannya kepada Anda.

Anda tidak perlu mengkonfigurasi proxy. Yang perlu Anda pastikan adalah bahwa server OpenVPN Anda menyediakan rute default (dan dengan metrik yang lebih rendah sehingga lebih disukai) melalui server OpenVPN.

push "redirect-gateway XXXX def1"

Anda mungkin juga perlu memberi tahu server OpenVPN ke NAT koneksi sehingga ia akan membagikan alamat IP server - karena 10.x.x.x tidak dirutekan secara global. Anda dapat melakukan ini dengan menambahkan perintah (di server)

iptables -t NAT -A POSTROUTING -o ethX -j MASQUERADE

Jika Anda ingin memastikan Anda terhubung melalui OpenVPN, Anda ingin memasang firewall pada klien VPN Anda yang mencegah lalu lintas keluar melalui antarmuka normal (kecuali lalu lintas ke server OpenVPN).

iptables -I INPUT -j DROP
iptables -I INPUT -s SERVER.IP -j ACCEPT
