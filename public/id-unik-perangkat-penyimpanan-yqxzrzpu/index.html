ID unik perangkat penyimpanan

Saya telah menemukan sistem lisensi yang entah bagaimana terkait dengan media penyimpanan. Meskipun mendasarkan ini pada UUID pada filesystem cukup mudah, saya tidak percaya ini masalahnya (lihat di bawah). Contoh tipik dari perilaku ini adalah sebagai berikut:

  1. Pada setiap boot, file id dibuat.
  2. Sistem akan memeriksa file kunci untuk lisensi. Jika kuncinya cocok dengan id, maka semuanya baik-baik saja.
  3. Jika tidak ada kunci, atau jika kunci tidak cocok dengan id (seperti, disalin dari file sistem yang berbeda), lisensi tidak valid.

Saya telah mengamati perilaku berikut:

  • Menghapus sepenuhnya kartu CF dan menginstal ulang menghasilkan file id yang sama, sehingga tidak diperlukan file kunci baru.
  • Menggunakan kunci yang sama pada CF yang berbeda tidak akan bekerja, karena sebenarnya tidak cocok dengan konten file id baru.

Ini membuat saya menyimpulkan bahwa ada cara untuk mendapatkan id unik dari kartu CF entah bagaimana caranya. Bagaimana seseorang bisa mencapai ini di linux?

PS: Saya sengaja meninggalkan detail produk itu sendiri, karena saya bertanya-tanya lebih dalam secara umum bagaimana sistem seperti ini dapat membangun id unik dari kartu CF.

Kartu tersebut akan memiliki banyak informasi yang terdaftar dengan cara hanya-baca.

Artikel Membaca Informasi Kartu SD Internal mengatakan ini:

Informasi Kartu SD Internal

Informasi tentang kartu SD dikodekan di kartu internalnya pendaftar. Salah satunya adalah Identifikasi Kartu (CID) Daftar, a Kode 16 byte yang berisi informasi yang secara unik mengidentifikasi SD kartu, termasuk nomor seri kartu (PSN), nomor ID produsen (MID) dan tanggal pembuatan (MDT). Register CID diset ketika kartu diproduksi dan tidak dapat diubah setelah disetel. (Menurut untuk spesifikasi kartu SD informasi hanya untuk ditulis sekali, Namun jika kartu tidak sesuai dengan spesifikasi ini informasi dapat diubah!)

Cara membaca CID dari kartu SD

Salah satu cara membaca CID adalah dengan menggunakan laptop dengan slot kartu SD. Kartu pembaca di laptop biasanya terhubung langsung melalui bus PCI (atau bus IDE). Ini tidak akan bekerja melalui pembaca kartu USB karena perintah untuk mengambil informasi kartu dicegat dan tidak dipahami oleh pembaca kartu. Tidak semua komputer dengan slot kartu built-in akan berfungsi, beberapa pembaca kartu internal terhubung melalui bus USB.

Dengan asumsi Anda memiliki perangkat keras yang tepat, ada beberapa metode untuk Anda dapat digunakan untuk mendapatkan informasi kartu. Dengan Linux, membaca internal Informasi kartu SD sederhana. Masukkan kartu dan lihat di bawah /sys/block/mmcblk0/device/ (lokasi ini dapat berubah tergantung pada Anda alat). Anda akan melihat beberapa atribut yang tersedia yang mencakup CID register dan informasi di dalamnya.

Seperti yang dikomentari orang lain, ini mungkin terkait dengan cara kartu CF telah dihapus, tetapi kemungkinan besar perangkat lunak mengidentifikasi kartu CF berdasarkan firmware di kartu -

Kabin informasi ini biasanya dibaca menggunakan "udevinfo" - misalnya, jika kartu CF adalah / dev / sdb udevinfo -a -p $ (udevinfo -q path -n / dev / sdb) - ini kemungkinan besar akan menyertakan informasi yang cukup untuk mengidentifikasi kartu secara unik.

Catatan : pada distro berbasis debian udevadm info menggantikan udevinfo:

udevadm info -a -p $( udevadm info -q path -n /dev/sdb )
