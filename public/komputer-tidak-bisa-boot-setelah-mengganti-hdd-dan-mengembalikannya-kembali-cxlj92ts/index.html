Komputer

Komputer tidak bisa boot setelah mengganti hdd dan mengembalikannya kembali

1) Saya memiliki HDD yang ada di komputer saya yang mem-boot ganda linux dan windows menggunakan cara bios tradisional.

2) Saya kemudian mengeluarkan HDD yang ada dan menukar HDD baru untuk menginstal linux dengan uefi. Itu boot dan reboot dengan baik.

3) Saya mengeluarkan HDD baru dengan uefi linux dan mengembalikan HDD lama dengan bios dual boot. Boot HDD lama baik-baik saja.

4) Saya menukar HDD lama dengan HDD baru yang memiliki linux uefi di atasnya dan menolak untuk boot.

Adakah ide tentang apa yang bisa menjadi masalah dan bagaimana mengatasinya?

Ini terdengar seperti masalah urutan boot, setiap kali hard drive ditukar, urutan boot harus diubah di bios. Entri UEFI di bios terpisah dari hard disk Legacy dalam urutan boot bios. Setiap kali Anda melepaskan disk UEFI dan menggantinya dengan disk lama, entri akan dihapus dari urutan boot.

Setelah menginstal disk UEFI Anda harus masuk ke pengaturan bios dan mengatur opsi boot pertama bukan ke hard disk 1, melainkan ke bootloader UEFI (grub) yang ada di partisi EFI hard disk 1. Sayangnya proses ini sangat bervariasi karena untuk implementasi UEFI yang buruk di sebagian besar motherboard, jadi saya tidak dapat memberikan Anda petunjuk khusus tentang cara melakukannya.

Bagus ya untuk diberitahu tentang UEFI. Versi Windows apa yang Anda gunakan? Win8 benar?

~ Apakah Anda mendapatkan bootloader GRUB, setidaknya, pada drive baru kedua?

Jika ya, Anda mungkin hanya perlu perbaikan startup sederhana, default, Win untuk mengevaluasi kembali sektor boot Windows (permulaan dan berakhir) setelah instalasi Linux. Selain itu, "Linux" tidak selalu berfungsi baik dengan UEFI atau mendeteksi partisi GPT.

~ Saat Anda menukar di antara dua drive, apakah Anda menggunakan port SATA yang sama setiap kali? yaitu: Port 0,1,2,3 seperti yang diberi label di MOBO?
