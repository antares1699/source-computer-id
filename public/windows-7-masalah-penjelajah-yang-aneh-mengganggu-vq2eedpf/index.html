Setiap kali Anda membuat folder baru, Anda harus menekan tombol enter DUA KALI karena pertama-tama akan muncul kesalahan bahwa "tidak ada file seperti itu ..". Baik. Kemudian jika Anda mencoba menghapus beberapa, ini juga terjadi. Mengganti nama sama saja.

Semua barang di PC legal, tidak ada crack / perangkat lunak lain yang hadir. Itu terjadi begitu saja, begitulah. Sudah menginstal ulang PC lain karena saya pikir * Saya * mengacaukan sesuatu dan itu hanya sesuatu yang FUBAR. Namun, saya juga tidak ingin menginstal ulang laptop saya.

Cara saya melihatnya, entah ada aplikasi jahat di komputer Anda, atau Windows menjadi buruk. Opsi kedua mungkin memerlukan penginstalan ulang.

Taruhan terbaik Anda adalah (dalam urutan keputusasaan):

  • Lihat apakah masalah terjadi kapan booting dalam mode Aman.
    Jika tidak terjadi: Maka masalahnya adalah dengan aplikasi terinstal atau pengemudi (kemungkinan besar video).
    Jika itu masih terjadi: Windows mungkin buruk.
  • Pindai aplikasi yang diinstal dan hapus instalan apa pun yang tampak meragukan, Gunakan Revo untuk itu.
  • Menggunakan autoruns untuk memverifikasi semua aplikasi yang dimulai dengan Windows.
  • Menggunakan Proses Explorer untuk memverifikasi bahwa tidak ada program yang mengunci file direktori
  • Menggunakan Pemeriksa Berkas Sistem (SFC) ke memindai file sistem dan mengganti versi yang salah dari mereka.
  • Segarkan Windows 7 dengan melakukan peningkatan instalasi.
  • Format ulang dan instal ulang Windows.

EDIT

Karena di sini ada dua orang dengan masalah rahasia yang sama, mungkin berguna untuk mengetahui kesamaan Anda.

Dapatkah Shiki menambahkan pada pertanyaannya rincian perangkat keras, versi dan bahasa Windows, dan daftar aplikasi yang terpasang? Kemudian TFM dapat mengomentari daftar tersebut.

Berikut adalah cara seseorang mendapatkan daftar semua aplikasi yang diinstal dengan menggunakan antarmuka baris perintah Instrumentasi Windows (WMIC) pada file teks:

  1. Mulai Prompt Perintah.
  2. Ketik wmic dan tekan Return.
  3. Anda akan melihat prompt baru: wmic: root \ cli>
  4. Menjalankan
    /output:c:\appList.txt product get name,version
    (Untuk menghentikan gunakan perintah berhenti.)
  5. Daftar aplikasi ditemukan di file c: \ appList.txt.
    Ini akan membutuhkan beberapa pengeditan teks untuk mengurangi jumlah baris.

EDIT2

Setelah banyak penelitian, tampaknya Explorer di Windows 7 memiliki banyak masalah dengan menyegarkan daftar file, yang menyebabkan masalah dengan semua operasi file. Masalah ini sudah diketahui umum tetapi tetap tidak disadari oleh Microsoft. Satu harapan bahwa mereka akan menghilang dengan paket layanan yang akan datang.

Solusi yang jelas adalah menggunakan pengganti pihak ketiga untuk Explorer.

Beberapa solusi yang tidak jelas dieksplorasi dalam utas yang sangat panjang yang ditemukan di sini:
Windows Explorer tidak menyegarkan saat memindahkan / menghapus .
Banyak orang mengklaim telah menemukan solusi, yang berhasil untuk beberapa tetapi tidak untuk yang lain.
Saya mencantumkan di bawah sebagian besar solusi, yang saya anggap adalah urutan kemungkinannya berfungsi:

  1. Hapus komputer dari Homegroup
  2. Hapus semua pembagian jaringan
  3. Menghapus centang klien MS untuk windows, dan mematikan semua opsi berbagi
  4. Hapus IconCache.db
  5. Urutkan file di Explorer berdasarkan nama, bukan berdasarkan tanggal
  6. Bangun kembali Indeks Pengalaman Windows.
