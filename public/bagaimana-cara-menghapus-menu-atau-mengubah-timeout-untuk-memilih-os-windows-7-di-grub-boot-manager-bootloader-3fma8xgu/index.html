Bagaimana cara menghapus menu atau mengubah TIMEOUT untuk memilih OS (Windows 7) di grub boot manager / bootloader?

ketika saya menyalakan notebook saya (Lenovo V580c) jendela ini muncul:

dan TIMEOUT untuk memilih salah satunya adalah 30 detik. hanya ada windows 7 di laptop ini (selain FreeDOS default yang awalnya diinstal di atasnya), jadi saya tidak ingin melihat Menu ini setiap kali saya menyalakan notebook saya. atau setidaknya ubah TIMEOUT menjadi 5 detik (misalnya).

Lantas, Bagaimana cara mengetahui nama dan versi bootloader ini? dan bagaimana mengubah konfigurasi seperti TIMEOUT dan menu acara?

  • tekan Alt + F2 , ketik gksudo gedit /etc/default/grub tekan Memasukkan dan masukkan kata sandi Anda.

  • Anda akan melihat konten berikut:

    GRUB_DEFAULT=0
GRUB_HIDDEN_TIMEOUT=0
GRUB_HIDDEN_TIMEOUT_QUIET=true
GRUB_TIMEOUT=10
GRUB_DISTRIBUTOR=`lsb_release -i -s 2> /dev/null || echo Debian`
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"
GRUB_CMDLINE_LINUX=""

  • Anda dapat mengubah default dari 0 ke angka apa pun, sesuai dengan entri di menu booting Grub (entri boot pertama adalah 0, kedua adalah 1, dll.)

  • Anda dapat mengubah "batas waktu tersembunyi" (tidak ada menu); dan juga menampilkan hitungan mundur (TIMEOUT_QUIET=false)
  • Anda dapat memaksa menu grub untuk ditampilkan dengan mengomentari dua baris GRUB_HIDDEN dengan # di awal baris

  • Dan atur batas waktu menu grub (default adalah 10 detik)

  • Lakukan perubahan Anda, tekan Ctrl + S untuk menyimpan dan Ctrl + Q untuk keluar.

  • Penting : Buka terminal dengan Ctrl + Alt + T dan ketik gksudo gedit /etc/default/grub untuk menerapkan perubahan yang baru saja Anda buat.
  • Mulai ulang dan Anda akan melihat batas waktu / entri default Anda berubah.

Itu GNU GRUB, versi 1.xx atau lebih tinggi, juga dikenal sebagai GRUB 2. Dengan boot berbasis BIOS tradisional, ia hidup di Master Boot Record dan di sektor yang biasanya tidak digunakan yang terletak setelah MBR tetapi sebelum permulaan yang pertama partisi. Ini juga biasanya memiliki file konfigurasinya dan beberapa modul tersedia sebagai file dalam salah satu partisi disk Anda, tetapi bagian ini secara opsional dapat disematkan di dalam sektor disk tersebut.

Instruksi yang diberikan oleh user345866 akan dapat diterapkan untuk mengkonfigurasi GRUB dari dalam OS Linux, tetapi tampaknya Anda tidak memilikinya. Fakta bahwa bootloader Anda menyertakan opsi (No SLIC) membuat saya berpikir bahwa versi Windows Anda mungkin merupakan versi bajakan.

(SLIC adalah versi lama dari lisensi Windows yang disematkan BIOS, dan ternyata dapat digandakan dan "dapat ditransplantasikan" ke sistem lain. Bootloader Anda mungkin seperti itu. Versi modern adalah MSDM: ini terkait dengan sistem tertentu lisensi dikeluarkan untuk, menggunakan kriptografi yang cukup kuat.)

Dalam hal ini, bootloader GRUB dapat digunakan sebagai mekanisme pengiriman untuk alat yang digunakan untuk melewati pemeriksaan lisensi Windows, dan konfigurasinya mungkin sepenuhnya tertanam dalam sektor disk yang biasanya tidak digunakan antara MBR dan awal partisi pertama. Jika demikian, maka konfigurasi GRUB tidak dapat diubah tanpa alat dan file konfigurasi yang semula digunakan untuk "memaketkan" instalasi GRUB tersebut. Jadi, kecuali Anda menemukan file grub.cfg di salah satu partisi disk Anda, Anda mungkin kurang beruntung.
