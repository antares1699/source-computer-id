Apakah layak / mungkin ISP saya melakukan pembatasan bittorrent per klien?

Tampaknya sangat aneh bagi saya, tetapi saya telah menjalankan beberapa pengujian, dan akan menyukai beberapa ide pengujian lainnya. Saya memiliki kotak linux di sini yang menjalankan Deluge, tempat sebagian besar aktivitas bittorrent saya terjadi, dan kotak Windows tempat saya melakukan lebih sedikit. Saya telah melakukan tes ini:

Mengunduh torrent yang sama, melalui porta penerusan yang sama di windows dan linux.

Mengunduh torrent yang sama, melalui porta penerusan yang berbeda di windows dan linux.

Mengunduh torrent yang sama, menggunakan perangkat keras dari kotak windows di linux (menggunakan file livecd).

Mengunduh torrent yang sama, Menggunakan kotak linux, dengan VM windows yang berjalan di kotak virtual. (Saya tahu, bukan NYATA uji)

Hasilnya selalu sama. Di bawah windows, menggunakan uTorrent 3.1, saya melihat lonjakan lalu lintas dan kemudian lambat hingga sedikit untuk sisa pengunduhan. Di bawah linux, menggunakan deluge 1.3.3, lalu lintas mempertahankan kecepatan tinggi selama pengunduhan. Saya bingung dengan perilaku ini. Apakah saya telah melewatkan beberapa ujian yang seharusnya saya lakukan? Bagaimana mungkin ISP saya untuk membuat profil aktivitas bittorrent berbasis windows dan bukan lalu lintas linux?

Banyak ISP saat ini melakukan semacam pembentukan lalu lintas pada lalu lintas dan biasanya ini pada lalu lintas P2P karena alasan yang cukup jelas. Jadi ya, kemungkinan besar ISP Anda memberi pengaruh pada koneksi.

Saya akan menyarankan untuk melakukan pengambilan paket dari dua mesin sehingga Anda bisa mendapatkan gambaran yang lebih baik tentang apa yang sebenarnya terjadi, Anda mungkin melihat hal-hal seperti paket RST yang berlebihan dari rekan torrent jarak jauh Anda. Dengan perangkat keras yang tepat, ini dapat dipalsukan oleh ISP Anda untuk mengelabui perangkat lunak agar mengira koneksi telah ditutup, yang secara efektif membentuk kecepatan. Lakukan pencarian cepat untuk "sandvine rst tcp" jika Anda ingin membaca lebih banyak tentang itu.

Tentunya ada banyak metode lain yang dapat digunakan ISP Anda, itu hanya sebuah contoh.
