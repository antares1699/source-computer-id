Tentang cache memori Linux

Saya menjalankan skrip python untuk melakukan beberapa statistik dan sebenarnya memori yang digunakan rendah, sekitar 10%. Dan tidak ada proses lain yang membutuhkan lebih banyak memori. Namun, ketika saya menggunakan free -m dan itu menunjukkan bahwa hampir 95% memori telah digunakan. Intinya adalah bahwa skrip saya harus banyak membaca dari file, jadi saya bertanya-tanya apakah ada mekanisme cache memori Linux yang menyebabkan masalah? echo 1 >> /proc/sys/vm/drop_caches berfungsi, tetapi tampaknya secara manual. Bagaimana saya bisa mengurangi biaya memori dan tidak membuat efek buruk pada membaca file?

Kernel Linux secara otomatis menyimpan file dalam memori untuk efisiensi. Ini bukan hal yang buruk. Saat menjalankan free -m Anda akan melihat di sebelah kanan jumlah memori yang digunakan dalam cache. Jika aplikasi membutuhkan memori, kernel akan membebaskan beberapa cache. Anda tidak boleh mencoba dan mengelolanya sendiri.

Saya tidak berpikir Anda memiliki masalah yang sebenarnya di sini.

Mengapa Anda peduli berapa banyak total memori yang digunakan? Apakah itu menyakitkan?

Linux mencoba menggunakan memori Anda seefisien mungkin. Jika Anda memiliki banyak memori gratis, dan Anda membaca banyak file, itu akan menggunakan memori gratis untuk menyimpan file-file itu. Jika Anda memiliki banyak memori aktif, itu akan menggunakan lebih sedikit memori untuk caching. Begitulah cara kerjanya. Jika Anda tidak ingin memori digunakan sama sekali, keluarkan chip dari komputer Anda.

Ada beberapa kasus yang jarang terjadi ketika banyak penyimpanan dalam cache bisa berakibat buruk. Mungkin ledakan membaca selalu terjadi tepat sebelum ledakan alokasi memori. Mungkin sesuatu tentang kasus spesifik Anda berarti tidak ada kemungkinan manfaat, dan Anda perlu memeras biaya kinerja 0,1% terakhir itu. Mungkin Anda menggunakan sistem tertanam yang menggunakan RAM dengan masa pakai terbatas. Jika Anda memiliki kasus seperti itu, beri tahu kami.
