Bagaimana cara 'menyelaraskan' hasil fitur "Teks ke Kolom" di Excel?

Jika saya menggunakan fitur Teks ke Kolom pada data berikut, menggunakan ";" sebagai pembatas:

foo; bar; qux; baz; roti panggang;
quux; selai; kacang;

Saya akan mendapatkan hasil "rata kiri" di kisi sel yang dihasilkan:

 |foo   |bar   |qux   |baz   |toast |
 |quux  |jam   |beans |      |      |

Namun, saya ingin mereka "rata kanan":

 |foo   |bar   |qux   |baz   |toast |
 |      |      |quux  |jam   |beans |

Bagaimana saya bisa melakukan ini?

CATATAN: Saya tahu bahwa "rata kanan" mungkin bukan istilah yang tepat, malah menyiratkan

|   foo|   bar|   qux|   baz| toast| 
|  quux|   jam| beans|      |      |

tapi bukan ini yang kucari. Jadi, jika ada yang bisa menyarankan istilah yang lebih baik untuk apa yang saya gambarkan, silakan lakukan.

Tambahan: Sebagai pendekatan alternatif, jika ada yang tahu cara menggunakan Excel untuk mengatur ulang sel seperti itu

 |a   |b   |c   |d   |    |    |    |    |    |
 |n   |m   |o   |p   |q   |    |    |    |    |
 |e   |f   |g   |h   |i   |j   |k   |l   |    |
 |n   |m   |o   |p   |q   |    |    |    |    |
 |x   |    |    |    |    |    |    |    |    |

menjadi

 |    |    |    |    |    |a   |b   |c   |d   |
 |    |    |    |    |n   |m   |o   |p   |q   |
 |    |e   |f   |g   |h   |i   |j   |k   |l   |
 |    |    |    |    |n   |m   |o   |p   |q   |
 |    |    |    |    |    |    |    |    |x   |

maka itu juga akan berhasil.

Rumus berikut akan memungkinkan konversi cepat data Anda ke formulir yang Teks-ke-Kolom akan langsung diurai dengan rata kanan seperti yang Anda gambarkan:

Rumus D5 (menambahkan titik koma jika tidak ada):

=IF(RIGHT(B5,1)<>";",B5&";",B5)

Rumus G5 (menambahkan jumlah titik koma yang diperlukan):

=REPT(";",5-(LEN(D5)-LEN(SUBSTITUTE(D5,";",""))))&D5

Menyalin hasil yang diikuti dengan Tempel-Khusus-sebagai-Nilai harus mampu menghasilkan bahan mentah yang sesuai untuk konversi Teks-ke-Kolom.

Solusinya tergantung pada jumlah kolom maksimum yang tetap; di sini, lima. Rumus G5 dapat digeneralisasikan dengan menambahkan sel 'sejumlah kolom untuk menghasilkan' di tempat lain pada lembar dan merujuk sel baru ini alih-alih nilai 5 yang di-hardcode.

Selain itu, jika Anda dijamin bahwa data akan selalu memiliki tanda titik koma, langkah perantara dari =IF(RIGHT(B5,1)<>";",B5&";",B5) tidak berguna.

EDIT: Berdasarkan pengamatan Some_Guy di komentar, metode ini juga akan berfungsi jika semua baris dikonstruksikan kekurangan tanda titik koma.

Seperti yang dinyatakan, ini bukan fungsi standar teks ke kolom atau apakah ada cara yang melekat untuk melakukan ini di excel, yang saya tahu. Namun, VBA ini akan melakukannya untuk Anda ( dengan asumsi Anda tidak memiliki ruang kosong di antara sel yang diisi ) -

Sub test()
Dim lrow As Integer
lrow = Cells(Rows.Count, "A").End(xlUp).Row

Dim lcol As Integer
lcol = Cells("1", Columns.Count).End(xlToLeft).Column

Dim lfcol As Integer
Dim dif As Integer
For i = 1 To lrow
    lfcol = Cells(i, Columns.Count).End(xlToLeft).Column
    dif = lcol - lfcol
    For j = lfcol To 1 Step -1
        If dif = 0 Then Exit For
        If Not Cells(i, j) Is Nothing Then
            Cells(i, j + dif) = Cells(i, j)
            Cells(i, j) = vbNullString
        End If
    Next
Next
End Sub

Berikut ini rutinitas VBA lain untuk melakukannya. Lakukan Teks Anda ke Kolom, lalu pilih rentang persegi panjang tempat Anda memasukkan data (mis., kolom A- (bidang maks) × baris) dan jalankan makro ini. Lihat Bagaimana cara menambahkan VBA di MS Office? untuk bahan ajar.

Sub Copy_Right()
    For Each rr In Selection.Rows
        For cn = Selection.Columns.Count To 1 Step -1
            If Len(rr.Cells(1, cn)) > 0 Then Exit For
        Next cn
        ' cn is now the (relative) column number of the last cell in this row
        ' that contains (non-blank) data.
        my_offset = Selection.Columns.Count - cn
        ' my_offset is how many columns to the right we need to move.
        ' If my_offset = 0, the row is full of data (or, at least,
        ' the last column contains data; there may be blank cells
        ' to its left), so there’s nowhere to move it.
        ' If cn = 0, the row is empty, so there’s nothing to move.
        If cn = 0 Or my_offset = 0 Then
            ' Nothing to do.
        Else
            For cn = Selection.Columns.Count To 1 Step -1
                If cn > my_offset Then
                    ' Copy data to the right.
                    rr.Cells(1, cn) = rr.Cells(1, cn - my_offset)
                Else
                    ' Set the cells on the left to blank.
                    rr.Cells(1, cn) = ""
                End If
            Next cn
        End If
    Next rr
End Sub

Ini akan menangani sel kosong tertanam (misalnya, the;quick;;fox;) dengan benar. Jika tidak, perbedaan antara jawaban ini dan yang lainnya hanyalah preferensi pribadi yang sewenang-wenang, dan yang lainnya mungkin lebih unggul dalam hal-hal yang saya tidak mengerti.
