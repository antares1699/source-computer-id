Apakah OpenSSL versi 0.9.8 saya terpengaruh oleh heartbleed?

Saya membaca buku masak OpenSSL, dan di sini yang saya lihat di sana

openssl version
OpenSSL 1.0.1 14 Mar 2012

Dan ini versi mu Ubuntu 14.04 server

openssl version
OpenSSL 0.9.8w 23 Apr 2012

Bukankah Maret lebih awal dari April?

Inilah yang saya temukan di situs web heart bleed:

Versi OpenSSL apa yang terpengaruh?

Status dari berbagai versi:

OpenSSL 1.0.1 through 1.0.1f (inclusive) are vulnerable
OpenSSL 1.0.1g is NOT vulnerable
OpenSSL 1.0.0 branch is NOT vulnerable
OpenSSL 0.9.8 branch is NOT vulnerable

Bug diperkenalkan ke OpenSSL pada Desember 2011 dan telah keluar sejak OpenSSL rilis 1.0.1 pada 14 Maret 2012. OpenSSL 1.0.1g dirilis pada 7 April 2014 memperbaiki bug.

Jika menggunakan paket Ubuntu openssl version menunjukkan versi dan tanggal dibekukan saat rilis; versi tambalan hanya ditampilkan dengan nama paket di dpkg atau apt-cache. CVE-2014-0160 (penasihat OpenSSL) 7 April 2014: Batas yang hilang periksa penanganan ekstensi detak jantung TLS dapat digunakan untuk mengungkapkan hingga 64kB memori ke klien atau server yang terhubung (a.k.a. Hati). Masalah ini tidak memengaruhi versi OpenSSL sebelumnya 1.0.1 . Dilaporkan oleh Neel Mehta. Diperbaiki di OpenSSL 1.0.1g (Terkena 1.0.1f, 1.0.1e, 1.0.1d, 1.0.1c, 1.0.1b, 1.0.1a, 1.0.1)
