Apa yang harus dilakukan administrator Gitlab untuk mengaktifkan Halaman Gitlab?

Halaman GitLab dinonaktifkan. Minta administrator sistem Anda untuk mengaktifkannya

Tidak ada apa-apa tentang ini dan saya benar-benar tidak tahu bagaimana caranya.

Dengan asumsi Anda menjalankan omnibus install. Sesuai dokumentasi dalam kasus yang paling sederhana, tanpa domain khusus atau SSL, untuk mengaktifkan halaman, yang harus Anda lakukan adalah, mengatur pages_external_url di file konfigurasi /etc/gitlab/gitlab.rb.

####################
# GitLab Pages
####################

## Define to enable GitLab Pages
pages_external_url "http://pages.foo.com/"

Sesuai komentar @Melroy, Anda mungkin juga perlu menyetelnya

gitlab_pages['enable'] = true

Satu hal lagi, karena informasi berikut tidak ditulis di mana pun tetapi tampak "jelas", Anda harus mengizinkan gitlab-runner membangun situs statis Anda setelah setelah menetapkan gitlab_pages['enable'] = true dan gitlab_pages['inplace_chroot'] = true saat menggunakan image buruh pelabuhan gitlab dan menjalankan gitlab-ctl reconfigure

Dalam kasus saya, saya menambahkan untuk memaksa Pekerjaan yang sebelumnya berhasil menggunakan tombol Retry untuk menerbitkan build statis. Segera setelah ini, menu Project -> Settings -> Pages diperbarui dengan tautan yang dapat digunakan.

  • gitlab_pages.py yang mencantumkan semua situs statis yang diaktifkan
