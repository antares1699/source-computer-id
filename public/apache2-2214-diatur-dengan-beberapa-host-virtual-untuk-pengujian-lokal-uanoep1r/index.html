Apache2 2.2.14 diatur dengan beberapa host virtual untuk pengujian lokal

Saya telah menyiapkan host virtual Apache 2 dengan metode ini.

di bawah /etc/apache2/sites-available saya telah membuat dummy_site_1.conf

dummy_site_1.conf

<VirtualHost local.dummy_site_1:80>
    ServerName local.dummy_site_1
    ServerAdmin webmaster@localhost
    DocumentRoot /var/www/dummy_site_1
    ErrorLog /var/log/apache2/dummy_site_1__error.log
    TransferLog /var/log/apache2/dummy_site_1__access.log
    <Directory /var/www/dummy_site_1>
        AllowOverride Options Limit None
        Options -Indexes FollowSymLinks
    Order deny,allow
    Allow from all
    </Directory>
    DirectoryIndex index.php
    #DISABLE HTTP TRACE
    #RewriteEngine On
    #RewriteCond %{REQUEST_METHOD} ^TRACE
    #RewriteRule .* - [F]
</VirtualHost>

Saya juga menambahkan baris ini ke httpd.conf:

ServerName local.dummy_site_1

Sekarang semua bekerja dengan pengaturan ini, saya mengetik local.dummy_site_1 di URL dan saya dapat melihat situs saya.

Masalahnya adalah ketika saya mencoba menambahkan konfigurasi situs lain dummy_site_2.conf dan membuat pengaturan yang sama

dummy_site_2.conf

<VirtualHost local.dummy_site_2:80>
    ServerName local.dummy_site_2
    ServerAdmin webmaster@localhost
    DocumentRoot /var/www/dummy_site_2
    ErrorLog /var/log/apache2/dummy_site_2__error.log
    TransferLog /var/log/apache2/dummy_site_2__access.log
    <Directory /var/www/dummy_site_2>
        AllowOverride Options Limit None
        Options -Indexes FollowSymLinks
    Order deny,allow
    Allow from all
    </Directory>
    DirectoryIndex index.php
    #DISABLE HTTP TRACE
    #RewriteEngine On
    #RewriteCond %{REQUEST_METHOD} ^TRACE
    #RewriteRule .* - [F]
</VirtualHost>

Saya juga menambahkan baris ini ke httpd.conf:

ServerName local.dummy_site_2

Saya mendapatkan kesalahan ini:

user@host:/etc/apache2$ sudo /etc/init.d/apache2 restart
 * Restarting web server apache2                                                                                                                                                                     [Wed Sep 15 14:00:43 2010] [error] (EAI 5)No address associated with hostname: Could not resolve host name local.dummy_site_2 -- ignoring!
 ... waiting [Wed Sep 15 14:00:44 2010] [error] (EAI 5)No address associated with hostname: Could not resolve host name local.dummy_site_2 -- ignoring!

Mengapa? apakah saya salah mengonfigurasi ini?

Anda tidak perlu 'ServerName local.dummy_site_2' di httpd.conf jadi keluarkan, tetapi tambahkan ini sebagai gantinya:

NameVirtualHost ip-address-of-server:80

misalnya: NameVirtualHost 192.168.1.10:80

Setelah Anda melakukan perubahan, Anda perlu memulai ulang Apache.

Jika semuanya masih tidak berfungsi seperti yang diharapkan, tambahkan baris ke setiap file conf virtualhost Anda di bawah baris nama server sehingga terlihat seperti ini:

ServerName local.dummy_site_1
ServerAlias *.local.dummy_site_1 local.dummy_site_1

dan

ServerName local.dummy_site_2
ServerAlias *.local.dummy_site_2 local.dummy_site_2

Sekali lagi, restart Apache untuk memeriksa perubahannya.

EDIT: Saya konyol - baru saja melihat sedikit masalah lainnya - SEMUA deklarasi VirtualHost Anda harus menggunakan alamat IP dari server virtual - misalnya:

<VirtualHost 192.168.1.10:80>

Sebenarnya, Anda bisa menghapus alamat IP dan atau nama situs sekaligus.

contoh:

NameVirtualHost *:80
NameVirtualHost *:443

<VirtualHost *:80>
ServerAdmin emailaddress@domain.tld
DocumentRoot "/path/to/vhost/documents"
ServerName servername.domain.tld
ServerAlias serveralias.domain.tld
ErrorLog "/path/to/logfiles/servername.domain.tld-error.log"
CustomLog "/path/to/logfiles/servername.domain.tld-access.log" common
<Directory "/path/to/vhost/documents">
allow from all
Option +Indexes
</Directory>
</VirtualHost>

<VirtualHost *:443>
ServerAdmin emailaddress@domain.tld
DocumentRoot "/path/to/vhost/documents"
ServerName servername.domain.tld
ServerAlias serveralias.domain.tld
SSLEngine On
SSLCertificateFile /path/to/certificate/file.cert
SSLCertificateKeyFile /path/to/certificate/file.key
ErrorLog "/path/to/logfiles/servername.domain.tld-error.log"
CustomLog "/path/to/logfiles/servername.domain.tld-access.log" common
<Directory "/path/to/vhost/documents">
allow from all
Option +Indexes
</Directory>
</VirtualHost>

(Sekarang, mari kita lakukan contoh vhost yang digabungkan dengan mod_proxy; ip = numeric-octet, di bawah)

<VirtualHost *:80>
ProxyPreserveHost Off
ProxyPass / http://ip.ip.ip.ip/
ProxyPassReverse / http://ip.ip.ip.ip/
ServerName servername.domain.tld
</VirtualHost>

Pada dasarnya, selama info DNS Anda cocok dengan vhost yang Anda konfigurasikan, maka semuanya harus berfungsi.
