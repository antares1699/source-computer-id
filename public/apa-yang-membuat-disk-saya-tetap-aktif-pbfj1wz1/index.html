Apa yang membuat disk saya tetap aktif?

Saya telah mengonfigurasi hard drive saya untuk tidur setelah 1 menit tidak ada aktivitas. Ini dulu bekerja dengan sangat baik; server hampir selalu sunyi senyap. Namun belakangan ini drive-nya lebih sering terjaga.

Dapatkah saya memeriksa apa yang membuat mereka tidak tidur? Ini adalah server rumah.

rc.local:

hdparm -M 128 -S 60 /dev/sda
hdparm -S 60 /dev/sdb

Memperbarui: Server menjalankan Ubuntu 10.04. OS, dengan / tmp dan / var /, menggunakan SSD, sedangkan sda dan sdb hanya digunakan untuk penyimpanan / backup.

Perbarui 2: Dari sysctl.conf saya:

vm.swappiness = 1
vm.vfs_cache_pressure = 50
vm.dirty_writeback_centisecs = 1500
vm.dirty_ratio = 20
vm.dirty_backgrounds_ratio = 10

3 Jawaban

3
3

Ini sebenarnya bukan jawaban atas pertanyaan yang Anda ajukan, tetapi dapat membantu mengatasi masalah yang mendasarinya. Satu alat yang saya ketahui adalah iotop, tetapi mungkin tidak membantu menangkap aktivitas singkat dari suatu proses.

Membuat hard drive tidur saat tidak digunakan bisa jadi sulit, karena ada begitu banyak sumber aktivitas yang potensial. Hanya karena Anda mengira sistem Anda tidak melakukan apa-apa, bukan berarti ada sesuatu yang tidak tertulis di latar belakang. Berikut beberapa penyebab umum:

  • Pembilasan jurnal. Linux membersihkan jurnal ext3 (atau ext4) secara berkala apakah perlu di-flush atau tidak. Ini dapat disetel dengan opsi pemasangan commit.
  • Log sistem. Di bawah konfigurasi default, beberapa log sistem dihapus setiap kali baris ditulis. Cron job adalah pelaku kebiasaan karena memicu entri log auth. Anda akan menginginkan - sebelum semua nama file log di /etc/syslog.conf.

Jika Anda benar-benar ingin disk Anda berputar, periksa noflushd . Ini benar-benar berfungsi (atau setidaknya dulu, itu belum banyak dipelihara akhir-akhir ini dan mungkin memiliki masalah dengan kernel kontemporer). Tetapi berhati-hatilah karena ini adalah peretasan, dan ini benar-benar berat: ini hanya menghentikan kernel dari menulis sampai cache penuh atau disk tetap aktif untuk pembacaan.

Jika Anda sedang mencari keheningan, teknik itu banyak membantu adalah untuk menangguhkan disk dari elastis alih-alih memasangnya secara langsung. Ini tidak akan membantu mengurangi konsumsi daya, tetapi kecuali jika Anda memiliki drive ≥10krpm (yang membutuhkan lebih banyak kipas), CPU dan motherboard mungkin merupakan penguras daya utama bahkan saat idle.
