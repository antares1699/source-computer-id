Pembaruan Debian grub2 menghapus opsi boot Windows

Sejak saya memperbarui grub ke grub 2, saya tidak lagi mendapatkan opsi untuk boot ke Windows (yang sayangnya terkadang diperlukan untuk plugin browser MSIE berpemilik yang perlu saya gunakan untuk bekerja).

Bagian /boot/grub/menu.lst yang relevan:

### END DEBIAN AUTOMAGIC KERNELS LIST

# This is a divider, added to separate the menu items below from the Debian
# ones.
title           Other operating systems:
root


# This entry automatically added by the Debian installer for a non-linux OS
# on /dev/hda1
title           Windows NT/2000/XP
root            (hd0,0)
savedefault
makeactive
chainloader     +1

Namun ini tidak muncul lagi. Saya memiliki beberapa entri di /boot/grub/grub.cfg dengan entri seperti ini:

menuentry 'Debian GNU/Linux, with Linux 2.6.32-5-amd64' --class debian --class gnu-linux --class gnu --class os {
        insmod part_msdos
        insmod ext2
        set root='(hd1,msdos1)'
        search --no-floppy --fs-uuid --set e638c434-4884-412f-a141-2c194f881fae
        echo    'Loading Linux 2.6.32-5-amd64 ...'
        linux   /boot/vmlinuz-2.6.32-5-amd64 root=UUID=e638c434-4884-412f-a141-2c194f881fae ro  quiet
        echo    'Loading initial ramdisk ...'
        initrd  /boot/initrd.img-2.6.32-5-amd64
}

Apakah saya harus mengubah file itu? Jika demikian, apa sintaks yang benar untuk boot Windows? Jika tidak, apa masalahnya?

Ini didasarkan pada Ubuntu tetapi harus valid untuk paket grub2 Debian juga:

/boot/grub/grub.cfg ditimpa setiap kali Anda menjalankan /usr/sbin/update-grub2 atau saat dipanggil setelah pembaruan kernel. grub2 seharusnya sudah otomatis mendeteksi instalasi windows melalui /etc/grub.d/30_os-prober dan menambahkannya ke menu boot. Entri kustom yang lebih baru harus dibuat pada /etc/grub.d/40_custom

Memperbarui:

Anda dapat mencoba menambahkan Windows Loader secara manual dengan menambahkan yang berikut ini ke /etc/grub.d/40_custom:

menuentry "Windows on /dev/hda1" {
insmod ntfs
set root='(hd0,1)'
search --no-floppy --fs-uuid --set YOURUUID
chainloader +1
}

"YOURUUID" adalah UUID dari partisi Windows Anda. Anda bisa mendapatkannya dengan melakukan ls -l /dev/disk/by-uuid/ | grep hda1. Setelah mengedit file, lakukan "sudo update-grub2" untuk menghasilkan grub.cfg baru.

Itu tergantung pada os-prober, yang mungkin tidak diinstal, dalam hal ini lakukan:

sudo apt-get install os-prober

... dan jalankan sudo update-grub2 lagi.
