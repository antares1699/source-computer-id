Manfaat & Kepraktisan Mesin Virtual [tutup]

Saya telah hidup di bawah batu selama 5 - 8 tahun terakhir, dan saya baru saja meningkatkan kecepatan dengan virtualisasi, mesin virtual, dan platform seperti VMware atau VirtualBox. saya berpikir Saya mulai memahaminya, tetapi ada dua konsep tentang VM yang membuat saya tersedak karena suatu alasan; yang satu berkaitan dengan manfaat intrinsiknya bagi perusahaan, dan yang lain berkaitan dengan kepraktisannya saat digunakan.

Jadi saya memahami bahwa Anda dapat memiliki beberapa VM yang berjalan pada mesin fisik yang sama, semua melakukan "hal" mereka sendiri seolah-olah mereka adalah mesin yang terpisah. Jadi katakanlah saya ingin membuat jaringan virtual server kecil yang bagus di kantor rumah saya sehingga saya dapat memiliki kotak pasir besar untuk dimainkan di dalam untuk semua proyek saya. Saya menyiapkan 2 mesin fisik sebagai server, dan pada masing-masing dari mereka saya memiliki, katakanlah, 4 mesin virtual yang berjalan (1 mungkin server aplikasi, yang lain mungkin RDBMS, yang lain mungkin menjadi perantara pesan, dll.).

  1. Selain menghemat uang saya (karena saya hanya perlu menyediakan 2 mesin fisik daripada membeli 8 server), manfaat / tujuan instrinsik apa yang disediakan VM ini di sini? Saya tidak peduli seberapa pintar mesin virtual, jika komputer saya hanya memiliki 8GB RAM, dan saya memiliki 4 VM yang berjalan di atasnya, VM tersebut hanya mendapatkan 8GB RAM untuk dibagikan di antara mereka. Menempatkan 4 VM di server saya tidak memberinya 4 kali lipat jumlah RAM! Jadi, sejauh yang saya lihat, VM menghemat uang Anda untuk perangkat keras, tetapi akhirnya menghambat perangkat keras tersebut karena Anda memiliki beberapa mesin yang bersaing untuk mendapatkan sumber daya yang sama. Apa yang kulewatkan di sini?

  2. Dapatkah VM berkomunikasi satu sama lain melalui jaringan Anda menggunakan protokol tingkat tinggi seperti TCP / IP, HTTP, FTP, dll.? Jika tidak, tidak masuk akal untuk menerapkan semua VM ini jika Anda harus memberi mereka potongan data / permintaan yang terpisah dan independen untuk diproses.

(1) Selain menghemat uang saya (karena saya hanya perlu menyediakan 2 mesin fisik daripada membeli 8 server), manfaat / tujuan instrinsik apa yang disediakan VM ini di sini?

Ya, mereka berbagi sumber daya. Mesin Virtual biasanya dialokasikan sejumlah memori tertentu saat startup. Sehingga 8GB dapat ditetapkan dalam potongan 2GB masing-masing di antara 3 mesin. (Dengan sisa 2GB terakhir sehingga OS host memiliki sesuatu untuk digunakan.)

Manfaat lainnya termasuk save state. Banyak jika tidak semua VMM memungkinkan Anda untuk mengkloning mesin virtual, bahkan saat "berjalan". Jadi, Anda dapat mengkloning status segera setelah boot dan jika macet, Anda dapat memulihkannya, alih-alih mem-boot dari sistem yang rusak. Ini juga dapat digunakan dalam beberapa kasus untuk menghemat waktu boot, karena mesin virtual dapat dipulihkan ke status berjalan.

Keuntungan lainnya adalah kemampuan untuk menukar VM yang sedang berjalan di antara mesin fisik, memungkinkan 100% uptime bahkan jika sistem host perlu dibuat offline karena alasan tertentu.

(2) Dapatkah VM berkomunikasi satu sama lain melalui jaringan Anda menggunakan protokol tingkat tinggi seperti TCP / IP, HTTP, FTP, dll.?

Ini tergantung pada VMM yang Anda gunakan, dan implementasi emulasi jaringannya yang khusus. Tapi secara umum, ya. VirtualBox misalnya mendukung emulasi kartu jaringan dalam banyak hal, termasuk pass-through atau NAT.

Salah satu manfaat VM adalah dapat menjalankan sistem operasi lama di perangkat keras terbaru. Sistem operasi yang lebih lama (atau versi yang lebih lama) mungkin tidak memiliki driver yang tersedia untuk perangkat keras saat ini. Virtualisasi menghadirkan OS tamu dengan NIC tervirtualisasi, pengontrol disk, dll. Yang didukung oleh tamu, terlepas dari perangkat keras sebenarnya.

Cara lainnya adalah dapat menguji sistem operasi baru atau rilis aplikasi secara terpisah dari siaran langsung tanpa harus membeli perangkat keras pementasan dan penerapan sebanyak mungkin.

Banyak manfaat lain yang bermuara pada fleksibilitas. Perangkat keras virtual dapat diubah ukurannya sesuka hati agar sesuai dengan tugas tertentu sambil menjaga isolasi dari aplikasi lain yang sebelumnya Anda dapatkan dengan memiliki perangkat keras terpisah - jika server database web Anda mogok, itu tidak memengaruhi sistem akuntansi Anda.

Semua sistem VM yang saya tahu mendukung jaringan di sistem operasi yang dihosting, ada beberapa mode, tetapi misalnya, para tamu dapat dialokasikan alamat IP terpisah dalam kisaran yang sama dengan host dan tidak berbeda dengan komputer fisik yang terpisah.
