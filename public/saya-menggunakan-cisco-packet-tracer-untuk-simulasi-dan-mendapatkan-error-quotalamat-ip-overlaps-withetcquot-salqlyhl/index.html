Pengalamatan Anda memang salah :-(

Entri pertama benar, tetapi entri kedua salah secara fundamental. Awalan dan mask tidak masalah, tetapi jaringan dan jangkauan tidak. Anda tidak dapat memiliki jaringan .4 dengan awalan 27.

27 awalan mengatakan - 27 bit alamat paling signifikan menentukan jaringan. Sisa 32 bit diisi dengan 0. Jadi oktet terakhir harus: xxx00000. Ketika Anda memasukkan alamat ip kedua Anda, router menghitung alamat jaringan untuknya (membersihkan 5 bit terakhir) menjadi 192.168.1.0, yang sama dengan jaringan pertama Anda, sehingga terjadi kesalahan 'tumpang tindih'.
