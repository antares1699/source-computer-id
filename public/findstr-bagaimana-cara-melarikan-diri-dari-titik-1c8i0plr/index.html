findstr: Bagaimana cara melarikan diri dari titik?

Saya tidak mengerti mengapa findstr tidak berfungsi seperti yang saya inginkan.

Saya memiliki file berikut di direktori pengujian saya:

aaa.jpg
bbb.png
ccc.svg

aaa_s.jpg
bbb_s.png
ccc_s.svg

aaa_small.jpg
bbb_small.png
ccc_small.svg

Dan saya memiliki baris berikut untuk diteruskan langsung ke cmd.exe:

for /f "delims=" %f in ('dir /b /a:-d ^| findstr /ile "gif jpg png svg" ^| findstr /ie "_s.*"') do echo "%f"

Menurut saya, itu harus sesuai dengan file berikut:

aaa_s.jpg
bbb_s.png
ccc_s.svg

Namun, sebenarnya itu cocok

aaa_s.jpg
bbb_s.png
ccc_s.svg

aaa_small.jpg
bbb_small.png
ccc_small.svg

Ya, saya memahami bahwa masalahnya adalah titik tersebut berfungsi sebagai "karakter apa pun" alih-alih diperlakukan secara harfiah.

findstr /ie "_s.*"

Bot bagaimana saya bisa menghindarinya? Menggunakan garis miring terbalik, yaitu mengubahnya menjadi

findstr /ie "_s\.*"

tidak bekerja.

Bagaimana cara saya keluar dari wildcard . untuk mencocokkan karakter . literal?

Anda berada di jalur yang benar dengan menggunakan \, namun Anda memerlukan . tambahan untuk menemukan karakter yang tersisa.

findstr /ie "_s\..*"

Contoh (ketik dan shell cmd):

F:\test>dir /b /a:-d | findstr /ile "gif jpg png svg" | findstr /ie "_s.*"
aaa_s.jpg
bbb_s.jpg
ccc_s.jpg
aaa_small.jpg
bbb_small.jpg
ccc_small.jpg

F:\test>dir /b /a:-d | findstr /ile "gif jpg png svg" | findstr /ie "_s\..*"
aaa_s.jpg
bbb_s.jpg
ccc_s.jpg

Jadi file batch Anda harus berisi:

for /f "delims=" %f in ('dir /b /a:-d ^| findstr /ile "gif jpg png svg" ^| findstr /ie "_s\..*"') do echo "%f"

Untuk tugas khusus ini, Anda juga dapat keluar dari "." dalam findstr dengan tidak menggunakannya jika Anda mengganti:

dir /b /a:-d → where .:*_s.*

findstr /ie "_s\.*" → findstr /ei "gif jpg png svg"

where .:*_s.*|findstr /ei  "gif  jpg  png  svg"

for /f delims^= %f in ('where .:*_s.*^|findstr /ei  "gif  jpg  png  svg"')do echo "%~nxf"
