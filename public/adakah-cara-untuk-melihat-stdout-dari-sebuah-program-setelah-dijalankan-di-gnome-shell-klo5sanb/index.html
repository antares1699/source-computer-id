Adakah cara untuk melihat stdout dari sebuah program setelah dijalankan di gnome-shell?

Jadi, jika saya menjalankan program melalui menu di gnome-shell, apakah ada cara untuk melihat stdout dan stderr? Atau apakah ada semacam peretasan untuk mencapai fungsi ini?

Atau semuanya baru saja dikirim ke /dev/null?

Biasanya, skrip start-up gdm / sesi mengalihkan stderr & stdout ke:

~/.xsession-errors

atau

~/.cache/gdm/session.log

Dengan versi systemd dan gdm terbaru, semuanya dialihkan ke systemd journal, jadi salah satu cara untuk mendapatkan keluaran itu adalah:

journalctl -b _PID=$(pgrep gnome-session)

journalctl -f

di tab terminal yang selalu saya biarkan terbuka (dan terbuka secara otomatis saat boot) jadi saya memiliki umpan balik waktu nyata dari semua pencatatan dari systemd di komputer saya.
