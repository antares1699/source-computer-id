Debian dan Ubuntu sering tidak melacak revisi kecil yang benar-benar terbaru dari sebuah paket yang dikembangkan secara aktif kecuali ada perbaikan fitur yang signifikan atau pembaruan keamanan (dan bahkan dalam perubahan seperti itu mereka akan sering mendukung pembaruan untuk perubahan signifikan pada revisi mereka membawa daripada mengambil revisi yang sama sekali baru. Hal ini terutama berlaku untuk Debian / Stable dan Ubuntu / Dirilis secara umum dan Debian / Pengujian dan Ubuntu / Next mendekati waktu rilis (ketika mereka dalam keadaan "fitur berhenti") .

Untuk mendapatkan versi yang lebih baru, Anda memiliki beberapa opsi:

  1. Ambil risiko menggunakan rilis distribusi nanti (yaitu Debian / Pengujian atau Ubuntu / Berikutnya - Squeeze dan Natty masing-masing saat ini) meskipun masih secara resmi dalam pengujian. Ini umumnya tidak disarankan, terutama untuk lingkungan produksi.
  2. Gunakan Apt Pinning (lihat https://help.ubuntu.com/community/PinningHowto ) untuk hanya mengambil paket yang Anda inginkan dari revisi nanti sambil mempertahankan sistem Anda yang lain pada tingkat stabil / rilis saat ini. Ini tidak terlalu rawan masalah dan opsi 1. Anda mungkin masih perlu melakukan langkah kompilasi jika LibC telah diperbarui secara signifikan antara rilis, tetapi ini masih lebih mudah daripada menggunakan sumber hulu karena Anda mendapatkan versi tweak Debian / Ubuntu (terkadang mereka mengubah skrip init dan bagian utilitas terkait agar lebih sesuai dengan sisa tata letak sistem mereka dan alat standar yang dipilih) dan apt / aptitude dapat menurunkan sumber yang diperbarui untuk Anda ketika ada pembaruan di repositori.
  3. Kompilasi dari sumber hulu seperti yang disarankan klakson. Ini akan memberi Anda versi stabil (atau beta / alpha) terbaru dan terhebat, tetapi lebih banyak upaya di awal dan tanggung jawab akan ada pada Anda untuk memantau proyek untuk pembaruan keamanan dan alasan lain yang perlu dikompilasi ulang.

Ubuntu / Natty saat ini mengusung 1.7.2.3-2 (yang mungkin berarti "1.7.2.3 dengan setidaknya dua pembaruan porta belakang dari revisi selanjutnya"), sama seperti Debian / Queeze.

Anda juga dapat mencoba versi dari Debian / Sid (meskipun saat ini versi yang sama) tetapi ini tidak disarankan. "Tidak stabil" diberikan nama itu karena suatu alasan - paket dapat rusak pada waktu tertentu karena ada secara khusus untuk menemukan kerusakan yang signifikan sebelum paket dipromosikan ke Pengujian.

Cara yang mungkin berhasil adalah mengunduh dari ke hulu dan berjalan seperti biasa

$ autoreconf
$ ./configure --prefix=/PATH/WHERE/YOU/PUT/YOUR/STUFF
$ make install

di dalam direktori sumber yang telah dibongkar.

EDIT

Karena dalam pengeditan Anda sekarang secara eksplisit menulis bahwa Anda tidak ingin membangun dari sumber dan ingin 1.7.3 diberi tag pada 2010/10/21 segala sesuatunya terlihat berbeda.

Paket git Ubuntu sepertinya datang langsung dari Debian dan Debian baru saja melakukan migrasi 1.7.2.3-2 ke TESTING seminggu yang lalu. Anda mungkin beruntung dengan meminta versi benjolan di pelacak bug Debian, dan bisa langsung menggunakan paket itu di Ubuntu saat itu.

hingga /etc/apt/sources.list saya dapat menginstal Git 1.7.2.3 menggunakan Synaptic.
