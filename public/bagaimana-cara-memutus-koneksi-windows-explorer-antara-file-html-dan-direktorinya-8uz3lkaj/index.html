Bagaimana cara memutus koneksi Windows Explorer antara file HTML dan direktorinya?

Saya benar-benar terkejut dengan fitur Windows yang ramah pengguna ini, di mana saya menyimpan halaman web (di Firefox), menghasilkan file HTML dan direktori untuk CSS, JS, PNG dan materi lainnya, dan di mana file dan direktori itu berada. direkatkan secara efektif.

Jika saya menyalin file ke lokasi lain, direktori juga akan disalin.

Jika saya menghapus direktori, file tersebut juga terhapus.

Jika saya mencoba mengganti nama direktori, sebuah peringatan akan muncul yang mengatakan: Jika Anda mengganti nama folder ini, file HTML terkait mungkin tidak berfungsi dengan benar . Ini tentu saja berbeda dengan asosiasi antara ekstensi file dan aplikasi.

Jika saya hanya tertarik dengan file HTML, tidak ada cara untuk menyalin file itu saja, atau hanya menghapus direktori. Tentu saja saya dapat memulai kembali dan mengunduh halaman dengan "html saja", tetapi maksud saya adalah:

  • Apa yang sedang terjadi? Bagaimana cara kerjanya? Apakah Pengalihan Folder ini?

  • Bagaimana cara membuat atau menghancurkan file asosiasi dua objek sistem file seperti file dan direktori?

Saya mencoba ini juga dengan Command Prompt DOS, Altap Salamander, dan dengan Linux. Asosiasi tersebut tampaknya merupakan fitur Windows Explorer. Tapi saya lebih suka bekerja dengan File Explorer (sementara di Windows).

OK saya baru saja menemukan jawabannya. Ini adalah fitur Windows 2000 khusus untuk menyimpan halaman HTM (L), menurut posting 2001 ini:

JSI Tip 3841. Apakah yang dimaksud dengan fitur File Terhubung Windows 2000?

Ia juga mengatakan bahwa itu dapat dimatikan dengan menambahkan kunci registri REG DWORD ini dengan nilai 1:

HKEY_CURRENT_USER \ Software \ Microsoft \ Windows \ CurrentVersion \ Explorer \ NoFileFolderConnection

Dari sini dapat disimpulkan bahwa Anda juga dapat memutuskan koneksi untuk halaman HTML tertentu dengan mengganti nama EKSTENSI halaman html, mis. ke foo.htmlXXX. Saya menguji ini, dan Anda harus mengklik OK dua kali: untuk mengonfirmasi bahwa Anda mengubah ekstensi, dan untuk mengonfirmasi bahwa Anda memutuskan koneksi dengan direktori HTML terkait, yang persis seperti yang saya inginkan. Kemudian cukup hapus direktori itu, dan ubah ekstensinya kembali ke foo.html.
