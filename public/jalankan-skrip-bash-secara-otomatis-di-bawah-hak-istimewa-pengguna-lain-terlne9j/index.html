Begitu pula saya mendapatkan hak itu.

Pengguna "bla" ingin menjalankan perintah sebagai Pengguna "blub"?

Kemudian sesuatu seperti ini bisa berhasil:

sudo -u blub /path/to/command

Jika Anda ingin ini dilakukan tanpa diminta kata sandi, tambahkan baris berikut ke / etc / sudoers (melalui perintah visudo):

bla    ALL=(blub) NOPASSWD: /path/to/command

Anda dapat bekerja dengan izin bit SUID dan SGID ( perhatikan bahwa ini mungkin sangat berbahaya )

berikan izin untuk mengeksekusi file ini ke grup tempat pengguna yang Anda jalankan skripnya, dan atur melalui SUID siapa saja yang menjalankan skrip ini untuk menjalankannya sebagai pengguna yang memiliki skrip

sudo chmod 4751 /path/to/my/file
sudo chgrp somegroup /path/to/my/file

Kemudian, Anda hanya perlu menjalankan skrip sebagai penggunaB siapa anggota somegroup dan itu akan dijalankan dengan as penggunaA dan somegroup

ls -l /path/to/my/file

apache   ALL=(nagios) NOPASSWD: /usr/local/nagios/libexec/pangoo_lib/change_state_simulation
