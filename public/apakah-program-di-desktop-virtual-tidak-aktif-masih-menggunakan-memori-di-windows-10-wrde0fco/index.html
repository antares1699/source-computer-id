Apakah program di Desktop Virtual Tidak Aktif masih menggunakan memori di Windows 10?

Sungguh, saya bertanya-tanya apakah ada gunanya beberapa desktop, di luar organisasi pribadi pengguna, Seperti apakah ada beberapa jenis peningkatan kinerja atau sesuatu?

Secara khusus, Jika saya memiliki Chrome dengan 10 tab terbuka, atau dalam kasus saya Visual Studio 20XX, secara terpisah di 3 dari desktop virtual bagaimana dosis ini mempengaruhi memori sistem (RAM), apakah jendela di desktop virtual yang tidak aktif diminimalkan secara efektif atau lebih seperti mereka di-hibernasi atau ditangguhkan (singkatnya diambil dari memori)?

Contoh:

  • Virtual Desktop 1 (Proyek utama yang sedang saya kerjakan)
    • Chrome dengan 10 tab
    • Visual Studio 20XX
    • berbagai utilitas kecil sedang berjalan
  • Virtual Desktop 2 (proyek saya menunggu tanggapan dari klien)
    • Chrome dengan 10 tab (terkait dengan proyek)
    • Visual Studio 20XX
  • Virtual Desktop 3 (barang internal perusahaan)
    • Chrome dengan 3 tab (terkait dengan proyek)
    • program pengatur waktu perusahaan

Akankah sistem bertindak seolah-olah saya memiliki 23 tab, 2 contoh Visual Studio 20XX dan program kecil lainnya (dan secara umum menjadi lamban karena saya harus banyak membuka) atau akankah itu bertindak seolah-olah satu-satunya hal yang terbuka adalah yang ada di Virtual Desktop yang aktif (jika Virtual Desktop 1 aktif, apakah program di Virtual Desktops 2 dan 3 masih berjalan juga)?

Secara pribadi saya tidak melihat poin ke Virtual Desktop jika itu tidak menghemat memori saya atau memungkinkan mesin saya berjalan lebih baik. Secara keseluruhan, ini sepertinya cara yang lebih rumit untuk mengelompokkan berbagai hal di bilah tugas.

Tidak ada manfaat memori untuk menggunakan desktop virtual, ini murni organisasi.

Microsoft hanya mengimplementasikan fitur yang ditawarkan oleh sistem operasi lain selama beberapa dekade. Saya menduga bahwa beberapa orang akan menggunakannya dengan berat tetapi sebagian besar akan mengabaikannya.

Satu-satunya tempat yang saya anggap berguna desktop virtual adalah pada hari-hari pengembangan Solaris saya hampir 20 tahun yang lalu ketika kami menjalankan Smalltalk.

Sebagian besar sistem pengembangan Smalltalk menggunakan beberapa jendela secara bersamaan membuat peralihan tugas ke tugas non-pengembangan mengganggu karena Anda akan terus-menerus mengganti semua jendela pengembangan Anda setelahnya.

Jika Anda menghabiskan sebagian besar waktu Anda hanya dalam beberapa jendela maka fitur tersebut tidak akan banyak menguntungkan Anda, tetapi jika Anda melakukan banyak multi-tasking yang tersebar di lusinan jendela terkait, itu adalah alat yang berguna.
