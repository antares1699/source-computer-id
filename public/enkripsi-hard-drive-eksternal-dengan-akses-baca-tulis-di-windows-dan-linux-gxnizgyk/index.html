Enkripsi hard drive eksternal dengan akses baca + tulis di Windows dan Linux

  • Saya pernah menggunakan NTFS + Bitlocker di masa lalu untuk hard drive eksternal USB, dan mudah digunakan dalam baca + tulis di Windows.

    Tetapi kurang mudah digunakan di Linux: NTFS baca + tulis membutuhkan ntfs-3g (yang menurut sebagian orang tidak 100% dapat diandalkan?) Dan Bitlocker di Linux memerlukan akses baca + tulis Dislocker (yang memiliki kurang dari 1000 bintang di Github sehingga ini mungkin bukan sistem yang paling aman / teruji yang tersedia; juga ada persyaratan untuk memiliki Dislocker dalam baca + tulis penuh yang saya tidak yakin akan memenuhinya).

  • Saya dapat menggunakan partisi ext4 terenkripsi pada hard drive eksternal USB, tetapi kemudian akan rumit untuk digunakan di Windows (mungkin tidak mungkin!).

Pertanyaan: apakah ada sistem enkripsi, yang didukung di luar kotak pada Windows + Ubuntu, yang memungkinkan hard drive USB eksternal dapat dibaca + ditulis dengan mudah di Windows dan Ubuntu?

Tentu saja, saya ingin hard drive eksternal USB tidak berguna bagi orang yang akan mencurinya, tanpa memiliki kuncinya.

Gunakan container VeraCrypt. Jadikan NTFS atau exFAT sistem file pada drive dan kemudian wadah dengan salah satunya juga.

Di samping wadah, Anda dapat meletakkan file teks dengan alamat Anda sehingga jika Anda kehilangan drive, seseorang yang menemukannya dapat mengembalikannya kepada Anda. Kedua, Anda dapat meletakkan binari Windows dan Linux VeraCrypt di sebelah wadah sehingga dengan memasang drive selalu Anda akan dapat membuka wadah.

Jadi ini di atas akan memenuhi kebutuhan Anda untuk membuka container di Windows dan Ubuntu.

Beberapa kemungkinan dijelaskan di bawah ini.

Enkripsi LUKS di Windows

Anda dapat menggunakan Linux LUKS enkripsi disk juga dari Windows. Untuk mengaktifkan Windows untuk bekerja dengan penggunaan LUKS LibreCrypt . Proyek ini masih dipertahankan, meskipun penulis memikirkannya penulisan ulang untuk menyelesaikan beberapa masalah keamanan.

Enkripsi Bitlocker di Linux

Proyek yang mengaktifkan Bitlocker non-TPM di Linux adalah cryptsetup , yang sekarang memiliki dukungan awal untuk Bitlocker. Baca lebih lanjut tentang itu di artikel Alat Enkripsi untuk Windows dan Linux .

CipherShed - garpu TrueCrypt lainnya

Proyek sumber terbuka CipherShed tersedia untuk Windows, Mac OS X dan GNU / Linux. Karena paket untuk OS X dan Linux belum ada, kompilasi dari sumber diperlukan untuk platform tersebut.

Cryptomator - alternatif cloud

Sumber terbuka Kriptomator adalah alat untuk mengenkripsi data yang disimpan di penyedia cloud, sebagai alternatif dari disk USB. Tersedia untuk Windows, macOS, Linux, Android dan iOS.

Di luar kotak, tidak.

Bitlocker adalah salah satu opsi, tetapi hanya tersedia di edisi Pro Windows dan dukungan Linux tidak terbukti dapat diandalkan, seperti yang Anda perhatikan. Menurut pengalaman saya, Dislocker berfungsi dengan baik, tetapi perlu diketahui bahwa rilis resmi terakhir tidak mendukung enkripsi Bitlocker yang lebih baru dan lebih kuat yang ditambahkan dalam versi Windows 10 terbaru (190x?). Dukungan telah ditambahkan pada cabang master, tetapi ternyata tidak dirilis.

Pertimbangkan Truecrypt atau Veracrypt. Mereka adalah solusi enkripsi pihak ketiga asli Windows (VC adalah garpu TC) dengan dukungan Linux yang baik: port Linux resmi tersedia, tetapi ada juga dukungan oleh cryptsetup yang merupakan alat enkripsi disk standar de facto Linux.
