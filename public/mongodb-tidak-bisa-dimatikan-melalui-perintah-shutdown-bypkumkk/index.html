Mongodb tidak bisa dimatikan melalui perintah --shutdown

saya butuh bantuan dengan masalah mematikan mongod dengan anggun. Saya menjalankan mongodb di buruh pelabuhan.

saya menjalankan mongod seperti dengan perintah:

mongod --config /etc/mongod.conf

dan ketika saya menutup dengan:

mongod --shutdown

saya mendapatkan pesan:

Sepertinya tidak ada server yang berjalan dengan dbpath: / data / db

pesannya benar, karena jalur db saya bukan "/ data / db".

dan jika saya mencoba mematikannya seperti:

mongod --config /etc/mongod.conf --shutdown

saya mendapatkan pesan:

proses pembunuhan dengan pid: 9

Apa masalahnya ?

Saya telah menemukan bahwa jika Anda tidak menggunakan default --dbpath, Anda harus menentukannya sebagai bagian dari shutdown. Misalnya, jika Anda memulai layanan mongod dengan /Mongo/Data sebagai dbpath, Anda harus mematikannya dengan:

mongod --dbpath /Mongo/Data --shutdown

Ini berfungsi bahkan jika Anda tidak menggunakan port default 27017.

Anda bisa menggunakan:

pskill mongod

Atau untuk beberapa distro * nix dan linux:

pkill mongod

Ketika mongod mendapatkan perintah SIGTERM (sinyal default dari perintah kill adalah SIGTERM), ia akan mematikan database dengan baik.

Anda juga dapat memeriksa PID dari proses mongod dengan:

ps -ef | grep mongod

Kemudian dimatikan oleh:

kill <PID of mongod>

Tapi JANGAN gunakan kill -9 untuk proses mongod.
