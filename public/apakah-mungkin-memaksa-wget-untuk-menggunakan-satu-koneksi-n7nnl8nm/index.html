Apakah mungkin memaksa wget untuk menggunakan satu koneksi?

Saya mencoba mengunduh sekitar 20 GB data dengan FTP dengan akun pribadi saya (scp tidak tersedia dan tar tidak dimungkinkan).

Saya menggunakan perintah standar berikut:

 wget -m ftp://user:pass@host.com/db/

Ini berfungsi beberapa waktu dan kemudian gagal saat mengautentikasi untuk unduhan.

Saat mencoba menyambung dengan klien FTP ke server, ia merespons dengan:

530 Maaf, jumlah klien maksimum (15) untuk pengguna ini sudah terhubung.

Jadi sepertinya wget membuka banyak koneksi. Apakah mungkin untuk menggunakan hanya satu koneksi ke server dengan wget?

Secara default, wget menggunakan pasif FTP, yaitu menggunakan satu koneksi kontrol dan membuka yang baru untuk setiap transfer file.

Di aktif FTP, server membuka koneksi yang diperlukan. Jika Anda dapat menerima koneksi masuk, itulah cara yang lebih disukai untuk mentransfer file melalui FTP.

wget hanya mengunduh satu file pada satu waktu, jadi sebaiknya hanya menggunakan dua koneksi. Sepertinya koneksi tidak ditutup dengan benar. Coba jika mengatur mode transfer ke "aktif" memperbaikinya.

Untuk mengaktifkan FTP aktif, gunakan sakelar --no-passive-ftp.
