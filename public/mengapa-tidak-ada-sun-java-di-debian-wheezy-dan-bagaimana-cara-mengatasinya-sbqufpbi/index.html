Saya tidak yakin itu karena alasan perizinan, karena Sun / Oracle Java ada di Squeeze dan Sid.

Jika Anda membutuhkan Sun / Oracle Java di Wheezy, cara termudah mungkin adalah menginstal versi Squeeze atau Sid. Anda dapat membuat apt melakukannya untuk Anda dengan menjepit .

Memasang pin memungkinkan Anda menjalankan paket tertentu dari satu versi (stabil, pengujian, tidak stabil) tanpa perlu mengupgrade seluruh sistem Anda.

Saya tidak melakukan itu saat ini, karena saya menemukan paket opendjk memadai untuk semua yang saya butuhkan, tetapi saya menggunakan penyematan di masa lalu untuk sejumlah paket lain.

Di http://wiki.debian.org/Java ada link ke helper untuk menghasilkan paket debian dari tarbal upstream.

Ini mungkin jawaban terbaik: membuat paket setiap kali orang-orang di Oracle membuat versi baru, tetapi saya lebih suka yang berbeda: memiliki paket dengan skrip untuk melakukan tugas yang membosankan dalam menginstal dari tar.gz.

Inilah sebabnya mengapa saya membuat paket yang tidak berisi kerangka kerja java, tetapi yang memberi tahu sistem justru sebaliknya.

Dengan cara ini saya dapat memutakhirkan hanya dengan mengunduh tar.gz secara manual dan menjalankan skrip.

Saya tahu bahwa paket dan skrip dapat ditingkatkan, tetapi bagi saya lebih dari cukup. Ia bekerja di i386 dan amd64.

Jika Anda ingin mencobanya, Anda dapat mendownloadnya dari http://babel.ls.fi.upm.es/~vpablos/others/packages/debian/

Saya tidak tahu cara yang benar untuk menginstal sun-java, tetapi ini adalah solusi saya (tidak terlalu bagus) untuk Wheezy.

Dalam contoh ini, versi Java saya adalah JDK 64 bit.

  1. Unduh jdk-6u30-linux-x64.bin dari www.oracle.com

  2. Ekstrak jdk-6u30-linux-x64.bin hingga /usr/lib/jvm

    cp jdk-6u30-linux-x64.bin /usr/lib/jvm
cd /usr/lib/jvm
chmod 777 jdk-6u30-linux-x64.bin
./jdk-6u30-linux-x64.bin
rm jdk-6u30-linux-x64.bin

  3. Buat tautan simbolis ke direktori baru /usr/lib/jvm/jdk1.6.0_30

    ln -s /usr/lib/jvm/jdk1.6.0_30 /usr/lib/jvm/java-sun

    (Ini akan menghemat waktu untuk pembaruan berikutnya. Hapus saja tautan /usr/lib/jvm/java-sun dan buat tautan simbolis baru.)

  4. Buat tautan simbolis untuk browser web:

    ln -s /usr/lib/jvm/java-sun/jre/lib/amd64/libnpjp2.so /usr/lib/mozilla/plugins/javaplugin.so

    Uji dengan browser web yang menunjuk ke www.java.com

    rm /usr/bin/java
ln -s /usr/lib/jvm/java-sun/bin/java /usr/bin/java
rm /usr/bin/javac
ln -s /usr/lib/jvm/java-sun/bin/javac /usr/bin/javac

    dan seterusnya..

Semoga Anda menemukan solusi yang lebih baik.
