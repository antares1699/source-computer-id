Bagaimana cara mengakses server yang berjalan di VirtualBox dari host?

Saya menjalankan aplikasi web di VM.

Saya dapat melihat aplikasi di localhost:8888 melalui browser dari dalam VM.

Saya tidak dapat melihat aplikasi dari host di URL yang sama. Itulah tujuannya.

Saya mencoba VBoxManage modifyvm "VM name" --natdnshostresolver1 on tetapi tidak ada pengaruh.

Konfigurasi saya adalah:

  • Tuan rumah: Windows 7
  • Tamu: Ubuntu 16.04.1 LTS
  • VirtualBox: 5.1.4r110228

URL localhost:8888 hanyalah URL yang valid di dalam VM Anda. Di luar VM itu, Anda harus menggunakan ip.ad.dr.ess:8888 atau hostname:8888. Dalam kasus terakhir, ketergantungan pada nama host yang menyelesaikan ke ip.ad.dr.ess di jaringan Anda diperkenalkan.

Secara default, saat Anda membuat VM di VirtualBox, adaptor jaringan 1 disetel ke NAT. Ini akan memberi VM alamat IP 10.0.2.15/24 dan rute default 10.0.2.2 (yang seharusnya menjadi host Anda). Singkat cerita, jendela tidak akan merutekan ke alamat yang disediakan NAT, tanpa melewati lebih banyak rintangan terlebih dahulu.

Mematikan VM dan mengubah setelan adaptor jaringan VM ke "dijembatani" akan memungkinkan host Anda (dan semua host di jaringan lokal Anda) untuk berkomunikasi dengan VM tersebut tanpa menambahkan rute khusus atau rintangan untuk dilewati.

Mulai ulang VM. Verifikasi konektivitas Anda ke localhost:8888. Juga verifikasi konektivitas Anda ke hostname:8888 (dengan asumsi nama host not ditambahkan ke alamat loopback / localhost di file / etc / hosts) dan ip-address:8888. EDIT: Berdasarkan detail / etc / hosts yang diposting, Anda are mengikat VM Anda dan karenanya webapp ke jaringan loopback. Konfigurasi ini sama sekali tidak dapat digunakan di luar VM.

Jika server web dan aplikasi Anda tidak bergantung pada IP tempat pertama kali dikonfigurasi / diinstal, dan Anda memiliki server DHCP di LAN Anda, satu-satunya informasi lain yang Anda perlukan untuk terhubung ke VM adalah alamat IP yang ditugaskan untuk itu melalui DHCP.

Untuk menemukan alamat IP VM: Buka jendela terminal / konsol dari desktop Ubuntu (ctrl-alt-T), kami menggunakan perintah nama host, ifconfig, dan grep /etc/hosts untuk menyatukan semua bagian dan menemukan alamat Anda. Dari terminal terbuka, ketik hostname lalu tekan enter. Ini mungkin gc. Di terminal yang sama, mengetik ifconfig eth0 | awk '/Bcast/{print $2}' akan menampilkan alamatnya. Anda harus memiliki baris di / etc / hosts dengan alamat yang diikuti dengan nama host. grep [hostname] /etc/hosts atau grep [address] /etc/hosts harus menemukan baris yang Anda butuhkan. Jika Anda menemukan apa pun selain alamat Anda saat ini yang diikuti dengan nama host, Anda perlu mengubah file / etc / hosts untuk menambahkan informasi yang benar.

Host di LAN Anda sekarang seharusnya dapat terhubung ke IP VM Anda di port 8888. Untuk host / klien tersebut untuk mengakses aplikasi web melalui hostname:8888 akan bergantung pada file host klien, DNS lokal, atau bahkan layanan mDNS yang mungkin disediakan di DHCP Anda server.

Untuk mencapai ini, saya pikir Anda akan jauh lebih baik menggunakan mode jembatan. Jika itu tidak mungkin atau tidak diinginkan, Anda memerlukan rute statis pada host Anda.

Untuk menambahkan rute statis, lakukan ini: Lihat di properti jaringan host Anda. Anda harus menemukan adaptor yang disebut Jaringan Hanya Host VirtualBox. Temukan alamat IP-nya. Ini mungkin 192.168.56.1, yang menurut saya adalah default, tetapi bisa jadi sesuatu yang lain. Sebut saja a.b.c.d. Selanjutnya, dapatkan alamat IP tamu Anda. Sebut saja w.x.y.z. (3 oktet pertama harus sama di kedua alamat.) Ketika Anda memilikinya, buka prompt DOS pada host Anda, dan ketik perintah ini:

route add w.x.y.z mask 255.255.255.0 a.b.c.d

Jika berhasil, Anda dapat menjalankan perintah yang sama dengan tombol -p untuk membuatnya permanen (jika tidak, perintah ini akan hilang saat Anda mem-boot ulang host).
