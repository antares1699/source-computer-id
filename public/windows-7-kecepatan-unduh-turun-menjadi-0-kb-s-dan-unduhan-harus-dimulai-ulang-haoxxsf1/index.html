Windows 7, kecepatan unduh turun menjadi 0 kb / s dan unduhan harus dimulai ulang

PC saya memiliki SSD sebagai C: dan drive sata untuk D:

SSDnya baru dan jadi saya telah menginstal ulang Windows 7 64-bit. Saya telah menginstal driver jaringan terbaru dari pabrikan motherboard, sebenarnya ini adalah driver yang sama yang saya gunakan sebelumnya. Sejak penginstalan, saya mengalami masalah aneh. Saya akan mulai mengunduh dan itu akan mencapai kecepatan maksimum untuk koneksi saya dan kemudian berhenti. Unduhan tidak gagal, itu hanya berhenti. Saya hanya melihat kecepatan jaringan menurun dan datar ketika saya melihat aktivitas jaringan. Ini terjadi secara acak selama pengunduhan. Saya harus memulai ulang proses pengunduhan dan kemudian (biasanya) akan berjalan dengan baik. Misalnya, jika saya memulai unduhan 700MB, itu akan berhenti di sekitar 40MB. Jika saya menjeda / membatalkan lalu memulai ulang unduhan ini, biasanya unduhan ini akan terus berlanjut.

Saya pikir itu mungkin drive SSD, tetapi saya mengatur unduhan untuk disimpan di D: dan masalah tetap ada. Ini mempengaruhi sejumlah besar lalu lintas jaringan - Pembaruan Windows, mengunduh melalui browser, menyalin file dari komputer lain, dll. Masalah dimulai sebelum dan setelah saya menginstal pembaruan, dan saya sudah mencoba mematikan firewall sampai tidak berpengaruh.

Saya tidak sepenuhnya yakin apa lagi yang bisa saya lakukan. Saya mencoba menginstal ulang lagi, tetapi tidak menyelesaikan masalah. Saya juga mencoba menukar kabel jaringan.

Sepertinya masalah dengan SSD ... dan kemungkinan besar masalah dengan ISP Anda. Saya mengalami masalah serupa dan menemukan bahwa itu adalah masalah dengan MTU yang dimiliki ISP saya. saya menulis kata-kata kasar tentang masalah ini beberapa waktu yang lalu ... tetapi berikut adalah langkah-langkah umum untuk mendiagnosis / menyelesaikan:

Pertama ... lihat pengaturan MTU Anda saat ini: Buka prompt perintah dengan hak administratif dan kemudian jalankan perintah sebagai berikut:

netsh interface ipv4 show subinterfaces

Anda akan melihat sesuatu seperti:

   MTU  MediaSenseState   Bytes In  Bytes Out  Interface
------  ---------------  ---------  ---------  -------------
  1500                1  8864896253  1116506364  Local Area Connection

Dengan pengetahuan itu, Anda akan mengetahui bahwa MTU Anda saat ini disetel ke 1500. Jika ISP Anda bekerja dengan itu ... Anda harus dapat melakukan ping ke situs seperti msn.com atau google.com dengan ukuran paket 1500 tanpa memecah paket.

ping www.google.com -f -l 1500

jika Anda melihat pesan kesalahan seperti: Paket perlu difragmentasi tetapi set DF.

Anda tahu Anda harus lebih kecil. Jadi, kurangi 8 & coba lagi ... sampai Anda menemukan MTU yang valid. Setelah Anda menemukan salah satu yang memberi Anda balasan tanpa mengeluh tentang perlunya dipecah-pecah ... inilah waktunya untuk mengubah MTU Anda. Kembali ke command prompt admin ... dan kami melakukan ini: (ganti 1464 dengan apa pun yang Anda buat)

netsh interface ipv4 set subinterface "Local Area Connection" mtu=1464 store=persistent

Jika Anda hanya ingin mengujinya ... tanpa melakukan perubahan ... lewati bit store=persistent ... dan reboot akan mengembalikannya seperti semula. Anda juga dapat mengaturnya kembali secara manual ke apa pun yang Anda mulai dengan ...

Saya tidak tahu apakah itu akan berhasil untuk semua orang ... tetapi "berhasil untuk saya" (c)

Saya mengalami perilaku aneh ini kembali dengan Windows Vista, dan baru-baru ini dengan Windows Server 2012 R2 dan Windows 8.

Ternyata fitur penyetelan otomatis global TCP yang diperkenalkan kembali di Windows Vista tidak selalu berfungsi baik karena kompatibel dengan lingkungan jaringan yang ada. Anda mungkin ingin mencoba menonaktifkan global auto-tuning di command prompt administrator.

netsh interface tcp set global autotuning=disabled
