Vim: tidak dapat memetakan ulang <leader> menjadi koma atau spasi

Saya sudah mencoba

let mapleader = '<space>'
let mapleader = ' '
let mapleader = ','

Tak satu pun dari mereka berhasil. VIM bahkan tidak menunjukkan koma di bilah status (bagian bawah layar?), Tetapi berfungsi dengan sempurna dalam mode penyisipan. Spasi hanya maju satu karakter.

Mengapa ini bisa terjadi?

4
joaquinlpereyra — Sumber
  • Di file mana Anda menulis pengaturan ini? Hanya untuk memastikan, coba let mapleader="," dengan tanda kutip ganda, bukan tunggal.
    — ap0
  • mapleader adalah sesuatu yang tidak dapat Anda ubah secara dinamis, yaitu jika Anda mengubah mapleader, ia tidak kembali dan mengubah semua pemetaan yang sudah ditentukan. Jadi, mengulangi pertanyaan, file apa yang Anda setel ini? Apakah sebelum atau sesudah pemetaan Anda? Kamu adalah mengaturnya dalam sebuah file, benar?
    — Ben
  • Oke, teman-teman, maaf tidak menjawab, terjebak dalam sesuatu selama beberapa hari. Saya malu untuk mengatakan bahwa saya TIDAK mengatur ini dalam sebuah file. Apakah itu ada hubungannya? Saya baru saja menekan ":" dan kemudian pengaturannya. Pemahaman saya yang seharusnya bekerja sebaik memasukkan ~ / .vimrc untuk sesi ini, setidaknya.
    — joaquinlpereyra
  • Tidak, menyetel mapleader ke nilai hanya memengaruhi pemetaan yang ditentukan setelah Anda menyetel mapleader. Jadi, jika Anda membuat pemetaan secara manual setelah ini, pemetaan itu akan berfungsi seperti yang Anda harapkan, tetapi tidak ada pemetaan sebelumnya (mis. Dari .vimrc Anda) yang ditafsirkan ulang.
    — Ben

Anda perlu menyetel mapleader di ~/.vimrc Anda, sebelum pemetaan apa pun ditentukan atau plugin dimuat. Dari :help mapleader:

Perhatikan bahwa nilai "mapleader" digunakan pada saat pemetaan dilakukan ditentukan. Mengubah "mapleader" setelah itu tidak akan berpengaruh pada yang telah ditentukan pemetaan.
