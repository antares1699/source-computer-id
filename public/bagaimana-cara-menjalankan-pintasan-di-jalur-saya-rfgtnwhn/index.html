Bagaimana cara menjalankan pintasan di jalur saya

Saya telah menambahkan pintasan ke file yang dapat dieksekusi ke dalam folder di jalur saya, tetapi saya tidak dapat menjalankannya seperti yang saya harapkan.

Apakah ada cara untuk melakukan ini? Saya punya banyak exe kecil yang saya ingin tetap teratur dalam folder instalasi mereka.

Saya ingin mengeksekusi exes dari dan folder ini. Jadi saya membutuhkan mereka di jalan saya.

Saya tidak ingin menambahkan 10 direktori lagi ke jalur saya.

Adakah cara untuk mencapai ini?

Salah satu cara untuk melakukannya adalah buat Symbolic Links ke file (program) yang dimaksud. Misalkan Anda memiliki satu folder (C:\bin) yang Anda tambahkan ke jalur sistem Anda. Lalu di folder itu Anda dapat membuat Symbolic Links ke file executable yang ingin Anda akses dengan cepat.

Sekarang, anggaplah ada beberapa aplikasi yang ingin Anda akses cepat, baik dari baris perintah atau dari kotak Windows / Run. Misalnya, daftar program Anda mencakup:

  "C:\Program Files\7-Zip\7z.exe"
  "C:\Program Files\AutoHotkey\AutoHotkey.exe"
  "C:\Program Files\iTunes\iTunes.exe"
  "C:\Program Files (x86)\Microsoft Visual Studio 11.0\Common7\IDE\devenv.exe"
  "C:\Program Files (x86)\Notepad++\notepad++.exe"

Untuk membuat tautan simbolik di atas, Anda dapat:

  mkdir c:\bin
  cd /d c:\bin

  mklink 7z.exe "C:\Program Files\7-Zip\7z.exe"
  mklink ahk.exe "C:\Program Files\AutoHotkey\AutoHotkey.exe"
  mklink iTunes.exe "C:\Program Files\iTunes\iTunes.exe"
  mklink devenv.exe "C:\Program Files (x86)\Microsoft Visual Studio 11.0\Common7\IDE\devenv.exe"
  mklink npp.exe "C:\Program Files (x86)\Notepad++\notepad++.exe"

Anda akan melihat bahwa saya telah "mengganti nama" AutoHotKey.exe menjadi ahk.exe dan Notepad++.exe menjadi npp.exe dalam contoh ini, sehingga memberikan nama singkatan untuk mereka.

Seperti banyak hal di Windows, ada cara lain yang benar-benar valid untuk mencapai hal yang sama. Satu akan dibuat Windows Alias ​​Baris Perintah . Yang lain akan digunakan AutoHotKey . Yang lain lagi adalah membuat Pintasan Windows dan tambahkan ekstensi .lnk ke variabel lingkungan PATHEXT sebagai @Pengen N menjelaskan dalam jawabannya. Atau, Anda bahkan bisa menggunakan doskey.exe untuk mencapainya, seperti yang dijelaskan dalam ini menempatkan pada StackOverflow .

Secara pribadi, saya terutama menggunakan alias karena saya cenderung sering menggunakan Windows Command Line. Saya juga menggunakan AutoHotKey, bagaimanapun, ketika situasi membutuhkannya.

Anda perlu memperbarui variabel lingkungan sistem PATHEXT. Variabel tersebut mencantumkan jenis file yang dianggap dapat dieksekusi untuk tujuan pencarian jalur. Tambahkan ;.LNK ke nilai variabel itu, dengan demikian membuat pintasan (.lnk) menjadi sesuatu yang akan dicari di PATH Anda. Perhatikan bahwa Anda mungkin perlu memulai ulang perintah perintah yang berjalan agar perubahan diterapkan.
