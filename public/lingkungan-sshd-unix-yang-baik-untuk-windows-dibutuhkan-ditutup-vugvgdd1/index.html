Lingkungan sshd + unix yang baik untuk Windows dibutuhkan [ditutup]

Pekerjaan saya adalah 95% administrasi Unix, tetapi terkadang saya mengelola mesin Windows dari jarak jauh. Meskipun saya cukup akrab dengan solusi VNC dan RDP, ada kalanya saya menginginkan sesuatu yang sedikit lebih ringan karena alasan bandwidth, serta memiliki lingkungan skrip dan set alat yang lebih familiar.

Saya tahu ada beberapa lapisan Unix untuk platform Windows mis. cygwin, uwin, WSU, dan ada beberapa server sshd. Namun, saya tidak yakin mana yang memberikan pengalaman konsol yang paling "mirip-unix", dalam hal shell dan utilitas standar, mis. ps, atas, tcpdump. Apakah lingkungan ini memungkinkan seseorang untuk menjalankan utilitas baris perintah saham normal seperti net, ipconfig, fsutil juga?

Semua pengalaman, saran, dan petunjuk diterima.

Untuk pengalaman konsol yang paling mirip Unix, saya pasti akan menggunakan OpenSSH dengan Cygwin . Dengan cara ini, semua perintah Windows Anda tersedia, serta perintah Unix Anda.

Menginstal OpenSSH dengan Cygwin sangatlah mudah. Cukup centang kotak OpenSSH di bawah bagian Net saat memilih paket:

Setelah penyiapan, jalankan ssh-host-config di Cygwin. Ini akan menanyakan berbagai pertanyaan:

  • pemisahan hak istimewa - jawaban Iya
  • buat pengguna sshd lokal - jawaban Iya
  • instal sshd sebagai layanan - jawaban Iya
  • Cygwin tty Anda (CYGWIN =) - jawaban ntsec tty

Sekarang buka c:\cygwin\etc\defaults\etc\sshd_config.

Ubah #Protocol 2,1 menjadi Protocol 2 untuk alasan keamanan.

Sekarang Anda dapat memulai layanan sshd. Pergi ke Panel Kontrol -> Alat Administratif -> Layanan dan mulai Cygwin sshd:

Anda ingin mengatur layanan untuk memulai secara otomatis sehingga terus berlanjut selama reboot.
