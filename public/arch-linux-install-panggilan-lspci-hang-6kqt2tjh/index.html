arch linux install: panggilan lspci hang

Saya mencoba untuk dual-boot Arch Linux (archlinux-2018.02.01-x86_64.iso & archlinux-2018.03.01-x86_64.iso) pada mesin ( ASUS B071ZL3996 ) yang memiliki Windows prainstal. Saya telah berjuang melawan kurangnya koneksi jaringan selama penginstalan (saya dapat terhubung melalui koneksi kabel dan nirkabel di sisi Windows). Saat memecahkan masalah, saya mencoba untuk memeriksa kartu jaringan dengan lspci. Setiap kali saya menyebutnya, bagaimanapun, itu hang instalasi. Secara khusus, ini mengembalikan kesalahan berikut:

xhci host controller not responding, assume dead

Saya tidak begitu yakin apa yang membuat ini, karena tampaknya terkait dengan kesalahan USB (berdasarkan apa yang telah saya lihat), tetapi ini hanya terjadi untuk panggilan lspci. Adakah yang pernah menemukan ini sebelumnya?

Coba daftar hitam nouveau sebelum booting; itu adalah kematian dari semua penginstalan pada laptop yang menggunakan pengaturan NVIDIA Optimus, yang mungkin terlihat seperti ini.

Tekan e sebelum memilih Arch dari menu boot ISO dan tambahkan modprobe.blacklist=nouveau ke parameter kernel.
