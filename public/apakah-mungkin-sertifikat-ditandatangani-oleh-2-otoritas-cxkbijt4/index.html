Apakah mungkin sertifikat ditandatangani oleh 2 otoritas?

Untuk menjelaskan situasinya sedikit:

Saya sedang membangun aplikasi iOS yang menggunakan Penyematan SSL . Saya telah membuat otoritas sertifikat yang ditandatangani sendiri yang menerbitkan sertifikat SSL ke server web saya, dan sertifikat CA dibundel dengan aplikasi untuk verifikasi. Saya ingin menggunakan letsencrypt untuk membuat sertifikat SSL untuk server web sehingga dipercaya secara implisit oleh browser web, tetapi sertifikat mereka tidak akan ditandatangani oleh CA saya sehingga ini tidak akan berfungsi dalam aplikasi. (Perlu dicatat bahwa sertifikat yang dikeluarkan oleh letsencrypt berumur sangat pendek, jadi mereka tidak dapat digunakan secara langsung untuk penyematan SSL).

Jadi saya ingin membuat sertifikat menggunakan letsencrypt dan kemudian menandatanganinya dengan CA saya. Apakah ini mungkin?

Sertifikat hanya boleh berisi satu tanda tangan. Namun, karena Anda tetap menggunakan penyematan SSL, Anda tidak perlu memiliki CA Anda sendiri, karena di dalam aplikasi iOS Anda cukup memeriksa sidik jari kunci publik. Selama Anda menggunakan pasangan kunci yang sama saat memperbarui sertifikat dengan letsencrypt, sidik jari kunci publik sepenuhnya mengidentifikasi sertifikat juga setelah memperbarui.

Apakah mungkin sertifikat ditandatangani oleh 2 otoritas?

Tidak. Hanya ada ruang untuk satu penerbit, satu pengenal kunci otoritas, dll.

Lihat juga Sertifikat dengan Banyak Penandatangan? di milis PKIX. PKIX adalah PKI Internet yang disebut oleh IETF. PKI lain mungkin berbeda.

Jika Anda menemukan PKI lain yang memungkinkannya, maka kemungkinan tidak akan berfungsi / interop dengan browser dan agen pengguna lain seperti curl , wget , dll. Mereka tidak akan tahu cara menangani sertifikat.
