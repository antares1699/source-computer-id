Tidak dapat terhubung ke server mysql biru di mesin virtual

Saya memiliki mesin virtual Azure yang memegang sistem Ubuntu, dengan server MySQL dan saya ingin mengakses database dari aplikasi web. Ketika saya menginstal mysql-server, saya mengeksekusi

$ iptables -I INPUT -p tcp --dport 3306 -m state --state NEW,ESTABLISHED -j ACCEPT
$ iptables -I OUTPUT -p tcp --sport 3306 -m state --state ESTABLISHED -j ACCEPT

untuk mengizinkan koneksi tcp. Di file /etc/mysql/my.cnf saya menambahkan bind-address = 168.63.178.87, yang merupakan "Alamat IP Virtual" yang diberitahukan panel Azure kepada saya.

Di mysql, pengguna yang saya gunakan untuk terhubung memiliki semua hak istimewa pada database yang diperlukan (menurut saya):

mysql> SHOW GRANTS FOR 'user'@'%';
+-------------------------------------------------------------------------------------------------------------------+
| Grants for user@%                                                                                            |
+-------------------------------------------------------------------------------------------------------------------+
| GRANT ALL PRIVILEGES ON *.* TO 'user'@'%' IDENTIFIED BY PASSWORD 'pass_ommited' |
| GRANT ALL PRIVILEGES ON `database`.* TO 'user'@'%'                                                          |
+-------------------------------------------------------------------------------------------------------------------+

Tetapi ketika aplikasi mencoba untuk menyambung ke database, saya menerima kesalahan:

 CDbConnection failed to open the DB connection: SQLSTATE[HY000] [2003] Can't connect to MySQL server on '168.63.178.87' (4)

(Ini adalah aplikasi kerangka Yii, jika itu penting).

Saya memiliki akses SSH penuh ke mesin virtual, menggunakan alamat IP yang diberikan dan port tertentu. Azure juga memberi tahu saya "alamat IP Internal", yaitu 10.78.28.85. Saya telah mencoba mengikat server MySQL ke alamat itu dan ke 0.0.0.0, tetapi tidak ada yang berhasil.

Saya menduga masalahnya adalah pengalihan yang dilakukan untuk mengakses instance spesifik saya di server, tetapi saya tidak tahu cara mengatasinya.
