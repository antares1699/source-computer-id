Bagaimana cara kerja menyalin file berkaitan dengan RAM, dan dapatkah saya mempercepatnya?

Saya selalu bertanya-tanya mengapa komputer saya menyalin file ke drive eksternal pada awalnya dengan kecepatan tinggi, dan kemudian 'hang' sebentar sebelum melanjutkan. Ketika saya memutakhirkan RAM dari 1GB ke 2GB, saya perhatikan 'gantung' ini jarang terjadi. Saya sekarang sampai pada 'kesimpulan' bahwa RAM digunakan untuk menyimpan data dan kemudian menyalinnya ke drive, jadi ketika saya menyalin sesuatu yang lebih besar dari RAM saya, itu perlu masuk dalam 'batch'. Apakah ini masuk akal? Apakah ini yang terjadi? Dan juga, apakah ini berarti bahwa jika saya menambahkan beberapa GB RAM, kemungkinan besar saya dapat menyalin file yang lebih besar dan memiliki kecepatan yang lebih konstan?

Saya menggunakan Ubuntu Linux.

Anda menyaksikan " Penyangga Data ", yang merupakan bagian standar dari (secara praktis) semua komputer.

Dari tautan Wikipedia itu:

Dalam ilmu komputer, buffer adalah wilayah penyimpanan memori fisik yang digunakan untuk menyimpan data sementara saat sedang dipindahkan dari satu tempat ke tempat lain.

Buffer sering digunakan bersama dengan I / O ke perangkat keras, seperti disk drive ...

Singkatnya: Ya, Anda berada di jalur yang benar dalam pemikiran Anda, dan mendapatkan lebih banyak RAM akan membantu mempercepat transfer file. Perbedaannya tergantung pada banyak hal di luar RAM (ukuran file, jumlah file, dll.) Dan media penyimpanan jangka panjang (disk) adalah penghambat untuk mempercepat 99% waktu.

  • "Buffer sering digunakan ..." - Artikel Wikipedia tidak terlalu akurat. Daripada "sering digunakan", buffer adalah jarang tidak digunakan . Tentang satu-satunya buffer waktu yang tidak diperlukan adalah saat mentransfer unit data yang sangat kecil, seperti nilai byte. Jika tidak, data untuk transfer blok, sektor, dan bingkai / paket pesan selalu buffer dalam memori utama sebelum transmisi / penulisan dan saat diterima / dibaca. Ada juga buffering (tersembunyi) yang terjadi secara internal di perangkat periferal seperti pengontrol disk (antara antarmuka SATA dan kepala R / W) dan sakelar Ethernet (simpan & maju).
