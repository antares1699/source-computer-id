Kesalahan saat memasang share Arch Linux NFS di Windows 7 Ultimate

Saya mencoba memasang share NFS melalui LAN:

  • Server: Arch Linux x64
  • Klien: Windows 7 Ultimate x64

Server sudah berhasil mengekspor share NFS ke mesin Linux lainnya, jadi saya cukup yakin tidak ada masalah. Saya menambahkan entri pada /etc/exports untuk mesin Windows baru dan memulai ulang layanan nfs-server.

Di komputer Windows, saya mengaktifkan yang berikut ini di "Add / Remove Programs":

  • Layanan untuk NFS
  • Subsistem untuk Aplikasi berbasis UNIX

Kemudian, ikuti panduannya sini Saya mencoba menjalankan perintah berikut untuk memasang share NFS sehingga akan muncul sebagai drive Z:\:

mount //192.168.1.99/data/authorized Z:

Saya mendapatkan hasil sebagai berikut:

cmdlet New-PSDrive at command pipeline position 1
Supply values for the following parameters:
Root:

Itu menunggu saya untuk memasukkan sesuatu untuk "Root:". Saya menduga ini berarti root berbagi NFS, jadi saya memasukkannya (/data/authorized), meskipun seharusnya sudah diambil dari perintah awal saya.

Saat itulah saya mendapat kesalahan ini:

New-PSDrive : Cannot find a provider with the name 'Z:'.
At line:1 char:6
+ mount <<<<  //192.168.1.99/ Z:
    + CategoryInfo          : ObjectNotFound: (z:String) [New-PSDrive], ProviderNotFou
    + FullyQualifiedErrorId : ProviderNotFound,Microsoft.PowerShell.Commands.NewPSDriv

Bagaimana cara agar share berhasil dipasang?
