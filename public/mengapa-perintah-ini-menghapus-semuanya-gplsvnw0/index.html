Mengapa perintah ini menghapus semuanya?

Saya menjalankan perintah ini di kotak Unix:

find . name CVS -exec rm -fr {} \;

Saya ingin menghapus file apa pun bernama CVS dalam direktori mana pun dari direktori saat ini dan itu menghapus semuanya.

Untungnya yang harus saya lakukan untuk memulihkan adalah memeriksa kembali dari CVS. Bayangkan jika saya menetapkan / sebagai direktori awal!

Saya pikir alasannya adalah saya menggunakan name dan bukan -name. Saya hanya menjalankannya kembali sebagai

find . -name CVS -exec rm -fr {} \;

Dan tampaknya berfungsi dengan baik. Apa yang sebenarnya terjadi jika name digunakan sebagai lawan -name?

Anda kehilangan tanda hubung sebelum -name, oleh karena itu dicari jalur bernama ., name dan CVS, di mana titik mereferensikan folder saat ini, sehingga menghapus semua.

Utilitas find secara rekursif menurunkan pohon direktori untuk setiap jalur yang terdaftar.

Anda dapat dengan mudah menguji dengan menggunakan echo sebelum perintah yang ingin Anda jalankan:

find . name CVS -exec echo rm -fr {} \;
