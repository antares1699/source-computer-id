Bash RegEx di OSX Vs Linux (keanehan)

Ini tentang pengembangan Bash dan pengkodean skrip Bash portabel yang menggunakan RegEx .

Menggunakan Bash RegEx, di Mac, saya dapat melakukan ini:

coconut-mac$ a='bananacoconutman'; [[ "$a" =~ banana(.*?)man ]] && echo FOUND ${BASH_REMATCH[1]}
FOUND coconut

Bagus. Berguna di banyak tempat. Suka.

Ketika saya mencoba melakukan ini, gagal:

coconut-mac$ a='<title>coconut</title>'; [[ "$a" =~ \<title\>(.*?)\</title\> ]] && echo FOUND ${BASH_REMATCH[1]}

Itu tepat perintah yang sama dijalankan sempurna di penguin:

coconut-linux$ a='<title>coconut</title>'; [[ "$a" =~ \<title\>(.*?)\</title\> ]] && echo FOUND ${BASH_REMATCH[1]}
FOUND coconut
  • Mengapa?
  • Bagaimana cara memperbaikinya untuk membuat skrip portabel?

Di Mac saya, info bash / =~ RET mengatakan:

Operator biner tambahan, =~', is available, with the same precedence as == 'dan `! ='. Saat digunakan, string ke kanan operator dianggap sebagai ekspresi reguler yang diperluas dan dicocokkan dengan semestinya (seperti pada regex3)).

man 3 regex mengatakan:

Operator pengulangan (?', * ', +', or bounds) cannot follow another repetition operator. A repetition operator cannot begin an expression or subexpression or follow ^' atau `| '.

Saya tidak melihat dokumentasi analog di GNU regex's man 3 regex atau info regex.

Jika saya menghapus ? dari (.*?) Anda dan melakukan hal berikut, ini berfungsi pada kedua OS:

$ a='<title>coconut</title>'; [[ "$a" =~ \<title\>(.*)\</title\> ]] && echo FOUND ${BASH_REMATCH[1]}
FOUND coconut
