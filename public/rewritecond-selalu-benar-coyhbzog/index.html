rewritecond selalu benar

Universitas saya menyediakan file public_html di direktori Linux setiap siswa sehingga setiap siswa dapat memiliki halaman web. Saya ingin memasukkan semua skrip PHP saya ke dalam file itu dan menempatkan indeks di sub-direktori bernama webroot. Saya mencoba mencari cara untuk memiliki file .htaccess di public_html yang akan mengarahkan SEMUA permintaan di folder itu menjadi dialihkan.

Ada banyak saran untuk mengarahkan ulang file apa pun yang tidak ada tetapi saya ingin mengalihkan terlepas dari keberadaan file tersebut. Dapatkah saya menggunakan sesuatu seperti RewriteCond TRUE?

Anda dapat meletakkan ini di file .htaccess Anda.

RewriteEngine On
RedirectMatch permanent ^~username/$ http://example.com/~username/php/index.php

Itu harus mengarahkan setiap permintaan root ke file indeks Anda (tidak diuji). Sesuaikan agar cocok dengan url dan domain public_html Anda.
