chroot di OSX sebagai OS yang berbeda

Saya bertanya-tanya apakah ada yang bisa menggunakan chroot di OSX untuk menjalankan OS lain (ubuntu, centos). Saya tahu mereka sangat berbeda, tetapi hampir semua yang saya ingin gunakan untuk ini tidak akan peduli tentang apa pun pada level kernel, jadi saya berharap akan ada cara untuk melakukan ini tanpa menggunakan VM.

Berdasarkan pencarian google saya, saya melihat pertanyaan ini ditanyakan, tetapi tidak ada jawaban nyata selain "coba VM". Akan sangat ingin melakukan ini tanpa VM.

Sayangnya, sebagian besar program melakukan peduli dengan kernel, baik secara langsung maupun tidak.

Sistem operasi yang berbeda memiliki perbedaan yang signifikan bahkan dalam fitur yang paling umum (epoll vs inotify vs kqueue, clone vs vfork vs rfork ...).

Bahkan jika program menggunakan fitur minimal yang terlihat sama dengan userspace (mis. Stdio, akses file dasar), program (atau lebih tepatnya, libc) masih perlu mengetahui cara memanggil fungsi tersebut di kernel, yaitu sama sekali berbeda antara Linux (syscall lama biasa) dan OS X (campuran syscall dan Mach IPC).

Dan itu dengan asumsi Anda bisa mendapatkan kernel untuk mulai menjalankan program di tempat pertama - kernel OS X tidak memahami format ELF, hanya Mach-O (ditambah semua jenis barang lama Mac OS).

Berbagai emulator ada - misalnya, FreeBSD memiliki emulasi Linux, dan Wine menjalankan program Windows di Unix, dan tampaknya ada Emulator OS X untuk Linux - tetapi saya tidak dapat menemukan yang dapat menjalankan program Linux di OS X.

Berikut uraiannya bagaimana fungsi dasar Linux diemulasikan di FreeBSD .

chroot mengubah direktori root; itu tidak mengubah format file yang dapat dieksekusi, antarmuka biner aplikasi, dll, yang semuanya berbeda antara OS X dan linux. Menjalankan OS lain membutuhkan lebih dari sekadar chroot; itu membutuhkan VM.
