Mengapa data dalam hard disk tidak dapat dipindahkan?

Saya memahami bahwa untuk membuat partisi baru, perangkat lunak partisi pada dasarnya memindahkan data "ke atas" sehingga lebih banyak ruang dapat disediakan untuk partisi. Namun,

karena satu dan lain hal, beberapa data tidak dapat dipindahkan. (sumber)

Saya mendapat kesan bahwa selama komputer tidak berjalan (penjadwalan partisi saat restart), pada dasarnya semua jenis data dapat dipindahkan. Saya bertanya-tanya alasan apa yang mungkin ada sehingga data tidak dapat dipindahkan?

Atau lebih tepatnya, apa Tipe data di hard disk tidak bisa dipindahkan?

  • Pertanyaan yang sangat bagus. Satu-satunya hal yang dapat saya pikirkan sekarang adalah data yang digunakan oleh mengatakan bootloader yang diharapkan berada di lokasi fisik tertentu pada disk oleh beberapa program. Selain fakta bahwa, seperti yang disebutkan dalam jawaban lain, jika komputer tidak berjalan maka tidak mungkin untuk melakukan sesuatu , petunjuk besar di sini seharusnya adalah dokumentasi Red Hat secara eksplisit menunjukkan bahwa ada pilihan perangkat lunak untuk melakukan tugas-tugas ini, dan perilakunya berbeda. Apa yang tidak dapat dipindahkan bervariasi dari perangkat lunak ke perangkat lunak. Disk defragmenter yang berbeda memiliki ide yang berbeda tentang apa yang dapat dan tidak dapat dipindahkan, dan akan menolak untuk memindahkan benda yang berbeda.

Misalnya: Di tahun-tahun MS-DOS / PC-DOS / DR-DOS dan DOS + Windows, sebagian besar utilitas defragmentasi disk menolak untuk memindahkan file apa pun yang diberi tag dengan atribut system atau hidden. Ini karena file BIOS, ibmbio.com atau io.sys, ditempatkan secara khusus pada disk, karena keterbatasan dalam kode bootstrap loader FAT VBR yang digunakan oleh DOS. File BDOS, ibmdos.com atau msdos.sys dimuat oleh program loader yang lebih mampu di BIOS dan tidak memiliki semua batasan penempatan yang sama. Tetapi defragmenter disk tidak tahu file apa itu, mengingat variasi nama, apalagi mengatasi variasi halus di antara beberapa versi DOS dan hal-hal tambahan seperti file paging Windows 3.x, volume terkompresi SuperStor, dan sebagainya. Jadi mereka kembali ke mudah diterapkan asumsi itu apa saja file yang ditandai dengan atribut tersebut, karena semua yang disebutkan di atas biasanya secara default, oleh karena itu ditetapkan oleh administrator sistem sebagai tidak dapat dipindahkan.

Windows NT, sejak versi 4.0 jika memori berfungsi, mendefinisikan API defragmentasi standar yang seharusnya diterapkan oleh FSD dan program defragmentasi disk dapat digunakan. Jadi pada Windows NT saat ini apa yang dianggap tidak dapat dipindahkan umumnya adalah provinsi dari driver sistem file (FSD) daripada utilitas defragmentasi disk. Jadi apa yang dianggap tidak dapat dipindahkan dalam - katakanlah - volume NTFS sangat bergantung pada versi NTFS FSD yang digunakan. Salah satu fitur yang disebut-sebut pada rilis Windows NT 5.1 adalah peningkatan pada NTFS FSD yang memungkinkan direktori defragmentasi, MFT, bitmap volume dan entri MFT metadata lainnya, file terenkripsi, dan file di mana ukuran unit alokasi volume lebih besar dari 4KiB .

Semua variasi ini, dari perangkat lunak utilitas hingga perangkat lunak utilitas dan dari versi FSD ke versi FSD, yang berada di luar cakupan dari apa yang coba dijelaskannya, adalah apa yang dicakup oleh Red Hat dalam empat kata "satu alasan atau lainnya".

File yang tidak dapat dipindahkan biasanya menjadi masalah saat Anda menjalankan defrag atau utilitas serupa lainnya pada partisi yang terkait dengan lingkungan operasi yang sedang berjalan. Di Windows, file-file ini, misalnya, registri, file halaman, file metadata, dll.

Menggunakan utilitas defrag atau partisi pada partisi yang tidak aktif dalam banyak kasus berarti file yang tadinya tidak dapat dipindahkan ini sekarang dapat dipindahkan ke awal partisi dan / atau didefragmentasi.

Dalam konteks pertanyaan Anda tentang menginstal OS baru, Anda harus menjalankan beberapa defrag pass pada volume yang tidak terpasang (menggunakan livecd, misalnya) setelah itu Anda akan dapat menggunakan gparted atau pengelola partisi instalasi OS untuk mengecilkan partisi yang ada.
