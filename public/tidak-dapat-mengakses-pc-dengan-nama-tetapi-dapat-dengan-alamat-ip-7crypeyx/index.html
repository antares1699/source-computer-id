Tidak dapat mengakses PC dengan nama, tetapi dapat dengan alamat IP

Saat ini saya mengalami masalah dengan komputer saya B dan C dapat mengakses komputer SEBUAH dengan nama (mis. \\computerA), tetapi komputer D tidak bisa menggunakan nama.

Komputer D hanya dapat mengakses SEBUAH melalui alamat IP-nya (mis. \\192.168.0.xxx).

Komputer D aktif Windows 10 dan SEBUAH masuk Windows XP .

Ini bukan masalah besar, tapi saya ingin menyelesaikannya dan selalu menggunakan nama komputer.

Anda harus memeriksa ulang bahwa file Penemuan Jaringan diaktifkan. Berikut langkah-langkah untuk menyalakannya (berdasarkan Windows 10 ):

  1. Pergi ke Panel kendali dengan membuka Pengaturan berbagi lanjutan (dari Mulailah tombol).
  2. Tipe jaringan dan buka Jaringan dan pusat Berbagi dan pilih Ubah setelan berbagi lanjutan .
  3. Aktifkan penemuan jaringan dan Simpan perubahan .

Lihat: Apa itu penemuan jaringan?

Penting juga, saat Anda terhubung ke jaringan, untuk memilih jenis Lokasi jaringan seperti: Rumah , Kerja , Publik , dan Domain . Berdasarkan ini, Windows menetapkan status penemuan jaringan dan mengkonfigurasi file Windows Firewall port untuk negara bagian itu.

Lebih disukai pilih Rumah lokasi, dimana Penemuan jaringan dihidupkan, yang memungkinkan Anda melihat komputer dan perangkat lain dan memungkinkan pengguna jaringan lain untuk melihat komputer Anda di jaringan.

Anda dapat mengubah lokasi Anda, dengan memilih opsi di Jaringan dan pusat Berbagi (antara Jaringan kerja , Jaringan rumah , atau Jaringan publik ).

Lihat: Memilih lokasi jaringan

Jika komputer Anda dapat dilihat melalui \\192.168.0.xxx), maka jaringan lokal Anda seharusnya berfungsi, jadi mungkin Anda hanya perlu masuk ulang ke kanan Kelompok Kerja . Jika itu semua tentang penemuan nama, Anda harus memeriksa apakah alamat IP penyiaran Anda cocok.

Penyelesaian masalah

Jika ini masih tidak berhasil, periksa:

  • Anda telah menonaktifkan firewall lain (untuk tujuan pengujian, nonaktifkan sementara anti-virus Anda),

  • komputer Anda merespons ping dari komputer lain, mis. di baris perintah (ubah IP menjadi IP komputer Anda yang relevan):

    ping 192.168.0.2
ping 192.168.0.3

  • Anda terhubung ke jaringan yang sama,

  • IP Anda berada dalam rentang yang sama (dan file topeng jaringan pertandingan),

    Ini diperlukan untuk paket siaran yang dikirim ke alamat IP penyiaran yang sama (mis. 192.168.255.255 ).

    Semua komputer harus berada di jaringan lokal atau subnet yang sama.

  • komputer berada dalam satu kesatuan kelompok kerja dan homegroup (bisa jadi tidak wajib),

  • periksa apakah komputer Anda memiliki port berbagi file terbuka (port 139 dan 445), mis.

    telnet 192.168.0.xxx 139
telnet 192.168.0.xxx 445

    Jika telnet hilang, instal melalui (run as admin ):

    dism /online /Enable-Feature /FeatureName:TelnetClient

  • Anda sudah Dukungan Berbagi File CIFS diaktifkan di Fitur Windows

  • masuk kembali atau mulai ulang komputer Anda, sehingga komputer Anda dapat masuk ke jaringan lokal

Saya tahu ini adalah pertanyaan yang sangat lama, tetapi saya juga tahu bahwa orang-orang masih mengalami masalah dengan ini. Jawaban teratas memang menyentuh ini di akhir posting mereka, tetapi saya rasa tidak cukup penekanan diberikan pada betapa pentingnya hal ini sebenarnya. Masalahnya adalah Windows XP menjalankan SMB 1.0 dan tidak mendukung SMB 2.0. Windows 10 menjalankan SMB 2.0 dan tidak mendukung SMB 1.0 tanpa mengaktifkannya dalam menambah / menghapus program. Ini adalah ketidaksesuaian mendasar antara kedua sistem operasi. Pada sistem Windows 10, tahan tombol logo Windows Anda dan tekan R, ketik appwiz.cpl dan tekan enter. Klik "Turn Windows features on or off" dan gulir ke bawah ke SMB 1.0. Klik + di sebelahnya dan aktifkan klien SMB 1.0 / CIFS lalu tunggu sampai selesai dan reboot. Perlu diingat, ini akan memperkenalkan kerentanan keamanan pada sistem Windows 10, tetapi Anda perlu melakukan ini untuk mengakses share apa pun di sistem Windows XP tersebut. Dari sana Anda harus bisa memetakan dengan nama komputer atau alamat IP. Jika nama komputer tidak berfungsi, kemungkinan router Anda rusak dan perlu di-boot ulang. Secara pribadi, saya lebih suka memetakan semuanya berdasarkan alamat IP. Jika itu masih tidak berhasil, Anda mungkin perlu memberikan kredensial pada mesin Windows 10 untuk terhubung ke mesin XP.
