Fragmentasi terjadi ketika file yang sudah ada di disk diubah sedemikian rupa menjadi lebih besar dari sebelumnya, dan tidak ada file yang lebih kecil dari Anda. ukuran kelompok harus berakhir terfragmentasi. untuk menjawab pertanyaan Anda dengan benar, kami perlu mengetahui ukuran cluster Anda, persentase drive yang digunakan, dan apakah file berubah setelah alokasi awal ruang disk.

Operasi IO pada banyak file kecil vs beberapa file besar jauh lebih lambat, karena semua overhead sistem file (dan waktu pemindaian / pencarian), jadi jika Anda menganggap bahwa kinerja yang menurun, maka ya. ini sangat bergantung pada apakah Anda mengakses 5000 dari mereka sekaligus, atau hanya beberapa pada satu waktu.

Ext4 khususnya tidak memerlukan defragmentasi cara yang sama dilakukan oleh sistem file windows karena caranya mengalokasikan ruang ke file, jadi selama partisi Anda tidak 90% penuh, saya tidak akan khawatir tentang itu.

Apakah ada implikasi menyimpan file dalam jumlah besar?

Saya akan menambahkan bahwa Anda harus memperhatikan jumlah maksimum file yang dapat ditangani oleh sistem file Anda. File (di ext4 yang Anda gunakan) menggunakan inode. Tabel inode dialokasikan secara statis saat sistem file dibuat dan satu-satunya cara untuk meningkatkannya adalah dengan membuat ulang sistem file.

Anda dapat memeriksa inode Anda saat ini digunakan / nomor gratis dengan df -i atau tune2fs -l /path/to/device

Nilai default harus menjadi masalah hanya saat menyimpan file yang sangat kecil.

Kecepatan akses menurun secara signifikan. Ini adalah masalah umum dalam proyek Java, di mana Anda memiliki banyak file yang relatif kecil. Lebih efisien untuk menyimpannya dalam satu arsip tunggal, aksesnya lebih cepat meskipun harus dibongkar.

Dalam kasus Anda, file Anda tidak kecil, jadi mungkin tidak akan ada masalah dengan fragmentasi.

Anda mengerjakan proyek arsip, jadi Anda mungkin harus menggunakan ZIP atau TAR / GZIP . Jika file Anda sudah dikemas, Anda dapat menggunakan TAR untuk menyatukannya menjadi satu file yang lebih besar. Temukan beberapa "unit pengepakan" yang masuk akal - sehingga Anda dapat dengan mudah menemukan semua file Anda nanti.
