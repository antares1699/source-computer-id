Bagaimana cara membuat share anonim di Windows 10?

Saya mencoba membuat share anonim di komputer Windows 10 saya. Sekarang jika saya mengklik kanan pada folder yang saya bagikan, beralih ke tab berbagi, saya dapat melihat di bagian bawah jendela properti People without a user account and password for this computer can access folders shared with everyone.
Tetapi ketika saya mencoba mengaksesnya dari Mac atau PC, saya masih mendapatkan perintah login.
Saya juga mencoba banyak hal yang saya temukan di web:

  1. Selain berbagi ke semua orang, buka pengaturan lanjutan, berikan ANONIM dan akses tamu.
  2. Edit kebijakan grup, aktifkan akun tamu dan buat beberapa jaringan lain konfigurasi terkait.

Tidak ada yang berhasil sampai sekarang. Sepertinya tidak mungkin dan bodoh, karena jendela properti dengan jelas mengatakan People without a user account and password for this computer can access folders shared with everyone.

Adakah yang pernah berhasil membuat folder berbagi anonim 100% di Windows 10?

Terima kasih sebelumnya.

Metode mudah:

  1. Jaringan dan Pusat Berbagi> Pengaturan berbagi lanjutan> Semua Jaringan> Matikan berbagi yang dilindungi kata sandi (ini memungkinkan akun tamu)

  2. Buat folder berbagi> klik kanan> properti> berbagi> berbagi lanjutan> Bagikan folder ini> Izin> centang 'kontrol penuh'> OK> tab keamanan> tambahkan "Semua Orang", centang "kontrol penuh"> OK> OK

Tidak perlu mengaktifkan berbagi folder Publik, mengacaukan izin keamanan (selain 'Semua Orang'), dan dapat membiarkan Network access: Sharing and security model for local accounts disetel ke Classic

Saya tidak merekomendasikan penggunaan 'Berikan akses ke' atau 'Bagikan ...' "wizard" karena terkadang akan berbagi folder induk (misalnya jika folder bersama Anda ada di desktop Anda, itu akan berbagi folder "Pengguna") atau gagal untuk menerapkan izin keamanan "Semua Orang" yang diperlukan (misalnya jika folder bersama Anda ada di C: \)

Langkah-langkah ini tampaknya berhasil:

  • aktifkan pengguna tamu
  • jalankan gpedit.msc, masuk ke Computer Configuration \ Windows Settings \ Security Settings \ Local Policies \ Security Options, ubah nilai Network access: Sharing and security model for local accounts menjadi guest only

Mengenai poin ke-2, saya membaca dokumen Microsoft berikut ini:

https://docs.microsoft.com/en-us/windows/security/threat-protection/security-policy-settings/network-access-sharing-and-security-model-for-local-accounts

Kesimpulan saya setelah membacanya adalah, jika Anda menjalankan server jaringan, Anda harus menggunakan "klasik", jika tidak, tidak masalah menggunakan guest only karena Anda mungkin tidak menginginkan kontrol akses sama sekali, seperti saya.

Ini masih sangat bodoh, tanpa membuat perubahan, pengguna anonim tidak dapat mengakses saham mengapa sih panel properti mengatakan People without a user account and password for this computer can access folders shared with everyone.

Berikut adalah metode alternatif yang saya gunakan untuk melakukannya di Windows 10 Pro. Metode ini melibatkan pengaktifan fungsionalitas berbagi folder Publik yang dibangun ke dalam Windows, membuat folder Bersama baru dan mengatur izin Berbagi dan NTFS yang identik dengan folder Publik di bawah direktori Pengguna. Kemudian nonaktifkan share Publik. Metode ini tidak mengubah kebijakan keamanan lokal atau pengaturan registri (yang telah saya lihat diposting di seluruh Internet)

  1. Buka "Jaringan dan Pusat Berbagi" dan klik tautan "Pengaturan berbagi lanjutan".
  2. Luaskan "Semua Jaringan".
  3. Centang "Aktifkan berbagi sehingga siapa pun dengan akses jaringan dapat membaca dan menulis file di folder Publik"
  4. Klik "Matikan berbagi yang dilindungi kata sandi".
  5. Buat folder "Bersama" di drive pilihan Anda di drive apa pun yang Anda pilih.
  6. Aktifkan share dengan mengklik tombol “Advanced Sharing…”.
  7. Setel izin Berbagi ke "Semua Orang", "Kontrol Penuh".
  8. Tetapkan izin Keamanan [NTFS] sama dengan folder "Publik" di bawah direktori C: \ Users.
  9. Untuk "Interaktif", "Layanan" dan "Batch", setel 2 izin khusus untuk mencocokkan izin di Pengaturan Keamanan Lanjutan, Tunjukkan Izin Lanjutan.
  10. Opsional: Nonaktifkan berbagi di direktori "Pengguna" yang diaktifkan saat berbagi Folder Publik diaktifkan.
  11. Opsional: Jika beberapa subnet / VLAN perlu mengakses fileshare, masuk ke Windows Defender Firewall, Advanced FirewallSettings, klik "Inbound Rules" dan filter berdasarkan File andPrinter sharing group dan tipe profil. Di bawah tab Scope, modifyeach Inbound firewall rule dan ubah "localesubnet" di bawah "RemoteIP Address" untuk menyertakan subnet tambahan yang dibutuhkan akses untuk berbagi.
  12. Uji akses ke folder "Bersama" yang baru dibuat.
  13. Periksa dalam "Manajemen Komputer" untuk status Sesi. Ini menunjukkan sebagai "Tamu" adalah akun yang digunakan untuk mengotentikasi.
  14. Ulangi proses untuk beberapa folder bersama dengan akses anonim / tamu.
