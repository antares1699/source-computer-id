6to4 vs ISATAP vs Adaptor Teredo

Saya memahami bahwa ketiga adaptor yang disebutkan dalam judul terkait dengan pengalihan IPv4 ke IPv6.

ISATAP: Mengirimkan paket IPv6 antara node tumpukan ganda di atas jaringan IPv4.

Teredo: konektivitas IPv6 ke host IPv4.

Pada mesin windows saya, ipconfig / all mengembalikan beberapa adaptor ISATAP, satu adaptor 6to4, satu Teredo Tunneling Pseudo-Interface dan dua adaptor Tunnel Antarmuka ISATAP yang Dapat Digunakan Kembali.

Bagaimana ini berbeda atau terkait untuk menyediakan fungsionalitas IPv6 ke host di jaringan IPv4? Wikipedia dan situs lain tampaknya berbelit-belit dalam penjelasan mereka untuk istilah-istilah ini. Penjelasan bahasa yang sederhana akan dihargai.

6 hingga 4 adalah mekanisme di mana router dengan alamat IPv4 publik dapat menjadi gateway / penyedia IPv6 untuk seluruh rangkaian LAN. Awalan IPv6 dimulai dengan 2002: diikuti oleh 32 bit alamat IPv4 publik. Ini memberikan prefiks /48 yang dapat digunakan untuk menyediakan 65536 LAN dengan masing-masing /64. Masalahnya adalah untuk mencapai internet IPv6 normal, router 6to4 perlu menggunakan relai 6to4 publik, dan relai ini tidak selalu dapat diandalkan. Tambahkan ke ini bahwa lalu lintas masuk dan keluar menggunakan relai yang berbeda sebagian besar waktu, dan keandalan jaringan 6to4 meninggalkan banyak hal yang diinginkan. Hari-hari ini menggunakan 6to4 sangat tidak disarankan.

Teredo adalah mekanisme bawaan dalam sistem Windows yang digunakan untuk memberikan sistem tunggal di belakang akses NAT IPv4 ke IPv6. Seperti 6to4, ia menggunakan relai publik. Teredo menggabungkan ini dengan protokol pengaturan menggunakan server Teredo (secara default dihosting oleh Microsoft) untuk mendeteksi dan menerobos IPv4 NAT. Ini tidak terlalu dapat diandalkan, tetapi Teredo hanya digunakan ketika secara eksplisit menghubungkan ke alamat IPv6 dan tidak saat menghubungkan ke nama host dan mencari alamat di DNS. Ini terjadi misalnya saat menggunakan BitTorrent, dan di sana keandalan yang lebih rendah tidak terlalu menjadi masalah.

ISATAP adalah teknologi yang dikelola untuk menyediakan IPv6 pada jaringan IPv4. Ini mengemulasi konektivitas IPv6 pada infrastruktur IPv4. Penemuan Router IPv6 biasanya menggunakan multicast. Ini tidak mungkin dilakukan pada infrastruktur yang didasarkan pada terowongan IPv6-over-IPv4, jadi ISATAP menyelesaikannya dengan cara yang berbeda. Router ISATAP disediakan di jaringan dan alamat IPv4-nya diketahui oleh host ISATAP (biasanya menggunakan nama host isatap. Di DNS). Host ISATAP kemudian dapat menggunakan alamat tersebut untuk mengatur konektivitas IPv6 mereka.

Dalam semua kasus, lebih baik mendapatkan IPv6 asli di jaringan Anda. Lebih disukai melalui ISP Anda, dan sebaliknya melalui tunnelbroker seperti tunnelbroker.net atau sixxs.net .
