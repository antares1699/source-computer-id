Bagaimana cara "mematikan" semua jaringan di Linux dari bash?

Saya ingin menulis skrip bash / shell yang "mematikan" kemampuan mesin untuk berkomunikasi melalui kartu jaringannya, secara efektif mematikan semua jaringan. Dari baris perintah:

# To turn all networking off
sh networking.sh OFF

# Then to turn it back on
sh networking.sh ON

Satu-satunya cara yang dapat saya pikirkan untuk melakukan ini adalah di level port melalui IPTABLES, tetapi karena belum pernah bereksperimen dengan itu sebelumnya, saya tidak yakin apakah saya sedang menuju ke jalan yang benar atau apakah saya salah jalan . Terima kasih sebelumnya.

Kebanyakan linux sudah dilengkapi dengan skrip untuk melakukan ini. Mereka adalah bagian dari proses startup / shutdown.

Dalam rasa Fedora / RHEL, skrip biasanya diakses melalui /etc/init.d/. Jadi, Anda dapat mengeksekusi "/etc/init.d/network stop" untuk menghentikan jaringan. Untuk memulai, gantikan "mulai" dengan "berhenti".

Dari baris perintah, preferensi untuk ragam ini adalah menggunakan perintah service: "service network stop" atau "service network start."

Perintah-perintah ini terbatas untuk pengguna root. Saya akan merekomendasikan untuk tidak mengizinkan pengguna biasa melakukan sesuatu seperti ini.

Seperti yang Anda sebutkan, Anda dapat menggunakan iptables untuk ini

Aturan ini akan menghapus semua paket masuk dan keluar

iptables -A INPUT -j DROP
iptables -A OUTPUT -j DROP

Untuk memeriksa aturan yang dikonfigurasi

iptables -L

Untuk menghapus aturan ini

iptables -D INPUT -j DROP
iptables -D OUTPUT -j DROP

  • Bukan metode termudah karena perintah pengelolaan jaringan lebih baik dan benar-benar akan menurunkan jaringan (tidak hanya memblokir semuanya). Dengan -A itu pada dasarnya hanya akan bekerja dengan iptables yang sebelumnya kosong.
