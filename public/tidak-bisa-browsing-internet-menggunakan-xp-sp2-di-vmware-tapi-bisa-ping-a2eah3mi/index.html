Oke, jika ping google.com berfungsi maka Anda setidaknya yakin bahwa:

  1. Permintaan DNS UDP berfungsi (untuk menyelesaikan google.com ke alamat IP)
  2. Paket ICMP bekerja (untuk menerima balasan ping echo)

dan ketika keduanya bekerja, itu cukup aneh daripada TCP (digunakan oleh HTTP) tidak bekerja juga. Mungkin ini masalah browser dan bukan jaringan? (mengkonfigurasi proxy atau sesuatu?)

Untuk mengecualikan browser dari persamaan Anda dapat mencoba telnetting ke port 80, seperti:

C:/> telnet google.com 80
GET / HTTP/1.1
Host: google.com

Pada dasarnya Anda harus memahami bagaimana NAT bekerja di VMware.

  • Sambungan baru (Sambungan A) akan dibuat di sistem hosting Anda (yang tidak memiliki akses Internet jika Anda menggunakan Windows 7).

  • Koneksi jaringan (B) dibuat di mesin tamu yang menghubungkan ke koneksi di host (A).

Masalahnya adalah, biasanya A tidak dapat mengakses Internet dan terkadang pengaturan gateway B salah.

Jadi, hal-hal yang perlu Anda lakukan untuk memperbaiki masalah NAT Internet:

  1. Pastikan Anda menghubungkan A ke Internet. Anda dapat melakukan ini dengan membagikan koneksi default Anda ke A
  2. Setel IP gateway B ke alamat IP A.

Ini kemudian harus bekerja. Atau Anda dapat dengan mudah mematikan firewall pada host (terkadang ini tidak akan berfungsi karena port telah berubah karena penggunaan NAT).
