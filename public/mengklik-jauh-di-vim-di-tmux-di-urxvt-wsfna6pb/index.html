Mengklik jauh di vim di tmux di urxvt

Saya menggunakan vim di dalam tmux di dalam urxvt, dan mouse berfungsi dengan baik untuk mengklik dan memilih teks, kecuali jika saya ingin mengklik terlalu jauh ke kanan.

Tampaknya terkait dengan jarak dalam jumlah kolom dari kiri. Ketika saya melampaui kolom ~ 200 (tidak yakin tentang jumlah pastinya), mengklik tidak apa-apa.

Perhatikan bahwa ini tidak terkait dengan jendela vim: dengan dua jendela vim masing-masing mengambil ~ 150 kolom, mengklik tidak akan berfungsi setelah kolom ~ ke-50 di jendela kedua. Ini terkait dengan seluruh sesi vim.

Perhatikan juga bahwa mengklik jauh di panel tmux besar (> 200 kolom) bekerja dengan sempurna.

Di .tmux.conf saya, saya memiliki baris ini:

set -g default-terminal "screen-256color"

dan di .vimrc saya, saya memiliki ini:

if &term =~ "^screen"
    autocmd VimEnter * silent !echo -ne "\033Ptmux;\033\033]12;7\007\033\\"
    let &t_SI = "\<Esc>Ptmux;\<Esc>\<Esc>]12;5\x7\<Esc>\\"
    let &t_EI = "\<Esc>Ptmux;\<Esc>\<Esc>]12;7\x7\<Esc>\\"
    autocmd VimLeave * silent !echo -ne "\033Ptmux;\033\033]12;14\007\033\\"
end

Ini mengubah warna kursor tergantung pada mode pengeditan vim, dan berfungsi, yang berarti bahwa tmux benar-benar menyetel $TERM ke "screen-256color" - tetapi saya tidak tahu apakah ini ada relevansinya dengan masalah mouse saya.

Saya menjalankan Ubuntu 12.04, vim 7.3, tmux 1.6 dan rxvt-unicode 9.14.

Adakah yang tahu tentang apa yang menyebabkan masalah ini? Terima kasih.

Memperbarui: Saya sekarang menggunakan xterm dan berhasil. Saya pikir urxvt diperlukan untuk dukungan unicode, tetapi percaya atau tidak, xterm juga memiliki dukungan unicode.

Jadi, terima kasih Rekaman sekrup , Aku mendapat sebuah jawaban untuk ini. Direproduksi di bawah ini:

Protokol mouse asli xterm hanya mendukung hingga 223 kolom dan garis. Versi protokol yang lebih baru menggunakan pengkodean UTF-8, yang berubah keluar menjadi ide yang buruk, dan dukungan terminal yang lebih modern protokol yang sama sekali berbeda berdasarkan urutan pelarian SGR itu jauh lebih waras.

tmux secara otomatis mendukung protokol asli dan protokol SGR kedua cara (ke terminal yang dijalankan di dalamnya, dan ke aplikasi yang berjalan di dalamnya.) tetapi hanya meminta mode UTF-8 dari terminal luar jika 'mouse-utf8' diaktifkan di file konfigurasi.

Anda dapat menggunakan alat vttest untuk bereksperimen dengan protokol mouse yang berbeda di dalam dan di luar tmux untuk mendapatkan gambaran yang lebih baik tentang apa yang sedang terjadi.

Menurut dokumentasi untuk opsi 'ttymouse' Vim, ia akan bertanya terminal untuk dukungan original-xterm-protocol jika $TERM adalah varian dari xterm, mlterm atau screen (dan tmux melaporkan dirinya sendiri sebagai screen, jadi BAIK). Jika terminal mendukung fitur termcap "RV" untuk menentukan Nomor versi xterm, Vim akan menggunakannya untuk secara otomatis meningkatkan ke Protokol mouse 'xterm2' atau 'sgr' ... tetapi biasanya hanya xterm yang mengiklankan "RV" mendukung, jadi tmux ditinggalkan dari proses peningkatan otomatis.

Di ~/.vimrc saya, saya memiliki kode berikut untuk mengaktifkan dukungan mouse yang layak, alih-alih mengandalkan deteksi otomatis Vim:

" Screen/tmux can also handle xterm mousiness, but Vim doesn't
" detect it by default.
if &term == "screen"
set ttymouse=xterm2
endif

if v:version >= 704 && &term =~ "^screen"
" Odds are good that this is a modern tmux, so let's pick the
" best mouse-handling mode.
set ttymouse=sgr
endif

(Catatan, set ttymouse=sgr adalah perbaikan yang sebenarnya, di sini; saya sarankan menambahkannya ke .vimrc Anda seperti yang dilakukan Tim.)
