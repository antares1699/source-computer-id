Putty di Linux tidak bekerja dengan nama pengguna di bidang nama host

Saya sedang menyiapkan VM berbasis Linux Mint 14 Cinnamon dev env dan baru saja menginstal dempul untuk terhubung ke server Ubuntu saya. Masalahnya adalah ketika saya mencoba menggunakan "username @ hostname" atau "username @ ip" (jelas tidak ada tanda kutip) maka saya mendapatkan pesan error "Tidak dapat membuka koneksi ke username @ hostname: Nama atau layanan tidak dikenal".

Jika saya hanya menggunakan nama host atau IP, koneksi akan baik-baik saja - ia meminta nama pengguna dan kemudian masuk dengan benar menggunakan kunci pribadi saya.

Pada mesin host Windows 8 saya, saya menggunakan "username @ hostname" di bidang Nama Host Putty dan bekerja dengan sempurna.

Apakah dempul Linux menggunakan sintaks yang berbeda di bidang nama host? Saya memeriksa sekeliling tetapi tidak dapat menemukan apa pun.

Sepertinya Anda tidak dapat menentukan nama pengguna di GUI dempul. Namun, Anda tidak perlu menggunakan dempul jika yang ingin Anda lakukan adalah menghubungkan ke host linux lain melalui ssh. Buka saja terminal dan ketik:

ssh user@host

Ini akan melakukan apa yang dempul lakukan untuk Anda di windows.

Jika Anda ingin dempul untuk menyimpan koneksi, tambahkan baris seperti berikut ke dalam file ~/.bashrc Anda (berikut ini diadaptasi dari saya):

alias myserver='ssh myuser@myserver'
alias petitbonum='ssh bob@petitbonum' 
alias docpad='ssh terdon@docpad.local'
alias badabing='ssh -Y lacoloc@1.2.3.4'

Jelas, ubah nama pengguna dan server dan / atau IP menjadi apa pun yang Anda butuhkan. Kemudian, Anda membuka terminal baru dan mengetik, misalnya, myserver dan tekan enter. Ini akan menjalankan perintah yang ditentukan dalam alias yang Anda tempatkan pada .bashrc.

Mungkin PuTTY versi sebelumnya di Linux tidak mengizinkan ini, tetapi versi 0.63 memungkinkan Anda menentukan nama pengguna di bagian Sambungan pada dialog konfigurasi awal.

Sambungan -> Data -> "Nama pengguna masuk otomatis"

Anda dapat menyimpannya di bawah "Sesi Tersimpan".
