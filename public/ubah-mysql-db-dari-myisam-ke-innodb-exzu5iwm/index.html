Ubah MySQL DB dari MyISAM ke InnoDB

Jadi saya rasa saya mengalami masalah dengan komponen Joomla karena saya tidak memiliki setup db saya untuk menggunakan mesin InnoDB. Saya pikir ini karena saya tidak mengubah apa pun dalam menjalankan MySQL dan percaya MyISAM menjadi defualt. Apakah ada cara untuk memeriksa ini terlebih dahulu?

Kedua, jika yang saya yakini benar adalah, apakah ada yang tahu sumber daya / panduan yang baik untuk mengubah DB dari satu mesin ke mesin lainnya? Saya telah menemukan beberapa tetapi segera menunjukkan bahwa saya adalah pemula SQL yang masif dan ini terbukti sebagai semacam baptisan api. Panduan yang telah saya lihat sejauh ini semuanya agak bervariasi dalam pendekatannya dan banyak perintah yang semuanya dipompa bersama sehingga agak sulit untuk mengetahui apa yang sedang terjadi. Keuntungan besar adalah server ini virtual dan saya telah mengambil banyak snapshot sehingga kesalahan apa pun tidak besar.

Saya menggunakan Lamp stack dengan Ubuntu 9.10 x64, Apache 2.2.12, MySQL 5.1.37 dan PHP versi 5.2.10-2

Sebagai jawaban untuk bagian pertama dari pertanyaan Anda. Cara tercepat untuk menentukan mesin penyimpanan whoch secara default adalah dengan mengeluarkan yang berikut ini;

$mysql -h localhost -u user -p -e "SHOW ENGINES"

Ini akan menghasilkan daftar mesin yang didukung yang mengaktifkan server db yang Anda sambungkan. Yang default akan memiliki "Support DEFAULT" yang menunjukkan mesin mana yang dikonfigurasi sebagai layanan default.

Sehubungan dengan bagian kedua dari pertanyaan Anda. Ada beberapa cara untuk melakukannya. Cara pertama adalah menggunakan SQL, jadi Anda bisa mengeluarkan;

ALTER TABLE mytable ENGINE = InnoDB;

Ini bisa memiliki sejumlah kekurangan. Ini akan memakan waktu lama jika meja Anda terlalu besar. Lihat http://dev.mysql.com/doc/refman/5.1/en/alter-table.html untuk info lebih lanjut tentang itu. Tetapi jika itu metode yang cepat dan kotor, itu akan baik-baik saja.

Pilihan lainnya adalah menggunakan mysqldump untuk membuang semua database MyISAM Anda. Anda kemudian dapat mengubah dump SQL yang dihasilkan menggunakan find replace;

sed -i 's/MyISAM/INNODB/g' file.sql

Namun, seperti yang disarankan Patkos, saya hanya akan membuang sintaks buat dan kemudian membuat db baru tetapi mengatur mesin defaultnya untuk skema itu menjadi Innodb dan mengimpor ulang data
