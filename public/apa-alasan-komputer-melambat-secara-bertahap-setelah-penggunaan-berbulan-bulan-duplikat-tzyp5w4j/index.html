Alasan Komputer Melambat secara bertahap

1) Registri yang tidak diinginkan karena perangkat lunak pihak ketiga

2) Banyak program yang Anda instal juga menginstal aplikasi latar belakang. Karena pengkodean yang buruk saat Anda mencopot pemasangan program yang dapat dilakukan tugas latar belakang masih dibiarkan berjalan. Berkali-kali a tugas latar belakang dibagikan dan uninstaller tidak tahu apakah itu harus dihapus atau tidak.

3) Waktu boot Windows melambat karena beberapa perangkat lunak yang Anda instal (atau yang dipasang sendiri) berjalan start-up secara default.

4) Data semakin terfragmentasi karena terus menambahkan perangkat lunak baru. Sangat Disk terfragmentasi memperlambat komputer menyebabkan melambat

5) Menghapus instalan aplikasi baik melalui panel kontrol atau folder menu mulai aplikasi memiliki diketahui meninggalkan sisa-sisa program yang dihapus. Ini sisa-sisa dapat memperlambat komputer Anda sementara Windows mencoba mencari tahu apa hubungannya dengan mereka. Selain itu, residu program lama tetap tertinggal di registri sistem, menyumbat PC Anda inti.

Baca ini untuk alasan lainnya, Sumber daya diambil dari sini.

Satu hal yang tidak Anda sebutkan adalah OS yang Anda gunakan. Hal lain yang tidak Anda sebutkan adalah apakah Anda defragmentasi hard disk Anda. Seberapa relevan itu tergantung pada OS.

Fragmentasi file meningkat dari waktu ke waktu menghasilkan perlambatan bertahap karena membutuhkan lebih banyak waktu untuk membaca file yang bagian-bagiannya tersebar daripada diatur secara berdekatan. Setiap kali file lama dihapus atau diedit dan yang baru ditambahkan, alokasi file dapat menjadi semakin terfragmentasi.

Dengan asumsi Anda menggunakan Windows, mungkin saja file defragmenter yang disediakan tidak dijalankan secara otomatis sesuai jadwal. Kemungkinan juga itu tidak terlalu agresif.

Ada defragmenter pihak ketiga yang disebut MyDefrag yang menggunakan API defrag Windows. Saya menggunakannya dan merasa sangat bagus. Tidak hanya mendefragmentasi tetapi juga mengoptimalkan posisi file pada drive sehingga akses lebih efisien.

Beberapa alasan komputer menjadi lambat, yang sebelumnya tidak lambat, adalah:

  • Peningkatan jumlah file. Ini bisa menjadi signifikan terutama jika Anda menjalankan program antivirus real-time atau program cadangan di latar belakang, atau program pengindeksan seperti Windows Search atau Nero Scout.

  • Apakah Anda benar-benar mencopot semua perangkat lunak yang tidak digunakan? Apakah ini termasuk add-on browser seperti toolbar, dll.?

  • Anda mungkin memiliki perangkat lunak atau layanan yang berjalan di latar belakang yang tidak Anda ketahui. Pertama, lihat services.msc dan lihat apakah ada yang tidak Anda butuhkan sedang berjalan. Peringatan - teliti layanan apa pun dengan hati-hati sebelum menonaktifkan - Anda tidak ingin menonaktifkan sesuatu yang diperlukan agar Windows dapat beroperasi dengan benar. Ambil Process Explorer dan Autoruns dari Sysinternals. Process Explorer dapat menunjukkan kepada Anda apa yang sedang berjalan pada saat itu, dan Autoruns dapat menunjukkan kepada Anda apa yang dimuat saat komputer Anda dinyalakan. Sekali lagi, lakukan penelitian Anda dan pastikan Anda tidak menonaktifkan hal-hal penting untuk Windows.

  • Apakah Anda menjalankan server yang terhubung dengan komputer lain, seperti server web atau database? Apakah ada peningkatan jumlah mesin yang terhubung ke komputer Anda dari waktu ke waktu?

  • Anda mungkin terkena virus atau terkena beberapa jenis malware. Pastikan program antivirus menjadi bagian dari jadwal pemeliharaan Anda.

    • lebih banyak penggunaan memori
    • lebih banyak penggunaan disk
    • lebih banyak penggunaan cpu
  • mesin pemindai virus perlu memindai lebih banyak area (memori, disk, registri, virtualisasi, cpu, tumpukan, dll.)
  • perangkat lunak pemindai virus perlu melindungi dirinya sendiri agar tidak dapat dihentikan oleh virus
    • lebih banyak penggunaan memori
    • lebih banyak penggunaan cpu

Untuk memperluas jawaban @Dennis, bahkan jika Anda adalah mendefragmentasi ada file tertentu, terutama pada versi Windows yang lebih lama, yang tidak dapat didefrag selama operasi normal. Mereka hanya dapat didefragmentasi selama boot Windows. File tersebut termasuk kumpulan registri dan file log. Jadi, meskipun Anda menjaga registri tetap bersih, registri mungkin masih terfragmentasi di disk.

Pengindeksan karena hasil pencarian cepat adalah area lain yang cenderung menjadi lebih haus sumber daya seiring berjalannya waktu.

Jumlah font yang diinstal juga dapat memengaruhi kinerja. Font baru mungkin diinstal dengan perangkat lunak baru tetapi mungkin tidak dihapus saat Anda menghapus perangkat lunak.

Log sistem menjadi lebih besar seiring waktu dan dapat menjadi terfragmentasi pada disk.
