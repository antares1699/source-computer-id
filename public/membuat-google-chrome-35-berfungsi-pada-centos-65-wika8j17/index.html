Membuat Google Chrome 35 berfungsi pada CentOS 6.5

Saya mencoba menginstal Google Chrome 35 di CentOS 6.5 saya. Setelah googleing selama beberapa jam saya menemukan bahwa Google telah memutuskan untuk tidak lagi mendukung rilis EL 6. Karena itu saya mendapatkan kesalahan berikut.

Kesalahan: Paket: google-chrome-stable-35.0.1916.153-1.x86_64 (google-chrome) Membutuhkan: libstdc ++. So.6 (GLIBCXX_3.4.15) (64bit)

Sebagai default libstdc ++. So.6.0.13 hadir dengan OS dan yang terbaru adalah gcc 4.9 dan libstdc ++. So.6.0.20. Saya mengunduh sumber gcc terbaru untuk dibuat. Itu dibangun dengan sukses dan saya mengujinya. tapi

gcc --version

mengembalikan 4.4.7. Saya memeriksa direktori instal saya yang saya setel saat melakukan konfigurasi adalah / usr / bin / gcc49 / bin / gcc. ketika saya menjalankan perintah yang sama di sini itu menunjukkan 4.9. Saya kira ini berarti bahwa libstdc ++ versi terbaru juga telah dipatuhi dan dibangun sebagai bagian dari proses.

Masalah saya sekarang adalah setelah mendapatkan versi terbaru dari gcc google chrome masih mengarah ke versi lama. apakah ada cara sederhana untuk mengarahkan pemeriksa ketergantungan rpm untuk menunjuk ke direktori tertentu untuk diinstal.

Terima kasih
