Cara membuat g ++ dari sumber * dan * meneruskan opsi ke `configure`

Saya ingin membangun kembali g ++ dengan pengaturan khusus saya sendiri.

Inilah yang saya lakukan sejauh ini:

sudo aptitude install build-dep g++-4.2
mkdir trial && cd trial
apt-get source g++-4.2

Sekarang saya ingin mengkonfigurasi pengaturan khusus saya. Untuk g ++ ini biasanya dilakukan oleh mis.

sh configure --prefix=/home/voku/testing

Namun sayangnya tidak ada skrip configure di dalam direktori unduhan (dan juga tidak ada konfigurasi.ac).

Apakah ada langkah tambahan yang saya perlukan untuk mendapatkan skrip configure?

Apa yang sebenarnya saya cari adalah cara untuk mengatakan:

  • dapatkan sumbernya
  • terapkan semua tambalan yang Anda suka
  • buat semuanya siap untuk dibuat

EDIT : Saya juga bertanya tentang Stack Overflow - tapi sepertinya tidak ada yang tahu di sana.

EDIT : Mungkin saya bisa menjalankan salah satu alat yang benar-benar membuat paket .deb untuk g ++.

Setelah itu - itulah yang saya asumsikan - akan ada skrip konfigurasi yang diperlukan.

Tapi kedengarannya mengerikan:

  • buat paket g ++ .deb
  • bersihkan semuanya lagi
  • kemudian gunakan skrip konfigurasi yang dihasilkan dengan opsi saya sendiri
  • lalu bangun lagi

Jika Anda ingin membangun ulang paket Debian dengan opsi yang berbeda, ini akan sedikit lebih rumit, karena pengemasan Debian dari GCC cukup rumit. Secara umum, Anda akan mencari panggilan configure di debian/rules, tetapi dalam paket sumber gcc-4.2 tindakan menarik sebenarnya terjadi pada debian/rules2. Cari variabel bernama CONFARGS di sana. debian/README.maintainers juga merupakan bacaan yang baik jika Anda ingin melakukan peretasan serius pada paket tersebut.
