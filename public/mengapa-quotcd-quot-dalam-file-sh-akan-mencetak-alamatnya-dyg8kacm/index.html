mengapa "cd -" dalam file .sh akan mencetak alamatnya

Saya menggunakan "cd -" untuk beralih antar folder. Suka:

cd /etc/                                                                                                                          
cd -

tapi kenapa "cd -" akan mencetak alamatnya (seperti pwd)? Bagaimana saya bisa menghentikannya mencetak alamat?

SUS4 mengatakan :

Jika - digunakan sebagai operan, ini harus setara dengan perintah:

    cd "$OLDPWD" && pwd

yang berubah ke direktori kerja sebelumnya dan kemudian menulis namanya.

Jadi, jika Anda tidak ingin setengah dari perilaku pencetakan, lakukan saja bagian pertama sendiri:

cd "$OLDPWD"

Atau ekuivalen ekspansi tilde

cd ~-

Seringkali jika Anda membutuhkan ini, pushd/popd adalah pilihan yang lebih baik meskipun kurang portabel.

Cara paling sederhana adalah tidak menggunakannya. Jalankan perintah di shell terpisah:

(cd /etc/; ... )

Ini tidak mengubah direktori kerja saat ini dari shell 'main'. Ini biasanya lebih kuat daripada menggunakan OLDPWD karena Anda dapat memiliki banyak perintah cd di sini seperti

(cd /etc; ... ; cd /tmp; ... )
