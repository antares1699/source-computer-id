Apa arah komputasi cloud yang muncul sebenarnya adalah tentang perusahaan yang mengalihdayakan layanan ke perusahaan pihak ketiga, alih-alih memiliki ruang server dan staf TI mereka sendiri, sehingga mereka dapat memusatkan upaya mereka pada apa yang terutama mereka lakukan.

Ini seperti bagaimana pabrik dalam revolusi industri berhenti menghasilkan tenaga mereka sendiri menggunakan mesin uap dan mulai menggunakan listrik dari perusahaan pembangkit listrik.

Ini penjelasan yang bagus: http://www.cloudpro.co.uk/cloud-essentials/1209/what-cloud-computing-quick-guide-subject

Kata cloud computing dilemparkan sebagai kata kunci dan dapat berarti banyak hal yang berbeda bagi banyak perusahaan / orang yang berbeda. Bagi beberapa 4shared / rapidshare adalah komputasi awan.

Definisi komputasi awan yang lebih konkret adalah ketika tidak ada file yang menyentuh komputer Anda. Misalnya di google docs, file tidak hanya disimpan di server google, tetapi juga tidak pernah harus diunduh oleh Anda. Ini menghilangkan kebutuhan akan sistem file dan memungkinkan penggunaan sistem operasi seperti Google Chrome.

"Cloud" adalah kata kunci pemasaran yang memiliki arti samar yang biasanya dikaitkan dengan "melakukannya secara online" atau "layanan yang dapat diakses melalui Internet".

Arti yang mungkin dari "berbagi file komputasi awan" semuanya mencakup:

  • Sewa ruang penyimpanan di fasilitas colocation, letakkan server di dalamnya, dan konfigurasikan dengan perangkat lunak penyajian file yang sesuai (dengan mempertimbangkan keamanan). Ini bisa menjadi sesuatu yang sederhana seperti hanya mengekspos rsync melalui port terbuka, menjalankan server FTP dengan banyak akun, atau mengalami masalah untuk menginstal dan menjalankan (dan mungkin mengembangkan) aplikasi pengunggahan dan pengunduhan file berbasis Web / AJAX.

  • Sewa instance VM dari "penyedia komputasi awan" seperti Amazon dan lakukan hal yang sama, hanya saja tidak pada perangkat keras fisik.

  • Siapkan server (atau PC dengan perangkat lunak server) di rumah Anda dan lakukan hal yang sama, tetapi jauh lebih lambat karena hulu hunian Anda kemungkinan besar 1/10 dari hilir Anda, dan kemungkinan besar Anda tidak memiliki perangkat keras server yang kuat di rumah.

  • Dapatkan ponsel dengan sistem operasi terbuka atau "jailbroken" dan lakukan hal yang sama, tetapi JAUH lebih lambat karena 3G (dan bahkan 4G) memiliki latensi yang relatif tinggi, dan ponsel Anda kemungkinan besar merupakan CPU ARM inti tunggal dengan kurang dari 1GByte RAM.

Dalam arti tertentu, layanan seperti 4shared dan Rapidshare adalah berbagi file komputasi awan, tetapi perbedaan utamanya adalah A) mereka telah dikonfigurasi (dan memiliki skala) untuk menawarkan layanan kepada masyarakat umum, dan B) mereka umumnya hanya mendukung file transfer melalui browser Web.
