Tanpa boot dengan 4+ GPU

Saya memiliki bangunan yang ditujukan untuk penambangan. Saya menggunakan Ubuntu 16.04. Spesifikasi saya adalah sebagai berikut:

  • mobo: ASUS TUF Z270 Mark 1 LGA1151
  • CPU: Intel Pentium dual core G4600 3.6 LGA 1151 GHz
  • RAM: (2x) Patriot Memory Viper Elite Series DDR4 4GB 2400MHz
  • PSU: Corsair HXi Series, HX1200i, 1200 Watt
  • GPU: (5x) XFX AMD Radeon RX 580

Sistem bekerja dengan baik jika ada <= 3 GPU yang terhubung melalui PCIe VER 007 16x ke 1x anak tangga bertenaga.

Jika saya menghubungkan satu atau lebih GPU tambahan, saya mendapatkan satu kode bip, dan peringatan bahwa periferal seperti keyboard saya tidak dapat ditemukan, diikuti oleh layar hitam.

Setelah membaca sedikit, ada beberapa saran untuk mematikan Boot Aman. Saya melakukan itu tanpa hasil.

Anda harus mengaktifkan Opsi decoding 4g di motherboard Anda BIOS opsi untuk mem-boot sistem dengan lebih dari 4 GPU di papan. Opsi ini memungkinkan sistem untuk mengaktifkan atau menonaktifkan I / O yang dipetakan memori untuk perangkat PCIe 64-bit ke ruang alamat 4GB atau lebih besar, karena kartu VGA utama harus selalu dipetakan di bawah alamat 4GB.

Anda juga harus memperbarui BIOS , jika Opsi decoding 4g tidak tersedia di BIOS motherboard Anda. Jika Anda membutuhkan bantuan tentang cara memperbarui BIOS, Anda dapat memeriksa wiki Pembaruan BIOS . Setelah itu, dalam beberapa kasus, juga menonaktifkan grafik terintegrasi (jika ada) dapat membantu.
