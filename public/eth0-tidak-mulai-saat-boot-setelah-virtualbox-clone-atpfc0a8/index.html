eth0 tidak mulai saat boot setelah Virtualbox clone

Setelah mengkloning VM CentOS saya di Virtualbox (saya menggunakan opsi Reinitialize the MAC address of all network cards), VM asli dan klon keduanya sekarang memiliki 127.0.0.1 sebagai IP mereka.

Ada yang tahu bagaimana cara memperbaikinya? IP vbox dulu mengikuti IP jaringan saya (yaitu, jika host adalah 192.168.0.2 maka vboxnya adalah 192.168.0.3 tetapi sekarang vboxnya adalah 127.0.0.1) Saya menggunakan Bridged Adapter BTW.

EDIT: Oke, sekarang saya menyadari eth0 tidak berjalan saat start up. Pertanyaan saya sekarang adalah bagaimana cara mengaktifkan eth0 saat startup? Perhatikan bahwa ini digunakan untuk memulai secara otomatis sebelum saya melakukan klon.

Untuk pertanyaan Anda ini how enable eth0 on startup?

Buka file /etc/sysconfig/network-scripts/ifcfg-eth0 itu

ubah ONBOOT=no

Untuk

ONBOOT=yes

jadi edit dua file

  1. /etc/udev/rules.d/70-persistent-net.rules
  2. / etc / sysconfig / network-scripts / ifcfg-eth0 - dari heavyd

ubah alamat MAC di kedua file agar sesuai dengan VM VirtualBox yang ditetapkan jika Anda mengklik klik kanan cloneVMvirutalBox => Setting => Jaringan => Adaptor 1 ketuk ada Advanced akan memberikan alamat MAC.

Semoga ini akan membantu ketika seseorang menemukan situs ini.

CentOS mengikat skrip konfigurasi jaringan ke alamat MAC tertentu, jadi ketika Anda mengubah alamat MAC Anda, pada dasarnya Anda merusak skrip tersebut. Anda dapat memperbaikinya dengan mengedit skrip sebagai root:

sudoedit /etc/sysconfig/network-scripts/ifcfg-eth0

Dalam skrip cari baris HWADDR baris. Perbarui nilainya agar sesuai dengan alamat MAC baru Anda dan kemudian reboot.
