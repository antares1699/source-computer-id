Mengapa NTFS melarang penggunaan tanda titik dalam nama direktori? [Tutup]

Di Linux, saya dapat memiliki folder bernama temp..

Jika saya mencoba melakukan mkdir temp. pada mesin Windows, periode trailing terpotong.

Jika saya mencoba menyinkronkan direktori dari Linux ke Windows yang memiliki tanda titik, direktori akan gagal (ditemukan di seafile oleh rekan kerja).

Mengapa demikian?

NTFS tidak melarangnya, hanya Win32 API yang mengizinkannya. Seperti yang dicatat Alex dalam komentar, jika Anda menggunakan \\?\... untuk melewati penguraian biasa dan memberikan jalur mentah ke API NT, Anda dapat membuat direktori dengan tanda titik.

(Mengapa ini berhasil? Artikel Win32 File Namespaces mengatakan: "Untuk file I / O, awalan" \? \ "ke string jalur memberi tahu Windows API untuk menonaktifkan semua penguraian string dan mengirim string yang mengikutinya langsung ke sistem file. Misalnya, jika sistem file mendukung jalur besar dan nama file, Anda dapat melebihi batas MAX_PATH yang sebaliknya diberlakukan oleh API Windows. ")

Adapun Mengapa periode trailing dihapus, sepertinya tidak ada dokumentasi resmi, tetapi itu mungkin harus melakukan sesuatu dengan kompatibilitas dengan program MS-DOS atau Windows 3.11 lama. Di dunia nama file 8,3, ekstensi kosong dan tidak ada ekstensi sama sekali adalah hal yang sama - dalam kedua kasus, nama file XYZ akan disimpan sebagai XYZ····· dan ekstensi sebagai ··· (titik mewakili byte nol), jadi mungkin ada program yang mengandalkan fakta ini. Ketika Windows 95 mendapat dukungan nama file yang panjang, itu mulai menyimpan nama dan ekstensi sebagai satu string, menyebabkan xyz dan xyz. menjadi nama file yang berbeda. Mungkin harus menghapus periode tambahan untuk menghindari kerusakan program semacam itu.
