Bendera "Nonaktifkan komposisi desktop" menonaktifkan efek Aero (batas kaca transparan) selama aplikasi dijalankan, yang sepertinya hanya ada di "tema visual"

Benar; itu memberitahu Windows untuk tidak menggunakan kaca Aero atau Manajer Jendela Desktop (atau dengan ekstensi, akselerasi perangkat keras) saat program itu dijalankan. Oleh karena itu, ketika program yang dikonfigurasi seperti itu dijalankan, semua jendela di layar berbagi hal yang sama bingkai penyangga alih-alih masing-masing mendapatkan miliknya sendiri. Ini hanya diperlukan dengan sedikit program yang mengharapkan hal-hal menjadi cara yang sangat spesifik dan menggunakan metode hard-code (dan mungkin tidak terdokumentasi) daripada menggunakan antarmuka yang disediakan. Untungnya ini umumnya terbatas pada program yang sangat lama.

Apa sebenarnya "tema visual" itu? Dari apa yang bisa saya tebak, "tema visual" adalah gaya batas jendela kustom di sekitar area konten jendela yang dibuat menggunakan hook dan panggilan API WDM, meskipun saya siap untuk dikoreksi.

Anda benar; tema atau gaya visual yang mewah, rendering grafis dari antarmuka Windows chrome (batas serta kontrol seperti tombol, tombol radio, kotak centang, scrollbar, dll.)

Di Windows XP, itu adalah tema Luna (gambar 3) yang memiliki tepi Windows yang tebal, bulat, biru dan glossy, merah [x]. Di Windows Vista dan 7, ada tema Aero (gambar 4) yang mencakup "Kaca" efek transparansi, serta tema Windows Basic (gambar 5) yang tidak, masih merupakan tema / gaya.

Bagaimana menonaktifkannya memengaruhi sistem?

Opsi ini hanya memiliki penggunaan Windows, matikan tema dan gunakan tema Windows Classic (gambar 1) saat program sedang berjalan. Ini biasanya tidak diperlukan, tetapi sekali lagi, secara teoritis mungkin ada program (mungkin sangat tua) yang akan macet jika dijalankan pada Windows bertema. Perhatikan bahwa tema Windows telah ada cukup lama (Windows XP adalah yang pertama menyertakannya dan dirilis pada 2001), jadi dalam banyak kasus, hanya program yang ditulis sebelumnya yang akan bermasalah kecuali seperti yang disebutkan sebelumnya, mereka baru tetapi buruk. terprogram (Saya menggunakan "buruk" di sini untuk apa pun yang ditulis menggunakan panggilan langsung dan semacamnya, bukan API; mungkin program yang tidak kompatibel dapat ditulis dengan baik, tetapi hanya dikhususkan).

tetapi mengubah opsi "Nonaktifkan tema visual" sepertinya tidak akan menghasilkan apa-apa.

Untuk melihat efeknya, Anda memerlukan tiga kondisi agar benar:

  1. Windows harus dikonfigurasi untuk menggunakan tema (Anda tidak akan melihat apa pun jika saat ini disetel ke tema Klasik)
  2. Program harus memiliki opsi kompatibilitas yang dicentang
  3. Program harus berjendela sehingga Anda dapat melihatnya; jelas, Anda tidak akan melihat perubahan apa pun untuk program layar penuh

Efek dari opsi kompatibilitas lebih jelas untuk opsi Disable desktop composition daripada opsi Disable visual themes karena ketika komposisi desktop dinonaktifkan untuk suatu program, itu tetap mati secara universal hingga program itu ada (seperti bagaimana menggunakan program video yang menggunakan akselerasi perangkat keras akan mematikan Aero sampai ada). Namun, mematikan tema hanya memengaruhi apa yang sebenarnya ditampilkan di layar, jadi jika disetel untuk program layar penuh, Windows mematikan tema, tetapi Anda tidak dapat melihatnya karena program itu layar penuh, tetapi jika Anda menekan Alt + Tab untuk beralih ke desktop, ini akan menyalakannya kembali dan kemudian mati lagi saat Anda beralih kembali ke program. Ini hanya terlihat jika program target berjendela; maka Anda akan melihat bahwa semua jendela tidak bertema.

Kata kuncinya di sini adalah kesesuaian . Opsi ini ada untuk menyediakan mode kompatibilitas untuk beberapa program lama yang tidak akan berjalan dengan benar dengan elemen windows modern. Ini adalah tradisi Microsoft untuk mengabaikan prinsip-prinsip kompatibilitas mundur dan menggantinya dengan opsi seperti ini. Perhatikan bahwa terkadang opsi yang sama berguna untuk baru program yang memiliki gaya visual mewah yang bertentangan dengan tema windows modern. Ini biasanya terjadi karena pengkodean yang buruk / tidak fleksibel atau perangkat lunak yang ketinggalan zaman.

Gunakan opsi ini hanya jika Anda memerlukan versi aplikasi tertentu dan tidak dapat menjalankannya dalam sistem Anda. Cara yang lebih disukai untuk menyelesaikan masalah kompatibilitas adalah dengan meningkatkan aplikasi Anda ke versi terbaru. Opsi ini mungkin membantu menyelesaikan masalah tetapi tidak menjamin apa pun.

Penting untuk memahami perbedaan antara menonaktifkan tema visual untuk satu aplikasi (melalui pintasan atau pengaturan khusus yang dapat dieksekusi) dan menonaktifkan tema visual secara global untuk pengguna (pada Advanced System Settings>Advanced>Performance). Pilihan kedua bukan untuk kompatibilitas melainkan untuk peningkatan kinerja seluruh sistem. Tema visual di versi baru windows sangat boros sumber daya.

Perhatikan bahwa opsi kompatibilitas hanya tersedia di Windows hingga versi 7. Dari versi 8, fungsionalitas ini dinonaktifkan (lihat diskusi di forum Microsoft )
