Apa itu string build Windows dan bagaimana menemukannya?

Build awal sistem operasi Microsoft Windows memiliki "string build" yang telah dikenal yang dilampirkan padanya untuk tujuan identifikasi. Situs web yang berfokus pada pengujian perangkat lunak Beta biasanya memiliki daftar lengkap (kurang lebih) dari string build ini.

Berikut ini hanya beberapa contoh tampilan string build ini untuk Windows 10, dalam urutan rilisnya ke penguji Beta.

  • 6.4.9879.fbl_release.141103-1722
  • 10.0.9926.fbl_awesome1501.150119-1648
  • 10.0.10014.winmain_prs.150205-1859
  • 10.0.10061.fbl_impressive.150410-2039
  • 10.0.10163.winmain_prs.150630-1700
  • 10.0.10166.fbl_impressive.150703-1818
  • 10.0.10176.16384.th1.150705-1526
  • 10.0.10240.16384.th1.150709-1700

Saya memahami bahwa kelompok angka pertama mewakili nomor versi mayor, kelompok kedua adalah minor, dan kelompok ketiga adalah nomor build. Ketiganya dapat dengan mudah diungkapkan dengan menjalankan applet winver dari prompt Run. Tapi apa arti dari gobbledygook berikutnya?

Apa arti dari fbl_release, fbl_awesome, winmain_prs, dan th1?

Set digit kedua dengan tanda hubung tampak seperti tanggal. Apakah itu tanggal waktu kompilasi?

Saya saat ini menjalankan Windows 10, versi rilis resmi yang saya dapatkan melalui Pembaruan Windows, sebagai bagian dari penawaran peningkatan gratis. Saya meningkatkan ke Windows 10 dari Windows 8.1. Alasannya adalah saya ingin membandingkan milik saya dengan yang terakhir dalam daftar itu.

Dengan menjalankan perintah systeminfo di cmd, saya mendapatkan bacaan ini:

Apa itu string build Windows?

Ini adalah sekumpulan karakter yang secara unik mengidentifikasi build Windows.

Apa yang membangun? ...

Build mengacu pada membangun solusi. Yaitu: menyusun kode sumber menjadi kode objek dan menghubungkan kode objek ke dalam kode mesin yang dapat dieksekusi. Ini adalah produk akhir dari upaya analisis, desain, dan pemrograman. Namun, ini mungkin bukan produk akhir, karena mungkin direvisi beberapa kali sebelum itu terjadi. Jika ya, itu dikenal sebagai rilis final, rilis emas, atau dalam kasus perangkat lunak Microsoft: RTM (rilis ke manufaktur).

Pada akhirnya, build adalah solusi untuk masalah yang dapat diselesaikan dalam perangkat lunak! Oleh karena itu lebih dikenal hanya sebagai perangkat lunak. Dalam hal ini adalah perangkat lunak sistem Windows - sistem operasi yang kemungkinan besar berjalan di komputer Anda sekarang dan membuatnya berguna. Anak-anak masa kini menyebutnya semua "aplikasi", awalnya merupakan bentuk singkat dari "aplikasi" atau "perangkat lunak aplikasi", tetapi sekarang juga memiliki arti lain. Grup hanya menyebutnya "program". Untuk menggunakan kembali pepatah Swedia, "anak yang dicintai memiliki banyak nama."

Itu juga merupakan nama konferensi pengembang Microsoft tahunan.

Bagaimana build string dapat dibaca di Windows?

String build disimpan secara internal sebagai nilai string di registri Windows. Ada dua jenis string build yang harus diperhatikan. Satu hanya disebut string build dan hanya menyertakan nomor build, jadi ini hanya numerik. Yang lainnya terkadang disebut string build lengkap, karena ini juga menyertakan nama cabang dan waktu kompilasi, atau lebih tepatnya build cap waktu.

Kunci berikut di registri Windows menampung string build lengkap dari sistem yang sedang berjalan.

HKLM\Software\Microsoft\Windows NT\CurrentVersion

Ada dua variabel yang menarik di sini:

BuildLab:   10240.th1.150729-1800
BuildLabEx: 10240.16412.amd64fre.th1.150729-1800

Apa arti dari build string?

Secara umum, string build berisi segmen berikut: versi mayor, versi minor, nomor build, nama cabang, tanggal pembuatan, waktu pembuatan.

Secara umum, skemanya terlihat seperti ini:

[major].[minor].[build].[branch].[date]-[time]

Saya akan memeriksa beberapa contoh sekarang.

  • 6.4.9879.fbl_release.141103-1722

Di sini versi mayor 6, versi minor 4, nomor build 9879. Nama cabangnya fbl_release. Dibangun pada 3 November 2014, pukul 17:22 waktu setempat.

  • 10.0.10014.winmain_prs.150205-1859

Versi mayor di sini adalah 10, versi minor 0, nomor build 10014. Ini adalah salah satu build Windows pertama yang mendapat nomor versi mayor 10. Microsoft membalik nomor versi mayor dari 6 menjadi 10 hanya satu build sebelumnya (build 10009) . Inilah yang memberi nama pada Windows 10. Nama cabangnya adalah winmain_prs dan ini dianggap sebagai cabang utama. Dibangun pada 5 Februari 2015, pukul 18:59 waktu setempat.

  • 10.0.10240.16384.th1.150709-1700

Di sini kami memiliki versi mayor 10, versi minor 0 dan nomor build 10240. Saya tidak begitu yakin apa yang diwakili oleh 16384. Nomor ini muncul di string build baik Windows 8.1, Windows 8, Windows 7, serta Windows Vista saat itu adalah build RTM atau hampir RTM. Saya menyebutnya "angka ajaib". Tapi saya rasa itu semacam "membangun mikro" atau membangun subversi. Yang ini dibuat pada 9 Juli 2015, 20 hari sebelum ketersediaan umum Windows 10. Itu didorong ke penguji Windows Insider pada 15 Juli, dan mulai diluncurkan pada 29 Juli bagi mereka yang memesan salinan pemutakhiran gratis Windows 10.

  • BuildLab: 10240.th1.150729-1800
  • BuildLabEx: 10240.16412.amd64fre.th1.150729-1800

Ini yang saat ini saya gunakan untuk mengetik jawaban ini. Ini juga merupakan RTM build, karena ini adalah build 10240. Tapi seperti yang bisa Anda lihat dari tanggal pembuatan, yang ini sebenarnya lebih dari yang sebelumnya yang dikeluarkan pada 15 Juli untuk penguji. Lihat tanggalnya! Tercantum 29 Juli 2015. Itu tanggal rilis Windows 10! Dan lihat angka ajaibnya! Ini 16412! Bandingkan dengan 16384. Jadi dengan ini saya tahu ini adalah rilis yang lebih baru.

Nilai BuildLabEx menyertakan sedikit lebih banyak informasi daripada contoh sebelumnya. Ini termasuk angka ajaib atau subversi build, dan nama arsitektur prosesor. Jadi dalam hal ini, itu adalah AMD64, a.k.a. EM64T atau Intel 64, yaitu: prosesor 64-bit. "Fre" kemungkinan besar adalah singkatan dari "free", yaitu: kernel produksi, sebagai lawan dari kernel pengembangan yang "diperiksa".

Jadi skema yang diperluas terlihat seperti ini:

[major].[minor].[build].[subversion].[architecture].[branch].[date]-[time]

Tapi apa arti nama cabangnya? ...

  • winmain_prs
  • fbl_impressive
  • fbl_awesome
  • fbl_release
  • th1

winmain_prs adalah cabang "Windows main". Ini adalah cabang utama, stabil dan siap untuk rilis yang dikembangkan dari semua versi Windows yang lalu dan yang akan datang (setidaknya dalam kasus keluarga Windows NT). "Prs" mungkin berarti mitra, atau produksi siap dan stabil. Saya benar-benar hanya menebak-nebak! Tidak banyak informasi tentang ini di web.

Saya cukup yakin bahwa "fbl" adalah singkatan dari "build masa depan". Ini sedang dalam proses, cabang yang tidak stabil. Mereka menggunakan nama-nama lucu seperti "mengesankan", "luar biasa" dan "rilis" (yang terakhir tidak terlalu lucu). Menurut saya, "mengesankan" dan "mengagumkan" digunakan untuk menambahkan fitur baru, sedangkan "rilis" digunakan untuk men-debug dan menstabilkan sebelum diintegrasikan secara terbalik ke cabang utama.

Cabang "th1" baru. Saya tidak yakin apa artinya. Tetapi saya tahu bahwa hanya build RTM yang tergantung di cabang ini. Jadi ini mungkin semacam indikasi perubahan model rilis Windows. Saya mengacu pada pergeseran ke Windows sebagai layanan.

Pembaruan: Build 10525 baru-baru ini dirilis di cabang "th2". The "th" tampaknya merupakan kependekan dari "threshold" yang merupakan codename untuk Windows 10. "th1" tampaknya merupakan rilis publik umum, sedangkan "th2" adalah cabang rilis pratinjau orang dalam.

Mengapa string build berguna?

Mereka memungkinkan identifikasi yang tepat dari build Windows, hingga ke cabang dan waktu kompilasi. Hal ini membuat pelacakan perubahan kecil menjadi sangat mudah, karena setiap perubahan pada kode, betapapun kecilnya, yang dikompilasi, ditautkan, dibuat, dan dirilis juga mendapatkan string build baru yang mengidentifikasinya. String build sebagian besar berguna untuk penguji Beta. Karena perubahan kecil dan sering pada kode dilakukan selama fase pengujian siklus pengembangan.

Setelah rilis final dibuat, string build lengkap menjadi sedikit menarik. Sebaliknya, hal utama yang menarik adalah nomor mayor, minor, dan build. Ini membantu mengidentifikasi "versi" dari sistem Windows yang sedang berjalan. Inilah yang digunakan aplikasi Windows pihak ketiga untuk mengidentifikasi versi Windows tempat mereka diinstal atau dijalankan. Ini berbeda dari nama pemasaran Windows, seperti "Windows XP" atau "Windows Vista".

fbl_release

dan acara serupa dari cabang mana yang dirilis. Misalnya jika Anda adalah mitra Microsoft yang mengembangkan aplikasi (ISV), Anda mendapatkan build dari cabang Impressive yang Anda tampilkan di postingan Anda. Cabang lain digunakan untuk mitra OEM dan winmain biasanya merupakan cabang utama tempat semua tanda digabungkan di beberapa titik sebelum pengiriman RTM.
