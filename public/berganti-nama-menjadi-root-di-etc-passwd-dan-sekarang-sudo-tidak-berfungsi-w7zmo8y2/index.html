Berganti nama menjadi root di / etc / passwd dan sekarang sudo tidak berfungsi

Saya tidak sengaja mengubah baris untuk root di / etc / passwd dan mengubah namanya. Untuk memperbaikinya saya mencoba menggunakan sudo untuk memodifikasi file ini, tetapi sudo tidak mengizinkan saya karena sudo tidak dapat menemukan pengguna root sejak saya mengganti namanya.

Saya saya di ubuntu.

Coba:

sudo -u new_wrong_root_user

Jika Anda tidak bisa, reboot di live-cd, lalu di terminal:

mkdir target
mount /dev/sda1 target # sda1 if / is the first partition of the first drive
vim target/etc/passwd
