Mengubah sekumpulan MP3 menjadi mono?

Saya memiliki banyak MP3 stereo yang ingin saya ubah ke mono. Apa cara terbaik untuk melakukannya? Saya lebih suka sesuatu yang membiarkan proses batch mereka. Saya ingin menjaga kualitas sedekat mungkin dengan aslinya. File saya juga dalam bitrate yang berbeda, jadi saya tidak ingin membuat semua file 320kpbs ketika beberapa hanya 128.

Juga, adakah cara cepat untuk melihat file mana yang stereo dari seluruh perpustakaan saya?

Mengonversi dari stereo ke mono berarti melakukan pengkodean ulang, jadi mempertahankan kecepatan bit yang sama tidak ada artinya. Faktanya, mengonversi MP3 128 kbit / dtk -> MP3 128 kbit / dtk baru akan menghasilkan kualitas yang sangat buruk, bahkan jika yang kedua adalah mono (dan karena itu memerlukan bit rate yang lebih rendah untuk kualitas subjektif yang sama).

Secara umum, saya akan menggunakan pengaturan Variable Bit Rate (VBR) untuk MP3, yang menargetkan kualitas tertentu dan membiarkan encoder mengatur kecepatan bit apa pun yang diperlukan (audio yang benar-benar senyap memerlukan bit rate yang lebih rendah daripada whalesong, yang membutuhkan bit rate lebih rendah daripada dubstep). Dari baris perintah, ffmpeg dapat mengonversi audio ke mono dengan opsi -ac 1, seperti:

ffmpeg -i input.mp3 -c:a libmp3lame -q:a 2 -ac 1 output.mp3

Lihat halaman ini untuk panduan penggunaan -q:a. Perhatikan bahwa tabel pada halaman itu ditujukan untuk audio stereo; laju bit sebenarnya yang akan Anda lihat akan sedikit lebih rendah. Biasanya, saya merekomendasikan 3-4, tetapi karena Anda melakukan encoding dari MP3 daripada CD asli, Anda harus membidik sedikit lebih tinggi.

Ini tentu saja dapat diotomatiskan dengan sangat mudah. Di Linux / OSX / seperti UNIX lainnya, untuk mengonversi direktori MP3:

for f in *.mp3; do ffmpeg -i "$f" -c:a libmp3lame -q:a 2 -ac 1 mono-"$f"; done

Untuk melakukannya secara rekursif:

find . -type f -name "*.mp3" -exec ffmpeg -i '{}' -c:a libmp3lame -q:a 2 -ac 1 mono-'{}' \;

Jika Anda memiliki GNU Parallel dan mesin multi-inti, Anda mungkin menemukan ini berguna:

find . -type f -name "*.mp3" | parallel ffmpeg -i {} -c:a libmp3lame -q:a 2 -ac 1 mono-{}

Lame dapat mengonversi ke file ke mono menggunakan tombol -m, keuntungannya adalah pelestarian tag meta (Saya tidak yakin apakah ffmpeg dapat melakukannya juga).

Berikut adalah contoh yang agak rumit yang menentukan bitrate dari sebuah mp3 sebelum melakukan transcoding dengan salah satu opsi kualitas bitrate variabel.

cat s2mono.sh
#!/bin/bash

mp3file=$@

mp3size () {
    du -sk "$1" | awk '{print $1 * 8 }'
}
mp3length () {
    id3info "$1" | \
        awk '/=== TLEN/ { if ($NF > 0) { len=int( $NF/1000) }} END {print len}'
}
mp3rate () {
    echo $(( `mp3size "$1"` / `mp3length "$1"` ))
}

bitrate=`mp3rate "$mp3file"`
if [ $bitrate -gt 155 ]; then VBR='-V4'; fi
if [ $bitrate -gt 190 ]; then VBR='-V2'; fi
if [ $bitrate -gt 249 ]; then VBR='-V0'; fi

echo downsampling $mp3file
lame --silent $VBR -m m --mp3input "$mp3file" \
      "$(basename "$mp3file" .mp3 )-mono.mp3"

Saya menjalankan ini di folder di mana ukuran total semua mp3 adalah 80 MB dalam format stereo. Mp3 mono yang dihasilkan hanya menggunakan 32 MB.

 find . -iname \*mp3  -print0 |xargs -0 -L1 sh s2mono.sh

Membangun jawaban ini , Anda bisa mendapatkan solusi yang lebih portabel (tanpa ketergantungan tambahan, tidak ada masalah dengan tag ID3v1 vs ID3v2) hanya dengan menggunakan file dan sed. Saya juga mengisi lebih banyak pengaturan bitrate.

Dalam skrip ini, Anda harus menentukan file input dan output pada baris perintah.

#!/bin/bash
#
# Usage: (this script) stereo-input.mp3 mono-output.mp3
#
# concept via https://superuser.com/a/566023/253931
#

# gnu/linux
bitrate=`file "$1" | sed 's/.*, \(.*\)kbps.*/\1/'`

# osx
# bitrate=`afinfo "$1" | grep "bits per second" | sed 's/.*: \(.*\)000 bits per second.*/\1/'`

BR='-V9'
if [ $bitrate -gt  75 ]; then BR='-V8'; fi
if [ $bitrate -gt  90 ]; then BR='-V7'; fi
if [ $bitrate -gt 105 ]; then BR='-V6'; fi
if [ $bitrate -gt 120 ]; then BR='-V5'; fi
if [ $bitrate -gt 145 ]; then BR='-V4'; fi
if [ $bitrate -gt 170 ]; then BR='-V3'; fi
if [ $bitrate -gt 180 ]; then BR='-V2'; fi
if [ $bitrate -gt 215 ]; then BR='-V1'; fi
if [ $bitrate -gt 230 ]; then BR='-V0'; fi
if [ $bitrate -gt 280 ]; then BR='-b320'; fi

echo "mono-izing file with detected bitrate '$bitrate': $1"
lame --silent $BR -m m --mp3input "$1" "$2"
