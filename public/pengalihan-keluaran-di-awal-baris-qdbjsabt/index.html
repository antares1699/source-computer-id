Pengalihan keluaran di awal baris

Saya terbiasa menentukan pengalihan keluaran di akhir baris seperti:

echo blah > output.txt

Baik di Windows dan * nix.

Baru-baru ini saya melihat semakin banyak contoh yang dialihkan di awal baris, seperti:

> output.txt echo blah

Sekali lagi, ini tampaknya berfungsi baik di Windows dan * nix.

Apakah ada keuntungan untuk mengarahkan keluaran di awal baris? Apakah itu berperilaku identik dengan mengarahkan di akhir baris atau adakah perbedaan? Mengapa ada dua pilihan?

Untuk contoh sederhana seperti itu, tidak masalah di mana pengalihan terjadi. Ini adalah pilihan gaya pengkodean di sini. Itu Status manual Bash :

Operator pengalihan (...) mungkin mendahului atau muncul di mana saja dalam perintah sederhana atau mungkin mengikuti perintah.

Apa itu perintah sederhana ?

Perintah sederhana adalah jenis perintah yang paling sering ditemui. Ini hanya urutan kata-kata yang dipisahkan oleh tanda kosong, diakhiri oleh salah satu operator kontrol shell (lihat Definisi ). Kata pertama umumnya menentukan perintah yang akan dijalankan, dengan kata-kata lainnya menjadi argumen perintah itu.

Urutan perintah pengalihan penting bila ada lebih dari satu, jelas.

Misalkan Anda ingin menulis count: 1 ke sebuah file.

Untuk Windows, echo count: 1>file.txt akan menulis count : ke file (tanpa 1) karena 1 diartikan sebagai stdout. Anda bisa menggunakan echo count: 1 >file.txt, tapi kemudian ada spasi tambahan yang tidak diinginkan. Menempatkan pengalihan di depan menghindari masalah: >file.txt echo count: 1.

Tidak yakin, apakah ini juga berlaku untuk 'nix, tapi saya menduga demikian.
