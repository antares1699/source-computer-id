Dapatkah saya meniru beberapa perangkat keras untuk mesin virtual dengan menjalankan program pada sistem operasi host?

Begitu...

Ada mesin virtual yang pada dasarnya menjalankan sistem operasi di beberapa lingkungan simulasi (yang mungkin atau mungkin tidak sesuai dengan perangkat keras sebenarnya yang ada saat ini). Ada beberapa kemungkinan untuk menghubungkan perangkat keras nyata (misalnya drive USB) ke mesin virtual.

Lalu ada beberapa yang disebut "perangkat USB virtual" (misalnya Arduino Leonardo) yang dapat diprogram untuk merespons seperti perangkat USB yang sebenarnya, tetapi pada dasarnya hanyalah sebuah chip mikro di ujung lain kabel USB yang menjalankan beberapa program ( mereka mungkin atau mungkin tidak terhubung ke komponen lain, untuk pertanyaan ini, anggap saja tidak).

Tentu saja, seseorang dapat menghubungkan perangkat USB virtual ke VM (karena mereka adalah bagian dari perangkat keras yang sebenarnya).

Apakah mungkin untuk melewati bagian perangkat keras dan menjalankan program khusus pada sistem operasi host yang terhubung ke mesin virtual seolah-olah itu adalah perangkat keras nyata (katakanlah, sebagai plugin untuk perangkat lunak virtualisasi, atau menggunakan program berdiri bebas menggunakan beberapa API)?

Jika ya, apakah ada solusi untuk referensi lebih lanjut?

Jika tidak, mengapa mesin virtual tidak menyediakan kemampuan ini (misalnya, apakah ada batasan perangkat keras)?

Jawaban singkatnya: Ya.

Ini sepenuhnya tergantung pada Hypervisor, sungguh.

Semua perangkat keras virtual disediakan untuk mesin virtual oleh Hypervisor. Jika Hypervisor mendukung plugin dan penambahan perangkat keras virtual melalui plugin tersebut, maka ya, Anda dapat menambahkan perangkat keras virtual ke lingkungan Anda dengan cara ini.

VMware's ESXi mendukung ini, dalam pengalaman pribadi saya. Anda dapat mengunduh dan menginstal perangkat jaringan virtual seperti switch dari Cisco, dan saya yakin saya ingat pernah melihat perangkat keras khusus lainnya yang tersedia di produk itu juga, meskipun sudah beberapa tahun sejak saya mengelola tumpukan virtual VMware.

Saya tidak tahu apakah ada Hypervisor lain yang mendukung ini, tetapi saya rasa HyperV Microsoft mungkin juga melakukannya, karena mereka mencoba mempertahankan paritas fitur dengan VMware sebanyak mungkin.

Saya sarankan melihat berbagai API VMware sebagai tempat yang baik untuk memulai.

Jika Anda siap menggunakan hypervisor tertentu, Anda harus membaca dengan teliti dokumentasi untuk sistem itu dan menentukan apakah mereka memiliki kemampuan yang serupa. Setiap hypervisor akan berbeda, dan perangkat lunak yang ditulis untuk satu hypervisor tidak mungkin berfungsi di hypervisor lain tanpa modifikasi ekstensif.
