Bagaimana ffmpeg menentukan bitrate streaming individu?

Misalnya, saya memiliki video yang menampilkan ffmpeg

  Duration: 00:46:43.72, start: 0.540000, bitrate: 2593 kb/s
    Stream #0.0[0x1e0]: Video: mpeg2video (Main), yuv420p, 720x576 [PAR 16:15 DAR 4:3], 9000 kb/s, 25 fps, 25 tbr, 90k tbn, 50 tbc
    Stream #0.1[0x80]: Audio: ac3, 48000 Hz, stereo, s16, 448 kb/s

Ukuran file adalah 908843008 byte, jadi 2593 kb / s sudah benar. Tetapi mengapa dikatakan bahwa bitrate streaming video adalah 9000 kb / s?

Aliran MPEG biasanya menggunakan pengkodean bitrate variabel untuk menghemat ruang di bagian video yang mudah dienkode, sehingga memungkinkan peningkatan kualitas. FFMpeg melaporkan bitrate yang disimpan di header urutan aliran video, yang merupakan bitrate maksimum di seluruh file (atau, dalam banyak kasus, bitrate maksimum yang dikonfigurasikan encoder - file sebenarnya mungkin tidak memiliki bagian yang menggunakan bitrate setinggi ini). Nilai 9000kbps adalah tipikal streaming MPEG2 dari (atau ditujukan untuk) DVD: DVD memiliki bitrate gabungan video & audio maksimum 9800kbps, jadi ini memungkinkan ~ 500kbps untuk audio dan beberapa cadangan untuk paket navigasi & overhead lainnya.

Secara umum, ffmpeg menggunakan file header untuk informasi di tempat yang bisa (saat mengenali format header), dan sebaliknya kembali ke penghitungan dari durasi dan ukuran file.

Format pembungkus (mis. Mov, mp4) tidak memiliki bitrate-nya sendiri - bitrate adalah karakteristik streaming individual - tetapi memiliki durasi. ffmpeg kemudian dapat menghitung bitrate dari ukuran file dan durasi.

Namun, streaming itu sendiri akan memiliki bitrate yang ditentukan di header streaming, sehingga dapat dibaca langsung dari metadata. Ini berarti bitrate mungkin salah ditentukan di header, atau mungkin 9000kb / s di awal, dan menurun ke bitrate yang lebih rendah, atau aliran mungkin terpotong sehingga file tidak lengkap, oleh karena itu matematika tidak berfungsi seperti diharapkan.

Jika Anda mengekstrak komponen video (misalnya: ffmpeg -i myfile.mov -vcodec copy -an testfile.m2v, penggunaan berbeda-beda tergantung versi perangkat lunak), Anda mungkin dapat menyelidiki lebih lanjut dan mencari tahu dari mana kecepatan bit yang ditampilkan berasal.
