Apa persamaan Linux dari FAT dan bagaimana kaitannya dengan memblokir perangkat?

Baru mengenal Linux dan saya mengalami kesulitan memahami beberapa konsep OS dasar. Apa versi Linux dari FAT, dan bagaimana kaitannya dengan apa yang disebut "blok" atau perangkat blokir ini?

Di Linux, Anda memiliki banyak sistem file yang berbeda untuk dipilih: ext3, ext4, ext2, xfs, ReiserFS, ReiserFS4, bahkan FAT dan NTFS.

Perangkat blok sesuai dengan hard disk Anda dan / atau partisi di atasnya (sda vs. sda1), dan di sana Anda meletakkan sistem file pilihan Anda. Setelah itu, Anda dapat memasang perangkat itu ke /mnt, ke /usr atau di mana pun Anda membutuhkannya.

FAT adalah sistem file, cara mengatur penyimpanan disk dengan struktur tersembunyi yang memungkinkan sistem operasi untuk menemukan data. Kebanyakan distribusi Linux secara native menggunakan sistem file bernama ext4, namun Linux dapat bekerja dengan banyak sistem file yang berbeda, termasuk FAT.

Kata filesystem juga digunakan untuk memaksudkan partisi disk bernama tertentu (misalnya partisi boot). Ini juga digunakan untuk mengartikan keseluruhan kumpulan dari filesystem yang saat ini sedang digunakan ("dipasang") oleh sistem operasi komputer.

Dalam sistem operasi mirip Unix, seperti Linux, (hampir) semuanya adalah file. Ada file yang mewakili disk dan partisi disk. File-file ini disebut file perangkat dan berada di folder / dev. Perangkat dikategorikan ke dalam perangkat mode blok dan perangkat mode karakter tergantung pada cara penulisan atau pembacaannya.

Melengkapi jawaban benar lainnya: filosofi Unix (diwarisi oleh Linux) adalah bahwa "semuanya adalah file". Semuanya (untuk perkiraan pertama) adalah perangkat blok, termasuk disk tetapi juga printer, perangkat USB, RAM ....

FAT adalah salah satu cara untuk mengatur data dalam partisi disk. Apa yang Anda lihat sebagai huruf drive di DOS / Windows (C :, D :, dll.) Adalah partisi disk. Linux dapat menggunakan disk FAT, tetapi juga disk yang diatur dengan cara berbeda seperti yang dijelaskan @glglgl. Perhatikan bahwa Windows mengenali partisi yang sama dengan Linux, jika Anda menghubungkan hard disk yang sama ke keduanya (tidak pada waktu yang sama!). Dan seperti di Windows, di mana Anda dapat menetapkan partisi disk yang sama dengan D: atau E: atau Z :, Anda dapat memasangnya di Linux sebagai / media / mydisk atau / media / externaldisk atau / mnt / secureisk atau apa pun yang Anda inginkan.

Ada banyak komplikasi (misalnya RAID dapat mengenali beberapa disk fisik sebagai satu partisi) tetapi itulah penjelasan dasar untuk kasus-kasus sederhana.
