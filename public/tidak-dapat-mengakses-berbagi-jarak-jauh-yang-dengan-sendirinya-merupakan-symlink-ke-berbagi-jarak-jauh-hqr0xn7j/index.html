Tidak dapat mengakses berbagi jarak jauh yang dengan sendirinya merupakan symlink ke berbagi jarak jauh

Saya memiliki skenario ini:

Domain A:

  1. Server file yang disediakan oleh pihak ketiga, yang menyediakan volume yang tersedia melalui CIFS. Server file ini tidak memiliki alamat IP eksternal. izin ditetapkan untuk semua orang.
  2. Seperangkat mesin Windows yang dapat mengakses server file.

Satu mesin, menjalankan Windows 2012, saya membuat tautan simbolis dari Drive C: ke folder di server file. Saya kemudian membuat folder ini sebagai dibagikan. Saya juga memastikan semua jenis tautan diizinkan:

C:\>fsutil behavior query SymlinkEvaluation
Local to local symbolic links are enabled.
Local to remote symbolic links are enabled.
Remote to local symbolic links are enabled.
Remote to remote symbolic links are enabled.

Ketika saya mencoba mengakses share ini dari komputer lain di domain yang sama, menjalankan Windows 2012 atau Windows 2003, saya mendapatkan pesan kesalahan

… tidak dapat di akses. Anda mungkin tidak memiliki izin untuk menggunakan ini Sumber daya jaringan. hubungi administrator server ini untuk mencari tahu jika Anda memiliki izin akses.

Tautan simbolis tidak dapat diikuti karena tipenya dinonaktifkan.

Saya perlu bekerja dengan cara ini karena pada akhirnya saya perlu mengakses share dari mesin yang berjalan pada domain yang berbeda, yang dapat mengakses mesin tempat saya membuat symlink; Saya tidak dapat mengakses server file "asli" dari luar domain itu.

Pertama, Anda melakukan tautan simbolis dengan "mklink / d link_name nama_folder (Ini bisa menjadi nama UNC untuk sumber daya bersama).

Kemudian, Anda melakukan (pada setiap klien) "set perilaku fsutil SymlinkEvaluation R2R: 1" untuk mengaktifkan remote ke link simbolik jarak jauh.

Ini hanya berfungsi dari Windows 7 dan seterusnya. Ini tidak bekerja untuk XP.
