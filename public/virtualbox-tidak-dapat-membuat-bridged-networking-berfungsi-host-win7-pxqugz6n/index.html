VirtualBox: Tidak dapat membuat Bridged Networking berfungsi (host Win7)

Saya mencoba menyiapkan server LAMP virtual, termasuk berbagi file antara OS tamu (Server Ubuntu) dan OS host (Windows 7) menggunakan samba. saya berpikir masalah saya adalah bahwa saya tidak bisa mendapatkan jaringan Bridged (atau Host-Only) untuk bekerja di VirtualBox.

Saya dapat mem-boot VM Linux dengan baik dengan NAT, tetapi kemudian tidak dapat mengakses layanan apa pun di dalamnya secara langsung (kecuali setelah port-forwarding port 80) (pemahaman saya adalah bahwa port-forwarding berfungsi karena saya tidak menjalankan server web pada OS host, dan oleh karena itu dapat meneruskan lalu lintas ke port 80 yang tidak digunakan).

Saya tidak berpikir bahwa lalu lintas samba port-forwarding (dari host ke tamu) akan berfungsi karena saya pikir OS host menggunakan port tersebut.

Ketika saya mematikan NAT dan mengaktifkan jaringan yang dijembatani, saya mendapatkan kesalahan. VM gagal untuk boot, dengan dialog muncul (judul: VirtualBox - Error) yang mengatakan "Gagal membuka sesi untuk mesin virtual UbuntuServer. Kesalahan konfigurasi: Gagal mendapatkan alamat MAC (VERR_CFGM_VALUE_NOT_FOUND).

Saya berharap setelah ini diselesaikan maka samba akan berfungsi dengan baik :)

Setiap saran tentang ini akan sangat bagus (cara memperbaikinya akan sangat bagus, langkah pemecahan masalah selanjutnya akan bagus juga :))

(Saya mulai dengan default: NAT sebagai adaptor # 1, lalu menambahkan adaptor Bridged sebagai # 2. Setelah saya menyadari bahwa adaptor bridged / host-only tidak berfungsi, saya menonaktifkan adaptor # 1 (NAT) meninggalkan adaptor Bridged sebagai # 2)

Saya menyelesaikannya dengan berbeda dari jawaban-jawaban ini. Menggunakan VirtualBox 4.3.10, saya hanya perlu melakukan hal berikut:

  1. Siapkan VM saya untuk menggunakan DHCP atau IP Manual.
  2. DALAM pengaturan VBox, jika jaringan Nat tidak memiliki nilai, tambahkan jaringan NAT dengan pengaturan default.
  3. Dalam pengaturan VBox, jika Jaringan Hanya-Host kosong, tambahkan satu yang disebut Adaptor Ethernet Hanya-Host # 2 (Saya berasumsi # 1 juga akan berfungsi)
  4. Mulai ulang VM Anda. Ini akan mendapatkan IP dan sekarang akan dapat mengakses internet menggunakan jaringan Bridged per instance VM.

Cukup aneh dan tidak jelas bahwa ini tidak berhasil di luar kotak.

Tampaknya ada masalah dengan banyak proses VBoxNetDHCP yang dikurangi jumlahnya tetapi beberapa tetap ada di daftar proses server host ketika VirtualBox sedang ditutup. Jenis proses zombie.
Saya memiliki Server Host VirtualBox di Win7 dan dua tamu dengan Win7. Setelah seminggu, mesin tidak dapat dijangkau melalui jaringan. Saya telah mengonfigurasi 2 antarmuka:

  1. Jembatan jaringan
  2. Jaringan NAT

Perbaikan saya saat ini adalah mematikan semua VM tamu, menutup konsol manajemen VirtualBox dan bunuh satu VBoxNetDHCP di server host (yang lain berhenti sendiri). Kemudian saya memulai VirtualBox dan VM lagi dan internet kembali.

Sebelumnya reboot dari seluruh hostserver juga berfungsi, tetapi itu adalah solusi yang lebih buruk.
