Nonaktifkan umpan balik saat mengetik kata sandi pada prompt sudo

Ketika saya mengetikkan kata sandi saya pada prompt baris perintah sudo, ini menunjukkan * untuk setiap penekanan tombol baru-baru ini. Apakah ini perilaku default baru? Bagaimana cara mematikannya lagi?

Tidak ada opsi pwfeedback yang disetel dalam file /etc/sudoers saya. Saya menggunakan Linux Mint 19 yang pada dasarnya adalah Ubuntu Bionic, versi paket sudo adalah 1.8.21p2-3ubuntu1 dan sudo biner saya menunjukkan hash sha256 yang sama dengan yang ada di paket.

Terima kasih atas sarannya!

Saya menemukan ini di utas lain dan memastikan bahwa, jika pengembang Mint memperkenalkan kembali fitur buruk ini dalam pembaruan baru, modifikasi Anda sendiri akan menimpanya:

Jawaban yang diterima hanya dengan menghapus /etc/sudoers.d/0pwfeedback akan bekerja dengan tepat sampai pembaruan berikutnya untuk mintsystem mengembalikan file itu ke tempatnya.

Cara untuk kembali ke perilaku lama, yang direkomendasikan oleh orang yang bertanggung jawab untuk yang baru, adalah dengan mengganti pengaturan di file lain yang akan dibaca nanti:

echo 'Defaults !pwfeedback'|sudo tee /etc/sudoers.d/9_no_pwfeedback

Sumber: https://unix.stackexchange.com/questions/491173/linux-mint-how-to-set-the-terminal-password-to-be-invisible

Setelah beberapa penelitian lebih lanjut tentang file di direktori /etc/sudoers.d, saya menemukan jawaban yang benar di sini:

Linux Mint. Bagaimana cara mengatur kata sandi terminal menjadi tidak terlihat?

Ia mengatakan:

LinuxMint menambahkan perilaku tersebut pada /etc/sudoers.d/0pwfeedback.
Anda dapat melakukan seperti yang saya lakukan - hapus file karena hanya berisi penyesuaian itu:

sudo rm -rf /etc/sudoers.d/0pwfeedback

Menurut @ SyntaxxxErr0r, mengganti nama file juga berfungsi: jika diakhiri dengan ~ atau berisi karakter ., itu tidak akan diuraikan.

Baru-baru ini, @elsewhere menyarankan solusi lain yang lebih tahan masa depan dan stabil di seluruh pembaruan paket dan oleh karena itu saya memilih jawaban itu sebagai yang paling membantu.

Hanya untuk memperjelas di sini, sebenarnya, ya, mengganti nama file sebenarnya dapat menonaktifkan file yang sedang diurai. Artinya, jika memiliki ~ atau a. sesuai README yang ditemukan di direktori itu sendiri:

# This will cause sudo to read and parse any files in the /etc/sudoers.d 
# directory that do not end in '~' or contain a '.' character.

Saya pikir Anda bisa menghilangkan umpan balik dengan mengedit file /etc/sudoers.

Cari baris yang mungkin terlihat seperti ini:

Defaults env_reset,pwfeedback

dan hapus ,pwfeedback dari baris. Ambil salinan cadangannya terlebih dahulu.

Anda mungkin perlu mengatur ulang terminal Anda menggunakan perintah reset.

Jika ini tidak berhasil, kemungkinan lain adalah mengganti nama file /etc/sudoers.d/pwfeedback ke sesuatu yang lain, menyimpannya untuk berjaga-jaga.
