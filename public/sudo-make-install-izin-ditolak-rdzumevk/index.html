sudo make install: izin ditolak

Saya punya pertanyaan paling menjengkelkan tentang membangun dari sumber. Saya telah mencari jawaban untuk waktu yang lama dan tidak ada tempat yang dapat saya temukan ... Saya telah mengumpulkan perangkat lunak dari sumber sebelumnya dan ini hanya membingungkan saya.

Jadi saya mencoba menginstal python 2.7.2 dari sumber. Saya bisa melakukan ./configure sukses, juga make tampaknya berjalan dengan baik. Tetapi ketika saya melakukan sudo make install, saya mendapatkan berbagai kesalahan ...

Pertama, sudo make install memberi saya ini:

make: stat: GNUmakefile: Permission denied
make: stat: makefile: Permission denied
make: stat: Makefile: Permission denied
make: stat: install: Permission denied
make: *** No rule to make target `install'. Stop.

Jadi saya melakukan chmod +rx Makefile*. Tidak berhasil.

Kemudian, sudo ls . berkata

ls: cannot access .: Permission denied

Kemudian ls -d . mengatakan izinnya adalah drwxr-x---

Kemudian, sebagai tindakan putus asa, chmod +rx .. Itu memberi saya:

make: stat: Modules/config.c.in: Permission denied
make: *** No rule to make target `Modules/config.c.in', needed by `Makefile'. Stop.

Jadi sedikit kemajuan ... Apa yang terjadi di sini? Sepertinya masalah izin. Saya berasumsi bahwa sudo akan menjadi solusinya tetapi jelas ada sesuatu yang lain yang terjadi di sini ... Saya mencoba sudo -s tetapi saya mendapatkan masalah izin itu lagi ...

Saya menggunakan Ubuntu 10.04LTS.

Saya yakin apa yang Anda pukul sebenarnya adalah kebijakan NFS yang disebut "root squash". Tidak jarang pengguna membutuhkan root pada host tertentu, tetapi Anda tidak ingin mereka memiliki hak istimewa yang sama pada volume NFS bersama. Dengan demikian, memberikan akses root lokal tidak memungkinkan pengguna untuk menjalankan secara kasar pada volume NFS yang terpasang dan mengakses / memodifikasi file pengguna lain - server NFS akan memetakan uid root dari 0 hingga 65534 (nobody), itulah sebabnya Anda bahkan tidak dapat mencantumkan direktori di contoh Anda.

Jadi, itu memberi Anda beberapa opsi:

  1. Anda dapat mencoba membuat semua file dapat diakses oleh nobody dalam build Anda dengan melakukan chmod -R o+rw . di direktori tingkat atas, memberikan izin baca / tulis "orang lain" pada file Anda (mungkin atau mungkin tidak cukup - misalnya Anda mungkin perlu o+x direktori jika perlu mencarinya).
  2. Jika Anda memiliki akses ke file /etc/exports server NFS, maka Anda dapat menonaktifkan root squash (mungkin bukan ide yang bagus).
  3. Buat saja di direktori lokal seperti /usr/local/src, /usr/src, /tmp dll, atau buat saja direktori lokal Anda sendiri (Anda bahkan dapat menjadikannya milik pengguna Anda) seperti /scratch/myuser. Ini mungkin opsi yang menyebabkan sakit kepala paling sedikit.
