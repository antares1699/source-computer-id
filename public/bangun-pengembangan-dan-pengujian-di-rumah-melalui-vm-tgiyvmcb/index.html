Untuk menggunakan semua VM secara bersamaan di mesin yang sama, Anda memerlukan mesin besar seperti di jawaban lain, katakanlah 2GB untuk pengujian windows dan 512MB untuk pengujian linux + 3GB untuk windows + 1GB untuk linux Anda akan mencapai a Mesin 8GB dan Anda memerlukan cpu yang agak cepat untuk menjalankan semua 4 VM (tidak terlalu cepat) pada saat yang bersamaan.

Saya pikir Anda dapat bekerja dengan baik dengan mesin yang jauh lebih kecil dan saran saya adalah menjalankan 2 mesin kecil yang berbeda, masing-masing dengan 1 host dan 1 tamu, satu untuk pengujian dan satu untuk mengembangkan karena saya pikir lebih baik memiliki 2 layar, karena 4 mesin pada layar yang sama bukanlah impian saya (atau bahkan 5 jika Anda berencana untuk menjalankan 4 VM pada host seperti yang saya pikir saya mengerti) ... Jika Anda memiliki mesin kecil / lama seperti P4 atau Athlon, saya pikir Anda bisa letakkan 2 atau 3 GB di atasnya untuk menjalankan mesin uji. Saya menjalankan XP guest (virtualbox) pada host ubuntu pada mesin Athlon dan P4 1GB dan 2GB dengan sangat baik - tentu saja P4 2GB lebih baik! tetapi Athlon 1GB OK untuk bekerja dengan aplikasi Database di xp dan browser + Voip + openoffice + thunderbird di linux pada saat yang bersamaan. Jadi saya pikir itu akan cukup untuk mesin pengujian host + vm dengan RAM 1GB lebih banyak karena Anda mungkin ingin menjalankan windows 7 atau Vista di dalamnya. Untuk mesin pengembangan, itu tergantung pada apa yang akan Anda gunakan tetapi inti ganda dengan 4Gb seharusnya bagus untuk menjalankan kedua OS menurut saya.
Bagaimanapun, saya akan selalu menggunakan linux sebagai host karena berbagai alasan:

  • selalu lebih cepat dalam semua pengujian saya (dengan RAM rendah dan tinggi)
  • windows lebih bergantung pada RAM daripada linux dan di VM mudah untuk mengubah RAM yang dialokasikan
  • jika Anda mendapat masalah, linux hampir selalu boot dan Anda dapat menjangkau disk Anda setidaknya dengan terminal atau live-cd untuk memperbaikinya dan lebih mudah dan lebih cepat untuk menghapus, menginstal ulang (tanpa instalasi / aktivasi terbatas) dan mengkonfigurasi daripada windows jadi menggunakan windows di VM Anda "melindunginya" dari keharusan untuk mengkonfigurasi ulang dan menginstal ulang (jika Anda mengambil salinan dari HD virtual yang baru diinstal)
  • jika Anda perlu menguji konfigurasi yang berbeda, cukup salin HD virtual dan windows Anda baru lagi sehingga Anda dapat mengkonfigurasinya secara berbeda dan menguji tanpa menginstal ulang
  • linux jauh lebih mudah untuk mengelola / mengubah ukuran partisi dan membaca / me-mount partisi NTFS tanpa masalah
  • Windows boot di VM jauh lebih cepat daripada di mesin sebenarnya
  • Yang ini bersifat pribadi tetapi saya menggunakan linux lebih banyak daripada windows jadi saya tidak perlu menjalankan VM sepanjang waktu ...

Simpan disk virtual Anda di partisi terpisah (atau lebih baik, disk jika memungkinkan) untuk melindunginya dan akan lebih mudah untuk mengubah ukurannya jika diperlukan dan jika di disk terpisah, disk akan bekerja lebih cepat.
Jika Anda ingin menjalankan semua 4 mesin sebagai VM, saya akan menambahkan 512MB pada setiap mesin dan menggunakan linux sebagai host.
Mengenai jenis VM, saya menggunakan virtualbox dan saya sangat puas dengannya (mungkin karena saya sudah lebih mengetahuinya) tetapi mesin VMWare juga bagus. Saya menguji VMWare di awal dan kelihatannya lebih kuat tetapi saya pikir virtualbox lebih mudah dan lebih cepat.

Saya sangat setuju dengan saran untuk menggunakan ESXi; ia bekerja persis sebaik produk ESX lengkap, dan gratis.

Ini dapat berjalan pada perangkat keras tingkat konsumen (cukup periksa HCL), dan akan menggunakan perangkat keras Anda jauh lebih baik daripada solusi virtualisasi tipe-2 (yaitu yang berjalan di atas OS yang ada, seperti VMware Workstation).

Satu-satunya downside adalah Anda akan membutuhkan komputer lain untuk terhubung ke host Anda, karena tidak akan memiliki konsol fisik yang dapat digunakan ... tetapi itu benar-benar sepadan dengan biaya tambahannya.

Selain inti CPU dan RAM, pastikan juga untuk membeli disk sebanyak yang Anda bisa: empat VM yang tersebar di empat disk kecil berfungsi secara dramatis lebih baik daripada empat VM yang sama yang berada di satu VM besar.

Jika Anda hanya ingin menggunakan satu komputer dan masih mendapatkan kekuatan penuh hypervisor tipe-1 ( dan jika Anda bisa mendapatkan salinan berlisensi ...), Anda juga memiliki opsi untuk menginstal Windows Server 2008 R2 di komputer pengembangan Anda dan mengaktifkan peran Hyper-V. Bahkan jika itu akan terlihat seperti Anda menjalankan hypervisor dalam Windows (sama seperti VMware Workstation), yang sebenarnya justru sebaliknya, karena Hyper-V sebenarnya menempatkan Windows dalam hypervisor, memungkinkannya memanfaatkan perangkat keras sepenuhnya.

Pastikan saja tidak coba grafik 3D kelas atas, sebagai efeknya cukup tidak menyenangkan ; tetapi jika Anda ingin membaca dokumentasi, mengirim email, memeriksa web ... atau sebenarnya menggunakan VM Anda, itu akan berfungsi dengan baik, dan Anda tidak memerlukan komputer fisik lain.

Saya sarankan Anda mencoba Citrix's XenServer daripada VMWare's ESXi. Versi gratisnya memiliki serangkaian fitur yang lebih kaya (kloning dan templat adalah fitur-fitur mematikan jika Anda suka banyak menguji). Jangan salah paham, ESXi bagus tetapi XenServer hanya menawarkan lebih banyak dengan harga yang sama ($ 0) dan fitur-fiturnya ada jika Anda membutuhkannya.

Satu-satunya alasan saya memilih ESXi daripada XenServer adalah jika Anda memerlukan fitur PCI-Passthrough.

Sedangkan untuk perangkat keras, gunakan cpu quadcore dan banyak ram (> = 8GB). Pastikan Anda mendapatkan kartu jaringan yang kompatibel dengan produk vitualisasi Anda. Yang cocok adalah Intel NIC seperti Intel PWLA8391GTBLK Pro 1000GT, murah dan bekerja dengan hampir semua hal.
