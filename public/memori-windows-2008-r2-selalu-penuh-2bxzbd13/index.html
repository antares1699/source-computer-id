Memori Windows 2008 R2 selalu penuh

Kami memiliki Windows 2008 R2 di Server HP dengan RAM 24GB. Satu-satunya perangkat lunak penting yang diinstal adalah SQL Server 2008 R2 dan hampir tidak ada fitur atau peran yang diaktifkan di Windows ini.

Tapi masalahnya RAM selalu hampir penuh. Ketika kita melihat kinerja di Task Manager atau bagian memori dari Resource Monitor atau program pihak ketiga, kita melihat bahwa sekitar 97% -99% dari 24GB RAM digunakan tetapi penjumlahan dari semua proses 'memori yang digunakan kurang dari 1GB.

Jika persentase RAM ini digunakan, mengapa windows tidak dapat menunjukkan proses apa yang menggunakan jumlah memori ini? dan jika tidak, mengapa windows mengatakan bahwa RAM sudah penuh?

SQL Server menggunakan semua memori yang tersedia. Ini bukan bug, ini memang disengaja. Itu juga mengapa SQL Server harus diinstal pada mesin khusus (dan tidak bersama dengan IIS misalnya).

Saat berjalan dalam waktu lama, SQL Server menyimpan data sesuai dengan kueri yang dieksekusi, sehingga lebih cepat untuk dieksekusi nanti. Ini berarti bahwa saat Anda memulai server, penggunaan memori akan rendah, tetapi akan bertambah seiring waktu.

Anda dapat menyesuaikan batas memori untuk digunakan oleh SQL Server, tetapi Anda tidak boleh melakukannya kecuali Anda memiliki alasan yang serius.

Ini kutipan dari Cara menyesuaikan penggunaan memori dengan menggunakan opsi konfigurasi di SQL Server di situs web Dukungan Microsoft:

Saat Anda memulai SQL Server, penggunaan memori SQL Server dapat terus meningkat dan tidak berkurang, bahkan saat aktivitas di server rendah. Selain itu, Manajer Tugas dan Monitor Kinerja mungkin menunjukkan bahwa memori fisik yang tersedia di komputer terus menurun hingga memori yang tersedia antara 4 hingga 10 MB.

Perilaku ini saja tidak menunjukkan kebocoran memori. Perilaku ini normal dan merupakan perilaku yang dimaksudkan dari kumpulan buffer SQL Server.
