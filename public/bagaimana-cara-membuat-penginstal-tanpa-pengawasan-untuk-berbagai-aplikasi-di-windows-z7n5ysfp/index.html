Ini tergantung pada program apa yang ingin Anda instal. Ada berbagai sistem penginstal di luar sana (NSIS, MSI, Installshield, ... off the top of my head) dan masing-masing biasanya memiliki sakelar baris perintah yang dapat Anda gunakan untuk melakukan instalasi diam-diam. Untuk paket Penginstal Microsoft (MSI), terdapat tombol /q untuk melakukan penginstalan tanpa GUI. Jadi baris sampel mungkin terlihat seperti ini:

"E:\Microsoft.NET\netfx.msi" /q

Alih-alih membuat skrip batch, ada alat yang lebih mudah untuk membuat instalasi Windows kustom tanpa pengawasan yang akan mempersiapkan semua perangkat lunak Anda untuk Anda. NLite dan Panduan Tanpa Pengawasan MSFN kepikiran.

Alternatif lain adalah membuat cadangan penuh setelah semua program yang Anda inginkan diinstal dan dikonfigurasi menggunakan Norton Ghost atau Acronis True Image . Saya selalu berakhir dengan menendang diri saya sendiri dengan solusi ini. Tepat setelah saya pikir semuanya baik-baik saja dan saya melakukan pencadangan, selalu ada 1 perangkat lunak yang saya lupa instal.

Saya suka menggunakan AutoIt. Biasanya saya hanya menggunakan sakelar jika penginstal mendukungnya. Universal Silent Switch Finder (USSF) adalah alat kecil yang bagus untuk memberi tahu Anda sakelar mana yang diperlukan penginstal. USSF dapat ditemukan di SoftPedia .

Dengan AutoIt apa yang akan Anda lakukan adalah membuat skrip yang mensimulasikan penekanan tombol / klik mouse. Misalnya, dalam skrip Anda akan menyuruhnya untuk menjalankan ccleaner.exe, tunggu jendela aktif, pilih tombol berikutnya / kirim tekan tombol "enter", dll. (Atau gunakan / S switch)

Jika Anda hanya membutuhkan satu file (Seperti jika Anda membuat addons nLite) maka saya tidak pernah berhasil menggunakan AutoIt dengan sendirinya, karena akan ada dua file dalam paket. Jadi saya hanya menggunakan SFX (Self Extracting Archive) untuk menambahkan script AutoIt dan file setup ke dalam satu file.

Omong-omong, file bantuan AutoIt sangat informatif. Anda akan menemukan contoh dan penjelasan yang bagus di dalamnya, jika Anda harus memilih untuk menggunakannya. Semoga ini bisa membantu Anda

Sunting: Anda juga dapat mengkompilasi skrip AutoIt ke .exe. Itu berarti tidak perlu menginstal AutoIt untuk menggunakan skrip.

File batch untuk menginstal perangkat lunak secara otomatis paling cocok untuk mendukung beberapa workstation karena waktu dan upaya yang diperlukan untuk mengubah installer, menguji sistem untuk keberadaan perangkat lunak versi saat ini atau yang lebih lama, switch baris perintah untuk membuat installer berjalan "diam" atau "senyap", dan perintah pembersihan untuk memindahkan atau membuat pintasan, menghapus autorun "pembantu" atau pembaru membuat pemeliharaan file batch ini memakan waktu. Dan menyimpan dan memulihkan pengaturan individu hampir tidak mungkin untuk dikelola dengan skrip batch, bukan karena Anda tidak dapat melakukannya, tetapi karena menemukan semua kunci registri dan file% appdata% adalah tugas yang berat.

Saya menyarankan agar Anda menggunakan ketarin.exe yang tersedia dari http://ketarin.canneverbe.com untuk selalu memperbarui program dan utilitas shareware / freeware. Setelah Anda mengkonfigurasi Ketarin, Anda akan dapat menjaga sistem yang ada tetap mutakhir pada semua program besar dan kecil Anda dan jika Anda harus membangun kembali sistem Anda, Anda dapat menginstal Ketarin terlebih dahulu, menginstal file database perangkat lunak Anda, menjalankan Ketarin dan unduh semua aplikasi Anda dalam satu gerakan.

Setelah itu, Anda harus menginstal aplikasi Anda, tetapi proses itu tidak memakan waktu sebanyak yang dibutuhkan untuk menemukan semua unduhan. Dan Ketarin adalah alat yang berguna untuk pemeliharaan sistem yang sedang berlangsung, membuat kurva pembelajaran dan upaya untuk membuat database Anda bermanfaat. Ada templat yang tersedia dalam Ketarin untuk mengunduh sejumlah besar program dan Anda dapat membuat templat Anda sendiri. Saya pikir Anda mungkin akan ketagihan dan menganggap usaha itu menyenangkan.

Saya belum mencobanya, tetapi Ketarin memungkinkan Anda untuk menjalankan perintah sebelum dan sesudah mengunduh aplikasi, sehingga Anda pada prinsipnya dapat menjalankan tes, mengkonfigurasi mode diam, dan pembersihan sesudahnya dengan meletakkan perintah atau referensi ke file batch langsung di Ketarin.

Jika Anda benar-benar tertarik untuk menulis skrip untuk melakukan penginstalan diam-diam, http://www.msfn.org/ , http://appdeploy.com , http://msdn.microsoft.com dan http://www.installshield.com/ adalah situs yang menarik untuk dikunjungi. Saya masih menggunakan file batch cmd.exe lama biasa, tetapi Microsoft telah mengembangkan PowerShell untuk membuat hal-hal seperti ini lebih mudah. Ini menggunakan Instrumentasi Manajemen Windows untuk mengetahui bagaimana sistem Anda dikonfigurasi, sehingga skrip Anda dapat membuat keputusan yang tepat tentang apa yang harus dilakukan.

Ada juga alat untuk memungkinkan Anda memodifikasi atau membuat penginstal .msi, yang memiliki cara standar untuk mengubah (atau "mengubah") penginstal dan membuatnya berjalan secara diam-diam. Anda juga dapat membuat penginstalan standar dari utilitas paket zip yang membuatnya lebih mudah untuk menghapus penginstalan program tanpa penginstal ini. Namun menurut saya ini hanya cocok jika Anda mendukung sejumlah besar pengguna, katakanlah, 20 atau lebih.
