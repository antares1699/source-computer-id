Silakan periksa file konfigurasi sshd jika Autentikasi Kunci Publik diaktifkan

cari di / etc / ssh / sshd_config untuk

#PubkeyAuthentication yes

Jika dikomentari hapus awalan '#'. jika disetel ke 'tidak' ubah ke 'ya'

Seperti yang disebutkan @Darius, harap periksa juga pengaturan AuthorizedKeysFile Ini harus dikomentari untuk perilaku default. Kemudian buat file authorized_keys di ~/.ssh/ dan tambahkan string kunci publik Anda ke file ini. Bagaimana melakukan ini dijelaskan di sini dalam dokumentasi Debian

Jaga izin yang ditetapkan pada folder ~/.ssh/ di direktori home Anda. Hanya pemilik yang harus memiliki izin baca / tulis.

chmod -R go-xwr ~/.ssh

dapat memperbaiki izin
