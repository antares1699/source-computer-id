Firmware vs OS RAID (alias Windows 8 Disk Mirroring vs Intel Onboard)

Jadi Windows 8 keluar dan saya memiliki motherboard baru. Saya ingin membuat kopling RAID 1 antara dua HDD - untuk tujuan penyimpanan saja (OS saya menggunakan SSD) - tetapi saya tidak tahu rute mana yang terbaik untuk diambil.

Motherboard saya (chipset Z77) dilengkapi dengan Intel Firmware RAID, tetapi karena saya hanya ingin menggunakan RAID saya untuk penyimpanan, saya bertanya-tanya apakah saya lebih baik menggunakan Windows 8 Disk Mirroring?

Adakah yang bisa menyarankan mana yang lebih baik? Atau mungkin pro dan kontra masing-masing, apakah itu terlalu kontroversial? Saya hanya tidak bisa melihat manfaat Firmware RAID melalui Windows Disk Mirroring.

tl; dr: Paragraf terakhir.

Ada beberapa kebingungan di sini tentang berbagai hal yang dapat membuat perangkat mirip RAID atau RAID. Mari kita bahas opsi dalam urutan preferensi dan kinerja.

Terbaik pertama: Kartu RAID asli, dari vendor seperti LSI, Pinang, dan lainnya. Kartu ini biasanya memiliki memori khusus untuk caching dan hampir selalu memiliki unit cadangan baterai opsional. BBU memberikan daya ke cache, sehingga jika terjadi kehilangan daya yang tidak terduga, data apa pun yang belum ditulis ke disk tidak akan hilang. Beberapa kartu tidak mengizinkan Anda mengaktifkan cache tulis kecuali jika BBU dipasang. Beberapa dari kartu ini bahkan memiliki cara yang lebih canggih untuk meningkatkan kinerja. Misalnya, banyak kartu LSI baru memiliki add-on opsional yang disebut "CacheCade", yang dapat menggunakan SSD untuk cache baca. Kartu RAID asli juga memungkinkan Anda untuk menetapkan suku cadang, dan dapat secara otomatis mulai membangun kembali larik jika terjadi kegagalan drive.

Selanjutnya adalah RAID perangkat lunak yang disediakan OS. Linux dan edisi Windows tertentu dapat membuat perangkat menggunakan berbagai tingkat RAID. Software RAID adalah solusi yang memadai dalam banyak kasus, tetapi memiliki beberapa kekurangan. Pertama, beberapa edisi Windows tidak dapat melakukan booting dari Dynamic Disk, fitur yang membuat perangkat lunak RAID. Beberapa distribusi Linux (kebanyakan lebih lama) juga mengalami masalah booting dari RAID perangkat lunak. Kedua, RAID perangkat lunak tidak memiliki cache tulis yang didukung perangkat keras. Ini tidak akan menyelamatkan Anda dari kerusakan data yang disebabkan oleh kehilangan daya secara tiba-tiba. Ketiga, bergantung pada beban kerja dan jenis RAID, ini dapat menghabiskan banyak waktu CPU. Hal ini terutama berlaku saat menggunakan jenis RAID yang memiliki paritas, seperti 6. Saya tidak yakin tentang Windows, tetapi RAID perangkat lunak Linux juga memungkinkan Anda mengalokasikan drive cadangan.

Berikutnya adalah RAID berbasis firmware / chipset, seperti LSI 1068 dan RST berbasis Intel "67", "68" dan "77". Seperti kartu RAID perangkat keras, mereka mengekspos drive normal ke OS, dan tidak memerlukan driver untuk melakukan operasi dasar. Semua operasi I / O, termasuk paritas, dilakukan dari dalam firmware itu sendiri, artinya mereka tidak menggunakan waktu CPU. Sayangnya mereka juga tidak memiliki keuntungan dari RAID perangkat keras khusus, seperti cache tulis atau BBU. Sangat disayangkan bahwa Intel RST yang lebih lama memiliki reputasi yang rapuh dan rewel. Firmware RAID biasanya tidak memungkinkan Anda mengalokasikan drive cadangan. Kecuali Anda melakukan sesuatu yang konyol dan menggunakan RAID 0, Anda mungkin tidak akan melihat peningkatan kinerja apa pun menggunakan firmware RAID.

Akhirnya, dan yang terburuk, adalah fakeraid. Ada dua kelas fakeraid. Yang pertama tidak memerlukan driver, tetapi biasanya hanya menampilkan RAID 0 dan 1. Yang kedua membutuhkan driver untuk semua operasi, dan melakukan semuanya dalam perangkat lunak. Fakeraid dapat ditemukan dalam kartu tambahan yang murah, di beberapa perangkat NAS kelas konsumen yang murah, dan bahkan pada beberapa motherboard (biasanya dengan chipset AMD atau Nvidia). Seperti firmware RAID, Anda hampir pasti tidak akan melihat peningkatan kinerja dengan fakeraid.

Pertanyaan Anda berkaitan dengan dua opsi tengah: RAID perangkat lunak Windows versus RAID firmware Intel pada papan Z77. Biasanya saya akan mengatakan ini adalah kinerja dan keandalan, karena kedua opsi telah membuktikan diri mereka sendiri di dunia nyata dan tidak terlalu buruk. Karena sepertinya Anda tidak menggunakan mirror sebagai perangkat boot, saya biasanya akan mengatakan bahwa RAID perangkat lunak Windows mungkin adalah cara yang paling masuk akal untuk digunakan. Namun, RAID firmware Intel seri 77 memiliki trik: itu dapat menggunakan SSD sebagai cache , saat Anda memasang pilihan sopir. Artinya, caching hanya tersedia saat driver dimuat. Array itu sendiri selalu tersedia. Jika Anda merasa ingin menempelkan SSD lain di sistem Anda, itu mungkin berhasil dengan baik.

  • Itu hidup di tengah-tengah antara RAID "palsu" dan RAID "asli". Ini nyata karena ini adalah implementasi on-chip aktual dari 0, 1, 10, dan 5, tanpa memerlukan perangkat lunak atau driver eksternal. Ini adalah komponen aktual dari jembatan selatan, bersama dengan bit pengontrol drive lainnya. Beberapa orang menganggapnya "palsu" karena ini adalah implementasi yang sangat sederhana dengan manajemen minimal dan wawasan tentang cara kerjanya. Ini bukan komponen terpisah oleh pihak ketiga, dan tidak memerlukan instalasi perangkat lunak / driver (== tidak ada penggunaan CPU), dan mereka mengambilnya sedikit keluar dari kategori standar RAID "palsu".

  1. Jangan gunakan pengontrol Jmicron [fake RAID] untuk apa pun selain AHCI yang bodoh, sebaiknya hanya untuk drive DVD.

Baik Windows Mirroring dan Intel FakeRAID akan memiliki kinerja yang hampir sama. Anda tidak akan melihat manfaat nyata satu sama lain karena fakta bahwa mereka akan menggunakan sumber daya sistem (CPU) Anda untuk mengelola pengontrol RAID.

Ini berbeda dari pengontrol RAID perangkat keras di mana pengontrol Anda akan melakukan penghitungan apa pun, mengangkat penggunaan sumber daya dari CPU Anda.

Namun, Anda juga perlu mempertimbangkan bahwa Pengontrol FakeRAID dan Pencerminan Windows adalah fungsi sederhana dan Anda mungkin tidak akan melihat perbedaan kecepatan antara keduanya dan juga Pengontrol RAID Perangkat Keras. Satu-satunya perbedaan yang akan Anda lihat adalah bahwa Anda akan memilikinya penggunaan cpu yang lebih rendah . Jumlah ini adalah perbedaan yang sangat rendah sehingga dapat diabaikan.

Anda mungkin melihat beberapa perbedaan nyata jika Anda menggunakan RAID5 / 6 di mana paritas dihitung dan ditulis. Ini membutuhkan lebih banyak perhitungan dan kemudian mungkin ingin menjalankan beberapa tolok ukur untuk melihat apakah FakeRAID lebih baik atau lebih buruk daripada Windows Mirroring. Sebagian besar Pengontrol RAID Perangkat Keras akan selalu menang saat menggunakan RAID5 / 6.
