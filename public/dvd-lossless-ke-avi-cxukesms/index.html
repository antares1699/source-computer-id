Ada beberapa hal yang perlu dipahami di sini:

Pertama, AutoGK cukup dapat disesuaikan, Anda dapat menyalin DVD ke file AVI 2GB yang kualitasnya mendekati kualitas DVD sumber.

Kedua, AVI adalah format yang cukup buruk untuk mencoba melakukan hal-hal yang tidak merugikan. Sulit untuk menyematkan subtitle dan aliran audio alternatif.

Ketiga, sebagian besar waktu keluaran video akan menjadi 704 piksel (bukan 720) setelah ripping dan konversi - dan ada alasan bagus. Banyak konten DVD yang akhirnya memiliki setengah garis hitam di bagian atas dan bawah, yang menjadi masalah bagi pembuat enkode, tampilan, dan hanya efisiensi umum dari ripping. Akibatnya, banyak pembuat enkode akan menghapus 8 piksel dari atas dan bawah (dengan demikian 720-16 = 704). Ingat, ini sebenarnya hal yang bagus!

Keempat, Anda akan dilayani dengan baik dengan menggunakan MP4 atau MKV sebagai wadah, bukan AVI. Mungkin itu tidak sesuai dengan keseluruhan rencana induk Anda, tetapi Windows 7 dapat memutar file MP4 di Windows Media Player sekarang. Kedua format ini memiliki cara untuk menangani masalah dari masalah kedua yang dijelaskan di atas.

Terakhir, mengapa tidak hanya menyimpan .VOB dan file terkait yang didekripsi jika lossless adalah kuncinya dan Anda tidak peduli dengan ukuran file?

vobcopy -l

Akan membuat satu file VOB besar dari DVD Anda, di direktori kerja, dengan nama file berdasarkan label DVD. Jika Anda memiliki libdvdcss, itu bahkan akan berfungsi pada DVD terenkripsi.

Untuk menentukan nama file keluaran:

vobcopy -l -o output.vob
