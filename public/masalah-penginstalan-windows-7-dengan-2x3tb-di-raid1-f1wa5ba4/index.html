Masalah penginstalan Windows 7 dengan 2x3TB di RAID1

Tidak yakin apa yang saya lakukan salah. Membangun satu PC sebelumnya berhasil dengan cara yang sama tetapi yang ini tampaknya sedang berjuang.

Saya memiliki komponen berikut:

  • Mobo: ASUS P8Z68-V / Pro Gen 3 (diperbarui ke firmware terbaru)
  • RAM: 16 GB (2 x 8 GB)
  • Sumber Daya listrik: Corsair HX850
  • HDD:
    • Drive 2 x 3TB pada pengontrol intel z68
    • 1 x 128GB SSD pada pengontrol Marvel
  • Grafik: Sapphire 7950 (tidak menggunakan grafik on-board)

Masalah

  • Saya menyiapkan disk 3 TB saya di RAID1
  • kontroler tampaknya mengenali mereka dengan baik selama boot sebagai satu volume raid1 2,7TB
  • pengaturan windows melihat dua disk, keduanya 746 GB, tetapi hanya akan membiarkan saya menginstal ke satu dan tampaknya berfungsi dengan baik.
  • Windows tampaknya menginstal dengan baik
  • setelah penginstal reboot, saya menerima "windows gagal untuk memulai" kode referensi kesalahan 0xc000000e dan "\ Windows \ system32 \ winload.exe
  • setiap kali saya melakukan penginstalan, entri "win7" tambahan baru ditambahkan ke menu boot; semua mengarah pada kesalahan ini.

Apa yang saya coba:

  • memperbarui BIOS ke firmware terbaru
  • mencoba memperbaiki penginstalan
  • mencoba membersihkan / menghapus drive penyerbuan / merampok ulang
  • mencoba memformat drive saat menginstal
  • mencoba untuk menghapus menu entri (tidak tahu bagaimana melakukan itu)

Tidak peduli berapa kali saya menghancurkan array serangan, memformat disk, dll., Entri boot terus menumpuk. Tahu di mana aku salah?

Peringatan / Catatan Sebelum Anda Mulai

  • Anda akan kehilangan semua data di hard drive Anda selama proses ini. Cadangkan jika perlu.
  • Jika Anda sudah memiliki drive dalam konfigurasi RAID, Anda mungkin ingin menghapusnya (sekali lagi, kehilangan semua data) untuk mengikuti proses ini.
  • Tidak ada jaminan atau jaminan, dll. - berhasil untuk saya, tetapi cobalah di rumah hanya jika Anda tahu konsekuensinya.

Cara Memasang Windows 7 pada Chipset Intel Z68 dengan Drive Lebih Besar Dari 2TB

Panduan ini akan memandu Anda melalui proses penginstalan Windows 7 x64 di komputer baru menggunakan yang berikut ini:

  • Drive yang lebih besar dari 2TB dalam Konfigurasi RAID 1 (untuk contoh kami, kami menggunakan HDD 2 x 3TB)
  • Drive SSD
  • Prosesor Chipset Intel Z68
  • Motherboard yang mendukung Chipset Z68 (dalam kasus saya, sebuah Asus P8Z68-V / Pro Gen3 )
  • Windows 7 Professional 64-bit
  • Stik USB yang Anda miliki (ukuran apa pun bagus, diformat ke FAT32)

Pada Mesin yang Baik: Pasang Driver Standalone Intel RST ke Drive USB FAT32

  • Pada mesin yang bagus, format drive USB ke FAT32
  • Pergilah ke Pusat Unduhan Intel RST untuk Windows 7
  • Unduh versi terbaru f6flpy-x64.zip ( 12.9.0.1001 pada 3/10/2014 ini ) Ini adalah driver yang akan Anda gunakan saat menginstal Windows 7.
  • Buka zip file ke folder di stik USB Anda

Di Mesin Baru: Perbarui Firmware Motherboard Anda

  • Ikuti apa pun rekomendasi pabrikan Anda untuk mendapatkan dan flash ke BIOS terbaru. Untuk ASUS P8Z68-V / Pro Gen3, lihat halaman unduh untuk motherboard . Unduh driver BIOS terbaru.

Buat Array RAID

(Ini mengasumsikan Anda memiliki PC Anda seluruhnya, perangkat keras bijaksana, yang berada di luar cakupan artikel ini)

  • Selama boot, saat layar penyimpanan Intel ditampilkan, tekan Ctrl + saya untuk masuk ke layar penyiapan RAID
  • Ikuti petunjuk untuk membuat larik RAID1 dari dua drive besar Anda
  • Pada titik ini, drive harus dilihat sebagai ukuran sebenarnya (kira-kira 2,7TB menggunakan drive 3TB)

Menambahkan Driver Khusus ke Pengaturan Windows 7

  • Colokkan Drive USB Anda
  • Boot dari CD-ROM Windows 7 dalam mode UEFI (pilih opsi UEFI untuk drive dvd Anda dari menu daftar boot - akan ada dua opsi, satu dengan dan satu tanpa UEFI).
    • Anda harus mem-boot dari mode UEFI, karena ini adalah satu-satunya mode yang dapat membuat drive GPT.
  • Pilih Instal Kustom (sekali lagi, kehilangan semua data)
  • Saat Anda mencapai layar yang menampilkan drive disk Anda, Anda akan melihat mereka menunjukkan ukuran yang salah.
  • Klik tombol "Muat Driver".
  • Arahkan ke tempat di drive USB di mana Anda memiliki driver Intel RST dan klik OK.
  • Klik Next untuk memuat driver.

Pada titik ini, larik RAID1 Anda akan ditampilkan sebagai satu drive besar 2,7 TB.

Ubah RAID Array dari MBR ke GPT

  • Dari layar yang menanyakan drive mana Anda ingin menginstal windows, Tekan Bergeser + F10 untuk menampilkan prompt perintah
  • Tipe diskpart untuk membuka utilitas partisi disk
  • Tipe daftar disk untuk mencantumkan semua drive
  • Temukan drive yang terdaftar sebagai drive besar dan temukan nomornya (di saya, volume RAID besar adalah disk 0)
  • Tipe pilih disk # (di mana # adalah jumlah volume besar Anda, mis. pilih disk 0 untuk contoh saya)
  • Tipe bersih . Ini akan menghapus semua data tentang drive dan memastikannya sebagus drive mentah.
  • Tipe konversi gpt untuk mengubah disk menjadi disk GPT dari disk MBR.
  • Tipe keluar untuk keluar dari diskpart dan keluar sekali lagi untuk keluar dari prompt perintah
  • Restart komputer (ya saya tahu Anda sedang dalam pengaturan Windows; tidak, Anda tidak perlu menyelesaikannya)

Periksa untuk Memastikan Konversi GPT Berhasil

  • Masuk ke pengaturan windows lagi, muat driver intel lagi, dan buka command prompt dengan menggunakan Bergeser + F10 lagi.
  • Tipe daftar disk . Kali ini, periksa untuk memastikan bahwa disk yang Anda ubah memiliki tanda bintang (*) di kolom "GPT".

Instal Windows

  • Buka instalasi windows dari UEFI BIOS lagi dan muat kembali driver Intel.
  • Drive besar Anda sekarang akan ditampilkan sebagai ruang yang tidak terisi.
  • Klik Maju dan klik Baru untuk membuat partisi baru.
    • Anda akan menerima pesan bahwa Windows dapat membuat partisi tambahan. Ini adalah baik ; itu berarti Windows membuat struktur partisi GPT.
  • Instal Windows ke partisi utama drive RAID seperti biasa.

Boot ke Windows dan Perpanjang Drive

Ketika Windows pertama kali boot, itu tidak akan mengenali drive sebagai drive 3 TB penuh - 760 GB akan dibiarkan tidak terisi.

  • Dari menu start, buka "Computer Management" (pintasan: type compmgmt.msc dan tekan enter)
  • Buka "Manajemen Disk".
  • Klik kanan struktur drive besar Anda dan pilih "Perpanjang".
  • Perpanjang untuk mengisi sisa ruang.

Anda sekarang memiliki larik RAID1 3TB yang sepenuhnya dikenali di Windows 7 64-bit.

Tindakan yang Direkomendasikan Lainnya

  • Instal Perangkat Lunak Intel RST dari pabrikan motherboard Anda atau situs web jika Anda merasa paham.
  • Instal juga pusat manajemen Intel (biasanya dibundel dengan perangkat lunak RST). Inilah yang akan memungkinkan Anda menggunakan SSD untuk mempercepat array RAID1 Anda (fitur hebat).

Saya bisa menjelaskan mengapa windows melihat dua disk, keduanya 746 GB.

Jika Anda mempartisi drive menggunakan file kuno MBR maka Anda dibatasi hingga 2,19TiB.
3TB (mungkin agak kurang karena pemasaran cenderung membulatkan) - 2.19TiB (karena membungkus) sekitar 750GB.

Untuk menghindari hal ini, Anda perlu mempartisi drive menggunakan GPT . GPT didukung oleh windows 7. Jika Anda menggunakan PC dengan UEFI daripada BIOS Anda bahkan dapat boot dari itu. (Jika Anda memiliki BIOS lama daripada UEFI biasanya tidak selalu berfungsi.)

Jika Anda menggunakan SSD sebagai disk OS maka boot seharusnya tidak menjadi masalah, baik dari BIOS atau EUFI. Setelah Anda menginstal windows, Anda dapat pergi ke manajer disk, mempartisi kedua drive 3TB dengan GPT dan mengatur volume cermin pada mereka.

(Catatan, biarkan BIOS dalam mode AHCI untuk ini. Tidak dalam mode RAID.)

Sistem konfigurasi:

  • Gigabyte GA-H87M-D3H Motherboard diperbarui ke firmware v.F7 terbaru
  • 2 x Seagate 4.0TB SATA HD (Model # ST4000VN000) RAID 1

  • 2 x G. Keterampilan DDR3-1600 PC3-12800 RAM 8GB
  • Windows 7 64-bit Ultimate

Karena ukuran HD, saya tidak bisa menginstal Windows. Sebelum melanjutkan, harap dicatat bahwa saya ingin menginstal Windows pada partisi 500 GB (bukan keseluruhan 4 TB).

Catatan BIOS (khusus untuk motherboard saya tetapi mungkin berguna sebagai petunjuk untuk MB lain):

  • RAID dikonfigurasi sebagai ROM RAID Lama per instruksi manual, yang mirip dengan instruksi di atas berlabel "Buat RAID Array"

Catatan Instalasi Windows:

  • Instruksi MB mengatakan bahwa saat menginstal dengan Windows 7, tidak perlu menginstal driver RAID / AHCI yang terpisah. Saya menemukan ini tidak benar karena ketika saya mencoba menginstal menggunakan disk Windows tanpa "Memuat Driver" seperti yang disebutkan di atas, HD saya dibagi menjadi dua drive, masing-masing sekitar 1,7 TB, bukan satu drive logis 4 TB. Saya mencoba driver Intel RST Standalone yang disebutkan di atas dan driver Intel RST yang disediakan oleh Gigabyte ([DVD Drive]:\\BootDrv\iRST\). Keduanya bekerja dengan baik. Akhirnya saya menggunakan driver yang disediakan oleh Gigabyte. Terlepas dari itu, untuk melihat satu drive logis 4 TB, petunjuk yang menyebutkan "Memuat Driver Intel RST harus diikuti, terlepas dari apakah Anda akhirnya menggunakan driver Intel atau Gigabyte.

  • Dalam instruksi, ada satu hal yang dikatakan untuk menggunakan "UEFI BIOS lagi". Ini agak membingungkan karena UEFI agak baru bagi saya. Ada banyak tempat lain untuk membaca tentang latar belakang teoritis jadi hanya untuk kepraktisan, ini pada dasarnya berarti ketika memilih drive boot, pastikan untuk mem-boot "UEFI: DVD / CDROM". Juga, pastikan bahwa saat Anda membuat partisi tempat Anda ingin menginstal Windows, di sana harus menjadi 3 partisi:

    • Partition 1 - System
    • Partition 2 - MSR dan
    • Partition 3 - Primary,

    dalam urutan itu. Instalasi harus dilakukan pada Partition 3 - Primary. Dalam kasus saya, karena saya ingin menginstal pada partisi 500 GB, saya menentukan 500 GB saat membuat partisi baru. Ada dua halaman yang sangat membantu menjelaskan Cara Menginstal Windows 7 Menggunakan UEFI dan Bagaimana Melakukan Instalasi Bersih dengan Windows 7 .

  • Setelah saya dapat mengetahui pengaturan RAID yang tepat, memuat driver RAID yang tepat, dan mengatur partisi dengan benar, Windows akan mulai menginstal. Namun, selama salah satu rebootnya, Windows akan menampilkan layar biru kematian (BSOD) dan saya tidak dapat melanjutkan. Saat menjelajahi jaring, ada banyak saran tentang apa yang bisa salah. Pada dasarnya, tidak ada satu penyebab spesifik karena BSOD sangat tidak jelas. Namun, dalam kasus saya, hanya ada dua penyesuaian sebagai berikut:

    • MB tidak menyetel RAM ke 1600MHz. Untuk memperbaikinya, saya masuk ke BIOS> M.I.T. > Pengaturan Memori Lanjutan> Pengganda Memori Sistem> 16. Ini mungkin bukan penyebab BSOD tetapi perlu disebutkan karena memori adalah penyebab BSOD yang sering dilaporkan. Saya belum memiliki kesempatan tetapi akan menjalankan MemTest86 + untuk memeriksa memori.
    • Booting ke disk Instalasi Windows, saya menggunakan alat yang disediakan. Perbaiki komputer Anda> Gunakan alat pemulihan ... Saya mencoba Diagnostik Memori Windows dan Perbaikan Startup Windows, dalam urutan itu, dan kemudian komputer akhirnya dapat melakukan boot ulang dan menyelesaikan penginstalan dengan sukses. Instruksi di Cara Melakukan Perbaikan Startup di Windows 7 sangat membantu untuk langkah ini.
