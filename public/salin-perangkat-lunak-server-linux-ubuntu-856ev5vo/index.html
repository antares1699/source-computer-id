Ada pendekatan lain yang bisa Anda gunakan:

  1. buat VM baru, dan instal versi Ubuntu yang sama persis ke dalamnya. Gunakan CD alternatif untuk menginstal, dan buat instalasi minimal.
  2. login ke server jarak jauh Anda, dan jalankan dpkg --get-selections \* > /tmp/serverpkgs.lst.
  3. transfer /tmp/serverpkgs.lst ke VM baru yang Anda buat
  4. di VM baru Anda, jalankan sudo dpkg --set-selections < /tmp/serverpkgs.lst
  5. masih di VM, jalankan sudo apt-get dselect-upgrade

Sekarang Anda seharusnya sudah menduplikasi perangkat lunak yang diinstal - yang meninggalkan Anda dengan konfigurasinya. Ini mungkin membuktikan bagian yang lebih rumit: karena saya tidak tahu apa yang digunakan, tidak ada "jawaban global" untuk itu. Umumnya, file konfigurasi disimpan di bawah /etc, jadi Anda mungkin ingin cd / && sudo tar czf /tmp/server-cfg.tar.gz etc/*, kemudian mentransfer /tmp/server-cfg.tar.gz ke VM Anda, membongkar ke beberapa direktori sementara (menggunakan misalnya cd /tmp && tar xzf server-cfg.tar.gz, yang akan membuat /tmp/etc/*), dan periksa hati-hati file konfigurasi mana yang akan disalin ke pohon /etc Anda.

Mengatasi secara langsung antara dua platform perangkat keras yang berbeda secara teoritis dimungkinkan (yang harus Anda lakukan hanyalah menyalin file dan menginstal bootloader) tetapi jarang merupakan ide yang baik. Saya telah berhasil melakukannya dalam beberapa kesempatan, tetapi saya juga telah berhenti dan gagal pada banyak kesempatan lainnya.

Masalahnya adalah ketika Anda menginstal OS, pengoptimalan tertentu dan keputusan lain terkadang dibuat berdasarkan jenis perangkat keras yang Anda instal. Jika Anda menduplikasi ke perangkat keras yang sama persis, maka tidak ada masalah. Jika Anda menggandakan ke perangkat keras yang sangat berbeda, maka mungkin Anda akan mempunyai masalah. Beralih dari mesin fisik ke mesin virtual mungkin merupakan contoh "perangkat keras yang berbeda". Ini mungkin atau mungkin tidak berhasil. Kemungkinan besar, itu tidak akan berhasil pada awalnya, tetapi jika Anda benar-benar tahu apa yang Anda lakukan maka Anda dapat menyesuaikannya sampai berhasil.

Tetapi jika Anda melakukannya sungguh sangat tahu apa yang Anda lakukan, maka Anda mungkin tidak akan bertanya apakah itu mungkin. Jadi, dengan logika itu, jawabannya mungkin tidak .

Tetapi jika Anda cukup gigih, Anda mungkin bisa mengubahnya menjadi file Iya . Apa yang Anda coba lakukan sering kali disingkat sebagai "fisik ke virtual" atau "P2V" (atau bahkan lebih pendek). Tidak ada kekurangan pendapat tentang subjek dan banyak dan banyak orang memberikan instruksi yang saling bertentangan tentang bagaimana melakukannya. Jadi untuk informasi lebih lanjut, saya akan tunduk pada Google yang perkasa .
