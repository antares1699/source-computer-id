Ruang penyimpanan di Windows 10 dan berjenjang

Saya memiliki sejumlah disk di desktop Windows 10 saya (pro jika penting) dan telah bermain-main dengan ruang penyimpanan. Saya membutuhkan beberapa saran tentang cara mengonfigurasi volume penyimpanan untuk kasus penggunaan saya. Saya tahu UI Windows 10 terbatas, tetapi tampaknya Anda bisa mendapatkan fitur tiering dari Powershell. Saya senang membaca dokumen untuk ruang penyimpanan dan mencoba mengungkap fitur tersebut, tetapi ingin mendapatkan masukan jika strategi saya tepat.

Inilah yang saya miliki saat ini:

  • 2x256GB SSD - tidak terisi, adalah volume aplikasi + boot RAID0 lama saya
  • 1x5TB HDD - media + game + aplikasi (Saya ingin menghentikan drive ini)
  • 1x8TB HDD - media + dokumen + foto
  • 2x8TB HDD - disk baru yang tidak digunakan
  • CPU: i7 4790k pada mobo chipset Z97, mungkin meningkatkan ke Ryzen3 ketika saatnya tiba
  • Drive boot saya adalah disk NVMe 512GB

Saya ingin membuat dua volume dari drive di atas. Idealnya, inilah yang saya inginkan:

  • game + volume awal: Tingkat SSD bergaris 256 GB + Tingkat HDD bergaris 3 TB
  • volume media + dokumen + foto: 256GB tingkat SSD cermin + Tingkat HDD 'paritas' 14TB

Semua yang ada di game + volume awal saya bisa kehilangan karena saya bisa mengunduh ulang datanya. Saya misalnya akan meletakkan file 'memori' VM saya, database plex, dll. Pada disk awal. Sesuatu yang sensitif akan saya backup ke volume lain.

Saya ingin setidaknya satu redundansi drive pada volume media + dokumen + foto. Saya memiliki dokumen + foto saya dicadangkan di luar kantor. jadi ini hanya menyelamatkan saya dari sakit kepala jika saya kehilangan drive.

Apakah pengaturan ini tampak masuk akal dan apakah itu sesuatu yang dapat saya capai pada mesin Windows 10 pro?

Saya juga membaca bahwa ruang penyimpanan memiliki kinerja kumpulan 'paritas' yang sangat buruk di versi Windows sebelumnya. Apakah ini masih benar? Saya belum dapat melakukan pengujian sebenarnya dari konfigurasi paritas karena 2 dari 4 HDD saya digunakan. Saya pikir saya akan bertanya di sini jika seseorang memiliki pengalaman melakukan ini sebelumnya.

Solusi lain apa yang harus saya perhatikan?

Untuk membuat Windows 10 Tiered Storage Spaces, lihat referensi berikut:

  • Blog Nils Schimmelmann: Intel Smart Response Technology vs Windows 10 Tiered Storage Spaces
  • Cara Mengonfigurasi Ruang Penyimpanan Bertingkat di Windows 10
  • vWorkspace Hyper-v dengan kolam penyimpanan berjenjang - Part1

Di sini saya mereproduksi skrip PowerShell Nils Schimmelmann untuk membuat dua tingkatan Ruang Penyimpanan, untuk SSD dan HDD:

#Variables
$StoragePoolName = "My Storage Pool"
$TieredSpaceName = "My Tiered Space"
$ResiliencySetting = "Simple"
$SSDTierName = "SSDTier"
$HDDTierName = "HDDTier"

#List all disks that can be pooled and output in table format (format-table)
Get-PhysicalDisk -CanPool $True | ft FriendlyName,OperationalStatus,Size,MediaType

#Store all physical disks that can be pooled into a variable, $PhysicalDisks
$PhysicalDisks = (Get-PhysicalDisk -CanPool $True | Where MediaType -NE UnSpecified)       

#Create a new Storage Pool using the disks in variable $PhysicalDisks with a name of My Storage Pool
$SubSysName = (Get-StorageSubSystem).FriendlyName
New-StoragePool -PhysicalDisks $PhysicalDisks -StorageSubSystemFriendlyName $SubSysName -FriendlyName $StoragePoolName

#View the disks in the Storage Pool just created
Get-StoragePool -FriendlyName $StoragePoolName | Get-PhysicalDisk | Select FriendlyName, MediaType

#Create two tiers in the Storage Pool created. One for SSD disks and one for HDD disks
$SSDTier = New-StorageTier -StoragePoolFriendlyName $StoragePoolName -FriendlyName $SSDTierName -MediaType SSD
$HDDTier = New-StorageTier -StoragePoolFriendlyName $StoragePoolName -FriendlyName $HDDTierName -MediaType HDD

#Identify tier sizes within this storage pool
$SSDTierSizes = (Get-StorageTierSupportedSize -FriendlyName $SSDTierName -ResiliencySettingName $ResiliencySetting).TierSizeMax
$HDDTierSizes = (Get-StorageTierSupportedSize -FriendlyName $HDDTierName -ResiliencySettingName $ResiliencySetting).TierSizeMax 

#Create a new virtual disk in the pool with a name of TieredSpace using the SSD and HDD tiers
New-VirtualDisk -StoragePoolFriendlyName $StoragePoolName -FriendlyName $TieredSpaceName -StorageTiers $SSDTier, $HDDTier -StorageTierSizes $SSDTierSizes, $HDDTierSizes -ResiliencySettingName $ResiliencySetting  -AutoWriteCacheSize -AutoNumberOfColumns

#Alternatively try adjusting the sizes manually:
#New-VirtualDisk -StoragePoolFriendlyName $StoragePoolName -FriendlyName $TieredSpaceName -StorageTiers @($SSDTier,$HDDTier) -StorageTierSizes @(228GB,1.816TB) -ResiliencySettingName $ResiliencySetting -AutoWriteCacheSize -AutoNumberOfColumns

Contoh berikut menunjukkan cara membuat volume Multi-Tangguh di mana tingkat SSD dicerminkan dan tingkat HDD dikonfigurasi dalam Paritas.

New-StorageTier -StoragePoolFriendlyName Pool1 -FriendlyName SSD_Tier -MediaType SSD -ResiliencySettingName Mirror
New-StorageTier -StoragePoolFriendlyName Pool1 -FriendlyName HDD_Tier -MediaType HDD -ResiliencySettingName Parity
$ssd_tier = Get-StorageTier -FriendlyName SSD_Tier
$hdd_tier = Get-StorageTier -FriendlyName HDD_Tier
New-VirtualDisk -StoragePoolFriendlyName Pool1 -FriendlyName "VirtualDisk" -StorageTiers @($ssd_tier,$hdd_tier) -StorageTierSizes 100GB, 300GB
