Bagaimana cara mempertahankan penyorotan kecocokan grep saat menyalurkan pencarian ke grep?

Jika saya menjalankan grep foo bar.txt, grep menyoroti setiap kemunculan "foo" di bar.txt. Tetapi terkadang saya ingin menggunakan find untuk menentukan file mana yang dicari oleh grep. Jadi saya melakukan sesuatu seperti ini:

find . -iname "*.abc" | xargs grep foo

Atau ini:

find . -iname "*.abc" -exec grep foo {} \;

Dalam kedua kasus tersebut, grep dengan benar menemukan kemunculan "foo" dalam file yang ditentukan, tetapi keluarannya tidak memiliki sorotan sama sekali.

Bagaimana saya bisa tetap menggunakan find untuk memilih file untuk pencarian grep tanpa kehilangan sorotan?

Saya menjalankan Gnome Terminal 3.4.1.1 di Ubuntu 12.04, dengan bash sebagai shell saya.

Di kedua perintah, masalahnya bukan grep tetapi perintah lain menghasilkan keluaran aktual (masing-masing xargs dan find).

Anda dapat menyelesaikan ini dengan langsung menelepon grep untuk setiap nama file:

IFS=$'\n'
for FILE in $(find . -iname "*.abc"); do
    grep foo $FILE
done
unset IFS

Atau sebagai satu baris:

IFS=$'\n';for FILE in $(find . -iname "*.abc");do grep foo $FILE;done;unset IFS

Bagaimana itu bekerja:

  • IFS=$'\n' mengatur pemisah bidang internal ke karakter baris baru (atau spasi dalam nama file akan menyebabkan masalah).

  • for FILE in $(COMMAND); do COMMANDS done memutar melalui file yang ditentukan oleh COMMAND, menetapkan variabel FILE ke file saat ini dan menjalankan COMMANDS.

  • grep foo $FILE mencari foo pada $FILE dan mengirimkan hasilnya langsung ke layar.

  • unset IFS menyetel pemisah bidang internal kembali ke nilai defaultnya (tidak diperlukan dalam skrip).
