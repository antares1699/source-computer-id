Saat menulis mis. skrip shell, saya ingin mengubah izin mereka (terutama file dapat dieksekusi izin) dari dalam Sublime Text 2.

Bagaimana saya bisa mencapai itu?

Berikut ini adalah perintah pengeditan izin tujuan umum untuk file yang saat ini sedang diedit. Untuk penjelasan lebih detail tentang plugin dan mengedit menu Sublime Text 2, lihat posting ini .

Ini akan menambahkan file Ubah Mode perintah di Sunting Tidak bisa. Saat dipilih, pengguna akan diminta untuk memasukkan string argumen yang valid ke chmod (misalnya u+rwx atau 755; defaultnya adalah string izin oktal 4 digit yang saat ini disetel seperti 0644), yang kemudian diterapkan ke file yang sedang diedit.

Pilih Alat »Plugin Baru , masukkan konten berikut dan simpan sebagai chmod.py di ~/Application Support/Sublime Text 2/Packages/User/:

import sublime, sublime_plugin, subprocess

def chmod(v, e, permissions):
    subprocess.call( [ "chmod", permissions, v.file_name() ] )

def stat(filename):
    proc = subprocess.Popen( [ "stat", "-f", '%Mp%Lp', filename ], stdout=subprocess.PIPE )
    return str(proc.communicate()[0]).strip()

class ChangeModeCommand(sublime_plugin.TextCommand):
    def run(self, edit):
        if sublime.platform() != 'osx':
            return

        fname = self.view.file_name()

        if fname == None:
            sublime.message_dialog("You need to save this buffer first!")
            return

        perms = stat(fname)

        def done(permissions):
            chmod(self.view, edit, permissions)

        sublime.active_window().show_input_panel(
            "permissions to apply to the file " + fname + ": ", perms, done, None, None)

Untuk menyisipkan item menu untuk perintah ini, tambahkan berikut ini ke ~/Application Support/Sublime Text 2/Packages/User/Main.sublime-menu, gabungkan dengan konten file yang ada jika file sudah ada:

[
    {
        "id": "edit",
        "children":
        [
            {"id": "wrap"},
            { "command": "change_mode" }
        ]
    }
]

Ini pada dasarnya bekerja di Linux juga, tetapi perintah stat bekerja secara berbeda dan menunjukkan banyak informasi yang tidak diperlukan.

stat -c %a filename

akan melakukannya dan mengembalikan sesuatu seperti '644'.
