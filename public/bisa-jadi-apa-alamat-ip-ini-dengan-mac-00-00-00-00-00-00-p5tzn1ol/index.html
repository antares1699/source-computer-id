Bisa jadi apa alamat IP ini dengan MAC: 00: 00: 00: 00: 00: 00?

Saya memiliki cukup banyak pengaturan di luar kotak dari openwrt, meskipun saya mengikuti 'mengamankan panduan router Anda', jadi set kata sandi, ssh pada port yang berbeda (jadi keamanan melalui ketidakjelasan). Namun, saya melihat beberapa alamat IP aneh yang terhubung ke antarmuka WAN:

root@OpenWrt:~# cat /proc/net/arp
IP address       HW type     Flags       HW address            Mask     Device
192.168.1.67     0x1         0x0         00:00:00:00:00:00     *        eth0.2
192.168.1.115    0x1         0x0         00:00:00:00:00:00     *        eth0.2
192.168.1.4      0x1         0x0         00:00:00:00:00:00     *        eth0.2
192.168.1.1      0x1         0x2         00:1f:90:XX:XX:XX     *        eth0.2
...

Saya tahu 192.168.1.1 adalah router ISP saya yang saya hubungkan dengan router OpenWRT saya, tetapi saya tidak yakin apa yang lainnya. Juga mereka memiliki alamat MAC yang aneh. Adakah yang tahu apa itu? Set top box saya memiliki alamat 192.168.1 tetapi saat ini bukan salah satunya. Mungkinkah itu telah diberikan alamat IP yang berbeda pada waktu yang berbeda dan berakhir di tabel ARP saya?

Saya menggunakan BARRIER BREAKER (14.07-rc3, r42056)

4

8

Bendera 0x0 dan alamat HW 00:00:00:00:00:00 berarti ARP gagal.

Untuk menguji ping alamat IP yang tidak digunakan di jaringan lokal dan kemudian relist. Alamat IP itu sekarang juga harus ditampilkan dalam keluaran cat /proc/net/arp. Bisa juga menggunakan perintah arp yang seharusnya menunjukkan (tidak lengkap) untuk HWaddress untuk alamat IP yang dimaksud.
