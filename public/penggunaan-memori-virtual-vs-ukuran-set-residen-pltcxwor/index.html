Penggunaan memori virtual vs ukuran set residen

Ketika saya meluncurkan php-script dan melakukan ps aux dan memiliki jumlah yang sangat besar untuk VSZ, sekitar 450MB. Sementara RSS - Ukuran set penduduk - sekitar 10MB. Dari wikipedia, Ukuran set penduduk adalah is the portion of a process's memory that is held in RAM. Tetapi apakah ukuran memori virtual mencakup memori yang disimpan di tempat lain, misalnya di sistem file? Apa sebenarnya yang terdiri dari memori Virtual?

Memori virtual (seperti namanya) adalah abstraksi atas arsitektur memori fisik yang mendasarinya. Setiap proses mendapatkan ruang alamat virtualnya sendiri, dan ketika alamat virtual diakses, mereka pertama kali diubah ke alamat perangkat keras oleh Unit Manajemen Memori (MMU). Data dalam ruang alamat virtual dari suatu proses dapat berada di memori utama, pada disk, keduanya atau beberapa memori eksternal lainnya. Namun, setiap halaman yang belum semuanya ada di memori utama (MM) dimuat ke MM ketika mereka diakses oleh suatu proses. Dengan demikian, proses tidak membaca langsung dari disk, tetapi beberapa atau sebagian besar sumber dayanya dapat berada di memori eksternal pada waktu tertentu, karena halaman dapat ditukar untuk memberi tempat bagi yang lain.

Memori virtual mungkin:

  • pada RAM jika data baru saja diakses atau jika tidak ada tekanan untuk melepaskan RAM
  • di area swap (partisi swap atau file swap) jika sudah keluar halaman
  • pada sistem file jika datanya sesuai dengan file yang dipetakan memori (misalnya: perpustakaan bersama)
  • tidak ada di mana pun (yaitu, tidak menggunakan sumber daya, terutama pada sistem yang melakukan over-commit memory seperti Linux) jika halaman yang dialokasikan belum diakses

Poin keempat sering diabaikan.
