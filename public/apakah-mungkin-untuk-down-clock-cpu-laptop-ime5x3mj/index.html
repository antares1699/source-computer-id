  • Status prosesor minimum saya adalah 0%. Jika ini benar-benar berarti Windows dapat mengurangi CPU ke 0Hz saat idle, itu berarti CPU tidak dapat menjalankan instruksi lagi - lalu bagaimana cara kembali ke keadaan normal !?
    — BlueRaja - Danny Pflughoeft
  • @BlueRaja kasus itu akan benar jika status prosesor maksimum dapat disetel ke 0%. Memiliki batas minimal berarti bisa penurunan serendah itu, tetapi tidak pernah harus tetap di sana. Menetapkan Maksimum yang rendah akan menjadi sesuatu yang secara teori dapat mencegahnya berjalan. Tapi kondisi minimum ... itu tidak akan mencegahnya berjalan. Yang saya ingin tahu, apakah yang terjadi jika minimum lebih tinggi dari 0%? Apa yang dilakukan komputer KEMUDIAN saat menganggur? Apa yang dilakukan untuk mempertahankan CPU setidaknya status prosesor minimum?
    — Bon Gart
  • @Bon Untuk menjawab pertanyaan Anda: en.wikipedia.org/wiki/Idle_%28CPU%29 tetapi Anda tampaknya salah paham dengan pertanyaan saya. Jika CPU berjalan pada 0Hz, itu tidak dapat dijalankan apa saja instruksi. Itu berarti OS tidak bisa melakukannya apa pun . Sama sekali. Tetapi jika OS tidak dapat melakukan apa-apa, bagaimana cara menginstruksikan CPU untuk meningkatkan kecepatan clock lagi? Pasti ada interupsi, atau sesuatu, yang berarti jam tidak benar-benar berjalan pada 0Hz. Saya merasa bahwa mengurangi maksimum tidak benar-benar mengurangi kecepatan clock, ini hanya mengurangi waktu berjalan minimum dari proses idle ..
    — BlueRaja - Danny Pflughoeft
  • @BlueRaja Meskipun 0% mungkin tidak selalu berarti 0Hz, itu tidak terbayangkan. OS mungkin mengkonfigurasi pengatur waktu untuk menyebabkan interupsi, katakanlah, 100 kali per detik, dan kemudian menghentikan CPU (yang mendekati 0Hz). Ketika interupsi terjadi, ia memeriksa apakah ada pekerjaan yang harus dilakukan, dan jika demikian, memperbarui frekuensi clock.
    — RomanSt
