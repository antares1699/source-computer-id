ClamAV dirancang untuk menjadi file sesuai permintaan pemindai, dan hanya akan berjalan saat Anda menjalankannya (yang berarti Anda mungkin tidak akan melihat prosesnya jika Anda tidak menjalankannya secara manual). Juga, ketika Anda menginstal file clamav paket, Anda hanya menginstal garis komando pemindai dan mesin pemindai.

Jika Anda ingin menginstal paket lengkap ClamAV, saya sarankan Anda menggunakan ini

sudo aptitude menginstal clamav clamav-daemon clamav-freshclam clamtk

Rincian paket apa melakukan apa:

  • clamav - mesin dan pemindai virus ClamAV berbasis baris perintah
  • clamav-daemon - memungkinkan ClamAV menjadi file dalam akses scanner, yang berarti berjalan secara otomatis tanpa campur tangan Anda
  • clamav-freshclam.dll - mengaktifkan pembaruan otomatis untuk ClamAV
  • clamtk - Antarmuka GUI untuk ClamAV

Dengan semua paket ini diinstal, ClamAV akan bekerja seperti kebanyakan paket AV lainnya. Seperti yang dikatakan alex, setelah Anda menginstal paket-paket ini, jalankan ps seharusnya memungkinkan Anda untuk melihat daemon ClamAV berjalan.

Utas ini agak tua, tetapi tampaknya relevan karena clamav masih agak canggung untuk dipahami.

Aspek akses clamav adalah clamonacc

Anda harus menjalankannya & kemudian mengawasi logfile-nya.

Ini membutuhkan clam-daemon berjalan, karena ia memonitor akses file melalui api kernel faninotify dan AIUI kemudian meneruskan permintaan tersebut ke daemon untuk dipindai.

clamonacc menggunakan clam.conf tetapi tampaknya tidak menghormati pengaturan yang dikonfigurasi melalui clamTK (misalnya, untuk tidak memindai dotfiles).

clamonacc tidak dijalankan secara otomatis & saya belum melihat peluncur layanan untuk itu (mungkin perlu membuatnya).

Saya mengalami kesulitan menemukan hubungan yang benar antara pemilik proses clam-daemon, proses clamonacc & akun pengguna desktop, seperti yang saya inginkan semua file yang dibuat oleh pengguna untuk diperiksa, bukan hanya yang kebetulan berada di grup yang sama dengan clam-daemon.

Mungkin ini karena desain utama clamav untuk pemindaian server, tetapi saya mencoba mengonfigurasi ini sebagai pengguna akhir & tampaknya jauh lebih kasar daripada yang saya harapkan (bahkan setelah sepuluh tahun lagi dari pertanyaan OP)
