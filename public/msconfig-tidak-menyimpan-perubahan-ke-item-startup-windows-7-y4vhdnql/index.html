Saya pernah mengalami situasi yang sama. Coba ini.

Dua program yang saya gunakan adalah: Penjelajah Proses Windows , dan Armor Online Gratis oleh Emsisoft. Windows Process Explorer adalah program mandiri yang akan memberi Anda informasi mendetail tentang proses yang berjalan di latar belakang. Online Armor adalah suite keamanan (uji coba 30 hari; tetapi setelah itu memungkinkan Anda memilih versi lengkap atau mode freeware terbatas) yang memiliki penggunaan memori rendah untuk saya dan tidak terlalu memperlambat startup, tetapi memerlukan beberapa pelatihan jika Anda ingin menggunakan semua itu. fitur keamanan. Saya sarankan untuk mendapatkan keduanya.

Windows Process Explorer menunjukkan kepada saya proses induk dan proses anak apa pun yang terkait dengan pelanggar autorun jadi saya tahu apa yang harus dicari di tab Autoruns Online Armor Free. Tab Autoruns akan mencantumkan berbagai dll, drv, sys, dan exe yang terkait dengan program dan layanan startup. Di sana, Anda dapat memperoleh lebih banyak informasi tentang file tersebut untuk menemukan kunci registri jika Anda mau, tetapi kunci registri terkadang dapat dihidupkan kembali. Yang perlu Anda lakukan hanyalah Memblokir proses induk atau proses anak, tergantung mana yang melakukan apa yang Anda inginkan.

Misalnya, di Windows Process Explorer, proses induk untuk TeamViewer adalah TeamViewer_Service.exe dan salah satu proses anaknya adalah TeamViewer.exe. Jika saya pergi ke tab Autoruns di Online Armor Free, itu akan menampilkan TeamViewer_Service.exe dan saya dapat mengaturnya ke Block. Maka itu tidak akan berjalan otomatis saat startup, bahkan jika file atau layanan lain mencoba untuk mengaktifkannya kembali dalam autorun.

Online Armor menangani autoruns yang tidak diinginkan dengan memblokir autorun itu sendiri; itu tidak bertindak sebagai sakelar on / off untuk autoruns (tidak seperti beberapa program di luar sana). Memastikan bahwa jika autorun diaktifkan kembali saat reboot, autorun masih tidak akan melakukan apa pun kecuali autorun muncul dengan nama yang berbeda; tetapi bahkan ini dapat ditangani dengan cara lain. Ini telah menyelamatkan saya dari masalah bahkan dengan program yang mengaktifkan kembali autorun jika saya menjalankan program secara manual (jika saya perlu menggunakannya). Dan itu telah bekerja bahkan pada layanan startup yang keras kepala yang hampir tidak ada yang berhasil.

Meskipun jawaban ini mungkin kualitasnya lebih rendah, dan Anda mungkin sudah membahasnya, saya tidak melihat ini disebutkan. Demi kelengkapan:

Pertama, pastikan antivirus, antimalware, atau apa pun yang tidak melindungi startup / layanan Anda. Terkadang aplikasi ini membuat "sangkar" untuk mencegah malware menyapu registri / startup Anda. Jika Anda menemukan ini masalahnya, coba nonaktifkan Anda dari dalam aplikasi pelindung.

Selanjutnya, dalam pengaturan (atau beri tahu ikon) untuk aplikasi yang ingin Anda hentikan saat boot: pastikan untuk mematikan "mulai aplikasi ini dengan windows". Setidaknya beberapa dari mereka harus memiliki pengaturan untuk ini. Membiarkan setelan untuk "aktif" ini di tingkat aplikasi terkadang dapat memaksa msconfig untuk mematuhinya.

Sekarang, bersama dengan beberapa aplikasi bagus yang disarankan di sini, ccleaner memiliki kemampuan untuk sebenarnya menghapus entri startup sepenuhnya. Menghapus centang untuk "menonaktifkan" adalah taruhan yang lebih aman, tetapi saya telah menemukan bahwa di masa lalu, menghapusnya sepenuhnya cenderung cukup aman untuk kegagalan (dengan asumsi bahwa pengaturan aplikasi mereka mengatakan tidak ada proses saat boot). Cadangkan registri Anda atau buat titik pemulihan, lalu hapus startupnya.

Alasan di balik ini adalah bahwa menonaktifkan meninggalkan kunci yang ada, sedangkan menghapus kunci sering membutuhkan perangkat lunak menginstal ulang untuk memasukkannya kembali. Jika itu sesuatu yang tidak akan Anda lewatkan, hapuslah. Jika masih kembali, ini bisa menjadi tanda bahwa aplikasi yang mendasarinya melindungi (atau membajak) boot winder Anda.

Bukan mencari suara, hanya ingin menambah teori lengkap pada jawaban yang sudah bagus.
