Bagaimana cara mengosongkan memori virtual di Ubuntu?

Statistik terkini menegaskan bahwa:

100Mb RAM digunakan dari 312Mb dan 31Mb dari Virtual Mem juga digunakan.

Pokoknya saya bisa menghentikan OS menggunakan memori virtual, bukan RAM? Saya tahu bahwa ini terjadi ketika RAM menjadi jenuh, dan OS tidak memiliki tempat lain untuk menyimpan data selain hard disk. Tapi anggota Virtual sepertinya tidak turun, malah sepertinya dia lebih suka tinggal di sana.

Saya tidak yakin apakah ini masalah atau tidak, karena mungkin itu tidak selalu berarti bahwa mem virtual digunakan selama ram.

Bantuan akan sangat dihargai! Terima kasih! :)

Lihat nilai swappiness Anda di /proc/sys:

$ cat /proc/sys/vm/swappiness
60

Anda dapat memaksa Ubuntu untuk tidak menggunakan swapfile hingga benar-benar diperlukan dengan mengubah nilai ini menjadi nol:

$ sudo sysctl vm.swappiness=0
vm.swappiness = 0

Mengutip Anda:

100Mb RAM digunakan dari 312Mb dan 31Mb dari Virtual Mem juga digunakan.

Anda belum menyebutkan penggunaan SWAP dan hanya menyebutkan bahwa 1/3 dari RAM Anda digunakan.

Satu hal penting yang harus diingat pada sistem Linux, adalah bahwa RAM Anda akan sering kali terlihat "lebih banyak digunakan" daripada yang sebenarnya karena Kernel akan menggunakan ram untuk apa pun yang dapat membuat hidup Anda lebih cepat (mis .: informasi sistem file / inode cache / etc ...). Ini tidak terlalu bermasalah karena informasi yang disangga ini dianggap sebagai warga negara kelas dua dan akan dihancurkan jika perlu ..
