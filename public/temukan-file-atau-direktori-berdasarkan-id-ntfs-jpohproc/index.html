Temukan file atau direktori berdasarkan id NTFS

Katakanlah saya menerima output berikut dari chkdsk:

Deleted invalid filename Flatland: A Romance of Many Dimensions (88216) in directory 403737.

Apakah ada cara untuk menemukan nama direktori ini dengan id 403737, dan lokasinya saat ini? (dan pertanyaan terkait, apa nama yang tepat untuk nomor "ntfs id" ini?)

4
IQAndreas — Sumber
  • Kedengarannya seperti nomor inode. Dalam POSIX Anda tidak dapat mencari file dengan nomor inode untuk menemukan entri / entri direktori yang mengarah padanya. Anda hanya dapat mencari entri direktori untuk menemukan nomor inode-nya. MS Windows sepertinya serupa dalam hal ini.
    — Celada
  • Saya berharap chkdsk melaporkan nomor referensi file, tetapi saya juga berharap mereka jauh lebih besar jadi saya tidak yakin. Coba fsutil file queryfilenamebyid c:\ 403737
    — Harry Johnston

Dengan asumsi itu Nomor Rekam MFT , Anda dapat menggunakan alat MS bernama nfi untuk membuang informasi ntfs dan kemudian utilitas seperti grep atau awk ( perlu diinstal / bukan kemenangan asli ) untuk memfilter hasil yang Anda minati saja.
nfi dapat diunduh sebagai bagian dari Alat OEM

nfi c: |awk "/File 72\r/{c=4;{print}next}c-->0" menghasilkan output berikut di laptop saya:

Berkas 72
\ WINDOWS \ Temp
$ STANDARD_INFORMATION (penduduk)
$ FILE_NAME (penduduk)
$ INDEX_ROOT $ I30 (penduduk)

Perlu diketahui bahwa ini adalah pencarian linier 'brute force' jadi mungkin butuh waktu lama untuk menemukan entri dengan angka tinggi.

bagi mereka yang mungkin mencari tahu tentang ini di masa mendatang: Saya baru tahu bahwa di Linux, ada yang bisa

ntfsls -iaR /dev/sdb1

-saya = show inode (mis., record MFT no.), -Sebuah = tampilkan semua (tidak tahu apa fungsinya tetapi tidak membahayakan), -R = recurse ke subdirektori, / dev / sdb1 adalah file perangkat partisi atau file image NTFS.

Ini menghasilkan daftar file yang bagus dan panjang serta nomor catatan MFT mereka.

ntfsls ada dalam paket ntfsprogs (setidaknya di Fedora.)
