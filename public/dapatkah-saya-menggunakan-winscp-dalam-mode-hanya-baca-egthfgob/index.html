Dapatkah saya menggunakan WinSCP dalam mode hanya-baca

Saya menggunakan WinSCP untuk melihat sistem file melalui FTP.

Sebagian besar waktu, saya hanya ingin melihat file di sistem file - tetapi terkadang saya perlu mengubah atau menghapus file.

Apakah ada pengaturan di WinSCP (atau klien Windows FTP umum lainnya) yang memungkinkan saya memilih apakah akan mengakses sistem file FTP dengan izin baca-saja atau baca-tulis.

Ini akan memungkinkan saya untuk menghindari perubahan yang tidak disengaja pada sistem file FTP.

Jika saya mengajukan pertanyaan dengan cara yang salah, maaf - saya kira ungkapan yang paling umum adalah "bagaimana saya dapat melihat sistem file FTP tanpa izin menulis?" Tapi saya cukup terikat dengan WinSCP, jadi yang ideal adalah solusi yang melibatkan itu.

TERPECAHKAN

Buat file .ftpaccess di direktori yang harus bersifat read-only untuk akun namapengguna@domainanda.com.

Isi dari file .ftpaccess:

<Limit WRITE>
DenyUser username@yourdomain.com
</Limit>

Itu saja.

Tolong dicatat:

  • Perintah dalam file .ftpaccess bersifat rekursif (yaitu mempengaruhi semua file di direktori saat ini dan setiap subdirektori).

  • Syaratnya, server mendukung ProFTPD.

Lebih lanjut tentang ProFPTD di sini:

http://www.proftpd.org/docs/howto/Limit.html

Peringatan: ProFTPD cukup kuat. Jika Anda melakukan kesalahan dan karenanya Anda tidak dapat mengedit file Anda melalui FTP lagi, Anda masih dapat mengedit file .ftpaccess melalui baris perintah.
