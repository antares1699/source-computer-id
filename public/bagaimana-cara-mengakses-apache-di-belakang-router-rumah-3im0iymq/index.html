Bagaimana cara mengakses Apache di belakang router rumah?

Saya memiliki IP publik 89.103.xxx.yyy , lalu router dan jaringan rumah dengan 192.168.1.zz Router saya disetel ke portforward every 8056 di alamat ip 192.168.1.5 (PC saya), tempat apache berjalan. Mendengarkan Apache diatur ke:

Listen 127.0.0.1:80
Listen 192.168.1.5:8056

Ketika saya mencoba:

Listen 89.103.xxx.yyy:8056

Saya mendapatkan pesan "akhir apache tak terduga" (Saya menggunakan easyphp untuk windows).

Ditambahkan : Ketika saya mencoba mengakses 89.103.xxx.yyy melalui browser, itu meminta router meminta otorisasi untuk mengakses pengaturan web router. Dan ketika saya mencoba 89.103.xxx.yyy: 8056, browser mengatakan bahwa koneksi habis waktu = (

Kamu tidak butuh untuk mengubah port Apache mendengarkan sama sekali! Dalam kasus saya, Apache mendengarkan di port 80 secara normal.

Kamu harus:

  1. Konfigurasikan semua router dan modem (beberapa modem memiliki router internal, seperti DLINK) untuk menerima beberapa port yang tidak biasa sebagai '8081' ke '8082' DAN mengalihkan permintaan tersebut ke IP tempat server apache Anda berjalan, di port 80. Hindari menggunakan 8080, 8081 atau lebih tinggi.

  2. Letakkan host (kedua jaringan) di DMZ (host di "ZONA DEMILITARIZED", dengan kata lain, di luar kebijakan)

Ada 2 jaringan dalam kasus saya:

  • 10.1.1.1 (modem)
  • 192.168.1.1 (router nirkabel)

Contoh:

Misalkan router modem yang jaringannya 10.1.1.1 dan Apache PC AT 10.1.1.5. Anda harus mengkonfigurasi NAT - Server Virtual yang mengakses 10.1.1.1 / NAT / Server Virtual:

PortStart......PortEnd......type....... dest IP........ dest PORT
...8081..........8082........TCP...... 10.1.1.5............... 80

DMZ Host: 10.1.1.5 ← UP yang digunakan oleh jaringan Apache PC. Untuk mendapatkannya, jalankan ipconfig pada prompt cmd.

Untuk router sekunder (nirkabel) yang jaringannya 192.168.1.1: DMZ Host: 192.168.1.10 ← IP dari PC yang menjalankan Apache di subnet router nirkabel.

Untuk melihat pratinjau situs, jangan gunakan nirkabel rumah Anda atau atau komputer yang terhubung ke jaringan rumah Anda, karena yang akan Anda lihat hanyalah halaman router modem. Anda harus mengakses situs Anda dari di luar jaringan rumah (mis .: gunakan internet GSM seluler Anda, jangan pernah WiFi).

Skema untuk mengakses situs, jadi, misalkan IP internet saat ini adalah 199.123.4.75:

  • http://199.123.4.75:8081 atau

  • http://199.123.4.75:8081/mywebsite

  • http://199.123.4.75:8082 atau

  • http://199.123.4.75:8082/mywebsite

Ingat, Apache masih mendengarkan di port 80. Yang melakukan keajaiban adalah router, terutama yang pertama (modem) yang mengarahkan port. NAT - Server Virtual. DMZ juga penting. Untuk kedua konfigurasi router.
