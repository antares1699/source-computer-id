Mencegah proses latar belakang yang sudah berjalan dihentikan setelah menutup klien SSH

Saya memiliki proses yang berjalan di mesin Linux saya (pemerasan Debian) yang membutuhkan waktu berjam-jam (atau berhari-hari) untuk menyelesaikannya.

Saya tidak ingin menghentikannya untuk memulai ulang lagi dengan screen, tmux atau dengan pengalihan keluaran ke nohup.

Apakah ada cara yang aman untuk meletakkannya di latar belakang dengan ^Z dan bg sehingga akan dilanjutkan setelah saya menutup sesi ssh?

1 Jawaban

7
7

menggunakan

commandtoexecute &> /dev/null &

itu akan menjalankan proses Anda di latar belakang, dan mencetak semua output ke / dev / null.

Ganti /dev/null dengan file lain untuk melihat hasilnya.

misalnya commandtoexecute &> /tmp/file1 &

menggunakan tail -f /tmp/file1 untuk dipasang ke keluaran lagi

Anda juga dapat mengarahkan stdin, lihat ini http://www.tuxfiles.org/linuxhelp/iodirection.html

Jika Anda ingin melepaskan dari proses yang semuanya sudah berjalan. Menggunakan disown <pid> di mana pid adalah id proses Anda.

Anda juga dapat mengubah terminal ke terminal lain:

  1. memulai layar
  2. dapatkan pid dari proses Anda
  3. jalankan reptyr <pid>
  4. lepaskan menggunakan CTRL + A + D

reptyr: https://serverfault.com/a/284795
