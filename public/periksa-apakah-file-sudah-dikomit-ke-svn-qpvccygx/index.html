Periksa apakah file sudah dikomit ke svn

Bagaimana cara memeriksa apakah file / folder sudah dikomit ke svn? Saya memiliki skrip shell di mana saya ingin memastikan bahwa file / folder yang dipilih oleh pengguna untuk check-in belum dilakukan. Misalnya jika saya mencoba check-in folder yang sudah berkomitmen menggunakan

svn tambahkan folder saya

Saya mendapatkan peringatan berikut:

svn: peringatan: 'myfolder' sudah di bawah kendali versi

4

Gunakan kode pengembalian svn info. Ini 0 jika di bawah kontrol versi, bukan nol sebaliknya.

Shell saya menampilkan kode pengembalian bukan nol setelah perintah, jadi terlihat seperti ini:

$ svn info trunk
Path: trunk
URL: https://(...)/trunk
Repository Root: https://(...)
Repository UUID: 651713a4-5a46-7e42-a99e-f31e79777eab
Revision: 213
Node Kind: directory
(...)

$ touch foo
$ svn info foo 
foo:  (Not a versioned resource)

svn: A problem occurred; see other errors for details

rc: 1

Bagaimana Anda melakukannya dalam skrip bash (serupa di shell lain), dengan menekan keluaran info:

svn info <filename> 1>/dev/null 2>&1
echo $?

Saya menggunakan ini:

if [ $(svn status $file | awk '{ print $1 }') == "?" ]; then
    echo "File is not under version control"
fi

Dengan cara ini Anda juga dapat menanyakan kode status svn yang berbeda yang dapat dilihat di sini: http://svnbook.red-bean.com/en/1.7/svn.ref.svn.c.status.html

Saya ingin memecahkan masalah serupa: Sebagai bagian dari skrip yang menghasilkan file, secara otomatis menambahkannya ke subversi jika ini belum pernah dilakukan sebelumnya.

Saya menemukan yang berikut berhasil:

if ! svn info ${file} 1>/dev/null 2>&1 ; then
  svn add ${file}
fi

Menguji nilai keluar di Bash membawa saya ke jawaban ini.
