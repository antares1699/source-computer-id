Saya sudah menanyakan pertanyaan di stack overflow, tetapi tidak mendapatkan jawaban apapun dari siapapun. Jadi saya juga menanyakan pertanyaan yang sama di sini.

Saya mencoba membuat program yang akan menghitung ukuran semua folder yang ada di lokasi yang ditentukan atau lokasi saat ini menggunakan skrip batch dan saya menulisnya ke dalam file .csv

Saya mengikuti pertanyaan ini di stackoverflow

https://stackoverflow.com/questions/21711180/how-to-list-all-folder-with-size-via-batch-file

dan kode saya

`@echo off
 setlocal disabledelayedexpansion
 set "folder=%~1"
 Set "Value=0"
 if not defined folder set "folder=%cd%"
  (for /d %%a in ("%folder%\*") do (
    set "size=0"
    for /f "tokens=3,5" %%b in ('dir /-c /a /w /s /x "%%~fa\*" ^| findstr /b /c:"  "')do if "%%~c"=="" set "size=%%~b"
   setlocal enabledelayedexpansion
   echo(%%~nxa , !size!
   endlocal
  ))>>foldersize.csv 
endlocal
exit /b`

Pada partisi NTFS, panjang jalur maksimum adalah sekitar 32.760 karakter (sedikit kurang dari 32.767). Namun, MAX-PATH masih 260. Jalur yang lebih panjang dari "MAX-PATH" masih dapat diakses dan disebut sebagai "jalur yang diperpanjang".

Menurut halaman ini: Batasan Panjang Jalur Maksimum

Untuk menentukan jalur panjang-diperpanjang, gunakan awalan "\\? \". Misalnya, "\\? \ D: \ jalur sangat panjang".

Juga perlu diingat bahwa "jalur panjang-diperpanjang" selalu merupakan jalur "mutlak". Anda tidak dapat menentukan "bentuk relatif" dari "jalur panjang yang diperpanjang".

Saya tidak memiliki jalur \ file yang lebih dari 260 karakter, jadi saya tidak tahu apa yang terjadi dengan jalur "sangat panjang", tetapi seharusnya berfungsi.

Jika Path yang Anda gunakan adalah path UNC, halaman yang ditautkan di atas mengatakan ini:

Awalan "\\? \" Juga dapat digunakan dengan jalur yang dibuat sesuai dengan konvensi penamaan universal (UNC). Untuk menentukan jalur seperti itu menggunakan UNC, gunakan awalan "\\? \ UNC \". Sebagai contoh,
"\\? \ UNC \ server \ share", di mana "server" adalah nama komputer dan "share" adalah nama folder bersama. Awalan ini tidak digunakan sebagai bagian dari jalur itu sendiri.

Contoh agar sesuai dengan jalur UNC yang Anda berikan dalam komentar Anda, harus ditentukan seperti ini:

dir /-c /a /w /s /x "\\?\UNC\10.0.0.10\folder1\folder2" | findstr /b /c:" "
