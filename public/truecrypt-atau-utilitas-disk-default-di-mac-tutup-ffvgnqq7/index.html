Truecrypt atau Utilitas Disk default di Mac? [Tutup]

Windows secara default tidak datang dengan opsi folder proteksi kata sandi (selain Win7 ultimate), jadi saya biasa bersumpah dengan Truecrypt yang hebat. Tetapi saya telah membaca di beberapa tempat bahwa Mac OS X secara default memiliki cara melindungi folder menggunakan Default Disk Utility .

Jadi pertanyaan saya adalah mana yang lebih baik, menggunakan TrueCrypt di Mac atau hanya tetap menggunakan aplikasi Disk Utils default? Adakah yang bisa memberi tahu saya kelebihan satu sama lain?

Ringkasan dari jawaban yang sangat membantu di bawah ini:

  • jika Anda mencari penggunaan lintas platform, Truecrypt adalah alat pilihan yang jelas
  • jika Anda mencari kenyamanan, dan bermaksud untuk tetap menggunakan platform Mac, gunakan aplikasi Disk Utils default.

Seperti biasa tergantung "lebih baik".

Jika Anda perlu mengakses data terenkripsi hanya di Mac Anda sangat baik diatur dengan DiskUtility DMG Images, atau Sparsebundles dengan enkripsi AES 256. (Saya biasanya merekomendasikan untuk tidak menggunakan SparseImages lagi.)

Anda dapat membuat penggunaan DMG terenkripsi atau SparseBundles sangat nyaman dan transparan dengan menggunakan produk perangkat berbagi yang disebut Spionase .

Jika Anda ingin / perlu lintas platform, TrueCrypt adalah alat pilihan. Ia juga menawarkan lebih banyak pilihan mengenai algoritma enkripsi dan memungkinkan denyability yang masuk akal dengan menggunakan partisi tersembunyi. (Artinya menyangkal bahwa ada data terenkripsi sama sekali.)

Anda bahkan dapat menjadi paranoid total dengan menggabungkan dan mengalirkan kedua teknik tersebut. DiskUtilities SparseBundle secara umum lebih kuat terhadap kerusakan daripada file gambar monolitik SparseImages atau TrueCrypt. Ini mungkin, atau mungkin tidak memengaruhi pilihan Anda.
