Ukuran yang tidak valid ditampilkan untuk SSD

Saya memiliki drive SSD Intel X25-M 80GB yang diinstal pada kotak Linux yang menjalankan Debian 5.0. Sistem file pada SSD adalah ext4. Saya tidak punya masalah dengan drive, dan telah menggunakannya cukup lama sebagai tempat penyimpanan file swap serta tempat untuk menyimpan beberapa file database besar yang memerlukan akses acak.

Namun, saya memperhatikan pembacaan ukuran aneh ini dari ls:

> ls -lh /ssd
total 34G
-rwx------ 1 root root 23G 2010-12-11 18:00 file1.dat
-rwx------ 1 root root 22G 2010-12-11 18:00 file2.dat

Saya mengubah nama file di sini untuk alasan keamanan, tetapi selain itu, inilah keluaran ls. Jelas, pembacaan ukuran total salah.

Berikut adalah output yang sesuai dari df

> df -h
Filesystem            Size  Used Avail Use% Mounted on
/dev/sda1              74G   35G   36G  50% /ssd

Meskipun pembacaan yang salah ini, drive tampaknya berfungsi dengan baik, dan utilitas diagnostik tidak menemukan yang salah dengan drive. Apakah ini semacam bug yang diketahui di ext4, atau lebih mungkin ada sesuatu yang salah secara fisik dengan drive?

Kemungkinan besar juga tidak. Ukuran yang ditampilkan oleh ls adalah panjang file, namun kebanyakan filesystem modern mendukung "Sparse Files" di mana blok kosong sebenarnya tidak ditulis ke disk. Coba jalankan du -h / ssd / * dan lihat ukuran file yang dilaporkan. 'du' akan menampilkan jumlah disk yang sebenarnya digunakan, yang seharusnya cukup dekat (tidak termasuk overhead sistem file & blok cadangan) dengan yang dilaporkan oleh 'df'.
