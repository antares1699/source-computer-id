Apakah Robocopy menyimpan file log secara default?

Saya menyalin sejumlah besar file menggunakan Robocopy, dan menerima catatan pada akhirnya bahwa dua file gagal. Saya tidak mengatur sakelar untuk menyimpan file log.

Apakah ada file log yang disimpan di lokasi default secara otomatis? Saya jelas memilih untuk tidak menyalin ulang semuanya, untuk mendapatkan informasi ini.

Robocopy tidak menyimpan file log default. Namun itu hanya menyalin file dengan perbedaan secara default. Jadi jika Anda mencoba salinan yang sama lagi itu akan melewatkan yang sudah berhasil disalin.

Gunakan tombol V saat menjalankan robocopy

/ V: Menghasilkan log keluaran Verbose, menampilkan file yang dilewati.

Anda mungkin juga ingin memasukkan tombol ini

/ FP: Sertakan Nama Path Lengkap file dalam output

Satu hal yang mungkin berguna bagi Anda adalah mencoba menyalin lagi dengan sakelar /L. Sakelar itu mirip dengan sakelar -n/--dry-run di rsync - ia menghasilkan log tentang apa yang akan terjadi tetapi tidak benar-benar menyalin apa pun. Ini bagus karena jauh lebih cepat, tidak merusak, dan bisa sangat serbaguna bila dikombinasikan dengan sakelar lain seperti /V, /LOG:file dan /LOG+:file.
