Bagaimana saya bisa menghapus lebih cepat? Backspace batas kata?

Saya suka mengetik dengan cepat. Saya seorang delete-keyer kronis. Saya akan mengetik terlebih dahulu, memikirkan kembali hasilnya nanti dan tekan delete. Sepertinya saya kecanduan untuk selalu melakukan sesuatu dengan cepat.

Bagaimanapun, saya harus sering mengetuk tombol hapus. Apakah ada cara untuk melakukan backspace ke batas kata yang asli?

Di kotak obrolan, saya dapat menggunakan alt + shift + kiri untuk mendapatkan sebuah kata, cmd shift kiri untuk mendapatkan seluruh baris, dll. Tetapi saya tidak dapat melakukannya di shell python misalnya.

Ada ide?

⌥⌫ (opsi-hapus) menghapus kata mundur dan ⌥⌦ (opsi-hapus maju) kata maju di sebagian besar tampilan teks.

M-delete menghapus kata mundur dan M-d kata maju secara default di readline. Anda dapat menduplikasi pintasan OS X dengan menggunakan opsi sebagai kunci meta dan menyesuaikan ⌥⌦ ke \033d.

Anda juga dapat menentukan pintasan baru dengan membuat ~/Library/KeyBindings/ dan menyimpan daftar properti seperti ini sebagai DefaultKeyBinding.dict.

{
    "~z" = (selectParagraph:, delete:);
    "~u" = (moveToBeginningOfParagraphAndModifySelection:, cut:);
    "~k" = (moveToEndOfParagraphAndModifySelection:, cut:);
    "@\UF728" = deleteToEndOfLine:;
}

Di terminal OS X, Anda biasanya dapat menggunakan penekanan tombol Emacs.

Sepertinya Anda bisa melakukannya ESC + Menghapus untuk melakukan penghapusan berbasis kata. Apa pun yang mendukung pustaka readline akan menerimanya ( termasuk BASH dan python dari pengujian saya ).

Untuk mempermudah, Masuk ke Terminal → Preferensi → Profil → Keyboard → Klik Gunakan Opsi sebagai Meta Key. Kemudian Anda bisa memukul Alt + Menghapus dan hapus kembali satu kata.

Google untuk GNU Readline untuk mempelajari lebih lanjut tentang bagaimana Anda dapat mengkonfigurasi ini.

Kombinasi yang Anda sarankan adalah asli, tetapi tidak semua program dilengkapi untuk bekerja dengannya, dan karenanya kombinasi tersebut diabaikan.

Tidak ada yang benar-benar dapat dilakukan tentang itu kecuali mendorong pengembang untuk memprogramnya ke dalam aplikasi yang Anda gunakan.

Hal yang sama berlaku untuk pengguna Windows. Kombinasi tersebut bekerja di OS itu sendiri serta sebagian besar program yang diperlengkapi untuk menghadapinya, tetapi terkadang, program akan mengabaikan kombinasi tersebut. Biasanya ini terjadi di tempat-tempat di mana pemilihan teks dianggap sebagai ide sekunder seperti shell terminal dan prompt perintah.
