tail -v file sambil mengecualikan daftar kata

Saya ingin mengikuti file log yang terus-menerus ditulis tetapi ingin mengecualikan berbagai entri yang 'umum' jadi saya hanya melihat kesalahan, dll. Saat mereka dilempar. Saya bisa menyalurkan ke grep -v "pattern" tetapi idealnya ingin menggunakan file yang berisi entri untuk dilewati. Ada ide?

Anda dapat menggunakan grep -v "pattern" seperti yang Anda katakan. Tapi bukannya pattern digunakan

`cat file_with_entires_to_skip`

Ingatlah untuk memasukkan backtics. Jadi perintah Anda akan terlihat seperti:

tail -v FILE | grep -v "`cat FILE_WITH_ENTRIES_TO_SKIP`"

Opsi -f hingga grep memungkinkan seseorang menentukan file yang berisi pola, satu pola per baris. Lihat halaman manual grep(1).

Selain itu, perintah unbuffer yang tersedia di beberapa sistem akan menonaktifkan buffering keluaran yang biasanya terjadi di jaringan pipa. Penambahan filter grep sebaliknya dapat menunda keluaran pipa Anda. Lihat halaman manual unbuffer(1) untuk lebih detil.

Grep saya menyertakan opsi --exclude-from=FILE yang memungkinkan saya menambahkan pola pengecualian dari file. Aku mendapat:

my-macbook-pro:Sun America ianchesal$ grep --version
grep (GNU grep) 2.5.1

Jadi Anda bisa melakukan:

tail -f <file> | grep --exclude-from=excludes.txt
