Perlu menyetel variabel mailx untuk menentukan alamat Dari

Menjalankan Oracle Linux 5.8 (yang hanya bermerek RedHat EL 5.8) Saya harus mengubah alamat Dari. Tetapi kami memiliki skrip yang menggunakan mailx yang tidak dapat ditulis ulang untuk menggunakan flag tambahan, jadi saya ingin menggunakan variabel internal sebagai gantinya, yang saya lihat di halaman manual linux.die.net di mailx adalah alternatif dari -r bendera:

alamat -r
Mengatur alamat Dari. Mengganti dari variabel yang ditentukan dalam file lingkungan atau startup. Pelarian Tilde dinonaktifkan. Opsi alamat -r diteruskan ke agen transfer surat kecuali SMTP digunakan. Pilihan ini ada hanya untuk kompatibilitas; ini disarankan untuk mengatur variabel from secara langsung.

(Sumber: http://linux.die.net/man/1/mailx )

Bagaimana kita bisa menggunakan variabel mailx ini? Saya mencoba menambahkan ini ke /root/.mailrc, no go:

set from=FromAddress@Example.com

Saya juga menambahkannya ke /etc/mail.rc tanpa emas. Jadi saya beralih ke Anda, Pengguna Super ...

Saya baru saja menguji ini di Ubuntu 14.04 dan menggunakan parameter -S berhasil untuk saya:

-S variable[=value]
          Sets the internal option variable and, in case of a string option, assigns value to it.

Jadi, perintah berikut menyetel dari alamat ke "test@exmaple.org":

echo "This is my test" | mail -S from="test@example.org" -s "This is my subject" "receiver@exmaple.org"
