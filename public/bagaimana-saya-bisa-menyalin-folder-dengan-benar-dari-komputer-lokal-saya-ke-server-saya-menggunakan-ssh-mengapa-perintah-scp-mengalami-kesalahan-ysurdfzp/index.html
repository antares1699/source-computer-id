Bagaimana saya bisa menyalin folder dengan benar dari komputer lokal saya ke server saya menggunakan SSH? Mengapa perintah scp mengalami kesalahan?

Saya masuk ke server web saya melalui SSH menggunakan alat bernama MobaXterm (Ini seperti Putty tetapi dengan fitur canggih, ini tidak begitu penting ...)

Masalah saya adalah saya ingin mengirim direktori dari komputer lokal saya (Windows) ke direktori tertentu dari server ini (Linux).

Jadi saya telah mencoba menggunakan scp perintah seperti yang ditunjukkan di sini: http://www.hypexr.org/linux_scp_help.php

Saya telah melakukan dengan cara ini:

-jailshell-4.1$ scp -r C:\Users\Andrea\Documents\Betrivius\candycane-0.9.6\candycane /home6/XXX/public_html

ssh: Could not resolve hostname C: Name or service not known

Tapi, seperti yang Anda lihat, saya mendapatkan kesalahan.

Mengapa? Apa yang saya lewatkan? Bagaimana cara memperbaiki masalah ini?

Agaknya Anda menjalankan ini di shell MobaXterm lokal (yaitu lokal ke mesin windows). Jika demikian, Anda perlu memperbarui sintaks perintah scp untuk mengakomodasi lingkungan itu, meskipun sintaksnya salah.

MobaXterm menyediakan akses ke drive lokal di cangkangnya melalui mountpoint /drives/X, dengan X menjadi huruf drive windows.

Selain itu, sintaks umum (yang disederhanakan) dari scp, terlepas dari lingkungannya adalah:

penggunaan scp:

scp [[user@]host1:]/path[/filename] [[user@]host2:]/path/[filename]

Urutan pertama ([[user@]host1:]/path[/filename]) adalah bagian 'FROM'; sumber file atau file yang akan disalin. Yang kedua adalah 'TO'; tempat tujuan. Tanda kurung menunjukkan argumen opsional.

Ini benar-benar hanya versi ssh yang diaktifkan dari perintah cp standar; dengan bagian sebelum : yang berisi informasi spesifik ssh, dan info setelahnya yang berkaitan dengan operasi cp.

pengguna:

user menggunakan nama pengguna lokal Anda saat ini - dapat ditinggalkan jika itu adalah nama pengguna yang benar baik di sumber maupun tujuan.

tuan rumah:

host default ke komputer lokal (localhost); jadi jika Anda menyalin ke / dari mesin lokal DAN nama pengguna di localhost yang akan digunakan adalah pengguna saat ini, Anda dapat menghilangkan argumen itu untuk file lokal.

file / jalur:

path/filename default ke direktori home pengguna yang ditentukan dan dapat dihilangkan jika diinginkan. nama file dapat ditinggalkan saat melakukan salinan rekursif seperti dalam situasi Anda - cukup berikan jalur dalam kasus itu. Umumnya jalur lengkap dan nama file ditentukan di bagian DARI, dan hanya jalur di bagian KEPADA (defaultnya akan mempertahankan nama file yang sama).

Saya menggunakan remoteuser dan remotePC sebagai nama pengguna dan nama host PC jarak jauh; ganti dengan nama yang benar.

Benar sintaks dalam contoh Anda:

 scp -r "/drives/C/Users/Andrea/Documents/Betrivius/candycane-0.9.6/candycane" remoteuser@remotePC:/home6/XXX/public_html

Gunakan perintah man scp atau situs ini untuk detail tambahan (argumen, info penggunaan lainnya).

Opsi GUI

Sebagai alternatif, MobaXterm menyediakan browser file grafis (dibangun di sftp atau scp berdasarkan opsi konfigurasi) yang menyediakan transfer file dua arah seret dan lepas ketika Anda telah terhubung ke host jarak jauh melalui ssh. Detailnya di sini .

Untuk menjawab

Mengapa perintah scp mengalami kesalahan?

, kesalahannya

ssh: Could not resolve hostname C: Name or service not known

, dan perintah yang digunakan adalah

scp -r C:\Users\Andrea\Documents\Betrivius\candycane-0.9.6\candycane /home6/XXX/public_html

Saya akan mengatakan:

sintaks C:\Users\.... salah. Dalam konteks ini Anda memiliki dua kemungkinan untuk mendeskripsikan jalur:

  1. foo/bar/...: berarti foo adalah direktori (folder) yang terlihat dari jalur Anda, mungkin jalur Anda saat ini (pwd) atau dalam variabel env jalur (echo $PATH).

  2. /foo/bar/... berarti foo adalah direktori (folder) yang ada di root sistem file Anda.

Saat Anda menggunakan opsi pertama (karena tidak ada "/" sebelum "C:"), shell mencari C: di jalur, tetapi tidak dapat menemukannya. Akibatnya, ia mencoba untuk mencocokkan sintaks lain:

ssh: Could not resolve hostname C: Name or service not known
. Dalam situasi ini ssh mengeluh karena ia tidak dapat menemukan alamat IP untuk C: (karena ia mengira bahwa C: adalah nama host). Menemukan alamat IP yang sesuai dengan nama host disebut " resolusi nama host "; di sini resolusi ini gagal, jadi Anda memahami kesalahannya.
