Bagaimana Anda memigrasi lingkungan dev setelah Anda mendapatkan laptop baru?

Apakah ada cara bebas repot untuk bermigrasi dari satu laptop ke laptop lain, mengingat saya akan menggunakan Ubuntu pada keduanya? (Selain menggunakan vm sepanjang waktu? Saya juga mencoba menggunakan vm secara fisik menggunakan Vmware, tetapi pengalaman ini tidak mulus)

Ya, Anda bisa, sampai batas tertentu

ini mengasumsikan bahwa Anda memiliki file /etc/apt/sources.list dan /etc/apt/sources.list.d/ yang sama, yang pertama adalah untuk repositori biasa, dan yang terakhir adalah tempat ubuntu menambahkan AKP. Salin ini dari sistem sumber Anda dan ke sistem tujuan Anda, dan jalankan perintah pembaruan apt-get / aptitude / ubuntu pusat perangkat lunak - lihat jawaban ini di AU untuk lebih jelasnya

Saya telah menyesuaikan instruksi untuk mentransfer daftar paket dari ubuntugeek Anda dapat membuang daftar paket dari sistem sumber ke file teks bernama paket terinstal dengan

sudo dpkg --get-selections | grep '[[:space:]]install$='| awk '{print $1}' > installedpackages

dan salin installpackages ke sistem tujuan kemudian instal ulang dengan cat installedpackages | xargs sudo aptitude install di lokasi paket terinstal di

Menyalin / etc / mungkin merupakan ide yang bagus juga untuk file konfigurasi utama.

Dan tentu saja, salin /home/username untuk mendapatkan salinan lingkungan kerja Anda, seperti folder rumah dan desktop Anda.

Daftar paket membuang dan menginstal ulang diambil dari ubuntugeek.com

Ini harus mencakup sebagian besar file pada instalasi 'desktop' ubuntu. Mungkin ada beberapa hal spesifik yang akan terlewatkan (apache virtualhosts muncul dalam pikiran) tetapi pengaturan terkait paket dan desktop harus dikosongkan.

Anda dapat menyalin seluruh sistem file dari satu partisi ke partisi lain. Misalnya. menggunakan kombinasi dd dan nc .

root@old# dd if=/dev/sda1 bs=4M | nc -l -p 4444
root@new# nc old 4444 | dd of=/dev/sda1 bs=4M

Ini mengasumsikan bahwa Anda menjalankan sistem langsung, mis. dari CD penginstalan, di sistem baru. Dan Anda memiliki nama host atau alamat IP untuk sistem lama, sehingga Anda dapat menyambungkannya melalui TCP. sda1 diasumsikan sebagai partisi dari sistem file linux; sesuaikan seperlunya jika Anda menggunakan partisi yang berbeda, atau beberapa partisi. Jika Anda dapat menghubungkan kedua drive ke mesin yang sama, Anda dapat menggunakan dd tanpa nc.

Setelah menyalin sistem file seperti yang disebutkan di atas, Anda bisa

  1. Sesuaikan ukuran sistem file menggunakan resize2fs jika partisi baru lebih besar.
  2. Pasang sistem file di suatu tempat dan chroot ke dalamnya untuk menginstal bootloader. Anda mungkin harus mount --bind filesystem tambahan seperti /dev dari dalam sistem live.
  3. Sekali lagi di dalam chroot, sesuaikan set driver yang diinstal agar sesuai dengan perangkat keras baru.

Saya telah memigrasi sistem file saya melalui beberapa konfigurasi perangkat keras yang berbeda, termasuk beberapa hdds. Peralihan dari 32bit ke 64bit sedikit lebih menantang, tetapi dengan Gentoo saya bahkan berhasil tanpa harus menginstal ulang sistem saya dari awal.
