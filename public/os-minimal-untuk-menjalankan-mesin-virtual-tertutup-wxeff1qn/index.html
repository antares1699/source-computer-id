OS minimal untuk menjalankan mesin virtual [tertutup]

Apakah ada sistem operasi minimalis yang dibangun murni untuk tujuan menjalankan mesin virtual (lebih disukai open source / jika tidak gratis)?

Jika tidak, katakanlah saya memutuskan untuk melakukan sesuatu seperti Linux Dari Awal . Berapa minimum yang bisa saya lakukan sementara masih bisa menjalankan sesuatu seperti Kotak Virtual (atau middleware mesin virtual yang Anda rekomendasikan)?

Tujuannya adalah untuk memiliki mesin yang dapat saya gunakan murni untuk menjalankan mesin virtual untuk konfigurasi pengujian pada perangkat lunak (paralel atau sebaliknya) dalam pengembangan. Saya ingin melakukan ini tanpa harus menjalankan OS penggunaan umum lengkap di bawah mesin virtual, jika memungkinkan (mencoba untuk mendapatkan performa sebanyak mungkin untuk setiap mesin virtual).

NetBSD sering dianggap cukup "minimalis" karena dilengkapi dengan perangkat lunak tambahan yang sangat sedikit yang telah diinstal sebelumnya, namun saya merasa sangat baik untuk menjalankan Xen untuk memvirtualisasikan banyak Sistem Operasi yang berbeda, termasuk MS-Windows.

Berikut adalah panduan langkah demi langkah gratis yang saya tulis yang menjelaskan cara menjalankan teknologi virtualisasi dengan perangkat lunak open source gratis (termasuk alternatif untuk MS-Windows yang juga gratis dan open source, dan bukan WINE):

Panduan langkah demi langkah untuk menginstal Xen di NetBSD
http://www.lumbercartel.ca/library/xen/

Banyak orang juga memberi saya umpan balik (dan beberapa saran; kredit disertakan di akhir panduan ini ) hal ini. Sejauh ini, panduan saya telah sangat menyederhanakan proses penginstalan Xen dan menjalankannya untuk kebanyakan orang. ( Saya juga menggunakan panduan ini saat menginstal Xen untuk klien saya yang membutuhkan virtualisasi di Unix karena menghemat waktu saya. )

Saya harap panduan ini bermanfaat bagi Anda.
