Server 127.0.0.1:3128 membutuhkan nama pengguna dan kata sandi. Server mengatakan: cntlm untuk orang tua

Saya baru saja menginstal cntlm di Ubuntu 12.04 baru saya. Di file konfigurasi etc/cntlm.conf, saya memasukkan konfigurasi jaringan saya seperti di bawah ini:

Username: myUsername
Domain : myDomain
Password: myPassword

Proxy : myProxy:8080

Listen: 3128

Tetapi ketika saya menetapkan 127.0.0.1:3128 sebagai proxy jaringan browser saya dan ketika saya ingin mengunjungi situs web, menu pop up cntlm muncul dan menginginkan nama pengguna dan kata sandi.

Apa solusinya? Bagaimana saya bisa mendapatkan nama domain jaringan saya? Saya pikir nama domain saya atau keterangan lainnya mungkin salah.

1) Instal CNTLM ke direktori default (di Windows, C: \ Program Files (x86) \ cntlm \).

2) Jalankan cmd.exe

3) Jenis: cd "C: \ Program Files (x86) \ cntlm \"

4) Jenis: cntlm.exe -H -d your_domain -u your_username

Ini akan menanyakan kata sandi Anda. Masukkan kata sandi Anda dan cntlm akan memberi Anda beberapa hash. Sesuatu seperti ini:

c:\Program Files (x86)\Cntlm>cntlm -H -d your_domain -u your_username
Password:
PassLM          4E9C185900C7CF0B6FFCB2044F81920C
PassNT          6E9F120B83EEA0E875CE8E6F9730EC9A
PassNTLMv2      2A0B7C2457FB7DD8DA4EB737C4FA224F

Sekarang Anda memiliki hash kata sandi. Simpan ke editor teks.

5) Jenis: cntlm -M http://www.google.com

Sekali lagi masukkan kata sandi Anda. Ini akan memberi Anda sesuatu seperti itu

c:\Program Files (x86)\Cntlm>cntlm -M http://www.google.com
Password:
Config profile  1/4... Credentials rejected
Config profile  2/4... OK (HTTP code: 302)
----------------------------[ Profile  1 ]------
Auth            NTLM
PassNT          6E9F120B83EEA0E875CE8E6F9730EC9A
PassLM          4E9C185900C7CF0B6FFCB2044F81920C
------------------------------------------------

Sekarang Anda melihat bahwa profil 2 berhasil. Karena dikatakan OK untuk profil 2. Ini mungkin berbeda pada sistem Anda.

Triknya adalah,

  • jika Auth adalah NT, maka Anda harus menggunakan PassNT saja
  • jika Auth adalah LM, maka Anda harus menggunakan PassLM saja
  • jika Auth adalah NTLM, maka Anda harus menggunakan PassLM dan PassNT
  • jika Auth adalah NTLMv2, maka Anda harus menggunakan PassNTLMv2 saja

Sekarang kami mendapatkan semua yang kami inginkan. Untuk konfigurasi saya, Auth mengatakan NTLM jadi saya akan menggunakan PassNT dan PassLM di file konfigurasi cntlm.ini.

Ini adalah contoh file konfigurasi sesuai dengan jawaban yang diberikan:

#
# Cntlm Authentication Proxy Configuration File
#

Username yourusername
Domain yourdomain

Auth NTLM
PassNT 6E9F120B83EEA0E875CE8E6F9730EC9A
PassLM 4E9C185900C7CF0B6FFCB2044F81920C

Workstation yourhostname.yourdomain

# Most probably proxy.yourdomain:8080
Proxy  yourProxyIP:yourProxyPort

NoProxy  localhost, 127.0.0.*, 10.*, 192.168.*

Listen  3132

Gateway yes

Sekarang Anda dapat menggunakan alamat IP dan porta 3132 komputer Anda sebagai proxy.

Catatan 1: Jangan gunakan domain \ nama pengguna sebagai nama pengguna Note 2: Jika ada maka jangan lupa gunakan extension '.intra' setelah domain anda (yourdomain.intra)

Di file cntlm.conf, pastikan untuk mengganti bidang Password dengan salah satu bidang berikut:

  • PassLM (jika Anda menggunakan hash LM)
  • PassNT (jika Anda menggunakan hash NT)
  • PassNTLMv2 (jika Anda menggunakan hash NTLMv2)

Kemudian, Anda tidak boleh menulis sandi teks biasa, tetapi hash yang sesuai, yang dapat dihitung dengan cntlm -H -d <domain> -u <user>

Saya memiliki masalah yang sama.

Pertama-tama, coba perintah ini:

sudo cntlm -M http://www.google.com

Lihat hasilnya. Jika satu-satunya metode baik aku s Auth = LM , maka Anda perlu mengubah file konfigurasi di (hapus tanda komentar pada baris ini):

Auth         LM
Flags        0x06820000

Jika Anda tidak tahu cara mengubah file konfigurasi, buat ini:

gksudo gedit /etc/cntlm.conf

Ingatlah untuk memulai ulang CNTLM:

sudo service cntlm restart
