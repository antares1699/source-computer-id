Mengapa ada Windows 8 edisi 32-bit? [Tutup]

CPU 64-bit telah ada selama sekitar 10 tahun sekarang, mungkin lebih. Sebagian besar komputer memiliki CPU seperti itu (menurut saya CPU 32-bit bahkan tidak diproduksi lagi).

Dan ada banyak komputer yang memiliki RAM lebih dari 4GB (terutama di sektor konsumen), yang OS 32-bitnya tidak dapat bekerja (mari kita abaikan PAE demi percakapan).

Dalam kondisi seperti ini, mengapa ada Windows 8 versi 32-bit? Semua edisi (reguler, Pro, Enterprise, dan RT) memiliki versi 32-bit dan 64-bit. Selain edisi RT, memiliki versi 32-bit tidak masuk akal bagi saya.

Bukankah kita harus sepenuhnya menyerah pada OS 32-bit? Apa masih terlalu dini?

Sebagai administrator Mesin Virtual, saya tidak sepenuhnya keberatan dengan build 32bit. Jika Anda membuat mesin yang tidak membutuhkan lebih dari 4GB memori, maka dengan menggunakan OS 32bit, semua perangkat lunak Anda menghabiskan lebih sedikit memori saat dijalankan. Itu karena untuk build 64bit, perangkat lunak menyimpan penunjuk alamat 64bit, bukan yang 32bit. Jadi mereka akhirnya memakan lebih banyak memori hanya untuk melakukan hal yang sama.

Dalam kasus Windows 8, ini mungkin juga ada hubungannya dengan kompatibilitas ke belakang, yang merupakan hal Microsoft yang sudah berjalan lama. Mungkin beberapa server 32-bit yang lebih tua (tapi germo) akan mendapat manfaat dari peningkatan dari versi windows yang lebih lama? Ini kasus malas, tapi begitu juga banyak admin Windows.

Akhirnya, saya akan menawarkan spekulasi diam bahwa Windows 8 adalah proyek perangkat lunak yang berkembang dari seri yang berjalan lama. Cabang 32bit mungkin lebih merupakan teritip daripada produk yang serius.

Edit:

Mungkin jawaban terbaiknya adalah orang membayar untuk 32bit Windows 8 . Apakah itu masuk akal atau tidak, mungkin tidak terlalu penting bagi orang yang melakukan penjualan. Saya berani bertaruh jika tidak ada yang membelinya, itu akan hilang dengan cepat.

Bahkan jika CPU Anda 64 bit, mungkin ada beberapa perangkat yang Anda tidak dapat hidup tanpanya, yang hanya ada driver 32 bit. Tanpa Windows versi 32 bit, Anda tidak beruntung, karena Windows membutuhkan driver dengan bit yang sama dengan OS.

Satu-satunya alasan saya dapat menduga adalah memberi Intel atau AMD kesempatan untuk menghasilkan prosesor bertenaga super rendah, yang mungkin lebih mudah untuk mencapai tingkat konsumsi daya seperti ARM jika itu adalah prosesor 32-bit klasik, untuk digunakan di tablet , smartphone atau perangkat portabel lainnya.

Intel telah mencoba masuk ke pasar ponsel selama bertahun-tahun dan ini mungkin Windows yang mencoba membantu mereka.

Bukankah kita harus sepenuhnya menyerah pada OS 32-bit? Apa masih terlalu dini?

Karena CPU 64-bit x86 dapat menjalankan kode 32-bit, dan karena versi Windows 64-bit akan menjalankan program berbasis 32-bit "lawas", Anda tidak perlu khawatir jika Anda menjalankan program yang relatif modern.

Edisi Windows 32-bit memungkinkan eksekusi program berbasis 16-bit yang lama. Kemampuan ini dihilangkan di Windows versi 64-bit yang dimulai dengan Windows XP x64. Software seperti ini sudah sangat tua.
