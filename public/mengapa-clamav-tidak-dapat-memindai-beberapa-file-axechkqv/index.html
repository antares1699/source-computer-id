Mengapa ClamAV tidak dapat memindai beberapa file?

Setiap kali saya mencoba menjalankan pemindaian ClamAV pada file di banyak direktori bin saya, gagal dengan

Tidak dapat membuka file atau direktori ERROR

pesan. Saya tahu mungkin saja ClamdScan memindai direktori bin tetapi saya tidak tahu caranya. Ada saran?

$ clamdscan /bin/true
/bin/true: Can't open file or directory ERROR

----------- SCAN SUMMARY -----------
Infected files: 0
Total errors: 1
Time: 0.000 sec (0 m 0 s)

Saya tidak dapat menemukan perbedaan apa pun antara file konfigurasi AppArmor di mesin tempat ia bekerja dan di mesin tempat ia tidak bekerja. Saya berhasil memindai dengan solusi dengan menambahkan:

/bin/** r,

untuk:

/etc/apparmor.d/local/usr.sbin.clamd

Tetapi baris ini tidak ada di file izin lokal clamd AppArmor mesin lain jadi saya masih bingung dan solusi tembakan seperti itu hanya mengatakan ClamAV dapat mengakses semuanya di /bin dan di bawahnya terutama karena jika ini adalah solusinya maka saya punya untuk menambahkan /sbin /usr/bin dan /usr/sbin ke AppArmor juga, dan itu sepertinya kontra-intuitif.

Itu adalah AppArmor yang mencegah saya mengakses bin. Aku berubah:

/etc/apparmor.d/local/usr.sbin.clamd

dengan menambahkan:

/bin/** r,

Untuk itu dan mampu memindai file di / bin. Jika saya ingin menambahkan direktori lain, saya hanya perlu menambahkan baris ke file itu untuk memasukkannya. Baris AppArmor di atas berarti / bin dan semua folder anak, secara rekursif, dengan akses hanya-baca, yang merupakan semua yang dibutuhkan ClamAV. File yang dimaksud adalah petunjuk AppArmor untuk izin khusus apa yang akan diterima / usr / sbin / clamd yang merupakan daemon layanan ClamAV. Jadi hanya ini yang perlu saya lakukan untuk membuatnya bekerja. Terima kasih @Ramhound untuk petunjuknya!
