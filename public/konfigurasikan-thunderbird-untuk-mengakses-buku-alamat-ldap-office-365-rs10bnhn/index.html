Konfigurasikan Thunderbird untuk mengakses buku alamat LDAP Office 365

Saya telah mengonfigurasi Thunderbird di Win10 untuk mengakses akun Office 365 saya. Mengirim dan menerima email (melalui IMAP) berfungsi, dan TB menampilkan folder langganan dengan benar.

Sekarang saya ingin mengonfigurasi TB untuk mengakses direktori kontak / buku alamat Office 365 melalui LDAP, dan karenanya, saat saya menulis pesan, memiliki fitur "pelengkapan otomatis" untuk alamat email penerima lokal.

Bagaimana saya bisa melakukan itu?

Cara termudah yang saya temukan adalah dengan menggunakan Konektor Davmail .

Ini adalah applet Java yang berjalan di latar belakang pada PC Windows 10 Anda. Bagi Thunderbird, tampilannya seperti IMAP untuk menerima email, SMTP untuk mengirimnya, dan LDAP untuk kueri buku alamat. Di sisi Office365 itu berbicara tentang protokol asli.

Saya telah menggunakannya selama sekitar lima tahun sekarang, dan IME itu solid dan dapat diandalkan.

Prasyarat penginstalan adalah Oracle Java. Unduh JRE dari Halaman unduhan Java . Pada saat penulisan, file unduhan yang sesuai adalah jre-10.0.1_windows-x64_bin.exe. (Jika Anda diminta untuk membuat akun Oracle, ada tulisan kecil "Lewati langkah ini" jika Anda tahu harus mencarinya.)

Pengaturan konfigurasi untuk Davmail

  1. Utama

    • Protokol pertukaran: Otomatis
    • URL OWA (Exchange): https://outlook.office365.com/EWS/Exchange.asmx
    • Port IMAP lokal: YA 143
    • Porta SMTP Lokal: YA 25
    • Port HTTP Caldav: YA 1080
    • Port LDAP: YA 389
    • Penundaan penyimpanan sampah (POP): 30
    • Tunda tetap terkirim (POP): 90
    • Penundaan monitor folder IDLE (IMAP): 2
    • Yang lainnya: TIDAK atau Kosong

  2. Jaringan

    • Alamat mengikat: 127.0.0.1
    • Yang lainnya: TIDAK atau Kosong

  3. Maju

    • Edit pemberitahuan Caldav: YA (Anda mungkin lebih suka TIDAK)
    • Tampilkan spanduk pembuka: YA
    • Aktifkan KeepAlive: YES
    • Yang lainnya: TIDAK atau Kosong

Pengaturan konfigurasi untuk Thunderbird

  1. Alat> Opsi> Pengalamatan

    • Server direktori: [Edit direktori ...]> Tambah

      1. Umum

        • Nama: Office 365
        • Nama host: localhost
        • Base DN: ou = People
        • Nomor port: 389
        • Bind DN: {alamat email Office365 Anda}

      2. Maju

        • Jangan mengembalikan lebih dari: 100 hasil
        • Cakupan: Subpohon
        • Filter pencarian: (objectclass = *)
        • Metode login: [Sederhana]
