Cara menetapkan IP statis ke Komputer di Active Directory

Saya ingin tahu bagaimana Anda dapat menetapkan alamat IP statis ke komputer menggunakan Active Directory.

Saya tahu itu mungkin dengan pengguna yang menggunakan tab Dial-in, tetapi melakukannya di komputer tidak berpengaruh (kecuali saya melewatkan parameter kebijakan tertentu).
Saat ini saya menggunakan reservasi DHCP, tetapi memerlukan alamat IP dalam rentang DHCP dan menuliskan alamat MAC untuk mengetiknya di server.

Saya mencoba menemukan cara yang tidak memerlukan penggunaan alamat MAC yang tidak jelas dengan menggunakan nama komputer AD.
Selain itu, saya ingin rentang DHCP saya hanya mencakup alamat yang benar-benar digunakan oleh klien DHCP dan tidak bergantung pada pengecualian.

Hutan dan domain AD saya memiliki tingkat fungsionalitas Windows Server 2008 R2 (karena saya memiliki komputer Win 7) tetapi saya menggunakan Windows Server 2012.

Terima kasih semuanya atas bantuan Anda.

Penggunaan DHCP Anda sudah menjadi cara terbaik untuk menetapkan alamat. Tidak mungkin menggunakan Active Directory itu sendiri untuk menetapkan alamat IP karena komputer harus memiliki alamat IP sebelum itu dapat berkomunikasi langsung dengan AD.

Tidak ada jalan lain untuk melibatkan alamat MAC dalam penetapan alamat IP dinamis. Sampai komputer memiliki alamat IP, secara efektif tidak dapat berkomunikasi melalui protokol IP (OSI lapisan 3 ), sebagai gantinya mengandalkan siaran dan Alamat MAC . Untuk mendapatkan alamat IP tanpa konfigurasi statis pada mesin, harus ada layanan di jaringan yang memberikan alamat IP berdasarkan alamat MAC. Biasanya ini adalah DHCP, meskipun ada layanan lain.

Beberapa perangkat lunak DHCP lain dapat mengatur reservasi pada atribut lain dari paket penemuan DHCP selain alamat MAC. Misalnya perangkat lunak kami WinGate dapat mengatur reservasi berdasarkan nama komputer (nama NetBIOS).

Selama Anda menyetel server DHCP untuk menetapkan DNS AD Anda ke klien, itu juga tidak merusak AD Anda, dan sebenarnya kami menggunakan WinGate DHCP di Active Directory kami di sini.
