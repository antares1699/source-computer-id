Komputer tidak menjaga waktu meskipun menyala dan terhubung ke Internet

Baru-baru ini komputer Windows 8.1 saya tidak bisa menghitung waktu. Pada awalnya saya mengira itu adalah baterai CMOS tetapi karena komputer saya selalu hidup dan diatur untuk memperbarui dengan internet, itu seharusnya dapat menjaga waktu.

Jika saya pergi ke Ubah Pengaturan Tanggal dan Waktu -> Waktu Internet -> Ubah Pengaturan -> dan Perbarui maka akan menampilkan waktu yang tepat. Namun, setelah jangka waktu tertentu, waktu akan berubah menjadi waktu acak seperti dari jam 9:20 sampai 13:05. Karena menit juga tidak aktif, saya pikir ini bukan masalah zona waktu. Tapi apa masalahnya?

Dua skenario:

Masalah perangkat keras

  • Motherboard Anda mungkin bermasalah dan perlu diperiksa. Meskipun motherboard Anda baru, baterainya mungkin sudah tua dan biasanya harganya murah. Coba ganti baterai dan lihat apakah itu menyelesaikan masalah. Selain itu, coba perbarui BIOS atau periksa waktu BIOS dan tunggu beberapa saat untuk melihat apakah itu sinkron. Baterai CMOS.

Masalah perangkat lunak

  • Oke, saya melihat Anda mencoba memasang pin pada server NTP untuk menyinkronkan waktu. Coba jalankan program anti-malware. Untuk beberapa alasan, mungkin itulah masalahnya. Saya mengalami masalah ini ketika PC saya terus mengubah zona waktu karena saya memiliki beberapa malware.

Coba ini di cmd: w32tm /resync Jika Anda mendapatkan pesan "Komputer tidak mensinkronisasi ulang karena tidak ada data waktu tersedia kesalahan", maka Anda harus dapat mencoba lagi hingga berhasil.

Satu hal kecil lagi. Coba sambungkan ke hotspot atau akses internet lain. Coba gunakan ponsel Anda sebagai hotspot dan lihat bagaimana server NTP merespons.

Layanan Windows Time biasanya mencatat dua peristiwa di log genap:

  • saat jam sudah disesuaikan
  • ketika sumber memberikan waktu telah berubah

Mencari acara ini dengan Event Viewer harus mengungkapkan penyebabnya (asalkan terkait perangkat lunaknya). Perhatikan bahwa jika jam Anda dimanipulasi oleh malware, mungkin cukup pintar untuk menyembunyikan aktivitasnya dari log.

Coba dulu gunakan server waktu lain dan lihat apakah masalahnya kembali. Jalankan juga sfc / scannow untuk memeriksa integritas sistem.

Anda juga bisa di Panel Kontrol / Alat / Layanan Administratif, atur Startup layanan Windows Time ke Otomatis, lalu mulai, jika belum dimulai.

Jika masalah terjadi lagi, jalankan Command Prompt (cmd) sebagai Administrator, dan memposting hasil perintah W32tm /query /configuration. Jika perlu, perintah untuk memaksa sinkronisasi ulang jam adalah W32tm /resync.

Verifikasi interval pembaruan Waktu Internet, dengan memulai regedit dan menavigasi ke HKEY_LOCAL_MACHINE\SYSTEM\ControlSet001\Services\W32Time\TimeProviders\NtpClient dan konsultasi nilai SpecialPollInterval, yang berisi waktu pemungutan suara dalam hitungan detik. Standarnya adalah 604800 detik, yang berarti 7 hari, dan Anda dapat mengubahnya menjadi setiap beberapa jam, jika perlu. Utilitas gratis Pembaruan Waktu Internet dapat menyederhanakan tugas ini (jalankan sebagai Administrator).

[EDIT}

Saya membandingkan konfigurasi W32tm yang Anda posting dengan milik saya, dan menemukan bahwa sebagian besar identik, kecuali Anda memiliki satu penyedia waktu lebih banyak dari saya: VMICTimeProvider.

VMICTimeProvider adalah penyedia waktu Hyper-V. Penyedia ini dibebankan dengan sinkronisasi waktu tuan rumah menuju tamu dan biasanya dipasang di mesin virtual. Saya merasa bingung bahwa Anda telah menginstal ini, kecuali Windows 8.1 Anda berjalan di dalam mesin virtual. Apakah ini masalahnya?

Pada teori bahwa saat ini penyedia yang menyebabkan masalah, dan jika ini adalah komputer fisik, saya sarankan Nonaktifkan Hyper-V di Windows 8 , bahkan hanya sebagai ujian sementara dari teori ini.

Anda juga bisa melakukan hack yang seharusnya di mesin virtual untuk menonaktifkan penyedia sinkronisasi waktu Hyper-V di infrastruktur sinkronisasi waktu Windows sambil tetap membiarkan layanan berjalan dan diaktifkan di bawah Hyper-V.

Untuk melakukan ini, buka regedit dan posisikan ke kunci HKLM\SYSTEM\CurrentControlSet\Services\W32Time\TimeProviders\VMICTimeProvider. Temukan item DWORD bernama Enabled dan setel ke 0, lalu reboot.

Ini menghentikan W32Time dari menggunakan layanan integrasi sinkronisasi waktu Hyper-V untuk sinkronisasi momen-ke-momen. Saya tidak tahu apakah itu akan berpengaruh di komputer host, tetapi mungkin patut untuk dicoba.
