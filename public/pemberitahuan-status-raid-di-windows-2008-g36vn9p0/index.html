Saya belum pernah melakukan ini, tetapi saya merasa ada kemungkinan Anda dapat menemukan cara untuk sampai ke sana dengan sukses.

Saya sebenarnya mendasarkan perasaan saya pada benang dikutip oleh Oliver Salzburg, yang mengatakan bahwa ID Peristiwa 26 (Informasi) dihasilkan di win7 / 2008 (mungkin dengan volmgr sebagai sumber) ketika volume toleransi kesalahan tidak lagi dapat diakses. Ini juga menyatakan bahwa Windows 8 menambahkan Event id 142 baru yang lebih informatif selain Event 26.

Saya pikir Anda harus mencoba mengirim diri Anda sendiri sebagai permulaan semua peristiwa yang dihasilkan oleh volmgr, lalu saring yang tidak Anda inginkan. Namun, saya tidak memiliki Server 2008 R2 untuk dimainkan untuk itu, jadi tidak bisa memberikan instruksi yang tepat.

Berikut adalah beberapa link yang mungkin membantu dalam mengaitkan tindakan dengan peristiwa:

Mendapatkan konten log peristiwa melalui email pada pemicu log peristiwa
Notifikasi email saat hard disk penuh
Jalankan Tugas sebagai Respons terhadap Acara yang Diberikan
Menggunakan Deskripsi Peristiwa sebagai kriteria untuk aturan
