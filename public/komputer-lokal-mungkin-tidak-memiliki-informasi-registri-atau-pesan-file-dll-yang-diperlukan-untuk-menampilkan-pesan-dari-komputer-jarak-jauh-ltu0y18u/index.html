Komputer

Komputer lokal mungkin tidak memiliki informasi registri atau pesan file DLL yang diperlukan untuk menampilkan pesan dari komputer jarak jauh

Kami memiliki mesin klien secon (XP-Pro, SP3) dan mesin server servlet. Saat memutakhirkan Norton Antivirus dari 11.4 ke RU6, salah satu aplikasi kami yang bernama 'LeaftradeLauncher' tidak berjalan lagi. Ketika saya memeriksa syslog aplikasi secon, ini adalah beberapa entri yang saya temukan:

"Deskripsi untuk ID Peristiwa (0) di Sumber (Peringatan Leaftrade) tidak bisa ditemukan. Komputer lokal mungkin tidak memiliki registri yang diperlukan informasi atau pesan file DLL ke menampilkan pesan dari remote komputer. Anda mungkin bisa menggunakan / AUXSOURCE = bendera untuk mengambil ini deskripsi; lihat Bantuan dan Dukungan untuk detailnya. Informasi berikut ini adalah bagian dari acara: LeaftradeLauncher :: Tem.InitClient berhasil. Diinisialisasi dengan server: servlet "

Ketika saya mencarinya di Google, saya menemukan beberapa situs yang menyebutkan bahwa kami perlu memeriksa di services.msc, untuk melihat apakah layanan dinonaktifkan atau tidak. Beberapa situs menyebutkan bahwa file DLL yang bertanggung jawab untuk menghasilkan acara untuk aplikasi ini rusak, atau akun yang Anda gunakan tidak memiliki hak untuk memuat DLL.

Ini sebagian besar terjadi dengan aplikasi pihak ke-3 yang diinstal dengan akun pengguna dan akun layanan tidak mendapatkan izin untuk menarik info tentang DLL yang diperlukan seperti itu.

Log peristiwa windows benar-benar tidak berisi deskripsi acara. Ini hanya berisi nomor peristiwa, dan Windows mencoba untuk menyelesaikan nomor tersebut ke deskripsi saat Anda melihat log. Aplikasi yang menambahkan entri log harus menginstal tabel terjemahan angka-ke-deskripsi.

Pesan kesalahan "Deskripsi untuk ID Peristiwa (0) di Sumber (Peringatan Leaftrade) tidak dapat ditemukan." Berarti Windows tidak dapat menemukan tabel untuk Leaftrade Alerts, atau tabel melewatkan entri 0. Ini adalah pesan kesalahan dari Windows itu sendiri , sehingga Anda bisa mendapatkan pesan kesalahan serupa untuk Sumber yang berbeda.

Namun perlu dicatat bahwa Windows berhasil menangkap satu bagian dari acara yang berasal dari Leaftrade Alerts. Bagian "LeaftradeLauncher :: Tem.InitClient berhasil. Diinisialisasi dengan server: servlet" tidak datang dari Windows, tetapi langsung dari LeaftradeLauncher. Jadi, tampaknya LeaftradeLauncher sendiri berpikir bahwa ini dimulai dengan sukses.
