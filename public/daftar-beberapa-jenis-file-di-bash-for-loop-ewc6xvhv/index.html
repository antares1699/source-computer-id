daftar beberapa jenis file di bash for loop

Saya menulis skrip yang mengubah gambar dalam folder. Skrip menggunakan for loop:

i="1"
for file in *.jpg; do
     outputFile=$(echo "image"$(echo $i))
     convert "$file" -resize 50x50 $outputFile
     i=$[i+1]
done

Yang ingin saya lakukan adalah mengizinkan skrip dijalankan pada beberapa ekstensi jenis file. Sekarang saya sudah mencoba melakukan:

for file in *.jpg *.JPG *.jpeg; do
....
done

Masalah yang saya miliki dengan ini adalah bahwa jika Anda berada di folder dengan semua * .JPG dan tidak ada * .jpg, skrip masih menabrak i + 1 meskipun tidak ada gambar, karena tetap menjalankan loop for, setelah tidak menemukan apapun * .jpg itu akan menjadi * .JPG.

Bagaimana cara menargetkan beberapa ekstensi file tanpa menjalankannya sendiri per jenis? Contoh, apakah sintaksnya seperti ini:

for file in [*.jpg | *.JPG]; do
...

?
Dengan begitu folder keluaran saya selalu berisi gambar berlabel seperti ini:
image1.jpg
image2.jpg
image3.jpg
dll ..

Alih-alih berakhir dengan folder kehilangan image1.jpg karena tidak ada * .jpg untuk dijalankan terlebih dahulu.

Solusinya

Gunakan nullglob. Letakkan baris berikut sebelum loop for:

shopt -s nullglob

nullglob berarti jika tidak ada file seperti itu, glob dihapus dari daftar. Amati tanpa nullglob:

$ echo  *.jpg *.JPG *.jpeg
test.jpg *.JPG *.jpeg

Sekarang, dengan nullglob:

$ echo  *.jpg *.JPG *.jpeg
test.jpg *.JPG *.jpeg

Script yang Direvisi

Skrip lengkapnya bisa jadi:

i=1
shopt -s nullglob
for file in *.jpg *.JPG *.jpeg; do
     convert "$file" -resize 50x50 "image$i.jpg"
     i=$((i+1))
done

Tiga catatan:

  1. Dalam definisi outputFile, semua pernyataan echo itu tidak diperlukan.

  2. Mengikuti konvensi, saya menambahkan ekstensi, .jpg, ke nama file output Anda. Jika Anda benar-benar tidak menginginkan ekstensi tersebut, hapus saja.

  3. Saya mengganti $[...] dengan bentuk yang lebih standar: $((...)).

  1. Karena Anda ingin mencari lebih dari satu berbeda (case insensitive), jawaban shopt -s nullglob sangat bagus untuk digunakan.

  2. Jika Anda hanya mencari

    @(pattern-list)
    , *.JPG, dan enam kombinasi lain dari huruf besar dan kecil, hal termudah untuk dilakukan adalah

    for file in *.[Jj][Pp][Gg]

    yang mencari file yang namanya

    • (sesuatu), diikuti oleh
    • . (titik), diikuti oleh
    • J atau j, diikuti oleh
    • P atau p, diikuti oleh
    • G atau g

    dengan kata lain,

    • (sesuatu), diikuti oleh
    • . (titik), diikuti oleh
    • "jpg", dalam beberapa kombinasi huruf besar dan kecil

  3. Jadi salah satu cara untuk menyelesaikan masalah Anda adalah

    shopt -s nullglob
for file in *.[Jj][Pp][Gg] *.[Jj][Pp][Ee][Gg]
do …

  4. Trik […] bagus untuk diketahui, tapi ada cara yang lebih baik. Jika saya memberi tahu Anda bahwa ada opsi shell bernama nocaseglob, bisakah kamu menebak apa fungsinya?

    shopt -s nullglob nocaseglob
for file in *.jpg *.jpeg
do …

    jadi sekarang j artinya

    • (sesuatu), diikuti oleh
    • . (titik), diikuti oleh
    • "jpg", dalam beberapa kombinasi huruf besar dan kecil

    dan perintah di atas akan cocok dengan shopt -s nullglob nocaseglob for file in *.jpg *.jpeg do … , *.JPG, *.Jpg, *.jpeg, *.jPeG, dll.

  5. Cara lain adalah dengan menggunakan opsi shell extglob untuk mengaktifkan operator pencocokan pola yang diperluas. Anda masih menginginkan nullglob dan nocaseglob. Semua operator pencocokan pola yang diperluas terlihat seperti

    (some_special_character)  (  pattern-list  )

    di mana spasi ditambahkan untuk kejelasan, dan pattern-list adalah daftar pattern-an dipisahkan oleh | karakter. Salah satu yang terlihat paling menarik untuk tujuan Anda adalah

    @(pattern-list)

    yang berarti cocok dengan salah satu pola yang terdaftar. Jadi sekarang skripnya menjadi

    (some_special_character)  (  pattern-list  )

    Ini mungkin terlihat seperti mengetik lebih banyak, tetapi pengorbanannya bergeser seiring dengan bertambahnya jumlah ekstensi. Mempertimbangkan,

    for file in *.@(jpg|jpeg|jfif|exif|tif|tiff|gif|bmp|png|svg)

    sedikit lebih pendek dari

    for file in *.jpg *.jpeg *.jfif *.exif *.tif *.tiff *.gif *.bmp *.png *.svg

  6. Di mana Anda belajar melakukan j? Tentu, $(some_command(s) …) pasti bisa berguna, dan mungkin kadang-kadang $(echo …), tetapi Anda terlalu sering menggunakannya. Sangat bagus untuk dikatakan

    outputFile="image$i"

    dan Anda mungkin bahkan tidak membutuhkan tanda kutip ( dalam pernyataan penugasan ).

  7. Oke, Anda bisa dimaafkan sekali ini saja , karena Anda tahu bahwa nilai outputFile adalah jinak, karena Anda menyetelnya dari string tidak kosong yang konstan, karakter non-khusus (image) ditambah variabel yang Anda setel ke string konstan karakter non-kosong, tidak khusus (1) dan hanya melakukan operasi aritmatika sepele di atasnya. Tapi, sebagai aturan umum, Anda harus melakukannya selalu mengutip semua referensi untuk variabel shell, kecuali Anda memiliki alasan yang sangat bagus untuk tidak melakukannya dan Anda yakin tahu apa yang Anda lakukan. Jadi, idealnya, perintah convert Anda adalah

    convert "$file" -resize 50x50 "$outputFile"

    (dengan kutipan sekitar $outputFile.)
