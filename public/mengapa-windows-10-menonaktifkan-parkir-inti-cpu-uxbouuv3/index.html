Kapan dan Mengapa Windows 10 Menonaktifkan Core Parking?

Microsoft menonaktifkan parkir inti di paket daya windows 10 default pada tahun 2017.

Satu situs web menyatakan:

“Jika Anda menonaktifkan parkir inti pada CPU Anda, komputer Anda mungkin memiliki benchmark yang lebih cepat. (Melaporkan frekuensi gambar yang lebih tinggi oleh gamer dan power user). ”

Meskipun Microsoft telah memilih untuk memberikan lebih banyak kekuatan pada kinerja perangkat keras, ia telah menyerahkan opsi rencana daya khusus kepada vendor, administrasi server, dan pengguna super.

“Core Parking paling efektif pada sistem yang memiliki prosesor idle negara dengan konsumsi daya yang sangat rendah. " ( sumber )

Juga catatan Microsoft untuk administrator server:

Saat algoritme parkir inti memilih inti untuk dilepas (dengan asumsi peningkatan intensitas beban kerja terjadi), mungkin tidak selalu memilih core dalam subset afinitized aktif (atau subset) untuk unpark, dan dengan demikian mungkin akan berakhir dengan pembongkaran inti yang tidak benar-benar digunakan. ( sumber )

Sementara banyak pengguna memperdebatkan manfaat dari parkir inti, Microsoft telah memilih untuk menyerahkan pilihan parkir inti kepada vendor, karena mereka dapat membuat rencana daya sendiri dan pengiriman dengan produk. Dengan hal-hal seperti Turbo Boost Index dan Frequency Scaling Index, prosesor biasanya dapat dikontrol secara lebih responsif dengan core parking dinonaktifkan.

Sini adalah contoh yang baik dari vendor, AMD, dengan produk Razon di tahun 2017 yang menciptakan kekuatan khusus untuk penggunaan optimal prosesor di sana.

Apakah Windows 10 Datang dengan Core Parking Disabled?

  • Windows 10 menggunakan rencana daya untuk mengatur Kinerja Prosesor yang akan mengakibatkan inti diparkir atau tidak tergantung pada konfigurasi pengaturan skema daya.

Note the below information assumes static core parking options not heterogeneous architecture. Please see further reading links for more info.

Mengapa Pengaturan Registri Parkir Inti Tersembunyi?

  • Microsoft menggunakan perintah powercfg untuk mengontrol pengaturan ini. Silakan lihat jawabannya sini untuk informasi lebih lanjut.

  • Untuk memperlihatkan opsi tersebut, buka cmd yang ditinggikan dan jalankan:

    powercfg -attributes SUB_PROCESSOR CPMINCORES -ATTRIB_HIDE

  • Selesai. Tidak perlu mengedit registri.

Apa itu Skema Kekuatan?

Skema daya adalah kumpulan pengaturan yang mengontrol daya penggunaan komputer Anda. Anda dapat menggunakan skema daya untuk mengurangi daya konsumsi perangkat individu atau seluruh sistem.

Opsi Konfigurasi Statis Microsoft untuk Parkir Inti

Anda dapat menggunakan opsi konfigurasi statis yang didokumentasikan di bagian ini untuk menyesuaikan perilaku mesin parkir inti.

  • CPMinCores menentukan persentase minimum prosesor logis (dalam format persyaratan dari semua prosesor logis yang diaktifkan pada sistem di dalamnya setiap node NUMA) yang dapat ditempatkan dalam status tidak terparkir di mana saja waktu.

  • CPMaxCores menentukan persentase maksimum prosesor logis (dalam format persyaratan prosesor logis dalam setiap node NUMA) yang bisa di negara bagian tidak terparkir pada waktu tertentu.

  • CPIncreaseTime menentukan jumlah waktu minimum yang harus dilalui sebelum prosesor logis tambahan dapat dialihkan dari negara terparkir ke negara bagian tidak terparkir. Waktu ditentukan dalam satuan jumlah interval pemeriksaan waktu kinerja prosesor.

  • CPDecreaseTime menentukan jumlah waktu minimum yang harus dilalui sebelum prosesor logis tambahan dapat dialihkan dari negara bagian yang tidak diparkir ke negara bagian yang diparkir. Waktu ditentukan dalam satuan jumlah interval pemeriksaan waktu kinerja prosesor.

  • Konkurensi BPK menentukan ambang batas untuk menentukan konkurensi simpul.

  • CPDistribution menentukan pemanfaatan, dalam persentase, untuk digunakan dalam distribusi konkurensi untuk memilih jumlah prosesor logis mendistribusikan utilitas ke.

  • CPHeadroom menentukan nilai pemanfaatan yang akan menyebabkan mesin parkir inti untuk melepaskan prosesor logis tambahan jika prosesor yang paling jarang digunakan dari rangkaian prosesor yang tidak terparkir lebih banyak pemanfaatan. Hal ini memungkinkan peningkatan konkurensi terdeteksi.

  • CPLatencyHintUnpark menentukan jumlah minimum inti yang tidak terparkir saat petunjuk sistem latensi rendah terdeteksi.

Profil Daya

  • Anda dapat menggunakan kerangka kerja Penyediaan Windows untuk mengonfigurasi pengaturan daya prosesor yang dijelaskan dalam bagian ini .

  • Ruang nama prosesor dibagi menjadi tiga set daya identik konfigurasi prosesor yang disebut profil daya. Profil kekuatannya adalah digunakan oleh mesin prosesor daya untuk menyesuaikan kinerja dan algoritma parkir pada berbagai kasus penggunaan sistem.

Windows 10 mendukung profil berikut:

  • Profil default adalah set konfigurasi yang paling sering aktif.
  • LowLatency adalah profil yang diaktifkan selama boot dan selama waktu peluncuran aplikasi.
  • LowPower adalah profil yang diaktifkan selama fase buffering dari skenario pemutaran media.
  • Dibatasi adalah profil yang diaktifkan oleh fitur penghemat baterai di Windows 10 untuk edisi desktop (Home, Pro, Enterprise, dan Pendidikan).

Apa itu Parkir Inti dan Bagaimana Cara Menyesuaikannya?

  • Silakan lihat tautan dan informasi di atas untuk semua pengaturan yang tersedia.

  • Singkatnya, parkir inti memungkinkan prosesor Anda untuk tidur mode. Tujuan utama dari parkir inti adalah untuk memungkinkan komputer / laptop / perangkat untuk hanya menggunakan prosesor saat dibutuhkan, dengan demikian hemat energi.

Microsoft, pertama di Windows Server 2008 R2, mulai menggunakan parkir inti melalui mesin manajemen daya prosesor (PPM) dan penjadwal untuk mengatur jumlah inti secara dinamis tergantung pada pengaturan rencana daya dan tugas yang dijadwalkan untuk dilaksanakan.

Jumlah minimum inti untuk utas yang akan dijadwalkan seperti yang dipilih oleh mesin PPM. Biasanya inti yang diparkir tidak memiliki utas yang dijadwalkan, dan mereka masuk ke status daya yang sangat rendah saat tidak memproses interupsi, DPC, atau pekerjaan lain yang terikat secara ketat. Jadi inti yang tersisa bertanggung jawab atas sisa beban kerja. Parkir inti berpotensi meningkatkan efisiensi energi selama penggunaan yang lebih rendah, itulah sebabnya ini umumnya diatur pada paket daya Windows.

Untuk menonaktifkan parkir inti, setel parameter Performa Prosesor Inti Minimum Parkir Inti ke 100 persen dengan menggunakan perintah berikut:

Powercfg -setacvalueindex scheme_current sub_processor CPMINCORES 100
Powercfg -setactive scheme_current

Untuk mengurangi jumlah inti yang dapat dijadwalkan hingga 50 persen dari jumlah maksimum, setel parameter Inti Maksimum Parkir Inti Kinerja Prosesor ke 50 sebagai berikut:

Powercfg -setacvalueindex scheme_current sub_processor CPMAXCORES 50
Powercfg -setactive scheme_current

Contoh Status Prosesor

C0 - Active mode, processor executing code.
C1 - AutoHALT processor IA core state (Pacakage C0 state).
C1E - AutoHALT processor IA core state with lowest freqency and voltage operating point (Pacakage C0 state).
C2 - All processor IA cores in C3 or deeper. Memory path open. Temporary state before Package C3 or deeper.
C3 - Processor IA execution cores in C3 or deeper, flush their L1 instruction cache, L1 data cache, and L2 cache to the LLC shared cache. LLC may be flushed. Clocks are shut off to each core.
C6 - Processor IA execution cores in this state save their architectural state before removing core voltage. BCLK is off.
C7 - Processor IA execution cores in this state behave similarly to the C6 state. If all execution cores request C7, LLC ways may be flushed until it is cleared. If the entire LLC is flushed, voltage will be removed from the LLC. 
C8 - C7 plus LLC should be flushed.

Pada prosesor multi inti, distribusi aktivitas ke inti sangat dinamis, dan ini adalah hal baik yang membantu Turbo Boost .

Akibatnya, inti terparkir dalam interval yang sangat singkat, tidak cukup lama bagi Anda untuk melihat inti tertentu tetap terparkir.

Dengan mencoba memaksa inti untuk tetap terparkir dalam jangka panjang, Anda menyebabkan pemanasan yang tidak merata di dalam prosesor Anda. Jangan lakukan itu! Biarkan penjadwal menyebarkan aktivitas, dan dengan demikian menghasilkan panas, secara merata ke seluruh prosesor.
