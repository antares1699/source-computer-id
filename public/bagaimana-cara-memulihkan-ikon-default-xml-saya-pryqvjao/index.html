Ya kali ini tidak ada hubungannya dengan cache icon. Seperti yang ditunjukkan posting ini dengan benar kepada saya: http://www.sevenforums.com/microsoft-office/152972-no-icon-xml-files.html#post1315413

masalahnya adalah karena IconHandler, tampaknya diinstal oleh beberapa versi Office suite. Jadi itu cukup untuk menghapus sub-kunci IconHandler dari kunci xmlfile, dan meninggalkan nilai string yang telah ditentukan sebelumnya dalam kunci DefaultIcon (seperti yang sudah Anda miliki).

Office menggunakan Penangan Ikon 32-bit, yang memecah ikon XML pada sistem 64-bit.

Untuk menghentikan Office agar tidak mencoba menampilkan ikon Anda untuk file XML, Anda dapat menggunakan skrip registri ini:

Windows Registry Editor Version 5.00

[HKEY_CLASSES_ROOT\xmlfile\DefaultIcon]
@=hex(2):25,00,53,00,79,00,73,00,74,00,65,00,6d,00,52,00,6f,00,6f,00,74,00,25,\
  00,5c,00,53,00,79,00,73,00,74,00,65,00,6d,00,33,00,32,00,5c,00,6d,00,73,00,\
  78,00,6d,00,6c,00,33,00,2e,00,64,00,6c,00,6c,00,2c,00,30,00,00,00

[-HKEY_CLASSES_ROOT\xmlfile\ShellEx\IconHandler]

Ini menetapkan nilai default kunci DefaultIcon ke %SystemRoot%\System32\msxml3.dll,0 dan menghapus kunci IconHandler.

Sebelum menggunakan skrip registri ini, pastikan Anda menginginkan kunci default tersebut (menurut saya Windows Vista / 7/8 menggunakan nilai itu secara default), dan secara opsional buat cadangan kunci IconHandler Anda.
