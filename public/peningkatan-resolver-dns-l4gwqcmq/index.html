Peningkatan resolver DNS?

Saya baru-baru ini menemukan konfigurasi /etc/resolv.conf berikut yang / seharusnya meningkatkan kecepatan resolusi DNS:

nameserver 1.1.1.1
nameserver 2.2.2.2
nameserver 3.3.3.3
option rotate 
option timeout:1

Saya memiliki bendera "opsi" dalam pikiran jelas selain menggunakan 3 server DNS dalam konfigurasi (biasanya hanya dua yang digunakan). Yang pertama seharusnya memutar / round-robin di semua 3 server DNS dan yang terakhir seharusnya mengatur batas waktu pencarian ke 1s (defaultnya adalah 5s). Saya hanya ingin tahu apakah ada di antara Anda yang menggunakan konfigurasi ini dan apakah Anda melihat peningkatan? Beberapa orang mengatakan bahwa ini tidak akan membantu perbaikan sama sekali tetapi saya tidak begitu yakin mengapa.

Ini akan memperburuk keadaan dalam setiap kasus tetapi kasus di mana server nama depan Anda sendiri gagal. Berikut adalah contoh bagaimana hal itu memperburuk keadaan:

Misalnya Anda mencoba menyelesaikan situs yang server nama utamanya lambat atau rusak. Biasanya ini terjadi.

  1. Anda mencoba server nama depan Anda.

  2. Ia mencoba server nama utama situs.

  3. Waktu keluar utama.

  4. Server nama depan Anda mencoba yang kedua.

  5. Itu mendapat tanggapan.

  6. Anda mendapat tanggapan.

Tetapi dengan perubahan Anda, ini terjadi:

  1. Anda mencoba namserver pertama Anda.

  2. Server nama depan Anda mencoba server nama utama situs.

  3. Anda waktu istirahat.

  4. Anda mencoba server nama kedua Anda.

  5. Server nama kedua Anda mencoba server nama utama situs.

  6. Server nama pertama mencoba server nama sekunder situs dan mendapatkan jawabannya. Sayang sekali Anda sudah kehabisan waktu.

  7. Anda kehabisan waktu di server nama kedua.

  8. Anda mencoba server nama ketiga.

  9. Server nama ketiga Anda mencoba server nama utama situs.

  10. Server nama kedua Anda memiliki jawabannya, sayang sekali Anda sudah kehabisan waktu.

  11. Anda kehabisan waktu di server nama ketiga.

  12. Server nama ketiga Anda memiliki jawabannya, tetapi Anda tidak mendengarkan lagi.

Anda kehabisan waktu cara terlalu cepat dan sangat mungkin untuk menyerah sebelum namserver mendapatkan jawabannya, hanya untuk mengulangi prosesnya. Sisi baiknya, sekarang semua server nama Anda memiliki hasil di cache mereka.

IIRC, diperlukan waktu dua detik secara default untuk server DNS yang pertama kali dicoba sampai batas waktu sebelum mencoba yang berikutnya dalam daftar jika tidak ada tanggapan. Oleh karena itu, sepertinya opsi terakhir akan mengurangi batas waktu menjadi satu detik sebelum mencoba opsi berikutnya dalam daftar.

Opsi lain untuk merotasi server DNS mana yang dicoba pertama kali hanya akan bermanfaat jika semuanya secepat satu sama lain dan Anda mencoba memuat permintaan keseimbangan.

Saya akan menggunakan namebench, sebagai Oliver telah menyebutkan , untuk mencari tahu seberapa cepat masing-masing dan memasukkannya ke dalam daftar dalam urutan kecepatan dan menghapus opsi putar, kecuali mereka secepat satu sama lain.
