Menggunakan kunci ssh untuk koneksi tetapi tanpa opsi -i

Saya bekerja dengan host serverals dan saya tidak ingin menentukan kata sandi setiap kali saya ingin terhubung ke host mana pun. Kemudian saya telah membuat kunci dengan:

Enter file in which to save the key (/home/robe/.ssh/id_rsa): my_key 
Enter passphrase (empty for no passphrase): 
Enter same passphrase again: 
Your identification has been saved in my_key.
Your public key has been saved in my_key.pub.
The key fingerprint is:
e6:d9:a3:85:38:b1:b2:a9:4d:3b:c1:33:65:df:57:25 robe@computer
The key's randomart image is:
+--[ RSA 2048]----+
|                 |
|              E .|
|               o |
|      o       .  |
|   . o..S.   .   |
|    =  *.+. .    |
|    o++ + +.     |
|   o.= . o .     |
|  ..=.  .        |
+-----------------+

lalu saya salin kunci_saya.pub ke kunci_otorisasi file di setiap host dan saya dapat terhubung dengan:

ssh -i /path/to/my_key  user@host

Tapi di sini masalah saya, saya ingin terhubung tanpa menentukan kuncinya. Saya ingin menemukan cara saya menjalankan:

ssh user@host

Catatan : Saya ingin menyimpan kunci id_rsa dan id_rsa.pub default saya. Kunci my_key adalah opsi alternatif untuk mengelola host lain.

Anda mencari file konfigurasi ~/.ssh/config. Misalnya, tambahkan entri ini di file ini.

Host host
    HostName host.example.com
    User johndoe
    IdentityFile ~/.ssh/host.key

Sekarang Anda bisa mengetik

ssh host

Dan ssh secara otomatis akan menggunakan nama pengguna dan kunci yang Anda tentukan oleh parameter IdentityFile. Oleh karena itu, perintah itu setara dengan:

ssh johndoe@host.example.com -i ~/.ssh/host.key

Lihat http://www.openbsd.org/cgi-bin/man.cgi/OpenBSD-current/man5/ssh_config.5 untuk semua detail konfigurasi untuk file itu.

Jika Anda berada di * nix, Anda dapat menggunakan ssh-agent / ssh-add untuk menyimpan identitas Anda di memori untuk sesi Anda.

  1. Jalankan ssh-agent dari terminal

pengguna @ mesin $ ssh-agent bash

Ini akan membuat proses bash dengan ssh-agent berjalan

Jika Anda tidak ingin membuat proses lain, maka

pengguna @ mesin $ ssh-agent -s

SSH_AUTH_SOCK = / tmp / ssh-shtzxe4347 / agent.4347; ekspor SSH_AUTH_SOCK; SSH_AGENT_PID = 4348; ekspor SSH_AGENT_PID; echo Agen pid 4348;

Anda harus menjalankan perintah dengan huruf tebal untuk memperbarui variabel env Anda.

2. Setelah salah satu langkah sebelumnya

pengguna @ mesin $ ssh-add

untuk menambahkan .ssh / id_rsa Anda ke agen, jika kunci Anda memiliki kata sandi, itu akan menanyakannya

http://linux.die.net/man/1/ssh-agent http://linux.die.net/man/1/ssh-add

Desktop Linux modern memiliki GUI untuk ini, dan akan memulai agen sejak login.

Di Windows, Anda dapat menggunakan Pageant (bagian dari Putty) sebagai ganti ssh-agent.
