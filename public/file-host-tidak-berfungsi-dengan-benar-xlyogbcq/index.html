File Host tidak berfungsi dengan benar

Saya mengalami beberapa masalah dengan file host saya.

127.0.0.1 localhost
127.0.0.1 whatever
127.0.0.1 sub.localhost
127.0.0.1 example.com

localhost dan whatever keduanya berfungsi. Namun yang lain berperilaku aneh.

Ping sub.localhost

c:\>ping sub.localhost
Pinging sub.localhost [127.0.0.1] with 32 bytes of data:

Ping example.com

C:\>ping example.com
Pinging example.com [127.0.0.1] with 32 bytes of data:

Semua tampak baik-baik saja, tetapi menelusuri example.com akan membuka halaman example.com yang benar @ 192.0.32.10 yang bukan merupakan perilaku yang diharapkan ... (jelas sub.localhost tidak akan berfungsi sama sekali)

Ada ide mengapa ini terjadi?

PEMBARUAN 1 : Menghapus Konfigurasi Proxy dari browser menyelesaikannya untuk sub.localhost. example.com namun masih tidak berfungsi ...

PEMBARUAN 2 : Beberapa info lagi ...

C:\Documents and Settings\amatos>ping example.com
Pinging example.com [127.0.0.1] with 32 bytes of data:

C:\Documents and Settings\amatos>ping www.example.com
Pinging www.example.com [127.0.0.1] with 32 bytes of data:

C:\Documents and Settings\amatos>nslookup example.com
Server:   ac1.-------.pt  (# suppose this is the router address...)
Address:  192.168.202.2

Non-authoritative answer:
Name:    example.com
Address:  192.0.32.10

Anda mengatakan example.com membuka 192.0.32.10 - seharusnya 127.0.0.1, jadi Anda salah menulis atau file host tidak diterapkan dengan benar.

Saya tidak yakin tentang masalah example.com, tetapi untuk yang lain, saya akan mengatakan bahwa jika ping berfungsi, mereka berfungsi dengan baik, jadi cukup periksa kembali apakah webserver yang Anda gunakan memiliki header host (atau pengaturan apa pun pada Anda server) disiapkan dengan benar.

Edit -

Tidak yakin mengapa ini terjadi, Hal pertama yang akan saya coba adalah browser yang berbeda seperti Firefox karena ini memiliki pengaturan jaringan sendiri yang tidak boleh terpengaruh oleh pengaturan sistem lain. Jika ini menyelesaikannya, kembali ke IE dan pada tab terakhir (Advanced), pilih Reset untuk menghapus apa pun yang mungkin mengatur ini.

selanjutnya, Anda dapat mencoba menyesuaikan prioritas Anda tentang apa yang dicentang.

Buka Registry Editor dan arahkan ke:

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Tcpip\ServiceProvider

Cari HostPriority dan pastikan angkanya lebih rendah dari DNSPriority. Anda juga dapat mengubahnya ke angka yang lebih rendah seperti 5 untuk memastikannya menjadi prioritas di atas yang lainnya.

Jika masih tidak berfungsi, saya hanya akan mengatakan untuk memeriksa hal-hal seperti ejaan Anda, tetapi sulit tanpa melihat mesin secara langsung.

Hapus cache ARP Anda:

  1. Buka jendela cmd.
  2. arp -a akan menampilkan semua entri ARP Anda.
  3. arp -d akan menghapus entri (atau yang spesifik).

Saya juga akan menghapus entri DNS:

ipconfig /flushdns
arp -d

ping <whatever>
