Menduplikasi izin akun pengguna mirip Linux di Windows 7

Saya ingin tahu apa saja praktik terbaik untuk membuat akun pengguna terbatas di sistem Windows 7, seperti lingkungan pengguna di Linux, di mana Anda memiliki superuser dan pengguna normal lainnya. Pada akhirnya saya ingin pengguna Windows biasa dibatasi untuk menyentuh apa pun di partisi tempat instalasi Windows (ini termasuk folder WIN, Program Files, dll), untuk dikunci hanya di sesuatu seperti folder rumah dan izin untuk menjadi. diperlukan untuk semua yang lebih maju (instalasi, konfigurasi, dll).

Untuk menguraikan lebih lanjut tentang komentar saya, pertanyaan Anda membuat saya agak bingung karena saya berasumsi, apa yang Anda minta untuk menjadi fakta dengan Windows 7.

Di Windows 7, akun Administrator (setara dengan root di Linux) tidak tersedia untuk digunakan langsung secara default. Jadi biasanya Anda memiliki akun dengan hak administratif (setara dengan /etc/sudoers). Pengguna tersebut dapat melakukan tugas sebagai Administrator dengan mengkonfirmasi a UAC cepat.

Pengguna biasa tidak akan dapat melakukan tindakan administratif apa pun. Selain itu, Anda dapat mengontrol hak akses pada sistem file untuk setiap akun pengguna atau grup pengguna, seperti yang Anda lakukan di Linux.

Mengenai keinginan Anda untuk mengamankan peningkatan hak istimewa dengan kata sandi. Sepengetahuan saya itu tidak mungkin (secara sengaja). Seperti yang Anda ketahui, pada sistem Linux, otorisasi elevasi hak istimewa Anda di-cache untuk sementara waktu. Jadi jika Anda menggunakan sudo dan mengotorisasi diri sendiri, jika Anda menggunakan sudo lagi beberapa detik kemudian, Anda tidak perlu memasukkan kata sandi lagi.
Di Windows, bukan itu masalahnya. Anda harus mengkonfirmasi prompt itu setiap. tunggal. waktu.
Jadi itu sudah sangat menyebalkan ( juga dengan desain ). Saya kira arsitek Windows memutuskan untuk tidak meminta kata sandi. Satu-satunya saat Anda harus memberikan kredensial Anda adalah ketika Anda masuk ke akun Anda. Arsitek Windows memutuskan itu sudah cukup.

Tentu saja ada kalanya kredensial Anda diminta lagi. Seperti ketika Anda mengakses sumber daya melalui jaringan atau saat masuk ke layanan tertentu (SharePoint, SQL Server, ...) tetapi itu yang diharapkan.

Juga, harap dicatat bahwa ada utilitas baris perintah di Windows yang disebut dengan sudo runas yang mungkin menarik bagi Anda (saya, secara pribadi, hampir tidak pernah menggunakannya).

Dibiarkan dalam konfigurasi defaultnya, menjalankan akun dengan izin USER di Windows 7 pada dasarnya akan memiliki efek yang Anda gambarkan, gmunk.

Kontrol Akses Pengguna (UAC) dan hak default akun PENGGUNA mengatur izin akun kurang lebih persis seperti yang Anda gambarkan. Tidak perlu konfigurasi khusus.

MEMPERBARUI:

Tergantung. Seperti biasa.

Tergantung pada apa yang Anda lakukan dengan komputer, persyaratan keamanan Anda akan berubah.

  • Apakah ini komputer rumah yang akan menjadi satu-satunya komputer di rumah? Tambahkan beberapa perangkat lunak antivirus dan Anda akan baik-baik saja.
  • Apakah ini workstation perusahaan? Jika jaringan perusahaan diamankan dengan benar, keamanan workstation individu dalam jaringan menjadi kurang penting dan biasanya sedikit ditolak untuk memungkinkan komunikasi dengan seluruh jaringan.
  • Apakah ini desktop yang tidak akan berpindah dan akan selalu terhubung ke jaringan yang sama? Mungkin di suatu tempat antara pengaturan perusahaan dan pengaturan rumah satu komputer. Anda ingin dapat berbagi printer dan mungkin file dan media antar komputer, jadi Anda ingin memastikan mereka saling percaya tetapi tidak menerima komunikasi dari komputer yang tidak tepercaya serta antivirus yang lebih kuat.
  • Apakah ini laptop yang akan bepergian? Anda pasti menginginkan keamanan yang sangat, sangat kuat. Firewall lokal diinstal di komputer (firewall Windows baik-baik saja, Anda mungkin lebih memilih yang lain) dan perangkat lunak antivirus yang bagus.
  • Apakah pengguna harus dapat menginstal sesuatu? Mungkin tidak berdasarkan pertanyaan Anda.
