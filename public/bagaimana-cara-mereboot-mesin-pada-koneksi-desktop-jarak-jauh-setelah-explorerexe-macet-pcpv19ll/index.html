Bagaimana cara mereboot mesin pada koneksi desktop jarak jauh setelah explorer.exe macet?

Bagaimana cara me-reboot mesin di file Desktop Jarak Jauh koneksi setelah explorer.exe crash?

Saya terjebak dalam mencoba melakukan ini. Setelah saya mencari di Google sedikit dan menemukan bahwa orang-orang mengatakan untuk menggunakan Ctrl + Alt + ESC atau Ctrl + Alt + Del untuk menyalakan Pengelola tugas . Keduanya tidak berhasil. Jadi sekarang yang saya miliki hanyalah desktop kosong dan tidak ada kombinasi tombol yang dapat berfungsi sama sekali.

Bagaimana cara menampilkan Pengelola Tugas sehingga saya dapat memulai contoh baru explorer.exe atau bagaimana cara memulai ulang mesin melalui pintasan keyboard?

jika Anda menggunakan Windows Remote Desktop Connection,

  1. Sebelum Anda menekan hubungkan, klik opsi tarik-turun,
  2. Pergi ke tab Program,
  3. Centang kotak yang bertuliskan "Mulai program Mengikuti di Sambungan"
  4. Masuk ' penjelajah '(tanpa tanda kutip) ke dalam Jalur Program dan Nama File Kotak
  5. Hubungkan Biasanya

Saya memiliki masalah serupa dengan salah satu instance VM kerja kami dan sangat sulit untuk mendapatkan I.T. untuk mem-boot ulangnya sepanjang waktu, dan sepertinya saya tidak dapat membuka Pengelola Tugas saat instance dalam keadaan ini.

Yang perlu Anda gunakan adalah Ctrl + Alt + Akhir dan itu akan memungkinkan Anda mengakses pengelola tugas. Kemudian, klik File -> New Task (Run...) dan ketik explorer.exe dan tekan Ok. Explorer akan mulai kembali dan Anda dapat melanjutkan bekerja seperti sebelumnya. Jika Anda benar-benar ingin mem-boot ulang saat ini (meskipun biasanya tidak diperlukan), Anda dapat melakukannya dengan mengeklik Start dan mengetik command prompt dan menekan Memasukkan , untuk membuka jendela prompt perintah. Kemudian ketik shutdown /r dan tekan Memasukkan dan itu akan mereboot komputer Anda setelah jeda waktu yang singkat.
