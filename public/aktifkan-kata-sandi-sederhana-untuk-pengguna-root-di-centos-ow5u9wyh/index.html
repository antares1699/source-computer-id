Tapi saya mendapatkan kesalahan ini:

BAD PASSWORD: it is based on a (reversed) dictionary word

Adakah yang punya ide bagaimana mengaktifkan sandi semacam ini?

Lihat ini:

Anda dapat menggunakan PAM (Modul Otentikasi yang Dapat Dicolok) untuk mengkonfigurasi file pemeriksaan kekuatan kata sandi sederhana dan kebijakan pengubahan kata sandi untuk semua pengguna. /etc/pam.d/system-auth menyediakan pengaturan penting untuk otentikasi sistem.

pam_cracklib - Ini adalah modul pemeriksaan kekuatan kata sandi sederhana untuk PAM. Selain memeriksa kata sandi biasa, ia menawarkan dukungan untuk kata sandi dan dapat memberikan kata sandi yang dibuat secara acak. pam_passwdqc - Modul ini menyediakan fungsionalitas hanya untuk satu grup manajemen PAM: perubahan kata sandi. Dalam hal parameter tipe modul, ini adalah fitur "kata sandi". pam_chauthtok () - Fungsi layanan mungkin meminta pengguna untuk kata sandi baru, dan verifikasi bahwa kata sandi tersebut memenuhi minimum tertentu standar. Jika kata sandi yang dipilih tidak memuaskan, layanan fungsi mengembalikan PAM_AUTHTOK_ERR.

Pengaturan Kekuatan Memeriksa Kata Sandi

Modul PAM pam_cracklib default menyediakan pemeriksaan kekuatan kata sandi. Itu menolak kata sandi jika salah satu dari yang berikut ini kondisi yang ditemukan:

Palindrome - Apakah kata sandi baru merupakan palindrome dari yang lama? Kasus Ubah Saja - Apakah kata sandi baru adalah yang lama dengan hanya perubahan kasus? Serupa - Apakah kata sandi baru terlalu mirip dengan yang lama? Sederhana - Apakah kata sandi baru terlalu kecil? Diputar - Apakah kata sandi baru versi rotasi dari kata sandi lama? Sudah digunakan - Apakah kata sandinya digunakan di masa lalu? Kata sandi yang sebelumnya digunakan dapat ditemukan di / etc / security / opasswd.

Cara Menggunakan pam_passwdqc - Modul PAM Kontrol Kualitas Kata Sandi

Edit file /etc/pam.d/system-auth: cp /etc/pam.d/system-auth / root / backup / system-auth vi /etc/pam.d/system-auth

Temukan garisnya:

kata sandi diperlukan pam_cracklib.so try_first_pass retry = 3 dan ganti dengan baris berikut:

kata sandi diperlukan pam_passwdqc.so min = dinonaktifkan, dinonaktifkan, 12,8,7 retry = 3 Dimana,

min = N0, N1, N2, N3, N4 - min = dinonaktifkan, dinonaktifkan, 12,8,7 adalah kata sandi kebijakan. Setiap file (N0, N1..N4) digunakan untuk tujuan yang berbeda. Itu kata kunci yang dinonaktifkan dapat digunakan untuk melarang kata sandi jenis tertentu terlepas dari panjangnya. Setiap nomor berikutnya harus diisi tidak lebih besar dari yang sebelumnya. N0 digunakan untuk password yang terdiri karakter dari satu kelas karakter saja. Kelas karakter adalah - angka, huruf kecil, huruf besar, dan karakter lainnya. N1 digunakan untuk password yang terdiri dari dua karakter kelas karakter yang tidak memenuhi persyaratan frasa sandi. N2 digunakan untuk kata sandi. Frasa sandi harus terdiri dari cukup kata-kata (lihat opsi frasa sandi di bawah). N3 dan N4 digunakan untuk kata sandi yang terdiri dari karakter dari tiga dan empat karakter kelas, masing-masing. Saat menghitung jumlah karakter kelas, huruf besar digunakan sebagai karakter dan angka pertama digunakan sebagai karakter terakhir dari kata sandi tidak dihitung. Tambahan agar cukup panjang, kata sandi harus berisi cukup karakter yang berbeda untuk kelas karakter dan panjang minimum mereka telah diperiksa. retry = 3 - Jumlah kali modul akan meminta kata sandi baru jika pengguna gagal memberikan kata sandi yang cukup kuat dan masukkan dua kali untuk pertama kalinya. Lihat file bantuan /usr/share/doc/pam_passwdqc-1.0.2/README dan orangnya halaman pam_passwdqc untuk opsi konfigurasi terperinci.

  1. vi /etc/pam.d/system-auth sebagai root.

  2. Cari dua baris berikut:

    password    requisite     pam_cracklib.so try_first_pass retry=3
password    sufficient    pam_unix.so md5 shadow nullok try_first_pass use_authtok

  3. Komentari baris pertama dari dua baris:

    #password    requisite     pam_cracklib.so try_first_pass retry=3

  4. Hapus use_authtok di baris kedua. Jika tidak, Anda akan mendapatkan kesalahan “passwd: Informasi otentikasi tidak dapat dipulihkan”.

    password    sufficient    pam_unix.so md5 shadow nullok try_first_pass

  5. Itu dia. Coba ubah sandi Anda lagi.
