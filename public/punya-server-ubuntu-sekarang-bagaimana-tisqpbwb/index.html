Jangan khawatir, di lingkungan server tidak ada sistem windowing yang terpasang. Beberapa alasan mengapa:

  • Konsumsi sumber daya - tergantung pada penggunaan server, Anda biasanya ingin sebagian besar sumber daya digunakan oleh program yang melayani klien, dan tidak menjalankan server itu sendiri.

  • Keamanan - semakin sedikit Anda menginstal, semakin sedikit aspek server Anda yang terbuka untuk diserang.

antara lain Anda mungkin ingin mempertimbangkan stabilitas (jika menggunakan perangkat lunak jendela terbaru), waktu boot yang lebih cepat (biasanya dapat dimodifikasi di inittab untuk menghindari GUI), dan masalah PEBKAC. Pada beberapa layar masuk GUI Anda akan melihat opsi seperti mematikan, memulai ulang, mengubah pengelola jendela, dan sebagainya. Dengan antarmuka berbasis teks, pejalan kaki akan kesulitan mengacaukan sistem jika menara terkunci. Tidak banyak yang dapat Anda lakukan dengan prompt login hitam. Dengan GUI di sisi lain, orang dapat dengan mudah mematikan sistem. Di Linux, Anda akan melihat sebagian besar administrasi lebih cepat di baris perintah, dengan asumsi Anda tahu perintah dasar juga.

Beberapa tautan berharga:

Proyek Dokumentasi Linux

Yang pertama adalah pendamping Anda untuk mengelola Ubuntu 10.4 dari baris perintah. TLDP adalah referensi klasik untuk semua hal tentang Linux. Saya akan tetap menggunakan panduan server pada awalnya karena TLDP memiliki sumber daya yang berkaitan dengan distribusi berbasis Redhat juga, yang mungkin membingungkan Anda pada awalnya.
