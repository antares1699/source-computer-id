Cetak file PDF dengan jalur file di footer

Saya ingin mencetak PDF di windows, tetapi ingin mencetak jalur file PDF di footer dokumen yang dicetak (seperti yang bisa saya lakukan di Word).

Kami menggunakan Adobe Reader atau Foxit Reader.

Apakah Anda tahu cara untuk mencapai ini?

Oke. Izinkan saya juga mencobanya. Karena kita di sini di superuser.com (dan bukan stackoverflow.com), saya akan melakukannya tanpa bahasa pemrograman , dan gunakan hanya skrip (batch) sebagai gantinya.

Langkah pertama: Gunakan Ghostscript untuk membuat halaman footer PDF yang menampilkan string 'footer' dengan jalur

Kami akan menggunakan yang relatif sederhana Ghostscript.dll perintah untuk itu. Perintah tersebut mengambil cuplikan PostScript pendek sebagai masukan baris perintah (-c "...") dan mengeluarkan PDF sebagai 'myfooter.pdf'. Kami akan menggunakan surat sebagai ukuran media (ganti dengan a4 atau apapun yang Anda butuhkan). Ini dia:
(Saya berasumsi kami menggunakan Windows. Jika salah satu pembaca menggunakan Linux, gunakan gs daripada gswin32c.exe.)

gswin32c.exe ^
 -o myfooter.pdf ^
 -sDEVICE=pdfwrite ^
 -sPAPERSIZE=letter ^
 -c "72 12 moveto /Helvetica findfont 9 scalefont setfont (File: d:\\c\\b\\a.pdf) show"

Sekarang kami memiliki halaman PDF yang hanya berisi footer, dengan sisa kanvas tanpa konten apa pun. Anda dapat membukanya di penampil dan melihat bahwa string dimulai pada 72 titik PostScript (72 pt == 1 inci) di kiri sudut kiri bawah, dan 12 poin ke atas.

Langkah kedua: Gunakan PDF ToolKit untuk menghamparkan halaman footer dengan PDF asli Anda

Kami akan menggunakan PDF ToolKit pdftk.exe perintah sekarang. Utilitas baris perintah ini dapat melakukan banyak hal pada PDF. Fitur yang kita butuhkan di sini: overlay halaman dari dua dokumen PDF yang berbeda satu sama lain untuk menyediakan file 'stempel' atau a 'Latar Belakang' ke file aslinya. Ini dia:
(Jika pembaca menggunakan Linux, gunakan pdftk alih-alih pdftk.exe.)

pdftk.exe ^
  d:\c\b\a.pdf ^
  stamp myfooter.pdf ^
  output d:\c\b\result-with-footer-a.pdf

(BTW, pdftk juga berasal dari pustaka iText, yang dirujuk oleh jawaban lain.)

Langkah ketiga: Buat skrip batch yang membuat kedua langkah sebelumnya terjadi secara otomatis

Ini tersisa untuk kesenangan berolahraga Anda ... ;-)

Anda mungkin ingin memutar string sejauh 90 derajat, sehingga terbaca secara vertikal di tepi kiri halaman. Untuk kasus ini ubah perintah Ghostscript seperti ini: gswin32c.exe -sDEVICE=pdfwrite -o myfooter.pdf -dAutoRotatePages=/None -c "12 72 moveto /Helvetica findfont 9 scalefont setfont 90 rotate (File: d:\\c\\b\\a.pdf) show showpage".

Berikut adalah file batch windows yang dibangun @ Jawaban Kurt :

@echo off
if [%*]==[] goto :usage
if not exist outdir mkdir outdir

for %%a in (%*) do (
 call :makeFooter %%a
 call :addFooter %%a
 )
goto :eof

:makeFooter
  gswin32c ^
  -o outdir\footer_%1 ^
  -sDEVICE=pdfwrite ^
  -c "5 5 moveto /Helvetica findfont 12 scalefont setfont (%~n1) show"
  goto :eof

:addFooter
  pdftk ^
  %1 ^
  stamp outdir\footer_%1 ^
  output outdir\%1
  del outdir\footer_%1
  goto :eof

:usage
  echo. &echo. -={ Usage: %0 [list of pdf files or pattern] }=-
  echo. &echo. %~n0 foo*.pdf &echo.

  • ubah: nama file keluar sekarang tetap sama dengan input, juga menghapus file footer sementara

Anda mungkin perlu mengubah gswin64c untuk penginstalan ghostscript 64bit.

Satuan dalam garis -c dari :makeFooter berada dalam titik. Jadi 5 5 /moveto adalah 5pts masuk dan 5pts ke atas dari kiri bawah halaman, dan /Helvetica ... 12 adalah menggunakan font Helvetica 12pt.

Tidak ada cara untuk memperbarui ini secara otomatis seperti bidang di Word. Tapi yang bisa Anda lakukan adalah memproses file setelah dibuat, dan dipindahkan ke lokasi selain dokumen sumber. PDF benar-benar dirancang untuk dicetak dan diubah untuk memungkinkan formulir dan hal-hal lain hanyalah sebuah retasan.

Saya menduga bahwa Anda akan menginginkan ini menjadi semacam proses otomatis dan generik, jadi Anda perlu menggunakan perpustakaan terpisah untuk melakukan ini. Salah satu yang paling produktif adalah iText yang merupakan perpustakaan open source java yang memungkinkan pengeditan file PDF. Ini tidak terlalu sulit untuk digunakan dan saya akan memberikan contoh di bawah kode yang Anda perlukan untuk membuat latihan ini berhasil.

Anda akan membutuhkan JDK dan iText dari sourceforge. Berikut adalah kode java

import java.io.*;
import com.itextpdf.text.pdf.*;
import com.itextpdf.text.*;
public class AddFooter {
    public static void main(String[] args) throws Exception {
        String fileName;
        try {
            fileName = args[0];
        } catch (Exception e) {
            System.out.println("You need to pass the filename after command");
            return;
        }
        int         bottomHorizontalOffset = 100;
        int         bottomVerticalOffset = 10;
        String      workingDir = System.getProperty("user.dir");
        String      file = workingDir + "\\" + fileName;
        String      stamped = workingDir + "\\" + fileName + "_stamped.pdf";
        stamped = stamped.replaceAll("\\\\", "\\\\\\\\");
        file    = file.replaceAll("\\\\", "\\\\\\\\");
        PdfReader   pdfReader   = new PdfReader(file);
        PdfStamper  pdfStamper  = new PdfStamper(pdfReader, new FileOutputStream(stamped));

        for(int i = 1; i <= pdfReader.getNumberOfPages(); i++) {
            PdfContentByte content = pdfStamper.getUnderContent(i);
            BaseFont bf = BaseFont.createFont(BaseFont.HELVETICA, BaseFont.CP1252, BaseFont.NOT_EMBEDDED);
            content.beginText();
            content.setFontAndSize(bf, 12);
            content.setTextMatrix(bottomHorizontalOffset, bottomVerticalOffset);
            stamped = stamped.replaceAll("\\\\\\\\", "\\\\");
            content.showText(stamped);
            content.endText();
        }
        pdfStamper.close();
    }
}

Kode ini memiliki beberapa masalah, dan karena saya tidak tahu semua spesifikasinya maka ini dia:

  1. PDF baru dibuat dengan nama yang berbeda, dan itu adalah jalur dan nama di footer
  2. Anda dapat mengubah variabel ... bawah untuk memposisikan teks di bagian bawah
  3. Anda perlu menyesuaikan font dan ukurannya sesuai dengan dokumen Anda
  4. Ini dikodekan untuk windows, jadi beberapa tweak diperlukan untuk * nix
  5. Itu diharapkan untuk dijalankan dari direktori dimana file tersebut berada, begitulah cara mendapatkan path.
  6. Ini TIDAK akan memutakhirkan secara otomatis jika Anda memindahkan file, pada kenyataannya Anda tidak dapat menjalankan ini untuk kedua kalinya dan menghilangkan teks asli.
  7. Kode ini digabungkan dengan cepat, tetapi dapat dikompilasi dan berfungsi, jadi jika Anda membutuhkan lebih banyak, ini akan menjadi titik awal yang baik.

Karena pdf tidak memiliki kolom 'autoupdate', Anda tidak ingin mengubah asosiasi pdf agar mengarah ke skrip untuk menjalankan ini, karena setiap kali Anda membuka file tersebut, footer akan ditulis ulang.
