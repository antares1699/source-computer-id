Mengapa bersikeras untuk meluncurkan paket MSI?

Saya telah membuat program di Autohotkey, yang membuat beberapa tombol pintas yang ingin dipasang secara permanen oleh seluruh perusahaan dengan 400 orang. Cara saya membayangkan program bekerja adalah bahwa pintasan ke program ditambahkan ke menu Start di bawah folder Startup, jadi program ini dimulai secara otomatis dan transparan saat komputer dihidupkan. Program ini akan terus berjalan sepanjang hari.

Saya seorang freelancer tanpa akses ke departemen TI, tetapi secara tidak langsung saya telah melihat email yang mereka desak bahwa saya harus membuat file MSI untuk meluncurkan program ke semua komputer ini. Masalah pertama yang saya alami saat membuat paket MSI adalah saya tidak tahu bagaimana file MSI dapat dibuat berdasarkan file .EXE. Yang kedua adalah bahwa mereka mengatakan bahwa sangat mahal bagi mereka untuk membuat file seperti itu (saya percaya itu berarti harganya sekitar $ 20.000 untuk menyelesaikannya), oleh karena itu mereka tidak ingin sering memperbarui, sementara saya percaya itu sangat penting untuk dapat dengan cepat menyesuaikan program dengan kebutuhan baru dan memperbaiki bug.

Pada hari Rabu, saya akan mengadakan pertemuan dengan mereka, dan oleh karena itu saya ingin tahu argumen seperti apa yang mungkin mereka miliki untuk menggunakan paket MSI, dan mengapa mereka tidak dapat begitu saja membuat pintasan ke file .EXE saya di semua komputer ini (ini seharusnya tidak terlalu sulit, bukan?). Pintasan harus mengarah ke file .EXE, yang terletak di server file sehingga seseorang dapat dengan mudah menggantinya untuk memperbarui program (komputer selalu memiliki akses ke server file, artinya tidak ada laptop).

Apa argumen mereka?
