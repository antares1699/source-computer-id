/ etc / shadow dalam format lama, di mana salt disimpan?

Di / etc / shadow saya memiliki entri seperti admin:YtChlvAGYzva2:16318:0:99999:7:::. Saya tahu kata sandi asli dan ingin membuat hash yang sama. Namun, menjalankan openssl passwd -crypt password memberi saya hasil yang berbeda setiap kali saya menjalankannya. Saya menganggap garam terlibat, jadi di mana saya dapat menemukan garam yang digunakan untuk membuat hash asli?

Edit: Saya mendapatkan hash asli menggunakan perintah berikut:

openssl passwd -crypt -salt Yu password

Salt dan kata sandi terenkripsi keduanya digabungkan menjadi string YtChlvAGYzva2.

Dari Shadow Password Howto :

Ketika seorang pengguna memilih atau diberi kata sandi, itu dikodekan dengan nilai yang dihasilkan secara acak yang disebut garam. Artinya apapun kata sandi tertentu dapat disimpan dengan 4096 cara berbeda. Garam nilai tersebut kemudian disimpan dengan sandi yang disandikan.

Saat pengguna masuk dan memberikan kata sandi, garam adalah yang pertama diambil dari sandi yang disandikan yang disimpan.

String kata sandi yang lebih panjang yang Anda lihat dengan sistem modern pisahkan hash menggunakan $ . Tetapi untuk sistem yang lebih lama, itu baru saja dihaluskan ( Wikipedia ):

Versi Unix sebelumnya menggunakan file kata sandi (/ etc / passwd) untuk menyimpan hash kata sandi asin (kata sandi diawali dengan dua karakter garam acak). Dalam versi Unix yang lebih lama ini, garamnya juga disimpan dalam file passwd (sebagai cleartext) bersama dengan hash dari file kata sandi asin.

Dalam contoh Anda, saya yakin bahwa saltnya adalah "Yt" dan sandi terenkripsi adalah "ChlvAGYzva2". Ini benar-benar dua karakter pertama dari string.
