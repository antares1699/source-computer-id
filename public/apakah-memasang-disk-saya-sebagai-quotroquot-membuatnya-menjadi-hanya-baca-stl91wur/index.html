Apakah memasang disk saya sebagai "ro" membuatnya menjadi hanya-baca?

Saya mendapat instalasi dualboot dengan Ubuntu 10.04 dan Windows 7 Ultimate. Saya memiliki 2 drive dengan pengaturan partisi berikut:

500 GB drive
2 partitions:
    C:/   320GB NTFS (Has Win 7 on it)
    D:/   180GB NTFS


160 GB drive
1 main partition:
    1     160GB EXT4 (Has Ubuntu 10.04 in it)

Di Ubuntu, saya secara otomatis memasang seluruh drive 500GB, menggunakan saluran /etc/fstab. Saya telah mengonfigurasi baris itu sebagai berikut:

UUID=XXXXXXXXXXXXXXXX /media/WinEmily         auto    ro,auto,user,exec 0 0

Di mana UUID adalah salah satu dari drive 500GB dan /media/WinEmily adalah titik pemasangan saya yang ada.

Sekarang pertanyaan saya adalah, apakah ada cara bagi Ubuntu untuk tetap menulis ke salah satu partisi dari drive 500GB. Saya mencoba mencegahnya sehingga itu hanya akan memiliki akses baca, apa pun yang dicoba. (Kecuali tentu saja melepas dan memasang kembali dengan akses baca / tulis.)

Memperbarui
Saya berbicara tentang operasi file normal, mengedit dan menghapus file dari sistem file. Tidak menghancurkan seluruh sistem file menggunakan brute force.

Jadi apakah ada perintah seperti ini, yang akan memungkinkan penulisan meskipun dipasang sebagai hanya-baca.

supersuperuserdo rm --doitanyway /media/WinEmily/file.txt

Jika Anda memasang sistem file hanya-baca, tidak ada cara untuk mengubahnya melalui operasi sistem file. Bahkan jika Anda root. Jika Anda ingin melakukan modifikasi apa pun (menulis ke file, menghapus file, mengubah izin, memperbarui waktu akses, dll.), Anda harus memasang ulang sistem file baca-tulis (mount -o remount,rw /dev/foo).

Anda masih dapat mengakses perangkat yang mendasarinya dan menulis padanya. (Itu ide yang sangat buruk, tentu saja.) Disk dan partisi hanya dapat diakses oleh root kecuali Anda telah mengubahnya secara eksplisit (biasanya ide yang buruk juga).

Satu hal yang tidak terlalu sering muncul adalah bahwa tindakan pemasangan itu sendiri mungkin menulis ke disk. Hal ini dapat terjadi bahkan dengan mount read-only untuk filesystem journal: jika filesystem tidak di-unmount dengan bersih, tindakan mount filesystem dapat memutar ulang jurnal dan melakukan antrian (berlaku untuk ext3; saya tidak tahu tentang ntfs) . Ini berarti Anda tidak dapat dengan mudah memeriksa disk mesin yang ditangguhkan (misalnya, gambar disk dari mesin virtual yang dijeda yang dilihat di host, atau sistem hibernasi dari CD penyelamat).

Dalam kondisi normal:

Jika Anda ingin melindunginya dari diri Anda sendiri, artinya operasi yang tidak disengaja, ini sudah cukup.

Tidak ada perintah yang layak seperti supersuperuserdo rm --doitanyway /media/WinEmily/file.txt yang dapat menyebabkan file.txt dihapus.

Untuk menyebabkan kerusakan harus ada dua hal: sudo, dan mount -o rw atau /dev/sda1.

Sekarang, jika masalahnya adalah orang lain yang memiliki akses ke komputer Anda, Anda harus percaya bahwa mereka tidak akan menggunakan kata-kata mount -o rw dan /dev/sda1, atau tidak memberi mereka akun admin. Dengan akun admin mereka dapat melakukan apapun yang mereka inginkan. Tingkat perlindungan lain dalam hal ini adalah dengan memasuki pengaturan BIOS, melarang booting dari CD / USB, dan melindungi sandi pengaturan BIOS itu sendiri.
