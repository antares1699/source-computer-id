OS X: apakah Anda benar-benar perlu menyeret aplikasi ke folder "Aplikasi"?

Saat Anda menginstal aplikasi OS X dari file .dmg, ini sangat umum, saat Anda memasang / membuka file .dmg .dmg , untuk melihat ikon aplikasi diikuti dengan panah yang menunjuk ke folder "Aplikasi", seperti ini:

CoolApp => Applications

Memberi petunjuk kepada Anda bahwa Anda harus menyeret & melepaskan aplikasi ke dalam folder Applications /.

Namun untuk banyak aplikasi, seperti Skype, Anda bisa meletakkannya di mana saja Anda mau, seperti di desktop Anda.

Untuk beberapa aplikasi, Anda bahkan tidak dipaksa untuk memasukkan kata sandi root sistem. Namun bahkan aplikasi ini biasanya menyarankan agar Anda meletakkan aplikasi di utama (seluruh sistem) Aplikasi/ map.

Selain membuat aplikasi tersedia untuk semua pengguna (yang tidak merupakan argumen yang bagus pada sistem yang memiliki akun pengguna unik), apa lagi yang saya lewatkan jika saya hanya menginstal aplikasi di beberapa folder akun pengguna biasa (yaitu : tidak seluruh sistem)?

Saya kira pertanyaan saya sebenarnya adalah: apakah aplikasi OS X yang dirancang dengan benar dijamin berfungsi bahkan jika Anda melakukannya tidak seret ke folder Aplikasi / di seluruh sistem?

Untuk sebagian besar aplikasi Macintosh yang didesain dengan benar, menerapkannya ke folder / Applications adalah tidak perlu.

Namun, ini dianggap sebagai praktik terbaik. Saat seseorang bekerja di Mac Anda (EG, salah satu karyawan di Apple Genius Bar), menempatkan aplikasi Anda di satu tempat, waras, dan sistem yang dikenali membuat hidup mereka jauh lebih mudah. Masuk akal juga bahwa aplikasi yang dirancang dengan buruk dapat berperilaku tidak baik saat tidak diinstal ke / Applications, meskipun saya biasanya tidak menemukannya. Beberapa aplikasi baru mendeteksi ketika diluncurkan dari tempat selain / Aplikasi (biasanya .DMG dari unduhannya) dan secara otomatis menginstal sendiri dengan benar.

Jika Anda ingin dengan cepat mengakses aplikasi yang diterapkan di folder / Aplikasi (atau Anda tidak ingin menggali miliaran entri untuk menemukan aplikasi yang Anda inginkan), saya sarankan menggunakan Alias ​​(Command-L di Finder), dan menempatkan Alias ​​di mana pun Anda suka. Misalnya, saya menggunakan folder Dock ("Tumpukan") yang berisi Alias ​​untuk semua kebutuhan peluncuran aplikasi saya, dengan folder untuk Pengembangan, Produktivitas, Multimedia, Game, Internet, dll.

Setiap kali seseorang mengajukan pertanyaan seperti ini (atau, sebagai alternatif di dunia Windows, "apakah saya perlu menggunakan folder 'Dokumen Saya' untuk file saya"), saya selalu bertanya balik: Apakah Anda benar-benar membutuhkan tidak untuk melakukannya? Salah satu yang tidak ditangani oleh mekanisme in-box aliasing ("pintasan" di Windows)?

Ini memang tidak perlu. Namun yang terbaik adalah menyeretnya keluar dari .dmg (diskimage) tempat mereka berada ke dalam folder di hard drive Anda. Jika tidak, Anda harus menyimpan image disk dan memasangnya setiap kali Anda ingin meluncurkan aplikasi. Tapi seperti yang dikatakan John, ini adalah praktik praktis, simpan barang di tempat yang mudah ditemukan. Biasanya mereka akan mandiri dan lari dari mana saja. Saya hanya meletakkan beberapa aplikasi yang saya tahu istri saya tidak akan ingin menggunakannya di folder Home / Applications saya sendiri (yang juga merupakan tempat standar) untuk aplikasi yang tidak ingin saya instal sistem secara luas. Ini mirip dengan saya memiliki direktori / bin di sana untuk program baris perintah saya sendiri (utilitas yang saya tulis sendiri misalnya) agar tidak "mencemari" sistem terlalu banyak. Jadi jika Anda ingin menginstalnya hanya untuk satu pengguna, letakkan di folder Aplikasi pengguna tersebut di direktori home-nya. Itu adalah lokasi yang jelas untuk tujuan itu ...
