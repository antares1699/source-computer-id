Mengapa Windows memiliki Unix-style etc / hosts? [Tutup]

Mengapa Windows memiliki %systemroot%/system32/drivers/etc/hosts, dibandingkan dengan sesuatu yang lebih "Windowsy"? Rasanya seperti peninggalan BSD dari turunan Windows mendapat beberapa bit dari ribuan tahun yang lalu. Apakah ini murni hal kompatibilitas?

Ya, itu datang dengan adopsi Microsoft TCP / IP berdasarkan API soket BSD seperti yang diterapkan sesuai dengan spesifikasi WinSock.

Winsock mengikuti model Windows Open System Architecture (WOSA); itu mendefinisikan antarmuka penyedia layanan standar (SPI) antara antarmuka pemrograman aplikasi (API), dengan fungsi yang diekspor dan tumpukan protokol. Ia menggunakan paradigma soket yang pertama kali dipopulerkan oleh Berkeley Software Distribution (BSD) UNIX. Itu kemudian diadaptasi untuk Windows di Windows Sockets 1.1

http://msdn.microsoft.com/en-us/library/windows/desktop/ms740673(v=vs.85).aspx
