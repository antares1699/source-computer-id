Apakah ada cara untuk sepenuhnya menghilangkan dukungan 32-bit pada jendela 64-bit?

Saya ingin menonaktifkan seluruh syswow64. Tujuan saya adalah membuatnya mustahil untuk menjalankan aplikasi 32-bit apa pun, atau memuat DLL 32-bit apa pun pada sistem windows 64-bit. Mirip dengan aplikasi Windows 16-bit lama, yang dapat diemulasikan pada beberapa sistem 32-bit lama, tetapi tidak lebih pada 64-bit.

Di Linux, saya cukup menonaktifkan emulasi 32-bit baik di konfigurasi kompilasi kernel atau dengan beberapa pengaturan sysctl. Tapi apa yang bisa dilakukan di Windows?

Apakah itu mungkin?

Saya tahu, bahwa program 32-bit mana pun — dan mungkin beberapa 64-bit — tidak akan berfungsi lagi, tetapi itu bukan masalah bagi saya. Sebenarnya itu tujuan saya.

Tujuannya adalah untuk bereksperimen di lingkungan Windows hanya 64-bit. Memulai perdebatan tentang kegunaannya, atau dibutuhkan atau tidak, apa adanya sangat offtopic sini. aku ingin sebuah larutan , tidak a perdebatan .

Metode paling sederhana untuk mencapai hal ini adalah dengan tidak mengizinkan aplikasi dipasang oleh pengguna biasa. Jika Anda hanya menginstal aplikasi 64-bit, maka hanya aplikasi 64-bit yang akan berjalan, kecuali proses 32-bit yang masih digunakan oleh Windows, tentunya.

Tidak mungkin menonaktifkan subsistem WoW64, yang bukan "emulasi", pada Windows versi konsumen. Mungkin dengan Server Windows . Saya akan menganggap ini adalah Windows Feature pada Windows Server yang dapat dengan mudah dinonaktifkan. Artikel tertaut menyediakan perintah, yang merupakan versi khusus untuk versi Windows Server yang lebih lama.
