Sed untuk windows

Saya mencoba menggunakan SED untuk pertama kalinya. Untuk memperumit masalah, saya menggunakannya di Windows. Saya mengunduh dari sumber ini . Karena saya tidak memiliki hak penginstalan pada mesin kerja saya, saya membuat folder di profil saya untuk file yang dapat dieksekusi dan menambahkannya ke PATH saya.

Saya mengalami masalah saat mencoba menggunakan SED untuk pertama kalinya. Itu terus gagal dengan kesalahan:

sed: -e expression #1, char 1: unknown command: `''

Saya telah mencoba beberapa sampel berbeda dari ini bagaimana , tapi saya tidak bisa mendapatkannya. Saya tidak menemukan orang lain yang mengalami masalah ini. Apa yang harus saya coba? Saya telah mencoba merangkumnya dalam tanda kutip, tanda kutip ganda, dan tanda kutip belakang. Sepertinya tidak ada yang membantu.

Tidak seperti Unix, shell baris perintah Windows tidak melakukan pemisahan kata dan tidak menghapus tanda kutip; program hanya menerima satu string yang berisi seluruh baris perintah. Artinya tidak semua program dapat mengikuti aturan kutipan yang sama.

Dalam hal ini, versi GnuWin32 dari sed hanya mendukung satu gaya kutipan - " tanda kutip ganda ". Misalnya, dalam pengujian saya, berikut ini berfungsi dengan baik:

sed "s/foo/bar/"

Anda juga bisa mendapatkan sed dari Cygwin , bersama dengan sistem mirip Unix yang lebih lengkap: shell, editor, alat lainnya. Akan lebih mudah jika Anda menggunakan shell Unix untuk belajar, untuk menghindari masalah sintaks di masa mendatang.
