Yang saya maksud dengan "komprehensif" semua URL: tidak hanya yang terdaftar oleh fungsi riwayat browser biasa, yang akan saya sebut sebagai URL "utama", tetapi juga semua URL lain yang diakses oleh browser dalam proses menyajikan URL utama (yaitu pengalihan, file media , lembar gaya, skrip, bingkai, dll.).

Saya kedua jawaban lainnya rekomendasi untuk Fiddler.

Selain itu, banyak browser memiliki plugin "header HTTP langsung" dan alat pengembangan web lainnya. Menginstal ini akan memungkinkan Anda melihat semua permintaan, seringkali dalam bentuk ringkasan yang bagus. Cari tempat penyimpanan plugin browser Anda "header HTTP langsung" (tautan menuju ke plugin Chrome - sini adalah Firefox, sini adalah satu untuk IE). Jenis plugin ini hanya akan memberi Anda informasi yang diminta browser, berbeda dengan Fiddler yang akan memberi Anda informasi tentang semua yang diminta oleh setiap aplikasi (yang bagus; tetapi Anda harus sedikit memilahnya, tidak yakin apa yang Anda persyaratannya).

Di Chrome, jika Anda membuka Alat -> Alat Pengembang, tab "Jaringan" dan "Sumber" akan menampilkan informasi ini, meskipun hanya untuk tab saat ini.

Di IE (setidaknya 11), bagian jaringan di Alat -> Alat Pengembang F12 juga akan memberi Anda informasi itu.

Di Firefox, saya tidak ingat, tapi saya yakin ada yang serupa.

Di Safari, sepertinya Anda bisa pergi ke Periksa Elemen -> Panel Sumber Daya .

Namun, tidak satu pun dari ini benar-benar memiliki kemampuan untuk menghasilkan daftar berformat bagus tanpa setidaknya sejumlah menyalin / menempel / mengedit di pihak Anda. Alat browser built-in mungkin yang paling dekat sejauh mengasosiasikan dengan "URL utama", dengan plugin header HTTP Live datang di samping untuk memantau hanya sesi (tetapi tidak begitu banyak asosiasi dengan URL utama), dan Fiddler memberikan Anda banyak informasi yang tidak disortir.
