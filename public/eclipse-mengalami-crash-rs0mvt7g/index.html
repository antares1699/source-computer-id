Nima, Apakah Anda menggunakan versi 64 bit dari JDK dan versi 64 bit dari Helios? . Jika tidak, cobalah memiliki keduanya pada 64 bit, sehingga Anda dapat memanfaatkan OS Anda. Juga, edit file eclipse.ini Anda untuk meluncurkannya dengan menunjuk ke vm tertentu. misal:

-vm
myjdk/1.6.0_22/bin/javaw.exe

[ASK] Masalah diselesaikan dengan menghapus file .snap dari yourworkspace\.metadata\.plugins\org.eclipse.core.resources

Penjelasan untuk file snap: Saat Eclipse berjalan, informasi tentang apa yang telah berubah di file ruang kerja secara bertahap masuk ke berbagai file "snapshot" (termasuk .snap). Pada shutdown Eclipse normal, ruang kerja lengkap status disimpan dan file .snap dihapus. Saat Eclipse mogok, file snapshot digunakan selama startup berikutnya untuk memulihkan jatuh.

Jadi, tampaknya terjadi beberapa kesalahan yang menyebabkan Eclipse jatuh. Kemudian, terjadi kesalahan saat memulihkan dari kemacetan berikutnya memulai. Ini adalah masalah yang sangat mungkin tidak terkait, tetapi secara manual menghapus file snapshot Anda menipu Eclipse agar berpikir bahwa a crash tidak terjadi, sehingga rutinitas pemulihan crash tidak berjalan.

dari http://www.eclipse.org/forums/index.php/mv/msg/39563/129039/#msg_129039
