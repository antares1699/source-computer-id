Apakah perubahan dalam tabel perutean memengaruhi koneksi TCP / IP yang ada?

Jika pertanyaannya tidak cukup jelas, pertimbangkan situasi ini:

  1. Buat koneksi TCP / IP dari komputer C ke server X
  2. Ubah gateway default (komputer C) dari gateway A ke gateway B (kedua gateway dapat mencapai server X)

Sekarang, gateway mana yang akan dilalui paket? Jika tergantung pada OS, tolong beri tahu saya perilaku Windows dan Linux.

Dengan asumsi bahwa tumpukan jaringan OS segera menerapkan perubahan, maka paket akan melalui gateway saat ini.

Rute default, dan perutean secara umum, hanya IP dan IP tanpa status. Ini hanya penggerak paket. Yaitu mendapat paket dari IP A ke IP B. Tidak ada status yang disimpan antar paket, itu untuk TCP.

TCP tidak peduli rute apa yang diambil, status TCP didasarkan pada IP / Port sumber dan IP / Port tujuan sehingga koneksi TCP Anda tidak boleh terputus. Itu kecuali jika gateway Anda adalah NAT dan mengubah gateway membuat paket IP melalui NAT lain.

Ujian sebenarnya dari pertanyaan ini adalah menyiapkan skenario dan melihat apa yang terjadi, tetapi saya berasumsi Anda bertanya karena penasaran.
