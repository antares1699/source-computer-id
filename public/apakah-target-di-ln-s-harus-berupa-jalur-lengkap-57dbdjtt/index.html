Apakah target di ln -s harus berupa jalur lengkap?

Saya sudah mencoba memanggil ln di direktori ~/download seperti ini:

ln -s ./abc ~/abc [EDITED]

tapi tidak berhasil.
Saya melakukan ls -al di rumah saya dan mendapatkan tautan yang salah dari abc.

ln [OPTION]... [-T] TARGET LINK_NAME   (1st form)

Jadi pertanyaan saya di sini adalah: Apakah ln membutuhkan jalur penuh dari target? Saya tidak menemukan dokumen atau petunjuk apa pun di halaman man. Jika demikian, di mana saya dapat memperoleh informasi lebih lanjut.

Ini cukup sederhana: Anda membuat tautan simbolis pada tahun ~ yang mengarah ke ./abc di direktori itu . Nama jalur dari tautan simbolis harus valid dalam konteks direktori tempatnya berada¹. Jika Anda memang ingin mengarahkan ke file yang ada di sekarang saat ini direktori, Anda harus menggunakan jalur lengkap.

  1. Tautan harus valid jika Anda ingin menggunakannya. Itu bisa menunjukkan ketiadaan atau ke file yang tidak ada. Itu bisa menjadi hal yang baik, karena file tersebut mungkin telah ada sebelumnya dan dapat muncul kembali kapan saja, dan tautan simbolis akan mengarah ke sana lagi.

ln dapat menggunakan nama jalur absolut atau relatif; perbedaannya penting untuk tautan simbolis, tetapi karena Anda meminta untuk membuat tautan keras, itu akan menyelesaikan ke file yang sama dengan cara apa pun.

Periksa keluaran ls -li pada ~/abc dan ~/download/abc - Anda akan melihat bahwa jumlah tautan adalah 2 dan nomor inode sama untuk kedua file.
