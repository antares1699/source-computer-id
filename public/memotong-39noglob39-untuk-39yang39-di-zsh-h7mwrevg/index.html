memotong 'noglob' untuk 'yang' di zsh?

Saya ingin menggunakan perintah 'which' untuk melihat eksekusi mana yang sebenarnya dipanggil, tetapi alias 'noglob' defensif tetap menghalangi. Apakah ada pintasan yang dapat saya gunakan untuk menemukan file yang dapat dieksekusi yang dipanggil saat perintah dialias?

Saya pikir ini harus agnostik shell, tetapi saya menggunakan zsh jika itu penting.

Anda dapat menggunakan which -a COMMAND (atau where COMMAND atau whence -ca COMMAND) untuk menemukan semua kemunculan COMMAND di jalur perintah.

Sebagai contoh:

% alias ls='noglob ls'
% ls () /bin/ls
% which ls
ls: aliased to noglob ls
% which -a ls
ls: aliased to noglob ls
ls () {
        /bin/ls
}
/bin/ls

Karena alias diganti di baris perintah sebelum apa pun dieksekusi, perintah kedua dalam daftar adalah yang Anda cari (dengan asumsi tentu saja bahwa baris pertama entah bagaimana memiliki alias dengan nama yang sama)

  • Bisakah Anda menjelaskan mengapa ini berhasil? Tidak ada entri pria untuk where atau whence.
    — peter.babic
  • Di zsh which, where dan whence semuanya adalah perintah bawaan, dengan which setara dengan whence -c dan where setara dengan whence -ca. Dokumentasi mereka dapat ditemukan di man zshbuiltins. Meskipun ada halaman manual untuk which, itu bukan untuk bawaan, tetapi untuk perintah eksternal (biasanya /bin/which atau /usr/bin/which). Lihat juga keluaran dari perintah which -a which.
    — Adaephon

Gunakan which -p misalnya:

✗ which ls
ls: aliased to ls --color=tty
✗ which -p ls
/bin/ls

Dari https://linux.die.net/man/1/zshbuiltins :

(dari mana) -p

Lakukan pencarian jalur untuk nama meskipun itu adalah alias, kata khusus, fungsi shell, atau bawaan.
