Bagaimana file dan aplikasi Linux diatur?

Saya adalah pengguna baru Linux (Ubuntu) dan saya ingin tahu apakah seseorang dapat memberikan beberapa saran di mana harus menginstal sesuatu, folder mana yang tidak tersentuh, yang merupakan arti dari setiap folder dan seterusnya.

Perhatian pertama saya adalah, haruskah semuanya masuk ke folder rumah saya? Saya telah menginstal "secara manual" Komodo Edit (ini adalah IDE) dan telah masuk ke folder home saya, saya benar-benar tidak suka ide memiliki aplikasi di sana. (di windows saya dulu memiliki partisi file kerja / gambar / unduhan ... dan kemudian partisi OS dengan semua aplikasi).

Jadi, adakah tempat di mana saya dapat menginstal perangkat lunak ini? Adakah saran untuk memesan folder utama saya? Mungkin saya harus membuat folder aplikasi di direktori home saya?

Terima kasih sebelumnya. :)

pd: sebagian besar waktu saya menggunakan apt untuk menginstal sesuatu, tetapi saya tidak selalu menemukan perangkat lunak yang saya inginkan di sana ...

/home/<yourlogin> adalah 'My Documents' efektif Anda yang setara dengan windows. Desktop Anda ada di sana, gambar Anda, dll ...

Biasanya Anda mengunduh sesuatu di sana, dan mengkompilasi / menjalankan rpms dari lokasi itu yang kemudian meminta akses pengguna super (elevasi) dan akan menginstal aplikasi mereka ke tempat-tempat seperti:

/usr/bin /usr/lib dll ...

  • /bin Berisi Perintah OS Global, seperti PATH di windows
  • /sbin Berisi Perintah Global OS untuk Pengguna Super (membutuhkan elevasi)
  • /usr/bin Berisi Aplikasi
  • /usr/sbin Berisi Aplikasi yang membutuhkan ketinggian
  • /dev Berisi Perangkat (contoh: /dev/sda0 adalah Scsi Drive A, partisi 0, /dev/floppy adalah floppy drive Anda, dll ...
  • /etc adalah tempat konfigurasi sistem Anda, biasanya dalam file .conf

Kurasa itu primer dasar.

Selain itu, mengawali folder dengan . akan membuatnya 'tersembunyi' yaitu: /home/aren/.ssh, Anda masih dapat membukanya tetapi Anda harus menggunakan ls -a untuk membuatnya muncul dalam daftar saat mencari.

Wikipedia memiliki artikel bagus tentang ini . Kutipan dari bagian paling berguna dari artikel itu:

Di FHS semua file dan direktori muncul di bawah direktori root "/", bahkan jika mereka disimpan di tempat yang berbeda perangkat fisik. Catat bagaimanapun itu beberapa dari direktori ini mungkin atau mungkin tidak ada di sistem Unix tergantung apakah pasti subsistem, seperti X Window Sistem, sudah diinstal.

Mayoritas direktori ini ada di semua sistem operasi UNIX dan umumnya digunakan di banyak file cara yang sama; Namun demikian, deskripsinya di sini adalah yang digunakan khusus untuk FHS, dan tidak dianggap otoritatif untuk platform selain Linux.

  • / Akar hierarki primer dan direktori akar dari seluruh sistem file hirarki.
  • /bin/ Biner perintah penting yang perlu tersedia secara tunggal mode pengguna; untuk semua pengguna, mis., kucing, ls, cp.
  • /boot/ File pemuat boot, mis., Kernel, initrd; seringkali terpisah partisi [8]
  • /dev/ Perangkat penting, misalnya, / dev / null.
  • /etc/ File konfigurasi seluruh sistem khusus host (namanya berasal dari dan lain-lain [9]).
  • /etc/opt/ File konfigurasi untuk / opt /.
  • /etc/X11/ File konfigurasi untuk Sistem X Window, versi 11.
  • /etc/sgml/ File konfigurasi untuk SGML.
  • /etc/xml/ File konfigurasi untuk XML.
  • /home/ Direktori home pengguna, berisi file yang disimpan, pribadi pengaturan, dll.; seringkali terpisah partisi.
  • /lib/ Pustaka penting untuk binari di / bin / dan / sbin /.
  • /media/ Mount poin untuk media yang dapat dilepas seperti CD-ROM (muncul di FHS-2.3).
  • /mnt/ filesystem yang dipasang untuk sementara.
  • /opt/ Paket perangkat lunak aplikasi opsional [10].
  • /proc/ Sistem file virtual yang mendokumentasikan kernel dan status proses sebagai file teks, mis., waktu aktif, jaringan. Di Linux, sesuai dengan Procfs meningkat.
  • /root/ Direktori Home untuk pengguna root.
  • /sbin/ Sistem biner penting, misalnya, init, ip, mount.
  • /srv/ Data khusus situs yang dilayani oleh sistem.
  • /tmp/ File sementara (lihat juga / var / tmp). Seringkali tidak diawetkan antara reboot sistem.
  • /usr/ Hierarki sekunder untuk data pengguna hanya-baca; berisi mayoritas utilitas (multi-) pengguna dan aplikasi. [11]
  • /usr/bin/ Biner perintah non-esensial (tidak diperlukan dalam pengguna tunggal mode); untuk semua pengguna.
  • /usr/include/ Standar menyertakan file.
  • /usr/lib/ Pustaka untuk binari di / usr / bin / dan / usr / sbin /.
  • /usr/sbin/ Biner sistem non-esensial, misalnya, daemon untuk berbagai layanan jaringan.
  • /usr/share/ Arsitektur-independen (bersama) data.
  • /root/ Kode sumber, misalnya, kode sumber kernel dengan headernya file.
  • /usr/X11R6/ X Window System, Versi 11, Rilis 6.
  • /usr/local/ Hierarki tersier untuk data lokal, khusus untuk host ini. Biasanya memiliki subdirektori lebih lanjut, misalnya, bin /, lib /, share /. [12]
  • /var/ File variabel — file yang isinya diharapkan terus menerus berubah selama pengoperasian normal sistem — seperti log, file spool, dan file email sementara. Terkadang a partisi terpisah.
  • /var/lib/ Informasi negara. Data persisten diubah oleh program saat dijalankan, misalnya, database, metadata sistem pengemasan, dll.
  • /var/lock/ Mengunci file. File melacak sumber daya saat ini digunakan.
  • /var/log/ File log. Berbagai log.
  • /var/mail/ Kotak surat pengguna.
  • /var/run/ Informasi tentang sistem yang berjalan sejak boot terakhir, mis., pengguna saat ini masuk dan berjalan daemon.
  • /proc/ Kumparan untuk tugas-tugas yang menunggu untuk diproses, misalnya, mencetak antrian dan email yang belum dibaca.
  • /var/spool/mail/ Lokasi tidak berlaku lagi untuk kotak surat pengguna. /var/tmp/ File sementara menjadi dipertahankan antara reboot.
  • /var/www/ Hierarki file situs web (lokasi default untuk situs web yang dilayani oleh Apache).

Di bawah Linux, direktori di folder root / ditata sesuai dengan Standar Hirarki Sistem File . Biasanya seseorang menggunakan manajer paket yang disediakan oleh distribusi untuk menginstal dan menghapus perangkat lunak. Ada juga beberapa tempat di mana file konfigurasi disimpan (terutama di /etc/) dan terkadang ini perlu diedit oleh pengguna. Jika tidak, seharusnya ada sedikit kebutuhan untuk dipusingkan di luar direktori home Anda.

Ketika Anda menginstal perangkat lunak dengan tangan, Anda biasanya tidak ingin menginstal ke direktori sistem seperti /usr/bin/, /bin/ atau /sbin/ atau /usr/sbin/. Sebaliknya perangkat lunak yang diinstal secara manual harus menggunakan /usr/local/ sebagai gantinya untuk menghindari bentrokan dengan perangkat lunak yang diinstal oleh manajer paket dan yang secara efektif memiliki struktur yang sama seperti / (yaitu untuk pemasangan manual, gunakan /usr/local sebagai awalan)

Ketika Anda menginstal perangkat lunak di luar struktur biasa, shell mungkin tidak menemukan executable dan binari mungkin tidak menemukan pustaka yang ditautkan dengannya. Binari dicari di semua direktori yang terdaftar dalam variabel shell $PATH yang dapat Anda sesuaikan. Hal yang sama berlaku untuk $LD_LIBRARY_PATH yang menyimpan daftar direktori untuk mencari pustaka saat menyelesaikan dependensi tautan (lihat man ld).

(Catatan: di atas mengandung banyak kata "biasanya" dan "biasa". Ini karena tidak ada yang memaksa Anda untuk mengikuti beberapa standar (masuk akal) - Anda bebas menembak diri sendiri.)
