Mampu mengakses antarmuka web modem kabel

Diperbarui

Saya memiliki modem kabel Motorola SBG900 yang duduk di depan Linksys WRT54G yang menjalankan DD-WRT. Server DHCP berjalan di router Linksys, bukan modem.

Saya dapat mengakses GUI web DD-WRT melalui alamat IP router tetapi tidak dapat mengakses GUI web modem tanpa terhubung langsung ke sana.

Konfigurasi apa yang diperlukan untuk dapat mengakses GUI web modem ketika terhubung ke router.

Berikut pengaturannya:

  • IP modem: 10.0.0.1
  • IP WAN Router: 10.0.0.2
  • IP Internal Router: 10.0.0.3 (Gateway IP yang digunakan oleh komputer yang terhubung ke router):
  • Baik router dan modem sudah menyala subnet yang sama (255.255.255.0).
  • Kabel ethernet dari modem terhubung ke port WAN di router.
  • Awalnya saya percaya hanya router menjalankan server DHCP. Sejak modem baru-baru ini disetel ulang, saya yakin itu mungkin menjalankan server DHCP sekarang demikian juga. Sulit untuk mengatakannya sejak aku tidak dapat mengakses GUI dari jarak jauh. :)

Seperti yang dikatakan heavyd, alamat untuk modem harus berada di subnet yang berbeda dari yang dikontrol oleh router Anda.

Hanya untuk memperjelas, file Subnetmask menunjukkan bagian mana dari alamat IP yang merupakan subnet . Subnet mask 255.255.255.0 berarti tiga angka pertama di alamat IP adalah subnet.

Jadi dalam pengaturan Anda, baik router dan modem memiliki subnet 10.0.0.x. Saat Anda memasukkan alamat 10.0.0.1, router melihat bahwa alamat tersebut ada di subnet yang dikontrolnya dan tidak pernah mengirim paket untuk alamat tersebut ke port WAN-nya.

Jadi, seperti yang dikatakan heavyd, solusinya adalah mengubah alamat IP router ke apa pun alamat IP pribadi yang tidak dimulai dengan 10.0.0.

Misalnya, Anda dapat mengubah alamat IP router Anda menjadi 10.3.3.3 dan membiarkan subnet mask di 255.255.255.0. Dalam hal ini subnet router adalah 10.3.3.x. Kemudian ketika Anda memasukkan alamat 10.0.0.1, router akan melihat bahwa alamat tersebut tidak ada di subnet yang dikontrolnya dan akan mengirim paket untuk alamat itu ke port WAN-nya.

SEBUAH router digunakan untuk menghubungkan 2 subnet yang berbeda bersama-sama. Untuk mendapatkan modem Anda yang dihubungkan ke komputer Anda oleh sebuah router, modem tersebut harus berada di subnet yang berbeda dari komputer Anda.

Pada dasarnya, router Anda memetakan setiap subnet ke antarmuka di router dan mengirimkan lalu lintas apa pun yang diterimanya yang ditujukan untuk subnet itu keluar dari antarmuka itu. Karena modem Anda berada di subnet yang sama, semua permintaan yang ditujukan untuk modem sebenarnya dirutekan ke LAN, bukan ke antarmuka WAN. Ubah IP modem Anda menjadi sesuatu seperti 192.168.10.1 (atau apa pun selain 10.0.0.x) dan seharusnya berfungsi dengan baik.
