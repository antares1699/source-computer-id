Adakah cara untuk menghapus / mengganti nama Chrome “Bookmark Lain”?

Folder Chrome "Bookmark Lain" di Bilah Bookmark memiliki nama yang jelek dan menghabiskan terlalu banyak ruang layar. Apakah ada cara untuk melakukannya Singkirkan ?

4

  • Alasan lebih lanjut ini penting, selain keburukan: Saat jendela sangat tipis (yaitu saat lebar layar 50%) semua bookmark lainnya terpotong, jadi Anda hanya melihat yang paling kiri, tetapi karena alasan tertentu "Bookmark Lainnya "ditampilkan secara penuh! Ini mungkin berarti beberapa tombol penting disembunyikan hanya untuk yang benar-benar tidak berguna! Menghapus isinya 100% bermanfaat jika Anda menggunakan beberapa jendela dengan lebar setengah!

Jika istilah " Bookmark lainnya "membuat Anda merinding, dan tidak keberatan melakukan retasan kecil, Anda bisa mengubah teks di database lokal.

Kami ingin mengubah file locales/en-US.pak (tentu saja, akan berbeda jika Anda tidak menggunakan bahasa Inggris AS).

Sebelumnya, lakukan a cadangan dari file .pak yang ingin Anda ubah.
Kemudian tutup Chrome (dan pastikan tidak chrome proses tetap dalam memori).

Menggunakan Ubuntu, ada di direktori /...install-dir.../google/chrome/locales/.

Ubah file menggunakan biner editor (Linux: digunakan okteta.dll ), editor yang hanya mengubah byte di lokasi tertentu (tidak ada konsep baris, ...; ukuran file tidak berubah).

Kemudian dari

Untuk

Berhati-hatilah untuk tidak mengubah apa pun di luar teks "Bookmark lain"

  • Tidak ada pembatas \0: panjang istilah mungkin telah disimpan di suatu tempat di file pak (saya tidak akan mengacaukannya)
  • Lengkapi dengan spasi jika istilah baru lebih pendek dari yang asli
  • Jelas istilah baru tidak bisa lebih lama dari yang asli ...

Simpan file yang dimodifikasi. Buka Chrome.

Kamu harus lihat

NB

Anda juga bisa menjalankan perintah seperti

 sed -i 's/Other bookmarks/Oops!          /' en-US.pak

itulah perubahannya.

Pada s/X/Y/, X dan Y harus memiliki panjang yang sama ...

Penafian

  • dengan resiko Anda sendiri. Itu a pak file di lokal, dan kami hanya menimpa beberapa teks tanpa mempengaruhi ukuran atau pengkodean, jadi itu tidak masalah. Tapi siapa yang tahu ...

  • itu pak file mungkin / mungkin akan diganti selama pembaruan Chrome berikutnya ... </sup>
