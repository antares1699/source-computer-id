Menjalankan Hyper-V dan VMWare bersama-sama

Saya memiliki dua perangkat lunak virtualisasi di salah satu laptop saya yang didedikasikan untuk pengembangan Windows 8.
VMWare untuk virtualisasi umum dan Windows 8 seperti Hyper-V untuk WP8 SDK.

Sayangnya perangkat lunak ini tidak saling menyukai.
Apakah ada cara untuk membuat mereka ikut bermain?

Masalah sebenarnya adalah jika saya menginstal SDK, VMWare bahkan tidak akan memperbarui atau menjalankan.
Hanya ada satu cara untuk membuatnya berhasil. Aktifkan dan nonaktifkan Hyper-V. Yang mana satu restart sepanjang waktu.

Dari Catatan Rilis VMWare Workstation 9 :

Virtualisasi Bersarang

Workstation 9 meningkatkan implementasi virtual Intel VT-x / EPT atau Ekstensi AMD-V / RVI. Ini memungkinkan pengguna untuk menjalankan ESX sebagai OS tamu dan menjalankan sistem operasi 64-bit yang bersarang di ESX menggunakan lebih sedikit sistem sumber daya.

Catatan: Jika Anda mengaktifkan ekstensi virtualisasi secara virtual mesin yang berjalan di Workstation 8, Anda mungkin perlu menonaktifkan file ekstensi, tingkatkan mesin virtual ke perangkat keras virtual terbaru versi (kompatibel dengan Workstation 9), lalu aktifkan kembali ekstensi.

Hyper-V

Hyper-V telah ditambahkan ke sistem operasi guest Workstation 9 daftar. Ini memungkinkan pelanggan untuk menjalankan Windows 8 dengan Hyper-V diaktifkan, atau instal Server Hyper-V. Ini dapat digunakan untuk tujuan pendidikan atau untuk membangun prototipe Awan Hibrida. Fitur ini TIDAK DIDUKUNG dan mungkin tidak akan pernah. Microsoft tidak mendukung pengumpulan hypervisor mereka yang membuatnya sangat sulit - jika tidak VMware tidak mungkin memperbaiki masalah yang mungkin terjadi dalam hal ini konfigurasi. Untuk itulah kapabilitas ini telah diimplementasikan semata-mata untuk melihat apakah kami bisa melakukannya!

VMWare dan Hyper-V keduanya merupakan implementasi dari konstruksi perangkat lunak yang dikenal sebagai a hypervisor . Perangkat keras komputer modern memiliki fitur pengoptimalan khusus untuk hypervisor untuk mendukung peningkatan kinerja. Ini adalah sirkuit khusus yang dibangun ke dalam cpu dan motherboard komputer Anda.

Perangkat lunak hypervisor modern, seperti VMWare dan Hyper-V, bergantung dan mengandalkan perangkat keras khusus ini. Jika satu sistem menggunakan perangkat keras itu, yang lain tidak bisa.

Singkatnya, tidak ada cara untuk menggunakan VMWare dan Hyper-V secara bersamaan.
