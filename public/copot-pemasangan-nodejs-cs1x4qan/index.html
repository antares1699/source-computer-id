Copot pemasangan Node.js

Beberapa waktu yang lalu, saya menginstal Node.js versi 0.2.1 menggunakan perintah ini di Mac OSX:

./configure
make
sudo make install

Saya baru saja menginstal Homebrew, jadi sekarang preferensi saya adalah menggunakannya untuk mengelola penginstalan saya. Saya menginstal Node.js versi 0.4.5 hari ini dengan perintah berikut:

simpul buatan

Tetapi saya perhatikan bahwa saya masih memiliki versi lama Node.js yang tergeletak di direktori ini:

/usr/local/include/node/
/usr/local/lib/node/

Apa cara yang tepat untuk menghapus Node.js yang diinstal menggunakan teknik sudo make?

Terima kasih sebelumnya

Tidak ada cara yang "tepat". make install hanya menempatkan file secara langsung di tempatnya dan tidak ada manifes untuk melacak apa yang diinstal pada saat itu. Anda hanya perlu mencari file yang relevan dan menghapusnya.

Satu hal yang dapat Anda lakukan adalah menemukan file yang dibuat dalam beberapa menit dari biner yang Anda tahu adalah bagian dari paket, karena itu akan memberi Anda titik awal untuk Anda filter.
