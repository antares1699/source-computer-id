Mengapa direktori menghilang ketika saya melakukan SSHFS? Bagaimana cara mengatur share SSHFS di Max OSX 10.9?

Saya menjalankan Max OSX 10.9.3 dan saya mencoba menyiapkan berbagi file SSHFS antara MacBook Pro saya dan sistem file jarak jauh. Namun, ketika saya mencoba melakukannya, tidak berhasil.

Anehnya, itu membuat direktori target menghilang. Adakah orang lain yang melihat ini terjadi? Apakah ini bug?

Pertama, lihat bahwa saya dapat melakukan ssh secara normal ke mesin target:

% ssh remoteuser@XXX.XXX.XXX.XXX            # <--- SSH to remote system works! See below.
remoteuser@XXX.XXX.XXX.XXX % ls -altr remoteDir
total 8
drwxr-xr-x 26 remoteuser remoteuser 4096 Jun 22 01:00 ..
drwxrwxrwx  2 remoteuser remoteuser 4096 Jun 22 01:08 .
remoteuser@XXX.XXX.XXX.XXX % exit
%                                           # <--- Logged out of remote system

Selanjutnya, saya membuat direktori secara lokal dan memverifikasi itu dibuat:

% pwd
/mnt

% ls
total 0
drwxr-xr-x  31 root  admin  1122 Jun 18 18:34 ../
drwxr-xr-x   2 root  admin    68 Jun 23 08:11 ./

% sudo mkdir share1 
% ls
drwxr-xr-x  31 root  admin  1122 Jun 18 18:34 ../
drwxr-xr-x   4 root  admin   136 Jun 23 08:50 ./
drwxr-xr-x   2 root  admin    68 Jun 23 08:50 share/

Sekarang saya mencoba mengatur share SSHFS:

% sudo sshfs remoteuser@XXX.XXX.XXX.XXX:remoteDir /mnt/share1
remoteuser@XXX.XXX.XXX.XXX's password:
%

Baik. Ini terlihat telah berfungsi. Tidak ada kesalahan. Jadi mari kita lihat bagian yang kita buat, oke?

% ls
ls: share1: No such file or directory
total 0
drwxr-xr-x  31 root  admin  1122 Jun 18 18:34 ../
drwxr-xr-x   3 root  admin   102 Jun 23 08:12 ./

Apa? Bukan hanya File Sharing tidak berfungsi, tetapi direktori share1 tampaknya telah lenyap! (Meskipun sistem file sepertinya tahu itu hilang, yang aneh).

Kemana perginya / mnt / share1 dan bagaimana cara menyiapkan SSHFS ini?

Lihat opsi konfigurasi https://code.google.com/p/macfuse/wiki/OPTIONS

Saya menambahkan 'allow_other' dan itu berhasil untuk saya. Ini string penghubung saya:

sshfs -p 22 user@example.net:/home/user/ ~/RemoteHome -oauto_cache,reconnect,defer_permissions,negative_vncache,allow_other,volname=RemoteHome

Kecuali jika Anda secara eksplisit ingin berbagi folder itu dengan pengguna lain dari sistem Anda, Anda juga dapat memanggilnya tanpa sudo; dengan cara itu Anda juga dapat menghilangkan opsi allow_other - pengguna yang memanggil perintah (Anda) akan memiliki akses eksklusif (selain root tentunya).
