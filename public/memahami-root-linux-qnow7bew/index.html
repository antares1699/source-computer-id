Memahami Root Linux

Saya telah menggunakan Linux (Ubuntu) selama sekitar 2 minggu sekarang dan saya masih berjuang dengan beberapa konsep dasar seputar file akar pengguna:

  1. Beberapa operasi terminal (seperti membuat subdirektori di dalam direktori FHS seperti /memilih ) mengharuskan saya untuk memberi awalan perintah dengan sudo - kenapa? Saya rasa apa yang membuat saya tersedak adalah: jika saya sudah masuk sebagai pengguna sistem yang valid, mengapa saya harus menjadi superuser / root untuk memodifikasi hal-hal yang menurut sysadmin sudah saya anggap layak untuk saya akses?

  2. Apakah ada GUI (Gnome, KDE) yang setara dengan sudo? Adakah cara untuk mengambil peran superuser melalui konteks grafis, bukan dari dalam shell baru?

  3. Saya tidak dapat mengakses direktori / root yang login sebagai diri saya sendiri ... tetapi saya menginstal sistem untuk memulai dan tidak pernah diminta untuk membuat akun root! Bagaimana cara saya masuk sebagai root dan mendapatkan akses ke / root?

jika saya sudah masuk sebagai valid pengguna sistem, mengapa saya harus menjadi superuser / root untuk memodifikasi hal-hal yang sudah dimiliki sysadmin menganggap saya layak untuk mengaksesnya?

Siapa bilang mereka punya?

Jika Anda melihat /etc/passwd, Anda akan melihat bahwa ada lebih banyak pengguna di sistem Anda daripada yang Anda pikirkan. Misalnya, milik saya terlihat seperti ini:

┌─[pearson@Bragi] - [~] - [Mon Jan 03, 11:29]
└─[$]> cat /etc/passwd
root:x:0:0:root:/root:/bin/bash
bin:x:1:1:bin:/bin:/bin/false
daemon:x:2:2:daemon:/sbin:/bin/false
mail:x:8:12:mail:/var/spool/mail:/bin/false
ftp:x:14:11:ftp:/srv/ftp:/bin/false
http:x:33:33:http:/srv/http:/bin/false
nobody:x:99:99:nobody:/:/bin/false
dbus:x:81:81:System message bus:/:/bin/false
pearson:x:1000:1000::/home/pearson:/bin/zsh
hal:x:82:82:HAL daemon:/:/bin/false
ntp:x:87:87:Network Time Protocol:/var/empty:/bin/false
avahi:x:84:84:Avahi daemon:/:/bin/false
temp:x:1001:1001::/home/temp:/bin/bash
mysql:x:89:89::/var/lib/mysql:/bin/false
mongodb:x:101:2::/var/lib/mongodb:/bin/false
usbmux:x:140:140:usbmux user:/:/sbin/nologin
kdm:x:135:135::/var/lib/kdm:/bin/false
deluge:x:125:125:Deluge user:/srv/deluge:/bin/false

Sebagian besar digunakan oleh berbagai daemon (program yang berjalan tanpa interaksi pengguna); mereka cenderung memiliki izin yang sangat terbatas, karena mereka tidak perlu berbuat banyak. Jika mereka mencoba melakukan sesuatu yang buruk, baik secara tidak sengaja karena bug perangkat lunak atau sengaja karena eksploitasi keamanan, mereka tidak akan bisa jauh.

Poin yang lebih besar adalah bahwa pengguna seharusnya hanya memiliki akses ke apa yang mereka butuhkan.

Sekarang, jika pertanyaan Anda adalah, "Mengapa saya perlu mengetik sudo padahal saya sudah ditambahkan ke sudoers?", Jawabannya adalah sudo menjalankan sesuatu sebagai root, bukan sebagai Anda. Jika kami membuat semua file dapat diakses oleh pengguna Anda secara langsung, atau Anda hanya menjalankan sebagai root secara normal, itu banyak lebih mudah untuk secara tidak sengaja melakukan Hal-Hal Buruk (rm -rf /* terlintas dalam pikiran). Selain itu, praktik keamanan yang sangat buruk untuk mengizinkan aplikasi apa pun yang Anda jalankan melakukan apa pun yang diinginkannya ke sistem Anda - begitulah banyaknya spyware yang diinstal pada mesin Windows sebelum UAC.

Apakah ada GUI (Gnome, KDE) yang setara ke sudo?

gksu, gksudo, kdesu, kdesudo. Merupakan ide yang sangat baik untuk mempraktikkan penggunaan ini untuk aplikasi grafis, karena mereka melakukan penyelesaian khusus untuk mencegah masalah seperti ini .

Bagaimana cara masuk sebagai root

Jangan. Jika Anda membutuhkan shell root, Anda dapat menggunakan sudo -s, sudo -i, atau sudo su.

Ubuntu dikirimkan dengan akun root terkunci, jadi Anda harus mengubah kata sandi untuk masuk (sudo passwd root). Setelah Anda selesai melakukannya, Anda dapat mengunci (sudo passwd -l root) dan membuka (sudo passwd -u root) akun root sesuka Anda. Tapi sungguh, tetap terkunci; Anda akan mencegah serangkaian serangan seperti itu.

└─[$]> cd /root

  1. sudo memungkinkan Anda menjalankan program sebagai root. Pengguna sistem biasa memiliki hak istimewa terbatas untuk alasan keamanan. Anda tidak ingin pengguna akun tamu Anda dapat mengubah file sistem, bukan? Jika Anda ingin memberikan hak istimewa yang lebih tinggi kepada pengguna Anda, Anda harus melihat ke dalam memodifikasi file "sudoers".

  2. gksu (secara teknis GUI untuk su, tetapi akan membantu Anda)

  3. Tidak seperti kebanyakan distribusi Linux lainnya, Ubuntu tidak mengizinkan login root secara default. Semua akses root seharusnya dilakukan melalui sudo. Lihat sini untuk mengatasinya (Penafian: Belum mencoba ini, jangan gunakan Ubuntu sendiri). Jika tidak, jika Anda hanya ingin mengakses direktori / root Anda, Anda dapat melakukannya dari shell (atau memulai pengelola file Anda sebagai root dengan sudo - menurut saya secara default adalah Nautilus di Ubuntu).

Saat melakukan sudo Anda untuk sementara waktu mengambil hak akses root. Umumnya, Anda ingin memeriksa izin file terlebih dahulu. Di Ubuntu, /opt misalnya milik root dan memiliki izin rwxr-xr-x. Artinya, pengguna Anda tidak boleh menulis (yaitu membuat direktori) ke direktori.

Akun root hadir, hanya karena "diperlukan" untuk melakukan tugas administratif.
