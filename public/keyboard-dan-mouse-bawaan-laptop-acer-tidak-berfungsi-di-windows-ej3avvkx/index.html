Keyboard dan mouse bawaan laptop Acer tidak berfungsi di Windows

Saya telah diminta untuk membantu dengan laptop Acer. Ini telah bekerja dengan baik dengan konfigurasi dual-boot (Arch Linux / Windows XP) untuk beberapa waktu. Kemudian pengguna memutuskan untuk mengganti XP dengan Windows 7, yang masih berfungsi dengan baik. Beberapa minggu kemudian ketika booting windows 7, mouse dan keyboard PS / 2 laptop tidak merespon.

Solusi tampaknya adalah memformat partisi Windows lagi dan menginstal ulang XP, tetapi mouse dan keyboard juga tidak merespons di XP. Kami mencoba Windows 2000, tetapi tidak berhasil.

Satu-satunya cara untuk mendapatkan input agar berfungsi di Windows adalah dengan meluncurkannya dalam mode aman atau dengan keyboard dan mouse eksternal terpasang sebelum boot (Windows tidak akan mengenali mereka saat runtime dan solusi yang disebutkan bekerja 1 dari 3 kali paling baik).

Dalam mode aman, Windows tidak memberikan informasi debug atau pesan kesalahan yang terkait dengan masalah tersebut. Saya harus menyebutkan bahwa keyboard berfungsi dengan baik di BIOS dan keyboard dan mouse (bawaan dan eksternal) berfungsi dengan baik sepanjang waktu dari Arch Linux OS. Saran apa pun akan dihargai.

Edit 1

  1. Konfigurasi BIOS dalam ini laptop sangat terbatas, saya ragu itu itu yang menyebabkan masalah.

  2. Mengapa stok Windows XP (versi yang sama yang bekerja sebelumnya) instal driver yang berbeda dengan tepat perangkat keras yang sama? (ya, disana dimana tidak ada perubahan perangkat keras sama sekali)

  3. Saya menyelesaikan penginstalan dengan menghubungkan mouse dan keyboard eksternal (USB) sebelum mem-boot sistem: itu satu-satunya cara untuk melakukannya.

    Saya mulai berpikir bahwa Windows entah bagaimana mempertahankan beberapa konfigurasi dari penginstalan sebelumnya di partisi NTFS lain yang tidak saya instal. Ini akan menjelaskan mengapa Windows 7 dapat mengacaukan instalasi XP dan 2000 saya yang bersih: sayangnya memformat partisi tersebut bukanlah pilihan pada saat ini.

Edit 2

Seperti yang saya sebutkan di postingan asli:

"Satu-satunya cara agar input berfungsi di Windows adalah meluncurkannya dalam mode aman ..."

Artinya, ini berfungsi dengan baik dalam mode aman! Dan itu bekerja di Linux dan BIOS juga ... Jadi tolong berhenti memposting "ini jelas merupakan kegagalan perangkat keras" jenis jawaban.

Saya pribadi belum pernah mendengar tentang ini.

Jika Anda yakin itu berfungsi di instalasi Linux, itu berarti tidak ada koneksi yang salah dan ini adalah masalah berbasis perangkat lunak.

Yang terbaik yang dapat saya pikirkan adalah bahwa daripada PS / 2 biasa, keyboard / mouse dikenali sebagai perangkat berbasis USB dan beberapa bagaimana driver / root hub USB menjadi rusak atau versi yang salah diinstal.

Jika keyboard eksternal adalah USB, saya sedikit bingung, tetapi jika itu adalah PS / 2, ini mungkin sesuai dengan teori saya.

Yang terbaik yang dapat saya pikirkan adalah menggunakan pemulihan sistem atau mencoba menghapus pembaruan secara manual satu per satu.

Terakhir, jika semuanya gagal, instal ulang Windows dan terapkan pembaruan satu per satu hingga Anda mencapai kegagalan dan Anda harus mengidentifikasi di mana letak masalahnya.

Secara pribadi saya akan mulai dengan membuka pengelola perangkat dan mencopot / menghapus (atau memutar kembali) driver untuk semua hub dan konektor akar USB kemudian melakukan pemindaian pasang dan pakai yang akan mengatur ulang / menginstal ulang. Ini mungkin tidak menyelesaikan masalah tetapi harus menjadi awal yang baik.
