Gagal memulai iptables.service: 'Tidak ada file seperti itu'

Saya mencoba menggunakan iptables secara langsung daripada firewalld untuk mengelola aturan firewall. Namun systemctl start iptables mencetak:

[root@centos-new ~]# systemctl start iptables
Failed to issue method call: Unit iptables.service failed to load: No such file or directory.

Saya menjalankan CentOS 7 yang baru diinstal sehingga hampir tidak mungkin untuk membiarkan sesuatu yang penting dihapus oleh saya. Apa kemungkinan masalahnya?

firewalld diinstal secara default pada sistem CentOS7. Anda perlu menonaktifkannya dan menginstal iptables sebagai gantinya. Langkah-langkah ini harus dilakukan:

systemctl mask firewalld
systemctl stop firewalld
yum install iptables-services
systemctl enable iptables
systemctl start iptables

Secara opsional, jika Anda ingin menangani IPv6 juga:

systemctl enable ip6tables
systemctl start ip6tables
