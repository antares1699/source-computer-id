Lan ke Wan mengatur cara menghubungkan dua subnet

Saya berharap seseorang dapat membantu saya dengan masalah yang saya alami dengan jaringan rumah saya. Saya memiliki modem / router Miliar 7700N dan saat ini saya menambahkan router dual band Western Digital N600. Secara default (mengikuti konfigurasi DVD yang disediakan oleh WD), Western Digital mengatur sendiri dalam konfigurasi LAN ke WAN dan menetapkan sendiri alamat IP dengan subnet yang berbeda. Yang ingin saya simpan jika semuanya bekerja dengan baik.

Jadi ini pengaturan saya:

Miliar 7700N:

  • AKU P: 192.168.1.1
  • Pintu gerbang: 192.168.1.1
  • DCHP: di Access point: off
  • Tersambung ke ISP

Western Digital N600:

  • IP Klien (ditetapkan oleh Miliar 7700N DHCP): 192.168.1.115
  • AKU P: 192.168.0.1
  • Pintu gerbang: 192.168.0.1
  • DHCP: di
  • Jalur akses: di
  • Saya menghubungkan port WAN Western Digital ke port LAN Billion 7700N.

Internet berfungsi dengan baik untuk semua perangkat baik yang terhubung ke Billion 7700N melalui ethernet atau ke Western Digital N600 melalui Ethernet / Wi-Fi. Ketika saya terhubung melalui Western Digital N600, saya dapat melihat subnet lainnya (ping dan mengakses drive bersama) tetapi tidak sebaliknya. Ketika saya terhubung ke Billion 7700N, saya bahkan tidak dapat melakukan ping ke mesin mana pun yang terhubung ke Western Digital N600. Saya bahkan tidak dapat melakukan ping 192.168.1.115 ke alamat IP yang ditetapkan ke Western Digital N600.

Saya membaca beberapa posting di Net yang secara default apa yang akan saya dapatkan dengan pengaturan ini. Tapi saya yakin pasti ada jauhnya sehingga saya bisa menghubungkan kedua subnet ini dua arah? Saya telah menambahkan rute statis di Miliar untuk 192.168.0.0/24 dengan gerbang 192.168.0.1 tetapi itu tidak membantu.

Terima kasih Paul atas tanggapan Anda. Ada alasan kuat untuk memiliki 2 subnet. Jika ada sesuatu di sini yang menjadi alasan saya ingin tetap seperti itu:

  • Saya dapat menggunakan semua port LAN pada N600.
  • Tenang rasanya mempelajari cara membuat penyiapan ini berfungsi.

Saya telah menambahkan rute statis ke 7700N tetapi saya rasa saya mengatur gateway ke 192.169.0.1. Saya akan memeriksa dan memastikan gateway adalah 192.168.1.115 dan melihat apakah saya dapat membuatnya berfungsi.

Bisakah Anda menjelaskan relevansi menonaktifkan NAT pada N600? Terima kasih lagi.

Kecuali ada manfaat yang belum Anda jelaskan karena memiliki dua subnet, saya sarankan Anda hanya memiliki satu.

Langkah-langkah untuk melakukan ini adalah

  1. Hubungkan port LAN N600 ke port LAN 7700N.
  2. Nonaktifkan DHCP pada N600.
  3. Aktifkan DHCP di 7700N.
  4. Berikan IP secara manual pada N600 dalam kisaran 192.168.1.x (di luar kisaran DHCP).

Ini kemudian akan menempatkan semua perangkat dalam kisaran 192.168.1.x, dan menjadi jauh lebih sederhana.

Jika Anda menginginkan kedua subnet, Anda harus melakukannya

  1. Hubungkan port N600 WAN— tidak port LAN — ke port LAN 7700N.
  2. Aktifkan DHCP di 7700N (karena N600 terhubung melalui port WAN, server DHCP ini tidak akan terlihat oleh perangkat di N600, jadi Anda tidak akan mengalami konflik).
  3. Tambahkan rute statis ke 7700N yang menyatakan bahwa jaringan 192.168.0.x dapat diakses melalui gateway di 192.168.1.115.
  4. Nonaktifkan NAT pada N600.

Jika NAT diaktifkan pada N600 maka perangkat pada 192.168.0.x akan tampak berasal dari 192.168.1.115 ke perangkat pada jaringan 192.168.1.x. Alamat mereka akan diberi NAT ke alamat router. Ini akan mencegah Anda mengakses apa pun di 192.168.0.x dari 192.168.1.x.

Anda melakukan NAT ganda. Jangan lakukan NAT ganda. Anda sebenarnya tidak menginginkan dua subnet terpisah di setiap sisi WD N600, terutama tidak dengan gateway NAT di antaranya, karena efek samping alami NAT (secara teknis disebut "NAPT" dalam kasus ini) adalah tindakannya seperti bit firewall Stateful Packet Inspection (SPI), dan tidak mengizinkan aliran / koneksi dimulai dari sisi "upstream" / "publik" dari NAT ke sisi "downstream" / "private" dari NAT.

Nonaktifkan NAT dan DHCP pada perangkat hilir (WD N600 Anda). Jadikan itu hanya sebagai jembatan, memungkinkan klien untuk mendapatkan sewa DHCP mereka dari Miliar 7700N.

Jika WD N600 Anda tidak menyediakan cara untuk menonaktifkan NAT, nonaktifkan DHCP dan hentikan penggunaan port WAN-nya. Cukup sambungkan port LAN-nya ke port LAN Billion 7700N. Jika tidak memungkinkan Anda untuk menonaktifkan DHCP, setel panjang kumpulan sewa alamat IP DHCP ke nol (atau satu, jika tidak mengizinkan kumpulan panjang-nol).
