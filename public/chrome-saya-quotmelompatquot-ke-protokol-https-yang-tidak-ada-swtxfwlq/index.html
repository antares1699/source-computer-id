Chrome saya "melompat" ke protokol HTTPS yang tidak ada

Chrome saya (40.0.2214.115m) melompat ke https yang tidak ada:

Saya mengetik

http://johnwhitech.homeip.net:8080/

Dan Chrome dialihkan ke

http s: //johnwhitech.homeip.net: 8080 /

yang tidak ada. Saya lakukan tidak memiliki HTTPS di mana-mana.

Dalam penganalisis jaringan, saya melihat bahwa server saya menjawab pengalihan 307 yaitu tidak benar (apache dikonfigurasi dengan benar dan tidak memberikan jawaban ini, browser lain tidak mengalihkan, paket HTTP berputar normal).

Saya mencoba memasukkan permintaan dengan fiddler, ini dia:

HUBUNGKAN johnwhitech.homeip.net:8080 HTTP / 1.1

Host: johnwhitech.homeip.net:8080

Koneksi: tetap hidup

Agen Pengguna: Mozilla / 5.0 (Windows NT 6.3; WOW64) AppleWebKit / 537.36 (KHTML, seperti Gecko)

Chrome / 40.0.2214.115 Safari / 537.36

Ditemukan jabat tangan ClientHello yang kompatibel dengan SSLv3. Fiddler mengekstrak parameter di bawah ini.

...

Hal aneh lainnya: di kolom HOST dari fiddler, jika saya coba dengan firefox, saya melihat nama server saya. Dengan chrome, saya hanya melihat "Terowongan ke ..." bahkan untuk permintaan pertama. Protokolnya memang HTTP (lihat di awal paket), tetapi mengapa pada hearth saya chromes mengirimkan SSL "ClientHello" bersamanya?

Apakah ini perilaku normal?

Satu-satunya hal yang saya lihat yang mungkin terkait dengan SSL adalah bahwa saya memasang sertifikat yang ditandatangani sendiri, pertama melalui setelan chrome, lalu di seluruh sistem. Sertifikat ini berfungsi dengan baik, tetapi mungkin karena nama domain cocok dengan sertifikat di daftar, chromes mencoba SSL?

Terima kasih semuanya!

EDIT3: Perilaku ini juga terjadi tanpa konektivitas internet (cache dihapus), jadi jelas bukan di sisi server

EDIT4: Apa yang saya lihat di tab "JARINGAN" Alat Pengembang Chrome. Entri pertama normal, entri kedua melalui HTTP yang langsung dialihkan ke HTTPS. Ini terjadi tanpa koneksi jaringan, khususnya server tidak mengirim HSTS.

Chrome dengan jelas menyatakan alasannya: HSTS. HSTS, setelah diterima, tongkat. Ini adalah bagian penting dari konsep. Artinya, jika Chrome menerima tajuk HSTS dari server Anda, meskipun hanya sekali, itu akan tetap ada hingga kedaluwarsa.

Untuk memeriksa cache HSTS Chrome, buka chrome://net-internals/#hsts. Dari sana, Anda dapat meminta cache. Chrome memiliki header HSTS cache dari halaman Anda, kecuali jika dilaporkan "Not Found". Untuk menghapusnya, gunakan alat yang disediakan di halaman yang sama.
