Haruskah root memiliki file di direktori home (linux) saya?

Pertanyaan ini dimulai dengan menanyakan mengapa file riwayat saya tidak berfungsi dengan benar. Kemudian saya perhatikan itu -rw------- 1 root root dan belum diperbarui sejak 2012-09-11. Saya mengubah kepemilikan, masalah diperbaiki.

Tetapi sekarang saya melihat beberapa file lain yang dimiliki oleh root:

.gitconfig
.pearrc
.viminfo

Bisakah saya mengubahnya dengan aman menjadi milik pengguna biasa saya, bukan root? Saya menggaruk-garuk kepala mencoba mencari tahu apakah ada sisi negatifnya, atau konsekuensi keamanan.

Kehilangan riwayat tujuh minggu sebenarnya cukup menyakitkan, karena saya banyak bersandar padanya (misalnya untuk mengingatkan bagaimana terakhir kali saya membuat arsip). Apakah masuk akal untuk menyiapkan tugas cron untuk mengirimi saya email jika menemukan file apa pun di direktori beranda saya yang dimiliki oleh orang lain selain saya? Diulang: apakah ada alasan yang baik bagi root untuk memiliki file di direktori home saya?

MEMPERBARUI: Terima kasih atas tanggapannya sejauh ini. Saya melihat bagaimana file yang dimiliki root dapat ditempatkan di sana karena melakukan sesuatu dalam sesi sudo. Inti dari pertanyaan saya adalah: Dapatkah saya selalu memperlakukannya sebagai kesalahan yang perlu diperbaiki? Atau adakah alasan yang sah untuk meninggalkan file milik root di direktori home saya begitu saya menyadarinya? Terima kasih!

Root biasanya tidak perlu memiliki file di direktori home Anda. Itu tentu saja generalisasi. Seperti yang telah disebutkan dalam jawaban dan komentar lain, file dapat dimiliki oleh root jika dibuat atau dimodifikasi oleh program yang berjalan di bawah sudo.

Untuk membuat daftar file mana yang dimiliki oleh root, atau milik grup root, coba jalankan daftar semua file dengan informasi pemiliknya di terminal dan ambil baris yang berisi "root": ls -al | grep root.

Di komputer saya, di bawah akun pengguna biasa, saya punya beberapa file dan bahkan direktori (.compiz-1 /) yang muncul sebagai milik root, jadi ini cukup umum. Melihat dan di dalam file-file itu - Anda mungkin perlu melakukan ini sebagai root - saya dapat menebak dengan baik bagaimana mereka sampai di sana. Beberapa di antaranya adalah file log dari program yang saya jalankan menggunakan sudo program. Ingat, menggunakan sudo menjalankan program sebagai root tetapi tetap $HOME menunjuk ke direktori home Anda. Menggunakan gksu atau kdesu untuk program GUI melakukan hal yang sama, tetapi juga memastikan lingkungan grafis menggunakan pengaturan Anda (yang merupakan bagian dari mengapa Anda tidak boleh menggunakan sudo untuk mereka).

Direktori .compiz-l / tampaknya merupakan cadangan dari .compiz /, yang dimiliki oleh pengguna saya. Dugaan saya adalah bahwa pada titik tertentu saya menjalankan peningkatan atau melakukan sesuatu yang lain untuk memicu pencadangan pengaturan Compiz sebagai root dan sekarang saya dapat menghapus file-file itu. Jika ragu, cukup pindahkan atau ganti nama file / direktori dan lihat apa yang terjadi setelah logout / login atau menjalankan program yang menurut Anda menggunakannya.

Mari kita lihat file yang Anda daftarkan dan bandingkan dengan yang sama di komputer saya.

-rw-r--r-- 1 henrik henrik 211 apr  9  2011 .gitconfig

File ini kemungkinan besar harus dimiliki oleh pengguna, tanpa akses tulis ke orang lain, karena ini jelas merupakan file konfigurasi per pengguna. Jika root memiliki file ini dan Anda tidak memiliki akses tulis, Anda harus menjalankan git sebagai root agar dapat memperbarui konfigurasi git global Anda, yang sangat tidak nyaman. Saya tidak keberatan orang lain memiliki akses membaca ke file ini karena tidak mengandung rahasia apa pun, hanya nama saya, email, dan beberapa preferensi.

-rw-r--r-- 1 root root 143 dec 17  2011 .pearrc

File ini juga merupakan file konfigurasi, tetapi bukan sesuatu yang biasanya saya gunakan langsung dari akun pengguna saya karena itu milik proyek PHP PEAR. Karena saya harus menginstal ekstensi PHP menggunakan sudo, file ini mungkin juga dimiliki oleh root jadi saya tidak sengaja mengacaukannya.

-rw-rw-r-- 1 henrik henrik 30709 nov  7 00:29 .viminfo

Ini adalah file riwayat Vim, berisi log dari hampir semua yang telah Anda lakukan di Vim sehingga dapat memulihkan status dengan mudah. Jika tidak dimiliki atau setidaknya dapat ditulis oleh Anda, saya ragu Vim akan dapat memperbaruinya.

Secara umum, ya, file yang dimiliki root di direktori home Anda adalah kesalahan, jika file tersebut digunakan oleh program yang biasanya Anda jalankan tanpa root. Bahkan mungkin menjadi sulit untuk melakukan backup penuh dari file konfigurasi di direktori home Anda jika file tidak dimiliki atau dibaca oleh Anda.
