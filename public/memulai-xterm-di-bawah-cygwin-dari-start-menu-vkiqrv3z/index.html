Memulai xterm di bawah Cygwin dari Start Menu

Saya menjalankan Cygwin pada Windows 7. Bagaimana cara membuat entri menu mulai (atau pintasan desktop) untuk memulai xterm?

Saat ini, saya harus memulai server Cygwin / X, dan setiap kali saya menginginkan xterm baru, saya harus mengklik kanan pada ikon server X di baki, dan membuka Aplikasi-> xterm, yang merepotkan. Saya mencoba menjalankan C: \ cygwin \ bin \ xterm.exe setelah mengatur DISPLAY =: 0.0. Ini menampilkan xterm, tetapi memberikan banyak kesalahan saat memulai, dan lingkungan saya (variabel env, dll.) Tampaknya sangat berbeda dari xterm yang dimulai dengan menggunakan ikon baki server X.

Anda juga perlu memulai shell login bash di xterm baru:

mkshortcut -D -n xterm -a "xterm -display :0 -e /bin/bash -l" /usr/bin/run

Ini berhasil untuk saya.

Anggap Anda selalu menggunakan DISPLAY =: 0, coba yang berikut ini:

mkshortcut -D -n xterm -a "xterm -display :0" /usr/bin/run

Ini akan membuat pintasan di Desktop Anda. Menghapus bendera -D akan menambahkan pintasan ke tingkat atas Start Menu sebagai gantinya. Lihat man mkshortcut untuk opsi lebih lanjut.

mkshortcut berfungsi, tetapi PATH hilang /usr/local/bin:/usr/bin: pada awalnya. Untuk mengatasinya, Anda dapat mengedit /etc/bashrc atau ~/.bashrc dan menambahkan yang berikut ini di akhir file:

PATH=/usr/local/bin:/usr/bin:$PATH

Jika Anda dapat menjelaskan mengapa bagian PATH ini hilang saat xterm dimulai melalui pintasan ini, itu mungkin membantu menemukan solusi.
