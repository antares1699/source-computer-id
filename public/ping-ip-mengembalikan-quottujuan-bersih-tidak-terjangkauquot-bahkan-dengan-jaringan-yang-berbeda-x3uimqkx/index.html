Ping IP mengembalikan "tujuan bersih tidak terjangkau", bahkan dengan jaringan yang berbeda

Saya tidak bisa masuk ke facebook, jadi saya melakukan ping ke domain dan mendapatkan ini:

ping www.facebook.com

Ping www.facebook.com [69.171.228.14] dengan 32 byte data:

Permintaan waktu habis.

Balas dari 204.15.23.57: Jaringan tujuan tidak dapat dijangkau.

Tetapi ketika saya melakukan ping ke IP ini, itu bisa dijangkau.

Ping 204.15.23.57 dengan 32 byte data:

Balas dari 204.15.23.57: byte = 32 kali = 190ms TTL = 51

Saya menyadari ada yang salah dengan komputer saya, karena saya telah mencoba menggunakan jaringan yang berbeda (pada jaringan yang sama perangkat lain dapat terhubung ke facebook), mengatur ulang tabel perutean router, menghapus cache arp komputer saya ... tetapi tidak ada yang berfungsi.

Apakah ada yang punya ide?

Apa yang diperlihatkan oleh perintah Anda adalah sebagai berikut:

  1. Komputer Anda telah menentukan alamat www.facebook.com menjadi 69.171.228.14
  2. Itu mencoba mengirim paket ping ke sana
  3. Dalam perjalanan paket, router (mungkin milik Anda, di alamat 204.15.23.57) tidak dapat menemukan rute ke 69.171.228.14
  4. Agar Anda tahu tentang ini, router 204.15.23.57 memberi tahu Anda tentang hal ini dengan pesan balasan ICMP Destination net unreachable.

Dalam perintah kedua Anda, Anda melakukan ping ke router yang memberi Anda balasan bersih yang tidak dapat dijangkau, bukan host facebook, oleh karena itu tidak mengherankan Anda mendapatkan balasan darinya.

Seperti yang telah Anda tunjukkan dengan membantu - ini adalah hasil dari file host yang kacau. Anda juga bisa melihat masalahnya jika Anda menggunakan nslookup untuk menanyakan server DNS secara langsung:

nslookup
server 8.8.4.4
www.facebook.com

Ini akan menunjukkan kepada Anda alamat sebenarnya dari host facebook dan Anda akan memperhatikan bahwa itu berbeda dari yang diberikan kepada Anda (69.171.228.14), jadi Anda akan tahu untuk melihat di file host saat komputer menyelesaikan alamat IP melalui file host atau permintaan DNS.

Karena itu tidak baik bahwa virus dapat memperbarui file host, karena itu seharusnya hanya dapat ditulis oleh superuser / sistem (Administrator, SYSTEM atau root di Linux). Oleh karena itu implikasinya adalah virus / trojan mempunyai akses tingkat tinggi ke sistem.

Anda dapat mencoba membersihkan DNS Anda. Buka CMD dan:

ipconfig /flushdns

Periksa server DNS IPV4 dan IPV6 (jika tersedia) Anda. Saya mengalami masalah yang sama di masa lalu dan saya membuat kesalahan dengan IPV6 saya. Tapi pada akhirnya, flush yang bagus berhasil!

Dalam kasus saya, saya harus menonaktifkan IPv6 untuk menghilangkan masalah ini. Untuk beberapa alasan, semua permintaan ping secara default ke IPv6.

Jika saya melakukan "ping google.com -4" Saya akan mendapat balasan. Itu memaksanya untuk menggunakan IPv4 Jika saya melakukan ping tanpa -4, itu tidak akan berhasil dan begitu pula yang lainnya (browser, email, dll).

Meskipun mungkin ada perbaikan yang lebih baik, untuk saat ini saya baru saja menonaktifkan IPv6 di properti NIC dan itu langsung menyelesaikan masalah.
