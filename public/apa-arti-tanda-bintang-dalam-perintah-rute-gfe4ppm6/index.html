Kepingan salju menunjukkan bahwa tidak ada rute default (beberapa router lain) ke jaringan yang terkait dengan antarmuka lokal. Artinya, paket IP apa pun yang menuju ke salah satu jaringan tersebut (subnet) dapat dijangkau melalui antarmuka yang ditunjukkan (eth0, wlan0, lo).

$ (route ;echo; route -n ) | grep -v '^[KD]'
10.3.0.0        *               255.255.255.224 U     0      0        0 eth0
192.168.0.0     *               255.255.255.0   U     2004   0        0 wlan0
link-local      *               255.255.0.0     U     2      0        0 eth0
loopback        localhost.local 255.0.0.0       UG    0      0        0 lo
default         192.168.0.1     0.0.0.0         UG    2004   0        0 wlan0

10.3.0.0        0.0.0.0         255.255.255.224 U     0      0        0 eth0
192.168.0.0     0.0.0.0         255.255.255.0   U     2004   0        0 wlan0
169.254.0.0     0.0.0.0         255.255.0.0     U     2      0        0 eth0
127.0.0.0       127.0.0.1       255.0.0.0       UG    0      0        0 lo
0.0.0.0         192.168.0.1     0.0.0.0         UG    2004   0        0 wlan0

Dengan kata lain 10.3.0.0/27 ada di eth0 dan 192.168.0.0/24 di wlan0, tetapi untuk menjangkau orang lain (tidak termasuk 127. *), mesin ini harus mengirim barang ke 192.168.0.1

Baru saja menjawab pertanyaan serupa:

Mengapa hanya ada satu Default Gateway?

Gateway "*" atau 0.0.0.0 menunjukkan bahwa jaringan dapat dijangkau secara langsung - tidak diperlukan gateway. Gateway default adalah alamat tempat paket dikirim jika tidak cocok dengan rute yang lebih spesifik - yaitu jika ditujukan untuk router inti & internet yang lebih luas.
