Komputer

Jika seseorang masuk ke SSH di komputer saya, dapatkah saya mengakses komputer mereka?

Misalnya ada 2 komputer, A dan B. Komputer A adalah server SSH. Komputer B terbuka dan koneksi / terowongan SSH ke komputer A. Apakah mungkin, dari Komputer A, menggunakan koneksi / terowongan SSH untuk mengakses komputer B dari komputer A? Juga, bagaimana Anda bisa memutuskan komputer B dari SSH tanpa mematikan server?

Jawaban dasarnya adalah tidak: pengguna di komputer klien (B) dapat menjalankan perintah di komputer server (A) (oleh karena itu administrator di klien dapat menjalankan perintah di server), tetapi administrator di server hanya dapat mempengaruhi apa yang dilakukan pengguna pada klien.

Namun ini tidak berarti bahwa komputer klien dilindungi sepenuhnya dari administrator server. Administrator server jahat dapat memasukkan beberapa kode serangan dalam file yang dapat dieksekusi yang akan disalin oleh pengguna ke komputer klien dan dijalankan di sana.

Selain itu, jika pengguna telah membuka terowongan selain koneksi ssh, ini mungkin menyediakan vektor serangan lain. Misalnya, terowongan X11 dapat memungkinkan administrator server untuk mengontrol aplikasi X11 dari jarak jauh yang berjalan secara lokal di mesin klien (misalnya dengan memasukkan penekanan tombol atau data papan klip). OpenSsh memberikan beberapa perlindungan (lihat deskripsi -X, -Y dan ForwardX11Trusted di Buka manual klien ) melalui Ekstensi X11 KEAMANAN , tetapi kontrol ini tidak memberikan perlindungan mutlak (X11 tidak dirancang dengan mempertimbangkan isolasi aplikasi, jadi Anda tidak dapat benar-benar memiliki keamanan tanpa mengorbankan fungsionalitas).
