Bagaimana cara membandingkan konten dari dua file pst yang memeriksa alamat email yang sama?

Apakah mungkin untuk mengotomatiskan perbandingan dua file pst?

Inilah situasi saya ... Seorang mantan karyawan yang akan segera membawa laptopnya sehingga isinya dapat disalin / dicadangkan (itu adalah laptop perusahaan, tetapi dia akan membawanya).

Ada komputer lain di kantor yang memeriksa alamat email yang sama dengan versi Outlook yang sama.

Karyawan tersebut menghapus email pribadinya sebelum membawa laptopnya (katanya banyak). Kami mencoba mencari tahu apakah dia menghapus lebih dari sekedar email pribadi.

Tentu saja kami memiliki semua email di mesin di kantor, tetapi singkatnya baris demi baris melalui kotak masuk, adakah cara untuk membandingkan dua file pst untuk mengetahui pesan mana yang hilang dari cadangan yang dimiliki versi lengkap?

Sayang sekali tidak ada perangkat lunak pembanding untuk melakukan pekerjaan itu. Ketertarikan saya adalah untuk memeriksa bahwa ekspor email ke file pst baru telah dilakukan secara lengkap dan akurat. Ekspor berguna karena akan memperbaiki kesalahan dalam file pst dalam satu klik, tidak seperti scanpst.exe, yang perlu dijalankan beberapa kali, dan juga akan menghasilkan file pst yang jauh lebih ringkas daripada menjalankan pemadat dalam pengelolaan file data Outlook.

Saya telah membaca di https://www.experts-exchange.com/questions/26658072/Compare-Outlook-PST-files.html , dan 4Team telah mengkonfirmasinya kepada saya, bahwa DuplicateKiller 4Team entah bagaimana akan membandingkan konten file pst, tetapi saya tidak yakin bagaimana dan tidak dapat menyelidikinya karena program itu beroperasi sebagai tambahan untuk Outlook, tetapi setelah saya menginstal DuplicateKiller di sistem - Outlook 2007 dan XP (yang dikatakan 4Team didukung) - Saya tidak dapat lagi membuka Outlook kecuali dalam mode aman - yaitu menjalankan outlook.exe / safe - yang tentu saja tidak akan menampilkan addin.

Saya rasa Anda akan menemukan bahwa jika Anda mengekspor ke file csv, Anda hanya dapat mengekspor satu folder di pst pada satu waktu, dan bukan seluruh file .pst. Setidaknya, itu yang terjadi dengan Outlook 2007. Dan saya curiga bahwa mengekspor konten pesan ke file csv bisa menjadi berantakan, meskipun saya belum melakukannya sejauh itu. Mungkin solusi yang lebih baik adalah mengarsipkan file pst untuk dibandingkan menggunakan Mailstore atau Mailstore Home (yang terakhir gratis, tetapi hanya akan mengarsipkan file pst yang dipasang di Outlook), lalu mengekspor arsip untuk setiap file pst dari Mailstore atau Mailstore Home ke direktorinya sendiri sebagai file .eml atau .msg. Email tersebut kemudian diekspor ke direktori dengan subfolder dll sesuai dengan struktur folder di file pst. Email diberi nomor tetapi tidak menunjukkan tanggal pengiriman. Kemudian Anda dapat menggunakan Beyond Compare untuk mengidentifikasi email yang tidak muncul di kedua file pst dan menggunakan fungsi salin ke folder di Beyond Compare untuk mengumpulkan email tersebut di direktori atau direktori terpisah. Saya belum mengambil sejauh itu, tetapi saya pikir ini pada dasarnya adalah metode yang sama seperti yang disarankan di atas, tetapi menggunakan Mailstore atau Mailstore Home alih-alih fungsi ekspor Outlook ke csv untuk mengatasi fakta bahwa Outlook hanya akan mengekspor satu folder di a waktu sebagai file csv.
