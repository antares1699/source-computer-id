`Mv` yang terlalu bersemangat mengganti nama folder / usr / lib saya

Saya mengalami masalah ketika file libiconv.2.dylib saya di / usr / lib rusak dan banyak aplikasi gagal. Saya mendapat salinan folder friends / usr / lib dan akan menukarnya. Rupanya saya kurang memikirkannya dengan baik karena ketika saya mengeluarkan sudo mv /usr/lib /usr/lib.old itu menyebabkan beberapa masalah. Sekarang Finder tidak mau terbuka, Terminal.app terus mengatakan "Operasi Ilegal" dan hal-hal lain seperti Archive Utility tidak berfungsi. Saya cukup yakin bahwa jika saya me-reboot mesin saya, itu tidak akan hidup sama sekali.

Bagaimana saya bisa mengganti nama ini kembali, atau mengganti nama salinan teman saya menjadi hanya / usr / lib (currenty /usr/lib.new) ???

Program Anda berhenti bekerja karena mereka mungkin terkait dengan perpustakaan (ketergantungan keras) yang tidak dapat ditemukan lagi. Ini menghentikan program bekerja.

Jika Anda dapat memulai terminal, Anda harus dapat secara manual menentukan jalur ke perpustakaan lama Anda di dalam variabel LD_LIBRARY_PATH sehingga Anda dapat memulai program lain.

$ export LD_LIBRARY_PATH=/usr/lib.old

Kemudian pindahkan perpustakaan Anda kembali ke tempat semula dan unset LD_LIBRARY_PATH.

$ unset LD_LIBRARY_PATH

Uji apakah semuanya berfungsi seperti sebelumnya.

Dan jangan pindahkan semua /usr/lib/ lagi jika Anda memiliki masalah dengan satu pustaka.
