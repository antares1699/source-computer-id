Bagaimana cara mempercepat pembuatan koneksi SSL?

Saya memiliki pengaturan OpenSSL di PC Linux saya menggunakan Apache2.

TIDAK, saya belum mengaktifkan penyelesaian DNS. Saya memiliki sertifikat di mesin, LOKAL.

Namun, devtools Chrome memberi tahu saya bahwa Initial Connection took 1.64 s. Itu sangat menunggu saya ketika harus menyegarkan halaman setelah waktu tertentu berlalu.

Bagaimana cara mempercepatnya atau begitukah seharusnya?

PS: Saya menggunakan ssl gratis dari StartCom.

  • Titik sakit dalam SSL / TLS adalah pertukaran kunci. Ini adalah bagian yang membatasi throughput, dan tidak jarang hanya dapat melakukan 500 atau lebih pertukaran kunci per detik karena Diffie-Hellman secara komputasi mahal (enkripsi massal dengan AES atau TripleDES relatif mudah, dan di situlah 1000 koneksi bersamaan muncul ). Tetapi mesin semi-modern apa pun (termasuk ponsel dan tablet) harus dapat melakukan lebih dari 1,5 detik untuk satu pertukaran. Masalah Anda mungkin terjadi di tempat lain kecuali Anda memilih ukuran kunci yang keterlaluan. Misalnya, RSA-2048 membutuhkan komputasi 100x lebih banyak daripada RSA-1024.

Ada beberapa area untuk meningkatkan koneksi SSL (dan latensi lalu lintas SSL umum). Beberapa di antaranya dapat memengaruhi keamanan dalam beberapa cara kecil atau hebat.

(Ini untuk ssl.conf menggunakan Apache)

Nonaktifkan pencarian Domain, dan izinkan Non-SNI menjangkau domain primer

  • SSLStrictSNIVHostCheck mati

Alih-alih host bernama, gunakan alamat IP untuk entri: 443 VHOST

Cache Sesi SSL

  • SSLSessionCache shmcb: / run / httpd / sslcache (512000)
  • SSLSessionCacheTimeout 300

Diizinkan untuk pesanan Protokol SSL lebih cepat , kompatibilitas umum, keamanan sedang:

  • SSLProtokol semua -SSLv3
  • SSLCipherSuite ALL: + HIGH: + TLSv1:! ADH:! EXP:! SSLv2:! MEDIUM:! LOW:! NULL:! ANULL
  • SSLHonorCipherOrder aktif

OCSP Stapling (mempercepat SSL)

  • SSLUseStapling aktif
  • SSLStaplingResponderTimeout 5
  • SSLStaplingReturnResponderErrors mati
  • SSLStaplingCache shmcb: / run / ocsp (128000)

HSTS (dapat melakukan preload sehingga permintaan http: // menjadi https: // permintaan di browser

  • Header selalu menyetel Strict-Transport-Security "max-age = 63072000; includeSubdomains; preload"

Kirimkan untuk pramuat di sini: https://hstspreload.appspot.com/

Pertimbangkan penerapan lebih lanjut berikut ini:

  • HTTP / 2 dengan HPACK
  • Kompresi Brotli

Masalah non-SSL lainnya yang dapat membantu mempercepat koneksi

Direktif lain untuk Apache di httpd.conf

  • HostnameLookups Off

Jangan gunakan StartCom.

Sebagai gantinya, gunakan Let's Encrypt, dengan opsi konfigurasi otomatis --apache.

Ini akan meminta Anda semuanya ramah pengguna dan Anda dapat sepenuhnya menyesuaikan konfigurasi Apache Anda setelah mengatur Let's Encrypt SSL.

Jadi, masalah diselesaikan dengan menghapus StartCom SSL dan konfigurasi otomatis apache dengan Let's Encrypt.

Saya tidak begitu tahu apa masalah awalnya di sini, tetapi Let's Encrypt tampaknya dioptimalkan dengan lebih baik.
