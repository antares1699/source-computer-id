Buat daftar file yang diawali dengan garis bawah terlebih dahulu saat menggunakan `ls -l`

Saya menggunakan ubuntu (rapi), dan ketika saya menggunakan ls -l, file-file tersebut terdaftar, tetapi tampaknya pengurutan algo mengabaikan karakter khusus apa pun. Selama berabad-abad saya telah menggunakan garis bawah untuk menandai folder khusus. Dan menurut saya, mereka selalu didaftar pertama.

Sekarang, garis bawah sepenuhnya diabaikan.

Anggaplah saya memiliki file fileA, _fileB dan fileC dalam sebuah folder. Saat ini, ls -l memesannya seperti ini:

malbert@dredg:/tmp/1$ ls -l
total 0
-rw-r--r-- 1 malbert domain users 0 2011-08-03 15:27 fileA
-rw-r--r-- 1 malbert domain users 0 2011-08-03 15:27 _fileB
-rw-r--r-- 1 malbert domain users 0 2011-08-03 15:27 fileC

Tapi saya ingin ini diurutkan seperti berikut:

malbert@dredg:/tmp/1$ ls -l
total 0
-rw-r--r-- 1 malbert domain users 0 2011-08-03 15:27 _fileB
-rw-r--r-- 1 malbert domain users 0 2011-08-03 15:27 fileA
-rw-r--r-- 1 malbert domain users 0 2011-08-03 15:27 fileC

Saya telah menggali halaman manual ls dan tidak dapat menemukan apa pun. Apakah ada suatu tempat opsi collation untuk seluruh sistem? Atau semacamnya?

Saya membayangkan itu mematuhi pengaturan LC_COLLATE di lingkungan Anda atau, jika tidak ada, pengaturan LC_ALL atau LANG sebagai gantinya.

Perintah locale(1) akan memberi tahu Anda pengaturan apa yang akan dilihat program, berdasarkan lingkungan Anda.
