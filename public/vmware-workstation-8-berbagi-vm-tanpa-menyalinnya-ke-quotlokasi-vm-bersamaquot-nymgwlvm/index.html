VMWare Workstation 8, Berbagi VM tanpa menyalinnya ke "Lokasi VM bersama"

Saya memiliki komputer induk (Win7-64) dengan tiga harddisk berbeda. Ini menjalankan beberapa VM yang didistribusikan di antara harddisk tersebut untuk kinerja yang lebih baik. Sekarang dengan VMWare Workstation 8 dimungkinkan untuk "berbagi" VM. Saya ingin menggunakan fitur ini, tetapi saya harus mengonversi VM menjadi "VM Bersama". Ketika saya mencoba WMWare ini memaksa saya untuk menyalin VM ke satu folder yang telah ditentukan sebelumnya, sehingga membuat upaya saya untuk mendistribusikan VM di antara harddisk tidak berarti. Apakah ada kemungkinan untuk menyimpan VM di tempatnya dan tetap membagikannya?

Saya tidak berpikir Anda benar-benar dapat melakukan itu, meskipun saya juga sangat berharap mereka menambahkannya.

Saya baru saja mulai menggunakan Workstation 8, tetapi sepertinya bagian "berbagi" di dalamnya agak kurang fiturnya. itu berfungsi dengan baik, kecuali kontrol atasnya sangat kurang berkilau.

Tentu saja Anda dapat mengubah di mana lokasi default itu adalah tempat VM bersama pergi, tetapi tidak ada pengaturan yang memungkinkan Anda untuk menggunakan lokasi yang berbeda (sayangnya) segera setelah Anda memindahkannya, Workstation tidak akan tahu di mana itu, dan jika Anda tambahkan kembali, itu hanya akan menjadi VM biasa, bagikan lagi, dan itu langsung kembali ke tempat yang sama.

Saya tahu ini bukan jawaban yang sebenarnya, tetapi hanya ingin memberi tahu Anda bahwa sepertinya (hingga saat ini) hal itu tidak dapat dilakukan. Mudah-mudahan mereka merilis pembaruan atau sesuatu yang akan menambahkan lebih banyak fungsionalitas (seperti kontrol pengguna! Mis. Menyembunyikan vms dari pengguna tertentu, atau hanya mengizinkan pengguna yang memulai VM benar-benar mengontrol VM, selain admin ... dll.)

Saya belum mencoba fitur ini, jadi jika VMware tidak mengizinkan Anda untuk menentukan banyak lokasi, Anda dapat melakukan beberapa trik NTFS seperti Mount Points, Junction Points, Symbolic Links, untuk membuat program percaya bahwa data sebenarnya berada di direktori itu. , tetapi sebenarnya direktori tersebut adalah penunjuk ke lokasi sebenarnya.

Ada banyak alat untuk membuat tautan ini, tetapi favorit saya adalah Tautan Ekstensi Shell .

Jadi, apa yang mungkin dapat Anda lakukan adalah setelah VMware menempatkan VM Bersama di direktori VM Bersama, Anda dapat memindahkan VM kembali ke drive khusus dan membuat tautan simbolis di lokasi dan nama yang ditempatkan VMware sebagai VM Bersama, dan memiliki tautan simbolis mengarah ke folder drive khusus. VMware seharusnya tidak lebih bijak.
