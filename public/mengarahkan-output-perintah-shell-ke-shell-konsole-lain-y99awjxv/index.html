Saya menulis skrip shell dan saya ingin menulis perintah di skrip tetapi mengarahkan output ke sesi konsole lain. (menggunakan kde)

Pertama, cari nomor yang mengidentifikasi sesi tty saat ini; mari pertimbangkan nomor tersebut sebagai X:

| terminal 1               | terminal 2                |
 ------------------------------------------------------
| $ tty                    | $                         |
| /dev/pts/X               |                           |

Kemudian, gunakan nomor yang diberikan untuk menulis ke tty yang dipilih:

| terminal 1               | terminal 2                |
 ------------------------------------------------------
| $ tty                    | $ echo "foo" > /dev/pts/X |
| /dev/pts/X               |                           |

Dan itu akan menghasilkan:

| terminal 1               | terminal 2                |
 ------------------------------------------------------
| $ tty                    | $ echo "foo" > /dev/pts/X |
| /dev/pts/X               |                           |
| foo                      |                           |

Cetak keluaran terminal ke file, bukan ke terminal tempat Anda saat ini berada:

commands >>outputfile

Di terminal baru, gunakan tail untuk menampilkan konten outputfile dan perbarui secara otomatis saat lebih banyak teks ditambahkan:

tail -f outputfile

Terlambat tapi mungkin lebih baik daripada tidak pernah ... :-)

mkfifo ~/MyOutput           # Create the FIFO (Special file) 

./MyScript.sh > ~/MyOutput  # In the terminal/console 1
cat ~/MyOutput              # In another tty/console

dari info coreutils 'mkfifo invocation':

A "FIFO" adalah jenis file khusus yang memungkinkan proses independen untuk berkomunikasi. Satu proses membuka file FIFO untuk menulis, dan lain untuk membaca, setelah itu data dapat mengalir seperti biasanya pipa anonim di cangkang atau di tempat lain.

Catatan samping:

  • Lebih baik membuat FIFO bukan di direktori home Anda :-)
  • Anda dapat memiliki lebih dari satu FIFO untuk proses yang sama (std out, std err).
  • Sampai Anda tidak akan meminta dump dari FIFOs itu akan menampung semua output.
    Seperti yang akan Anda tanyakan pertama kali, itu akan terjadi siram semua.
  • Anda dapat mengarahkan (atau menambahkan) ke file lain cat MyOutput >> NewRealFile
  • Anda juga dapat menggunakan cat MyOutput dari terminal lain!
  • Peringatan : Jika Anda meminta 2 program berbeda (atau contoh) untuk mengarahkan output ke FIFOs yang sama, fluks akan digabungkan (tidak a priori cara untuk membedakan dari program mana baris itu berasal).
  • Peringatan : Jika Anda bertanya 2 kali atau lebih (mungkin dari terminal yang berbeda) ini akan memberikan satu baris untuk setiap permintaan yang membagi output ke pemohon.
  • Ini dapat memiliki kegunaan yang lebih kompleks: Bagaimana cara meluncurkan program dengan benar dan memungkinkan untuk mengarahkan output yang sedang berjalan (stdout dan stderr) di tahap selanjutnya?

Gunakan perintah tty di setiap terminal untuk mengidentifikasinya:

$ tty
/dev/pts/0

$ tty
/dev/pts/1

Dengan asumsi TTY ini, untuk mengarahkan stdout pertama ke yang kedua, jalankan ini di terminal pertama:

exec 1>/dev/pts/1

Catatan: Sekarang setiap keluaran perintah akan ditampilkan di poin / 1

Untuk mengembalikan perilaku default stdout dari pts / 0:

exec 1>/dev/pts/0
