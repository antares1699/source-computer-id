  • Apakah ini masih akan berhasil? cp /usr/bin/passwd . ; chmod +x ./passwd ; ./passwd

  • @ f.ardelian Masalahnya, passwd memiliki beberapa sihir khusus yang disebut "setuid" di atasnya - itu berarti bahwa ketika seseorang menjalankan file, mereka menjalankannya sebagai pemiliknya (yaitu, root.) Hal ini memungkinkan pengguna biasa untuk mengubah /etc/shadow file yang berisi kata sandi. Jika Anda menyalin file ke direktori home pengguna, file tersebut tidak lagi diseteluid, dan oleh karena itu tidak lagi dijalankan secara otomatis dengan hak akses root. Untuk mempelajari lebih lanjut, cari informasi tentang "izin Unix" dan "setuid".
    — JamesTheAwesomeDude

passwd -n 9999 user akan mencegah user mengubah kata sandinya selama hampir 274 tahun.

Jika Anda ingin memiliki user tanpa kata sandi, yang tidak dapat mengubah kata sandinya, buka /etc/shadow sebagai root, temukan baris yang dimulai dengan nama pengguna, dan ubah konten antara titik dua pertama dan kedua menjadi U6aMy0wojraho.
(sumber: https://help.ubuntu.com/community/PasswordlessGuestAccount )
