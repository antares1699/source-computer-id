debian stretch upgrade - mariadb menghapus libmysqlclient.so.18 - bagaimana cara mendapatkannya kembali?

Saya mengupgrade ke debian stretch dari jessie.

Database mysql saya bermigrasi dan saya dapat membacanya. Saya memiliki proyek django yang mengandalkan libmysqlclient.so.18 untuk mengakses mysql.

saya telah menginstal paket mariadb-server dan libmariadbclient18.

apa yang harus saya lakukan untuk memastikan django dapat mengakses database saya?

ini adalah kesalahan yang saya dapatkan:

    raise ImproperlyConfigured("Error loading MySQLdb module: %s" % e)
django.core.exceptions.ImproperlyConfigured: Error loading MySQLdb module: libmysqlclient.so.18: cannot open shared object file: No such file or directory

saya mencoba ln -s /usr/lib/x86_64-linux-gnu/libmariadbclient.so.18 to /usr/lib/x86_64-linux-gnu/libmysqlclient.so.18 yang mengubah kesalahan sebagai berikut:

    raise ImproperlyConfigured("Error loading MySQLdb module: %s" % e)
django.core.exceptions.ImproperlyConfigured: Error loading MySQLdb module: /usr/lib/x86_64-linux-gnu/libmysqlclient.so.18: version `libmysqlclient_18' not found (required by /usr/local/lib/python2.7/dist-packages/_mysql.so)

Saya ingin mengucapkan terima kasih telah membuat utas ini, ini sangat membantu. Masalah saya hampir identik dan itu terjadi setelah peningkatan distribusi ke Debian Buster - Saya memilih untuk memutakhirkan semua paket mariaDB dari versi 10.1 ke 10.3 sambil memilih untuk menyimpan file konfigurasi asli.

Berikut adalah solusi alternatif (perbaikan terbaru) yang berhasil untuk saya:

ln -s libmariadb.so.3 /usr/lib/x86_64-linux-gnu/libmariadbclient.so.18

Solusi ini didokumentasikan oleh mariaDB di bawah ini: https://jira.mariadb.org/browse/MDEV-13934
