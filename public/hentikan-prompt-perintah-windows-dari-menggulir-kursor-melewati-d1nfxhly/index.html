Hentikan prompt perintah Windows dari menggulir kursor melewati

Saya menggunakan Windows 10 untuk proyek saya saat ini setelah lebih dari satu dekade menggunakan macOS / Linux. cmd tampaknya datang sedikit dari masa Windows XP, tetapi saya mengalami satu masalah khusus.

Di Terminal macOS, jika saya menggulir kembali ke atas untuk memeriksa sesuatu, ketika saya ingin kembali, saya dapat melemparkan gulungan ke arah lain dan itu akan berhenti di akhir teks saat ini di mana kursor berada. Jika saya melakukan hal yang sama di Command Prompt, ternyata ada ribuan baris teks kosong di bagian bawah di luar kursor dan saya perlu menggulir kembali dengan hati-hati untuk kembali ke kursor. Mengetik sesuatu akan memfokuskan kembali ke baris yang benar, tetapi hanya jika ada kursor; jika saya menjalankan rangkaian pengujian saya, ia menunggu perintah tanpa kembali ke shell, dan tidak akan menggulir ke titik yang benar saat mengetik.

Saya telah melalui pengaturan dan tidak dapat melihat sesuatu yang jelas untuk diubah. Adakah cara agar saya dapat mengubah perilaku agar sesuai dengan Terminal macOS, atau apakah saya harus mempelajari kembali memori otot saya?

Mulai Windows 10 build 18298 dan yang lebih baru, Anda dapat membuka properti dan mengaktifkan file "Nonaktifkan Scroll-Forward" kotak centang

Nonaktifkan Scroll Forward:

Saat dihapus (status default), Konsol dapat menggulir ke mana saja dalam buffer Konsol, bahkan di bawah baris teks terbaru yang ditampilkan Jika dicentang, Konsol tidak akan mengizinkan pengguliran di bawah keluaran baris terakhir, mirip dengan cara kerja sebagian besar * terminal NIX

  • Fitur Konsol Eksperimental Baru
  • Cara Mengaktifkan atau Menonaktifkan Scroll Forward di Jendela Konsol di Windows 10
  • Nonaktifkan Gulir Maju di Konsol di Windows 10
