Bagaimana cara memperbaiki masalah ketidakcocokan cipher dengan sftp?

Saya mengalami masalah saat server ditingkatkan ke RHEL 6.5, dan kami memiliki skrip bash otomatis pada penginstalan RHEL 5.9 yang terhubung melalui perintah sftp dan ssh.

Sekarang, server baru hanya mendukung ciphers aes128-ctr, aes192-ctr, dan aes256-ctr.

Saya dapat memperbarui perintah ssh untuk menggunakan opsi -c dengan aes256-ctr dan ini berhasil:

ssh -c aes256-ctr ${remote_host} ${my_command}

Namun, ketika saya mencoba melakukan yang setara untuk sftp:

sftp -oCipher=aes256-ctr ${remote_host} <<< $'get /home/me/* me/'

Saya mendapatkan pesan kesalahan bahwa klien tidak mendukung cipher yang sama dengan server:

tidak ada sandi yang cocok ditemukan: klien arcfour256, aes128-cbc, 3des-cbc, blowfish-cbc, cast128-cbc, arcfour, aes192-cbc, aes256-cbc server aes128-ctr, aes192-ctr, aes256-ctr

Perintah ssh dan sftp ini dijalankan pada kotak RHEL 5.9 yang sama, jadi saya bertanya-tanya mengapa ini berfungsi untuk ssh dan bukan sftp?

Juga, bagaimana cara memperbaikinya sehingga saya dapat terhubung menggunakan sftp tanpa perubahan sisi klien?

Direktif Cipher adalah untuk SSH versi 1 (yang tidak digunakan saat ini).

Untuk SSH versi 2, gunakan Ciphers:

sftp -oCiphers=aes256-ctr

Lihat ssh_config halaman manual .

Meskipun perhatikan bahwa sftp mendukung sakelar -c juga. Jadi tidak perlu menggunakan -o.

Lihat sftp halaman manual :

-c cipher

Memilih cipher yang akan digunakan untuk mengenkripsi transfer data.
Opsi ini langsung diteruskan ke ssh (1) .

Opsi ini didukung sejak OpenSSH 5.4 . Perubahan itu menyamar sebagai "Dukung sebagian besar argumen baris perintah scp (1) di sftp (1) ".

Perhatikan bahwa argumen baris perintah -c pada dasarnya setara dengan perintah Ciphers (sementara itu dapat kembali ke Cipher). Kutipan dari ssh halaman manual :

-c cipher_spec

Memilih spesifikasi penyandian untuk mengenkripsi sesi.
Protokol versi 1 memungkinkan spesifikasi satu sandi. Nilai yang didukung adalah "3des", "blowfish", dan "des". Untuk protokol versi 2, cipher_spec adalah daftar sandi yang dipisahkan oleh koma urutan preferensi. Lihat kata kunci Ciphers dalam ssh_config (5) untuk lebih informasi.
