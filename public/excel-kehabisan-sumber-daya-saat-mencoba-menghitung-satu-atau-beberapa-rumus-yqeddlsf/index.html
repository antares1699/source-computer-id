Excel kehabisan sumber daya saat mencoba menghitung satu atau beberapa rumus

Saya menerima pesan kesalahan "Excel kehabisan sumber daya saat mencoba menghitung satu atau lebih rumus" di SATU dari komputer saya (Dari 2).

Lembar kerja saya berisi: (Di dalam satu lembar kerja, bukan buku kerja, ada rumus di lembar kerja lain)

  • 1.000.000 rumus

  • tabel Pivot berdasarkan 900.000 baris data

Ketika saya menjalankan excel / vba pada tindakan "Hitung Lembar" (Hanya lembar kerja saat ini), program akan memunculkan pesan kesalahan berikut:

Excel ran out of resources while attempting to calculate one or more formulas

tidak saya "menyegarkan" tabel pivot di (Excel atau VBA) atau "Hitung Lembar" di (Excel atau VBA)

Saya memiliki 2 komputer:

  • keduanya menjalankan 64 bit Windows 7,

  • keduanya menjalankan Excel 2007 32Bit,

  • Saya menjalankan Excel tepat setelah memulai Windows,

  • PC pengembangan saya dengan RAM 2GB dapat berjalan tanpa masalah,

  • PC lain dengan RAM 6GB menampilkan pesan kesalahan ran out of resources

  • Berjalan pada kumpulan data yang sama, file excel yang sama

Saya juga memperhatikan bahwa pada PC pengembangan saya, ia menggunakan ~ 1.2G RAM, dan PC yang tidak berfungsi, ia menggunakan RAM 900M sebelum mengklik Tindakan "Segarkan" / "Hitung".

EDIT

Komputer Tidak Bekerja dapat menangani data dalam 100.000 baris data

  1. Mengapa ini berfungsi untuk komputer dengan memori lebih sedikit tetapi tidak berfungsi untuk komputer dengan memori lebih banyak? (pertanyaan utama)
  2. Apa yang dapat saya lakukan untuk mengurangi memori yang digunakan oleh Excel? (sub pertanyaan) (Selain menghapus data)

Setiap bantuan dihargai, tolong arahkan saya ke arah yang benar atau berikan petunjuk.

EDIT : Saya berpikir untuk menghapus rumus, dan memindahkan logika ke vba, dan melakukannya dengan menyimpan data di cache mungkin per 10.000 baris. Tetapi ini tidak akan menyelesaikan masalah saya jika tabel pivot "menyegarkan" akan menampilkan kesalahan yang sama.

Alasannya mungkin karena perbedaan fragmentasi memori 32-bit pada 2 Pcs (seringkali sulit untuk menggunakan semua 2 GB yang tersedia secara teoritis).
Anda dapat mengurangi jumlah memori yang digunakan oleh tabel pivot dengan mengurangi jumlah kolom dan / atau mengurangi memori yang dibutuhkan oleh beberapa kolom (string teks adalah kandidat yang baik).
(Anda dapat mengukur jumlah memori yang digunakan oleh pivot cache dengan VBA PivotCache.memoryUsed)
Saya berasumsi Anda membuat cache pivot dengan membaca data dari kueri langsung ke cache pivot daripada meletakkan data kueri ke lembar kerja dan mendasarkan pivot pada lembar kerja, yang akan menggunakan lebih banyak memori.
Anda tidak mengatakan apa yang> rumus 1000000 jadi saya tidak punya saran untuk perbaikannya.
Jika Anda ingin menggunakan data dalam jumlah besar di pivot Excel dll, Anda akan mendapatkan hasil yang lebih baik dari versi 64-bit Excel 2010 yang tidak memiliki batas 2 GB.

Sepertinya Anda membuat lembar Excel yang sangat besar. Aplikasi 32-bit hanya dapat menggunakan hingga 2GB RAM, kecuali mereka dapat dikonfigurasi untuk menggunakan hingga 3,2GB dengan "sakelar" khusus. Jadi Excel 32-bit sebenarnya kehabisan memori setelah 2GB.

Jadi tidak masalah jika Anda memiliki RAM 16 atau 32GB. Anda membutuhkan MS Office 64-bit agar bisa bebas menggunakan RAM di atas 2GB. Hanya dengan begitu RAM tambahan 6GB Anda akan benar-benar berguna untuk Excel. Jika tidak, macet karena harus mencari cara untuk menjalankan dengan hanya 2GB RAM, dan kemungkinan besar, akan mulai menggunakan pagefile pada hard drive.

Salah satu kemungkinannya adalah meningkatkan ukuran file halaman Windows. Saya tidak pernah membiarkan windows mengatur pagefile. Saya mengaturnya ke 1.5X RAM fisik dan mengatur ukuran pagefile minimum dan maksimum SAMA. Ini mencegah fragmentasi RAM. Jadi jika Anda memiliki RAM 4GB, atur ke 6144 / 6144MB min / max.
