Apakah ada alasan untuk TIDAK menggunakan XAMPP (atau paket server serupa)?

Sampai beberapa minggu yang lalu saya selalu biasa mengatur server rumah saya satu layanan pada satu waktu (mis. Httpd, ProFTPD, dll.). Sekarang saya sudah mulai menggunakan XAMPP dan bertanya-tanya apakah ada alasan mengapa saya seharusnya tidak akan menggunakannya.

Saya pernah mendengar bahwa menggunakan paket server seperti XAMPP dapat menimbulkan risiko keamanan. Bagaimana ini mungkin jika file konfigurasi masih ada untuk saya edit secara manual jika perlu?

Apakah ada masalah potensial lain yang harus saya waspadai?

Masalah terbesar adalah bahwa di luar kotak itu tidak terkunci dengan sangat ketat.

Dari Wikipedia :

Secara resmi, desainer XAMPP dimaksudkan itu untuk digunakan hanya sebagai alat pengembangan, untuk memungkinkan perancang situs web dan programmer untuk menguji pekerjaan mereka komputer mereka sendiri tanpa akses apa pun ke Internet. Untuk membuatnya semudah ini mungkin, banyak keamanan penting fitur dinonaktifkan secara default.

Jika Anda tahu apa yang Anda lakukan dan meluangkan waktu untuk mengamankannya dengan benar, itu sama tidak amannya dengan aplikasi web dan kata sandi yang telah Anda tetapkan pada layanan Anda.

Jika Anda tidak tahu apa yang Anda lakukan, atau Anda tidak meluangkan waktu untuk mengamankannya dengan benar, ini berpotensi menjadi lubang keamanan yang besar.
