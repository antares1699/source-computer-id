Macports, Fink, Hombrew: Proses Latar Belakang?

Jika saya menginstal manajer paket seperti Macports, Fink atau Homebrew, bagaimana pengaruhnya terhadap startup dan jalannya sistem saya?

Sepertinya jawabannya adalah "Tidak sama sekali saat Anda tidak menggunakannya" tetapi saya khawatir mereka akan merusak perangkat lunak lain (seperti Mono) atau menjalankan proses latar belakang. Apakah ketakutan saya benar-benar salah tempat? Apakah mereka hanya program seperti yang lainnya?

Mereka tampaknya menempatkan jejak mereka di sekitar OS di beberapa tempat.

Saya terutama dapat berbicara untuk MacPorts, tetapi saya yakin ini berlaku setidaknya untuk Fink dan kemungkinan Homebrew juga. Tidak ada proses latar belakang sama sekali untuk penginstal paket MacPorts. Ini hanyalah sebuah sistem yang melacak port yang diinstal dan memelihara dependensi dalam direktori /opt/local yang berdiri sendiri (dengan beberapa pengecualian). Jika Anda menginstal semacam daemon atau layanan yang perlu dijalankan sebagai layanan latar belakang, maka itu cerita yang berbeda dan keluaran port install biasanya akan memberi Anda instruksi tentang cara memuatnya.

Saya tidak dapat melihat cara menginstal MacPorts (atau Fink, dll) secara inheren akan merusak instalasi Mono Anda, yang menginstal sendiri pada /Library/Frameworks dengan symlink untuk executable pada /usr/bin. Saya akan mengatakan ketakutan Anda salah tempat.

Dimungkinkan untuk menginstal proses daemon melalui Fink, MacPorts dan Homebrew tetapi dalam semua kasus, Anda perlu menjalankan launchctl secara manual untuk menambahkan proses itu ke launchd

Anda lebih cenderung membuat masalah sendiri dengan menginstal secara manual dari tarbal sumber daripada oleh manajer paket. Atau dengan menggunakan dua atau lebih manajer paket. Sebagai contoh, jangan mencoba dan menginstal permata ruby ​​melalui porta dan melalui perintah permata karena yang diinstal oleh perintah permata tidak akan memperbarui basis data port permata yang diinstal dan sebaliknya.
