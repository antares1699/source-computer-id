Anda mungkin mengacu pada variabel lingkungan .

Lingkungan diwarisi oleh setiap proses dari induknya. Di Windows, lingkungan pertama dibuat oleh proses logon. Jadi, jika Anda meluncurkan editor atau program lain, itu akan mewarisi dari Windows lingkungannya, menambahkan variabelnya sendiri, dan akan meneruskan semuanya ke anak cmd mana pun yang diluncurkan di akunnya sendiri.

jika kamu mau istilah agar selalu ada dalam proses cmd apa pun yang pernah diluncurkan saat login Anda, buka variabel Panel Kontrol / Sistem / Tingkat Lanjut / Lingkungan. Di sana Anda memiliki pilihan pengaturan istilah untuk Anda sendiri logon atau untuk sistem (artinya setiap proses diluncurkan dengan atau tanpa login).

Jika Anda mencari GUI yang lebih baik daripada yang ditawarkan oleh Windows, freeware Editor Lingkungan Cepat adalah pilihan yang sangat baik.

Saya tidak terbiasa dengan set term, tetapi Anda selalu dapat menyetel variabel lingkungan di panel kontrol Sistem.

Anda mungkin mencari nilai autorun di registri yang terletak di sini:

HKEY_LOCAL_MACHINE\Software\Microsoft\Command Processor

Dan / atau:

HKEY_CURRENT_USER\Software\Microsoft\Command Processor

Ini harus berupa REG_SZ atau REG_EXPAND_SZ yang berisi perintah yang dapat dieksekusi di Command Prompt. Anda mungkin ingin mengatur ini ke, katakanlah %USERPROFILE%\autorun.bat untuk menjalankan serangkaian perintah.

Perhatikan bahwa nilai HKLM akan dijalankan sebelum nilai HKCU jika keduanya disetel.

Sumber: http://technet.microsoft.com/en-us/library/cc779439%28WS.10%29.aspx

Hati-hati terhadap http://blogs.msdn.com/b/oldnewthing/archive/2007/11/21/6447771.aspx meskipun.
