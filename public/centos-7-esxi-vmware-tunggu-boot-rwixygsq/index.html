CentOS 7 ESXi / VMware Tunggu Boot

Saya menginstal CentoOS 7.0 1406 pada hypervisor VMWare ESXi 5.5. Saya menginstal "Sever dengan GUI" dan memilih KDE. Namun saat boot saya mendapatkan pesan berikut dalam waktu kurang dari satu detik, sebelum proses puncak (menunggu 15 menit, beberapa kali):

sd 0:0:0:0: [sda] Assuming drive cache: write through
sd 0:0:0:0: [sda] Assuming drive cache: write through
sd 0:0:0:0: [sda] Assuming drive cache: write through

Dengan flashing "_" tapi tidak ada kemampuan untuk memberikan input.

Drive dienkripsi. Saya tidak tahu apakah enkripsi drive CentOS akan mempengaruhi ini (saya belum diberi kemampuan untuk memasukkan frase untuk mendekripsi drive)

Saya mencoba menginstal ulang dari awal, pesan kesalahan yang sama. Saya memiliki server Windows 2012 yang berjalan di hypervisor tanpa masalah. Saya mencari-cari di Google, dan ini tampaknya tidak menjadi masalah dengan CentOS secara umum, jadi saya membayangkan itu adalah cara saya mengkonfigurasi mesin virtual saya, atau menginstal OS.

Saya menggunakan RAM 2 GB dan 1 inti virtual. Saya mencoba boot dengan menghapus akses vm ke jaringan, dan ini tidak mengubah apa pun.

  1. Apa masalahnya?

  2. Apa yang dapat saya lakukan untuk memecahkan masalah instalasi / mesin virtual saat ini?

  3. Bagaimana cara mereset mesin virtual, dan menginstal ulang OS untuk menghindari masalah ini di masa mendatang?

Di menu boot GRUB, tekan e untuk mengedit konfigurasi boot dan menghapus rhgb.

Sunting: Anda dapat menonaktifkannya secara permanen dengan menghapus rhgb di /etc/default/grub dan menjalankan grub2-mkconfig -o /boot/grub2/grub.cfg untuk memperbarui grub.cfg boot Anda.
