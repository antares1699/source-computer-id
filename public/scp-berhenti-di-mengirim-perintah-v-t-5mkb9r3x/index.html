SCP berhenti di `mengirim perintah -v -t .`

Saya mencoba mengirim file dari komputer saya ke server menggunakan perintah berikut:

scp -v file.txt username@server:~/file.txt
  • Saya menggunakan nama pengguna saya dan alamat server sebenarnya pada perintah di atas.

Ada ~/.bashrc, yang mungkin mengganggu perintah jarak jauh (seperti scp).

module load LANG/PYTHON/2.7.6
module load TOOLS/TMUX/2.1

Jika Anda ingin melakukan sesuatu seperti ini, Anda harus melakukannya hanya pada shell interaktif (bukan untuk merusak skrip). Anda dapat menambahkan ke awal ~/.bashrc Anda:

[ -z "$PS1" ] && return
Bagikan
