Bagaimana cara menonaktifkan nginx dari Gitlab bunlde?

  • Saya telah menginstal server Debian / Jessie dengan layanan apache 2.4
  • Saya mengunduh dan menginstal paket bundel Gitlab untuk Debian 8 (Jessie)
  • Saya mengaktifkan unicorn dan mengkonfigurasi virtualhost di apache untuk digunakan subdomain tertentu.

Sekarang jika saya me-restart server saya, nginx yang dibundel akan dimulai sebelum apache dan mendengarkan pada port 80 sebelumnya.

Saya mengubah konfigurasi nginx untuk mendengarkan pada port 666 tetapi solusinya tidak benar-benar bersih. Apakah ada cara untuk menonaktifkan layanan ini dari bundel?

setelah peningkatan singkat (peningkatan gitlab-ctl) dari gitlab, saya sekarang dapat melihat lebih banyak konfigurasi yang memungkinkan untuk nginx.

Jadi dari / etc / gitlab / gitlabrc ada baris:

#nginx['enable'] = true

Dikomentari secara default, jika nginx terus berjalan, Anda dapat menghapusnya dan mengubahnya menjadi false seperti ini.

nginx['enable'] = false

Dan mulai ulang layanan dengan

gitlab-ctl restart
