Bagaimana cara membagi file WebM tanpa pengkodean ulang?

Saya memiliki file WebM yang perlu saya pisahkan, tetapi saya ingin melakukannya tanpa encoding ulang. Menempatkannya dalam wadah MKV tidak berfungsi, karena titik di mana saya perlu membaginya bukanlah bingkai utama, jadi tidak berfungsi.

Apakah ini mungkin? Saya sangat ingin melakukannya tanpa encoding ulang jika memungkinkan, karena saya tidak ingin kehilangan kualitas apa pun.

  • Jika titik di mana Anda ingin membaginya bukan bingkai utama, maka tidak, ini tidak mungkin. Semua non-bingkai utama bergantung sebagian pada bingkai sebelumnya dalam urutan (mereka pada dasarnya merekam perubahan ke gambar sebelumnya daripada keseluruhan gambar - ada b-bingkai yang memperumit ini, tetapi prinsip yang sama berlaku) dan mengubahnya menjadi sebuah I-frame (keyframe) melibatkan pengkodean ulang.

1 Jawaban

6
6

Cukup mudah menggunakan perintah avconv dari baris perintah. Anda tidak perlu khawatir tentang pemisahan bingkai utama jika, seperti yang ditentukan kemudian oleh penanya, Anda memotong outro (akhir video). Berikut contohnya:

$ avconv -i videoandoutro.webm  -t 00:03:30  -codec: copy  videoonly.webm

Itu akan mengambil videoandoutro.webm dan membuat file baru, videoonly.webm dan memotongnya tepat setelah 3 menit dan 30 detik.

Memotong intro juga sebenarnya tidak jauh lebih sulit, tetapi melakukan itu akan mengalami masalah keyframe.

$ avconv -ss 00:01:01 -i jossintro.webm  -t 00:00:30  -codec: copy oatybar.webm

Di atas akan mengambil file .webm bernama "jossintro.webm" dan memotong irisan mulai sekitar 1 menit, 1 detik masuk. Outputnya dimasukkan ke dalam "oatybar.webm", yang dipangkas menjadi (sekitar) 30 detik. Hasilnya sebenarnya akan sedikit lebih lama karena waktu mulai dibulatkan ke bingkai utama sebelumnya.

[Perhatikan bahwa avconv didasarkan pada ffmpeg, sehingga alat itu kemungkinan besar juga akan berfungsi, tetapi saya hanya mengujinya dengan avconv.]
