Pada server linux tanpa kepala, deteksi jika disk optik yang dimasukkan adalah disk audio

Pada server Linux tanpa kepala (fedora terbaru jika ini diminati), bagaimana saya dapat mendeteksi bahwa disk optik yang dimasukkan adalah disk audio, bukan disk data atau DVD video? Tidak ada lingkungan desktop yang terpasang, dan saya memilih untuk tidak memasangnya.

Apa yang saya lihat adalah pemasangan gagal, tetapi saya rasa itu juga bisa menjadi kasus untuk disk data yang rusak:

[root@server ~]# mount /dev/sr0 /mnt/cdrom
mount: /dev/sr0 is write-protected, mounting read-only
mount: /dev/sr0: can't read superblock
[root@server ~]# echo $?
32

Ada saran?

Cara untuk mencapai ini adalah dengan menggunakan file cdparanoia.dll alat baris perintah. Di Ubuntu Linux Anda dapat menginstalnya melalui

sudo apt install cdparanoia

lalu lari

cdparanoia -Q

Dimana "-Q" adalah singkatan dari "query". Jika tidak ada disk audio di drive, Anda akan mendapatkannya

cdparanoia III release 10.2 (September 11, 2008)
004: Unable to read table of contents header
Unable to open disc.  Is there an audio CD in the drive?

Kalau tidak, akan terlihat seperti ...

Table of contents (audio tracks only): track        length             begin        copy pre ch
===========================================================
  1.     8315 [01:50.65]        0 [00:00.00]    no   no  2
  2.    12830 [02:51.05]     8315 [01:50.65]    no   no  2
  3.    15642 [03:28.42]    21145 [04:41.70]    no   no  2
  4.    16238 [03:36.38]    36787 [08:10.37]    no   no  2

Bagaimana cara mendeteksi jika disk optik yang dimasukkan adalah disk audio?

#!/bin/bash
checkforaudio=`gvfs-info -f cdda://sr0/ | grep "filesystem::type: cdda"`
if [ "$checkforaudio" == "  filesystem::type: cdda" ]; then
    vlc cdda:///dev/sr0
    else
        checkfordvd=`find /media/*/ -name "VIDEO_TS" | grep "VIDEO_TS"`
        if [ ${checkfordvd:(-8)} == "VIDEO_TS" ]; then
            vlc dvd:///dev/sr0
            else
            pcmanfm
        fi
fi
exit 0
