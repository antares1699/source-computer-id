Mengapa koneksi ODBC 32bit saya tidak muncul di odbcad32.exe?

Saya memiliki sistem Windows 7 64bit yang harus menjalankan aplikasi 32bit yang mengakses MSSQL Server 2005 express melalui sambungan ODBC. Setelah instalasi semuanya berfungsi dengan baik. Sekarang beberapa minggu setelah instalasi saya mendapatkan laporan bahwa aplikasi tidak lagi terhubung ke database. Saya memeriksa C:\Windows\SysWow64\odbcad32.exe untuk melihat apakah ada masalah dengan pengaturan ODBC, dan koneksi ODBC tidak lagi terlihat. Jadi saya memeriksa registri untuk melihat apakah itu benar-benar dihapus. Kunci registri adalah HKEY_LOCAL_MACHINE\Software\Wow6432Node\ODBC Ketika saya membuka bahwa nilai-nilai itu ada di sana, mereka tidak dapat diakses oleh odbcad32 atau aplikasi saya sendiri.

Saya mencoba menjalankan ini sebagai administrator komputer sehingga perizinan seharusnya tidak menjadi masalah, dan selain itu saya memeriksa untuk memastikan bahwa mereka tidak berubah, tetapi saya masih tidak dapat membuatnya berfungsi. Apakah ada yang punya ide tentang cara mengembalikan nilai ke applet odbcad32? Itu kemungkinan besar akan memperbaiki masalah perangkat lunak juga.

kupikir artikel ini di majikan saya Situs, tentang menggunakan ODBC 32-bit pada Windows 64-bit, dapat membantu Anda ...

Tidak ada yang Anda katakan, dalam komentar Anda di bawah atau dalam pernyataan masalah asli Anda, dengan jelas menunjukkan bahwa Anda menggunakan Administrator ODBC 32-bit. Perhatikan bahwa program 32-bit dan 64-bit diberi nama yang sama, odbcad32.exe . Perbedaannya terletak pada lokasi direktori yang berlawanan dengan intuisi - System32 (untuk komponen 64-bit) dan SysWow64 (untuk komponen 32-bit).

Kamu mungkin mengalami masalah lama tetapi berulang dengan korupsi kecil di Windows Registry. Korupsi mengambil bentuk entri yang berisi string 4 karakter ini -

@=""

Entri ini tidak terlihat di mana pun kecuali file ekspor Registry - tetapi mereka dapat menyebabkan sejumlah perilaku yang tidak diinginkan.

CATATAN: Pada mesin Windows 64-bit Anda, secara alami ada beberapa komplikasi yang terkait dengan Registry 32-bit. Artikel Microsoft KB ini mungkin cukup untuk membantu Anda melalui ini. </em>

Saya menyarankan Anda menggunakan Editor Registri 64-bit untuk mengekspor cabang berikut (tempat entri ini cenderung ditemukan) -

HKEY_LOCAL_MACHINE\Software\ODBC
HKEY_LOCAL_MACHINE\Software\WOW6432Node\ODBC
HKEY_CURRENT_USER\Software\ODBC
HKEY_CURRENT_USER\Software\WOW6432Node\ODBC

Edit file ini di editor teks mana pun (Notepad.exe atau Wordpad.exe umumnya baik-baik saja), dan hapus semua baris yang terdiri dari string 4 karakter tersebut, di atas. Kemudian, hapus segmen pohon Registri yang Anda ekspor, dan impor dari file yang diedit - dengan demikian memulihkan segmen pohon, tanpa kerusakan.

Saya mengalami masalah yang sama dan inilah cara saya memperbaikinya untuk DSN Sistem saya.

Untuk referensi, berikut adalah dua jalur di registri untuk mengetahui apakah ODBC 64-bit atau 32-bit pada OS Windows.

64-bit:

\ HKEY_LOCAL_MACHINE \ SOFTWARE \ ODBC \ ODBC.INI

32-bit:

\ HKEY_LOCAL_MACHINE \ SOFTWARE \ WOW6432Node \ ODBC \ ODBC.INI

Dalam folder ODBC.INI terdapat sub-folder dengan nama DSN dengan semua kunci terkaitnya, dan sub-folder lain bernama ODBC Data Sources yang hanya memiliki kunci (Default): (value not set).

Di folder ODBC Data Sources juga harus ada kunci dengan entri DSN Name, Type dari REG_SZ dan nilai Data dari nama ODBC Provider/Driver (dalam kasus saya itu iSeries Access ODBC Driver). Saya menyiapkan tes lain ke server MS SQL, dan nilai Data adalah SQL Server.

Bagaimana saya menemukan ini adalah saya benar-benar menghapus semua folder DSN dan folder ODBC Data Sources dan membuat ulang DSN saya dari awal. Itu membuat folder dan kunci terkait, dan sekarang DSN saya muncul dan dapat digunakan.

Tanpa kunci dengan nama penyedia di ODBC Data Sources, DSN tidak akan muncul di odbcad32.exe. Saya juga dapat membuat ulang masalah dengan menghapus entri ODBC Data Sources.

Semoga ini membantu seseorang jika solusi lain tidak memperbaikinya. Saya mencoba mengekspor kunci registri saya dan tidak melihat @="" seperti yang dijelaskan dalam solusi lain. Tolong beri tahu saya jika instruksi ini tidak cukup jelas dan saya akan mencoba memposting tangkapan layar. Terima kasih!

Akhirnya menemukan resolusi untuk kegagalan nyata untuk menginstal driver ODBC 32 bit sekitar 4-5 tahun yang lalu. Jawaban 7 di atas menunjukkan masalah "Perbedaannya terletak pada lokasi direktori yang berlawanan dengan intuisi - System32 (untuk komponen 64-bit) dan SysWow64 (untuk komponen 32-bit)." Saat menjalankan odbcad32.exe untuk menampilkan jendela Sumber Data ODBC, menjadi jelas bahwa perintah Panel Kontrol menjalankan versi 64 bit di System32 \, meskipun label "Sumber Data ODBC (32 bit)". Saya juga memeriksa Registry untuk menemukan data kunci yang relevan terpasang dengan benar di HKLM_Software_Wow6432Node.

Solusi saya adalah membuat pintasan ke C: \ Windows \ SysWOW64 \ Odbcad32.exe di Desktop saya. Ini kemudian menampilkan jendela Sumber Data ODBC untuk driver 32 bit. Pada instalasi Win7 Professional OS saya, tab Drivers mencantumkan driver 25+ 32 bit.

Kesimpulan - Microsoft memberi kami perintah yang menunjuk ke folder yang salah, mungkin bingung dengan penamaan folder intuitif mereka.
