Alternatif untuk Ubuntu

Setelah pertempuran mencoba untuk membuat monitor saya berfungsi di Ubuntu, saya telah melepaskan Ubuntu sebagai platform pengembangan utama saya. Saya mencari OS yang lebih stabil dengan kriteria berikut:

  • GCC
  • Python dan perpustakaannya
  • Haskell
  • JVM
  • llmv
  • Dukungan Nvidia (Jika ada hal seperti itu)
  • Dukungan beberapa monitor default

Fancy GUI adalah yang paling tidak menjadi perhatian saya dan saya membutuhkan ini secepatnya! Saya telah melihat ke dalam FreeBSD dan fitur mang sangat menarik, saya pikir saya tidak yakin seberapa baik pustaka python bekerja di FreeBSD. Apakah ada yang punya saran bagus? Apa pun dihargai.

Masalah Anda adalah dengan X. Kecuali Anda beralih ke Mac OS X atau Windows, Anda akan menjalankan X pada OS apa pun yang Anda gunakan. Jadi, pertanyaannya adalah apakah Anda bisa mendapatkan pengalaman X yang lebih baik dengan meninggalkan Ubuntu.

Salah satu tujuan desain ekstensi "RandR" adalah untuk memungkinkan monitor penyumbat panas ke Just Work. Tapi itu tidak berfungsi dengan baik untuk Anda.

Masalahnya, X.org telah membuat langkah besar dalam waktu yang relatif singkat belakangan ini. Ada kemungkinan yang masuk akal bahwa X.org terbaru di Ubuntu 9.10 dapat menyelesaikan masalah Anda. Mengapa tidak mengunduh image CD Ubuntu 9.10 dan mencobanya?

Untuk FreeBSD, saya belum mencobanya, tapi saya yakin Python-nya sama bagusnya dengan yang lain. Dan, ia memiliki kernel yang sama sekali berbeda dari Ubuntu (bukan Linux) sehingga driver memiliki dasar yang sangat berbeda ... apakah itu akan membantu Anda? Tidak yakin. Tapi saya periksa, dan nVidia mendukung FreeBSD, jadi setidaknya saya tahu Anda bisa mendapatkan driver.

Semoga berhasil.

Saya akan merekomendasikan Fedora 12 (yang baru saja merilis beta-nya; final akan keluar hanya dalam waktu kurang dari sebulan)

Ini memiliki banyak monitor dan dukungan nVidia, untuk mengutip pengumuman rilis beta :

Urutan startup grafis Fedora sekarang bekerja lebih baik pada sistem dengan banyak monitor. Juga pada beberapa sistem monitor, desktop sekarang akan secara otomatis tersebar di semua monitor secara default, daripada membuat semua monitor menampilkan output yang sama, termasuk pada chip NVIDIA (di mana beberapa rentang monitor tidak dimungkinkan tanpa perubahan konfigurasi manual di Fedora 11). Sistem dengan chip grafis NVIDIA juga mendapatkan dukungan awal untuk menangguhkan dan melanjutkan fungsionalitas melalui driver Nouveau default.

Ini memiliki dukungan Haskell:

Name       : haskell-platform
Arch       : x86_64
Version    : 2009.2.0.2

Untuk JVM (java), ini memiliki:

java-1.6.0-openjdk

dan

java-1.5.0-gcj

Untuk python, ada:

python-2.6.2

Dan untuk llvm:

Name       : llvm
Arch       : x86_64
Version    : 2.6
Release    : 0.5.pre1.fc12

Untuk gcc, ada:

gcc-4.4.2

dan

compat-gcc-34-3.4.6

Jika itu tidak berhasil, mungkin Anda dapat mencoba kembali ke versi lama dari Rilis Jangka Panjang Ubuntu. Ubuntu 8.04 LTS masih menjadi tujuan saya untuk Linux.
