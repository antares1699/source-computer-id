Menarik lampiran dari Outlook ke dalam aplikasi

Halo Saya punya pertanyaan tentang kemampuan drag and drop Outlook 2010 ke berbagai aplikasi. Saat ini saya dapat mengambil lampiran dan menjatuhkannya di desktop tidak ada masalah. Jika saya mengklik dua kali pada lampiran di Outlook, itu akan membukanya di aplikasi default masing-masing. Tidak ada masalah disana. Apa yang menurut saya terjadi adalah file tersebut disimpan sementara ke lokasi sementara kemudian dibuka dari sana. Masalahnya adalah ketika saya mencoba dan menyeret lampiran ke aplikasi lain.

Contohnya termasuk dokumen teks ke Notepad ++, file CAD ke AutoCAD, dll. Aplikasi ini biasanya dapat melakukan drag and drop langsung dari desktop dan membuka file secara langsung. Sebaliknya, dari Outlook saya harus menyeretnya ke desktop, lalu menyeret file desktop ke dalam aplikasi. Proses beberapa langkah yang memerlukan pergerakan jendela, dll. Saya ingin menghindari ini jika memungkinkan.

Saya rasa yang saya tanyakan adalah, dapatkah Outlook mengunduh lampiran saya secara otomatis ke lokasi permanen dan menautkannya ke email sehingga ketika saya menyeret dan melepas dan langsung membuka aplikasi saya? Atau bahkan jika saya harus mengunduh setiap file email secara manual, cukup buka jendela penjelajah lokasi itu sehingga saya dapat menyeret dan melepaskan dari sana?

Saat aplikasi memulai seret, aplikasi menentukan apa yang tersedia untuk diseret dengan meletakkan format yang tersedia di IDataObject. Ketika 'target drop' menerima event drag drop, ia dapat menanyakan format yang tersedia untuk melihat apakah ia dapat menerima data.

Ini cukup rumit dan oleh karena itu sebagian besar target penurunan hanya menerima format CF_HDROP yang paling dasar, yang tampaknya tidak disediakan oleh Outlook.

Satu-satunya cara untuk menyiasatinya adalah dengan menggunakan aplikasi lain yang dapat membaca file PST dan menangani CF_HDROP di sumber drop atau menggunakan alat seperti di atas untuk menghapus lampiran dari email Anda dan meminta Explorer menangani drag and drop.

Komentar yang lebih bermanfaat: https://connect.microsoft.com/IE/feedback/details/867235/html5-fileapi-file-input-does-not-support-dragging-and-dropping-from-outlook dan perpustakaan Java untuk menerima aplikasi http://www.wilutions.com/outldd/example.html

Tl; dr: Outlook seharusnya tidak menyediakan CF_HDROP karena CF_HDROP hanya berlaku untuk file yang ada di disk lokal. Jika menyediakan CF_HDROP, itu harus menyalin file ke disk lokal terlebih dahulu, yang akan menjadi lambat.

Sebaliknya, target penurunan lainnya harus menerima apa Pandangan menyediakan, yaitu:

  • CFSTR_FILECONTENTS
  • CFSTR_FILEDESCRIPTOR
