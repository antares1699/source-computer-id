ssh-keygen dan ssh-add masalah frasa sandi

Ketika saya menjalankan ssh-keygen dan menekan kembali alih-alih memasukkan kata sandi maka saya dapat menambahkan kunci dengan ssh-add ~/.ssh/id_rsa.pub. Saya diminta untuk memasukkan kata sandi untuk ~/.ssh/id_rsa.pub. Saya cukup menekan enter dan semuanya baik-baik saja.

Namun - ketika saya memasukkan kata sandi pada tahap ssh-keygen, dan kemudian memasukkan kembali kata sandi yang sama pada tahap ssh-add gagal dengan bad passphrase, try again

Anda perlu menambahkan pribadi kunci ke ssh-agent, bukan kunci publik. Nama kunci default adalah id_rsa, jadi Anda bahkan tidak memerlukan opsi apa pun:

ssh-add

Jika Anda telah menamai kunci Anda dengan nama lain (mis.) github_dsa, gunakan:

ssh-add github_dsa

Catatan untuk interloper internet: jika Anda tidak menggunakan ssh-keygen untuk membuat kunci, pastikan kunci Anda dalam format SSH. Untuk PuttyGen, misalnya, ada opsi ekspor yang perlu Anda gunakan.
