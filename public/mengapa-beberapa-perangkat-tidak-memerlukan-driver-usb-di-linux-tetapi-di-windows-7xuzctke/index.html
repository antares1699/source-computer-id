Mengapa beberapa perangkat tidak memerlukan driver usb di Linux, tetapi di Windows?

Saya telah melihat beberapa proyek sekarang, yang menggunakan antarmuka USB. Mereka biasanya membutuhkan driver khusus untuk Windows, tetapi tidak untuk sistem berbasis Linux. Mengapa demikian? Maksud saya, bahkan Windows memiliki beberapa driver generik untuk kelas perangkat USB dasar, jadi apa dukungan Linux yang tidak didukung oleh Windows?

Anda juga akan menemukan bahwa di Linux, distribusi standar (dan terkadang pembuatan kernel) cenderung menyertakan banyak driver perangkat keras yang menarik dan spesifik, sebagian besar karena kontribusi pengguna dari waktu ke waktu. Windows tidak benar-benar memiliki cara bagi pengembang sewenang-wenang untuk membuat driver untuk perangkat keras umum kemudian memasukkannya ke dalam semua distribusi baru Windows. Ini adalah perangkat lunak dan model distribusi yang berbeda. Dengan Linux banyak hal yang membuatnya menjadi arus utama dari waktu ke waktu.

Bukan hanya karena Linux memiliki lebih banyak driver "generik" (tetapi ini sering kali benar, terutama ketika kontributor tertarik untuk menulis driver dengan cakupan seluas mungkin daripada vendor yang menulis driver berpemilik untuk perangkat keras khusus mereka), itu hanya dikirimkan dengan koleksi yang dapat disumbangkan orang dan dengan cepat masuk ke distribusi utama, selamanya.

Saya melihat semua jenis hal aneh yang dikirimkan dengan Linux. Misalnya, CAN adalah antarmuka dan protokol serial untuk digunakan dalam otomasi dan kontrol industri. Penggunaan relatif jarang, tetapi kernel yang dikirimkan bersama Ubuntu memiliki dukungan CAN generik bawaan, dan modul kernel antarmuka khusus vendor kartu pengontrol dikirimkan dengan standar Ubuntu. Distribusi semacam itu tidak mungkin dilakukan dengan sistem operasi yang lebih terkontrol seperti Windows atau OSX. Saya tidak akan tahu tentang ini seandainya saya tidak mengerjakan proyek robotika baru-baru ini dan, sangat mengejutkan saya, tidak hanya driver antarmuka yang dibangun ke dalam kernel, tetapi modul untuk kartu pengontrol vendor saya juga sudah (secara mengejutkan) disertakan .

Di luar topik:

Perlu juga dicatat bahwa bahkan untuk driver yang tidak mengirimkan standar, kedua OS tersebut jauh lebih baik daripada sebelumnya. Distribusi Linux yang populer telah bekerja keras untuk membuat mendapatkan perangkat lunak semudah mungkin agar tetap kompetitif, jadi dalam banyak kasus, driver yang belum ada di sistem dapat dengan mudah diperoleh melalui repositori paket (15 tahun yang lalu itu banyak lebih sulit untuk mendapatkan perangkat keras acak untuk bekerja di Linux). Windows, dalam beberapa tahun terakhir, telah menjadi lebih baik tentang memiliki repositori besar driver tersedia online untuk download otomatis ketika perangkat keras terpasang - hari ini, setidaknya bagi saya, jauh lebih jarang untuk menemukan "driver untuk perangkat ini tidak dapat terletak "di Windows.

Ini hanya akan menjadi lebih baik (di semua OS) juga. Kemudahan penginstalan perangkat lunak dan driver ada di mana-mana saat ini (terutama karena kemajuan dalam platform gaya "aplikasi" yang didorong oleh perangkat seluler - dan sekarang lihat; Anda memiliki Pusat Perangkat Lunak Ubuntu, Pasar Windows 8, iTunes Store, Google Play, bahkan perangkat lunak individual seperti Chrome, Firefox, semuanya memiliki fitur baru yang berfokus sepenuhnya pada kemudahan menemukan dan memasang perangkat lunak baru) yang hampir diharapkan; Ini adalah bug jika Anda harus mencari-cari perangkat lunak, yang dulunya merupakan bonus jika Anda tidak melakukannya.

AFAIK Linux menyertakan semua "driver" di kernel, tetapi Windows memisahkannya. Jadi Linux memang memiliki driver tetapi sudah disertakan dengan jantung sistem. Juga Linux menggunakan driver yang lebih umum sementara Windows memiliki driver yang lebih spesifik untuk menargetkan perangkat tertentu.
