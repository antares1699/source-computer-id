Secara umum, apakah SFTP tidak aman?

Saya telah melihat beberapa komentar di internet di mana orang-orang mengatakan sesuatu seperti:

"Saya telah mencopot pemasangan server SFTP, ini jauh lebih baik, ini menghilangkan kerentanan keamanan"

Apa saja kerentanan ini, jika ada?

SFTP berjalan melalui SSH, meskipun beberapa server menggunakan executable terpisah untuk mereka. Kerentanan apa pun yang mungkin dimiliki dapat dieksekusi adalah apa yang Anda lindungi dari (meskipun saya sendiri belum pernah mendengarnya). Protokol itu sendiri seaman server SSH Anda.

SFTP memiliki dua arti.

Arti lama SFTP adalah Secure FTP lama, yang hanya mengamankan saluran kontrol dan membiarkan saluran data tidak terenkripsi.

SFTP baru adalah SSH FTP yang mengamankan semua lalu lintas dengan SSH.

SFTP baru bagus, SFTP lama jelek.

Alternatif lain adalah SCP, yang biasanya lebih cepat dari sftp.

Satu item lain yang terkait. Saya perhatikan bahwa ketika Anda membuka SFTP, jaringan saya mulai mendapatkan banyak serangan (kebanyakan dari China). Dugaan saya adalah mereka hanya mencoba mengenai SSH yang juga perlu dibuka. Namun, jika saya mengatur FTP dengan TSL / SSL, saya tidak mendapatkan serangan seperti itu. Itu mungkin menyiratkan bahwa FTP dengan TSL / SSL lebih aman. Maksud saya, bukan karena itu sebenarnya lebih aman, tetapi secara praktis karena serangannya lebih sedikit.

Sekarang, yang saya lakukan adalah mengubah port SSH saya ke nomor lain dan itu pasti mengurangi jumlah serangan yang melewati router saya ke server saya, tetapi saya masih mendapatkan lebih banyak serangan menggunakan SFTP daripada menggunakan FTP dengan TSL / SSL. Hanya pengamatan yang mungkin menarik minat seseorang untuk melihat utas ini.

Namun, jika Anda menyiapkan server atau berbagi jaringan atau sesuatu, saya akan merekomendasikan menggunakan port yang berbeda untuk SSH daripada default.
