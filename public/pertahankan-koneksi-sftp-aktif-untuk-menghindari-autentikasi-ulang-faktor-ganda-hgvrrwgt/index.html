Pertahankan Koneksi SFTP Aktif untuk Menghindari Autentikasi Ulang Faktor Ganda

Saya menggunakan WinSCP untuk terhubung melalui SFTP ke server Linux dengan otentikasi faktor ganda. Ketika saya terhubung, saya harus mengkonfirmasi koneksi di ponsel saya. Setiap transfer file juga dihitung sebagai "koneksi" individu dan untuk setiap file saya harus memeriksa ponsel saya dan menyetujui tindakan tersebut.

Adakah cara untuk menjaga koneksi tetap "aktif" sehingga saya tidak terhubung kembali setiap kali saya ingin melakukan transfer? Pertanyaan ini tampaknya serupa tetapi tidak terjawab, dan secara khusus menyebutkan faktor ganda Google. (Untuk referensi, server saya menggunakan Duo Security jika itu membuat perbedaan.)

Anda dapat mencoba menggunakan agen yang mengingat kredensial Anda, seperti Pageant, yang biasanya dipasang bersama WinSCP. Bergantung pada implementasi 2FA spesifik dan apa yang sebenarnya dilindungi oleh 2FA, itu mungkin atau mungkin tidak memperbaiki masalah.

Jika itu tidak berhasil - yang mungkin tidak akan berhasil - Anda terjebak membatasi WinSCP ke satu koneksi. Sayangnya, itu akan membatasi fungsionalitas WinSCP: transfer latar belakang tidak akan berfungsi lagi. Ini memiliki beberapa efek:

  • Anda tidak akan dapat menjelajahi sistem file saat operasi unggah / unduh apa pun sedang berlangsung.
  • Operasi SSH, seperti pintasan untuk membuat tarball, masih memerlukan koneksi terpisah, artinya Anda harus mengautentikasi lagi.
  • Anda hanya dapat mentransfer satu file dalam satu waktu.
  • Anda hanya dapat mengantrekan satu objek pada satu waktu (file atau direktori) - artinya, tidak ada lagi antrian. Semuanya terjadi di latar depan, dan Anda harus menunggu hingga selesai sebelum melanjutkan.

Untuk mengubah status ini, Anda perlu menonaktifkan transfer latar belakang sepenuhnya. Ini tidak sesederhana mungkin; ada dua tempat yang Anda perlukan untuk menyesuaikan pengaturan:

  1. Dialog preferensi (Transfer -> Latar Belakang)
  2. Dialog transfer file pada waktu transfer

Info berguna tambahan di situs web WinSCP: Antrian Transfer

Di halaman Transfer -> Preferensi latar belakang, ubah jumlah maksimum koneksi simultan menjadi 1, nonaktifkan pemrosesan antrian secara default, dan pastikan semua pengaturan yang terkait dengan transfer latar belakang dinonaktifkan.

Pada waktu transfer, Anda akan melihat dialog transfer:

Pastikan Transfer on background adalah un diperiksa.

  • Catatan samping: Anda lebih baik menggunakan kartu pintar dengan PGP daripada 2FA untuk keamanan SSH terbaik. Anda bahkan bisa mendapatkan perangkat keamanan "universal" seperti YubiKey yang mendukung lebih dari sekadar fungsionalitas kartu pintar, dan umumnya lebih kecil. Pastikan Anda membuat kunci pribadi pada kartu itu sendiri dan menggunakan beberapa kartu. Jika Anda memiliki kunci pribadi yang sama di beberapa kartu, Anda melakukan sesuatu yang sangat salah.

Coba sesuaikan pengaturan " Keepalives "dari WinSCP. Di file Keepalives kotak, Anda dapat memerintahkan WinSCP untuk mengirim beberapa data tiruan dalam periode reguler untuk menghindari terputusnya sambungan. Periode ini ditentukan dengan Detik antara keepalives. Temukan detail lengkapnya dengan tangkapan layar di sini: http://winscp.net/eng/docs/ui_login_connection
