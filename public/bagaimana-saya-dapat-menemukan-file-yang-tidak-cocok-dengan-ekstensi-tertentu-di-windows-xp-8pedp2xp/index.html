Bagaimana saya dapat menemukan file yang TIDAK cocok dengan ekstensi tertentu di Windows XP?

Saya perlu mencari dan menghapus semua file di subdirektori tidak .SQL file. Saya tidak tahu cara mencari <>.SQL atau !.SQL di Pencarian Windows.

Sebagian besar solusi yang saya lihat di sini adalah menemukan ekstensi tertentu dan melakukan sesuatu dengannya. Saya mencari yang sebaliknya.

Saya tidak tahu cara melakukannya dari Windows Search, tetapi dari baris perintah:

dir /s /b /a-d | findstr /v /r ".*\.sql"
  • dir / s - daftar direktori rekursif
  • / b - daftar kosong, hanya nama file (tidak ada info ukuran direktori dll di keluaran)
  • / a-d - filter berdasarkan atribut, bukan direktori (hapus sub direktori dari daftar)
  • | (pipa) - kirim output dari daftar direktori ke
  • findstr - utilitas pencarian teks
  • / v - hanya mengembalikan baris yang tidak cocok
  • / r - gunakan ekspresi reguler
  • ". *. sql" - cocok dengan berapa pun jumlah kali diikuti dengan titik diikuti dengan sql
Bagikan

Jika Anda bersikeras menggunakan pencarian Windows yang tidak memiliki kemampuan asli atau dukungan untuk fitur semacam itu, Anda dapat menjalankannya dengan cara ini:

*.a OR *.b OR *.c OR *.d OR *.e OR *.f OR *.g OR *.h OR *.i OR *.j OR *.k OR *.l OR *.m OR *.n OR *.o OR *.p OR *.q OR *.r OR *.t OR *.u OR *.v OR *.w OR *.x OR *.y OR *.z

Pencarian berikutnya akan mengecualikan semua file dengan ekstensi .s *, sementara semua file lainnya terdaftar.

Mungkin ada cara yang lebih baik untuk melakukannya, tetapi ini juga akan berhasil.
