Bagaimana cara mengatasi masalah VirtualBox yang menampilkan VM dengan UNMOUNTABLE_BOOT_VOLUME?

Saya mengunduh beberapa gambar VM untuk VirtualBox dari

https://dev.windows.com/en-us/microsoft-edge/tools/vms/mac/

dan mengekstrak file, dan menyiapkan VM baru di VirtualBox 5.0.12 menggunakan file .vmdk yang diekstrak. IE 11 pada Windows 7 berfungsi dengan baik, tetapi karena saya menginstal sesuatu, saya pikir saya akan menginstal salinan VM lain yang segar, jadi saya mengekstrak file lagi dan mencoba menggunakan salinan lain dari file .vmdk untuk mengatur VM lain .

Tetapi VirtualBox mengeluh bahwa itu adalah UUID yang sama dan tidak dapat membuat VM lain, jadi saya mencoba dan menggunakan IE 10 untuk gambar Win 7 sebagai gantinya (bukan IE 11, tetapi IE 10).

Dan kemudian VirtualBox akan mem-boot VM dan setelah 1,5 menit atau lebih, menampilkan layar biru pada VM:

Sepertinya terkait dengan masalah UUID itu. Jadi saya mem-boot ulang Mac saya, dan sekarang menyiapkan VM baru menggunakan "IE 9 di Win 7", dan VM ini juga memiliki masalah yang sama yaitu UNMOUNTABLE_BOOT_VOLUME. Jadi saya pikir entah bagaimana, VirtualBox mengalami beberapa masalah dan tidak mengatur VM benar mulai saat ini. Saya mem-boot ulang Mac dan memulai ulang VM dan itu adalah layar biru yang sama. Bagaimana ini bisa diselesaikan?

Saya juga mencoba chmod -R a+rw untuk semua file untuk file .vmdk yang diekstraksi dan untuk semua file baru yang dibuat VirtualBox, sehingga semua file memiliki izin menulis untuk semua orang, tetapi itu tidak menyelesaikan masalah.

Saya ingin memberikan beberapa wawasan tentang apa yang mungkin terjadi dengan UNMOUNTABLE_BOOT_VOLUME.

Dengan file .zip yang disediakan, jika Anda menggunakan program zip default Mac untuk mengekstraknya (hanya dengan mengklik dua kali pada file zip di OS X El Capitan), maka pertama-tama akan membuat file .ova, dan kemudian akan segera membuka kompres lagi, untuk 2 file: satu file .vmdk dan satu .ovf.

Sekarang, Anda tidak dapat menggunakan file .vmdk tersebut sebagaimana adanya. Anda harus mengklik dua kali pada .ovf agar dapat dibuka oleh VirtualBox, dan kemudian akan mengkloning gambar .vmdk dan membuatnya menjadi VM.

Saya pikir Anda juga dapat menggunakan sesuatu seperti Unarchiver pihak ke-3, dan mengekstraknya, dan itu akan membuat file .ova dari file .zip. Dalam hal ini, cukup klik dua kali untuk VirtualBox untuk menggunakannya, dan saya pikir dalam kasus ini juga tidak akan memiliki masalah UNMOUNTABLE_BOOT_VOLUME.
