Bagaimana cara mengubah nama komputer di komputer desktop Ubuntu?

Ada tutorial di Forum Ubuntu: tautan

Ini adalah cara singkat dan manis yang tetap sangat penting. Saya t didasarkan pada pengalaman saya saat mengubah nama host suatu sistem. Itu nama host secara efektif hanya nama sistem. Anda tidak biasanya perlu mengubahnya, tetapi jika Anda harus melakukannya karena alasan apa pun, itu tidak terlalu sulit sama sekali.

Meskipun mengubah nama host bukanlah tugas yang sulit, itu bisa berbahaya karena jika sistem tidak dapat menyelesaikan nama hostnya sendiri, tidak mungkin sudo. Jadi, jika dilakukan dengan tidak benar, mengubah a nama host dapat mengunci Anda dari sistem Anda sendiri. Pertama, dan paling banyak yang terpenting, kami membuat cadangan. Buka aksesori / terminal dan tempel perintah ini masuk, lalu tekan enter:

sudo cp /etc/hosts /etc/hosts.bak && sudo cp /etc/hostname /etc/hostname.bak

Anda HARUS membuat cadangan, untuk berjaga-jaga. Jika Anda tidak bisa boot ke file yang bisa digunakan sistem setelah menyelesaikan cara ini, atau sudo tidak berfungsi, lihat instruksi di akhir panduan ini. Jika Anda tidak sepenuhnya akrab dengan terminal linux, mungkin ada baiknya untuk menulis ini ke bawah atau cetak.

Ok, sekarang kita sudah melakukan backup, kita bisa edit / etc / hosts:

sudo nano /etc/hosts

Anda akan melihat apa yang Anda cari dengan cukup cepat, di bagian atas file itu akan terlihat seperti: Kode:

127.0.0.1       localhost
127.0.1.1       old_hostname

Ubah old_hostname menjadi hostname baru pilihan Anda. Sekarang simpan dan keluar.

File kedua yang akan diedit adalah / etc / hostname. Ini melakukan apa yang tertulis di timah, file tersebut akan memiliki satu baris di dalamnya dan baris itu adalah nama host Anda. Ubah ke nama host baru Anda, simpan dan keluar.

Sekarang reboot, Anda harus memiliki sistem yang berfungsi dengan baik di mana sudo bekerja. Anda dapat menguji sudo dan nama host baru Anda dengan melakukan, cukup:

sudo hostname

CATATAN: Perintah ini biasanya tidak perlu dijalankan sudo, kita cukup menguji nama host yang diubah DAN sudo secara bersamaan.

Jika tidak berhasil, lihat di bawah. Anda DID mengambil cadangan, bukan?

  • Masuk ke menu grub dengan menekan esc saat bootup, atau izinkan untuk ditampilkan secara otomatis jika Anda telah mengaturnya seperti ini
  • Pilih mode pemulihan
  • Jika diminta, pilih "drop to single user mode" atau serupa (catatan: ini hanya berlaku pada hardy menurut saya)

  • Saat Anda berada di shell root, ketik perintah berikut:

    cp /etc/hosts.bak /etc/hosts && cp /etc/hostname.bak /etc/hostname

Kemudian tekan ctrl-alt-del dan Anda harus mem-boot ulang kembali ke sistem yang berfungsi sempurna.
