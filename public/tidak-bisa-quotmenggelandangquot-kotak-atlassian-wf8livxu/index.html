Tidak bisa "menggelandang" kotak atlassian

Windows 8.1, versi gelandangan dan VirtualBox terbaru. Menjalankan perintah di git CLI di Windows.

Saya dapat memulai ubuntu/trusty64 tanpa masalah, dan tidak perlu masuk ke atlas.hashicorp.com.

Tetapi ketika saya mencoba ubuntu/xenial64 saya mendapat kesalahan.

vagrant init ubuntu/xenial64; vagrant up --provider virtualbox

Selain itu, trusty 64 tidak menyebabkan peringatan apa pun tentang login dengan vagrant login, tetapi karena xenial 64 memang menyebabkan peringatan seperti itu, saya segera membuat akun di atlas.hashicorp.com.

Saya menggunakan vagrant login untuk masuk, mendapatkan pesan "Anda sekarang masuk."

Saya menghapus gelandangan dari folder dan mengulangi perintah init dan up, tetapi sekali lagi saya mendapatkan kesalahan yang sama.

Sepertinya saya menemukan jawabannya.

Versi terbaru dari Vagrant (tampaknya 1.8+) memiliki versi curl yang disertakan tidak berfungsi di Windows 7, 8, 8.1, atau 10 dengan adanya!

Tapi cara mengatasinya sangat mudah. Instal saja ini, Paket Microsoft Visual C ++ 2010 SP1 yang Dapat Didistribusikan Ulang:

https://www.microsoft.com/en-us/download/details.aspx?id=8328

Halaman itu tidak mencantumkan Windows 8.1 sebagai sistem operasi yang kompatibel, tetapi berfungsi untuk saya. Tidak diperlukan restart sistem, dan mesin virtual saya sudah aktif dan berjalan.

Atau, menurunkan ke versi Vagrant 1.7.4 juga tampaknya menyelesaikan masalah: https://releases.hashicorp.com/vagrant/

Saya menemukan kedua solusi tersebut di sini: https://github.com/mitchellh/vagrant/issues/6852

Masalah duplikat: https://github.com/mitchellh/vagrant/issues/6754 https://github.com/mitchellh/vagrant/issues/6861
