Bagaimana cara mengakses halaman web yang berjalan di server jarak jauh?

Saya memiliki instance Ubuntu 14.04 LTS yang berjalan di mesin komputasi Google. Saya menginstal manajer Cloudera pada instance dan menjalankan halaman web administrasi di localhost: 7180. Saya menggunakan port forwarding dan xauth untuk mengaksesnya dari laptop saya tetapi sangat lambat. Apakah ada cara saya bisa langsung mengaksesnya dengan menggunakan alamat ip dari browser saya. Saya mencoba melakukan http: //A.B.C.D: 7180 (A.B.C.D adalah ip eksternal instance saya) di browser saya, tetapi tidak berfungsi.

Terima kasih sebelumnya. maafkan saya jika pertanyaan saya konyol atau terlalu jelas.

Seperti yang dikatakan Bolwerk, Anda dapat mencoba salah satu metode tersebut, tetapi metode termudah adalah mengaktifkan tunneling lokal. Buka terminal Anda dan lakukan ini.

ssh -L 9999:localhost:7180 A.B.C.D

Sekarang Anda akan masuk ke server secara normal. Periksa apakah port 9999 menjalankan layanan apa pun. jika Anda melihat 9999 di sana, Anda siap melakukannya.

netstat -tulpn

Kami tidak perlu mengkonfigurasi proxy SOCKS apa pun. Buka browser apa pun yang Anda suka dan ketik http: // localhost: 9999 dan itu akan bekerja dengan sangat baik. Setelah Anda selesai menggunakan tutup sesi SSH terminal.

Saya tidak tahu apa-apa tentang Cloudera atau batasan akses Google Compute Engine. Anda bahkan tidak mengatakan apakah Anda memiliki ssh. Berikut beberapa hal yang dapat Anda coba ...

Pilihan 1

Dengan asumsi laptop Anda adalah Linux atau BSD atau bahkan mungkin berbasis Mac, Anda memiliki SSH, memerlukan GUI, dan memiliki akses SSH di server, Anda dapat membuat terowongan ssh di komputer lokal Anda. Buka terminal (xterm baik-baik saja) dan jalankan

ssh -D 8080 -C -N you@remote.machine

Jangan tutup terminal sampai Anda selesai melakukan proxy!

Kemudian buka kromium dengan:

chromium --proxy-server="socks://localhost:8080"

Jika Anda tidak memiliki chromium, Anda perlu mengkonfigurasi kaus kaki secara manual (misalnya, di Firefox). Saya menggunakan chromium hanya untuk tujuan ini dan Firefox untuk hampir semua hal lainnya.

Anda harus menjelajah seolah-olah Anda orang lokal! http: // localhost: 7180 harus bekerja.

Setelah selesai, ctrl + C keluar dari terowongan ssh yang Anda buat atau tutup terminal.

pilihan 2

Jika hal di atas tidak bisa dijalankan: dengan asumsi Anda memiliki ssh dan Anda dapat hidup dengan administrasi berbasis teks yang dapat Anda lakukan

ssh -t remote.machine "lynx localhosthost:7180"

dalam sebuah xterm. Anda mungkin perlu menginstal lynx di Ubuntu.

Pilihan 3

Anda mungkin tidak dapat mengontrol port yang Anda miliki aksesnya, tetapi karena tampaknya Anda memiliki akses ssh dan mesin tersebut adalah milik Anda sesuka Anda, mungkin instal x2go. Ini adalah solusi desktop jarak jauh Linux yang cukup cepat dan menggunakan ssh.
